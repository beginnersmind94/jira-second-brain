# NXT-2909 — Authentication - Retain last authenticated user for login

## Description
As a user I want my context and username (profile) to be automatically retained (persistent) on my computer/browser so that I don’t have to enter my username each time and can save time by just setting my password to sign in.

We should retain the user account that was last authenticated after the user has either signed out or inactivity thresholds are met and signed out automatically. I would expect to see the user’s profile picture, name, and username displayed. There should only be an entry text box for the password. We can have an option like many sites that says “Not John?” This would, of course, remove the loaded profile and reset the context selector and user credential controls. 

The benefit of this is adding to the user experience and the feeling of customization. In some ways, this is functioning not as a user signing out, but a user just locking their screen. In most cases, our users are returning to work the next day and want to just sign back in.

Things to consider/explore …

* Should we have multiple sign out options of “Lock” and “Sign Out Completely”? Each could follow the retention or clearing of prior user based on that selection?
* There is the possibility for the need of a setting to control if we want to retain the last user (for security purposes). However, for now, let’s proceed with no setting to control.

## Acceptance Criteria
1. Previously authenticated user profile should be retained per browser instance
2. Previously authenticated user should be displayed when a user signs out
3. Previously authenticated user should be displayed when the system signs out due to inactivity
