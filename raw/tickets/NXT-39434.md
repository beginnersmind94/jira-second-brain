# NXT-39434 — Student & Adult Import – Validate Phone Numbers

## Description
As a SchoolCafé admin, I want to ensure we import valid phone numbers so that we can have cleaner data in the system.

*Flow:*

* Open SchoolCafé and browse to Settings > Imports & Exports > Imports
* Click Add New or click Edit for a Student or Adult Import
* Complete the process of adding or editing a student or an adult

*Notes:*

* This story comes from Dallas and Customer Care team during implementation of Salem
* The goal is to remove all non-numeric data and take the first 10 numbers only so that the system has clean data

*Requirements*:

* The following changes are for both Student and Adult Imports, related to phone numbers
* 
* Student Import
* Make the changes for the following fields:
** GuardianCellPhoneNumber
** GuardianHomePhoneNumber
** GuardianWorkPhoneNumber
* 
* Put validators on the import routine that do the following:
** Strip out anything that is not a number, including spaces, (, ), -
** Trim to only 10 digits
** Use this value to import
** Example:
*** Source value: (281) 254-8567
*** Import value: 2812548567
** Example:
*** Source value: (543) 558-9657 ext. 3
*** Import value: 5345589657
* If the field had more than 10 digits after stripping everything out:
** Log as an Optional exception since they had extra numbers so the user can know we modified their data
** Treat it the way we do exceptions for optional fields by marking as Optional exception and still importing the row
** This exception will show up in the exceptions table
* If the field has less than 10 digits after trimming
** Then make it blank as it is a malformed phone number
** Treat it the way we do exceptions for optional fields by marking as Optional exception and still importing the row
** This exception will show up in the exceptions table
* 
* Adult Import
** Do the thing for the field PhoneNumber on the Adult Import that you did for the phone fields in the Student Import
* 
* Permissions: No changes necessary

## Acceptance Criteria
* When importing, for the fields above:
** it will not import anything that is not a number
** it will not import more than 10 digits
** it will not import less than 10 digits
* Optional exceptions still allow the row to import
* Exceptions show in the exceptions table
