# NXT-65006 — System Accounts (Proposed #4)

## Description
*As a product manager,* 

*I want* developers to create and maintain a system accounts database 

*so that* district financial managers can view all available system accounts and make informed mapping decisions.

## Acceptance Criteria
* *Authoritative catalog available*
A single, [complete catalog|https://cybersofttechnologies-my.sharepoint.com/:w:/g/personal/rahul_mehta_cybersoft_net/EZdA4m_hj4BPpv72wA8A82oB9MCtBeHu-uVUeTprpifx4Q?e=xc7pet] of system accounts exists and is used by all features that need it (e.g., GL mapping, reports).
* *Standard categories*
Each account belongs to one of the defined categories: All Meal Sales, All Inventory (Asset), All Inventory (Expense), Materials Used, All Reimbursements, All A La Carte Sales, Core Financial Accounts.
* *Context/help note (“source of data”)*
Each account can include a short help note explaining where the data comes from (e.g., “This data is sourced from XYZ report”). If no note exists, nothing is shown in the UI. 
* *Toggle behavior*
Accounts can be toggled on or off
* *Search & filter*
• filter accounts by category,
• search by words in the account’s description or code/label,
• accounts toggled on vs off 
* *Extensible without disruption*
New accounts can be added later without impacting existing mappings or reports.
* *Consistency across districts*
By default the catalog is global/standardized for all districts
