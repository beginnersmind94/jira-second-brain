## Acceptance Criteria
The user should be able to search an item based on following search parameters:

*Quick Filter* (Default Value: All), other values should list as shown below. The system should show the results based what user selected

* All- show all the transactions
* Past 3 Months - show all the transaction created or updated in the last 3 months (system date - 90 days)
* Past 6 Months - show all the transaction created or updated in the last 6 months (system date - 190 days)

* Past Year - show all the transaction created or updated in the past year (Ex: 1st Jan 2020 until 31st Dec 2020)
* Date Range - enable the date range so that user can select

!image-20211122-231023.png|width=361,height=283!



Date Range Values should show following error- in case user enter invalid dates:

!image-20211122-231458.png|width=708,height=115!



*Manufacturer* :

 The user should be able to select one or more manufacturer details. The brand field is disabled until user select at least one manufacturer is selected

 Minimum of 1 manufacturer should be selected- otherwise system shall NOT include this in the search criteria (to treated like ALL)

The system should allow the user to select one or more manufacturer as needed



*Brand*:

The user should be able to select one or more brands which belongs to the manufacturer selected

When more then one manufacturer is elected, brand applicable for all the selected manufacturer should be listed in the drop down

If no brand is selected - the system shall not consider that in the search criteria (to treated like ALL)



*Item Category*:

Default Value: All

The values should be listed based on the Item Category Field (Originated in Product Information section)

user should be able to select one or more item category as needed



*Show Single Item Recipe (Check Box)* - when the user selects this, the system should only show the items which are flagged as Single Item Recipe

*Buy American (Check Box)* - when the user selects this, the system should only show the items which are flagged as Buy American



*Result Grid Display*

* User should be able to select columns to be displayed from available fields by clicking on “Columns & View button”. 
* Default fields in view are:
** Item Code
** Item Name
** GTIN Number
** Data Source
* User should not be able to de-select/uncheck default fields
* An option to Select ALL fields should be available to the user
* While on selection of fields to view, the user should be able to reset to the default fields
* View a count of the number of fields selected
* The user should have an option to close the card and return to the Item list view
* The user should be able to see in the results grid the fields  and values as per field selection



*Generic Requirement:*

 When the system loads the page for the 1st time, it should all the transactions (Quick Filter = All, Manufacturer = All, Brand = All, Item Category= All)

 The check box parameters are deselected by default when the screen is loaded for 1st time

When user apply more than one parameters, it works based on AND condition
