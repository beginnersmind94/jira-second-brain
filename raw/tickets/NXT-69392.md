# NXT-69392 — Fin - Display Data Point Business Rules in Account Mapping

## Description
*As a* financial administrator viewing the Account Mapping interface,  
*I want to* see contextual business rules for each data point when I open its details drawer,  
*So that* I can understand how the data point affects financial entries and verify the correct GL account mapping.

## Acceptance Criteria
# *Rules display for system-generated data points*  
When a user opens a new ‘Rules’ action button in the main grid for any system-generated data point, a drawer displays with an "About This Data Point" section containing:
#* A description of the data point's purpose
#* The source system or process that generates entries
#* A journal entry template showing which accounts are debited and credited
# *Multiple journal entry support*  
For data points affected by multiple transaction types (e.g., Inventory, Due from Federal Funds, Refunds, Sales Tax), the system displays all applicable journal entry scenarios, each with its own source label and debit/credit template.
# *-Editable rules for user-created data points-*  
-When viewing a user-created data point, the journal entry template section is editable, allowing the user to select which data points should be debited and credited for manual entries.-
# *Consistent terminology*  
All journal entry templates -use "This data point"- to refer to the currently selected data point, ensuring clarity regardless of the data point name's length or complexity.
## *Updated to display the actual name instead of “this data point”* 
# *Manual entry indicators*  
Data points requiring manual journal entries clearly indicate "(Manual)" in their source field (e.g., "Benefits Entry (Manual)") to remind users that these entries are not system-generated.
# -Two-step transaction completion-  
-For certain liability accounts (Returned Checks, Refunds, Sales Tax Owed), the rules display indicates that a second manual ledger entry is required to complete the transaction. These entries are identified by their GL Entry Type Code:-
#* -Pay Returned Checks (PRTNCHK)-
#* -Pay Refunds (PAYRFND)-
#* -Sales Tax Paid (TAXPAID)-
