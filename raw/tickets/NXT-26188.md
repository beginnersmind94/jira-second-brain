# NXT-26188 — Accountability - Patron Balances - Create API to get BalanceAsofDate based on date/balance range

## Description
As a central office admin, I would like to be able to generate a list of accounts whose balances were within a certain range on a particular day in the past (or as of today), so that I can better audit my system or help troubleshoot issues.

*Summary*

In order to allow users to pull up this list (formerly called the ‘Patron Balance’ report in Classic), we will need to integrate a new API in the both Accountability and Account Management.

*Mockup*

None (API only)

*Details*

The new API needs to return a given set of accounts within:

# a certain date and 
# within a certain balance range.

Relevant story to be taken up in a future sprint by Account Management/Eligibility team:

* [https://primeroedgenext.atlassian.net/browse/NXT-5944|https://primeroedgenext.atlassian.net/browse/NXT-5944|smart-link] 
* In context, that story requires us to add the following fields and colums to the Accounts page:

{quote}*_Balance Range_*

_Two fields - Min and Max._

_Default for both are blank._

_Will filter results based on current balance - must fall within the range._

_Column does not need to be sortable but it would be nice to have._

*_As of Date_*

_Default: Today’s date._

_Dates in the past can be selected, likely need a calendar selector._

_The resulting information (Eligibility, Balances, grade, site, status) all need to comply with this date._

_Column should be sortable._{quote}

## Acceptance Criteria
Verify that the new API returns  balances based on a given set of accounts within a certain date and balance range.
