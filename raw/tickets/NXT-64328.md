## Acceptance Criteria
* Update the Contracts tab for internal vendors
** move Date Range to the top row
** shorten the Shipping and Delivery Groups if necessary
*** use the *+2* type label if not all groups can be displayed in the column
* Provide a checkbox option when creating a new internal vendor contract for ‘Seasonal Contract’
** if Seasonal Contract is selected, allow the date range to overlap with the main contract
** if Seasonal Contract is selected, disable the ‘No End Date’ option
** Provide an info icon with an explanation about seasonal contracts
*** “Seasonal contracts will be considered an extension of the current active contract and can be created without ending or inactivating the other contract. 
The seasonal contract will replace the main contract during the specified date range.”
** Only show the Seasonal option when there is a currently active “main” contract
* Allow seasonal contracts to appear as “Active” alongside the main contract, but display with Seasonal label
** when the seasonal contract is active, display the main contract as “Paused”
* Update the Orders functionality to use the seasonal contract when those dates are active
