# NXT-8084 — INV - Physical Inventory - Save as Complete Validations (PI Close Day: Current)

## Description
As an inventory user I want validations when I attempt to save an inventory as complete so the system ensures my transactions are completed before I save a new snapshot of the perpetual inventory.

This is a back end story for the checks that occur when saving an inventory as complete, when the Physical Inventory Close Day setting is set to ‘*Current*’
Front end story: [https://primeroedgenext.atlassian.net/browse/NXT-7833|https://primeroedgenext.atlassian.net/browse/NXT-7833|smart-link]

*Requirements*

When ‘Save as Complete’ is clicked, 

* Saves a snapshot of the perpetual inventory at the counted date and time.
* Saves the count discrepancies
** calculates perpetual according to the counted date & time
* Validation for ensuring the Counted Date/Time is after the most recent previously reconciled physical inventory
* Alert if the Counted Date/Time precedes the Started Date/Time

## Acceptance Criteria
Save as Complete will check for pending inventory transactions as listed in the description
