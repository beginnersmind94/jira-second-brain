# NXT-30128 — IM - Items - Create New Item - Item Type Questions/Toggles (Step 4)

## Description
As a District Admin I want the system to ask me questions about the item I am creating so I can define the item types accordingly.

UX to be updated before Sprint Planning (JM 5-16)

*Requirements*

* Upon successful save of the header after Unit Info step ([NXT-30046|https://primeroedgenext.atlassian.net/browse/NXT-30046]), display some banner questions below the header
** There are a couple different flows related to which questions are displayed
** If user skipped Product Info, they can only toggle POS Info to Yes
*** provide a hover tooltip on other toggles: “Must configure Product Info”
* Question 1: Is this a POS Item?
** If *YES*, enable the POS Info tab
*** color the tab as red to indicate missing mandatory attributes
** if *NO*, color the tab as gray and do not load the tab data
* Question 2: Is this a Food Item?
** If *YES*, enable the Food Info tab and load Serving Measures ([NXT-30099|https://primeroedgenext.atlassian.net/browse/NXT-30099])
*** color the tab as red to indicate missing mandatory attributes
** if *NO*, color the tab as gray and do not load the tab data
*** Do not show Question 3 in this case; Skip to Question 4
* Question 3: Is this a Menu Item?
** if *YES*, we will enable the Menu Item card on the Recipe / Menu Item tab (different story)
*** do *not* load the Recipe / Menu Item tab yet; the user must also answer Question 4
** if *NO*, we will assume it’s not a Menu Item and will not load the section
*** however, users will have the option to turn it into a Menu Item later
* Question 4: Is this a Single Item Recipe?
** ***Note: Only Food Items* – this question is not displayed if the user specifies the item as Non-Food
** if *YES*, we will enable the section on the Recipe / Menu Item tab (different story)
** if *NO*, we will assume the item is not a recipe and will not load the recipe card/section
*** however, users will have the option to turn it into a Single Item Recipe later
* Once all questions are answered *OR* the user has saved the item anyway, remove the questions and load all the tabs
** Food Info
*** gray/disabled for non-food
** Recipe / Menu Item Info
*** gray/disabled only if both Question 3 and 4 are NO
** POS Info
*** gray/disabled if user said NO
** Product Details
*** this is loaded regardless of any item type distinctions/answers
* Save the answers to each question
** This will be used to filter/search items later (i.e. search all Food items missing necessary info)
** Consider also saving “status” (Incomplete, Partially Complete, Complete) which considers the percentage of mandatory attributes provided
