# NXT-37111 — IM - Items - Units - Save & Continue - Inventory Ready

## Description
As a user configuring items I want a series of completion tasks in the new Units card so I can configure any serving or pack size data points that aren’t input on a specific row or add flow.

*Requirements*

* Provide questions/prompts to complete the overall required fields for saving the initial Units card and proceeding to item creation
* Execute required checks before enabling official Save
** 1. Confirm Inventory readiness / Base Unit (this story)
** 2. Prompt for Costing - [https://primeroedgenext.atlassian.net/browse/NXT-37568|https://primeroedgenext.atlassian.net/browse/NXT-37568|smart-link] 
** These checks will always apply when saving the card, but these are presented as “steps” when creating the first time
** Once saved, these 2 checks will be handled differently in edit/update scenarios.
*** The checks will still occur, but will be managed by icons/banners
* Confirm Inventory Readiness / Base Unit - *few scenarios*
** +If pack sizes are incomplete/invalid+ (not linked, not added, etc.), we can use this step to confirm that the user is not configuring pack sizes
*** Ex: “This item is not inventory eligible with the current pack size configuration. Continue anyway?”
**** “To make pack sizes valid, link the pack and servings using weight or sub-quantity”
** +If there are fewer than 3 pack sizes and+ +*no*+ +inventory-able serving sizes+ linked (using sub-qty)
*** “Is [_description of lowest unit_] the smallest inventory-able unit?” *Yes/No*
**** Yes > Confirms base unit and proceeds
**** No > Emphasize the ‘Add Pack’ button 
** +If there is an+ +*inventory-able serving*+ (linked via sub-qty) which is linked to the current base unit via sub-qty 
*** *Note: there must still be fewer than 3 pack sizes
*** “Is [_description of lowest unit_] or [_description of linked serving size_] the smallest inventory-able unit?
**** if lowest unit > confirms base unit and proceeds
**** if serving size > convert the serving size into a pack size (it’s both) and define it as the Base Unit
*** **Exception:* this step/option is invalid once the item is in Inventory tables and Pack Size is locked
** +if there are no pack size units+ but there is at least one inventory-able serving 
*** [https://primeroedgenext.atlassian.net/browse/NXT-38477|https://primeroedgenext.atlassian.net/browse/NXT-38477|smart-link] 

----

*General Rules:*

* Sub-Qty is for pack sizes, except the one case where serving size is linked to the base unit
* Global Info (Brand, Manufacturer, Product Code) is always optional, but ideally available
* Inventory flag is triggered if Pack Sizes are added
* Item Categorizations are only required if Inventory is enabled 
* at least one unit should be configured before moving on
