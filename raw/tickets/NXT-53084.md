# NXT-53084 — MVR - Users Grid Pages

## Description
As a SchoolCafé user, I would like to be able to setup my managed view for recurring distribution to a set of users so that I can get periodic notifications like on custom reports.

*Notes:*

* We are doing this for Users page, but code as if it were being done by any page in front office or back office
* Do not hard code assumptions that it is only for Users page or only for System module
* This is part of the Custom Reports Framework and will work like a Custom Report
* Note that there is a many to one relationship of MVRs to MV
* See use cases and flows below to better understand user experience

*Requirements:*

* On System > Users, do the following:
* Add, if not already there, a button to the Managed View ellipsis
** Title: Distribute
** Tooltip: 
*** If there are no linked MVRs: Create Managed View Report
*** If there are linked MVRs, for each one: Distribute [MVR name} now
** On click:
** First:
*** Check if the user is viewing a managed View
**** If yes, then proceed
**** If no, then show the user this message:
***** You can only distribute a saved managed view. Select one or create one to first.
***** Do not allow them to proceed until they are viewing a managed view
** Second:
*** Check if an MVR has been created for it already
**** If yes, requirements covered in Distribute Now cloned story
**** If no, then we need to setup the MVR first, so continue
**** Show a confirmation popup:
***** Text: You need to first setup a managed view report distribution for this view. Continue?
***** Values: Yes / No
***** If no, do nothing
***** If yes, open the Managed Views Distribution drawer to create an MVR (covered in another story)
* 
* Security:
* For distribute button: Use View rights on R&P for the page it is on, which is Users in this case
** So, user must have View rights to Users page



h2. *Use Cases:*

There are 4 scenarios for when a user wants to distribute an MVR:

# There is no managed view saved
## User is forced to save a managed view 
# There is a managed view saved but the user is not looking at it
## User is forced to select a managed view or create one
# User is a looking at a managed view but there is no MVR saved for it
## User is forced to create an MVR for it
# User is looking at a managed view with a saved MVR
## User is allowed to distribute the MVR



h2. *Example Flow:*

{color:#ff5630}_Note: Red text is deviation from happy path_{color}

*Happy Path (#4):*

* User browses to the Users page
* User selects a managed view to view
* User decides to distribute it
* User clicks the Distribute button
* System confirms that the user is looking at a managed view 
* System confirms that the managed view has an MVR
* System executes the distribution
* System shows a success message to the user



*Saved MVR Path (#3):*

* User browses to the Users page
* User selects a managed view to view
* User decides to distribute it
* User clicks the Distribute button
* System confirms that the user is looking at a managed view 
* System confirms that the managed view {color:#ff5630}does not have an MVR{color}
* {color:#ff5630}System notifies the user to create an MVR{color}
* {color:#ff5630}User starts the Add New process and creates the MVR{color}
* {color:#ff5630}System redirects the user back to the Users page{color}
* {color:#ff5630}User starts the process over and is now in the happy path{color}



*Saved Managed View Path (#2):*

* User browses to the Users page
* User selects a managed view to view
* User decides to distribute it
* User clicks the Distribute button
* {color:#ff5630}System confirms that the user is not looking at a managed view{color}
* {color:#ff5630}System notifies the user to select a managed view or to save one{color}
* {color:#ff5630}User selects one from the list{color}
** {color:#ff5630}Alternatively, the user creates a managed view and then selects it{color}
* {color:#ff5630}User clicks the Distribute button{color}
* {color:#ff5630}System confirms that the managed view does not have an MVR{color}
* {color:#ff5630}System notifies the user to create an MVR{color}
* {color:#ff5630}User starts the Add New process and creates the MVR{color}
* {color:#ff5630}System redirects the user back to the Users page{color}
* {color:#ff5630}User starts the process over and is now in the happy path{color}



*No Saved Managed View Path (#1):*

* Same flow as #2

## Acceptance Criteria
* Add Distribute button to the Managed View ellipsis and has tooltip
* On Distribute button click, if user is not looking at a managed view, show them a message and don’t allow them to distribute or create an MVR
* On Distribute button click, if user is looking at a managed view with no MVR saved, show them a message and allow them to create an MVR
* On Distribute button click, if user is looking at a managed view with an MVR saved, then allow them to distribute the MVR
