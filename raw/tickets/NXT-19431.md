# NXT-19431 — INV - Manage Orders - Receipts - Prevent Quick Receipt if a Physical Inventory has been completed

## Description
As a food service director I want the system to prevent receipt adjustments if a physical inventory has been completed since the receipt was created, so that users are not adjusting data from the past.

We will check Receipt Date against the most recent completed physical inventory. If the Receipt Date is earlier than the Physical Inventory completion date, do not allow edits.

*Requirements*

* *This story applies to only Quick Receipts* (Regular receipts are [NXT-18705|https://primeroedgenext.atlassian.net/browse/NXT-18705])
* Check the Quick Receipt for any items which are included on the completed Physical Inventory
** Alert user on load of Receive step
*** display a message at the top: “This Quick Receipt contains items from a previously reconciled Physical Inventory. Please remove these items or change the transaction date.”
* Do not allow a new receipt to be created if the transaction date is earlier than the most recent Physical Inventory completed date
** only if the receipt contains items which are also on the physical inventory
* Do not allow an existing receipt to be adjusted or cancelled if the transaction date is earlier than the most recent Physical Inventory completed date
** only if the receipt contains items which are also on the physical inventory
* Do not allow an item to be added to a receipt if that item is on the physical inventory AND the receipt date is earlier than physical inventory completed date
* *Allow* header level changes which do not affect receipt items (changing in [NXT-19432|https://primeroedgenext.atlassian.net/browse/NXT-19432])
** Delivered By, Invoice #, Invoice Amount, Comments
** These fields are enabled when user enters Edit mode, but Item level fields are not enabled
* Provide a message in any scenarios in which the user is prevented from adjusting the receipt
** Message: “This receipt contains items from a reconciled Physical Inventory and therefore cannot be edited.”
*** subject to change or additional messages

## Acceptance Criteria
* Add logic to check receipts against reconciled physical inventories
* Disallow the editing or creating of receipts if they contain items which have been reconciled on a physical inventory 
* Show a message in any scenario where user is unable to edit/create/cancel/etc.
