# NXT-33415 — Acct Mgmt - Accounts - Add filter to view latest PNA students only

## Description
As a central office admin, I would like to be able to easily view a list of students who are ‘Paid’ status but have ‘Not Applied’ for free or reduced benefits, so that I can contact them directly, encouraging them to apply.

*Summary*

Add a new drop-down to the ‘Accounts’ page, which allows users to immediately view only those accounts which are ‘Paid Not Applied’ (These are Paid students who have not submitted an application for the current academic year), based on the Insights data.

*Details*

* In Account Management → Accounts, add a new drop-down menu to the Filters: *Insights*
** Type: Single-selection
** Options:
*** *All* 
**** Default value
*** *Paid Not Applied*
**** We can pull this list directly from the Insights module. Per Mahesh: _“it will be a API call to one of the Insights API to get 'Student ID' and then lookup into database(FO tables) with Student ID to get details as per the report”_
* When ‘All’ is selected, ignore this selection and continue searching based on the remaining filters (when the user clicks Apply)
* If ‘Paid Not Applied’ is selected, do the following:
** Hide and ignore all other fields except Site Code/Site Name
** When the user clicks Apply, get the list of PNA students from Insights, for the site(s) selected, and populate the grid
