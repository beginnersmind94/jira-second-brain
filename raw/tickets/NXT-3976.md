# NXT-3976 — AcctMgt - Patron Details - Restrictions - Item Restrictions

## Description
*User Story*

As a district administrator I want to see and set any menu item restrictions on an account, so that the patron can be prevented from purchasing (or allowed to purchase) specific items when they go through the line.

*Requirements*

* In the Restrictions section in the Patron Details page, expand the section to have a line item for *Menu Items*.
* This should show all menu items restricted for this patron. Items can be added by the user -2 ways:-

# *-When the user adds an allergen for this patron on the Allergies/Medical tab, all menu items with that allergen should automatically be added to this list-*
## -This should occur even when adding an allergen while adding the account-{color:#ff5630} *(IGNORE until future story)*{color}
# The user should be able to type in a menu item name (with auto-fill/complete) manually

* User should also be able to edit the list so that with the Save button, data saves appropriately on the account.
** -If the user attempts to remove an item that has a marked allergen, block the attempt and inform the user why. i.e. if the ‘Milk’ allergen was added, and the user tries to remove the menu item ‘Chocolate Milk’, throw an error: ‘This item cannot be removed because it is associated with an allergen’-{color:#ff5630} {color}{color:#ff5630}*(IGNORE until future story)*{color}
* History should also track changes here.

## Acceptance Criteria
* Menu Item Restrictions section expanded upon in the Restrictions tab.
Page shows current menu item restriction details.
* Edit mode allows user to type specific menu item names to enter them in and/or select to remove current menu items.
* -Items cannot be removed if there is a related allergen--error message displays.- {color:#ff5630}*(IGNORE until future story)*{color}
* Data saves appropriately. 
* History is tracked.
