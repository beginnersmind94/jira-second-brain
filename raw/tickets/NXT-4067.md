# NXT-4067 — Security - Security Policy

## Description
*User Story*
As a PrimeroEdge admin, I can create a security policy or multiple security policies and have them apply to groups or individuals based on security needs.

*Requirements*

* A security policy can be created in PrimeroEdge
** Security policies consist of a group of security settings that are applied to anyone the policy is assigned to
* Security policies can be assigned to groups
** Groups can only be assigned to a policy once
** Individuals can be excluded from a policy
* Security policies can be enabled/disabled
* Security policies can be deleted
* Policy is applied by group over organization
** For example, if there are two policies, one that applies to a specific group and one that applies to an organization, the users in the group will have the policy for the group applied instead of the policy for the organization
* There is a platform-level permission for accessing security policies
* There is a default policy present that applies to the entire organization
** The default policy applies the recommended settings put forth by PrimeroEdge
** The default policy can be edited by users
** The default policy can be disabled, but only if there exists at least one other policy and there are no users without a policy
*** If disabled and other policies are deleted, the default policy is automatically enabled
** The default policy cannot be deleted
* The following settings are determined by the security policy:
** Whitelisted IPs
** Password Strength Requirements
** Muti-Factor Authentication Settings
** Session Settings
** Note: These settings will be individually added in other stories
