# NXT-5749 — Recipe - Recipe Code Generation and Create Recipe

## Description
As a user, I want the item code to be generated automatically.

Prefix will be based off datasource and have an R, for example, LR would be Local Recipe so LRXXXXX.

[~accountid:5e93d1a4272c6a0c0e681d53]  worked on NXT-4291. I know some of these columns need to be updated. But I wanted to keep them for reference

{quote}1.	Have the NextItemCodeNumber per datasource stored in the database. [~accountid:5f46ca4dea5e2f0039c27b0f] , please review the option of a separate table to store these as this new table will have equal number of reads and writes.
2.	Have MinimumCodeLength column per DataSourceType to enable shorter codes for local items compared to global/shared.
3.	At the time of creating the item, start a transaction, get the NextItemCodeNumber by DataSourceId, adjust the length based on MinimumCodeLength and finally add the prefix.
4.	Check for a possible duplicate code before inserting the Item or catch any unique index errors and retry the process. I would suggest the second option as inserting item is not as frequent as say, placing orders or making payments. Also, two persons managing the same datasource attempting to add an item at exactly the same time is a remote possibility.
5.	Before ending the transaction, upon successful item creation, update the NextItemCodeNumber to NextItemCodeNumber + 1.{quote}

## Acceptance Criteria
Recipe Code is generated automatically per datasource.
