# NXT-36040 — Build the Roles & Permissions Stepper - Design

## Description
This story has been broken up into 2 parts - A design part and an implementation part in the next sprint. Here is the original story. Acceptance Criteria has been updated accordingly.

As a SchoolCafé admin, I want a more intuitive way to add roles and permissions so that I can do it faster with less mistakes.

*Flow:*

* Open SchoolCafé and browse to Settings > Roles & Permissions (R&P)
* Click on + New and browse to that page (fly out)
* Complete the form, select your resource permissions as you step through the stepper
* Once complete, submit changes and return to R&P page

*Notes:*

* This stepper needs the hierarchy updates to be completed in order to finish the story
* All the updating will go back to the CRUD table, this is just a better interface to working with that table

*Requirements*:

* Requirements will be by page and in flow
* On the R&P page, the following changes will occur when the + New button is clicked and the page / fly out is opened
* 
* Add a stepper control similar to the one used for imports + New
** Same UI / UX and flow
** Stay consistent
** See screenshot 1 for example
** Instead of icons, can use numbers
** Step Names
*** Modules
*** Pages
*** Sub-Pages
* 
* Step 1
* Page title: Add Custom Role
* Step Title: Modules
* Step Body Content:
* Role Name – same as before, no changes
* Description – same as before, no changes
* Remove the existing modules drop down and replace with this grid
* Grid:
** Make UI consistent with other pages
** Shows modules per team grouping
** Show only modules they have purchased / licensed
** For each module, need to select the CRUD
** Each row has an “All” toggle
** If they select Yes, select all the checkboxes in the row
** If they select No, unselect all the checkboxes in the row
** Default select to Yes for each row and therefore default check Yes for each row’s checkboxes
** CRUD checkboxes work the same as the R&P Gen Permissions table
** Each row has a “Continue Editing” Option
** If select “Complete”, then when click Next:
** Apply their selection to all the children for that module
** Do not show them any more screens for that module, they are completed with editing permissions for that module
** Default Workspace selection to Complete
** Default the rest to Continue
** If select “Continue”, then when click Next:
** Apply their selection to all the children for that module
** Show them the next level of screens for that module because they want to continue editing permissions for that module
* Previous button closes the page and makes no updates to the system
* Next button:
** On click, commit the changes to the database by saving the role name and description and saving the permissions for each resource for each module per the selections made in the CRUD
* See Screenshot 2 for example UI
* 
* Step 2
* Page title: Add Custom Role
* Step Title: Pages
* Step Body Content:
* 
* Inside of the Step 2 stepper, we need another stepper called Inside Stepper
* Inside Stepper control lists all the modules as steps
** Show only the modules they selected “Continue” from Step 1
** Possible modules (show in this order if shown):
** Workspace
** Accountability
** Eligibility
** Account Management
** Item Management
** Inventory
** Menu Planning
** Production
** Insights
** System
** Stepper should show percent complete / done
** When hit next on a module, then that module is complete
* The inside stepper has the CRUD grid like on Step 1, but it shows pages for a given module instead
* Grid:
** Same Grid and UI as on Step 1, but show module and page names instead of team and modules
** Grid will show only one module at a time in a table b/c there are over 200 pages
** For each page in that module, need to select the CRUD
** Default load all the saved values from the way they are in the database
** In step 1, all resources had values set when they clicked next, so just load those values from the database in this step and update the grid checkboxes per those values
** Each row has an “All” toggle, same as step 1
** CRUD checkboxes same as step 1
** Each row has a “Continue Editing” Option same as step 1
** If select “Complete”, then when click Next:
** Apply their selection to all the children for that page
** Do not show them any more screens for that page in next step
** If select “Continue”, then when click Next:
** Apply their selection to all the children for that page
** Show them the next level of screens for that page in next step
* Previous button goes back to previous step in Inner Stepper and makes no updates to the system
** On first step, disable, there is nowhere to go
** Each module has a previous button that does something different than the outer Step 2 previous button
* Next button:
** On click, commit the changes to the database by saving the permissions for each page to the CRUD table
** Each module has a next button that does something different than the outer Step 2 next button
** On last step, disable, there is nowhere to go
* 
* The excel attachment has a list of all the pages that need to be added sorted by which module
* 
* Outer Previous button goes back to Step 1 in the Outer Stepper and makes no updates to the system
* Next button:
** Disable until all the Inner Stepper steps have been completed (they clicked next button)
** On click, go to Step 3 of Outer Stepper; no writing to database necessary since handled by Inner Stepper
* See Screenshot 3 for example UI
* 
* Step 3
* Page title: Add Custom Role
* Step Title: Sub-Pages
* Step Body Content:
* 
* Inside of the Step 3 stepper, we need another stepper called Inside Stepper just like in Step 2
* Inside Stepper control lists all the modules as steps just like in Step 2 with the same rules (just copy the inside stepper from Step 2)
* Add the module name as a title for the grid above it
** We didn’t do this before since the module was already in the grid
* The inside stepper has the CRUD grid like on Step 2, but it also shows sub-page rights for a given page
* Don’t show a page if there is no sub-page content
** Note: most pages don’t have additional sub-page content
* Grid:
** Same Grid and UI as on Step 2, but add the sub page content to it
** Grid will show only one module at a time like in Step 2
** For each sub-page in that module, need to select the CRUD
** Default load all the saved values from the way they are in the database
** In step 1, all resources had values set when they clicked next
** In step 2, they updated those values in the database
** So just load those values from the database in this step and update the grid checkboxes per those values
** Each row has an “All” toggle, same as step 2
** CRUD checkboxes same as step 2
** Each row has a “Continue Editing” Option, but it is different from the first two steps
** Disable the Continue button since there is nothing left to continue to
** Auto select the Complete button and disable it too
** When hit next, don’t worry about applying changes to children because there are none
* Previous button goes back to previous step in Inner Stepper and makes no updates to the system
** On first step, disable, there is nowhere to go
** Each module has a previous button that does something different than the outer Step 2 previous button
* Next button:
** On click, commit the changes to the database by saving the permissions for each page to the CRUD table
** Each module has a next button that does something different than the outer Step 2 next button
** On last step, disable, there is nowhere to go
* 
* The excel attachment has a list of all the sub-pages that need to be added sorted by page, use this for your list of pages
* 
* Outer Previous button goes back to Step 2 in the Outer Stepper and makes no updates to the system
* Next button:
** Disable until all the Inner Stepper steps have been completed (they clicked next button)
** On click, go to Step 4 of Outer Stepper; no writing to database necessary since handled by Inner Stepper
** Change the text of the Next button to “Complete”
** On click, close the stepper and return the user to the Roles & Permissions page
