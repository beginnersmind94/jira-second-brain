# NXT-70194 — Revenue Category Groups & Program Filter

## Description
*Parent Page:* Account Mapping (Financials > Configuration > General Ledger Configuration)

*Scope:* This story adds revenue-specific category grouping and a Program filter to the existing Account Mapping page. It does not modify the existing page structure, edit drawer, inline GL mapping, Rules drawer, progress banner, filters (Base Type, Status, Entry Type, Search), section dividers, or batch save behavior.

## Acceptance Criteria
h2. 1. Revenue Category Groups

The system displays 77 system revenue data points, grouped by category (not by program):

* Student Meal Sales – Free (6)
* Student Meal Sales – Reduced (2)
* Student Meal Sales – Paid (3)
* Adult & Staff Meal Sales – Taxable (18)
* A La Carte – Student / Tax Exempt (7)
* A La Carte – Adult/Staff/Visitor / Taxable (21)
* Federal Reimbursements (7)
* State Reimbursements (7)
* State Programs (2)
* Operating Revenue (3)
* Year-End Close (1)
* Other Revenue (user-created items)

----

h2. 2. Program Filter

A new "Program" dropdown filter is added to the existing filter bar, positioned after the Entry Type dropdown and before the Search input.

||Option||Label||
|All Programs|Default selection|
|SBP|School Breakfast|
|NSLP|National School Lunch|
|ASP|After School Snack|
|CACFP|Child & Adult Care|
|SFSP|Summer Food Service|
|SMP|Special Milk|
|FFVP|Fresh Fruit & Veg|
|State Programs|State Programs|

Behavior:

* Follows the same Apply-to-filter pattern as the existing Base Type, Status, and Entry Type filters.
* When a specific program is selected and applied, only data points associated with that program are shown across all category groups.
* Category groups containing matching data points auto-expand. Groups with no matching data points collapse or are hidden.
* Each visible category group header updates its account count to show: "{filtered count} of {total}" (e.g., "3 of 7").
* A context banner appears below the filter bar when a program filter is active:
** Program name
** Total filtered data point count with active/inactive breakdown (e.g., "12 data points — 10 active, 2 inactive")
** "Clear Filter" action: resets Program to "All Programs" and applies
* The context banner is informational only — it does not contain any actions.
* Program filter composes with existing filters: if Base Type = Revenue and Program = NSLP are both applied, results show only Revenue-typed NSLP data points.

Search behavior with Program filter active: Search results respect the applied Program filter. When search returns results, groups containing matches auto-expand; groups without matches collapse or hide.

----

h2. 3. Save Warnings

This extends the existing batch save behavior. When the user clicks "Save Changes" in the existing save bar:

* If any active data points remain unmapped, display a warning dialog before committing:
** Message: "You have {count} active data points that are not mapped to a GL account. Unmapped data points will not generate journal entries during posting. Save anyway?"
** Actions: *Save Anyway* / *Go Back*
* If all active data points are mapped, save proceeds without additional prompts.

----

h2. 4. Discard Changes

A *Discard Changes* button is added to the existing save bar, visible whenever pending changes exist. This provides a way to abandon all staged changes without needing to trigger the navigation guard.

* Confirmation dialog: "Discard all unsaved changes?"
* Actions: *Discard* / *Cancel*
* On Discard: Reverts all pending changes to last saved state. Toast: "All changes discarded."

----

h2. 5. Navigation Guard

If the user has any pending unsaved changes (from individual edits in the drawer or inline mapping) and attempts to navigate away from the page:

* Dialog message: "You have unsaved changes that will be lost."
* Actions: *Discard & Leave* (navigates away, reverts all pending changes) / *Stay on Page* (returns to page with pending changes intact)
