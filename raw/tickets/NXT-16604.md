# NXT-16604 — Serving Date Navigation for Plan and Record

## Description
When a menu planner creates menus for the school sites to serve to their students, they also assign those menus to a calendar, so that the menu items are associated with a date. (Example: Hamburger on Monday, Taco on Tuesday, Pizza on Wednesday, etc)

A School Manager goes to a calendar to select those dates, and enters in the quantity of each menu item that they’re intending to serve, and also to enter their final “end of day” information. A Calendar is the easiest method of selecting the dates, and the manager needs to be able to view what status is on different dates. We want it to be evident to a manager when their tasks are complete, or when new menus have been assigned to their site. 

!image-20210622-140337.png|width=100%!

Requirements:

* Site Code / Site Name will be combined into a single select / search dropdown menu similar to Account Management:

!image-20210622-143725.png|width=547,height=430!

* Calendar component needs to be developed as a shared component
* Assumption:
** Menus have to be assigned via Menu Planning
* [https://fullcalendar.io/demos|https://fullcalendar.io/demos|smart-link]  is the calendar used in Menu Planning

## Acceptance Criteria
There will be a method of selecting the phase of task that I need: Production Plan, Production Record, Production Withdrawal

There will be a clear key or legend that coordinates with the phases that my menu is in: Plan Incomplete, Plan Complete, Record Complete, Withdrawal Complete

There will be a method of navigating from the current month to future months or past months.
