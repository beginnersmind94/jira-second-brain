# NXT-18534 — Income Surveys - Surveys - Add support for Student Benefits search

## Description
As a district admin, I would like to be able to view, group, and export lists of students whose surveys indicated that their households wish to receive Optional Benefits for a given academic year, so that I can provide that data to any appropriate third-parties.

*Summary*

# Add a multi-select ‘Benefits’ filter and ‘Benefits’ column to the Income Surveys grid.
# Add an ‘Info Sharing’ column to the Income Surveys grid.

*Requirements*

* Formerly, this was a separate page (see example from Classic below). However, we can leverage our existing Income Surveys grid to meet this need.
* In Income Surveys, do the following:
** {color:#6554c0}*(May not yet be shown in mock-up)*{color} In the list of filters, add a new drop-down filter: _Benefits_
*** Menu should be *multi*-select
*** Menu should show all of the benefits associated +only with the selected Academic Year+, even inactive ones
*** Default value: “All”
** In the list of columns, add a new column: _Benefits_
*** This column should be hidden by default
*** When the user chooses to show this column, the column should show any benefits that were selected in association with each survey record
**** If none exist for the survey, this column can be blank
** In the list of columns, add a new column: _Info Sharing_
*** This column should be hidden by default
*** When the user chooses to show this column, the column should show Yes/No in association with each survey record
**** If no data exists for the survey, this column can be blank
** When a user applies the changes, only show those surveys where those benefits were accepted or specific info sharing preferences were indicated.

## Acceptance Criteria
Verify that both a ‘Benefits’ filter has been added to the Income Surveys page with a default value of ‘All’

Verify that both a ‘Benefits’ column and ‘Info Sharing’ column have been added to the Income Surveys column search

Verify that these columns display any benefits tied to the survey record, and are hidden by default

Verify that when a user applies these options, the grid only shows those surveys where those benefits were accepted or info sharing filter is set to the selected value (even if the columns are hidden).
