# NXT-31553 — Add Detection Success or Failure Indicator to Cart Screen

## Description
As a cashier, I want to see the success or failure of when the Edge device detects a tray properly so that I can see trends in failures and no that the Edge device is working even if there are no results.

*Application*: POS

*Summary*

We will add a visual indicator on transaction/cart screen next to current green dot to show if success or failure of tray detection.  We had this as an original request, but didn’t do it. No need to wait on Amniscient for this one.

*Flow:*

* While cashier is using POS to sell food to students:
* Detection cycle begins when student walks up to cashier
** Cashier selects student
** Edge detection triggered
** Edge sends results to POS
** POS shows the Results Detected image in the lower left part of the screen next to the green dot (new)
** POS saves results and optionally loads the cart
** Transaction is completed

*Assumptions:*

* This was completed in classic and you can copy the code from it
* Get with Saketh on the classic code
* You can leverage existing Results received MQTT message as a trigger
* UX involvement might be required
* It is assumed in these requirements that you are using a green image for success, a red image for failure and a yellow image for everything else

*Requirements:*

* Before triggering the edge device, show no image
* When Edge device is triggered, show the yellow image
* When results are returned
** Show the green image if count of items > 0
** Else show the red image
* When transaction is completed, show no image
*  
* Security – no additional requirements necessary

*Mockup*:

* Ask UX team if you feel necessary

## Acceptance Criteria
* Before triggering the edge device, it shows no image
* When Edge device is triggered, it shows the yellow image
* When results are returned, it shows the green image if count of items > 0
* When results are returned, it shows the red image if count of items = 0
* When transaction is completed, it shows no image
