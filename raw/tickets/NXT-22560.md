# NXT-22560 — INV - Vendor - Items - Contract Info Side Panel

## Description
As a FSD I want the Contract Items to be fully functional so I can manage all aspects of contract items.

Other additions:

* Pricing Conditions ([NXT-7241|https://primeroedgenext.atlassian.net/browse/NXT-7241])

*Requirements*

* Make the side panel a fixed component
** Provide a non-editable mode as default
** -Provide an edit option- ([NXT-22561|https://primeroedgenext.atlassian.net/browse/NXT-22561])
** Rename this section to ‘Contract Item Information’
* Change column order:
** Vendor Item #
** Purchase Price 
*** Delivery Fee
*** Processing Fee
*** Net Off Invoice 
** Product Lead Time
** Min Order Qty
** Max Order Qty
* Resolve existing issues

+Issues with current functionality:+

* Requirement validations are not being given to the user
** Mandatory attributes are not clearly communicated
** Ex: go to Add Items to Contract, select an item, type a Vendor Item # and click Add. Nothing will happen, because Purchase Price is required but the system does not say so
** Mandatory attributes are Vendor Item # and Purchase Price
* Min/Max Order Qty are not being handled correctly
** See below image. User is given a message about Max Qty being greater than Min Qty, even though the user has not entered a value for Max Qty
*** this is an issue if the user types something then removes it
** Both of these fields are optional. They are independently optional as well. The only validation is making sure Max is greater than Min *IF* the user enters both.
** Min and Mix should never be the same value
*** system is currently allowing this
* ‘Add Info to Item’ is the wrong button text
** the user is not adding info to the item. They are adding the required info to add the item to the contract
** text should be ‘Add Item to Contract’
* Currency values should be displayed as such
** any field with a currency value should be formatted as a currency
** Ex: “$11.00” not simply “11”
