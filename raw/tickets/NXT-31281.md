# NXT-31281 — IM - Create New Item - Pack Size Config Enhancements

## Description
Make UI enhancements to the Pack Size Configuration section to improve the experience.

See related story: [https://primeroedgenext.atlassian.net/browse/NXT-30168|https://primeroedgenext.atlassian.net/browse/NXT-30168|smart-link] 

*Requirements*

* UI Changes
** Capture Highest/Lowest outside grid before configuring
** Place buttons in a single location (header)
*** Highlight mandatory attributes on button click if not provided 
*** *Note:* removing ‘Complete’ button for now, using single Save button (for Product Info)
**** Instead of Complete button, show the save indicator as green, being saveable 
**** Note: indicator likely needs to change for UX standardization
** Convert weight into higher measures for higher units (i.e. 4000 grams → 8.81 Pounds) *view mode only*
*** We will not convert in Create/Edit mode, but will convert after save for view mode
**** Ex: 1,400 Ounces > 87.5 Pounds
*** This should be only API work
** Improve typing experience (!)
*** right now it will miss characters if you type too quickly 
*** Example: I quickly type ‘Package’ but only ‘Pkage' is entered
** Make font smaller if needed
*** probably not necessary with Highest/Lowest
* Rules
** Weight should be editable on the lowest unit
** Unit Type radio option is only provided the first time 
** If user changes Unit Type radio option once more than one row has been insert, prompt before changing
*** “This will restart Pack Size configuration. Are you sure?”
** Enforce interdependent relationship between Pack Size and Food questions
*** Do not allow users to access question Save & Continue button until either skipping or saving Product Info
