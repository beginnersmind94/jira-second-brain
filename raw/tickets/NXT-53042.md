# NXT-53042 — OpenID - District Authentication - Notify - SchoolCafe Enabled

## Description
As a Customer Support user, I want to be able to setup a district’s OpenID authentication provider so that they can use SSO

*Notes*:

* This story is covering the non completed functionality from its parent story regarding what to do when enable / disable SchoolCafe Authentication



*Requirements*:

* These requirements apply to the System > Security page
* 
* On the grid in the tab
** Preferred
*** If the user clicks to make the SchoolCafe the preferred method:
**** Run the mass password reset process described below



*Mass Password Reset:*  

* If we enable SchoolCafe password usage for the first time, or re-enable after some time, there will probably be a lot of users with no password or with expired passwords
* So we must follow our normal password setting and expiration process for all accounts affected
* Therefore, do the following
* Check all accounts with no password
** For each one, follow the process to set password for new accounts which includes sending them emails etc.
* Then check for accounts with expired passwords
** For each one, send them the expired password emails and follow that process

## Acceptance Criteria
* Email sent to users when SchoolCafe is enabled and they don’t have a password
* Email sent to users when SchoolCafe is enabled and they don’t have an expired password
