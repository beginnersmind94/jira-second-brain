# NXT-30656 — Onboarding - Global Catalog (GTIN)

## Description
As Item Manager Admin I need to be able to onboard new Items using Global Catalog Item information  using an excel file in order to speed the process of bringing customers working with SchoolCafe’s Backoffice.

## Acceptance Criteria
*Entry Point:*

* System → Import Dashboard → Import Set Up
* The user should be able to on board Items using Global Catalog Items information based on the templates defined by the system Item template.

[^Global Catalog - GTIN (294b4f3a-fbcb-451b-860b-b4c98fda97ac).xlsx]

*onboarding process of Items details from Global Catalog shall follow the following procedure/steps*

*Step 1 - Pre-Upload*

* Select Import Type
** Options
*** *IM  Global Catalog*

* 
** Select Data Source
*** By Default set to Local
If School District has more than one allow the user to select from drop down box
*** Only display Data Sources Created for/by the school district
*** Items cannot be imported to CN (Future Global Catalog)

 

*Step 2 - File Upload*

* The user shall select between downloading a system template or uploading  a new template
** If user selects to download template the system will download a template (excel format)  into the user's download folder (template user will use to fill in data)
If the user selects upload a template then the user shall be prompted  to drag and drop file from user location  to drop area (File with data fill in)
If user pick Select then the system shall prompt the user for the location where the file is located and select the file to upload.
** Once file is uploaded the system shall provide the user with visual confirmation of success or failure
*** It shall Provide a message and the name of file uploaded and file size
** The user shall be provided with an option to remove/delete the file uploaded
*** User shall be provided with confirmation of action to remove/delete uploaded file
** The user shall be provided with an option to move to  the next step (Step 3 - Field Mapping)
* The user shall have the option to move to the prior screen at any time during the process

 

*Step 3 - Field Mapping*

* Fields shall be mapped by selecting fields from source file (loaded template) and dragging and dropping to the mapped area
All mandatory fields shall be mapped
* Once fields are mapped the user shall be provided with:
**  an option to preview the file data as mapped;
an option to return to the previous step
an option to continue to next step (Step 4 - Review)
* If user selects the previous option the system shall remove the uploaded file and  move to Step 2
if the user selects continue then the system shall move to step 4

 

*Notes:*

* All fields mark in red are *MANDATORY*

* *Step 4 - Review*

* The system shall display:
**  map fields between Source and Target (Source - SchoolCafe)
** Current File Name (Data File use)
*** User shall be able to rename the file
* The system shall provide the user with an option
**  to enable (Activate) the file mapping;
*** an option to edit the mapping;
*** and an option to save the mapping
* If user selects to save the mapping then the user shall receive visual confirmation of success or failure of the saving process and ask the user for acknowledgement
* If the mapping is successful then the user shall be directed to the Import dashboard where the user shall have the option:
** to disable the mapping
** to import the data according to mapping;
*** If user selects Import then the system shall use created mapping for import process
**** The user shall load data file ( drag and drop area or selecting from specified file location)
Once file is loaded the system shall display data as mapped
**** Then user shall provide the user with option
***** Cancel the import process
Move the previous page
***** Complete the import file process
****** User shall be presented with information about file import and be presented with options
******* to go to import dashboard
To go to Monitoring page where the user can view the progress of file import and view log file

.*General:*

* Check for duplicate  GTIN. Duplicate GTIN are not allowed
** If duplicates are found flag them as duplicate in log transaction file and do not store information
* Default Item Status = Active
* Data Source:
** Default is Local
** If local, the data for this is editable.
** If Global Catalog or CN database is the source, then other fields are not editable unless a copy is made.
** GTIN \[GTIN describes a family of GS1 (EAN.UCC) global data structures that employ 14 digits and can be encoded into various types of data carriers].
*** 14 character 
** The import file most be populated (has data to import)
** *IMPORT:*
*** The system shall take the GTIN code from source file and search for GTIN code in Global Catalog and import all item information in SchoolCafe  data source (local). Similarly as it is done in Item Management’s Global Catalog -> Item Catalog Page
**** Item Detail
**** Nutritional Information
****  Allergen Information
**** Stock Item 
**** Pack Size (Hierarchical) information
*** Validation Rules:
****  1. If GTIN less than or greater than 14 then display message “GTIN should be 14 digits” 
****  2. If GTIN does not match Global Catalog display message “GTIN does not match Global Catalog “
****  3. If GTIN match successful  the display message “GTIN Import Successful”
****  4. If GTIN already exist in local database in local data source and display message “GTIN already exist in Local “data source”
**** 5. If Product code of the GTIN already exist in local database  in local data source and display message “Product Code already exist in Local “data source”
**** If Item Description (Name) of the GTIN already exist in local database  in local data source and display message “Item Name already exist in Local “data source”
**** When error is found log the error and proceed with next GTIN



All the white spaces - Start and End of the fields should be trimmed

Once the import is completed, the user should be able to view the log as per UX design (attached below). The summary should list the total number of records in Excel, how many are processed, how many are created, how many are updated, how many are exceptions, Start Date/Time, End Date/Time and Status (as Completed)

Excel - user should be able to download Excel with all the original data + Import Status (Success/Fail) + exception messages associated with that record
