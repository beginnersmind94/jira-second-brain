# NXT-55503 — DMF - Dashboard - Select Which Modules to Import

## Description
As a Customer Support role user, I would like to be able to select which modules I am importing so that I can skip the ones I don’t want

*Notes:*

* After our first prototype review, Dallas asked that we be able to optionally select which modules/databases to import
* In practical terms, this really means to decide if we want to import front office and back office sinch the framework and system must be imported no matter what

*Requirements:*

* On the main grid, make these changes:
* When click Add New button, do the following:
* Add checkbox list below the District
** Data: List of modules/databases from the nested grid
*** Framework
*** System
*** Front Office
*** Item Management
*** -Inventory-
**** -Can only select if select Item Management-
**** -Note: we aren’t doing Inventory yet, so we can add it back to the list once done-
*** Family Hub
** Auto select all of them by default
** Make all the checkboxes read-only except Family Hub so that they always have to do those for now
*** Later, we will make Item Management optional, but for now leave required
*** Therefore, they can select / unselect only Family Hub for now
* 
* Update the nested grid
** If the user unselected Back Office, then make those fields read only and skip them
** If the user unselected Front Office, then make those fields read only and skip them
* 
* Migration
** If the user unselected Back Office, then don’t run them during the migration process
** If the user unselected Front Office
*** Don’t run them during the migration process
*** Do run the any BO scripts that FO has dependency on

## Acceptance Criteria
* User is able to optionally select if they want to migrate front office or back office data
