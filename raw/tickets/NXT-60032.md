# NXT-60032 — TH - Sec - SC Auth - Setting

## Description
*As a* SchoolCafé admin  *I want* to control if my district allows usage of the mobile app in SchoolCafé *So that* I can prevent users from trying to use it if we don’t want them to

h3. Comments:

* We need to add a setting in System settings that can be used to control if districts are using the mobile app or not.
* The idea is that even though we have the mobile app, some districts might be security conscience and not want to allow their data outside their domain, so we need a way to turn off access for those districts
* We will then use a security setting that when a user tries to login or authenticate their mobile app, we can check this setting and return a code/status back to the app saying yes/no
* Note: in this story, we create the setting. Usage of the setting will be done in other stories.

h3. Requirements:

* Do the following in System Settings:
* In System Settings, add a new property
** Setting Name: Enable TeamHub
** Type: Internal
** Code: TEAMHUB
** Value: Yes/No
** Default: Yes
** Category: Configuration
** Module: System
* 
* *Security*
** Only Customer Support roles can make this change

## Acceptance Criteria
* In System Settings, add a new property with the values defined above
* Only Customer Support roles can make this change
