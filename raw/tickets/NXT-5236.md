# NXT-5236 — Direct Certification - Summary

## Description
*User Story*

As a district admin I want to see a high-level overview of the impact a direct certification (DC) file had on my student population.

*Mockup*

[https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=47124%3A0&scaling=scale-down-width|https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=47124%3A0&scaling=scale-down-width|smart-link] 

*Requirements*

* User is able to click on any of the DC files in the List ([https://primeroedgenext.atlassian.net/browse/NXT-5235|https://primeroedgenext.atlassian.net/browse/NXT-5235|smart-link]) and see a summary of information.
* Show the following information:
** File Number
** Program
*** This will be one of the following:
**** Assistance Program
**** Other Source Categorical Program
** Type
*** This will be one of the following:
**** Direct Certification
**** DC Medicaid (Free)
**** DC Foster
**** Even Start
**** Head Start
**** Homeless
**** Migrant
**** Runaway
**** Pre-K
**** DC Medicaid (Reduced)
** Status
*** This will be one of the following:
**** Pending
**** Preprocessed
**** Processed
**** Deleted
** Entry Method
*** One of the following:
**** File Import
**** Manual
** Import Date
*** Date/time the file was uploaded.
*** User that processed the file (Firstname + Lastname)
** Process Date
*** Date/time the file was processed.
*** User that processed the file (Firstname + Lastname)
** Approval Date
*** ‘File Date’ in Classic.
*** It’s the date/time designated if this file was backdated during the upload process.
**** User will have an option to say ‘this file represents information as of X date,’ which may be a date in the past.
** Statistics
*** Show the impact on the student population - what was the percent that were Economically Disadvantaged before, what was it after?
**** Example: Eco Dis Impact: 8% → 22%
**** Economically Disadvantaged is the percentage of the student population that has an Eligibility of Free and Reduced against the active student population as of the file approval date.
*** Matched Students
**** Number of students who were successfully matched based off information on this file.
**** If multiple Approval Types are present in the file, show the number matched per each.
***** Example: The file had SNAP, TANF, and Medicaid in the same file.
****** Show that X were SNAP, Y were TANF, Z were Medicaid.
**** Break this number down further to show the impact on the overall student population.
***** Show many changed from Free, Reduced, Paid.
***** We can likely use a pretty pie chart here.
*** Total Records on File
**** How many entries were in this file?
***** Break this down by Approval Type if multiple were present, like the Matched students.
*** Match Rate
**** Show the percentage of matched students based off the total records on file. Round off to 2 decimals, round to nearest.
**** Example: 500 matched students, 1500 on the file. Match Rate = 33.33%
** A matched student list should also be available - this is covered in a separate story.

## Acceptance Criteria
User can 'enter' a DC file to see more information.
User can see the impact the file had on the student population via a summary of information tied to the particular DC file.
The Information is accurate - any comparisons against student population need to keep the file approval date in mind.
