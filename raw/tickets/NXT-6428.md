# NXT-6428 — Users Phase 2 - Users - Create, View and Edit User Information

## Description
*User Story*

As a PrimeroEdge admin, I would like to have a screen for viewing and editing existing users in operation in a way that that more closely fits within the new UI/UX style and overall product.

*Details*

* Users page (accessed from main *Users* screen) should be updated to fit the new UI/UX style
** This applies to both viewing (View Mode) and editing (Edit Mode) individual users
* Users page connects with Keycloak to properly update user information
* User Information can be viewed
** Profile Image
** First/Middle/Last Name
** Email
** Work Phone
** Cell Phone (if applicable)
** Active/Inactive status
** Notification if password is expiring within 30 days (can be deferred if needed)
** Organization & Permissions section
*** Org. Level
*** Region(s)
*** Area(s) (if applicable)
*** Site(s) (if applicable)
*** Role(s)
* Add an edit button on that page that allows an admin to edit User Information.
** All fields listed above can be edited except Email address, plus:
*** Random password can be generated and emailed to user (handled in separate story)
** User can save or cancel changes
** Any saved changes are properly reflected  when viewing the user’s information

## Acceptance Criteria
* Users page is properly updated to reflect new UI
* Users page properly hooks into Keycloak
* User information above can be viewed
* User information can be added, edited, and saved as specified above
