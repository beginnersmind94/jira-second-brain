# NXT-19970 — Income Survey - Edit Survey Details

## Description
As a district admin, I would like to be able to edit submitted Income Surveys, so that I can view individual details as well as make corrections or changes as needed.

*Summary*

Allow users to edit individual details for a survey.

*Requirements*

* In Eligibility → Forms → Surveys, when a user clicks the ‘Details’ icon next to a survey and lands on the Survey Details page, display an edit pencil for the following sections (except ‘Comments’):
** Students
*** User should still be able to both add and search (may not be fully pictured within mock-up)
*** At least one student must remain. Message: “Survey must contain least one student.”
** Household
** Contact Info
** Details
*** (*If* applicable. This should be hidden if the system setting to show ‘Details’ is disabled)
** Comments
*** For this section, only an ‘Add’ button is needed (to add new comments), because existing comments cannot be edited.
** Documents
*** Design may need to be renegotiated. Please let Daimien know if a mock-up change is required.
** Adult Signature
** Other fields: 
*** Received Date
*** Language
* For each section, open a drawer that allows the user to change details in the section.
** Existing validation rules for each section should apply . i.e. Only a student’s first and last name are required, one income response must be selected, no fields in Contact Info are required (but some must be validated whenever the user begins typing in a value), etc.
** Save the details when the user clicks a ‘Save’ button in each drawer, except for those sections where additional actions (Add/Delete) are required. Those can be saved immediately.
* If Eligibility is not determined yet, we can hide that section.
* This story is complete when everything on the Survey Details page is functional except for the Process, Pending, Delete, and Restore buttons.

## Acceptance Criteria
Verify that the Survey Details page displays individual sections with information corresponding to each step of the survey, each with a button to edit or add details

Verify that the validation rules for each section match the corresponding section on the Add Survey wizard

Verify that information is saved when data is either added or the user clicks the ‘Save’ button
