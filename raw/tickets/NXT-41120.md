# NXT-41120 — PNA and Sale Data ADF Optimization

## Description
These 2 pipelines will only run when a change is detected in the data.
It will check if any new transaction has occurred in the last 1 hour for Sale Data ADF and if any eligibility change has occurred for any student.
Neither the stored procedures used to generate the parquet are modified nor the ML code base.

## Acceptance Criteria
Currently the ADF Pipelines are set to run on hourly basis and this add more cost to services 

The expectation is to reduce the frequency + Optimize the service so that the the latency could be reduce and it will save the cost on the pipeline execution
