# NXT-67674 — Account Deletion Logic

## Description
*As a* system
*I need to* prevent deletion of accounts that are in use
*So that* data integrity is maintained

## Acceptance Criteria
* When a user attempts to delete an account, check if it has any posted transactions
* If posted transactions exist, block deletion and show error: "This account has posted transactions and cannot be deleted"
* Are you sure? This account is mapped to [System Account Name]. You'll need to assign a new account to this system account before transactions can post.”
* If user clicks Yes in the confirmation modal, proceed with deletion
* After successful deletion, remove the account from all lists and dropdowns
* -If the deleted account was mapped to a system account, the mapping is removed-
* -A warning banner appears: "1 account requires mapping" (or updates the existing warning count)-
* -The system account that lost its mapping shows a warning indicator (page dealt with in separate story)-
* If user clicks No or Cancel in the confirmation modal, close the modal without deleting
