# NXT-7038 — Application Entry - 7. Submit: Pending Application

## Description
*Story* 

As an application processor, i want a way to set an application as pending for later.

*Mockups*

*Requirements* 

* User has a way to mark an application as ‘Pending’
* User can select from a list of pending reasons and enter a comment.
* Application is added into the system with a status of Pending, and does not get processed.
* The Eligibility Panel for Applications *with* missing hard stop info or pending reasons should read:
** _Due to pending reasons, we cannot reach an eligibility determination._
* Submit button will only be enabled if NO pending reasons exist and required fields are completed 
** If any hard stops exist for the fields below, Submit is disabled (Only option is to Add as Pending)

|*Field/Reason*|*Step*|*Hard Stop*|
|Incorrect categorical information|Benefits|Assistance Programs format > Incorrect|
|Incorrect household member(s) information|Household|Household size incorrect/missing|
|No signature|Submit|Signature confirmation box is NOT checked|
|SSN information missing/incorrect|Submit|Income apps > Incorrect format OR no SSN not checked|

* Pending > Triggers popup > “Pending Reasons” and comments
** The system hard stops that are missing from the application are populated in the pending reasons: _field checks in the table above_ 
*** Reasons added by the system cannot be removed by the admin (the 'x' is not available on the chip)
** Reasons added by admin can be removed by clicking the 'x' next to the reason
** The admin can add more Pending Reasons (from the API Pending Reasons list) 
** The admin must enter a comment 
** Popup options > ‘Cancel’ or ‘Pending’
*** Cancel > back to submit page to make changes 
*** Pending > application is put in Pending state with the reasons listed. Those reasons will also be tracked and shown in application details.  

Add as Pending > no benefit extension 

* Benefits *not* extended to students in the application since the application has not been processed due to pending reasons.
** Once application is processed as complete, benefits will be extended to matched students

Summary screen after adding as pending

* User should be shown the same message (as below) for the determination panel.
** Status: 'Pending' with message > _Due to pending reasons, we cannot reach an eligibility determination._

----

API Pending Reasons list:

* Reasons Exists > Admin can select any of the reasons from this section to put application in *Pending* status 
** Missing PFD information
** Restored application
** Incomplete
** Incorrect income information
** Incorrect categorical information
** Incorrect household member(s) information
** No signature
** SSN information missing/incorrect
** Possible duplicate information
** Reapplying after verification
** Other (please specify in comments)

## Acceptance Criteria
- applicant can submit application  
- submission of a pending application does not extend benefits to matched students
- submit is disabled if pending reasons or incomplete required fields
- Add as Pending button is available at any time for the admin (always enabled)
