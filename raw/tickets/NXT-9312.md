# NXT-9312 — Verification - Tracking - No Response Applications

## Description
*User Story*

As a district admin I want to see applications that have failed to respond and take action against them.

!image-20210120-220701.png|width=1431,height=833!



*Requirements*

Add a 2nd tab to the Tracking page.

* Second tab: No Response Applications
** Grid should contain applications that are:
*** Not complete.
*** Have been sent a follow-up notice.
**** We’re not going to make this dependent on the time since the follow-up notice; as soon as a follow-up notice is sent, we’ll let them process as No Response.
** Grid columns, typical search and filter options:
*** Application #
*** Eligibility
*** Basis
*** Type
**** This will be either: Sampled or Cause
*** Confirmation
**** This is a placeholder for future functionality.
*** Selection Date
**** Date the application was added to a sample.
*** 1st Notice
**** Date/time for when the 1st Notice was successfully created.
*** Follow Up Notice
**** Date/time for when the 2nd Notice was successfully created.
*** Days Since Last Notice
**** How many days has it been since the last notice was sent? This can be for either the 1st or follow-up notice.
*** Additional Columns (should be hidden by default, but can be made visible)
**** Sample Number
**** Student Names
***** single cell containing the name(s) of all students on the application
**** Applicant
***** applicant first + last name
**** Contact Information
***** contact information provided by the applicant
***** street address, phone numbers, email
** Action: No Response
*** User should be able to select applications from the grid, including all applications, and select the No Response option.
*** This will mark the application as complete, with no response.
*** Update the Application’s verification panel - 
**** Verification Result: Free/Reduced to Paid
**** Reason for Change: No Response
**** Track the Eligibility Change Date
***** This will be the date the new eligibility takes place.
***** *Adverse Action days need to be kept in mind for this - check the system setting for adverse action days.*
**** Track the Official who did this.
**** Track the Completion Date (date + time)
*** Any student on the application will need to be updated appropriately.
**** Current eligibility should be issued an end date which matches the Change Date.
**** New eligibility record should be inserted.
***** Start date = Change Date.
***** Price Type = Paid
***** Reason = Verification
*** The application should move out of the No Response grid at this point, and be available under the Verification Complete grid.
*** Inform the user of the success. Text messages below.

Confirmation on the Tracking page: Are you sure you want to mark the selected applications as No Response? Households have until the end of the verification season to respond.

Success on the Tracking page: Selected applications have been marked as No Response. They are available under the Completed tab.

## Acceptance Criteria
No Response Applications tab added.
Grid shows information by academic year, current AY by default.
Grid shows applications that are currently incomplete for verification.
Grid shows information for selected applications per the requirements.

User can complete an application as No Response.
Verification status updates appropriately for student, application, and verification.

Adverse Action Days should be used for reduction in status for the students.

Actions aside from History are only available for current academic year.
