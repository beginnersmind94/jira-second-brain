# NXT-5220 — Eligibility - Applications - Pending Applications

## Description
*User Story*

As an applications processor I want to view applications that are pending because of incomplete information so that I can work on collecting information, moving those applications to a complete/processed status and extending the correct eligibility to those families.

*Mockup*

[https://www.figma.com/proto/EVGaMgaqmoII4zO9Qff1Cq/Eligibility-4.0?page-id=3419%3A41216&node-id=3429%3A46258&viewport=285%2C48%2C0.13&scaling=scale-down-width&starting-point-node-id=3429%3A46258|https://www.figma.com/proto/EVGaMgaqmoII4zO9Qff1Cq/Eligibility-4.0?page-id=3419%3A41216&node-id=3429%3A46258&viewport=285%2C48%2C0.13&scaling=scale-down-width&starting-point-node-id=3429%3A46258|smart-link] 

*Summary*

* From Eligibility → Forms → Applications, complete the flow for the Pending tab by splitting it into ‘Pending Applications’ and ‘Pending Students’
* User should have a way to click into Application Info to complete processing or delete applications if needed.

*Requirements*

* Within the Applications page, add a new tab: _Pending Applications_
* Additionally, rename the existing _Pending_ tab to _Pending Students_
* Within the Pending Applications tab title, display the number of total pending applications in a circle value
** Ex: Pending Applications (100)
* Within the tab itself, display a grid with the following default columns:
** Application #
** Reason
*** Can be semi-colon or comma-separated
** Date Added
** Application Status
** Added By
** Actions -
*** View
**** User is taken to the Application Info screen. From here, the user can edit/process the application.
*** Delete
**** User is given a confirmation prompt. Text:
***** _This will delete the application. If there are any students on this application, this action will return them to their previous eligibility. Do you want to continue?_
***** Options: Cancel, Delete Application
****** Cancel will exit out of the prompt and do nothing.
****** Delete Application will update the status of the application to 'Deleted'. User then receives a confirmation message and the application will no longer be listed under Pending. Any students on the application need to have their eligibility records updated appropriately.
** Additional hidden columns (can be adjusted if needed):
*** Student ID
*** Student Name
*** Comments (show latest)
*** Date of Birth
* Out of scope: Refuse Benefits. *Will add this to a separate story in the future.*
** -Options: Cancel, Refuse Benefits-
*** -Cancel will exit out of the prompt and do nothing.-
*** -Refuse Benefits will open the Refuse Benefits drawer to allow the user to update the status of the application to 'Refuse Benefits'.-
**** -User then receives a confirmation message and the application will no longer be listed under Pending.-
**** -Any students on the application need to have their eligibility records updated appropriately (New status is either Reduced/Paid - Refused Benefits).-

## Acceptance Criteria
Applications page contains the tabs ‘All’, ‘Pending Applications’, and ‘Pending Students’ (formerly just ‘Pending’)

The All tab is still the default tab and is unchanged, and the Pending Students tab is unchanged other than it being renamed

In Pending Applications, the user sees the grid and filtering buttons displaying above, along with other columns they can show/hide if needed

User can click into an app and be taken to App Info to view/process. User can also delete a pending application.

Either action (Process/Delete) refreshes the grid
