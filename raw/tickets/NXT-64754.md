# NXT-64754 — SC - PBIRD - Activity Report – Enhancements

## Description
As a user viewing the Activity Report, I want to initiate the creation of a distribution configuration when one does not exist so that I can schedule and distribute this report.

----

h2. *Description*

Enhance the Activity Report page by adding a contextual actions menu (vertical ellipsis) next to the *Generate* button. This menu will serve as the entry point for Power BI report distribution and future report-related actions.

Users will continue to configure and generate the Activity Report as they do today. After validating the report output, users can access the ellipsis menu and select *Distribute* to begin the process of creating a distributed Power BI report (handled in a separate story).

----

h2. *User Flow*

# User logs into SchoolCafé
# Navigates to:
#* *Accountability → Reports → Activity Report*
# User configures report filters
# User clicks *Generate* to preview the report
# User closes the report viewer
# User clicks the *ellipsis (⋮)* button next to Generate
# User selects *Distribute*
# System initiates distribution flow _(handled in separate story)_

----

h2. *Out of Scope*

* Power BI distributed report creation flow
* Power BI Reports management page
* Any additional dropdown options beyond *Distribute*

----

h2. *Important Notes*

* Current *Generate* behavior remains unchanged
* No impact to Power BI report rendering or viewer experience

----

h2. *Functional Requirements*

h3. *UI Placement*

* A *vertical ellipsis (⋮) button* is displayed to the *right of the Generate button*
* The ellipsis follows existing UI/UX standards used elsewhere in the application

----

h3. *Dropdown Behavior*

* Clicking the ellipsis opens a *dropdown menu*
* Dropdown appears aligned to the ellipsis button
* Dropdown closes when:
** User clicks outside
** User selects an option

----

h3. *Menu Options & Extensibility*

* The dropdown contains:
** *Distribute* (first option)
* Additional menu options will be added in future stories
* The implementation must support easy addition of new menu items without rework
* Use existing shared component patterns for:
** Ellipsis menu
** Dropdown styling and behavior

----

h3. *Interaction*

* Clicking *Distribute*:
** Triggers navigation or modal to begin distribution flow
** No business logic for distribution is handled in this story

----

h3. *Permissions*

* Users with *View permission to the Activity Report screen*:
** Can see the ellipsis button
** Can open the dropdown
** Can click *Distribute*

## Acceptance Criteria
* Given a user has access to the Activity Report screen, when the page loads, then the ellipsis button is visible next to the Generate button
* Given the user clicks the ellipsis button, when the dropdown opens, then the Distribute option is displayed
* Given the dropdown is open, when the user clicks outside the menu, then the dropdown closes
* Given the dropdown is open, when the user selects Distribute, then the system initiates the distribution flow
* Given the user does not have access to the Activity Report screen, when they attempt to access the page, then the ellipsis button is not available
