# NXT-33112 — User Experience and Application Monitoring

## Description
The SchoolCafe application is in need of tools to help a support team quickly understand potential customer problems or existing customer issues.  These tools should help a support team ensure the customer experience is optimal and/or is restored as quickly as possible to the desired level.

## Acceptance Criteria
* The tool shall be able to send alerts to email, text messages, RingCentral based on metrics hitting a threshold
* The tool shall be able to generate support tickets based on metrics hitting a threshold
* The tool shall be able to collect, report, and generate alerts based for:
** HTTP errors: 5xx, 4xx, etc…
** Real User Monitoring (RUM) metrics
** Core Web Vital (CWV) metrics
*** Largest Contentful Paint (LCP)
*** First Input Delay (FID)
*** Cumulative Layout Shift (CLS)
* The tool shall be able to monitor and report on the metrics and performance over time
* The tool shall not impact the user’s experience
