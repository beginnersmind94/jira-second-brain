# NXT-21253 — Build Model Enhancements

## Description
As a site administrator, I want to build a new model for my site so that I can get more accurate results

*Application*: Face and Food Detection Website

*Flow:*

* Browse to the Food Detection Settings page and then the Model Management Page
* Click on the Create Model button and come to this page
* Type in a description and optionally add tags
* View information about the food items
* Select which food items I want in the model
* Click the Create Model button
* Validate required fields and other requirements
* Start the Model to build
* Make this model read only
* Go back to Model Management

 

*Background:*

* A preliminary page for this already exists
* We are expanding upon that page and adding new functionality
* Copy over the Description, Tags and button fields as-is with all their functionality as nothing will change with these
** I will not repeat the code details for the existing functionality, so be sure to keep it
* The Menu Items will change into a grid and have additional functionality
* The current page is a popup, but I expect you will need a real page since the grid will be big
* It is not needed to add a new menu link for this page

 

*Assumptions:*

* Here you will need to know if a given food item is trained for a site
* To know this, you will need to tap into the trained images linked to menu items
* It is assumed there is a database table that links images to menu items
* You should be able to get counts from that table
* Definition of an eligible menu items = A menu item that has at least one trained image

 

*Requirements:*

* Page Title: Build Model
*  
* Header Fields:
** Description – Copy from existing page, same everything, no changes
** Tags – Copy from existing page, same everything, no changes
** Clear – button, clears all changes to the grid and puts it back to like when freshly loaded
** Create – button, described in detail below
** Select All – button
*** When clicked, select all Menu Items that are eligible
*** Do not select items that are not eligible
*** A menu item is eligible if it has at least one trained image, else it is not eligible
*  
* Body Fields:
* Menu Items – grid
** Dataset = all the menu items from the selected district using menu API
*** this is the same list used in food labeling
** Be able to sort, paginate, filter upon columns, export to excel
**  
** Columns:
** Include – checkbox, clickable
*** If checked, add to the model
*** If not eligible, cannot check the Include box
**** If not eligible, either make it read only or hide upon load or in the validator prevent them from checking it; it is best to make read only or hide upon load
** Menu Item – text name of the food item, RO
** Trained – text, RO
*** Value = count of images trained, whole number >= 0
**** Know from Food labeling, count the number of times in database table that an image is linked to this menu item
*** Comment: Shows if the item is eligible
**** If > 0, it is eligible, else
**** This determines if the item can be used in the model
**** Don’t allow them to include in the model if = 0
** Meta Data – textbox
*** Used for the meta data to send to the model when build it
*** Auto load from the food menu item if it has it
*** If the menu API doesn’t have meta data, then don’t add this column
*  
*  
* Create button functionality:
* Remember, this button already exists, so copy over all its functionality
* What is described below are simply validators that need to run before execute the button
* Upon click, ran validators check:
** Description
*** Must have a value
** Menu Items:
*** There must be >0 menu items checked
**** If no menu items checked, show this error message and stop: “At least one menu item must be selected.”
***  
*** If you are blocking the checking of the ineligible menu items, you don’t need this, but if you aren’t then do the following:
*** No menu items with 0 trained images can be checked
**** if yes, show a warning message: “You have selected food items that are not trained.  These items will not be included in the model if you continue.  It is recommended that you cancel and go to the food training section to train these items.”
*** then give them the option to continue or cancel
**** if they cancel, do nothing
**** if they continue, uncheck the items w/o training and proceed to Schedule Model bullet point
*  
* If pass validators, then show them one last prompt: “Continuing will overwrite the model on all your edge devices with this model once it is complete.  Are you sure you want to continue?”
** Give them “Continue” and “Cancel” buttons
*** If Continue, go ahead with the existing code to build the model
*** If Cancel, do nothing
