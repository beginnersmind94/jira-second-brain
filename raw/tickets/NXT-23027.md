# NXT-23027 — IM - Item Details - Hierarchy - Slide-out Enhancements

## Description
Enhance the current item hierarchy visualization slide-out to better improve appearance and experience.

*Points:*

* specific guidelines regarding sizing, spacing
* “Child item per parent” label improvement
* how to display 3+ items in a single row
* preferred colors
* interactions (mouseover, click, etc.)
* dedicated Close button

*Requirements*

 Notes from previous story:

* We will use this control to display the visualization in the slide-out
** We will modify this slightly to disable certain features
*** These nodes will not be “movable”
** We will likely not use the animated edges
** We may not want to use graphical background
** emphasis will be put on more immediately related items
*** distantly related items or lesser used unit types such as mixed pallet or display shipper will not be emphasized in the graph
** Note: the same slide-out will be displayed for each item in the family tree
*** However, different nodes will be emphasized/centered depending on which item is being viewed when opening the slide-out
* There should be the ability to open items in a new tab from the visualization (*Note:* this is a “nice to have” not must-have)
** this would not be the case for the immediate related items, since these can opened from the Details card
** for other items, there would be some option after clicking on the item, to open this item in a new tab
* Each node will have a label that corresponds to the value of its ‘Unit Type’ (Each, Case, etc.)
** Display Item # on the node as well
** *Note:* look into displaying Item Description as a tooltip
* The node corresponding to the selected item should be centered and emphasized
** If we are viewing the ‘Case’ item, the Case node should be centered
** this could be highlighted a different color
** we +could+ show a smaller button for items with a Product Type of ‘Display Shipper’ or ‘Mixed Pallet’ (these are less likely to be used)
*** for these types we will display the Code instead of the Text
**** DS for Display Shipper
**** MX for Mixed Pallet
** Inactive items will be displayed in a gray node which is smaller than the others
** If there are many items in a single “row” we should reduce the size to fit all items
* Node links will be displayed
** these lines will represent relationships between different unit types
** these lines will have a label which represents the quantity of child item per parent
*** this label will show the number of child items per parent, Ex: “55”
