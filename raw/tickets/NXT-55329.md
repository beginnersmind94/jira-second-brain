# NXT-55329 — Elig - DC File Deletion - Move to Background Process

## Description
Running into timeouts while deleting large DC files, need to move to a background process.

*requirements*

* When the user initiates a delete:
** Inform the user this may take some time and that they will be notified when it is completed.
*** Text: File has been queued for deletion. For larger files, this may take some time. You will receive an Inbox message when the file has been removed.
** DC File needs to be put into a new status → ‘Deleting’
** User has a brief window to undo the delete and remove it from the queue
*** Message at the top of the DC File Details:
**** This file is in the queue to be deleted. [UNDO]
**** UNDO button should only be available for 30 seconds, then removed.
**** Add it to the background process queue at that point.
** User cannot interact with the DC File otherwise → disable other options
* Background process needs to handle the safe removal of the file and re-processing of eligibility
* Once complete:
** Inform the user with a Workspace Inbox Message:
*** Subject: DC File #12345 Deleted
*** Body: Direct Cert/Other Source Categorical File #12345 has been removed from the system. Eligibility records have been rolled back accordingly.
** Update the status of the DC File → ‘Deleted’

## Acceptance Criteria
* User is able to delete a DC file.
** User can still only delete the latest DC file.
** User cannot queue up another file for deletion.
** User has a brief window (30s) to undo the deletion.
** User can see that the DC file is in the queue.
** User cannot interact with the file aside from the Undo button.
* Background process handles the file deletion.
* User is informed when the file is removed.
