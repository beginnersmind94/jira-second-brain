# NXT-36787 — PR- Plan/Record- Add menu item Variety Menu item

## Description
When adding a variety menu item, there is no need to enter in a count for the user, Since they will be turning on the menu items on the plan or record page, they will enter the counts there.

## Acceptance Criteria
* Allow user to select the serving size
* Hide the ability to add Planned Counts and do not display the column to enter
* User can add item without adding counts since there is no counts for a variety menu item and it is only a place holder
** Change logic where the system does not expect any planned counts, currently if user does not enter counts, it is switching the plan or record to the previous status EX: plan complete goes to plan incomplete etc
** Currently, we are defaulting the “child” menu item in the variety to a count of zero if any counts are entered for the parent
** Currently API is sending the counts as 0 (This should not happen)
* Provide a way for the user to delete the whole variety menu item, currently once it is added it is added for good'
** If a “child” menu item is toggled on, disable the delete, they should only be able to delete if all are turned off (Toggle off all Menu items to delete)
* When user toggles variety menu item on, allow alignment for all columns to match normal menu items, and also resize text box to match normal menu items
* If user toggles a menu item on, enters counts and saves it, save the counts. However if they come back and turn the toggle off and save it, clear the counts in the background (Currently it is still storing the counts) 
* If the variety menu item is added on the record page, and the user toggles on some of the menu items. insert a 0 for those turned on toggled items on the plan page once they save the record
