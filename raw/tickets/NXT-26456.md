# NXT-26456 — Acc. - End of Day - Edit Checks Task

## Description
As Agatha the Site Manager I want to see if there are any edit checks that need to be resolved so I can quickly answer them.

*mockup*

[https://www.figma.com/proto/Yo99YhQm5Xk91Ie7b0PRkD/Front-Office-6.0?node-id=66%3A5775&scaling=scale-down-width&page-id=11%3A1711&starting-point-node-id=44%3A5320|https://www.figma.com/proto/Yo99YhQm5Xk91Ie7b0PRkD/Front-Office-6.0?node-id=66%3A5775&scaling=scale-down-width&page-id=11%3A1711&starting-point-node-id=44%3A5320|smart-link] 

*requirements*

* Add the Edit Checks task and populate it.
* Rename the task status ‘Not Done’ to ‘In Progress’
* Add a special permission for accessing the Settings button.

*Edit Checks Task*

* Task Name: Edit Checks
* Summary:
** Show the number of each serving exception.
** Examples:
*** 5 Edit Checks Found
* Details:
** Show how many have been resolved.
** Example:
*** 3 Edit Checks Resolved
* Status
** Status is based on them completing them all.
** 0% resolved = Not Started
** 100% resolved = Done
** Anything in between = In Progress
** If 0 exist, status = Done
* Actions
** Done / Not Done
** History

## Acceptance Criteria
* Not Done → In Progress
* Special permission for the Settings button added and programmed
* Edit Checks task created.
* This task is only shown if it is Active.
* Summary populates based on whether any edit checks exist for that site + date.
** No edit checks exist, text = No Edit Checks Found
** If edit checks exist, text = X Edit Checks Found
* Details populates how many have been resolved.
** If none exist, leave blank.
** X Edit Checks Resolved
* Status is based on how many have been resolved.
** Edit checks exist but 0 resolved, status = Not Started
** Edit checks exist but >0 resolved, status = In Progress
** Edit checks exist but all are resolved, status = Done
** 0 edit checks exist, status = Done
