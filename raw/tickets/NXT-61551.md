# NXT-61551 — IM - Item Onboarding- Validations for Associated Column fields (Inventory Units)

## Description
As a district manager that is working in the item onboarding tool, I do not know what can be and not be removed based on what the system is requiring. Provide proper validations and stopping points so that as a user I am not allowed to remove critical information.

## Acceptance Criteria
* If user does not include pack Size section in edit sessions, grey out the “# of Inventory Units” and select None/Decide Later
** Provide a header tool tip “Pack Size Info Section not selected”
* If user selects Non food item, the tool is asking users to enter “# of Inventory Units”, but there is no enforcement
** Provide a soft warning “orange Alert” for user to select “# of Inventory Units” but do not require, Tool tip: “# of Inventory Units not selected”
** On “Units and Sub-Qty” screen, users MUST make a selection for # of units if item is NON FOOD, provide a red alert, Tool Tip: “# of Inventory Units is Mandatory”
* On initial info screen, do not allow user to change the # of inventory units if any pack size units have been entered on the “Units and Sub-qty” screen
** This applies to both food and non food items
