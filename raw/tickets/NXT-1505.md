# NXT-1505 — Eligibility - Reports - Add Roster tab & grid to allow user to view classic "Eligibility Roster"

## Description
*Background*

Necessary for R5. If possible, we want to leverage grid columns similar to how Accounts page will be used to allow users to create reports by adding/removing columns and filters.

*Details*

* -In Eligibility, add a new page above Forms called- *-Roster:-*
* In Eligibility → Reports, add a new tab called *Roster*. It can be the last (far-right) tab.
* In this new Roster -page- tab, add the following filters:
** ‘As of Date’
*** Default value: today’s date
*** Must be a current or past date
*** The grid then returns the list of students with their eligibility status and reason *as of that date*.
** Site Code/Name 
*** Default value: ALL
*** Multi-select
** Grade
*** Default value: ALL
*** Multi-select
** Status
*** The student account’s status
*** Default value: ALL
*** Options: Active, Inactive, All
** Eligibility
*** Default value: ALL
*** Multi-select
** Basis
*** Default value: ALL
*** Multi-select
*** Nice to have (not mandatory): Have this cascade from Eligibility (i.e. The ‘DC SNAP’ basis only applies to Free students, so it should not appear if the user has chosen only the Paid eligibility option)
** Birth Date
** Buttons: Apply/Reset
* We need to load the default grid results upon page load, as the UX team needs this in order to eventually move the filters to a drawer.
** When the user clicks the Apply button, reload the grid with the resulting Students. 
** With the Reset button, allow them to reset the Filters.
* For the grid itself, add the following columns (hidden columns in _italics_, default in *bold*):
** _Site Code_
** *Site Name*
** *Student ID*
** *Student Name*
** *Eligibility*
** _Basis_
*** Add distinction/explanation for this and the ‘Reason’ via tooltip (may not be reflected in mock-up): ‘This may not be applicable for surveys’)
** *Grade*
** *Birth Date*
** *Start Date*
*** Start date of their current eligibility status
** *End Date*
*** End date of their current eligibility status
** _Status_
** *Application #*
** *Survey #  - (*We should only show this if the district is accepting surveys based on their setting value (“Yes” or “Mixed”))
*** For corresponding Eligibility/Basis, only show those that are not blank.
** _Work Phone_
** _Home Phone_
** _Cell Phone_
** _Email Address_
** _Benefits_
* Allow the user to Field Search through as many columns as possible.
* Allow the user to export the roster to Excel as well.
** File Name when exporting: *Eligibility_Roster_As_of_MM-DD-YYYY* (where date is the ‘As of Date’ selected by the user)
* There are no actions in this grid, but we may want to make an Action(s) column that contains buttons to view the Application # and Survey # at some point.

## Acceptance Criteria
Verify that the Roster tab has been added to the Eligibility module as part of Eligibility → Reports

Verify that the Roster tab has the set of filters shown above, with them operating as expected

Verify that when the user hits Apply, a grid populates with the default columns shown in *bold* above

Verify especially that the data is all based on the ‘As of’ filter date

Verify that the user can also show/hide the columns shown in _italics_ above

Verify that the user can field search through as many columns in the grid as possible

Verify that the list can be exported to Excel

Verify that the list has the naming convention defined above
