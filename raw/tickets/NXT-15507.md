# NXT-15507 — Verification - Tracking - Delete application from sample

## Description
*User Story*

As Vera the director I want to delete an application and select a different one instead to ensure a higher response rate and an easier workflow.

*Mockup*

[https://www.figma.com/proto/Y3tlwN2yvRc67304adzyyB/Eligibility-3.0?node-id=236%3A0&scaling=scale-down-width&page-id=1%3A254608|https://www.figma.com/proto/Y3tlwN2yvRc67304adzyyB/Eligibility-3.0?node-id=236%3A0&scaling=scale-down-width&page-id=1%3A254608|smart-link] 

*Requirements*

* Add an action for Incomplete applications - 
** Delete
*** Confirmation prompt + comment required.
**** ‘Are you sure you want to remove this application?’ CANCEL / REMOVE
**** Comment box, 500 characters.
**** Application should be updated to be Deselected with appropriate History entries.
*** Option to replace with another application.
**** Checkbox ‘Replace with another application?’
**** If selected, then add another application with the same basis (error-prone, categorical, etc.) if possible. If not, add another randomly selected application.
**** Deselected applications should not be selected as a replacement.
*** Inform the user on success, including the number of the replaced application if selected. 
**** Messaging: Application #12345 has been deselected for verification and been replaced by Application #4242.
**** Messaging if the application is not replaced: Application #12345 has been deselected for verification and has not been replaced. Please recalculate your sample(s) on the Sampling page if necessary.
**** This info message should hang around long enough for the user to notice it, if not dismiss it themselves. 
** 5% of sampled applications can be removed per the rules.
*** This should be controlled by the system setting: MAXRMVSAMP
** The delete option is always available for ‘Cause' applications.
** Replace option is only available for Sampled apps.
** If too many apps have been deleted, disable the delete button for Sampled apps and show the following tool-tip:
*** You can only delete 5% of sampled applications. If more applications need to be removed, please adjust the system setting: *Maximum Percent of Applications that can be Removed from Verification Sample*

!image-20210610-205546.png|width=654,height=502!

Success message if the option isn’t selected: Application #12345 has been deselected for verification and has not been replaced. Please recalculate your sample(s) on the Sampling page if necessary.

## Acceptance Criteria
User can delete an application and has the choice to add another application in its place.
Delete option is only available for the current academic year.

Only 5% of applications can be removed, to a minimum of 1 application. (The actual percentage will be controlled by the system setting MAXRMVSAMP.)

Look at the total number of applications pulled, not just a single verification sample in the case of rolling verification.
