# NXT-53516 — VND - AcctMgt - Addition of Security PIN

## Description
As Dana the Director I want to use a secure PIN for vending accounts and have these automatically generated. 

*Requirements*

* Add a new attribute to accounts:
** Security PIN
*** 4-6 numbers for length
*** We need a post-deployment script to generate a 4-digit Security PIN for every existing account.
*** Should be randomly generated on account creation as 4 digits:
**** Imports
**** Add Student/Adult
** If AMSECPIN is set to YES:
*** User can view/edit the Security PIN or generate a new random number in AcctMgt
**** Must be between 4 - 6 numbers in length
*** Available with the other ID #s
* Control this with a system setting:
** Name: Accounts - Use Security PIN
*** Code: AMSECPIN
*** Values: Yes/No (Default)
*** Description: This controls whether a separate Security PIN is available for Accounts. This can be used as part of Vending with the Vendnovation Interface and will be used elsewhere in the future.
*** Category: Account Management
*** Module: Account Management
*** Internal/Common: Common

Notes:

* We did not implement this in PrimeroEdge.
* VendNovation have provided a Postman collection to help with internal testing.

## Acceptance Criteria
* Security PIN attribute added to Accounts
* Every existing account gets a randomly generated 4-digit security pin
* System setting added
* If the system setting is set to Yes:
** User can see the Security PIN in Account Management
** User can edit the Security PIN in Account Management
