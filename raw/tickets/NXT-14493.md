# NXT-14493 — Balance Import

## Description
This Story is to cover the Balance @ Imports.

The user tires to Import Balance by navigating System--> Configuration-->Import Dashboard.

Once on the landing page, the user based will select the template “Balance Import“ under Import Type and proceed further.

Next step is to select if the Balance import is related to Adults or students (Based on the selection, appropriate table must be selected and updated). 

Secondly, the option to ignore Zero Balances must given (Again, Based on the selection, appropriate action must be done on the database). The end use must have the option to either Ignore or Not Ignore the Zero Balance.

Finally, option to ignore the previous balance import must be given (This means the user can select the latest or the existing balance).

After the appropriate options are selected, mapping is completed.

Once the import configuration is saved successfully similar to that of Student Import, The configuration must be accessable from landing page to edit if necessary.

Also, the transactions for this must be logged in Import Monitor and the details logs similar to that of Student import must be available.



{color:#ff5630} _Note : Overwrite previous balance import : Every transactions is a normal transaction, however every import is flagged separately in the database. So overwriting it will update previously flagged import._ {color}

{color:#ff5630}_Ignore Zero Balance : The context here in general is to update the amount in the database with the  amounts in the file. In case of Zero on the file, we leave the database entry as is._ {color}{color:#6554c0}*_The amounts in the database will be replaced by the ones on the file._*{color}
