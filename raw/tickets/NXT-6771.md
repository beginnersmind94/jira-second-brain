# NXT-6771 — System - Users Phase 2 - Add User  - Contact Info

## Description
As a District Admin, I would like to be able to easily grant PrimeroEdge access to a new user in my organization, so that they can quickly begin working in the system.

+Summary+:

Utilize the existing breadcrumbs (System → Configuration → Users → Add User button) and update the existing Add User page to use the new UI template *for the Contact Information section only*.

+Details+:

* When a user selects the Add User button in the Users page, redirect them to a new *Add User* page using the new template.
* Add a *Contact Info* section that displays the following fields:
** First Name (Required)
** Middle Name
** Last Name (Required)
** Email (Required)
*** Email address should be unique amongst all active or inactive users.
*** If possible, we should check to see whether the email is valid as well (belongs to that specific organization)
** Cell Phone
*** We should automatically display the field data to use the standard (XXX) XXX-XXXX format for the user
** Work Phone (Required)
*** We should automatically display the field data to use the standard (XXX) XXX-XXXX format for the user
** Active toggle
*** Affects the user status display on the Users screen. Can be handled when all sections are completed and the user data can be committed.
* Utilize the same validations as previously. Error messages should preferably display as the user tabs/exits a field rather than when clicking the Save button.
** {color:#ff5630}Email address must be unique{color}
** {color:#ff5630}Email address must be in a valid format {color}
** {color:#ff5630}Email address does not belong to this organization{color}
*** This case may need to be handled in a separate story
** {color:#ff5630}Cell Phone must be in a valid format{color}
** {color:#ff5630}Work Phone must be in a valid format{color}

Verify that when a user navigates to System -> Configuration -> Users and selects Add User, they are presented with the new design template for the Add User page.

Verify that the page contains a User Information section that displays the following fields:
First Name (Required)
Middle Name
Last Name (Required)
Email (Required)
Cell Phone
Work Phone (Required)

Verify that we automatically display phone field data to use the standard (XXX) XXX-XXXX format for the user

Verify the section utilizes the same validations as previously, with error messages displaying as the user tabs/exits a field rather than when clicking the Save button.

Verify email address must be unique
Verify email address must be in a valid format
Verify Cell Phone must be in a valid format
Verify Work Phone must be in a valid format

Verify that there is an Active toggle for modifying the user's status
