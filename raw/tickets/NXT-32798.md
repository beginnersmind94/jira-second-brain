# NXT-32798 — PR- Consolidated Plan- Non prepared item counts can not change/ allow links to all plans from page

## Description
As a Prep site, when I have my non prepared items showing on the consolidated plan. I should not be able to make changes to them. I also would like to have links to all the plans from the receiving sites that the prepared items are coming from.

## Acceptance Criteria
Display only counts for non prepared items with no text box (User has no way to change the counts for the planned quantity of non prepared items)

Display links on all the site name headers for the prep site as well as the receiving sites

Link should take user to the plan for that particular site, serving date, meal type, and menu line in a new tab

As previously implemented, if user makes changes to the plans and the consolidated plan is already saved, do not effect the consolidated plan and open plans in new tabs

* IF consolidated plan is not saved, user can only open one plan at a time in an overlay (prep site and receiving site plans)
** Do not allow calendar navigation in overlay
** Do not allow back button in overlay
** Do not allow menu hyperlink to be accessed
** Do not allow Navigation to other tabs
** User can enter counts, edit counts, add menu items and the save button is a save and close 
** If user clicks cancel, provide a warning for unsaved changes (same as current production plan implementation)
* Once user save and close, automatically refresh the consolidated plan to display the newest information
** If prep site happens to be changed, the data should be “transferred” to the new prep site and no longer active on the old prep site
