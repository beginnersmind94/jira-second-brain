# NXT-5173 — System - Configuration - Charge Limits - Allow user to set Charge Limits

## Description
As a manager or district admin, I would like to be able to set charge limits for students and adults in my district, so that patrons can be prevented from running up a large bill and not paying for it.

+Summary+:

Allow a user to set charge (negative balance) limits that apply to all students and adults of specific price types, school types, or grade groups.

+Details/Use Cases+:

* In the Configuration category, add a page titled *Charge Limits*.
** Basic functionality and text descriptions will be carried over from Classic.
* Allow the user to modify the charge limits of both patron types: Students and Adults.
** Allow a user to modify the Student charge limits
*** If the setting code PRICEMODE is set to RDA Group, display the Grade Groups of the district as the columns and allow the user to set the charge limits in each column for ‘Free’, ‘Reduced’, and ‘Paid’ students.
*** If the setting code PRICEMODE is set to Site Type, display the +*School Type*+ *Site Types* of the district as the columns (no other kinds of site types should display) and allow the user to set the charge limits in each column for ‘Free’, ‘Reduced’, and ‘Paid’ students (in that order).
** Allow a user to modify the Adult charge limits
*** PRICEMODE *does* *not* *apply* to adults.
*** As with students, display the School Type Site Types of the district as the columns and allow the user to set the charge limits in each column for ‘Staff’, ‘Visitor’, and ‘Prog Adult’ adults (in that order).
** Columns should display in the order of GradeIds contained in the group (may need a display order).
** Values for any charge limit (student or adult) must be between $0.01 and $9,999.99. Otherwise, we should display an error. 
** Saving should happen in-line, with a success message displaying when above criteria are met.
** Updates should happen in the database immediately. There is no delete functionality on this page.
