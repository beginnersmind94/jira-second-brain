# NXT-44913 — SIS Auto Sync – Custom – API DB and PreFetch Data

## Description
JAS 01/12/24 - We are not going to do this, but keeping story in backlog for now in case want it later

-We need to create a backend database table structure to house the list of source APIs and its meta data plus a collection of sample data so that the SIS Import setup process will go faster.-

*-Notes:-*

* -This should be done before doing the new API tab in the Add New overlay-
* -That tab will need to call these tables- 
* -The objective of this story is to overcome the delays expected in the API tab when a user wants to preview the columns or the data. By pre-fetching and storing the data locally, we can eliminate the delay.-

*-Requirements-*-:-

* -Store in the database the following about SIS Provider API calls-
* -APIs-
** -Info we need such as:-
** -District-
** -Provider (Aeries)-
** -endpoint name-
** -Endpoint Alias?-
** -IsPII (true/false if has PII data)-
** -We will show the name or Alias to the end user-
* -For each API endpoint in that table, have a API fields table that shows:-
** -columns available for that endpoint-
** -we will show this columns in the API tab previews and in Field Source table in Field Mapping-
** -there is a many to one relationship of this table to that one-
* 
* -The data in the APIs table, we will do manually and selectively-
* 
* -The data in the API Fields table will need to be updated programmatically-
** -We can set up a scheduled job to periodically call each API and get a list of endpoints- 
** -Or we can asynchronously do the same when a user selects the API in the API tab-
* 
* -We need a database table with sample data for each API endpoint- 
** -Needs to be restricted to district-
** -table will have district, api endpoint, column and field value and whatever other data you need-
** -the objective of this table is to have sample data to show in the preview tab so that we don’t have to make a full call to the API endpoint and retrieve data in real time-
** -This data can be retrieved at the same time the API endpoint is updated-
** -Or it can be triggered and done asynchronously by the API tab when the use selects the API-

## Acceptance Criteria
* Data is stored locally so that there is no delay in previewing in the API tab
