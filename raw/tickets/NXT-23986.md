# NXT-23986 — Framework - Advanced Search/Filters (R6 Integration)

## Description
As a product I want to provide users with the ability to filter/search information a consistent, organized, and elegant way so that users know what to expect as the interact with filterable data grids throughout the application.

Value is that we reduce the noise on the page with so many filters. This puts the data representation front and center. The user will be able to filter the information, select additional columns, and eventually select preset Views to load information.

We should provide properties on the control to allow for pre-determined/selected filters that should be applied on the page load. This will provide the user with their typical, expected results. If there are no results that should be preloaded, we will have a property to automatically display the filter panel so that the first action is getting user input for loading information. Wherever possible, let’s try to load information into the grid data sets to start.

Order of priority and flow of the filter actions is very important and should be part of the user story. Top to bottom flow should be given for the team to implement.
