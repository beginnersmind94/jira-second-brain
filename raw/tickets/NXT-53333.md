# NXT-53333 — VND - Additional Settings - Max vending transactions, Serve Meal By, Allow 2nd Meals, Show Actual Price

## Description
As Dana the Director I want to allow students to authenticate themselves at the vending machine with different methods to fit our district’s needs.

*Requirements*

* Add a new System Setting:
** Setting Name: Maximum vending transactions per account per day
** Setting Code: MAXVEND
** Module: Accountability
** Category: Vending
** Setting Description: Controls how many transactions each account can have per day via vending machines.
** Values: Text field, we expect numbers only. Default: 9999
* Setting MAXVEND needs to be used as part of our response to the /selection endpoint.
** Check to see how many vending transactions the account has had today.
** If it is > MAXVEND value - “Approved” sent back is false.

----

* Add a new System Setting:
** Setting Name: Serve Meal By
** Setting Code: SERVEMLBY
** Module: Accountability
** Category: Vending
** Setting Description: Controls whether meals at vending machines are served based on Meal Timings or by the Meal Type. This impacts pricing at the machines.
** Values: Meal Timing, Meal Type
* Setting SERVEMLBY needs to be used as part of our response to the /customer endpoint.
** Meal Timing → “UserCredentials” checks against the Student ID #. “PINNumber” sent back is the PIN.
** Meal Type → “UserCredentials” checks against the Student ID #. “PINNumber” sent back is the DOB.

----



* Add a new System Setting:
** Setting Name: Allow 2nd Meals
** Setting Code: VNASEMEAL
** Module: Accountability
** Category: Vending
** Setting Description: Controls whether 2nd Meals are allowed to be sold to students via vending machines.
** Values: Yes, No
* Setting VNASEMEAL needs to be used as part of our response to the /selection endpoint.
** Yes→ 
*** Check to see if this is a 2nd meal. 
*** Check to see if 2nd meal is allowed to be served.
*** Check all account restrictions per normal on this.
*** If everything is fine the “NewPrice” sent back is based on the second meal pricing.
** No → 
*** Check to see if this is a 2nd meal.
*** If it is, “Approved” sent back is false.
* The setting also has to impact how we are recording the transactions.
** If the setting is set to yes, we need to make sure we’re checking to see if it’s a second meal, and recording the transaction appropriately with 2nd meal pricing.



----

* Add a new System Setting:
** Setting Name: Show Actual Price
** Setting Code: SHOWVNACP
** Module: Accountability
** Category: Vending
** Setting Description: Controls what price is shown at the vending machine. Set to Yes to show the price based on the account pricing and meal type.
** Values: Yes, No
* Setting SHOWVNACP needs to be used as part of our response to the /selection endpoint.
** Yes→ “NewPrice” sent back is based on the account’s eligibility and meal type.
** No → “NewPrice” sent back is the same as the “Price” they sent to us.
