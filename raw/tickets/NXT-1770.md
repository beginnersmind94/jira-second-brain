# NXT-1770 — INV - Settings - Site Par Levels - View Par Levels

## Description
As a site ordering manager I want the ability to configure par levels for certain items so that the system can provide recommended order quantities to stay at the par levels defined.

This page will be used for sites to define their “par level” for items. Par levels are quantities of items that a site or warehouse would ideally like to always have on-hand. 

For example, a school may want to always have 500 cups on hand. If they configure their Par Level as 500 cups, when they use some of those cups and they now have 200, the system can alert them to order 300 more as soon as possible.

*Requirements*

* Add a new page under the Settings category, titled ‘Site Par Levels’
* Provide a header containing a Site dropdown
** Mandatory selection
** Single selection (only one site can be selected)
** ‘Select’ is default
* Within the same header, separated by a vertical separator, display dropdowns to help filter items
** Storage Category 
** Item Category
** Vendor 
** *All of these are optional and will have ‘All’ as the default selection
*** Single selection as well
** Apply button to apply the selections to the item list
* In a grid below, provide an list grid that contains the items
** this should populate as soon as a Site is selected
** this will be the non-editable state of the item list
** this would show the list of items for which some Par Level has been saved
** use pagination for this grid
** provide the Field Search button
* Edit button ([NXT-17773|https://primeroedgenext.atlassian.net/browse/NXT-17773])
** Somewhere above or within the grid, provide an Edit button which would change the page to edit mode
* Display an informational banner above the grid, in line with the Edit button
** “Currently viewing par levels for \[Site]” …… TBD
* In the item list grid, provide the following columns
** Item #
** Item Description
** Storage Category
** Item Category
** Current Vendor
*** this will display the vendor to which the item is currently configured for ordering for that site
** Par Level
*** this will display the currently saved par level value for the item
*** this should also show a label for the Whole Unit description
* If there are no par levels currently configured for any items, display a label
** “There are no par levels configured for the selected site.”



Note on mockup: do not require Item Category groupings
