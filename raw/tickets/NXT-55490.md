# NXT-55490 — Family Hub - Super Admin - Configure Districts - Add 'Documents' section

## Description
As a Cybersoft or district admin, I would like to have the ability to securely house documentation within the Family portal website, so that I do not have to transfer financial documents through less private forums such as Freshdesk.

*Summary*

# Add an ‘Upload Document' feature to the /agreements page while logged-in as a district admin
# Add a ‘Documents’ section which houses the online agreement (if uploaded manually) as well as any other documents uploaded by the user.

*Details*

# In the Agreements page (accessed by logging in as a district admin and replacing {color:#6554c0}_/dashboard_{color} with {color:#6554c0}_/agreement_{color} in the URL), add a new button: *Upload Documents*
## When clicked, open file explorer and allow user to upload a document
### Existing, or reasonable restrictions (file must be Excel, Word, PDF, no SQL files, etc.) can apply if needed.
## Once a file has been uploaded, add a button to save the file (or just save it automatically) and a success message: “File uploaded successfully! A member of our team will reach out to you if any additional information is required.”
## Send an email alert to the email address(es) in the setting _Agreement Submission - Cybersoft Internal Contacts._
### To save development time, we can reuse the existing Agreements notification template by replacing all instances of “An Agreement…” with “A Document…” to make it a more generic notification.
## -Optional:-
### -Allow users to delete uploaded files-
### -Exception for the above point: Do not maintain the user’s uploaded files/history in this page, and do not show other users' uploaded files either. In other words, a user always sees the same empty page until they upload a file, and everything is cleared again once they logout. The user should only be able to delete files that they have uploaded in the current session, in case they made a mistake. Once they logout though, the only way to delete the file will be upon request.-
## This button should only appear on the page that displays when an agreement has already been uploaded. If the district still needs to submit their agreement, continue showing the Agreement form when the user navigates to {color:#6554c0}_/agreement_{color}, without any changes.

## On the Super Admin side, in _Configuration → Configure Districts_, display a new section: *Documents*
## In parentheses, display the number of uploaded documents, including the Agreement if it has been submitted or uploaded by the user, next to the title. i.e. +*Documents (2)*+ as a clickable link.
## When clicked, open a pop-up table similar to the existing View Agreement History hyperlink. It should contain the following:
### Columns: *_File Name | Uploaded By | Uploaded On | Action(s)_*
#### Filename: Name of the file uploaded
#### Uploaded by: firstName + lastName of the user who uploaded the file on the Agreements page.
#### Uploaded on: Date that the file was uploaded, in MM/DD/YYYY 00:00 AM/PM format
#### Action(s): Initially, just show a View icon. (Delete may come later)
##### Allow the user to view the file within a viewer, rather than force them to download it (like we do with Agreements). If a viewer is too complicated with our structure, allow users to view PDF in the viewer and download all other file types.
### *Upload* button
#### Display the same kind of upload button that the district admin sees. When the file has been uploaded, display the logged-in Super Admin user’s information in the table above.
##### Success message can just say, “File uploaded successfully!”
#### -Optional:-
##### -Allow users to delete uploaded files-

## Acceptance Criteria
Verify that district admins, who belong to districts where an agreement has been submitted, can go to the /agreement page and upload documents

Internal contacts receive an email about a new document being uploaded (very similar to existing functionality for new agreement submission)

Verify that super admins see the new Documents section on the Configure Districts page

Verify that the section allows super admins to view and upload documents
