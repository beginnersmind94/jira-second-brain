## Acceptance Criteria
* Redesign the Contributions slideout with better UX and using the Component icons and colors from Menu Planning
* Provide just 5 main rows, one for each food component
** unapplied components should have the icon at 50% opacity
** switch to 100% opacity when the component is enabled
* Use the icons and colors from Menu Planning as headings for the rows
* Display the Cups or Oz. Eq. label and icon after each Contribution dropdown
** [Material Icon link for scale icon|https://fonts.google.com/icons?selected=Material+Symbols+Outlined:scale:FILL@1;wght@400;GRAD@0;opsz@24&icon.query=scale&icon.size=24&icon.color=%235f6368] to use for Oz. Eq.
* make the Contribution value dropdown just big enough to support the range of values 
* Display the text and icons for unused values in gray
** Switch the text and icons to black once a value is input (see mockup examples)
* Organize all the “variety” type options into a single Variety column
** Fruits: Fruit Juice checkbox
** MMA: nothing
** Grains: Grain Based Dessert checkbox
*** if checked, enable the GBD dropdown below
** Milk: Variety of Milk dropdown
** Vegetables: 
*** Vegetable Juice checkbox
*** vegetable type dropdown
* By default, display just one row of values for Vegetables, but provide an + (add) button
** the add button will add another row below the default one, where an additional veg type can be selected
* Ensure the ‘Any Subgroups’ option for VMI works in the new design
** For the vegetable component, add an option to the top of the subgroup dropdown
*** “Any Subgroups”
** if ‘Any Subgroups’ is selected, disable the +Add button to add additional rows
*** tooltip on disabled icon button: "Multiple subgroups not allowed when 'Any Subgroups' option is selected.
** When this is selected, modify the Contribution matching logic to consider any vegetable subgroup as a valid match for child items
*** e.g. if VMI is “1 Vegetable, any subgroup,” then a child item with 1/4 Red/Orange and 3/4 Starchy is a valid match
** *Hide* the 'Any Subgroups' option from the drop-down if any other specific subgroup is first selected. 
* Ensure the Recommendation logic and behavior is carried into the new design
** for Menu Items, the business rules can be a little different: [https://primeroedgenext.atlassian.net/browse/NXT-41959|https://primeroedgenext.atlassian.net/browse/NXT-41959|smart-link] 
** the Recommendation shows in a column to the right side, and displays a simple text value containing the recommended value for each relevant component
*** in the new design this may require making dropdown selections or adding records (for Vegetables) on behalf of the user
** the Apply Recommendation button applies the recommended values to the editable cells
