# NXT-48049 — System - Global Managed Views 

## Description
As an Item Management user I want the Manage Views tool to be defined by the module itself rather than simply allowing custom user views, so that the system can provide very tailored, useful pre-loaded views out of the box.

*Notes:*

* We have a Manage Views control for grids already
* Currently, a site user can set a view with the columns they want, etc. and save it and later come and click on it and have the view applied to the grid
* In this story, we want to be able to setup a managed view in the backend (No UI required to create/edit/save the view) that will be saved in the db, but not be tied to a specific user, but be available to all users in all sites, hence it is a global view for a grid
* The user will then be able to click on a global managed view and use it the same ways as one of their own saved managed views

*Requirements:*

* Backend:
* Create a table to store global managed views
** It should have all the columns as the user saved managed views except the user and district info
** Ensure it is tied to a grid
* Write APIs that can be called by other modules so that they can get their managed view settings by grid 
* 
* Document what other modules will need to do to use this feature 
* 
* Front End
** The front end usage of this feature will be covered in another story

*Technical Details*

* FrontEnd
** Add New Check Box Save As GlobalView (Permission to Custom Support Role Only)
** When Save As GlobalView is Selected, Disable other 2 checkboxes 
** In Managed Views Screen, Move Edit button to ellipse
** Replace the first column (current a Tick mark is showing). Instead show it the Letter as displayed in below screenshot or figma
** Remove the restrictions for sharing (currently the view that is shared can’t be shared again). Read the description from screenshot
** Enable Delete button restrictions - Read the description from screenshot. (In addition to above requirements, If the logged in user is haing *Delete Shared Role Views* enabled, he would be able to delete Role based share View. Show a dialog says “*Deleting a Shared Role View will affect other users*”
** Currently Only Usershared view is displayed, need to display role shared view as well. The data is already fetching. Even the view is duplicated that’s fine show it by type (1 for user & 1 for role (even if the view is in multiple roles and all those roles assigned to user)). 
* Backend
** Add a special Permission Under system Module “*Delete Shared Role Views*”
** In Create/Update endpoints, check if IsSytemView is true, then modify regionid & userid as -10
** Modify Get endpoints to include regionid & userid -10 value along with user specific views
** Verify Conditions for sharing if userid is already in shared list, then don’t insert again
** Modify Get endpoints to include roleid as well in the where conditions
** When sharing see the unique conditions and apply the distinct values as necessary.
