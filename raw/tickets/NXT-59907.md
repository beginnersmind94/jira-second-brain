# NXT-59907 — TH - Sec - SC Authentication

## Description
*As a* TeamHub user *I want* to securely authenticate my login credentials (username and password) in SchoolCafé for my District *So that* I can access the correct account and district-specific data for out district

h3. Comments:

We need to develop a secure API endpoint on our server that our mobile app’s login screen will use to authenticate users. This endpoint must not only validate the username and password but also ensure that the provided Platform (e.g., SchoolCafé or PrimeroEdge) and District are valid. This ensures that the user is granted access only to the correct environment and data set. The API must adhere to industry-standard security practices including encryption, rate limiting, and secure logging.

h3. Requirements:

* *API Endpoint Setup*
** Create an API endpoint
** The endpoint is accessible via a POST request (e.g., {{/api/auth/login}}).
** All communications must be secured using HTTPS.
* *Request Payload*
** The API accepts the following fields in the request body:
*** {{username}} (string, required)
*** {{password}} (string, required)
*** {{platform}} (string, required; valid values: "SchoolCafé" or "PrimeroEdge")
**** use this to make trying to log in to correct platform
*** {{district}} (string or identifier, required)
*** Any other fields needed by the API endpoint
** The API returns an HTTP 400 error if any required field is missing.
* *Validation & Processing*
** Validate that the provided District exists on that platform
** Validate that the district is in active status
** Encrypt and validate the provided password against the stored (encrypted) credentials.
** If the platform/district combination/status is invalid, return an appropriate error message with HTTP status code 401 or 403.
** Implement rate limiting to prevent brute force attacks.
** Log each authentication attempt (both successful and failed) securely, ensuring no sensitive data (like raw passwords) is logged.
* *Successful Authentication*
** On valid credentials, the API returns an HTTP 200 response including:
*** A JSON Web Token (JWT) for session authentication.
*** User profile details and the active district data.
*** Licenses
*** Roles and Permissions
** The JWT must be signed securely and include an expiration time.
* *Failed Authentication*
** If the username, password, platform, or district is invalid:
*** Return an HTTP 401 Unauthorized response.
*** Provide a generic error message that does not expose sensitive details.
* *Non-Functional Requirements*
** The API response time should be under 2 seconds under normal operating conditions.
** Ensure scalability to handle multiple simultaneous login requests.
** Maintain comprehensive API documentation for mobile app integration.
* *Security*
** Ensure data is properly encrypted
** The endpoint will be used exclusively by our mobile app; ensure the necessary token authorization/handshake is implemented so that only our app can access it.
