# NXT-3698 — Add Item - Duplicate Check Functionality

## Description
*User Story*

As a Item Management User, when creating an Item, I want to check the duplicates already in the system

*Mockup*

[https://nqyc9v.axshare.com/start.html#p=itemmanagement_-_super_user&id=n0hwa7&g=1|https://nqyc9v.axshare.com/start.html#p=itemmanagement_-_super_user&id=n0hwa7&g=1]

*Requirements*

* After clicking ‘Add New’ and ‘Add item manually’ display
* Item Name as a text field
* Short Name - Optional field
* “Add Item as”
** Ingredient - 'Default' value
** Product
* After typing in the Item Name and selecting the type, click continue, Short Name is optional
* Datasource
** Shows all datasources the user has access to
** Default - Local
* Check if there are any other items that have the same Item Name
** If not, go to the next screen
** If yes, then display a message and the results of the duplicates
*** “We found “x” similar items based on the item name. If you want to edit an item below, please click on the item name you want to edit. Or click ‘Save & Continue’ to create as new”
*** Display the columns:
**** Item Name (show 10-15 characters, then show a triple dot) - If you hover over, then it will display the full name via tooltip
**** Item Short Name (if applicable)
**** Item Code
**** Data Source
** Click continue to go to the ‘creation screen’
** If you click on an Item Name, go to the view screen for that item
** !image-20200511-140225.png|width=1050,height=591!

## Acceptance Criteria
* After typing in Item Name, check for duplicates from all datasources this instance has access to. Check to see for exact duplicates. Partial duplicates would show as lesser priority 
* If an exact match and local datasource, user cannot continue - Display message (An item with the same name already exists. Please change the name or view the item that is already available)
* If an exact match and a shared or global datasource, the user can continue
* If it's a partial match, the user can continue
