## Description
GC - PE - Global Catalog - Pending Updates - Update Items button functionality from Review Items popup

scope of this story : 

while working on this story, i noticed, UI is already completed, mainly this story needs backend logic to update the items, track the history for changes  and create the GC link if its not already exists., 

# clicking on review items button 
## asks for confirmation 
### clicking on ok will proceed with the update items action
### clicking on cancel will cancel the action. 
## clicking on ok button will close the popup  - all selected updates are applied to selected items
### if user changes any selection while reviewing the individual items and mark as reviewed then those changes should not be applied with bulk update.
### status changed to “completed” for selected items
### displays a green success message to the user on main page(popup closes and success message displayed on the top of the main page)
# updates can be seen on ingredient details page from menu planning
# history tab of the ingredients shows the history records for each tab and for all accepted changes. 
# in one of those discussions , it came up that saving the data in a table like this  helps us to quickly pull the items for updates in the grid (2nd time onwards,.)
