# NXT-48056 — Roles & Permissions - New Screen - Edit Role Enhancements

## Description
We need to add a couple of enhancements for role editing that came up during testing

*Requirements*:

* On System > Roles & Permissions:
* When click Edit on a role row:
* On the general permissions and special permissions tabs, do the following:
* Right now, if they browse away from the tab, we lose changes made
* We want to not lose changes, so we either need to force them to save before browsing to another tab or enforce data persistence on both tabs
* Either way is fine, but probably the best thing to do is if they make changes on the tab and click one of the other tabs, to ask them if they want to save changes first
* 
* On the Users tab, we need to show Status as a column since it has been brought up that there are valid scenarios in which a user can be Inactive status but they still want to add him to a new role
** So we need to let the user know if they have inactive users
** Also, it will help with role maintenance as they can see the status and decide if they want to delete the user from the role
* Add Status as a column
** Values: any valid status
* 
* Add a back button to the top that allows the user to go back without having to hit the Cancel button

## Acceptance Criteria
* Users don’t lose changes when browsing between tabs
* Users can see User status on Users tab
