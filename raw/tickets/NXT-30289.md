# NXT-30289 — PR- System- Plan and Record- Allow changes to X days in the past

## Description
As a Food Service Director, I would like my managers to only be able to correct/finish production plans or records for a certain number of days in the past

Currently, the system allows a user to edit or change production plans and records whenever a user would like

## Acceptance Criteria
Use the setting placeholder currently in the system> settings >production> plan and record to control this setting

*Plan/Record - Allow Changes X Days in the Past*

This setting will affect all production plans and records and only allow users to change information or data on those plans and records for as many days in the past as it is set to

The input for this setting is numerical, default to 100 days

Based on default of 100, user will be able to make edits/changes for 100 days in the past

User has the ability to *view* the plans but can not make edits or save any new information (if more than 100 days in past)
