# NXT-58290 — LMA - Device Management Page

## Description
As a Customer Support role member, I would like to ensure we don’t add more vending devices than what are licensed so that we can charge for any extras.

*Flow:*

* User logs in and browses to the Device Management (DM) page
* User sees a list of devices and sees the license status for each one
* User optionally adds a new device and optionally selects a license
* User optionally edits an existing device and optionally applies a license or removes one

*Requirements:*

* These requirements apply to System > Device Management screen

*Device Management page*

* On the main grid on the DM page, add the following:
* Add column to grid:
** Name: License
** Values:
*** Green = linked to a license whose license key status is New or Active
*** Yellow = Not linked to a license 
*** Red = linked to a license whose license key status is Cancelled or Expired
** Location: After Status

*+ New Drawer*

* On the + New drawer, do the following:
* Add a License selector to the bottom of the page
** Label: License
** Single Select drop down list
** Optional
** Data: list of all licenses that are of type Device whose parent license key has a status of New or Active
*** Filter out district and site licenses
*** Filter out cancelled or expired license keys
** Value formatting
*** format the values like this: [device type] - [license key] - [count of available] (see bolt code)
** Inform user if there is no license available to select from (see bolt code)
** Do not allow the user to select a license that has 0 available 
* Update Save button
** Add logic to link the device to the license if selected
** Ensure the main grid is updated with the change
** Ensure license linking is added to History log



*Edit Drawer*

* Add the same License selector to this drawer as well with the same rules
* Additionally, ensure the user can deselect a license
* Update Save button
** Add logic to link the device to the license if selected a new one
** Add logic to delink if they removed it
** Ensure the main grid is updated with the change
** If delinked, make sure the available count for the license is updated accordingly
** Ensure license linking is added to History log



*Renew Update*

* After doing this, you should have the code you need to link devices to licenses
* We need to update the Renew button on the License Management page
** When user clicks Save automatically link the devices from the old license to the new license
** You don’t have to unlink them from the old license
* Likewise, we need to do the same for Bulk Renew

## Acceptance Criteria
h3. *1. Device Management Grid*

* *Add column*:
** *Name*: {{License}}
** *Location*: After {{Status}} column
** *Values / Color Coding*:
*** *Green* → Linked to a license whose *License Key Status* is {{New}} or {{Active}}
*** *Yellow* → *Not linked* to a license
*** *Red* → Linked to a license whose *License Key Status* is {{Cancelled}} or {{Expired}}

----

h3. *2. + New Drawer*

* *Add a License selector* at the bottom:
** *Label*: {{License}}
** *Type*: Single-select dropdown
** *Optional field*
* *Dropdown Data Source*:
** Only *Device licenses*
** Parent *License Key Status*: {{New}} or {{Active}}
** Exclude *District* and *Site* licenses
** Exclude *Cancelled* or *Expired* license keys
* *Value Formatting*: {{[device type] - [license key] - [count of available]}}
* *Behavior*:
** Inform user if *no license is available* (see Bolt Code)
** Disable selection of licenses with *0 available*
* *Save Button Logic*:
** If a license is selected → *link device to license*
** Update *main grid* to reflect the linked license
** *Log license linking in History table*

----

h3. *3. Edit Drawer*

* *Add same License selector* with same rules as +New drawer
* *Additional behavior*:
** Allow *deselection* (delink license)
* *Save Button Logic*:
** If a *new license* is selected → *link device to new license*
** If *license removed* → *delink device from license*
** Update *main grid* to reflect changes
** If delinked → *update available count* for that license
** *Log license linking/delinking in History table*

----

h3. *4. Renew / Bulk Renew Integration*

* After device-to-license linking is implemented:
** *Renew Button (License Management page)*:
*** When user clicks *Save* → automatically *link the devices from the old license to the new license*
*** Do *not* unlink from the old license
** *Bulk Renew*: Same behavior as Renew → automatically *link devices from old license to new license*
* *Purpose*: Ensure continuity of linked devices without manual relinking

----

h3. *5. Business Rules / Notes*

* Only *Device licenses* are selectable
* Cannot select a license with *0 availability*
* Linking/delinking must *update immediately* in the grid and license counts
* All linking/delinking actions are *logged in the History table*
* Renew / Bulk Renew automatically preserves device links
