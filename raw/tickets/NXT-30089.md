*requirements*

* Add / program the Done action for each task.
** This toggles the button between a Done and Not Done state. 
** This is not the same as the Status. This is the user ‘signing off’ that they completed the task.
* This can be done regardless of the Status of the task.
* User can roll the Done button back to a state of not being done.
* Add / program the History action for each task.
* When clicked, load a drawer containing the following details for that task:
** Task Name
** Done / Not Done state. 
** Who
** When

## Acceptance Criteria
‘Done’ button programmed

Can toggle between Done and Not Done

* This is not the same as the task status, it’s the user signing off on the task.

History button added, loads a drawer showing the … history: 

* Task Name
* Done / Not Done state
* Who pressed the button
* When did they press the button
