# NXT-54309 — SC - GDSN Connect - Exception Handling Approach - Enabler

## Description
_(none)_

## Acceptance Criteria
*Manu Navigation:*

IM - Items - Upload Icon

The system is throwing parsing failed error when the user tries to upload the samples.



*Change*: The system should assess and update the logs on the failed records with appropriate exception on following cases: 
1. Data Length (Ex: 9,3), (Ex: Protein - Invalid Data Type) 
2. Wrong Data Types or conversion Error ((Ex: Protein - Error during data Parsing) 
3. Duplicate Records with in the file (more than 1 record with same naming convention) (Ex: Item Description - Duplicate entry”

  

*Approach:*

Change the stored procedure to verify each row to meet the data quality standard 

If the row is qualified, proceed with next row 

If the row is not qualified - record a remark as “Field Name - Description of Remark” 

Exit the procedure only when all the rows are processed through data validation
