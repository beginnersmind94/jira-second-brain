# NXT-9297 — Data Exchange - SIS Import - Update a student account

## Description
As a district I want to update student account information fields/attributes if any information sent from the student information file is different than current values.

More Detail:

* Logic should compare the current value of each field/attribute of the student record with the provided information on the file. If different, it should update the information with the values provided.
* If a value is provided as blank or null input and it currently contains information, the field should be updated to empty on the student record. This is the safest and best data integrity to ensure matching between what the student information has sent.
* Common changes to a student account are updates to the contact details (address, phone, email)
* Less common changes to a student account are updates to the enrollment details (school, grade) which usually occurs during yearly changes to the enrollment
* Rare changes to a student are for name, birth date, gender, etc as this would indicate a potential error in data entry that was originally provided

## Acceptance Criteria
1. Values in the provided import file that are different from the current value should be updated on the student account.
2. Empty values provided that are not required fields should be updated as blank/empty values.
3. Empty values provided that are required fields should be ignored as it will impact the data integrity of the student record.
