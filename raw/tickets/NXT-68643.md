# NXT-68643 — Fin - Revenue and Expenditure (P&L) Report – Parameters, Multi-Site Exports 

## Description
As a *District Finance User*, I need to generate a Revenue and Expenditure (P&L) report for a specific fiscal year and reporting period, for one or many sites, and receive an Excel workbook where each selected site has its own P&L worksheet.

## Acceptance Criteria
h3. 1. Page Header & Helper Text

* Page title: *“Revenue and Expenditure (P&L) Report”*
* Helper text directly under title:
* “Choose a fiscal year, reporting period, and sites. When multiple sites are selected, each site gets its own P&L worksheet in the Excel file.”
* Tooltip “You’ll be notified once your file is ready and can download it from the Document Center.”

----

h3. 2. Parameter Panel (top card)

h4. 2.1 Fiscal year

* Label: *“Fiscal year”*
* Control: dropdown.
* Options: fiscal years formatted like {{2024–2025}}, {{2023–2024}}, etc.
* Default to current fiscal year.
* Changing fiscal year updates the available period values (months/quarters) but does not reset sites or file format.

h4. 2.2 Reporting period

The reporting period area consists of *two controls* in the second column:

* Left dropdown (type): label *“Reporting period”*
** Options:
*** {{Single month}}
*** {{Quarter}}
*** {{Fiscal YTD}}
*** {{Custom date range}}
* Right control (value), behavior depends on type:
** *Single month* (default)
*** Value dropdown lists months ({{July}}, {{August}}, …, {{June}}) as interpreted within the selected fiscal year.
*** Period display in history: e.g., {{May 2025}}.
** *Quarter*
*** Value dropdown: {{Q1}}, {{Q2}}, {{Q3}}, {{Q4}}.
*** Period display in history: e.g., {{Q2 2025}}.
** *Fiscal YTD*
*** No separate value dropdown; UI may show disabled text like {{FY 2025 YTD}}.
*** Period display in history: {{FY 2025 YTD}}.
** *Custom date range*
*** Replace value dropdown with two date pickers: *From* and *To*.
*** Period display in history: {{MM/DD/YYYY – MM/DD/YYYY}}.
* Reporting period selection is required.

h4. 2.3 Sites/Locations selector

* Label: *“Sites/Locations”*.
* Control: multi-select dropdown with search and “select all” behavior.
* When opened, the panel contains:
** A *search box* at the top.
** A “Select all (N sites)” row.
** List of sites, each with a checkbox and site name.
** This list will vary based on the user’s permissions

*Behavior*

* *Search*
** Filters the visible site list as user types, case-insensitive.
* *Select all (filtered)*
** With no search term, “Select all (X sites)” selects all sites in the district.
** With a search term, “Select all (N sites)” selects all currently filtered sites only.
* *Closed-state summary text*
** No sites selected → {{No sites selected}}.
** All sites selected → {{All sites selected (X sites)}}.
** One site selected → show the site name, e.g., {{Lincoln HS}}.
** More than one site selected → {{Multiple (N sites)}}.

h4. 2.5 Download

* On clicking the export button, it will create a Power BI report from where the user can generate a PDF.

----

h3. 3. Buttons and Behavior

Buttons appear left-to-right: *Reset | Apply | Generate Report*

h4. 3.1 Reset



* Resets parameters to defaults:
** Fiscal year = current fiscal year.
** Reporting period = {{Single month}} + current fiscal month.
** Sites/Locations = all sites selected.
** File format = {{PDF}}.

*3.2 Generate Report*

* Triggers creation of an Excel workbook with the following rules:
** *One workbook per Generate action.*
** If a single site is selected:
*** Workbook contains one P&L worksheet named with that site (e.g., {{Lincoln HS}}).
** If multiple sites are selected:
*** Workbook contains *one worksheet per site*, each named with the site’s name.
*** If ‘All’ is selected, consolidate data from all sites within the district in one report.
*** If a line in the report doesn’t have any data, indicate so using a ‘-‘ and use zero in the backend for calculations

----

h3. 5. Consistency With GL and Journal

* Layout of the parameter card, warning line, button row, and Recent Reports table must align visually with the *General Ledger Report* and *Journal Report* pages.

For the backend side of the report:

* user selects a reporting period (example: january 1 to january 31)
* create a revenue table
* revenue table must include one row for each gl account that meets both of these conditions:
** account type = revenue
** account appears in at least one manual entry line (debit or credit) where the posting date is inside the selected reporting period
* row label must use the gl account name exactly as stored in the gl accounts table
* for each revenue gl account row, system must calculate:
** description = gl account name
** current period = (sum of credits in period) – (sum of debits in period)
** prior period = (sum of credits in prior period) – (sum of debits in prior period)
** year-to-date = cumulative (credits – debits) from fiscal year start up to the end date of the selected reporting period
** *% of Revenue:* 
*** *if total revenue > 0 → (current period ÷ total revenue) × 100, display as "XX.XX%"*
*** *if total revenue ≤ 0 → display "-" and show warning banner above report: "Total revenue is zero or negative for this period. Percentage calculations have been suppressed."*
** % change from prior period =
*** if prior period ≠ 0 → (current – prior) ÷ prior
*** if prior period = 0 → -
* total revenue = sum of all current period revenue values across all revenue gl accounts
* these exact revenue values (each row + totals) must be used by the dashboard
* create an expenses table
* expenses table must include one row for each gl account that meets both of these conditions:
** account type = expense
** account appears in at least one manual entry line (debit or credit) where the posting date is inside the selected reporting period
* row label must use the gl account name exactly as stored in the gl accounts table
* for each expense gl account row, system must calculate:
** description = gl account name
** current period = (sum of debits in period) – (sum of credits in period)
** prior period = (sum of debits in prior period) – (sum of credits in prior period)
** year-to-date = cumulative (debits – credits) from fiscal year start up to the end date of the selected reporting period
** *% of Expenses:* 
*** *if total expenses > 0 → (current period ÷ total expenses) × 100, display as "XX.XX%"* 
*** *if total expenses ≤ 0 → display "-" and show warning banner above report: "Total expenses are zero or negative for this period. Percentage calculations have been suppressed."*
** % change from prior period (if selected period is not a custom date range, e.g. Quarter 1, January) =
*** if prior period ≠ 0 → (current – prior) ÷ prior
*** if prior period = 0 → -
* total expenses = sum of all current period expense values across all expense gl accounts
* these exact expense values (each row + totals) must be used by the dashboard
* *Note: If the user selects a custom date range, do not show any columns involving comparison with prior periods.* 
* If no records are available, indicate in export that “No records were found” 
* if total revenue ≤ 0 AND total expenses ≤ 0 → "Total revenue and expenses are zero or negative for this period. Percentage calculations have been suppressed."
