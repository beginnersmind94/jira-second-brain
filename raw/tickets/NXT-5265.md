# NXT-5265 — UI DEV - Verification - Tracking - Confirmation Review

## Description
*User Story*

As a district admin I have not had my software approved to avoid the confirmation review from happening. I need to be able to confirm each application selected for verification.

*Mockup*

[https://www.figma.com/proto/Y3tlwN2yvRc67304adzyyB/Eligibility-3.0?node-id=1110%3A5773&scaling=scale-down-width&page-id=1110%3A0|https://www.figma.com/proto/Y3tlwN2yvRc67304adzyyB/Eligibility-3.0?node-id=1110%3A5773&scaling=scale-down-width&page-id=1110%3A0|smart-link] 

*Business Information*

This is an optional configuration where applications need to be ‘confirmed’ by an official prior to contacting the household about their selection for verification.

It’s controlled by a system setting that sometimes needs to be turned on if the district performed poorly in a prior Administrative Review. It’s also required unless software has been approved by the state as having a high degree of accuracy.

!image-20201224-212548.png|width=909,height=94!

*Requirements*

* If the system setting for Verification Confirmation is turned on:
** Tracking pages:
*** Show the confirmation status of applications selected for verification - sampled, not for cause.
*** If confirmed, show Confirmed.
*** On the Inactive tab & No Response - 
**** If not confirmed, an action button is available for the user to perform the confirmation.
***** Button text: Confirm
**** If they hit the button, the confirmation status should be updated.
*** On the other tabs - 
**** We show Not Confirmed as text, no action available.
** Application’s verification section:
*** Update when the confirmation was done, and by who.
** History tracking for who confirmed and when.
** Actions will not be possible on an application without it being Confirmed.
** Page would check the confirmation setting to hide or un hide the Confirmation Column.
Selection has no limitation. Notify button would be turned on if the selection has atleast one confirmed record.

## Acceptance Criteria
If the system setting is on, the information on confirmation status should be available in the application verification panel and the tracking pages.

Information is shown for who confirmed it and when.

History tracking is also done for this action.

Actions on the application (other than Confirm) are not available until after the application is confirmed.

Page would check the confirmation setting to hide or un hide the Confirmation Column.
Selection has no limitation. Notify button would be turned on if the selection has at least one confirmed record.
