# NXT-65571 — Prod APIs Environment for 2.0

## Description
As a district admin accessing the production SchoolCafe website,
I want the site to be connected to the production API instead of the QA API,
so that all logins (including with Family Hub credentials) and data retrieval happen in the live production environment and reflect accurate, real-time information.

## Acceptance Criteria
1.  API Endpoint – The production website must point to the production API (e.g., https://schoolcafeapis.perseusedge.com/token) for authentication and data retrieval.
	2.	Environment Check – Verify that no production URLs reference QA endpoints (https://schoolcafeqaapis.perseusedge.com).
	3.	Login Validation – District admins can log in successfully via the production API using their Family Hub credentials, without errors.
	4.	Feature Verification – All features and data accessible to district admins function as expected post-login.
	5.	Regression Testing – All login flows for all user roles work correctly after the endpoint update.
	6.	Deployment Verification – Post-deployment smoke test confirms the production site uses the correct API endpoint.
