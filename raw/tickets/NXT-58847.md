# NXT-58847 — MVR - Optionally Save Filters

## Description
As a MVR user, I would like to optionally decide to not use the filters for the search grid in the managed view (MV) and managed view report (MVR) so that I can use the  managed view’s other functionality

*Notes:*

* We want to give the user the choice to save the filters in the managed view or not
* And then for the MVR to respect that
* This request comes from FO team and they feel they have some managed views where they don’t want to save the filters 

*Requirements:*

* Use the Users grid to work on this and make sure it works
* On a managed view (MV), when click the save or save as buttons, it opens the save screen
* On that Save screen, put a new field:
** Save Filter Values
*** Toggle
*** Default Yes
*** If user selects yes:
**** include the filter values in the MV configuration saved to the DB
*** If no:
**** Do not include the filter values in the MV config in the DB
* 
* When running an MVR, make sure that the respects these values
** If yes, it uses the filters
** If no, it doesn’t use them

## Acceptance Criteria
* User is able to decide if they want to include filters on any managed view
* Default is Enabled for the toggle
* When running an MVR, If user selects to include filters, it uses the filters
* When running an MVR, If user selects not to include filters, it doesn’t use them
* All existing managed views continue to work as is (don’t break)
