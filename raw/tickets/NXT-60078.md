# NXT-60078 — WS - Meals

## Description
As a Workspace user, I want to view individual KPI metrics in an easy-to-understand card format so that I can quickly assess performance.

h2. Comments

* At dev mgr. request, we have combined all things related to this KPI in one story. These are:
** KPI Card
*** Values, actions, filter connections, widget visibility, benchmarking
** School Performance Grid
*** Value integration
** Trend Chart
*** Value integration
** APIs to support this functionality
* Out of scope for this story
** Details page (covered separately)

*Component*: KPI Card

The card displays:

* KPI name 
* KPI value
* Trend indicator
* Performance status
* Benchmark goal

h2. User Flow

# User logs into Workspace and goes to Insights page
# User views KPI card on dashboard
# User can:
#* See current value
#* View trend direction
#* Compare to benchmark
#* Track goal progress
#* Click for details

h2. Requirements

h3. Card Display Elements

* For the card, show these things:
* Status indicator
** This is a green or red color bar at the top of the card
** Green if values is greater than or equal to goal
** Red otherwise
* KPI name
** Read only label with the name of the KPI
* Trend Icon
** Related to status indicator, not really a trend measurement
** Green if values is greater than or equal to goal
** Red otherwise
* KPI Value:
** Current KPI value and its unit indicator (%, $, #)
* Expected
** This is a simple text that says “Expected” and is related to the Goal
* Goal Comparison Section:
** Benchmark (goal) value and its unit indicator (%, $, #)
** Value comes from KPI Config drawer for the card with the same KPI name

h3. Interactions

* Filters:
** Ensure card is updated when change period
** Ensure card is updated when change location
* Configurations:
** Ensure card is updated when change benchmark configuration
** Ensure card is hidden or shown based upon KPI Configuration toggle
** Ensure card is in the order specified on the KPI Configuration screen
* Click Actions:
** Details:
*** On click, open details panel (covered in another story)

h3. Data Requirements

* API for KPI Performance Data:
** Name
** Current value and Unit type
** Benchmark Goal
** Goal Status (green/red)

* All communications between the drawer and APIs occur over HTTPS
* The system ensures that data is retrieved and processed securely
* The drawer interacts with versioned API endpoints (e.g., {{/api/v1/...}}) to ensure future compatibility and backward support
* API endpoints implement rate limiting to prevent abuse and safeguard against denial-of-service attacks

h3. Performance & Usability

* Ensure the card displays properly in these layout options: Full width, Compact, Mobile responsive, Expanded
* The card should load within 2 seconds under normal operating conditions
* Interactions such as opening configuration panels and switching between views are smooth and responsive

h3. Security

* Inherits page’s view permissions, no need to set permissions on the cards directly

h3. Error handling

* Use Error Boundaries and Try/Catch in Asynchronous Code
* Show User-Friendly Error Messages in modal dialogs, toast notifications, or inline messages without disrupting the entire experience
* Log errors to the database

## Acceptance Criteria
h4. *Card Display Elements*

* The *status indicator* is a color bar at the top of the card:
** Green if the KPI value is greater than or equal to the goal.
** Red otherwise.
* The *KPI name* is displayed as a read-only label with the name of the KPI.
* The *trend icon* visually reflects the KPI’s status:
** Green if the KPI value is greater than or equal to the goal.
** Red otherwise.
* The *KPI value* displays:
** The current KPI value.
** The unit indicator associated with the value (%, $, #).
* The *goal comparison section* includes:
** The Benchmark (goal) value.
** The unit indicator (%, $, #).
** The benchmark value is retrieved from the KPI Configuration drawer for the KPI with the same name.
* The "Expected" label appears next to the goal value to indicate what the expected benchmark is.

----

h4. *Interactions*

* *Filters:*
** The KPI card updates when the period selection changes.
** The KPI card updates when the location selection changes.
* *Configurations:*
** The KPI card updates when benchmark configurations change.
** The KPI card is hidden or shown based on the KPI Configuration toggle.
** The KPI card appears in the order specified on the KPI Configuration screen.
* *Click Actions:*
** Clicking the KPI card opens a details panel (functionality covered in another story).

----

h4. *Data Requirements*

* *API for KPI Performance Data* must provide:
** KPI name.
** Current KPI value and its unit type.
** Benchmark goal value.
** Goal status (green/red).
* *Security and API Requirements:*
** All communications between the KPI cards and APIs occur over HTTPS.
** Data retrieval and processing must be handled securely.
** The KPI card interacts with versioned API endpoints (e.g., {{/api/v1/...}}) to ensure future compatibility and backward support.
** API endpoints must implement rate limiting to prevent abuse and denial-of-service attacks.

----

h4. *Performance & Usability*

* Ensure the card displays correctly in the following layouts:
** Full width
** Compact
** Mobile responsive
** Expanded
* The KPI card must load within 2 seconds under normal operating conditions.
* Interactions such as opening configuration panels and switching views must be smooth and responsive.

----

h4. *Security*

* The KPI card inherits the page’s view permissions, and no additional permissions need to be set at the card level.

----

h4. *Error Handling*

* Implement Error Boundaries to catch UI errors.
* Use Try/Catch blocks for all asynchronous API calls to handle failures gracefully.
* User-Friendly Error Messages:
** Errors should be displayed using modal dialogs, toast notifications, or inline messages without disrupting the user experience.
* Database Logging:
** Errors must be logged into the database for tracking and troubleshooting.
