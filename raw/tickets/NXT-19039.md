# NXT-19039 — UIonly- Production - Production Record - Modify Food Used

## Description
With the ingredient and stock item information associated with each other, the software is capable of doing the math to determine how much food will be required to generate the quantity of portions that each site needs to prepare for their students. The resulting information is based directly on the recipes that have been created/generated in the software. 

There are times when modifications must be made to recipes, depending on the availability of all of the products. Sometimes I might substitute half the quantity of shredded cheddar with shredded colby jack… or I might need to use turkey lunchmeat because of not having enough ham to make sandwiches. In these instances, I need to be able to edit the food that was used during the production of my menu.

## Acceptance Criteria
There should be a way for me to select to “Edit” the food used in conjunction with each menu item on my production record. 

It should provide me with the full quantity and listing of food, so that I may adjust quantities, add new information, and/or remove a single item from the list. 

It will need to have a method for me to save the updated information. 

It should also have an automated notation regarding the fact that the food used has been modified, as well as a time/date stamp and the user logged into the software at the time of modification.

The pencil in this screenshot is what a user would click on to make changes to the designated “Food Required” list below the menu items on the Production Record Page:

!image-20210615-175615.png|width=1508,height=460!

After clicking the pencil, the user will receive a pop up window, displaying the “System Generated” information for the required food items. To make a change, the user will select the “Manual Override” radio button, which populates everything into the text box, and that will allow the user to make edits and save.
