# NXT-43900 — MP- Find and Replace- Replacing menu item with a VMI that has sub items already on menu

## Description
As a menu planner, when I replace a menu item with a VMI, and that VMI has sub items that are already on the menu, the VMI should “inherit” all of the menu items that are already on the menu.

## Acceptance Criteria
* Allow menu item to be replaced by a VMI 
* IF VMI child items already exist on the menu, all sub items should now be included as part of the VMI
* Display Info message “ Menu item(s): *Menu Item Name*, *Menu Item Name* has now been added as part of the Variety Menu item” (green Font) at the top of the menu page once user access any of the menus that were effected in this way
!image-20231009-051737.png|width=284,height=52!
** If there happens to be multiple menu items that were already on the menu then display the multiple menu item names within the message 
** Allow message to be removed
* All menus should be able to be find and replaced at the same time
* These changes should sync to production and need to be confirmed for any plan incomplete or plan complete records
** We should not effect any plans that are already part of a consolidated plan *complete*
* Rules of the 2 Variety Menu Item with Same scenarios should apply from FInd and Replace as well.
