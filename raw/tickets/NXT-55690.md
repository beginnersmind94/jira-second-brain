# NXT-55690 — IM - Item Onboarding - Initial Item Info

## Description
As an onboarding district admin I want a step for creating new items so I can insert the number of records I wish to create.



!IO - Initial Info Early-20241016-135707.png|width=91.66666666666667%,alt="IO - Initial Info Early-20241016-135707.png"!

## Acceptance Criteria
* Provide an initial step in the Item Onboarding tool to insert a new record for each new item being created
* Capture Item Description 
* Provide Food/Non-Food column
** default to Food
* Provide ‘# of Inventory Items’ radio selection
** default to None (same behavior as Units card)
* Provide ‘Is this a Direct Menu Item’ radio selection
** default to “Not a Menu Item (or decide later)”
** For Non- Food items, “Not a Menu Item” should be selected and grayed out as non food items can not be made to menu items at this time
* Provide a Save button to save the records and officially create the placeholder items
** validate against any duplicate descriptions, missing data, etc. (only applicable to Item Description)
* *Note:* if any of the non-Description fields cannot be developed at this time, do not implement and wait for a later sprint
** confirm with PO & Team
