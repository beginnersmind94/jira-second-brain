# NXT-45938 — Imports - Display Connection Error in View Log

## Description
As a SchoolCafé user I would like to know why there is no data in View Log sometimes so that I don’t have to call customer support

*Notes:*

* Sometimes SIS Provider servers go down for maintenance and are otherwise unavailable and this causes us to not have any history log, source data, exceptions etc. b/c those processes were unable to run
* This ends up giving us a blank screen in view log and users don’t know why
* We want to warn the users what happened

*Requirements:*

* This functionality applies to Import Monitor records
* When user clicks View Log and the import failed b/c of provider issues and we don’t have data, show the user a message explaining the issue.
* Text: Unable to retrieve data from SIS Provider due to issues beyond our control.



+*UI Changes*+

# *Import process - View log* 

       When any SIS API provider did not yield data because of any internal reason, API will log the End user message and error message stack trace into DB and return 412 status code (no records) like below. 

On click of “View Log” UI need to show the below highlighted props from the response into the grid.    

!image-20240429-222507.png|width=91.67%,alt="image-20240429-222507.png"!

*EndUserMessage* - to be showed in the Grid 

*Entity* - to be showed when hovered over the above error message (Refer to same design as error activity) 

!image-20240429-222743.png|width=91.67%,alt="image-20240429-222743.png"!

# *Preview data*

       Upon clicking of Preview data when the SIS API Provider did not return any data due to any internal reason, API will return 412 status.

UI need to catch this and show in the grid 

*EndUserMessage* - to be showed in the Grid 

*Entity* - to be showed when hovered over the above error message (Refer to same design as error activity)
