# NXT-42421 — INV - Orders - Provide the Receipt Report before receipt creation

## Description
As a site manager I want the Receipt report before I create a receipt so I can use it when receiving goods in person.

## Acceptance Criteria
* Report UI Updates: 
** Increase the vertical spacing between header fields
** Remove the ‘($)’ from column headers if the cell values are already including the $ 
** NOI should be displayed in parentheses as a negative value
* Provide the Receipt Report before any receipts have been created (banner row button)
* Generate with all ordered items
* Leave the Received Quantity column blank for users to write it in
* Provide lines for input fields where users can write
* Show ‘Total Price’ as ‘Estimated Total’ for the pre-receipt version of the report
* Remove banner row Report button once a receipt is created
** existing card level Report button is enabled for specific receipts
