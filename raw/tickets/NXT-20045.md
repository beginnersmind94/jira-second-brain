# NXT-20045 — Viewing a report - Data Query Framework

## Description
As a system I need to display the Data Query report when the user clicks on a report from the list of reports.

Dev notes: As per the [high level design|https://primeroedgenext.atlassian.net/wiki/spaces/NEXT/pages/2169143343/Data+Query+high+level+design] this screen would utilize the event driven UI approach. The following would be the flow for it:

# User will be redirected to this screen which should initiate two things.
# -Should register the signalR client with the hub and create the UI that awaits the broadcasted message from SignalRHub.-
# UI component should make a GET call to the api end point passing the report id and receive report’s data in response back.
# -Should make and API call to the Process endpoint by passing the report id that needs to be ran, which will publish the message to the rabbit mq.-
# -The Eligibility sync agent will contain a subscriber that will be listening to the message, when a message arrives this agent queries the database and gets the data for the report.-
# -The sync agent then broad cast the data for the report to the signalR hub.-
# The data is then displayed on the UI.

## Acceptance Criteria
# When I click on the report and able to see the data for that report in a screen.
# When I click on Excel button, I should be able to get the excel file with the reports data.
