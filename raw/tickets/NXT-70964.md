# NXT-70964 — Insights - KPI - ENP - Dashboard - PE

## Description
*I am a* district user *that I want to* view Eligible Not Participating (ENP) performance in the Insights dashboard
*so that I* can monitor actual performance against expected benchmarks across sites and timeframes.

----

h2. *Notes*

* This story applies to *Insights – PrimeroEdge data source*.
* ENP (Eligible Not Participating) is part of the *front office KPIs* being integrated into Insights.
* This story covers the *Dashboard screen only*.

h3. *Data & Source of Truth*

* The source of truth for ENP data is available in:
** *POS → Daily Reports → Patron Participation*
* To generate the correct dataset:
** Filter *Eligibility* to *Free and Reduced only*
** Generate report
* This must be used by developers and QA to validate ENP values.

h3. *Stored Procedure Integration*

* The front office team has already created the stored procedure:
** PrimeroEdge: [https://dev.azure.com/Cybersoft-Technologies-Inc/PrimeroEdge%20Classic/_workitems/edit/109336|https://dev.azure.com/Cybersoft-Technologies-Inc/PrimeroEdge%20Classic/_workitems/edit/109336]
* The dashboard must use this stored procedure as the data source for ENP values.

h3. *KPI Definition & Formatting*

* ENP is a *percentage-based KPI*.
* Format must be *0.00% (two decimal places)* on the dashboard.
* Values must be:
** Rounded to two decimal places
** Between *0 and 100*
* The dashboard must display:
** Actual value
** Expected value (*ENP Goal from benchmark configuration*)

h3. *KPI Direction*

* This KPI is *Lower is Better*:
** Values closer to *0% indicate better performance*
** Higher values indicate worse performance
* Performance indicators, comparisons, and trends must respect this direction.

h3. *Filtering Behavior*

* All ENP components must respond to *Timeframe and Site filters*:
** KPI card
** School Performance grid
** Performance Trends chart

h3. *Interaction Behavior*

* Clicking on ENP values (card or grid) must open an *ENP details drawer* (covered in a separate story).

----

h2. *Out of Scope*

* SchoolCafé implementation (separate story)
* Benchmark configuration UI (covered in separate story)
* ENP details drawer implementation
* Schoolie Update

h2. *User Flow*

# A user logs into PrimeroEdge.
# The user navigates to Insights.
# The user lands on the Insights dashboard.
# The user selects timeframe and site filters.
# The system displays KPI cards including ENP.
# The user views ENP:
#* Actual value
#* Expected value
#* Performance indicator
# The user may:
#* Adjust filters → dashboard updates

#* Performance Trends chart → selects ENP

h2. *Requirements*

h3. *1. KPI Card – ENP*

* Add a new KPI card:
** Label: *ENP*
** Location: *added to end by default*
* Card must display:
** ENP percentage (actual value) for selected filters
** ENP Goal (from benchmark configuration)
** Trend indicator
* Values must be formatted as *0.00%*
* Card must update based on:
** Timeframe filter
** Site filter
* On click:
** Open *ENP Details Drawer*

----

h3. *2. KPI Card – Performance Indicators*

* Card must display color states:
** *Green* → Actual ≤ Goal
** *Red* → Actual > Goal
** *Grey* → No benchmark configured
* This KPI follows *inverse performance logic*:
** Lower values are better
** Higher values are worse
* If no benchmark exists:
** Card displays grey

----

h3. *3. Filters (Timeframe & Site)*

* Dashboard must include:
** Timeframe selector
** Site selector
* Changing filters must update:
** ENP KPI card
** School Performance grid
** Performance Trends chart

----

h3. *4. Screen Configurator*

* Ensure ENP is added to *Configure KPIs drawer*
* User must be able to:
** Toggle ENP card on/off
** Reorder KPI cards via drag-and-drop
* Changes must:
** Persist per user
** Apply to dashboard layout

----

h3. *5. School Performance Grid*

* Add *ENP column*:
** Label: *ENP*
** Location: *last column*
** Format: *0.00%*
* Column must display ENP per site
* Grid must update based on:
** Timeframe filter
** Site filter
* Grid must support:
** *Horizontal scrolling when columns exceed screen width*
* On click:
** Open *ENP Details Drawer*

----

h3. *6. Performance Trends Chart*

* Add *ENP* to KPI dropdown:
** Location: *bottom of dropdown list*
** Order must align with benchmark configuration screen
* Selecting ENP must:
** Display ENP trend line
** Update based on timeframe and site filters

----

h3. *7. Trend Hover Behavior*

* Hovering over the ENP trend line must display tooltip including:
** Date
** *Students (count)*
** *Eligible (count)*
** *ENP (percentage formatted as 0.00%)*

----

h3. *8. Data Integration*

* Ensure ENP is added to:
** Backend API responses
** Dataset used by dashboard
* Data must support:
** Student count
** Eligible count
** Actual ENP values
** ENP Goal (benchmark)
** Trend calculations
** Chart rendering

----

h3. *9. Performance Requirement*

* Dashboard must load and update without performance degradation
* Must support large districts with many sites and long timeframes
