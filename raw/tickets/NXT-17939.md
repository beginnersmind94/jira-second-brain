# NXT-17939 — MP- Menu Planner- Save State (Locally)

## Description
As a Menu Planner, I want to make sure that whenever I work on menu planner, if I get called away from my desk and I have been working on menu planner, the system will save me information if my login is getting timed out. 



There will be two different processes on how the system timeout should save information from the Menu Planner Page. 

Scenario 1:
A user starts working on Menu planner and has put items on only one day. If login timeout occurs, menu will be saved with timestamp as the name and can be edited in the traditional menu creation page. 

Scenario 2: 

If there are multiple days with information, it will create a 5 day menu cycle or 7 day if "include weekends" is selected. Menu cycle name will also be defaulted to time stamp of last save. In addition, all menus within the menu cycle will also receive the same time stamp with the additional wording Week 1 Day 1, Week 1 Day 2. Menu Cycle days with no information will be included in the Menu cycle but will have no Menu name saved for day.

## Acceptance Criteria
Save Menu planner instance locally and local DB. 

We will not be saving menu or menu cycles to the system DB.

Retain last menu planner session whether done from system timeout or user leaves menu planner page.

Menu planner page will need a “reset filters” to restart menu planner process.

Store last session, user can resume last session no matter what the time frame unless user has “save and continue” “save” or “reset filters”
