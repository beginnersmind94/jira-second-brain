# NXT-30285 — Import - Vendors - Platform File Upload to Staging Only (1/3)

## Description
{color:#ff5630}This story is intended to cover the Platform team’s portion of the import.  BackOffice will have to complete their piece for the import to work end-to-end.{color}

There are 3 steps involved in an import:

# Platform uploads the file to a staging repository
# The module team exposes an API to be notified that step 1 is complete
# The Platform notifies the module mechanism via the API identified in step 2.

This story is for step 1.

## Acceptance Criteria
*Step: Pre-Upload*

* A new “Import Type” shall be used for this import named “Inventory - Vendors”.
* A dropdown parameter named “Data Source” shall be present containing one entry named “Local”.
** The “Local” Data Source entry shall be selected by default
** There is lack of clarity from the Back Office request on additional values within the “Data Source” dropdown.  Until there is, we will treat it as if there are no additional values.
* The user shall be prompted to download a template using the standard verbiage
** The “No” radio button shall be set by default
** The file template to be used shall be attached to and provided by the Back Office team in story [https://primeroedgenext.atlassian.net/browse/NXT-30256|https://primeroedgenext.atlassian.net/browse/NXT-30256|smart-link] .  The fields marked in red font in the attachment are mandatory.

*Step: File Upload*

* No changes from existing behavior

*Step: Field Mapping*

* No changes from existing behavior
* No code mapping required

*Step: Review*

* No changes from existing behavior



*Business Rules and Error Log Rules*

* The “Business Rules” and “Error Log Rules” as stated in [https://primeroedgenext.atlassian.net/browse/NXT-30256|https://primeroedgenext.atlassian.net/browse/NXT-30256|smart-link] shall be the responsibility of the Back Office team.
