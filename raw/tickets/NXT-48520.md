# NXT-48520 — PR- Autofill- Final Menu Check when submitting autofill to match menu to plan

## Description
As a production manager, when I am submitting my autofill, if there happens to be changes that happen on the menu, I would like those changes to reflect if/when I click the autofill button to bring the latest information on the plan page.

## Acceptance Criteria
* When clicking autofill on the slideout to confirm the autofill, do a final check against the menu to make sure any changes that are done on the menu are reflected on the autofill insertion.
* Apply all checks
** Adding/removing serving groups
** Adding/removing menu items
** Changing serving sizes
** Changing meal counts or menu item counts (Menu Planning counts option only)
* All new changes should be inserted into the plan page once the autofill is confirmed and the plan page is refreshed
* Any new serving groups added would obviously not warrant any counts as the previous plan did not have those counts either (Previous plan option)
** This would not apply to the previous record option as we can not make changes to the menu with record complete menus.
** This would apply to the menu planning counts option as we can still change the menu item counts
* IF user enters counts on the plan but has not saved the counts and then clicks autofill, display a pop up message that asks the user the following 
** !image-20240320-073612.png|width=728,height=127!
** IF user clicks the continue button, discard all counts from the UI and open the autofill slideout to start the autofill process
** IF user clicks the save button, save all the counts and refresh the plan page for them to save the counts and now they can access the autofill with all saved counts and decide what they would like to do
