# NXT-50141 — LMA - Subscription - License Administration

## Description
As a License Management (LMA) role member, I would like to subscribe to License Management events so that I can stay aware of what’s happening.

*Notes:*

* We will leverage the event subscription framework for this story
* Do it like we have done for Users, Custom Reports, Imports, etc.

*Flow:*

* User browses to System > License Management
* User clicks on Subscribe button
* License Key Subscriptions drawer opens
* User selects self, users or roles to be notified
* Selects criteria under which scenario to be notified
* Optionally sees which users & roles are already subscribed
* Optionally deletes the notification
* User clicks Save
* Later, if one of the criterion are met, the System triggers the notification process
* System notifies by sending message to user according to notification criteria and timing
* User browses to inbox and reviews the notification

License Key event subscription process

* When a License Key event is triggered, check to see if there are any subscription notification for it
* If Criteria are met:
** Verify if need to send immediately or schedule for the future
** If immediately:
*** Then prepare a notification message
*** There are no files to save, just notification text
*** Send the message to the framework for messaging with the event subscription
*** Framework will determine the notification settings for each user and use them for distribution
*** Distribution framework will send the message to each recipient to their notification methods selected for this event
**** For roles, get each role member and send individually
** If scheduled:
*** Schedule to run using the scheduler and do everything the same but at that time
