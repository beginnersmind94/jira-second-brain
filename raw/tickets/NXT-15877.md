# NXT-15877 — INV - Create Orders - Generate Incomplete Orders

## Description
*User Story*
As an Inventory user I want the system to generate and split orders for me based on order criteria such as delivery date, vendor, etc., so I can begin to review and place orders.

This story is a back end story for generating the orders when user clicks ‘Next’ on the Shopping Cart. These orders will be presented on the Confirm Orders screen ([NXT-5714|https://primeroedgenext.atlassian.net/browse/NXT-5714]).

*Requirements*

* Generate orders based on item quantities entered
** Orders will be created by grouping items together which have the same attributes listed below:
*** Vendor
*** Delivery Date
**** Delivery Date is determined and calculated according to:
***** Vendor Delivery Schedule for the site
***** Vendor Lead Time per the contract
***** Product Lead Time per the contract
* Generate Order # for each order
** Order Number should be long enough for future central realm functionality, but no leading zeroes should be shown to the user. Order Number should appear as “1001” or “12345”
* Orders will be initially created with a Status of ‘Incomplete’
* Orders will be populated on the Order Review/Confirmation screen ([NXT-5714|https://primeroedgenext.atlassian.net/browse/NXT-5714])
