# NXT-14644 — Accountability - Transactions - All Transactions - Search

## Description
As Sophie the Site Manager I want the ability to search all transactions collectively without having to go into each session.

*mockup*

[https://www.figma.com/proto/m3QGmo7Fjys7um36XDhwAY/Accountability-5.0?node-id=504%3A13485&scaling=scale-down-width&page-id=499%3A13482&starting-point-node-id=504%3A13485|https://www.figma.com/proto/m3QGmo7Fjys7um36XDhwAY/Accountability-5.0?node-id=504%3A13485&scaling=scale-down-width&page-id=499%3A13482&starting-point-node-id=504%3A13485|smart-link] 

It’s going to be something along the lines of our existing search pages, with the data a grid version of the Participation and Payment reports together.

*requirements*

* User has an ‘All Transactions’ page on the Transactions page.
* User can search with the following filters. Do multi-select for as many as possible, with an All option as default.
** Program (multi-select)
** Meal Type (multi-select)
** Transaction Type (multi-select)
** Date (usual assortment of options)
** Site Types (multi-select)
** Site(s) (multi-select)
*** The following are also available, but should have their options updated as sites are changed:
**** Grades (multi-select)
**** Homerooms (multi-select)
**** Rosters (multi-select)
**** Terminals (multi-select)
** Eligibility (multi-select)
** Menu Item
** Payment Type (multi-select)
** Review Reasons (multi-select)
** Additional options as checkboxes:
*** Only Show Exceptions
*** Only Show Suspicious Transactions
*** Only Show Reimbursable Meals
*** Everything
* User can then run the Search.
* Results grid is handled in a separate story.

## Acceptance Criteria
‘All Transactions’ page added.

Search filters added per requirements.

Search is possible.
