# NXT-6026 — Application Entry - Guided Process (6. Review)

## Description
*User Story* 

As an admin, i can review the information entered in previous steps, which are organized by section and easily navigate back to a section to make edits if needed.

*Mock Ups* 

[https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=1306%3A27072&scaling=min-zoom|https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=1306%3A27072&scaling=min-zoom]

*Requirements* 

* Review page is organized in sections based on the data entered in previous steps thus far 
** Students, Benefits, Household, Contact Info, Details
*** Benefits:
**** Case Benefits - if no, then no display. If yes, display answered fields
**** Refuse Benefits - display answer (Yes or No)
* Admin can select the edit pencil to navigate back to a section and make changes as needed 
** reference stepper story to jump 
* Applicant can go to previous page or next step
* If a section is blank - should only be for Contact Info.
** “_Contact information was not indicated in the application._ ”
* Eligibility Determination panel 
** HH Size 
** Income total
** (no outcome column)

## Acceptance Criteria
- admin is guided to next step 
- admin has option to go to previous step 
- admin can review information entered thus far from previous steps 
- display of answers in shown on review page
- admin can edit any section info by using the section link 
- the section link takes the applicant to the correlating section to make edits
- Multi-language, ADA compliant.
