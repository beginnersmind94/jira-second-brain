# NXT-19251 — INV - Create Order - Item Quantities - Update to reflect Item Hierarchy

## Description
As a Site Manager I want the Create Order screen to show related items on individual rows so I can see the various item hierarchies when ordering items.

*Requirements*

* +On Enter Item Quantities screen+
* Remove ‘Broken Unit Quantity' column
** Update to remove any references to Broken Units, as Broken Units will no longer exist
* Add a ‘Product Type’ column
** This will display one of 6 options: Pallet, Mixed Pallet, Display Shipper, *Case, Package, Each* (bolded ones are most common)
*** This will be pulled from the details of the item in Item Management (See image below)
** display this column after Item Description column, before Vendor
* Group related items together in the grid
** Items which are linked together from Item Hierarchy in Item Management should be displayed as consecutive rows
** This should at least be done on initial pageload
** Sort the *related* items by Product type descending (Pallet > Case > Package > Each)
* Rename ‘Whole Unit Quantity’ to simply ‘Quantity’
** for the unit label, make sure this is referencing the new ‘Unit Description’ free text value (not the ‘Whole / Purchase Unit’ dropdown value which INV1 is removing)
* Change ‘Number’ to # hash sign
** change ‘Item Number’ to ‘Item #’
** change 'Vendor Item Number' to ‘Vendor Item #’
* Make the Item Description column as wide as possible
* +On the Shopping Cart+
** remove Broken Unit Quantity
** group related items together if related items are in the cart
*** Ex: Case item and Package item are both in the cart. Show the Case item right above the Package item
** display the Unit Description _after_ the order quantity textbox
* +Update Order Quantity validations+
** Minimum and Maximum Order Quantity validations should apply to the single Order Quantity field after BU are removed
** Update tooltip message to remove the “WU & BU” part
*** Revised example: “Increase/Decrease order quantity by 5 to meet minimum order quantity.”
