# NXT-17958 — API - Enhancing the the notification end point in Administration API

## Description
As a system, I would would need to enhance the existing system’s API , Notification End Point to include the scheduling functionality. This would require the following end points to include the scheduling details.

# Config - POST: This end point would now get an additional details for the schedule as an object which will contain values if the notification type is 'Schedule', else it would be null.
# Config - GET: The current notification configuration object would now include an additional object that will contain scheduling info if the notification type is 'Schedule'.
# Config/Search- GET: The current notification configuration objects would now include an additional object that will contain scheduling info if the notification type is 'Schedule'.
# Config - PUT : This end point would now get an additional details for updating the schedule of a notification configuration object if the notification type is 'Schedule', else it would be null.
# Config - DELETE: This end point will need to delete the notification configuration schedule when a notification configuration of notification type 'Schedule' is deleted.

To enhance the functionality of the above end points the NotificationManager and NotificationRepository needs to be enhanced to accommodate the functionality to store and retrieve info from NT_ConfigSchedule table.
