# NXT-2967 — System - Management - Roles - Add New Custom Role

## Description
As a district administrator, I would like to be able to easily add a custom role to the list of roles within my organization, so that I can give my users the specific permissions they need.



+*Summary:*+

* On the *Roles* page, when the user (with permission) selects the Add New button, open a drawer to add a new role.
* On this drawer, require the user to enter a role name, description, and at least one organization level. Rules:
** Role Name should be max 50 chars and must be unique
** Role Description should be max 200 chars and does not need to be unique
** Organization Levels are standard: Site, District, Area, Sub-Region (may not be pictured in mock-up)
* When the above rules are met and the user selects ‘Save': 
** Close the drawer 
** Display a success message
** If no custom roles already exist, display a new section for Custom Roles
** Add the role to the section
* Otherwise, the user should be able to Cancel out the changes as needed.
* The user should now be able to click the role and assign permissions to it.



Mock-up (click three dots icon next to a role and click Copy first): [https://nqyc9v.axshare.com/start.html#p=roles___permission&id=fjbxyo|https://nqyc9v.axshare.com/start.html#p=roles___permission&id=fjbxyo]

## Acceptance Criteria
Verify that when a user clicks the Add New button, the Add Role drawer opens

Verify the design of the drawer contains the 3 fields above

Verify that Role Name must be unique

Verify that Role Name must be < 50 chars

Verify that Role Description does not need to be unique

Verify that Role Name must be < 200 chars

Verify that at least one Org. Level must be selected (if applicable)
