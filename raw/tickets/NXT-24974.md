# NXT-24974 — MP > Menus > Archive / Edit (based on production status)

## Description
Archive & Edit functionality of Menus - dependent on Production functionality

If a user enters the page of a menu that:
1.) has been assigned to production, and
a.) is in Plan Incomplete status
b.) is in Plan Complete status
User should receive a message that the changes they’re about to make will impact the menus already assigned to production.

2.) has been assigned to production, and
a.) is in Record Complete Status
b.) is in Withdrawal Complete Status
User should receive a message that this menu can not be changed, and the user can choose to Archive the existing menu so that they can edit a duplicate of it (Archive/Edit button), or they can choose to copy (Copy button) this menu and work with their copy.

If scenario 2 is in place, the following should occur:
1.) If the user chooses to archive and edit the menu, the existing menu should be saved with the current date/time stamp added to the menus name, and the existing menu name should be used in creating a copy of the menu for them. The menu items on existing menu will be in place on the created copy. This allows the user to continue using the menu with the original name.
2.) If the user chooses to manually make a copy of the menu, then the normal procedure for copying a menu will take place. This includes the user needing to make a unique name for the new menu that they're creating.
