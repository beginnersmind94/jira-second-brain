# NXT-70311 — FIN - Add Entry Amounts to Manage Periods Table

## Description
As a Child Nutrition Director, I want to see posted and pending entry amounts on the Manage Periods page, so that I can assess the financial impact of each period at a glance without navigating to the journal.

## Acceptance Criteria
h3. 1. Amount Column

* Add a "Posted Amounts" column to the Manage Periods table.
* For each period row, display the total debits (or credits — they should be equal) across all entry types for that period.
* Amounts must be formatted as currency (e.g., $4,320.50).

h3. 2. Expandable Breakdown

* When the user expands a period row, show amounts broken down by entry type and site.
* Each row in the breakdown must show: entry type, site, debit amount, credit amount.

h3. 3. Pending Amounts

* If a period has pending entries (entries that could not post because the period was closed), display the pending total separately, visually distinct from posted amounts (e.g., "$1,200.00 pending").

h3. 4. Dirty Period Amounts

* If a period is marked dirty (source data changed after posting), display both the currently posted amount and the expected amount based on updated source data, so the user can see the delta.
* Example: "Posted: $3,100.00 | Updated: $3,250.00 (+$150.00)"

----

h2. Out of Scope

* Notification behavior (separate story).
* Journal-level detail (user navigates to journal for full entries).
* Exporting amounts from this page.

h2. Dependencies

* FIN - Financial Period Management (existing Manage Periods page)
