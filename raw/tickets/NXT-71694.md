# NXT-71694 — Elig - DC - Split Street Address for DC Matching

## Description
As Dana the Director I want to better control how Street Address is matching on the DC Scorecard to catch scenarios where points are not allocated due to City/Town being different in some parts.

!image-20260406-171548.png|width=1024,alt="image-20260406-171548.png"!

*requirements*

* Split Street Address up into:
** Street Address (7 pts)
** City (1 pt)
** State (1 pt) 
** Zip Code (1 pt)
* Update DC Matching to make use of the individual fields and the new default points
* Update the DC Scorecard to show the individual fields
* User should be able to see prior DC Scorecards with no issue - if we matched on the “full” Street Address and gave 10 pts, show that

## Acceptance Criteria
* Street Address split
* DC Matching algorithm updated
* DC Scorecard updated
* Prior Scorecards not impacted
* User can adjust points from the DC Match Configuration page
