# NXT-36439 — Acc - Sessions - Delete Session

## Description
As Dora the Director I want to completely remove a session. This will save the time it will take to individually roll off each transaction for me, or for Support to run a script to do this on my behalf.

[https://www.figma.com/proto/QRowEx99f8nKGMmpZPeLRq/Front-Office-6.3?node-id=264%3A5546&scaling=scale-down-width&page-id=264%3A5195&starting-point-node-id=264%3A6224|https://www.figma.com/proto/QRowEx99f8nKGMmpZPeLRq/Front-Office-6.3?node-id=264%3A5546&scaling=scale-down-width&page-id=264%3A5195&starting-point-node-id=264%3A6224|smart-link] 

*requirements*

Inside the new More Actions dropdown inside of Reconciliation

* Add a new option:
** Delete Session
* User will be given a confirmation prompt and a mandatory comment box.
** Text on the pop-up: 
*** *This action will completely remove the session and all transactions.* Affected accounts will show a transaction adjustment. 
*** Enter a comment below. You can track the removed session and its comments in the *Transactions > Adjustments > Session* page. 
*** If you proceed, an *Undo* option is available for 30 seconds.
** Comment box: standard length, required field
** Box options: Cancel, Proceed
* User needs an option to cancel the delete.
** Give the user 30 seconds to click an Undo button as part of the success toaster.
** Message: Session will be deleted shortly. [Undo]
*** If they click on Undo, cancel the delete on the session.
* If the user doesn’t cancel within 30 seconds, start the deletion process.
** All transactions need to be adjusted off with the comment entered.
** All payments adjusted off.
** All sales adjusted off.
** These should be available on the Adjustments page with the comment.
** A Session adjustment needs to be added also which will include the comment.
** Session status is set to Deleted.
** Opening/Closing Balances removed.
* Inside a deleted session:
** All actions are disabled aside from Session History
** Summary sessions → show 0 counts, Transaction entry fields disabled
* Rollups need to be run for the date the session was on.
* New session status needs to be available on the Sessions page.

Role Permission

* Add a role permission for this action.
** Sessions - Delete Session

## Acceptance Criteria
* Delete Session option added.
** Requires a confirmation from the user.
** Requires a comment.
* Undo button available for 30 seconds before deletion begins.
* Deletion to be done via adjustments.
* Adjustments of all types must be available under the Transactions > Adjustments page.
** *Absolutely essential that we have adjustments for everything done in the session. We may add an ‘Undo’ option for this in the future.*
* Deletion must be available under Session Adjustments.
* Adjustments should use the comment from the user in this format: Session Deleted: comment
** We should already be tracking the user + time on this as part of adjustments in general.
* Rollups run afterwards.
* Role Permission added.
