# NXT-41896 — DC Import – Handling Mandatory Exceptions

## Description
As a SchoolCafé admin, I want to be able to import all the successful DC rows even if some fail so that I can get started with using the system.

*Flow:*

* Open SchoolCafé and browse to Settings > Imports & Exports > Imports
* Find a DC Import you want to use
* Click on Import for that row
* Follow the import process
* Validate the fields that are mandatory
* When the import routine comes across a row that fails because of a required field, skip it, log it as an exception and keep on importing the rest of the records
* Notify the user when the import finishes

 

*Notes:*

* This story corresponds to how we handle exceptions of mandatory fields like when importing in Student Import process 



*Requirements*:

* When click on Import button for a DC Import:
* When running, if there is a record that fails because of a required field, do the following:
** Currently it causes the import to fail – don’t do this, keep going
** Skip that record in the import file
** Add that failure to import that record as an exception with the same verbiage as Student Import
** Continue processing the remaining records
** This is all just like Student Import
* When creating the import file and testing if the import file is accurate
** Continue to test the same way you do currently, no changes
* 
* For each field, put a required length validator per the database schema
** Probably the same as is for Student, but verify
* 
* When viewing the History log from Import Monitor, ensure that these exceptions show in the Exception tab as Mandatory
* The following are the Mandatory columns:
** ???
* For each one:
** Validate the required field is present
** Validate it does not exceed the max length
* Just like Student Import, do not import the row if these do not validate, but log them
* For SchoolCode
** Validate that it is a legitimate SiteCode just like for Student Import
** If it is a valid SiteCode, log it as an exception and do not import the row

## Acceptance Criteria
* When importing, if there is a record that fails, the import routine skips it and continues
* Failed record imports are marked as exceptions and visible in the exceptions tab
