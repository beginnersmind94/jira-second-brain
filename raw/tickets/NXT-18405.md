# NXT-18405 — Eligibility - Reports - Income Surveys-Economically Disadvantaged

## Description
As a district admin, I need to be able to view the list of accounts at my school who are economically disadvantaged, so that I can provide the necessary data required by my state.

*Summary*

Add the “Economically Disadvantaged” Income Survey report, while also slightly restructuring the Eligibility - Reports navigation.

*Requirements*

* +*Note:*+ This page has changed somewhat in Classic to also include the scenario where students were approved based on a Meal Application submission, but we will handle that case in a future story, possibly via Report Options.
* In Eligibility - Reports - Income Surveys, display the _Report Type_ Economically Disadvantaged (default selection). 
** This tab (Income Surveys) should only display if the setting to show the district’s Income Surveys is set to ‘Yes’ or ‘Mixed’.
* Tab contents and behavior:
** _Select Site(s)_: display -Site Code +- Site. (Site code requirement removed- will be taken up in a future enhancement that PO will create) 
*** Default Option: ALL (individual sites can be selected as well)
*** -_Date as of_--: Allow user to select any current or past date-
**** -Default date: today’s date-
*** _Generate Report_ button
** When the user selects ‘Generate Report’ within the Economically Disadvantaged context, display the Economically Disadvantaged report. We can follow the Classic structure and set of columns (see screenshots below):
*** Site Code
*** Site
*** Free
**** In this column, display the count of students who have been matched to the ‘Free’ status based on the submission of an Income Survey.
*** Reduced
**** Same as above, except for ‘Reduced’ students.
*** Paid
**** Same as above, except for ‘Paid’ students.
*** DC Students
*** Enrollment
*** DC %
**** This is the # of DC students divided by total number of students
*** Total F/R
**** Sum of all free and reduced students in the given report criteria 
*** Total F/R%
**** This is the 'Total F/R' divided by all Free, Reduced, and Paid students in the given report criteria 
*** Also, the report should display the report option selections in the header or footer of the report

## Acceptance Criteria
* Verify that a new tab, *Income Surveys*, is added to the Reports page (but only displays based on their setting to show Yes/Mixed surveys)
* Verify that when the tab is selected, the user can select the Site(s) and Date of the report, as well as generate it
** Verify the default selection for ‘Site(s)’ is ‘ALL’ and the ‘Date’ is today’s date by default
* Verify that when the user generates the report, the 'Economically Disadvantaged' report is displaying accurately based on their selections
* Verify that the report is displaying the report option selections in the header and/or footer of the report
