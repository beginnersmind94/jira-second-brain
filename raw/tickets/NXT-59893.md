# NXT-59893 — IM - Items - Units card - Allow 'Pound' as a pack size base unit

## Description
As a food service director I want the Units card to allow Pound as pack size base unit, so I can inventory some items by weight.

[Freshdesk Issue/Request 276604|https://primeroedge.freshdesk.com/a/tickets/276604]

## Acceptance Criteria
*Bug*:

* Ensure proper validations are present when a record cannot be saved in the Units card
* For invalid Pack Size unit descriptions, show a proper error message and disable the save action
** current behavior just removes Sub-Qty, and does not allow user to understand the problem

*Enhancement*:

* Modify the pack size description logic to allow ‘Pound’ as a valid unit
** do not allow Ounce, Gram, etc.
* Pound can only be used as the base unit description
** disallow lower units to be created
** disallow higher units to be modified to use Pound
* Do not disrupt the logic for inventory-able servings
