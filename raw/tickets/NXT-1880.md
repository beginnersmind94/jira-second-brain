# NXT-1880 — Menu Planning - Configuration - Menu Item Quick Lists 

## Description
Manage, view, add, and edit menu item quick lists.

As the user handling the process of creating menus, I would like to have the ability to set up lists of menu items that I can add to my menus in one quick process.  One example of a use for a quick list would be to have a list of breakfast items that are used frequently together, so that I don’t have to search them out singly and add them to the menu one item at a time. Or if I have items that go on my menus every time I serve a particular food (such as on Taco Tuesday, the district might always have tacos, the side salad, quesadillas, refried or charro beans, spanish rice, and guacamole). I can add that “Taco Quick List” to my menu quickly and easily, all at one time. 

* I will need a page to manage the creation and editing of these lists.
* When creating a quick list, I will need the ability to search for, select, and add menu items to the list. 
* When editing a quick list, I will need the ability to search for, select, add and delete menu items to/from the list.
* I should have the ability to delete a quick list when I want to, with no dependencies, other than to confirm an “Are you sure you wish to delete this quick list?” message.

## Acceptance Criteria
As the user handling the process of creating menus, I would like to have the ability to set up lists of menu items that I can add to my menus in one quick process.  One example of a use for a quick list would be to have a list of breakfast items that are used frequently together, so that I don’t have to search them out singly and add them to the menu one item at a time. Or if I have items that go on my menus every time I serve a particular food (such as on Taco Tuesday, the district might always have tacos, the side salad, quesadillas, refried or charro beans, spanish rice, and guacamole). I can add that “Taco Quick List” to my menu quickly and easily, all at one time. 

* I will need a page to manage the creation and editing of these lists.
* These lists are independent of any meal type, site group.
* On quick items list configuration page, display name, how many menu items, and what the data source is.
* When creating a quick list, I will need the ability to search for, select, and add menu items to the list. 
* When editing a quick list, I will need the ability to search for, select, add and delete menu items to/from the list.
* Columns to display on the menu item quick list page (Field Search for all Fields except projected servings)
** Menu Item Name (Recipe Code)
** Projected Servings --- This will be a free text field ( Optional)
** Serving Size
** Meal Contributions
** Allergens
** Data Source 
* When adding menu items, add the menu item to the category they belong to. 
* If user designates a projected serving count, divide by number of serving groups EX: if serving count is 100 and there are two serving groups it would 50/50 if 3 then 34, 33,33, Use this story for reference on distribution of counts ([https://primeroedgenext.atlassian.net/browse/NXT-26994|https://primeroedgenext.atlassian.net/browse/NXT-26994|smart-link] )
* When quick list is created, on the configuration page, there should be way to preview the quick list and see all the items as well as what category they are in. 
* I should have the ability to delete a quick list when I want to, with no dependencies, other than to confirm an “Are you sure you wish to delete this quick list?” message.
