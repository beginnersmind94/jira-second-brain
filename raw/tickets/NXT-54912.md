# NXT-54912 — MVR - Users Grid Pages - Delete Update

## Description
As a SchoolCafé user, I would like to be able to setup my managed view for recurring distribution to a set of users so that I can get periodic notifications like on custom reports.

*Notes:*

* These requirements were in the parent story, but we separated out as a separate story to facilitate QA
* The parent story needs to be completed first or in parallel

*Requirements:*

* On System > Users, do the following:
* On the Saved Views tab for Managed Views drawer:
** Ensure that any Managed View that has a Managed View Report for it has a “D” in the letters column
*** D for Distribution
*** Tooltip for D = Distribution
** Actions Updates
*** Edit
**** Make no changes
**** Edit is for the MV only, not for the MVR
**** If the user wants to edit the MVR, they can go to the MVR dashboard
*** Share
**** Make no changes
**** Share is for the MV only, not for the MVR
*** Delete
**** We have a little bit of biz logic on this one
**** The delete is for the MV, not the MVR, but if there is a linked MVR to the MV, then we need to handle it
**** In order to keep the design more simple, we will simply stop the user from deleting and tell them to unlink (delete) the MVR from the MV first
**** Update the Delete button as follows:
***** Ensure user has Delete rights
****** if no, then disable
***** Check if MV has MVR linked
***** If yes:
****** Disable the Delete button
****** Put tooltip: Delete distribution in Managed View Reports first
***** If no, then proceed as normal

## Acceptance Criteria
* On the Saved Views tab for Managed Views drawer, ensure that any Managed View that has a Managed View Report for it has a “D” in the letters column
* On the Saved Views tab for Managed Views drawer, disable the Delete button if MV has MVR linked and show tooltip
