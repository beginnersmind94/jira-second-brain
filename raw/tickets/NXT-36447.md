# NXT-36447 — Monitor of Monitors - Scheduled Job

## Description
for all the monitoring scripts we have, create one new one that monitors them and reports if they didn’t run

## Acceptance Criteria
As a SchoolCafé admin, I want to ensure that all the scheduled monitoring jobs are running per schedule so that I can proactively monitor the application and ensure higher customer satisfaction.

*Flow:*

* User browses to Support > Monitoring Dashboard > Job Scheduler
* User clicks the + New button
* User Selects System Module and selects Monitor of Monitors
* User sets the schedule and saves
* The system sets up a recurring job at the time and days selected by the user
* The system runs the jobs per schedule and output is viewable in the Monitoring pages

*Notes:*

* This story assumes that the screens for Job Scheduler and Add New Job have been created

*Requirements*:

* *Job Scheduler Page updates:*
* Ensure that Monitor of Monitors is in the Monitor drop down list
** Via API, not hard coded
* Ensure that after a Monitor job is created, that it shows up in the grid with the correct values
* 
* *Add New Monitor Page updates:*
* Add or ensure that System is a value in the Modules list
* Add Monitor of Monitors to the list of values for the Monitor drop down list
* When the user selects System, the list of values for Monitor are restricted to that Module so ensure that Monitor of Monitors is one of the values visible for System
** Schedule section: no additional requirements
** Save button:
* On click, create the scheduled job per the dates and time selection in the schedule section (what to do in detail follows below) and then redirect to Job Scheduler screen
* 
* *Job Requirements*:
* The scheduled job will do the following:
* For every job that has been scheduled, verify that it has run at its scheduled time
** Time frame: in order to not keep repeating error reporting, verify only those jobs since the last time this monitor ran
* 
* *Emails:*
* Collect all the exceptions (monitor jobs that didn’t run) and send an email to the new support email we created in the previous story
** remember that there are 2 emails, one for prod, one for non-prod
* Email:
* Subject: Monitor of Monitor Exceptions – District Name
* Body:
* Environment (Prod, UAT, etc.)
* Date / Time Monitors checked
** This is the date/time the monitor of monitor runs
* 
* List all the exceptions
* Table Columns / Fields:
* Monitor Name
* Scheduled Time
* The date / time that it was supposed to run, but didn’t
* 
* *Auto Correct:*
*  If the monitor of monitors finds a scheduled job that did not run, call the job immediately and see if you can get it to run
* 
* Permissions: no extra requirements
