# NXT-65000 — Ledger Entry Rules (Proposed #5)

## Description
*As a* district business administrator configuring Financial Oversight,
*I want* to view the debit/credit rules tied to each type of ledger entry,
*So that* I can clearly understand how transactions will be posted and ensure correct account mappings.

Note: In terms of functionality, this is the GL Entry Types tab under configuration in Financials for PrimeroEdge

The full list of rules for system accounts can be found [here|https://cybersofttechnologies-my.sharepoint.com/:b:/g/personal/rahul_mehta_cybersoft_net/EVJ13VdumHpGv0PL82CL2mEB-bTzNUuLco71gdE6-sLonw?e=KeHWB7]

## Acceptance Criteria
* Users can view a *list of entry types* that the system will post transactions for (e.g., Meal Sales, Adjustments, Inventory).
* For each entry type, users can see:
** The *system accounts* involved,
** The *account type/category* (e.g., All Meal Sales, All A La Carte Sales), and
** The *credit/debit designation* for each account.
* Users can expand any entry type to see the *detailed transaction rules* (e.g., “Debit Cash in Bank OR Prepaid Account, Credit Meal Sales”).
* Users are able to *add new accounts* under an entry type.
* Users are able to *add new templates* for ledger entry types to customize posting rules.
* The UI clearly indicates *read-only system rules* versus those that can be edited or extended.
