# NXT-47718 — File upload with Multi region data should qualify to update duplicates in base and child tables 

## Description
Steps to reproduce:

Upload a file with single region and verify the data in base and child tables

Upload a same file with different region name and ID then verify in the base and child tables, where data is not creating successfully.

[https://apitests.primeroedge.co/GCQAAPIs/api/GDSNConnectSC/Uploadfile?RegionId=51&RegionName=testregion&Username=Suresh|https://apitests.primeroedge.co/GCQAAPIs/api/GDSNConnectSC/Uploadfile?RegionId=51&RegionName=testregion&Username=Suresh]

## Acceptance Criteria
Expected:

The item name should be treated as unique only with in the region, while the same item should be allowed to create in other region (in reality most district will have large items with same item names)

The target tables (Serving Size, Nutrition, Allergens, Meal Credits, Pack Size, Units and Images) should have the corresponding values written as well.

These IDs shall be used to write the items on SchoolCafe as well

Solution Ideas: if needed, can we merge region ID and Import Item ID and keep it as unique ID for validation purposes?
