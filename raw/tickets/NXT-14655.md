# NXT-14655 — Accountability - Default Menu Grids - Additional Columns and Filtering

## Description
As a user I need better clarity on the site types, sites, and devices that are assigned to default menus so there is less ambiguity from the grid view.

*Additional columns:*

* Site Types
** If site type assigned = 1, show site type name
** If site type assigned > 1, Show “\[X Sites Types]” where X is the total of site types associated 
* Sites
** If site assigned = 1, show site name
** If site assigned > 1, Show “\[X Sites]” where X is the total of sites associated 
* Devices
** If devices assigned = 1, show device name
** If devices assigned > 1, Show “\[X Devices]” where X is the total of devices associated

*Filters (order as listed below)*

* Site Type - Multi-select dropdown; All option by default
* Site - Multi-select dropdown; All option by default; filtered by any site types selected
* Devices - Multi-select dropdown; All option by default; filtered by any site types and/or sites selected
* Menu - Multi-select dropdown; All option by default; filtered by any site types, sites, and/or devices selected

Note: Similar filtering is already provided when adding/editing details of a default grid. We can likely re-use these filters within the page as well.

*New column display order should be:*

* Meal Type
* Menu
* Site Type
* Sites
* Devices
* Days
* Actions

## Acceptance Criteria
* Additional columns added
* Filters should display default menu results
* Logic of information displayed within each new column follows description of = 1 (show name) and 1> (show count)
