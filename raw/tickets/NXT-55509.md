# NXT-55509 — DMF - Dashboard - Precheck

## Description
As a Customer Support role user, I would like to be able to precheck for commonly known issues before trying to import so that I work on problems more than one at a time

*Notes:*

* After our first prototype review, the following was brought up as a “would like to have”
* The issue is that in the current process:
** We can only find issues one at a time
** We try and import, it fails, we look at what happened, go to source data and correct and then start over only to find that next step has a problem too, and so on
** This is very inefficient
** We would like to try and find as many import issues as we can as fast as we can and deal with them all at once in order to cut down on the import cycle

*Requirements:*

* On the nested grid, make these changes:
* Add a button at the end of the row
** Make it an icon
** Tooltip: Precheck
** On click:
*** Open the Precheck drawer



*Precheck Setup*

* In the backend, create a way to check for common issues that cause failures
* Do it in a way that is scalable
** Need to be able to add checks in the future
** Needs to not be horribly slow
* An example of a precheck:
** We know that users require email address but in PE, it is not required, so many user accounts don’t have it and will cause import failure 
** Our precheck would analyze the source data and see if there are any user accounts missing email address
* As we go along, each time we do an import and see failures, we can add that cause to the list of prechecks
** Eventually, we should be able to fix most of the problems before actually trying to migrate



*Precheck Drawer*

* Create a new drawer
** This drawer will show default comments and allow the user to add/edit/delete from the comments drawer
** Note: this is essentially the same exact thing as doing the Validation that we did for student import, so copy how you did that if you can
* Title:
** [Module Name] Prechecks
** Ex: System Prechecks
* Size: Full Page
* Use sliding motion to open
* Fields:
** Everything exactly the same as the Student Import Validator except no file upload dialogue
** Validate button
*** Does same as in Student Import but it runs the prechecks and outputs the exceptions into the file
** Validations grid
*** Called Validated Files in Student Import
** Be able to download the validation is a must have
** Being able to preview is a very nice to have

## Acceptance Criteria
* User is able to precheck / validate source data for commonly known issues and see the results
