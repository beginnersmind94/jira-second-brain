# NXT-48797 — SSO - Microsoft (Social)

## Description
As a Customer Support user, I want to be able to setup Microsoft social as a district’s authentication provider so that they can use SSO

*Notes*:

* This story is for setting up Microsoft Social in Keycloak, not for ADFS
* This feature will be done mostly the same way as OpenID and the requirements will reference this

*Flow:*

* User browses to System > Security page
* User clicks the Microsoft Social auth type Add new button
* The system adds Microsoft Social to the grid like it does for SchoolCafe simple authentication
* System saves the config to Keycloak
* User tests the config
* User optionally sets Microsoft Social as the default SSO provider
* User optionally views History
* User optionally sets Microsoft Social as their own SSO provider
* User optionally sets Microsoft Social as someone else’s SSO provider if they have rights
* User optionally makes Microsoft Social active or inactive
* User optionally deletes Microsoft Social

*Requirements*:

*Security Screen*

* These requirements apply to the System > Security page > Authentication tab:

** If they have Microsoft Social in the grid, then disable this button

**** Auth type = Microsoft Social
**** Description = Login with Microsoft’s Social SSO Provider (This is not Windows or ADFS SSO)
**** Mark In-Active and Not preferred by default
*** Note: they can have only one Microsoft Social provider, so do not allow them to add multiple of this type
** Note: there is no Add New drawer for this type since there are no config settings to capture for Keycloak since Keycloak has all the necessary config settings already

## Acceptance Criteria
* Admin is able to add Microsoft Social as an authentication method for their district
* Admin is able to assign Microsoft Social as the preferred method
* Admin is able to enable / disable Microsoft Social method
* Admin is able to delete Microsoft Social method if there are other rows in the grid
* Users are able to set Microsoft Social in their profile if it is a valid option
* Admins are able to set Microsoft Social for a user if it a valid option
* Changes to Microsoft Social auth show up in History page
* Passwords are handled the same was as for Microsoft Social 
* If user is Microsoft Social method, they redirect to their provider when click next on login screen
* User is redirected back to our website with valid auth token after logging in
