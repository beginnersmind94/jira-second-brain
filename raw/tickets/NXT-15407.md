# NXT-15407 — [AcctMgt] - ID Cards - Search Filters & Grid - APIs

## Description
Within the ID Cards page, we need to have several ways for a user to specify which ID cards are needed.

This story is to implement the APIs so that the UIs created in [https://primeroedgenext.atlassian.net/browse/NXT-4637|https://primeroedgenext.atlassian.net/browse/NXT-4637|smart-link] are connected and a search can be performed. Everything except being able to print the ID.



*Main mock-up:*

[https://www.figma.com/proto/yCRz6zcMFIjX3cY1BxgQI7/Accounts?page-id=165%3A0&node-id=165%3A1&viewport=845%2C297%2C0.125&scaling=min-zoom|https://www.figma.com/proto/yCRz6zcMFIjX3cY1BxgQI7/Accounts?page-id=165%3A0&node-id=165%3A1&viewport=845%2C297%2C0.125&scaling=min-zoom|smart-link] 

*Requirements:*

* Add the following search filters:
** Account ID
** Site Code + Site
** Homeroom
** Roster
** Grade
** Template
*** The placement of this particular filter may change and it may not be selectable until the user hits the ‘Print’ button. This will be determined when the mock-up is completed.
* When the search filters are selected, the list of accounts with viewable/printable ID cards should display in the grid appropriately.
* Results:
** Should be filtered by the above Site/Grade/Homeroom/Roster/ID (if individual).
** Can be both Students and Adults.
** Should be active accounts only.
* Results should be a grid showing the following information:
** Patron ID
** Last Name
** First Name
** Middle Name / Initial
** Site Code + Site Name
** Grade
** Homeroom
** Roster
** Placeholder Print icon

## Acceptance Criteria
Verify that the filters above are added to the ID Cards page

Verify that when the filters are selected, the search grid updates appropriately

Verify that results include Students and Adults, Active accounts only.

Multi-language support (can be deferred).
