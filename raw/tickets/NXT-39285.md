# NXT-39285 — GDSNConnect - PE - Update Local Ingredients (UI)

## Description
*Assumptions*:

The database shall contain 3 data sets:

# Ingredients without Stock Items
# Stock Items without Ingredients
# Ingredients Linked with Stock Items

## Acceptance Criteria
Pending Updates should have a new Dropdown as shown in the screen attached. The data shall refresh once the user has selected the data provider dropdown:

# 1WorldSync
# 3rd Party Source(s)

User should be able to select the list of attributes that are changed and be able to import the changes to local ingredients or stock items

Local items changes should be able to captured in Audit Log

!image-20230509-230319.png|width=1263,height=674!

*UI Rules*:

The data should be interpreted in the UI based on the following details

# Ingredients without Stock Items (Flag this as Ingredient in API) - UI shall show clickable hyperlink against Ingredients ONLY
# Stock Items without Ingredients (Flag this as Stock Item in API) - UI shall show clickable hyperlink against Stock Items ONLY
# Ingredients linked with Stock Items  (Flag this as Linked Stock Items in API)  - UI shall show clickable hyperlink against BOTH Ingredients and Stock Items

The API should render the the above 3 data sets and render the details in Pending Updates screen as shown in the screenshot

The user should be able to click on the counts against each item that has new values



*Data Processing Rules:*

Items Marked as Non-Food - the attributes related to Serving Size and Nutrition Details, Contributions (Meal Credits) shall NOT be listed in UI

Items Marked as Food - shall list all the attributes including Serving Size and Nutrition Details, Contributions (Meal Credits)

Allergens - the data shall have || symbol which should be treated as Separator. Each Allergen should be listed as an update

Meal Credit (Referred as Contribution in UI) - the data shall have || symbol which should be treated as Separator. Each credit should be listed as an update



*Approach*:

Compare the data between Local Items and New Data Sets

The Attribute comparison shall be managed via SPs

Use District_Menu_ID (Ingredient Codes) to list the attributes under Ingredients page

Use District_Item_ID (Stock Item ID) to list the attributes under Stock Items page



*QA Guidelines*

The Updated data should be referenced in UI based on the attached Mapping sheet (Ingredient and Stock Items)

Regression Testing on Bulk Updates (Review Items) Audit Log, Performance and Pagination, Data Filters , Navigations to Items and Stock items etc
