# NXT-58238 — FOH Dash - Claim Readiness - Configuration Options

## Description
As Dana the Director I want to indicate which areas have an impact on overall claim readiness, as not all of them are important to my operation.

*mockup*

!image-20250119-201159.png|width=1100,height=737,alt="image-20250119-201159.png"!

*requirements*

* User can configure what areas make a site “ready”
** Areas that can be included/excluded and their respective default % value:
*** Sessions - 25%
*** Edit Checks - 35%
*** Reverse Edit Checks - 15%
*** Serving Exceptions -10%
*** Review Reasons - 5%
*** Applications - 5%
** All are on/included by default.
* User can adjust the following:
** Included checkbox
** Total Confidence %
* User can see a total for the Confidence %s - add each of the Included %s together
* User cannot save their changes on this screen unless the total % = 100
** Text if false: Please adjust Confidence %s so that it equals out to 100%.
* User has a brief explanation for each Area (text below).
* User has text at the top of the page to explain how the configuration works.
* Access locked to Edit page permission
** Tooltip for disabled button: You do not have Edit permissions on this page. Check Roles & Permissions if this is in error.



Brief explanation text for each one to be included:

* Sessions: Number of Sessions that are not in Reconciled/Deposited.
* Edit Checks: Number of unresolved Edit Checks per site.
* Reverse Edit Checks: Number of low meal count instances per site.
* Serving Exceptions: Number of duplicate meals & eligibility variances.
* Review Reasons: Number of transactions marked for review.
* Applications: Number of unprocessed applications.

Top of the page explanation:

* You can adjust which areas are included and their percentage impact on overall Claim Confidence here. More detailed explanations of each Area are available in the Confidence Rating section of the main page.

## Acceptance Criteria
* User can see and configure the Readiness Areas
** Toggle included/excluded
** Change % value
** Total configured % value
* User cannot save changes unless the Confidence % total = 100
* User has info explaining each Readiness Area
* User has an explanation note at the top of the page
* Access to this area and saving should be tied to an Edit page permission
** Button is disabled otherwise
