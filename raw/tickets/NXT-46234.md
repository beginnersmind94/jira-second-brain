# NXT-46234 — Site Configuration - Add Grades Management Tab

## Description
As a SchoolCafé Custom Support role user, I would like the ability to add and delete grades myself so that I don’t have to go to Dev team and ask them to do it for me

*Notes:*

* Try to write this as a component that can be reused in the onboarding questionnaire (covered in another story). 
** Does not need to be a new part/page. See if we can integrate this within the Site Types section of the questionnaire.

*Requirements:*

* These requirements apply to the System > Sites & Users > Site Configuration screen
* Create a new tab
** Title: Grades
** Location: right of Grade Types tab
* Fields:
* Add New button
** On click:
** open the Add New drawer
* Grid
** data: list of grades that are in the database
** Columns:
** Grade
*** the name of the grade
** Actions
*** buttons:
*** Edit
**** On click, open the Edit Grade drawer
*** Delete
**** On click:
**** Check if the grade is in a Site Type/Grade group
**** If yes, show the user this message “Remove grade from grade group first” and allow them to close the popup, but don’t delete it
**** If no, show them a confirmation popup and then delete from the system if they say yes
* Security
** View permissions managed via roles and permissions view property for the page
** Default Customer Support and SchoolCafé admin role with add, edit or delete
** Default all others to view only



*Add New Drawer*

* Create a drawer for when the user clicks + New button
* Title: Add New Grade
* Fields:
** Grade Code
*** Textbox
*** Max Length = max length in corresponding database table column
*** Should accept numbers, characters, and if possible, special characters (PK, EE, N/A etc.)
** Description
*** Textbox
*** Max Length = max length in corresponding database table column
*** Should accept numbers, characters, and if possible, special characters (PK, EE, N/A etc.)Save
** Cancel



*Edit Grade Drawer*

* Fields:
* Grade Code
** Same as Grade Code from Add New
** Preload the value
** Make read only, user can’t edit 
* Description
** Corresponds to Description from Add New
** Pre-load value of Description
* 
* On save, update instead of insert



*Onboarding Screen*

* Add the same grid with add/edit/delete functionality to the Onboarding flow
*

## Acceptance Criteria
* In Questionnaire, a section for managing grades is integrated within Site Types step
* In Site Configuration, a tab is added and called Grades
* SchoolCafé Admin and Customer Support role users have default add/edit/delete access
* All other roles default to View only
* Users with permissions can add a grade with the code and description
* Users with permissions can edit only the description
* Grade Code management added to Onboarding screen
