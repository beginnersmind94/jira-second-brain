# NXT-61059 — AI Onboarding - Digital Recipe - Edit Recipe Section

## Description
As a customer support person, I would like to use AI to digitize a recipe document so that I can speed up onboarding

*Notes:*

* This story covers what happens when a user selects a recipe in the file analysis section and how to edit a recipe
* Submitting the changes are in another story
* Explanation of user experience deviation from existing recipe screen:
** Existing screen puts the ingredients in the step on the right side, thus linking an ingredient to the step
** Recipes we want to import almost all have an ingredient list and a separate step list and they are not linked
** So it was decided to match the flow in our design to the output the AI engine can give us
** Therefore in this design, we too are treating the ingredients as a separate object from steps
** But, we are allowing the user to link them using the Step selector in the Ingredients section

*User Flow:*

* User browses to Item Management and then Recipes
* User clicks on the AI button and it opens the AI Recipe drawer
* User ensures the New chip is selected
* User selects format and a file to upload and clicks upload
* AI analyses the file and returns the results
* System saves the results, loads the File Analysis section
* System automatically selects the first recipe and loads it into the edit recipe section
* Optionally, the user can select another recipe and the system loads it in this screen

*Requirements:*

* On the Report AI screen, do the following:
* This story covers the right side of the screen which is for editing a recipe
* When the user selects a recipe in the file analysis section, do the following:
** Load the values from the database table for each field as they are recorded
** Note: Values could be the original ones the AI engine selected or it could be the edited and saved value by the user
* Basic Information and Servings sections are the only sections that have required fields
* On load:
** Check status
** If Complete, lock the fields down and do not let them edit it and show the sections as contracted if you can to discourage them from trying
* 
* *Basic Information section*
* Expandable/Contractible 
* Name field:
** Required
** Validate that the recipe name does not already exist
*** If it does, do not allow them to proceed with submitting the recipe until they change it
** Max length and other validators same as the name field on the + NEW Recipe screen
* HACCP Process field
** Required
** Default value:
*** Check if there is a selected value in the database table
*** If yes, use it
*** If no, default select None value
* Tags:
** Optional
** Same requirements as on the + NEW Recipe screen
* Prep and Cook Time
** Optional
** Validate positive numbers only
* 
* *Servings Section*
* Expandable/Contractible 
* # of Servings
** Required
** Validate positive numbers only
** Same requirements as on the + NEW Recipe screen
* Amount
** Required
** Validate positive numbers only
** Same requirements as on the + NEW Recipe screen
* Description
** Required
** Same requirements as on the + NEW Recipe screen
** Textbox the user can select existing value from or type their own
* Strict Batching
** Optional
** Same requirements as on the + NEW Recipe screen
* 
* *Ingredients Section*
* Expandable/Contractible 
* Note: it is possible for a user to save a recipe without ingredients, so this entire section is optional
* Columns:
** Source Ingredient: the name found by the AI engine
** Source Quantity: the count and measurement type found by the AI engine
** SC Ingredient:
*** the item in the district’s database that corresponds to the source ingredient
*** If there is a value in the DB table, load it
*** Load the Item values for the district
*** If the SC Ingredient matches the Source Ingredient, auto-select it
*** Note: name = SC Ingredient, not SC Menu Item
** Measure
*** the corresponding measurement for the SC Ingredient
*** Values:
**** Must select the SC Ingredient first
**** Load the values for the selected SC Ingredient
**** If there is a value in the DB table, load it
**** If the SC Unit matches the Source Quantity, auto-select it
**** Example Value: 1
** Unit
*** the corresponding unit for the SC Ingredient
*** Values:
**** Must select the SC Ingredient first
**** Load the values for the selected SC Ingredient
**** If there is a value in the DB table, load it
**** If the SC Unit matches the Source Quantity, auto-select it
**** Example Value: Cup
** Step
*** List of steps below
*** First step should be zero, always
*** If they add a step, make sure the list of steps is updated to reflect it
*** If the user selects a step, add that menu item to the Step Ingredient column below
*** Default all ingredients to Step 0 if they don’t have a step
*** Validate: If the user has selected an SC Item and Unit, they must select a step also
** Delete button
*** On click:
**** Ask to confirm
**** If no, do nothing
**** IF yes, remove the ingredient from the list
**** If the ingredient is in a step, remove it from that step
** Add Ingredient button
*** On click:
**** add a row
**** Note: when manually adding an ingredient, there is no source values, so no need to show those rows
**** Use Case: AI engine missed one ingredient, so user manually adds it to save time
* 
* *Steps Section*
** Expandable/Contractible 
** List of steps for the recipe
** Step 0 logic:
*** There must always be at least one step
*** First step is always Step 0
*** If DB table doesn’t have a step, add one by default
*** If no description for step 0, the default the text to: Preparation Step
** Step Description
*** Required
*** Load step description from DB table
** Ingredients
*** List of ingredients linked to the step from the ingredients section
*** If a user selects a step in the ingredients, it is added to this section
*** If a step is deleted, it unlinks the ingredient to that step
** Delete button
*** On click:
**** Ask to confirm
**** If no, do nothing
**** IF yes, remove the step from the list
**** If the ingredient is in a step, unlink it
**** Update step numbers of rows behind it if they delete one in the middle
**** User cannot delete Step 0
** Add Step button
*** On click:
**** add a row
**** Ensure the step number is the next available number
* 
* *Allergens Section*
** Expandable/Contractible 
** List of allergens found in the document
** User can also add their own by manually typing
* 
* *Action buttons at the bottom:*
* Select what to import checkboxes
** Covered in the Submit story
* Discard
** Ask to confirm
** If no, do nothing
** If yes:
*** Don’t save any changes the user made
*** Mark the status as Discarded
*** Contract all the sections
* Submit
** Covered in the Submit story
* Save Changes
** Ask to confirm
** If no, do nothing
** If yes:
*** Save any changes the user made to the DB table
*** Mark the status as In Progress and update the File Analysis section
*** Leave the edit screen as is so user can continue
** Use Case: User is editing and realizes there is a missing ingredient, so they save it, then go add the ingredient, and come back and finish adding it
