# NXT-54107 — POS - Offline Mode, API Toggles

## Description
As Cena the Cafeteria Manager I have a poor wifi connection and the constant API checks cause significant service delays at POS.

*requirements*

* Establish a true “Offline POS” mode
** No API calls should happen. More on this below.
** Bottom banner for Offline POS should be shown to the user.
* Add a manual toggle for the user to move themselves into Offline Mode
** Heartbeat should not attempt to bring them back online if this has been purposefully enabled
* Sub-toggles to turn off particular API Calls while in Detailed Service
** Pictures
** Second Meals
** Visiting Students
* These toggles / offline mode itself come into play while the user is in Detailed Service
** Do not perform the respective API calls while they are in the Detailed Service window
*** Pictures → Just use the default picture (can we generate one without the ‘No Photo!’?)
*** Second Meals → Don’t check the server for second meals, only local data. Stop the routine check for meal data from other machines.
*** Visiting Students → Don’t check the server - if they’re not in the local, tell the user the account cannot be found.
* Check and automatically move the user into Offline Mode if they’re legitimately offline
** Only the ‘heartbeat’ needs to fire to see if they’re back online
*** But don’t go back online if the user manually toggled into offline mode

*Discussion 9/11/24*

* We will not add the option to Opening Balance for now

## Acceptance Criteria
* Offline Mode added
** Toggle to enter Offline Mode
** Toggles to turn off Pictures, Second Meals, Visiting Students
* User has access to the Offline Mode toggles from:
** Admin Tools
** -Opening Balance-
* This will not be persistent between sessions - user has to turn on these options each time they open a session.
** Disable the toggles as part of Closing Balance.
* POS → Detailed Service updated to use the toggles. Do not fire the APIs if the toggle is on.
* POS → Detailed Service can fall into Offline Mode if internet connection is offline.
** Routine check to try and go back Online, but not if the user manually entered Offline Mode.
