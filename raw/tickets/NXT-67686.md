# NXT-67686 — Track Login & Logout for each Session in Table Storage

## Description
*Current Situation:*
Our SchoolCafe application does not currently capture user login and logout activity. For auditing and compliance purposes, it is important to record this information.

*Proposed Enhancement:*
We need to implement a mechanism to log and store user session activity with the following key details:

* *UserName (Email)*
* *RegionId*
* *AuthenticationType (SchoolCafe, OpenID, Microsoft..)*
* *Login Date & Time*
* *Logout Date & Time*
* *Session ID*
* *Client IP Address* (if available)

*Implementation Details:*

* When a user logs in or logs out, the application should send a non-blocking (asynchronous) request to the backend containing the above details.
* The backend should also handle these requests asynchronously and store the session data in a persistent storage system for auditing purposes.

## Acceptance Criteria
Able to log the login & logout details for each user
