# NXT-53379 — MP - Menu - UI Updates for Permissions (UI ONLY)

## Description
As a District admin, I would like permissions set aside for on the menu page for what my users can and can not do.

## Acceptance Criteria
Update the Menu Planning module *Menu* page and subpages to show/hide page elements based on logic of the Permissions

* +Menu+ (List)- Page
** *View:* view icon (check for Menu Planner)
*** if user does not have Menu Planner permission do not show the sub-menu and route directly to standard 
** *Edit:* Pencil icon
*** if user does not have Menu Planner permission do not show the sub-menu and route directly to standard 
*** If user does not have edit permission menu list page but does have menu planner permissions, route directly to menu planner
*** If user does have menu planner permissions or Edit permission on Menu list page, hide the edit pencil
** *Add:* +New button
** *Delete:* Status (Active) toggle
*** If user does not have delete access, hide the toggle and provide label for active and inactive in status column
**** also disable toggle on Menu Details page 
** +Menu Details+ - Sub Page 
*** *Edit:* every editable field on page
*** *Add:* Add Menu Item (from both methods)
*** *Delete:* remove menu item
*** +Copy+ (Action): Copy button
*** +View Analysis+ (Action): Analysis button
