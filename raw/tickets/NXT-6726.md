# NXT-6726 — Application Details - Actions - Match Student(s)

## Description
*User Story*

As an applications processor I want match students as part of a specific application.

*Mockup*

TBD

*Requirements*

* Option to match pending students from a single application
* Find Matches
** For all the pending students in the app - attempt to find a student in the student database to match this against.
** Use the same matching criteria as we do for identifying a student.
** If a match is found: 
*** Show the information on the potential match.
**** ID number, name, etc. Same information that we’d be showing for the Potential student so the user can do a visual confirmation.
*** Show a visible indicator that a match has been found. (A green check mark?)
* Process Matches
** Execute any potential matches that are present in the application
*** Link the student to the application, process eligibility appropriately.
*** Add a comment into the application’s history for the successful identification and the benefits being extended.
**** Pending Student (Last Name, First Name) successfully linked to Student #ID (Last Name, First Name).
**** Comment should be appropriately timestamped and have the user who actioned this.
