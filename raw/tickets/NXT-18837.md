# NXT-18837 — IM - Item Details - Hierarchy - Link Items

## Description
As a FSD I want a search item tool so I can find related items and add them to their correct item hierarchy.

*Requirements*

* On the Hierarchy Details card, provide a ‘Link Item’ button
** This is displayed in the top right corner of the card header
** *Note:* if we open an item which has *no* linked items, the item may not have a ‘Unit Type'
*** in this case, the user must select the Unit Type for the item before the Link Items function can be used
** If there is not a valid Unit Type selected and the user clicks the Link Items button
*** show an error message saying “Please first provide a Unit Type.”
* When the Link Item button is clicked, the details card will change 
** the details grid will disappear
** display the message: “Are you adding a parent or child item for \[Unit Type] Item # \[Item Number] \[Item Description]?
*** Example: “Are you adding a parent or child item for Case Item # DST1234 - Rice Krispies Treats 10.8oz box?”
*** Example 2: “Are you adding a parent or child item for Mixed Pallet Item # DST9876 - Rice Krispies Treats Variety Pack Pallet?”
** Provide two buttons beneath the message
*** Parent button
*** Child button
* Once the user selects Parent or Child
** hide the buttons
** load a smart search field ([NXT-19692|https://primeroedgenext.atlassian.net/browse/NXT-19692])
* Once an item is selected from the smart search results
** populate item attributes into a preview grid
*** this grid will just be one row and will only show the info for the selected item
** grid columns:
*** Item #
*** Item Description
*** Unit Type
**** this will display the value if one exists
**** if no current value exists, display a dropdown here containing the eligible unit types
***** the eligible types would be limited to the units which are applicable
***** Example: if we are adding a child to a ‘Case’ item, only ‘Package’ and ‘Each’ would be shown
*** GTIN
*** Manufacturer Name 
*** Brand Name
*** Display an ‘Add’ button
**** this button is available as long as the item has a Unit Type
**** if Unit Type is still unselected, show a hover tooltip on the Add button which says ‘Please select a Unit Type’
***** *optional enhancement: highlight the dropdown if it is unselected
** User can select a different item from the Smart Search and that will replace the previously selected item in the preview grid
* Once the user has clicked ‘Add’ from the preview grid
** remove the Smart Search
*** move the preview grid to the top of the card
** disable the Unit Type as a label now 
** display a question to the user underneath the grid
*** Message: How many units of \[unit type]\[child item] per each \[unit type]\[parent item]?
**** Ex: How many units of *package* item # DST100-A per each *case* item # DST1234?
*** Provide a numeric entry box where the user can type this number (Child Items per Parent)
** display a ‘Link Items’ button
*** this is enabled once a value has been provided for ‘Child Item per Parent’
* Once Link Items button is clicked:
** clear the card of all Link Item components
** again display the hierarchy details grid with the newly added item ([NXT-19300|https://primeroedgenext.atlassian.net/browse/NXT-19300])

## Acceptance Criteria
* Users can add a new linked item by searching items
* Item results will be restricted to only eligible items
* Users must specify Number of Child Items per Parent
* Users can link items and view updated hierarchy
