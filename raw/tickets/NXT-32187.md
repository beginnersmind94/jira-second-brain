# NXT-32187 — DataQuery Framework - Reports Should be PII Protected

## Description
All reports/data models shall have PII protected if they contain PII.



Dev notes:

# The new field will be added to \[DQ_DataModelField] table called IsPIIEnabled that would indicate a field being PII protected.
# Product management team should be responsible for going through each field in each data models and indicating a field being IsPIIEnabled.
# Add a new back end endpoint that should take a report id and determine the fields that are IsPIIEnabled in it and return an array of strings that contains the fields that are IsPIIEnabled.

## Acceptance Criteria
* All reports / data models that contain PII shall be protected from roles who do not have permission to view PII
* As a reporting user, I shall be able to see, select, and add PII fields to a report within and from the “Columns” drawer when managing reports regardless of the permission “Protect Personally Identifiable Information (PII)” on the role(s) I belong to.
* As a reporting user, I shall be able to see PII data on a report only if at least one of the roles I belong to has the permission “Protect Personally Identifiable Information (PII)” set to False.
* As a reporting user, if I do not have permission to view PII data, all PII fields on a report shall display “****”
* If a report created by a user who has permission to view PII data shares a report with me and I do not have permission to view PII data, I shall not be able to view the PII data on the shared report.
