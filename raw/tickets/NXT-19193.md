## Acceptance Criteria
*Done when:*

Report meets the below criteria for displaying required information.

*Entry Point*

* Once on IM module and report option is selected the system should redirect the user to reports landing page.
* Once on reports landing page  then select “Locally Grown Usage Report”. The system should open the report page for the user.

 *Filters*

* Organization Level \[District for which the user has permissions to]
* Group  By option \[Dropdown box]
** District
** Area (Will included later when it becomes available)
** Site
* Date Range
** Start Date (Dates taken from Completed Production Records)
** End Date (Dates taken from Completed Production Records)

*Actions:*

* Clear Filter \[ Reset values on filter section]
* Generate Report When user click on the report is generated on the screen 
** Screen layout should look like Figma link included on comments
* Table Filter \[user should be able to filter transactions by each of the fields listed on the mockup]
* Export (File on PDF format)  
** Report layout should look like Figma link included in the comments reports
*** It should cover field criteria, used in the header section:
**** Title
**** Organization Level
**** Group By
**** Date Range
**** Power by SchoolCafe
**** Generated date
**** Pagination 

*Header*

* Organization Level
* Group By
* Date Range

*Fields:*

* Item Code
* Item Description
* -Pack Size-
* Manufacturer
* Brand
* Total Qty Used  \[Production -  Plan Plan] (use 1 as default value for now)
* Cost$ \[IM - Standard/fare market price]
* Group by subtotals 
* Total Cost $  \[Qty used x Cost$]

*Footer*

* Pagination should be present

General:

* Qty Used is 1 for now (There will another story to cover source of Qty Used)
* Fair Market Value = Total Cost (for now) there will be another story to cover the calculations
* Counts  ( Completed Production Record )
