# NXT-15016 — Acct.Mgmt - Outreach - Quick Broadcast tab - Create Campaign

## Description
As a central office admin, I would like to be able to set up a recurring broadcast (a “campaign”) to specific students and adults within my district, so that I can use SchoolCafé to automatically market a variety of district programs to an updated subset of our population.

*Summary*

In the Quick Broadcast tab of the Outreach Center, let’s add the options to schedule a campaign, allowing users to now both select their audience and schedule how often they want the letter to be sent to them via email.

We will combine this function within Quick Broadcasts so that the Campaigns tab is where the user goes to view, edit, and/or delete existing campaigns.  

*Details*

* Below the filters, display a scheduling section that allows the user to indicate whether they want to send a Quick Broadcast or a Campaign.
** If Quick: No change in the look, feel, and functionality on the page.
** If Campaign: 
*** Campaign Name - A name for the campaign so that it can be tracked and updated later (covered in a future story)
*** A button that redirects the user to the System Notifications page to edit the schedule. (NOTE: This button should be labeled ‘Set Schedule’, which may not be reflected in the mock-up)
**** Allow the user to set up an Outreach Center business process, including start and end dates, and timings.
**** Behavior: send a broadcast message out in this interval, between the start and end dates.
**** There should be a back button which takes the user back to the Outreach Center with their filters maintained.
**** Each time the letter is sent, it should go to the latest list of accounts according to the filters selected.
**** The timing should be based on the district’s local time zone.
* The next story will contain a way to view existing campaigns and delete them.
* For the rest of the page:
** Change the ‘Quick Broadcast’ tab title to ‘Broadcast Message’
** Change the ‘Campaign’ title to ‘Campaigns’
** Edit Template button should still be shown in any scenario.
** Footer message can be hidden when Campaign is selected.
** Create Campaign button: When the Campaign Section is selected, the Broadcast Message should say ‘Create Campaign’ instead.
** This button should be disabled until the user sets the schedule (may not be reflected in mock-up).
*** When this button is clicked:
**** Save the campaign, which will perform this operation according to the selected frequency scheduled in System Notifications (getting the latest filter results each time)
**** Reset the filters. 
**** Display a toast message: "Campaign created successfully\!"
** Add ‘Campaign Name’ column to the History (shown by default with the other columns). Broadcasts that are sent out as part of a campaign should also display in the History tab.

## Acceptance Criteria
Verify that the user can now choose between a Quick Broadcast and a Campaign

Verify that when Campaign is selected, the user has scheduling options as described above

Verify that a Campaign can be successfully created with these options

Verify that a Campaign continues sending the message according to the Frequency (interval specified)

Verify that this Campaign does so with a refreshed list, according to the same filters

Verify the rest of the page details (footer, buttons, tabs) are updated accordingly

Verify that campaign messages that get sent are also displayed in the History tab. 

Verify that ‘Campaign Name’ column is added to the History tab as well.
