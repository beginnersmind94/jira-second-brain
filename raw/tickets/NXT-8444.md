# NXT-8444 — MP > Menus - enhancements for Menu screen based on current behavior -UI

## Description
1. When calculating the counts with Choose > 1, the total and percentages are not correct. The total should simply be the sum. Example: Choose = 2, and number of meals = 100. This means 200 total selections. With 2 Items, each at 50, the total should be 100 (currently displayed as 200). The percentages would be 50/200 or 25% each, and the total percentage should be 50.

2. We should disable the capability to remove a MIC from the MIC list on the Menu. If the user clears or sets the Choose number to 0, we should force the value to be 1.

3. When adding a new Menu, please mark it as Active by default.

4. After adding a Menu, please refresh the screen so the user can continue working with the newly created Menu.

## Acceptance Criteria
1. Counts and percentages with Choose > 1 are correct
2. MIC cannot be removed from MIC grid
3. Creating Menu is Active by default
4. User can directly work with a Menu after creating it.
