# NXT-36449 — Monitoring Scheduler – P1 Monitors Tab

## Description
As a SchoolCafé admin, I want to be able to view, schedule, edit and delete monitoring jobs so that I can proactively monitor the application and ensure higher customer satisfaction.

*Flow:*

* User logs in and browses to Support > Monitoring Dashboard > Job Scheduler
* User views a list of scheduled monitoring jobs
* Optionally, the user can create a new scheduled job
* Optionally, the user can edit an existing scheduled job
* Optionally, the user can delete an existing scheduled job
* Optionally, the user can view the output of the scheduled job
* Optionally, the user can view the history of completed jobs

*Notes:*

* This story is based off the System Configuration Notification page and is following that page’s flow, layout and functionality
* We are only changing the appropriate fields as necessary, so when in doubt, copy what that page does
* This page is the user front end to managing scheduled jobs
** These jobs are created in other stories and not covered here
** Those other stories will include requirements to show their output on this page
* This page follows the UX of the System Configuration Notification page with the exception that it has 2 tabs:
** Tab 1 is a history report of completed jobs
** Tab 2 is basically the same as the System Configuration Notification
* 
* On Tab 1, note that the design is based upon the Export Monitor
** A filter section at the top to filter the grid below
** A grid section that shows completed jobs
* 
* On tab 2, note that it will have the following key sections and functionality:
** A + New button that is used to create new scheduled jobs
** A filter section at the top to filter the grid below
** A grid section that shows scheduled jobs
** An edit button on each row
** A delete button on each row
** A history button on each row

** Card should look exactly like the cards on Item Management > Reports
** Card title: Job Scheduler

** On click, open the Job Scheduler page

** Title: Job Scheduler
