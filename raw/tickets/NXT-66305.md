# NXT-66305 — API - Include other GTINs / Product Code in Multiple GTIN Search Criteria

## Description
_(none)_

## Acceptance Criteria
Navigation: DB Enabler

Changes Required: Applicable for Cybersoft Data Source Only

Add a new key as SearchKey_GTIN

Introduce a new sql job that update  search key based on the following fields where the searchKey_GTIN=NULL

Each Key is separated by Pipe Symbol

# GTIN Code (gtin)
# Purchase Unit GTIN (gtincode)
# Inventory Unit GTIN  brokenunitgtincode
# Product Code (productcode)
# Stock Item Product Code (stock_productcode)
# Vendor Item Number (vendoritemnumber)
# Vendor UPC (vendor_upc)
# Vendor Broken Unit UPC (vendor_upcbroken)



*Changes in in Search API:*

The Multi GTIN Search should be mapped to this new field (SearchKey_GTIN) for Cybersoft source, while USDA Consolidated remains unchanged

Include the SearchKey_GTIN value in API response 

*Changed in View API:*

Include the SearchKey_GTIN value in API response 

*Changes in Update API (Cybersoft):*
If either of the following field is modified, then the search key should be amended. 

# GTIN Code (gtin)
# Purchase Unit GTIN (gtincode)
# Inventory Unit GTIN  brokenunitgtincode
# Product Code (productcode)
# Stock Item Product Code (stock_productcode)

*Out of scope:* The following fields cannot be amended in search key_GTIN as they are maintained at inventory module:
Vendor Item Number (vendoritemnumber)

# Vendor UPC (vendor_upc)
# Vendor Broken Unit UPC (vendor_upcbroken)
# Vendor item number
