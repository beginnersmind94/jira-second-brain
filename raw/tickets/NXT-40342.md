# NXT-40342 — Custom Reports – Add New Report

## Description
As a SchoolCafé user, I would like to be able to create and save custom reports so that I can use the information in my decision making and reviews.

*Notes:*

* The dashboard already has an + New button with some useful code, but we are completely redoing it, so I will write these requirements assuming we are starting from scratch
* This report covers creating a new custom report from scratch, but users can also create a new custom report from other “Grid Pages”
** Requirements written for + New from scratch only
** Other story will handle the + New from “Grid Pages”
* In this feature, users can create the following things:
** Custom Report
*** Linked to an existing dataset
* Future stories will include:
** Create a Report Group
*** Link reports to the report group
** Create a Dynamic Query
*** Write custom SQL to create own dataset

*Flow:*

* Open SchoolCafé and browse to Settings > Dashboard > Custom Reports
* View list of reports
* Click + New
* Create the configuration and save
* Return to the updated list of custom reports

*Requirements*:

* All these requirements are for the + New Overlay

 

*New Overlay:*

* Remove the page that the + New link goes to, but save the code b/c you can reuse a lot of it
* The new + New screen is an overlay, per the Figma
** Overlay Title: Custom Report
** Overlay should be “framework” style, reusable on other pages without any additional coding or hard coding
** Should be “plug and play” as much as possible for other modules
** Overlay will use a Stepper control like on the Imports & Exports, per Figma

 

*Stepper:*

* Save state on screens
** When using Previous and Next to move back and forth, don’t make the user reselect / redo things, keep state

h3. Selection Screen

* Fields:
* There is a hidden field for this page:
** Report Type = Custom Report
** Don’t need to show it to the user, but make sure it is saved to the database
* Name
** Same as is already there
** Textbox
** Required
** Always show this field
* Description
** Multi-line textbox
** Optional
** Always show this field
* Owner
** Textbox/list
** Required
** Values: user accounts for the district
** Use our usual single person select user textbox or list
** Default to the person who is logged in
** They can change to someone else
** Only one person can be the owner
** The owner is the one used for “My reports”
** Always show this field
* 
* Module
** Single select drop down list
** Required
** Already exists
** Add missing modules
** If coming from a grid screen, code should pass in the module and it should auto select
* Data Source
** Single select drop down list
** Required
** Values: List of data sources preconfigured by dev team
** Filtered by Module selection
** If coming from a grid screen, source screen should pass in the necessary API information and name in order to auto select the data source
* Contains Personal Identification Information (PII)
** Toggle
** Default No
** If No, it has no impact on anything
** If yes, then it impacts the visibility of the report and the security of the output file
*** If yes, only persons in the PII role can view the View Report in the View Report Page
*** If yes, the file must be encrypted or password protected when it is generated
*** These notes will be repeated in the other stories where needed
** 
** If they select PII = Yes, you need to check the user’s roles and see if they are in the PII Role
*** If they are in the PII role, then allow them to continue normally
*** If they are not in the PII role, disable the Next button and show them a message “Only users in the PII Role are allowed to create custom reports with PII data.”
*** This will essentially block them from creating a report that has PII data
* Cancel

h3. Security and PII

* Security
** This drawer uses the Add permission for the Custom Reports
** Custom Reports Add role permission is required to use this page; also allow Cybersoft Admins
** Don’t show this drawer in Roles and permissions CRUD tables since it uses the Add permission
* 
* PII Security Trimming
** If on the selection page, they select that the data contains PII data, then you must also verify that the user is in the PII Role
** If they are not, then they cannot move past the first tab
** If they are in the role, then everything is normal
** Therefore, if they select PII = Yes on first tab, then they must be in both the PII role and have Add rights to this drawer
