# NXT-54658 — PEI - Auth - Enforce Roles & Permissions

## Description
We need to ensure that PrimerEdge Insights (PEI) FO Command Center is only usable by users in a role with access so that we don’t have unauthorized usage

*Requirements:*

* Put this code on the FO Command Center page
* Ensure that the user is in a role that has access to the Command Center
* If they do not have rights:
** Do not render the page
** Show them the access denied message on the page like in SC
*** Copy same one from existing SC page
* 
* Note:
** All users can see the dashboard page since it doesn’t have anything important on it
** Therefore, we only need to enforce security on the Command Center page

## Acceptance Criteria
* Users cannot visit the Command Center page if they do not belong to a role that has been given rights
* They are shown the access denied message instead of the page content
