# NXT-23322 — IM - Item Category Maintenance

## Description
As a user a need to a way maintain  current Item Categories to be used  in order to keep consistency thru process assignment of Item Categories as well as to keep up-to date information

## Acceptance Criteria
This story is done when:



*Entry Point*

* A  Item Category option is added  under IM configuration  area
* Option Item Category

*Result Grid*

* Should display fields as mockup above
* User should be able to search and sort fields on result grid 
* User should be able to edit Item Category Description only
** Only for user created item Categories (e.g Local). -CN cannot be edited or deleted
* User should be able to deactivate Item Category ONLY if Item Category is not used within recipe or Item
* User should be presented with a confirmation message
“Are you sure you want to deactivate this Item Category” is Yes proceed
with deletion if No cancel process and return to item configuration page

*Actions*

* *ADD* button \[When clicked a Slide Out Card is displayed]
* Fields on Slide Out Cards
** Data Source \[Dropdown box that list available data sources to the user]
*** Is a Mandatory field
*** Only one Data Source can be selected
** Item Category Description \[Free text box]
*** Is a Mandatory field
*** Only can be added to user/district created data source (e.g. Local)
** Item Category Code \[Free text box]
*** Is a Mandatory field
*** Only can be added to user/district created data source (e.g. Local)
* *SAVE* 
** Validation occurs
*** Item Category Description is Mandatory and no duplicates are allowed
*** Item Category Code is Mandatory and no duplicates are allowed
*** If the Item Category  Description or Item Code already exist warn the user with a message “Item Category Description / Code already in use, please use a different  description / code”
*** One Data Source is Selected (Mandatory)
*** The newly added Item Category Description and code are listed on the result grid
** *X icon*  \[When Clicked the Slide Out Card is close and transactions are canceled]
** Pagination should be present

* *Export* \[PDF Format]
** Report Title  \[Item Category Configuration]
** Should cover same fields as the result grid \[Item Category Description, Item Category Code, Data Source Name]
** Header
*** Updated by \[User Name]
*** Generated on \{System date/time]
** Footer
*** Power by SchoolCafe for \{School District Name]
*** Pagination should be present

*General:*
Once user selects configuration a po-up window is presented to the user. This pop-up windows asks for data source and  configuration option desired. When user click/press apply button then system re-directs the user to appropriate configuration area
