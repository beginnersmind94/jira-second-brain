# NXT-23963 — INV - Reports - Price Details - Reconciliation

## Description
As a Site Manager or Food Service Director I want a report to pull the Price Details of Reconciliation transactions to see how items changed as a result of Physical Inventory.

*Note:* The API for retrieving batch details can be used regardless of transaction type. However, the APIs for retrieving each transaction type need to be created. 

* This story is only for retrieving *Reconciliation* (Physical Inventory) transactions

*Requirements*

* Display the Price Details Report ([NXT-20418|https://primeroedgenext.atlassian.net/browse/NXT-20418]) when only ‘Reconciliation’ is selected for Transaction Type
* Load the Reconciliation transactions for the selected date range
** there should be one transaction for each reconciled inventory
** display Reconciliation Transaction # (even if it is otherwise not customer visible currently)
** display oldest transactions first

*Report header shows* : Site name, Transaction Type, Transaction#, Actual Transaction date.
*Item details table shows:* 

* Price detail information is grouped by Item #
* columns are Batch#, Received Date, Quantity, Price, Line total. Item total for each item
* Transaction Total at the end as a last row

*We have another option to generate the report for only items with multiple batches involved* : if selected only pull the transactions and item involving more than one batch. 

* This can be a separate story if it makes sense
