# NXT-130 — Configuration - POS Periods - [Already in System] - Link to System from POS

## Description
As a Central Office Admin I want to set the period billing and claiming periods for POS. Periods control the date range of when sales transactions can be created or modified. Period Frequency are set for a fiscal year and can be weekly, bi-weekly, semi-monthly, or monthly. A periods The most common is monthly. The user can preview the periods and change or combine periods if wanted. Periods are contiguous and cannot overlap.

Only a link to the System Page

## Acceptance Criteria
POS Periods is launched from the Configuration from the left panel
POS Periods is reached through the tabs of the Configuration page.
****The Periods page in System configuration page is launched.****
There is a current implementation is system


The user can pick the Fiscal Year and the day and date it ends on.
The user can pick the frequency (Weekly, Bi-Weekly, Semi-Monthly, Monthly.
For Weekly and Bi-Weekly the user picks the day the period end on M-F
For Semi-Monthly the user picks which day the period ends on 14, 15, or 16
The names for the periods are Auto generated
When selected the period name contains the date range
The user can preview the periods before accepting them
The user can split and merge periods.
When a Period is split a new period is created with an End date of the Period it was created from and the End and Start dates of split period and new period are blank.
When a period is merged the end date of previous period set to end date of deleted period.
Periods are contiguous and don't overlap.
When created the period status is open.
The previous fiscal years periods configuration can be copied
