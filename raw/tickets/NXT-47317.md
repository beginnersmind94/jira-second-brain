# NXT-47317 — Files – View History

## Description
As a SchoolCafé admin, I would like to be able to View the audit history of File import configuration changes so that I can see who made a change and when.

*Note:*

* We already do audit / history tracking a bunch in System team, so please copy what they are doing
* Do not reinvent this, use the framework they have created
* This is the backend part of File View History and needs to be done before the front end part

*Requirements*:

* For Add New and Edit File Import Configurations, do the following
* When you save the configuration on the review screen, do the following:
* When doing Add New:
* Save all the label / value pairs from the config screens to the history table
* Use the multi-level logging from System
* Fields to save:
** Pre-Upload
** List these fields as children:
*** Import Type
*** File Format
*** Any other field on the screen (ex: DC imports)
** Field Mapping
*** List each mapped field as a child of Field Mapping same way you do for API Config
**** For any that have code mapping, list the code mapping pairs as a child of the mapped field
*** Ex: 
**** SchoolCode
***** 1101 > 01
***** 1102 > 02
** Review
** List these fields as children
*** Import Name
*** Enabled
* Be sure all fields for all import types are included
* 
* When doing Edit:
* Record in the same format as Add New anything that was added, edited or removed
* Do not record anything that was not modified
* 
* When a user deletes a configuration, record the deletion in the history log as well
** Even though user won’t see it, Dev team can go see who did it when in the database in case there is an issue



* For File Imports tab’s grid, do the following:
* In the ellipsis, add a button to the actions:
** Title: View History
** On click, open the History drawer
* 
* Create a new drawer
** Title: File Import History
** Size: same as in other system screens
** This is a rea-only screen, no edit
* 
* Use the components created by System team to display the audit history
* On page load, show the history of all Add & Edit actions for that specific File import

## Acceptance Criteria
* All additions, edits and deletes are recorded in the history log for File Config Imports
* History log table uses the framework created by System team
* When a user deletes a configuration, it is recorded in the history log
* User can view the changes in the view log
