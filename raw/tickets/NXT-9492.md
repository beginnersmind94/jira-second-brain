# NXT-9492 — MP > Menus > edit Menu screen - add Copy feature

## Description
As a Menu Planner
I need to be able to copy Menus
so that I can work quickly.

Add a “Copy…” button near the “Save” button that opens a popup like this:

!image-20210104-215945.png|width=1193,height=593!

which allows the user to change the copied Menu Name, Meal Pattern, Meal Type and/or Site Group. The user can also select which (appropriate) Serving Groups the copy should have. (Note that these may differ from the source Menu.) One or more serving groups must be selected, error message “Please select one or more Serving Groups“.

On Cancel, abort the operation and return the current Menu

On Copy, make a copy of the Menu and navigate to the copy:

* if the Menu name is not unique giving a message “A copy could not be created. The name must be unique.“ and keep the popup open.
* on success give message “Menu saved successfully.“ and navigate to the copied Menu

If the source Menu and potential copied Menu’s Meal Pattern and Meal Type have a common selected Serving Group (or is not changed) the resulting copy Menu should have the same groups, number of meals and Menu Item Counts for the common Serving Groups where that information was provided on the source Menu.

Where there are new Serving Groups selected (from changing Meal Type, or Meal Pattern, etc.) the resulting Serving Group meal counts will be set to 100, and the Menu Items will not have any counts.

## Acceptance Criteria
Copy button added next to the save button
which displays a popup similar to mockup
User can change the name, Meal Pattern, Meal Type, Site Group, or the selected Serving Groups
Selecting different Meal Pattern or Meal Type changes the list of available Serving Groups
Serving Group meal counts and Item counts can be copied as well as the list of Menu Items
selected Menu Item Categories and choose numbers are copied on success.
If new Serving Groups are added, they will have 100 meals and no Item counts.
