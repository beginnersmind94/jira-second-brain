# NXT-71522 — K12 - Ins - Inventory Turnover Rate Card

## Description
*I am a* district user
*and I want to* view the Inventory Turnover Rate card on the K12 Insights screen
*so that I can* monitor actual performance for inventory turnover across my selected sites and date range.

----

h2. *Notes*

* This story covers the *Inventory Turnover Rate KPI card only*.
* KPI cards on the mobile Insights screen can include up to 5 pieces of information:
** Name
** Actual Value
** Expected Value
** Trend Line
** Border Color
* The *Inventory Turnover Rate* card:
** shows Name
** shows Actual Value
** shows Expected Value
** does not show Trend Line
** shows Border Color
* Inventory Turnover Rate is a *number-based KPI*.
* *Important:* This KPI currently only works for *PrimeroEdge (PE)*.
** SchoolCafé (SC) is not yet fully integrated with the Insights dashboard for this KPI.
** QA should test this functionality using *PE districts only*.
** A future story will address platform-based visibility (PE vs SC).

----

h2. *Out of Scope*

* Changes to site filter behavior
* Changes to date range filter behavior
* Other KPI cards

----

h2. *User Flow*

# A user logs into the *K12 Insights* mobile app.
# The user navigates to the *Insights* screen.
# The user optionally adjusts:
#* Site filter
#* Date range filter
# The user taps *Apply*.
# The screen refreshes and displays KPI cards including *Inventory Turnover Rate*, if enabled.
# The user may open the *configuration drawer* and:
#* show or hide the Inventory Turnover Rate card
#* reorder the card
# The card updates based on the selected filters and saved configuration.

----

h2. *Requirements*

h3. *1. Add Inventory Turnover Rate Card*

* Add an *Inventory Turnover Rate* KPI card to the K12 Insights mobile screen.
* The card must display:
** Name
** Actual Value
** Expected Value
** Border Color
* The card must *not display a Trend Line*.

----

h3. *2. Card Formatting*

* Inventory Turnover Rate must be displayed as a *number*.
* Formatting must be consistent with the corresponding Insights dashboard KPI formatting for this metric.

----

h3. *3. Border Color Behavior*

* The card border color must display:
** *Green* when performance is good
** *Red* when performance is bad
** *Grey* when no comparison or benchmark status is available
* Border color logic must align with the desktop Insights implementation for this KPI.

----

h3. *4. Filter Responsiveness*

* The Inventory Turnover Rate card must update when the user applies changes to:
** Site filter
** Date range filter

----

h3. *5. Configuration Drawer Integration*

* The Inventory Turnover Rate card must appear in the *configuration drawer*.
* The user must be able to:
** show or hide the card
** reorder the card
* The card’s visibility and order must persist consistent with existing mobile app behavior.

----

h3. *6. Platform Constraint (PE Only)*

* The Inventory Turnover Rate card must function for:
** *PrimeroEdge (PE)* districts only
* The card should not be validated for SchoolCafé at this time.

----

h3. *7. Performance Requirement*

* The card must load efficiently on mobile.
* Applying filters and loading KPI cards must not introduce noticeable delay.
