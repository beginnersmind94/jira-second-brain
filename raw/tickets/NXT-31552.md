# NXT-31552 — Add Tray Detection Failure Notification to Cart Screen

## Description
As a cashier, I want to be notified when the Edge device does not detect a tray properly after 3 times so that I can take appropriate action to align it and trigger the camera detection again.

*Application*: POS

*Summary*

We need to incorporate Amniscient’s enhancement for tray detection.  At the time of service using the transaction screen, the Edge device detects if the tray does have food and if not sends a “No data (empty array)”.  This story covers what to do when it receives these 3 consecutive times which is it will notify the cashier of the problem.

*Flow:*

* While cashier is using POS to sell food to students:
* Detection cycle begins when student walks up to cashier
** Cashier selects student
** Edge detection triggered
** Edge does not detect food
** Edge notifies POS that food is not detected
** POS “records” that food not detected (new)
** Transaction is completed
* Detection cycle with no detection happens a 2^nd^ time in a row
* Detection cycle with no detection happens a 3^rd^ time in a row
* POS notifies the cashier in a message that food detection has failed 3 times in a row (new)

*Assumptions:*

* This was completed in classic and you can copy the code from it
* Get with Saketh on the classic code
* You can write code without waiting on Amniscient for this
* You will need to write code to track these failures in code

*Requirements:*

* Take action on Food Not Detected results
** If receive the “No Data” message, “store” it somehow in code
** Logic for when get the “No Data” message:
*** Add +1 to the NoDataCount
*** Check if NoDataCount >= 3
*** If Yes:
**** Display warning message (see below)
**** Set NoDataCount = 0
*** If No:
**** Do nothing
* Update to existing Edge Results notification
** When receives existing results message, at the end of the current logic, add a NoDataCount = 0 statement to reset the not detected count
* Show message
** When NoDataCount >= 3, the cashier needs to be notified (see above)
** Open a popup message with text and a close button
** Text: Three consecutive Food Not Detected messages have been received.  Please ensure trays are placed in the proper location.
** Allow them to hit OK or close button and close the popup
* Security – no additional requirements necessary

*Mockup*:

* None

## Acceptance Criteria
* If we receive the “No Data” message, we “store” it and count it
* If we receive the “No Data” message, we count it up to 3 and then displays the message
* If we receive the “No Data” message, we count it and if less than 3 do nothing
* If we display the message to the cashier, it resets the count to 0
* If we receive a normal detection results, it resets the count to 0
* The user can close out of the message return to work with the click on one button
* Existing code continues to work
