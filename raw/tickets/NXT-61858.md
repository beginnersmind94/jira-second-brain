# NXT-61858 — IM - Item Onboarding - Import File Validations

## Description
As a district admin using the Import file to insert my data for my items into the onboarding tool, I would like for there to be a exception file that provides me information about what issues are wrong

## Acceptance Criteria
* Validations for field inputs
** Item Type (Food or Non Food)
** # of Inventory Units (Pack Size Info Section Enabled)
** Is this a Direct Menu Item? (Menu Item Info Section Enabled)
* Conditional Validations
** Non- Fluid/ Fluid (ONLY for Food Items)
** IF Serving Size Quantity (Integers & Decimals) is entered, validate to make sure Serving Size Unit measure is present
*** ONLY for Food Item
** IF Serving Size weight (Integers & Decimals) is entered, validate to make sure Serving Size Weight Unit is present
*** Only for Food Item
** Require both sets of fields to be present if user enters Menu Item Info (Menu Item Name)
* ANY fields that are required with a Yes/No, allow blank to indicate NO
* No contribution selections does not mean it is complete 
** Currently, system is marking any items that come in as contributions are complete even when nothing is indicated
* No other validations are required for an item to come in with Partial Data
** Partial is spelled incorrectly on the tool
