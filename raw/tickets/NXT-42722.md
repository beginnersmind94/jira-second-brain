# NXT-42722 — GDSNConnect - Additional Field Mapping (Meal Credit Mapping)

## Description
_(none)_

## Acceptance Criteria
The user should be able to see the following values in UI when the upload file has the following values:

# *Meal Credit:*

JUICE_FRUIT -  tag it as Fruit in Meal Contribution section, if an menu item is tagged, enable the check box as Fruit Juice at Menu item as well

MMA_ Extra tag it as MMA in Meal Contribution section only
MMA_Grain tag it as MMA in Meal Contribution section only


GRAIN_DESSERT- tag it as as +Non Whole Grains-Rich in Meal Contribution section+, if menu item is tagged, enable the check box as Grain Based Dessert at Menu item as well


 GRAIN_WHOLE_RICH_DESSERT - tag it as as +Whole Grains-Rich+ in Meal Contribution section, if menu item is tagged, enable the check box as Grain Based Dessert at Menu item as well

PLAIN_DESSERT tag it as as +Non Whole Grains-Rich+  in Meal Contribution section, if menu item is tagged, enable the check box as Grain Based Dessert at Menu item as well



 JUICE_VEG_DG - tag it as Dark Green veg  in Meal Contribution section and enable Veg Juice check box at Menu item as well
 JUICE_VEG_RO - tag it as Red Orange Veg in Meal Contribution section + enable Veg Juice check box at Menu item as well
 JUICE_VEG_OTHER - tag it as Other Veg in Meal Contribution section + enable Veg Juice check box at menu item as well
 JUICE_FRUIT -  Tag it as Fruit in Meal Contribution section +Fruit Juice Flag check box at menu item as well (Edited)

*Component Credit Calculations:*

the meal credit should be round down to nearest value as attached in the excel and the result value should be shown in the UI against the list of possible values as listed

If the range of values fall below or the above the ranges defined in UI, pls ignore them
