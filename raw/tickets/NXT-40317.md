# NXT-40317 — eOrder Format – Amfire

## Description
As a SchoolCafé user, I would like to generate an eOrder in Amfire format so that I can electronically order food from vendors that require that transmission format.

*Notes:*

* We are porting over this page: [https://test.primeroedge.co/Inventory/EOrderDetails.aspx|https://test.primeroedge.co/Inventory/EOrderDetails.aspx] from Classic
* We need to ensure that we fully support each of the format file types
* We will do a separate story for each format file type
* This one is for Amfire 
* There is no front end required for this story

*Flow:*

* Open SchoolCafé and browse to Inventory > Orders > eOrders
* User clicks Generate eOrder button and the overlay opens for them
* User selects a vendor configured to use this file format from the Add New screen
* User views list of orders, select the ones they want
* User clicks Generate eOrder button
* System formats the order using this file format
* System drops the file in a specified location and updates the FTP framework with the necessary information so that it can send the file
* User redirected to dashboard screen and grid of eOrders updates with new one

*Requirements*:

* Using the code from Classic that already does this same thing, write the code to generate eOrders in this file format
** Note that any necessary meta data is created in the eOrder Configuration process
* Ensure that this code is available for the EOrder generation process
** When a user generates an eOrder, it will grab the code to process the orders into this format and then pass the file to the FTP server to send

## Acceptance Criteria
* When a user generates an eOrder, it is able to generate a file in this format correctly
