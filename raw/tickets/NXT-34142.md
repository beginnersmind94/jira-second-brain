# NXT-34142 — Team Hub - Perform Full Push & Pull of Messages

## Description
At various points in the mobile application, it will need to do a full push / pull and reconciliation of messages with the server so that the app stays in sync with the server.

 

*Flow:*

·         Upon login, or

·         Upon Refresh button click in Messages screen

·         Full Push / Pull is called and app messages are reconciled to server’s messages

 

*Background:*

·         This code is called from the login screen and the messages screen

·         Put here as a separate story so can be reused for both screens

 

*Requirements*:

·         Fields: None, backend code only

·          

·         Source triggers this code to run:

·         App performs the following actions asynchronously and in the background so as to not impact app UX behavior and performance:

·         Performs a full push of any inbound or outbound messages that have pending actions on them using Messaging API

o   Do this first so as to update the server with any pending status updates, pending sent messages, etc. as the server is the one version of truth

·         Then perform a full pull of all Inbound and Outbound messages from the server using Messaging API

o   Any pending changes would have been sent in the push, so can pull fresh from the server to get full list

·         App updates local DB and reconciles to make sure in sync with server

o   The main point here is we want the local app to reflect what is on the server

o   If you want to do a full overwrite or incremental is a technical decision, just make sure the local looks like the server

·         App starts the Messaging Load Timer

o   Timer is counted in minutes

o   We will start with 2 minutes

o   Please make the number easily configurable /changeable, preferably not hard coded

o   Messaging Load Timer is explained more fully in the Background Refresh story

·          

·         Security – already covered in previous stories

## Acceptance Criteria
·         App performs the following actions asynchronously and in the background so as to not impact app UX behavior and performance:

·         Performs a full push of any inbound or outbound messages that have pending actions on them using Messaging API

·         Then perform a full pull of all Inbound and Outbound messages from the server using Messaging API

·         App updates local DB and reconciles to make sure in sync with server

·         App starts the Messaging Load Timer

·         After complete, the app has the same messages as the server
