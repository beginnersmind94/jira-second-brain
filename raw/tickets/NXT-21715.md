# NXT-21715 — Logging Report

## Description
As an administrator, I want to be able to see the logging table so that I can determine which terminals are having errors and what the end users are doing with the FFD settings.

*Application*: Face and Food Detection Website

*Flow:*

* Browse to PrimeroEdge and click the link for Food and Face Detection
* Browse to the Reports
* Browse to Logging Report
* View data

 

*Assumptions:*

* There was a user story for logging from ExpressPoint ([https://primeroedgenext.atlassian.net/browse/FFD-49|https://primeroedgenext.atlassian.net/browse/FFD-49|smart-link] )
* It is assumed that logging was done per the story and that there is a database table with this information
* It is assumed you will be able to pull the data from that table

 

*Requirements:*

* Create a new reports page with the title Messages and Logs
* In the menu navigation, under Reports:
** Create a new menu link under Reports named Messages and Logs for the new page
*  
* All fields are read only, there are no action buttons except grid behaviors and hyperlinks
*  
* Grid of log entries that come from the terminals
** Usual sort, paging, filtering, export to excel functionality
*** default sort time descending (most recent first)
** Columns:
*** Site (name)
*** Terminal (name)
*** Date / Time – from the date time stamp of when the message was created
*** Message – show the error or message text
*  
* Filters for the grid
** Site
** Terminal
** Date / Time Range – like filter on other pages
** Message
*  
* Security and roles:

o    Copy from Accuracy Report, do exactly as there

* Unauthenticated users cannot view screen

 

*Mockup*:

* Should look the same as Accuracy Report
