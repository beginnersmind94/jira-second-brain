# NXT-25482 — Accountability - Transactions - Adjustments - Payment Adjustments report

## Description
As a Central Office Admin I want to see what adjustments are being made to accounts and sales so I can see if anything is being done incorrectly.

As a Site Manager I want to see the same, but for the sites I have access to.

*requirements*

* Add Report Options for ‘Serving Site’ or ‘Enrollment Site’ for the Payment Adjustments report to use.
** Serving Site
*** Default selection.
*** It will look for adjustments done for payments at the selected site(s) and return data.
** Enrollment Site
*** This looks for adjustments done for students who are enrolled at the selected site(s) and returns data.
** Think of it this way - it relates to students visiting another site and eating a meal there.
*** Student 12345 is enrolled at Site B. They visit Site A and eat there. An adjustment is done on this transaction.
*** The adjustment will show if the report is run for Site A if Serving Site is selected.
*** The adjustment will show if the report is run for Site B if Enrollment Site is selected.
* Add the Payment Adjustments report as an option to the Adjustments page:
** Use the search criteria to pull back any ‘sale adjustments’ transactions that have been performed which match.
** The report shows the following information:
*** *Serving Site*
**** Code + Name of the site the sale was recorded at.
*** *Enrollment Site*
**** Code + Name for the student’s enrollment site on that date.
*** *Account Name*
**** Firstname + Lastname
*** *Account ID*
**** Account ID #
*** *Transaction #*
**** The transaction number of the sale inside of it’s session.
*** *Payment Method*
**** The method code for the payment - cash, card, etc.
*** *Payment Type*
*** *Payment Date*
*** *Adjustment Date*
**** Full datetime for when the transaction was performed. Time included.
*** *Original Amount*
**** $$, 2 decimals.
*** *New Amount*
**** $$, 2 decimals.
*** *User*
**** Firstname + Lastname of the user who performed the adjustment
*** *Comments*
**** If a comment was entered when the adjustment was done, show that string.
** Show a *total* for the *Amount* column.
** Sort order:
*** Enrollment site, Adjustment date
** Report should show the typical headers and footers:
*** Generation datetime and user.
*** Report name.
*** Search criteria.
*** Footer about the report by SchoolCafe.
