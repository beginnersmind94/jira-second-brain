# NXT-11763 — Eligibility - DC File Search - Precedence, Effective Dates, Error Messaging

## Description
As a district admin I want manual DC matching (via File Search) to make use of DC Precedence orders, effective dates from the file, and to tell me if there’s an issue.

*mockup*

n/a

*requirements*

* Update the DC File Search page to comply with DC Precedence ordering.
* Update the Matching on that page to make use of the effective date from the file.
** Example: File has an effective date of 8/1/20. Matching a student to this should backdate the eligibility start to 8/1/20. (Or to the start of their enrollment records, whichever is greater.)
* User needs to receive feedback for the above.
** If a match is not going to happen due to precedence ordering, tell the user this when they are shown the scorecard.
*** This match will not extend eligibility to the student as the current eligibility is higher in the precedence order.
** Include the effective date in any success messaging.
** Provide an error for the user if they attempt a blank search.
*** Update the error text: At least one field must have entered in order for a search to return results.

## Acceptance Criteria
User is informed of any precedence issues that will result in no extension of eligibility.
The file matching method takes DC precedence into account.
The file matching method uses the effective date of the DC file to backdate eligibility.

User receives feedback if they haven't entered anything into the search field.
