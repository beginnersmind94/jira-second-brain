# NXT-5962 — System - Periods - Allow user to Split and Delete Periods

## Description
As a a district financials administrator, I would like to be able to specify my district's fiscal year and view or manage periods within that year, so that I can view reports and claims for the right date ranges.

+Summary+:

* Allow a user to split period details and delete them when previewing periods, before AND (in regards to deletion) after creating periods.

+Details+:

* In the Create Periods section of the Generate Periods page, allow the user to perform several actions to update previewed or previously saved periods:
** Delete individual periods
*** Period row should be deleted. Remaining future period dates within the fiscal year should adjust accordingly so that the whole year is covered.
*** Show a Y/N confirmation when user chooses to delete a period row. “Are you sure you want to delete \{\{PeriodName}}?”
*** At least one period has to remain and must span the whole Fiscal Year.
** Split periods
*** New period row should be created with a start date that is selected by the user and an end date that covers the previous end date.
*** Previous period should have a start date that ends 1 day prior to the new (split) period’s start date.
*** Future periods should be unchanged, unless the user changes the new (split) period’s End Date. In this case, we should adjust future period dates accordingly when the user saves the change after splitting.
*** Period Names may need to be automatically updated by the system. (see Classic for example) They must be unique at a Period Type level.
** Deleting & Splitting Periods can only be done for current or future dates.
* When the user selects the ‘Create Periods’ button, allow them to commit the changes and update the period data for the selected fiscal year.

## Acceptance Criteria
Verify Create Periods section can be accessed on the Generate Periods page when: Fiscal Year dates are set, Period Configuration are set, and Preview Periods button is selected.

Verify that previewed period details display in a table that matches selections from Period Configuration section.

Verify user can modify any Period Name and that it must be unique

Verify user can modify an End Date for a period and update it, and that updates all future date ranges so that the year is covered.

Verify that user can split periods (where Start Date =/= End Date) by selecting a new start dates, so that a new period is created. Verify that this updates all future date ranges so that the year is covered.

Verify user can commit the updated details by clicking the 'Create Periods' button.

Verify that the periods are then saved into the database.
