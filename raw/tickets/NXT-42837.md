# NXT-42837 — DataExchange Import – Fixed Width – Custom Formats

## Description
As a SchoolCafé Admin I want to import fixed width files of any format so that I don’t have to try and convert to delimited file.

*Requirements*:

 

*Updates to Pre-Upload tab:*

* Predefined Format:
** Add Custom as a value
* 
* If they select Custom:
** Redirect the user to the Custom Fixed Width File-Upload screen
* Else
** Don’t make any changes to the code
* 
* Download template dialogue
** Add a download template dialogue like we do for delimited file formats
** Make sure it is linked to the custom fixed width file format excel template we want them to use

 

*New File-Upload tab:*

* There are 2 file upload sections:
* Data File
** This is the same as already exists, so just copy it
** The user will upload a fixed width data file just like before they uploaded a delimited file
* Custom File Format
** This is a new file upload concept
** The user will upload an excel file that we provided them
** The excel file is in a preformatted template with these values at a minimum
*** Name
*** Length
*** Description
* 
* Next button
** Validate the Custom File Format file if they uploaded a file
*** Verify that the file has at least 3 rows (1 header row and at least 2 data rows)
*** Verify that the file has at least 2 columns (name and length)
*** Verify that the file has text in those first 2 columns and 3 rows
* 
* Allow the user to proceed (click next) without uploading files
** If they do that, then it means they want to build it manually on the next tab by using the add and delete buttons on the source table
** This is just a shortcut for them

 

*File Mapping tab:*

* Use the same one you created for Minnesota
* Ensure they can add and delete to the source table so that they can manually build it
*  
* All the normal functionality for imports will be there for these imports:
** Import button on Imports tab
** Edit, clone and delete on Import tab
** View Log button and exception count on Import Monitor
** View source data and exceptions in View log
** Etc.

## Acceptance Criteria
* User can choose Custom format in Pre-Upload
* User can download a Custom format template
* User can upload a custom format template in File-Upload
* User can manually add and delete and customize the fixed width configuration on File Mapping screen
* The configuration is saved to the database and is usable for executing the runtime routine
* User can manually import a file using this configuration
* User can see exceptions and source data in View Log
* User should able to upload data file first, till then Upload custom file should be disabled.
* Show Message: Click + button to add a field. Put it above the Field table. When their is no fields in Mapping fields page.
* Validation should be their for add new fields for custom files.
