# NXT-35490 — District Onboarding - Allow user to access certain System module pages - Part 2

## Description
As an Onboarding user, I need limited access to multiple pages within SchoolCafé, so that I can save time by not having to communicate back-and-forth with a project manager while they use those pages on my behalf.

*Requirements:*

* Though Onboarding is done in the *Workspace* module, we need to allow users to access hyperlinks for pages that are in the *System* module, so that they are able to complete certain tasks before being able to access all of SchoolCafe. 
** Can we accomplish this by moving the Onboarding wizard pages from the Workspace module to the System module? 
** Can we duplicate the pages below in the Workspace module and just disable them after the user finishes these onboarding tasks?
*** What if we gave them a hidden, extra ‘Onboarding’ role that had access to these duplicate pages, and then removed the role once the tasks were completed?
* These are the System pages that the user needs access to in the beginning:
** Site Configuration 
** Users
** Roles and Permissions
** Manage Settings
** License Management (to apply licenses to the created sites)
** Security (to set password policy)
** Additionally we need to keep in mind:
**# Do not allow the user to navigate or view any other page except the one where they land.
**# Add a ‘Return to System Setup’ button that, when clicked, returns the user to this task selection page.

## Acceptance Criteria
Verify that the user can access the System pages above in order to complete their Onboarding tasks
