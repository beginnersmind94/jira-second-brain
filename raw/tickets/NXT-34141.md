# NXT-34141 — Team Hub - Perform Initial Data Load

## Description
After successful login, we need the app to perform a full push and pull of data from the messaging API for the logged in user and save it to the App’s messaging DB so that they can see their inbound and outbound messages.

*Flow:*

·         Open the App

·         See Splash screen and then login screen

·         Select my school district and login

·         App authenticates user

·         App asynchronously, in the background performs a full push pending messages to be sent and then a full pull of all Inbound and Outbound messages from the server using Messaging API

·         App saves changes to local DB and reconciles to make sure in sync with server

 

*Notes:*

·         User authentication is done in a different story

·         This story adds messaging to that process

·         Messaging already exists on SchoolCafé and we trying to leverage that on a mobile app, so feature set should closely stay in line

·         Understand that we do not have one centralized database used by all customers, but rather have many on-premise installations. Therefore, you may have to develop a special API that feeds a list of districts and feeds the URL for the correct district Messaging API

·         Understand that we have 2 different platforms, Classic and SchoolCafé (AKA 2.0) that you will interface with on messaging, and that although each district uses only one platform, you will need to be able to work with both

·         The actual requirements for the full push / pull of data are in a separate story because reused in other places

 

*Assumptions:*

·         This story assumes that there is a server database containing messages for each district

·         This story assumes that there is a Messaging API that can be called to provide push / pull services

·         This story assumes that you somehow are storing messages locally on the app and refers to this as the Local DB, and however that is actually done is up to the dev team

 

*Requirements*:

·         Fields: No UX fields, all backend work

·          

·         Update the login button of the login screen as follows:

·         Upon successful authentication, do the following:

·         Call the Full Push / Pull code (covered in another story) and let it do its thing in the background without affecting the user experience

·         Security – already covered in previous stories

## Acceptance Criteria
·         Upon successful authentication,

·         App calls the Full Push / Pull code asynchronously and in the background so as to not impact app UX behavior and performance

·         App updates local DB and reconciles to make sure in sync with server

·         App starts the Messaging Load Timer

·         Messages are accurate compared to the server
