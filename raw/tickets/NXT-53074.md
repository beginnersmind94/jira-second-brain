# NXT-53074 — OpenID - District Authentication - Notify Delete Button

## Description
As a Customer Support user, I want to be able to delete a district’s OpenID authentication provider so that they can use SSO

*Notes*:

* Covers delete notifications that weren’t done for original story



*Requirements*:

* These requirements apply to the System > Security page > Authentication tab
* OpenID deletion:
** If they click on an OpenID row:
*** Show message and if they confirm:
**** For each user account:
***** Remove OpenID from their list of authentication methods
***** Send a notification to their workspace inbox and their email address:
* 
* If their new preferred method is SchoolCafe Auth, run them through the password check process and notifications done earlier

*Message Text:*

Your district administrator has removed OpenID single sign on as a permitted login type. Users are no longer be able to login using their district username and password.

The permitted authentication methods by your district are: [show list like: SchoolCafe Authentication, Email Two-Factor Authentication, etc.].

Your default authentication method is: [name].

To update your authentication settings, go to the Authentication tab in your user profile.



* {color:#ff5630}-On the grid:-{color}
** {color:#ff5630}-SMS 2FA rule:-{color}
*** {color:#ff5630}-SMS 2FA cannot be the only preferred method, so-{color}
*** {color:#ff5630}-User cannot delete an authentication method if it would leave the SMS 2FA as the only remaining method-{color}
*** {color:#ff5630}-First, validate that they have more than one row-{color}
*** {color:#ff5630}-2nd, validate that the SMS 2FA is not the only remaining row-{color}
**** {color:#ff5630}-If it is, then make them select another preferred method and don’t allow them to proceed, by showing this message: Deleting this method would leave SMS 2FA as the only remaining method which is not allowed. Please add another method before deleting.- {color}
**** {color:#ff5630}-Give them an OK button to click on to close the message and do not delete.-{color}

## Acceptance Criteria
* If user deletes OpenID Authentication, then system notifies impacted users of the change
* If user deletes OpenID Authentication and the preferred auth type is SchoolCafe Authentication, then it goes through the new password process for those who don’t have one and the change password process for those who have an expired one
