# NXT-38355 — Onboarding - Tours - Add API to track users' progress for a tour

## Description
As a new SchoolCafé user, I would like to have the option to continue a tutorial where I last left off, or have it no longer appear if I have already completed it, so that I can more quickly complete our onboarding of SchoolCafé.

*Summary*

Add an API that keeps track of where users last left off regarding a tour. 

*Mockup*

N/A

*Requirements*

* When a user clicks ‘Skip’ or ‘Finish Tour’ in a Tour, have an API keep track of that. When the user returns to that page, do not display the tour for them any more.
* When a user clicks ‘Exit’ in a Tour, refreshes a page or is logged out while in the middle of a tour, have an API keep track of that. When the user returns to that page, instead of starting the tour at the initial step, have it display the last step where they clicked ‘Exit’ or navigated away. 
* Finally, integrate the APIs on the 2 existing pages where tours have been implemented.
** *Note:* If we need to implement this on one page at a time, that is fine. We will eventually have either a page or a button that allows users to review a tour on-demand, and we may also use this feature as a training tool which can be tracked (we can implement a form of gamification where we reward users who have gone through an entire ‘training’ tour).

## Acceptance Criteria
Verify that when a user skips or finishes a tour, they will no longer be shown the tour when they refresh or return to the page.

Verify that when a user Exits a tour, refreshes a page or is logged out while in the middle of a tour, they will be returned to that specific step within the tour when they refresh or return to the page.

Verify that this is implemented in as many pages at the start as possible.
