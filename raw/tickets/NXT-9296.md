# NXT-9296 — Data Exchange - SIS Import - Create a student account 

## Description
As a district I want newly enrolled students sent within the student information file to create a student account with the provided details. 

More Details: 

* Logic can determine a new student based on the Student ID provided in the file. If the Student ID is not identified we can proceed with creating a new student account with the details provided.
* -To prevent duplicate student creation, we should explore/brainstorm how to compare student names, DOB, enrollment info, guardian info, etc to see if somehow it is a same student that was somehow assigned a new ID. We do not need to stop the import of the record, however, we may (in the future) provide a quick merge functionality of the new and previously identified record. This allows us to retain any prior historical information in the case that the new Student ID is requested. This is mainly a note of future ideas for the team to review/discuss/prioritize (can be Phase 2).-

## Acceptance Criteria
1. File import process creates a new student account with the provided details IF the Student ID DOES NOT exist for a student account type.
2. File import process should not create a new student account if the Student ID exists for student account type.
