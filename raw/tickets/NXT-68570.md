# NXT-68570 — Fin - Lock GL Account Base Type After Activity Is Posted

## Description
As a district finance administrator,
I want the GL account “Base Type” (Asset / Liability / Equity / Revenue / Expense) to be editable *only* on accounts that have never been used,
so that we can correct configuration mistakes before go-live without silently changing the accounting behavior of accounts that already have posted history.

## Acceptance Criteria
h3. Account Categories

* *When* a user attempts to edit the *Base Type* of an existing Account Category:
** *If* any GL account assigned to that category has *one or more posted transactions in any fiscal year*,
** *Then* block the save and show this error message:
** *“Base Type cannot be changed once transactions have been posted to accounts under this category.”*
** If conditions are met for category deletion, 
*** Mention : “*This category has accounts with posted transactions, so you can only reassign it to a category with the same base type.*”
*** Provide the option to bulk reassign to another category
*** Inform user that if its a new category, they have to go back to the categories page and create a new category

----

h3. Individual GL Accounts Page

* *When* a user edits a GL Account and attempts to change its *Account Category*:
** *If* that GL Account has *any posted transactions in any fiscal year*, *and*
** The new category’s *Base Type* is *different* from the account’s current Base Type,
** *Then* block the save and show this error message:
** *“This account has posted transactions, so you can only reassign it to a category with the same base type.”*
