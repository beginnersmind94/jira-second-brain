# NXT-4023 — Authentication - Set Password Landing Screen

## Description
*User Story*
As a new PrimeroEdge user, I would like to easily be able to set a password that fits defined parameters and then continue on to the login screen.

*Requirements*

* When a new user clicks on the link in their email, they should be logged in and taken to a landing page where they are able to set their password.
** They should not have access to any other operations until the new password is created and accepted.
* Landing page should be simple
** PrimeroEdge logo is present, but no other menus or options
** There's a welcome message for the user ("Welcome to PrimeroEdge <User Full Name>\!")
** There's a smaller message with directions ("Please set a new password before continuing.")
*** Parameters are defined by organization’s password settings
* There are two fields present: Enter Password and Confirm Password
** Both fields are validated against the set constraints
*** "Your password does not meet the required strength. Please see the requirements above."
** Both fields need to match
* There is a button for advancing
** Once clicked, the password is saved to the account and can be used to log in
** The user is logged out and returned to the login page, where they are then required to enter their password to log in with it for the first time
*** (Optional) The user’s email should be automatically filled in if coming from this screen
* If the user exits the page without saving their password, the email link is considered expired and will no longer work correctly
** Admins can send a new one (covered by another story)
* If the user clicks on an email link that has expired, they get a different page:
** All of the fields and messages above are hidden
** There is a message present ("The link to set your password has expired. Please reach out to your Administrator to receive a new one.")
* If user clicks on an email link after a password has been set, they get a different page:
** All of the fields and messages above are hidden
** There is a message present ("You have already set your password. Please click the button below to log in.")
*** There is a ‘Return to Homepage’ button present that takes the user to the login page
*** There is a second message below the button ("If you have forgotten your password, please follow the instructions in the Forgot Password link.")
**** The words "Forgot Password" direct to the forgotten password page (covered in a separate story)

## Acceptance Criteria
* Emails with valid links take user to set password screen with all elements as described above
* Password validation properly occurs with default PE password requirements
* Password validation properly occurs with custom password requirements
* Passwords can be entered, saved, and used to log in

* Emails with expired links take user to the same screen but with a different message displaying
* Links for users who have already created their password take user to the same screen but with a different message displaying
