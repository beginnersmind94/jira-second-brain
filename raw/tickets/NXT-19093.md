# NXT-19093 — Income Surveys - Add New Survey - Comments Step

## Description
As a district admin, I would like the ability to quickly and easily add a manual income survey on behalf of a household, so that their income data is entered into our system for reference at any time.

*Summary*

Within the ‘Add New Survey’ function, add a step to help add a new survey record for a set of students by helping them to indicate any last details before reviewing submitting the survey.

*Requirements*

* Display the *Comments Step* (which is similar to the Submit step of the application wizard), with the following details:
** Language
*** List all languages that the district has added through System → Configuration → Languages
*** Allow the user to select the preferred language for the household submitting the survey
*** Default value: English
** Survey Accepted
*** Allow the user to select the date that they are accepting/accepted the survey
*** Any *past* date within the current academic year can be selected
*** Default value: today’s date
** Upload Documents
*** Allow user to upload one or more documents that will be attached to the survey
*** Normal size, type, etc. rules apply
** Comments
*** Allow user to enter a 500-character comment that will be attached to the survey
** Income Status
*** *Only* show this if the district is using the _Income Range Type_ setting value of ‘USDA Income Range’. Otherwise, hide this text.
*** According to the household income/special assistance program details entered by the user, display the proper income status (Free/Reduced/Paid) based on those USDA guidelines.

## Acceptance Criteria
Verify that in the Income Survey page, when a user selects ‘Add New Survey’, the wizard page displays

Verify that the user can proceed through the survey and access this step

Verify that the step functions as outlined in the requirements above

Verify that the user can associate a language with the survey, based on whatever languages are available in the district.

Verify that the default value for this Languages field is English

Verify that the user can select the date that they are accepting/accepted the survey

Verify that the default value is today’s date, but that any date within the current academic year can be selected

Verify that the user can upload one or more documents that will be attached to the survey, where normal rules apply

Verify that the user can enter a 500-character comment that will be associated with the survey

Verify that the Income Status for the survey displays when the district is using the _Income Range Type_ setting value of ‘USDA Income Range’. Verify that the Income Status is hidden otherwise

Verify that according to the household income/special assistance program details entered by the user, we display the proper income status (Free/Reduced/Paid) based on the USDA guidelines

Verify that the selections that the user makes on the step are retained if the user proceeds to the next step and then goes back to this previous step, or vice-versa
