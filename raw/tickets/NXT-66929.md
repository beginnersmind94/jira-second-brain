# NXT-66929 — Cafeteria Manager

## Description
As a Cafeteria Manager, I need limited read-only access to Financial Oversight functions so that I can view relevant financial information without the ability to modify critical financial data

## Acceptance Criteria
<Remove this>

*Given* I am logged in as a Cafeteria Manager
*When* I access the Financial Oversight module
*Then* I should have the following permissions:

*Dashboard Access:*

* I can VIEW the Financial Oversight dashboard
* I cannot EDIT, ADD, DELETE, or have FULL access to dashboard elements

*Configuration Module - LIMITED VIEW ACCESS:*

* I can VIEW configuration settings but cannot modify them
* I do not have FULL access to the Configuration module
* GL Code Structure: VIEW only
* Account Mapping: VIEW only  
* Import/Export Templates: VIEW only
* Version History: VIEW only
* I cannot EDIT, ADD, or DELETE any configuration items

*Transaction Management Module - LIMITED VIEW ACCESS:*

* I can VIEW transaction management functions but cannot modify them
* I do not have FULL access to Transaction Management
* Manual Entries: VIEW only
* Post from POS/Inventory: No access (cannot VIEW, EDIT, ADD, or DELETE)
* Transaction Search: VIEW only
* Void/Reverse: No access (cannot VIEW, EDIT, ADD, or DELETE)
* I cannot EDIT, ADD, or DELETE any transactions

*Reports Module - VIEW ONLY ACCESS:*

* I can VIEW all reports but cannot modify them
* I do not have FULL access to reports
* Profit & Loss: VIEW only
* Balance Sheet: VIEW only
* Site-Level Analysis: VIEW only
* Export to ERP: No access (cannot VIEW, EDIT, ADD, or DELETE)
* I cannot EDIT, ADD, or DELETE any reports

*Period Management:*

* I have no access to period management
* I cannot VIEW, EDIT, ADD, DELETE, or have FULL access to period settings

*System Behavior:*

* Most permission checkboxes remain unchecked when Cafeteria Manager role is selected
* Only VIEW permissions are enabled for accessible modules
* Master toggle shows "Enable All" by default (since most permissions are disabled)
* I receive appropriate "Access Denied" messages when attempting restricted actions
* My limited permission set is maintained when switching between roles and returning to Cafeteria Manager
