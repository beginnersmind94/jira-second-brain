# NXT-3205 — Reconciliation - Reconcile Info - Closing Balance - [Require Update as per POS Data Sync on Opening Bal]

## Description
*User Story*

As the manager I need to be able to see and modify the Closing balance entered for a session including Cash, Checks, and Cards

*Mock-up*

[https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=1137%3A22760&scaling=scale-down-width|https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=1137%3A22760&scaling=scale-down-width]

*Requirements*

* Denominations counts or amounts can be entered
* Checks and cards can be viewed. 
* A summary of all cash, checks, and cards is shown
* The user can clear all Cash entries with a single click
* The user can print/export the closing balance information
* Cards are only shown if enabled in settings.
* Limit denominations to 5 digits left of the decimal $99,999.99
* selecting a check or card entered allows editing the amount or check/card number [https://primeroedgenext.atlassian.net/browse/NXT-6746|https://primeroedgenext.atlassian.net/browse/NXT-6746|smart-link] 
* When saved session history is saved [https://primeroedgenext.atlassian.net/browse/NXT-6745|https://primeroedgenext.atlassian.net/browse/NXT-6745|smart-link] 
* Closing balance can be Exported [https://primeroedgenext.atlassian.net/browse/NXT-6747|https://primeroedgenext.atlassian.net/browse/NXT-6747|smart-link] 
* Closing balance can be printed. [https://primeroedgenext.atlassian.net/browse/NXT-6748|https://primeroedgenext.atlassian.net/browse/NXT-6748|smart-link]

## Acceptance Criteria
The denominations are listed
The user can enter denomination counts OR denomination Amounts
If the user enters a count the amount is calculated. and displayed
If the user enters an amount the count is calculated and displayed
When amounts are entered that result in a partial coin amount the user will be displayed a message "The entered amount results in a incorrect coin count. Please recount and enter the correct amount" The amount is not accepted until corrected. Example Entering $5.20 for Quarters results in 20.8 as the count for quarters.
All cash amounts and counts can be cleared with a single click
Checks entered in transactions are displayed
Cards entered in transactions are displayed
Changes can be saved
Session status is changed to CLOSED.
Cash denomination changes can be canceled (prior to saving)
selecting a check or card allows the amount and card/check number to be updated
The closing information can be printed or exported.
The session History is saved when the closing balance is saved
