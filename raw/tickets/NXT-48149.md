# NXT-48149 — Subscriptions - Dashboard - History

## Description
As a SchoolCafe user, I would like to be able to see what subscriptions have been executed so that I can know who is being notified of what.

*Notes:*

* This is the dashboard story and first part of the front end for subscriptions
* It will cover the History tab of the Event Subscriptions page
* This story relies on the backend framework story to track when event subscriptions are executed
* The objective of this story is to be able to view a list of subscriptions that have been executed

*Requirements:*

* On the Event Subscriptions > History tab:
* If the tab doesn’t exist yet, create it
** Title: History
* 
* Add a filter section to the top
** Section should be collapsible with the arrow
** Fields:
*** Modules
**** List of modules
**** multi-select
**** Default select all
*** Events
**** list of event subscriptions
**** multi-select
**** Default select all
*** User
**** Use this one if you can
*** Date Range
**** Use the reporting dates from Custom Reports
**** Default: today()
*** Apply
*** Clear
* 
* Add a grid to the page
** Grid is read-only
*** No “Add New”, edit or delete for this grid
** Grid needs sorting, default by date with most recent first
** Grid needs paging, default 10
** Columns:
*** Module
*** Event
**** Show the user friendly name with spaces
*** Distributed
**** The date / time it was sent
*** Criteria
*** Timing
*** Users
**** List of users that received the event subscription notification
**** this could be a long list, so limit the number of characters (50???)
**** Use the tooltip to show the full list on hover over for each row
*** View icon
**** On click, show the message sent in a drawer the way you do for email monitor
*** Note: don’t show Roles column, since distribution is to individuals

## Acceptance Criteria
* Users are able to see a list of subscription notifications that have been sent
* Users can see to whom and when and for what a subscription event was sent
