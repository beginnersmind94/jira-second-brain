# NXT-29867 — PR- Consolidated Plan (Prep site) actions and scenarios

## Description
As a prep site manager, There are a few actions and scenarios that need to happen for me to be able to consolidate my production plan with other site information.

## Acceptance Criteria
* Prep site must complete their own plan for the consolidated tab to appear. If prep site happens to go back to their production plan and go to plan incomplete, consolidated tab is removed
** Prep site will not have their own plan if they do not have the same menu line as the receiving site. Consolidated tab will appear when user clicks on the link from the calendar. Display on Plan tab (There is no Plan for this Menu Line)
* All receiving sites must have their plans completed before prep site can consolidate the plan (Give a Save Button once all sites have completed plans)
** Once plan is saved/consolidated, display (save and delete) buttons once page is refreshed
* If plan is consolidated at the prep site, and the receiving site modifies their plan or happens to change status to plan incomplete, do not effect the previously consolidated plan
** Display this informational message to the receiving site on the their production plan they modified to tell the user that they must contact the prep site to update manually. “The selected Prep site was previously consolidated. Please notify the Prep site of the changes made to this production Plan. Status changed to Plan Complete.”
* To display new information on the consolidated plan, user must delete the consolidated plan and the new counts will then appear.
** If the change to the receiving site is still in plan complete, display new counts
** If the change to the receiving site is plan incomplete, display Not Available (in red) for all count text boxes.
** Display error message on the consolidated “Production Plan has not been completed for the following site or sites: (Receiving Site Name/ Names)
* On the consolidated plan, prep site manager is allowed to change the counts either for their site or the receiving sites. 
** Allow textboxes prefilled (based on plans) with counts that user can change
** All counts that are “changed” would not effect the original production plans for the receiving or prep site. Changes to counts can be edited after saved consolidated plan
** User can be allowed to edit the Preparation Date (Default to the Plan/Record Date) (Allow change from Current date to Record Date)
* This will trigger Prep Site Distribution Tab to be enabled (separate story)
