# NXT-64771 — CC - AI - Schoolie Recap

## Description
As an Insights role user, I would like to see a summary of my performance generated by AI in natural language on the home screen so that I can get more understandable feedback.

h2. User Flow

* User opens the application and is directed to login screen
* On login success, user is redirected to home screen
* User sees the quick actions card for Daily Recap and clicks on it
* Daily Recap drawer opens
* App gets the district and site access and a calculated time range
* App calls WS Insights API
* System gathers the correct data and a prepared AI prompt and sends to the AI engine
* AI engine analyzes data using prompt and returns a paragraph style text summary
* App shows the summary to the user

h2. Requirements

* Ensure the screen supports dark mode

*** Check if user is in Insights role
*** If No for either, then hide the card

* Fields:

## Acceptance Criteria
h3. Quick Actions Card

* Title: “Daily Recap”
* Subtitle: “View AI operational analysis”
* Replaces the “Production Record” quick card on the home screen
* On Tap: Opens the Daily Recap screen
* Card is visible only if:
** District is licensed for Insights AI (system setting; future will check actual license)
** User has the Insights role
* If either permission check fails, hide the card

h3. Daily Recap Screen

* Name: Daily Recap
* Accessible only from the quick action card; not included in the menu system
* Permissions same as quick card
* Functions like WS Daily Recap, but with AI summary as the sole content
* Loader displayed while waiting for AI summary, with optional “waiting for AI” message
* Must support dark mode

h3. Fields and Controls

* Period dropdown (single select) with values: Today, Yesterday, This Week, Last Week, This Month, Last Month, School YTD, Combined
* Default value:
** If after 10:30 am, default to Today
** If before 10:30 am, default to previous operating day
* Combined value uses same dates as School YTD
* Icon button (View/Search/Go) next to Period dropdown
** On click: initiate AI engine call for selected time frame

h3. Data Set and Filtering

* Filter results by district and site access
* If user has access to only one site, show results for that site only
* Apply date range based on selected Period dropdown value
* API call parameters: district, sites, calculated timeframe
* For default daily results:
** After 10:30 am → Today
** Before 10:30 am → Previous operating day (weekend/holiday logic applies)
* On screen load:
** Use Period dropdown value to determine time frame
** Call WS Insights API with filters
** Prepare AI prompt and send dataset to AI engine
** Receive and display AI-generated text summary

h3. AI Prompt Behavior

* Use WS Daily Recap prompt text excluding “improvements” section
* If Combined is selected:
** Use modified prompt instructing AI to analyze data separately for Daily, Weekly, Monthly, and YTD
** AI summary must include specific verbiage for each time frame

h3. Error Handling

* If AI retrieval or processing fails:
** Display user-friendly error message (do not leave loader indefinitely)
** Log the error for troubleshooting

h3. Performance

* AI-generated summary should load within a reasonable timeframe for a smooth user experience
