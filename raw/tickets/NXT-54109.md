# NXT-54109 — FH - Menus - Serving Groups, Serving Sizes, Grade Checks for Item Display

## Description
[https://primeroedge.freshdesk.com/a/tickets/254291|https://primeroedge.freshdesk.com/a/tickets/254291]

[https://primeroedge.freshdesk.com/a/tickets/256153|https://primeroedge.freshdesk.com/a/tickets/256153]

*Current Behavior*

* Online Menus are showing items available to students in grades, when the item is marked as ‘Not Served’ on the menu.

*Expected*

* Should only see items that are available to be served to that grade.



*requirements*

* Update the Menu API to pull over the following for each item:
** Serving Groups (K-5, K-8, etc.)
** Serving Sizes
* Update the online menus to:
** Perform a check what the selected Grade falls into vs Serving Group Grade Mappings established in [https://primeroedgenext.atlassian.net/browse/NXT-54690|https://primeroedgenext.atlassian.net/browse/NXT-54690|smart-link] 
*** Example: Grade 4 is selected. 
*** Item has Serving Groups: PK, K-8. 
*** This falls into K-8 - use that Serving Group's data.
** Check to see if the Serving Size for that Serving Group is set to NOT SERVED
*** If true, do not display the menu item on the menu.
*** If false, show the menu item on the menu.





!image-20240911-130642.png|width=1716,height=616,alt="image-20240911-130642.png"!

* Example from screenshot:
** Pizza items have a serving size of NOT SERVED
** It is served to the 6-8 serving group only. 
** Someone using the online menus selects Grade 4. 
** Grade 4 falls into the K-5 Serving Group
** We would not show this item on the menu.

## Acceptance Criteria
* User can select a particular grade on the online menus, and be shown only the menu items that are due to be served to that grade.
