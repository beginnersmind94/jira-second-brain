# NXT-3225 — ** System - Sites - Site Types - Edit Site Type

## Description
As an administrator, I would like to be able to modify the details of existing site types within my district, so that I can more easily manage non-traditional site types within PrimeroEdge.



* In the Site Types page, allow the user to select an existing site type to view its details by selecting its name in the list
* When the button is selected, open an interface that allows the user (having appropriate permission) to add a new Site Type successfully using the following fields:

# ({color:#ff5630}required{color}) Site Type Description - Form field
# ({color:#ff5630}required{color}) Site Type Type - Drop-down. Default value should be ‘School Site Types’
# ({color:#00b8d9}optional{color}) Grades - Multi-select field that should allow the user to add/remove grades from the site type. List of grades should come from the database.  

* Existing field validation rules for description, as well as existing site type types can be carried over from PrimeroEdge Classic
* Within the ‘Add/Edit Site Types’ form, add an *Update* button and *Cancel* button. 
** When the field validations are successful and the user selects the 'Update' button, update the new site type details and return the user to the main Site Types list/view.
** When the user selects the ‘Cancel’ button, return the user to the main Site Types list/view.

Compared to the PrimeroEdge Classic layout (below) this will be mostly the same, with the biggest change being that we’re adding the ability to add/remove grades to the edit screen rather than the grid as was the case previously.

## Acceptance Criteria
1. Within the Site Types page, a site type's details is accessible to users with the proper role permission when they select the Site Type Description
2. When the site type is selected, the user is presented with three fields: Site Type Description (req.), Site Type Type (req.), and Grades (optional). 
3. When field validations are successful and the user selects the 'Update' button, the site type details/grades are updated and the logged-in user is returned to the Site Types main page.
4. If the validation is not successful or the description field is blank, the 'Update' button is disabled.
5. When the logged-in user selects the ‘Cancel’ button, the user is returned to the main Site Types list/view.
