# NXT-48153 — Subscriptions - Users Dashboard (My Subscriptions)

## Description
As a SchoolCafé admin, I would like to subscribe to notifications of User CRUD operations so that I can be informed if anyone makes a change.

*Note:*

* This story is leveraging the subscription framework in order to create notifications for when a user adds, edits or deletes a user

*Flow:*

* Browse to Users
* Click on Subscribe
* Subscriptions drawer opens
* Select self, users or roles to be notified
* Select criteria under which scenario to be notified
* Optionally see which users & roles are already subscribed
* -Optionally edit the notification configuration-
* Optionally delete the notification 
* Save
* Later, if one of the criterion are met, the System triggers the notification process
* System notifies by sending message to user according to notification criteria and timing
* User browses to inbox and reviews the notification

*Requirements*:

* On the System > Users page, do the following:
* 
* Add a button to set notifications
** Title = -Subscriptions- My Subscriptions
* On click:
** Open the User Subscriptions Drawer



User Subscriptions Drawer:

* Create a new drawer
** Title: -User Subscriptions- My Subscriptions
** Size: half page
* Fields:
* Grid
** Use the inline editing grid instead of the usual because would be weird to open an add or edit drawer inside another drawer
** No sorting, filtering or paging needed for the grid since will have few rows
** Grid has an Add button that will add a row
** Users can edit the grid inline
** Grid columns:
*** -Users-
**** -Multi-Select list of users that will receive the notification-
**** -Ensure user can start typing name to filter down the list-
**** -Optional if have selected a role, required if not-
**** -Auto-insert the user logged in-
**** -Note that we do not want to add notifications to users that do not have view access to this page-
***** -Therefore, if you can, trim the users list to only those who have rights to the page-
***** -If you can’t, then when setting up the notification and check if the user has rights-
****** -If they don’t, then do not add the subscription and show the user a message: Subscription not added for [name of user] because they do not have rights to view this page-
*** -Roles-
**** -Multi-Select list of roles that will receive the notification-
**** -When sending the notification, get the list of users from that role and send to each one-
**** -Optional if have selected a user, required if not-
**** -Note that we do not want to add notifications to roles that do not have view access to this page-
***** -Therefore, if you can, trim the roles list to only those who have rights to the page-
***** -If you can’t, then when setting up the notification and check if the role has rights-
****** -If it doesn’t, then do not add the subscription and show the user a message: Subscription not added for [role name] because it does not have rights to view this page-
*** Criteria {color:#36b37e}*(Get this using an endpoint passing the page name)*{color}
**** Select list
**** Values: Additions, Changes, Deletions
**** Required to have at least one selected
**** No default selected
*** Distribution  {color:#36b37e}*(These values are not going to change, Hard code these ones)*{color}
**** Select list
**** Values: Inbox, Email, Toast
**** Required to have at least one selected
**** You can hard code these values
*** Timing {color:#36b37e}*(These values are not going to change, Hard code these ones)*{color}
**** Single select dropdown list
**** Values:
***** Immediately
***** End of Day
***** End of Week
***** You can hard code these
**** No default selected
**** Required if row is checked
**** If do End of Day, schedule to distribute at 5 pm every day
**** If do End of Week, schedule to distribute at 5 pm every Friday
*** Delete 
**** tooltip: Delete
**** On click, show the usual are you sure confirmation box
**** If yes, then delete the event subscription {color:#bf2600}*(If the even subscription is deleted,  ensure all the scheduled events for this subscriptions are also deleted for all the users)*{color}
**** Apply the Users page delete checkbox from R&P page to this button
***** They must have delete permission to perform this operation
***** You can either disable the button if they don’t have it or do a check if they click on it
*** Save / Edit
**** For each row, must have at least one user *OR* role … somebody must be notified
***** the user and roles fields establish to whom the notification is for
***** users can add other people besides themselves 
**** Validate have selected at least one criteria
**** Validate have selected a timing
**** On save, register the subscription with the subscription framework
**** Note: We have this same function on the user profile page, so if you have done that one, you want to do the same function here as well
* Cancel
** On click, close the drawer and don’t save
* 
* Ensure these subscriptions show up in Profile Subscriptions section
* 
* Backend
* Setting it up:
* You must register your event subscription with the Subscription framework
** Create a unique subscription event: 
*** Name: User Accounts
*** Criteria: Additions, Update -Changes, Deletions-
* 
* At run time:
* When a user is added, edited or deleted, check to see if there are any notification configurations are set for the User Accounts
** You will get this from the Subscription framework
* Add code to the Add, Edit and delete buttons to check for subscriptions to those events
* If there are notifications, then check the criteria to see if any are met
** Criteria are met in one of 3ways:
*** A new account is manually added
*** An existing account is edited
*** -An existing account is manually deleted-
*** Excludes imports from data exchange
** You will check this by comparing the criteria against the activity you are doing:
*** Save button on Add New
*** Save button on Edit User
*** -Delete button on a User-
* If Criteria are met:
** Verify if need to send immediately or schedule for the future
** If immediately: 
*** Then prepare a notification message
*** There are no files to save, just notification text
*** Send the message to the framework for messaging with the event subscription
*** Framework will determine the notification settings for each user and use them for distribution
*** Distribution framework will send the message to each recipient to their notification methods selected for this event
**** For roles, get each role member and send individually 
** If scheduled:
*** Schedules to run using the scheduler and does everything the same but at that time
*** Can run at End of Day or End of Week
*** These are Summary notifications
*** Include all the events for that day or week in one message (don’t spam them with a lot of emails)
* 
* The user will receive a notification when it is done based upon the type they selected
** User can browse to it and open it from Workspace Inbox if Inbox
** User can get an email if by email
** User will see a toast message if by Toast
* 
* Message:
** Note: Copy formatting of existing System messages
** Note to dev: the Footer text needs to be small. See existing messages for formatting

*** Subject: User Account Notification
*** Body:

Hi [user first name],

Notification for User Accounts

Criteria: [Add or Edit or Delete] [Note: if summary email, show Adds, then Edits, then Deletes in same email]
