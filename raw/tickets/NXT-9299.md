# NXT-9299 — Data Exchange - SIS Import - Deactivate (withdraw) a student account

## Description
As a district I want to inactivate (withdraw) a student account that is currently active if the student is not present on the student information file.

More Detail:

* The most common approach for activating/deactivating student accounts is the presence of the student on the import file. This will be the default and initial approach to activation/deactivation. An alternate method is to provide an enrollment flag on the file (Active, Inactive). Note: this user story will only handle the scenario of the Student ID not being present on the import file.
* Inactivation (withdrawal) of a student means that the Student ID (account) already exists within the system. The account simply moves from the active (enrolled) status to being made inactive (withdrawn).
* This scenario often occurs if a student moves physical addresses and begins attending a new school district (becoming withdrawn).

## Acceptance Criteria
1. If a currently active student account (Student ID) is not present within the provided student import file, the student should be made inactive (withdrawn).
