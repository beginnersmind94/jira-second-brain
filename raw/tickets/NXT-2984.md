# NXT-2984 — Employee Details Record - Duplicate Trainings

## Description
As an admin, i want a message when entering training events in the employee record if the training i entered appears to be a duplicate. 

* Check training event name, date, topic/subtopic
* Reference training event history when adding a new event to the record
* When adding a training event to an individual employee record, use the pop up message to prevent duplicates.
* User clicks save and the message below pops up
**  "Uh oh\! Looks like <Employee Name> attended <Event> on <Date>. Saving may create duplicate records. Do you want to make changes or save anyway?"
*** "Make Changes" - like a 'cancel' and goes back to the record to make changes
*** "Save Anyway" - inserts the record as is
* Same date - hard stop 
* Different date - warning 



_Do we want to show only the events that are not already added for the current year or we would show everything and let the FE take the decision/logic check to have the duplicate value check? Also would there a possibility of multiple occurring events within an year, which is a permitted duplicate?_

* User *cannot* edit training hours for events (different than current classic functionality)
** When the event is selected from the dropdown, the hours populate (pulled from training events operation - future story)
* Look for duplicates by checking - Training Date, Training Event
** Trigger message if user tries to record duplicate training
*** Same date - hard stop 
*** Different date - warning 
** If training event date is in the future, we will notify user that this is a potential duplicate and they can check the completed box on the date/passed date of the event to record.
* The notification will have the option for the user to ‘Record Anyway’ or ‘Cancel’ - if they want to attend the same training event the same day, then we should notify and let them confirm.
