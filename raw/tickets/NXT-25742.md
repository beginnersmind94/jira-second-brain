# NXT-25742 — System - Roles - Improve Edit Role User Experience

## Description
It is not very intuitive for a user to know how to edit a Role.  Currently, the user is required to click on the number that is in the “Module Permissions” column.

+*Requirements*+: 

* EDIT action is enabled for each role to update permissions for various modules for that role
* If user does not have permissions to EDIT a role, EDIT button must be in disabled state
* Disable permissions number expansion
* Custom Role edit action is enabled

## Acceptance Criteria
* User shall see EDIT button for each role with additional actions buttons next to EDIT
* User shall not be able to EDIT button if user does not have permissions
* User shall be able to see the expanded actions button with COPY and HISTORY actions
* User shall be able to view EDIT button for both standard & custom roles
