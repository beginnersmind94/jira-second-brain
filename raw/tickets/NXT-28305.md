# NXT-28305 — POS - Create Menu Item for Dynamic Menu Reimbursable Meal

## Description
Dynamic Menus needs reimbursable meal items to use as the price bases when a reimbursable meal is formed. The Menu Items need to be available for all users of dynamic menus initially we need items for Lunch, and Breakfast, Items should also be created for all the other program Meal Types. The reason there are not items for the other meal types is that there is not a meal pattern defined for OVS for those meal types. The current implementation will use the SNP meal patterns to form reimbursable meals for summer and CACFP meals.

*There are 3 parts.* 

# Adding the Flag to POS Info in Item Management
# Creating the Items in Item Management
# POS using the flag and meal type to determine pricing the reimbursable meal

*Add a flag to POS Info*

* Dynamic Menu Reimbursable Meal, default = No
* Set this to *Yes* if this item is used for pricing when a reimbursable meal is formed

*Item Details*
The Item Names & Short Item Names will be

* DM Lunch, DM Breakfast
* DM Summer Lunch, DM Summer Breakfast
* DM CACFP Lunch, DM CACFP Breakfast
Data Source will be 
* Local
Item Category will be (DM Item may need to be added to Item Category list)
* DM Item

These fields are currently required in Item Management and will be optional in a later release

The POS will determine the Menu Item to use by Looking at the added flag *Dynamic Menu Reimbursable Meal* If that flag is *YES* and the *Meal Type* matches the Meal Type being served the prices from that menu item will be used.

## Acceptance Criteria
*Item Management* 

Reimbursable Meal Items are created for Lunch, Breakfast, and Snacks for use by Dynamic Menus

There can only be one DM marked item per meal type per district.

*POS*

When a reimbursable meal is formed in dynamic menus the Name, and POS pricing for the Meal Type set for the meal sold.
