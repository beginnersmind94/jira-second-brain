# NXT-36316 — API Error - Monitoring Page

## Description
As a SchoolCafé admin, I want to be able to view the API errors so that I can proactively monitor the application and ensure higher customer satisfaction.

*Flow:*

* User logs in and browses to Support > Monitoring Dashboard > API Errors
* User views a list of error messages

*Notes:*

* This story assumes the Monitoring Dashboard has been created
* This story’s UI/UX is based off the Web Error page and is following that page’s flow, layout and functionality
* We are only changing the appropriate fields as necessary, so when in doubt, copy what that page does
* Try to make it as close as possible to that page
* This page is the user front end way of viewing API errors

*Requirements*:

* *Monitoring Dashboard Page updates:*
* Add a card to the Monitoring Dashboard page
* Card looks the same way the Web Error card looks
** Card title: API Errors
** Description: View API errors
* On click, open the API Error Monitor page
* 
* *API Monitor Page Requirements*:
* Create the page and link to it from the Monitoring Dashboard
* 
* Base the UI/UX off the Exports page and stay consistent with the other pages under Monitoring Dashboard
* 
* Sections:
* *Header Section*
* Title: API Error Monitoring
* Navigation: System > Support > Monitoring > API Error Monitoring
* 
* *Filters Section*
* Create a filters section at the top of the screen like System Configuration Notification page
* Filter fields:
** Start Date
** Start date and time to filter the results
** Default to today at 1 minute after midnight
** End Date
** End date and time of when to stop
** Default to Today at 1 minute before midnight
** Error codes
** A multi-select list of error codes
** Default no selection or All
* Reset button
** Resets the filters same as for the System Configuration Notification page
* Apply button
** Applies the filters same as for the System Configuration Notification page
* Collapse / expand arrows
** Expands and collapses the filters section same as for the System Configuration Notification page
* 
* *KPI Section*
* This section is to the right of the filters section, at the same level, above the grid
* Show a chart of the Top error codes
** I recommend a bar chart with count vertical and error codes horizontal, but I am open to suggestion on what looks best
** Without being able to see what it looks like in real world, I recommend we try top 10 error codes, but if there is not enough space to make it look good, we can do fewer. If there is room, we can do more.
* The filters in the filter column update the dataset for this chart and force it to refresh when user hits Apply
* 
* *Grid Section*
* Show a grid of web errors filtered to this district and environment
* Columns:
* Note: at the time of this writing, I still don’t know what values we can get from the error log, so this is a tentative list; it is to be reviewed during refinement
* Module
** Show which module the page is in
** Empty if not related to a module
* API
** What API did the error occur in
** Empty if not available
* Error Type
** Ex: Web, API, etc.
* Error Code
* Date / Time
** The moment it occurred
* User
** Account the error occurred to
* Action
** For each row, have an icon button
** Icon: use our standard view icon
** On click, show the API Error detail page (another story)
* 
* Permissions:
* In General resources screen for Roles & Permissions
** Visible = true
** Hierarchy: put under System
** Title: Web Error
** Default permissions: all four checkboxes checked for SchoolCafé Admin
