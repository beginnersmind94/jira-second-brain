# NXT-30046 — IM - Items - Create New Item - Product Info - Pack Size Config (Step 3)

## Description
As a District Admin I want the option to provide general Product Info about an item I am creating so I can populate important attributes.  

*Note:* this story has large impacts on the way items and units are saved to the DB. This also has impact on the syncing between IM and Inventory.

* Meeting Notes (Update 5-24-22 *JM*) TBD in 69 or 70
** Change ‘Smallest’ to ‘Lowest’
** Switch ‘Lowest’ to be the first radio option, and selected by default
** If ‘Highest’ is selected, automatically add a second row

*Requirements*

* Once the dropdowns in [Product Info (Step 2)|https://primeroedgenext.atlassian.net/browse/NXT-30037] are populated, enable the Unit Info grid 
* Display a section for ‘Pack Size Configuration’ (AKA ‘Unit Info’)
** This will allow users to add *up to 3* item units, starting with one unit
* Provide the initial unit row
** Unit Type flag - *Highest, Smallest*
*** this will allow the user to build the hierarchy up or down
*** this also determines whether we require *Weight* or *Fair Market Value*
*** this only applies to the first unit
** Unit Description - freetext
*** the description for the first unit
** GTIN - 14 digit numeric value (optional)
** Fair Market Value* (*only for Highest unit*)
*** Dollar value for the highest level unit (often the orderable unit)
** Weight* (*only for Smallest unit*)
*** Weight for the lowest level unit in the hierarchy
* *Unit Type: Smallest* scenario
** Buttons: *Add Unit* or *Enter Fair Market Value*
** *Enter Fair Market Value* will enable the FMV field and allow user to enter a value
*** this item could be saved as a single-unit item at this point
** *Add Unit* will add another row and conclude the entry for the first unit
*** adding another unit will insert a row above the smallest unit with the same columns *plus* a Sub-Qty column on the smallest unit
*** Hide the Unit Type column
*** Calculate the Weight according to the Sub-Qty
*** Again provide the Add Unit button if user wishes to add a 3rd and final unit, or the Enter FMV button to enter a price and conclude
* *Unit Type: Highest* scenario
** After entering Fair Market Value, enable the *Add Unit* button to enter a second item unit
*** Insert another row below the highest unit with the same columns *plus* a Sub-Qty column 
*** Calculate the FMV according to the Sub-Qty
*** Buttons: *Add Unit* or *Enter Weight*
*** *Enter Weight* will enable a Weight field on the lower item and allow user to configure
**** this item could then be saved as a 2-tier pack size
*** *Add Unit* will add another row below and conclude the 2nd row
**** this would add a 3rd row which would automatically have the *Weight* field enabled  
* +Clarifications/Rules+
** *Mandatory:* at least one unit is configured including Weight and Fair Market Value
*** Otherwise the entire Product Info card can be skipped
** Current maximum is 3 units (most items will be 2 units)
** Unit Description is unique per item family
** GTIN should be unique for all items
** Fair Market Value is only entered on the top-level unit (either user-specified or inferred highest) and calculated for the others according to the sub-quantity
** Weight is only entered on the lowest level unit and calculated for the others according to sub-quantity
** The ‘Unit Type' column will only be shown when there is a single unit

## Acceptance Criteria
* Display the Pack Size (unit info) grid in the Product Info card once the selections are made
** Do not allow users to skip the Unit Info separately
* Display an initial unit row
* Allow users to specify the first unit as highest or smallest, and therefore allow to build up or down
* Capture the sub-quantity for all 2 or 3 tier unit hierarchies
* Allow pack size configuration to be saved as long as there is a description, weight and FMV (for one or multiple units)
* Users only enter Fair Market Value on the highest unit; it is calculated for others
* Users only enter Weight on the smallest unit; it is calculated for others
* Allow users to configure up to 3 units
