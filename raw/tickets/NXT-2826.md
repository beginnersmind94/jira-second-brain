# NXT-2826 — Inspections Dashboard - Create Screen and Widgets

## Description
*User Story*
As an Inspections user, I have a dashboard screen to land on when I first enter the Inspections module.

*Mockup*
[https://nqyc9v.axshare.com/start.html#p=inspection_dashboard&id=0rb254&g=1|https://nqyc9v.axshare.com/start.html#p=inspection_dashboard&id=0rb254&g=1]

*Requirements*

* The Inspections Dashboard screen is the first screen users see when navigating to the Inspections module
** Dashboard display is dependent on user’s role. While this story won’t show this, any backend architecture should be considered
* The Dashboard has widgets to display data to the user
** Widgets describe the different displays within the screen
*** For example, the calendar is a widget
** Widgets for this story are placeholder and show no actual data

## Acceptance Criteria
* Dashboard page is available and can be accessed by users
* Placeholder widgets are present and have basic funcionality

Note: Dummy data can be used to validate acceptance criteria
