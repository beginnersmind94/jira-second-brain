# NXT-53698 — Sys/POS - Sync Internal - Research/implement down to seconds, override options at POS

## Description
|Previous Sale/Transactions sync interval to POS|POSPSSYNTM|System Setting|

*requirements*

* Research if we can lower this setting down to seconds and have it work safely
* Updates to the system setting:
** Rename: POS Data Sync Interval
** Description: This setting controls how frequently data is sent to/from the POS machine. The shortest interval available is 30 seconds. If this setting is set under 30 seconds, POS will still transfer data at 30 seconds. This will not change the interval for balance updates - account updates intervals are always 60 seconds.
** Default value: 60 seconds, but need to test if we can safely push this down to something lower
* POS needs to continue operating with the setting change
* Add a setting at the POS
** Override POS Data Sync Interval
*** Options: Yes/No (Default)
** If set to Yes, user can specify a different sync interval in seconds from 60 → 600
* Remove POS Site Setting: Enable SQL Accounts Sync in POS
* Remove setting  POSACCSYTM: Account documents sync interval setting
** Hard code the account updates to happen every 60 seconds

## Acceptance Criteria
* Research how fast we can get data sync down to
* System Setting updated:
** Name, description, default value changed to 60 Seconds
* POS operates with system setting value changed from minutes to seconds
* POS setting added to let the terminal use a different sync interval
* Remove POS Site Setting: Enable SQL Accounts Sync in POS
* Setting POSACCSYTM removed, POS updated to check every 60 seconds
