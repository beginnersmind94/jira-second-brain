# NXT-36672 — Monitoring Scheduler – P5 Run Monitor Now

## Description
As a SchoolCafé admin, I want to be able to manually run an existing monitoring job so that I can force it to run during troubleshooting or if it failed to run when it was supposed to.

*Flow:*

* User logs in and browses to Support > Monitoring > Job Scheduler
* User views a list of scheduled monitoring jobs
*  User clicks Run Now icon on an existing scheduled job

*Notes:*

* This is a continuation of the story to create the Job Scheduler page
* This story assumes that story has been created and that the Monitor tab was created and the grid on that page has the edit button
* This story assumes that the Add New Monitor story has been completed and that users are able to add new monitoring jobs already
* Note that each row in the grid already has a Run Now button that does nothing, so update that button

*Requirements*:

* Requirements for the Job Scheduler > Monitors tab
* Requirements are for what happens when a user clicks the Run Now button on a row in the grid of scheduled jobs
* All this functionality is like the Import button on the SIS Auto-Sync, so copy that functionality and do what that button does
* For the Run Now button, on click do the following:
** Show them the confirmation button like in SIS Auto-Sync, show a confirmation popup with the same UI
** If they select yes, delete the scheduled job
*** Start the job manually right then
*** Ensure job is logged and shows up in history with all the correct values
** If they say no, then cancel the popup and do nothing
* 
* Permissions: no new requirements
