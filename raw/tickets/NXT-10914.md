# NXT-10914 — Eligibility - Notifications - Direct Certification 

## Description
*User Story*

As a central office user I want to send Direct Certification notifications so that I can provide notices to families of eligibility approval.

Utilize the Eligibility > Notifications page (see related link for page/routing of the Notifications page)

*Mockup*

Posting current mockup that is in place based on notifications from the Matched and Extensions page(s). We will be shifting the notifications to be viewed/triggered from with Eligibility > Notifications > Direct Certification (tab).

!image-20210225-234429.png|width=1094,height=620!

*Requirements*

Add to the Notifications page from 10913.

* User can perform a Direct Certification Search that looks for student DC matches.
* User has the following options:
** -Academic Year-
*** *Note 3/10/21: Academic Year selector not required, only use current academic year.*
** Approval Date
*** User can specify an optional date range that will filter results based on the dates selected vs the Approval Date for the DC record.
** Exclude student who already have Direct Certification notifications
*** This would omit any student that has previously been sent a notification for their DC match/approval
** -Exclude students who were notified with the same eligibility status based on another approval method-
*** -Excludes students who have already received a notification for that particular eligibility. Example: Student A has previously received a letter for being Reduced. They would not show up in the grid for a 2nd match of Reduced.-
*** *REMOVED 3/12/21 TO ITS OWN ITEM DUE TO TECHNICAL/DATABASE*
* User sees search results in a grid with the following columns:
** ID #
** First Name
** Last Name
** Site
** Grade
** File Number
** Case Number
** Approval Date
** Notification Date
** Print
** Email
** View

*More Details:*

* Harness the Direct Certification letter template (in the System module) to be populated with information from an approved student.
* User should be able to select multiple students from the list to notify them. Column selection needs to apply to all students within the entire grid of results (not just the page)
* User should be given a choice for how they’re sending the notification letters: _(Note: This will be taken from how the platform control is created.)_
** Email
** Print
* User should have an option for grouping by household. _(Note: This will be taken from how the platform control is created.)_
** Options:
*** Individual Letter
*** Household Letter
**** This will grab any household members who are also part of the selected list and collate their information into the template grid.
**** Only 1 letter per household is required.
***** Use the youngest student’s contact information for populating address fields.
* Letters should populate and dispatch appropriately to a local Print Dialog or to the email service. _(Note: This will be taken from how the platform control is created.)_
** The letter should be based on the primary language of the student. If a template does not exist for that language, use English as the default.

*Direct Certification - Notify (Current screens)*

Direct Certification will contain the list of direct certification student matches and/or extensions that are available for notification.
