# NXT-37511 — Local - Ingredients - Creation of New Ingredient using GTIN

## Description
_(none)_

## Acceptance Criteria
Navigation: Menu Planning - Ingredients

Creation of ingredients using GTIN:

If the GTIN is valid - the system should show the confirmation message and populate the details on the screen without prompting for any additional fields

IF the GTIN is invalid, show the i icon and prompt for other fields as-is

Show the i icon if the GTIN is invalid

Regression Scenario: Check loading of Information either on Gtin, Gtin code or Broken Unit GTIN

!image-20230316-063106.png|width=1607,height=951!

Assumption - user always uses base unit GTIN to populate ingredient details

Agreed Solution:

Populate the information on the fly without inserting any NULL value in DB (use of Find Icon next to GTIN Field)

If the the system find a new match - the positive flow will continue with Ingredient creation

* If the user choose cancel on the popup message, then existing validation will follow (prompting user to enter item name 

If the system identifies existing match, load the existing item

* If the user choose cancel on the popup message, then proceed with new item creation, but upon saving the validation will follow (prompting user to enter a different item name due to existing item with same description)
* If the description is different, the user should be allowed to create a new ingredient
