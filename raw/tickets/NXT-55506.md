# NXT-55506 — DMF - Dashboard - Progress Log

## Description
As a Customer Support role user, I would like to be able to track which features are imported and their status so that I can track my progress

*Notes:*

* After our first prototype review, Dallas asked that we be able to communicate to end users what “features” were migrated and wait failed
* To accomplish this, we will make the following updates below

*Requirements:.*

*Database or ADF*

* Store somewhere a list of tables that will be updated in the destination DBs
** Module Name
** Feature name (provided by PO)
** Table name
* 
* Create a History log table that can be used at run time for the migration routine to log which table is updated and its status



*Runtime*

* When running the scripts to insert into the destination tables:
* For each table, after completing, log into the history log table that the table completed successfully 
* If it failed, log its failure
* This should be done so that an API can be written that shows a list of all the table activity from the first until the last with the following fields:
** Module Name
** Feature
** Table
** Status
** If possible, would be great to have start and end time stamps, but can live without



*Dashboard*

* On the nested grid, make these changes:
* Add a button at the end of the row
** Make it an icon
** Tooltip: Progress
** On click:
*** Open the Progress drawer





*Progress Drawer*

* Create a new drawer
** It will show the step by step import progress of each module
** It will show a progress indicator for each module
** It will show estimated time remaining
* Title: Migration Progress
* Size: Half Page
* Use sliding motion to open
* Drawer uses 3 accordions to divide up data
** Top accordion is for selecting what to view
** Middle accordion is for the KPI and progress stuff
** Bottom accordion is for the raw data in a grid
** Each accordion can be expanded or contracted
* Fields:
** First accordion
*** Title: Selection
** Single Select Drop Down List
*** Values: list of modules
**** Framework
**** System
**** Front Office
**** Back Office
**** Family Hub
**** Do not include any modules that are being skipped by the migration routine
**** First row is empty text / null value
*** Default selection:
**** Check and see if there are any migrations running
**** If yes, auto select this one and load the page with its data
**** If no, then select the last one that was ran and load the page with its data
**** Else, select the empty record by default
** Apply
*** If empty text, blank out all the subsequent fields
*** Else, load the remaining fields with the data for that module
** 
** Second accordion
*** Title: Progress
** KPI Card:
*** Label: Total
*** Value: Count of tables to be imported for that Module
*** Example: 180
** KPI Card:
*** Label: Completed
*** Value: Count of tables completed for that Module
*** Value: Percent Completed
*** Example: 90 (50%)
** KPI Card:
*** Label: Estimated Time
**** If the status is Completed, Cancelled or Error, then change the label to: Actual Time
*** Value: 
**** Time in hours and minutes (no seconds)
**** To get value, follow this process:
**** First see if it completed successfully
***** If yes, show that actual duration
***** If no, continue with the logic
**** Then see if it _previously_ completed the same module for the same district
***** If yes, then use that duration
***** If no, continue with the logic
**** Then see if it _previously_ ran _but failed_ or was cancelled
***** If yes:
****** Get the duration for that failed migration
****** Count the steps that were completed 
****** Calculate the % complete for that failed migration
****** Divide duration by % complete to calculate total estimated time
****** 
****** Example:
****** Module A has 50 tables
****** Previous migration failed at number 10 and took 60 minutes
****** Therefore, it completed 20% (10/50) and has 80% remaining (100%-20%)
****** Therefore total estimated time is 60 minutes / .2 = 300 minutes or 5 hours
****** -Therefore remaining estimated time is 300 - 60 = 24o minutes or 4 hours-
***** If no, continue with the logic
**** Else it has never run at all, so need to find comparable
***** Get an _average_ time of all previously run migrations that were successful for that module and how that value
**** Else, there is no possible answer, so:
***** Show the text: No Data
*** Example: 03:40 
**** example says 3 hours and 40 minutes
** KPI Card:
*** Label: Est. Time Remaining
*** Value: 
**** 0 for anything that is not in running status
**** Else (it is currently running)
***** Get the actual duration so far
***** Get the estimated duration
***** Subtract the duration from the total estimated to get the estimated remaining time
***** Check est. remaining time and make sure it is greater than zero
***** If yes, show that number
***** If not, then try to recalculate:
****** Get the % complete and % remaining
****** Calculate remaining time using % remaining vs. Actual duration
*** Example: 01:20 
**** example says 1 hour and 20 minutes
** Progress Bar
*** Show a progress bar across the screen that shows % complete
** 
** Third accordion
*** Title: Details
*** Default contracted
** Show a grid
*** Columns:
**** Order
***** The number of its execution 
***** Ex: 1 for the first one, 2 for the second
***** Need this so users can see the order the tables were executed in and also so they can sort for last one first
**** Feature
**** Table
**** Status
**** Duration (if possible)
*** Usual paging and sorting
*** Filters:
**** Do the table Show / Hide Filters icon for inline editing
*** Default sort on Order DESC (last one first) so that they can easily see which table failed without having to go to the last page in the grid
*** If there are no steps, for example, it hasn’t run yet, then show the usual No Data message

## Acceptance Criteria
* User can go to a page and view the progress of a running, completed, failed or cancelled migration
* On this page, they can see:
** Total steps
** Completed steps and % complete
** Actual or estimated duration
** Estimated time left
** A graphical representation of the progress
** A grid with the details of the features and tables and their status
