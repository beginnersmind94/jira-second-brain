## Acceptance Criteria
The user should be able view “Brands” configuration button as shown in UI

Upon clicking the button the system shows the slide out as shown in UI



*Adding a New Brand:*

User should be able to enter the brand name, and select the manufacturer

upon clicking the Save button - it should be added to configuration

Save button is enabled only upon selecting the Manufacturer

In case of duplicate entry - system should alert the user with error message (“Brand already exists, please enter a different brand”)
User can close the slide out by clicking on “X” icon



*Editing a Brand:*

User should be able to click on Edit icon and update the description as needed

The unique validation should be applied in case of duplication again

User can close the slide out by clicking on “X” icon



*Deactivating a Brand:*
The user should be able to click on the icon and de-activate the entry only when the entry is not used in any Item
the deactivated brand are not listed in the dropdown while creating or editing an item

Deactivate item should be disabled  for manufacturer which are used



*General Rules:*

The system should store the values with a flag that user has created the configuration
Only user added entries should be listed in this configuration
Brand will have 1:1 mapping with manufacturer, while user may create link more multiple brands to manufacturer (but one transaction at time)
