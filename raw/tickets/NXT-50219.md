# NXT-50219 — LMA - District - Dashboard

## Description
As a SchoolCafé Admin role member, I would like to view the licenses my district has so that I can know what we have access to and when they will expire.

*Notes:*

* {color:#ff5630}Warning - The Bolt code has been updated a bunch for this screen since it was originally turned in, so get the latest{color}
* For Licenses and Reports, those tabs are for Cybersoft employees and the scope is for all Districts
* For this page, the users are District users and Customer Support role
* On this page, we can see what licenses a district has and their history of licenses

*Flow:*

* LMA role user browses to System > District Licenses
* User sees a list of licenses generated for the district

*Requirements:*

*Resources, Roles & Permissions*

* Add District Licenses page to the resources
** Ensure it has View and Edit permissions
*** There is no Delete on this page
* Ensure this permission is selectable in roles
* Add the view and edit permission to the default permissions for Director and Admin roles only
* Update all pre-existing Director and Admin roles to have the default permission

*Header*

* Create the District Licenses screen if it does not already exist
* Title: District Licenses
* Button:
** Label: Subscription
** On click:
*** Open the Subscription drawer (covered in another story) 

*-Filters-*

* -Search-
** -Like elsewhere, a search against all the fields in the grid-
** -Starts trimming at 3 characters-
* -Status-
** -Values: the License Key Statuses (New, Active, Cancelled, Expired)-

{color:#bf2600}[Ravi]: Filters section is not needed as we are only displaying the active status licenses in the district dashboard{color}

*Body*

* Table of panels:
* No paging, sorting
* Data:
** List of all the generated license keys for the district regardless of status
** For each license key, a list of all licenses that are children of that key
*** Seen when expand a license key
** For each site or device license in that list, a list of all sites linked to that license
*** Seen when click Show Sites after expanding a license 
** Default Sort:
*** If you can, it would be best to do them in this order: New, Active, Expired, Cancelled
**** If not, then sort by Status alphabetically
*** And then sort by Expiration Date

*License Key Panel*

* Visual Indicator
** To the left of the license key
** Color matches status colors (green for Active, etc.)
* License Key
* District Name
* Effective Date
* Expiration Date
* Days Until Expire
** Calculated: Days between End Date and Today()
** Color red if less than 30 and show as urgent
* Status
* Hide/Show chevron
** On click: Show/hide the licenses linked to the key

*Licenses Panel*

* If expanded, show the licenses linked to the license key
* Separate District licenses from Site & Device licenses because they have different actions
** For District licenses, show:
*** License Name
*** Enable / Disable
**** Toggle button
**** Use the same rules and functionality of what is on the Configuration tab currently
**** Only License Management role or Customer Support role can click this button
**** Default is Enabled
**** User cannot disable System or Workspace (same as now)
*** View History button
**** On click:
***** Open the View License History drawer (covered in another story)
** For Sites and Devices, show the same but also:
*** Show if a Site or Device license
*** Show Counts
**** Format n of n used (ex: 0 of 2 used)
**** Shows how many licenses there are and how many are linked to a site or device
*** Show Sites Button
**** On click:
***** Show the View Sites panel (covered in another story)

*Security*

* Customer Support role and License Management role can do anything
* District Licenses page permissions from R&P, so ensure user is a role with that permissions
** View permission necessary to view the screen, subscribe and view history
** Edit permission for all other actions (turn on/off and add sites)

## Acceptance Criteria
h3. *1. Resources, Roles & Permissions*

* Add *District Licenses* page to *Resources*.
* Ensure *View* and *Edit* permissions exist.
* *No Delete* permission.
* Permissions must be selectable in *Roles*.
* *Default permissions*: add *View + Edit* to *Director* and *Admin* roles.
* Update all pre-existing *Director* and *Admin roles* to include the default permission.

----

h3. *2. Header*

* Title: *District Licenses*
* Button:
** Label: *Subscription*
** On click: open *Subscription drawer* (covered in another story)

----

h3. *3. Filters*

* *Search*:
** Works across all fields in the grid
** Minimum 3 characters to start search
** Trims whitespace before search
* *Status filter*:
** Values: *New, Active, Cancelled, Expired*

----

h3. *4. Body / Table of Panels*

* *No paging, no sorting* (sorting handled separately)
* *Data*:
** Show *all license keys* for the district, regardless of status
** Expand a license key → show all *licenses linked to that key*
** Expand a license → show all *sites linked* (for site/device licenses only)
* *Default Sort*:
** Preferably: *New, Active, Expired, Cancelled*
** If not possible, then: *Status alphabetically* → then *Expiration Date ascending*

----

h3. *5. License Key Panel*

* *Visual Indicator*:
** Color matches status (green = Active, etc.)
* Display the following:
** *License Key*
** *District Name*
** *Effective Date*
** *Expiration Date*
** *Days Until Expire*:
*** {{Days = Expiration Date – Today()}}
*** Color red if < 30 days → show as urgent
** *Status*
* Chevron toggle:
** On click → expand/collapse *licenses linked to the key*

----

h3. *6. Licenses Panel*

* *Expanded view* shows licenses linked to the license key
* *Separate District licenses* from *Site & Device licenses*

h4. *District Licenses*

* Show:
** *License Name*
** *Enable / Disable* toggle button
*** Use same rules as *Configuration tab*
*** Only *License Management* or *Customer Support* roles can toggle
*** Default = Enabled
*** Cannot disable *System* or *Workspace* licenses
* *View History button*:
** Opens *View License History drawer* (covered in another story)

h4. *Site & Device Licenses*

* Show:
** *License Name*
** *Type* (Site / Device)
** *Counts*: {{n of n used}}
** *Show Sites button* → opens *View Sites panel*

----

h3. *7. Security*

* *Roles allowed*: *Customer Support* and *License Management*
* *District Licenses page permissions*:
** *View permission* → required to view the page, Subscription drawer, and View History
** *Edit permission* → required for Enable/Disable toggles and adding sites
