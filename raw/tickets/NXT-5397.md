# NXT-5397 — Meal Exceptions - Resolve Duplicate Meals

## Description
When multiple reimbursable meals are served to a student for the same meal type and date I need a way to correct them.

From the view list I need to be able to select a meal and charge it to the account, delete it from the account, or mark as keep as reimbursable. By default they are marked as reimbursable.

The transaction item is updated when the save is selected.

## Acceptance Criteria
Transaction History is updated
When a transaction status is changed the students account is updated to reflect the change.
Adults do not have duplicate Meals.
When save selected 'Selected duplicate meals are resolved'
   If the update does not apply 'Selected duplicate meals did not update properly. Please verify the changes.'
If Keep(Check) is selected for an item, Keep is disabled for the other duplicate Items
All duplicate items can be set to charge($)
All duplicate items can be deleted (Trash Can)
Selections can be unselected before Save is selected.
