# NXT-7504 — System - Roles - Edit Role (2) - Save changes to Role History drawer

## Description
As a PrimeroEdge admin, I would like to be able to have an audit trail of changes that were made to a custom role, so that I can quickly identify when issues could have arisen from that role being modified.

*Summary*

When a user selects the Save button on the Edit Role screen after making a change(s), save the old and new values of that session along with the current date and time.

*Details*

* In the Edit Roles screen, when the user hits the Save button after checking any boxes/toggles, track the changes they have made. (We should capture special permissions as well.)
** Old values: for whatever has been checked/unchecked/toggled on/toggled off, capture the previous value of that field.
** New values: for the same field, capture the corresponding new value.
** Date/time: Capture the mm/dd/yyyy and 00:00:00AM/PM datetime of the change
** User: UserId (to get First and LastName) of the person who has made the change.
* Example:
** Date: 11/11/2020 11:30AM
** Changed by: Daimien Burks
** Old Value
*** Site Configuration - Sites - Add - Enabled
*** Site Configuration - Site Types - View - Disabled
*** System - Receive email notifications after every update - Enabled
*** Workspace - Broadcast Messages - Full Control - Enabled 
** New Value
*** Site Configuration - Sites - Add - Disabled
*** Site Configuration - Site Types - View - Enabled
*** System - Receive email notifications after every update - Disabled
*** Workspace - Broadcast Messages - Full Control - Disabled
* Formatting/conversion (using hyphens, specific datetime format, etc.) can happen on History drawer UI if needed. 
** More importantly, this story is complete when we track every change made in the Edit Roles page, with its old and new value.
