# NXT-67675 — Add New Account Form

## Description
*As a* district financial manager
*I want to* add a new GL account through a form
*So that* I can expand my chart of accounts

## Acceptance Criteria
* When I click "Add New Account", a popup form appears
* The form has fields: Account Code (required), Description (required, 50 character limit), Account Category (required dropdown)
* Account Code must be unique across all accounts
* Description must be unique across all accounts
** The Description field shows a live character count as I type (e.g., "35/50 characters used")
*** 50 character limit
* Form validates uniqueness before allowing save
* When I select an Account Category from the dropdown, the Base Type is automatically populated based on that category's base type
* Base Type field is read-only and displays automatically with one of these values: Revenue, Expenditures, Assets, Liabilities, Fund Balance
* After clicking Save, the system validates the Account Code against the configured state account structure
* -If the Account Code doesn't match the structure format, show a warning modal: "This account code doesn't match your configured account code structure. Do you want to continue?"- (since we will display a warning as the user is typing, this is redundant so removed by PO)
** If I click Continue in the warning modal, the account saves and the form closes
** The new account appears in the GL Accounts table with a warning triangle icon next to the account code
** When hovering over the warning icon, show tooltip: "This account code doesn't match your configured account code structure format"
** The validation warning banner count updates to reflect the new non-compliant account
* If the Account Code matches the structure, the account saves without warnings
* If validation fails for duplicate Account Code or Description, show appropriate error messages and prevent saving
* After clicking Cancel in the form or warning modal, the form closes without saving
