# NXT-3751 — Inspections Scoring - Static Tables

## Description
*User Story*
As an Inspections user, I have a method for including scoring in static table questions to allow for evaluating an inspection.

*Requirements*
* Some column types for Static Tables can have scoring added to them
** Occurs for List and Rating column types
** Only available if scoring is set to Custom Scoring
* If scoring type is set to Custom Scoring, List options have a field for score next to them
** Field determines how much each option is worth toward the score (0-99)
** If left empty, field defaults to 0
** Field can be edited when Template is edited
* If scoring type is set to Custom Scoring, Scale columns have a toggle for enabling scoring
** By default set to false
** Toggle can be edited when Template is edited
* A Static Table can have multiple scoring columns
* Total possible points is updated based on the number of scoring columns and the number of rows in the table
** Each List column adds the highest possible point value
** Each Scale column adds the highest value in the Scale
* When an Inspection is run, points associated with each column is applied toward the Inspection's final score
** List entries apply the value of the selection made
** Scale entries apply the value of the rating that is given
** If no selection is made, no points are added
* All Static Table scores are included in the final score of the Inspection

## Acceptance Criteria
* Score can be applied to Scale and List columns when creating a Template with Custom Scoring, and can be saved and edited
* Score from both column types is properly applied when running and submitting an Inspection from the Template
