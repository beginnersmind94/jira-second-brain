# NXT-5241 — Direct Certification - Unmatched (Potential) - Search & Review

## Description
*User Story*

As a district admin I want to see potential DC matches (the Unmatched list) that have met appropriate thresholds so I can make a decision on extending eligibility.

*Requirements*

* Add a page called ‘Unmatched’
* Add a tab called ‘Search & Review’
* User is able to view any potential matches that have been generated per [https://primeroedgenext.atlassian.net/browse/NXT-7681|https://primeroedgenext.atlassian.net/browse/NXT-7681|smart-link] 
* User can select probability Confidence levels (High, Medium) which will act as filters.
** These ranges are specified in the configuration story. [https://primeroedgenext.atlassian.net/browse/NXT-5243|https://primeroedgenext.atlassian.net/browse/NXT-5243|smart-link] 
* User can alternatively select a probability range, from 0 to 100, and can then search for results.
* Results in the grid will need filters to let users find specific students. Contains filters for the most part, and a date picker for the DOB columns.
* Grid should show the following information for each potential match:
** Student ID
** First Name
** Last Name
** Birth Date
** Eligibility
*** This is the current eligibility.
** Potential Match (File Data)
*** First Name
*** Last Name
*** Birth Date
*** Approval Type
**** SNAP, TANF, etc.
*** File Date
** Probability
*** This will be a percentage up to 100%. 
**** We may actually record well past 100% in the DB, but sensibly we should cap at a 100% match.
*** User will need a way to analyze the probability to see where the points came from - this will be handled in another story.
** Any additional filters (columns) that are hidden by default will be added to the ‘File Details’ side.
** Default sort for the results is by Probability % descending, then last name of the student, first name.
* Page needs to show the user any potential DC match (this is not the same as the Extensions we have previously done).

## Acceptance Criteria
Unmatched page added.
Search filters available for probability, including a choice between the set High and Medium thresholds, and a 0-100 percentage slider.
Results should relate to the probability threshold selected.
Grid should show potential matches that have not been acted on yet.
