# NXT-26109 — Platform - Permissions - Resource Name Hierarchy implementation

## Description
As a Primeroedge admin, I should be able to create and update resources based on the hierarchy of the resource which system shall automatically provides/updates the name of the resource

## Acceptance Criteria
* User shall be able to select the resource hierarchy which guides the resource name
* User shall be able to update the resource name when the hierarchy of the selection changes
* User shall be able to delete and add resource which will reflect the naming criteria
