# NXT-53378 — MP - Menu Cycle - UI Updates for Permissions (UI ONLY)

## Description
As a District admin, I would like permissions set aside for on the menu cycle for what my users can and can not do.

## Acceptance Criteria
Update the Menu Planning module Menu Cycle page and subpages to show/hide page elements based on logic of the Permissions

* +Menu Cycle+ (List)- Page
** *View:* view icon (check for Menu Planner)
*** if user does not have Menu Planner permission do not show the sub-menu and route directly to standard 
** *Edit:* Pencil icon
*** if user does not have Menu Planner permission do not show the sub-menu and route directly to standard
*** If user does not have edit permission for Menu Cycle List page but does have Menu Planner permissions, route directly to Menu Planner
*** If user does not have Menu Planner permissions or Edit permission on Menu Cycle list page, hide the edit pencil
** *Add:* +New button
** *Delete:* Status (Active) toggle
*** if user does not have Delete permission, hide the toggle switch and show a text label for Active/Inactive
**** also disable toggle on Menu Cycle Details page 
** +Menu Cycle Details+ - Sub Page 
*** *Edit:* day & menu selection
**** consider permissions for Menu Details for menu-level edit pencil 
*** +Copy Cycle+ (Action): Copy button
*** +View Analysis+ (Action): Analysis icon
