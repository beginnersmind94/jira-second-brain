# NXT-56173 — SC - Bulk Update UI Changes (Images)

## Description
_(none)_

## Acceptance Criteria
*Navigation*: Items- Item Upload Icon - Pending Updates Tab

The UI should have a Column named as “Status” with dropdown values as as “New” and “Queued”



*Single Image Update:*

The image shall be updated to Queued and Processed Immediately, the records is marked as completed, it will be cleared from the grid upon closing the slide out 



*Bulk Image Update:*

All the selected records (lets say 50), they shall be updated with status as Queued

The user should be able to click on filter and view all the records that are queued status 

All the records marked for bulk updates shall be processed in the backend. 

An auto refresh shall be used in UI (runs once per minute) to clear that records that marked as Queued once its moved to completed. The completed records shall not reflect in UI

Once a file is updated if the new images are included - the record for that status will be reset to new (it will reflect in API as New)

When a new file is updated and there are no images - the record status will remain as completed, no new records will reflect in the UI
