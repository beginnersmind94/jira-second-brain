# NXT-43588 — MP- Menu Planner- Updating Frequently used, recently used and favorites to get the latest information

## Description
As a menu planner, currently if I want the latest information related to a menu item that is in one of my lists on the menu planner, I need to search for the menu item to get the latest information. That is not the correct approach and the menu items should be updated on the lists themselves.

## Acceptance Criteria
* Update menu items with latest information when on the menu planner
** When menu item is dropped, latest information should be available on the menu (EX: nutrition. contributions, allergens, Menu Item categories information)
** Create a job that will run a new refresh API call for the below sections
*** This should not effect saving menus or menu cycle if API fails
*** This should apply on both the create and edit scenarios
** Add a manual refresh under the menu item search bar that will trigger a manual API call and also refresh the search bar
*** This should apply on both the create and edit scenarios
* Apply this change to menu items in the following lists
** Recently Used
** Frequently Used
** Favorites
