# NXT-48823 — Temp User - Login & Authorization Updates

## Description
As a SchoolCafe user, I want to be able to login to the site using my temp account so that I can use the website

*Flow:*

* User browses to Login page
* User asked for username and hits Continue
* System checks if user is a temp user and then checks if the date is in between the start and end dates
** If not, the system stops the user and shows them a message
** If yes, then the system continues with the authentication process

*Requirements*:

* These requirements apply to the login page
* When the user inputs their username and clicks next, validate if they are a temp user
* If they are, validate if their account is in between start and end date
* Don’t allow temporary accounts to login before or after these dates
* If user tries to login before or after, then show this message: Your account does not have access at this time. Contact your administrator to request further access.
* If their account has expired, set it’s status as Inactive and log in History

## Acceptance Criteria
* Temporary accounts only work during the start and end dates and user is notified if they try to access before or after
* Expired temp accounts have their status set to Inactive and is logged in History
