# NXT-68663 — PE-SC-Reload the KPI page when the user adds or updates the Benchmark value for a site on the Benchmark configuration page.

## Description
As a district user, I need the KPI page to reflect the updated benchmark values when I add or modify the benchmark value for a KPI on the benchmark configuration page. 
Refresh the KPI page when the user closes the Benchmark configuration page, only when the user adds or modifies a benchmark value for a site.
When reloading the KPI page, retain the user-selected date range and school selection previously made on the KPI page.

## Acceptance Criteria
Reload the KPI page when the user adds or updates the Benchmark value for a site on the Benchmark configuration page.
Refresh the KPI page when the user closes the Benchmark configuration page, only when the user adds or modifies a benchmark value for a site.
It needs to work for both PE and SC.
When reloading the KPI page, retain the user-selected date range and school selection previously made on the KPI page.
