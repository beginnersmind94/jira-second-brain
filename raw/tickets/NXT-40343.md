# NXT-40343 — Custom Reports – Add New Report – Sharing Tab

## Description
As a SchoolCafé user, I would like to be able to share my custom reports so that others can save time from creating their own.

*Notes:*

* We are updating the existing Add New Custom Reports and adding in the sharing functionality to it
* This functionality will be used by action buttons on the dashboard and grid pages
* Try to develop this as something that can be shared in multiple places so that you can code once any changes

*Flow:*

* Open SchoolCafé and browse to Settings > Dashboard > Custom Reports
* View list of reports
* Click + New
* Create the configuration and set sharing and then save
* Return to the updated list of custom reports

*Requirements*:

* These requirements are for the Add New Custom Report overlay
* 
* Add a tab to the Stepper for the Sharing tab if it doesn’t already exist
** Title: Sharing
** Location: 3^rd^ step, behind Configuration
* 
* Tab requirements:
* User flow for this screen:
** Ask user if they want to share this report?
** If yes
*** Give them a role selector
*** Give them a people selector
** If no
*** Hide or make read-only the selectors
* Fields:
* Share Report
** Radio Buttons
** Values – Yes / No
** Required
** Default = No
** If yes, enable the people and role selectors
** If No, disable the people and role selectors
* SchoolCafé Users
** Multi-select list
** Values – active user accounts for the district
** Required if Share = Yes
** No default values
* SchoolCafé Roles
** Multi-select list
** Values – Roles for the district
** Required if Share = Yes
** No default values
* 
* If the Selection screen field PII Data = Yes:
** Show a message below the fields
** Message: This report contains PII data. Users that are not in the PII Role will not be able to view or receive the report even if it is shared with them.
* If the PII Data = No, then do not show a message
* Note: allow them add any user or role. We will control access on the View screen and when distributing
*

* Next



*Review Tab Updates:*

* Add the Sharing fields to the review tab HTML
* Save the sharing information into the database
* 
* Send a notification to any users that were added to share list users or roles when you save the report configuration:
** Text: [logged in username] has shared with you the [name of report] Custom Report. [break] View: [link to the View Report of this custom report]
** Note: Don’t send more than once to a given user (remember they might be in a role)
* 
* Update to the *Delete button on Dashboard*
** Add the Sharing information in the database to the list of things you delete
* 
* Update to the *Edit button on Dashboard*
** Ensure they are there or add all these requirements to the Edit screen
* 
* Update to the *View Configuration Details*
** Ensure the HTML is updated with the changes after you update it here
