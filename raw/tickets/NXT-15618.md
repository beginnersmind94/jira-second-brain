# NXT-15618 — Verification - Sampling - Actions - Edit Sites and Add Dates

## Description
*User Story*

As a district admin I need a way to correct mistakes, update counts, and make sure that my selected sample equals the sample size.

*Mockup*

[https://www.figma.com/proto/Y3tlwN2yvRc67304adzyyB/Eligibility-3.0?node-id=122%3A0&scaling=scale-down-width&page-id=1%3A254608|https://www.figma.com/proto/Y3tlwN2yvRc67304adzyyB/Eligibility-3.0?node-id=122%3A0&scaling=scale-down-width&page-id=1%3A254608|smart-link] 

*Requirements*

*these actions are only available for the current academic year*

* User needs a way to edit the selected/deselected sites.
** Add an ‘Edit Sites’ option off the Sampling page.
** User should be shown the Selected Sites grid as a drawer with the same options as when creating the sample.
** User can Save/Cancel.
*** On Save, it needs to recalculate again and see if enough applications have been selected.
*** If not, then we need to show the user that their total is short and that they should recalculate the sample. The message from above.
*** If a site is deselected and has applications Sampled - move them to Cause.
* User needs an option to add additional sample dates if they selected Rolling Verification.
** User cannot be allowed to add a sample before a date that’s already been generated. 
** Example:
*** Samples exist for:
**** 8/1/21 (Generated)
**** 9/1/21 (Not yet generated)
*** User CANNOT add a sample for 7/31/21.
*** User CAN add a sample for 8/15/21.
* If the system is actively calculating a sample for a district, the actions are blocked.

Success messages:

* Date added successfully. 
* Sites edited successfully.

## Acceptance Criteria
User can edit the selected sites. Recalculation needs to happen when the sites are edited. See linked story 5266.

User can add additional sample dates if they are operating under Rolling verification.

If the system is actively calculating a sample for a district, the actions are blocked.
