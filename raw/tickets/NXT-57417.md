# NXT-57417 — PEI - Auth - Login Authentication

## Description
As a PrimeroEdge Insights (PEI) user, I would like to be able to login directly to the PEI website instead of logging in through the PrimeroEdge website

*Notes*

* We will leverage the SchoolCafé login screen
* We only need to add district field to what is there
* Instead of logging in via KeyCloak, it will have to call PrimeroEdge authentication

*Requirements*

* Copy the SchoolCafe login screen
* Remove the logo
* Change the name to: Insights
* Keep username/login field the same
* Keep password field the same
* Remove the the version and about at the bottom of the screen
* Keep the remember me
* Add the District field
** Data: List of districts in PrimeroEdge
** Make autocomplete list where it filters as you type
** Note: PE is not one consolidated app and some of the districts are on premise, so you will have to figure out how to get a full list of districts
* Update the the Save button
** On click:
*** Validate required fields (District, email and password)
*** Get the district and figure out how to check the username/password on that district
*** Authenticate user
**** If yes, redirect to FO Command Center
**** If no, show failure message (same as SC)
* Update the Forgot Password
** On click:
*** Redirect to PE login forgot password screen
*** Don’t try to implement the forgot password stuff here, just redirect them

## Acceptance Criteria
* User is able to login to any PE district
* User is redirected to FO Command Center when authorized
* User is shown error message if not authorized
* User can go to PE if they forget their password
