# NXT-59554 — SC - Settings to Enable GDSN SSO Login

## Description
As a GDSN user, I want to be able to see/hide the SSO login button based upon a system setting so that I don’t try to login when I can’t.

*Notes:*

* This comes from KP and is needed for the Global Catalogue roadmap item
* We already have a button we created a few sprints back, but we need a way to show or hide it, so we will use a system setting.

*Requirements:*

* In System Settings, add a new property
** Setting Name: Enable GDSN SSO Button
** Code: GDSNSSOC
** Value: Yes/No
** Category: Configuration
** Module: Item Management
** Type: Internal
** Default: No
* 
* Make changes to the Global Catalogue page (see screenshot):
* Add in a check for the property you just created on the logon button
** If yes, show the button
** If no, don’t show the button

## Acceptance Criteria
-Currently there is no option to restrict the usage of SSO or GDSN API Integration-



-Changes Needed:-

-Introduce a new setting as “Enable GDSN SSO Login”, Default Toggle Value: No, User can change to Yes if needed-



-If the toggle value is Yes, then- *-SHOW-* -Log-On GDSN Connect Button-

-If the toggle value is No, then HIDE the Logon - GDSN Connect button-

-This option should be accessible only if primary GDSN permission is enabled-



-QA Considerations:-

-Turn on the setting and try clicking the button-

-Turn off the setting and try clicking the button-
