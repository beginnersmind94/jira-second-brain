# NXT-40204 — Scheduler – Dashboard

## Description
As a SchoolCafé user, I would like to be able to see the history of job scheduler activity so that I can quickly diagnose if there are any issues.

*Notes:*

* The purpose of this page is to show the history of Job Scheduler activities
* It can be used to diagnose if there are any issues
* Multiple features in SchoolCafé can leverage the Job Scheduler framework, and all their activity can be seen here
* It is just like the FTP Dashboard
* There are no add, edit or deletes on this page

*Flow:*

* Open SchoolCafé and browse to System> Support > Job Scheduler
* View list of Scheduler jobs
* Optionally view the details

*Requirements*:

*Monitoring Dashboard Changes:*

* Change the name of the existing card for Job Scheduler
** New title: Monitors
** Leave the description alone
* 
* Add a card to the Monitoring Dashboard
** Title = Job Scheduler
** Description: View details of all scheduled jobs
** On click, opens the Job Scheduler page



*Scheduler Page:*

* Create a new page
** Title = Job Scheduler
* Put tabs
** Start with History tab
** Will add more later
* Filters Section:
* Source
** Multi-select drop down list
** Optional
** Comes from Job Scheduler Source field
** Values:
*** List of unique source names from the job scheduler
* Date Range
** Single select drop down list like what we have on Export Monitor
** Optional
** Values:
*** Last 24 hours
*** Last 7 days
*** Last 30 days
* Status
** Multi-select drop down list
** Values: list of statuses for Job Scheduler
* 
* Create a grid:
* Data:
** List of scheduled jobs executed for the given filters
* 
* Columns:
* Source
* Job Name
* Executed
** Date and time it was sent or attempted to send
* User
** Name of user that created it
* Status
* Actions
** View Details
*** Requirements covered in the View Job Scheduler Details story
* 
* Grid has paging
* Grid has sorting
* Grid has filters
* Grid has export to Excel or CSV
* 
* Security
** Add page to resources
** Security is managed via the resource in Roles & Permissions (R&P)
** In R&P front end, this page shows up under Support > Monitoring Dashboard
** R&P Edit permission should be disabled since there is no edit
** R&P Delete permission should be disabled since there is no Delete
** R&P Add Permissions should be disabled since there is no Add
** R&P View Permissions applied to the whole page



----

*Technical Details*

* Currently the data is fetching from EO_Distribution History. We need to modify the logic to save the history into Azure Table Storage and retrieve the data from there
* Modify Notification Schedule Project to save the distribution history to Azure Table Storage
