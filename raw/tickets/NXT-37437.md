# NXT-37437 — District Onboarding - Enhance new password screen UX

## Description
As a new Onboarding user, I need to be able to easily set a new password, so that I can get started and finished with actual onboarding more quickly.

* N/A 

*Requirements:*

# For new users who have accepted the invitation to begin onboarding, let’s update the Password creation screen UX by doing the following:{color:#ff5630} {color}
## Allow users to see their password as they’re typing it in. We can do this by adding a ‘Show/Hide’ button to the New Password and Confirm Password fields, like we already do on the homepage.
!image-20230313-210716.png|width=1182,height=549!
## Warn users about special character requirements when setting a password (we should verify that every default requirement is shown in the ‘Password must have’ section, not just the 2 shown above. (Also, are we showing updated password requirements here? Not sure if new user screen is using the same UI as the existing users' Change Password screen)
## Add a success toast message when selecting the ‘Change Password’ button.
### Text: ‘New password created successfully!’ This can be shown in a modal instead (with an OK button) as “New password created successfully! You will now be logged out. Please log back in to continue.”
## Add an error message letting the user know if password/confirm password fields do not match-, or if the password selected is the same as the existing password-. 
### Texts: ‘These passwords do not match’. -‘New password cannot be the same as the existing password’-

## Acceptance Criteria
Verify that users can show/hide their password in the New Password and Confirm Password fields on this screen

Verify that users are given the current password rules in the ‘Password must have:' section (may just be sanity testing required)

Verify that users receive an error message if password/confirm password fields do not match, -or if their new password = the old password-.

Verify that users get a success toast message when updating their password and logging in to the Onboarding wizard
