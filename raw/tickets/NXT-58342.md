# NXT-58342 — Elig - Applications - Subscriptions Framework POC

## Description
As Elena the Eligibility Processor I want to be notified whenever there are 10 new FRE Applications waiting to be processed so I can handle them all in one session. Right now I have to check manually multiple times a day.

----

Platform team has recently finished up work on a Subscriptions framework that can be bolted on to various actions with scripts behind them. We can write these scripts and tie them to this new subscription framework, for example: Notify the user after 5 instances of an event/action taking place.

Other use cases for example, on a screen with CRUD operations, they can set it so that a user is notified every time something is created or deleted by adding a listener to the respective Create/Delete actions.

They’d like to see some real world applications for this however, and the following use case should work:

* Will need to setup 3 subscriptions, criteria: 
** Notify the user when they have 5 FRE Applications in status = NEW
** Notify the user when they have 10 FRE Applications in status = NEW
** Notify the user when there are any FRE Applications in status = NEW

Rough workflow on this:

{noformat}add a listener in the message queue
setup a register
	configuration tables already exist
back end
	setup listener
	registers with the scheduler
	runs the query
	hand results to distribution framework
front end
	add My Subscriptions screen{noformat}

Ravi is available to help with KT on this.

----

*requirements*

* User can setup Subscriptions to let them know when 5 or 10 new FRE applications are available for processing
* User is sent an email when this is true and they ask for Inbox/Email distribution: 
** Email Subject: 5/10/X New FRE Applications Available For Processing
** Email Body: Hi $name, it looks like there are 5/10/X new FRE Applications available for processing.

* My Subscriptions available on the Applications grid
* User can setup the following Criteria for Distribution: 5 New Applications, 10 New Applications, Any New Applications
* User is sent an email/toast/inbox msg when the criteria are met
