# NXT-10965 — [TaggedDashboard] - Dashboard (Insights) - Account Management - Enrollment Summary

## Description
As an account administrator, I would like to be able to easily analyze enrollment data for a given date range, so that I can more effectively target my marketing toward certain demographics within my district.

*Summary*:

On the Dashboard, add a section that allows a user to see the number of enrolled students broken down by a variety of criteria.

*Mockup*:

[Link|https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=422818%3A277812&scaling=scale-down-width] - {color:#ff5630}+*_NOTE:_*+{color}{color:#ff5630} _If Pie Charts and Horizontal Bar Chart Components are_ {color}{color:#ff5630}*_not_*{color}{color:#ff5630} _completed in the story book, we can replace them with Donut and Vertical Bar Charts._{color}

*Details*:

* In the Account Management module Dashboard, add a section for *Enrollment* *Summary*
** This section can contain a table/grid/other control that  shows the count of all +student+ accounts in the district grouped/broken down by the following criteria:
*** Eligibility
**** Free
**** Reduced
**** Paid
*** Grade 
*** Gender
**** Male
**** Female
**** Non-Binary
*** Site Type
*** Status
**** Active
**** Inactive
** Note that these widgets do *not* include users or adults, _only_ +student+ accounts.
* This grid does not need to be subject to the view options (last 1, 7,30, X days). It can be beneath the widgets as its own table and display data as of today only--at least for now. It should only be as of Today.

## Acceptance Criteria
This story is complete when a user can navigate to the Account Management Dashboard and view an Enrollment Summary section for their district which contains a count of every student in the district based on the individual criteria above
