# NXT-32252 — Users - Add Columns and Filters for Roles and Sites

## Description
The ability to view and filter which roles and sites users belong to is needed on the Users page.

See attachment for mockup.

!image-20220628-182401.png|width=1310,height=483!

## Acceptance Criteria
* A column shall be added to the Users page which will display which roles the user is a member of
** The column shall be positioned to the right of the “Users Status” column
** The columns header shall be labelled “Role(s)”
** Multiple roles shall be listed in alphabetical order and separated by commas and a space.  Example:  “Director, Cashier, Central Office”
** The end-user shall be able to filter this column
* A column shall be added to the Users page which will display which district(s) or site(s) the user has access to
** If the user’s organization level is defined as “District”, the district the user has access to shall be displayed.
** If the user’s organization level is defined as “Site”, the list of sites the user has access to shall be displayed.
** The column shall be positioned to the right of the “Organization Level” column
** The column header shall be “Access To”
** Multiple sites shall be listed in alphabetical order and separated by commas and a space.
** The end-user shall be able to filter this column
