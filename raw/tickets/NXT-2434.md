# NXT-2434 — Inspections - Create Inspections Screen Slide Out UI

## Description
*User Story:*

As an Inspections user, I have a functional, intuitive form for creating a new Inspection that can be accessed on any screen.

*Mockup:* 

[https://nqyc9v.axshare.com/start.html#p=inspections&id=c7gcdx&g=1|https://nqyc9v.axshare.com/start.html#p=inspections&id=c7gcdx&g=1]

(Click Create Button)

*Requirements:*

* There is a Create button that produces a form for creating a new Inspection
** Create Inspection Form appears if user selects Inspection from the dropdown
** The form slides out from the right side of the screen
** There are two buttons, one of which is context dependent
*** Cancel - Closes the form
*** Start Now - Creates the inspection (If schedule checkbox is not checked)
*** Schedule - Creates the inspection (If schedule checkbox is checked)
* Users can create an Inspection to start immediately
** Requires Create New Inspection permissions (Inspector, Supervisor, Admin)
** Will start immediately if the “Schedule for later?” checkbox is unmarked
** The following fields are present and required:
*** Inspection Name - 150 characters
*** Template
**** Only active Templates are present
*** Site
** Once the Start Now button is clicked, the Inspection is created in an In Progress status
* Users can schedule an Inspection to begin later
** Requires Scheduling permissions (Supervisor, Admin)
** Scheduling is done if the “Schedule for later” checkbox is enabled
** The following fields are present and required
*** Inspection Name - 150 character limit
*** Template
**** Only Active Templates are present
*** Site
*** Inspection Date
**** Earliest possible selection is the next day (Today +1)
*** Inspection Due by (Optional) - Functionality in future sprint
*** Inspector
*** Inform Site? - Functionality in future sprint
**** Radio Buttons - Exact Date, 1 Week Window, Do Not Inform
*** Recurrence - If checked, reoccurs based on user’s settings - functionality in future sprint

## Acceptance Criteria
* Create Inspection form present when clicking the Create button
* User can use the form to create a new inspection that goes straight to In Progress
* User can use the form to schedule an inspection for a future date
* Validation occurs for Inspection Name to ensure no two inspections are named the same for a single district
