# NXT-66346 — GC Portal -Populate Matching Status

## Description
_(none)_

## Acceptance Criteria
*Navigation*: GC Portal - Item Search

*Changes Required:*

The Match status should render only for GTIN match. 

Region ID, Application ID should be included in the matching DB (GTIN_Match_Result)

Each time a search is performed, existing log should be deleted and new log should be created

Search Endpoint needs a application ID as a flag to differentiate the application in which the search is performed

The Matching Status should be populated in following format:

*USDA BF API: Match count is based on USDA BF Source Only*

Total GTINs: XX | Total Matched: XX | No Match Found: XX

*Cybersoft Catalog: Match count is based on Cybersoft Source Only*

Total GTINs: XX | Total Matched: XX | No Match Found: XX



*UI Change*: The matching should be included on the Top of the grid

!image-20250731-031216.png|width=1596,height=233,alt="image-20250731-031216.png"!

*Definitions:* 

Total GTINs - Total number of GTINs input by user

Branded Foods - Total Number of GTIN matched in USDA Branded Foods Source

Total Matched - Total Number of Matched (USDABF+CNDB+Cybersoft)
No match Found: Total GTINs - Total Matched
