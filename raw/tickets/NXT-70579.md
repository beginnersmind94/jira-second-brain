# NXT-70579 — GC- PE - Enhance search by uploading a file match criteria to include new column data ingredient name, manufacturer, brand

## Description
# Add new columns to template manufacturer, Brand.
# match logic will consider ingredient name +manufacturer +brand values only for items for which we didnt find a match using GTIN, Product Code, VIN.
#  use ingredient name, and manufacturer and brand columns data for matching items.
## Name + Manufacturer + Brand -- Match on all 3 (normalized comparison)
## Name + Manufacturer ---Match on Name + Manufacturer
## Name + Brand ---Match on Name + Brand
## Name only --- Match on Name only
## Manufacturer only or Brand only ---Do NOT attempt match
# its not going to be exact match or string matching ,normalize the 3 column values first and then compare - use advanced algorithm to find a match with confidence score. exact matches are going to be matched automatically,
# if only one match found with a good confidence score >=80 use it as a match 
## Include a row for listing the counts matched on Ingredient Name in the match count table.
# If more than one match found for an item list top 3 items in the report and flag them “Needs Review”. 
##  These items are listed in the grid, we can see them in the report under needs review items section. 
## Include a row for listing the Needs Review  in the match count table, 
## needs review count will the number of items from uploaded file with a match using secondary logic 
### 3 items from file - search logic found 9 items in catalog(3 for each item), listed then all 9 items are listed in the results grid and count  3 is listed in the  match status count table  for needs review row,
