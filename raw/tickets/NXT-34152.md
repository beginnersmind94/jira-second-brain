# NXT-34152 — Team Hub - Compose a Message

## Description
As a school admin, I want to view the specific message details and optionally action it so that I can stay up to date with the app and coworkers.

*Flow:*

·         After logging in, user redirected to Home Screen

·         There are six ways to get to the Compose Message screen

o   On Home Screen, in the messages control, click Compose button

o   On Messages Screen, click the Compose button

o   On Message Details Screen, click the Compose button

o   On Message Details Screen, click the Reply button

o   On Message Details Screen, click the Reply All button

o   On Message Details Screen, click the Forward button

·         From any of those ways, it opens the Compose Message screen

·         Admin composes their message

·         Admin clicks Send or Discard

·         Admin redirected to Messages Screen

 

*Assumptions:*

·         User has logged in

·         To code this, you will have to have completed the push / pull messages stories and one of the source triggers (Home Screen or Messages Screen)

·         To code this, you will have to have added the buttons and their actions in the related screens (Home, Messages, Message Details)

 

*Requirements*:

·         Fields:

·         Screen Name: Send Message

·         Back – Image button

o   Use same back button as rest of site

o   On click, redirect user to Messages Screen

·         From – Label

o   Make look like traditional email From field

o   Read-only

·         To – List Box

o   Values: List of district user accounts from System > Sites & Users in the website; show names, not Emails

o   Can select more than one account

o   Allow start typing and select user from the list

o   Validator: Do not allow anyone not in the list of user profiles to be in To list

o   Validator: If there is text that does not validate to a user, then flag the field as invalid

o   Validator: Field is required to have a value

·         Subject – Text Box

o   Make look like traditional email Subject field

o   Editable, single line, max length = ?

o   Validator: Field is required to have at least one character in length

·         Body – Text Box

o   Make look like traditional email body field

o   Editable, multi-line, max length = ?

o   Validator: Field is required to have at least one character in length

·         Discard – Image button

o   Use a traditional delete or trash icon

o   On click, show confirmation message

§  If click No, close message and do nothing

§  If click Yes, continue

o   If confirms discard, the app deletes / forgets this message and redirects the user to the Messages screen

·         Send – Image button     

o   Try to put this button on opposite side of the screen from the Discard button so as to prevent accidentally clicking wrong button

o   On click, do the following:

o   Run the field validators and enforce

§  If fields validate, continue:

o   Once validated and before sending:

§  Set the date / time stamp of message before sending

§  Set State to Unread

o   Once validated, show a circling or progress graphic to the user to indicate to them that the app is trying to send the message

o   Call message push API to send the message to the server

§  Upload the message

§  Receive confirmation

o   When receive success confirmation, hide progress image and show a complete graphic for a short period

o   If receive an error during the process

§  Show a Send Failed error message to the user

§  Let them click OK to move past it

§  Leave user on Send message screen with everything still completed and allow them to decide what to do

§  Clear the date / time stamp value

§  Clear State value

·          

·         Notes:

o   There are no CC or BCC fields

o   Date / time stamp of when message sent is not shown on the screen since message not sent yet

o   Message State is hidden since not sent yet

·          

·         Biz Logic:

·         You need to know what button sent the user to this screen and act accordingly when the screen loads:

·         Compose

o   Used on Home, Messages and Message Details

o   If comes from this button, do the following:

o   Insert Admin’s account in the From field

·         Forward

o   Used on Message Details

o   If comes from this button, do the following:

o   Insert Admin’s account in the From field

o   Leave To field blank

o   Insert the Subject Line from the original message and put a “Fwd: ” in front of it

o   For the body field, do the following:

§  Copy the way Outlook and Gmail do it

§  Insert some space from the type

§  Insert the From, To, Subject and date time stamp below the space

§  Insert the body from the original message below the From, To, Subject and date

·         Reply

o   Used on Message Details

o   If comes from this button, do the following:

o   Insert Admin’s account in the From field

o   Insert the Subject Line from the original message and put a “Re: ” in front of it

o   Insert the original sender’s name in the To field

o   For the body field, do the following:

§  Copy the way Outlook and Gmail do it

§  Insert some space from the type

§  Insert the From, To, Subject and date time stamp below the space

§  Insert the body from the original message below the From, To, Subject and date

·         Reply All

o   Used on Message Details

o   If comes from this button, do the following:

o   Insert Admin’s account in the From field

o   Insert the Subject Line from the original message and put a “Re: ” in front of it

o   For the To field, do the following:

§  Insert the original sender’s name in the To field

§  Insert all the other people from the original message’s To field

§  If you can, remove the Admin’s email from the To field

o   For the Body field, do the following:

§  Copy the way Outlook and Gmail do it

§  Insert some space from the type

§  Insert the From, To, Subject and date time stamp below the space

§  Insert the body from the original message below the From, To, Subject and date

·          

·         Security

o   Copy all the security on the Home Screen and do the same for this screen

o   If user not logged in, redirects the user to the login screen
