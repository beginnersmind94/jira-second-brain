# NXT-31555 — Warn Cashier to Check Camera Alignment When Login

## Description
As a cashier, I want to be reminded to check camera alignment and to be easily able to do it so that I don’t forget and end up with bad pictures all session long.

*Application*: POS

*Summary*

We will add in code to the startup of POS that checks if it is in food detection mode and show a warning screen reminding the cashier to ensure alignment of the camera and if possible show them a shot of it or the live stream of it or at least takes them to the camera alignment screen.

*Flow:*

* When the cashier opens POS and after the rest of the loading occurs:
** If Food detection is on for the terminal, the system shows the cashier a message to verify camera alignment
** Cashier optionally views image or goes to the alignment screen
** Cashier closes message
** Else, the system does nothing

*Assumptions:*

* This was completed in classic and you can copy the code from it
* Get with Saketh on the classic code
* If you are going to show the image, you can call Amniscient api to get the image
* POS subscribes to the correct MQTT Topic and send start alignment trigger to the Edge and once alignment is done, send STOP message

*Requirements:*

* When the cashier opens POS, wait until all the other pre-processing and loading is complete and then add in the following
* Check if Food detection is enabled for the terminal
** If No, do nothing
** If yes, display the message box / screen
*  
* Message screen minimum functionality:
* Show message box
** Open a popup message with text and a close button
** Text: Please confirm the food camera detection is properly aligned.
** If possible, would like a link to go to the camera alignment screen
** Allow them to hit OK or close button and close the popup
*  
* Message screen preferred functionality:
* Show message screen
** Open a popup screen with text, a close button and a placeholder for a live feed from the camera
** Text: Please confirm the food camera detection is properly aligned.
** Show the live feed from the camera like you do in the camera alignment tool
*** Basically, show that functionality in the popup minus the text from that screen
*** If do this, don’t need link to camera alignment screen

·         API Flow per Amniscient:

·         POS subscribes to the correct MQTT Topic

·         POS sends start alignment trigger to the Edge

·         POS sends STOP message once alignment is done

* Allow them to hit OK or close button and close the popup
*  
* Security – no additional requirements necessary
