# NXT-3447 — System - Notification Templates - Allow user to edit a Notification Template

## Description
As a district administrator, I would like to be able to easily manage all of the letter templates (for letters that will be going out to my parents and students) in one place, so that I can easily make changes as needed.

* In *Notification Templates* page, when the user selects a letter, show a modal/window to allow them to edit the content of that letter
* We can retain the rich-text editor of the PrimeroEdge Classic version, and any upgrades that can be made should be acceptable
* We should also retain the same list of fields, which are pulled from PrimeroEdge. {color:#ff5630}This part will require lots of testing.{color}
* Fields will be dependent on letter type.
* Please note that letters are often printed in large bulk (10,000+ letters) and with images, so files should be as light as possible.
* When the user (with edit permission) makes changes to the content of the template and save it, the changes made should be reflected when selecting this letter again from the list and (in the future, per NXT-3451) the History tab.

## Acceptance Criteria
Verify that a user with permission can view the letter selected on the Notification Templates page.

Verify that a user can edit the details (including fields) of the letter and have it reflect appropriately.

Verify that when the user saves the letter, all changes are reflected in the Notification Templates list and when selecting the letter again.

Verify that fields display appropriate data when an actual letter is printed (this step by have to be deferred until other notification pages are done)
