# NXT-35775 — System - Users - Allow option to enable new user account without email verification

## Description
As a central office user who creates new user accounts I want the ability manually activate a user’s account and bypass the need for them to validate via email because some of my users do not get an email account in time from the IT department for them to start using the system when I need them to.

Details:

* District employees are often needed to work immediately. Often, the IT department falls behind creating their email account because they are just cashiers who don’t usually have a need to do their email.
* The need is to allow (if desired) the central office to manually make the account active so that the user can login and do their work. If the central office chooses, they can leave it as is and use the standard method.
* Keycloak should honor the selection of account activation just as it would the email confirmation from the user.

## Acceptance Criteria
* When creating a new user, you should be able to set the active flag on the user account which enables immediate use by the user
* When editing a user account, you should be able to set the active flag on the user account which enables use by the user
