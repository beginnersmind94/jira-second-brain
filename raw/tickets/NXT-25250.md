# NXT-25250 — IM > Menu Item Report (with multiple options)

## Description
A report is needed to obtain a list of Menu Items that have been created in Item Management, with some of the following options available:
1.) Those that do not have POS Pricing set up
2.) Those that DO have POS Pricing set up, -and what those prices are- 
3.) A view containing ALL menu items with both recipe serving size and Menu Item Serving Size(s) displayed so that users can look for any inconsistencies
-4.) A list of menu items with their serving size(s) and component information- (research out scope for now)
-5.) Whether the menu item is a dual entry or single ingredient recipe-

Ideally, a page similar to some in other areas of SchoolCafe that allow the user to select what columns they would like to have displayed on the page, to include all aspects of the menu item section of recipes in Item Management.

## Acceptance Criteria
* Users should a page similar to some in other areas of SchoolCafe that allow the user to select what columns they would like to have displayed on the page, to include all aspects of the menu item section of recipes in Item Management. 
* User Shall be able to filter Menu Items by
** *Filters:*
*** Item Category (Menu Item Category)
*** POS Pricing (Has Items with POS Pricing setup / Has Items without POS Pricing setup)
*** Status (Active / Inactive)
*** Menu Item Name/Code
*** Preparation Site (Flag default set false)
*** Include recipe Serving Size (Flag, default set to true)
*** Favorites (Flag default set false)
*** Summary (Flag default set false)
* Columns
** As per mockup
** Menu Item Name/Code (Default - always on)

*General*

* Use default serving size 
* Allergen \[ is a concatenated string]
* Display recipe serving size on  report grid
* Menu Item flags should be part of the column & views  \[Favorites, prep-site, summary]
* Menu Item Type  (Menu Item / Variety Menu Item)
* Consider only single ingredient menu items
* Add POS pricing flag (indicate if it has price \[Set (green) or Not Set(red)-)
* Report Shall follow all header and footer standards

*Export*

* User shall be able to export search results (PDF or Excel format) 

*Note:*

* Content under mockup fields do represent actual values
* Screen navigation shall be keyboard friendly
