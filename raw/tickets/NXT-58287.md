# NXT-58287 — Acc - Edit Checks Page - Reverse Edit Checks - Calculations

## Description
As Dana the Director I want a way to focus on sites that are well under the expected meal count based on their monthly average, and a list of sites that fall into this category would speed up my workflow.

*requirements*

* User can set and save the average number of meals by site, by meal type.
** User can also ask the system to calculate this number over the last 30 days.
** User can see the meal types the district has served over the last 30 days - these will be the only meal types shown to the user.
** User can leave an average meal count blank for a site type + meal type combo.
*** Example: Texas Elementary, Breakfast, can be left blank. This will then be ignored for search results.



----

* *Results Grid*

||*Site Code*||*Date*||*Meal Type*||*Meal Count*||*Site Avg Count*||*% Under Avg*||*Options*||
|163 - Texas Elm|01/17/2025|Breakfast|0|89|100%|Sessions|
|163 - Texas Elm|01/17/2025|Lunch|150|300|50%|Sessions|

So for 163 - Texas Elementary:

* User has configured the Site Average Count for:
** Breakfast → 89
** Lunch → 300
* They could’ve had the % Under Average Meals set to 50% and hit Apply.
* The results grid would’ve shown them that Breakfast was 50% under, and Breakfast is 100% under average.
* If they set the % Under Average Meals to 51% they would have only seen Breakfast.



*Calculations*

* Calculating Average Meals for the user:
** Check the last 30 days
** Sum how many days they’ve served each meal type
** Sum the meal count
** Average = (Total Meals by Meal Type) / Number of Service Days
** Save this number.
** User can override and manually set a different number.
* % Under Avg
** (Meal Count / Avg Meal Count) * 100
*** (150 / 300) * 100 = 50

Discussion at Refinement (1/21/25)

* Rounding → go up to the nearest whole number
* User can leave a site average blank - we will ignore this meal type when finding results.
