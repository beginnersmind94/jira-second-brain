# NXT-20636 — Platform - BO - Onboarding Imports Details Page

## Description
*Persona*: CyberSoft/ISD Administrator

*Access level:* School District/Central Admin

*Description*:

As a school admin, I should be able to view system Imports summary widgets with its overall progress and take necessary actions

*Mockup:*

[https://www.figma.com/proto/PLmXuylGMphY0Bz7WBDCOp/Systems-4.0?node-id=637%3A9929&scaling=min-zoom&page-id=548%3A8289|https://www.figma.com/proto/PLmXuylGMphY0Bz7WBDCOp/Systems-4.0?node-id=637%3A9929&scaling=min-zoom&page-id=548%3A8289|smart-link] 

*Requirements*:

* Site admin logs into Onboarding Summary Dashboard page for Imports widgets displayed in “Dashboard” tab
* Imports tab displays all Imports that has to be configured for given district
* Imports table displays all imports progress % for each import along with actions
* Imports widget has VIEW DETAILS button which opens Imports tab in the onboarding page.

*Progress Calculation*

·        Each row is 20% aggregates to 100%

*Updates:*

* *Columns* should be displayed as:
** *Activities*
** *Status*
** *Actions*
*** Remind Me
*** View
* Activities should include
** Sites Import
** Student Import Configuration (SIS)
** Student Data Imports (SIS)
** Eligibility Import
** Balance Import
* Each Activity must be follow the precedence
* Without Sites import, Student Import configuration should be greyed out and vice versa for Eligibility, Balance imports
*  Once Site Imports is partially complete (atleast one site) Student Imports should be enabled for configuration
* VIEW should open that config page for the respective activity in a pop-up “*Configuration*” screen of school café
* Student Import Configuration screen when the user clicks on VIEW button of Student Import configuration in SchoolCafe
* Each pop-up for the VIEW should contain CLOSE button for user to return to Onboarding dashboard

## Acceptance Criteria
* User shall be able to see the Imports widget in Onboarding dashboard with activities
* User shall be able to see VIEW ALL button to open *Imports* *Config* page in SchoolCafe
* User shall be able to see status of the imports for each import type with hierarchy
** Ex: When user did not setup Site or Students import, balance/eligibility should be greyed out
* User cannot take action unless  pre-requisite configs are complete
* User shall be able use “Remind Me” action to set a reminder for themselves to receive notification
* User shall be able to view and close the import config to return to onboarding dashboard
* User shall be able to click on VIEW action to bring up respective Import Config page.
