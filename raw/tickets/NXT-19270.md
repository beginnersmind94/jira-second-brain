# NXT-19270 — INV - UI - Receipt Detail - Create/Edit Receipt - Remove Broken Units (UI)

## Description
As a Site Manager I want the Receipts Detail screen to not include broken units so I can see accurate item information when viewing receipts.

*Requirements*

* +On Receipt Detail screen+ - in *Create/Edit* mode
* Remove Broken Units from the 2 quantity fields (Order, Receive)
** remove label text in parentheses ‘(WU, BU)’
** remove BU textbox from Create/Edit screen
*** also remove from the Lot Number sub-grid
** Update to remove any references to Broken Units, as Broken Units will no longer exist
** for the unit label (e.g. Basket, Bag, etc.), make sure this is referencing the new ‘Unit Description’ free text value (not the ‘Whole / Purchase Unit’ dropdown value which INV1 is removing)
* Add a ‘Product Type’ column
** This will display one of 6 options: Pallet, Mixed Pallet, Display Shipper, *Case, Package, Each* (bolded ones are most common)
*** This will be pulled from the details of the item in Item Management (See image below)
** display this column after Item Description column, before Vendor
* Group related items together in the grid
** Items which are linked together from Item Hierarchy in Item Management should be displayed as consecutive rows
** Sort the *related* items by Product type descending (Pallet > Case > Package > Each)
* Change ‘Unit Price’ to ‘Item Price’
* Move ‘Vendor Item #' column after Item Description column
* *Possibly* remove Item Category column
** Only if there is not enough width/space after Broken Unit changes
* In the header, ‘Receipt Location’ should simply say ‘Site’
** this should be the case *everywhere* on the order/receipt

+On Add Item screen+

* Change Item Number to Item #
* Change Vendor Item Number to Vendor Item #
