# NXT-3555 — System - Management - Program Configuration - Add Programs tab

## Description
As a Central Office Admin, I would like to be able to easily mark program classifications for my entire district, site by site.

*Summary*: Allow the user to configure the program classifications for their district, and which sites are for feeding and reimbursement, for a specific academic year.

Requirements:

* Allow the user to select the ‘Programs’ tab of the Program Configuration page (or as default landing tab) and be able to manage:
** program classifications for the district
** individual site classifications, along with feeding site and reimbursement status.
* Academic Year:
** All options should be saved for the specific academic year. Allow the user to choose their academic year if desired (but display the current academic year by default)
* Program Classifications:
** Display all programs that exist nationally (*the full list is unchangeable seed data and can be copied from classic*)

* Allow the user to toggle on/off participation in a specific program
* Add the ability to do a mass-update on the program level to change all of the toggles nested below. i.e. If the user changes a classification at the program-level, update the classification of all related sites as well. User should also be able to go to an individual site and change its classification if needed.
** Note that NSLP has these feature already, but we need to implement it in a different way for the other programs.
* Site-level Classifications:
** If the program is configured for the district, allow the user to toggle on/off participation for a specific site (same drop-down options as NSLP above)
** Allow the user to toggle Feeding site status on/off for the site
** Allow the user to toggle Reimbursement status on/off for the site

* Updates should happen instantly (no save button required) for the academic year selected.
* Programs should display in the same order as shown in Classic (or the order above). 
* Sites should display in alphabetical order.

* *Not required*: 
** 'Copy configuration from' option. This will be addressed in a separate story.
** Summary of how many sites are Configured/Not Configured for a single meal type. (see Classic screenshot #2) 
** All Sites/Active sites radio buttons. (we will have a search functionality instead)

## Acceptance Criteria
Verify Programs tab contains new features explained above

Verify an individual academic year can be selected, with the data displaying based on the selection.

Verify drop-down shows, and data displays based on, current academic year by default (academic year can be displayed in descending order from the current year)

Verify full list of programs shows for the district

Verify program configuration can be toggled on/off for each program

Verify program configuration can be toggled on/off for each site

Verify that if the user changes a classification at the program-level, we update the classification of all nested sites as well. 
Verify that the user is still able to go to an individual site and change its classification if needed.

Verify that feeding status can be toggled on/off for each site

Verify that reimbursement status can be toggled on/off for each site

Verify that updates happen immediately
