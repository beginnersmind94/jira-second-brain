# NXT-70454 — SCTV+ - [Big Card Menu Planning Widget ] Add 2×3 Column Support and Dynamic Scaling to Big Card Type (Max 5 Items per Widget)

## Description
h3. Story Description

This PR enhances the *Big Card Menu Planning widget* by introducing support for a *2×3 column layout* along with full dynamic scaling capabilities.

The goal is to improve layout flexibility, increase scalability, and ensure rendering stability across different widget and item configurations.

----

h3.  Key Enhancements

* Added support for *2×3 layout per column* (in addition to the previously supported 2×2).
* Enabled *vertical and horizontal widget stacking* with dynamic layout behavior.
* Each widget supports *up to 5 items per widget* in the standard configuration.
* Implemented validation rules to prevent invalid widget combinations that could cause rendering or UI layout issues.
* Improved overall scalability of the Big Card layout to better handle larger content sets.

----

h3.  Component-Level Improvements

All components inside the *Menu Item View* were updated to be fully flexible and scalable, including:

* Image frame
* Food components
* Calories
* Title
* Description

Each component dynamically adjusts based on:

* Number of widgets displayed
* Number of items inside the widget
* Stacking direction (vertical or horizontal)

This ensures proper spacing, alignment, readability, and visual consistency whether the layout contains fewer widgets or higher-density configurations.

----

h3.  Scalability Improvements

* Menu items are now fully scalable, allowing users to add the maximum supported number of items per widget.
* Layout behavior adapts automatically to maintain UI stability and prevent overflow or rendering issues.
* The system now better supports larger and more complex widget configurations without breaking the UI.

----

h3.  Outcome

Deliver a more flexible, scalable, and stable Big Card Menu Planning widget that supports expanded configurations while maintaining clean and consistent rendering across all supported layouts.

## Acceptance Criteria
h2.  Acceptance Criteria

h3. Layout & Configuration

# The Big Card Menu Planning widget supports a *2×3 column layout* per column.
# The system allows *vertical and horizontal stacking* without layout breaking.
# Each widget supports *up to 5 menu items*.
# The layout renders correctly when:
#* Only 1 widget is present
#* Multiple widgets are present
#* The maximum supported configuration is used

----

h3. Dynamic Scaling

#* Number of widgets displayed
#* Number of items inside the widget
#* Stacking direction

#* Image frame
#* Food components

#* Title
#* Description

----

h3. Validation & Stability
