# NXT-32222 — Create/Change Password - Display Acceptable Special Characters

## Description
When a user creates a new password or changes an existing password, we need to show the user which special characters are acceptable so that they don’t have to guess.

+*Existing UI*+

!image-20220624-172237.png|width=1273,height=599!

## Acceptance Criteria
* In the password change/create UIs, the text “At lease x special character” shall be changed based on the below conditions.
** If the “Minimum Special Characters“ value for the password policy in effect is 1, the message shall be:  “At least 1 special character (>, <, and % are not allowed)”
** If the “Minimum Special Characters“ value for the password policy in effect is >  1, the message shall be:  “At least x special characters (>, <, and % are not allowed)”
*** where “x” is the number of characters associated with the selected password policy
