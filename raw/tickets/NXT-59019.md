## Acceptance Criteria
*Navigation*: SC - Items - Item Upload - Item Creation:

Currently, the Item Creation is manually exported from GDSN Connect and imported.

As a change, this feature shall fetch the item details via API and insert the data into staging table directly

*Use Case Flow:*

The user should be able to upload the vendor item upload template as attached

[^Vendor Upload - 100 items - Part1.xlsx] 



The SC UI shall have new radio button called as “Vendor Item Template” 
By uploading the file, the information should be translated in JASON Format as attached.
[^sample jason export.txt]  



The system should call the Item Sync API (GDSN Connect) and sync the data in the corresponding region.

TBC - how do we keep checking for matching status?

Upon returning the results, the data should be translated in GDSN Template format and inserted into staging table (template attached for reference purposes only)
[^ItemsExport_202504151757_SchoolCafe_v1.0 - 10 items.csv] 


The inserted data should be parsed and processed as Translated Data and Exception Log downloads

Regression:
Once Processed, user can download the translated data template and proceed with import
User can also view the logs via user interface (view icon)
