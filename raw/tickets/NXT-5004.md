# NXT-5004 — Data Exchange - Configured Field Validations

## Description
*Summary*

As a PrimeroEdge user making an import, I have a pre-check present in the file mapping screen to alert me to any issues in my data.

*Requirements*

* System should  check to validate the mappings for below criterias
** Errors:
*** Missing entries - Indicates lines that will not be imported if it is a required field{color:#36b37e}*-- (Taken care in Field Mapping page)*{color}
*** Duplicate entries - By row, or for a field that must be unique{color:#ff5630}*--Need discussion with Amar and Bhanu*{color}
*** Wrong data type - E.g. Letters in a phone number field{color:#36b37e}*-- (Taken care in Field Mapping page)*{color}
*** Data fields are exceeding the allowed character limit{color:#36b37e}*--(Will be taken care in respective staging table)*{color}
*** *D*ata fields with special characters{color:#ff5630}*--Need discussion with Amar and Bhanu*{color}
*** Data is not matching with configurations{color:#36b37e}*-- (Taken care in code, column mapping is created in json format with respective data types)*{color}
*** Data field is set for “Not in Use” or “Ignore” option set for mandatory fields{color:#36b37e}*-- (Taken care in Field Mapping page)*{color}
* User should be to update configuration any number of times by clicking the field{color:#36b37e}*--(API already available to insert and update)*{color}
* Merged fields data types should match{color:#36b37e}*-- (Data type will always be string)*{color}

## Acceptance Criteria
1. Validations can be run and successfully outputs errors by the appropriate columns on the File Mapping Screen'
2. Validate appropriate error message is shown when config is not allowed (displayed in RED)
2. Successful validations enables green checkmark right next to the field
3. When validation fails, Mapped field should be made available in DRAG column (available fields)
4. Disable "Next" option for user until all validations are resolved
5. Data Format mismatch must be validated
6. Allow users to configure "Not In Use" or "Ignore" for non-mandatory fields
