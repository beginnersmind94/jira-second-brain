*Requirements:*

* Add the following options for the Edit Check Worksheet / Report:
** Exceptions Only
*** This would filter the report down to only days where either the meal count is greater than the attendance or eligible counts.
** Exclude Non-Serving Days
*** This would filter the report down to only days where some type of service happened.
**** Believe we will want to check for the presence of any type of sale or transaction.



Collating some information from the other stories for UI purposes:

Page options:

* Area selector
** If they’re not using areas, we don’t need to show this.
** All would be the default.
* Site type selector
** All would be the default, otherwise check the configured site types.
* Site / Site Code selector
** All would be the default.
** Matt: I’m not clear if we have the ability to check site permissions yet. If we do and we can use it, let’s do so.
* Program selector
** The available choices here will be SNP and SFSP. No All option. SNP default.
** These should check to see if they’re configured at the district - if they’re not configured, then we shouldn’t show the programs.
* Meal Type selector
** This is tied to each program, and also to whether they’re configured or not.
** All can be the default selection here.
* Report Options
** Edit Check Worksheet
*** This will cause the ‘Generate Report’ to pull the Worksheet instead of the Report.
** Exceptions Only
** Exclude Non-Serving Days

Mandatory page selectors:

* All dropdowns must have a selector.
* A date (or range) must be selected.
** Date range should not span more than a single year.

Notes about options layout:

* Report options embedded in an accordion, open on page load.
* When the user hits generate report, the accordion should close.
* The report will be embedded on the page, and if possible, we’ll scroll down to the top of the report.

## Acceptance Criteria
User can filter the report for Exceptions only.

User can filter the report to serving days only.
