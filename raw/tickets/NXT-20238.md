# NXT-20238 — INV - Withdrawals - Create Withdrawal - Production Type (Step 1)

## Description
As a production manager I want a Production type withdrawal so I can withdraw items for all the meal lines for a single day.

*Note:* Production Withdrawals may result in items being withdrawn in decimal values, therefore [https://primeroedgenext.atlassian.net/browse/NXT-19983|https://primeroedgenext.atlassian.net/browse/NXT-19983|smart-link] is a prerequisite for this story

*Requirements*

* Add logic to the ‘Withdrawal Type’ selection on the first step of the Create Withdrawal function to consider Production selection
** use radio option as long as we only have 2 
* Change the below selections if Production is selected as the Withdrawal Type
** Provide a ‘Production Date’ date picker
*** *Note: this is not a transaction date/time. This is the date to grab the production data.
*** only a single day date selection for Production Withdrawals
*** *enhancement: showing the actual weekday name as well
*** Only current/past dates can be selected (no future dates)
*** Only ‘Record Complete’ menu lines can be pulled from Production
*** Show an error and prevent/disable Next button if there are no valid production records
**** no ‘Record Complete’ menu lines, OR
**** all menu lines already withdrawn
** Remove Transaction Date & Time
** Remove Reason
** Remove Comments
* +UI Changes+ (separate story for Transfers & Additions)
** Center align header
** Remove white box for background (only ‘Step 1’ should be boxed)
** Use sticky footer
* Proceed to a different second step screen if Production has been selected as the Withdrawal Type ([NXT-25244|https://primeroedgenext.atlassian.net/browse/NXT-20238])
* Possible error messages:
** No valid Production Records found (no records at all, or no records in ‘Record Complete’ status)
*** Also for invalid Production dates (weekends, holidays)
** All Production Records are already withdrawn
