## Acceptance Criteria
Navigation: DB Enabler

Changes Required:
Clone and Use dbo.branded foods table (from CN26)

For each GTIN Matched, Create a custom end point that inserts the API response into a new table as brandedfoods_consolidated and insert 1 row per record that has all the fields in the API response. 
Note- this table shall be act as foundational db to avoid future API calls.


*Usage of Serving Size:*

The actual serving size and its corresponding nutrition values should be used in the API, if not available 100 Gm equivalent measure can be used. If both are available, both the serving size and corresponding nutrient should be rendered in API

While inserting the DB, if the GTIN is already present in the table, only update the record.

Have additional fields like created update, updated date and updated user etc
