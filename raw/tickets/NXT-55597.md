# NXT-55597 — Subscriptions Changes - My Profile & Users Page

## Description
When a subscription event is added by an Admin, we should not allow those events to be edited by the user. In order to restrict the users to delete/edit the events created by Admin we need to explicitly specify if a subscription is tied with more than 1 user or tied to a Role. 

This can be done by adding a new column to the existing grid (similar to CustomViews as Global (G) and Mine as (M)))

From Backed verify if the Subscription is tied to more than 1 user or tied to a Role then return SubscriptionMode as G otherwise return as M

In frontend on UsersPage & MyProfile page Add a new column and show Subscription Mode as G or M. If it’s G don’t allow user to edit/delete. If it’s M then allow user to edit/delete

While modifying also verify once update there is no duplicate entries possible

## Acceptance Criteria
No duplicates needs to allow. If a subscription event is tied with more than 1 user or a role then don’t allow to edit/delete
