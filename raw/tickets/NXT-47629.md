# NXT-47629 — Insights: FO - KPI Report (API) - Inclusion of Site ID and Site Type ID as Parameter in API

## Description
Currently the Insights API runs the aggregate based on district ID. when we need to run this for site level or site type level we do not have a parameter to fetch the aggregates.

The aggregates are fetched at monthly and weekly ranges to project the trend on Insights

## Acceptance Criteria
The following API should have a provision to extract the aggregates based for a specific site ID (Ex: MOUNTAIN VIEW ELEMENTARY SCHOOL) or Site Type ID (Ex: High School)

*​/api​/InsightsAggregate​/MealEquivalentAggregate*

/api/InsightsAggregate/MealAdpAggregate

*Impact of Not having Site ID and Site Type ID in API:*

The page goes to not responding / API gets Timed out mode as the aggregates are fetched for the all the sites even the UI needs the values for one site ID or site type ID at a time
