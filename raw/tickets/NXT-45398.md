# NXT-45398 — Imports - GDSN Connect - Backend

## Description
As a SchoolCafe admin, I would like to be able to import the GDSN Connect Items Export from InTeam so that I can include them in our Global Catalogue.

*Notes:*

* This story is a backend part of the GDSN Connect Items export
* The front end part is in another story
* This story is for the InTeam Items export only and not the InTeam Recipes export
* Try to develop the backend so that both Manual Import and API Import can use the same
* A lot of these requirements reference the InTeam specifications which are attached to this story

*Assumptions:*

* Data Exchange team will import the raw data into a temp database
* Back Office team will consume our imported data and make any transformations / changes necessary 

*Requirements:*

* Create the database tables and data necessary for front end Field Mapping to use it
* Create the database tables to store the GDSN Connect data on Import
* Use the ITEM Data v4.0.0 table on pages 19-35 of the attached requisites document
* Use the Field Name column for the column name
* Use the Data Type column for the SQL data type
* Use Step 2 (Transfer) to determine if the field is required
** use M only for Required
* 
* Write the APIs so that the front end can use

## Acceptance Criteria
* Tables exist to support file mapping
* Tables exist to store data on import
* APIs exist so front end can consume
* All the fields exist with correct datatypes
