# NXT-5205 — Application Details - Household Members

## Description
*User Story* 

As an application processor, i can see basic application information on the Application Info tab. 

*Mock Ups* 

[https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=87%3A2599&scaling=min-zoom|https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=87%3A2599&scaling=min-zoom]

*Requirements* 

* Household Members (section) should match what’s in the application and guided based on settings and conditionals 
** For example: total child income or alaska PFD setting is enabled, the appropriate UI for that setting and captured data will show here. 
** Name, Income Type, Amount, Frequency, HH Size, total Income (calculated)
*** With respect to conditionals/settings
* HH size and income not required for categorial: assistance program applications
** -_This section is not required due to the application basis._- _This section is not required due to the basis of the application._
** If the user adds HH info for a categorical: program app, we will show info provided, but that’s an edge case.

## Acceptance Criteria
- Fields above are shown in the household members section 
- section is reflective of whats in the application (settings/conditions)
- Multi-language, ADA compliant.
