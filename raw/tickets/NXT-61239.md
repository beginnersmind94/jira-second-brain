# NXT-61239 — WS-PE-Insights-AI Insights Summary on Home screen

## Description
As an Insights role user, I would like to see a summary of my performance generated by AI in natural language on the home screen so that I can get more understandable feedback.

*Notes:*

* This story is for adding the natural language summary to the top of the home screen
** Not Insights screen, but the Workspace home page

*Flow:*

* User browses to the home page
* System preps the data using the site access and a calculated time range
* System sends the data and a prepared AI prompt to the AI engine
* AI engine analyzes data using prompt and returns a paragraph style text summary
* System shows the summary to the user

*Requirements:*

* On the home page in Workspace, do the following:
* 
* Summary Insights section 
** Add the AI summary section to the page per Dallas’s Bolt mockup 
** It is a summary (paragraph) text section Should have expand / collapse (show/hide) ability Default to show expanded 
** Do not show the section if the district has AI turned off 
** There is a system setting for districts to enable\disable AI 
** If disabled, don’t show the AI section 
* Data Set 
** The results are dependent on the user’s site access 
** If the user only has access to one site, results show only for that one site, etc. 
** The results are dependent on time as follows: 
*** If after 10:30 am, show results for Today 
*** Else, show results for previous operating day 
**** Note: For Sunday and Monday, that would normally be Friday if it wasn’t a holiday 
** Pass in a dataset with these site and time filters 
* On page load: 
** System gets the dataset using the site and time filters 
** System sends the dataset and a previously prepared AI prompt to the AI engine to perform the analysis 
** AI engine does its analysis and returns the result 
** System loads the result in the AI section 
** Show a loader icon while waiting on the AI engine. If you can, include text to let them know we are waiting on the the AI engine.
* 
* Prompt Text:
** Ask AI to summarize in paragraph format what went well and what didn’t go well for that site(s) 
* 
* Add security to the button:
** This section inherits the page’s role based security, no additional security needed beyond the AI district check
* Error Handling:
** If there is an error in retrieving or processing the AI summary (e.g., failure to contact the AI engine), an appropriate error message should be displayed to the user.
* System Performance:
** The system must load the AI-generated summary within a reasonable amount of time, aiming for a smooth user experience without significant delays.

## Acceptance Criteria
# *AI Summary Section on Home Page*:
#* The AI summary section must be added to the top of the home page in Workspace as per Dallas’s Bolt mockup.
#* The AI summary must be a paragraph-style text section that summarizes the performance data.
#* The section should have expand/collapse (show/hide) functionality.
#** The default state of the section must be "expanded."
#** The user should be able to collapse and expand the section by clicking a button/icon.
#* The AI summary section must only be displayed if AI is enabled for the district (based on the district’s AI setting).
#** If AI is disabled, the section should not appear on the home page.
# *Data Set Handling*:
#* The AI summary content must be based on the user’s site access:
#** If the user has access to multiple sites, the summary should aggregate results for all sites the user has access to.
#** If the user has access to only one site, the summary should display results for that site.
#* The summary should also depend on the time:
#** If the time is after 10:30 AM, the summary should show data for "Today."
#** If the time is before 10:30 AM, the summary should show data for the "previous operating day."
#** For Sunday and Monday, the results should default to the previous Friday (unless it’s a holiday).
# *AI Engine Integration*:
#* On page load, the system must:
#*# Retrieve the dataset using the site and time filters.
#*# Send the dataset along with a prepared AI prompt to the AI engine for analysis.
#*# The AI engine should return the following:
#*#* A paragraph-style summary of what went well and what didn’t go well for the site(s).
#*# The system should load the returned result into the AI summary section.
#* A loader icon should be displayed while waiting for the AI engine to process the data.
#* Optionally, the loader icon should include text (e.g., "Analyzing data...") to inform the user that the system is waiting for the AI engine’s response.
# *Role-Based Security*:
#* The AI summary section must inherit the page’s role-based security, with no additional security required beyond the AI district setting.
#* Only users with the appropriate role and district AI settings should see the AI summary section.
# *Error Handling*:
#* If there is an error in retrieving or processing the AI summary (e.g., failure to contact the AI engine), the system should display an appropriate error message on the home page.
#* The error message should inform the user of the issue and suggest any necessary steps or contact points.
# *System Performance*:
#* The system must load the AI-generated summary within a reasonable amount of time to ensure a smooth user experience.
#* The system must not cause significant delays or performance issues during the page load or while waiting for the AI engine’s response.
# *Prompt Text for AI Engine*:
#* The prompt text that will be sent to the AI engine should request the AI to summarize in paragraph format what went well and what didn’t go well for the site(s) specified.
#** Example prompt: "Please summarize in a paragraph what went well and what didn’t go well for the given site(s) based on the provided data."
# *Behavior on Expand/Collapse*:
#* Clicking the expand/collapse button must toggle the visibility of the AI summary section with a smooth sliding motion.
#* The section should default to expanded when the page loads, but users should have the ability to collapse and expand it.
# *Prompt Handling Based on Time and Site Filters*:
#* The system must ensure that the prompt to the AI engine is dynamically created based on the current site access and time frame selected (as described in the Data Set Handling section).
#* The system should pass the appropriate site and time filters when making the request to the AI engine.
