# NXT-47954 — POS - Picture Mode for Bulk Sales

## Description
As Carmen the Cashier I want to see the student photos while performing a bulk sale as it is a quicker identification method and will increase productivity. As Sally the Sales Engineer it is also a good marketing feature.

*mockup*

[https://www.figma.com/proto/MFg1OFhP2GtXhGajDnPKRC/Dec-2023-Systems-FO?page-id=1943%3A16680&type=design&node-id=1943-16681&viewport=294%2C783%2C0.13&t=QeAQF7Nf7Z0gbycZ-1&scaling=scale-down-width|https://www.figma.com/proto/MFg1OFhP2GtXhGajDnPKRC/Dec-2023-Systems-FO?page-id=1943%3A16680&type=design&node-id=1943-16681&viewport=294%2C783%2C0.13&t=QeAQF7Nf7Z0gbycZ-1&scaling=scale-down-width|smart-link] 

*requirements*

* User has an option to choose the new picture mode by default at the terminal
** Add an option to pick a default serving mode for Bulk Sales within the POS
** Option name: Bulk Sales - Use Picture Mode
** Toggle - On/Off, default On
* User can see 4 x 6 picture cards by default
** Card details:
*** Large picture of the account
**** With an enlarge option
*** Name
*** Balance
** Actions to add payments and additional items are also available
** Alerts (Allergens/Special Instructions) are visible and can be seen
* User can select the cards to indicate who ate a meal
** Cards should update visually to inform the user they have been selected.
* User can see which accounts have been selected in a list
* User can de-select an account by either selecting the card again or removing from the list
* User can record the sale against all selected accounts and be taken into the remainder of the existing bulk sales workflow.

## Acceptance Criteria
* Option added to POS to set default bulk sales to Picture Mode.
* User can toggle between Picture Mode and normal bulk sales mode.
* User can see cards showing: big picture, name, id number, balance, alerts/notes
* User can add funds to the account
** Adding additional items is in a separate story
* User can select/pick cards to indicate who will be sold against
** Card updates visually
* User can select all accounts, and then de-select all accounts
* User can see how many accounts have been selected
* User can see a list of everyone who has been selected
* User can de-select anyone from this list or by re-selecting their card
* User can see if an account has already had a meal, and cannot serve a second meal with this feature
* User can finish the sale and be taken to the remainder of the workflow
