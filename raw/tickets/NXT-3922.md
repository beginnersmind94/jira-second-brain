# NXT-3922 — [AcctMgt] - Preview - Add Payment

## Description
*User Story*

As a district administrator I want a way to quickly add a payment to a particular patron.

*Requirements*

* User can select an ‘Add Payment’ option on the preview card.
* The action slides the preview card to show the same payment functionality as the ‘Add Payment’ drawer (via Account Details).
** The user does not leave the preview card. They stay on it. 
** There is a back button that takes the user back to the main preview (with basic details)
* User can: 
** select the Payment Type
*** Options:
**** Cash
**** Card
***** Ask for Last 4 digits of Credit Card
**** Check
***** Ask for Check Number
** enter a payment amount.
** See what the balance will be after the payment.
** Add a comments box.
** Add Cancel and Process (Save) buttons.
*** Process should create the transaction and impact the account balance accordingly.
** All functionality/test cases should match the existing account action found in the Account Info page.
* When a payment is successfully made and the user closes the drawer, the grid should be refreshed as well.

## Acceptance Criteria
Payment option added.
Box prompts user to select a Payment Type.
User can enter an amount.
User can enter a check number or last 4 credit card number if appropriate.
User can enter comments.
User can see the balance before and after.
Transactions record appropriately.

When a payment is successfully made and the user closes the drawer, the grid should be refreshed as well.

All functionality/test cases should match the existing account action found in the Account Info page.
