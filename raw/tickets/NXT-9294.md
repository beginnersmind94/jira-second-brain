# NXT-9294 — Verification - Sampling - Schedule Sample Wizard

## Description
*User Story*

As a district admin I want a way to create a verification sample based off my selections for how we’re doing it this year.

*Mockup*

[https://www.figma.com/proto/Y3tlwN2yvRc67304adzyyB/Eligibility-3.0?node-id=122%3A0&scaling=scale-down-width&page-id=1%3A254608|https://www.figma.com/proto/Y3tlwN2yvRc67304adzyyB/Eligibility-3.0?node-id=122%3A0&scaling=scale-down-width&page-id=1%3A254608|smart-link] 

*Requirements*

User needs a way to make a verification sample pool.

* Add the option to Schedule Sample(s) to the Sampling page.
** Schedule Sample leads to a 4-step wizard with next/previous/cancel actions. User won’t be allowed to jump around on the pages of this wizard.
*** Step 1 - Verification Type
**** ‘What type of verification method are you planning on using this year?’
***** A single sample (Standard Verification)
***** Multiple smaller samples (Rolling Verification)
*** Step 2 - Sample Process
**** ‘What sample method are you using this year?’ (radio buttons, only 1 can be selected)
***** Standard (formerly Error Prone)
***** Alternate One (formerly Random)
***** Alternate Two (formerly Focused)
*** Step 3 - Verification Dates
**** If the user selected Rolling Verification in step 1, ask them to enter verification sample dates.
***** User can enter multiple dates.
***** Same date constraints - must be between 7/1 and the Final Sample date.
***** ‘Which dates should be used for the rolling samples?’
**** ‘What date should be used for the final sample?’
***** Date picker, default to 10/01.
****** Check the system setting for this default value.
***** Date range: current academic year only.
***** If the user changes the date from the system setting value - they must enter a comment. 
****** ‘What exceptional circumstances have led to this date change? Please enter a comment.’
****** 250 character comment box.
***** *If any dates fall on a weekend -* 
****** Show a caution tooltip with the following:
******* The selected date is on a weekend. Please consider changing the date to an operating day.
**** ‘What date should students be counted on for the Verification Collection Worksheet?’
***** Date picker, default to 10/31.
****** Check the system setting for this default value.
***** Date range: current academic year only.
**** In-line validation for date selection errors. Student count date should be after the sample date.
*** Step 4 - Exclude Sites
**** 'If any sites/schools should be excluded for verification, deselect them below. Sites operating under special provision programs (CEP/Provision II) should remain selected.'
**** Load all active school sites into grid.
***** This will be a multi-select of all their sites that are schools and are active.
****** All of those sites are selected by default
****** User can uncheck as needed.
****** User can search the grid to quickly find the site they may want to deselect. Search by name, or by site code.
***** Show the number of selected sites, and give the user a select all option.
****** Select boxes, a select-all box.
***** Grid columns:
****** Site Number, Site Name
***** Default sort: Site Number
** User can cancel at any time.
*** User will need a confirmation prompt if they cancel after beginning the wizard. (ADDED 2/4/21)
*** ‘Are you sure you want to cancel sample generation? The information will not save if you leave.’
*** CANCEL GENERATION / STAY HERE
** User can only generate at Step 4.
* The user should receive a success message about the sample being added.
** Verification samples are now scheduled and will run at the end of the selected sampling date(s).
* Sample should be added to the Sampling grid and be ready for the mechanism to automatically generate it later that day.
* Generate Sample button should be blocked at this point if any samples exist for the current academic year.
** Show the user what they need to do to resolve the issue.
*** Existing samples must be deleted before this action can be taken.

## Acceptance Criteria
Wizard added to the Sampling page.

Options available for each step per the requirements.

If the wizard is completed, user can see the samples in the grid.

Sample dates are ready for the background process to pick up and process.

Generate Sample wizard is blocked if any samples are present.

--

Given a user is on the sampling page & info 
And the academic year is defaulted to the current year 
When the Generate Sample button is tapped 
Then take the user through a 4 step wizard 
Step 1 - Verification Type
‘What type of verification method are you planning on using this year?’
•	Standard Verification
•	Rolling Verification
Step 2 - Sample Process
Q - ‘What sample method are you using this year?’ (radio buttons, only 1 can be selected)
•	Standard (formerly Error Prone)
•	Alternate One (formerly Random)
•	Alternate Two (formerly Focused)
Step 3 – Verification dates  
Q- What date should the sample be generated on?
Generate Date with date picker
Q - What date should the students be counted on for the verification collection worksheet?
Verification collection date with date picker 
Step 4 – Exclude Sites 
Q- What sites should be excluded for your sample?
Search by site name and site number 
And Show Cancel and previous buttons to either cancel or take the user back to previous wizard
And Generate sample button at the last step of the wizard should generate the sample pool based on the options chosen

Rules - 

User has options to select the sampling process they'll be using.
User can generate a traditional sample.
Sample is generated accurately, using system settings to control the percentage that must be selected.
Applications that are randomly selected are noted as being part of a verification sample for that year.
User cannot generate / remove samples for prior years.
User can only generate 1 sample with the Traditional method.
User can remove the sample for the current year.
