# NXT-57928 — MP - Menu - Secondary History information

## Description
As a menu planner, I would like additional information to be included in my history.

## Acceptance Criteria
* Include Serving Size information when items are added – API side 
* ON a VMI case where a child menu item is already on the menu and and VMI is added and  consumes the child, we need to display that correctly. 
** There is a scenario where if multiple VMI are added but only 1 has a particular child, they both look like they are replacing the 1 child menu item
!image-20250107-221115.png|width=1429,height=671,alt="image-20250107-221115.png"!
* On performed actions, provide accurate information to the user and not always display Menu Updated
** All actions should be specified on the performed action IF AND ONLY IF they are performed independently (All multi actions show Menu Updated)
*** Menu Item(s) Added- If ONLY menu items added and no other action was performed on the save
*** Menu Item(s) Deleted- If ONLY menu items deleted and no other action was performed on the save
*** Menu Item Count(s) Added- If ONLY menu item counts added and no other action was performed on the save
*** Menu Item Count(s) Deleted- If ONLY menu item counts deleted and no other action was performed on the save
*** Serving Size Updated- If ONLY menu item serving size changed and no other action was performed on the save
*** Menu Item Category Updated- If ONLY menu item category changed and no other action was performed on the save
*** Serving Groups Updated- IF ONLY Serving Groups Updated and no other action was performed on the save
*** Menu Name Updated- IF ONLY Menu Name Updated and no other action was performed on the save
*** Offered With Added- IF ONLY Offered with is added and no other action was performed on the save
*** Offered With Deleted- IF ONLY Offered with is deleted and no other action was performed on the save
* Any actions performed from Find and Replace that change the menu, replacement, deletion, removal of offered with relationships must be captured
