# NXT-65011 — GL Account Mapping in UI (not wizard)

## Description
*As a district financial manager,*
I want to map my general ledger accounts to the system accounts used by the software
so that I can generate accurate financial reports like the P&L, Balance Sheet, and Site-level breakdowns.

## Acceptance Criteria
*Dashboard/Summary Section:*

* Dashboard shows mapping progress with counts: Total GL Accounts, Mapped, Unmapped, Structure Issues
* Users can search all accounts by name, code, or description
* Users can filter by account category to focus on specific account types

*Mapped Accounts Section:*

* Displays successful mappings with System Account, GL Account, Parent Account, and View Details columns
* Users can view and edit existing mappings between system accounts and imported GL accounts
* Users can delete existing mappings

*Unmapped System Accounts Section:*

* Shows system accounts that haven't been mapped yet
* Users can search system accounts by description or code
* Users can filter system accounts by type
* Table shows System Account, System Code, Type, and Action columns
* Users can click "Map" button to open mapping modal for individual system accounts
* Users can toggle system accounts on/off to enable/disable accounts they don't want to use
* Users can add new custom system accounts to the mapping list

*Mapping Modal:*

* Allows users to search and select from imported GL accounts
* Shows system account details (description, code, type, category) and selected GL account details
* Users can create mappings between system accounts and imported GL accounts

*Structure Issues Section:*

* Identifies mapped accounts where the imported account code doesn't follow the defined account code structure
* Users can view and fix structure compliance issues for mapped accounts

*Contextual Information:*

* An info icon appears next to each system account; hovering shows contextual help text explaining where the data is sourced from.
