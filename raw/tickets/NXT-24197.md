# NXT-24197 — MP > Menu Planner > Frequently Used

## Description
In the Menu Planner page, there is a section labeled Frequently Used on the left hand side of the page.

The functionality of this, should be that as I build a menu cycle in the Menu Planner, if I have added a particular Menu Item a minimum of 4 times during the course of building my cycle, then this item should show up in the “Frequently Used” section.

The items that show up there should remain in that section, even if I finish and save the current menu cycle, and begin building another. (The reason for this, is that if I have created my Middle School menus, and am ready to create my High School menus for the same time period, there are generally several items that will be used in that cycle as well.)

There can be a Maximum quantity of items that display in that area… so as certain items are used more frequently than others, the lesser used items get bumped out of the list and replaced. I feel that there should not be a need to have more than 10 items in that list.

## Acceptance Criteria
When building cycles in the Menu Planner pages, Menu Items that I use frequently will appear in the “frequently used” section of the page. There will be a maximum quantity of 10 menu items listed in that area, and they will be replaced as other items get used more often than some. 

IF I have added a particular Menu Item a minimum of 4 times during the course of building my cycle, then this item should show up in the “Frequently Used” section.

If user removes menu item and is no longer 4 times, restore back to recently used. 

Display last 5 items that are frequently used, do not store more than 10. 

Display most recent 5 items at a time, allow “load more” option below to give user chance to load additional items if applicable. 

Keep previous 5 items and display additional 5 with scroller option. Only allow 5 items to display at any point in time.
