# NXT-7245 — INV - Vendor - Add Items to Vendor/Contract - Item Attributes

## Description
As a Procurement Specialist I need a way to search items from Item Management and add those items to a vendor & contract so I can begin listing the items that I will be receiving from the vendor.

Continuation of [Story 5709|https://primeroedgenext.atlassian.net/browse/NXT-5709]

+Story 2+ - Effort: *8*

* API: 2 
** updating DB when adding item
* DB: - (covered in story 1)
** new tables for new attributes (Product Lead Time, Min/Max Order Qty, etc.)
* UI: 5
** Right side fixed panel 
** 8 attributes, Add To Contract button
** removing item from list once added to contract
* QA: 5 

*Requirements*

A right-side panel containing Item-Contract attributes

* Once an item is selected, allow the user to input the 2 mandatory and other optional fields on the right side panel then Add to Contract
** This side panel will contain the following attributes
*** Vendor Product Code - MANDATORY
**** an additional item code that can be defined per vendor contract
**** Validation: Vendor Product Code is *unique within the vendor* 
*** Purchase Price - MANDATORY
**** this is different from the Standard Price defined in Item Management
**** This will be the price used on orders
**** The Purchase Price is the Whole Unit price
**** should support up to 4 decimal places
*** Minimum Order Quantity - Optional
**** should be less than Max Order Qty
*** Maximum Order Qty - Optional
**** a limit on how many of this item can be placed on an order
**** should be greater than Min Order Qty
*** Product Lead Time - Optional
**** this needs to be an optional field that can be added for an item if the user wants to set a different lead time than the one provided for the vendor
**** ideally this field would not be provided by default, and the user could click a button to add it
*** Charges/Fees
**** Delivery Fee ($)
***** up to 2 decimals
**** Processing Fee ($)
***** up to 2 decimals
*** Discounts
**** Net Off Invoice ($)
***** up to 2 decimals
** Once the above mandatory fields have been populated, user can click Add to Contract and the item is removed from the item list.
*** *If item cannot be removed from the list, at least provide some indicator that the item is added
* User will remain on the Add Items page after adding an item to a contract so they may continue adding items.

## Acceptance Criteria
User can search for items by Product Code, Name, 
User can add item to the vendor contract
User can view the attributes obtained from Item Management
User can define the attributes that are specific to the vendor contract
