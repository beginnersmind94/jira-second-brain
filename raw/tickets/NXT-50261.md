# NXT-50261 — LMA - Sites - Licenses

## Description
As a Customer Support role member, I would like to ensure we don’t add more site licenses than what are licensed so that we can charge for any extras.

*Notes:*

* For this page, the users are District users and Customer Support role
* On this page, we can see what licenses a site has and when they expire

*Flow:*

* LMA role user browses to System > Sites & Users > Sites tab
* User clicks Edit button on a site row and opens Edit screen
* User clicks Licenses chevron and that section opens up
* User sees a list of licenses granted to the district
* User can optionally select/unselect a license

*Requirements:*

* These requirements apply to System > Sites screen

*+ New Drawer*

* On the + New drawer, do the following:
* Add a License selector section to the expand/collapse sections below
** Label: Licenses (Optional)
** Optional
** Data: list of all licenses that are of type Site whose parent license key has a status of New or Active
*** Filter out district and device licenses
*** Filter out cancelled or expired license keys
** Card fields
*** Checkbox
*** License name
*** License key
*** n days left
**** End date - Today()
*** n of n available
**** Update the text when they select the checkbox
** Visually indicate if the license has none left (see bolt code)
** Do not allow the user to select a license that has 0 available 
** Visually indicate when a record is selected
* Update Save button
** Add logic to link the site to the license if selected
** Ensure the main grid is updated with the change
** Ensure license linking is added to History log



*Edit Drawer*

* Add the same License selector to this drawer as well with the same rules
* Additionally, ensure the user can deselect a license
* Update Save button
** Add logic to link the site to the license if selected a new one
** Add logic to delink if they removed it
** If delinked, make sure the available count for the license is updated accordingly
** Ensure license linking is added to History log



*Renew Update*

* After doing this, you should have the code you need to link sites to licenses
* We need to update the Renew button on the License Management page
** When user clicks Save automatically link the sites from the old license to the new license
** You don’t have to unlink them from the old license
* Likewise, we need to do the same for Bulk Renew

## Acceptance Criteria
h3. *1. + New Drawer*

* *Add a License selector section* in the expand/collapse area
** *Label*: {{Licenses (Optional)}}
** *Optional field*
* *Dropdown / Card Data Source*:
** Only *Site licenses*
** Parent *License Key Status*: {{New}} or {{Active}}
** Exclude *District* and *Device* licenses
** Exclude *Cancelled* or *Expired* license keys
* *Card Fields / Behavior*:
** Checkbox to select license
** Display:
*** License name
*** License key
*** {{n days left}} = {{End Date - Today()}}
*** {{n of n available}}
** Update text dynamically when checkbox is selected
** Visually indicate *0 available licenses* (see Bolt Code) → cannot select
** Visually indicate when a license is *selected*
* *Save Button Logic*:
** If a license is selected → *link the site to the license*
** Update *main grid* to reflect linked licenses
** *Log license linking in History table*

----

h3. *2. Edit Drawer*

* *Add same License selector* with same rules as +New drawer
* *Additional behavior*:
** Allow *deselection* (delink license)
* *Save Button Logic*:
** If a *new license* is selected → *link site to new license*
** If a *license removed* → *delink site from license*
** Update *main grid* to reflect changes
** If delinked → *update available count* for that license
** *Log license linking/delinking in History table*

----

h3. *3. Renew / Bulk Renew Integration*

* After site-to-license linking is implemented:
** *Renew Button (License Management page)*:
*** When user clicks *Save* → automatically *link the sites from the old license to the new license*
*** Do *not* unlink from the old license
** *Bulk Renew*: Same behavior as Renew → automatically *link sites from old license to new license*

----

h3. *4. Business Rules / Notes*

* Only *Site licenses* are selectable
* Cannot select a license with *0 availability*
* Linking/delinking must *update immediately* in the grid and license counts
* All linking/delinking actions are *logged in the History table*
* Renew / Bulk Renew automatically preserves site links
