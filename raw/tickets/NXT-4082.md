# NXT-4082 — Security - Session Settings

## Description
*User Story*
As a PrimeroEdge admin, I have the ability to determine settings for automatic timeout or logout for organization users.

*Requirements*
* There is a Session Settings section when creating a Security Policy
* The following settings are available:
** Session Lifetime - Sets the limit a user can be logged in consecutively in days (increments of 5)
** Idle Session Timeout - Sets how long in hours a user can be idle before being automatically logged out (increments of 1 up to 10)
** Concurrent Sessions - Sets the number of sessions a user can be logged into at once
* All settings have a No Limit setting

## Acceptance Criteria
* Session Settings section present in Security Policy
* Settings can be saved and edited as part of a security policy
* All settings available and functioning properly
