# NXT-21255 — Add Model Configuration to Database

## Description
We need to update the Build Model so that it saves the Model Configuration to the database so that it can be used in many other places in the app.

*Application*: Face and Food Detection Website

*Flow:*

* Follows flow from Build Model story
Upon clicking Create button, if validates, will save the model configuration to the database as well as do the other stuff that are already coded in

 

*Background:*

* The Create Model page already exists so don’t take anything away from it
Here, we are adding in key functionality that is needed in other places
Here we will add the functionality to save the Model Configuration to the database
When a model is built, it is given a set of food items with names, images and metadata along with a description
We need to save all this info

 

*Assumptions:*

* In order to explain what we need, I have put it in technical terms the way I understand them
You can do however you feel is best
Per my comment that I am explaining in technical terms as I understand them, for the purposes of these requirements, It is assumed that:
There are database tables that have Model configuration info stored in them based off of the Model Configuration schema below
It is assumed there is a model object that can be created from this schema and used in other places (stories to come)
Further, the DB Tables are for models that are completed, after the build process, and are used to manage models, linking them to sites, managing POS relationships, etc. (stories to come)

 

*Requirements:*

Front End Developer:

* On the Build Model Management, add the following after validation is complete and before start the build
* Save the following data to the database
** This data composes what we call a Model Configuration:
** Model ID
*** Unique ID for the model that is built
After a model is built, Model ID is what is used by the system to associate the model to many things
** Description – text, set by user when create the model
Tags – text, optionally added by user when create the model
District ID – unique id of the district
District Name – name of district per District ID
User ID – unique id of the user account that created the model
User Name – display name of user per User ID
Status – ID or text, status of the model
Menu Items Count – count of linked menu items
Date Time Created – date/time the model build was created (when the Create button was clicked on the Build Model page)
** Build Start Date/Time
*** When saving to the DB the first time, won’t have this value
blank or null until Build starts
*** When build starts, update this value with real build start date time stamp including time
**** You will need to update that code to make this happen
** Build End Date/Time
*** When saving to the DB the first time, won’t have this value
blank or null until Build starts
*** When build ends, update this value with real build end date time stamp including time
**** You will need to update that code to make this happen
** Time to Build
*** When saving to the DB the first time, won’t have this value
blank or null until Build starts
*** When build ends, update this value with real build end date time stamp including time
**** You will need to update that code to make this happen
*** Shows end datetime – start datetime measured in hours
Save as 2-digit decimal like 16.45
** Menu Items list:
*** Menu ID (ID of food item)
Menu Item (Food Name)
Metadata (ignore if not shown on Build Menu)
Count of Images linked
*** Linked Images list:
**** Note: this is linked to the Menu Item
Note: this will tell us which images are actually used in the model
Image ID
Menu Item ID
link to image (don’t need if can easily build link from Image ID in code later)

 

Backend Development:

* Create data structures to store the model configuration (see below)
Write code necessary for front end developer to add the data from above

 

Data Structures (minimum) as I see them:

* Model table
** Model ID - Unique ID for the model
Description – text
Tags – text
District ID – unique id of the district
District Name – text, may not need if prefer to join to that table when doing select statements
User ID – unique id of the user account that created the model
User Name – text, may not need if prefer to join to that table when doing select statements
Status – ID or text
Date Time Created – date/time stamp
Build Start Date/Time – date/time stamp
Build End Date/Time – date/time stamp
Time to Build – decimal (or similar), 2 digits
Menu Items Count – whole number or similar
* Model Menu Items list table
** Model ID – Foreign Key to Model table Model ID
Menu ID – Unique ID of food item
Menu Item – text, (food name) may not need if prefer to join to that table when doing select statements
Metadata – text, may be null
Count of Images linked – whole number or similar
* Model Linked Images list table
** Menu Item ID – Foreign Key to Menu Items table Menu ID
Image ID – unique ID of the image
Image Link – hyperlink text

## Acceptance Criteria
* Data is saved to database
* Data can be used by View Model page
* Does not impact the building of a model code that already exists
