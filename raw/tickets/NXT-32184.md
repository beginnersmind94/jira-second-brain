# NXT-32184 — DataQuery Framework - Reports Execution Page and Behavior Improvement

## Description
DataQuery Framework - Reports execution layout and behavior change

+*Existing UI*+

## Acceptance Criteria
* Items per page shall contain 5, 10, 25, 50, 100
* The value that the user selects for “Items per page” shall persist until the user changes it (persisting in browser storage is acceptable)
* The user shall be able to export the entire dataset of the report to Excel.
* The grid shall grow horizontally with horizontal scrolling enabled when columns are added.
** The idea behind this to initially have a visually appealing display when the user is adding columns.
* The “Action” column shall always be the rightmost column and shall scroll/grow with the grid
* The “Save” button shall not be enabled if the report has not been modified.
* The user shall be able to filter the columns and have the column filters persisted with the report when saved
** As of 9/2/2022, date filtering does not work.  Date filters will be corrected in the next sprint via another story.
* The user shall be able to sort the columns and have the column sort order persisted with the report when saved
* The “Create Reports” header immediately above the grid shall be renamed to the name of the report.
** This report name shall be editable
* The “Create Reports” header under “System | Management” shall be renamed to “Reports”
* The existing “Report” header at the top of the table shall be renamed to show the datamodel used by the report: The text shall be “Data Source: \[name of datamodel]”.
* See “Reports User Flow.pdf” for a UI markup containing more acceptance criteria.  The complete set of acceptance criteria shall be a union of the above text and the notes in the mentioned attachment(s).
