# NXT-33919 — INV - Withdrawals - allow decimal values for base unit

## Description
As a Site Manager I want the ability to enter decimal values for my “broken units” so I can enter exact quantities for the lowest pack size units.

*Ex:* Case unit and Bag unit. Case cannot be counted in decimals, but Bag can be counted with up to 2 decimal places, i.e. 5.45 Bags

*Requirements*

* Allow decimal place values for *lowest unit* on 
** Withdrawals
** For Transfers, we will *not* allow decimals for now (future enhancement)
* Allow decimal values to be entered on Incomplete inventories for only the base/lowest unit
** allow up to 2 decimal places
** UI: should be clear only base unit accepts decimal values
* Allow decimals for single row when base unit is selected in dropdown
** Add UI rules to prevent clearing of decimals accidentally
* Update QOH to reflect partial base unit quantities (i.e. 5 Case, 4.25 Bag)
