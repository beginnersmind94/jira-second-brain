# NXT-55575 — FH - Low Balance Alerts - Update for alert recurrence

## Description
!image-20241004-205041.png|width=783,height=537,alt="image-20241004-205041.png"!

Scenario from production - user set the alert cadence to 0 days, and got email alerts every 15 mins when the background job fired.

There’s a note on the page about only sending again after a transaction happens, but the logic behind the note is not being used.

*requirements* 

* Use the logic with a small twist
** Alerts should be sent out at the frequency set by the user
*** If ‘continue to alert me every ? days’ is set to 3, alert the user every 3 days if the balance is below the alert limit.
** Min value should be 1 day, not 0.
** Only send another low balance alert on the same day to the account holder if a transaction has happened and the account is still below the limit specified.
** Update the note:
*** Note: Low Balance Alerts are sent once per day, and will only send again on the same day when there are transactions on this account after the last alert was sent.

## Acceptance Criteria
* User isn’t spammed with low balance reminders on every background job trigger.
** User should only receive 1 reminder per day unless there is a balance change and balance is below threshold limit.
* Min value for the day frequency changed from 0 to 1.
