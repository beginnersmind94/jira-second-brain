# NXT-41621 — Student Import - Auto Code Mapping – Updates

## Description
As a SchoolCafé Admin I want to be able to delete a row from the code mapping and have the Is Mapped value auto set so that it is more intuitive to me and I have to make less clicks.

*Requirements*:

* If we can do this for all the imports, would be great, but at a minimum do for Student Import
* 
* -On the File Mapping tab, when you click edit on a mapping row, do the following per the Figma:-
* -Add the name of the field to the title-
** -Ex: Gender Code Mapping-
* 
* Change the 3^rd^ column name from Mapping to Actions
* Add Delete button
** On click, remove the row
* 
* If the user adds a value to External code and has a value for Internal code, automatically consider it “Mapped”
* Conversely, it they don’t have values for both, consider it “Unmapped”
* 
* Close button
** treat it like the Save button with all the same validations
* 
* Updates to the Save button
** Allow the user to click save button when there is no rows in the mapping grid and have it counted as completed
*** They may not want to map anything if they have already cleansed their data
*** Therefore, when they hit save and the drawer validates ok, the edit button on the File Mapping tab should turn black and be considered complete
** If they have no mapping, save and show a black edit icon
** if they have mapped all the rows, save and show a black edit icon
** If they have a row with one field with a value and the other field with no value, show an error message “Please correct unfinished mapping row” and do not allow them to close the drawer



+*Code Mapping UI:*+

*For the Auto Code Mapping-Student Import:*
1)When all the codes are auto-mapped.
Check the payload of auto code API and if the list has data and all are mapped then only turn the pencil to black otherwise keep it red.
The user should not click the Save button to turn the pencil black, the way it is working currently.

    a)When the user adds the required code for the unmapped ones and clicks the Save button, then turn the pencil to black. (current functionality).
    b)When the user does not add the codes for the unmapped ones and clicks the Save button, then turn the pencil to black. (current functionality).
    c)When the user does not add the codes for the unmapped ones and clicks the close button, then prompts a modal dialog box.
              Some values are not mapped properly for [ColumnName]. Continue? 
              Buttons option: Yes and No
              When Yes is clicked, close the modal popup and close the Code Mapping slider, save and change the color of the pencil to black.
              When No is clicked, close the modal popup and change the color of the pencil to black.
              Modal pop-up header text: Code Mapping - [ColumnName] 
              [ColumnName] is a variable.

    a)When there are no codes to map (The grid is empty) and the user clicks the close button, then prompts a modal dialog box.

              Buttons option: Yes and No
              When Yes is clicked, close the modal popup and close the Code Mapping slider, save and change the color of the pencil to black.
              When No is clicked, close the modal popup.
              Modal pop-up header text: Code Mapping - [ColumnName] 
              [ColumnName] is a variable.
