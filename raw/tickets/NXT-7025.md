# NXT-7025 — Direct Certification - Summary - Preprocessing and Confirmation

## Description
*User Story*

As a district admin I want to review the impact a direct certification file will have on my student population and give final approval before any student eligibility is updated.

*Mockup*

[https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=68333%3A28&scaling=scale-down-width|https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=68333%3A28&scaling=scale-down-width|smart-link] 

*Requirements*

If the user is viewing the summary of a direct cert file that is in a status = Preprocessed there is a bit of a pivot to how the Summary information would show.

* -User should be able to see the progress of the file.-
** -Is it in a queue?-
*** -How far back in the queue is it?-
** -Is it still gather statistical data?-
*** -What statistics are being gathered?-
** -Is it currently performing student matches?-
*** -How far along in that process is it?-
* -Progress can be shown as a progress bar.- 
** -Not sure if we can generate an estimated time to completion yet on this but keep in mind that will likely be a future ask.-
** -A percentage would be good however.-

The user has actions for a file that’s preprocessed - they can delete (or cancel) the file, or review it and approve the full processing.

* Cancel Import / Delete
** This can be done at any time of the process, whether it’s fully preprocessed or still pending analysis.
** Deleting the file will be a soft delete - we still want them to see what they’ve done or cancelled.
*** Confirmation prompt for deleting.
**** ‘Are you sure you want to cancel this file import?’
***** Cancel Deletion / Confirm Deletion
***** If they confirm, status should update to Deleted.
* If they Process the file, then we need to extend eligibility to all students who are on the file.
** The Approval Date needs to be used while processing the start date for student eligibility.
** DC Precedence order needs to be used - if the Type of this DC file is lower in the order than the student’s current eligibility, no update should happen.
** *Discussion point - do we need to update student’s enrollment records if they weren’t active at the approval date?*
** Status for the file should be updated to Processed.
* Update the audit log for the file to show:
** Who, when, action taken.

## Acceptance Criteria
(REMOVED 12/3) Progress Bar is available to show the user where the file is regarding preprocessing.
User can choose to delete/cancel or process the file.
Eligibility for students updates appropriately if the file is processed, paying attention to DC Precedence order and the Approval Date of the file.
Track who took an action inside a Preprocessed file and when.
