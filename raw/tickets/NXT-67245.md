# NXT-67245 — GC> Item Catalog in PE >Item search by uploading a file with GTIN/Product Code/Vendor Item# information and preferred name 

## Description
As an implementation manager, I want the ability to upload a file (.csv,.xls,.xlsx) containing GTINs, Product Codes, or Vendor Item numbers, and ingredient and stock item preferred name so that I can quickly import and search multiple items at once without manually typing them in or applying multiple filters.

PE - 3

## Acceptance Criteria
*Toggle Options:*
•	Allow users to choose between Search by Filters or Upload a File when searching the Item Catalog.
•	Search by filters will be the default option and displays the filters by default (current behavior)
*File Upload Capability:*
•	The Upload a File option should support.xlsx,.xls,.csv file formats(all standard formats we have for file upload).
•	A Browse button should allow users to select a file from their device.
•	A Get Template link should provide a downloadable file template with the correct format for users.
•	Downloaded excel template will give 5 columns

|GTIN|Product Code|Vendor Item Number|Preferred Ingredient Name|preferred stock item name|
| 124 -priority| abc| 456| | |
| | bcd123 -priority| 859| | |
| | | 745 - priority| | |

item 1 - gtin -124

item2 - abc

item3 - 456

124,abc,456 - current behavior - item1 

*Search Execution:*
•	After uploading a valid file, clicking Search should display the details from file: Total Records, Number of GTINS, Number of Product codes, Number of Vendor item numbers (as shown in the below screenshot)
•	Display the progress bar for showing the progress of how many records are processed  and how many are remaining.
•	Searching logic is going to be same not changing with story, it should work how its currently working when searched using the GTIN text field.
•	Results from Upload a File should display in the same Search Result table as the Search by Filters option.
•	Results that match GTIN/Product Codes/Vendor Item numbers are displayed in the grid. -Current behavior, should work, not changing with this story.
•	The user should still be able to review and import items after file-based search. -Current behavior, should work,  not changing with this story.
•	If no matches are found, the system should display No data available.- Current behavior
*Error Handling:*
•	If the uploaded file is in an unsupported format, the system should show a clear error message.
•	If the file content is empty don’t start the processing the file.
