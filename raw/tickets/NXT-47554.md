# NXT-47554 — Grades - History Log

## Description
As a SchoolCafé Custom Support role user, I would like the ability to see who made what changes to Grades so that I can quickly debug issues

*Notes:*

* Normally we add history log to grid rows, but in this case we are going to do it at the tab / page level instead of item level as item level won’t show deletions and this would likely be what users are looking for
* So we will show history for all grades in one screen instead
* Multi-level logging might be best for this

*Requirements:*

* These requirements apply to the System > Sites & Users > Site Configuration > Grades tab
* 
* Add New button
** Title / tooltip: History
** On click:
** open the usual history drawer
* Security
** View permissions managed via roles and permissions view property for the Site Configuration page



*History Drawer*

* Create a drawer for when the user clicks view history button
** Title: History
* Show the usual History drawer
* Use the multi-level view if it makes sense
* Show all the Additions, Deletions, and Edits to all the Grades for the site
** Note just one grade row, but all of them in one screen
* 
* This button is not necessary for the onboarding screen, but won’t hurt if you can’t hide it

## Acceptance Criteria
* Users can see add additions, edits and deletions since the site was created for grades in one drawer view
