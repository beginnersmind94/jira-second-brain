# NXT-47834 — API – Schedule by Time, Not Interval (Background Processing)

## Description
As a SchoolCafé admin, I would like to be able to schedule my imports by the time of day instead of interval so that I can avoid running them at the same time I am doing service.

*Note:*

* We already have the ability to do scheduling by interval and will keep that
* Now we are adding the option to schedule by time of day instead of interval

*Requirements*:

* For Add New and Edit API on the Scheduling tab, do the following:
* Add a radio button list
** Values: Interval, Time of Day
** Default selected: Interval
** Like other fields, only enable if they select Scheduler toggle button
** If they select Interval:
*** Show the existing fields that are already there
*** Hide the Time of Day field (new field)
** If they select Time of Day:
*** Hide the start time, end time, frequency fields (keep the Select day field)
*** Show the Time of Day field
* Add a the Time of Day field
** Show only if the select Time of Day radio button
** Title: Time of Day
** Multi-Select list
** Values:
*** Midnight
*** 01 AM
*** 02 AM
*** …
*** 11 AM
*** Noon
*** 01 PM
*** …
*** 11 PM
* 
* Backend:
* Enable to set the schedule by times of day from the Time of Day field

## Acceptance Criteria
* User can choose if they want to run import by interval or by times of the day
* If they do by interval, everything works like it does now
* If they do by Time of day, the import runs at the times selected
* This change is implemented for all the Import Types and all the providers
