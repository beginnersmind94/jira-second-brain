# NXT-19300 — IM - Item Details - Hierarchy - Details card

## Description
As a user I want a details section of the hierarchy of the item I am viewing so I can see the details of the item and its related items.

This is one of two stories to capture hierarchy info. This section will be where users are making any changes or configuration to item relationships.

*Requirements*

* Add a new card section titled ‘Hierarchy’ on the Item Management > Item Details page
** there will be several different views inside this card
*** for this story we will cover what is visible in this section when the page is loaded
*** linking new items is covered in [https://primeroedgenext.atlassian.net/browse/NXT-18837|https://primeroedgenext.atlassian.net/browse/NXT-18837|smart-link] 
* Displaying items (logic)
** by default, the item which we are viewing will always be in the grid
** if this item is not linked to other items, it will be the only item in the grid
** we will also display any *immediately* related items
*** items have parent-child relationships
*** we will show all parent items and child items of the selected item
* Provide a grid that spans the width of the card
** this grid will list the items related to the selected item and will provide details in grid columns
** this grid will still display the selected item in the list as well
** add a ‘Link Item’ button to the top-right corner of the grid (header) ([NXT-18837|https://primeroedgenext.atlassian.net/browse/NXT-18837])
** +Grid columns (left to right):+
** Status 
*** this will display a label [(Storybook link)|https://devuicomponents.perseusedge.com/?path=/docs/components-label--default] to indicate whether the item is a Parent or Child
**** use different color for Parent and Child labels
**** for the selected item, show a label of ‘Selected’ or leave it blank/empty
** Item #
** Item Description
*** this will likely need to be a fixed width 
*** text will wrap 
** Quantity of Child Items per Parent
*** this will display the numeric value for how many child items are in the parent item
**** this is the replacement for ‘Broken Units per Whole Unit'
*** display a dash in this column for the selected item
** Unit Type
*** this will display that item’s value for 1 of the 6 unit types (Pallet, Case, Each, etc.)
*** for items which have no linked items, this value could be null
**** if the item does not have a value for this, display ‘None’ as the label and provide an Edit icon
**** when user clicks the Edit icon, change the label into a dropdown
***** by default ‘None’ is selected and is an option
***** display all 6 possible values as selection
**** display save and cancel icons in place of the edit icon
**** after saving, return to a label
**** the Edit icon and functionality can remain until this item is linked to another
** Action column
*** this will contain a View icon for all the related items 
**** do not show a view icon for the selected item
*** when the user clicks the view icon, it would navigate to the Item Details of that item ([NXT-19301|https://primeroedgenext.atlassian.net/browse/NXT-19301])

## Acceptance Criteria
* Add new Hierarchy card section
* Hierarchy card will display the selected item and show any parent or child items
* Hierarchy details will display attributes which are relevant to item linkage
* Unit Type can be selected in the grid if it does not exist for the item
** Only if the item has not been linked to any other items yet
