# NXT-66927 — Finance Coordinator

## Description
As a Finance Manager, I need granular permissions to manage financial oversight functions so that I can configure GL structures and manage transactions while maintaining appropriate access controls.

## Acceptance Criteria
*Given* I am logged in as a Finance Coordinator
*When* I access the Financial Oversight module
*Then* I should have the following permissions:

*Dashboard Access:*

* I can VIEW the Financial Oversight dashboard
* I cannot EDIT, ADD, or DELETE dashboard elements
* I do not have FULL access to dashboard configuration

*Configuration Module - FULL ACCESS:*

* I have FULL access to the Configuration module
* I can VIEW, EDIT, ADD, and DELETE all configuration settings
* GL Code Structure: FULL access (VIEW, EDIT, ADD, DELETE)
* Account Mapping: FULL access (VIEW, EDIT, ADD, DELETE)  
* Import/Export Templates: FULL access (VIEW, EDIT, ADD, DELETE)
* Version History: FULL access (VIEW, EDIT, ADD, DELETE)

*Transaction Management Module:*

* I can VIEW, EDIT, and ADD transaction management functions
* I cannot DELETE transactions or have FULL access
* Manual Entries: I can VIEW, EDIT, and ADD manual entries
* Post from POS/Inventory: I can VIEW only
* Transaction Search: I can VIEW only
* Void/Reverse: I can EDIT transactions (void/reverse capability)

*Reports Module:*

* I can VIEW all reports but cannot modify them
* I cannot EDIT, ADD, DELETE, or have FULL access to reports
* Profit & Loss: VIEW only
* Balance Sheet: VIEW only
* Site-Level Analysis: VIEW only
* Export to ERP: VIEW only

*Period Management:*

* I can EDIT period settings
* I cannot VIEW, ADD, DELETE, or have FULL access to period management

*System Behavior:*

* When I click FULL access on a parent module, all child permissions are automatically enabled
* When I modify individual permissions, the FULL access indicator updates accordingly
* All permission changes are logged and auditable
* I can save my permission configurations for future sessions
