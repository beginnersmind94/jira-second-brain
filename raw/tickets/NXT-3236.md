# NXT-3236 — System - Organization - Edit Area [Regions]

## Description
As an administrator, I would like to be able to edit existing areas, or sub-regions, of my district, so that I may be able to better group sites for reporting and administration purposes.

* In the main Regions view/list page, allow users to select the region/area names like a  button
* When users with appropriate access to the edit functionality on the page select this button, open the form that allows the user to edit area details
* Existing fields (included which ones are required), buttons, and logic from PrimeroEdge Classic can be carried over
* When the user attempts to select the *Update/Update Region* button, if all required fields have appropriate data, save the changes and return the user to the main list/view.
** Otherwise, if some required details are still missing or inappropriate, the button should be disabled.
* Add a *Cancel* button that, when selected, returns the user to the main view.

## Acceptance Criteria
1. When the user navigates to the Regions view/list page, region/area names should function like hyperlinks/buttons that should be accessible to users with appropriate permission.
2. When selected, the button should open a form to edit the area details, which has the same fields, buttons, and logic as in PrimeroEdge Classic.
3. The 'Update/Update Region' button should only be enabled if all required and appropriate field data is entered.
4. When selected, the button should add the area and return the user to the main Regions list/view.
5. The user should be able to cancel their changes and be returned to the main Regions list/view
