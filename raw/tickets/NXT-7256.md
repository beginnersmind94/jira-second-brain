# NXT-7256 — INV - Orders - Order Detail - Create Receipt - Header

## Description
As an operations manager I want the ability to create a receipt on a specific order so I can receive products into my inventory.

This a component of the Order Detail page, in which all details of a specific order are displayed. 
For any order that is in Open status (not Incomplete, Pending Approval, Closed), the ability to create a receipt for the order should be available. 

On the Receipt tab of the Order Detail page, there should be a ‘Create Receipt’ button, which will change the view-only state of the page into a section where items can be received by entering received quantities or checking a checkbox.

*Requirements*

Load the header details of the order and receipt 

API is part of this story

* +Populated+:
** Site (Receipt Location)
*** Include Delivery Location if configured for the site

** Order #
** Order Amount
** Receipt Amount

* +User-Input+
** Transaction Date/Time (Received Date/Time)
*** Must be between the Order Date and the current date
**** i.e. cannot be prior to the order date, and cannot be in the future
** Order Delivered By
*** free text
** Invoice #
*** alphanumeric
*** mandatory
*** must be unique within district/region (for now)
**** validate uniqueness on Confirm/Save
**** Show message if not unique: “Invoice # has already been used. Please enter a unique Invoice #.”
** Invoice Amount
*** numeric $ value
*** mandatory
** Comments - optional

## Acceptance Criteria
* Receipts can be created for Orders which are in Open status
* Provide Receipt-level fields (Invoice, Total Amount, etc.)
* Provide Received Quantity fields for each item
* Provide Lot Number option for each item
* Provide warning prompt if Invoice Amount and Receipt Amount do not match
