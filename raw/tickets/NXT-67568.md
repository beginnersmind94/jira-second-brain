# NXT-67568 — IM - Bulk Item - Serving Info Validations & Multiple Servings

## Description
As a Nutritionist, once I am in update mode on the Bulk Item Tool, there should be some validations that prevent me from making particular changes to my serving size weight or description based on how the serving size has been used or if it is linked to a pack size (used as a dual unit). I would also like to able to add multiple serving sizes

!image-20251023-051915.png|width=1032,alt="image-20251023-051915.png"!

## Acceptance Criteria
* Add all validations that prevent serving size description or weight from being removed 
** Based on onboarding rules, serving description or weight can not be removed if nutrients or contributions are already entered and only changes can be applied
** To match the units card, if serving size is associated with recipe, only allow the amount and the weight to be changed (This only applies to Update session)
* Allow max 2 servings to be added per item (Applies to both onboarding and update)
** Provide a + after the first serving size description to insert another row
** This description must be a completely new description and can not be another amount EX: 1 each, 2 each
*** Check uniqueness against currently existing servings in the tool or on the item (If item has more than 2 servings externally)
* Allow second serving size to be deleted if none of the above validations apply
* Do not allow changes to fluid or non fluid and grey out the column (Only on update)
* IF item is added to the session that has more than 2 serving sizes, always prioritize serving with nutrients/contributions
* Provide more width for item description and shorten the columns for all others
