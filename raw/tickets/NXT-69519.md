# NXT-69519 — Fin - Template Entries

## Description
As a user, I want a template of debit/credit rules so I can quickly create manual entries for recurring transaction types.

## Acceptance Criteria
Requirements:

# *Mode toggle*: Segmented control offers "Template Entry" (default) or "Manual Entry". Switching modes clears all entered data with confirmation dialog if any fields are populated.
# *Location*: Required dropdown. "All Locations" creates a district-level entry; specific site selection creates a site-level entry.
# *Posted Date*: Required, defaults to today. If date falls in a closed fiscal period, show inline error: "Selected date is in a closed period" and block save.
# *Template filter*: Chips for All / Default Templates / Custom Templates. If no templates exist for selected filter, show: "No templates available. [Create one in Account Mapping]"
# *Lists all entry descriptions from account mapping.* Templates with inactive or unmapped data points show ⚠️ prefix, specific status subtitle ("Contains inactive data points", "Contains unmapped data points", or "Contains inactive and unmapped data points"), and are disabled.
# *Description*: Required textarea, max 255 characters, counter shown. Placeholder: "e.g., January breakfast program deposits"
# *Line Items grid*: Hidden until template selected (show: "Select a template to populate line items"). After selection, shows "X items from template" badge. Accounts are pre-populated but fully editable — users can change account selections, delete rows, or add new rows via "+ Add Line" button. Minimum 1 line item required. If template rows are modified, badge updates to "Modified from template" for audit clarity.
# *Amount fields*: Two decimals, no negatives, thousand separators on blur. Tab order flows: Debit → Credit → next row's Debit. Empty fields treated as 0.00.
# *Balance validation*: Validates on every field change. Totals row shows running sums. Green "✓ Balanced" when equal; red "Unbalanced" with difference amount when not.
# *Save button*: Disabled until: all required fields complete AND entry balanced. On click: if validation fails, scroll to first error. On network failure, preserve all data and show toast: "Save failed. Please try again."
# *Cancel*: If any field modified, show confirmation: "Discard unsaved changes?" with Discard / Keep Editing options.
# *Scroll*: If template has >5 line items, Line Items grid scrolls internally with sticky header row.
