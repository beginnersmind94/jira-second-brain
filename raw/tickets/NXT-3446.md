# NXT-3446 — System - Management - System Settings - Allow user to update default setting details

## Description
As a district or Cybersoft administrator, I would like to be able to easily manage settings related to the entire website in one location, so that I can quickly make the changes I want for PrimeroEdge to function as needed.

* In the *System Settings* page, allow a user to update the values of the settings related to the district.
* When the user selects the ‘Save Setting’ button (which should be added if it does not exist already), save the settings modified by the user. 
* Messages/functionality can be carried over from PrimeroEdge Classic. 
* When the user selects a setting to view details related to that setting, and the setting details are modified in the resulting pop-up, when the user selects the ‘Save Changes’ button, save the settings modified by the user.
* Values should be retained when the page is reloaded.
* Success messages and behavior can be carried over from PrimeroEdge Classic for now.

## Acceptance Criteria
Verify that settings can be modified when the user selects the 'Save Setting' button in the System Settings page grid

Verify that settings can be modified when the user selects the 'Save Changes' button in the setting information pop-up

Verify that appropriate error/success messages display as expected (being carried over from PrimeroEdge Classic) and data is retained
