# NXT-5266 — Verification - Sampling - Actions - Delete and Recalculate

## Description
*User Story*

As a district admin I need a way to correct mistakes, update counts, and make sure that my selected sample equals the sample size.

*Mockup*

[https://www.figma.com/proto/Y3tlwN2yvRc67304adzyyB/Eligibility-3.0?node-id=122%3A0&scaling=scale-down-width&page-id=1%3A254608|https://www.figma.com/proto/Y3tlwN2yvRc67304adzyyB/Eligibility-3.0?node-id=122%3A0&scaling=scale-down-width&page-id=1%3A254608|smart-link] 

*Requirements*

*these actions are only available for the current academic year*

* User needs an option to delete the entire verification sample, including all pools in 1 single click.
** User can select a Delete action.
*** *-If any applications have been completed the actual deletion will need to be blocked and the user advised on why.-*
*** If any applications have been completed, the action will result in an error message listed below:
**** At least one application has been completely verified. Due to this sample deletion is not permitted.
*** Confirmation prompt with an option to keep applications or discard. 
**** Confirmation prompt: Are you sure you want to delete and reset the verification sample? Applications added for Cause will be available after samples have been regenerated.
**** -Checkbox option: Keep the previously picked applications for future samples.-
**** CANCEL / DELETE
*** Deactivate/”delete” the sample.
*** -If the user did not select to keep applications, deselect all applications.-
**** -Track who reset/removed the sample, and what apps were taken out as a result.-
**** -Applications marked as ‘for cause’ should remain marked.-
* User needs a way to update the application counts in case something has changed.
** Recalculate Sample
*** This will recalculate all samples based on their date ranges.
*** Confirmation prompt.
**** Are you sure you want to recalculate the samples?
**** CANCEL / RECALCULATE
*** If they confirm, begin recalculating the samples.
**** check to see if the number of applications is correct or not
**** check to see if any applications are inactive and deselect them if they were selected
**** Add more apps if required
**** If any apps are at sites that are no longer part of the sample - change them to ‘Cause’ applications.
*** Status should update to ‘In Progress’ while being calculated.
*** If the application count is not correct afterwards, update the status to Incomplete



*Discussion with Haritha Manne, 8/3/21*

* Cause applications can be deleted with the samples, however…
** Cause applications must be returned when samples are next created.
* Delete confirmation message has been updated above.



*Success messages:*

* Deleting samples: Verification samples have been successfully deleted.
* Recalculating samples: Samples will be recalculated shortly.

*Error messages:*

* Deletion fails due to an application being complete: At least one application has been completely verified. Due to this sample deletion is not permitted.
* Deletion fails due to a sample being calculated/generated: At least one sample is currently generating. Please wait until generation is complete before deleting all samples.
* Recalculation fails due to samples being in progress or scheduled: All samples are currently scheduled for calculation. Please try again after they have finished.

## Acceptance Criteria
User can delete the sample and deselect all applications. -Applications selected for cause should remain marked as selected for cause.-

Cause applications should be returned to the sample as part of the next sample generation.

User can recalculate the counts tied to the sample. This would update all counts, including number of applications, and add more applications if the user asked the system to do so and it was required.

If the system is calculating a sample for that district, actions should be blocked.
