# NXT-6442 — Data Exchange - Pre-Upload Step

## Description
*User Story*

As a PrimeroEdge user, I would like to have a screen where I can select from a variety of file import types and have the ability to download the appropriate template when needed, so that I can easily begin the process of uploading as much of my organization’s data as possible.

*Mockup*

[https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=3024%3A51698&scaling=min-zoom|https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=3024%3A51698&scaling=min-zoom]

*Requirements*

* In the System module - Configuration category, add a new page: *Data Exchange*
* When the user navigates to the page, display the screen in stepper format, landing them on the *Pre-Upload step* first.
* On the Pre-Upload step, display an Import Type drop-down menu that includes the following options (for the different types of imports available):
*# --Select Type-- (Default, must be changed)
*# Student Import
*# DC Import
*# Inventory
*# Item Management Data
*## When this option is selected, display a Data Source drop-down menu where the user has to select a data source.
*## Once the data source has been selected, proceed to displaying the Download Template option as normal.
*# Production
*# Marketplace
*# Student Eligibility
*# POS Balance
* For any option is selected, display a Download Template button. 
** Only display the button after an Import Type has been selected
** When selected, the button should download a .csv OR .xlsx file for the option associated with it. If needed (either due to effort or architectural decision), the functionality for this button can be spun off to a separate story if needed.
*** The template will include all possible column headers in the file, but the rows will be blank for the user to fill out.
*** The template will be used on the next step, where the user will upload a file for the related Import Type.
* Once a selection is made, add or enable a Next button to allow the user to proceed to the next step: Upload File

## Acceptance Criteria
* Verify that within the Configuration category of the System module, we have added a Data Exchange operation

* Verify that when selected, the user is directed to the Data Exchange screen and lands on the Pre-Upload step

* Verify that there are two options on the page: choosing an Import Type and downloading a template file

* Verify that the Import Type drop-down contains at least the 8 options above 

* Verify that download template button does not appear until an Import Type has been selected

* Verify that download template button allows the user to download an .xls or .csv file (can be blank for now if needed, as long as file download is possible) for every Import Type

*Verify that, once any one of import selections indicated in the requirements are selected, the user can continue to the Upload File step
