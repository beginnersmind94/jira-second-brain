# NXT-41630 — Student Import - Auto Code Mapping – Don’t Add Correct Codes When Mapping

## Description
As a SchoolCafé Admin I don’t want to see correct codes in the auto code mapping so that I don’t have to map them.

*Requirements*:

* If we can do this for all the imports, would be great, but at a minimum do for Student Import
* 
* On the File Mapping tab, when you click edit on a mapping row, do the following:
* 
* Do not add rows for values that are correct when doing the auto mapping
* For example, for Gender, don’t add rows for “F” or “M” since those are the correct values
* We want to only add rows for values that are not correct

## Acceptance Criteria
* Correct codes are not added as rows to the code mapping drawer
