# NXT-71517 — SC - PBIRD - Activity Report – Distribute Now

## Description
As a Director, I want to distribute the Activity Report using its existing Power BI distribution configuration so that I can send the report to the configured recipients without recreating the setup.

----

h2. *Description*

When a user initiates distribution from the Activity Report page and a Power BI distribution configuration already exists for that report, the system will prompt the user to confirm distribution.

If the user confirms, the system will generate the report and distribute it using the existing saved distribution settings associated to that report.

This story covers only the *existing configuration path* from the Activity Report page.

----

h2. *User Flow*

# User navigates to:
#* *Accountability → Reports → Activity Report*
# User configures filters and generates the report
# User clicks *ellipsis (⋮)* → *Distribute*
# System checks for an existing distribution configuration
# Existing configuration is found
# System displays confirmation message
# User selects:
#* *Yes* → Report is generated and distributed using saved configuration
#* *No* → Dialog closes and no action is taken

----

h2. *Out of Scope*

* Creation of a new distribution configuration
* Editing an existing distribution configuration
* Power BI Reports dashboard actions
* Workspace and mobile integration

----

h2. *Requirements*

h3. *Existing Configuration Check*

* When user clicks *Distribute* from the Activity Report page:
** System checks whether a Power BI distribution configuration already exists for that report
* This story applies only when:
** An existing configuration is found

----

h3. *Confirmation Dialog*

* If an existing configuration exists, display confirmation dialog informing the user that the report will be distributed using the configured distribution settings
* Suggested confirmation text:

{quote}*This will distribute the report to the configured users. Do you wish to continue?*{quote}

* Dialog must include:
** *Yes*
** *No*

----

h3. *User Decision*

* If user selects *No*:
** Close dialog
** Remain on Activity Report page
** No report is generated or distributed
* If user selects *Yes*:
** Generate the report
** Distribute the report using the saved distribution configuration tied to that Activity Report

----

h3. *Distribution Behavior*

* Distribution must use the existing Power BI distribution configuration, including:
** Recipients
** Distribution method
** Format
** Other saved distribution settings
* Do not prompt user to re-enter configuration details
* When distributing a report from the Activity Report page:
** The system must pass the *current filter state* into the report execution consistent with Managed View Reports

----

h3. *One-to-One Relationship*

* There is a one-to-one relationship between the Activity Report and its Power BI distribution configuration
* Once configured, the report can be distributed ad hoc using that saved configuration

----

h3. *Business Logic Reuse*

* Reuse existing distribution framework and business logic where applicable
* Distribution behavior should align with established report distribution patterns already used in the platform

----

h3. *User Feedback*

* After user confirms the action:
** Provide standard feedback that the report distribution has been initiated
* If distribution runs asynchronously:
** Follow the standard async processing and notification behavior used by existing report distribution flows

----

h3. *Error Handling & Monitoring*

* Any failures during:
** Report execution
** Report distribution (Download, Email, Scheduled, Ad hoc)
* Must:
** Surface user-facing error feedback consistent with Managed View Reports
* Additionally:
** Errors must be logged and displayed in the *Monitoring Dashboard*
** Specifically within the relevant monitoring screens used for report failures
* Behavior must align with:
** Existing Custom Reports monitoring implementation



h2. Technical Assumption Note

*Technical Assumption*
SchoolCafé Power BI Distributed Reports will use *Power BI paginated reports* hosted in *Power BI Report Server*. Paginated reports are defined by *RDL (Report Definition Language)*, which is an *XML-based report definition*. For this implementation, the Power BI artifact should be treated as the *source report definition*, while SchoolCafé continues to store distribution, scheduling, ownership, and related application configuration in its own Power BI report framework and database structures.

*Implementation Direction*
Developers should assume that Power BI report metadata and report content can be accessed programmatically through the *Power BI Report Server REST API*. If report-definition-level access is required, the implementation should retrieve the report content/definition from Report Server rather than creating a separate hard-coded per-report configuration structure in the UI.

## Acceptance Criteria
* Given a user clicks Distribute on the Activity Report page, when an existing Power BI distribution configuration is found, then the system displays a confirmation dialog
* Given the confirmation dialog is displayed, when the user selects No, then the dialog closes and no action is taken
* Given the confirmation dialog is displayed, when the user selects Yes, then the system generates the report and distributes it using the saved distribution configuration
* Given a distribution is initiated from the Activity Report page, when processing begins, then the user receives standard feedback that the distribution has started
* Given the report has an existing configuration, when distributed from the Activity Report page, then the user is not required to re-enter recipients, format, or other distribution settings
