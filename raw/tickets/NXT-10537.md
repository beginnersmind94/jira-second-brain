*Requirements*

* When the Session is a Summary Sale session the Transaction Grid should have a row per Person Type.
* Each row will have a Meal Count, the Meal Price, Tax if applicable, and the total amount for the Person Type
* Meal Counts are updated in the summary and cards based on edited information
* After the updates are recorded meal counts for the session are updated to the changes.
* When returning to the grid the Session information updates and the Meal Counts update.

## Acceptance Criteria
User has an area where the transactions are shown by Person Type.
List shows Person Type, Meal Count, Meal Price, Tax and Total.
Selecting a person type opens it for editing.
The Meal Count sales field can be edited.
Selecting a count opens it for editing.
A comment must be entered if a count is changed.
When the commit button is selected the updates are saved and comment is saved
Counts are for Meals, 
    Changing a count will also change the Meal Count Card (Session Details, & Session Grid)
    Changing the count for Free will also Free Count Card (Session Details & Session Grid)
    Changing the count will also change the Total Sales Card (Session Details & Session Grid)

After saving the Session Details update to reflect the updates
When returning to the Grid the Session information updates and the Grid Summary cards update.

Taxes are calculated on Adults only and only for items with the Is Taxable set.
