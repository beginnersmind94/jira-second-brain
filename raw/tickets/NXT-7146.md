# NXT-7146 — Data Exchange - Field Mapping - Add PrimeroEdge Mapping Fields section

## Description
As a PrimeroEdge user, I would like to have an intuitive screen that allows me to easily associate fields in PrimeroEdge with the fields in my uploaded file, so that I am not overwhelmed by the amount of information on the screen.

*Summary*

# In the Data Exchange wizard, add a PrimeroEdge mapped field column (tentative name) that displays all of the PrimeroEdge fields related to the selected Import Type.
# (UI Only) Add Confirm Mapping section
# Implement the necessary controls that will allow a user to drag individual items from the column. *No need to implement connection between this section and the table yet. Only make it possible to do so in a subsequent story.*

*Details*

* In the Data Exchange → Field Mapping screen, add a *PrimeroEdge mapped field* (tentative name) section.
* Within this section, display all of the fields related to the selected Import Type.
** We can use whatever type(s) we have added support for up to this time and implement the others later.
** *For the purpose of the story/proof of concept, we can use the Student Import fields shown in the mock-up.*
** Fields that are required for the selected Import Type should have a red asterisk next to them.
** The fields should be implemented as a ‘drag and drop’ type.
*** It is not required that we allow the user to drag and drop fields from this section onto the Confirm Mapping sections, as that will be in a separate story.
** ‘Not in Use’ should still be a standard field that is included in all import types to indicate columns that exist in the file, but are not needed within PrimeroEdge.
* *(UI Only)* Also, add a Add Confirm Mapping section:
** By default, each column can show as ‘Not Validated’ 
** We may implement tools later to automatically match the field header with what we believe is the correct PrimeroEdge mapped field value, but that can be taken up in another story.
* This story is complete when the Confirm Mapping and PrimeroEdge mapped field sections/table are added and includes all of the potential fields related to the selected Import Type, all of which are ‘drag and drop’ style fields that can eventually trigger other actions on the page when dropped.
