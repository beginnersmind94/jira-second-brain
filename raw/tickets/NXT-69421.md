# NXT-69421 — Fin - Global Excel Import for GL Account Categories and GL Accounts

## Description
As a *district finance/admin user*, I want to *import my GL account categories and GL accounts from an Excel template in one step*, so that I can *quickly populate and maintain my General Ledger configuration without manual data entry*.

## Acceptance Criteria
h3. Import Button Behavior

* Provide an *Import* button in *Configuration > General Ledger Accounts*
** Updated on {{2026-02-09}}
* If GL Accounts import is configured: clicking Import redirects to System > Integration > Imports and Exports > [Actual Import Page] (user uploads file directly—no extra click on import type)
** If not configured: display warning: "Configure GL Accounts import type to continue. Have your template ready." (Updated on {{2026-02-05}})

h3. Import File Requirements

* The import file must be a filled version of the system-provided template and must include
** Sheet 1: GL Accounts

h3. Column Mapping Step

* After upload, display a *Column Mapping* step that supports separate mapping for the sheet:
** Sheet 1 (GL Accounts) required mappings
*** Account Code
*** Account Description
*** Account Category
*** Account Base Type
* Prevent the user from proceeding unless all required fields are mapped for each sheet

h3. Account Categories Import Logic

* Import Account Categories rows as create/update records using this key rule
** If Category Description is present, match on Category Description
* Display imported/updated Account Categories in the *Account Categories* grid with Category Name and Base Type

h3. GL Accounts Import Logic

* Import GL Accounts rows as create/update records using Account Code as the unique key
* On GL Account update, update Description and Category assignment
* Display imported/updated GL Accounts in the *GL Accounts* grid with Account Code, Description, and Category

h3. Validation Rules

* Validate each imported GL Account Code against the configured Account Code Structure using the same validation rules as the existing UI
* Do not block the import if Account Code Structure validation fails
* Any imported GL Account failing Account Code Structure validation must display in the GL Accounts grid with the existing warning indicator showing it needs correction

h3. Duplicate Handling

* If duplicate Account Codes exist within the same import file, still import the affected rows and flag them as Needs Fixing with inline error text “Duplicate account code in file”

h3. Data Handling Rule

* Treat Account Code as a string during import to preserve leading zeros and formatting exactly as provided

h3. Import Completion

* On completion, show an Import Results (Imports & Exports > Import Monitor > ‘General Ledger Accounts’ import record >Click on View Log button)
** Add a statistics section on top
*** GL Accounts: total rows, imported/updated count, needs fixing count
** Add the ‘Activity Monitor’, ‘Source Data’, ‘History Log’ and ‘Exceptions’ tabs
