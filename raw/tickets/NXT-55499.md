# NXT-55499 — DMF - Dashboard - Notes

## Description
As a Customer Support role user, I would like to be able to see which features are being imported for the modules I am importing and be able to make notes about it so that I track my progress

*Notes:*

* After our first prototype review, Dallas asked that we be able to communicate to end users 
** what “features” and things were migrated
** what wasn’t migrated
** what is partially migrated and needs additional work
* To accomplish this, we will make the following updates below

*Requirements:*

* In the database:
* Store in a table the ability to have default notes per DMF module
** should support html format
** should be linked to a database module in the nested grid
* When the user creates a new district migration in the dashboard grid, the system will copy these default notes to the corresponding row for each module, so create an API endpoint to provide this data
* 
* On the nested grid, make these changes:
* Add a button at the end of the row
** Make it an icon
** Tooltip: Notes
** On click:
*** Open the Notes drawer



*Add New Drawer*

* Update the start button
** Add code to insert the default comments into the comment box for the migration rows
*** Note: Do it by default when create the migration. Don’t wait to do it later.



*Notes Drawer*

* Create a new drawer
** This drawer will show default comments and allow the user to add/edit/delete from the comments drawer
* Title:
** [Module Name] Notes
** Ex: System Notes
* Size: Half Page
* Use sliding motion to open
* Fields:
** Large Rich Text Box
*** Name: Notes
*** Use the one in our library
*** On load, load the preexisting notes
*** Allow the user to modify the text as necessary
** Save button
** Cancel button

## Acceptance Criteria
* Dev team is able to save default notes in the database
* System loads default notes when create a new database
* User is able to see the default notes the first time they open it
* User is able to modify and save their notes
