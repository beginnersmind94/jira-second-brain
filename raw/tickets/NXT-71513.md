# NXT-71513 — SC - PBIRD - Email Now

## Description
As a Director, I want to email a Power BI report to users so that I can share report outputs directly with others.

----

h2. *Description*

Allow users to send a Power BI report via email from the Power BI Reports dashboard.

When the user selects *Email Now*, the system will open an email composition dialog where the user can define recipients, subject, and message body. Upon sending, the system will execute the report asynchronously and deliver the completed report via email.

This process should follow the same *distribution and execution behavior* as Managed View Reports, with the primary difference being the *email composition step and delivery method*.

----

h2. *User Flow*

# User navigates to:
#* *Reports → Power BI Reports*
# User clicks the *ellipsis (⋮)* on a report row
# User selects *Email Now*
# System opens *Send Message dialog*
# User:
#* Selects recipients (email and/or system users)
#* Updates subject line (optional)
#* Updates message body (optional)
# User clicks *Send*
# System begins asynchronous report execution
# When processing completes:
#* Report is sent via email to selected recipients

----

h2. *Out of Scope*

* Inbox delivery (covered in Download story)
* Workspace Dashboard implementation
* Mobile app behavior

----

h2. *Requirements*

h3. *Email Now Access*

* *Email Now* is available as an option in the row ellipsis menu _(covered in separate story)_

----

h3. *Email Dialog*

* Selecting *Email Now* opens a *Send Message dialog*
* Dialog must include:
** *To field*:
*** Supports selecting system users
*** Supports entering email addresses
** *Subject field*:
*** Default format: {{Email report - {Report Name}}}
*** Editable by user
** *Message body*:
*** Pre-populated with default message
*** Editable using rich text editor
* Default message should follow existing Managed View Reports pattern

----

h3. *Validation*

* At least one recipient must be provided before sending
* Send button is disabled until required fields are met

----

h3. *Send Action*

* When user clicks *Send*:
** Dialog closes
** System initiates asynchronous report execution

----

h3. *Asynchronous Processing*

* Report must be executed asynchronously
* User is not required to wait for completion

----

h3. *Delivery*

* Upon completion:
** Report is sent via *email* to all specified recipients
** Email includes:
*** Report output as attachment or link (consistent with existing behavior)

----

h3. *Business Logic Reuse*

* Email distribution behavior must follow Managed View Reports, including:
** Execution process
** Distribution framework
** Error handling
* Any applicable business logic should be reused

----

h3. *Permissions*

* Users must have access to the Power BI report to send it
* Follow existing Managed View Reports security behavior

## Acceptance Criteria
* Given a user selects Email Now, when the action is triggered, then the Send Message dialog is displayed
* Given the dialog is displayed, when the user enters recipients and clicks Send, then the report execution begins asynchronously
* Given the report execution is complete, when processing finishes, then the report is sent via email to the specified recipients
* Given required fields are not completed, when the user attempts to send, then the Send button remains disabled
* Given the email is sent, when recipients receive it, then it includes the report output
