# NXT-22129 — Exports- Reimbursement Claims-Oregon (SNP,SSO)

## Description
This story is to cover the tasks related Reimbursement Claims @ Exports.

The user will access the Claims Reimbursement-Export by navigating System--> Configuration-->Export Dashboard.

Once on the landing page,  (similar to that of Import dash board which is currently being redesigned to incorporate file/api based inputs), the user based on the access level (will see either the school ISD assigned or list of schools under their umbrella) will choose either File or API (check for naming consistency). Once selected, the user can select the the type of claim. Currently there’s e are 4 for each state.
The Standard report is also called SNP. There should be an additional report called as customized one which the user can customized based on this his needed. 

Note : When the user selects Customized, the template must be loaded from SNP as reference.

1. SPSF-SSO
2. SFSP
3. SNP
4. CACFP
5. Customized

Once the claim sub type is selected, the user lands into configuration page. Here "Export Data Category" will be based on the state and template you select like OR SNP Claim Report, "Export Name" to name the export, option to enable / disable the configuration must be provided.

Provision must be provided to export the file with various file structure options like qualifiers, delimiters etc must be provided along with the option to save the exported file (pop up to save the location , file name of the file.)

Also, another option to apply state defaults must be provided so that the user can just select that so that all the state defaults.

IF ANY DEFAULT setting is modified or edited, the state default must be indicated to reflect that is not in default anymore.

History of the claims must be provided in the exports Dash monitoring, however an option must be provided to select the dates and modify/edit the configuration must be provided..

Provision to apply filters for the reports must be provided.

In the Mapping screen, user must have an option to include the filed at source level if needed and these fields along with other fields must be available in case if the user wants to add more fields or delete fields.

{color:#ff5630}*Note : Once the user selects the State, The user must have option to download the respective state template.*{color}
