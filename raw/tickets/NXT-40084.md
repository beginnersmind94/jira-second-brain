# NXT-40084 — FTP – Dashboard

## Description
As a SchoolCafé user, I would like to be able to see the history of FTP activity so that I can quickly diagnose if there are any issues.

*Notes:*

* The purpose of this page is to show the history of FTP activities
* It can be used to diagnose if there are any issues
* Multiple features in SchoolCafé can leverage the FTP framework, and all their activity can be seen here
* It is like the export monitor
* There are no add, edit or deletes on this page

*Flow:*

* Open SchoolCafé and browse to Settings > Integration > FTP Monitor
* View list of FTP executions
* Optionally view the details

*Requirements*:

*Menu Changes:*

* Add a page to under System > Integration
** Title = FTP Monitor
** On click, opens the FTP Monitor page

*FTP Monitor Page:*

* Create a new page
** Title = FTP Monitor
* 
* Filters Section:
* Source
o   Multi-select drop down list
o   Optional
o   Comes from Source field in Email Distribution
o   Values: list of distinct sources from the table
* FTP Type
** Multi-select drop down list
** Optional
** Comes from File Type field in FTP Distribution
** Values:
*** sFTP
*** Active FTP
*** Passive FTP
* Date
** Single select drop down list like what we have on Export Monitor
** Optional
** Values:
*** Last 24 hours
*** Last 7 days
*** Last 30 days
* Status
** Multi-select drop down list
** Values: list of statuses for FTP executions
* 
* Create a grid:
* Columns:
* Source
* File Name
* FTP Type
** FTP Type field FTP Distribution fields
* Sent
** Date and time it was sent or attempted to send
* Sent By
** Name of user that sent it
* Status
** FTP success status
* Actions
** View Details
*** Requirements covered in the View FTP Details story
** View File
*** On click, download or open the file that was sent
* 
* Grid has paging
* Grid has sorting
* Grid has filters
* Grid has export to Excel or CSV
* 
* Security
** Add page to resources
** Security is managed via the resource in Roles & Permissions (R&P)
** In R&P front end, this page shows up under System > Integration
** R&P Edit permission should be disabled since there is no edit
** R&P Delete permission should be disabled since there is no Delete
** R&P Add Permissions should be disabled since there is no Add
** R&P View Permissions applied to the whole page
