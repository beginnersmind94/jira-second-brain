# NXT-6445 — Application Entry - Guided Process (1. Students) - Display and Add Pending Student

## Description
*Story* 

As an admin, i can manually enter the student information if the student is not found using the lookup feature. 

*Mockup* 

[https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=1296%3A25854&scaling=min-zoom|https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=1296%3A25854&scaling=min-zoom]

*Requirement* 

*  _Select 'Find a Student' to search for an existing student in the system. Can't find the student? Select 'Add a Student' to manually enter the pending student details. Then, answer the categorical and income questions to save the information._
* Add New student generates popup 
** Students popup explanation text: 
*** _TBD Instructional text - Find a student in the system or add a pending student by entering the information as it appears on the application._ 
* Student was not found in lookup > “Add a student” from the new student page > Manually add student details in Popup:
** Entry fields: ID, First Name, Middle Name, Last Name, Suffix, DOB, Site, Grade
*** Required fields: First Name, Last Name
** Categorical Status
*** Is this student a Foster, Homeless, Migrant, Runaway, or Head Start child?
**** No (default)
**** Yes > Dropdown option to select from the categories above
***** Categorical - blank until selected (selection is required if answering yes)
** Student Income
*** Does this student receive income?
**** Yes > Type and Frequency
***** Type 
****** Income (Work), Income (Assistance), Income (Other) 
****** Enter Amount under corresponding type and choose frequency
***** Frequency 
****** Weekly, Monthly, Every Two Weeks, Twice Per Month, Annually
****** Income - Frequency is blank until/unless a value greater than 0 is entered in the corresponding amount box, Frequency for an amount greater than 0 is required.
**** No > (default) don't have to fill out anything
* *Student Display Grid* 
** For a new application, the grid will be blank until a student is added.
*** Message in grid: “Students added to this application will show here.”
** Name, ID, Enrollment Site, Grade, Date of Birth, - IF Income, Foster/Homeless/Migrant/Runaway/Head Start show it
*** Name format: First Middle Last Suffix
*** Grade format: PK, KG, 01, 02, 03 ... 12
** Shows status of student as Pending
*** Pending - could not find student based on info provided (i.e. lack of info, new student, etc)
** Option to Edit and Remove student(s) 
*** Edit > Student popup comes up 
**** can edit the fields that were manually entered for the pending student (6643)
*** Trashcan > Are you sure you want to remove this student from this application? Yes/Cancel
** at least 1 student added to the application to go next

## Acceptance Criteria
- admin can add students if not found using lookup 
- admin must enter required first and last name, and answer the 2 questions
- students not located in the db are shown as 'Pending'
- if answered yes, admin must provide the details from the additional fields
- students show as active (found in db) or pending (not found in db)
- at least 1 student added to the application to go next
- Multi-language, ADA compliant.
