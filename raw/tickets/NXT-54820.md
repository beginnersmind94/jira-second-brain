# NXT-54820 — Payment Import - Student - RevTrak

## Description
As a SchoolCafé Admin I want to be able to import the RevTrak payment import file for so that I can see payments accurately in our Accountability module

*Notes:*

* This comes from a customer request to be able to import RevTrak payment imports the way PrimeroEdge does
* We are cloning from our work on CalPads and will do essentially the same thing we did for that
* Customers use a predefined template from RevTrack to do payment imports
* We need to accommodate this template
* It uses comma delimited files with quotes around the text
* It has no header row like CalPads
* Unlike CalPads import, we don’t have to worry about the unusual date format, so if you copy from that code, you don’t need the date handling part
* It is assumed that the RevTrak file format will always be the same, never change
* These requirements are designed to be able to handle these unchangeable format requirements

*Requirements*:

* For *Add New* and for *Edit* screens, when a user selects Payment Import - Students, do the following:
* 
* *Pre-Upload tab*
* Note: When the user selects Payment Import - Student as the Import Type, it then shows the File format field below it
* Add a value to the File Format field when the user selects Payment Import - Student 
** Label: RevTrak
* If the user selects RevTrak:
** Do not show the template download section
** Do show the info textbox and show this text: 
*** Import configuration will use a predefined format from this provider. The import file must comply with this format or will not work.
The text file must be in comma delimited format, with quotes around the text and no header row. The columns in order are Payment Number, Payment Date, Amount, Student ID. 
Example first row: “758612368723”,”12/25/2025 11:30:”,”4.99”,”11023”
* 
* *File Upload tab*
* If the user selected RevTrak on the previous screen, do the following:
* Note: Make the assumption that they are using the correctly formatted file
* Delimiter is auto set to Comma and made read only
* Qualifier works as normal
* 
* If you can, validate the file is a comma delimited file
** If it is not, then show an error message like you do for other formats
* 
* When they upload the file, insert as the first row the predefined column headers we stored in the database (see below)
** Note: This will enable us to continue the rest of the import functionality without modification
* 
* File Preview button
** Note: the file has no headers, so need to handle
** If the user wants to preview the data, use our predefined column headers (see below) in the grid
* 
* 
* *Field Mapping tab*
* Since you inserted the column names as the first row, you can use this tab normally without changes
* Automap the fields as follows, since they should always be the same
** Payment Number = ReferenceNumber
** Payment Date = Transaction Date
** Amount = Transaction Amount
** Student ID = AccountId
* 
* 
* *Review tab*
* No changes



* For *Impot Monitor* screen, when a user selects to import Payment Import - Student using RevTrak file format, do the following:
* If needed, insert as the first row the column names the same way you did for the configuration



* Setup column names in the database:
* In the database, save the file configuration format settings by specifying the column names we want to use and their order 
** Remember, file has no headers, but it is always in same format, so we will specify the column names ourselves
* Name and order:
** Payment Number
** Payment Date
** Amount
** Student ID

## Acceptance Criteria
* User can select RevTrak in the file format if doing Payment Import - Student
* If user selects RevTrak , on next screen, file preview uses our predefined column header names
* On file mapping screen, the page shows the predefined names in the source column
* On file mapping screen, the page auto maps the source fields to the database fields
* When importing, the import routine is able to handle not having column headers
