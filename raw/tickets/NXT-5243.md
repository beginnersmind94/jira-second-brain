# NXT-5243 — Direct Certification - Configure Matching Criteria, Points, Thresholds, and Algorithms

## Description
*User Story*

As a district admin I want to fine-tune the points awarded for direct cert matching, or to force a field to match in order for an automatic extension to happen.

*Mockup*

[https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=309497%3A205519&scaling=scale-down-width|https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=309497%3A205519&scaling=scale-down-width|smart-link] 

*Requirements*

* Add a configuration page called ‘Direct Certification’ under the Configuration area.
* Page should show information on threshold values used for matching, along with what the percentages are set to.
** Threshold values:
*** Automatic Match Percentage
*** Discard Percentage
*** High Confidence
*** Medium Confidence
** Value range: 0-100.
** Automatic value > high value > medium value > discard value.
** If the user attempts to save a value that breaks the above greater than line, block the save and inform them:
*** The threshold value for X must be greater than the value for Y.
*** Example: The threshold value for Medium must be greater than the value for Discard.
* Page should show information on the direct certification passes that are done, and show their percentage value if a data match happens.
** The passes are outlined in: [https://primeroedgenext.atlassian.net/browse/NXT-7681|https://primeroedgenext.atlassian.net/browse/NXT-7681|smart-link] but are also listed further below.
** Each pass has the option to be marked as ‘Must Match for Automatic Extension’ - this should be shown also.
** Each pass has the option to be enabled or disabled.
** Sort the passes alphabetically.
* User should be able to edit these values.
** If the user attempts a save on the percentages and the total number of points can’t reach any of the threshold values, inform the user.
*** The total percentage points for the enabled passes are not high enough to meet the threshold value for X.
**** Replace X with the lowest threshold they’re failing to meet.
* We need to track changes on these values - who changed it, old value, new value, timestamp.
* User should be able to view a history of changes on this page.



Additional notes from week of 12/21:

* Order for displaying the thresholds should be Automatic > High > Medium > Discard.
* Sort the Matching Passes alphabetically.
* The grids need to validate each other both ways - 
** If the user changes a threshold value and the total percentage points for enabled passes aren’t high enough, they need a warning about that.
*** ‘The new threshold values are higher than the total percentage points possible for a match.’

## Acceptance Criteria
Configuration page added.
Values are shown for threshold values, and matching pass information.
Matching Passes can be set as 'must match' which will be used as a blocker for automatic DC matching.
Matching Passes can be enabled or disabled - this will turn them off for DC matching.
User can edit all values.
History of changes is tracked for all values/flags.
History can be viewed on this page.
Role permissions added for view + edit.
