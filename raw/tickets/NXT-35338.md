*Requirements:*

* Update the existing feature selection page UI:
*# Display a loader symbol that displays before the API proc is completed.
*# Just like the landing page, move the content downward in the center of the page (needs top-margin padding)
*# Update the Readiness and Target Go-Live Date card:
*## Move it above the three suite cards
*## Make Readiness Date first in display order
*## Center-align both date fields
*# When less than 100% of objectives have been completed, allow the user to select the entire card (mockup uses a double border, but this is just an example. We can use whatever component in the library as long as it conveys that the user is selecting the whole card).
*## Do not allow the user to proceed to the next step until a suite is selected.
*## (*Will be done in a later story, but keep in mind*) Users should be able to go back and update individual tasks once they’ve completed them all.
*# Use gray disabled font on Front Office and Back Office modules and do not allow them to be selected. These will be completed in future stories.
* Make some text changes:
*# Change ‘Thank you for choosing SchoolCafe’ to ‘Set up your SchoolCafé Suite’
*# Change ‘Select the features you would like to get you up and running’ to ‘Below, select your readiness date (the date you would like to begin allowing other users to use SchoolCafé), your target go-live date, and specific features you would like to set up.’
*# Change the individual features' wording. New text:
*## *Create Sites*
Select this option to create sites within your district (~3 minutes)
*## *Create Roles*
Select this option to create various roles for your users (~1 minute each)
*## *Create Users*
Select this option to create additional users and assign them specific roles (~1 minute per user)
*## *Manage Settings*
Select this option to view and modify default settings (~5 minutes)
*# Change button label ‘Lets get started’ to ‘Let’s get started!' (ignore ‘Continue’ in mockup)
