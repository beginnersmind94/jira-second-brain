# NXT-58253 — Acc - Exceptions - Automatic Processing

## Description
As Dana the Director I want to setup automatic processing for Serving Exceptions based on selected search criteria.



Note from Matt: We may want to implement this similar to automatic reminders. Let the user save their search results, and the respective action like the ‘Resolve All’ option.

*requirements*

* User can setup automatic processing for Serving Exceptions.
** User can set the included sites.
** User can set the date range.
*** Limits: User cannot set more than 1 week. 
** User can set the frequency for the automation firing.
** Duplicate Meals:
*** User can include the time difference filtering from [https://primeroedgenext.atlassian.net/browse/NXT-58251|https://primeroedgenext.atlassian.net/browse/NXT-58251|smart-link] 
*** User can set how the meals should be handled (see screenshots below for the Resolve All drawer)
** Eligibility Variances:
*** User can set how the meals should be handled (see screenshots below for the Resolve All drawer)
* User can see / manage the automatic processing.
* History can be part of → [https://primeroedgenext.atlassian.net/browse/NXT-58252|https://primeroedgenext.atlassian.net/browse/NXT-58252|smart-link] 

*Discussion at Refinement (1/27/25)*

* Existing Scheduler framework does not support queuing - we’ll potentially have multiple jobs running at once, big concern for data adjustments.
** Framework will have to be expanded to support queuing.
