# NXT-15077 — API – Export Data imported through API

## Description
As a SchoolCafé admin, I would like to download the data imported via the API call so that I can debug issues.

*Note:*

* Keep PII in mind for Students Import
* These requirements for API imports only

*Flow:*

* Browse to Import Monitor
* Click on History button for one of the rows that are and API download
* View Source Data tab
* Click Download button
* System generates an excel file of the data that was imported in that specific import
* Close History

Requirements:

* This story applies to the Source Data and Exceptions tabs of the History button on Import Monitor
* At a minimum do for API Student Imports, but strongly prefer to do for all API Imports so don’t have to more stories in the future
** If you can do for Manual imports and sFTP, that would be awesome as well
* 
* Source Data tab:
* When user clicks Source Data tab:.
** Check if the import type is Student
*** If yes, verify they are in the PII role
*** If yes, proceed
*** If no, then stop the user with a message box 
**** Text: PII role required
* Add a download button to the grid
** On click:
** Download the file like we normally do
** Don’t worry about PII since we are controlling it at the tab level
** If possible, the columns should be in the order of the imported data columns, but not a hard requirement
* 
* Exceptions tab:
** Do exactly the same thing as for the Source Data tab:
** Enforce PII for Student Import and add the download functionality
