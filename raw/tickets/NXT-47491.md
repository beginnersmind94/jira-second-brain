# NXT-47491 — Roles & Permissions - New Screen - History

## Description
As a SchoolCafé admin, I want to be able to see who has made what changes to a given role so that I can debug any role related issues.

*Flow:*

* Open SchoolCafé and browse to Settings > Roles & Permissions (R&P)
* View list of roles
* Click on History on a role
* History drawer opens
* User sees list of history changes

*Requirements*:

* For the System > Roles & Permissions > roles grid, do the following:
* Add a new button to the Action(s) column on the rows
** Title: History
** On click, open the History drawer
* 
* Add a History drawer
** Title: History
** Use our standard History / Audit interface
** Show all the following:
*** Add New Role
*** Delete Role
*** Edits to resources check boxes for a role
*** Description change for a role
*** Adding and deleting users to a role
*** In the backend, be sure to capture all these changes
* 
* Security for tab and drawer
** Use the View permission for the R&P page to control who can see the page

## Acceptance Criteria
* All the requirements are met from above
* Users can see add add, edit and delete changes to roles in the History drawer
* Only users with R&P page View permission can view the page in read-only mode
