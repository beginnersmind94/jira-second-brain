# NXT-26991 — Custom Views - Provide out-of-box default views for Accounts Management - Accounts

## Description
As a central office admin, I would like to be able to have some custom views ready for me to use right out of the box, so that I do not have to manually create views just to recreate reports that I commonly accessed in PrimeroEdge Classic or other tools.

*Summary*

The objective of this story is to provide out-of-the-box views for first-time users of the Accounts page, and for a user to be able to “Reset” their view to original out-of-the-box view.

N/A

*Details*

* Create a default view for Accounts:
*# View Name - *Basic Details*
*## Default View
*## Columns - Account ID, Last Name, First Name, Site, Grade, Eligibility, Balance, Status, Action(s)
*# View Name - *Full Details*
*## Columns - All
*# View Name - *Birthday List*
*## Columns - Full Name, Birthday, Site, Grade, Action(s)
*## If we can’t deselect First/Last Name, then we can display those in place of Full Name, but it is preferable to not have to do that
*# View Name - *Allergen List*
*## Columns - Account ID, Last Name, First Name, Age, Site, Grade, Allergens/Restrictions, Special Instructions, Guardian Name, Phone Number, Email, Status, Action(s)
* +Notes+:
** These Views do not have to be shared with anyone by default.
** Filters would be default as well. 
** Only one of these views should be the default view, for _all_ users.
** Columns should be in the +order specified,+ if possible.
** Ideally, we should prevent the user from being able to ‘Save’ over these views (at least the Default view), and instead force them to have to ‘Save As’. This is so that they can always easily return to the stock views if they make a change they don’t like.
