# NXT-513 — Record Training for Employees - State

## Description
At the Central Realm, administrators can Record training events for individual or group

* Choose training event (All events including state are shown in CR for recording)
** Event Details displayed - Event Name, Description, Topic, Subtopic, Method, Source, Attachment(s) icon
** ability to sort headers in grid (if search generates alot of results)
* Employees show in grid *AFTER* event is selected
** If district event is chosen, then only employees within that district will show in the drop down with respect to the filters
* choose employee(s) from district or more than one district (at state level)
** Filter by: District (central), Area, Site Code, Site, CEU level, Job title 
** check (select/deselect all) employees within grid to record training 
** ability to sort headers in grid (if search generates alot of results)
* Review employees selected for the training record 
** options to edit (add, remove employees)
* Training date only available for <today> and past dates - cannot record training for future date 
* Confirm and record 
** review selected employees for the training event and confirm to record.
* Check for duplicates 
** Training Date, Training Event Name, unique State ID and/or Employee ID
*** ‘Training record(s) already exist for the selected employee(s). Proceeding may create duplicate records.’
**** ‘Cancel’ > 'Show List' automatically
**** "Proceed"
*** Show Event Date that employee attended the possible duplicate training
**** **Only show dates within same academic year.**

* Current limitations on recording is a ‘Batch Limit’ setting. Classic version has max of 25 - i would like the 2.0 version to have a batch limit of 50. The admin can adjust the batch limit to anything less than 50 in System settings and TW will abide.

## Acceptance Criteria
Admins can record training for employee(s) across districts. 
duplicates checked 
message appears if possible duplicates exist 
admin can remove the potential duplicates easily from the list before recording 
upon recording, the event shows in the employee(s) details under the training section
