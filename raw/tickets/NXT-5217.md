# NXT-5217 — Application Details - (Action) Refuse Benefits

## Description
*User Story* 

Application processor can indicate ‘refuse benefits’ after the application has been validated. 

*Mock Ups* 

*Requirements* 

* refuse benefits fields 
** Eligibility 
** Basis
*** these two are the current statuses for the application
** Result 
*** this is a choice for the user to make - it’s a reduction in benefits.
*** example: if the app is free, options in the dropdown are: reduced and denied.
*** if the app is reduced, the only option in the dropdown is denied.
** Start Date 
*** default to 10 days out to account for adverse action.
*** can be any day from current to end of academic year
** Comments 
* Refuse Benefits button
* Once done, students update accordingly. 
** Adverse action days should also be considered.

* Application status History tracking needs to happen for all of these actions.
** Action, user, timestamp.



Eligibility + Basis show the current status of the application. Result should be a choice between Reduced and Denied using this Free as an example. Start Date is a date picker.

## Acceptance Criteria
- Multi-language, ADA compliant
- ability to refuse benefits from refuse benefits page.
