# NXT-58324 — MP - Menu - Order of History Information and actions capture

## Description
As a menu planner, I would like for the information on the history to go in the order that I perform the operations down the page. This will allow me to easily be able to know where or what I did and see where mistakes were made or to quickly audit to make sure everything is correct.

## Acceptance Criteria
* Reorder History on Initial Creation to Include information in order of user input
** Meal Pattern
** Meal Type
** Site Group
** Menu Name
** Menu Template Name (If Applicable)
** Tags (If Applicable)
** Projected Serving Count as a Percentage
** Menu Is Active
** Menu Item Categories
** Serving Group Counts
** Serving Group OVS
* Always follow this pattern in order even on edit scenario if multiple actions happen on the same save including additional actions that happen later on 
* Order for additional actions as follows
** Add Menu Item or Add Menu Item from Quicklist
** Serving Size Information
** Menu Item Counts
** Offered With Relationships
* If items are added or replace from the Find and Replace, provide *serving size information*
* If items are added from Add Menu Items from Quicklist, provide *serving size information*
* If Menu is created from Template, provide all information for the user including menu item *serving sizes*, *menu item counts*, *offered with relationships* if already existing on the menu template at the time of using template to create menu
* If Menu is created from Copy, provide all information for the user including menu item *serving sizes*, *menu item counts*, *offered with relationships* if already existing on old menu before copy
* Any actions performed from Find and Replace that change the menu, replacement, deletion, removal of offered with relationships must be captured (Added from story 57928)
* Explore being able to add UI and API history changes into one entry instead of having multiple lines on the history
