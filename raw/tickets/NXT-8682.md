# NXT-8682 — ExpressPoint Website Payment Verfication

## Description
As a user I want the payments that are entered in ExpressPoint web to be entered into the session and the patrons account.

*Mockup:*

N/A

*Requirement:*

When a payment transaction is entered in ExpressPoint the transaction is recorded as part of the session and is entered into the account for the patron.

## Acceptance Criteria
Verify Couch DB is updated correctlyA session is created for the session date and site selected when the Open Service button is selected.
The session date defaults to the current date.
The open session data is saved and the status is set to open
A patron is selected and a cash payment is entered.
A 2nd patron is selected and a check payment is entered
A 3rd patron is selected and a card payment is entered.


Perseus Verification
After a session is open the user will go to the Perseus website and open the Reconciliation page.
Enter the search criteria for the site and date. There should be a session in the open state with the Opening balance and denominations entered from ExpressPoint.
The Terminal Name will be the ExpressPoint Device name, the Site and Site name will be for the site the session was opened, Opening balance will be Opening balance from ExpressPoint the Opening Date will be the date and time the session was created. 
Closing balance will be $0.00 Closing date will be blank, the status will be OPEN, Over/Under will be $0.00 - Opening Balance.

When the session is opened the denominations in the opening balance are the same as what was entered in ExpressPoint.
As each payment is recorded the transaction should be recorded
The cash payment should be what was entered on ExpressPoint for the 1st patron with a transaction ID of 1
The Check payment should be what was entered on ExpressPoint for the 2nd patron with a transaction ID of 2
The Card payment should be what was entered on ExpressPoint for the 3rd patron with a transaction ID of 3

The accounts for the 3 patrons can be opened and the payment transactions are shown in their account.
A 2nd patron is selected and a check payment is entered
A 3rd patron is selected and a card payment is entered.
