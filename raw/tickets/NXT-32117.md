# NXT-32117 — Elig. - Direct Cert - DC Processing/Config/Cards - Address Updates

## Description
As a district admin I want the information about DC address matching to make more sense to avoid confusion and calls to Support.

*not really a mockup*

!image-20220622-212731.png|width=1372,height=407!



*requirements*

Change the way Street Address is handled for DC Matching and update the scorecards / configuration screens appropriately.

* Street Address needs to be removed as a mapping option for DC File Imports.
** Existing files will need to be updated as part of this.
* Street Address field needs to show the combined and full street address. Points should show for it if it matches.
** line1 + line2 + city + state + zip
* The following fields should be nested under Street Address and:
** no points should be shown for them.
** their points cannot be configured on the DC Configuration page.
** Fields:
*** Line 1
*** Line 2
*** City
*** State
*** ZipCode
*** County Code

## Acceptance Criteria
# ‘Street Address’ is no longer available as a mapping field for imports.
# Additionally, ‘Street Address Full’ is no longer available as a map-able field in imports either.
# Street Address on the scorecards shows a combination of the full street address fields and highlights / allocates points if it matches.
## line1 + line2 + city +state + zip. ex: 123 Fake St Apt #1B, Houston, TX 77048
# The fields that make up the full street address need to be indented under Street Address (tabbed in).
# The fields that make up the full street address should show no match points, and give no match points.
# The fields that make up the full street address should have no points field to edit on the DC Match Config page.
