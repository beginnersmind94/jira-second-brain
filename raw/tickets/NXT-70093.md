# NXT-70093 — Base Type Fallback When Account Category Is Blank

## Description
*Intent:* Category is the single source of truth for Base Type. The Base Type column exists only as a fallback when Category is not provided, ensuring every imported account has a usable base type.

## Acceptance Criteria
*Fallback Logic*

* If a GL Account row in the import file has a blank or empty Account Category value, the system must use the Account Base Type column value from that row to assign the account's base type directly
* In this scenario, the rule "ignore the GL Account Base Type column during import" does not apply — the Base Type column serves as the fallback when Category is not provided
* Valid Base Type values for fallback: Asset, Liability, Revenue, Expense, Fund Balance
* "Equity" is accepted as a valid alias and maps to Fund Balance internally
* Matching is case-insensitive and leading/trailing whitespace must be trimmed before validation (e.g., "revenue", "REVENUE", " Revenue " are all valid)

*Default Category Assignment for Fallback Accounts*

* When a GL Account is imported via Base Type fallback, the system must assign it to a default category matching that base type
* The system must first check whether a category with that base type already exists on the Account Categories page
** If a matching category exists: assign the account to that category
** If no matching category exists: auto-create a default category named "[Base Type]" (e.g., "Revenue", "Expense") with the corresponding base type, and assign the account to it
* Auto-created default categories appear on the Account Categories page like any user-created category and are subject to the same logic when editing and deleting them
* Multiple accounts imported via the same base type fallback share the same default category
* Reassignment shares the same logic as normal account categories 

*Error States*

* If both Account Category AND Account Base Type are blank or empty for a GL Account row, import the row and flag it as Needs Fixing with inline error text "Category assignment required"
** The base type will then be derived from that category 
* If the Base Type fallback value does not match any valid base type or recognized alias after trimming and case normalization, import the row and flag it as Needs Fixing with inline error text "Invalid base type"

*Re-Import Behavior*

* On subsequent re-import, if the same Account Code is re-imported with a Category value provided, the Category assignment and its derived Base Type will overwrite the previous fallback Base Type
* On subsequent re-import, if the same Account Code is re-imported again with a blank Category, the system follows the same fallback logic described above using the Base Type column value from the new file
