# NXT-71506 — SC - PBIRD - Dashboard – Edit

## Description
As a Director, I want to edit an existing Power BI distributed report configuration so that I can update its settings without creating a new report.

----

h2. *Description*

Allow users to edit an existing Power BI distributed report from the Power BI Reports dashboard.

When the user clicks the *Edit* button from the grid row, the system will open the same Power BI stepper flow used for Add New, but in *edit mode* with the existing report configuration prepopulated.

Users may update the report configuration fields, and on save, the system will update the existing Power BI report configuration record in the database rather than creating a new record.

----

h2. *User Flow*

# User navigates to:
#* *Reports → Power BI Reports*
# User locates an existing Power BI report in the grid
# User clicks the *Edit* button on the row
# System opens the Power BI report stepper in *edit mode*
# Existing values are prepopulated
# User updates one or more fields
# User clicks *Save*
# System updates the existing record
# User is redirected back to the Power BI Reports dashboard

----

h2. *Out of Scope*

* Creating a new Power BI report
* Row ellipsis actions
* Favorites
* Dashboard filtering and download behavior

----

h2. *Requirements*

h3. *Edit Entry Point*

* Each row in the Power BI Reports grid includes an *Edit* button

----

h3. *Permissions*

* Users must have *Edit permission to the Power BI Reports dashboard screen* to interact with the Edit button
* If the user does NOT have Edit permission:
** The *Edit button is visible but disabled (grayed out)*
* Exception:
** Users with *Customer Support* or *Technical Support* roles:
*** Can always click the Edit button regardless of permissions

----

h3. *Screen Reuse*

* Use the same Power BI report configuration flow used for Add New
* Do not create a separate edit screen
* Stepper flow and behavior must be consistent with Managed View Reports

----

h3. *Prepopulation*

* All previously saved values for the selected Power BI report must be loaded into the stepper
* User may modify all available fields

----

h3. *Validation and Business Rules*

* Validation rules must follow the existing Power BI report configuration flow
* General business rules should be consistent with Managed View Reports where applicable

----

h3. *Save Behavior*

* On Save:
** Update the existing Power BI report configuration record in the database
** Do not create a new record
** Do not duplicate the report
* Save must use the *Power BI reporting framework and data structures*

----

h3. *Audit Logging*

* Any changes made during edit must:
** Be recorded in the *audit log*
** Include standard metadata (e.g., user, timestamp, changes made)
* Changes must be viewable via:
** *History drawer* (covered in separate story)

----

h3. *Post-Save Navigation*

* After successful save:
** Redirect the user back to the *Power BI Reports dashboard*
** Display the updated report in the grid

----

h3. *Cancel Behavior*

* If the user cancels edit:
** No changes are saved
** User is returned to the Power BI Reports dashboard

## Acceptance Criteria
* Given a user has Edit permission, when they click the Edit button, then the system opens the Power BI report stepper in edit mode
* Given a user does not have Edit permission, when the dashboard loads, then the Edit button is visible but disabled
* Given a user has Customer Support or Technical Support role, when they view the dashboard, then they can click the Edit button regardless of permission
* Given the stepper opens in edit mode, when the screen loads, then the existing report configuration values are prepopulated
* Given the user updates the configuration and clicks Save, when the save completes, then the existing Power BI report record is updated and no new record is created
* Given a report is updated, when the save completes, then the changes are recorded in the audit log and available in the history drawer
* Given the user cancels from edit mode, when the action completes, then no changes are saved and the user is returned to the Power BI Reports dashboard
