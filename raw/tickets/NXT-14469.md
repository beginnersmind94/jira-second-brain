# NXT-14469 — POS - Dynamic Menu Grid sizing based on rows/columns

## Description
As a POS cashier I want my POS menu to take up the full space of the menu area so that the buttons are easier to select when serving.

With the reduction of column support on POS menu grids, we are seeing that the max 5x6 grid is only taking up the top-left portion of the menu grid space. We need to expand that grid to consume all of the space horizontally. We should take the menu grid column definition and then apply a percentage width to reach 100%.

Example - 6 columns would be ~16.66% in button width if it is a 1x1 square. If the button was 3 columns, the width would be ~33.3%.

Same principal should be used for the button height to take up 100% vertical spacing.
