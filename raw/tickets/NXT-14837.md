# NXT-14837 — Platform - PII - Implement PII component at Field level

## Description
*Persona*: Perseus User

*Access level:* School District/Site

*Description*:

As a Perseus user, I should be able to view the sensitive data at field/section/page level only when I have access to a role that has view permissions. 

+*UX Screenshots:*+

!Field PII-20220426-123846.png|width=1280,height=497!

*PII Fields:*

* First Name
* Last Name 
* Parents Name
* Address
* Student Number
* Birth Date (DOB)
* Gender
* Ethnicity
* Race
* Eligibility

*Sensitive Sections: (HIPAA, FERPA compliance)*

* Allergens
* Eligibility
* Household Information

*Requirements:*

* All the fields identified in the above data fields needs to be masked/hide for user who do not have PII access roles/permission

## Acceptance Criteria
* User shall be able to see sensitive fields only when permissions are enabled for the roles specific to PII
* User shall see the <Sensitive Content> tag with a message displayed as “Sensitive Data can be only accessed with appropriate permission>
* User shall not see the sensitive data even at page source or downloadable files
