# NXT-3098 — **System - Organization - Users - List View

## Description
As an administrator I want to view a list of users related to each site in my school district, so that I can quickly search and make changes as needed.

The users list page contains a table of all users that are authorized to use PrimeroEdge within a school district.

PrimeroEdge Classic Reference: System > Sites and Users > Sites > Users



Page should allow the administrator to search for users within the district using a set of search fields. Selecting a user should open the individual user details (separate story).

We propose a cleaner view with better UX for the page controls (for example, avoid having a row of buttons as seen in the table header below, use a traditional search field to search within the table itself rather than individual filters on each column, etc.). Page should contain an edit feature to enable controls for the user to access buttons OR select user (if permissions of the user role allow). Buttons themselves can be addressed in a future story.
