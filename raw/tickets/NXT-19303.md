# NXT-19303 — IM - Item Details - Hierarchy Details - Add Unit Type attribute

## Description
As a user I need Unit Type as an attribute so that items can be classified as one of the 6 available unit types in accordance with GS1 data.

This story is to expand on the new ‘Unit Type’ attribute which is referenced in story [https://primeroedgenext.atlassian.net/browse/NXT-19300|https://primeroedgenext.atlassian.net/browse/NXT-19300|smart-link]. This story is to capture the back end of this new attribute.

*Requirements*

* Add a new attribute to the item titled ‘Unit Type’ 
** this field will be displayed only in the Hierarchy Details card section
** This attribute is conditionally mandatory
*** This must be provided in order to link items together
*** This must be provided if the item is to be used in Inventory
**** This may be used for other logic for other modules (future, other teams)
* This will be a dropdown and will contain 6 possible values:
** Each (EA)
** Package (PK)
** Case (CA)
** Display Shipper (DS)
** Mixed Pallet (MX)
** Pallet (PL)
* Assign a value to these unit types so the priority or sort order can be used later in transactions
** For example, Each is the lowest level item and should have a value of 1. Pallet is the highest level and would have a value of 6.
* **Note*: this attribute will be populated for all items imported from Global Catalog

## Acceptance Criteria
* Unit Type will be a new item attribute
* Unit Type will be restricted to one of 6 possible selections
* Unit Types will have an order/hierarchy
