# NXT-31838 — MP- Menu Planner- Editing Menu/Menu Cycle from Menu Planner

## Description
As a Menu Planner, I enjoy the interactive nature of the menu planner and would like to be able to edit my menu cycle after I create it. I would like to be able to come back to the menu Planner screen and make my edits from the Menu Planner screen and not just the edit menu/ menu cycle page.

## Acceptance Criteria
* Allow edit of any menu item on any menu in the menu cycle on the menu planner
* Allow edit to any menu name and or menu cycle name
* Once user lands on menu planner, if there happens to be more than one serving group already configured for menu or menu cycle, require user to select one to use for analysis while preserving the others as part of the menu in the background
** User can switch between serving groups to see the analysis associated with that particular serving group along with the analysis based on current meal counts and menu item counts for that particular serving group
** *IF serving group is not available for a particular day on the menu, display a message on the day “Serving Group not available for this menu/day”*
*** DO NOT ALLOW USER TO ADD MENU ITEMS TO THIS DAY
* DO NOT allow user to add more weeks in edit mode, user is stuck with the weeks that were created.
** IF user wants to add additional weeks, they will need to make a copy of the menu cycle of the standard menu cycle page
** Give a hover message over the plus to add extra weeks, “This menu cycle was created with (number of weeks) weeks, additional weeks can not be added”
* Allow delete/remove any menu item
** This will rearrange the menu item counts to be divided evenly between the menu items within the category irrespective if they have entered counts or not. 
** *THIS WILL EFFECT EVERY SERVING GROUP*
* Allow add menu items 
** This will rearrange the menu item counts to be divided evenly between the menu items within the category irrespective if they have entered counts or not. 
** *THIS WILL EFFECT EVERY SERVING GROUP*
* Update all analysis (same as current logic)
* Once edited, user can save and will now show the updated changes on the Menu Planner Page, menu cycle page,  the new counts and changes on the edit Menu Page
* Logic for not allowing edits based on production records complete should also take account here
* IF one of the menus in the cycle was used on a production record complete, Provide a message at the top of the menu planner
** *“Menu/Menus that cannot be edited have been used on a production record that is complete ”*
*** Keep these menus in only view mode, user will have no access to make changes
