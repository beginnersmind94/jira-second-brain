# NXT-39235 — KPI Reports - Data Rendering based on ADF Source (MEQ & Revenue)

## Description
Currently USDA - CNDB serving measures are not part of the rules

## Acceptance Criteria
Navigation - Insights - KPI Reports

The KPI report should be rendered based on new ADF data source

The data points should be automatically linked when new data flows in

The end user should be able to run the KPI reports based on the new data



Data Coverage:  MEQ and Revenue. the API should render the values based on Monthly & Weekly

*Meal Equivalent* should be total MEQ value for the given region / site type level or Site at a header level. Data should match with FO Dashboard (Example below). The API should return the MEQ valued based on the categories included in the snapshot.
