# NXT-47404 — GDSNConnect - Additional Exception Handing (Pack Sizes / Units))

## Description
Currently, Pack sizes entries are created with NULL entries. To avoid that, the following conditions should be incorporated

## Acceptance Criteria
Pack Size Table - To be created only for new entries, currently the entries are truncated and new entries are created each time

Condition: Add an entry to the table only when Either Purchase Unit Description is Not NULL or Inventory Unit Description is Not NULL

* If both are NULL - do not add an entry in Pack size table itself



Units Table: Food Items Vs Non Food Item Conditions

* Create entries in Units table only for the entries in Pack size table (which will have only valid entries with Either Purchase Unit Description is Not NULL or Inventory Unit Description is Not NULL)
* Include a condition where Not Null Description alone be used for conditional matches. Applies to all conditions where Purchase Unit Description not Equals to Inventory Unit Description
** Example: If Inventory Unit Description = Blank, Purchase Unit Description = Case, then Inventory Unit description cannot be treated as “Not Equals to” Purchase Unit description. 
** Reference from Pack size conditions:

|If PU Desc != IU Desc AND if Serving per IU > 1 AND IUs per PU=1|
|If PU Desc != IU Desc AND if Serving per IU = (1 or below) AND IUs per PU=1|
|If PU Desc != IU Desc AND if Serving per IU= (1 or below) AND IUs per PU>1|
|If PU Desc != IU Desc AND if Serving per IU > 1 AND IUs per PU>1|
