# NXT-7890 — Authentication - Forgot Password

## Description
*User Story*

As a PrimeroEdge user, when I want to login into PrimeroEdge but forgot my password, I would like to be able to use a “Forgot Password” option from the homepage, so that I can easily request a password reset and get back into my account. 

*Requirements*

* From the homepage, display a *Forgot Password* button that, when selected, allows the user to begin the process of resetting their password.
* Forgot password functionality flow:
** User selects ‘Forgot Password’ button
** A new UI Pop-up comes up requesting the registered email ID of the user
*** Pop-up may not be reflected in mock-up, but can be standard field+OK+Cancel style mock-up.
** User adds their email ID and clicks on “Request Password Reset” 
*** Success message: “Thank you\! if this email address belongs to a registered account, an email with instructions to reset your password will be sent shortly.” 
*** The system resets the password and sends an email notification IF the address belongs to a registered user
** Remaining steps should be covered by other stories:
*** When the user gets an email and clicks on password reset URL embedded in the email, they are redirected to a Set Password screen
*** Password reset page is loaded and password reset is successful.

## Acceptance Criteria
Verify that the 'Forgot Password' link on the homepage opens a pop-up requesting the user's email address

Verify that when an email address is entered and confirmed, the system validates whether it is is a valid address and then sends the Password Reset email to that user

Verify that the success message above is then displayed as a toaster on the homepage
