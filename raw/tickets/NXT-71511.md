# NXT-71511 — SC - PBIRD - Dashboard – Recently Run Reports

## Description
As a Director, I want to view recently executed and distributed Power BI reports so that I can access past outputs and verify successful delivery.

----

h2. *Description*

Allow users to view a history of *executed and distributed Power BI reports* from the dashboard.

This feature will display a grid of recent report executions and distributions, allowing users to review status and access the generated report output.

The functionality should follow the same *structure and behavior* as Managed View Reports Recently Run Reports, while displaying only *Power BI report data*.

----

h2. *User Flow*

# User navigates to:
#* *Reports → Power BI Reports*
# User clicks the *ellipsis (⋮)* on a report row
# User selects *Recently Run Reports*
# System opens the *Recently Run Reports drawer/modal*
# User views list of executed/distributed reports
# User clicks action button to:
#* View report
#* Download report
# User closes the drawer

----

h2. *Out of Scope*

* Configuration history (covered in History story)
* Editing or re-running reports
* Workspace Dashboard integration _(separate story)_
* Mobile app behavior _(separate story)_

----

h2. *Requirements*

h3. *Access*

* “Recently Run Reports” is available as an option in the row ellipsis menu _(covered in separate story)_

----

h3. *Data Scope*

* Display history of:
** Report executions
** Report distributions
* Each record represents a *single execution/distribution event*

----

h3. *Grid Structure*

* Grid must follow the same structure as Managed View Reports
* Columns:
** Module
** Report Name
** Date/Time
** Distributed (e.g., Email, Inbox, FTP, etc.)
** Comments
** Status
** Actions

----

h3. *Status*

* Status should reflect execution result:
** Success
** Failure
** (Other statuses as applicable)
* Display format should match existing patterns

----

h3. *Actions*

* Each row includes an action button to:
** *View report output*
** *Download report output*
* Behavior must match Managed View Reports implementation

----

h3. *Report Access*

* When user selects the action:
** System retrieves the generated report output
** Displays or downloads the report as appropriate

----

h3. *Business Logic Reuse*

* Any business logic implemented in Managed View Reports Recently Run Reports:
** Should be reused where applicable
* Includes (if applicable):
** Status handling
** Output retrieval
** Distribution tracking

----

h3. *Data Filtering*

* Display only:
** Power BI report executions
** Reports the user has permission to view

----

h3. *Read-Only Behavior*

* This view is informational only
* No modification of records is allowed

## Acceptance Criteria
* Given a user selects Recently Run Reports, when the view opens, then a list of execution and distribution events is displayed
* Given the grid is displayed, when the user reviews the data, then each row includes module, report name, date/time, distribution method, status, and actions
* Given a report execution is successful, when displayed, then the status shows as Success
* Given a user clicks the action button, when the system retrieves the report, then the user can view or download the report output
* Given the data is displayed, when filtering is applied internally, then only Power BI report executions are shown
