# NXT-4983 — AcctMgt - Add Patrons - Students - Wizard Step 1 - Information - Add & Edit

## Description
*User Story*

As a district admin I want to fill out general information for a student patron.

*Mockup*

[https://nqyc9v.axshare.com/start.html#p=category_based_on_patron_view_-_stepper&id=tj8d55&g=1&view=default|https://nqyc9v.axshare.com/start.html#p=category_based_on_patron_view_-_stepper&id=tj8d55&g=1&view=default]

*Requirements*

* Populate Step 1 of the wizard added via NXT-4982.
* This is mostly the same as the current General Information part of the Add Student drawer, with some information re-organization to provide better clarity and flow.
* Three sections, one is a placeholder:
** Section 1 - Student Information
*** Some fields have been rearranged here, please refer to the mockup for ordering.
** Section 2 - Enrollment and Eligibility
*** User can set the following:
**** Enrollment Site
**** Grade (populated by selected site)
**** Homeroom (optional)
**** Start Date (impacts both Enrollment and Eligibility)
***** Date limitations:
****** Must be within the same academic year.
****** Can be future dates and past dates.
**** Eligibility
**** Reason (populated by selected eligibility)
**** Comments (optional)
***** This will be a comment filled for the Eligibility grid. I don’t believe Classic has a length limit on this, but we may want to consider one. The wireframe shows a count of 250 which is likely good.
** Section 3 - Benefits (placeholder)
* Validations need to fire as normal for the ID numbers.
* User has a ‘SAVE & CLOSE’ option which will save, but exit the wizard. 
** These should be marked as an Incomplete Account somewhere so that they can be searched for later.
*** Item Management does something similar. We’ll add a future search filter for these accounts, we just need to note it somewhere if the user exits out without Contact Information.
** An account is considered incomplete until Contact Information is also added.
* ‘NEXT’ button available which will save, and take the user to Step 2.
* ‘CANCEL’ button should provide a confirmation prompt for the user.
** ‘Are you sure you want to cancel? Changes from this step will be discarded.’
* Stepper (top right component) can be clicked to allow navigation to a future step.
** This would be the same as the user clicking on NEXT. Validation + Save must happen.

## Acceptance Criteria
Fields provide appropriate information and validations.
Account is created on the Save & Close or Next actions.
Information is retained while navigating the wizard.
Multi-language support enabled (LANGUAGENAME_ENGTEXT ie: FR_Students, FR_ADULTS)
