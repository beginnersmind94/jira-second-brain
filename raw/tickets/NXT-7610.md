# NXT-7610 — Data Exchange - File Mapping - Allow cells to have multiple mapped fields

## Description
As a PrimeroEdge Admin, I would like to be able to have a single column on my import file be fed into multiple fields in PrimeroEdge, so that I can capture the data from files more accurately.

*Summary*

Allow users to drag multiple PrimeroEdge fields in the same ‘Drop’ cell and update database records as expected when mapping is completed.

*Details*

* On the Field Mapping step of the Data Exchange wizard, allow the user to drag multiple PrimeroEdge fields to a single cell in the Confirm Mapping table (no limit on number of fields per cell for now).
* User should be able to configure all of the fields within the cell as normal
* When the user proceeds to the next step/saves the configuration (at whatever point we commit the data), do the following:
** Make sure that the import will copy the data from the source file into each of the fields specified
** Other types of operations can be added in the future

## Acceptance Criteria
Verify that on the Field Mapping step of the Data Exchange wizard, we allow the user to merge 2 field headers, creating a single row in the various tables.

Verify that the user is able to configure all of the fields within the cell as normal

Verify that when the user proceeds to the next step/saves the configuration (at whatever point we commit the data), we make sure that the import will copy the data from the merged source file columns into each of the fields specified
