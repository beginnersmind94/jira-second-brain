# NXT-49652 — 2FA - District - Add & Edit - Google Authenticator

## Description
As a Customer Support user, I want to be able to setup a district’s 2-Factor authentication for Google Authenticator 2FA so that they can use a code sent to them after logging in via SSO Process to complete the login process

*Notes*:

* This story is for adding the 2FA concept to the Google Authenticator 2FA method

*Flow:*

* User browses to System > Security > Authentication tab
* User clicks Add New Google Authenticator 2FA (Or Edit on row) and opens that drawer
* User clicks to add a 2nd authentication step to take place after website login
* User selects the 2nd method and clicks save
* System updates Keycloak with the change
* User tests the method
* Dashboard grid updated to reflect the change

*Requirements*:

*Add New Screen:*

* These requirements apply to the System > Security page > Authentication tab:
* When user clicks Add New Google Authenticator 2FA and opens that drawer:
* 
* Add new fields:
* Use Two-Factor Authentication (2FA)
** Checkbox
** Default = No
** On click:
*** If select No, disable 2FA Methods
*** If select Yes, enable 2FA Methods
** If district has only one authentication type
*** Disable this field
*** Show tooltip: Enable another authentication method
* 2FA Methods
** Multi-Select drop down list
** Required if enabled
** Values:
*** The list of enabled authentication types on the dashboard screen
*** Remove the value for the current authentication method
**** Ex: Can’t select SchoolCafe Authentication as 2FA for SchoolCafe Authentication
** If there is only one method, default select it
* Info text icon next to 2FA
** Text: Selecting an authentication method will force the user to do a second authentication using the method you choose
* 
* Save button
** Update the Save action to save the 2FA value
** We don’t need to send an email to users because Add New defaults to disabled and user has to enable it on the grid, so email will be covered there



*Edit Screen:*

* These requirements apply to the System > Security page > Authentication tab:
* When user clicks Edit on a Google Authenticator 2FA row and opens that drawer:
* 
* Ensure 2FA and info button fields are there with same requirements
* Default select the 2FA field to what the user has saved if it is enabled
* 
* Save button
** Update the Save action to update instead of save the 2FA value
** If the user has changed the value for 2FA:
*** Show confirmation popup: This change will take immediate effect and impact all users that use this method. Are you sure?
*** If yes:
**** Update Keycloak accordingly
**** Send Inbox message to all user that use this authentication type
***** If 2FA = No: Your district has removed the requirement to perform two-factor authentication for your Google Authenticator one-time passcode on login. 
***** If 2FA = Yes: Your district has added two-factor authentication for your Google Authenticator one-time passcode on login. Moving forward, after you login with your one-time passcode, you will be asked to further authenticate using [2FA method one [, or 2FA method two, etc.]]. If you have not configured this method [or these methods], go to your user profile and update it [or them].
** Ensure changes are reflected in the grid

## Acceptance Criteria
* User can select if they want to enable 2FA for this method
* User cannot select 2FA if there is only one method for the district
* User can only select a method from one of the enabled methods for the district
* Inbox message is sent out to affected users on Edit
