# NXT-65485 — Create Stored Procedure to Apply Debit/Credit Rules and Post Transactions to General Ledger

## Description
*As a* finance/platform owner
*I need* a stored procedure that converts source transactions into *debit/credit lines* by applying *global rules keyed to system accounts* and resolving them to *tenant GL accounts* via the mapping table,
*So that* all modules post as per accounting rules and reports can rely on one source of truth.

## Acceptance Criteria
* When the stored procedure runs, it looks at the source transactions (e.g., meal sales, inventory postings, payroll entries) for the period you give it.
* For each transaction, it finds the matching debit and credit rule from our central rules table.
* It replaces the system account codes from the rule with the correct GL accounts for that customer, using their account mapping.
* It creates one debit line and one credit line for each transaction, making sure the amounts match so the ledger stays balanced.
* If a transaction can’t find a matching rule or account mapping, it still shows in the results, but is marked as needing a fix.
* The output will show all the information needed for the ledger entry — which accounts are debited and credited, the amount, the date, and where the transaction came from.
* Running the procedure again for the same period and data will always give the same results.
