# NXT-56978 — System - Management - Year Begin - Add New / Edit Task drawers

## Description
As a Cybersoft or district admin, I would like to be able to have a central location that explains every task which is necessary to have a complete year begin process, so that users will have smooth transitions between academic years.

*Summary*

* Add the ability to add and update tasks in the Year Begin page

[*Mockup link*|https://www.figma.com/proto/TxIF5PvP9D0IBNiTPkA2m5/UX-2024-25?page-id=105%3A1277&node-id=105-1362&node-type=canvas&viewport=226%2C415%2C0.13&t=uolVIp2PntEbLBpx-1&scaling=scale-down-width&content-scaling=fixed&starting-point-node-id=105%3A1362]

*Details*

* In _System → Management → Year Begin_, add the ability to create new tasks and edit existing tasks.
** When the user clicks +NEW, open a Create Task drawer.
** Create Task fields:
*** Name ⭐
*** Description (optional) ⭐
*** Link ⭐
**** Duplicate the list of pages from the Quick Search field in the top navigation. Can we just use the same control/field here?
*** Module ⭐
**** Same as above, but only the modules
**** Only show the modules that the district has licenses for
*** Assigned To (optional)
*** Due Date (optional)
*** Mandatory/Optional (flag) ⭐
*** Priority (High / Medium ({color:#0747a6}default value{color}) / Low) ⭐
*** Status (New ({color:#0747a6}default value{color}) / In Progress / Completed)
*** Task Order
**** Enter or select a numerical value. Lower-numbered items will have the item display towards the top of the grid.
*** Active (Yes ({color:#0747a6}default value{color}) / No) (Info icon)
** Edit Task - fields
*** Same as above
*** For now, Standard Task fields marked with a ⭐ cannot be edited except by users with Cybersoft/internal roles.
** Shortcut / Go-To Page
*** If not covered in the previous story, allow users to click the link to open a task’s related page in a new tab.
** Delete Task
*** If not covered in the previous story, allow users to delete *Custom* tasks from the grid.
* Remove all generic tasks that were added in the previous story (#56977). We will add some new default tasks in a future story.

## Acceptance Criteria
Verify that users can create tasks in the Year Begin page

Verify that user-created tasks are marked as Custom Task in the grid

Verify that those tasks can be edited completely by any user

Verify that certain Standard Task details cannot be edited except by Cybersoft users

Verify that Standard Tasks cannot be deleted except by Cybersoft users

Verify that all users can delete Custom Tasks
