# NXT-15747 — Reuse-Import Dash Board- API set up and Configuration

## Description
*A platform has to be designed and developed where all the API’s stored (API Configuration Page) and the user can trigger various API’s based on the requirements.*

*Once the API’s are triggered and the data is imported, the data is mapped to the respective template, transformed per requirement and  stored locally per business rules.*

This Story is to cover the task of retrofitting the File Import Platform to accept API’s as the mode of input along with the file based inputs.

An API configuration page has to be set up to store the API information, Vendor Information and map the vendors to the appropriate templates as only one vendor can be active at any given time.

# Provision to configure and Set up APi’s (consumer and provider) must be provided.
# Provision to add and edit vendors must be provided.
# Provision to tag every template (student, item etc) with a vendor must be provided so that this mapping can used to set up the trigger and importing appropriate template.
# Provision to enable vendors and select the default must be provided.
# Provision to save the API information (URL’s, credentials-user id-password, pass-phrases, certificates), alias, reconnect mechanisms etc) must also be provided.
