# NXT-52621 — PE - GDSN Connect - Exception Handling (Multiple Ingredient linked with Same Stock Items)

## Description
_(none)_

## Acceptance Criteria
Navigation: MP - GC - Pending Updates

Currently the multiple ingredients linked to same stock items are getting rejected , as a result users are not getting to see the ingredient updates in UI



Requirements:

Compare Ingredient Code + Stock Item Code, only if they are same, then that item should be flagged as duplicate (Regression: Item Name duplication still takes higher priority)

If the combinations are different, enable the item as processed, show them in the UI for user to review and accept

Regression Scenarios:

Item Name - Duplicate
More than Ingredient ID - in same import - Duplicate
More than one stock item ID (non linked) - Duplicate
Linked Items - Same Ingredient + Same stock item ID - Duplicate
*Linked Items - Different Ingredient Code + Same Stock Item - to be processed - new change*
