# NXT-57368 — PR - Plan - Menu with no menu items already assigned

## Description
As a production manager, in the event the menu planner does not want of the menu items on the menu served and removes the menu items, provide a message or banner on the plan/record that there is no plan for this day

## Acceptance Criteria
* This applies to menus that have already been assigned and are now showing on production
** IF menu items are all removed either from menu planner, standard or F&R, display a banner message that shows there is no plan 
!image-20241210-061546.png|width=2049,height=321,alt="image-20241210-061546.png"!
* This should apply regardless of whether counts have been entered or not as the menu item removal logic should still trigger menu items to be removed
* This does not apply if menu items have also been added from plan or record page before all “Menu” Menu Items are removed 
* Store categories on production side to ensure that any category used to add menu items from plan/record are available at all times regardless of what happens on the menu
** Choose counts will always be inherited from the menu and will remain whatever was last available based on menu data
** Update choose count on production (new table) if choose counts were changed on menu
* Only categories that have been used on production before menu items and categories are removed from menu should be available after the removal and can be used to add additional menu items
* In the scenario were items have been added to the record but there are no menu items on the plan, continue displaying the top portion of the plan page including status and meal counts and display a banner where current menu items are located “There are no menu items on the plan, however menu items have been added on the record”
** Allow menu items to still be added to the plan and if any menu items are added, no longer display the banner
* -All existing data should be consumed in the new production table and all categories should be reflecting there instead of case by case scenario-
