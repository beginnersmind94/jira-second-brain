# NXT-67407 — Elig - Apps - Controls for Automatic Pending Apps

## Description
As Dana the Director I am using FRE Apps for all my CEP sites as my state requires us to still process apps. Right now, all apps at CEP sites are going into Pending, which means I have to manually process them.

*requirements*

* Add a Configuration page to allow the user to see all Automatic Pending Reasons for applications
** Page name: Form Settings
* User can turn off Automatic Pending Reasons
* If a pending reason is turned off - proceed with the application as normal.
* Manual Entry workflows will not be changed as part of this - only Online app submission.

Text string for the top of the page:

{noformat}This page lets you control scenarios where an online application, with automatic processing, may be parked as pending. 

For example, applications with students at CEP sites are typically moved to Pending as they are not usually processed. But if you turn that option off, they will be processed per usual and will not be in Pending Applications.{noformat}

## Acceptance Criteria
* New settings page added
** Name: Form Settings
** Location: Between Eligibility Descriptions and Optional Benefits
* User can see all Automatic Pending Reasons for applications (View Permission)
* User can control them (Edit Permission)
* Role permission for updating these
** Provide it to the usual roles: Director, Central Office, SchoolCafe Admin, Internal Roles
** View and Edit
