# NXT-30719 — Contextual Help Widget - Freshdesk Help Widget - Design Spike

## Description
We will be utilizing the Freshdesk Help Widget within our app.  This story is intended to understand from a technical level how the widget can be embedded within our pages and which widget parameters we can make data driven in order to:

* Minimize future development work
* Simplify management of the content within the widget for the Cybersoft content team
* Create a configuration setting to show/hide the widget globally (or at a page level)



The idea is that the Platform team shall be responsible for defining the framework in which to embed the widget and manage the widget itself while the content team shall be responsible for the content of what is within the widget.  There could be opportunities for the Platform team to build features to help the content team more easily manage the content within the widget.



Reference for the Freshdesk widget: [https://developers.freshdesk.com/widget-api/#solutions|https://developers.freshdesk.com/widget-api/#solutions|smart-link]

## Acceptance Criteria
* An “Internal” setting named “Enable Contextual Help Widget” shall be defined which will determine if the widget is visible or not
** If the setting is set to true, the widget shall be displayed
** If the setting is set to false, the widget shall NOT be displayed
** If the setting is not present, the widget shall NOT be displayed
* The dev team shall identify how much offset is available to reposition the widget from its standard positioning.
* *The following is for research and exploration only.  This implementation will not be for production:* The attached widget script shall be implemented on the lower left corner of the Dashboard.
** The attached “SC Sample Widget.txt“ shall be embedded in the UI for the research/exploration
