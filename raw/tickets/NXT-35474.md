# NXT-35474 — PART 2 - Roles and Permissions - Limit Permission Changes To District

## Description
Currently, any change in permissions applies to all districts regardless of which district made the change. Each district needs to be able to control its own permissions.

PART 1: Roles and Permissions - Create Schema for Roles and Permissions and Implement API’s.

PART 2: Roles and Permissions - Integrate with Keycloak to fetch data from user Profile. and also manage resource catalog.

PART 3: Roles and Permissions - Update UI depending on changes required for new API’s.  and also testing included.

## Acceptance Criteria
Permission changes made by a district shall be limited to that district.
