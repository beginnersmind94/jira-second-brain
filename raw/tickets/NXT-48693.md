# NXT-48693 — OpenID - District Authentication - History

## Description
As a Customer Support user, I want to be able to view all authentication changes so that I can diagnose issues

*Flow:*

* User browses to System > Security 
* User clicks on History button
* User views the audit log in the History drawer

*Requirements*:

* These requirements apply to the System > Security page
* 
* Add a History button if it is not already there
** On click, show the audit / history drawer
* The drawer shows all the Add, Edit, Delete activities for all authentication methods
* Note: this is not on the rows under the action column b/c we want to show the user the scope of all changes including delete actions

## Acceptance Criteria
* Users are able to see all add, edit and delete operations for all authentication types
