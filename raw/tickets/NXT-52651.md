# NXT-52651 — Roles & Permissions - Warn User When No Role Permissions

## Description
As a Customer Support role user, I would like to be warned when I assign someone to a role if the role has no permissions that I can know it won’t work for them

*Requirements:*

* On the Users Add New and Edit screens, do the following:
* On the save button click, do the following
** Check if they added a new role
** For each (new?) role, check if the role has at least one permissions set for it
*** By permisson, I mean the checkbox for View, Edit, Add, Delete in the CRUD table is checked
*** If it has at least one, stop validating because the requirement has passed
*** If it has no CRUD permissions, then warn the user by:
**** Show the user a warning message: [Role name] has no permissions set for it. User will have no permissions granted to them from this role until permissions are assigned to it in the Roles & Permissions screen.
**** Allow user to close message
**** Allow user to continue to assign the role to the account

## Acceptance Criteria
* Users is warned when assigning a role to an account
* User is able to continue assigning the role to the account
