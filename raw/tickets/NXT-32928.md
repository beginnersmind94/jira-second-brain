# NXT-32928 — SchoolCafé - Add SSO Shortcut to redirect users to Family Hub

## Description
As a district admin, I would like to be able to access the Family Hub using my normal SchoolCafé session, so that I do not have to manage a separate username or password for yet another website.



+Background:+

In Classic, users had the opportunity to directly log into the Family Hub via a dedicated module link, which  when clicked, opened SchoolCafé in a new tab. It also automatically logged them into a/their district account.

!image-20220720-191128.png|width=1259,height=222!

+Details:+

Let’s repeat this in SchoolCafé by adding a shortcut which: 

# opens the Family Hub in a new tab
# Logs the user into their admin account automatically (_if_ it already exists. If it does not exist, create the account using their SchoolCafé profile details and grant them admin privileges).
## Details for creating new Family Hub accounts:
##* FirstName, LastName, Email for a new account can all be gained from their SchoolCafé profile 
##* Append ISD before SchoolCafé Username to create Family Hub username (this may not be necessary since we are using email address for usernames now)
##* Use their existing SchoolCafé password or use a random password
##* Use the ISD phone number as their phone number
##* Create/use a generic security question and answer for SSO users as their security question and answer
## They should not have access to their ‘My Profile’ page (since they can then change their email address and possibly break the connection)
## They should have access to all other district admin-related pages.

Add this shortcut as part of the System module. We will add it in to the System → Configuration category as a page link: *Family Hub*.

## Acceptance Criteria
Verify that a shortcut has been added which, when selected by a user, opens the Family Hub website in a new tab

Verify that if a district admin account for the user (tied to their email address) already exists, then they are automatically logged in to the Family Hub for their district using that account

Verify that if the user account’s exist but is not an SSO admin account, a new SSO admin account is created for them which they are then logged into, with the conditions listed above

Verify that these SSO admin users do not have access to their ‘My Profile’, but do have access to the District Profile and other related admin privileges ([User Story 114386|https://tfs.primeroedge.com/ProductDevelopment/Express/_workitems/edit/114386]: SchoolCafe - Admin - PrimeroEdge Sign-On users - Grant District Profile operation)

Verify the clean up tasks (users who then log out from Family Hub are taken to the Family Hub homepage, not SchoolCafe; SSO users cannot be found in user lookup, etc. See TFS references for details)
