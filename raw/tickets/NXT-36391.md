# NXT-36391 — MP- Menu/ Menu Cycle- Grain Based Dessert should be analyze based on inputted number (API ONLY)

## Description
Changes were made in Item Management that allow the user to now indicate how much of the grain is considered grain based dessert. Since we did not update the logic on the MP/PR side, we are still considering all grain on an item that has grain based dessert as the amount of grain based dessert to put on the analysis.

for above menu , expected grain based desert value : 1 but its showing as 2.75

## Acceptance Criteria
* Only Analyze the menu item based on the grain based dessert associated
** Previously grain based D was a flag but now it has an associated value
*** EX: Menu Item has 1 WGR and 2 non WGR, Grain based dessert can be up to 3 GBD or less and the user can pick how much of the total grains is considered GBD. Analyze for GBD based on what user inputs 
* This should apply to the menu analysis slide out and the nutrition report 
* This should apply to the menu Cycle analysis slide out, nutrition report and food component report
