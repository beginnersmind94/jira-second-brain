# NXT-36320 — Family Hub Payments - Scheduled Job - API

## Description
As a SchoolCafé admin, I want to ensure that Family Hub Payments are reconciled on a nightly basis so that I can proactively monitor the application and ensure higher customer satisfaction.

*Flow:*

* User browses to Support > Monitoring Dashboard > Job Scheduler
* User clicks the + New button
* User Selects Module and selects Family Hub Payments as a Monitor
* User sets the schedule and saves
* The system sets up a recurring job at the time and days selected by the user
* The system runs the jobs per schedule and output is viewable in the Monitoring pages

*Notes:*

* This story assumes that the screens for Job Scheduler and Add New Job have been created
* We have been told that the script already exists that does the checks, we just need to update it

*Requirements*:

* *Job Scheduler Page updates:*
* Ensure that Family Hub Payments is in the Monitor drop down list
* Via API, not hard coded
* Ensure that after a Monitor job is created, that it shows up in the grid with the correct values
* 
* *Add New Monitor Page updates:*
* Add or ensure that Family Hub is a value in the Modules list
* Add Family Hub Payments to the list of values for the Monitor drop down list
* When the user selects Family Hub, the list of values for Monitor are restricted to that Module so ensure that Family Hub Payments is one of the values visible for Family Hub
** Schedule section: no additional requirements
** Save button:
* On click, create the scheduled job per the dates and time selection in the schedule section (what to do in detail follows below) and then redirect to Job Scheduler screen
* 
* *Job Requirements*:
* These requirements are for the job itself
* Take the existing script and make the following updates:
* Define the days and times that the script runs
* Manually define the days and times that the script runs
** Later will add this to a front-end scheduler similar to SIS Auto Sync
* Change the email it sends errors to the new support email we created in the previous story
** remember that there are 2 emails, one for prod, one for non-prod
* 
* For every student that does not reconcile:
** Automatically call a script to force the system to push that student’s payments to SchoolCafé again
** If successful in pushing that student, then reconcile him again
** If successful in reconciling that student, then keep on going for the remaining students
*** If reconciles, add a record to the email that shows he failed but was fixed using the status column, Status = Repaired
** If fail to push or reconcile the student, then add him to the email exception body (described below)
*** Status = Failed
* 
* If there is any student not reconciled or was repaired, then send an email:
** Email:
** Subject: Family Hub Payment Exceptions – District Name
** Body: List all the payment exceptions
*** Table Columns / Fields:
*** Payment ID (from FH)
*** FH Payment Amount
*** FH Payment Date/Time
*** SC Payment Amount (can be null) 
*** Status: values = Repaired, Failed
* 
* Permissions: no extra requirements
