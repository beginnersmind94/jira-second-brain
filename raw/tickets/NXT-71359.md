# NXT-71359 — Insights - KPI - ENP - Configuration - SC

## Description
*I am a* district user *that I want to* configure benchmark values for Eligible Not Participating (ENP) in Insights using SchoolCafé data
*so that I* can evaluate whether schools are meeting expected student participation thresholds.

----

h2. *Notes*

* This story applies to *Insights – SchoolCafé data source*.
* ENP (Eligible Not Participating) is part of the *front office KPIs* being integrated into Insights.
* This story covers the *Benchmark Configuration screen only*.
* This story is a *companion to the PrimeroEdge ENP Benchmark Configuration story*.
* The *frontend behavior, UI, validation rules, formatting, grid behavior, autofill, bulk update, history, and save behavior must remain the same as the PrimeroEdge implementation*.
* Frontend developers should *not need to build new UI logic* for SchoolCafé.
* This implementation must reuse the same:
** benchmark configuration grid
** validation logic
** autofill behavior
** bulk update behavior
** history tracking
** save behavior

----

h3. *Data & Source of Truth*

* The source of truth for ENP data is available in:
** *Accountability → Reports → Participation*
** Click the *Non-Participation* tab
** Select *Free and Reduced* in the *Eligibility* field
* This must be used by developers and QA to validate ENP values.

----

h3. *Stored Procedure Integration*

* The front office team has already created the stored procedure:
** SchoolCafé: [https://primeroedgenext.atlassian.net/browse/NXT-67456|https://primeroedgenext.atlassian.net/browse/NXT-67456|smart-link] 
* The system must use the *SchoolCafé stored procedure* as the data source instead of the PrimeroEdge stored procedure.

----

h3. *KPI Definition & Formatting*

* ENP is a *percentage-based KPI*.
* Format must be:
** *Whole number percentage (no decimals)*
** Example: *25%*
* Values must:
** Be *integers only (no decimals)*
** Be within the range *0 to 100*

----

h3. *KPI Direction*

* This KPI is *Lower is Better*:
** Values closer to *0% indicate better performance*
** Higher values indicate worse performance
* Performance indicators, comparisons, and trends must respect this direction.

----

h2. *Out of Scope*

* PrimeroEdge implementation (covered in separate story)

----

h2. *User Flow*

* Same as *PrimeroEdge ENP Benchmark Configuration story*.

----

h2. *Requirements*

h3. *1. Shared Frontend Implementation*

* The SchoolCafé implementation must reuse the *same frontend behavior and UI* as the PrimeroEdge benchmark configuration.
* No new or separate frontend workflow should be introduced for:
** grid behavior
** manual entry
** validation
** autofill
** bulk update
** history
** save behavior

----

h3. *2. Backend Data Source Change*

* The backend must use the *SchoolCafé ENP stored procedure* instead of the PrimeroEdge stored procedure.
* This applies to:
** actual ENP values
** autofill values
** year-based values
** dataset population

----

h3. *3. Functional Consistency*

* All SchoolCafé ENP benchmark configuration behavior must remain identical to the PrimeroEdge implementation

----

h3. *4. Data Validation*

* SchoolCafé ENP values must match:
** SchoolCafé stored procedure output
** SchoolCafé Non-Participation report

----

h3. *5. Performance Requirement*

* The SchoolCafé-backed implementation must perform at the same level as the PrimeroEdge implementation and must not introduce performance degradation.

## Acceptance Criteria
h3. *1. Shared Frontend Implementation*

* SchoolCafé benchmark configuration uses the same UI and behavior as PrimeroEdge.
* No duplicate UI components are introduced.

----

h3. *2. Backend Data Source Change*

* System uses the SchoolCafé ENP stored procedure.
* Autofill values come from SchoolCafé data.
* Displayed values match SchoolCafé source of truth.

----

h3. *3. Functional Consistency*

* Validation rules match PrimeroEdge (0–100, integers only).
* Bulk update behaves the same as PrimeroEdge.
* Autofill behaves the same as PrimeroEdge.
* Save behavior matches PrimeroEdge.

----

h3. *4. Data Validation*

* ENP values match SchoolCafé stored procedure output.
* ENP values match the SchoolCafé Non-Participation report.

----

h3. *5. Performance Requirement*

* No performance degradation compared to PrimeroEdge implementation.
