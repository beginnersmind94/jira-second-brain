# NXT-31834 — MP- Menu Planner- Editing Menu from Menu Planner

## Description
As a Menu Planner, I enjoy the interactive nature of the menu planner and would like to be able to edit my menu after I create it. I would like to be able to come back to the menu Planner screen and make my edits from the Menu Planner screen and not just the edit menu page.

## Acceptance Criteria
* Start off with all scenarios that apply with the view option to start the edit process
* Allow edit of any menu item  or menu name on the menu on the menu planner
* Once user lands on menu planner, if there happens to be more than one serving group already configured for menu or menu cycle, require user to select one to use for analysis while preserving the others as part of the menu in the background
** User can switch between serving groups to see the analysis associated with that particular serving group along with the analysis based on current meal counts and menu item counts for that particular serving group
* IF menu item is marked as not served or has a null/0 count for selected serving group. show menu item as a basic item with a toggle, once toggle is switched to served, show as a normal chip and menu item with all information associated 
** IF user does turn on the not served or null/0 count menu item, rearrange the meal menu item counts based on a new added menu item 
** *ONLY effect that serving group that this action was done on*
** EX: Meal count for serving group is 200, there are 3 menu items in the category 2 of them have counts 125 and 75 and one is either null/0 or not served, run the analysis based on the 2 menu counts and the 3rd as 0, however, if they turn it on, the counts now would change to 67, 67, 66 respectively as previous logic has been implemented. 
* Allow delete/remove any menu item
** This will rearrange the menu item counts to be divided evenly between the menu items within the category irrespective if they have entered counts or not. 
** *THIS WILL EFFECT EVERY SERVING GROUP*
* Allow add menu items 
** This will rearrange the menu item counts to be divided evenly between the menu items within the category irrespective if they have entered counts or not. 
** *THIS WILL EFFECT EVERY SERVING GROUP*
* Update all analysis (same as current logic)
* Once edited, user can save and will now show the updated changes on the Menu Planner Page and the new counts and changes on the edit Menu Page
* On edit mode, user will no longer have a *save and continue* button since now they can be able to make edits directly from the menu planner 
** User will only have the save button and can continue to make additional edits and save again
* Logic for not allowing edits based on production records complete should also take account here
** *On the Menu page, when user clicks on the menu planner view provide a error message that says “One or more production records is complete, please use the standard option to make a copy of the menu”*
