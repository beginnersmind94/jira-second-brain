# NXT-8684 — ExpressPoint Website Bulk Sale Verification

## Description
We need to verify that when a Bulk Sale is recorded on ExpressPoint web that Couch DB is updated, the Reconciliation pages show correctly, and the patrons account is updated correctly.

## Acceptance Criteria
A session is created for the session date and site selected when the Open Service button is selected.
The session date defaults to the current date.
The open session data is saved and the status is set to open
Select a Classroom with Patrons in it.
Serve Meals, record payments, add alacarte items and save.


Perseus Verification
Verify Couch DB is updated correctly
After a session is open the user will go to the Perseus website and open the Reconciliation page.
Enter the search criteria for the site and date. There should be a session in the open state with the Opening balance and denominations entered from ExpressPoint.
The Terminal Name will be the ExpressPoint Device name, the Site and Site name will be for the site the session was opened, Opening balance will be Opening balance from ExpressPoint the Opening Date will be the date and time the session was created.
Closing balance will be $0.00 Closing date will be blank, the status will be OPEN, Over/Under will be $0.00 - Opening Balance.

When the session is opened the denominations in the opening balance are the same as what was entered in ExpressPoint.

Go to the transactions list.
The list will be the same as the patrons selected in the classroom sale
Open the details for each patron. The items selected for sale, and any payment entered will show in the details.

The accounts for the patrons can be opened and the items and payment transaction detail are shown in their account.
