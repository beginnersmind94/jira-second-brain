# NXT-56240 — SC - GDSN Connect - API Based Image Updates

## Description
_(none)_

## Acceptance Criteria
Acceptance Criteria:

The API should be able to send all the data in SC for a specific region and return the results

All the extracted API information should be stored in staging table for parsing of data

The exception engine should be run on a stored data in staging table

The refined data should be moved to GDSN Connect SC Updates table

Once the data is stored, the user should be able to render the images in UI as-is
