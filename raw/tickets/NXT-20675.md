# NXT-20675 — INV - Inventory - Additions - Reason configuration

## Description
As a FSD I want an Addition Reasons page so I can configure the reasons that could be associated with an inventory addition.

*Requirements*

* Display a configuration page titled Inventory Addition Reasons
** User navigates to this from the main Additions page ([https://primeroedgenext.atlassian.net/browse/NXT-1779|https://primeroedgenext.atlassian.net/browse/NXT-1779|smart-link])
* Display a header section that allows the user to create a new reason
** Provide a textbox titled ‘Addition Reason’
** Display an ‘Add’ button to add to the list
* Display a list section which displays a list of existing Addition Reasons
** Columns:
*** Addition Reason
*** Action column
**** Edit icon
***** this will enable the text field for the Addition Reason so user can edit
***** this will replace Edit icon with Save and Cancel icons 
**** Delete icon
***** this will remove the reason from the list
***** this icon will be hidden if the reason is used on any transaction
* Reason description should be unique by region

Note: this is the _Withdrawal_ reasons mockup but the page is the same other than the word ‘Addition’ instead of ‘Withdrawal’

## Acceptance Criteria
* User can view existing reasons
* User can add new reasons
* User can delete reasons if they are not used on a transaction
