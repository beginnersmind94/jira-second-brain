# NXT-60355 — IM- Item Onboarding-Inventory Options- Serving-Pack Link (Weights) (Step 2)

## Description
As a district admin, I would like a way for me to bulk update serving/pack size relationships. These could be either deciding to use serving as a base unit, provide how many servings are in the base unit, or provide weight.

!image-20250429-071325.png|width=1361,height=749,alt="image-20250429-071325.png"!

## Acceptance Criteria
* Provide a secondary action on Pack-Serving Link page to Edit Weights for items
* Filter and only display items that have *full* pack size information and are food items with proper serving size information (IN mockup showing all items to show flow)
* Provide Size Overview (This will be displayed on every step to emphasize changes) and should always dynamically update based on changes on the page
* Any items that were toggled on for “Use serving as Base Unit” on Step 1 will inherit the serving size weight as the base unit weight
* Display ghost text “Optional” base unit weight drop down for any items that already have “Servings per BU” entered on Step 1
* Any items that were neither toggled on to “Use Serving as Base Unit” OR were given “Servings per BU” MUST have Base Unit Weight 
** All items that were never eligible for Step 1 will also be required to enter Base Unit weight
** Provide alert if user goes past an item that is requiring Base Unit weight so it is clear that they must enter the weight to ensure a Pack-Serving Link
** Ensure the fluid/Non Fluid drop down logic for standard measures on weight/volume
* Provide a “Count Base Unit In” column that allows the selection of whether base units will be counted in whole or decimals
** Default to Whole
* Provide a way for user to go back to the first step to enter sub-quantities again if needed
** Provide a Unsaved changes prompt if user has made changes or entered new information on the weight step
