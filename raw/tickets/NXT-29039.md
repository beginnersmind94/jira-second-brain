# NXT-29039 — QA- File Imports - File-Drop/sFTP Capability for Customers

## Description
As a customer,  I would like the ability to import files into SchoolCafe in an automated fashion so that I don’t have to manually import files.

Following imports are eligible for SFTP Import:
1)Student Import (Non SIS API)
2)Allergen Import
3)Payment Import

Files with the file extensions configured with import will only be processed.
Only one file will be processed at a time.
The processed file will be deleted from the location irrespective of whether the import has been completed successfully or failed.


{color:#ff5630}I don’t think UX is needed for this.{color}

## Acceptance Criteria
* As a customer,  I would like the ability to import files into SchoolCafe in an automated fashion so that I don’t have to manually import files.
* As an end-user, I would accept 1 or more of the following options as acceptable:
** The ability to import by placing an import file or set of files onto a “fileshare” or something similar from which the file(s) would be automatically ingested
** The ability to import by transferring a file or set of files via sFTP to a location from which the file(s) would be automatically ingested
* I would like the ability to receive feedback for the file(s) I provided on their import status and to be able to see if there are any exceptions in the process
* I would like the data files that I provide for import to be separated from those of other import end-users so that other end-users are unable to see my data.

 =========

* Verify and validate that files with incorrect files extensions are not processed (updated test case)
* Verify and validate that the actual file is processed successfully.
** File is deleted after process and monitoring is updated.
* Verify and validate if any file name but actual configurations are processed (any file name but appropriate file config)
** File is deleted after process and monitoring is updated.
* Verify and validate that when multiple files of same type (different data)are dropped and only the actual files (appropriate configured) are processed successfully.
** File is deleted after process and monitoring is updated.
** Verify the status of the rest of the files (They all must be queued and processed in sequence.)
* Verify and Validate if incorrect file formats are placed, the file is deleted and reflects the same error in Monitoring dashboard.
** Verify the above with multiple files also.
* Verify and validate that when multiple files of same type (duplicate data) are dropped and only the actual files (appropriate configured) are processed successfully.
**  File is deleted after process and monitoring is updated.
** Verify the status of the rest of the files (They all must be queued and processed in sequence.
