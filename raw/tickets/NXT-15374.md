# NXT-15374 — Platform - PII - Implement PII Request Access

## Description
*Persona*: Perseus User

*Access level:* School District/Site

*Description*:

As a Perseus user, when a field/section/page is classified as PII and masked for viewing, I should be able to request access from the in-scope actions available.

*Requirements:*

* If the user cannot view PII, user should be able to request for PII access  by clicking on the “eye” icon at any of the data points (either at field/section)
* Modal prompt will be shown to user to capture scope (field/section) with cancel and submit buttons with a justification message.
* Respective Site Admin & CyberSoft administrators will be notified with an email and system alert notification for each request.
* {color:#ff5630}*Administrator views the request with link in email & system alert to open “Roles & Permissions” section under System*{color}
* Admin provides necessary permissions and “Approves” or “Rejects” the request with a reason.
* User is notified about the request update via email & system alert.

## Acceptance Criteria
* User shall be able to submit request the request for which data points (field/section/page)
* User shall be able to send notification for site admin/cybersoft admin for PII access
* Admin gets notified with detail PII access scope via email & system alert 
* Admin shall be able to review and take necessary action to “Approve” or “Reject” with a reason
* Users receives alert/email notification on request updates.
