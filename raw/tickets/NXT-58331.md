# NXT-58331 — PR- Plan/Record- Consider Status change if no menu item or category available 

## Description
As a production manager, I would like for the status to check against whether there are menu items or categories.

## Acceptance Criteria
* Status check against menu items and categories no matter what the change
* If Plan is complete before all categories or menu items are removed, roll back status to plan incomplete to show more accurate information for the user from the calendar and also not trigger the record tab and display the banner
* If Plan is incomplete and user adds serving group information and then saves, removes menu items and categories continue to leave the plan in complete status and display the banner 
* Use this an opportunity to switch all status check and changes to API code and no longer check/trigger from the UI
