# NXT-46749 — API – Photos – Aeries Incremental Update

## Description
As a SchoolCafé admin, I would like to be able to download student photos from Aeries if we don’t have one so that I can keep up to date with photos and not have to download them all every time.

*Note:*

* We already added in the ability to download photos for aeries
* This story is to enable the incremental updates for students that don’t have pics such as a new student

*Requirements*:

* For Add New and Edit API, do the following:
* When the user selects Import Photos, do the following:
* Show a field
** Toggle
** Title: Incremental
** If yes, the user wants to download photos only for students w/o photos in our DB
** If no, they want to do a full download
* If the user select any other type of import, do now show this toggle
* This toggle is for photos from any provider, although we only have Aeries right now
* When selecting students to use for the API endpoint, filter out the students that already have photos
** In other words, the list of students to get photos for are only those with no photos in our DB
* The rest of the import process is the same for photos

## Acceptance Criteria
* User can see and choose to download photos only for students with no photos
* This import can be scheduled and run per schedule
* The incremental import successfully imports photos for only students with no photos
* The full photo import still works w/o errors
