# NXT-67408 — Acc - Claims - Creation Workflow

## Description
As Dana the Director I want better support for Reimbursement Claims - a way to attach documents, reports, and a rollback feature.

*requirements*

* User has a guided workflow process for creating claims.
** Step 1: User can select programs.
** Step 2: User can trigger rollups.
** Step 3: User can generate reports and have them attached to the claim.
** Step 4: User sees a summary.
** More info on steps below.
* Requested actions are taken care of, and then the User is informed when the claim(s) are ready.

!image-20251013-210242.png|width=442,alt="image-20251013-210242.png"!



----

Step 1 - Programs

* User can see the programs they are actively using for the selected period. (Check Program Configuration)
* User can select whichever programs they want out of the available list. If only 1 is available, we should select it for them.

Step 2 - Rollups

* User can see if rollups are needed / suggested.
** We flag this on the Report Rollups page if any data > last rollup date exists.
* User can ask for reports to be triggered for the period.

Step 3 - Supporting Documentation

* User can select reports to be run for the period.
* Always run Detailed reports for the selected programs.
* They should be attached to the claim afterwards.

Step 4 - Summary

* User can review their selections and start claim creation.

## Acceptance Criteria
* User has a guided workflow for creating claims:
** can select programs;
** can trigger rollups;
** can select reports and have them available for download tied to the claim
