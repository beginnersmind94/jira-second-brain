# NXT-60122 — WS - Eco Dis Details

## Description
As a Workspace user, I want to view detailed KPI information so that I can understand performance metrics in depth.

h2. Comments

*Component*: KPI Details Panel

The panel shows:

* Name
* Date 
* Summary KPIs
* Detailed metric analysis
* Historical performance
* Contributing factors (optionally)
* School breakdowns (optionally)
* Action recommendations (optionally)

h2. User Flow

# User logs into Workspace and goes to Insights page
# User clicks on KPI card
# Panel opens

h2. Requirements

* KPI name

** If you can, use the full name: Economically Disadvantaged instead of Eco Dis. Eco Dis is on the dashboard only to save space
* Time period
** Date from the dashboard
* Close button



_Eligibility Summary Section:_

* Label: Eligibility Summary
** Grid

*** Label: Total Free/Reduced 
*** Count of students and percent of students against the total row
*** Note totals should match the Free and Eligible rows in the grid
*** Note: if you need to do as a sub total under those rows, that might be an option
** No paging, sorting, filtering



_Eligibility By School Section_

* Label: Eligibility By School
** Grid
** Columns:
*** School
*** Free

## Acceptance Criteria
h5. *Header Section*

* Displays an icon representing the KPI.
* Displays the full KPI name as a text label. If possible, use the full name, such as *"Economically Disadvantaged"* instead of "Eco Dis" (abbreviations are used only on the dashboard for space-saving).
* Displays the time period retrieved from the dashboard.
* Includes a close button that allows the user to exit the drawer.

h5. *Eligibility Summary Section*

* Displays a section labeled *"Eligibility Summary"*.
* Contains a grid with the following columns:
** Category
** Students – The count of students in that category.
** Percentage
* Includes a summary row at the bottom of the grid.
* Below the summary row, an additional row is displayed with:
** Label: *"Total Free/Reduced"*
** The total count of free and reduced students and their percentage against the overall total.
* The totals in this section must match the *Free* and *Eligible* rows in the grid.
* If needed, the total free/reduced row can be presented as a sub-total under the Free and Reduced rows.
* No paging, sorting, or filtering is applied.

h5. *Eligibility By School Section*

* Displays a section labeled *"Eligibility By School"*.
* Contains a grid with the following columns:
** School
** Free
** Reduced
** Paid
** Total
* No summary row is included.
* The sum of the columns (e.g., Free) should match the corresponding count in the Free row from the Eligibility Summary section above.
* No paging, sorting, or filtering is applied.

h5. *Summary Section*

* Displays an *"About This Metric"* section explaining the KPI.
* The explanation text is retrieved from Bolt.

* The KPI details drawer is read-only and does not contain any action items.
* The close button exits the drawer.

** Full width
** Compact
** Mobile responsive
** Expanded
