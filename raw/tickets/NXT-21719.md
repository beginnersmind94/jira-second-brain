# NXT-21719 — Update Model

## Description
As an administrator, I want to be able to be able to update the publishing of a model configuration so that I can add or remove sites to an already configured model

*Application*: Face and Food Detection Website

*Flow:*

* Browse to the Food Detection Settings page and then the Model Management Page
* Click on the Update button on one of model rows and be taken to the Create Model Page
* Since in “Update Mode”, the first tab is read only
* On the second tab, select the publishing information
* Select Update and it makes the necessary updates

 

*Background:*

* We need a way to add or remove a model from a site without having to rebuild a model
* We will leverage the previous code to do this

 

*Assumptions:*

* It is assumed you have done the stories for Add Tabs to Build Model, Add Publishing Models to Sites and Add Sites to Model Building
* If not, wait until then

 

*Requirements for Model Management page:*

* On the rows, under the actions column, add another button action and title it: Update
* The Update button, when clicked takes the user to the Create Model page
** It needs to send the Model ID to the page
** It needs to notify that page that it is the Update action

 

*Requirements for Create Model page:*

* You need to add in logic on page load to determine if the user clicked on Create Model button, Clone Model button or Update Model button
*  
* If clicked on Update Model, do the following:
** Get the Model ID of the model the user wants to clone
** Get the Model Configuration from the database using that ID
** Load all the fields on the page as from the values of the Model Configuration
** Make everything in the first tab read only
** The user can modify anything on the second tab
** Hide the Create button and put a new button in its place called: Update
*  
* Update Button:
** You need a way to know what changes the user made to this page
** Possible Changes: Add or Remove a site(s)
** If added a site(s), you will need to tell the Model Publisher to send this model to the new site(s)
*** Do it the same was it normally does, just skip the model building since the model is already built
*** This means you will need to update the Model to Sites table and need to call the publisher instead of the builder
** If removing a site(s), you will do the same thing as adding, but in opposite
*** you will not call build model
*** you will update the sites to model table and update the status value
*** you will call publisher to delete the model from the Edge device instead of publish to it
*  
* Security – already done
