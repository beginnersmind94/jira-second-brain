# NXT-5218 — Application Details - (Action) Delete/Restore

## Description
*User Story* 

As an Application processor, i can delete an application if needed.

*Mock Ups* 

*Requirements* 

* Delete option available 
** _Please select a reason and enter a comment to delete this application._
*** Reasons: 
**** All students are directly certified
**** Students approved on previous application
**** All students are at CEP sites
**** Duplicate application
**** Other (please specify in comments)
*** Comments box
*** Cancel > pick up where they left off
*** Delete> Application Status - Deleted 
*** The student eligibility should be restored to their prior eligibility
* After deleting the application, “Restore” action button should be made available in the application details. screen At this time, Delete icon should not be clickable action as it defines just a status that application is deleted.
** A restored application should return to a “Processed” status and should need to be re-processed by the user.
* Application status History tracking needs to happen for all of these actions.
** Action, user, timestamp.

## Acceptance Criteria
- Multi-language, ADA compliant
- application processor can delete an application
- app status is deleted 
- comment and reason required on delete
- app can be restored, but must be reprocessed
- history tracking for actions
