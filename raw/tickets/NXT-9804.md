# NXT-9804 — Data Exchange - File Import: File Upload Step

## Description
*User Story*

As a PrimeroEdge user, I would like to have a screen where I can quickly and easily select or drag and drop data import files, so that I can more quickly upload my data, review and trigger import job.

*Requirements*

* From File Import Dashboard, user should be able to see list of File Import Setup configurations in a tabular fashion with EACH row has “IMPORT” as available action.
* When user clicks on “*IMPORT*”. system should launch *“File-Upload” s*creen
* File Upload screen screen is rendered with *“Current File Name”* labelled field displaying File Import Setup configuration name to which user is trying to import the data file
* On right side of *File Import Name, File Upload widget* is available for user to *drag & drop* or browse the file explorer to *SELECT* the file:
*# Drag and drop a .csv or .xlsx, or .xls (if possible) file onto the Upload Files portion of the screen
** User can ONLY one file at a time and one is file is limited
** SELECT button is enabled for user to browse the file explorer and select the file which should filter for CSV, xlsx file formats by default
** Tooltip is displayed with allowed file types and max size as 250MB.
* When user adds the file, system *automatically uploads the file* to database with progress bar
* Once the user has successfully added a file, replace the Upload Files section of the page with the filename and file size of the file. (file size limit 250 MB for .xls, .xlsx and .csv file)
* Upload function:
** Remove File (Trash bin icon) - Allow user to remove the file and start this step over
** When user clicks on UPLOAD:
*** Loading/Progress bar is displayed with %
*** Progress bar should disappear once the upload is complete and a DELETE icon is enabled for user to delete
*** Upload successful banner message should be displayed and disappears after 10 secs. 
*** When user clicks on DELETE
**** Display a confirmation prompt, “Are you sure you want to delete ?”
**** If user selects DELETE button, display File Upload widget
**** If user selects CANCEL, return to uploaded file widget
*** The file is validated to ensure it is the proper type, and an error is displayed if it is not. File can either be validated upon drag+drag/browse OR when selecting the Upload button--whichever is more feasible.
*** Rules for successful upload:
***# File must be .csv, .xlsx, or (if possible) .xls. 
***# File should have a reasonable size limit (architect can advise)
*** Error messages: 
**** File type must be \[support file types].
**** File size must be no larger than 250 MB.
* Add a Previous button that allows the user to return to the Pre-Upload step. 
** *WARNING message that says “Going back to previous step will remove the files added for upload”*
* Only after file upload success & delimiter/qualifier are selected, Enable Next button that allows the user to proceed to the next step: *Preview* screen

## Acceptance Criteria
* Verify that within the File Import Dashboard page, when the user selects the IMPORT button on any of the row in File Import Setup config table should launch "File Upload" page for data file import

* Verify that "Current File Name" is displayed with value from File Import Setup configuration file

* Verify that a tooltip is displayed with allowed file types as txt, csv & xls, maximum size allowed is 250MB.

* Verify that the user has 2 options for importing a file on this step:
1. Drag and drop a .csv or .xlsx, or .xls (if possible) file onto the Upload Files portion of the screen
2. Use a browser to SELECT a file from their machine - launch file explorer to open the file with file types filter

* Verify that when user uploads/selects file, loading bar is rendered with % of progress and disappears after upload is complete.

* Verify banner text is displayed as "Import file upload is complete" message is displayed once upload the complete. 

* Verify that a user can only upload a ONE file at a time

* Verify that we once the file has been uploaded (and possibly validated at this point), we display 2 options for the user:
1. Delete File Icon - Allow user to delete the file and start this step over with File Upload drag and drop widget displayed
2. Progress bar should disappear and banner text of file upload success is displayed for 10secs

* Verify that File deletion confirmation prompt is displayed

* Verify the file is validated to ensure it is the proper type, and an error is displayed if it is not. Rules for successful upload:
1. File must be .csv, .xlsx, or (if possible) .xls. 
2. File should be less than the maximum size limit

* Verify potential error messages: 
1. File type must be in one of the formats [txt, xls, csv].
2. File size must be no larger than [250 MB].

* Verify each function behaves in a stepper format

*Verify that, once a valid file has been selected and uploaded, the user can continue to Preview step
