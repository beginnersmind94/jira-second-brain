# NXT-28476 — INV - Receipts - Edit Receipt - Update QOH for Commodity Contract Items - API Only

## Description
As a FSD I want the system to check *Available Quantity* when I edit an existing receipt so it can adjust the warehouse’s available quantity accordingly.

*Requirements*

* Create trigger upon *Editing* site receipt for *Commodity Contract Items*
* Adjust Available Quantity at the Commodity Contract according to the quantity received
** hit the API to reduce quantity or add quantity (if receipt quantity is reduced)
* Do not limit the receipt to only the available quantity
** this could cause the warehouse to go negative quantity
** this may be a setting in the future, to allow or disallow receiving more than available quantity

## Acceptance Criteria
* Check Commodity Contract available quantity when editing receipts
* Adjust commodity item *Available Quantity* according to receipt adjustments
