# NXT-15309 — Verification - Details - More Information Required

## Description
As Vera the Director I want to ask the family to submit additional documentation for their verification response, so that they can respond directly through the Family Hub rather than in the form of a separate email that I have to keep track of.

*Mockup*

[https://www.figma.com/proto/Yo99YhQm5Xk91Ie7b0PRkD/Front-Office-6.0?node-id=2349%3A23045&scaling=scale-down-width&page-id=2349%3A23043&starting-point-node-id=2349%3A23045|https://www.figma.com/proto/Yo99YhQm5Xk91Ie7b0PRkD/Front-Office-6.0?node-id=2349%3A23045&scaling=scale-down-width&page-id=2349%3A23043&starting-point-node-id=2349%3A23045|smart-link]  

(In classic, this didn’t exist.)

*Requirements*

* If an applicant has responded via the family portal, show the user that information.
** Add an ‘Applicant Response’ section to the top of the Verification Details page.
*** Show any messaging back from the household (if necessary, we can show this in the comments, if the ‘Message from the applicant’ in the mockup will not work).
*** Show any documents that were submitted via the portal.
* After a household has responded via the family portal, users need a way to accept or ‘reject’ the response and ask for a new response.
** Action - Accept / Reject, user picks one.
** If accepting, the verification completion process continues as it already does currently.
** If rejecting, the user can enter a custom message back to the household.
*** Here is the default message, which the user can also overwrite: _“More information is required for us to process your response. Please re-upload any supporting documentation.”_
*** Any submitted documentation should be kept attached.
*** Any ‘responded’ flags should be removed from the application.
** History should be updated with the action taken.
** Status should be updated.
** Create a special permission for ‘Reject’ button in Roles and Permissions: ‘Can reject a Verification Response’.
* *This will need to integrate with the team doing the 2.0 Family Portal to allow for this re-submission.*
** *We will handle this in a separate story where:*
*** *The parent will get a ‘rejected’ letter asking for more information (also likely another push alert if they have the Family Hub app, and a high-priority message)*
*** *The parent will have a ‘reactivated’ Verification Response page to resubmit details.*
*** *There will be a last action of ‘Previous response was rejected’*

## Acceptance Criteria
User can ask a household to submit additional information.

User can enter a custom message that will integrate with the parent portal.

Verification status should update appropriately - if we’re tracking that a ‘response’ has come in, then it should be updated.

Documentation should be retained.

Status should be updated.

History updated with the action taken.
