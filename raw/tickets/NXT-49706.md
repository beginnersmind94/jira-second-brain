# NXT-49706 — 2FA - Profile

## Description
As a SchoolCafe user, I would like to be able to setup 2FA in my profile so that I can be in compliance with my district’s requirements.

*Notes*:

* This story is for the user setting up 2FA in their user profile
* It is for all authentication methods
* It assumes you have already done one-time passcode stories

*Flow:*

* User browses to Workspace > My Profile > Authentication tab
* User views if an authentication type has 2FA or not
* User clicks on 2FA to open the configuration drawer
* User sets up configuration in the drawer and saves

*Requirements*:

*Authentication Tab*

* On the Workspace > My Profile page, do the following:
* 
* On the grid, 
** If an authentication method row has 2FA, do the following:
** Add a button to the Actions column
*** Label: 2FA
*** Tooltip: Configure 2FA
*** On click:
**** Open the Configure 2FA drawer
*** Only show 2FA button if the method has 2FA, else hide
* 
* If the user has configured the 1st factor, but not the 2nd factor:
** Make the Configured = No for that method even though the first factor is configured
** Put tooltip: Configure 2nd Factor



*Configure 2FA Drawer*

* On the Workspace > My Profile > Authentication, do the following:
* When click 2FA button on a row, do the following:
* 
* Create a drawer, if not already exists:
** Title: Configure 2FA
* Fields:
** Info button:
*** This is a list of 2nd factor authentication methods permitted by your district for this authentication method. Ensure each one is configured and select the one you prefer as your default method.
** Grid
** Columns:
*** Preferred
**** Same as Profile Authentication grid
**** If there is only one method in the grid, automatically mark it as preferred
*** Configured
**** Same as Profile Authentication grid
*** Name
**** Same as Profile Authentication grid
*** Action
**** Configure
**** On click:
***** Open Configure [auth method] drawer
***** Depending on auth method, open that specific drawer (covered below)
** Save
*** Validate:
**** They must have at least one 2FA configured 
***** If not, show message: Configure at least one method
**** They must have at least one preferred
***** If not, show message: Select a preferred method
*** Saving:
**** Update the status of the authentication, if necessary
***** Remember, for auth methods with 2nd factor, both 1st and 2nd must be configured, so if it wasn’t configured b/c of 2nd, check and see if should set the Configured Status to Yes



*Configure 2FA Type drawer*

* Title: name of the original drawer
* use the sliding motion
* If the user clicks Configure on the Configure 2FA drawer, it will open this drawer like an overlay
* In this drawer, the user does the config for the specific method
* Open the config drawer for the type of method they are using as the 2nd form factor
** These drawers have already been done in previous stories, so reuse them
** For example:
*** If the 2nd factor is SMS OTP, then open the SMS OTP config drawer
*** If the 2nd factor is Google Authenticator, then open the Google Authenticator config drawer
* Ensure this drawer has a back button that goes back to the previous drawer

## Acceptance Criteria
* User can see if an authentication type has 2FA enabled
* User can configure the 2nd factor
