# NXT-30317 — Production - Production Adjustment (Save Affected Plans) (Step 3)

## Description
As a menu planner, I need to be able to make changes to multiple sites within my district at one time. This could be because multiple sites are having an event such as grandparents day, or because of product that is not getting delivered from the vendor, but they’ve offered a substitute item, which I now need to enter on the menus of sites that will be receiving that item. 

Making the selections for Meal Type, Site Group, Menu Line, and the Date or Date Range that I want to impact, and then clicking the button labeled “Select Menu Item,” opens a window that allows me to choose a menu item. I then can search for the menu item, and click on the selection that I want to add to the menus.

After that, I can select a different serving size (if there are more than one) and display all serving groups and Meal Patterns that are selected in the date range and criteria

*Populate all plans that are available for menu item selected to be added to (DO NOT INCLUDE PLANS THAT ALREADY HAVE THIS MENU ITEM)*

Once Plans are selected, user will click on Submit Adjustments, and there will be a page that will show an overview of all affected plans

## Acceptance Criteria
This will only impact a site with a menu plan that fits into the criteria selected. Do not include plans on the list that already have this menu item

If the production plan is in a “Plan Complete” status, inserting the new menu item should go with a zero in the count, so that the production plan will remain in “Plan Complete” status. 

If it is in plan incomplete status, user will need to go into the individual plans and make the counts.

When user clicks “view” only allow read access, user cannot make changes. 

IF user at the time of adjustment, goes to the plan and adds that menu item, then the adjustment happens, disregard adjustment. 

Do NOT SHOW ANY RECORD COMPLETE PLANS.

Only categories that are part of a particular production plan should be available for the menu item to be added to.

Default the category to the category that the menu item is part of unless that category is not available on the production plan.

Use all serving groups available based on site group and menu line selection within the date range. Use whatever serving size is applicable per plan. 

EX: Breakfast Menu Line: Main ( There are 5 schools 2 of them have K-5 and 6-8 and 3 of them have K-12 and 9-12)

Display:

*Meal Pattern and Serving group populates based on all plans within that date range and any serving groups that are available*
