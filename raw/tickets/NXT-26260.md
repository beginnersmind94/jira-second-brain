# NXT-26260 — IM - Edit Item Substitution

## Description
As an authorized user I want to be able to search for similar items so I can select  from al list of possible substitute items and tag to another Item as as substitute item(s)

## Acceptance Criteria
*Entry Point*

* Item Managements → Item  → Item Name → Item Substitution toggle switch  (ADD/EDIT item )

*A screen shall be presented to the user that provides the following features:*

* On ADD / EDIT item screen a toggle switch shall be available to the user to select if Item substitution is going to be added
* By switching the toggle button to off:
**  the Item substitution accordion is not displayed
** No item substitution is added to the item
* By switching the toggle button to on:
**  the Item substitution accordion is displayed
** User shall be presented with an option to search by Item Name or code (Smart search box)
*** If a Item Name / Code is entered them
**** If a matching item(s) exist the item(s) should be displayed for selection
**** If more than one matching Items are available 
***** them a list of available items shall be displayed to the user.  
***** The user shall  be able to add item to the grid by clicking the ADD button  (more than one item can be added)
***** The corresponding conversion factor for any selected item  shall be enter by the user before adding and saving the record.
***** The user shall be able to indicate if the item is interchangeable (check box)

*A search and save to list option shall also be presented to the user to enable the search of items by filters  (ADD Button) Note: This should be rename with a more user relatable  field name*

* *Option to filter items by*
** Item Category (drop down listing Item Categories)
** Storage Category (drop down listing Item Categories)
** Item Description Like (Smart search on  Items - Item Name)
** Item # Like   (Smart search on  Items - Item Code-)

* *APPLY Button \[ Filter(s) to Search]*
** When press or clicked the user shall be presented with a list of items that meet the filter criteria
*** The user shall be able to select and item (s) and tag it(them) 
*** A single item shall be selectable by clicking add button to its right  
*** More than one item shall be selectable. When more than one item is selected  and ADD button shall be displayed indicating that all selected items will be added to the add list.
* *Items on selected List*
** User must enter equivalent quantity and unit type  for substitute item and source Item
** The user shall be offer the option to delete and remove the selected item from the list as substitute from 
* *CLEAR button*
** When clicked or pressed the system shall clear the field content.  
* *ADD button*
** When click or press the system shall add select item information to the list
* *SAVE button*
** Once the user is done selecting and adding items to the list and click or press the SAVE button the transactions shall be saved and the system shall return to add/edit mode
* *CANCEL button*
** If user press or click on CANCEL button no record/transactions shall be created.
** The system shall clear all item substitution fields and return to Item add /edit mode

*List (Filter by option)*

* *Headers*
** Item #
** Item Name
** Storage Category 
** Action

*List (Result Grid)*

* *Headers:*
** substitute Item#
** Substitute Item Name
** Conversion Factor (Quantity - Unit Type of substitute item  ) = (Quantity - Unit Type of item  )
** Interchangeable
** Action (delete from add list)

*Mandatory Fields*

* Sub Item#/Name
* Conversion factors for example
** 1 CASE (of substitute item) = 20 each  (of  source item)



*General:*

* List = Result Grid 
** 1 CASE (LI-17379) =  20 Each (LI-17374)
* Unit (Hierarchy Details - Unit Type)
* Page Navigation shall be keyboard friendly 
* Applies to List and Split view
* Applicable to Food Items and Non-Food Items
