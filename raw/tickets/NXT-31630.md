# NXT-31630 — Verification - Reporting - Add Backdated Applications report

## Description
As a central office admin, I want to see a list of applications that are causing my verification report figures to make no sense, so that I can make corrections as needed.

*Requirements*

* In Eligibility → Verification → Reporting, add a report that shows a list of applications that were processed after the sample date (typically 10/1) but backdated to before the sample date.
* Report name: *Backdated Applications*
* Report description: _The following applications were processed after the sample date but backdated to before the sample date._
* Columns:
** Application #
** Student ID #s
** Process Date
** Received Date
** Status
*** Active - An application should show as ‘*Active*’ if +at least one+ student on the app is _currently_ receiving benefits based on the result of the app (i.e. if the app was processed as ‘Reduced’, at least one student is still currently receiving that same ‘Reduced’ benefit).
*** Inactive - An app should show as ‘*Inactive*’ if +none+ of the students on the app are currently receiving benefits based on that app’s result.
* No custom views are necessary.

Verify that the Backdated Applications report has been added to the Verification Reporting page

Verify that these applications show a Status of ‘Inactive’ if *none* of the students on the app are still receiving benefits based on the app’s result
