# NXT-36512 — Roles & Permissions - New Screen - Part 1

## Description
As a SchoolCafé admin, I want a more intuitive way to set permissions for given roles so that I can control who has access to what.

*Flow:*

* Open SchoolCafé and browse to Settings > Roles & Permissions (R&P)
* View list of roles
* Optionally click on + New to create a new district role
* Optionally click on Edit on a role to edit permissions
* Optionally click on History on a role to see changes
* Optionally delete a custom district role

*Requirements*:

Updates to Resources:

* Since the permissions are tied to resources, do the following to the underlying resources
* Enable hierarchy by adding a parent / child relationship between resources
** Hierarchy:
** Module
*** Page
**** Sub Page
**** Tab
**** Section
**** Function
* Be able to define if a resource has: View, Add, Edit, Delete actions
** We can then use this to determine enabled / visibility on the R&P grid for the checkboxes
* Enable a resource to be visible / invisible in the permissions screen
** might be able to use the View tag in the previous bullet point
* 
* We need a way to distinguish between an internal role and external role
** Example Internal role: Customer Support role



Updates to R&P Dashboard:

* For the System > Roles & Permissions landing page, do the following:
* Remove the permissions stuff on the right side (will become a drawer)
* Keep + New button
** Opens up the + New drawer
* Make the left table full screen
* Get rid of the extra row on top of the grid
* Add Users to the columns
** values: count of users in that role
* Remove Type from the columns
* Add Edit action button on each row
** Title: Edit
** On click, open the Edit Permissions drawer
* Ensure the user can clone and delete a role the same way we do currently
* 
* Internal roles are only visible and editable by users with Customer Support role



Updates to +New drawer

* Remove Modules
* Keep name, description, cancel and save the same
* All roles created using this UI are external roles
