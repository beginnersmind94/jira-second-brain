# NXT-6697 — Data Exchange - File Upload Step - Warn user when existing configuration is detected

## Description
*User Story*

As a PrimeroEdge user attempting to configure an import type with an existing configuration, I would like to be warned about that before starting the process so that I can safely choose to either overwrite that configuration or keep it.

*Mockup*

[https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=4893%3A77674&scaling=min-zoom|https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=4893%3A77674&scaling=min-zoom]

*Requirements*

* When a user navigates to the Upload File step and the user successfully uploads a valid file and attempts to proceed, perform a check to determine whether there is an existing configuration for the type.
** If there is not, have the user flow proceed as normal according.
** If there is, display a warning message/modal with a Yes/No prompt: “An existing configuration was detected for this import type. Do you want to overwrite this configuration? All previous changes will be lost.”
*** If ‘No’, close the modal and cancel the user’s selection/reset the process
*** If ‘Yes’: 
**** Discard the previous configuration for the selected Import Type
***** Database record deletion can happen as needed
***** Success message/confirmation is optional here
**** Proceed with the rest of the file upload → field mapping flow as normal

## Acceptance Criteria
* On the File Upload screen, warn a user with a Y/N prompt when they have uploaded a file for an existing configuration and attempt to proceed to the next step
* If No, close the modal and cancel the user's selection/reset process
* If Yes, discard the previous configuration for the selected Import Type and (optional) display a confirmation
* User can then continue through upload process as normal
