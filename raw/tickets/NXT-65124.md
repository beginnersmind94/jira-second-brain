# NXT-65124 — Fin - Ledger View (UI)

## Description
Users need to view the transaction history and running balance for a specific GL account over a selected time period.

**Dev Notes:**

* This sprint covers MANUAL entries only
* System-generated entries (automated postings from POS/Inventory modules) will be implemented later
* Base type (Asset/Liability/Equity/Revenue/Expense) is determined by the account's assigned category in the general ledger configuration
* Balance calculation must follow GAAP standard accounting rules based on base type
* ~~Default fiscal year to begin from 7/1 to 6/30~~

## Acceptance Criteria
# **Page Header:**

- Page title displays "View Ledger"

# **Help/Info Section:**

- Collapsible info panel at top with header "What does this page show? (Click to expand)"
- When expanded, shows two sections side-by-side:

**Left section - "What is a Ledger?"**

- Header with bar chart emoji: "📊 What is a Ledger?"
- Text: "Think of a ledger like your bank account statement. It shows you the running balance for a specific account (like 'Student Prepaid Accounts' or 'Cash in Bank') over time. Every time money goes in or out, you see exactly how it affects your total balance."
- Example text in italics: "Example: If you want to see how much money students have in their lunch accounts right now, and what transactions changed that amount, you'd look at the Prepaid Account ledger."

**Right section - "Journal vs Ledger - What's the difference?"**

- Header with pencil emoji: "📝 Journal = Daily Diary"
- Text: "Shows ALL transactions that happened on specific days, like 'June 15th: Lincoln High lunch sales, Kennedy Middle supply delivery, etc.'"
- Header with bar chart emoji: "📊 Ledger = Account History"
- Text: "Shows how ONE specific account changed over time, like 'Everything that affected the Student Prepaid Account balance.'"

**Quick Tip:**

- Light bulb icon with text: "💡 Quick tip: Use the filters above to pick which account you want to see, then view how deposits and purchases affected that account's balance over your selected time period."

# **Filter Options:**

- Searchable dropdown labeled "Account" with red asterisk (required field)
- Dropdown populated with all GL accounts from the chart of accounts
- User can type to search/filter accounts
- Date range picker with two fields:
  - "From (Posted Date)" date field (format: MM/DD/YYYY)
  - "To (Posted Date)" date field (format: MM/DD/YYYY)
- Purple "Apply" button to execute filters
- When account is changed, date filters remain the same

# **Ledger Summary Line:**

- Below filters, shows: "Account: [Account Name] | Date Range: [MM/DD/YYYY] to [MM/DD/YYYY]"
- Account name is plain text (not clickable)

# **Ledger Table Display:**

- Table with columns (left to right):
  - Posted Date (format: MM/DD/YYYY, sortable with ↕ icon)
  - Transaction ID
    - Clicking the transaction ID shows the complete GL account breakdown for this transaction.
  - Location (text, e.g., "001 Lincoln High School" or "District")
  - Description (text)
  - Entered Date
  - Entered By (username)
  - Debit (currency format with $ and commas, shows empty/blank if $0)
  - Credit (currency format with $ and commas, shows empty/blank if $0)

**Reversal Display:**

- Original transactions that have been reversed show yellow badge with text "REVERSED" next to description
- Original transaction shows clickable link "Reversed by #XXXXX" (where XXXXX is transaction ID, displayed as brown/orange hyperlink that scrolls to that transaction in the current ledger view)
- Reversal transactions show yellow badge with text "Reversal of #XXXXX" (where XXXXX is clickable brown/orange hyperlink that scrolls to the original transaction in the current ledger view)
- Reversal transaction descriptions are prefixed with "REVERSAL: " followed by user-entered reason

**Account Balance Bar (Fixed at Bottom):**

- Bar fixed at bottom of viewport (stays visible when scrolling), styled consistently with existing UI theme elements
- Shows: "Account Balance for [Account Name]: [Balance Amount]"
- Below header, displays four rows vertically stacked with labels left-aligned and amounts right-aligned: 
  - Top Line (in header): “Current Balance (as of MM/DD/YYYY):” followed by dollar amount
    - Date shown is today’s date (system date when the page is rendered/refreshed).
    - Amount is calculated using all posted transactions for this account (and selected location) through today, following the account-type balance rules.
    - Displayed in large, emphasized text with positive-number styling per UI theme. This is the primary focal point of the balance section.
  - Row 1: “Beginning Balance (MM/DD/YYYY):” followed by dollar amount
    - Date shown is the From (Posted Date) from the filters.
    - Amount is calculated based on account type using all posted transactions before the From date (see balance calculation rules).
  - Row 2: “Total Debits:” followed by dollar amount
    - Sum of all debit amounts for this account within the filtered Posted Date range (inclusive).
  - Row 3: “Total Credits:” followed by dollar amount
    - Sum of all credit amounts for this account within the filtered Posted Date range (inclusive).
  - Row 4: “Ending Balance (MM/DD/YYYY):” followed by dollar amount
    - Date shown is the To (Posted Date) from the filters.
    - Amount represents the net balance at the end of the filtered period, calculated from Beginning Balance + period debits/credits per account-type rules.
    - Displayed with standard emphasis (secondary to the Current Balance at the top)
- All dollar amounts align vertically on the right side of the bar. Labels are left-aligned. All amounts use standard accounting format with dollar sign, commas, and two decimal places.
- If any balance amount (beginning balance or ending balance) is negative, display in red with parentheses: ($1,250.00)
- Balance Calculation Rules by Account Type: Account type (Asset/Liability/Equity/Revenue/Expense) is automatically determined by the account's assigned category in the chart of accounts.
  - Beginning Balance Calculation: 
    - For Asset accounts (Cash, Inventory, Prepaid, Receivables): - Beginning Balance = SUM(all debits - all credits) WHERE posted_date < start_date filter
    - For Expense accounts: - Beginning Balance = SUM(all debits - all credits) WHERE posted_date < start_date filter AND posted_date >= current_fiscal_year_start_date - If start_date filter is at or after the current fiscal year start date, beginning balance reflects activity since fiscal year start - If start_date filter is before the current fiscal year start date, beginning balance = $0.00 (expense accounts close at fiscal year end)
    - For Liability accounts (Accounts Payable, Accrued Expenses): - Beginning Balance = SUM(all credits - all debits) WHERE posted_date < start_date filter
    - For Equity accounts: - Beginning Balance = SUM(all credits - all debits) WHERE posted_date < start_date filter
    - For Revenue accounts: - Beginning Balance = SUM(all credits - all debits) WHERE posted_date < start_date filter AND posted_date >= current_fiscal_year_start_date - If start_date filter is at or after the current fiscal year start date, beginning balance reflects activity since fiscal year start - If start_date filter is before the current fiscal year start date, beginning balance = $0.00 (revenue accounts close at fiscal year end)
  - Ending Balance Calculation: 
    - For Asset accounts: - Ending Balance = Beginning Balance + Total Debits - Total Credits 
    - For Expense accounts: - Ending Balance = Beginning Balance + Total Debits - Total Credits 
    - For Liability accounts: - Ending Balance = Beginning Balance + Total Credits - Total Debits 
    - For Equity accounts: - Ending Balance = Beginning Balance + Total Credits - Total Debits 
    - For Revenue accounts: - Ending Balance = Beginning Balance + Total Credits - Total Debits

**Sorting Behavior:**

- Entries are sorted by Posted Date in descending order (newest first) by default
- User can click column headers to sort by that column
  - Entered Date
  - Transaction ID
  - Debits
  - Credits
  - Username
  - Allow user to type in to filter down to what they need
- Sorting selection persists when changing date filters (but resets when changing account)

**Pagination:**

- Ledger entries are displayed with pagination controls at bottom (Previous / Page X of Y / Next)
- Each page loads a limited number of entries (10 per page)
- Pagination maintains the current filter and sort state across pages

**Empty State Handling (No Transactions in Selected Date Range):**

- **When triggered**
  - Condition: there are **no transactions with Posted Date between From and To (inclusive)** for the selected account / location.
- **Transaction table**
  - Do **not** show any rows.
  - Instead, show the message centered in the table area:
    “No transactions found for this account in the selected date range.”
- **Account balance bar behavior**
  - The balance bar **shows and uses the normal balance logic**.
  - Values are calculated as if the table had zero rows in the period:
    - **Beginning Balance (From Date)**
      - Calculated per account-type rules using **all posted transactions before the From date**.
      - This may be **non-zero** if there were transactions before the selected period (e.g., July activity when August is selected).
    - **Total Debits**
      - `0.00` (because there are no transactions in the filtered date range).
    - **Total Credits**
      - `0.00` (because there are no transactions in the filtered date range).
    - **Ending Balance (To Date)**
      - Equal to the **Beginning Balance**, since there is no activity in the selected range.
      - Calculated as: `Ending Balance = Beginning Balance + period debits/credits (which are zero)`.
    - **Current Balance (as of today)**
      - Unchanged; always computed from **all posted transactions through today**, independent of the selected date range.
- **True “no history” case** (optional explicit rule)
  - If the account has **no posted transactions at all** (before or within the filter range):
    - Beginning Balance = 0.00
    - Total Debits = 0.00
    - Total Credits = 0.00
    - Ending Balance = 0.00
    - Current Balance = 0.00
