# NXT-35336 — Acc - Sessions - Move Session to another date

## Description
As Dora the Director I need to move a session to another date entirely as an error was made by a cashier. This will save time instead of having to delete and re-create all transactions, or I have to ask Support to do this for me with developer time.

*mockup*

[https://www.figma.com/proto/QRowEx99f8nKGMmpZPeLRq/Front-Office-6.3?node-id=264%3A5546&scaling=scale-down-width&page-id=264%3A5195&starting-point-node-id=264%3A6224|https://www.figma.com/proto/QRowEx99f8nKGMmpZPeLRq/Front-Office-6.3?node-id=264%3A5546&scaling=scale-down-width&page-id=264%3A5195&starting-point-node-id=264%3A6224|smart-link] 



*requirements*

* Change the Move Session action to a dropdown.
** Add Move Session as an action in this list.

Enhance the existing Move Session option:

Inside of Sessions, we have a Move feature.

!image-20221130-211218.png|width=582,height=324!

* Add text to the top of the drawer:
** You can move the session to another date, or to another site. 
* User should pick if they are moving the site, the date, or both.
* Depending on what they pick, the options become available.
** If they pick Date, a date picker can be used.
** If they pick Site, Site and Terminal dropdowns can be used.
* Comments are always required.
* Movement should be tracked in Session History.
** Track the original session date → new session date, who did it, when.
* Rollups need to be queued for any dates involved.
** Original session date, new session date. 
* Rules for the date picker:
** Date can be up to 7 days in the future.
** Selected date should not be in a closed period.
*** If it is, error message: Selected date is in a closed period. Period must be opened first.

Role Permission

* Add two Role Permission tied to the Move Session actions
** Move Session - Change Site
** Move Session - Change Date

## Acceptance Criteria
* Text added to Move feature
* Workflow updated to allow user to pick a date / site / both.
* Session History updates with changes.
* Rollups performed on dates involved.
* User can’t pick a date in a closed period.
* User can pick up to 7 days in the future.
* 2 Role Permissions added - if the user has 1, they can see Move Session. If not, they cannot see it.
