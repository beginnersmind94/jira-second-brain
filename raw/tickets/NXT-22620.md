# NXT-22620 — INV - Orders - Update Orders to check Pricing Conditions

## Description
As a Procurement Specialist I want the Create Order process to reference the Contract Item Pricing Conditions so I get the correct price for items when ordering.

*Note:* applying updated prices to edited transactions will be handled in separate stories.

*Requirements*

* With the new Pricing Conditions ([NXT-7241|https://primeroedgenext.atlassian.net/browse/NXT-7241]) being added for each Contract Item, there is the ability to configure prices by date range
* The Create Order logic needs to be updated to determine the order price for an item by checking the pricing conditions for the order’s delivery date
** Example Price Conditions:
*** $5.50 (1/1/2021 - 12/31/2021)
*** $6.15 (1/1/2022 - 12/31/2022)
** Example Order:
*** Order Date: 12/15/2021
*** Delivery Date: 1/2/2022
** In the example above, the item price should be $6.15 because the order’s delivery date is in January after the new prices will apply
* On the Create Order screen (Enter Item Quantities)
** the price for each item should be displayed according to the delivery date
** if a specific Delivery Date is selected in the header, all items should show their price for that date
* On the Confirm Order screen
** if the Delivery Date is changed for an order, update the prices according to the new date
