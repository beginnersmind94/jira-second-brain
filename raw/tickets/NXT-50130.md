# NXT-50130 — LMA - License Keys - Remove

## Description
As a License Management (LMA) role member, I would like to remove a license to a license key so that I can update it

*Notes:*

* The use case for the story is LMA role user accidently gave them a module they didn’t purchase and we need to remove it
* Another use case is the customer cancels a purchase of a module
* Business rules at play here: 
** We cannot delete license keys
** We cannot delete licenses from license keys once they have been linked to a district
* So instead of “updating” the licenses in the license key, we will generate a new one that cancels out the original.

*Flow:*

* District buys 10 site licenses for Accountability
* They then decided they don’t want them and we agree to refund and cancel the licenses
* LMA role user browses to System > License Management
* User sees a list of licenses generated for the software in the grid
* User clicks Remove License button on a row
* Remove License drawer opens and user puts -10 site licenses for Accountability and saves
* System generates a new license key with -10 site licenses for Accountability
* System automatically links the new license key with the district
* System cancels out the original license key
* License Keys grid updates with the new information
* When district users login, they now no longer have access to Accountability

**** -Always enabled for any status except New-
**** {color:#bf2600}*Update: Always enabled for only Active*{color}

**** Disabled for everyone else



*Remove License drawer*

* If the drawer does not already exist, create it
** Title: Apply License
** Width 1/2
** Use sliding motion



*Header*

* Show these fields are read only:
** District
** License Key
** Usage message (see bolt code)
** Effective Date
*** Defaults to today
** Expiration Date
*** Defaults to existing
* Show these fields as editable and required:
** Notes
*** Default the text per Bolt Code



h4. Current Licenses

* A table of current licenses
* Show read only:
** License name

* Show editable:

## Acceptance Criteria
h3. *1. Access and Navigation*

* From the *License tab grid*, clicking *Remove License* opens the *Apply License* drawer.
* *Remove License button* is enabled for all license statuses *except New*.
* *Disabled for all other roles* except License Management (LMA).
* Only *License Management (LMA) role* can click this button.

h3. *2. Drawer Layout*

* Title: *Apply License*
* Width: half page
* Drawer opens with a *sliding motion*.

h3. *3. Header Section*

* Read-only fields:
** District
** License Key
** Usage message (from Bolt code)
** Effective Date (defaults to today)
** Expiration Date (defaults to existing)
* Editable and required field:
** Notes (default text per Bolt code)

h3. *4. Current Licenses Table*

* Displays all *current licenses*.
* *Read-only fields*:
** License name
** Deployment value (Site, Device)
* *Editable fields*:
** Count
*** Default to actual existing count
*** User *cannot increase count*, only decrease
*** Reducing count indicates licenses to remove
*** Setting count to *0* removes the entire license
*** Maximum value = original license count (cannot remove more than available)
* *Revenue field is not displayed*.

h3. *5. Remove Licenses Button*

* Only *LMA role* can click.
* Label: *Remove Licenses* (bolt code previously had incorrect text).
* On click:
** Validate if there are any changes and required fields.
** Show confirmation message:
_"This will immediately update the license with your subtractions. Are you sure?"_
** If confirmed:
*** Generate a *new license key*.
*** Set all values same as the existing key except:
**** Start Date = today()
**** Notes
**** Generated Date = Now()
**** Generated By = current user
**** Licenses grid updated with reduced counts or removed licenses (count=0)
*** Automatically link the new license key to the district.
*** Cancel the existing license that was updated and append a note:
_"Canceled because it was overridden when removed licenses from it."_
*** Show success message.
*** Close drawer and return user to License Keys page with *updated grid*.
*** Log change as an *Update* in the History table.

h3. *6. Security*

* Only *License Management (LMA) role* can take actions on this page.
