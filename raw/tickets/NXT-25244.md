# NXT-25244 — INV - Withdrawals - Create Production Withdrawal - Menu Lines & Items (Step 2)

## Description
As an Inventory Manager I want the Production Withdrawal item review process to allow me to review multiple menu lines in a single process so I can get my job done more efficiently.

*Requirements*

* Update the Add Items (step 2) screen of the Create Withdrawal to handle multiple production records on the screen
* Populate all viable inventory items retrieved from the menu lines of the selected date
* Show a panel which includes all menu lines for the day as selectable items
** when a menu line is selected, load those items in the items list
** provide a status for each menu line
*** New - for all menu lines which have not yet been withdrawn
*** Selected - for the menu lines which have been selected to be confirmed on the next step
*** Withdrawn - for menu lines already withdrawn on a previous cycle 
** No need for search filters, as there will only be 2-5 menu lines on average
* Include the existing ‘Add Item’ control
** change the component to be a collapsible/expandable accordion style
*** collapse by default for Production type
** allow items to be added to the selected menu line
*** indicate added items as ‘Non-Production’ items
* Populate the Items grid 
** Show Quantity On Hand
*** include Alternate QOH popover
*** Show in True QOH, but may change to Highest Unit QOH later
** Use the updated format for pack size expand/collapse, etc.
** Allow quantities to be edited
*** quantities are pre-populated based on production data calculations
** Allow items to be deleted
*** Provide warning confirmation prompt
** Provide a Confirm button for the whole Menu Line 
*** this will select the specific menu line to be moved to Confirm step
*** this will also change the menu line status to ‘Selected' from ‘New’
** *Remove* the Comments icon for each item
*** Provide only on Confirm Step 3
* Provide validation error messages
** Not enough QOH - “Please check quantity entries. The Withdrawal Quantity is greater than the Quantity on Hand.”
*** highlight the affected items in red
** Completed Physical Inventory - “Physical Inventory is completed at 7/1/2022 12:17:03 PM. Production withdrawal cannot be done.”
** Closed Period - use existing message? already implemented for other transactions
* Provide Next button to proceed to Confirm step with the selected Menu Lines ([NXT-33861|https://primeroedgenext.atlassian.net/browse/NXT-33861])

## Acceptance Criteria
* Handle multiple production records (menu lines) on a single production withdrawal
* Allow confirmation for one or multiple menu lines at a time
* Show menu lines as selectable/viewable cards in a side panel
* Allow decimals for base (lowest) units of items
** Show QOH with decimal form for base units where applicable
