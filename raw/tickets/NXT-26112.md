# NXT-26112 — Platform - Permissions - Functions & Category Hierarchy Redesign

## Description
As a primero edge admin, I should be able to access resource catalog to add a new resources available for any district with functions resource type along with category so that the resource type can be accessed by any user with the assigned role for that resource.

+*UX Mockup:*+

[https://www.figma.com/file/PLmXuylGMphY0Bz7WBDCOp/Systems-4.0?node-id=1041%3A14610|https://www.figma.com/file/PLmXuylGMphY0Bz7WBDCOp/Systems-4.0?node-id=1041%3A14610|smart-link] 

+*Requirements:*+

* User shall be able to create functions under specific category and module
* When user clicks on create new Add Resource screen should provide an option to create resource of “Functions” resource type
* When selected “Functions” resource type, system shall dynamically add a drop down as 
* Adding resource type will show up under that specific module to map the permissions
* Once a specific function has been enabled, user shall be able to enable and update access permissions from roles & permissions page

## Acceptance Criteria
# User shall be able to click on “Add New” button with Resource Type drop down with “Functions” as an option
# User shall be able to create functions under specific category and module
# User shall be able to add new resource from drawer pop-up only after filling in required fields listed below
## Resource Name
## Select Resource Type as Functions
## Module
# Duplicate check - User shall NOT be able to create Resource when same resource name is provided with Module & Type combination
# User shall be able to update permissions of the newly created or existing functions under specific modules that has been categorized under. 
# User shall see the resource catalogue with “Module/Category” column instead of just “Modules”
