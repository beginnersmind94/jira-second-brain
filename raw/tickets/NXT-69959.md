## Acceptance Criteria
h2. 1. Data Points & Grouping

The system displays 77 system revenue data points, grouped by category (not by program):

* Student Meal Sales – Free (6)
* Student Meal Sales – Reduced (2)
* Student Meal Sales – Paid (3)
* Adult & Staff Meal Sales – Taxable (18)
* A La Carte – Student / Tax Exempt (7)
* A La Carte – Adult/Staff/Visitor / Taxable (21)
* Federal Reimbursements (7)
* State Reimbursements (7)
* State Programs (2)
* Operating Revenue (3)
* Year-End Close (1)
* Other Revenue (user-created items)

----

h2. 2. Default Table Display

* Each category group is collapsed by default.
* Each category group header displays the group name and total item count.
* Each data point row displays: Name, Status (Active / Inactive), Created By, Entry Type (Automatic / Manual), Program.

----

h2. 3. Search matches against Data Point Name

* Search matches against *Data Point Name* and fields only.
* Matching is case-insensitive, substring-based (e.g., "NSLP" matches any data point whose name contains "NSLP").
* Search does not match against Description, Group name, or ProgramType. Program filtering is handled by the dedicated Program filter.
* *Real-time filtering:* Table results update as the user types. No Apply click is required for search.
* Search results respect any currently applied filters. (e.g., if Program = NSLP is already applied and the user searches "Paid," results show only NSLP data points matching "Paid.")
* Clearing the search field immediately removes the search constraint and returns results to the current applied filter state.

----

h2. -5. Category Group Actions-

-Each category group header exposes three persistent actions:- *-Map All-*-,- *-Activate All-*-, and- *-Deactivate All-*-. All three are always visible regardless of the current state of items within the group. These are override operations — they apply uniformly to all data points in the group, overwriting any existing mappings or statuses.-

*-Map All (Bulk Mapping)-*

* -Opens a confirmation modal:-
** -Message:- -_"Map all data points in {Group Name} to the same GL account? All future transactions for this group will route through the selected account."_-
** -A single GL account selector.-
** -Actions:- *-Apply-* -/- *-Cancel-*
* -On Apply: Maps every data point in the group — including inactive items — to the selected GL account. These are staged as pending changes and do not commit until the user saves.-

*-Activate All-*

* -Confirmation dialog:- -_"Activate all {count} data points in {Group Name}? Activated data points become eligible for posting."_-
* -Actions:- *-Activate-* -/- *-Cancel-*
* -On Activate: Sets all data points in the group to Active. Staged as pending changes.-

*-Deactivate All-*

* -Confirmation dialog:- -_"Deactivate all {count} data points in {Group Name}? Deactivated data points will not generate journal entries during posting."_-
* -Actions:- *-Deactivate-* -/- *-Cancel-*
* -On Deactivate: Sets all data points in the group to Inactive. Staged as pending changes.-

*-Feedback & Change Tracking-*

* -After any bulk action, show an inline toast:- -_"{Action} applied to {count} data points in {Group Name}."_-
* -All bulk changes are reflected in the configuration change history.-

h2. 6. Rules Drawer

*Access:* Clicking "Rules" on a data point row opens a side drawer.

*Content:* Displays the journal entry rule(s) for the selected data point.

*Required rule formats:*

||Data Point Type||Rule Name||Debit||Credit||
|Meal Sales|Meal Sale|Cash in Bank and/or Prepaid Account|Revenue account|
|A La Carte|A La Carte Sale|Cash in Bank and/or Prepaid Account|Revenue account|

*Highlighting:* The selected data point's own account is highlighted in the debit/credit display.

View the file in the description for the final list to be used.
