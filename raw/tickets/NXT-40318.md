# NXT-40318 — eOrders – Dashboard

## Description
As a SchoolCafé user, I would like to create eOrders so that I can order my food electronically.

*Notes:*

* This page exists in classic as [https://online.primeroedge.com/Inventory/EOrderFilter.aspx|https://online.primeroedge.com/Inventory/EOrderFilter.aspx]
* We are porting that page over to 2.0, so all the functionally on that page needs to be added
* This page is related to the eOrder Configuration Dashboard
* It uses that configuration to send the orders

*Flow:*

* Open SchoolCafé and browse to Inventory > Orders > eOrders
* View list of eOrders
* Optionally generate a new one

*Requirements*:

*Menu Changes:*

* Add a page to under Inventory > Orders
** Title = eOrders
** On click, opens the eOrders Dashboard page

 

*eOrders Configuration Page:*

* Create a new page
** Title = eOrders Dashboard
* 
* Filters Section:
** Vendor
*** List of all active vendors for the district
** Site
*** List of sites for the district
** Delivery Dates
*** From
**** Date picker, no time
**** Part of Delivery Date range
*** To
**** Date, no time
**** Part of Delivery Date range
** Order Date
*** Date picker, no time
** Order #
*** Textbox
* 
* Create a grid:
* Data:
** All eOrders regardless of status
** Possible statuses are Sent, Open, Deleted
** Open is when it is scheduled
* Columns:
*  eOrder #
** Unique ID of eOrder
* From Delivery Date
** From Date field from the Generate screen
* To Delivery Date
** To Date field from the Generate screen
* Vendor
** Vendor field from the Generate screen
* Generated On
** Date/Time stamp of the moment the file was generated
* Orders
** Count of orders included in the eOrder
* Status
** Sent
** Scheduled
** Canceled
* Actions
** View File
*** See story for View eOrder File
** View Orders
*** See story for View eOrder Orders
** Delete
*** See story for Delete eOrders
* 
* Grid has paging
* Grid has sorting
* Grid has export to Excel or CSV
* 
* Security
** Add page to resources
** Security is managed via the resource in Roles & Permissions (R&P)
** R&P Edit permission not required since there is no edit
** R&P Delete permission not required since there is no delete
** R&P Add Permissions applied to the Generate button
** R&P View Permissions applied to the page in general
