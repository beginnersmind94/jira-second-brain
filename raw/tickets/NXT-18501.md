# NXT-18501 — Acc. - Claim Creation - Update to use Program Configuration date ranges, program reimbursable value

## Description
As Vera the Director I want the claim creation in the application to use the program dates I’ve configured when collecting meal counts.

*mockup*

n/a

*business info*

We’re trying to fix a particular program with this. We have users running SSO and SFSP during the same month at the beginning/end of summer, and they are having trouble generating separate claims as the dates and meal types overlap.

*requirements*

* Update the claim creation scripts to check for program configuration dates when generating claims.
** Example: Site A is configured for SSO from 7/1 - 7/26, and SFSP from 7/27 → 7/31
** When generating the SNP claim (for SSO), we should only look for summer meals within that date range (7/1 - 7/26). The SFSP claim should only check for meals between 7/27 → 7/31.
** Check for the Reimbursable flag per program instead of the site’s reimbursable flag.

## Acceptance Criteria
Claim creation scripts make use of the program/site configuration start and end dates, and the program’s reimbursable flag.
