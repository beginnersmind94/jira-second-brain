# NXT-6769 — Deposits - Create Deposit Slip

## Description
As the site manager or director I need to be able to create deposit slips for the bank based on the days transactions. Sometimes I deposit multiple days at one time.

*Mockup:*

[https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=5834%3A730&scaling=min-zoom|https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=5834%3A730&scaling=min-zoom]

*Requirements:*

* User can select the Area, Site, and a Date Range for the deposit.
* The user can enter a Bank Deposit slip number.
* If there are sessions within the date range that are not Reconciled or Deposited a message is displayed ‘Not all sessions in the date range are reconciled. Please reconcile the sessions or change the deposit date range.
* The Date of Deposit is the ending date of the date range
* Checks in for the deposit slip are auto selected from the sessions in the date range. [https://primeroedgenext.atlassian.net/browse/NXT-5373|https://primeroedgenext.atlassian.net/browse/NXT-5373|smart-link] 
* Checks can be added [https://primeroedgenext.atlassian.net/browse/NXT-5373|https://primeroedgenext.atlassian.net/browse/NXT-5373|smart-link] 
* Denominations are entered by the manager [https://primeroedgenext.atlassian.net/browse/NXT-5372|https://primeroedgenext.atlassian.net/browse/NXT-5372|smart-link] 
* The Summary displays the Cash Total, Check Total and calculates the Actual Deposit Total by adding them together. [https://primeroedgenext.atlassian.net/browse/NXT-85|https://primeroedgenext.atlassian.net/browse/NXT-85|smart-link] 
* The Summary also includes the Expected Deposit.
* The Expected Deposit is calculated by adding all the Cash and Checks from the sessions included in the deposit. Cash is calculated by subtracting the opening balance cash from the closing balance cash for each session included in the deposit. 
* IF there is a difference between the Expected Deposit and Actual Deposit Total a comment must be entered to save the deposit slip, A message is displayed saying a comment must be entered.
* An EDGE Reference number is created when saved (Its an increment of the previous reference number within the district) FORMAT SSSSMMDDYYNN-V
** SSSS = Site ID
** MMDDYY = Deposit Date
** NN = Deposit Number (one up number for by site for the deposit)
** V = Version of this deposit slip 0 for initial deposit slip. Plus one for each modification..
* The status of reconciled sessions for the date range and site are changed to DEPOSITED when the deposit slip is saved.
* A list of Session IDs included in the deposit slip is listed and saved.

Flow:

# User Selects Add Deposit from Deposit Lists Page
# Create Deposit slip page is displayed
# User enters Site Selection, Date range and clicks Create 
# System gathers data from reconciled sessions and calculates Expected Deposit and populates checks.
# User enters cash to deposit, verifies checks, enters comment if necessary and clicks save.
# Success Message displayed “Congratulations, deposit slip xxxxxxx successfully created”
# System returns to Deposit Lists page

## Acceptance Criteria
User enters a Single Site and a Date Range (required)
User can enter a Deposit Slip number (optional)
A message is displayed 'Some sessions within the date range are not reconciled. Please reconcile all the sessions or change the date range.' when all sessions in the date range for the site are not reconciled.
Only Reconciled sessions are included in the calculation for Expected Deposit and Checks 
The status for all sessions included is changed to DEPOSITED when the deposit slip is saved.
A comment is required for all deposit slips where the Expected Deposit and Deposit Total are not the same.
The Deposit Total is Total Amount of All Checks in all Sessions PLUS the (Closing Balance - Opening Balance) of all sessions included in the deposit slip.
When saved the Session IDs are saved with the Deposit Slip
When a new deposit slip is saved a Reference ID is generated. the initial Reference Id is 'SSSSMMDDYYNN-V' where SSSSMMDDYYNN is a one up number of the previous reference number used in the district. Leading zeros are shown, and V is the Version number V=0.
SSSS = Site Code
MMDDYY = Month/Day/Year
NN starts at 1
V starts at 0

Success message "Congratulations. Deposit Slip xxxxxxx successfully created"

New Deposit Slip
SAVE button is enabled when the Deposit Total = Expected Deposit.
SAVE button is enabled when the Deposit Total <> Expected Deposit and a Comment is entered
When the SAVE is selected the status of all Sessions used to create the deposit slip are changed to RECONCILED.
CANCEL can be selected at any time and all changes or modifications are ignored or reversed.

Modified Deposit Slip
When an existing deposit slip is modified the Reference ID is kept and the Version # is incremented by 1.
When a Deposit Slip is modified a Comment must be entered to save.
The SAVE button is not enabled unless a change has been made and a comment entered.
CANCEL can be selected at any time and all changes or modifications are ignored.
