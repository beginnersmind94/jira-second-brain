# NXT-30212 — Acc. - Sessions - Option to move session to another site

## Description
As Agatha the Site Manager I want to move a session I made to another site so I don’t have to re-enter the information.

*mockup*

[https://www.figma.com/proto/Yo99YhQm5Xk91Ie7b0PRkD/Front-Office-6.0?node-id=484%3A13087&scaling=scale-down-width&page-id=481%3A12603&starting-point-node-id=484%3A13087|https://www.figma.com/proto/Yo99YhQm5Xk91Ie7b0PRkD/Front-Office-6.0?node-id=484%3A13087&scaling=scale-down-width&page-id=481%3A12603&starting-point-node-id=484%3A13087|smart-link] 

*requirements*

* Add an option to Reconciliation:
** Move Session
* User can pick the following:
** Another site
*** Default can be the current site, with the current terminal
*** User needs to pick a terminal to move the session to as part of this
*** If the user changes the selected site, re-populate the terminal list with active terminals for that site. Default can be a site terminal
* User should confirm and enter a comment for this action.
* History tracking with the comment will need to be added for this. - This will be handled in [https://primeroedgenext.atlassian.net/browse/NXT-21237|https://primeroedgenext.atlassian.net/browse/NXT-21237|smart-link] in next sprint
* Update the Reconciliation layout to match UX standardization.

## Acceptance Criteria
Move Session option added

Inside of the option:

* User can change the site, default is current site.
* User can change the terminal, default is current terminal.
* If user changes the site, terminal list should refresh.
** Default terminal should be the site terminal.

User has a confirmation prompt and comment when actioning this.

History tracked for who actioned the move, when, and what was changed. Including before and after for changes. - this to be handled in [https://primeroedgenext.atlassian.net/browse/NXT-21237|https://primeroedgenext.atlassian.net/browse/NXT-21237|smart-link] in next sprint

Sites move successfully.

Reports update appropriately for all sites involved.
