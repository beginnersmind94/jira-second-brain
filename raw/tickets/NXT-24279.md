# NXT-24279 — IM - Recipe Category Configuration

## Description
As a user I need to be able to define my recipe categories in order for me to organize my recipes

!image-20211230-125739.png|width=876,height=793!

## Acceptance Criteria
Done when:

*Entry Point*

* A Item  option is added under IM configuration area
* Option Recipe Category 

*Result Grid*

* Should display fields as mockup above
* User should be able to search and sort fields on result grid
* User should be able to edit  Recipe Category Description only
** Only for user created Recipe Categories (e.g Local). -CN cannot be edited or deactivate
* User should be able to deactivate Recipe Category ONLY if Recipe Category is not used within recipe
* User should be presented with a confirmation message
“Are you sure you want to deactivate this Recipe Category” is Yes proceed
with deletion if No cancel process and return to Recipe Category configuration page

*Actions*

* *ADD* button \[When clicked a Slide Out Card is displayed]
* Fields on Slide Out Cards
** Data Source \[Dropdown box that list available data sources to the user]
*** Is a Mandatory field
*** Only one Data Source can be selected
** Recipe Category Description \[Free text box]
*** Is a Mandatory field
*** Only can be added to user/district created data source (e.g. Local)
* *SAVE*
** Validation occurs
*** Recipe Category Description is Mandatory and no duplicates are allowed
*** If the Recipe Category Description already exist warn the user with a message “Recipe Category Description  already in use, please use a different description ”
*** One Data Source is Selected (Mandatory) - Default to Local
*** The newly added Recipe Category Description is listed on the result grid
** *X icon* \[When Clicked the Slide Out Card is close and transactions are canceled]
** Pagination should be present

* *Export* \[PDF Format]
** Report Title \[Recipe Category Configuration]
** Should cover same fields as the result grid \[Recipe Category Description, Data Source Name]
** Header
*** Updated by \[User Name]
*** Generated on \{System date/time]
** Footer
*** Power by SchoolCafe for \{School District Name]
*** Pagination should be present

*General:*
Once user selects configuration a pop-up window is presented to the user. This pop-up windows asks for data source and configuration option desired. When user click/press apply button then system re-directs the user to appropriate configuration area
