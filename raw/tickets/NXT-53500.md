# NXT-53500 — OpenID - Login - Screen & Flow Changes

## Description
As a SchoolCafe user, I want to be able to login on one screen instead of 2 so that I can save a button click

*Notes*:

* After releasing the parent feature, some important users requested that we update the screen flow to go to a one step process instead of 2 step process
* This story covers the work necessary to accomplish that



*Requirements*:

* Do not delete the code you have written. We may still use it as an alternate login method.
* 
* Remove Login screen 1 from the flow
* Have the user go straight to screen 2
* Leave screen 2 as it is, except the following:
** Put a validator on the submit button on screen 2 that does the same checks from screen 1
** If they are logging in using SchoolCafe Auth and district has SSO preferred:
*** Show error message: District requires SSO
*** Show a button:
**** Login Using SSO
**** On click, do the same thing you do as the SSO Login from the More Options screen
** Remove the More Options button
** Add the Login With SSO button from the More options screen with all its functionality
** On Login With SSO button, require them to put in their username so that more options screen filters the options to the district (might already be done)

## Acceptance Criteria
* User is able to do SchoolCafe authentication username/password on the first screen they see
* -They can still do the More Options screen-
* If they try to do SC Auth and district requires SSO, don’t allow the user to login, but force them to go through SSO process
