# NXT-24774 — MP > Menu Planner > Menu Item Categories - how it should work

## Description
Only populate the "choose" options of a menu item if that category is added to the menu planner.
When adding menu items to the menu planner menus, the page is performing analysis on the fly, and currently assuming Choose 1 for ALL of the menu item categories that exist in the software (2 on Vegetable). This means that when I go in to edit a menu, or assign a menu to production, ALL of the Menu Item Categories are displaying, even if they have no menu items.
I only want the Menu Item Categories applied to a menu when a menu item from that category is placed on a menu. 
If a Menu Item Category does not have any menu items on the menu, then it should remain blank and unused.

## Acceptance Criteria
When using the Menu Planner to add menu items to menus, the menus should only populate the menu item categories that are associated with the Menu Items that have been added to the menus. 

When is item is removed, remove the category and choose count that was originally populated.
