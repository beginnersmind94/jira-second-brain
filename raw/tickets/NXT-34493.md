# NXT-34493 — PR- Production Plan- Autofill- Previous Records (Calendar Option)

## Description
As a production manager, there are plenty of times that I repeat the same menus over and over throughout the course of the school year. I would like to quickly fill in my production plan counts based on historical data. I could fill in my counts based on previous plans that are the same menu as the one currently on the production plan, copy of the menu, or other menus.

## Acceptance Criteria
* Provide a button on the plan page that will allow user to open the autofill slideout
* Provide a drop down that allows user to select autofill based on *Previous Records*
* Provide a radio option under the drop down once selected, for the user to pick whether they want to fill based on *Produced* or *Served* Counts
** Default to Produced Count
** Served count ( Produced- (All leftover))= Served
* Calendar option will display a calendar that would give color indicators as to what days have the same menu/plan, copy of the menu/plan, or a completely different menu/ plan (Other Menu/Plan)
** For all types of menus, divide the produced/served count by the number of serving groups to match the total autofilled
** Same Menu/Plan- Copy all counts from the date selected and populate the counts accordingly on the plan page. This includes the meal counts. 
*** In the instance of VMI, pull child item counts and toggle on same items from previous record
** Copy of Menu/Plan- Copy all counts from the selected date and populate the counts accordingly on the plan page. This includes the meal counts
*** In the instance of VMI, pull child item counts and toggle on same items from previous record
*** Copy planned counts only on matching serving sizes for a particular item, leave as blank if serving sizes from autofill date and current date do not match
** Other Menu/Plan- Copy all counts from the selected date for the menu items that *match* with the current production plan that is being autofilled. The meal counts will also be populated based on *matching* serving groups
*** In the instance of VMI, pull child item counts and toggle on same items from previous record
*** Copy counts only on matching serving sizes for a particular item, leave as blank if serving sizes from autofill date and current date do not match
* Consider all plan statuses include plan incomplete *EXCEPT* if plan incomplete does not contain *ANY* counts for menu items or meal counts
* Indicate on the calendar what date is selected as well on the page so that the user knows what date has been selected 
* Allow user to go as far back in the calendar as they would like to pull previous plan counts based on the types of menus/plans
* Display an option for user to override currently existing planned counts with whatever counts they use on the autofill feature
** Default to be selected
** If not selected, only insert counts for menu items that have not had a count entered for

* Only pull menus based on same Site, Meal Type, Menu Line combination
