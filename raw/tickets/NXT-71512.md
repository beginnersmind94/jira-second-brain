# NXT-71512 — SC - PBIRD - Download

## Description
As a Director, I want to download a Power BI report from the dashboard so that I can receive the generated report output for review and use.

----

h2. *Description*

Allow users to initiate a download of a Power BI report from the Power BI Reports dashboard.

When the user selects *Download* from the row ellipsis menu, the system will display a confirmation message. If confirmed, the system will run the report asynchronously and deliver the completed report to the user through their *Inbox*. The completed report will also be available through the *Workspace Dashboard* as a distributed report.

This behavior should follow the same standard process used by *Managed View Reports*.

----

h2. *User Flow*

# User navigates to:
#* *Reports → Power BI Reports*
# User clicks the *ellipsis (⋮)* on a report row
# User selects *Download*
# System displays confirmation dialog
# User selects:
#* *Yes* → Report execution begins
#* *No* → Dialog closes and no action is taken
# System notifies user that the report has been submitted for processing
# Report runs asynchronously
# When processing completes:
#* User receives notification that the report is complete
#* Report is delivered to the user’s *Inbox*
#* Report is available from the *Workspace Dashboard* distributed reports area

----

h2. *Out of Scope*

* Email Now
* Recently Run Reports UI
* Inbox implementation
* Workspace Dashboard implementation
* Mobile app behavior

----

h2. *Requirements*

h3. *Download Access*

* *Download* is available as an option in the row ellipsis menu _(covered in separate story)_

----

h3. *Confirmation Dialog*

* When the user selects *Download*, display confirmation dialog with the following text:

{quote}*This will run the report now and save the data in a file. You will be notified in your Inbox once complete. Do you wish to continue?*{quote}

* Dialog must include:
** *Yes*
** *No*

----

h3. *User Decision*

* If user selects *No*:
** Close dialog
** No further action occurs
* If user selects *Yes*:
** Begin asynchronous report execution process

----

h3. *Asynchronous Processing*

* Report execution must run asynchronously
* Do not require the user to wait on the page for report completion
* This is required because some report executions may take significant time depending on data volume

----

h3. *User Notification*

* After user confirms the action:
** Provide immediate feedback that the report execution has been initiated
* When report processing completes:
** Display standard completion notification to the user
** Notify the user that the report is available in their *Inbox*

----

h3. *Delivery*

* Completed report must be delivered to:
** User’s *Inbox*
** *Workspace Dashboard* distributed reports area
** Report is viewable in the Recently run reports

----

h3. *Managed View Reports Reuse*

* Download distribution behavior must work the same as *Managed View Reports*, including:
** Confirmation flow
** Asynchronous execution
** Inbox delivery
** Completion notification
* Any Managed View Reports business logic should be reused where applicable

----

h3. *Permissions*

* Users must have permission to access the Power BI report in order to initiate Download
* Follow existing Managed View Reports security behavior as applicable

## Acceptance Criteria
* Given a user selects Download, when the action is triggered, then a confirmation dialog is displayed with the expected text
* Given the confirmation dialog is displayed, when the user selects No, then the dialog closes and no report is executed
* Given the confirmation dialog is displayed, when the user selects Yes, then the report begins asynchronous execution
* Given the report execution has started, when the request is submitted, then the user receives feedback that the process has been initiated
* Given the report completes successfully, when processing finishes, then the user receives a completion notification and the report is available in their Inbox
* Given the report completes successfully, when the user opens the Workspace Dashboard, then the distributed report is available there as well
