# NXT-51986 — PE - GDSN Connect - Mark as Reviewed for Bulk Updates / Counts Overhaul

## Description
*Navigation*:

MP -GC - Pending Updates

*Use Case:*

Scope: All the attributes in Ingredients Tab, Stock items Tab

Counts are not factored in Images Tab (Out of Scope), but users can deselect and proceed with updates without taking them

 

Item Updates – Count Scenarios:

 

*File Upload or API Update:*

The system populates the initial count based on number of attributes that are different from local ingredients or stock Items:

# *Count of Ingredients* – Shows the count of Ingredient attribute where the difference value is 1 or more

a.      If the count is less than 1 – the system should NOT list that attribute

# *Count of Stock Item* - Shows the count of Stock Items attribute where the difference value is 1 or more

a.      If the count is less than 1 – the system should NOT list that attribute

# *Count of Images* - Shows the count of Images where the difference value is 1 or more

a.      If the count is less than 1 – the system should NOT list that attribute

 

*Updating a Single Item:*

 User can select the desired attribute and click on “Update Now” button

The counts in the main grid and the dialog should match – applicable to ingredient and stock item (image count is not listed in grid)

 The system should show the remaining attribute with status as New

 This iteration can continue until all the attributes are updated

 

*Updating Bulk Items:*

 *Scenario 1:*

*Users are accepting changes with all the attributes count without reviewing them (without Marking as Reviewed)*

The user selects multiple records at a time and click “Review Updates”

The counts in the main grid and the dialog should match – applicable to ingredient and stock item (image count is not listed in grid)

The user clicks on “Update Items” button

The attributes are accepted, the items are no longer shown in grid until new updates comes in again

 

*Scenario 2:*

*Users are accepting partial changes with selected attributes: (Mark as Reviewed)*

User can select the desired attributes and click on “Mark as Reviewed”

Item status is updated “Partially Reviewed”

The count of attributes shown in main grid shows the actual remaining attributes (Ex: 19) and while the review dialog should show the attributes selected (Ex: 2)

Upon accepting the items – status is reinstated to new

The remaining attribute count is refreshed on the grid

The user can continue this until all the attributes are accepted

 

 

*QA Regression:*

The user should be able to open an item and opt out few fields and mark as reviewed

The user should be able to open another item and opt out some other fields or even in different pagination. The user should be able to override mark as reviewed status with more fields unselected if they wish to do it. 

When the user clicks on reviewed items , the UI should show only the selected attributes count

For items that are marked as reviewed, the user should be able to review the updates by clicking review button (as-is)

Upon system updating the records, the system should only push the update for the selected attributes. Once it is completed, the status should be reinstated to New 

Regression: Serving Size Vs Nutrition Selections, Serving Size Vs Meal Credits section

Logout Scenario: Even if the user logs out and log in, the system should maintain the attributes selected, until they update. One its updated, the status resumes to New status

Out of Scope: Count of Attributes (Pending) need not be changed in UI when user opts out

Approach - user will see the deselected values even after log out, Changes required - DB, API Level
