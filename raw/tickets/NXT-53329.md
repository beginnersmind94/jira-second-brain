# NXT-53329 — VND - Settings - Serve Meal By

## Description
As Dana the Director I want to allow students to authenticate themselves at the vending machine with different methods to fit our district’s needs.

*Requirements*

* Add a new System Setting:
** Setting Name: Serve Meal By
** Setting Code: SERVEMLBY
** Module: Accountability
** Category: Vending
** Setting Description: Controls whether meals at vending machines are served based on Meal Timings or by the Meal Type. This impacts pricing at the machines.
** Values: Meal Timing, Meal Type
* Setting SERVEMLBY needs to be used as part of our response to the /customer endpoint.
** Meal Timing → “UserCredentials” checks against the Student ID #. “PINNumber” sent back is the PIN.
** Meal Type → “UserCredentials” checks against the Student ID #. “PINNumber” sent back is the DOB.

## Acceptance Criteria
* System Setting AUTHMETH checked as part of the /customer endpoint.

* Student ID/PIN → “UserCredentials” checks against the Student ID #. “PINNumber” sent back is the PIN.
* Student ID/Birthday → “UserCredentials” checks against the Student ID #. “PINNumber” sent back is the DOB.
* PIN/Birthday → “UserCredentials” checks against the account PIN #. “PINNumber” sent back is the DOB.
