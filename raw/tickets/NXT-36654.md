# NXT-36654 — Monitoring Scheduler – P7 History Tab

## Description
As a SchoolCafé admin, I want to be able to view the history of monitoring jobs so that I can proactively monitor the application and ensure higher customer satisfaction.

*Flow:*

* User logs in and browses to Support > Monitoring > Job Scheduler
* User clicks the History tab
* User views a list of monitoring jobs that has completed
* Optionally, the user can view the log

*Notes:*

* This is a continuation of the story to create the Job Scheduler page
* This story assumes that story has been created and that the History tab was created
* This story is based off the Export Monitor page and is following that page’s flow, layout and functionality
* We are only changing the appropriate fields as necessary, so when in doubt, copy what that page does
* This page is the user front end to viewing executed jobs
** These jobs are created in other stories and not covered here
** Those other stories will include requirements to show their output on this page
* 
* On this tab, note that the design is based upon the Export Monitor, and it has
** A filter section at the top to filter the grid below
** A grid section that shows completed jobs
** A View log button for each row

*Requirements*:

* On the History tab of the Job Scheduler page, do the following using the Export Monitor page as the example UI/UX:
* 
* Sections:
* *Filters Section*
* Create a filters section at the top of the screen like Export Monitor page
* Filter fields:
** Module
** Monitor
** Job Type
** Date Range
** Copy like Export Monitor
** Status
** Values for the filters are the same as the grid columns
* Apply button
** Applies the filters same as for the Export Monitor page
** Use icon like Export Monitor
* Collapse / expand arrows
** Expands and collapses the filters section same as for the Export Monitor page
** This is not currently on the Export Monitor page, but is on the first tab and is a good feature, so please add
* 
* *Grid Section*
* Show a grid of scheduled monitoring jobs
* Columns:
* Module
** Value comes from the Module field in the + New page when creating the job
** Where Export Monitor has Import Name, use Module
* Monitor
** Value comes from the Monitor field in the + New page when creating the job
** This is the name of the monitor
** Where Export Monitor has Import Type, put Monitor
* -Exceptions-
** -This is new from Export Monitor-
** -Show a count of exceptions found in the monitor-
* Start Time
** Same as Export Monitor
* End Time
** Same as Export Monitor
* Duration
** Same as Export Monitor
** Don’t add the (HH:mm:ss) to the column header
* -Triggered by-
** Don’t show this field since we added Exceptions column
** This table is already busy, full
** If they want to see who ran it, they can click the view log
* Status
** Same as Export Monitor
* Action
** View Log
** Same as Export Monitor
** The requirements for this button will be covered in a separate story, just add the button and have it do nothing for this story
* 
* Permissions: Covered in other story
