# NXT-21716 — New Image Handling Report

## Description
As an administrator, I want to be able to see info regarding treatment of new images so that I can determine level of effort, status of training and build reports on quality of service.

*Application*: Face and Food Detection Website

*Flow:*

* Browse to PrimeroEdge and click the link for Food and Face Detection
* Browse to the Reports
* Browse to New Image Handling Report
* View data

 

*Assumptions:*

* It is assumed that when an image is deleted, that only the file is deleted, that the data in the database is kept
* It is assumed you can identify when an image has been trained

 

*Requirements:*

* Create a new reports page with the title New Image Handling
* In the menu navigation, under Reports:
** Create a new menu link under Reports named New Image Handling for the new page
*  
* All fields are read only, there are no action buttons except grid behaviors and hyperlinks
*  
* This is a summary grid
** Data is summed up per day
** Don’t show a grid per image
*  
* Grid of uploaded image data
** Usual sort, paging, filtering, export to excel functionality
*** default sort time descending (most recent first)
** Columns:
*** Day – date, no time
*** Total Uploaded – count of total images uploaded that day
*** Total Trained – count of images from that day that received training (saved and moved to Trained Images)
*** Total Deleted – count of images from that day that were deleted
*** Total Pending – count of images from that day left in the new images category
*  
* Filters for the grid
** Date / Time Range – like filter on other pages
*  
* Security and roles:

o    Copy from Accuracy Report, do exactly as there

* Unauthenticated users cannot view screen

Should look the same as Accuracy Report
