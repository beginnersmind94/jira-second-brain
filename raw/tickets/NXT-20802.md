# NXT-20802 — INV - Quick Receipt - Display the read-only mode with labels (not disabled fields)

## Description
Per [https://primeroedgenext.atlassian.net/browse/NXT-18083|https://primeroedgenext.atlassian.net/browse/NXT-18083|smart-link], there was a point that was not correctly implemented:

*Requirements*

* Display a label-only version of the Quick Receipt page
** this would require a dedicated Edit button
** the page would be label-only when loaded in Incomplete status
** Edit button would enable the edit state of the Quick Receipt
** Save as Incomplete (or Confirm) button would convert back to label only

Currently when the user saves a Quick Receipt, the page remains the same but all of the input fields are in disabled mode. 
We should instead display these as labels like the view-only Receipt mode.

!18083 1.png|width=100%!

## Acceptance Criteria
* On click of Confirm button:
** Success message will be displayed
** Buttons will be removed
** Editable fields will turn to labels
** ‘Return to Receipt List’ button will be provided
