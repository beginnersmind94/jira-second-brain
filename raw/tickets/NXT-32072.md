# NXT-32072 — INV - Vendors - Add Items to Contract - Update for Pack Size Config

## Description
As a District Admin I want the Contract Item configuration to reflect the pack size units of my items so I can configure the correct unit for ordering.

*Requirements*

* Update the Adds Item to Contract screen to reflect the various units of each item
* Display each unit on its own row, with a checkbox available for each unit
** Show item units on individual rows but indicate items which are linked together
*** It does not need to be an actual TreeList grid, but if we can simply indicate when items are child units this will be sufficient
** Preserve item relationships for paging
*** “child units” should not be separated by page from their parent units
* Display Item # only once for the highest unit
* Replace Unit Type with Unit Description
* Show conversion (sub) quantity 
* +Business Rules+
* Items are still managed at the Item # level from user perspective
** Do not remove items from the add contract screen if only one unit has been added to the contract
*** Instead, remove the checkbox for the units which are already added
*** Show indication the item is already on the contract
* +UI Ideas+
** Provide filter options 
*** ‘Only Highest Units’ 
**** this would hide all units which have a parent unit
**** default value is enabled
***** users will toggle off if they want to see all pack sizes, since most of the time it will be the highest unit
** Remove some header search options
*** Weight, FMV, Unit Description
*** Optional: Item Category, Storage Category (since there are dedicated dropdowns)
** Move all unit-specific attributes to the right-most columns
*** but ensure they are always visible, by reducing width first from Category
