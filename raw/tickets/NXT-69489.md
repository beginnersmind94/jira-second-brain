# NXT-69489 — WS - Insights - KPI - Inventory Value - Dashboard Screen

## Description
As a district nutrition manager, I want to see an Inventory Value card on the Insights dashboard, benchmarked against the Inventory Total Goal for each site, so that, I can quickly evaluate whether schools are staying within acceptable inventory levels.

*Notes:*

* This story is for the Dashboard screen
* There is no benchmark screen update for this KPI
* Another story will cover the Inventory Total Details drawer

*Requirements:*

*Card Updates:*

* Ensure the Total Inventory value is added to the API and dataset for the dashboard screen
* 
* Add Inventory Total as a card to the screen
** Column name: Inventory Total ($)
** Location: Add to the end by default
** Type and formatting: Exactly the same as Revenue and Waste
** Card shows:
*** Total Inventory value (in $) for the selected Time and Site dropdown options
*** There is no benchmark or arrow
*** -Inventory Total Goal set in the benchmark configuration page-
** Card displays in:
*** -Green → if Total Inventory Value ≤ Inventory Total Goal.-
*** -Red → if Total Inventory Value > Inventory Total Goal.-
*** -Grey → if no benchmark is configured.-
*** Just show it in Grey
** Update the value for actual based upon time frame filter change
** Update the value for actual based upon location filter change
** On click: Open the Inventory Total Drawer

*School Performance Updates:*

* Add the Total Inventory column to the grid
** Column name: Inventory
** Location: Add to the end, after waste
** Type and formatting: Exactly the same as Revenue and Waste
* Update the value based upon time frame filter change
* Note: there will be too many grid columns, so ensure there is horizontal scroll
* On click: Open the Inventory Total Drawer

*Performance Trends Updates:*

* Add the Inventory Total value to the drop down list
** Update the order of the fields to match the order of the columns in the benchmark config screen
** Type and formatting: Exactly the same as Revenue and Waste
* Update the value based upon time frame filter change
* Update the value based upon location filter change
* Popup text: 
** Total inventory value:
Food Inventory:
Non-Food inventory:
Commodity:
Non Commodity:

*Schoolie Updates:*

* Ensure Inventory Total is accounted for when doing an AI Analysis on the dashboard screen
* Ensure Inventory Total is accounted for when doing an AI Analysis on the homepage daily recap screen

*Configure KPIs Updates:*

* Ensure Inventory Total is added to the configure KPIs drawer
* Ensure users can toggle on/off
* Ensure users can sort it
* Ensure toggle and sorting apply to the dashboard for the user

## Acceptance Criteria
Below is a *complete, implementation-ready Acceptance Criteria* for the *Inventory Total card on the Insights Dashboard*, aligned to your notes and scoped strictly to the *Dashboard screen*.

----

h2. 1. Data & API Integration

* The *Total Inventory value* must be added to:
** Dashboard API responses
** Dashboard datasets used for rendering cards, grids, trends, and AI analysis
* The value must be retrievable by:
** Selected *time frame*
** Selected *site(s)*
* Inventory Total data must be consistent with values used in:
** Benchmark configuration
** School Performance grid
** Performance Trends
** AI analysis inputs

----

h2. 2. Inventory Total Card

h3. Card Creation & Layout

* Add a new card named *Inventory Total ($)* to the Insights dashboard.
* Default position: *last card on the dashboard*.
* Formatting:
** Currency format identical to *Revenue* and *Waste*.
** Font size, spacing, and alignment follow existing card standards.

h3. Card Values

* The card displays:
** *Actual Inventory Total ($)* based on:
*** Selected *Time* filter
*** Selected *Site(s)* filter

h3. Card Color Logic

* Grey 

h3. Filter Interaction

* When the *Time filter* changes:
** Actual values refresh.
* When the *Site filter* changes:
** Actual values refresh.
* Updates occur without requiring a page reload.

h3. Interaction

* Clicking the card opens the *Inventory Total Details drawer*.

----

h2. 3. School Performance Grid Updates

* Add a new column:
** *Column name:* Inventory
** *Location:* After Waste
** *Formatting:* Currency, same as Revenue and Waste
* The column updates when:
** Time filter changes
* Horizontal scrolling is enabled to accommodate additional columns.
* Clicking a value in the Inventory column opens the *Inventory Total Details drawer*.

----

h2. 4. Performance Trends Updates

* Add *Inventory Total* to the KPI dropdown list.
* Reorder KPI dropdown values to match the order defined in the *benchmark configuration screen*.
* Formatting matches Revenue and Waste.
* Values update when:
** Time filter changes
** Location filter changes
* Popup/help text behavior:
** Total inventory value:
Food Inventory:
Non-Food inventory:
Commodity:
Non Commodity:

----

h2. 5. Schoolie (AI) Integration

* Inventory Total is included as an input in:
** Dashboard AI analysis
** Homepage daily recap AI analysis
* AI summaries consider:
** Actual Inventory Total
** Variance and trend over time (when applicable)

----

h2. 6. Configure KPIs Drawer

* Inventory Total is added as a configurable KPI option.
* Users can:
** Toggle Inventory Total *on/off*
** Change its *display order*
* User preferences:
** Persist across sessions
** Apply immediately to the dashboard
* Toggling Inventory Total off removes:
** The card
** Grid column
** Trend option

----

h2. 7. Validation & Edge Cases

* Inventory Total must:
** Display as *$0* if data exists but value is zero.
** Display *grey / no goal state* if benchmark is not configured.
* If API data is unavailable:
** Show standard dashboard error state.
* Inventory Total values must never display as null or undefined.

----

h2. 8. Performance & UX

* Dashboard load time remains within existing performance thresholds.
* Card, grid, and trend updates occur smoothly without noticeable lag.
* Inventory Total visuals are consistent with existing KPI cards.
