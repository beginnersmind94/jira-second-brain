# NXT-8875 — Application Details - History (Changelog)

## Description
*User Story* 

As an application processor, I would like see a history of changes made to the application at the time of entering and thereafter if modified, so that I can properly survive an audit of my applications when required.

*Mock up* 

[https://www.figma.com/proto/EVGaMgaqmoII4zO9Qff1Cq/Eligibility-4.0?page-id=3998%3A48568&node-id=3998%3A49797&viewport=304%2C48%2C0.25&scaling=scale-down-width&starting-point-node-id=3998%3A49797|https://www.figma.com/proto/EVGaMgaqmoII4zO9Qff1Cq/Eligibility-4.0?page-id=3998%3A48568&node-id=3998%3A49797&viewport=304%2C48%2C0.25&scaling=scale-down-width&starting-point-node-id=3998%3A49797|smart-link]

*Requirements* 

* In Eligibility → Forms → Applications, in the Details column, next to each application details icon, display a *history icon*.
** This change needs to be made in both the ‘All’ and ‘Pending Applications’ tabs
* When this history icon (button) is selected, redirect the user to that individual application’s history page.
* Application History shows changes made to an application after it was received.
** Every change that a user can make to an application should be tracked. i.e. 
*** Adding/removing students or household members, 
*** changing answers to categorical questions, 
*** changing household size/income, 
*** changing assistance programs and case numbers
*** changing optional benefits responses
*** leaving new comments
*** adding/removing documents
*** And anything else that can be done to an application, including marking it as pending, processed, deleted, or refused benefits. 
* Include details on the History page about ‘from/to’ values that changed, with the following fields
** Updated On
** Updated By 
** Previous Values
*** \{section/step} - \{currentValue}. i.e. Assistance - 
*** Blank is acceptable if no data exists
** New Values 
*** \{section/step} - \{newValue}. i.e. Assistance - SNAP - 12345678
** See Account Management → Account Info → History for example
* For any questionable syntax, please work with the PO.
* History Section should also have pagination

## Acceptance Criteria
Verify that the Applications page’s ‘All’ and ‘Pending Applications’ tabs Details column includes a clickable History icon next to each application, which redirects the user to the History page for that app

Verify that all changes made to an application's information are viewable on this page, with the columns specified above.

Verify that the page includes pagination
