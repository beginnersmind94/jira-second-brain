# NXT-71515 — SC - PBIRD - Workspace - PowerBI Integration

## Description
As a Director, I want Power BI reports to appear in my Workspace so that I can access my messages, starred reports, and distributed reports in one central location.

----

h2. *Description*

Ensure that Power BI report functionality is fully integrated into the *Workspace* screen.

This includes:

* Inbox messages for delivered reports
* Starred Power BI reports
* Distributed Power BI reports

All behavior must align with existing functionality for *Managed View Reports* and *Custom Reports*.

----

h2. *User Flow*

# User logs into SchoolCafé
# Navigates to:
#* *Workspace*
# User interacts with:
#* Messages (Inbox)
#* My Custom Reports → Starred
#* My Custom Reports → Distributed Reports
# User can:
#* View report messages
#* Reply or delete messages
#* Access starred reports
#* Access distributed reports

----

h2. *Out of Scope*

* Mobile app implementation _(separate story)_
* Creation of new messaging or dashboard components
* Report execution logic

----

h2. *Requirements*

h3. *Messages – Inbox Integration*

* Any Power BI report delivered to a user’s *Inbox* must:
** Appear in the *Messages → Inbox* section on Workspace
* Messages must behave the same as existing messages:
** User can *open* message
** User can *reply*
** User can *delete*

----

h3. *View All Behavior*

* When user clicks *View All* in Messages:
** Navigate to full Inbox view
* Power BI report messages must:
** Appear in *View All*
** Appear in the *Inbox submenu (left navigation)*
* Behavior must match existing Inbox functionality

----

h3. *Starred Reports Integration*

* Any Power BI report marked as *favorite (starred)* must:
** Appear in:
*** *Workspace → My Custom Reports → Starred Reports*
* Behavior must match:
** Managed View Reports
** Custom Reports

----

h3. *Distributed Reports Integration*

* Any Power BI report distributed to the user must:
** Appear in:
*** *Workspace → My Custom Reports → Distributed Reports*
* Applies to all distribution types:
** Scheduled
** Ad hoc (Download, Email, etc.)

----

h3. *Distributed Reports Behavior*

* Distributed Power BI reports must support the same functionality as existing reports:
** View report
** Download report
* Behavior must match:
** Managed View Reports
** Custom Reports

----

h3. *Consistency & Reuse*

* All Workspace behavior must:
** Reuse existing components
** Follow existing patterns
** Maintain consistency with Managed View Reports and Custom Reports

## Acceptance Criteria
* Given a Power BI report is delivered to a user’s Inbox, when the user views Workspace, then the message appears in the Messages → Inbox section
* Given a message is visible in Workspace, when the user clicks View All, then the message appears in the full Inbox view
* Given a message exists, when the user navigates to the Inbox submenu, then the message is visible there
* Given a Power BI report is marked as favorite, when the user views Workspace, then the report appears under My Custom Reports → Starred Reports
* Given a Power BI report is distributed to a user, when the user views Workspace, then the report appears under My Custom Reports → Distributed Reports
* Given a distributed report is visible, when the user interacts with it, then they can view or download the report
