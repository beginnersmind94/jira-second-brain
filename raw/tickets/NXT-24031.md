# NXT-24031 — Production - UI Technical Changes

## Description
Following Technical Tasks needed to be addressed as part of this Story.

* Update the UI-Components to the latest version.
* Remove urls from .env files and resue base url as needed for different API calls. (Example attached as image below)
* Remove unused packages from client/package.json
* Remove unused files and logic.
* Run ESLint report and fix all the lint errors
* Follow proper naming conventions for all files and variables.
* If there are Components that are carrying multiple functionality, then break into smaller components to improve readability and performance.
* Clean up routes. (i.e: There should only be module level and page level urls. If there are sections added as third level url example ‘item-management/item/add’, then break the url down as ‘item-management/item?action=add’)
* Reduce unnecessary pages and break it down as components. (Add page can be moved as a full window drawer or a paper component in item-management item page) 
* Follow proper folder structure. (Info provided below)
* In Transport layers, if there are multiple API methods using same functionality then create a common method to accept url, method, body and handle response. Response must handle 2XX, 4XX, 5XX code. (More info provided below)
