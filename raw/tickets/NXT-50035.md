# NXT-50035 — LMA - License Keys - Add

## Description
As a License Management (LMA) role member, I would like to add a license to a license key so that I can update it

*Notes:*

* {color:#ff5630}*NOTE: The bolt code has been updated to fix a few errors and has a slightly updated UI. Get the latest code before working on this screen*{color}
* 
* The use case for the story is the customer wants to add a new license after they purchased the old one and we want it to have the same lifecycle as the others they already purchased
* Business rules at play here:
** We cannot add licenses to license keys once they have been linked to a district
* So instead of “updating” the licenses in the license key, we will generate a new one that cancels out the original.

*Flow:*

* LMA role user browses to System > License Management
* User sees a list of licenses generated for the software
* User clicks Add Licenses button on a row
* Add License drawer opens and user selects which licenses he wants and saves
* System generates a new license key with the original values plus the new ones that were added
* System automatically links the new license key with the district
* System automatically cancels the existing license that the user added licenses too
* License Keys grid updates with the new information
* When district users login, they now have access to the additional licenses 

*Requirements:*

* On License tab grid, do the following:
* Grid Actions column:
** Add License
*** On click: 
**** Open the Add License drawer
*** Only enabled for Active Status
**** Note: if Status = New, they can just click the Edit button
*** Only License Management role can click this button



*Add License drawer*

** Use sliding motion



*Header*

*** Defaults to today

* Show these fields as editable and required:

*Add New License Section*

** Adding a license

*Add New License*

* Works the same as Generate New License Key
* When the user adds a new license, add it to the New licenses section below
* Only add licenses that don’t already exist



h4. Previous & New Licenses

** License name

* Add Licenses button

*** If confirmed:
***

**** Start date = today()

**** Licenses grid stuff

## Acceptance Criteria
h3. 1. Access and Navigation

* From the License tab grid, clicking Add License opens the Apply Licenses drawer.
* Add License button is only enabled for license keys with Active status.
* If the license key status is New, users should use the Edit button instead.
* Only License Management (LMA) role can click the Add License button.

h3. 2. Drawer Layout

* Title: Apply Licenses
* Width: half page
* Drawer opens with a sliding motion.

h3. 3. Header Section

* Read-only fields:
** District
** License Key
** Usage message (from Bolt code)
** Effective Date
** Expiration Date
* Editable and required field:
** Notes (default value per Bolt code)

h3. 4. Add New License Section

h4. Adding a License

* Works like the License Types section in Generate New.
* User selects which licenses to add.
* Only licenses that don’t already exist can be added.
* When added, new licenses appear in the New Licenses section below.

h4. Previous & New Licenses Table

* Displays all licenses with Site or Device deployment.
* Combined previous and new licenses into one table.
* Read-only fields:
** License name
** Deployment value (Site or Device)
* Editable fields:
** Count
*** Default to actual existing count
*** User cannot decrease count, only increase
*** Increasing count means adding more licenses
** Revenue
*** Default to existing revenue
*** Default to 0 for newly added licenses
*** Works same as Generate New screen

h3. 5. Add Licenses Button

* Only LMA role can click.
* On click:
** Validate all changes and required fields.
** Show confirmation message:
_"This will immediately update the license with your additions. Are you sure?"_
** If confirmed:
*** Generate a new license key.
*** Set all values same as the existing key except:
**** Notes
**** Generated Date = Now()
**** Generated By = current user
**** Licenses grid updated with added counts and new licenses
*** Automatically link the new license key to the district.
*** Cancel the existing license that was updated and append a note:
_"Canceled because it was overridden when adding licenses."_
*** Show success message.
*** Close drawer and return user to License Keys page with updated grid.
*** Log change as an Update in the History table.

h3. 6. Security

* Only License Management (LMA) role can take actions on this page.
