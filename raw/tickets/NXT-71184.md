# NXT-71184 — FIN — Reports: Trial Balance

## Description
> **For QA context:** A trial balance lists _every_ GL account with its debit and credit balances to verify that total debits equal total credits; a balance sheet only shows Assets, Liabilities, and Equity as a financial snapshot — no individual accounts, no Revenue or Expenditure detail, and no debit/credit columns.

_Last updated: May 6, 2026 — removed Account Code Range (From/To) filter from report parameters (3.1); Base Type and Account Category filters already cover filtering needs at our scale._

## User Story

As a district finance user, I want to run a trial balance report that shows year beginning balances, prior period activity, current period activity, and ending balances for each GL account, organized by base type and account category, so that I can verify my books balance and assess the district's financial position at any point in time.

---

## Context

The Financials 2.0 trial balance is designed around the job to be done: _verify my books balance and assess the district's financial position at any point in time, in a way that aligns with standard accounting practice._

The report is modeled after Oracle's General Ledger Trial Balance Report to ensure familiarity for districts that export to Oracle and to align with industry-standard reporting conventions.

The key design principles:

* **Structured hierarchy** — GL accounts are organized under base types and user-defined account categories, with subtotals at each level and a balancing grand total, giving users an immediate read on financial health.
* **Site as a filter, not a drilldown** — the user runs the report at the district level or for a specific site. Same report, same structure, different data scope.
* **Period-based with range support** — the report ties to fiscal year and period as configured in Periods. Beginning balances come from the Opening Balance Import, which is faster, auditable, and consistent with how finance teams actually close their books.
* **Oracle-aligned column structure** — year beginning balances, net beginning balance, prior periods, period activity, ending balances, and net ending balance — so the math is fully traceable: Year Beginning + Prior Periods + Period Activity = Ending Balance.
* **Visibility of all accounts** — GL accounts with a non-zero beginning or ending balance appear even if they have zero period activity, so nothing is silently hidden.

The report will be built using **Power BI Report Builder** (paginated reports), which natively supports row group hierarchies with toggle visibility.

## Requirements

### 1) Reports Landing Page Integration

The Trial Balance report must be added to the existing Reports landing page as a tile.

1.1 The Reports landing page must be updated to display the following tiles (sorted A–Z):

| Report Name | Description |
| --- | --- |
| General Ledger | Export detailed debits and credits with per-account worksheets. |
| Journal | Export a list of all financial postings in transaction order for a selected date range. |
| Revenue & Expenditure (P&L) | View revenue and expense activity summarized by category for a selected period. |
| Trial Balance | Verify that total debits equal total credits across all accounts for a selected period. |

1.2 Each tile must include the report name and the description stated above. The entire tile is clickable and opens the report's configuration page.

1.3 Hover state uses elevated shadow. Click leads to the specific report's configuration screen.

### 2) Report Configuration Page

2.1 Page title: **"Trial Balance Report"**

2.2 Helper text directly under title: _"Choose a fiscal year and reporting period to verify that total debits equal total credits across all accounts. Use the period range toggle to view activity across multiple periods."_

2.3 Tooltip: _"You'll be notified once your file is ready and can download it from the Document Center."_

### 3) Report Parameters

3.1 The user must be able to specify:

| Parameter | Notes |
| --- | --- |
| Fiscal Year | Required; defaults to the current fiscal year as configured in Periods |
| Period | Required; single period selection. Defaults to the current open period. |
| Period Range toggle | Optional; when enabled, replaces the single Period selector with From Period / To Period. Defaults to off. |
| From Period | Visible only when Period Range is enabled. Required; defaults to Period 1 of the fiscal year. |
| To Period | Visible only when Period Range is enabled. Required; defaults to the current open period. |
| Site | Optional; defaults to All (district-level rollup). Single selection. |
| Base Type | Optional filter; defaults to All. Allows selection of a specific base type (e.g., Assets only) |
| Account Category | Optional filter; defaults to All. Filters to GL accounts within a specific user-defined account category |

3.2 When Site is set to All, the report shows GL account balances summed across all sites (district-level rollup).

3.3 When a specific Site is selected, the report shows GL account balances for that site only. The report structure and hierarchy remain the same.

3.4 **Single period mode (default):** When the user selects a single Period, the report treats it as From Period = To Period. See section 6.6 for how each column group is calculated.

3.5 **Period range mode:** When the user enables the Period Range toggle and selects From Period / To Period, the report covers a multi-period span. See section 6.6 for how each column group is calculated.

3.6 When the selected period (or From Period in range mode) is Period 1 of the fiscal year, the Prior Periods columns show 0.00 (there are no prior periods).

### 4) Outcome

4.1 A district finance user can run a trial balance report that shows all GL accounts with year beginning balance, prior period activity, period activity, and ending balance in a single view.

4.2 The report covers all five base types: Assets, Liabilities, Equity, Revenue, and Expenditures.

4.3 Total debits must equal total credits at the grand total level — this is the primary proof that the books balance.

### 5) Report Structure and Hierarchy

The report hierarchy has three levels:

**Base Type → Account Category → GL Account**

* **Base Type** is system-defined: Assets, Liabilities, Equity / Fund Balance, Revenue, Expenditures.
* **Account Category** is user-defined: groupings of similar GL accounts created by the district (e.g., within Assets a district might create Current Assets, Investments, Receivables, Fixed Assets).
* **GL Account** is the individual account (e.g., Cash on Deposit - Banks, Inventory of Food).

5.1 The report must be organized by base type in the following order:

1. Assets
2. Liabilities
3. Equity / Fund Balance
4. Revenue
5. Expenditures

5.2 Within each base type, GL accounts are grouped under the user-defined account categories.

5.3 **Debits and credits are kept separate at every level of the report — they are never netted against each other in the debit/credit column pairs.** All subtotal and grand total rows independently sum the debit column and the credit column. The only place debits and credits are _compared_ is the grand total row, where they must be equal as a proof that the books balance. Net columns (Net Beginning Balance, Net Ending Balance) are provided as a convenience alongside the debit/credit pairs, consistent with Oracle's GL Trial Balance Report.

5.4 Each account category must display a subtotal row that independently sums the debit column and the credit column (not netted). The Net Beginning Balance and Net Ending Balance columns must also display values on subtotal rows, calculated as the subtotaled debit minus the subtotaled credit for their respective column pairs, with the DR/CR display rule from 6.2.

5.5 Each base type section must display a subtotal row that independently sums the debit column and the credit column (not netted). The Net Beginning Balance and Net Ending Balance columns must also display values on subtotal rows, calculated as the subtotaled debit minus the subtotaled credit for their respective column pairs, with the DR/CR display rule from 6.2.

5.6 The report must display a grand total row at the bottom showing total debits and total credits across all base types. These two totals must be equal for a balanced set of books. The Net Beginning Balance and Net Ending Balance columns must also display values on the grand total row, using the same debit-minus-credit calculation and DR/CR display rule from 6.2.

5.7 Each level of the hierarchy — base type, account category, GL account — must be independently expandable and collapsible.

5.8 The user can expand all or collapse all with a single control.

### 6) Column Structure

| Column | Description |
| --- | --- |
| Account Code | GL account code (e.g., `5.1010.000.000`) |
| Account Description | GL account name (e.g., "Cash on Deposit - Banks") |
| Year Beginning Balance — Debit | Opening debit balance as of the fiscal year start date configured in Periods |
| Year Beginning Balance — Credit | Opening credit balance as of the fiscal year start date configured in Periods |
| Net Beginning Balance | Year Beginning Debit minus Year Beginning Credit (single column; see 6.2) |
| Prior Periods — Debit | Total debit activity from Period 1 through the period immediately before the selected period (or before From Period in range mode) |
| Prior Periods — Credit | Total credit activity from Period 1 through the period immediately before the selected period (or before From Period in range mode) |
| Period Activity — Debit | Total debit activity during the selected period (or From Period to To Period in range mode) |
| Period Activity — Credit | Total credit activity during the selected period (or From Period to To Period in range mode) |
| Ending Balance — Debit | Net ending balance, populated only for GL accounts with a net debit position as of the end of the selected period (or To Period in range mode) |
| Ending Balance — Credit | Net ending balance, populated only for GL accounts with a net credit position as of the end of the selected period (or To Period in range mode) |
| Net Ending Balance | Ending Debit minus Ending Credit (single column; see 6.2) |

6.1 The ending balance for each GL account is calculated as a single net value and displayed on its natural side:

**Ending Balance = (Year Beginning Debit − Year Beginning Credit) + (Prior Periods Debit − Prior Periods Credit) + (Period Activity Debit − Period Activity Credit)**

* If the result is positive, the amount is displayed in the Ending Balance — Debit column; Ending Balance — Credit is blank.
* If the result is negative, the absolute value is displayed in the Ending Balance — Credit column; Ending Balance — Debit is blank.
* If the result is zero, both columns show 0.00 (subject to the zero-balance hide rule in 7.4).

This matches Oracle's convention of printing one side per account for beginning and ending balances.

6.2 **Net columns display the absolute value with a DR or CR suffix.** Net Beginning Balance = Year Beginning Debit − Year Beginning Credit. Net Ending Balance = Ending Debit − Ending Credit. The formula does not change per base type. The display rule:

* If the result is positive, display as `[amount] DR` (net debit balance).
* If the result is negative, display the absolute value as `[amount] CR` (net credit balance).
* If the result is zero, display `0.00` (no suffix).

This applies at every level: GL account rows, account category subtotals, base type subtotals, and the grand total. The DR/CR suffix tells the user which side the balance sits on without requiring them to interpret sign conventions — positive values are typical for Assets and Expenditures (DR), negative for Liabilities, Equity, and Revenue (CR).

6.3 Year Beginning Balance columns must be populated from the Opening Balance Import for the fiscal year. The year beginning balance represents the financial position as of the fiscal year start date configured in Periods. If no opening balance import exists, the Year Beginning Balance columns display _"Not set"_ with a visual indicator rather than zero.

6.4 Revenue and Expenditure accounts always show 0.00 in Year Beginning Balance columns, as these accounts reset at fiscal year start. This must be clearly indicated in the UI to avoid user confusion.

6.5 When the selected period (or From Period in range mode) is Period 1 of the fiscal year, Prior Periods columns show 0.00 for all accounts.

### 6.6) Column Calculations by Date Range Selection

The fiscal year start date, periods, and their date ranges are defined by the district in the Periods configuration. The examples below assume a district has configured FY 2025–2026 to start September 1, 2025 with 12 monthly periods (Period 1 = September, Period 2 = October, etc.).

**Example A — Single period mode, Period = November (Period 3):**

| Column Group | What it contains |
| --- | --- |
| Year Beginning Balance | Opening Balance Import — the district's balances as of the fiscal year start date (September 1, 2025) |
| Prior Periods | All activity from Period 1 (September) through Period 2 (October) |
| Period Activity | Activity for Period 3 (November) only |
| Ending Balance | Year Beginning + Prior Periods + Period Activity — the balance as of the end of November |

**Example B — Range mode, From Period = October (Period 2), To Period = December (Period 4):**

| Column Group | What it contains |
| --- | --- |
| Year Beginning Balance | Opening Balance Import — the district's balances as of the fiscal year start date (September 1, 2025) |
| Prior Periods | All activity from Period 1 (September) only — everything before From Period |
| Period Activity | Activity from Period 2 (October) through Period 4 (December) |
| Ending Balance | Year Beginning + Prior Periods + Period Activity — the balance as of the end of December |

**Example C — Range mode, From Period = September (Period 1), To Period = December (Period 4):**

| Column Group | What it contains |
| --- | --- |
| Year Beginning Balance | Opening Balance Import — the district's balances as of the fiscal year start date (September 1, 2025) |
| Prior Periods | 0.00 — there are no periods before Period 1 |
| Period Activity | Activity from Period 1 (September) through Period 4 (December) |
| Ending Balance | Year Beginning + Period Activity — the balance as of the end of December |

### 7) Display and Formatting

7.1 All currency values must be formatted consistently with two decimal places.

7.2 Negative values must be clearly distinguishable (e.g., parentheses or color).

7.3 GL accounts with zero activity in the selected period range must still appear in the report if they have a non-zero beginning or ending balance.

7.4 GL accounts with zero beginning balance, zero period activity, and zero ending balance may be hidden by default with an option to show them.

7.5 The report header must display: District Name, Report Title, Fiscal Year, Period (or From Period / To Period if range mode), Site (or "All Sites"), and the date and time the report was generated.

### 8) Export

8.1 The user can export the report to Excel (XLSX) and PDF.

8.2 The Excel export must produce a flat table of the currently visible data with all subtotals included. No row grouping outlines, no merged cells, no empty cells. Ready for immediate use without cleanup.

8.3 The PDF export must render the currently visible level of the hierarchy.

## Acceptance Criteria
**AC1 — Landing Page Tile** Given an authorized Finance user opens the Reports section, When the landing page loads, Then a "Trial Balance" tile is displayed with the description "Verify that total debits equal total credits across all accounts for a selected period." and tiles are sorted A–Z.

**AC2 — Configuration Page** Given the user clicks the Trial Balance tile, When the configuration page loads, Then the page title reads "Trial Balance Report," helper text is displayed under the title, and the tooltip about Document Center notification is present.

**AC3 — Default Parameter Values** Given the configuration page loads, When the user has not changed any parameters, Then Fiscal Year defaults to the current fiscal year, Period defaults to the current open period, Period Range toggle is off, and Site defaults to All.

**AC4 — District-Level Rollup** Given the user runs the report with Site = All, When the report renders, Then each GL account row shows values summed across all sites, grouped under base type and account category headers.

**AC5 — Site-Level Filter** Given the user runs the report with a specific Site selected, When the report renders, Then each GL account row shows values for that site only, with the same report structure and hierarchy.

**AC6 — Correct Column Structure** Given the report renders, When the user views any GL account row, Then they can see Year Beginning Balance (Debit/Credit), Net Beginning Balance, Prior Periods (Debit/Credit), Period Activity (Debit/Credit), Ending Balance (Debit/Credit), and Net Ending Balance columns.

**AC7 — Column Math** Given the report renders, When the user views any GL account row, Then the Ending Balance equals (Year Beginning Debit − Year Beginning Credit) + (Prior Periods Debit − Prior Periods Credit) + (Period Activity Debit − Period Activity Credit), displayed in the Ending Balance — Debit column when the result is positive or in the Ending Balance — Credit column when the result is negative, with the opposite column left blank.

**AC8 — Prior Periods in Single Period Mode** Given the district has configured FY 2025–2026 starting September 1 and the user runs the report with Period = November (Period 3), When the report renders, Then Prior Periods columns show total activity from Period 1 (September) through Period 2 (October), and Period Activity shows November only.

**AC9 — Prior Periods in Range Mode** Given the district has configured FY 2025–2026 starting September 1 and the user enables the Period Range toggle with From Period = October (Period 2) and To Period = December (Period 4), When the report renders, Then Prior Periods columns show total activity from Period 1 (September) only, and Period Activity shows October through December.

**AC10 — Prior Periods Zero When Period Is Period 1** Given the user runs the report with Period set to Period 1 of the fiscal year (or From Period = Period 1 in range mode), When the report renders, Then Prior Periods columns show 0.00 for all accounts.

**AC11 — Net Column Calculation** Given the report renders for an Asset account with Year Beginning Debit = 100,000 and Year Beginning Credit = 0, When the user views the Net Beginning Balance column, Then the value is 100,000. And for a Liability account with Year Beginning Debit = 0 and Year Beginning Credit = 50,000, the Net Beginning Balance is −50,000. The formula is always Debit minus Credit regardless of base type.

**AC12 — Base Type Grouping** Given the report renders, When the user views the report, Then GL accounts are grouped under base types in order: Assets, Liabilities, Equity, Revenue, Expenditures, each with a subtotal row.

**AC13 — Account Category Grouping** Given the report renders, When the user views GL accounts within a base type, Then they are grouped under the user-defined account categories, each with a subtotal row.

**AC14 — Beginning Balance Not Set Indicator** Given no opening balance import exists for the selected fiscal year, When the report renders, Then Year Beginning Balance columns show *"Not set"* rather than 0.00 for permanent accounts.

**AC15 — Revenue and Expenditure Beginning Balance Zero** Given the report renders for any period, When the user views Revenue or Expenditure accounts, Then Year Beginning Balance columns show 0.00 with a clear label indicating these accounts reset at fiscal year start.

**AC16 — Grand Total Balances** Given the report renders for a period with complete data, When the user views the grand total row, Then total debits equal total credits.

**AC17 — Zero Balance Accounts Hidden by Default** Given GL accounts with zero beginning balance, zero period activity, and zero ending balance exist, When the report renders, Then those GL accounts are hidden by default with an option to show them.

**AC18 — Parameter Filters** Given the user applies a Base Type, Account Category, or Account Code Range filter, When the report renders, Then only GL accounts matching the filter criteria are displayed, with subtotals reflecting the filtered data.

**AC19 — Excel Export** Given the report has rendered, When the user exports to Excel, Then the export contains all visible rows and columns as a flat table with subtotals and no empty or merged cells.

**AC20 — PDF Export** Given the report has rendered, When the user exports to PDF, Then the PDF reflects the currently visible state of the report.

## Edge / Empty / Error States

| Scenario | Behavior |
| --- | --- |
| No transactions exist for selected period range | Report renders with zero Period Activity; beginning and ending balances shown if available |
| No opening balance import for fiscal year | Year Beginning Balance columns show *"Not set"* with visual indicator |
| Period has not yet closed | Report renders with available data; indicator shown if period is still open |
| GL account has no account category assigned | GL account appears under the base type's default category |
| From Period is after To Period (range mode) | Validation error; report does not run |
| Account Code Range returns no accounts | Report renders empty with a message indicating no accounts match the filter |
