# NXT-47315 — Remove - API – View History – Back End

## Description
As a SchoolCafé admin, I would like to be able to View the audit history of API import configuration changes so that I can see who made a change and when.

*Note:*

* We already do audit / history tracking a bunch in System team, so please copy what they are doing
* Do not reinvent this, use the framework they have created
* This is the backend part of API View History and needs to be done before the front end part

*Requirements*:

* For Add New and Edit API Import Configurations, do the following
* When you save the configuration on the review screen, do the following:
* When doing Add New:
* Save all the label / value pairs from the config screens to the history table
* Use the multi-level logging from System
* Fields to save:
** Select & API
** List these fields as children:
*** Provider
*** Import Type
*** Provider API
** Field Mapping
*** List each mapped field as a child of Field Mapping
**** For any that have code mapping, list the code mapping pairs as a child of the mapped field
*** Ex: 
**** SchoolCode
***** 1101 > 01
***** 1102 > 02
** Schedule
** List this fields as children
*** Scheduler
*** Day
*** Start Time
*** End Time
*** Frequency
** Review
** List these fields as children
*** Import Name
*** Enabled
* 
* When doing Edit:
* Record in the same format as Add New anything that was added, edited or removed
* Do not record anything that was not modified
* 
* When a user deletes a configuration, record the deletion in the history log as well
** Even though user won’t see it, Dev team can go see who did it when in the database in case there is an issue

## Acceptance Criteria
* All additions, edits and deletes are recorded in the history log for API Config Imports
* History log table uses the framework created by System team
* When a user deletes a configuration, it is recorded in the history log
