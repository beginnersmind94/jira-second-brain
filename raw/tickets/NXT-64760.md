# NXT-64760 — SC - MT - District - Link Districts

## Description
As a Customer Support role user, I want to be able to view data related to a Central realm so that I can manage org level.

*Notes*

* This story covers adding Org Level to the Central screen and assumes the Central Screen story has been completed
* Users is covered in another story
* A Realm is a generic term for District, state, area, etc.

*Requirements*

* On the System > District + New and Edit screens, do the following:
* Add tabs to the screen:
** General
*** Everything that is there now will go under General
*** Update permissions so that User must be in customer support or technical support role, else it is disabled
**** Central/District users cannot be allowed to add or remove districts
** Realms
* Realms tab, do the following:
* Add a button:
** Label: + New
** Permissions: User must be in customer support or technical support role, else it is disabled
*** Central/District users cannot be allowed to add or remove districts
** On click:
*** Allow the user to select a child realm and add it to the grid
*** Cannot select itself, so filter itself out
*** Cannot add a realm that is already added as a child to some other realm
* -Add a grid to show the children of the current realm-
** -Data: All children of the current realm plus any of their children-
*** -Note: A state might have areas which might have districts-
** -Be able to add or remove a realm as a child to the current realm-
** -Grid should have paging, sorting and filtering-
** -Columns:-
*** -Name-
*** -Realm (State, District, etc.)-
*** -Actions-
* -Actions column:-
* -View button-
** -On click:-
*** -Validate user has rights to the realm-
**** -If not, show message: You do not have permission to view this realm-
*** -Open the realm for the user like you do when changing districts-
* -Delete button-
** -On each row in the grid, allow the user to remove a child realm-
** -Use the usual trash can icon-
** -Permissions: User must have page delete permission, else it is disabled-
** -On click:-
*** -Show confirmation button-
*** -If Yes, unlink the child realm and remove from the grid-
*** -Show a green success message if successful-
* 
* If there are no children in the realm viewing the page:
** Show a message: No realms linked.
* 
* All district users with view rights in their role to view everything in read-only mode

## Acceptance Criteria
h2. *1. Tabs*

* On *System > District + New and Edit* screens, add two tabs:
** *General*
*** Contains all existing fields currently on the page.
*** Permissions:
**** Editable only by *Customer Support* or *Technical Support* roles.
**** All other users see the tab and its content as *read-only*.
** *Realms*
*** Used for managing child realms linked to the current realm.
*** Permissions:
**** Editable only by *Customer Support* or *Technical Support* roles.
**** *Central/District users* cannot add or remove realms.

----

h2. *2. Realms Tab – Add Realm*

* Add a *+ New* button above the Realms grid.
** *Label:* {{+ New}}
** *Permissions:*
*** Enabled only for *Customer Support* or *Technical Support* roles.
*** Disabled for all other users.
* *On Click:*
** Opens a selector to choose a child realm to link to the current realm.
** *Filters:*
*** Exclude the current realm itself from the list.
*** Exclude any realm that is already a child of another realm.
** On confirmation, the selected realm is added as a *child* to the current realm and displayed in the grid.
** Display a *green success message* confirming the addition.

----

h2. *3. Realms Tab – Grid*

* The grid lists all *children of the current realm*, including *nested children* (e.g., a State → Areas → Districts).
* Grid supports:
** *Paging*
** *Sorting*
** *Filtering*

h3. *Columns:*

# *Name* – The name of the child realm.
# *Realm* – The type (e.g., State, County, District, -Area, Sub Area-).
# *Actions* – View and Delete controls.

----

h2. *4. Actions Column*

h3. *View Button*

* *Behavior:*
** On click, validate if the user has permission to view the selected realm.
** If the user lacks permission, show message:
*** “You do not have permission to view this realm.”
** If the user has permission, open the realm as if the user were switching to that district (same behavior as changing districts).

h3. *Delete Button*

* Displayed as a *trash can icon* on each row.
* *Permissions:*
** Enabled only for users with *Delete permission* on the page.
** Disabled for users without it.
* *On Click:*
** Show a confirmation dialog:
*** “Are you sure you want to unlink this realm?”
** If confirmed:
*** Unlink the child realm from the current realm.
*** Remove it from the grid.
*** Display a *green success message* confirming the removal.

----

h2. *5. Empty State*

* If no children realms exist for the current realm, show message:
** “No realms linked.”

----

h2. *6. Data Handling & Validation*

* *Data Source:* Grid pulls from all linked child realms of the current realm, including their own children where applicable.
* *Validation:*
** Prevent circular relationships (a realm cannot be its own child or ancestor).
** Prevent duplicate linking of the same child realm.
** All changes must be saved and reflected immediately upon success.

----

h2. *7. Permissions Summary*

||Role||Add||Delete||View||Edit (General tab)||
|Customer Support|✅|✅|✅|✅|
|Technical Support|✅|✅|✅|✅|
|Central / District|❌|❌|Read-only|❌|

----

h2. *8. Performance & UX*

* Tabs and grid load within standard performance thresholds (<3 seconds).
* Sorting and filtering occur client-side or with minimal latency.
* UI supports dark mode and follows the app’s existing styling conventions.
