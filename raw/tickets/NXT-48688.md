# NXT-48688 — GDSN Connect - ADF Integration (Data Exchange)

## Description
End User Flow:

* The user uploads to data via SchoolCafe (Item Management → Items → Upload Button(^)
* The data gets pushed to GDSN Connect DB (part of Global Catalog)
* The fire and forget method runs all the required validations and parses all the required data on to target tables in GDSN Connect DB and populate the data in SchoolCafe required format
* The ADF workflow is required to call the APIs further and kick start the API calls at Schoolcafe end show the import status in UI

## Acceptance Criteria
The ADF Integration should cover the following requirements:

* Design a new ADF that calls the following APIs 
** GdsnImportItems - this API will write all the parsed data from GDSN Connect DB to Schoolcafe DB
*** GdsnGetImportStatus - this API will return the status either as “Data Parsing Successful” or “Import Failed / Data Parsing Failed”
** GdsnImportImages - this API will translate images into azure blob objects (thumbnail and regular image)  
*** GdsnGetImportStatus  - this API will return the status either as “Image Processing Successful” or “Image Processing Failed”
** GdsnImportStatusSync - this API will write the GDSN Connect DB with Schoolcafe Item Import Status, Exception message, Item Code and Item ID
* Import Monitor:
** For Internal Purposes - there should be an entry in Import monitor that can be viewed to check if the ADF fails to process
** No configuration or Field Mapping applicable
