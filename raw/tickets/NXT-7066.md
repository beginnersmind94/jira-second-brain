# NXT-7066 — Revenue

## Description
*Note from Matt: This and the Site Revenue report could potentially be combined, and be a Dundas report. There’s a lot of overlap here - it’s all about how we’re showing the data.*

*User Story*

As a district admin I want to see revenue data to get an idea on site performance and as a measure for breaking even vs operating costs.

PrimeroEdge version:

!image-20201204-021808.png|width=1416,height=469!

!image-20201204-021944.png|width=1409,height=1308!

*Requirements*

* User can select site(s), date(s), meal(s).
* User can opt for a summary or a daily breakdown of data.
* User is shown revenue data broken out by:
** Date
** Account Type (student/adult)
** Meal
* User is shown totals, including any tax taken. 
* User is shown any additional income that may have been added outside of session/sale data.
* User is shown a grand total of revenue.

## Acceptance Criteria
User can select a site or multiple sites.
User can select a date or a date range.
User can select any configured meals for the district.
User can see a summary, or a daily breakdown.
User is shown accurate revenue data, including tax and additional income.
User is shown revenue totals.
