# NXT-5235 — Direct Certification List

## Description
Designate Assistance Programs from Other Source Categorical Eligibility. Designate federal, state, and/or local.

*User Story*

As a district admin I want to see all direct certification (DC) file imports that have happened year-to-date, and historically.

*Mockup*

[https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=30749%3A2444&scaling=min-zoom|https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=30749%3A2444&scaling=min-zoom]

*Business Information*

* School Districts import files with data from a 3rd party - usually the Department of Human Services - with information that needs to be matched against local student data.
* These .xls/.csv files are uploaded into the product and analyzed/algorithm’d to find matches.
* If a successful match is made, then eligibility benefits are granted to the identified student.
* This story is to show a list of all files that have been imported into the system.

*Requirements*

* Add a Direct Certification page to the Eligibility module.
* Page contains a grid showing information on all Direct Cert files that have been imported by that district.
** Columns
*** File Number
**** This is a friendly local reference number for  each DC file.
***** Friendly as in easy to reference - start at 00001 and increment with each ‘file’.
*** Program
**** This will be one of the following:
***** Assistance Program
***** Other Source Categorical Program (*)
*** Type
**** The type is tied to the Program selected above.
**** Assistance Program:
***** This will be one of the following and should use the local descriptions:
****** Direct Certification
****** SNAP
****** TANF
****** FDPIR
****** Medicaid (Free)
****** Medicaid (Reduced)
****** Foster
**** Other Source Categorical Program
***** This will be one of the following, use local descriptions:
****** Even Start
****** Head Start
****** Homeless
****** Migrant
****** Runaway
****** Pre-K
*** Status
**** This will be one of the following:
***** Pending
***** Preprocessed
***** Processed
***** Deleted
*** Import Date
**** Date/time the file was uploaded.
*** Process Date
**** Date/time the file was processed.
*** Approval Date
**** ‘File Date’ in Classic.
**** It’s the date/time designated if this file was backdated during the upload process.
***** User will have an option to say ‘this file represents information as of X date,’ which may be a date in the past.
*** User
**** First Name + Last Name of the user who performed the file upload.
*** Matched Students
**** Number of students who were successfully matched based off information on this file.
*** Academic Year
**** The academic year that the file was uploaded into.
** All columns should be sortable and filterable.

* Add search filters to the Direct Certification List page which will prune the results in the grid accordingly.
* Search Criteria / Filters:
** Academic Year
*** This will be a single year selection, ex: 2019 - 2020.
*** An ‘All’ option should also be available and is the default.
** Search by Approval
*** Approval Type
*** File Number
**** Text entry, will search for any files with that number.

## Acceptance Criteria
Direct Certification (DC) page added.
Grid with high-level information on all DC files available.
Grid columns are filterable, sortable.
Search for approval type / file number added.
