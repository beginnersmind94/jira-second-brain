# NXT-35402 — INV - Reports - Item History - Transaction status & enhancements

## Description
Make some needed small improvements to the experience of generating and reading the Item History report that should improve clarity.

* Add a transaction status label to the Transaction Type column for adjustments, reversals and cancellations (Reference Transaction History Report)
** put this label in parentheses after the Transaction Type value
*** e.g. 'Reconciliation (Reversal)
** Consider using stylized font (italics, smaller)
* Correct positive/negative numeric formatting for Adjustment Quantity
** If we are expecting this column to make sense with the Perpetual column
** use parentheses instead of minus sign, for the whole quantity string
* Clean up the generation page behavior 
** Ex: the item dropdown resets your search when you select an item (*R&D*)
*** it’s possible the user wants to search “Cheese” and select multiple items
*** *Update:* this is a component level behavior that would need to be discussed with Components team
** Consider slight design cleanup
*** Possibly check with [~accountid:5f99ce5044658b0071265de9] after date picker component is changed, to ensure proper sizing and alignment
** *Note: Date Range picker being changed in a [different enhancement|https://primeroedgenext.atlassian.net/browse/NXT-35374]
