# NXT-6782 — <To Be Rewritten> System - Users Phase 2 - Add User  - Save new user's details

## Description
As a District Admin, I would like to be able to easily grant PrimeroEdge access to a new user in my organization, so that they can quickly begin working in the system.

+Summary+:

Allow an existing user to add a new user if all required fields contain valid data.

+Details+:

* In the Add User screen, add *Save* and *Cancel* buttons to the bottom of the screen. 
* When the user selects the Cancel button, warn them with a Yes/No prompt (see mock-up; message may change slightly).
** If the user selects No, close the prompt.
** If the user selects Yes, do not commit any of their changes (including user image/avatar) and redirect them back to the main Users screen.
* When the user selects the Save button: 
** Commit the changes and add the user
** Redirect the user back to the main Users screen.
** *An email (based on NXT-4022) should be sent to the new user with their credentials plus a random password.*
** Display a success toast message (Not pictured in mock-up; Daimien can provide text if needed, or a generic ‘User added successfully\! An email has been sent to this user with their credentials.’ should be fine)
* The new user should now be locatable in the Users list
