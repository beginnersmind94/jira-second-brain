# NXT-70455 — SCTV+ - Enforce Strict Column Placement Rules for Menu Planning and Menu List Widgets-Web Dashboard 

## Description
This update enforces strict per-column placement rules for *Menu Planning* and *Menu List* widgets to prevent layout conflicts and rendering issues on device slides.

The goal is to improve slide stability, eliminate invalid configurations, and ensure consistent rendering behavior across all supported layouts.

----

h2. Summary of Changes

h3.  Menu Planning Widget (MP)

* Fixed the column row cap logic to count *actual Menu Planning widgets per column*, not total rows.
* Users can add *up to 2 Menu Planning widgets per column*.
* Columns containing a Menu Planning widget are capped at *2 total rows*.
* A maximum of *2 columns per slide* may contain Menu Planning widgets.
* Any violation triggers the existing *orange warning modal*.

----

h3.  Menu List Widget (ML)

* Strengthened the column guard:
** A Menu List widget can only be added to a *completely empty column*.
** No other widgets of any type may exist in that column.
* Removed the previous “warn but allow” behavior — the block is now *absolute*.
* Adding an empty slot (Top/Bottom) to a column that already contains a Menu List widget is *blocked*.
* Violations trigger the existing *blue warning modal* with updated messaging.

----

h3.  Items Limit (Menu Planning)

The number of items in a Menu Planning widget is capped at *5 items* when:

* The slide contains more than one Menu Planning widget, *OR*
* The focused Menu Planning widget shares its column with another widget.

The cap is enforced consistently across:

* Slider changes
* Manual text input
* On blur validation

If a user attempts to exceed the limit, the value is clamped to 5 and the warning modal is shown.

## Acceptance Criteria
h1. Acceptance Criteria

h2. Menu Planning Widget Rules

# A user can add a *first Menu Planning widget* to a column successfully.
# A user can add a *second Menu Planning widget* to the same column successfully.
# Attempting to add a *third Menu Planning widget* to the same column is blocked and shows the orange warning modal.
# Attempting to add Menu Planning widgets to a *third column* on the same slide is blocked and shows the orange warning modal.
# A column containing a Menu Planning widget cannot exceed *2 total rows*.

----

h2. Menu List Widget Rules

# A Menu List widget can be added only to a *completely empty column*.
# Attempting to add a Menu List widget to a column that contains any other widget is blocked and shows the blue warning modal.
# Attempting to add an empty slot (Top/Bottom) to a column that already contains a Menu List widget is blocked and shows the blue warning modal.
# The system does not allow bypassing the rule under any condition (no “warn but allow” behavior).

----

h2. Items Limit Enforcement (Menu Planning)

# When a slide contains more than one Menu Planning widget, the item count is capped at *5*.
# When a Menu Planning widget shares its column with another widget, the item count is capped at *5*.
# If a user attempts to increase items beyond 5:

* The value is automatically clamped to 5.
* The appropriate warning modal is displayed.

# The cap is enforced consistently via:

* Slider interaction
* Manual numeric input
* On blur validation

----

h2. Stability & Validation

# No invalid layout configuration can be saved.
# No rendering errors occur on device slides when using supported configurations.
# No console errors occur during widget addition, removal, or item updates.
