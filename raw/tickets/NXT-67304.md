# NXT-67304 — Optimize Nginx Route Configuration and Apply Performance Enhancements

## Description
Currently, the Nginx configuration defines all routes with regular expressions even after the main path. This adds unnecessary overhead and negatively impacts performance.

The goal of this ticket is to:

* Remove unnecessary regular expressions from Nginx route definitions.
* Apply additional Nginx properties and optimizations to improve performance.

## Acceptance Criteria
# *Configuration Changes:*
#* Remove redundant regular expressions from Nginx route definitions.
#* Implement performance optimizations (e.g., caching, compression, keep-alive, etc.) as required.
# *Functional Verification:*
#* Verify that *all pages and modules* of the application load correctly after the configuration changes.
#* Ensure no broken routes, missing assets, or errors occur.
# *Performance Validation (Optional but Recommended):*
#* Measure key performance metrics before and after changes to confirm improvements.
