# NXT-6570 — Item Management - POS Item Descriptions

## Description
*Description:*

Item Management needs to have a way to indicate what ‘items/menu items’ are servable through the POS. POS Menu Items have specific POS Menu Item attributes

*Mockup:*

[https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=4288%3A69517&scaling=scale-down-width|https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=4288%3A69517&scaling=scale-down-width]

*Requirements:*

* POS Specific Section
** Button Description English
** Button Description Spanish (optional)
** Is a Reimbursable meal
** Is Vendible indicator
** Is Orderable indicator (Ver 2)
** Meal Type (Derived from Program Configuration)
** Prices see [https://primeroedgenext.atlassian.net/browse/NXT-184|https://primeroedgenext.atlassian.net/browse/NXT-184|smart-link]
** Order Parameters if Is Orderable is  TRUE. (Ver 2)
** A POS Item can be set to a Default Reimbursable meal
** POS Item Category

## Acceptance Criteria
Above items are part of Item Management.
POS Info is added to Item Management Steps, After Product Info.
Button Name (English) - required
Button Name (Spanish) - optional
Meal Type - optional
There are Toggles for Is Meal, Is Vendable, Is Orderable, Is Default Reimbursable Meal.
IF Vending is not enabled for Vending 'Is Vendable' toggle is not shown (setting is ENBVENDINT)
IF Ordering is not enabled 'Is Orderable' toggle is not shown (setting is ORDMEALS)
For Default Reimbursable Meal to be set, Is meal must be set and Meal Type must also be set.
Clear will reset all values
Skip for Now allows the user to not enter or have to save any data and closes the frame.
POS Item Category 

When the Role permission 'POS Menu Items' is disabled the user can only view POS Information.
    All fields in POS Information are viewable.
