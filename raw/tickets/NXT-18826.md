# NXT-18826 — Production- Reports- Production Form (UI)

## Description
This report  gives the user a way to print production records and fill them out by pen before entering the information into the UI.

In the day to day workflow, the user would use this report to complete all information before production record is marked complete on the UI. However if they do not, and print both the production form and record, they will look identical except for the title of the report.

## Acceptance Criteria
Page Criteria for Report:
Site Code
Site
Site Group
Meal Type 
From and To (Date)

Report should display very similarly in format to the production record page with the following criteria
Display on report:
Site code with Site Name
Serving Date
Meal Service
Menu Line
Preparation Site
Offer VS Serve (True or False)
Meal Pattern
A text box showing:
Serving Group
Planned Count 
Actual Count

There also needs to be a display of any plan or record comments that users have added on UI side. 
Columns to display information are:
Locally Grown
Menu item (Recipe #) and a small size font underneath for serving size with (portioning tool)
Serving Group
Adult
Total Planned
Servings Produced
Total Served
Adult/ A la Carte
Reimbursable
Leftover
Comments

Items also need to display in same order of category as on the record.
