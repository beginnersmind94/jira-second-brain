# NXT-36442 — Device Management – FO – Additional columns in Terminals Table, update TerminalData API (API ONLY)

## Description
_This is Device Management Phase 1 for Front Office_



We need to add additional fields to our Terminals data table so that Device Manager can save the relevant information.

*Notes:*

* Device Management uses Status and Description in various screens and ways
* Our POS terminal does not have these fields
* This story is to add the columns to the table
* System team has multiple dependencies on this table and its APIs being created in order to finish the Device Manager feature

*Requirements*:

* Add the following to the terminals table:
** Description
*** max length = 255
** Status
**# Known values at this time:
**## Active
**## Suspended
**## Deleted 
* Update the TerminalData API to include:
** Status
** Description
** TerminalType
*** The name, not the ID number - join table to get description
* POS needs to interact with the Status column
** If the user renames their terminal, update the existing status to Suspended
** If the terminal was suspended on them and they’re launching POS, they need to do terminal setup again
*** Update the status to Active if they reclaim the POS
*** New terminals should be Active also

----

TerminalData API ultimately needs to provide the following:

* SchoolID
* SiteName
* TerminalType – the name, not the id, get via join
* TerminalName
* Status
* Description
* LastCommunicationDate
* AppVersion

## Acceptance Criteria
* Columns are added 
* Available APIs updated to include TerminalType description, and the new fields Description and Status
