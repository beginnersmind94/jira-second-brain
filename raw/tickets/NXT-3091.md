# NXT-3091 — **System - Regions List View

## Description
As an administrator, I would like to be able to view and create a list of areas to which all of my sites belong, so that I can properly organize my sites by area and run efficient area-specific reports of site groups instead of pulling a report one site at a time.

Classic PrimeroEdge path: _System > Sites and Users > Regions_

!image-20200121-221523.png|width=1039,height=299!

!image-20200121-221545.png|width=604,height=464!



Functionality-wise, this can be a straight port of the existing application. The main changes should be in terms of UI/UX improvements.

## Acceptance Criteria
1. User should be able to view existing regions/areas within a district when navigating to the page
2. User should be able to select a region and see a pop-up/expansion displaying the region's details
