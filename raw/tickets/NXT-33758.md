# NXT-33758 — MP- Menu Cycle- Standard/Menu Planner View Scenarios

## Description
As a Menu Planner, I enjoy the interactive nature of the menu planner and would like to be able to edit my menu after I create it. I would like to be able to come back to the menu Planner screen and view or make my edits from the Menu Planner screen and not just the edit menu page.

## Acceptance Criteria
* View: User can only view the menu from either option but will not be able to make any changes or edits. Everything is in view mode
* Give the option on the serving groups on the menu planner screen to pick what serving group they would like to see and show the daily analysis based on the standards of that particular serving group and their counts. 
** User can switch between serving groups at any time
** Display Projected Meal Count on a per day basis per serving group so the user knows what the analysis is running based on for example:
*** Projected Meal Count K-5: 100 
*** Projected Meal Count 6-8: 100
*** Projected Meal Count 9-12: 100 
** !image-20220811-201754.png|width=235,height=354!
** *IF serving group is not available for a particular day on the menu, display a message on the day “Serving Group not available for this menu/day”*
* When showing the menu or menus, display the daily analysis based on the counts entered on the edit menu page
** IF user has entered in a projected meal count and no menu item counts, divide the menu item counts based on meal counts and choose counts 
*** For example: User enters 250 projected meal count for 9-12 and there are 2 menu items in the entrée category, divide the counts to 125 and 125 respectively. 
**** Repeat for all categories with the same process
*** Follow the same logic currently be used when we can not perfectly divide the counts evenly between menu items and give the highest count to the first in the alpha order EX: 200 projected meal count with 3 menu items in a category would be divided into 67, 67,66.
** IF user has entered in a projected meal count and some menu item counts, display the menu items that do not have counts as a Not served items with a toggle to give them an option to “turn it on”. At this point, this menu item is not included in the analysis as it is technically not being served,  Turning it on will only be available in the edit mode and not view mode. *THIS ONLY APPLIES IF SOME MENU ITEM COUNTS HAVE BEEN ENTERED.*





* *Discussed in Refinement for clarity: If user creates 3 weeks of menus on menu planner originally, but skips week 2, we save the cycle as a 2 week menu cycle and we shift week 3 to week 2. When viewing or editing, we will continue to bring the menu cycle back in the same fashion.*
