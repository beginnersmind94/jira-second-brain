# NXT-69388 — Push POS Reimbursement Claims to Financials for Automation (Backend)

## Description
As a user, I want the software to create ledger entries for my reimbursement claims.

## Acceptance Criteria
*Requirements:*

* Ledger entries are created per Site.
* Financial periods must be open from the date of posting to the current date.
* *When creating these ledger entries in the database, each debit/credit line must be attributed to the originating data point / data source (e.g., the specific meal sales revenue data point or reimbursement data point) so the Profit & Loss report can calculate totals by data point even when one GL account is mapped to multiple data points.*

*Reimbursement Claim*

* GLEntryTypeCd: RMBRSCLM
* Posting date is the last day of the Claim Period.
* Debit: Due from Federal Funds (Asset)
* Credit: Reimbursement Program System Accounts (Revenue)

*Reimbursement Received*

* GLEntryTypeCd: RMBRSRCVD
* Posting date is the current date.
* Debit: Cash in Bank (Asset)
* Credit: Due from Federal Funds (Asset)

*Rollbacks*

* If a claim is recreated then, the user will be able to resubmit a claim. When the resubmit button is clicked the previous Reimbursement Claims and Reimbursement Receives will have rollback ledgers generated for them and the new claim will be posted.
** Should this be a setting?
