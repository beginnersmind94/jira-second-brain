# NXT-15475 — External notifications

## Description
External communication to parents and other entities needs to occur when certain conditions within the various modules are met.  We would like a mechanism that detects when these conditions occur and when they do, will send the communication related to the condition.

We would like for the various modules to be able to own and manage the definition of these conditions as well as the communication templates that are sent.  These condition definitions and templates should be able to changed at runtime by the various modules.

## Acceptance Criteria
* Modules shall have the ability to “register” functions that will return the values true or false based on conditions that exist within their data/system.
** The module’s “function” shall provide the list of recipients to receive the notification if the function returns true.
* Modules shall have the ability to define templates that will be sent out for external communication
* Modules shall have the ability to define when and how often the mechanism evaluates their function
* Modules shall have the ability to list, add, update, delete, enable, and disable these execution of these external notification jobs
* A mechanism shall be available that will periodically evaluate the module’s functions and, if true, will send out a notification using the defined template associated with each function.
* The existing platform notification mechanism shall be utilized, where possible, to enable this feature
* A “Developer’s Guide” or implementation instructions shall be made available so that the module’s development teams know how to use our mechanism.
