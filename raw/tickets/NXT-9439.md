# NXT-9439 — INV - Transfers (District) - Create Transfer

## Description
As an inventory  user I want a screen that would allow me to search for availability  of an item or list of items in a site or across district site’s inventories.

+*Site Selection*+

* From Site
** allow an ‘All’ selection option
** Limit of 1:1 transfers right now, eventually will do many to one
*** this will allow the District to perform a many-to-one transfer (multiple sites transferring to one receiving site)
**** This would generate multiple transfers
* To Site
** only allow specific Site selection

+*Transaction Details*+

* Date
* Time
* Comments
* Apply button
* District Transfer Number
** Create individual site-to-site transactions
** Once one of the site-to-site transactions has been touched, then it would lock the other ones and they would need to be edited one by one

+*Item Selection Criteria*+

* The user will have a smart search functionality to enter the Item # or Description of the item for which they are looking.
* This will only allow the user to add an item to the Transfer Item List if it matches an item existing in the system
* Once an item is found, the user can select it and *add* it to the Transfer Item List

+*Item List*+

* Item# 
** This is a selectable field - Check Box-
** When selected the To Site# - Name field is made available to the user.
* Item Description
** Alphanumeric field 
** Field length should be the same as defined in Item Management
* Whole Unit 
** Item Whole Unit description / abbreviation
* Broken Unit
** Item Broken unit description / abbreviation
* +*Transfer Quantity - Site sub-grid*+
* If a district is facilitating a Transfer Request, they will jump straight to this part (NXT-…..)
** i.e. the To Site and the Items have already been determined by the Transfer Request
* District users (who would obviously be able to see all sites QOH) will see a *sub-grid* that lists individual sites:
** Site
*** Code + Name
*** only list sites which have the item on-hand
** Whole Unit QOH
*** the site’s current Whole Unit Quantity of that item
** Broken Unit QOH
*** the site’s current Broken Unit Quantity of that item
** WU Request Qty - numeric entry field
*** the user will input the request quantity they want from _that specific site_
** BU Request Qty - numeric entry field
*** the user will input the request quantity they want from _that specific site_
