# NXT-17151 — UI-  Production - Production Plan

## Description
As a site manager, the production plan will reflect the menu that has been assigned to my site on the calendar. This will contain the menu items that have been assigned to me, as well as the grade or age groups that were assigned in the menu. 

When the user clicks on ‘Plan Incomplete’ it will bring them to this page

!image-20210706-153044.png|width=100%!



Display as: Title of the Menu (Link) if permissions allow - Site Code | Site Name | Date of Service | Meal Type | Menu Line

I must be able to enter in counts that reflect the quantity of servings that I estimate will be served of each menu item, for each grade group. In addition, I must also be able to enter a number for the estimated total quantity of meals per age/grade group and/or person type. 

!image-20210706-153107.png|width=100%!





The page should also allow access to the recipes that I will need for this service, and the technician’s worksheet that will coordinate with this service.

If there are carryovers from a previous date, the page should display the quantity that are being carried over, and those counts should be taken into consideration into the total counts displayed on the page.

There should also be a place to enter any comments that might correlate with my plan.  I should have the ability to confirm or decline “Offer Vs. Serve” on the page. There should also be the ability to add a menu item from this page, and the ability to save the data that I’ve input, and display the status of the plan. The Food preparation Site should also be displayed (in the event that my site is configured for preparation at another location)

The page must also contain information regarding the Meal Type, the date of the plan that I’m on, and the name of my site.

## Acceptance Criteria
This page will allow/display the following:

1.) Site Name

2.) Date of service

3.) Meal Type

4.) Line Name

5.) Offer vs. Serve (set by site configuration, but should be adjustable on this page)

6.) Food preparation Site should be displayed (in the event that my site is configured to have my food prepared elsewhere)

7.) Serving Groups with text boxes where I can enter planned counts for meals

8.) List of menu items that are assigned to the site (with text boxes for each age and/or grade group for planned counts per item)

9.) If there are carryovers from a previous date, the page should display the quantity that are being carried over

10.) Technician’s Worksheet Report

11.) Production Recipes

12.) The ability to Add Menu Items

13.) The ability to save the data that I’ve input

14.) The status of the plan
