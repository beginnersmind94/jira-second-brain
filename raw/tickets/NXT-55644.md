# NXT-55644 — DMF - View Log - Grid Enhancements

## Description
As a Customer Support role user, I would like to be able to view efficiently the migration execution steps so that I can quickly diagnose the issue causing the error

*Requirements:*

* In the database, add step number to the activity steps and number them in the proper order
* Create an API endpoint to provide this data per Module / Database
* 
* For the View Log overlay, on each tab, do the following:
* On each grid, add a column
** Colum Header = Step
** Value = the step number for that activity
*** Ex: 7 for the seventh step
* 
* On each grid, change the column header from Step Name to Run
* 
* On each grid, default the pagination to be greater than the number of steps required for a successful run
** Ex: System DB Source SQL Monitor has 47 steps, so make the default 50
* 
* On each grid, set the default sort order: 
** Run desc, then Step desc
** Therefore, the first row should be the last step of the last run
* 
* -On each grid, show only one run at a time-
* -To do this:-
** -Add a drop down list to filter the grid-
** -Label: Run-
** -Values: list of runs in descending order-
** -Default Select the latest run-
** -On Click, update the grid to show the data for the run selected-
** -Default the grid data to show only the latest run data-

## Acceptance Criteria
* Step is added as a column in each grid
* Step Name is changed to Run in each grid
* Default sort order shows the last step first in each grid
* Default pagination shows all the steps for a completed run in each grid
