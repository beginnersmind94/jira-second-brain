# NXT-4641 — [AcctMgt] - ID Cards - Settings

## Description
*User Story*

As a District Admin I want a way to control the settings tied to printing ID cards.

*Mock-up*

N/A

*Requirements*

* Add a way to set and change the following settings which should be tied into the printing/alignment of ID cards.
* *Most of these are in Classic and have already been carried over. This story is to verify they are working correctly and have some of them populate on the ID Cards page.*
** Print Multiple Cards on a Single Page (Standard: 10-15 per page)
*** Options: Yes / No
*** This controls whether or not multiple ID cards will be printed on the same page, or if each card will be its own page.
*** {color:#6554c0}When this is set to ‘No’, make the ‘1 Card Per Page’ option the default in the ‘Cards Per Page’ drop-down{color}
** Number of ID Cards Per Page
*** Value input, must be a number between 0 - 10.
*** This controls how many ID cards need to be on a single page.
*** {color:#6554c0}This relates to the ‘Cards Per Page’ menu in the UI. When ‘Multiple Cards Per Page’ is selected, it will display the number corresponding to this value.{color}
** The following are margin settings with values that affect how the card looks when printed out.
*** Card Top Margin
**** {color:#6554c0}Margin - Top default value in the UI{color}
*** Card Side Margin
**** {color:#6554c0}Margin - Left default value in the UI{color}
*** Horizontal Margin between Cards
**** {color:#6554c0}Margin - Right default value in the UI{color}
*** Vertical Margin between Cards
**** {color:#6554c0}Margin - Bottom default value in the UI{color}
*** These have already been added to the ID card UI. This story is to make sure that when the default values of any of them are updated in the Settings page, those are the values that show when the user goes to the ID Cards page. 
**** These can be changed by the user while on the ID Cards page, but when the page is refreshed, we should show the default values again.
* New setting:
** Orientation
*** Options: Horizontal (default), Vertical.
*** {color:#6554c0}This will be the ‘Orientation’ field’s default value in the ID Card UI{color}
* Hide any other card-related settings. Those can be ignored for now.
* We will utilize the ‘inch’ settings in a future release. For now, all margin guides are in pixels only.

## Acceptance Criteria
Settings to control aspects of ID cards added.
Settings have an appropriate impact on printing, alignment, etc.

When a user goes to the ID Cards page, they should see each of the values from the settings mentioned above, for the fields mentioned (Cards Per Page, Orientation, Margins)

Field values can be modified by the user, but should be reset to default when the user refreshes the page

Settings related to ID Cards that are not mentioned above should be hidden from the System Settings UI for now.
