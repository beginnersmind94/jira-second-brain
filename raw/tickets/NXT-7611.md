# NXT-7611 — Data Exchange - File Mapping - Allow user to merge field headers

## Description
As a PrimeroEdge Admin, I would like to be able to have a multiple columns on my import file be fed into a single field in PrimeroEdge, so that I can capture the data from files more accurately.

*Summary*

Allow users to merge multiple field headers so that they share a single ‘Drop’ cell and update database records as expected when mapping is completed.

*Details*

* On the Field Mapping step of the Data Exchange wizard, allow the user to combine multiple Field Headers. We can use an icon that denotes a merge.
* When a field is merged, combine the 2 rows in the File Preview table.
** Additionally, combine the related Confirm PrimeroEdge cells in the Confirm Mapping table.
* User should be able to then map all of the fields within the cell as normal
* When the user proceeds to the next step (at whatever point we commit the data), do the following:
** Make sure that the import will copy the data from both merged fields of the source file into each of the fields specified
** Other types of operations can be added in the future
* Disable Merge option for non-mergeable fields

## Acceptance Criteria
* Verify that on the Field Mapping step of the Data Exchange wizard, we allow the user to merge multiple source file headers to a single cell in the File Preview table.
* Verify that the UI adjusts appropriately (rows merged as expected)
* Verify that when the user proceeds to the next step (at whatever point we commit the data), we make sure that the import will copy the data from the source file into each of the fields specified
* The “PrimeroEdge” field name shall be populated in the “Mapped Fields” column, by default, with light-grey font color to indicate that the field has not been mapped.  When the source field is dragged/dropped to the “Mapped Fields” cell, the font color shall become green (indicating it is mapped). The legend shall contain an entry to communicate this to the user.
