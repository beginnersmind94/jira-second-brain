# NXT-5237 — Data Exchange - Direct Certification Import Type

## Description
*Summary*

As a district admin I should be able to import Direct Certification specific to a state and preserve configurations. so any other user within that state/org can import DC data file using same configuration.

[https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=3024%3A51698&scaling=scale-down-width|https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=3024%3A51698&scaling=scale-down-width|smart-link] 

!image-20201001-211218.png|width=1049,height=385!

*Requirements*

* User to be able to select “DC Import Type” from pre-upload screen within DE 
* *_User should have ability to select the “Import File Configuration”_* 
* Default template specific to that state should be available for user to download and reupload with DC data in file upload screen
* User selects delimiter and qualifier
* User uploads DC Import data file in File-Upload screen
** File types supported are XLS, TXT, XLSX, CSV
** File size should support upto 250 MB
** Only one file should be allowed to be uploaded
** Rest of the specs should follow student import specs
* System detects for previous configurations based on state and import type and display for user to use the pre-defined/previous configurations
** If they’re in Texas, then any active file configurations that are ‘public’ (pushed from Central) and for that State should be available.
* Attached list of mappings and specifications for configurations
** 2.0 DC Import mappings file & Config screens from 1.0 are attached
* System validates for datatypes, mandatory fields and other checks similar to student import
* Once user moves to next step after mapping, Preview screen should be displayed for user first 10 rows of data from their file, mapped up against the file configuration they’ve selected.
* User should be able to preview and select SAVE Options to perform standard DE operations.
* When user selects Config & Import, DC Import process begins with a link to monitor import status
* Once Import starts, Retain import job ID (reference number like File#) to be shared/reconciled with Eligibility POS screen

## Acceptance Criteria
1. Verify state specfic DC Import File template is downloadable
2. Verify valid DC import file is being uploaded by the user
3. Verify file types must be in acceptable formats - xls, xlsx, txt, csv.
3. Verify system detects previous configurations based on state and import type. 
4.  Verify system stores the import data and configurations
5. Verify system generates an unique File Number or reference ID for import job to displayed in Eligibility screen
6. A confirmation prompt is must before the import and configuration save
7. Verify system retains the data when user switches and returns from previous screen
