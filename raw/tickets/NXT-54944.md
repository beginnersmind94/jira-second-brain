# NXT-54944 — POS - Peripheral Devices - Identify and Manage Utility Tool (Research)

## Description
Freshdesk ticket: [https://primeroedge.freshdesk.com/a/tickets/258419|https://primeroedge.freshdesk.com/a/tickets/258419]

As Sally the Site Manager I have 4 peripheral devices plugged into each terminal - two barcode scanners, two pinpads - and I need the POS software to identify each device separately and keep them separated.

*requirements*

* Research into building a utility tool to identify and manage peripheral devices
** Need to be able to designate what the input device is for → ID Numbers, Cashier Keyboard
** Aim will be for sections of the POS to only accept input from certain devices → Cashier Keyboard can go into Lookup, Payment Screens. ID Numbers go into the Queue.





----

*Discussion Notes*

* We have many different ways to access the POS website - how do we handle this agnostically? Can we?
** Different browsers
** Different operating systems
** Computer vs Tablets/Handhelds
* The logical follow-ups to this:
** If we know what device is sending us input, can we make sure ID Number entry isn’t co-mingling?
** Can we set and then force peripheral data to go into the Queue, and let the cashier’s main keyboard be used for areas like the Payment Amount boxes?
** We can setup ‘left’ and ‘right’ for keypads and do POS Arrows.
