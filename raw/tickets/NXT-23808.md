# NXT-23808 — INV - Receipts - Apply logic to consider Receipt Price setting (API)

## Description
As a district admin I want the receipt price to consider the vendor contract setting so that my items populate with the correct price.

Setting is added as part of [https://primeroedgenext.atlassian.net/browse/NXT-16356|https://primeroedgenext.atlassian.net/browse/NXT-16356|smart-link] 

*Requirements*

* Add logic to the Create Receipt screen to populate the price (and charges + discounts) according to the new vendor setting
** Use the Transaction Date + Time to determine which date to use
** if the setting is set to ‘Delivery Date’ the receipt price should reference the price according to the order’s delivery date
*** this should be current functionality
*** this should also mean the receipt price matches the order price by default
** if the setting is set to ‘Receipt Date’ the receipt price should check the pricing conditions for the date that the receipt is being created
*** this will also check the charges and discounts according to the date as well (the entire price conditions, not just the item price itself)

## Acceptance Criteria
* Apply logic to the Create Receipt screen to consider new setting for how receipt price is determined
