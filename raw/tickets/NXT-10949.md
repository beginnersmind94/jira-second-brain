# NXT-10949 — [SchoolcafeTesting] - Applications - Automatic Processing and Notifications

## Description
As a district admin I trust the system to automatically process applications that have no glaring issues.

*mockup*

n/a

*requirements*

This is tied to the following system settings being switched on:

* Automatically Validate Online Applications if All Required Fields are Completed (ONLAPPROCE)

This setting has multiple values:

# Do Not Auto Process (No)
# All Applications (Yes)
# Only if all students are identified (yes, but)
# Only if at least 1 student is identified (yes, but)

If the setting is set to yes, then the system will need to automatically process applications as they come in. (2)

If it is set to no, then the system should not automatically process any applications. (1)

The identification pieces (3, 4) relate to pending students

* 3 - if all students are NOT pending, then automatically process the application
* 4 - if at least 1 student is NOT pending, then automatically process the application

## Acceptance Criteria
Application processing happens automatically dependent on the system setting

Setting value: Do not auto process
- no applications are automatically processed, they are left in New status

Setting value: All Applications
- all applications are automatically processed provided there are no hard stops

Setting value: All students are identified
- no pending students - all are linked to a student in the DB - then automatically validate the app

Setting value: At least 1 student
- if at least 1 student is linked to a student in the DB, automatically validate
