# NXT-64772 — CC - AI - Schoolie Prompt

## Description
As an Insights role user, I would like to see a summary of my performance generated by AI in natural language on the Insights screen so that I can get more understandable feedback.

h2. Notes

* This is not the same as the recap on the home screen
* It is a different API that does a different analysis
* It corresponds to the AI button on the WS Insights screen

h2. User Flow

* User logs in and goes to Insights
* User sees the summary button and clicks on it
* AI Summary drawer opens
* App gets the district and site access and a calculated time range
* App calls WS Insights API
* System gathers the correct data and a prepared AI prompt and sends to the AI engine
* AI engine analyzes data using prompt and returns a paragraph style text summary
* App shows the summary to the user

h2. Requirements

* Ensure the screen supports dark mode

_Insights Screen_

* Add a button to the top
** Use Icon
*** Title: “AI Summary”
** On click: Open AI Summary screen (covered below)
** Permissions Check
*** Check if district is licensed for Insights AI (currently a system setting, future will be actual license)
*** Check if user is in Insights role
*** If No for either, then hide the card



_AI Summary screen_

* Add a new screen to the app
** name: Schoolie Insights
** No need to put in the menu system
** Permissions: same as the AI Summary button
* On the screen, Should work just like the corresponding Schoolie Insights drawer in Insights
** Use a loader and then show the results on the screen
** Add the AI summary section to the screen same as Schoolie Insights drawer in Insights
** It is a summary (paragraph) text section 
** Don’t need the expand / collapse (show/hide) since this is the only thing on the screen
* Fields:
** back button
* Data Set 
** The results are dependent on the user’s district and site access 
*** If the user only has access to one site, results show only for that one site, etc. 
** The results are dependent on a date range
*** APIs need date range, but you will be showing data for only one day, but different start and end times
*** The results are dependent on time as follows: 
**** If after 10:30 am, show results for Today 
**** Else, show results for previous operating day 
***** Note: For Sunday and Monday, that would normally be Friday if it wasn’t a holiday 
** Get a dataset with these district, site and time filters 
* On screen load: 
** Use Period value on Insights screen to determine time frame
** App calls API and passes in the district, sites and timeframe
** System gets the dataset using the site and time filters 
** System sends the dataset and a previously prepared AI prompt to the AI engine to perform the analysis 
** AI engine does its analysis and returns the result in a text format
** System loads the result in the AI section 
** Show a loader icon while waiting on the AI engine. If you can, include text to let them know we are waiting on the the AI engine.
* 
* Prompt Text:
** Use prompt text from WS Insights Schoolie Insights drawer
* 
* Error Handling:
** If there is an error in retrieving or processing the AI summary (e.g., failure to contact the AI engine), an appropriate error message should be displayed to the user.
** Note that in WS, it just shows the loader forever when there is an AI issue; we do not want this
** Log the error
* System Performance:
** The system must load the AI-generated summary within a reasonable amount of time, aiming for a smooth user experience without significant delays.

## Acceptance Criteria
h3. *1. Dark Mode*

* Ensure the *Insights screen supports dark mode* consistently with the rest of the app.

----

h3. *2. AI Summary Button*

* *Location*: Top of the Insights screen
* *Type*: Icon button
* *Title*: {{AI Summary}}
* *On Click*: Open *AI Summary screen* (Schoolie Insights)

h4. *Permissions Check*

* District must be *licensed for Insights AI* (currently a system setting; future will use actual license)
* User must have *Insights role*
* *If either check fails* → hide the button

----

h2. *3. AI Summary Screen (Schoolie Insights)*

* *Screen Name*: {{Schoolie Insights}}
* *Menu*: Not included in main menu
* *Permissions*: Same as the AI Summary button (district license + Insights role)
* *Dark Mode*: Supported

----

h3. *3.1 Layout & Display*

* Loader *while waiting for AI engine*
* *Single summary section* (paragraph text)
** No expand/collapse needed
** Full-screen focus on AI summary
* *Back Button*: Returns user to Insights screen

----

h3. *3.2 Data & Filtering*

* *User Scope*: Filter results based on user’s district and site access
** Only show sites the user has access to
* *Date Range / Timeframe*:
** Use *Period value* from Insights screen to determine start/end times
** If *after 10:30 AM*, show *today’s results*
** Else, show *previous operating day*
*** For Sunday/Monday → normally Friday unless holiday
* *Dataset*:
** API receives district, sites, and timeframe
** System fetches dataset filtered accordingly
** Dataset sent to AI engine with pre-prepared prompt for analysis

----

h3. *3.3 AI Processing*

* *AI Engine*:
** Performs analysis on dataset
** Returns results as *text summary*
* *Loader Behavior*:
** Show *loader icon and optional message* while waiting
** Ensure loader *does not spin indefinitely* in case of error
* *Prompt*: Use *existing WS Insights Schoolie Insights drawer prompt text*

----

h3. *3.4 Error Handling*

* Display *friendly error message* if AI summary cannot be retrieved or processed
* *Log all errors* for diagnostics
* Avoid the behavior seen in WS where loader spins forever

----

h3. *3.5 System Performance*

* AI-generated summary must *load quickly*, providing smooth UX without noticeable delays
* Aim for *real-time-like response*, while handling backend processing efficiently

----

This version ensures:

* Dark mode compliance
* Clear permissions rules
* Accurate filtering for districts, sites, and time
* Proper API → AI → UI flow
* Loader with error handling
* Fast, reliable AI summary display
