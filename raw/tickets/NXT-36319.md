# NXT-36319 — Account Balances - Monitoring Page

## Description
As a SchoolCafé admin, I want to be able to view the output of the reconciliation of Account Balances so that I can proactively monitor the application and ensure higher customer satisfaction.

*Flow:*

* User logs in and browses to Support > Monitoring Dashboard > Account Balances
* User views a list of payments with their reconciliation status
* Optionally, the user can reconcile a payment

*Notes:*

* This story assumes the Monitoring Dashboard has been created
* This story’s UI/UX is based off the Exports page and is following that page’s flow, layout and functionality
* We are only changing the appropriate fields as necessary, so when in doubt, copy what that page does
* This page is the user front end way of reconciling the Account Balances
* There is a schedule job that also reconciles on a nightly basis, so you can get a lot of the code and logic from that if needed



*Requirements*:

* *Monitoring Dashboard Page updates:*
* Add a card to the Monitoring Dashboard page
* Card should look exactly like the other cards on the page and like those on Item Management > Reports
* Card title: Account Balances
* Description: View results of the Account Balances reconciliation
* On click, open the Account Balances page
* 
* *Account Balances Monitor Page Requirements*:
* Create the page and link to it from the Monitoring Dashboard
* 
* Base the UI/UX off the Exports page
* 
* Sections:
* *Header Section*
* Title: Account Balances Monitor
* Navigation: System > Support > Monitoring > Account Balances
* 
* *Filters Section*
* Create a filters section at the top of the screen like System Configuration Notification page
* Filter fields:
* Start Date
** Start date and time to filter the results
** Default to today at 1 minute after midnight
* End Date
** End date and time of when to stop
** Default to Today at 1 minute before midnight
* Student Name
* School
* Exceptions Only
** A yes/ no toggle or radio button
** Default selection is no
** If toggle yes, filter the grid to show only rows with Not Matched status
* o    Reset button
** Resets the filters same as for the System Configuration Notification page
* o    Apply button
** Applies the filters same as for the System Configuration Notification page
* o    Collapse / expand arrows
** Expands and collapses the filters section same as for the System Configuration Notification page
* 
* *KPI Section*
* This section is to the right of the filters section, at the same level, above the grid
* Show a simple card with a label / value format
** Title: Exceptions
** Value: count of exceptions
* The filters in the filter column update the count when the user hits Apply
* 
* *Grid Section*
* Show a grid of the Account Balances and their reconciliation status
* Columns:
* Student ID
* Student Name
* Student School
* Transaction Sum
** This is the balance per the source and is considered “the truth”
* Account Balance
** This is what our system says is the balance
* Reconciliation
** This compares the Account Balance vs. the Transaction Sum
** Values: Matched, Not Matched
** Highlight somehow if not matched
** Leave normal text if matched
* Action Button
** Icon: use our standard History icon
** On click, redirect to the View Trans History report in Front Office
** Auto-load the report settings and auto execute the report upon page load
* 
* Permissions:
* In General resources screen for Roles & Permissions
** Visible = true
** Hierarchy: put under System
** Title: Account Balances
** Default permissions: all four checkboxes checked for SchoolCafé Admin
