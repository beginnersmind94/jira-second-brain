# NXT-68312 — Fin - Configure Site-Level GL Sub-Accounts for Each District Account

## Description
As a *district finance administrator*, 

I want to define GL account codes for each school site (and optionally use one district-wide account instead), 

so that transactions can post to the correct sub-accounts while still keeping a clear, maintainable chart of accounts.

*Dev Notes:*

* Site list comes from active sites in Sites in System
* Site code segment position determined by Account Code Structure configuration
* Autofill logic must respect configured segment lengths and formats from Account Code Structure

## Acceptance Criteria
*Site Account Codes Section Display:*

* Edit GL Account drawer shows "Site Account Codes" section below Base Type field
* Section header displays "Site Account Codes" in bold
* Below header, gray text reads: "-Configure sub-accounts for each location. Each site should have a unique account code that includes its site code. If a site account code is left blank, that site will use the main account code for reporting."-
** Updated to “Set unique site account codes per location; blank sites default to the main account code.”
* Section contains checkbox labeled "Use this account for the entire district" (unchecked by default)
* Purple "Autofill Site Account Codes" button appears to the right of the checkbox
* Table displays below with three columns: LOCATION, SITE CODE, SITE ACCOUNT CODE
* One row appears for each active site in the district
* A search field above the Site Account Codes table lets users filter sites by location name or site code in real time.

*Manual Site Account Entry:*

* Each table row shows location name (non-editable), site code in blue/purple text (non-editable), and editable text input field
* Site Account Code input fields have placeholder text "Enter site account code..."
* Users can type or edit values in Site Account Code fields without validation while typing
* Clicking Update Account saves all entered Site Account Code values
* Reopening the drawer shows previously saved values
* *Site-level defaults*: If a site account code is left blank, that site automatically uses the main GL account code for reporting.

*Autofill Functionality:*

* Clicking "Autofill Site Account Codes" takes the Account Code value from top of form (e.g., 1100-0001-0000-000-00000)
* System identifies the site segment position from account code structure configuration
* System replaces site segment (000) with each site's actual code
* All Site Account Code fields populate with generated values (e.g., 1100-0001-0000-602-00000 for site 602)
* Users can manually edit any autofilled value after generation
* If Account Code field is empty or invalid, error message displays: "Cannot autofill: Account Code is invalid or empty"
* No fields populate if Account Code is invalid

*Uniqueness Validation:*

* Clicking Update Account validates no two sites have identical Site Account Code values
* If duplicates exist, save is blocked
* Red error message appears at top of drawer: "Each site must have a unique account code."
* Input fields containing duplicate values display with red border and red text
* Drawer remains open with all values preserved
* User can correct duplicates and save again
* Once duplicate value is changed to be unique, red border and text color return to normal
* Uniqueness check only applies within this GL account (same codes can exist across different GL accounts)

*District-Wide Checkbox Behavior:*

* Checking "Use this account for the entire district" immediately clears all Site Account Code input values
* Checked state disables all Site Account Code input fields (gray background, not editable)
* Checked state disables the Autofill Site Account Codes button
* Informational text changes to: "This account will be used district-wide."
* Unchecking the checkbox re-enables Site Account Code fields (white background, editable)
* Unchecking leaves all fields empty (no auto-population)
* Unchecking re-enables the Autofill button
* Unchecking returns informational text to: "-Configure sub-accounts for each location. Each site should have a unique account code that includes its site code.-"
** Updated to “Set unique site account codes per location; blank sites default to the main account code.” 
* Saving with checkbox checked stores sameAccountAcrossDistrict: true and does not save site account codes
* Saving with checkbox unchecked stores sameAccountAcrossDistrict: false and saves all entered site account codes
* When "Use this account for the entire district" is checked, the Site Account Codes section (including search and grid) is hidden, and all sites use the main GL account code for reporting.

*State Persistence:*

* Reopening Edit GL Account drawer for a saved account shows checkbox state matching last save
* If district-wide was enabled: checkbox is checked, all Site Account Code fields are empty and disabled, Autofill button is disabled
* If district-wide was disabled: checkbox is unchecked, Site Account Code fields show previously saved values, Autofill button is enabled, empty fields remain empty

*Table Scrolling:*

* Districts with more than 10 sites display scrollable table
* Table header row (LOCATION, SITE CODE, SITE ACCOUNT CODE) remains fixed while scrolling
* -Maximum visible height before scrolling: approximately 400px-

*Cancel Behavior:*

* Clicking Cancel closes drawer without saving
* All changes including autofilled values are discarded
* Next time account opens, previously saved values display

*Scope:*

* Site Account Codes section only appears in General Ledger Accounts configuration tab's Edit GL Account drawer
* Other configuration tabs (Account Code Structure, Account Categories, Account Mapping, Sites, Year End Mapping) remain unchanged
