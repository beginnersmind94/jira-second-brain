# NXT-42757 — Elig - Applications - Updates for State Income Guidelines for application entry and processing

## Description
As Anna the Application Processor I want applications to issue State Eligibility based on the State Income Eligibility Guidelines to stay in compliance.

*requirements*

Using this newly added setting: Enable State Income Guidelines (Code: STATEEIG) and if it is set to YES:

* If an application’s total income exceeds the Federal Income Guidelines (Paid + Denied), make use of the State Free and State Reduced guidelines to make a determination.
** If the total income falls within the ranges for State Free or State Reduced, record the State Eligibility accordingly.
** If the total income is beyond State Reduced, record State Eligibility as N/A. 

*Application Processing*

* Show a State Eligibility in any Determination spots.
** If no State Eligibility determination is reached, State Eligibility = N/A
** Show the State Free or State Reduced descriptions from the Eligibility Descriptions page

!image-20230830-222442.png|width=1505,height=416!

* Application Status History should also mention State Eligibility outcome as part of Application Status
** Ex: Notified (USDA Paid - Income, State Free - Income)
** Ex: Notified (USDA Paid - Income, State N/A)

!image-20230830-225913.png|width=974,height=215!



*Manual Application Entry*

* Show State Eligibility on the Submit step under Eligibility Determination

!image-20230830-222108.png|width=1182,height=677!

*Student Eligibility*

* Record the State Eligibility result along with the Federal Eligibility result on the student as part of processing.

## Acceptance Criteria
If the System Setting is/was on when the application was processed:

* Use State Income Guidelines to process State Eligibility accordingly.
* Show State Eligibility on Application Processing/Details, Application Status History, Manual Application Entry.
* Record the State Eligibility on the student.
* Record whether the setting was on at the time of processing.
