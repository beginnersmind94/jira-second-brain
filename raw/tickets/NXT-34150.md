# NXT-34150 — Team Hub - Delete a Message

## Description
As a school admin, I want to delete the specific message details so that I don’t see unwanted messages.

*Flow:*

·         After logging in, user redirected to Home Screen

·         On Home Screen, click the View All Messages Icon and redirected to Messages Screen and on that screen click on a message, or

·         On Home Screen, click on a message in the messages control

·         This screen opens up and user views automatically loaded message details of selected message

·         Admin click on Delete button and message is removed

·         Screen redirects to Messages

 

*Assumptions:*

·         User has logged in

·         To code this, you will have to have completed the push / pull messages stories and one of the source triggers (Home Screen or Messages Screen)

·         To code this, you will have to set up the Message Details screen covered in another story

 

*Requirements*:

·         Fields: No new fields added

·          

·         Update the pre-existing Delete button as follows:

o   If button wasn’t added, then add it

o   On click, show confirmation message

§  If click No, close message and do nothing

§  If click Yes, continue

o   If confirms delete, the app sends a push message to the server to delete this message from the server

§  Then the app gets a confirmation response that the server deleted the message

§  Then the app removes the message from the local copy of messages from the inbox or sent collection, respectively

·         If deleted, the app redirects the user to Messages Screen

·          

·         Security – no additional requirements necessary

## Acceptance Criteria
·         Delete is an Image button like a traditional delete or trash icon

·         On click, show confirmation message

·         If click No, close message and do nothing

·         If click Yes, continue

·         If confirms delete

o   The app sends a push message to the server to delete this message from the server

o   Then the app gets a confirmation response that the server deleted the message

o   Then the app removes the message from the local copy of messages from the inbox or sent collection, respectively

·         If deleted, the app redirects the user to Messages Screen
