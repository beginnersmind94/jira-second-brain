# NXT-20663 — UI Only - MP- Menus- Search criteria Display

## Description
As a user, I should be able to do a smart search for my menus without having to wait for all the menu results to populate.

Would like the same functionality that is occurring on the search page in Item Management when using the filters.

## Acceptance Criteria
When searching Menus, all menus disappear and regenerate a new search.
No loader is displayed (must display loader while waiting for results)
Find a way to allow smart search (seems that API call is being made everytime a new character is being entered)
All menu information should already be available when user starts trying to search.
If we are displaying total number of menus, we know how many menus so we should be able to have all menu names available.
Information for menu can be loaded once user selects a specific menu. 

When searching to add a menu to a menu cycle, there will be a smart search occurring once I arrive at a 3 letter minimum in my criteria, which will continue narrowing as I add quantities of letters. In addition, there will be a display of 30 results to select from.
