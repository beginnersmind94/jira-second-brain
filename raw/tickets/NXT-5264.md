# NXT-5264 — Verification - Tracking - Incomplete Applications

## Description
*User Story*

As a district admin I need to keep track of applications that have been selected for verification and the status they are in.

*Requirements*

* Add a page called 'Tracking' under the Verification menu.
* User should be able to select an academic year and see applications that are part of a sample for the selected year.
** Default: current academic year.
* Page should contain multiple tabs that represent the status an application goes through as part of the workflow.
** There are individual stories for each tab and the info displayed in the grid - this just covers the 1st tab.
* First tab: Incomplete Applications
** Grid columns, typical search and filter options, export option also:
*** Application #
**** searchable
*** Eligibility
**** dropdown-, multi-select-
**** options: free / reduced / denied
*** Basis
**** dropdown-, multi-select-
**** options: income (error-prone) / categorical / income
***** income (error-prone) would be an application that is considered … error-prone
***** income by itself would be an application is not in an error-prone bracket, likely selected at random or for cause
*** Type
**** dropdown-, multi-select-
**** This will be either: Sampled or Cause
*** Confirmation
**** This is a placeholder for future functionality.
*** Selection Date
**** Date the application was added to the sample
*** 1st Notice
**** Date/time for when the 1st Notice was successfully created.
**** There’s an action tied to this if it hasn’t been created but that is future functionality.
*** Follow Up Notice
**** Date/time for when the 2nd Notice was successfully created.
**** There’s an action tied to this if it hasn’t been created but that is future functionality.
*** Days Since Last Notice
**** How many days has it been since the last notice was sent? This can be for either the 1st or follow-up notice.
*** Additional Columns (should be hidden by default, but can be made visible)
**** Student Names
***** single cell containing the name(s) of all students on the application
**** Applicant
***** applicant first + last name
**** Contact Information
***** contact information provided by the applicant
***** street address, phone numbers, email
**** Sample Number

## Acceptance Criteria
Tracking page added.
Incomplete Applications tab added.
Grid shows information by academic year, current AY by default.
Grid shows applications that are currently incomplete for verification.
Grid shows information for selected applications per the requirements.
