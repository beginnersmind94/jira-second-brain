# NXT-14468 — Consolidate Export and Download tasks to Download.

## Description
Currently in Export File, download file is a two step process.First Export and after around 30-60 seconds, the file will be available and then user selects download to download the file. This operation must be reduced to one step process.

In simplified process, end user must have only download option. Once the download option is selected, the backend process of generating the file must be started and and once the file is generated and ready to download, It must give the end user the option of choosing where and how to save. Currently, it just downloads.

If possible, during the generation of the file, Hourglass symbol must be displayed so that the end user is aware that the file generation is in progress.

*Technical Details:*
ADF Export pipeline is triggered when the user clicks on the Export button.
When the API returns with HTTP Status Code=200, we need to call a javascript function every 30 seconds, which will call an API to see that the ADF Export Pipeline has succeeded. 
Once the ADF Export Pipeline has succeeded, will call the File Download API to download the file.
Need to clear the javascript function that is running every 30 secs.
This way we will no longer need the additional download button.
Both the export and download functionality will be achieved with the click of the Export button.
Need to show a progress indicator once the Export button is clicked till the file downloaded successfully.

## Acceptance Criteria
One step process to generate and download the file. 
Once download is selected, download button must be disabled until the file is generated and downloaded by the end user.
Hourglass must be displayed until the file is ready to be downloaded.
Once the file is ready for download, a pop-up window must be populated giving the user an option to choose the location to save and rename if necessary.
