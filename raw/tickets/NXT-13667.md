# NXT-13667 — Data Exchange - SIS Import - Error and data integrity handling - Display Logs-API

## Description
As a central office administrator or technical team member I want to see a log of import errors so that I can determine why particular records in my file did not process correctly.

Expectation would be that we can display the error logs within the current “View Log” functionality of the Import Monitor. We need to see if the error messages can be included within the pipeline logs. If not, we should create a separate tab for showing record processing logs/errors. It is not only about showing errors, but showing any records that triggered an update as well.

## Acceptance Criteria
* User can view error messages for records unable to be processed within the View Log page of the Import Monitor
* Messages should display based on text logged within the database
The summary section should be only visible for Student Import
History Log tab should be only visible for Student Import
Exception tab should be only visible for Student Import performed using SIS APIs (Aeries, Skyward, and PowerSchool)
