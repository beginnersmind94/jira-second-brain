# NXT-36317 — Email - Dashboard

## Description
As a SchoolCafé user, I would like to be able to see the history of Email activity so that I can quickly diagnose if there are any issues.

*Notes:*

* The purpose of this page is to show the history of sending Emails
* It can be used to diagnose if there are any issues
* Multiple features in SchoolCafé can leverage the Email framework, and all their activity can be seen here
* It is like the FTP Framework Dashboard
* There are no add, edit or deletes on this page

*Flow:*

* Open SchoolCafé and browse to Settings > Support > Monitoring Dashboard
* Click on Email Monitor
* View list of Emails sent
* Optionally view the details

*Requirements*:

*Menu Changes:*

* On the page System > Support > Monitor Dashboard, Add a box
** Title = Email Monitor
** Description = View records sent out by the Email Framework
** On click, opens the Email Monitor page

*FTP Monitor Page:*

* Create a new page
** Title = Email Monitor
* 
* Filters Section:
* Source
** Multi-select drop down list
** Optional
** Comes from Source field in Email Distribution
** Values: list of distinct sources from the table
* Date
** Single select drop down list like what we have on Export Monitor
** Optional
** Values:
*** Last 24 hours
*** Last 7 days
*** Last 30 days
* Status
** Multi-select drop down list
** Values: list of statuses for Email executions
* 
* Create a grid:
* Columns:
** Source
*** Example: eOrders
** Sent
*** Date / Time emailed
** Subject
** Recipients
*** Count of people it was emailed to
** Attachments
*** hyperlinks to files sent
*** On click, download or open the file(s)
*** Show the name of the file
*** Truncate names to 20 characters
** Status
* Actions
** View Details
*** Requirements covered in the View FTP Details story
** View File
* 
* Grid has paging
* Grid has sorting
* Grid has filters
* Grid has export to Excel or CSV
* 
* Security
** Add page to resources
** Security is managed via the resource in Roles & Permissions (R&P)
** In R&P front end, this page shows up under System > Support
** R&P Edit permission should be disabled since there is no edit
** R&P Delete permission should be disabled since there is no Delete
** R&P Add Permissions should be disabled since there is no Add
** R&P View Permissions applied to the whole page
