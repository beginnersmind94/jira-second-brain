# NXT-49344 — OTP - Profile - Google Authenticator

## Description
As a user, I want to be able to setup Google Authenticator One-Time Passcode so that I can use my phone app to complete login

*Notes*:

* This story is for One-Time Passcode and is for a user to configure Google Authenticator One-Time Passcode in their profile
* Other One-Time Passcode user configurations covered in other stories

*Flow:*

* User browses to Workspace > Profile
* User clicks the edit button for a Google Authenticator One-Time Passcode row
* User is redirected to Google Authenticator One-Time Passcode edit drawer and completes it
* User saves the config to Keycloak and system marks it as configured
* User tests the config
* User returns and optionally sets One-Time Passcode as the default authentication type

*Requirements*:

*On User Profile > Authentication tab*

* Grid Updates:
** Configured
*** Yes if they have saved the configuration
*** No if they have not saved the configuration
** Method
*** Value: Google Authenticator 
** Validator
*** Show this text: QR Code
** Actions:
*** Edit
**** On click, do the corresponding action listed below:



*Edit Google Authenticator Drawer*

* If they select to edit the Google Authenticator row:
** Open the Configure Google Authenticator drawer
** Title: Configure Google Authenticator 
** Note: on this screen, we need them to scan a QR code with their device in order to register it with Keycloak
* 
* Fields
** Show the user this text message: 
** You need to set up Google Authenticator to activate your account.
** Step 1: download and install Google Authenticator on your mobile
** Step 2: open Google Authenticator on your mobile
** Step 3: scan the QR code on this screen
** [show QR code here]
!image-20240409-191930.png|width=133,height=138,alt="image-20240409-191930.png"!
** Step 4: Enter the one time code provided by Google Authenticator and click Submit
* Textbox
** Label: One-Time Code
** Required if visible
** Numeric only
** Required length = number of digits of the code sent
** User must enter the code they received
* Textbox
** Label: Device Name
** Optional
** Default to: My mobile
** Needed by Keycloak
** Save to the DB when you save the configuration
* Button
** Submit
** On click:
*** Validate they entered in the correct code in the Textbox
*** If yes:
**** Save to the DB
**** Save the Configured value to Yes
**** Check if this is the first saved configuration
***** If yes, mark Preferred to True
**** Show a success message: Verification completed.
**** Close the drawer and return user to updated grid
*** If no:
**** Show failed message: Incorrect code. Try again.
**** Allow them to try up to 3 times
**** After 3rd failure:
***** Show them final failure message: Verification failed.
***** Close the drawer to end the process
*

## Acceptance Criteria
* User is able to scan QR Code in their app
* User is able to open their app, get the code and enter it on the screen to test
