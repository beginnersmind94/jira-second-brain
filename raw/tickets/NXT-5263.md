# NXT-5263 — Verification - Sampling - Page & Info

## Description
*User Story*

As a district admin I want a way to create a verification sample based off my selections for how we’re doing it this year.

As a District admin i would like to see a list of verification samples so that i can begin the verification process

*Requirements*

* Add a Sampling page.
** Page should show the user information about the sample(s) taken for the current academic year.
** User should also be able to see prior academic years.
* If a sample already exists, we want to show information to the user in a grid.
** Sample Number
*** Count starting at 1 for each academic year.
**** This will tie into Rolling Verification where multiple samples per year are possible.
** Method
** Sites
** Selected Applications
*** Number of applications pulled as part of that particular sample.
*** Blank until sample is generated.
** Approved Applications (Free/Reduced)
*** Number of applications that were approved as free or reduced in the time frame.
** Status
*** Complete or Incomplete, depending on whether or not enough applications have been selected.
*** Scheduled - for samples that have not yet been generated.
*** In Progress - for samples that are being recalculated / generated.
*** Tooltip for each status:
**** Complete - This sample contains the correct number of applications. (85/85)
**** Incomplete 
***** Too few - This sample does not contain the correct number of applications. (83/85) Use the Recalculate button to add more applications. 
***** Too many - This sample does not contain the correct number of applications. (87/85) Deselect applications on the Tracking page.
**** In Progress - This sample will be recalculated.
**** Scheduled - This sample is scheduled for a future time and will be generated then.
** Sampling Date
*** We’ll want to show the date range that the sample includes.
*** Example - 3 samples are planned for 8/1, 9/1, and 10/1.
**** Sampling Dates:
***** 7/1/21 - 7/31/21
***** 8/1/21 - 8/31/21
***** 9/1/21 - 10/1/21
** Generation Date
*** Timestamp for when that particular sample was made + User who made the sample.
*** For samples that have not yet been generated, show the scheduled time instead.
* User will need to be able to select samples from the grid for actions covered in NXT-5266.
* If no sample exists, then the grid can simply state that. ‘No sample has been generated for this academic year.’

Sort by Sampling start Date by default.

## Acceptance Criteria
Sampling page added.

Page shows information on samples for a particular academic year.

Current academic year is the default selection.

User can view past academic years.
