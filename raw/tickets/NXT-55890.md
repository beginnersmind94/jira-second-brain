# NXT-55890 — District Authentication - Users - Change Password

## Description
As a Customer Support user, I want to be able to change the password for a user so that they can continue using the website

*Notes*:

* We already have change password as part of profile authentication tab, so you can copy the code from that

*Requirements*:

* These requirements apply to the System > Security > Edit user overlay
* On the grid in the tab:
** Add a button
*** Label: Change Password
*** On click, open the Change User Password drawer
*** Only Customer Support role and Technical Support role users can use the button
**** Make it invisible to everyone else
* The change password and reset password links will be displayed one below other

*Change User Password drawer*

* Create a new drawer
** Title: Change User Password drawer
** Copy the interface over from the Profile Change Password drawer
** Do everything that you do there but for the selected user, not for the logged in user

## Acceptance Criteria
* Customer Support role users is able to change the password for any user account
* Change is logged in the history
* Non-CS and tech support role users cannot change the password
