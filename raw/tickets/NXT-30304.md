# NXT-30304 — Contextual Help Widget - UI Embedding

## Description
As an end-user, I would like the ability to easily access contextual help while I am using SchoolCafe.

## Acceptance Criteria
* A single contextual help (Freshdesk) widget shall be implemented in the following modules:
** Front Office
** Back Office
** System
* The content, UI treatment, and behavior of the “Contextual Help Widget“ shall be the responsibility of the Content team and not the Platform team.
* An “Internal” setting shall be defined for each module.  This setting will determine if the widget is visible or not for that module.
** If the setting is set to true, the widget shall be visible in all pages in the module
** If the setting is set to false, the widget shall NOT be visible in any of the pages in the module
** If the setting is not present, the widget shall NOT be displayed in any of the pages in the module
* The “Internal” setting for Front Office shall be named “Front Office - Enable Contextual Help”
* The “Internal” setting for Back Office Office shall be named “Back Office - Enable Contextual Help”
* The “Internal” setting for System Office shall be named “System - Enable Contextual Help”
* The code snippet for the widget which will be used in production shall be provided by the content team.
