# NXT-16851 — INV - Orders - Order Detail - Create Receipt - Charges & Discounts

## Description
As an operations manager I want a Charges & Discounts section on the Create/Edit Receipt screen, so I can view the calculated fees/discounts and so I can add any additional manual charges or discounts.

This section displays all Charges and Discounts, from the ‘Additional Charges & Discounts’ configuration screen ([NXT-16382|https://primeroedgenext.atlassian.net/browse/NXT-16382]) but also the 3 that are configured per item on the vendor contract.

*Requirements*

* Add a section below the receipt item list, titled ‘Charges & Discounts’
** This section will list all *active* charges/discounts 
* Charges & Discounts List:
** Provide header groupings to separate Charges from Discounts
** ‘Description’ column - this will display the description of the charge/discount
*** this would either be from the vendor contract (Processing Fee, Delivery Fee, Net Off Invoice)
**** under each section, display these charges/discounts before listing all of the others
*** or it would be from the ‘Additional Charges & Discounts’ screen, where users configure their own charges/discounts ([NXT-16382|https://primeroedgenext.atlassian.net/browse/NXT-16382])
** ‘Price’ column - this shows the saved value for the charge/discount as calculated or input on the receipt
*** Price is *not* editable for the 3 types that are defined on the contract (Processing Fee, Delivery Fee, Net Off Invoice), since these are calculated from the total of all items on the receipt
*** Price *is* *editable* for all the other Charges & Discounts, configured via the ‘Additional Charges & Discounts’ configuration screen ([NXT-16382|https://primeroedgenext.atlassian.net/browse/NXT-16382]) 
**** this should be a textbox where user can only type and save valid dollar amounts
*** Display in correct currency format, i.e. $75.25
*** For Discounts, show the amount as negative but do not require the user to type it negative
**** either show minus sign outside of textbox, or add minus sign when user leaves focus on textbox

No need for ‘Add’ button at this time

Mockup Note: for the Create Receipt (first time), we do not need any of the buttons in the header of the Charges & Discounts section.
