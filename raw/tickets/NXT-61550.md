# NXT-61550 — IM - Item Onboarding- Validations for Associated Column fields (Food Item)

## Description
As a district manager that is working in the item onboarding tool, I do not know what can be and not be removed based on what the system is requiring. Provide proper validations and stopping points so that as a user I am not allowed to remove critical information.

## Acceptance Criteria
* Do not allow food item to be marked as a non food item if serving info has been entered
* Grey out the selections and provide a tool tip hover “Serving Size has been provided”
* If user completely removes the serving size info, then unlock to allow selections to be changed
* Do not allow “Is this a Direct Menu Item?” to be changed if user has entered Menu Item Name as this is a mandatory field
** Grey out the selections and provide a tool tip hover “Menu Item Name has been provided”
** If user completely removes the menu item Name and saves, this will unlock the question to be changed if needed
* If user does not include Menu Item Info section in edit sessions, grey out the “Is this a Direct Menu Item” and select None/Decide Later
** Provide a header tool tip “Menu Item Info Section not selected”
* Do not allow Serving Info to be removed completely if Nutrients, Contribution, Ingredient info, or Menu Item Info have been entered
** User is allowed to change the amount, description and or weight but can not completely remove the serving info for an item
** IF all nutrient, contribution, ingredient, menu item info are removed, user can then be allowed to remove serving info completely
