# NXT-14909 — POS - Default Restrictions - Do not serve if configured

## Description
We need POS to bypass traditional categorical restrictions and from now on consider all items that are not flagged as reimbursable (Is Meal) as restricted if default restrictions are set OR if a student has configured quantity of items.

Confirmed that default restrictions are being logged within Couchbase (Accountability > Settings > Configuration > Default Restrictions).

## Acceptance Criteria
* Do not serve non-reimbursable items over the configured threshold from Default Restrictions (Accountability > Settings > Configuration)
* Do not serve non-reimbursable items over the configured threshold from the account (Account Management > Accounts > Details > Restrictions)
* User should be prompted when selecting the item that puts over the threshold indicating that there is a restriction (should already be in place within POS code from prior version)
