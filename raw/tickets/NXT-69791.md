# NXT-69791 — WS - Insights - KPI - Inventory Turnover Rate- Benchmark Config Screen (PE)

## Description
As a district nutrition manager, I want to be able to configure Inventory Turnover, so that, I can quickly evaluate whether schools are staying within acceptable inventory levels.

*Notes:*

* This story is for the benchmark screen. The Dashboard screen will be done separately.

*Requirements:*

*Grid Updates:*

* Add Inventory Turnover Rate as a column to the grid
** Column name: Turnover (@) [note: need to keep the name short, so don’t do full name]
** Location: After MEQs (Put it last)
** Type and formatting: decimal, only two digits allowed after the decimal (ex: 1.23)
* Ensure the Inventory Turnover Rate is added to the API and dataset

*Years List Updates:*

* Ensure the Inventory Turnover Rate textbox value is updated when changing years
* Ensure the Inventory Turnover Rate actual value is updated when changing years

*AutoFill List Updates:*

* Ensure the Inventory Turnover Rate textbox value is updated when updating using the autofill button

*Bulk Update Updates:*

* Add the Inventory Turnover Rate textbox to the bulk update popup
** Label: Inventory Turnover Rate (@)
** Location: Put it as the last one
** Type and formatting: decimal, same as on the grid
** Ensure type formatting same as on the grid
** Update the Apply Actions button to ensure the Inventory Turnover Rate is updated on the benchmark screen

*View History Updates:*

* Ensure Inventory Turnover Rate is accounted for in the backend API and database for all Add, Edit and Delete actions
* Changes to the Inventory Turnover Rate Value appear in the History Log when viewing it

*Save Changes Updates:*

* Ensure Inventory Turnover Rate is saved or updated in the database when the user clicks save
* Ensure this new value is reflected on the dashboard when closing the configuration

## Acceptance Criteria
h2. 1. Grid Updates

* The benchmark configuration grid includes a new column named *Turnover (@)*.
* The *Turnover (@)* column is positioned:
** After *MEQs*
** As the *last column* in the grid.
* Formatting rules:
** Value is displayed as a decimal number.
** Only *two digits* are allowed after the decimal (example: {{1.23}}).
** Values follow consistent numeric formatting across the grid.
* The Inventory Turnover Rate value is included in:
** Benchmark screen API responses
** Benchmark dataset models

----

h2. 2. Year Selection Updates

* When the user changes the selected *Year*:
** The Turnover textbox value updates for that year.
** The Turnover actual value displayed in the grid updates accordingly.
* Behavior matches the year-switching behavior used for other benchmark KPI fields.

----

h2. 3. Autofill Updates

* When the user clicks the *Autofill* button:
** The Turnover textbox value updates based on autofill rules.
** The grid refreshes immediately to show the updated Turnover value.
* Autofill behavior matches existing benchmark KPI autofill functionality.

----

h2. 4. Bulk Update Popup Updates

* The Bulk Update popup includes a new field:
** Label: Inventory Turnover Rate (@)
** Location: Last field in the popup
** Type: Decimal input
** Formatting: Two decimal places maximum
* Users can enter and update Turnover Rate values in bulk using the same workflow as other benchmark KPI fields.
* Clicking *Apply Actions* updates Turnover Rate values on the benchmark screen immediately.

----

h2. 5. View History Updates

* Inventory Turnover Rate is fully supported in:
** Backend APIs
** Database persistence
** History tracking
* Add, Edit, and Delete actions affecting Turnover Rate generate History log entries.
* History log displays Turnover Rate changes using the same formatting and structure as other KPI changes.

----

h2. 6. Save Changes Updates

* When the user clicks *Save* on the benchmark configuration screen:
** Turnover Rate values are persisted to the database.
** Updates apply correctly per site and year.
* After saving and closing configuration:
** The updated Turnover Rate value is available for display on the dashboard (dashboard story handled separately).

----

h2. 7. Validation Rules

* Turnover Rate values must:
** Be numeric decimals only
** Allow no more than two decimal places
** Reject invalid text or malformed inputs
* If save fails:
** Turnover Rate changes are not partially committed
** User sees the standard benchmark save error message

----

h2. 8. Performance & UX

* Adding the Turnover column does not degrade benchmark grid performance.
* Updates from year switching, autofill, and bulk apply occur within existing response thresholds.
* Column placement and formatting remain consistent with the benchmark grid design.

----

h2. 9. Explicit Out of Scope

* Display of Inventory Turnover Rate on the Dashboard
* Inventory Turnover Rate Details drawer
