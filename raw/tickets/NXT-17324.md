# NXT-17324 — AI Data Insights – Eligibility and Participation - Data requirements

## Description
As a system, AI insights need data to achieve prediction and recommendation intelligence. Data requirement includes both internal and external influence/alternate data points.

+*Eligibility/Certification*+

*a. Identify Influence factors/metrics/standards*
1.	District size
2.	Site size (number of sites)
3.	Geo / area details (rural vs other)
4.	Ethnicity by Geo/area
5.	Age/grade by Geo/area
6.	Affordability (SNAP, TANF, FDPIR, etc)
7.	Household size and Income
8.	Recent changes to affordability 
9.	Recent changes to Household size and income

*b.	Identify key subcategory* 

1.	Student per district
2.	Students per site
3.	Enrolled students per site
4.	Enrolled students per district
5.	Current eligibility for Free/paid/reduced

+*Paid Not Applied (PNA)*+
*a.	Influence factors/metrics/standards*
1.	Paid student affordability
2.	Paid student household size and income
3.	Applied status on file (with application/without application)
4.	Rejected to apply status on file

h4. *b.	key subcategory* 

1.	Paid Student breakfast
2.	Paid Student lunches
3.	Paid Student supper

+*Participation*+
*a.	Influence factors/metrics/standards*
1.	Eligibility
2.	Menu items
3.	Breakfast serving time
4.	Breakfast cancellations
5.	Service offered (indoor/outdoor) – weather evaluation
6.	Bus routes/delays/traffic factors
7.	Number of students registered through bus route
8.	Cancelations of servings due to compliance (time factors)
9.	Participation between schools in each district
10.	Labor sick time effect on participation

*b.	key subcategory* 

# Student breakfast
# Student lunches
# Student supper
# Adult breakfast
# Adult lunches
# Adult supper
# After school snacks
# Number of operating days
# Number of students in attendance
# Number of students with no access
# Non-program food sales

*Requirements:* 
a.	External data requirements (public/private)
1.	Data required from external sources (licensed)
2.	Data available and can be used (public/open)

b.	Custom data requirements (fillers/standards)
1.	Identify MVD data sets and evaluate the gaps for the required data sets

c.	Key KPIs / New Standards / New KPIs identified
1.	ADP
2.	PNA (Paid Not Applied) 
3.	Identify new standards as part of the data discovery
4.	Identify new KPI standards and definitions – based on influence factors * key subcategory

## Acceptance Criteria
1.	Identify MVD data sets and evaluate the gaps for the required data sets
2.	Identify new KPI standards and definitions – based on influence factors in each key subcategory
3.	Create MVDs for each category listed for Eligibility/Certification
4.	Create MVD’s for each category listed for PNA
5.	Create MVD’s for each category listed for Participation
6.	Create MVDs for Influence data/Alternate data
