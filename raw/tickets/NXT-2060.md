# NXT-2060 — Production - Reports - Production Record - Items Planned But Not Produced (UI)

## Description
Menus are assigned to sites, with the intention for those sites to serve the items as they’re assigned. 

Site Managers put their planned (or intended) counts next to each menu item, in advance of the actual serving date for the items. However, sometimes things occur that cause the site managers to NOT serve some or all of the menu items that they had planned/intended to serve… on those occasions, I will put a “Zero” in the “Quantity Produced” field on the production plan.

## Acceptance Criteria
This page should give me the ability to select by Area, Site, Site Group, Meal Type, and To/From dates. This way if I need to pull the report for all of my sites at one time after a snow day, or just a single site after having someone notify me of an issue, or perhaps just needing to check up on a new user in the district.

This report should give me the list of menu items from each date and site that were not produced. It will also give me any comments that were entered next to those menu items, so that if a user has put in a reason or information in regards to why something was not produced, it will appear on this report so that I can review it. 

Note: In the current version of the software, this report is “housed” on a page that has multiple other reports available, as well.
