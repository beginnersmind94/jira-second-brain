# NXT-36653 — Monitoring Scheduler – P6 View Monitor History

## Description
As a SchoolCafé admin, I want to be able to view the history of changes for existing monitoring jobs so that I can proactively monitor the application and ensure higher customer satisfaction.

*Flow:*

* User logs in and browses to Support > Monitoring > Job Scheduler
* User views a list of scheduled monitoring jobs
* User clicks history button on an existing scheduled job

*Notes:*

* This is a continuation of the story to create the Job Scheduler page
* This story assumes that story has been created and that the Monitor tab was created and the grid on that page has the edit button
* This story assumes that the Edit Monitor story has been completed and that users are able to edit existing monitoring jobs already
* Note that each row in the grid already has a history button that does not do anything yet

*Requirements*:

* Requirements for the Job Scheduler > Monitors tab
* Requirements are for what happens when a user clicks the history button on a row in the grid of scheduled jobs
* For the history button, on click do the following:
** Open the Monitor History Page
* 
* Requirements for the Job Scheduler > Monitor History Page
* Copy the page for System Notifications > History button click > Notification History page
* Try to leverage all that you can from that page including the UI/UX
** Make it just like that page
* Page Navigation: System > Support > Monitoring
* Page title: Monitor History
* 
* Have a grid of updates made to the job
* Updates are made when the user saves changes on the Edit Monitor page and saved to a log with the relevant information
* Grid columns:
** Field Name – which field was changed
** New Value – the new value in the changed field
** Old Value – the old value in the changed field
** Updated By – who is doing it
** Updated On – date / time stamp of when make the update
* The grid has filters at the top like on the original page
* There is a close button on the bottom of the page
** On click, return to the Job Scheduler > Monitors tab
* 
* Permissions:
* In General resources screen for Roles & Permissions
** Application: applies Job Scheduler view permission
** Write the security code to apply that permission to this button
