# NXT-11572 — Integration - HTTPS Endpoint for importing

## Description
Perseus will no longer support FTP locations for importing of files.  Instead, an HTTPS endpoint shall be made available for them to use.

*Developer discussion:*

Create a new controller(endpoint) to bypass authentication as BaseController needs authentication.
The payload will contain apim subscription key and regionId along with all the other header information and file content.
Each tenant will have an apim subscription key and it will reside in Primero_System DB(new table).
Validation needs to be made by apim subscription key and regionId in the API.
Metadata of the file will reside in mongodb.
The file will get uploaded to the incoming folder.

## Acceptance Criteria
Customers shall have an HTTPS endpoint available for importing files
This mechanism shall be performant and scalable.
Documentation on the how to use the API(s) shall be made available

Import APIs shall be exposed to the customer
Import APIs shall pass security requirements (authentication, authorization,..)
The scope is limited to file upload relevant to data exchange.
