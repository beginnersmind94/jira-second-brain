# NXT-7026 — Direct Certification - Manual Entry - Preprocessing

## Description
*User Story*

As a district admin I want to see the impact a direct certification file will have on my student population.

*Mockup*

N/A

*Requirements*

This is to handle what needs to happen and what we may need to track after a Manual Entry happens. We’ll likely be doing the same for File Imports as well.

* We need to gather statistics on the current student population as of the file submission date.
** Total number of active students
** How many by each eligibility (free/reduced/paid - keep future price types in mind, do this dynamically by checking the district’s price types)
** How many by each pricetype reason?
* Based on the students added to the manual entry - how do the above stats change?
** What were the Reasons prior - what did they move to?
** Example - Homeless Manual Entry for 5 students.
*** What eligibility were the 5 before? After?
*** What reason were they before? After?
* We need to store this statistical information somewhere. 
** We’re going to display it when the user reviews the file and approves it, and we don’t want these counts to change over time.

## Acceptance Criteria
Statistical information on how the eligibility population changes as a result of this file is tracked.
