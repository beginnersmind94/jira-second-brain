# NXT-26254 — IM - Item Details - Hierarchy - Disable 'OR' option for child items

## Description
As a FSD I don’t want the ability to create child items with ‘OR’ conditions so I do not have confusing item behavior throughout Inventory.

No API work required

*Requirements*

* Modify the Hierarchy section on the Item Details page in Item Management module
* Hide/disable the functionality that allows user to specify ‘OR’ condition for child items
* Assume ‘AND’ condition for all child items
* Hide the entire AND/OR column
** Note: may re-enable in the future
* Update all items to ‘and’ option ([~accountid:5f4fc459347294003eb63e8e] )
