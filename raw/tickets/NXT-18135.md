# NXT-18135 — Platform - Licensing - Audit Trail of Module License Updates

## Description
*Persona*: CyberSoft/ISD Administrator

*Access level:* School District/Site

*Description*:

As a Central office admin, I should be able to view list of products we are enrolled with along with changes made for specific licensed product in the audit history tab. 

*Mockup:*

[https://www.figma.com/proto/PK64OPHjiNeQP1Lm1jz9eu/Summary_Licensing?node-id=3%3A2155&scaling=min-zoom&page-id=0%3A1&starting-point-node-id=3%3A1550|https://www.figma.com/proto/PK64OPHjiNeQP1Lm1jz9eu/Summary_Licensing?node-id=3%3A2155&scaling=min-zoom&page-id=0%3A1&starting-point-node-id=3%3A1550|smart-link] 

*Requirements*:

* Admin logs into PrimeroEdge portal under License Management, each product displayed under License summary page should contain “History” page/drawer when they click either on HISTORY button from the licensing summary page for each product
* History should display all events occurred on the product loads in a page/drawer with pagination 
* Audit table should contain below columns
** Event Name
*** ex: New License has been added, License Type has been changed,  XXX site has been activated, Added license to site XXX
** Old Value
** New Value
** Updated By
** Timestamp
* All events should be recorded with detailed with description of event includes recording OLD & NEW values for UPDATE operations

## Acceptance Criteria
* User shall be able to see list of products with history button is enabled under “Actions” column at the end of the license summary table
* User shall be able to click history button to view the detailed event audit trail
* User shall be able to see the old and new values for the changes made on the product
* User shall be able to see the changes at site level that includes Activated/Deactivated and License addition of the sites shall be recorded in the history tab
* User shall be able to see the list of events sorted by Event timestamp in descending order
