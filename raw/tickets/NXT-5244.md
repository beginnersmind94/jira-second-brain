# NXT-5244 — Direct Certification - Extensions - Search & Review

## Description
*User Story*

As a district admin I want a way to find direct cert (DC) benefits to other household members.

*Mockup*

[https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=172630%3A45&scaling=scale-down-width|https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=172630%3A45&scaling=scale-down-width|smart-link] 

*Requirements*

* Add an Extensions page under Direct Certification.
* Add  to the side-nav also.
* Page should allow the user to Search for  potential extensions.
** These should be tracked by Academic Year (AY).
*** Add an Academic Year selector.
**** Default: Current AY.
**** No All option.
** Add a Search button that will comb the student database to look for potential extensions based off the following methods.
*** Methods to execute:
**** Household
***** Students must be in the same household already in the DB.
**** Address
***** Address must be an exact match for all students.
**** Guardian
***** Guardian name must be the same for all students.
*** Additional search option for the user to enable or disable:
**** Last Name Match Required
***** All students must have the exact same last name.
* Page should show the results of the search in a grid showing relevant information to make an informed decision.
** Eligible Student
*** Full Name (First + Last)
*** ID #
*** Eligibility (Pricetype + Reason)
*** File Number
**** The DC file number the approval was issued from.
** Potential Match Student
*** Full Name (First + Last)
*** ID #
*** Eligibility (Pricetype + Reason)
** Address
*** Show the address for all students involved in the extension.
** Guardian
*** Show the guardian name for all students involved. First + Last.
** Matching Criteria
*** Show what criteria we’re suggesting the match on. Could be 1, could be all 3.
**** Same Household
**** Same Guardian
**** Same Address
* We need to begin tracking history / audit log for all of these.
** First Noticed
*** When was this potential extension first picked up by the system? Full timestamp.
** Actions will be tracked in subsequent story.

## Acceptance Criteria
Extensions page added.
Search methods available, user can execute the search.
Grid shows appropriate results including the matching criteria.
Academic year available so user can review prior years.
