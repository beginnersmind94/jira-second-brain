# NXT-5712 — INV - Create Orders - Order Selections (Step 1)

## Description
*User Story*
As an Inventory user I want to be able to provide my order selections so I can start creating an order.

As part of the Create Order process, user provides Delivery information. This is the first step in the multi-step Create Order process.

On this first step, the selections being made will determine which items are populated in the item list. Only Site and Order Date are required fields

* Add ‘Create Orders’ page to navigation under Orders category

+*Delivery (Site) Information:*+

* Location (site)
** *Required
** destination site
*  Order Date
** *Required
** Default value: current date
** User can back date the Order Date if they have appropriate permission/setting (future story)
*** Use setting to allow backdating (setting already exists)
*** Setting Description: Orders – Number of Days Allowed for Back-Ordering
*** Setting type: numeric entry box to specify number of days
*** (Classic setting code: AOCBBDD)
* Vendor
** *Optional
** Default value: ‘All’ 
** Multi-select (checkbox preferred)
** Include all Vendors with active contracts and with selected site having active item ordering condition on those contract items
* Order Template
** *Optional
** Multi-select (checkbox preferred)
** Only show templates with items added/available for the selected site
* Delivery Date
** *Optional 
** This field is disabled by default
*** Use a toggle/switch to allow user to enable the date picker field
** Single selection, date picker
** Must be greater than or equal to Order Date selection
** When the field is initially enabled, try to calculate the Delivery Date
*** if no vendor is selected, pick the earliest date
*** if one or multiple vendors are selected, use Vendor Lead Time and Delivery Schedule to pick the earliest date

*Apply* button to populate the item list with the selection criteria.
