# NXT-30651 — Import - Allergen for Adults - Platform File Upload to Staging Only (1/3)

## Description
{color:#ff5630}This story is intended to cover the Platform team’s portion of the import.  The Front Office team will have to complete their piece for the import to work end-to-end.{color}

There are 3 steps involved in an import:

# Platform uploads the file to a staging repository
# The module team exposes an API to be notified that step 1 is complete
# The Platform notifies the module mechanism via the API identified in step 2.

This story is for step 1.

{color:#ff5630}UX is not needed.{color}

## Acceptance Criteria
*Step: Pre-Upload*

* The existing “Import Type” named “Allergen Import-Student” shall be re-used for this import.
** The “Import Type” name shall be changed to “Allergen Import”
* A radio button group shall be placed below the import type dropdown and shall be displayed when the user chooses “Allergen Import” (See attachment)
!image-20220527-103121.png|width=240,height=84!
** The radio button group label shall be “File Type”
** Radio button 1 shall be labeled “Student”
** Radio button 2 shall be named “Adult”
** The Student radio button shall be selected by default
* The Student radio button shall be selected by default.
* If the Student radio button is selected, the contents of the import file shall be imported as students
* If the Adult radio button is selected, the contents of the import file shall be imported as adults
* The user shall be prompted to download a template using the standard verbiage
** The “No” radio button shall be set by default, but the user shall not be automatically advanced to the “File Upload” step as a result of this default setting.
** The file template to be used shall be provided by the Front Office team. 

*Step: File Upload*

* No changes from existing behavior

*Step: Field Mapping*

* No changes from existing behavior
* No code mapping required

*Step: Review*

* No changes from existing behavior

*Business Rules and Error Log Rules*

* The “Business Rules” and “Error Log Rules” as stated in the Front Office story shall be the responsibility of the Front Office team
