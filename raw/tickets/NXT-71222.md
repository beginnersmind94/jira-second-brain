## Acceptance Criteria
* Add another sub-page to the Item Configuration page dropdown: Data Sources
* Display the data sources in a grid
** Data Source
** Abbreviation
** Item ID Prefix
** Recipe ID Prefix
** Menu Item ID Prefix
** Data Source Type
* List the existing 3 data sources here: CNDB, Cybersoft, Local
** CNDB & Cybersoft have a ‘Global’ data source type and require no prefix values nor any Action column functionality
** Local is editable only to modify the prefixes 
*** Local also has no abbreviation
* Provide an Action column 
** Edit
*** Open a slide-out with editable fields for the values above, excluding data source type
*** for Local, disable editability of Data Source name and do not show the Abbreviation row at all
** Manage Users
*** Open a slide-out with User, Username, Data Source Admin (Yes/No) and Action
*** Provide a row-level edit mode to modify Data Source Admin setting
*** Provide a Delete icon to remove a user with a confirmation prompt on click
*** Include a history icon to view user history (added, edited)
*** Provide the ability to add new users to the data source
**** UX TBD
** Manage Districts
*** Open a slide-out with District, Status and Action
*** Provide a Delete icon to remove a district with a confirmation prompt
*** Include history
*** Provide the ability to add new districts to the data source
**** UX TBD
** History (data source level)
* Provide the ability for multi-tenant users to add additional data sources
** for now, multi-tenant users may only add ‘Shared’ data sources
** use the same slide-out for Edit Data Source
