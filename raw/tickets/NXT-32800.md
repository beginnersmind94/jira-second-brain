# NXT-32800 — MP- Menu/ Menu Cycle Analysis- Not including 0/null count menu items on the analysis (API ONLY)

## Description
As a menu planner, when I 0 out an menu item or do not enter in a count, that means I am not going to be using this menu item. That item should not be accounted for in the component analysis

Currently, a 0 or not entered counts on a menu item is still considered as an input and we are still considering the menu item as an item that will be used/ served.  It does however, not consider not served items which is ideally what needs to happen to all these scenarios

USDA requires that menu items that are not truly being planned should not be included in the nutrition or component analysis

## Acceptance Criteria
Do not include 0 or null value count menu items as part of the component analysis 

Continue to display the menu items as part of the nutrition summary report in the menu item section

This should be applied to:

*Menu Analysis*

Analysis Slide out

Nutrition Report

Food Component Report

*Menu Cycle Analysis*

Analysis Slide out

Nutrition Report

Food Component Report
