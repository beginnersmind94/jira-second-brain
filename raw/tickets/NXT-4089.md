# NXT-4089 — out of scope Multi-Factor Authentication - Security Questions

## Description
*User Story*
As a PrimeroEdge user, I have a multifactor authentication option where I answer security questions that allows me to verify my identity when logging into PrimeroEdge.

*Requirements*

* There is a system in place that allows for security questions as a multi-factor option
** It works the same as any other multi-factor option in how it is activated
* On first login, if user selects security questions as their multi-factor option, or if it’s their only option, they answer three questions that will be used to validate their identity
** Questions are chosen from a list of pre-defined questions
* When activated, the user is taken to a screen where the questions are shown with text boxes
** User must answer all three correctly before they can proceed
*** Responses are not case sensitive, but otherwise must match exactly
*** Example: If original response to “What is your mother’s birthplace?” is “Houston”, then “houston” is acceptable, but “Huston” is not
** If user gets any of them wrong, an error is generated, all fields are cleared, and the user is able to try again.
*** Error: “One or more of the responses does not match our records. Please try again.”
*** The user can fail three times before being locked out
**** This value is configurable

## Acceptance Criteria
* Security question authentication can be set up and functions properly as detailed in the requirements
