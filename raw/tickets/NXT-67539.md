# NXT-67539 — User Story 4: View Changes (History)  (UI)

## Description
*As a* district business administrator
*I want to* view all changes made to my general ledger configuration
*So that* I can track them

## Acceptance Criteria
"History" tab exists as the last tab in General Ledger Accounts page that tracks all changes across account code structure, categories, accounts, mappings, and system accounts.

*Tab Access:*

* Clicking "History" tab from the General Ledger Accounts page opens this view
* Info banner at top explains: "All changes to your account code structure, categories, accounts, mappings, and system accounts are tracked with timestamps. Click 'View Changes' to see full details of each modification."

*Filters:*

* Filter row at top with dropdowns for:
** Change Type (Account Code Structure, Category, Account, Mapping, System Account, All)
** Action (Created, Updated, Deleted, Imported, All)
** Date Range (date pickers for from/to)
* RESET and APPLY buttons on the right

*History Table:*

* Table displays all changes with columns:
** Change Type (e.g., "Account Code Structure", "Category", "Account", "Mapping", "System Account")
** Action (e.g., "Created", "Updated", "Deleted", "Imported")
** Changed By (user name)
** Change Date (date and time, sortable with arrow indicator)
** View Changes (link or icon button)
* Each column has a filter icon dropdown for quick filtering
* Changes are sorted by Change Date descending (newest first) by default
* Pagination at bottom showing "1 - 5 of 7 items" with page navigation controls
* Items per page dropdown selector

*View Changes Modal:*

* Clicking "View Changes" opens a modal showing:
** For Account Code Structure changes: Read-only view of segment configuration before and after the change
** For Category changes: Before and after values of category code, description, and base type
** For Account changes: Before and after values of account code, description, category, and base type
** For Mapping changes: Which system account was mapped/unmapped to which GL account (full codes and names)
** For System Account changes: Details of what was created or modified
** For Imported changes: List of all items imported in that batch with their details
* Modal includes timestamp and user who made the change
* All views are read-only with no edit controls

*Change Type → Action Mapping:*

*Account Code Structure:*

* Created: New segment added to the structure
* Updated: Existing segment modified (name, length, type, order changed)
* Deleted: Segment removed from the structure

*Category:*

* Created: New account category added manually
* Updated: Category code, description, or base type changed
* Deleted: Category removed from the system
* Imported: Account categories added via bulk import

*Account:*

* Created: New GL account added manually
* Updated: Account code, description, category, or parent account changed
* Deleted: GL account removed from the system
* Imported: GL accounts added via bulk import

*Mapping:*

* Created: System account mapped to a GL account for the first time
* Updated: System account remapped from one GL account to a different GL account
* Deleted: System account unmapped (mapping removed)

*System Account:*

* Created: New system account added to the module
* Updated: System account name changed
* Deleted: System account removed from the module
