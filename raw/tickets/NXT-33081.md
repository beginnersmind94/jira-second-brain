# NXT-33081 — Acct Mgmt - Accounts - Add tabs for Basic and Advanced Search

## Description
As a district admin, I would like the option to be able to quickly view and edit details for *one* particular account whose ID I already know, so that I do not have to spend time wading through a list of filters that are more geared for going through a list of accounts.

*Summary*
Divide the existing Accounts page into 2 tabs: 'Basic' and 'Advanced', with a simple lookup being in the Basic tab and the current Accounts page content being moved to the 'Advanced' tab.

*Details*

* In Account Management → Accounts, split the current page into 2 tabs:
** *Basic*
*** Includes the ‘Search by Name or ID’ field
*** Grid displays when a name is selected from the field only (no Apply button necessary)
**** Grid does not load by default in this tab
*** No custom view required
*** No Export functionality required
** *Advanced*
*** Includes all the filters that currently exist in Accounts, but does not include the ‘Search by Name or ID’ field
**** Consider putting the Insights drop-down in a more prominent location among the list of filters. (*Deferred)*
*** Grid displays when Apply button is clicked
**** Grid also still loads by default on this tab
*** Custom views can load/be managed as normal
*** Export functionality can still be used as normal
* Add pagination (server-side) back to the grid on the Advanced tab. This important for speeding up performance, which has been a common complaint.
* Current role permission still applies (if the user previously had access to the Accounts page, they still have access to both the Basic and Advanced sections).

## Acceptance Criteria
* Verify that the existing Accounts page is now divided into 2 tabs: Basic and Advanced
* Verify that the existing ‘Search by Name or ID’ field is moved solely into the Basic tab
** Verify that using this field immediately causes a grid to populate and allows the user to still preview and view account details (if they have permission, as normal)
*** Columns shown in mockup are fine to show by default. There is no need to allow the user to show/hide additional columns.
** Verify that this grid can have columns added/removed (Custom View allowance is optional, not required)
* Verify that the Advanced tab includes all of the remaining filters and still functions as it does currently, but without the ID lookup field (Custom views are still required here)
* Verify that Advanced tab grids bring back server-side pagination in order to speed up performance
* Verify that current role permissions still work as expected
