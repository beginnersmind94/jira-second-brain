# NXT-36684 — API Error - Error Detail Page

## Description
As a SchoolCafé admin, I want to be able to view the specific details of API errors so that I can proactively monitor the application and ensure higher customer satisfaction.

*Flow:*

* User logs in and browses to Support > Monitoring Dashboard > API Errors
* User views a list of error messages
* User clicks on View error on one row and is redirected to this page
* User views error details on this page

*Notes:*

* This is an almost identical story to the Web Error Detail, so if you can, just copy all that code or even use the same page if you can
* This story assumes the API Error Monitoring page has been created
* This page is the user front end way of viewing a specific API error since the details are too much for a grid to handle

*Requirements*:

* *API Error Monitor Page Requirements*:
* That page already has the button
* Ensure that it links to this page when clicked
* 
* *API Error Detail Page Requirements*
* This is not a “stand alone page” like the others
* Use the fly out from the right like from the Web Error page or the System > Users when the user clicks on the History button
* It overlays the screen, and the user can x out
* We use this in several places on the website
* 
* *Header Section*
** Title: API Error Details
** Navigation: none
* Export
** On clicks, exports page to pdf
* Send
** On click, opens email to send using mailto: link
** Add the contents of the page to the body or as an attachment
** Subject is API Error Detail + the error id
* 
* *Body Section*
* Show these fields in Label / Value format
* Can use simple html if that works
* 
* Fields:
** Note: at the time of this writing, I still don’t know what values we can get from the error log, so this is a tentative list; it is to be reviewed during refinement
* Environment
** Values: Prod, UAT, etc.
* District
** Name of district
* Module
** Show which module the page is in
** Empty if not related to a module
* API
** What API the error occurred in
** Empty if not related to a page
* Error ID
* Error Type
** Ex: Web, API, etc.
* Error Code
* Date / Time
** The moment it occurred
* User
** Account the error occurred to
* Error Details
** The long description of the error with all the details
* Any other thing that adds value
* 
* Permissions:
* In General resources screen for Roles & Permissions
** Visible = false
** Title: Web Error Details
** Application: use the view permissions from Web Error Monitoring page
