# NXT-67677 — Account Category Detection and Sync

## Description
As a system
I need to prevent transactions from posting to orphaned accounts when their category is deleted
So that users maintain proper account categorization for financial reporting

## Acceptance Criteria
* *When a user attempts to delete a category from the Account Categories page:*
** *Side drawer opens* displaying the category to be deleted and a warning if accounts are linked: "This category has [X] accounts assigned to it. Categories cannot be deleted until all accounts are reassigned to another category."
** *Bulk reassignment required*: A dropdown shows all available categories (excluding the one being deleted). User must select a replacement category to bulk reassign all accounts, or click Cancel to manually reassign on the GL Accounts page first. Deletion is blocked if no replacement is selected.
** *Helpful tooltips*: Instructions clarify "Choose one option: Select a category above to bulk reassign all accounts now, then delete • Or click Cancel to manually reassign accounts on the GL Accounts page first" + Note: "If you need to create a new category, click Cancel and use the 'Add Account Category' button above."
* -Accounts previously assigned to the deleted category show a warning icon in the Account Category column with blank/empty value on the GL accounts page-
* -Orphaned accounts appear in a validation warning banner: "⚠️ [X] accounts need category assignment before transactions can post."-
* -Any attempt to post a transaction to an orphaned account is blocked with error message: "Cannot post transaction - account [code] requires category assignment"-
* -Users must reassign orphaned accounts to a valid category before those accounts can accept transactions-
* When a new category is added on the Account Categories page, it immediately appears in the Account Category dropdown on the Add New Account form
* When a new category is added on the Account Categories page, it immediately appears in the Account Category filter dropdown on the GL Accounts page
* The counts on the accounted categories update based on the changes and the deleted category is no longer in the grid
* Account category column values in the GL account page update based on changes made here
