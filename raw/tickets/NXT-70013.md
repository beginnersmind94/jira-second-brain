# NXT-70013 — ACC - Sessions > Add and Edit Transaction > Square changes

## Description
As a site manager, it is possible I may notice a discrepancy in the number of Square transaction in SchoolCafe vs what my Square Website Dashboard shows which is connected to the real money I receive. For accounting purposes, I would like an easy way to manually correct this discrepancy in my session through way of adding Square specific transactions.

## Acceptance Criteria
Within a session on the Add transaction and Edit Transaction slide out Section we will be modifying the Card section for Square Transactions.
Add and Edit does {color:#ff5630}*not*{color} interact with the Square API
The transaction itself can be deleted but the payment type can {color:#ff5630}*not*{color} be changed
The section will Require a card # and a comment for both adding and editing.
For Add Transaction:
Display Only Payment Amount and make it non-modifiable
{color:#36b37e}Tag{color} Add/Edit Square transactions as *_“Manual”_* for easier separation in reporting.

On both the Add and Edit, Add Note for District users that this is for accounting purposes only.
{color:#ff991f}For Editing:{color} “This is a 3rd party credit card transaction. Edits do not affect the card charge and are for SchoolCafé-side accounting only.”

{color:#4c9aff}For Adding: {color}”You are about to add a 3rd party credit card transaction that will affect reporting. This will not charge a card. To initiate a transaction, please use the POS.”
