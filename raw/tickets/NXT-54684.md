# NXT-54684 — PEI - Auth - Enforce SSO in PEI

## Description
We need to ensure that PrimerEdge Insights (PEI) users are logged in via PE and SSO only so that we don’t have unauthorized usage

*Requirements:*

* Ensure that district the user is coming from has logged into PE and is redirected from there
* If not:
** Log them out if they are logged in
** Redirect them to the log out screen
** On the logout screen, show this message:
*** Users are required to authenticate using PrimeroEdge.
* Do not allow them to log in directly
* 
* Get Profile info from the authentication exchange
* Get district info from the authentication exchange
* 
* Remove any code ported over that references using Keycloak authentication

## Acceptance Criteria
* Users must login using PE
* Credentials are sent over from PE and used to authenticate user
* If not properly authenticated, logged out and sent to logout screen
