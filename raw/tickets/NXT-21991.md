# NXT-21991 — Platform - BO - Onboarding Activities Page

## Description
As a district admin, I should be able to setup onboarding activities after logging into SchoolCafe based on the selection of activities

* Once customer completes welcome page selection, list of selected activities are activated in Customer Activities page
* Customer can start setup of each activity which *redirects* to existing pages.
* Other users who are not part of onboarding role for that specific district should not be able to setup any activities until onboarding is complete
* Selection questionnaire for activities that customer are presented with based on district modules licenses
* % of completion calculation - Each activities represents 1 out of total number of activities customer selected.
** Ex: if customer selects 4 activities, each activity completion is calculated as 1/4 which equals to 25%
* Upon completion of each activity, customer shall be able to close which returns user to activities page with the step completion mark
* After completion of each activity customer shall be able to update using EDIT option
* After completing all activities, “Go to Dashboard” is activated which takes them to Onboarding summary dashboard

## Acceptance Criteria
Customer shall be able to:

* See listed activities selected for each modules as part of onboarding process
* Each activity is enabled for setup (sequence will be implemented later in R5)
* “Lets Setup” button associated with activity redirects the user to load respective pages without side & top panes
* Tracker is shown at top banner with business days remaining for target go-live and progress percentage
* % of completion is updated upon each activity completion
* Activities page is loaded when customer returns and logs back into SchoolCafe
* Return to activities page for any updates by using EDIT of an activity 
* Each activity completion is clearly highlighted with EDIT option
* GO TO DASHBOARD option is enabled after completing all activities
* SKIP option is enabled to load customer dashboard at anytime.
* District user who are not part of onboarding role will be able to login only after 100% completion of onboarding activities or activation of users triggers the login (TBD  with team)
