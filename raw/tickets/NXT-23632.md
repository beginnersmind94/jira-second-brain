# NXT-23632 — INV - Vendor - Items - Pricing Condition Rules

## Description
As a Procurement Specialist I need the ability to add pricing conditions per date range so I can configure all the prices my items may have.

*Requirements*

* Validations
** date ranges can never overlap
*** if a new price is inserted that conflicts with an existing price, the existing price must be *automatically* modified to have an End Date the day before the new condition
*** Example 1
**** Current price: 1/1/2020 - 12/31/9999
**** New price: 1/1/2022 - 12/31/2024
**** The previous price should be modified so the End Date is 12/31/2021
*** Example 2
**** Current Pricing conditions present
***** 01/01/2021 - 01/01/2025
***** 01/02/2025 = 12/31/9999
**** New Pricing Conditions : 05/01/2024-01/01/2025
**** This would update the 1st condition to update to 01/01/2021 - 4/30/2024
Because we don’t allow gaps in between the pricing date ranges, there is a significant need for automatic updates.
* Disallow editing or deleting prior dates
** only the current price can be edited or deleted 
** if End Date is in the past, the price is locked
* The Latest Prices as per the Pricing Conditions for the Current/Sysdate should be displayed in the Item Grid under Purchase Price as well as the Contract Item Information slide-out.
* Rename the Purchase Price to Current Price once the item is added to the contract >> in both the Item grid as well as the Contract Item Information slide-out.
* No Pricing conditions can be added outside the contract dates.
