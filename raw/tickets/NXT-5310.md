# NXT-5310 — AcctMgt - Patrons Landing Page - Search Filters

## Description
*User Story*

As a district admin I want a way to filter my searches to save time.

*Mockup*

[https://nqyc9v.axshare.com/start.html#p=acm_landing_page&id=ravqth&g=1&view=default|https://nqyc9v.axshare.com/start.html#p=acm_landing_page&id=ravqth&g=1&view=default]

!image-20200816-165929.png|width=1355,height=378!

*Requirements*

* Add two search filters to the Patrons search area on the landing page.
* Filters available:
** Site Code / Site Name
** Grade
** Defaults for both: blank
** User can type into each filter to filter the available results.
*** -Example: User can type ‘Elementary' into the Site Code / Name field and the results will dynamically shift to only include sites with a name that includes ‘elementary’, or ‘501’ to pull up any site code (or name) with 501 in it.-
** Multi-select is possible on these filters. 
** Selected values should appear as chips that can be removed.
** Grade is populated based off site(s) selected. 
*** If all sites are selected, all grades are selected.
*** If Site A has grades PK-5 and is selected, then the Grade dropdown has PK, KG, 1, 2, 3, 4, 5 available.
* Selected filters impact the search results.
* Selected filters pass through with the search query to the split view.
* Search button remains - user will need to use it or hit enter to execute the search per our current setup.

## Acceptance Criteria
Search filters added to the Patrons area of the landing page.
Multi-select available on the two dropdowns.
Grade behaves appropriately based on selected sites.
Search results are appropriately filtered based on selections.
Filters pass through to the Split View if present.
Multi-language support enabled (LANGUAGENAME_ENGTEXT ie: FR_Students, FR_ADULTS)
Accessibility / ADA Compliance programmed appropriately (full KB nav possible, etc.)
