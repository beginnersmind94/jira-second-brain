# NXT-5238 — Direct Certification - Manual Entry

## Description
*User Story*

As a district admin I want to manually create a list of students that need to be issued direct certification (DC) benefits.

*Requirements*

* Add a ‘NEW IMPORT’ button to the Direct Certification page.
* The button takes users to a page where district admins will select the following options:
** Program
*** This will be one of the following:
**** Assistance Program
**** Other Source Categorical Program
** Type
*** These are tied to the selected Program as outlined in [https://primeroedgenext.atlassian.net/browse/NXT-5235|https://primeroedgenext.atlassian.net/browse/NXT-5235|smart-link] 
*** They’re also based on the list of DC/DA programs that are active per the Eligibility Descriptions page.
*** Use the local descriptions from that page too.
** Approval Date
*** This can be any date from the start of the academic year (7/1/XX) to current date.
** Entry Method
*** -File Import-
**** -This will be a placeholder.-
*** Manual Entry
* For Manual entry, the user will need a way to add students for processing.
** User can select students via the Student Lookup feature.
** User can add multiple students, and remove them from the selected list.
*** There’s no limit here - user can keep adding more and more students if they really want to.
** Show the following information for these students:
*** ID #
*** Name (First + Last + Suffix)
*** Site (Code + Name)
*** Grade
*** Eligibility (Type + Reason)
** Show a count of how many students have been added.
* User can leave a comment.
** Comments can be up to 250 characters.
* User can submit this information for pre-processing or cancel out.
** Options: Cancel, Submit
*** Confirmation prompt if they cancel out.
**** Are you sure you want to exit the import process?
***** Stay Here / Exit Process
*** If they Submit, then we need to take this information in for pre-processing, which is covered in [https://primeroedgenext.atlassian.net/browse/NXT-7026|https://primeroedgenext.atlassian.net/browse/NXT-7026|smart-link] .
**** The user would be taken on to see the Summary of the direct cert file, per the information in [https://primeroedgenext.atlassian.net/browse/NXT-5236|https://primeroedgenext.atlassian.net/browse/NXT-5236|smart-link] 
*** The file needs to be allocated a number.
**** Numbers are unique per district, start at 1. 
***** Format: XXXXX. Example: 00001.
*** We’ll also need to track the following if they process / submit:
**** User
**** Timestamp

## Acceptance Criteria
User can select the program they're uploading for. 
Type selections are based on selected Program.
Local descriptions are used for the program types.
User can manually add a list of students for direct certification preprocessing.
User can add students to the list and delete them.
Lookup feature is used to find students.
Student grid shows the latest information on the student's enrollment and eligibility.
Unique number (per district) allocated for each submitted file. File numbers increment sequentially.
User can also add a comment.
