# NXT-3202 — Reconciliation - Reconcile Info - Transactions

## Description
*User Story*

As a site manager I want to see all the transactions within a session for accuracy.

*Mock-up*

[+https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=1137%3A22760&scaling=scale-down-width+|https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=1137%3A22760&scaling=scale-down-width]

*Requirements*

* Add a Transactions area within the Reconciliation Info / Detailed view.
* Show a list of transactions that are tied to the session - this data may be from an ExpressPoint POS terminal or from other website functionality.
* Display information about the transactions in a grid. Information: Custom headers are selected from this list.
** Transaction Number
** Review Reason (if present)
** Patron ID #
** Name (Lastname, Firstname)
** Grade
** Transaction Type - (Debit/Credit)
** Transaction Date
** Sale Amount
** Payment Amount
** Check / Card # (Visible indicator for whether it’s Check or Card)
* -Add search fields to let users look for particular transactions by Transaction #, Patron ID #, Patron Name, Check / Card #.-
* Add filters to let users view subsets of transactions - by time, transaction type, grade, transaction type, sale amount, payment amount.
* Be able Sort columns
* Headers are customizable.
* Display area expands/contracts based on Headers selected for display
* User has ability to drag and drop columns for sorting order.

## Acceptance Criteria
User has an area where all transactions within a session are listed.
Headers are customizable.
List shows basic information about the transaction.
List is filterable.
Drag and Drop Headers can be used for grouping
Selecting a transaction opens it for viewing or editing.
A button is present to allow a user to add a transaction (later)
The user can select a transaction for editing (later)
Transactions cannot be deleted
