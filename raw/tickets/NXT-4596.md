# NXT-4596 — AcctMgt - Account Info - Add/remove Tags

## Description
*User Story*

As a district admin I want a way to view and set patron tags for an account.

*Requirements*

* In Accounts, show a ‘Tags’ column (hidden by default, but can be shown/hidden via Columns & Views button).
* In Account Info, show any tags that have been applied to a patron account. 
** This applies to Student, Adult, and Special Account types (though Special Account will be deferred until we re-enable the button to view S.A. account info).
** Add a field that displays in the Summary panel (view mode).
** In the Summary-Edit mode:
*** Show the current tags that may already be associated to the Account.
*** Allow user to be able to add additional tags to the patron.
**** While typing, the tag list should filter for matching tags.
*** Allow user be able to add additional tags for the Account Management module. (this may deferred based on how [https://primeroedgenext.atlassian.net/browse/NXT-15509|https://primeroedgenext.atlassian.net/browse/NXT-15509|smart-link] goes)
**** If they’re trying to add a tag that doesn’t exist, give the user the option to add a new tag. (Think of Jira Labels for a similar concept_
**** This may depend on how the Platform team has built the framework. Check with them to confirm.

## Acceptance Criteria
Verify the following for Student, Adult-, and Special Account- Types:

* Accounts page allows user to show/hide the Tags column (hidden by default).
* Account Info Summary view shows the set tags for the account.
* Edit view allows for existing tags to be added to that account.
* Edit view allows for new tags to simultaneously be created within Account Management and added to the account.
** New tags are now visible in the Tag management page.
