# NXT-49534 — MVR - Dashboard - Delete

## Description
As a SchoolCafé user, I would like to be able to delete a managed view report so that I can get rid of old reports.

*Notes:*

* Generally copy the delete process from Custom Reports as much as possible and the requirements included here assume you will do that

*Flow:*

* Open SchoolCafé and browse to Reports > Managed View Reports
* View list of MVRs
* Click Delete on a row
* User confirms their intentions, and the system deletes and records in History table

*Requirements:*

* On the grid in Reports > Managed View Reports, do the following:
* Action column updates:
* Button
** Label = Delete
** If button not there yet, add it
** On click:
** Ask if they are sure want to delete
*** Text: This will delete the distribution configuration for this managed view, but not the underlying managed view it is linked to. Are you sure?
** If confirmed, they want to delete:
*** First check if the have MVR Delete permission in the R&P Crud table
*** If they don’t, show them a message: “Delete permission is required” or something to that effect
*** Send a notification to any users that were in the users or roles lists in the distribution tab:
**** Text: [logged in username] has deleted the [MVR name] managed view report. You will no longer receive any scheduled distributions of this report and it will no longer appear in the list of managed view reports.
*** Unlink the report from the managed view
**** On the Managed View > Saved views, it no longer shows that it is distributed
*** Note: it does not delete the actual managed view, only the distribution report
* 
* Ensure you log the delete in the History log
* 
* Security:
** User must have delete rights to Manage Views Report dashboard to use this drawer

## Acceptance Criteria
* Users are able to delete an MVR
* Affected users are notified
