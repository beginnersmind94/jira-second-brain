# NXT-7629 — INV - Create Perpetual Inventory (back end)

## Description
As an Inventory user I need access to perpetual inventory so the system can track what I have on hand and how the transactions will update those quantities.

We need to create perpetual inventory in the system as a prerequisite to saving any Physical Inventories and affecting Perpetual.

Entire back end structure:

* Site Item
* Site Inventory (current QOH)
* Bin / Batch tables
* other core Inventory tables

Inventory Backend Rules
All transactions that effect inventory will be recorded.
Transactions will be insert only.
If an adjustment is made then we will cancel the original transaction by inserting a negation transaction and then add a new transaction with the correct amount.
Transactions will affect the bin inventory and site inventory.
Bin and site inventory tables totals should always match.

FIFO
IMWH_BinInventory_GetItemBatchesByFIFO
Inputs:
SiteId –Site where the quantity is coming from
ItemId
Whole units
Broken units
DestinationSiteId –used when transferring to check is destination site is a warehouse with bins

Logic:
Check is destination site id using bins. Note: may not have a destination site.
Check if the item can be ordered in broken units. This comes into play when transferring to a warehouse with bins

Search bin inventory for batches with remaining quantity > 0.
The order of the batches should be ascending (use oldest first). The date priority is Expiration Date, Best if Used by, packed, Received Date.

If the destination site is a warehouse using bins and the item is not allowed to be ordered in broken units then we should ignore the broken unit totals of the batches. We will not send the broken units to the warehouse.

If broken units are allowed to be transferred we should still ignore any partial broken units.

Loop through the batches starting with oldest and remove quantity until the desired target quantity is met.

If all of the batches are exhausted and we still need QOH to use then take the additional quantity out of the last batch.

If for some reason there are no batches at all they we will insert a new batch with the required quantity. The price will use the standard price of the item since we don’t have another price available.

FIFO for adding
IMWH_BinInventory_GetItemBatchesByFIFOForAdding
Inputs:
SiteId –Site where the quantity is coming from
ItemId
Whole units
Broken units

Logic::

Search bin inventory for batches with Received - remaining quantity > 0.
The order of the batches should be descending (use newest first). The date priority Received Date.

Loop through the batches above starting with newest and add quantity until the desired target quantity is met.

If all of the batches are exhausted and we still need QOH to use then take the additional quantity out of the last batch.

If for some reason there are no batches at all they we will insert a new batch with the required quantity. The price will use the standard price of the item since we don’t have another price available.
