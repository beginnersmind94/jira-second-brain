## Acceptance Criteria
* Consume servings that have been added either through the Item details or through the Serving info on the onboarding/update tool
* On the *Contributions Review*, display the serving size that already has a contribution associated with it (Update Mode)
** If there has not been any contributions (component and value) associated with a particular serving size, display “--” to indicate no serving size has been selected (Onboarding and Update Mode)
* Once in the “*Update Contributions*” edit mode, allow users to select what serving size will be used to indicate contribution information on the *Components* section, until a serving size selection has been made, grey out the ability to select components
** In onboarding mode, land user with all items that have multiple serving sizes in serving size edit mode
*** Serving size selection and confirmation will enable the ability to select components
** In update mode, all items that have multiple serving sizes will have a edit pencil to select/switch servings 
*** Again, if there has not been any contributions (component and value) associated with a particular serving size, display “--” to indicate no serving size has been selected
* Serving sizes flow through the rest of the “Update contributions” sections and can be confirmed or changed on the *Component* or *Values* section
* If user selects serving size and selects a component but does not add a value, when returning to the review page, clear the serving size selection and do not hold that selection as it is invalid (Current Item Details behavior)
* Space columns to handle the max amount of characters in all sections including review
** On Review mode, display veg components in their abbreviated format to better help with space 
** Move Missing label to resemble values labels (reference GBD in mockup)
** Set column width to minimum (for columns like Fruit, Milk)
