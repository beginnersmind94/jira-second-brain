# NXT-29591 — Notification Scheduler refactor

## Description
As a system notification scheduler is responsible for fetching all the notification schedules that needs to be notified. Currently the system is pulling all the active scheduled from the db and executing them if they’ve met the criteria but doing so would result in a job that will be fetching active schedule and publish it over an over, every time the job is triggered. To fix we need to make the cron job fetch the active schedules for 24 hours and store it in memory and then should run every 30 minutes and publish out the messages for the notification schedule that has met their criteria. Once the messages are published, those notification schedules should be removed from the memory reducing the list and removing the chance of sending the same message multiple time.

## Acceptance Criteria
When notification scheduler is running and able to publish messages in dev and qa environment.
