# NXT-24028 — IM-Convert Food Item to Non-Food Item

## Description
As a AUTHORIZED Nutrition Dept. user I want to be able to convert an Food-Item to a Non-Food Item  so in the event a mistake i wont have to re-enter all information again

!image-20211221-141456.png|width=981,height=684!

## Acceptance Criteria
*Entry Point:*

* Item Split / List  view → Item Detail card → Edit

*Actions:*

When on Edit on Item Detail Card (Split or List view) 

Switch Item Type  from Non-Food to Food  will trigger the system to validate if the item name is already used if so notify the user with a message “Item Already Exist” and return to previous screen, otherwise present the user with a  pop-up window asking to confirming the action. The pop up windows should display the message “ Are you sure you want to convert this Food Item to Non-Food Item?
if  the user  confirms (Convert) the system should proceed  with the conversion process. 

Conversion process

* The system should copy the Item Detail information
from the food-item and automatically create an Non-Food Item
* User should be allow to enter a new item name 
* Fields to copy over from Food Item to Non-food Item
** Item Name
** Item Form
** Data source
** GTIN
** Product Code
** Manufacturer Name
** Brand Name
** Tags
** Storage Category
** Unit Description
** Fair Market Value / Standard Price
** Item Category
** Valuation Group
** Unit Weight
* Status of Item/Recipe from which the new item was created should be set
to Inactive

If the user responds No (Discard)  to the question then pop-up window is closed and request / action is canceled and the screen returns to Item Detail page.

*Note:*

Do not add the convert options to single item recipes.

*General:*

* Validation rules and display information should be the same as when creating a Non-Food Item
* If Item is used in a dual-recipe (Menu Items) the conversion process is not allow. 
* Conversion from Non-Food to food is not allow
