# NXT-40776 — IM - Allow Pack Size adjustments until Item is Inventory Locked

## Description
As a District Administrator I want to be able to adjust pack sizes as long as my items are not locked according to Inventory rules, so I can make any necessary adjustments to hierarchy or sub-quantities.



!image-20230817-144127.png|width=100%!

## Acceptance Criteria
* Consider Inventory usage flag (isUsedinInventory) to determine item pack size editability 
** Manage this flag’s value according to existing Inventory business rules (item on a contract, item ordering conditions, etc.)
** Reverse flag if item is removed from Inventory without ever being used in a transaction
* *Allow Pack Sizes to be adjusted as long as item is not used in Inventory*
** *re-enable edit ability in the event the item is added to ordering conditions and then removed without usage (for example)* 
** Not working JM 9-20
* Show appropriate status and message on the Inventory Readiness banner communicating inventory data lock
** Show the correct ‘Publish’ toggle value
* Lock the pack size permanently once item is used in Inventory transactions (isUsedinTransactions)
* When an item is added to Inventory *Contract* or *Ordering Conditions*, it is considered _Inventory Synced_
** Pack sizes can be added, +Sub-Quantities can be edited+
** the +specific unit+ added to Inventory *cannot be deleted*
* When an item is on an Inventory Transaction, it is considered _Inventory Locked_
** Pack sizes *cannot* be added, deleted, etc.
** Sub-Qty cannot be edited
