# NXT-7109 — Direct Certification - Revert Import

## Description
*User Story*

As a district admin I make mistakes, and I definitely need a way to roll back a direct cert (DC) import so I can pretend it never happened.

*Mockup*

[https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=47124%3A0&scaling=scale-down-width|https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=47124%3A0&scaling=scale-down-width|smart-link] 

*Requirements*

* Users need an option on an Processed DC file to Revert / Delete the file.
** This would be a soft delete. We would need to track what happened in there.
* This should only be possible in reverse order - the latest file entered can be reversed. Then the one before that.
** If 10 files have been processed, the Revert option is only available on #10.
** If 10 is reverted, then the option is available on #9, then #8, etc.
* Confirmation prompt if the user initiates this action.
** ‘This will change the eligibility for all students involved in this file. Are you sure you want to proceed?’
*** Options: Cancel / Revert
*** There should also be a mandatory comment field, 500 character limit.
*** Revert needs to successfully roll back the eligibility of the students as if the file had never been imported.
* This Revert process should also be available for an individual student inside of the DC file.
** They will also need the same confirmation prompt for this.

## Acceptance Criteria
Revert/Delete button available for the entire file, and for individual students matched on the file.
Eligibility rolls back appropriately for all students involved with the file.
