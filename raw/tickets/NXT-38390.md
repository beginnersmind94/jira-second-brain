# NXT-38390 — POS - Device Settings - Low Spec

## Description
As Dora the Director I don’t want to have every school go to the latest POS build every time Cybersoft has a new version. I’d like to test it first at 1 terminal at 1 site to avoid the potential for a district-wide issue. This will save substantial aggravation for all parties involved.

The idea here is to provide a way to upgrade a single terminal on Production if we’re at all concerned about performance after an update. 

* We can update 1 machine, and have it be carefully used + monitored to be sure there’s no side effects or issues.

*requirements*

At POS

* Add an option to select a different POS build: 
** Location: Device Info > Admin
** Option: Select Build
*** Options: Stable (default), Experimental, Low Spec
* If the user changes the build and confirms, relaunch POS and download the appropriate build.
* Data needs to stream as normal to the server

Server

* Set up the alternate build paths so there’s something to download with each build option.

Build explanations:

* Stable will be, well, our main branch.
* Low Spec will be a cut off point for slow machines to use, like those seen at CPS.
* Experimental will be used to verify our updates with a handful of machines or to try out new features at selected lines only.

## Acceptance Criteria
POS has an option to take an experimental build, and updates appropriately.

User can change POS back to stable build.

Builds are setup - we can start pushing experimental builds out and update a single machine.
