# NXT-40310 — eOrders Configuration – History Log

## Description
We need to create a history table to track changes to the eOrder Configurations so that end users can monitor changes over time to their eOrders.

*Notes:*

* No front end on this story, this is the backend story
* This table will be updated when a user does one of the following: Adds, edits or deletes
* This table will be read from on the View History drawer

*Requirements*:

* Create a History table for eOrders
* 
* At a minimum, the table needs the following columns:
* eOrderConfigID
** linked to eOrders table unique ID so that you know which config these changes are for
* Event Type
** This is which action caused the record to be inserted
** Values:
** Add
** Edit
** Delete
* Description
** Value:
** The text of the values from the Add and Edit are coded like this:
** Format: FieldLabel=Value; FieldLabel=Value; FieldLabel=Value
** 
** Text for Add or Edit:
** Example text:
** Vendor=Gordons; FileFormat=EDI 850; etc.
** Note: this can get long if they add a bunch of email addresses and users, so make sure table column can handle a lot of text
** 
** The text of the Delete can be empty
* User
** The user account logged in that made the change
* Date/Time
** The date / time stamp of the change
*   
* *APIs Needed*
* At a minimum, you will need the following APIs / Stored Procedures:
* Receive the data from Add New, Edit and Delete eOrder Configuration
* Don’t need an update for this table
* Provide all the rows with all the columns for that given config for a given eOrderConfigID
** Used on View History drawer
