# NXT-45073 — Notifications - Toast Messages

## Description
Note: JAS 07/23/24 - Team could not do preferences part b/c of dependencies on another feature, so striking out what they can’t do and cloning story for part 2



As a SchoolCafé user, I would like to receive banner notifications when there are new messages so that I can know immediately if there is a new message.

*Notes*

* We want to show the small message banner boxes like Ring Central or Outlook do when there is a new notification

*Flow*

* User logs in and browses to any page
* System or someone sends the user a message
* System gets user preferences and sees user allows toast notifications
* User sees a message toast banner slide onto their screen for a few seconds
* The banner disappears or the user closes it

*Requirements*

* Add to the Notification framework the following:
* When the user receives a new message, be able to show a toast banner message box
* Banner should show on any page in the website
* Be able to optionally play a ding sound when it opens
* Banner should close in x seconds automatically
* Banner should have a close button
* Banner should show only if the user is logged in
* Banner should show:
** Title
** Body
** values set by code that sends it
* 
* -User should be able to use the Preferences tab to control-
** -Show banner yes or no-
** -Play sound yes or no-
** -Location of the banner-
** -Duration of banner in seconds-
** -Therefore, use these values when showing the banner-
* Logic:
** There is a new message for a user
** Check to see if they are logged in currently
*** If no, do nothing
*** If yes, continue
** -Check their preferences and see if they want to see banners-
*** -If no, do nothing-
*** -If yes, continue-
** Prepare the banner message to show adding title and body
** -Check if they want an audio message-
*** -If yes, play it when you show the banner-
** -Check the location of where they want to show it-
** -Show the banner in the location they specify and play sound if they choose yes-
** -Wait the seconds they specify and close it-
** Optionally close it if they click the close button
* 
* -If these are missing from the preference tab, then do these things first:-
* -Add the following questions to the Notification section if not already there-
** -Show banner for new messages-
*** -Enable/Disable Toggle-
*** -Default Enable-
*** -If the user selects yes, show the banner when there is a new inbox message-
** -Audio ping with banners-
*** -Enable/Disable Toggle-
*** -Default Enable-
*** -If the user selects yes, play a ding sound when show the banner-
** -Banner location-
*** -Single select list-
*** -Values: Upper Left, Upper Right, Lower Left, Lower Right-
*** -Use these values for the location of where to show the banner-
** -Banner visibility in seconds-
*** -Textbox that scrolls numbers-
*** -Default value: 5-
*** -Numbers only-
*** -Value range: 3 - 8-
*** -Use this value for how long to display the banner- 
*

## Acceptance Criteria
* -Banner uses values from preferences-
* -When the user receives a new message while logged in, site shows a banner message box if enabled-
* Banner should show on any page in the website
* -Be able to play a ding sound when it opens based upon user settings-
* Banner should close in specified seconds automatically
* Banner should have a close button
* Banner shows Title and body
