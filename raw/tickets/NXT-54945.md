# NXT-54945 — OpenID - District Authentication - Notify - Expired Passwords

## Description
As a Customer Support user, I want to be able to setup a district’s OpenID authentication provider so that they can use SSO

*Notes*:

* This story is covering the non completed functionality from its parent story regarding sending expired password emails for SchoolCafe Authentication

*Requirements*:

* These requirements apply to the System > Security page
* 
* On the grid in the tab
** On Preferred click:
*** If the user clicks to make the SchoolCafé the preferred method:
**** Run the mass password reset process described below
** OpenID deletion:
*** If they delete on an OpenID row:
**** If their new preferred method is SchoolCafé Auth
***** Run the mass password reset process described below



*Mass Reset*

* -Check all accounts with no password-
** -For each one, follow the process to set password for new accounts which includes sending them emails etc.-
* Then check for accounts with expired passwords
** For each one, send them the expired password emails and follow that process



*Update to Expired Password scheduled job*

* For our current scheduled job that checks for expired passwords and sends an email:
** Update the logic so that it doesn’t send a expiring password or reset password email when the user is doing SSO

## Acceptance Criteria
* -Email sent to users when SchoolCafé is enabled and they don’t have a password-
* -Email sent to users when SchoolCafé is enabled and they don’t have an expired password-
* -Email sent to users when SSO is deleted and SchoolCafé is preferred method and they don’t have a password-
* Email sent to users when SSO is deleted and SchoolCafé is preferred method and their password is expired
* System doesn’t send a expiring password or reset password email when the user is doing SSO
