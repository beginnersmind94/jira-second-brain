# NXT-20856 — DataQuery Framework - UI DataQuery Selector Component

## Description
End-Users would like the ability to recall previously-defined DataQueries and have the columns and their positions, the specified filters, and the sort order of their data replayed onto the UI.  This story would result in a UI component that would allow the end-users to select a DataQuery from a list of existing DataQueries.

* They can click on Custom user View button and a drawer should open with all the DataQueries saved by the user.
* Radio button box should be there in front of each DataQueries to select the default view for user.
* Delete button should be there to delete the unwanted Dataqueries.
* Download button should be there to download the save data.
* Edit button should be there to edit the DataQueries

## Acceptance Criteria
* Module development teams shall have a UI component to use on their DataQuery web pages that will allow an end-user to choose which DataQuery to execute and display.
** The user-defined DataQueries shall be sorted alphanumerically
* The component shall include the ability for an end-user to identify a DataQuery as their “Default” which would be used the next time the end-user returns to and loads the page.
** The “Default” shall have a UX treatment that indicates it is default.
* The component shall offer the end-user the ability to select the “Standard” DataQuery.
** The “Standard” option will have a separate UX treatment from user-defined DataQueries
** “Standard” DataQueries will be those that are defined at implementation time
** Modules shall be responsible for defining the “Standard” DataQuery
* UX treatment for this component shall be decided by the UX team
* Delete button should be there to delete the unwanted DataQueries.
* Download button should be there to download the save data.
* Edit button should be there to edit the created DataQueries
