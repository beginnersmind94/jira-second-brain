# NXT-48715 — OpenID - Edit User Settings

## Description
As a SchoolCafe admin, I would like to be able to change a password in Edit User only if they are using SchoolCafe Authentication so that I don’t get confused trying to set a password that will never be used.

*Notes*:

* Authentication Type is chosen by a user in their profile page but can also be set here by an admin
* A user can only choose the type of authentication available to them by the district

*Requirements*:

* On the System > Users > Edit User drawer, do the following:
* When in View Mode:
* 
* In security section:
** Show list of Authentication methods the user has access to
*** Put an asterisk next to the default one
** If the user selects authentication type SchoolCafe Authentication:
*** Show the Password row like you do now
*** Show the Password expiration date
*** Show the Password reset message like normally do in this section
** If the user’s authentication type is not SchoolCafe Authentication:
*** Hide the password row with the password reset, since it is not needed
* 
* When in Edit Mode:
* Under the Security section:
* Fields:
* Add the Grid
** Copy from the Profile grid if you have done that one already
** Title: Authentication Methods
** Shows a row for each of the methods chosen by the district on the Security tab
*** Note shows same rows as Profile grid
*** Note: not all the methods, just the ones enabled by the district
** No paging, sorting, filtering
** Columns:
*** Enabled
**** Checkbox
**** Default to what the user selected
*** Preferred
**** Button column
**** Do like the Star that we do for Managed Views
**** User can select only one
**** If select one, de-select the others
**** Default to what the user selected
*** Configured
**** Show value from row in Profile
*** Method
**** Show value from row in Profile
*** Actions:
**** Note: there is no edit or delete for this grid
**** 2FA actions covered in that epic
**** SSO methods don’t need user input, so no actions
**** If it is a SchoolCafe method row:
***** Show the Reset Password button
* 
* On Save click
** Ensure the validators work:
*** Must have at least one row marked as preferred and at least one marked as enabled
* 
* Changes are recorded in History log
* 
* If not already done, add the section collapse icons to the 3 sections on the page

## Acceptance Criteria
* Authentication Type field shows the type selected by the user
* User is able to change the password type to any and only ones chosen by the district
* If the user chooses SchoolCafe Auth, they must set a password
* The login screen uses this type of authentication selected by the user when they try to login
* Changes are recorded in History log
