# NXT-28323 — Production - Reports - Pre-Production Costing (Summary only by Site Group)

## Description
The Pre Production Costing Report gives the district information regarding the cost of foods in relation to the completed Production Plans from their sites. It looks at the quantity of ingredients calculated to be used to generate the servings, and pulls the costing from the Fair Market Value listed with the products associated to the ingredients.

Example:  If my Production Plan states that I plan to serve 23 apples, then the software would calculate the Fair Market Value of those apples, divided by how many broken units contained in the whole unit, and multiply that broken unit cost by those 23 apples.

The district should be able to pull this report with the following details:
Site Group: Default to All

Meal Type: Default to All

From Date

To Date

Report Options (First Criteria) : Details by Site, Details by Site Group, Summary Only by Site Group

## Acceptance Criteria
Display the date range for the report as well as the type of report it is (Summary only by Site Group)

These are the follow columns that need to display on the report 

* Site Group
* Meal Type
* Serving Date
* Menu Line
* Total Meal Count
* Purchase Cost
* Average Cost

When calculating Meal Count (Planned Meal Count), Combine all the meal counts for all plans based on site group and menu line for each day. 

When Calculating Purchase Cost:

Unit Cost (Cost Per serving) X Number of planned counts per item

If there are multiple serving sizes, multiple each unit cost with the appropriate planned counts per item associated with that serving size.

Combine all the purchase costs per item and add them all up to give you the Purchase Cost on this report. 

When calculating Average Cost:

Purchase Cost / Total Meal Count



Display one site group at a time for the entire criteria that is selected as shown on the report.

Only consider plan and record complete status. 

Any incomplete Plans will not be included in the report. We will need to give a message on the top of the report to indicate how many incomplete records there are. Message should be as this "There are (# of incomplete plans) incomplete records that were omitted from this report. Complete all plans to remove this warning."
