# NXT-16836 — Income Surveys - Range Configuration

## Description
As a district manager that needs for the income survey to use district input salary ranges I need to be able to input the ranges.

*Requirements*

Some configuration options are available for districts using Income Surveys. This will include any system settings that need to be added, and an edit to the Income Guidelines page depending on that setting.

* In Eligibility > Settings> Eligibility Guidelines, add a new tab to the page called ‘Survey Ranges'
* Show the current value of the *Income Range Type* Setting, +with the information on the page changing+ +*based on the setting value*+.
** *If* the setting is set to '_USDA Income Ranges'_ then show the following text:
*** “Income Surveys are using the USDA Income Ranges from the standard Income Eligibility Guidelines.” 
*** Nothing else needs to display on the tab other than this text.
** *If* the setting is set to '_Total Household Income_,' show the following text:
*** “Income Surveys are using the Total Household Income field and will ask for an exact amount.”
*** Nothing else needs to display on the tab other than this text.
** *If* the setting is set to ‘_District Income Ranges’_ then show a grid with frequencies for columns. Grid can look exactly like existing Eligibility Guidelines.
*** Default data for this can be a copy of that year’s Income Eligibility Guidelines.
**** If something already exists for that year’s custom range values, do not overwrite them.
**** If nothing exists, copy over the IEGs so the user has something to start with. 
** Grid has the typical frequencies available. (Yearly, Monthly, Twice a Month, Every 2 Weeks, Weekly).
** User can edit values and add new rows.

## Acceptance Criteria
Verify that the Eligibility Guidelines page is updated to have the Income Survey tab with editable functionality as described above.

Verify that the tab changes based on the system setting.
