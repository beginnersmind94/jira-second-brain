# NXT-33791 — MP- Menu Cycle- Lock access to multiple users when viewing or editing

## Description
As a menu planner, I would like a way to either be notified that changes have been made on the edit menu page or on the menu planner page so that I do not end up either overriding somebody's work or waste time doing work somebody else already did or locking down menus to only be accessed by a single user at a time. It also would prevent from multiple changes from happening at the same time.

## Acceptance Criteria
* Allow multiple users to view the menu cycle at the same time
* Allow users to view the menu cycle while another user is editing at the same time 
* DO NOT ALLOW multiple users to edit the menu cycle at the same time 
* As a user clicks on the edit and is on the editing view of the menu cycle from the standard edit or menu planner edit, that user has now essentially “checked out” the menu cycle
** Disable the edit option for ALL menus in the menu cycle from being available to other users while User 1 is editing menu cycle
*** Display a tool tip hover on the disabled edit pencil “*This menu has been checked out with Menu Cycle: (Name of Menu Cycle) by (Name of the User)*”
**** This applies either from the menu landing page or from the edit pencil on the standard menu cycle page
**** If user is already on the landing page with access to edit, and user 2 goes and starts editing a menu cycle, do another check on user 1 entry to the edit menu cycle page, display Warning message: “*This menu cycle has been checked out by (Name of the User).* *Please access this menu cycle in view mode*.”
*** Users can still view the menu cycle
* If user is accessing Menu Cycle and one of the menus in the menu cycle is already been checked out, allow view only mode for that menu either from the standard menu cycle page or the menu planner
** Display tool tip on the view only menu “*This menu has been checked out by (Name of the User). You can not edit this menu”*
* On production plan and record, there is a link to access the menu, default to allow the user to access the menu in edit mode, this will check out the menu and give no access to other users in edit mode.
*  However if the menu is checked out either in the menu cycle grouping or on its own, display the menu in view mode and display banner at the top of the screen “*This menu has been checked out by (Name of the User). You are now in view only mode*”
** For Menu cycle check out: “*This menu has been checked out with Menu Cycle: (Name of Menu Cycle) by (Name of the User). You are now in view only mode”*
** If the user2 is still on the menu once the user 1 “checks in”, they can refresh the page and it will put them into edit mode now that no user has the menu checked out
*** This only applies to production plan and record menu link
* User 1 “checks in” menu cycle back in along with all the menus within it either being logged/timed out or if they save and continue and gets redirected to the menu cycle landing page (Menu Cycle Search Page) by the menu planner mode
* User 1 checks in” menu back in either being logged/timed out or leaving the standard edit menu cycle page
** Any user will now be allowed to edit the menu cycle and any menus associated with it and the above process happens again
