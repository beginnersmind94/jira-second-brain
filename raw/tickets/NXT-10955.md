# NXT-10955 — Applications - Sessions - Sessions List

## Description
As a district admin I want to review the work my employees have done regarding application processing.

*mockup*

[https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=421061%3A834&scaling=scale-down-width|https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=421061%3A834&scaling=scale-down-width|smart-link] 

*requirements*

* User has a page where they can see all application processing sessions.
* User can filter by academic year.
* Grid shows the following:
** Processor
** Start Date
** End Date
** # Apps Processed
* Grid can be grouped/filtered as needed.
* User can click into any session and be taken to the Session Summary.



*New reqs after discussion 2/24/21*

* Academic year filter has no ALL option
** Default is current academic year
* User needs date range search fields
** Default is today’s date
** Dates cannot exceed the selected academic year

Place the new page under the Forms menu as a sub-menu item called Sessions.

Applications should also be under Forms if we can do that in here also.

## Acceptance Criteria
User has a page where sessions can be tracked.
User can find sessions by academic year.
User can search for sessions and see details per the requirements.
User can click into sessions to see the summary.
