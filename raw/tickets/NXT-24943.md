# NXT-24943 — PR- Prep site set for mutliple sites

## Description
As a user, I want to have the flexibility to have a prep site prep food for multiple schools in the district.

I would like to have a configuration page to select my prep site. 

*Navigation:*

Menu Planning> Site Groups configuration> Prep Site Configuration (New Tab)

## Acceptance Criteria
* Display all sites with the option for user to select their prep site per site
* Default the prep site per site to the site name
** EX: Site Name: Hundred high School  Prep site : Hundred High School
* Allow user to be able to configure multiple sites to a prep site at the same time (similar to site group configuration) (API)
* A (receiving site) can only be on one prep site list. You can not have multiple prep sites for 1 site. 
* Any prep site will have to have itself as a receiving site as well. A prep site can not prep for ONLY other sites. They must prep for themselves as well (They must have the option but they do not have to be forced to actually prepare food for themselves) 
* If Prep site was already configured for 1 or more sites and new site is added, effective date only applies for the new site/sites added
* Only effect production plans and record from the effective date and moving forward. (API) (For future use Cron Job)
** Display effective date on the site configuration tab on each individual site row under *Column: Prep Site Effective Date*
* Display history on each site to display changes done for prep site including effective date, site group changes, receiving site changes (API)
* Any changes done on these configuration pages should reflect on the individual production plan and record. Effect only production plans that are not complete. (These changes will be handled story ([NXT-30931|https://primeroedgenext.atlassian.net/browse/NXT-30931])
** User may still go back to the individual production plan and change the prep site and save. This will only effect that particular production plan and record. 
*** EX: If user manually changes production plan prep site but does not change the status to plan complete and is still in plan incomplete, if the configuration is changed again, it will effect that production plan and record.
