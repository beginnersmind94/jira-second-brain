# NXT-6837 — Favorites – Create Framework

## Description
We need to create a framework to save pages as favorites so that end users can quickly browse to the pages they want.

*Notes:*

* This is a backend only story
* Need to do first so front-end stories can be completed

 

*Requirements*:

* Create a backend structure in the database and supporting APIs that will allow users to save pages as favorites in SchoolCafé
* 
* At a minimum, the database needs to be able to save:
** User ID
** Name of the link
** URL
** Display order
* 
* At a minimum, the following needs to be exposed to the Front End
* FE needs to be a way to get a list URLs and names by user id and sorted by display order
* FE needs a way to save a single favorite (User ID, name and URL) to the list
* FE needs a way to delete a single favorite from the list
* FE needs a way to update the name of a favorite from the list
* FE needs a way to update the order of the favorites per User ID
* 
* Any page with a URL should be to stored
* We do not want to store drawers and overlays or any action (add/edit/delete) screen

## Acceptance Criteria
* Database exists
* Front End team can use the framework to save/edit/delete favorites
