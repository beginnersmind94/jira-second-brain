# NXT-49389 — OTP - District - Delete

## Description
As a Customer Support user, I want to be able to delete a district’s OpenID authentication provider so that users don’t try to use it

*Notes*:

* Covers delete for all One-Time Passcode methods

*Flow:*

* User browses to System > Security page
* User chooses which auth types they want to delete
* System deletes the method if it validates

*Requirements*:

* These requirements apply to the System > Security page > Authentication tab
* On the grid:
* Continue to enforce the one row rule done in the OpenID delete story
* Continue to enforce the preferred method rule done in the OpenID delete story
* 
* Action Column update:
* SMS One-Time Passcode rule:
** SMS One-Time Passcode cannot be the only preferred method, so
** User cannot delete an authentication method if it would leave the SMS One-Time Passcode as the only remaining method
** First, validate that they have more than one row
** 2nd, validate that the SMS One-Time Passcode is not the only remaining row
*** If it is, then make them select another preferred method and don’t allow them to proceed, by showing this message: Deleting this method would leave SMS One-Time Passcode as the only remaining method which is not allowed. Please add another method before deleting. 
*** Give them an OK button to click on to close the message and do not delete.
* 
* If deletion is allowed:
* If they click delete on a One-Time Passcode row:
** Show message: This will disable all users' ability to login using [method name] and they will only be able to login via the other methods you have selected. Are you sure?
** If they confirm:
*** For each user account:
**** Check preferred method
**** If this One-Time Passcode method:
***** Then they are affected, so
***** Change their preferred method to the district’s preferred method
**** If anything else, then don’t change their preferred method
*** For each user account:
**** Remove this method from their list of authentication methods
**** Send a notification to their workspace inbox and their email address:
***** Your district administrator has removed [method name] as a permitted login type so users are no longer be able to login using this method.

The permitted authentication methods by your district are: [show list like: Single Sign On, Email One-Time Passcode, etc.].

Your default authentication method is: [name].

To update your authentication settings, go to the Authentication tab in your user profile.
*** 
*** Log the change in the history log
*** Remove the row from the grid
** If they don’t confirm, then do nothing
**

## Acceptance Criteria
* User is able to delete an auth type if there is more than one
* User cannot delete the Preferred auth type
* If user deletes a One-Time Passcode method, then it updates all users with that auth type to use the new preferred auth type
* If user deletes One-Time Passcode method, then system notifies impacted users of the change
* Deletion is logged in history log
