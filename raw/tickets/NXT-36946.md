# NXT-36946 — PR-Plan/Record- Variety "Child" Menu Item Serving Size replacing individual menu item (API ONLY)

## Description
As a production manager, I have the ability to add variety menu items to my production plan/record. In the case that a “child” variety Menu item has a different serving size than the individual menu item that is already on the plan or record, the system should use the “Child” serving size.

## Acceptance Criteria
* IF individual menu item is already on the plan/record, and then user adds a variety menu item that includes that menu item, we will be replacing the menu item with the child item and toggle the child item on [https://primeroedgenext.atlassian.net/browse/NXT-39056|https://primeroedgenext.atlassian.net/browse/NXT-39056|smart-link] 
* When replacing the individual menu item with the “child” menu item, use the “child Menu Item serving size if it happens to be different than the individual menu item serving size
* In the case of a individual menu item that is already on the plan or record having a carryover, continue using the serving size that the carryover has
* Effect both the *plan* and *record* page
