# NXT-25734 — AcctMgt - Groups - Rosters -  Allow user to view and assign/remove tags

## Description
As a district admin, I would like to be able to easily view and categorize my Rosters, so that I can serve them accordingly.

*Summary*

* In Rosters (just like with Homerooms), add a Tags filter and column which the user can use to search for specific Rosters. Also, allow them to edit a roster’s tags.

*Details*

* In Groups → Rosters, add a Tags drop-down menu to the list of filters.
** Tags menu should be *multi-select*
** In this list, pull up all active Tags that have been created for this particular district (+*_if_*+ we have implemented active/inactive status for Tags. Otherwise just show all Tags for the district)
* When a user selects one or more Tags and then clicks ‘Apply’, only display those Rosters in the grid which have been assigned those Tags.
* In the Grid, add a Tags column which shows those assigned tags for each Roster.
** If the number of Tags is too wide for the standard column width, display the overflow as ‘+1','+2','+3', etc. 
*** When hovering over that content, display all Tags for the Roster in a floating tooltip.
* When a user clicks the edit button in the grid, open the Edit Roster control as a drawer instead of as a pop-up.
** Within the drawer, add the Tags field. (other fields remain unchanged)
** The user should be able to use this field to assign/remove Tags and then save the details for this individual roster.
* Other UI/UX changes to note:
** Filter design should be changed to an expansion panel with the title: ‘Filters’ (expanded by default).
** Add a Reset button, which when selected, returns all of the filters to their default selections (if we have to also clear the grid results, that’s fine).
** Change the ‘Find Roster(s)’ button label to ‘Apply’. Behavior remains unchanged.
** In the grid, change the edit icon from ‘Edit’ to ‘Edit2’
** Replace the Edit Roster buttons ‘Update’ and ‘Cancel' with a X and 'Save’ buttons.

## Acceptance Criteria
* Verify that a multi-select Tags field is added to the list of Roster filters, containing the active Tags for this specific district
* Verify that a Tags column is added to the list of Roster filters 
* Verify that this column shows a condensed view of the tags relating to each roster, with overflow showing as +X,X+1,X+2, etc. and the full list showing in a tooltip while hovering over it
* Verify that the Edit Roster functionality opens as a drawer instead of a pop-up, and that it now contains a Tags field that the user can use to assign/remove tags from the roster
* Verify that the other general UI/UX improvements have been made to the page:
** Expansion panel
** Reset/Apply
** Grid edit icon ‘Edit’ → ‘Edit2’
** Edit Roster buttons ‘Update’ and ‘Cancel' → X and 'Save’
