# NXT-52839 — PR- Plan- Deleting Prepared items from menu after plan consolidated 

## Description
As a production manager, I would like for my prepared items to not be removed from the plan if the menu planner removes it from the menu after the plan has been consolidated.

## Acceptance Criteria
* Allow prepared menu item to be deleted from menu at any time before record complete
* IF prepared menu item is deleted anytime before the consolidated plan is complete, remove from the plan and from the consolidated plan for site/sites
* IF consolidated plan is complete, allow menu item to be deleted from the menu but keep the menu item on the plan and consolidated plan
** Pretend like menu has been added from the plan by providing delete option and also adding a comment as we currently do as to when it was added to the plan 
*** Use time and date of the deletion from menu and user to add comment on plan 
** Delete icon should be disabled by default in this scenario and would only be enabled if consolidated plan is deleted
* This should be effected by any means of removal or replacing of menu items 
* When Adding menu items from the production side, either from plan or production adjustment, prepared menu items should be treated as normal menu items if consolidated plan is complete 
** Delete icon would be available as normal
** If consolidated plan is delete, trigger the prepared item flag again and display in the consolidated plan 
*** This should happen from any add menu item path whether from menu planning or production
