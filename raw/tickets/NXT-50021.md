# NXT-50021 — LMA - License Keys - Edit

## Description
As a License Management (LMA) role member, I would like to edit license keys for districts so that I can correct any mistakes I made

*Notes:*

* The user can only fully edit a License Key if it is not applied yet
* Once it is applied, it can be canceled or renewed with other processes
* This is to protect the integrity of the license and prevent changes being made w/o sales and CS teams knowing about it

*Flow:*

* LMA role user browses to System > License Management
* User sees a list of license keys
* User clicks Edit on License Key that is in New or Active status
* User redirected to the Edit License Key drawer
* User updates info and hits Update button
* System saves changes
* User goes back to the License Key dashboard and sees the updated key

*Requirements:*

* On System > License Management, do the following:
** When click on the Edit button on a grid row, open the Edit License Key drawer
** The Edit button is only enabled for New and Active Status keys

* If not already created, create the drawer

* Show all the Basic Information fields same as Generate License Keys screen

## Acceptance Criteria
h3. *1. Access and Navigation*

* From *System > License Management*, clicking the *Edit* button on a grid row opens the *Edit License Key* drawer.
* *Edit button is only enabled* for license keys with *New* or *Active* status.
* Drawer leverages the *Generate License Key* screen as a base.

h3. *2. Basic Information Section*

* All fields are *read-only except Notes* field.
* *If Status = Active:*
** Display all fields from the Generate License drawer.
** Additionally show the following *read-only fields*:
*** *Generated Date* (date only, no time)
*** *Generated By* (name of the user who generated the license)
*** *Status* (e.g., New, Active)
*** *License Key* (key number)
* *If Status = New:*
** Display all fields from Generate License drawer.
** All fields are *read-only except Notes*.
** Show *Generated Date, Generated By, Status, License Key* as read-only.
** Ensure Status field is *disabled* (correcting prior Bolt code bug).

h3. *3. License Types Section*

* Load all license type rows and values from the database.
* *If Status = New:*
** All rules and behaviors match the Generate License Key drawer.
* *If Status = Active:*
** Delete button is *disabled* (correcting prior Bolt code bug).
** Edit button is enabled.
** Only the *Revenue* field is editable.
** Changes to Revenue must update *Total Revenue* in real-time.

h3. *4. Update License Key Button*

* On save:
** Do not generate a new license number.
** Save all updates to the database.
** Save update to the *History table*.
** Updates to Revenue should recalculate and display Total Revenue correctly.

h3. *5. Security*

* Drawer is accessible by *LMA, Customer Support (CS), and Tech Support* roles.
* *If Status = Active:*
** All three roles can make changes to editable fields.
* *If Status = New:*
** Permissions are the same as Generate New License Key drawer:
*** CS and Tech Support can update *Notes* and *Revenue* only.
*** LMA role required to update all other fields.
