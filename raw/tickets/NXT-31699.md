# NXT-31699 — PR- Consolidated Production Plan (Prep Site View) (Different Menu line)

## Description
As a prep site manager, I want to see a consolidated plan for all planned counts for my site as well as the “prepared items” for any receiving site I am prepping for.

## Acceptance Criteria
Prepared items planned counts from receiving sites will show on the consolidated plan.

Only consider Plan Complete for receiving sites

*Prep site and receiving site are not on same meal type/Menu line:*

If prep site does not have the same meal type/menu line combination as receiving site/ sites, show receiving site menu lines on prep site calendar. 

EX: 

Prep site has Lunch- Main

Receiving site 1 has Lunch- Burger Line

Receiving site 2 has Lunch- Burger Line

Receiving site 3 has Lunch- Veggie Line

Receiving Site 4 has Lunch-  Pizza Line

Show on prep site calendar:

Lunch- Main (Plan Complete)

Lunch- Burger Line (show as Plan Incomplete)-- Only Consolidated Saved show as plan Complete

Lunch- Veggie Line (show as Plan Incomplete)-- Only Consolidated Saved show as plan Complete

Lunch- Pizza Line (show as Plan Incomplete)-- Only Consolidated Saved show as plan Complete

Combine “prepared items” for Burger line for 2 sites and show counts in separate columns since they are the same menu line. 

When user clicks on receiving site menu line, there will be no information in the production plan tab since there is no production plan for prep site with this menu line. (Indicate on the tab (There is not Production Plan for this Menu Line)

Change Status to Plan Complete on Plan Incomplete lines once Consolidated Plan is complete

There will also be no column for prep site on consolidated plan tab as this is not their menu line. Only show sites that have that menu line and are tied to specific prep site. 

Columns to Display:

Preparation Date (default to actual record date for prep) user can change if needed (Date Range to select from Current date- Record Date)

Menu Item

Serving Size

Site Name ( Receiving Site/ Sites) 

Total (Planned Counts for Each item for all sites) 

If menu items are not available for specific site, display —

If planned counts are not available for specific site that need “prepared items” display (Not Available) in red

*FROM other story: Sort Prep site first, then all sites in alpha order*

*FROM OTHER STORY: Word wrap all the sites to show all the information of the sites on the header*

*FROM OTHER STORY: Message “The selected Prep site was previously consolidated. Please notify the Prep site of the changes made to this production Plan. Status changed to Plan Complete.” needs to appear AFTER the user has saved. Keep the message on the screen with option for user to remove if they want*

*FROM OTHER STORY: Check consolidated plan status after every save. We should not be doing the check when the receiving site is loading the page but rather when they are saving*

*FROM OTHER STORY: Sort Menu items for Prepared items in Alpha order, Non prepared in alpha order separately*
