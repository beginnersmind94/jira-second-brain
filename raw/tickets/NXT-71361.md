# NXT-71361 — Insights - KPI - ENP - Drawer - SC

## Description
*I am a* district user *that I want to* open the Eligible Not Participating (ENP) details drawer from Insights using SchoolCafé data
*so that I* can review the actual value, expected value, program-level breakdowns, and school-level detail behind the KPI.

----

h2. *Notes*

* This story applies to *Insights – SchoolCafé data source*.
* ENP (Eligible Not Participating) is part of the *front office KPIs* being integrated into Insights.
* This story covers the *ENP Details Drawer only*.
* This story is a *companion to the PrimeroEdge ENP Details Drawer story*.
* The *frontend behavior, layout, interactions, formatting, sections, sorting, pagination, and drawer pattern must remain the same as the PrimeroEdge implementation*.
* Frontend developers should *not need to build new UI logic* for SchoolCafé.
* This implementation must reuse the same:
** drawer layout
** overview section
** Program by Eligibility matrix
** Details by School matrix
** sorting behavior
** pagination behavior
** about-this-metric section
** open/close behavior

----

h3. *Data & Source of Truth*

* The source of truth for ENP data is available in:
** *Accountability → Reports → Participation*
** Click the *Non-Participation* tab
** Select *Free and Reduced* in the *Eligibility* field
* This must be used by developers and QA to validate ENP calculations and displayed values.

----

h3. *Stored Procedure Integration*

* The front office team has already created the stored procedure:
** SchoolCafé: [https://primeroedgenext.atlassian.net/browse/NXT-67456|https://primeroedgenext.atlassian.net/browse/NXT-67456|smart-link] 
* The details drawer must use the *SchoolCafé stored procedure* as the data source instead of the PrimeroEdge stored procedure.

----

h3. *KPI Definition & Formatting*

* ENP is a *percentage-based KPI*.
* Format must be:
** *0.00% (two decimal places)*
* Values must be:
** Rounded to two decimal places
** Between *0 and 100*

----

h3. *KPI Direction*

* This KPI is *Lower is Better*:
** Values closer to *0% indicate better performance*
** Higher values indicate worse performance

----

* If launched from a *School Performance grid row*, the drawer must show only the data for that selected school.
