# NXT-3194 — Accounts - Student Info - Enable Refunds option

## Description
*User Story*

As an administrator I need to be able to issue refunds to patrons for amounts remaining in their accounts, so that they can both receive money back as well as have it accounted for in the system.

*Mock-up*

!image-20210308-182107.png|width=1453,height=839!

!image-20210308-182122.png|width=1453,height=839!

*Requirements*

* Within Account Details where there is a ‘More Actions’ drop-down menu, when a user selects the menu, show a button for *Refund*.
* When that button is selected, open a new slide out action panel appears where the user can refund money {color:#ff5630}-from the account selected to another account-{color}.
* Details
** Refund Type
*** Group button options for Full and Partial are provided to the user.
*** By default, no options should be pre-selected (must require the user to choose)
** FULL
*** If Full refund type is selected, this means the user wants to refund all of the account balance. We should populate the Refund Amount label with this information and then calculate the New Balance as $0.00
** PARTIAL
*** If Partial refund type is selected, this means the user wants to refund a portion of the available account balance. When selected, an input field for “Refund Amount” should be given, allowing the user to populate the amount up to the full account balance. 
** Previous Balance
*** Display the selected patron’s current balance
** Refund Amount
*** Displays the amount entered by the user (for Partial) or the full account balance (for FULL).
** New Balance
*** When “full” is selected or a dollar amount is entered into the refund amount field (partial), update the New Balance amount in real-time so the user is aware before committing to changes (save).
*** New Balance = Previous Balance - Refund Amount (either the full balance if FULL, or entered Refund Amount if PARTIAL)
** Comments
*** A comment is required for any transaction.

* Once the requirements above are met, the user should be allow to select the Save button to complete the transfer.
* ‘Close’ and/or ‘Cancel’ buttons that return them to the Account Info page without saving anything.
* Error handling:
** IF Comments IS NULL
*** Display “A comment is required in order to process a refund.”
** IF _Refund Amount_ <= 0 OR _Amount_ > _Balance_
*** Display “Please enter an amount between $0.00 and the current balance of this patron.”
** IF Balance <= 0
*** Immediately hide the button group, amount, comment field, and Save.
*** Display “This patron cannot be refunded because they currently have a negative or zero balance.”
*** Cancel button is only available for the user to close (no prompt required).
* When ‘Apply’ is selected, an un-editable *transaction* should then be entered into the patron’s account, with a transaction type and description of '*Refund*'.
* A *session* is also created if an open session for that site and user is not open, and a single transaction is entered into that session.
* If an *OPEN session* exists for that site and user, the transaction is added to the open session.
* This story is complete when a user can navigate to the Refunds menu from the Student Info page and perform a refund, with balance display changes and error handling where expected, and a transaction and session being created. A refund receipt that can be printed is also created.

## Acceptance Criteria
After selecting an account the user can select an account to refund.

The user can select FULL to refund the entire balance

The user can select PARTIAL to enter a refund amount to refund that is less than or equal to the account balance, greater than 0.01.

Appropriate error handling as described in the story description should exist.

New Balance should display based on calculation of previous balance - refund amount (full amount or partial amount)

A transaction and session are created with these details:
-The transaction date is the current date
-The transaction type is Refund
-The comment is saved
-The account has a transaction entered
-A session is created

User can back out of the transaction by selecting 'Close' and/or 'Cancel' to return to Account Info

A receipt must be created which is printable from the Transactions page (can be deferred if needed)
