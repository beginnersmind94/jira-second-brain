# NXT-71252 — PE-SC-Insights-Benchmark History Enhancement

## Description
The current API returns an excessively large response, necessitating a reduction in response size to ensure efficient functioning of the history feature with pagination.
Use the feature release version details for Benchmark History Search V2 from CommandCenter/DateFilter API to enable this feature.
If IsActive = true, then code needs to use the V2 API with the optimised response to build the benchmark history control.
If it is false, then it should work as it is currently working.
Virtual pagination should work as is.
