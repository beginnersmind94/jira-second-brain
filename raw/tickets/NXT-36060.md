# NXT-36060 — Error Reporting - Create Error Reporting Email

## Description
As a Cybersoft employee working in development, I want to be emailed all the errors so that I can quickly see if I need to address a problem.

*Flow:*

* An error occurs
* An email is sent with the error details

 

*Requirements*:

* Create a group email account in our internal domain for production errors
** Name: SchoolCafé Errors
** Administrators: FO, BO and System dev team leads
** Initial members: Johnathon, Dallas, Daimien, dev team leads, any other dev team members you want; more added later
** This email will be used for production applications
* Create a group email account in our internal domain for non-production errors
** Name: Non-Prod SchoolCafé Errors
** Administrators: FO, BO and System dev team leads
** Initial members: dev team leads, any other dev team members you want
** This email will be used for non-prod servers
** Start with demo and UAT environment, probably overkill for QA and dev
* These emails need to be in a configuration file or easily changeable so that they can be changed without having to recompile (no hard coding)
* Need to ensure it is easy to make sure don’t accidentally email non-prod email from a prod site
* (Daimien’s note:) There is an existing email that we send out to a user who has performed an import, when there are errors. The email goes out and shows a summary of the errors that occurred so that the user knows what failed and what they can do. We might be able to follow that template’s format for our emails as well.

## Acceptance Criteria
* Both email addresses exists and can add members to it
* Non-prod email no used by prod site
