# NXT-39786 — SIS Auto Sync – Remove SIS Connection Fields From Add New and Edit Screens - Dev

## Description
As a SchoolCafé admin, I would like the SIS Auto Sync connection fields to be in System Settings so that we don’t have to reconfigure them every time we do an SIS import configuration.

*Notes:*

* These fields currently exist in the Add New and Edit SIS Import configuration screen
* In another story, we added them to System settings so that it can be reused for multiple imports and not have to be re-setup every time
* Now in this story, we will remove them from the SIS Auto Sync Add New and Edit screens

*Requirements*:

* On the System > Integration > Imports & Exports > SIS Auto Sync > Add New screen, do the following:
* 
* Remove these 4 new fields:
** Provider
** URL
** Key
** Secret
* 
* Update the Save button:
** Put a validator on it to check the System > Configuration > Settings properties to ensure they have proper values
** Or, test the connection to ensure it works
** If fails, show an error message
*** Text: Please verify your SIS configuration in System Settings
*** Do not let them save the configuration until they have a valid SIS connection
* 
* Instead of these fields, use the corresponding values from the System > Configuration > Settings when you are creating a new one
* 
* Update existing SIS Import configurations to use the corresponding values from the System > Configuration > Settings
* 
* Everything you do for Add New needs to be done for Edit as well

## Acceptance Criteria
* Fields are removed from the screen
* When you create a new SIS Import configuration, it works
* The SIS imports that have been created already are updated to use the properties from System Settings
* Existing SIS imports continue to work
* Code validates there are SIS properties in System Settings and that it can connect to the SIS Provider
* The changes are applied to both Add New and Edit screens
