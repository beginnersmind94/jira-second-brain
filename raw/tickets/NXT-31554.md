# NXT-31554 — Add Rescan Button to Cart Screen

## Description
As a cashier, I want to be able to rescan a tray if I see that it failed to detect a tray properly so that I can try again after fixing the tray alignment.

*Application*: POS

*Summary*

We will show a button on the transaction/cart screen next to new red / green detection indicator created in the story labeled “Add detection success or failure indicator to cart screen” when they story produces a red image so that the cashier can rescan manually again if they want.

*Flow:*

* While cashier is using POS to sell food to students:
* Detection cycle begins when student walks up to cashier
** Cashier selects student
** Edge detection triggered
** Edge sends results to POS
** POS shows the Results Detected image in the lower left part of the screen next to the green dot (Other story)
** POS saves results and optionally loads the cart
** If image is red, show the cashier the rescan button
** If the cashier uses the rescan button, it starts the Edge detection process over
** Transaction is completed

*Assumptions:*

* This was completed in classic and you can copy the code from it
* Get with Saketh on the classic code
* You should complete the story labeled “Add detection success or failure indicator to cart screen” before doing this or do this one at the same time as that one
* UX involvement might be required for the rescan button
* It is assumed in these requirements that you are using a green image for success, a red image for failure and a yellow image for everything else, per the other story

*Requirements:*

* When the results image is red, show the Rescan button
* When the results image is any other color or is not shown, do not show the Rescan button
*  
* Rescan button functionality:
* Upon click:
** Call the Edge device to re-detect and follow the process normally
** Essentially, you are just mimicking the trigger that is when a student is identified that already exists
** Reset the results image to yellow
** Subtract 1 from the NoDataCount if you are showing this button because you had received a Tray Not Detected message
* Security – no additional requirements necessary

*Mockup*:

* Ask UX team if you feel necessary

## Acceptance Criteria
* When the results image is red, it shows the Rescan button
* When the results image is any other color or is not shown, it does not show the Rescan button
* Upon click, it mimics the trigger that is when a student is identified that already exists and calls the Edge device to re-detect
* Upon click, it resets the results image to yellow
* Upon click, it subtracts 1 from the NoDataCount if you are showing this button because you had received a Tray Not Detected message
* If don’t hit submit when have a red button, everything works normally
