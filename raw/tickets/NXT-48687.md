# NXT-48687 — OpenID - Users Imports update

## Description
We need to ensure that when a new user account is created during the Import routine, that the user’s default password setting is configured

*Requirements*

* When a user import is executed, ensure that the user’s authentication type is set to the district’s default setting
* All security, authentication, login, emails etc. should continue to work as normal
* If the default type is not SchoolCafe Authentication, don’t force the user to set a password

## Acceptance Criteria
* When a user import is executed, the user’s authentication type is set to the district’s default setting
* User receives usual emails and is able to login same as before the change if it is SchoolCafe Authentication
* If default is something different, then user does not have to set a password
