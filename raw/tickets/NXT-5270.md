# NXT-5270 — Verification - Tracking - Indicator and filter for responses via family portal

## Description
As Vera the director I want to know when a family has responded to an application specifically via the Family Hub portal (as opposed to an email/letter) so that I can quickly deal with those first in order to save time and improve our overall tracking.

*Mockup*

[https://www.figma.com/proto/Yo99YhQm5Xk91Ie7b0PRkD/Front-Office-6.0?node-id=120%3A6574&scaling=scale-down-width&page-id=120%3A6271&starting-point-node-id=120%3A6574|https://www.figma.com/proto/Yo99YhQm5Xk91Ie7b0PRkD/Front-Office-6.0?node-id=120%3A6574&scaling=scale-down-width&page-id=120%3A6271&starting-point-node-id=120%3A6574|smart-link] 

*Requirements*

* In Eligibility → Verification → Tracking, add a checkbox to only show the apps that have had a response sent via the Family Hub portal.
* Additionally, display an icon in the 'Application #' column next to the number of each app where a Family Hub response has been received
** In PrimeroEdge this was a green leaf, but in SchoolCafe we should use something different (i.e. an ‘FH’ icon).
* Checkbox name: *Show Family Hub responses only*
** This will replace the existing ‘Verification Incomplete Applications’ table header, which can be removed
** If unselected (Default state), show all apps regardless of if/where the response was received
** If selected, only show those apps with the icon displaying next to them
* When a new online Verification response is submitted through the Family Hub, it should now display this new icon. (Added on May 26th after meeting to discuss: add another text/icon displaying information about apps where the parent opted to refuse benefits.)
** So there will be one FH icon if the response was submitted through the Family Hub
** Additionally, if the parent chose not to a respond, there will be a second, NR, icon displaying next to the FH
** Both are impacted by the ‘Show Family Hub responses only’ toggle

## Acceptance Criteria
User can filter for apps with an online response.

User has an indicator on the Tracking grids to let them know that that single app has an online response.

User has an indicator to show specifically that an app received a ‘No Response’ response

Newly submitted responses now appear on this page with the new icon.
