h1. *Best Practices:*

When implementing error handling in a React web page, it’s important to both capture errors effectively and communicate them to the user in a clear, user-friendly manner. Here are some best practices:

* *Error Boundaries:*
** Use React error boundaries (components that implement {{componentDidCatch}} or use the {{ErrorBoundary}} pattern) to catch JavaScript errors in the component tree and display a fallback UI instead of crashing the entire application.
** Provide a friendly fallback interface that informs the user something went wrong, and optionally, a way to retry or navigate to a safe page.
* *Try/Catch in Asynchronous Code:*
** Wrap asynchronous operations (e.g., API calls, promises) in {{try/catch}} blocks. This helps you catch errors where they occur and manage them gracefully.
** Use error handling libraries or patterns like async/await with try/catch for better readability and maintainability.
* *User-Friendly Error Messages:*
** Display clear, non-technical error messages to users. Avoid showing raw error objects or stack traces.
** Use context-specific messages: For example, “We’re having trouble loading your data. Please try again later.” instead of “HTTP 500 Error.”
** Consider using modal dialogs, toast notifications, or inline messages to inform the user without disrupting the entire experience.
* *Logging and Monitoring:*
** Log errors to a monitoring service (such as Sentry, LogRocket, or your own backend) so you can analyze issues later.
** Ensure that logging does not expose sensitive user data and follows your security/privacy policies.
** Include additional context in logs, such as user ID, browser, and component details to aid debugging.
* *Graceful Degradation & Retry Options:*
** When possible, offer a way for the user to retry the failed operation.
** For non-critical failures, consider falling back to a cached version of data or a simplified UI, ensuring the app remains partially functional.
* *Consistent Error Handling Strategy:*
** Standardize how errors are handled across the application by defining a global error handler or using middleware (for example, in Redux or a custom hook) to centralize error management.
** Create a common error component that can be reused across different parts of the app to display error messages consistently.
* *Accessibility Considerations:*
** Ensure that error messages are accessible (e.g., using ARIA roles) so that screen reader users are informed of errors.
** Make sure any retry or navigation controls are keyboard-accessible.

By incorporating these best practices, your React web page will be better equipped to handle errors gracefully, provide useful feedback to users, and maintain a robust and user-friendly interface even when unexpected issues occur.

h3. b. Business Logic Layer (errorLoggingService.js)

This service function accepts error details and writes them to the database.

{noformat}// errorLoggingService.js
const { getConnection, sql } = require('./db');

async function logError({ errorMessage, errorStack, districtID, userId, additionalInfo }) {
  try {
    const pool = await getConnection();
    const result = await pool.request()
      .input('ErrorMessage', sql.NVarChar(4000), errorMessage)
      .input('ErrorStack', sql.NVarChar(sql.MAX), errorStack)
      .input('DistrictID', sql.Int, districtID)
      .input('UserId', sql.Int, userId)
      .input('AdditionalInfo', sql.NVarChar(sql.MAX), additionalInfo)
      .query(`INSERT INTO ErrorLogs (ErrorMessage, ErrorStack, DistrictID, UserId, AdditionalInfo)
              VALUES (@ErrorMessage, @ErrorStack, @DistrictID, @UserId, @AdditionalInfo);
              SELECT SCOPE_IDENTITY() AS ErrorLogId;`);

    return result.recordset[0].ErrorLogId;
  } catch (error) {
    console.error('Error logging failed:', error);
    throw error;
  }
}

module.exports = {
  logError
};

h3. a. Error Boundary Component

Create a reusable error boundary component that catches errors in the component tree using {{componentDidCatch}} and displays a modal dialog.

{noformat}// ErrorBoundary.jsx
import React from 'react';
import axios from 'axios';
import Modal from './ErrorModal';

class ErrorBoundary extends React.Component {
  constructor(props) {
    super(props);
    this.state = { hasError: false, errorInfo: null, showModal: false };
  }

  componentDidCatch(error, errorInfo) {
    // Update state so the next render shows the fallback UI.
    this.setState({ hasError: true, errorInfo, showModal: true });

    // Optionally log error details to the server.
    const logData = {
      errorMessage: error.toString(),
      errorStack: errorInfo.componentStack,
      districtID: this.props.districtID || 0, // Pass in from props if available.
      userId: this.props.userId || null,
      additionalInfo: 'Error caught in ErrorBoundary'
    };

    axios.post('/api/v1/log-error', logData, {
      headers: {
        'Authorization': process.env.REACT_APP_AUTH_TOKEN
      }
    }).catch((err) => {
      console.error('Failed to log error:', err);
    });
  }

  closeModal = () => {
    this.setState({ showModal: false });
  };

  render() {
    if (this.state.hasError) {
      // Render any fallback UI
      return (
        <>
          {this.props.children}
          {this.state.showModal && (
            <Modal onClose={this.closeModal}>
              <h2>Something went wrong.</h2>
              <p>We encountered an unexpected error. Our team has been notified.</p>
            </Modal>
          )}
        </>
      );
    }

    return this.props.children;
  }
}

export default ErrorBoundary;

h3. a. GET /api/v1/error-logs

This endpoint accepts optional query parameters and returns a list of error logs.

{noformat}// In your Express server file (e.g., app.js or index.js)
const express = require('express');
const bodyParser = require('body-parser');
const { sql, getConnection } = require('./db'); // use the db.js module from previous example
const app = express();
app.use(bodyParser.json());

// Simple token-based auth middleware
app.use((req, res, next) => {
  const token = req.headers['authorization'];
  if (!token || token !== process.env.APP_AUTH_TOKEN) {
    return res.status(401).json({ error: 'Unauthorized' });
  }
  next();
});

// GET /api/v1/error-logs - fetch grid data with optional filters
app.get('/api/v1/error-logs', async (req, res) => {
  const { errorLogId, districtID, userId, startDate, endDate } = req.query;
  try {
    const pool = await getConnection();
    let query = `
      SELECT 
        ErrorLogId, 
        DistrictID,
        UserId,
        Timestamp,
        d.DistrictName,
        u.UserName
      FROM ErrorLogs
      CROSS APPLY (SELECT 'District ' + CAST(DistrictID AS VARCHAR(10)) AS DistrictName) d
      CROSS APPLY (SELECT 'User ' + CAST(UserId AS VARCHAR(10)) AS UserName) u
      WHERE 1=1
    `;
    const request = pool.request();

    if (errorLogId) {
      query += " AND ErrorLogId = @errorLogId";
      request.input("errorLogId", sql.Int, errorLogId);
    }
    if (districtID) {
      query += " AND DistrictID = @districtID";
      request.input("districtID", sql.Int, districtID);
    }
    if (userId) {
      query += " AND UserId = @userId";
      request.input("userId", sql.Int, userId);
    }
    if (startDate) {
      query += " AND Timestamp >= @startDate";
      request.input("startDate", sql.DateTime, new Date(startDate));
    }
    if (endDate) {
      query += " AND Timestamp <= @endDate";
      request.input("endDate", sql.DateTime, new Date(endDate));
    }

    query += " ORDER BY Timestamp DESC";

    const result = await request.query(query);
    res.status(200).json(result.recordset);
  } catch (error) {
    console.error('Failed to fetch error logs:', error);
    res.status(500).json({ error: 'Failed to fetch error logs' });
  }
});
{noformat}

                <Button
                  variant="contained"
                  onClick={() => fetchErrorDetails(log.ErrorLogId)}
                >
                  View
                </Button>
              </TableCell>
            </TableRow>
          ))}
        </TableBody>
      </Table>

      <Drawer anchor="right" open={drawerOpen} onClose={() => setDrawerOpen(false)}>
        <div className={classes.drawerContent}>
          {selectedError ? (
            <>
              <h2>Error Details</h2>
              <p>
                <strong>Error ID:</strong> {selectedError.ErrorLogId}
              </p>
              <p>
                <strong>Error Message:</strong> {selectedError.ErrorMessage}
              </p>
              <p>
                <strong>Error Stack:</strong> {selectedError.ErrorStack}
              </p>
              <p>
                <strong>District ID:</strong> {selectedError.DistrictID}
              </p>
              <p>
                <strong>User ID:</strong> {selectedError.UserId}
              </p>
              <p>
                <strong>Date/Time:</strong>{' '}
                {new Date(selectedError.Timestamp).toLocaleString()}
              </p>
              <p>
                <strong>Additional Info:</strong> {selectedError.AdditionalInfo}
              </p>
            </>
          ) : (
            <p>Loading...</p>
          )}
          <Button variant="contained" onClick={() => setDrawerOpen(false)}>
            Close
          </Button>
        </div>
      </Drawer>
    </div>
  );
}

export default ErrorLogsPage;
