# NXT-37187 — IM - AI Script for Serving Size Exceptions

## Description
Currently the Serving Size description is parsed as_is. The PE and SC application does not know how to parse this value to derive the serving size (decimal  Number) and the measure to use



Serving Measure comes with various combination of Size and Measure put together in one field. The excel file is attached for reference

## Acceptance Criteria
The system should be able to parse the description by means of API and generate two variables out of description:

# Serving Size (Ex: 1 1/2 )
# Serving Measure (Ex: Cup or Each or Tablespoon)

The system should handle the attached list of exceptions as provided in the remarks column.
