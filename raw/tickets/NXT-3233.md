# NXT-3233 — System - Organization - Add Area [Regions]

## Description
As an administrator, I would like to be able to add areas, or sub-regions, to my district, so that I may be able to better group sites for reporting and administration purposes.



* In the main Regions view/list page, add an *Add Region* button
* When users with appropriate access to the add functionality on the page select this button, open a new form that allows the user to add a new area
* Existing fields (included which ones are required), buttons, and logic from PrimeroEdge Classic can be carried over
* Previously, the address section could not be added until other region details (name, description, org. level, etc.) were entered. Instead, let’s make all of those fields immediately configurable by the user in the ‘Add’ form
* When the user attempts to select the *Create Region* button, if all required fields have appropriate data, save the area and return the user to the main list/view.
** Otherwise, if some required details are still missing or inappropriate, the button should be disabled.
* Add a *Cancel* button that, when selected, returns the user to the main view.

## Acceptance Criteria
1. When the user navigates to the Regions view/list page, an 'Add Region' button should be enabled and accessible to users with appropriate permission.
2. When selected, the Add Region button should open a form to add a new area to the district, which has the same fields, buttons, and logic as in PrimeroEdge Classic.
3. A Cancel button should be available also, which when selected returns the user to  the main Regions list/view.
4. The 'Create Region' button should only be enabled if all required and appropriate field data is entered.
5. When selected, the button should add the area and return the user to the main Regions list/view.
