# NXT-7257 — INV - Orders - Manage Orders - Receipts - Quick Receipt - Order Step

## Description
As a site manager I want the ability to create a receipt without placing an order in the system so I can receive items that I purchased outside the system.

This will be a new feature that will allow users to create a receipt at the same time they create the order. (For context, this is called ‘Receipt without Order’ in PrimeroEdge). The purpose of this feature is to allow users to receive product that may not be orderable via the typical order process. We will call this ‘Quick Receipt’

*Requirements*

* On the Receipts tab of the Manage Orders page, add a new ‘Quick Receipt’ button in the top right corner
* The Quick Receipt button will open a version of the Create Orders page with some differences
** Header selections:
*** Vendor is mandatory and must be a single selection
**** only load vendors that are applicable to the selected Site
*** Delivery Date can be hidden/removed, as it is not required for Quick Receipt
*** If possible, we should remove options from the Order Templates dropdown if it does not contain any items offered by the selected vendor
**** *Future enhancement story
** Item list:
*** All contract validation alerts would be suspended (min/max order quantity)
** Remove Shopping Cart
** ‘Next’ button will replace the ‘Add to Cart’ button
*** make sure page is long enough that the Next button can be easily accessed
* Once the user clicks Next, it will immediately navigate to the Create (Quick) Receipt screen ([NXT-18046|https://primeroedgenext.atlassian.net/browse/NXT-18046])
