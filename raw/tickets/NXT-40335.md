# NXT-40335 — eOrders – Generate Part 3

## Description
This is a continuation of the eOrders – Generate Part 1 and covers the Completion Tab

*Completion Tab*

* If the tab hasn’t been created already, then create it
* The tab is used for taking any additional actions on the eOrder and submitting it
* Tab Title = Completion
* 
* Fields:
* Include Order Comments
** Checkbox
** Default to selected
** Do whatever the field does in Classic
* 
* Group fields as Notification
* Users
** Multi-select list
** Values – active user accounts for the district
** Optional
** No default values
** Note: This will enable them to send a message to SchoolCafé users that the eOrder has been sent
** Could be a big list, so make bigger than usual box size
* SchoolCafé Roles
** Multi-select list
** Values – Roles for the district
** Optional
** No default values
* 
* Group fields as Email
* Email Addresses
** Textbox
** Always optional
** No default value
** Tooltip: Optionally add comma delimited list of email addresses
** The user can type in a list of email addresses separated by commas
** Note: make this a larger textbox and ability to scroll since they may have a lot of email addresses they want to use
* 
* Group fields as Scheduling
* When to Send
** Radio buttons
** Values – Immediately / Scheduled
** Required
** Default = Immediately
** If No, this means they are just configuring and it will not send anything by default
* 
* If they select Scheduled, then enable the following field
* If immediately, disable field
* Date and Time to send
** Date time picker
** Required if When to Send = Scheduled
** They must pick a single date and time in the future
* 
* At the bottom of the tab, have these buttons:
* Previous
** On click, go back to Selection tab
** Don’t ask them if they are sure, just do it
** Save state on the screen in case they come back from the previous tab
* Send
** Validate the following:
** 
** If fail validation, show the user an error message and show the user which fields need correction
** 
** If validated, then do the following:
** Notifications:
** If they add any users to the Notify Users field or the Roles field, then send a message to their inboxes
*** Text: [user’s name] generated eOrder # [number] for [vendor name] at [date/time stamp]. To view order details, click [url to eOrders dashboard].
*** Be sure to not send more than one message to a user since could be in the roles and users
** 
** Email:
** If there are any email addresses in the list of Emails, then do the following
*** Email the file to each email address
*** Subject: eOrder #[number of eOrder]
*** Body Text: [user’s name] generated eOrder # [number] for [vendor name] at [date/time stamp].
*** Attach the file
** 
** If sending immediately, do the following:
*** Generate the file per the file format
*** Send the file via FTP
*** Update the eOrder history log???
*** Return the user to the dashboard with the new eOrder in the grid of eOrders
*** Look at the Classic page and make sure you are doing everything that they are doing there
** 
** Scheduling:
*** Use the job scheduler to schedule sending the FTP file
*** Status of eOrder is Open if it hasn’t sent yet
*** Once the schedule hits, then do everything like you would do if doing immediately
*
