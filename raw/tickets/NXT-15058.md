*requirements*

* Rename the Unmatched page to Potential Matches (page and nav links)
* Add a drawer to the Unmatched page in place of the current Scorecard View.
* Drawer should show the following:
** Photo of the student
** Full name, preferred name, Student ID #
** Current Eligibility + Reason
** Current Enrollment Site
** Matching Factors
*** Note: These are any criteria that resulted in points.
*** Factors are broken into 3 primary information categories:
**** Personal Information
**** Contact Information
**** ID Numbers
** Match Points 
*** This is the total number of points from the matching criteria
** Comments
* The drawer should also provide a way to view the full scorecard.
** ‘Comparison Scorecard’ → show the user the full scorecard with the updated organization of info.
* The drawer should have actions to Match and Mark as Reviewed.
** Mark as Reviewed is the new name for Dismiss.
* Make similar changes over on the Reviewed tab.
** The ‘Dismissed’ status needs to be updated to ‘Reviewed’
** Drawer can be implemented there also with the reverse action available.
*** If status = Matched, user can use action ‘Unmatch & Mark as Reviewed’
*** If status = Reviewed, user can user action ‘Match’
* Update the scorecard to have the 3 category layout.
** Personal Information (All the name fields, DOB, DOB Variable)
*** -Nest the name variations under Full Name as part of this, indent them.-
** Contact Information (Guardian, Guardian First Name, Guardian Last Name, Street Address, Phone Number1, Phone Number2)
** ID Numbers (SSN, State ID, Student ID)

## Acceptance Criteria
* Drawer available that shows high-level potential match information.
** Information segmented into 3 primary categories.
** -If a section results in 0 points, don’t show that section.-
** Total points shown by primary category.
*** Example: Personal Information 57
** Total points shown as ‘Match Points’
* Drawer has a link to the full scorecard analysis.
* Drawer has Match and Mark as Reviewed actions like the scorecard does.
** Comment required for matching.
* Drawer shows student information:
** Photo, full name, student id #, current eligibility + reason, current enrollment site.
* *Unmatched renamed to Potential Matches*
* Reviewed page updated similarly with the drawer available.
** Statuses updated to be Matched and Not Matched.
** Drawer / Scorecard allows the user to take the reverse action.
*** If status = Matched, user can use action ‘Unmatch & Mark as Reviewed’
*** If status = Reviewed, user can user action ‘Match’
