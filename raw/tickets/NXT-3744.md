# NXT-3744 — Inspections Scoring - Pass/Fail

## Description
*User Story*
As an Inspections user, I have a method for scoring Pass/Fail questions in both Pass/Fail scoring type and Custom Scoring type, to properly evaluate Inspections.

*Requirements*

* Pass/Fail questions can be scored in both Custom Scoring and Pass/Fail scoring types
* Template
** For Pass/Fail scoring type, the scoring is preset and cannot be edited by the user
*** Scoring settings can be edited - covered by another story
*** Total points possible is equal to the number of questions
*** There is no required/not required flag - all questions are required in this scoring type
** For Custom Scoring, users can set the value give to each option
*** Like with multiple choice questions, a field is present next to the Pass and Fail selections to apply a score
**** 0-99, no decimals
*** Leaving the field blank defaults the value to 0
*** N/A value cannot be edited
*** Pass/Fail question type can be set to required/not required
* Inspection
** For Pass/Fail scoring type, each response has a given value
*** Pass - Adds 1 to inspection score
*** Fail - Adds 0 to inspection score
*** N/A - Removes 1 from total possible score
** For Custom Scoring, value is added based on the values assigned to the responses in the temple
*** Selecting N/A removes the maximum value from the total possible score
**** For example, if Pass is worth 5 points, Fail is worth 3 points, an N/A response would remove 5 from the total possible score
** A selected option can be cleared for any question type

## Acceptance Criteria
* Scores can be set for Pass/Fail questions in Custom Scoring templates and saved and edited
* Scoring for Pass/Fail questions is properly applied for both Pass/Fail and Custom Scoring Inspections
