# NXT-44388 — MP/PR-Menu/ Plan/Record- 2 Variety Menu item with Same item scenarios

## Description
As a menu planner, there might be a chance that I put 2 variety menu items on the menu that might have some overlapping items that are the same. We need to handle this on the production side so that there are no issues.

## Acceptance Criteria
* Allow user to add 2 different Variety menu items with same items on the menu
** This only applies to VMI, all other rules still apply as in previous stories and scenarios
** This can be added via Menu templates, Quick list , regular add menu item or -Find and Replace-
* Allow user to add 2 different VMI with same menu items on production
* On the production side, user can only toggle on 1 of the child items
** Use this image as an example 
* !image-20231025-070836.png|width=1838,height=783!
** Since the Cereal Honey Nut Cheerios is toggled on, we will not allow it to be toggled on as part of the Ready to Serve Cereal VMI 
*** Display a tool tip on the grayed out toggle “Menu Item already in use on VMI: VMI Name”
** If user toggles  Cereal Honey Nut Cheerios off from the 1 OZ Variety Cereal, now the toggle should be unlocked and ungrayed and the tool tip should be removed and it can now be added on Ready to Serve Cereal VMI
* IF only 1 VMI is added on the menu and then another VMI containing some child items gets added on from either the add menu item in production or from production adjustment, execute the same checks and do not allow the child items to be toggled on if they are already toggled on as part of a different VMI on the same plan or record
* IF VMI is added in production, then child item is added from menu planning, execute checks to lock toggle on since the child item is now part of the menu 
** Checks should happen either direction from MP- PR 
* IF menu Item is already on Menu/production plan, then multiple VMI is added with the same menu item, insert counts (If available) onto the first VMI in the display order
* In the case of a carryover, insert the carryover on the VMI with the child that shows up on the list first in display order
** Add carryover in bubble format on record page
