# NXT-9547 — MP > Menu Cycle > Create/Edit Menu Cycle - improvements

## Description
As a Menu Planner, I want the Menu Cycle creation/editing to be as smooth as possible so I don’t need to be trained in detail for all the ins and outs of the system. I just want it to work logically.

# Remove Cancel button - if the user does not wish to save, they can simply navigate away.
# Give a “Not implemented yet.” message/dialog on clicking the Costing Report buttons.
# Remove prompt when removing a Menu from the Cycle.

As a menu planner, while on the page to create a menu cycle, if I choose to not proceed with the action of creating a menu cycle, then I should be able to simply navigate away from the page without taking the time to click on the "Cancel" button. If I have not yet clicked the "Save" button, then nothing has been created, which negates the need for using a "Cancel" button should I decide to not complete any action on the page. 

While working on this page, I want to view the different reports available to me. Currently, when I click on the dollar sign "$" - absolutely nothing occurs on the page. Since this report is not expected to be available for a time, it would be helpful to give the users a notification that the report has not yet been implemented.  A message such as: "This report is not currently available. Watch the Release Notes for more information." 

As a menu planner, to speed up processes while working on this page, I would also like to minimize unnecessary popup notifications. Currently, if I click on the small X to remove a menu from the list, there's a message that pops up and requires me to click the "Yes" button to remove that menu. Since I can also remove that menu simply by typing the name of a different menu into the blank, it renders the pop up message (and even the small X) unnecessary on the page.  While the small X is not horribly invasive, the pop up window does take extra time and attention, and needs to be removed.

## Acceptance Criteria
Cancel button should no longer be present.
Clicking on the dollar sign for a report should give an appropriate message regarding the status of that report.
Clicking on the X to remove a menu should still remove the menu, but should not give a pop up that also requires attention.
