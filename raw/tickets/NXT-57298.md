# NXT-57298 — PE - GDSN Connect - Sync Vendors & Items from using API

## Description
_(none)_

## Acceptance Criteria
Navigation: System - Import Export:

Acceptance Criteria:

Introduce a new radio button as “Ingredient/Item Export (APi)”

Upon selecting the data source, and click finish, the system shall perform the following:

# *Sync all the unique vendor names into GDSN Connect using Vendors/Sync API:* 

Upon clicking the button, The API should be able to send all the *Vendor* data in PE for a specific region and return the results

Vendors with at least 1 item should be tagged, if a vendor has zero items tagged against them, then that vendor do not need to be synced

Once the data is stored, the user should be able to render the the Vendor list in GDSN Connect UI

Upon successful sync , show an UI message says, “Vendor Sync process is successful”, if failed - show as “Vendor Sync Process Failed”

# *Sync all the Ingredients/Stock Items into GDSN Connect using Items/Sync API:* 

Export all the existing items in selected data source (Local) and translate them into message structure as defined in Swagger - [https://api.gdsnconnect.com/docs/cybersoft/#/Items/startItemsSync|https://api.gdsnconnect.com/docs/cybersoft/#/Items/startItemsSync|smart-link] 

Upon successful sync (Return of Sync ID as response), show an UI message says, “Ingredient/Stock Items: Sync process is successful”, if failed - show as “Ingredient/Stock Items: Sync Process Failed”
