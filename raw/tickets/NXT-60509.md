# NXT-60509 — Research - POS - Device Analytics

## Description
Research task to see what data we can get via POS.

----

As Matt the Product Manager I want a way to help prove when it is network or device bottlenecks that are resulting in poor POS performance, so districts can take action more quickly to achieve great performance.

*mockups*

at the bottom

----

*requirements*

*Tracking*

* Track the following information for the POS Terminal:
** CPU Usage
** RAM Usage
** Drive Space
** Connection Type (Ethernet/WiFi)
*** Are multiple connection types happening? Are both working at the same time?
** Speed or Ping to the Server
* Track the information at the following intervals:
** on POS open
** on POS close 
** every time the heartbeat goes off
*** Discussion point: is this too frequent?
* Upload tracked data to the servers
** Retention - 14 days max?

*Visibility*

* Data is viewable on the Device Management page
** Add a Performance column
** Metrics button
* Text updates on the main page:
** Replace mentions of ExpressPoint with POS
** Site Id → Site Code
** status → Status
* User has Performance Filters on the main grid
** Can check for machines that are hitting over a certain threshold for CPU or RAM
** Can check for machines that have no performance data
* Metrics Drawer
** Date filter - default: today
** Session # filter - default: All
*** Multi-select, all on by default
** Graphs showing CPU and RAM usage over time
** Summary section outlining machine specs, connection type, storage, ping at startup, if the connection type has changed today (flipped between wifi and ethernet)
** Table containing tracked data for the selected date and session
*** Time, CPU %, RAM %, Connection Type

ExpressPoint has a similar story being worked on right now, we should attempt to replicate where possible → [https://dev.azure.com/Cybersoft-Technologies-Inc/PrimeroEdge%20Classic/_workitems/edit/101533/|https://dev.azure.com/Cybersoft-Technologies-Inc/PrimeroEdge%20Classic/_workitems/edit/101533/|smart-link]

## Acceptance Criteria
Research task to see what device information we can grab via POS and send back to our servers.

----

*POS*

* Track performance data
* Track machine data
* Do not impact service while performing the checks
* Send to server
* Retain for 14 days

*Device Management*

* Main grid gets Performance Filters
* Text updates: Replace all mentions of ExpressPoint with POS, Site Id → Site Code, status → Status
* New column for Metrics

*Metrics*

* User can search by date and session #
* User can see a summary of machine specs:
** CPU, RAM, Drive Storage, Connection Type, Connection Type Changing, Last Tracked, Ping to Server on Startup
* User gets graphical charts for CPU and RAM usage over the selected date + sessions
* User has a table showing session tracking data
