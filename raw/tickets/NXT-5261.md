# NXT-5261 — Direct Certification - Unmatched - Actions

## Description
*user story*

As a district admin I want a way to act on a potential DC match, or to unmatch a previously matched student.

*mockup*

[https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=310084%3A3&scaling=scale-down-width|https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=310084%3A3&scaling=scale-down-width|smart-link] 

*requirements*

* Add actions to the Search & Review:
** Match
** Dismiss
** User needs to enter a comment while performing either action.
** If they match - eligibility needs to be updated appropriately for the student.
** If they dismiss - this should be noted somewhere so that the same offering doesn’t come back up the next time the user searches for potential matches.
* Each action taken needs to generate a history entry for that potential match.
** Action taken, timestamp, user.

## Acceptance Criteria
User can approve a match, or dismiss it.
We need to keep track of these - once acted on, the same extension shouldn't come up again as a potential match.
Eligibility is assigned and updated appropriately for the student being matched.
History tracking of actions taken, when the potential match was first logged, etc.
