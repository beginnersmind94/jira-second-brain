# NXT-49342 — OTP - Profile - SMS

## Description
As a user, I want to be able to setup my SMS One-Time Passcode so that I can use a code sent to my phone to complete login

*Notes*:

* This story is for One-Time Passcode and is for a user to configure SMS One-Time Passcode in their profile
* Other One-Time Passcode user configurations covered in other stories
* Note: on this screen, we need to get or confirm their phone number, test it by sending an SMS code and then saving to DB

*Flow:*

* User browses to Workspace > Profile
* User clicks the edit button for a SMS One-Time Passcode row
* User is redirected to SMS One-Time Passcode edit drawer and completes it
* User saves the config to Keycloak and system marks it as configured
* User tests the config
* User returns and optionally sets One-Time Passcode as the default authentication type

*Requirements*:

*On User Profile > Authentication tab*

* Grid Updates:
** Configured
*** Yes if they have saved the configuration
*** No if they have not saved the configuration
** Method
*** Values correspond to the One-Time Passcode  methods: SMS One-Time Passcode 
** Validator
*** Value is the phone number from the Add / Edit drawer
** Actions:
*** Edit
**** On click, do the corresponding action listed below:



*Edit SMS Drawer*

* If they select to edit the SMS row:
** Open the Configure SMS drawer
** Title: Configure SMS
* 
* Fields:
** Textbox
*** Label: Confirm Phone Number
*** Mask it with a 10 digit phone number format
*** If we have a cell phone number for the user in their account, default the textbox with this value
*** If we don’t, then require the user to enter it and save
** Button
*** Label: Send Confirmation Text
*** On click, send an SMS confirmation text to the number entered
*** Disable the textbox
*** Change the label of this button to Resend and leave it enabled
*** Show the verification fields:
* 
* Fields to show to verify the number
** After SMS test message sent, show these fields:
** Textbox
*** Label: One-Time Code
*** Required if visible
*** Numeric only
*** Required length = number of digits of the code sent
*** User must enter the code they received
** Button
*** Submit
*** On click:
**** Validate they entered in the correct code in the Textbox
**** If yes:
***** Save the phone number to the DB
***** Save the Configured value to Yes
***** Check if this is the first saved configuration
****** If yes, mark Preferred to True
***** Show a success message: Verification completed.
***** Close the drawer and return user to updated grid
**** If no:
***** Show failed message: Incorrect code. Try again.
***** Allow them to try up to 3 times
***** After 3rd failure:
****** Show them final failure message: Verification failed.
****** Close the drawer to end the process
*

## Acceptance Criteria
* User is able to set their SMS cell phone number
* User is able to test sending the code to their number
