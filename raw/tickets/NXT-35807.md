# NXT-35807 — Onboarding - Task List - Add nested tasks, Step# column

## Description
As a Cybersoft administrator, I would like users to perform onboarding tasks in a certain order, so that they don’t skip ahead to tasks that have a dependency on previously incomplete tasks.

*Requirements:*

* Update the existing task selection page UI:
*# Make the card into a traditional grid
*# Add a new column to the card grid: *Step*
*## Display this column to the left of the task name 
*### New columns in this grid: *Step, Task Name, Action*
*### We can remove the existing behavior where the user clicks on the task name/card area, and instead have them click an Edit pencil next to each task in the Action column. This edit pencil will replace the ‘Let’s Setup' hyperlink.
*## For System, use the following step/display order:
*### *Create Sites*
*### *Create Roles*
*### *Create Users*
*### *Assign Licenses*
*### *Manage Settings*
*### *Set Password Policy*
*## Display the description of the task on a separate line.
*### We might be able to use this storybook component, but without the expand/collapse icons: [https://devuicomponents.perseusedge.com/?path=/story/components-grid--with-grouping|https://devuicomponents.perseusedge.com/?path=/story/components-grid--with-grouping|smart-link]
*# For each step, +do not+ allow the user to access the next step *until* they’ve completed the previous step
*## We could possibly do this by displaying the text of the next step in gray and disabling clicks on it, as we did on the Feature Selection screen.
*# When a user ‘completes’ the previous step, they can then click the subsequent step in the list.
