# NXT-4028 — System - Users - Allow Admins to Reset Password

## Description
*User Story*
As a PrimeroEdge Admin, I have a way to either resend a user's password email or trigger a reset on their password if it becomes compromised.

*Requirements*

* One each individual user page, there's a button that the Admin can click that triggers a set password email to an existing user.
* If the user has already created a password for their account:
** Button reads: "Reset Password"
*** Add a confirmation dialogue when clicked: "Resetting the password will invalidate the user's ability to log in until they set a new password. Proceed?" (prompt may not be pictured in the mock-up)
*** User can confirm or cancel
* If the user has yet to create a password for their account:
** -Button reads: "Resend Password Email"-
*** -There is a confirmation dialogue when clicked: "Resending the email will invalidate any prior email that was sent. Proceed?"-
*** -User can confirm or cancel-{color:#ff5630} {color}
** -Clicking will send the same password email that a new user received, but with a new link- {color:#ff5630}(Deferred){color}
** Clicking will invalidate the user's current password and send a password email to the user
* In either case:
** User cannot log in until a new password is set
** If the previously sent email's link has not expired, clicking this button causes the link to expire, regardless of how long it's been active
** A success toaster message should display

## Acceptance Criteria
* Reset User's Password button is present for users who have set a password or users who have yet to send a password
* Both buttons reset password and send appropriate email
* Success message also displays 
* Once clicked, user cannot log in until new password is set
