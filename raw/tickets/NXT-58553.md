# NXT-58553 — WS-PE-Insights-AI Drawer on Insights Screen

## Description
As an Insights role user, I would like to see more detailed analysis of my performance generated by AI in natural language so that I can get more understandable feedback.

*Notes:*

* This story for a drawer on the Insights screen that show greater detail
** There is another story for adding the natural language summary to the top of the Insights screen
* Both backend and frontend are covered in this story

*Flow:*

* User browses to the page
* User sees the summary and wants more details
* User clicks the AI button to show the drawer
* System preps the data using the site and time filters
* System sends the data and a prepared AI prompt to the AI engine
* AI engine analyzes data using prompt and returns it
* System shows the details to the user

*Requirements:*

* On the Insights screen in Workspace, do the following:
* 
* In the Summary Insights section
* Add an icon to view details
** On click:
*** Show the AI Details drawer
*** Use a sliding motion to open
* 
* Drawer
** Title: AI Recap Details
** Add the drawer per Dallas’s Bolt mockup
** It has 3 sections:
*** Summary
**** It is a summary (paragraph) text section
*** What went well
**** this section shows the top 5 things that went well
*** What didn’t go well
**** this section shows the top 5 things that went well
** 
** Data Set
*** The results are dependent on the user’s site access
**** If the user only has access to one site, results show only for that one site, etc.
*** The results are dependent on the time frame selected on the Insights screen
*** Pass in a dataset with these site and time filters
* On page load:
** System gets the dataset using the site and time filters
** System sends the dataset and a previously prepared AI prompt to the AI engine to perform the analysis 
** AI engine does its analysis and returns the result
** System loads the result in the AI section
* Show a loader icon while waiting on the AI engine. If you can, include text to let them know we are waiting on the the AI engine.
* 
* Prompt Text:
** Ask AI to summarize in paragraph format what went well and what didn’t go well and any trends if the data is for multiple days
** Ask AI to analyze for the top 5 things that went well
** Ask AI to analyze for the top 5 things that didn’t go well
* 
* Security
** This section inherits the page’s role based security, no additional security needed beyond the AI district check
* Error Handling:
** If there is an error in retrieving or processing the AI summary (e.g., failure to contact the AI engine), an appropriate error message should be displayed to the user.
* System Performance:
** The system must load the AI-generated summary within a reasonable amount of time, aiming for a smooth user experience without significant delays.

## Acceptance Criteria
# *AI Details Drawer Icon*:
#* An icon to view detailed AI analysis must be added to the Summary Insights section.
#* The icon should be visually consistent with the other icons on the Insights screen and should be clearly identifiable.
#* When the user clicks the icon, the AI Details drawer should open using a sliding motion.
# *AI Details Drawer*:
#* The AI Details drawer must be implemented as per Dallas’s Bolt mockup.
#* The drawer should be titled "AI Recap Details" at the top.
#* The drawer should contain three sections:
#*# *Summary*: This section should display a paragraph text summary generated by AI.
#*# *What Went Well*: This section should display the top 5 things that went well, as identified by the AI engine.
#*# *What Didn’t Go Well*: This section should display the top 5 things that didn’t go well, as identified by the AI engine.
#* The drawer must open with a smooth sliding motion when the icon is clicked and close similarly when the user clicks it again.
# *Data Set Handling*:
#* The AI analysis and resulting details must be based on the user’s site access:
#** If the user has access to multiple sites, the AI engine should aggregate results across all accessible sites.
#** If the user has access to only one site, the results should be specific to that site.
#* The analysis and details must also depend on the selected time frame on the Insights screen:
#** The system should pass in the dataset with the appropriate site and time filters.
# *AI Engine Integration*:
#* On page load, the system must:
#*# Retrieve the dataset using the site and time filters.
#*# Send the dataset and a pre-defined AI prompt to the AI engine for analysis.
#*# The AI engine should return the following details:
#*#* A paragraph-style summary of what went well and what didn’t go well, and any trends (if the data spans multiple days).
#*#* A list of the top 5 things that went well.
#*#* A list of the top 5 things that didn’t go well.
#*# The system should load the returned result into the appropriate sections of the AI Details drawer.
#* A loader icon should be shown while waiting for the AI engine’s response, with optional text indicating the system is waiting for the AI engine to process the data.
# *Role-Based Security*:
#* The AI Details drawer must inherit the page’s role-based security settings.
#* No additional security is required beyond the AI district setting check to determine if the drawer is visible or not.
# *Error Handling*:
#* If there is an error in retrieving or processing the AI summary (e.g., failure to contact the AI engine), an appropriate error message should be displayed in the drawer, informing the user of the issue.
# *System Performance*:
#* The system must load the AI-generated details (summary, "What Went Well," and "What Didn’t Go Well") within a reasonable amount of time, ensuring a smooth user experience without significant delays.
#* The AI engine's response time should not negatively impact the user experience.
# *Prompt Text for AI Engine*:
#* The AI prompt should include the following:
#** Request the AI engine to summarize in paragraph format what went well, what didn’t go well, and identify trends if the data spans multiple days.
#** Ask the AI engine to analyze and display the top 5 things that went well.
#** Ask the AI engine to analyze and display the top 5 things that didn’t go well.
# *Drawer Behavior*:
#* The drawer should be initially hidden, with the user able to open it by clicking the icon.
#* The drawer should close if the user clicks the icon again, using the same sliding motion.
#* The sliding motion should be smooth and visually clear to the user when the drawer is opening or closing.
# *System Feedback*:
#* While waiting for the AI engine’s response, the system should display a loader icon in the drawer. If possible, the loader should include text such as "Analyzing data..." to inform the user of the processing status.
