# NXT-9540 — MP > Menus > edit Menu Cycle screen - add Copy feature

## Description
As a menu planner, I need to be able to copy menu cycles so that I can work quickly.

Add a "Copy" button near the "Save" button that opens a pop up like this:

!MPCycleCopy.jpg|width=200,height=183!

 

which allows the user to change the copied Menu Cycle Name, Meal Pattern, Meal Type, Site Group, Number of Weeks, and Days per Week. The user can also select which (appropriate) Serving Groups the copy should have. (Note that these may differ from the Menus in the source Menu Cycle.) 

On Cancel, abort the operation and return the current Menu Cycle.

On Copy, make a copy of the Menu Cycle and each of the menus within the Menu Cycle, and navigate to the copy of the Menu Cycle.

* if the Menu Cycle Name is not unique giving a message “A copy could not be created. The name must be unique.“ and keep the popup open.

* on success give message “Menu Cycle saved successfully.“

Logically this should be using the Menu Copy feature to copy each of the Menus for including Serving Groups, Meal Counts and Item Counts.

There needs to be a process to name each of the copied Menus. Use the copy’s Menu Cycle Name + “ Week x Day y”.

Copy the Menu Cycle, then copy Menus, if a Menu results in a non-unique name then do not copy that Menu and leave the Menu assignment blank for that day and give a message “One or more Menus could not be copied. These are left blank.”

## Acceptance Criteria
Copy button added next to the save button
which displays a popup similar to mockup
User can change the name, Meal Pattern, Meal Type, Site Group, or the selected Serving Groups
Menu Copy routines work the same way as in 9492
Menus are copied if possible as described
