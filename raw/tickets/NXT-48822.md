# NXT-48822 — Temp User - Edit User Settings

## Description
As a SchoolCafe admin, I would like to be able to edit a temp user settings so that I can extend their account.

*Notes:*

* If a temp user’s time has expired, changing the end date is how to give them more time
* The system should set the Status to Inactive when the time expires, so when you re-enable by setting the end date to the future, you need to reactive the account

*Requirements*:

* On the System > Users > Edit User drawer, do the following:
* Add Account Type like on Add New
* Add Start Date like on Add New
* Add End Date like on Add New
* Apply same requirements and business Logic like on Add New
** Except default the values to what they are in the DB
* 
* Allow the Admin to change a temp account to a perm account and the other way too
* 
* On Save click Updates:
** If validated:
** Check the account status if the user extended the End Date forward
*** If the account status is Inactive for a temp user and the end date is in the future, then ask the user if they would like to mark the account as Active
*** If they say yes, then update the status to Active
*** If they say no, then don’t update the status
** Check the account status if the user marked the End Date backwards
*** If the account status is Active for a temp user and the end date is in the past, then notify the user that their status will be marked Inactive and ask them to confirm if they are sure before saving
*** If they say yes, then update the status to Inactive
*** If they say no, then don’t save changes
** If changing a temp account to a perm account:
*** Update account type
*** Remove the dates
*** If in-active, ask user if want to make active and do it if they say yes
** If changing a perm account to a temp account:
*** Update account type
*** Require the dates
*** Enforce usual rules for temp accounts
* 
* Changes are logged to History

## Acceptance Criteria
* Account Type is added to the screen
* Start Date and End Date fields added if Temporary account type and required if visible
* Temporary dates and auth type are included in the save button
* User is given the option to set account status according to the requirements for temp users if they change the dates
* If end date is set to the past and saved, the system inactivates the account
* Users can be changed between temp and perm account types
