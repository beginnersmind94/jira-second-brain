# NXT-11636 — Integration - Enable support for code mapping

## Description
A user performing an import would like the ability to map a data value from a field in a source file into a corresponding value within Perseus.

For example, an import file could have a “Gender” column which contains the values “M” or “F”, representing “Male” and “Female”.  In Perseus, these values would be represented as “Male” or “Female”. As such, the user would like the ability to define a rule whereby when the import occurs, the values “M” from the source file are persisted in Perseus as “Male” and the values “F” are persisted as “Female”.

## Acceptance Criteria
The customer shall have the ability to define a rule that will allow mapping a data value from a source file's column into a known value of a Perseus destination column.
Customer shall have the ability to easily enable/disable/delete the rule.
2.0 Pilot stretch goal:  The customer shall have a UI available to define, manage, activate, deactivate mapping rules.  Otherwise, a script and/or config file shall be made available (using Azure Data Factory conventions?) to enable said rules.
