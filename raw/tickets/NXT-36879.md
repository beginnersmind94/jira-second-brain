# NXT-36879 — Student Import – Don’t Allow Optional Field to Stop Row Import

## Description
As a SchoolCafé admin, I want to be able to import the record if an optional field doesn’t validate so that I can get started with using the system.

*Flow:*

* Open SchoolCafé and browse to Settings > Imports & Exports > Imports
* Find a Student Import you want to use
* Click on Import for that row
* Follow the import process
* When the import routine comes across a row that fails because an optional field causes it to fail, update the field, log it as an exception and keep on importing the rest of the records
* Notify the user when the import finishes

 

*Notes:*

* This story comes from Dallas and Customer Care team during implementation of River Road. They have concluded that it is better to have a successful import with exceptions rather than completely abort the import because of a few bad records.
* Therefore, the point of this story is that we can’t let a bad import record cause the whole import to fail.
* Note this story applies only to failures caused by optional fields; required field issues are covered in another story

 

*Requirements*:

* When click on Import button for a Student Import:
* When running, if there is a record that fails because of an optional field, do the following:
** Currently it causes the import to fail – don’t do this
** If it fails because the text is too long, trim it to its max length
** Else if it fails because of any other reason, empty the field
** Record if you trim or remove text in the history log for that record
** Continue processing the remaining records
* 
* When creating the import file and testing if the import file is accurate
** Continue to test the same way you do currently, no changes
* 
* When viewing the History log from Import Monitor:
** Ensure the exception is logged and visible in the import log
** If there is not already an exception text and code for these, use these:
** If trimmed:
*** Exception text: [Field Name] trimmed
*** Assign it a code
** If text removed
*** Exception text: [Field Name] text removed
*** Assign it a code
* 
* Ensure that the number of exceptions are countable
** Future stories will need to display the amount of exceptions
* 
* Permissions: No changes necessary

## Acceptance Criteria
* When importing, if there is a record that fails because of an optional field, the import routine repairs the field and continues
* Record imports that are trimmed or emptied are marked as exceptions and visible in the history log
