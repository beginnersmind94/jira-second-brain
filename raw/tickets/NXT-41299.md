# NXT-41299 — FTP & Email – Framework - Part 2

## Description
We need to create an FTP Framework so that all modules can leverage an existing FTP and not have to write their own.

*Notes:*

* We need this for Data Query and for eOrder features for now, with more to come
* No front end for this story
* There are conceptually 2 parts to the framework:
** Configuration
** Execution
* Users will configure the FTP in the first phase
* Then later, they will use the configuration to execute a transfer

*Flow:*

* Source page needs to send something FTP
* It inserts a record into the FTP framework
* FTP framework gets the configuration needed to send the file
* FTP framework sends the file and notifies the source that it is complete

*Configuration Requirements*:

* This needs to be framework style, meaning that it can be used by multiple modules and features
* There needs to be a way to save an FTP Configuration

 

*Execution Requirements*:

* There needs to be a way to tell the framework it needs to execute an FTP file transfer
* There needs to be a way to tell the framework to use a certain FTP Configuration to send the file
* There needs to be a way to save files somewhere and link them to the execution
* The framework needs to be able to notify the source of the completion status
* All attempts to FTP files must be documented in a History table regardless of success
* The framework must be able to support sFTP, ActiveFTP and PassiveFTP
* 
* From my perspective, you might need these things at a minimum:
* Database FTP Configurations
** Columns:
*** ID
*** Name
*** Description
*** Source
*** Host
*** Port
*** Folder
*** Username
*** Password
*** FTP Type
** Stored Procedures:
*** Insert
*** Update
*** Delete
*** Get Config by ID
*** Get All Configs
* Database Config History
* Database FTP Execution
** Columns:
*** ID
*** ConfigID
*** User
*** Date/Time Created
*** Date/Time Executed
*** Status
** Stored Procedures
*** Insert
*** Update (Date/Time Executed and Status after runs)
*** Get Execution by ID
*** Get All Executions
