# NXT-22086 — Platform - Register an Edge Device in Device Management

## Description
As a site administrator, I want to be able to register Edge devices for Face and Food detection for my district so that I can use them for contactless face and food detection at the school’s POS devices.

*Application*: SchoolCafé Website

*Flow:*

* Browse to SchoolCafé and click the link for System in left menu and then Device Management link
Click on Register New device
Complete the form and submit

*Background:*

* This already exists in classic
** POS > Food and Face Detection > Device Management (see screenshot)
* You need to do the same thing for 2.0 that is done for Classic
Talk to Saketh if you have any questions
Just copy the code from classic for the APIs and flow
* In 2.0, we will have to add in some missing fields that are needed by other stories
** Fields missing:
*** Terminal
Registered
Device Mode

*Requirements:*

* System Settings > Device Management
** No changes from this field
*  
System Settings > Device Management > Register New Device
* Changes to existing fields as follows
** Model field
*** Already exists
Add the following new logic:
If the user selects a model that is an Edge Device, show the Edge Registration grouping (see below)
By default, the Edge registration grouping is not visible
if the user selects an Edge Device model and then selects something else, hide the Edge Registration grouping
** Register button
*** Upon click, add validators for new required fields in Edge Registration grouping
**** Validate only if have selected and Edge Device model
*** If validated and if it is an Edge Device model:
**** Add in code to register the device with Amniscient PaaS portal
***** Copy the code from Classic
Just do what they do, it is already done and works
**** Update the status per response from the API the same way classic does
It sets the Edge device’s mode the same way as in classic
*  
* Add a grouping section under the Register New Device grouping
** Title: Edge Registration Fields
* Add the following fields to the Edge Registration grouping
** Terminal
*** List, single select, Required, no default selected
*** Values = list of terminals at the select site
**** Site = Site Name field that already exists in the Register New Device section
** Registered
*** this is a date stamp text field, read only
when registering, it has no value
*** when editing, it will show the date that the device was registered
**** a device is registered when they successfully click the Register button
** Device Mode
*** List, single select, Required, default to Detection
Values = Detection, Learning
*** There are API calls connected to this field that you will need
**** See existing classic code for how and what to do

## Acceptance Criteria
* System Settings > Device Management > Register New Device
* Changes to existing fields as follows
** Model field:
*** If the user selects a model that is an Edge Device, it shows the Edge Registration grouping
if the user selects an Edge Device model and then selects something else, it hides the Edge Registration grouping
** Register button, only if have selected and Edge Device model:
*** Upon click, it validates required fields in Edge Registration grouping
If validated, it registers the device with Amniscient PaaS portal exactly the same was as in classic
It updates the status per response from the API the same way classic does
It sets the Edge device’s mode the same way as in classic
*  
The Edge registration grouping is not visible by default
There is a grouping section under the Register New Device grouping called Edge Registration Fields
* The following fields are added to the Edge Registration grouping
** Terminal
*** It is a list, single select, Required, no default selected
Values are list of terminals at the select site
** Registered
*** This is a date stamp text field, read only and blank before registering
When editing, it will show the date that the device was registered
** Device Mode
*** It is a list, single select, Required, default to Detection
Values are Detection, Learning
API calls connected to this field are called as needed
* Is visually consistent with the rest of the page
