# NXT-3047 — **System - Organization - Users Details - Edit User

## Description
As an administrator I want to edit the details of an existing user's account, so that I can easily make changes to it as needed.



PrimeroEdge Classic Reference: System > Sites and Users > Sites > Users > Select individual username



* This story should result in allowing the administrator to edit ALL details related to a specific user, using a set of required fields, in the 4 sections related to the user’s access level within PrimeroEdge: Profile, Password, Region, and Role. 
** Page should only allow user to edit if the permissions of the logged-in user’s role allows.
* Existing fields/checkboxes/menu content in each section can be carried over from PrimeroEdge classic.
* Password validation rules are addressed in a separate story
* *Required fields to update a user*: Username, First Name, Last Name, Organization Level, and at least one Role. 
* The layout for each of the 4 sections is negotiable.
* Aim for a cleaner view of the page with better UX for the page controls (for example, we could potentially reduce amount of scrolling required by splitting content into columns). 

(History tab and De-Activate User buttons can be skipped and addressed in separate stories if needed)

!image-20200203-202709.png|width=988,height=730!

## Acceptance Criteria
1. Users with edit permission can modify the details in any of the 4 sections of a user's details page
2. Required fields and one role must be selected for the user to be updated. 
3. If not all required fields contain data, the button to update the user's profile should be disabled.
