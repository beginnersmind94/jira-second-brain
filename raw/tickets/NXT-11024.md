# NXT-11024 — Notifications - SchoolCafe - Automatically send scheduled Applicant Determination Results

## Description
As a central office admin, I want parents/applicants to receive an automated notification letter via email when their application is submitted and automatically processed, so that I have fewer notifications to send out manually during the day.

*Mockup*

N/A

*Summary*

Add a new toggle setting for automated notifications. If this setting +*and*+ the district’s automatic processing setting are enabled, then (at a certain time, set by the user) when a submitted application fits the criteria to be automatically processed, let’s also automatically take the app straight to ‘Notified’ and send the notification email. This should also cause the letter to appear in the applicant’s Family Hub account.

*Requirements*

* In System → Settings, in the (Common) Eligibility → Applications section, add an automatic notification setting:
** Setting Name/Description: *Automatically notify completed applications*
** Type: Toggle
** Tooltip: “_When auto-processing is enabled and the system processes an app, this will also automatically send an approval/denial notice._”
** Default state: Disabled
** *(*[~accountid:5e17ecfbefa19b0e976be845] *will send the name of an existing setting we may leverage instead, if possible).*
* {color:#ff991f}-In Eligibility → Notifications, add a toggle to the grid (similar to in Verification Tracking):- {color}{color:#ff991f}*-Auto Approval/Denial Notice-*{color}
** {color:#ff991f}-Add a tooltip next to the toggle: “When auto-processing is enabled and the system processes an app, this will also automatically send an approval/denial notice if Notification Scheduling for Approval/Denial notices is set.”-{color}
** {color:#ff991f}-This toggle should be disabled by default, which means no functional change/auto-processing should take place-{color}
* If this toggle is enabled: then when an application is automatically processed *and* contains an email address, do the following: 
** Check System → System Notifications: If there is an active +*Real-Time scheduling*+ with ‘Email’ as a Channel:
*** email a copy of the active approval/denial notice letter to the applicant. 
**** The content should be ‘Letter’ or ‘Inline’ based on the ‘Content Type’
*** push the application status to _Notified_. 
*** Finally, verify that the notification letter also displays in Family Hub (this should happen automatically and only need testing).
* *Deferred*: 
* Not sending the notification for reprocessed applications if there is no eligibility change. For now, we can just send it in any scenario.
* Scheduled Email Timings
** For scheduled email timings, we will also use the scheduled notifications functionality in the System module. The Platform team has developed this infrastructure to automatically send notifications according to a schedule. We can coordinate with them as necessary.

## Acceptance Criteria
* Verify that a new automated notifications toggle setting has been added to Eligibility → Notifications according to the specifications above 
* Verify that the setting is disabled by default, and only functions when the automatic processing setting is also enabled
* Verify this new setting also affects categorical apps that may be auto-processed when that other setting is enabled 
* Verify that if there is an active Real Time scheduled notification for the Approval/Denial Notice
** impacted apps, when an email address exist (and not otherwise), will cause the system to trigger an approval/denial notice to be emailed to the app’s contact person, the app to be taken to ‘Notified’ status, and a notification created in the Family Portal
* Verify that the email is sent out at the timing/interval specified in the Notification Scheduling page
* Verify that the system behaves as it currently does when the setting is disabled
