# NXT-54443 — DMF - Realtime Status update

## Description
As a Data Migrator, I need the status of each step in real-time.
Currently, we need to wait for the ADF of each DB migration to add the current status in the respective table so that it is reflected in the UI.
In this story, we will provide a real-time status for each migration.

*Change:*
When the user clicks the New or Not Started or Re-run button, and once all the underlined APIs are successful, the UI will change the status from Not Started to Running.
API will add an entry to the DE_DistrictMigrationLogDetail table with StepName as Starting Data Migration for Source DB for System DB where Source DB and 
System DB will be variables. API will add the respective OrderId, MigrationLogId, ImportKeyId and StatusId as 1.
SQL query will use the ImporKeyId available in the DE_DistrictMigrationLogDetail table, to get the accurate status of ADF using the DE_ImportHeader table.
For ItemRecipe and Eligibility DB Destination migration, we need to create a new endpoint in the ItemManagemnet and Eligibility DB repo to insert the data in the DE_DistrictMigrationLogDetail table. This endpoint will be called once the Start API is successful.

## Acceptance Criteria
The status of each migration needs to be available in the UI in real time.
Users should not wait for 30 seconds for the grid to refresh to see the exact status.
