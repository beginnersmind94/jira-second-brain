# NXT-6350 — Messages - New Broadcast Message

## Description
*User Story*

As a PrimeroEdge admin, I would like to have a form for posting or editing a broadcast message.

*Requirements*

* Within the Broadcast Message page, when the user selects the Broadcast Message button, display a *New Broadcast* window on the page. 
** Window appears on the Broadcast Message screen - [https://primeroedgenext.atlassian.net/browse/NXT-5914|https://primeroedgenext.atlassian.net/browse/NXT-5914|smart-link]
* This same window can be accessed/reused for both creating a new message or editing an existing one
* The window should include the following fields:
** To - Allows the user to determine the audience for the message
*** User can search between -groups- ({color:#ff5630}deferred{color}) or individual names (First + Last) of *active* users.
**** List should be a smart search
**** Sort both role and user lists in ascending order
**** Add a ‘Select All’ option to each list that is selected by default but can be deselected. 
*** Any number of individuals can be added and the Recipients/To field should update as a result.
** Message Text - Entry for whatever message the user wishes to send
*** Rich text message 
*** 150-character limit
*** Allow user to add hyperlinks (and possibly images if feasible) that will display along with the message (Covered in separate story)
** Broadcast On - Allow user to post the broadcast at a future date and time.
*** Field should auto-populate with the current date and time of the user. It is required.
** Expires On - field that determines when the broadcast will no longer be displayed to the user (regardless of whether they dismissed it or not)
*** Field is blank by default and is optional (if left blank, banner displays forever until dismissed)
** Post - Posts the message either immediately or at the scheduled time.
* A successfully posted broadcast message should display:
** …in the table/list on the Broadcast Message page.
** …at the top of the screen of a user who is in the ‘To’ audience for the message.
*** Covered in [https://primeroedgenext.atlassian.net/browse/NXT-5138|https://primeroedgenext.atlassian.net/browse/NXT-5138|smart-link]
** …in the Broadcasts tab of the user’s Workspace → Inbox section.
*** Banner/message should only display on the scheduled date and time.

## Acceptance Criteria
* Verify that New Broadcast message form is available from the Broadcast Messages page when the Broadcast Message button is selected

* Verify that new window opens with To, Message text, Schedule/Expires options and Post buttons options available

* Verify both mailing groups and names (of active users only) can be selected in the TO field

* Verify Message text has a character limit and counter of 150

* Verify at least a hyperlink can be attached to a message

* Verify a message can be scheduled for a future date and time or posted immediately

* Verify that Broadcast On is auto-populated with the current date and time of the user. Verify field is required.

* Verify Expires On determines when the broadcast will no longer be displayed to the user (regardless of whether they dismissed it or not). Verify field is blank by default and is optional

* Verify that New messages can be created  successfully based on the scheduled time
