# NXT-70426 — Add Segment Column Checkbox to Financials Report Configuration

## Description
*As a* district administrator configuring the Financial Oversight module, *I want* to control whether GL account codes are split into individual segment columns in exports, *So that* my finance team receives data in the exact column structure they need to import into Oracle or Tyler Munis without manual reformatting.

## Acceptance Criteria
* The checkbox appears on the Reports tab, under the General Ledger & Journal section, as a toggle labeled *"Split account code by segment."*
* Below the toggle, display helper text in small, muted text: _"When enabled, each segment of the account code appears as its own column in the export, using the segment name as the column header. The full account code is also included as a reference column."_
* The toggle defaults to *ON* for all districts.
* *When ON:* Every ledger and journal export includes one column per segment (using the segment names defined in the district's account code structure configuration as column headers), plus one column labeled "Account Code" containing the full combined code.
* *When OFF:* Every ledger and journal export includes only the single "Account Code" column. No segment columns are produced.
* Changes to the toggle take effect on the next export generated. Previously exported files in the Document Center are not affected.
* *Validation gate:* The toggle is *disabled* (greyed out, non-interactive) if any *active* GL account does not conform to the configured account code structure. On hover over the disabled toggle, display a tooltip: *"[X] active account(s) do not match the account code structure. Go to GL Accounts to correct them."* where [X] is the count of non-conforming active accounts. "GL Accounts" is a link that navigates the user to *Configuration > General Ledger Accounts > GL Accounts*. Once all active accounts are valid, the toggle becomes interactive and retains its last-set state (or the default if never set).
* This checkbox setting only applies to ledger and journal exports. It does not affect other export types or on-screen display.
