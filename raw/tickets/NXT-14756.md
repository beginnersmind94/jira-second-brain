# NXT-14756 — Acct Mgmt - Notifications - Integrate Account Balance report within page

## Description
As a Manager, I would like to be able to print a list of accounts whose balances fall within a set of criteria, so that I easily can reach out to them for marketing or reminder purposes. 

*Summary*:

Within the Notifications page, integrate the necessary options to add an Account Balance report which displays a list of accounts and their balance (and potentially contact info), based on some criteria. Basic functionality can mimic classic.

*Details*:

* In the Account Management - Notifications page, add a toggle to display *Report Options.*
* When the toggle is off (default state), the page functionality remains the same as it currently is.
* When the user toggles this on, display the Report Options to allow users to print an Account Balance report (formerly ‘Patron Balance’ in Classic) 
** Report Options-
*** Display: Contact Info
**** Display the contact info of the Primary Guardian of the account across several columns (as needed). See Classic image below for an example.
*** Group by: Site
**** Group each account on the page by the enrollment site of the account.
*** Sort by:
**** Alphabetical: Within the grouping, display the accounts in alphabetical order by the account’s Last Name.
**** Homeroom: Within the grouping, display the accounts in alphabetical order by the account’s Homeroom.
*** Date as of:
**** If possible, display the balances of each account as of a specific date, which the user can enter here (default should be today’s date).
* When the user selects the ‘Apply’ button
** IF the ‘Report Options’ toggle is off, display the Notifications grid as normal.
** IF the toggle is on, display the Account Balance report. We can follow the same structure as Classic:
*** ID
*** Account Name (formerly ‘Patron Name’, which is First Name +Last Name)
*** Site
*** Grade
*** Homeroom
*** Status
*** No Charge Reason
*** Eligibility
*** Balance
*** Contact Info
**** Guardian Last Name
**** Guardian Last Name
**** Street Address
**** City
**** State
**** ZIP (formerly ‘Zip’)
**** Phone #
*** Site Total
*** Grand Total
*** Also, display the user’s report option selections in the header/footer of the report

## Acceptance Criteria
* Verify that, within the Acct Mgmt Notifications page, Report Options have been added
* Verify that the Report Options are toggled off/disabled by default
* Verify that when the toggle is enabled, the Report Options display as shown above
* Verify that when the toggle is disabled and the user selects ‘Apply', they view the Notifications grid as normal
* Verify that when the toggle is enabled and the user selects ‘Apply’, the Account Balance report displays, according to the options selected
* Verify that the report is updated appropriately/logically when various report options are changed
* Verify that the report is displaying the report option selections in the header/footer of the report
