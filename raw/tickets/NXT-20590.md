# NXT-20590 — Eligibility Notifications - Add Income Surveys tab, support bulk notifications

## Description
As a district admin, I would like to be able to notify multiple households that their Income Survey has been received/accepted at once, so that I can quickly finish my notifications tasks and be able to spend more time doing other tasks.

*Summary*

Add an Income Surveys tab to the Eligibility Notifications page and allow a user to email and/or print an Income Survey Notice for one or more surveys.

*Requirements*

* Within the Eligibility → Eligibility Notifications page, display a tab for *Income Surveys*
* This tab should only display if the Show Income Surveys setting is to the ‘Yes’ or ‘Mixed’.
* When the user selects the tab, display a UI similar to the Applications tab:
** User can enter a date range (within the current academic year only) to search for all surveys with a Processed Date within that range
*** Note: User can +only+ see surveys that are in Processed and/or Notified status. 
** User can select an option to exclude previously notified surveys.
*** This should omit any survey in that range that has previously been sent a notification after the last processed date.
* Upon searching, the user should see search results in a grid with the following:
** *Default Columns:*
*** Survey #
*** Status
*** Processed Date
*** Processed By (First Name and Last Name of the Processor)
*** Notified On (if available)
*** Language
*** View
**** Should display a letter icon to launch the notification viewer in order to preview.
** *Hidden columns* under Filter. Guardian information should be per the person who submitted the survey.
*** Eligibility
*** Student Names
*** Submitter Name
**** First and Last Name of the person listed in the survey’s ‘Contact Info’ step (if it exists, like the other contact fields below)
**** Otherwise we can follow sibling contact info rule from Applications.
*** Address
*** Email
*** Phone #
** All of these columns should have the same or similar sorting/filtering behavior as exists on the Applications tab.
** Notification Options
*** Print
*** Email
**** Only applicable if an email address exists
**** User should be able to select one, many, or all notifications within the grid and send out a Survey Notice for them. After notifying, a success message should display and the related survey(s') details, such as “Notified On” date should get updated.

## Acceptance Criteria
Verify that this tab should only display if the Show Income Surveys setting is to the ‘Yes’ or ‘Mixed’.

Verify that an ‘Income Surveys’ tab is added to the Eligibility Notifications page

Verify that the user can select a date range within the current academic year

Verify that the user can elect to include or exclude previously notified surveys

Verify that when the ‘Apply’ button is selected a grid appears, displaying all surveys based on the criteria above

Verify that the grid shows the default columns listed above, including a ‘View’ column with a selectable icon 

Verify that when selected, the ‘View’ button opens the Notification viewer for that individual survey

Verify that a user can print or email a ‘Survey Notice’ for one, more, or all surveys within the grid

Verify that when the user prints or emails out notices, the print dialog opens and/or the email goes out

Verify that the survey data (and grid, if needed) for the related survey(s) is then updated at that point
