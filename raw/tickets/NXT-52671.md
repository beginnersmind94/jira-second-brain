# NXT-52671 — PE - GDSN Connect - Pending Updates - Inclusion of Data Source ID (Upload)

## Description
_(none)_

## Acceptance Criteria
Currently, the data source ID is not captured while the user is uploading the pending updates data



UI Changes:

The user should be able to select the data source and then click on upload

* Show default as Local if available (applicable to region only)
* Show - “Select” otherwise, Validate the user “Select Target Data Source” in case if its not selected
* Validate the user with confirmation message before upload (Are you sure you want to proceed with uploading data with “selected data source”? OK and Cancel options
* The data should be uploaded against the corresponding data source ID (inclusion of new field in DB)
* Exceptions: If a specific Ingredient ID or Stock Item or Linked Stock Item are not found in the data source - mark them as exceptions (As: Item not found at data source - such records should not be inserted
** If IDs are blank in both the District Menu ID and District Item ID - then the should be remarked as “Invalid Item” - such records should not be inserted
* All the duplicate conditions will work as-is if an valid item ID is found
* Data Sources:
**  Region level - default is Local
** Central Realm - Prompt user to select

+Regression+:

The Pending updates should only load the items related to selected data source 

Central Realm scenario should be considered for shared data sources (Ex: PDE Shared)

The sync history, audit history, bulk updates should work as-is for both new and reviewed items
