# NXT-44392 — MP- Add Menu item from Quicklist- Business Rules for VMI and Child Items

## Description
As a menu planner, the system should prevent me from being able to add child items that are already part of a VMI onto the menu from the quicklist

## Acceptance Criteria
* We continue to allow users to add VMI and child items onto the quick list together if they would like to
* IF User adds a VMI either through the quicklist or through add menu item, restrict the user from being able to add any child item from the quick list (Exactly the same way we prevent from Add menu item)
** Gray out the check box and display tool tip that says “Menu item added on Variety Menu Item” 
* IF User adds a child Item either through the quicklist or through add menu item, then tries to add a VMI that also contains that child item, the VMI will take priority and consume the child item 
* IF User adds a child item and a VMI with child item at the same time as part of the quicklist, the VMI will take priority and consume the child item once they are added
** Info message “ Menu item(s): '*Menu Item Name”*, “*Menu Item Name*” has now been added as part of the Variety Menu item” (green Font)
*** Display at the top of the page once page reloads with newly added menu items, if there happens to be multiple menu items that were already on the menu then display the multiple menu item names within the message 
*** Only display on current refresh, if user leaves page and comes back do not display again, allow user to manually remove the message

This should apply to both adding to templates or menus
