# NXT-30215 — Acc. - Block session / deposit / transaction updates if POS Period is closed

## Description
As Vera the Director I don’t want anyone to be able to add/edit sessions, transactions or deposit slips if the POS Period is closed.

## Acceptance Criteria
When the POS Period is closed the following actions are blocked.

* Opening a session for editing
* Closing a session
* Reconciling a session
* Creating a Deposit Slip
* Editing a Deposit Slip
* Editing a Transaction dated during the closed period.
** Transaction details can be viewed, but cannot be saved. Save button = disabled.
** Note added to the left of the save button: The POS Period for this transaction is closed. You must open the period to save any changes.
* Adding a transaction dated during the closed period.

The buttons for these actions would be disabled. Save button disabled.

User is informed why via a tool-tip: This option is disabled because the POS Period is closed.
