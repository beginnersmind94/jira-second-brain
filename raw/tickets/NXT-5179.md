# NXT-5179 — Device Management – Edit POS Device

## Description
As a SchoolCafé admin, I want to be able to edit an existing POS device so that I make the necessary changes.

*Flow:*

* User logs in and browses to System > Management > Device Manager
* User clicks the edit button on a record
* Edit POS Device drawer opens
* User completes form and Saves
* System sends Edit POS Device to FO Team’s Edit POS Device API 

*Notes:*

* This is the continuation of the Device Manager stories.
** 1^st^, we created the APIs in Front Office
** 2^nd^, we updated the device manager screen
** 3^rd^, we created the Add New POS drawer and are able to add new devices
** Now, we will do the edit functionality for a POS device
* This screen is identical to Add New drawer, except we save changes to an existing record rather than creating a new one
* Therefore, if you can, copy that drawer as much as possible
* Note: this feature requires the work of both System and FO teams.
** By the time we start this story, FO team must have written the relative APIs for this drawer
** And the Add New functionality must have been completed

*Requirements*:

* *Remove Edit Device Registration*
* Right now, when the user clicks the Edit button, it opens the Edit Device Registration page
** We don’t want to do that anymore
** Take the link to that page away from the button
** Remove Edit Device Registration from the page resources
** Hide that page from the user
** Don’t completely delete it as it has all the code for the FFD device management
** Hold onto the code in case we need it one day
** In its place, add a link to the Edit POS Device drawer
* 
* *Edit POS Device Drawer*
* All of these requirements are for the Edit POS Device drawer
* Create a new drawer and link to the edit button on the grid rows
* Make it look exactly like the Add New POS Device drawer except change the page title
** Just copy that code exactly as is
** Only changes:
*** Title = Edit POS Device Drawer
*** Save button behaves differently, so covered in detail below
** 
* Using the design in the Figma, add the following fields
* Device Name
** Textbox
** Required
** We need to get max length from FO team
* Site Name
** Single select dropdown
** Values: site names list
** Required
** Sort alphabetically
** Note: Send ScoolID instead of name in API
* Device type
** Single select dropdown
** Values: Device Type API list
** Required
** Sort alphabetically
* {color:#ff5630}Use POS Beta Site{color}
** {color:#ff5630}Toggle{color}
** {color:#ff5630}Values: Yes, No{color}
** {color:#ff5630}Required{color}
** {color:#ff5630}Default = No{color}
* Description
** Multi line textbox
** Note Required
** Max length = 255
* Cancel
** On click, return to the Device Manager Screen
* Save button:
* On click, do the following:
** Validate required fields
** Validate that the Device Name does not already exist
*** Use DeviceName API from FO team for device name list
** 
** If validated:
** Use FO Add New POS Device API to send them the data
** When call API, add the following hidden fields:
***  User – logged in account, needed for history table
** When call API, send SchoolID instead of School Name
** 
** Show a message to the user that the POS Device is being updated
*** Wait a few seconds for the success response and
*** then show them a message that it has updated successfully for a second
*** then close the drawer and show them the device manager
** When they come back to the device manager screen, update the table with the new device in it
* 
* Security:
** Roles & Permissions actions:
** Don’t show this page in Roles and Permissions since it is effectively the Edit value of Device Manager page
** Therefore, anyone with Edit rights to Device Manager & Cybersoft admins can use this page
