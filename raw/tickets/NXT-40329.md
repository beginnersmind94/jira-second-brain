# NXT-40329 — eOrders Configuration – Add New Part 2 – Vendor Tab

## Description
This is a continuation of the eOrders Configuration – Add New Part 1 and covers the Vendor Tab

* Flow:
** Select the vendor and format type
** Type will drive the fields on the next tab
* 
* If the Vendor tab is not already created, then create it and do the following on it:
* 
* Fields:
* Vendor
** Single select list
** Required
** Values:
*** List of vendors
*** Get API from Back Office
*** Note: we have a business requirement that there should be only one eOrder configuration per vendor; the way we want to enforce that is here in this list:
*** Note: only vendors that have a value of “E-Ordering” for the field Preferred Ordering Mode should be in the list of Vendors; If they don’t have that value, then they are not allowed to eOrder
*** Remove from the list of values any Vendor name that already has a eOrder configuration
*** If all the vendors are configured and the list is empty, then show a message: All eligible vendors have eOrder configurations.
** Classic Mapping: Select a Vendor on ManageVendorEDISettings page
* 
* File Format
** Radio buttons
** Required
** Values:
*** Amfire
*** Cybersoft 1.0
*** Cybersoft 2.0
*** EDI
*** EDI (PFG)
*** EDI 850
*** US Foods
** No default selection
** Classic Mapping: File Format on ManageVendorEDISettings page
* 
* Description
** Multi-line textbox
** Optional
* 
* Cancel
** Close overlay
* Next
** Validate required fields and show the user validation error messages until they are all resolved
** Validate that the Vendor does not already have an eOrder configuration
*** If they do, show an error message and do not let them progress to the next screen
*** message text: An eOrder configuration file already exists for this vendor.
** Cannot advance unless they have selected all the required fields
** Go to next tab
