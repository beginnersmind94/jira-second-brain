# NXT-25240 — INV - Production Ordering - Display Placed Orders

## Description
As an Ordering Manager I want the placed production orders to appear on the Production Ordering screen so I can see which orders have already been placed.

*Requirements*

* Update the Production Ordering screen as orders are placed
* Amend the Vendor dropdown to show order status
** Show ‘Ordered’ next to the Vendor name in dropdown if all eligible items have been ordered
*** separate with hyphen, parentheses, etc.
** Show ‘Partially Ordered’ next to the Vendor name in dropdown if some orders are placed but items still remain
* (Optional) Display a banner message if user selects a vendor for which all orders are placed 
** “Orders for this Vendor are already generated.”
* Display a small grid displaying the orders created for the selected vendor
** *Do not display “deleted” orders
*** *for Merged orders, maybe link to the newly merged order, the one which actually contains the items >> +*View icon click should display the order details page for the order which it was merged to.*+
** This could be pulled from the Confirm Orders screen
** Order #
** Delivery Date
** Order Date
** Amount
** Status
** Created (or Initiated?) By
*** user who generated the production orders
** Production Start Date
** Production End Date
** Action column
*** with view icon to open the order in a new tab (like the Confirm Orders screen)
* **Note:* it is possible that orders may be placed for vendors while some items are still not yet ordered (this scenario does not occur in PE). We will address this with optimizations in [https://primeroedgenext.atlassian.net/browse/NXT-27832|https://primeroedgenext.atlassian.net/browse/NXT-27832|smart-link] 
* If Items have been ordered for the complete or greater quantity >> Do not show the item in the PO item list 
If items have been partially ordered or not ordered at all >> the item needs to be shown in the PO item list




+Mockup Notes:+

* Change Placed Orders to another tab
