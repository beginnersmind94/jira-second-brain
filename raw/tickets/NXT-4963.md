# NXT-4963 — Account Management - Search - Additional Information in Predictive, Impact on Student / Adult Toggle

## Description
*User Story*

As a district admin I want more information on the predictive results so that I can make an informed decision.

*Mockup*

[https://nqyc9v.axshare.com/start.html#p=acm_landing_page&id=ravqth&g=1&view=default|https://nqyc9v.axshare.com/start.html#p=acm_landing_page&id=ravqth&g=1&view=default]

!image-20200816-175434.png|width=484,height=396!

*Requirements*

* For the predictive search dropdown, show the patron type for each result.
* Show the full name and ID number for each result similar to the wireframes.
** We won’t worry about the additional filters that are nested in the wireframe for this story.
* If the user selects an exact patron, that patron type should be loaded on the search results.
** Example: They pick Mathew Reynolds from the above screenshot. 
*** The Students / Adults toggle on the search results page should be toggled over to Adults automatically.
** If the user searches for ‘Matt’ and there are results for both students and adults, default to Students.

## Acceptance Criteria
Predictive search updated to show patron type information, clearer visual for name and main patron ID #.
Search query has an impact on the toggle - exact results pick the appropriate student / adult toggle in the results page.
Multi-language support enabled (LANGUAGENAME_ENGTEXT ie: FR_Student, FR_Adult)
Accessibility / ADA Compliance programmed appropriately (full KB nav possible, etc.)
