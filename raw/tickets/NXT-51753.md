# NXT-51753 — Login - Dashboard 2.0 - Miscellaneous items carried over and based on initial feedback

## Description
*Summary*

As a Cybersoft Administrator, I would like users to be ‘wowed’ by a new landing page upon logging in, so that the product immediately leaves a good first impression upon both new and returning users.

*Details*

* On the new SchoolCafe landing page, do the following:
*# Add a (transparent/non-prominent) background image, similar to our family portal homepage. See the attached file _TabletopEdited.jpeg_ to for the image to use.
*# Enlarge some of the page’s contents
*## Increase the size of the SchoolCafe logo slightly (whatever we feel makes it more prominent. Maybe 3x the size of the greeting message beneath it?).
*## Increase the width of the welcome card slightly as well, as needed
*# Modify the ‘welcome’ message so that it changes based on the user’s time of day
*## If the user logs in between 12:00AM and 12:00PM local time, change the welcome message to, *_Good morning, {firstName}!_*
*## If the user logs in between 12:00PM and 5:00PM local time, change the welcome message to, *_Good afternoon, {firstName}!_*
*## If the user logs in between 5:00PM and 11:59PM local time, change the welcome message to, *_Good evening, {firstName}!_*
*# Condense message, task, and report icons into a single floating action button which shows all 3 options
*## Add a notification bell icon with floating buttons
*## Get the aggregate total of all the user’s unread messages, tasks, and reports
*## Display the count in a chip next to/overlapping the notification bell icon
*## When selected by the user, open a floating list of Messages, Tasks, and Starred Reports menu options (with their icons), which each redirect the user when clicked
*# Make the unread count more conversational. Beneath the bell icon, display a summary
*## If the user has +any+ unread messages, tasks, or starred reports, display that number as a (bold) X within this message: *_You have X new messages, X tasks, and X new reports to view._* (This may differ slightly from the mockup).
*### If there X = 0 for any of the types, display the word ‘no’ instead of '0'. If X = 1, it would be nice to display the type in singular form (“message” rather than “messages”) also. 
*#### For example, if a user has 2 new messages, 0 tasks, and 1 starred reports, the text should say: *_You have 2 new messages, no tasks, and 1 new custom report to view._*
*## If the user has none of the above, instead of displaying ‘no' multiple times, display the message as: *_You have no new messages, tasks, or custom reports to view._*
*### I would like to hide the unread chip in this case, or change it’s fill color from “error” to “success” if it can’t be hidden
*# Provide a way for users to quickly see their list of modules. We can accomplish this by rebranding the look-and-feel of the Search field (Behavior is otherwise unchanged)
*## Change the icon from a magnifying glass to the module icon
*## Change the field’s label to _Modules_
*## Change the field’s placeholder text to _Jump to any module or page_
*# Add carousel buttons within the Recommended section, so that the View All button can be removed and all potential recommended pages be shown.
*# Change the Favorites message to say “You haven’t favorited any pages yet.” if the implementation of that feature is now complete. Otherwise, ignore this.
*## Also, add carousel buttons here like with the Recommended section, in case the user has a lot of favorite pages to possibly display
*# Change the bottom button label to say ‘Take Me To Work’ next to a briefcase, rather than ‘View Full Dashboard’ next to the module icon.
*# Footer, help icons, etc. can still be shown (even though they are not pictured in the mockup below)

*# Carried over from previous story, as requested by Aas/Jyothi:
*#* Improve page performance as much as possible. I am fine with waiting to load everything between the welcome message and the ‘Take Me to Work’ button until it is complete.
