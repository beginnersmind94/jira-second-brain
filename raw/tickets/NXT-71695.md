# NXT-71695 — Elig - DC - Add New DC Types / Edit Existing Names

## Description
As Dana the Director I want to allow for new DC Types to be importable with proper labeling so that reports are easy to interpret.

We’ve started to receive new DC Types across a few states, primarily tied to S-EBT (both Free and Reduced). We need to future proof the software and allow this to be managed by districts.

!screenshot 2.png|width=657,alt="screenshot 2.png"!

*requirements*

* Merge the functionality from the Eligibility Descriptions page into the DC Precedence page
** then remove the Descriptions page
* User can edit the names of existing DC Types on the Precedence grid
* User can see the default names of the DC Types on the grid
* User can add new DC Types to either the Free or Reduced grids
* Updated names are reflected across the product
** Reports
** Notifications
** Account Management
** Import (mapping) / Exports
** POS
** FamilyHub

## Acceptance Criteria
* Eligibility Description features merged into the DC Precedence page
* User can edit existing DC Types on the Precedence grid
* Default (original name of DC Type) is available as a grid column
* User can add new DC Types to either Free or Reduced
** User can pick if it’s an Assistance Program or a Other Source Categorical
* Updated names are used across the entire product suite
