# NXT-57835 — CR & MVR - Custom Names (FD 268513 and 262956)

## Description
As a SchoolCafé user, I would like to optionally set the name for a Custom Report file so that I can control the name of the file in case I need to



** Only for distribution 

*Notes*

* We need to do this for the Add New and Edit screens for custom reports and for MVR
* This comes from a customer request. They want to be able to specify the file name.

*Requirements*

* Do for CR and MVR both to maintain integrity
* On the Distribution tab, do the following:
* In the top part, add the following fields:
* Radio Buttons
** Values: Yes, No
** Label: Custom File Name
** Default No
** If click Yes, enable the File Name Textbox
** If click No, disable the File Name Textbox
* TextBox
** Default disabled
** If click Yes radio button, enable
** If click No, disable
** Required if enabled
** Max length: whatever you use for file name in the DB
** Note: We will not try to do smart naming like [Today] like we do with templates. Whatever they name it is what it is
* 
* Backend:
* When generating the file for distribution, do the following:
* Update the code to check if they want a custom name
** If yes, use the name they provided
** If no, keep doing the way we currently do
* 
* Note, that we only need to change the name when actually distributing (attachment to email or FTP; inbox doesn’t matter)
* You should save the file in our storage with the unique name so that when they review the recently run reports, it doesn’t show the latest file that overwrote the original
* Else, find some way to maintain integrity for when 2 reports have the same name



{color:#0747a6}*Technical Details*{color}

The details in the requirements are pretty clear. We need to add a new Column CustomFileName in CR_CustomReport Table. Modify the logic to add/delete methods to have this column so that it can inserted and retrieved

In Notification SyncAgent, when the file is being saved and stored in blob we need to verify if the customfilename exists then use this name. Otherwise the flow will be same

## Acceptance Criteria
* Fields added to distribution tab
* If select No, the report names the files like always
* If select Yes, the report uses the new name for the distribution file
* Integrity is maintained for file names of previous reports meaning that even if they use the same name, previous reports show their original data
