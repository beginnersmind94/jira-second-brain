# NXT-50264 — LMA - District - Add Sites

## Description
As a SchoolCafé Admin role member, I would like to view a list of sites linked to a license so that I can know what we have access to and when they will expire.

*Notes:*

* This is a continuation of the Dashboard story and covers everything that happens when a user clicks on the Show Sites button
* Users are District users and Customer Support role
* In this area, we can see what sites a license is linked to
* Requirements are written assuming the drawer will be only usable for site licenses (Not Device, covered in another story)
* This area replaces the Assign drawer from the original License Management screen

*Flow:*

* LMA role user browses to System > District Licenses page
* User sees a list of licenses generated for the district
* User clicks the View Sites button which opens the View Sites panel
* User sees which sites are linked to the license
* User optionally adds a site
* User optionally removes a site

*Requirements:*

* These requirements apply to System > District Licenses page:
* Add a button to the License Types card if not already there
** Label: Show Sites
** On click:
*** Expand the Sites panel
*** Toggle the label to Hide Sites
*** If click Hide Sites, hide the panel and toggle the label back to Show Sites

*Show Sites panel*

* If there are no sites assigned, show a message explaining to click the Add Sites button
* Add Sites button
** Permission:
*** Customer Support and LMA role users can click it
*** District users with Edit permission to the page can click it
*** Else, disable the button
** On click:
*** Show the Add Sites popup (see below)
** Business logic:
*** If the district has filled the license count (used all the available licenses), we cannot allow them to add more sites to the license. They can’t add more sites than they have purchased. So, they must remove a site to add a site.
** If there are no available licenses, hide or disable the Show Sites button and instead show a message that the max has been achieved
* Assigned Sites list
** Data: list of sites that have been linked to the license
** Fields:
*** Name of the site
*** Delete button
**** On Click:
***** Show confirmation button
***** If confirmed:
****** Unlink the site from the license
****** If they had reached the max licenses, show the Show Sites button again

*Add Sites popup*

* If the popup does not already exist, create it
** Title: Add Sites
** Use animation
* 
* Fields:
* Available Sites availability
** Indicate in text how many sites they can add
* Add All
** Business Logic: User cannot add more sites than available licenses
** On click:
*** If there are enough available licenses:
**** Add all the sites to the license
**** Close the popup
**** Update the linked sites list on the main page with the new sites
*** If not, then disable the Add All button
* List of Available Sites
** Data: list of sites that have not been linked to the license
** Shows Name and Add Button
** If a site is already added, do not show it
** On Click:
*** Link the site to the license
*** Hide it from the available licenses list
*** Show it on the list of linked sites in the main page
*** If you reach the max count of available, close the popup

*Security*

* Customer Support role and License Management role can do anything
* District Licenses page permissions from R&P, so ensure user is a role with that permissions
** Edit permission required for Add Sites

## Acceptance Criteria
h3. *1. Show Sites Button*

* Add a *button* to the *License Types card* if it doesn’t exist.
* *Label*: {{Show Sites}}
* *On click*:
** Expand the *Sites panel*
** Toggle label to {{Hide Sites}}
* *If clicked again* (Hide Sites):
** Collapse the panel
** Toggle label back to {{Show Sites}}

----

h3. *2. Sites Panel*

* *If no sites are assigned*, show a *message*: “No sites assigned. Click Add Sites to link sites to this license.”
* *Assigned Sites list*:
** *Fields*:
*** Site Name
*** Delete Button
** *Delete Button behavior*:
*** Show confirmation popup
*** If confirmed:
**** Unlink site from license
**** If user had reached max licenses, re-enable {{Show Sites}} button

----

h3. *3. Add Sites Button*

* *Permissions*:
** Clickable by *Customer Support* and *License Management* roles
** District users with *Edit* permission for this page
** Disabled for others
* *On click*:
** Open *Add Sites popup*

----

h3. *4. Add Sites Popup*

* If it doesn’t exist, *create it*
* *Title*: {{Add Sites}}
* *Animation / sliding motion*

h4. *Fields / Logic*

* *Available Sites*:
** Indicate how many sites can be added (available licenses left)
* *Add All button*:
** *Business Logic*: Cannot add more sites than available licenses
** On click:
*** If enough licenses → add all sites, close popup, update main page assigned sites list
*** If not enough → disable the button
* *List of Available Sites*:
** Show *Name* and *Add Button*
** If a site is already added → do not show it
** On *Add click*:
*** Link site to license
*** Hide from available list
*** Show in main page assigned sites list
*** If max license count reached → close the popup

----

h3. *5. Business Logic*

* *Cannot assign more sites than purchased licenses*
* *If max reached*:
** Hide or disable {{Show Sites}} button
** Show message that maximum has been achieved

----

h3. *6. Security / Permissions*

* Roles allowed: *Customer Support* and *License Management*
* District Licenses page permissions apply (from R&P)
* *Edit permission required* for *Add Sites* and site assignment
* Users without required role/permission → buttons disabled
