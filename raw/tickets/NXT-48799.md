# NXT-48799 — OpenID - Add New User Settings

## Description
As a SchoolCafe admin, I would like to be able to apply the authentication for a new User so that they are consistent with the rest of the company

*Notes*:

* Authentication Type is chosen by a user in their profile page but can also be set here by an admin
* A user can only choose the type of authentication available to them by the district

*Requirements*:

* On the System > Users > Add New User drawer, do the following:
* Add a Security section
* In it, do the following:
* Fields
** -Authentication Type-
*** -Single select drop down list-
*** -Required-
*** -If District has only one auth type:-
**** -Auto select the one auth type-
**** -Make the field read-only since they can’t change it-
*** -If the district has a preferred (they must)-
**** -Auto select the preferred one auth type-
**** -Allow the user to change it to one of the others-
*** 
* 
* -If the user’s authentication method selected is SchoolCafe Authentication:-
** -Show the Password information row like you do now-
** -Move this to the security section from the top section-
* -Else for any other:-
** -Hide the password row with the password reset, since it is not needed-
** -DO NOT SEND USERS A PASSWORD EMAIL-
* 
* Add a grid
** Populate the grid with all the authentication methods the district has enabled and are active
*** one per row
*** Allow the user to enable or disable a method for a user
*** Allow the user to set the preferred method for a user
** no paging, sorting, filtering
** Columns:
** Enabled
*** Checkbox
*** Default Yes for each new row unless:
*** If district has specified that one method is required:
**** Check that row
**** Uncheck and disable all others
** Preferred
*** Checkbox
*** Default No for each new row except as follows
*** If district has specified that one method is required:
**** Check that row
**** Uncheck and disable all others
** Method
*** Value is the name of the authentication type (Ex: OpenID SSO, SchoolCafe, etc.)
** Description
*** Show the description from the District’s authentication type grid
*** In the grid, if SchoolCafe is one of the options, Add the the password will be randomly generated message to the Description
* 
* If not already done, add the section collapse icons to the 3 sections on the page
* On save:
** Ensure the new validations
** Ensure you don’t send password emails if they haven’t selected SchoolCafe authentication
** 
** {color:#bf2600}-Note: currently, there are 2 types of authentication emails that can go out to a user-{color}
*** {color:#bf2600}-One if they use SchoolCafe authentication which handles the password stuff-{color}
*** {color:#bf2600}-The other is for any other kind of method such as SSO-{color}
*** {color:#bf2600}-We are going to leverage the existing SSO email to communicate the 2FA changes-{color}
** {color:#bf2600}-Update the authentication email sent to the user (not the password one) as described in the New User Authentication Email below-{color}
* 
* Changes are recorded in History log
