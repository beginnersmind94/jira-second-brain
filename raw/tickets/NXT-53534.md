## Acceptance Criteria
* Update the UI/UX of the Identifiers column (TBD)
** Replace acronyms with icons where possible
** Collapse some identifiers as part of others
* Provide a popover legend for each row on click event of the identifier row
** show all identifiers for that unit, with an explanation for each
** Provide a clear explanation for each identifier:
*** +Serving Size+:  This is a serving measure associated with the item, to be used as an ingredient on recipes or for serving on menus.
*** +Preferred Measure+: This serving unit has been set as the preferred measure, which makes it the default when adding as a recipe ingredient.
*** +Nutrition Linked+: Nutritional values for the item are associated with this specific serving size and measure.
*** +Contribution Linked+: Food component contributions have been associated with this serving size and measure.
*** +Recipe Ingredient+: This serving measure is currently being used as an ingredient in a recipe.
*** +Pack Size+: An independently countable unit of the pack size, used to represent the item within Inventory
*** +Base Unit+: The smallest countable unit of the inventory pack size (e.g. Each, Loaf, Muffin). Base Units can be configured to count in decimals (e.g. 1.75 Bags).
*** +Cost/Value+: The unit to which the Cost/Value has been assigned. This field is used for Inventory fair market value and Production costing. 
*** +GTIN/UPC+: A unique GTIN/UPC barcode value has been associated with this unit.
*** +Menu Serving+: A serving measure enabled specifically for usage as a Menu Item.
*** +Default Menu Serving+: This is the default serving size and measure used when assigning as a menu item.
*** +Menu Exception+: This serving measure is associated with specific serving groups as part of the menu item exceptions.
*** +Recipe+: This unit represents the values of the entire cooked or prepared recipe.
*** +Strict Batching+: This recipe can only be scaled up or down in increments equal to the number of servings.
