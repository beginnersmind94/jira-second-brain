# NXT-68736 — Insights - Insights Dashboard Page - Inventory Turnover Rate  (Backend)

## Description
As an inventory manager , I want to track the inventory turnover rate for each period, so that I can understand how quickly inventory is used and identify inefficiencies in stock usage.

## Acceptance Criteria
* Using Data from the Inventory Usage report
** Inventory turnover rate is calculated as : (Beginning Inventory + Purchases During Period) – Ending Inventory /(Beginning Inventory + Ending Inventory) ÷ 2 (Note:  All values are dollar amounts not inventory Quantity)
** Purchases are calculated as: Receipts +transfer in+ add to inventory - (transfer out+ returned amount)
** When all schools and more than one school is selected : apply the above formula for all schools or selected sites and all items. 
** One school selected: apply the formula for the selected school items. 
** Inventory turnover rate when less than a month (today, yesterday, last week, this week) is selected : calculated for the current month, if current month has just started and its value is zero then it fall back to year to date.
** For any other time period selection, it will be calculated using the formula for the selected date range and school(s)
** SPs/APIs should be provided for breakdown of site(s ),time period, storage category(s) ,Item category(s),item type(s),Item(s) selection 
** SPs/APIs should cover the inventory turnover rate for both Quantity based and Dollar based options
