# NXT-48634 — Users - Managed Views - Sharing - Not Required

## Description
As a SchoolCafé user I want to be able to share my views with others based upon role membership, so that I don’t have to keep track of users as they come and go.

h3. *Notes:*

* We need to add roles to the sharing screen where they manage sharing with Users
* We already do this with Custom Reports, so do it the same way

*Requirements:*

* In the Managed Views drawer from the Users page, do the following:
* When click Save As, it shows the Save As drawer
* On this drawer, there is a sharing button
* If clicked, it shows the Users list
* We want to add a Roles list in addition to where the Users list is
** Do this just like Custom Reports (CR) sharing tab
** Copy over the biz requirements from CR in the dynamic between the Users and Roles tab
* 
* On the Edit Sharing drawer:
** Make this interface consistent with CR Sharing tab
** Have Users field (already there)
** Have Roles field
** You can get rid of the Users table at the bottom of the drawer to make it easier
** One request: on CR page, it only shows one user or role then the + count. Can we show all the roles? or at least more than one before the + count? Same for Users.
* 
* On the Manage Views Saved Views tab:
* If a site user belongs to a role that was linked to a shared view, then the Managed View is available to them the same way it is available to them if they were selected as an individual

## Acceptance Criteria
* Users can select roles in addition to users to share a managed view with
* Managed View is usable by a user in a role assigned to the view
* Edit Sharing drawer has Roles field added
* Edit Sharing drawer has consistent interface as CR
