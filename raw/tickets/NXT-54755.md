# NXT-54755 — SC - GDSN Connect - Exception Handling Changes

_(none)_

## Acceptance Criteria
Navigation: 

Item Management - Items - Item Upload Icon

Changes Required:

Status column is applicable in Exception download only

Mandatory failed items should be shown as “Failed” instead of showing it as completed. The should be  shown at GDSN template upload itself

*Scenarios:*

*Item Name Description is Duplicate:*

Check whether the item name is already found at corresponding local data source at target region. 

If yes - mark that item as Failed, show the message as-is (Duplicate Item Description) - Reflect the status as Failed

Changes should reflect in Translated File, View Log UI, Exception Download



*Mandatory Fields Missing:*

Check whether the item is missing with any of the mandatory fields (regression - Item Description, Mandatory Nutrients such as Calories, Total Fat, Saturated Fat, Carbohydrates and Protien)

If yes - mark that item as Failed, show the existing message as-is 

Changes should reflect in View Log UI, Exception Download (currently they are shown as complete)

Reflect the status as Failed



*Inventory Units are missing (new addition)*

Check Whether both Inventory Unit and Purchase unit are blank against the record

If yes - show only warning message as “Missing Pack size details” (this item can still show as completed status, need not show them as failed)
