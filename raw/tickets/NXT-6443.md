# NXT-6443 — Data Exchange - File Upload Step

## Description
*User Story*

As a PrimeroEdge user, I would like to have a screen where I can quickly and easily select or drag and drop import files, so that I can more quickly upload my data.

*Mockup*

[https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=4447%3A71492&scaling=min-zoom|https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=4447%3A71492&scaling=min-zoom]

*Requirements*

* After the user advances from the Pre-Upload screen of Data Exchange, take them to the next step in the process: *File Upload*
* Here, provide users with 2 options for importing a file:
*# Drag and drop a .csv or .xlsx, or .xls (if possible) file onto the Upload Files portion of the screen
*# Use a browser to select a file from their machine - launch file explorer to open the file
** User can only upload a single file at a time
* When the user has successfully added a file, replace the Upload Files section of the page with the filename and file size of the file. (file size limit 250 MB for .xls, .xlsx and .csv file)
* Then, display 2 options:
** Remove File (Trash bin icon) - Allow user to remove the file and start this step over
** Upload - Commit the upload. We should display a green checkmark-type of symbol to indicate success.
*** The file is validated to ensure it is the proper type, and an error is displayed if it is not. File can either be validated upon drag+drag/browse OR when selecting the Upload button--whichever is more feasible.
*** Rules for successful upload:
***# File must be .csv, .xlsx, or (if possible) .xls. 
***# File should have a reasonable size limit (architect can advise)
*** Error messages: 
**** File type must be \[support file types].
**** File size must be no larger than \[max file size limit].
* Add a Previous button that allows the user to return to the Pre-Upload step. 
** *WARNING message that says “Going back to previous step will remove the files added for upload”*
* Add a Next button that allows the user to proceed to the next step: File Mapping

## Acceptance Criteria
* Verify that within the Data Exchange page, when the user selects the Next button on the Pre-Upload step (and conditions are right), they land on the File Upload step

* Verify that the user has 2 options for importing a file on this step:
1. Drag and drop a .csv or .xlsx, or .xls (if possible) file onto the Upload Files portion of the screen
2. Use a browser to select a file from their machine - launch file explorer to open the file

* Verify that a user can only upload a single file at a time

* Verify that when the user has successfully added a file, we replace the Upload Files section of the page with the filename and file size of the file.

* Verify that we once the file has been uploaded (and possibly validated at this point), we display 2 options for the user:
1. Remove File (Trash bin icon) - Allow user to remove the file and start this step over
2. Upload - Commit the upload. We should display a green checkmark-type of symbol to indicate success.

* Verify the file is validated to ensure it is the proper type, and an error is displayed if it is not. Rules for successful upload:
1. File must be .csv, .xlsx, or (if possible) .xls. 
2. File should be less than the maximum size limit

* Verify potential error messages: 
1. File type must be [support file types].
2. File size must be no larger than [max file size limit].

* Verify that the page contains a Previous button that allows the user to return to the Pre-Upload step.

*Verify that, once a valid file has been selected and uploaded, the user can continue to the File Mapping step
