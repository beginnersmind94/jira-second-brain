# NXT-43552 — PR- Production Adjustment- Validation against VMI Child items 

## Description
This is a continuation of closing the gaps for the VMI and regular menu item relationship.

## Acceptance Criteria
* When user adds a variety menu item on the plan, toggled or untoggled child menu items should also not be able to be added separately from the production adjustment function
* Do not allow individual menu items to be added to the plan if these menu items are part of VMI (Already on plan)
** Grey out the plans that include the VMI with the “child” they are trying to add and display a tooltip over the the greyed out check box “Menu Item is already added on plan via VMI: Variety Menu Item Name” 
* IF a menu item is added previously, and a user decides to add a variety menu item that happens to include that menu item or menu items (if there are more than one already on the plan), add the variety menu item and automatically toggle onto those menu items and remove the previously added individual menu items
** If there happens to be already counts for those menu items previously, populate the counts with the toggled child menu item
** Use the category of the VMI (or whatever was selected when adding the VMI) and not what was previously added for the individual menu item 
** User can not toggle these child menu items off as they were already previously added on the menu, all they can do at that point is 0 out the menu item if they happen to not want to use the menu item ( Same logic as they would have to previously do had they not added the VMI)
*** Provide tool tip “ Menu Item originally on Menu”
* IF Menu Item happens to be added via carryover and that was the reason the menu item was on the plan, then the user removes the carryover from the “source” record, the system should toggle off the child menu item 
** At this point, this becomes just a normal child menu item that can be toggled on or off
