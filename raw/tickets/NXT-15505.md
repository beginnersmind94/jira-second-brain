# NXT-15505 — Verification - Automation - Mark applications as 'No Response' automatically at the end of verification

## Description
As Vera the Director I want all applications to be automatically finished as ‘No Response’ at the end of verification if no response has been received. This will ensure I stay in compliance.

*mockup*

n/a

*requirements*

* User needs a setting they can toggle that will automatically mark all applications as ‘No Response’ at the END of verification.
** We might need a system setting to control this.
*** Code - VERAUTNRES
*** Description - ‘Mark all applications with no response as ‘No Response’ at the end of Verification?’
**** Options; Yes / No - default: No
* If the setting is enabled, we’ll want to check all Incomplete applications at the date specified in the following system setting:
** ‘Verification Response End Date’
** The next day, perform the ‘No Response’ action on those applications.
** This background job should check from 10/16 - 12/16 to see if the date in the system setting has been reached.
** -If there is a response, do not perform the action.-

## Acceptance Criteria
User can configure an option to automatically mark apps as no response at the end of verification.

Process uses the Verification End Date setting to determine when to do this. Technically, they have all day on that final day to get a response in, hence the time included.

Process performs the ‘No Response’ action on any apps that do not have a pending response filled in.
