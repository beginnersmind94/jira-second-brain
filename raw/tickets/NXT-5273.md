*requirements*

* User can click off the tracking page and into a ‘Verification Details’ screen relating specifically to that application.
** Action takes the user to a page where they can review information about the pre-verification details, the current verification status, a quick view of application information.
*** High level information up top:
**** App #
**** Current Verification Status (Complete, Incomplete)
**** If the application is:
***** incomplete: show the Last Action taken and date (example: 2nd Follow-up Notice - 1/20/21)
***** complete: show the Outcome and Result
****** examples: Reduced (Income) to Paid (Income), No Change
**** Available actions (covered in other stories)
*** Milestone Dates (any historical dates are covered by the cloned story)
**** Confirmation (if one happened) - date + name
***** this is tied to a system setting for ‘verification - confirmation required’ - if it’s not turned on, we don’t need to show this
**** 1st Notice
***** date the original 1st notice was dispatched
**** Follow Up Notice(s)
***** multiple dates will be possible, we need to show them all
**** Completion Date
**** Completion Notice
***** date it was originally sent, and any subsequent sends
**** Eligibility Change Date
***** If there was a change in eligibility - show what date that takes effect on.
*** Pre-Verification Application Information
**** Show the eligibility determination + reason.
**** Students (ID, Name, Categorical)
**** Household Members (Name, Income Type, Amount, No Income indicator, PFD for Alaska)
**** Household Size, Total Income + Frequency, Child Income (if being used)
**** If the app was categorical, show Categorical Details
***** Case Type, Name, #.
*** Post-Verification Application Information
**** This is to show if any app details were changed as part of the Verification Workflow.
**** -Highlight any changes between the Pre-Verification info and the Post-.-
***** -Example: Income on Household Member 3 was changed, highlight that section.-
**** Show the following:
***** verification outcome + reason.
***** Students (ID, Name, Categorical)
***** Household Members (Name, Income Type, Amount, No Income indicator, PFD for Alaska)
***** Household Size, Total Income + Frequency, Child Income (if being used)
***** If the app was categorical, show Categorical Details
****** Case Type, Name, #.
*** Verification Results
**** Mostly milestone dates and the outcome.
***** Result
***** Change Reason
***** Eligibility Change Date
***** Completion Date
***** Completion Notice Date

## Acceptance Criteria
Verification Details available from the tracking grids.

Information shown for a particular application’s journey through verification.

* Status, App #
* Outcome (if complete) or Last Action (if incomplete)
* Pre-Verification Info
* Post-Verification Info (if verification is complete for that app)
* Verification Results
