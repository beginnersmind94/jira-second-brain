# NXT-5211 — QA Test Execution - Application Details - Notifications History

## Description
*User Story* 

As an applications processor, I can see every letter generated from the system tied to this application, basically the notification history. 

*Mock Ups* 

*Requirements* 

* User has a section in Application Details that shows every letter generated from the system tied to this application
* Fields
** Letter Name
** Notification Method (email, print, both)
** Notification Date (datetime)
** Generated By (user firstname + lastname)
* User can preview any of these letters.
** This should pull the saved copy of the letter that was previously sent.
** This should NOT create a new letter.
* User can send the notification to print or email.

## Acceptance Criteria
Notifications section added to Application Details.
Shows all letters tied to this application.
User can preview.
User can send again.

History tracking for any sends should happen if the user does re-send the notification.

If the user re-sends the notification:
These notifications should be tied to the application and student(s) involved.
