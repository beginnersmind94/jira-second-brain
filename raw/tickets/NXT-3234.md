# NXT-3234 — **System - Organization - Users Details - Add User

## Description
As an administrator I want to be able to add a new user to my district, so that they can begin using PrimeroEdge until they are imported in automatically.

PrimeroEdge Classic Reference: System > Sites and Users > Sites > Users > Add User button



* This story should result in allowing the administrator to select the *Add User* button to add a new user, using a set of required fields, in the 4 sections related to the user’s access level within PrimeroEdge: Profile, Password, Region, and Role. 
** Page should only allow user to access the Add User button if the permissions of the logged-in user’s role allows.
* Existing fields/checkboxes/menu content in each section can be carried over from PrimeroEdge classic.
* Password validation rules are addressed in a separate story
* *Required fields to add a user*: Username, First Name, Last Name, Organization Level, and at least one Role. 
* The layout for each of the 4 sections is negotiable.
* Aim for a cleaner view of the page with better UX for the page controls (for example, we could potentially reduce amount of scrolling required by splitting content into columns). 

(History tab and De-Activate User buttons can be skipped and addressed in separate stories if needed)

## Acceptance Criteria
1. Users with add permission can access the 'Add User' button on the Users page to add a new user and modify the details in any of the 4 sections of a user's details page
2. Required fields and one role must be selected for the user to be added successfully. 
3. If not all required fields contain data, the button to update the user's profile should be disabled.
