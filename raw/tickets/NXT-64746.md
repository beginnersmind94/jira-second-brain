# NXT-64746 — WS - Insights - KPI - Inventory Turnover Rate - Dashboard Screen

## Description
As a district nutrition manager, I want to see an Inventory Turnover card on the Insights dashboard, benchmarked against the Inventory Turnover Goal for each site, so that, I can quickly evaluate whether schools are staying within acceptable inventory levels.

*Notes:*

* This story is for the Dashboard screen
* There is no benchmark screen update for this KPI
* Another story will cover the Inventory Turnover Details drawer

** Card displays in:

** Update the value for actual based upon time frame filter change
** Update the value for actual based upon location filter change
** On click: Open the Inventory Total Drawer
** 
** Inventory Turnover Rate for the selected Time and Site dropdown options. [+PBI for Inventory Turnover rate calculation+|https://dev.azure.com/Cybersoft-Technologies-Inc/PrimeroEdge%20Classic/_workitems/edit/107442]

## Acceptance Criteria
h2. 1. Data and API Integration

* Inventory Turnover Rate must be included in the dashboard API response and dashboard dataset.
* Inventory Turnover Goal benchmark values must be retrievable per:
** Site
** Academic year

----

h2. 2. Inventory Turnover Rate Calculation Rules

h3. Purchases During Period

* Purchases During Period must be calculated as:
Purchases = Receipts + Transfer In + Add to Inventory − (Transfer Out + Returned Amount)

h3. Inventory Turnover Rate Formula

* Inventory Turnover Rate must be calculated as:
Turnover = ((Beginning Inventory + Purchases During Period) − Ending Inventory) ÷ ((Beginning Inventory + Ending Inventory) ÷ 2)
* All values are dollar amounts (not quantities).

h3. Site Selection Rules

* If one school is selected:
** Apply the turnover formula using only that school’s data.
* If multiple schools or All Schools is selected:
** Aggregate all inputs across the included schools and items, then apply the formula once on the totals.

h3. Timeframe Rules for Actual Value

* If the selected time filter is less than one month:
** Today
** Yesterday
** This Week
** Last Week
* Then calculate turnover using current month-to-date.
* If current month-to-date is zero because the month just started:
** Fall back to year-to-date.
* For all other time filters:
** Calculate turnover using the selected date range and selected school(s).

----

h2. 3. Inventory Turnover Rate Card

h3. Card Creation and Placement

* Add a new KPI card labeled Inventory Turnover Rate.
* Default placement is at the end of the card list.
* Formatting:
** Display as a decimal with two digits (example: 1.23).

h3. Card Displays

* Actual Inventory Turnover Rate for the selected Time and Site filters.
* Expected/Benchmark value from the benchmark configuration page.

h3. Card Color Logic

* Green when:
Actual Turnover Rate ≥ Benchmark Goal
* Red when:
Actual Turnover Rate < Benchmark Goal
* Grey when:
** No benchmark is configured for the selected site(s).

h3. Filter Refresh Behavior

* When the time filter changes:
** Actual value refreshes.
** Benchmark/expected value refreshes using academic year rules.
* When the site filter changes:
** Actual value refreshes.
** Benchmark/expected value refreshes.

h3. Card Click Behavior

* Clicking the card opens the Inventory Turnover Rate Details drawer (covered in another story).

----

h2. 4. Benchmark / Expected Value Logic on Card

h3. Multi-Site (All Schools or Multiple Selected)

* Benchmark value must equal:
Average of the turnover goals configured for all included schools for the selected academic year.

h3. Single Site Selected

* Benchmark value must equal:
Turnover goal configured for that school for the applicable academic year.

h3. Academic Year Selection Rules for Benchmark

* Today, Yesterday, This Week, Last Week, This Month, Last Month, Year-to-Date:
** Use the current academic year benchmark.
* Prior Year:
** Use benchmark from one academic year prior.
* Specific numbered year in the past:
** Use benchmark for the selected year.
* Custom Date Range:
** Use the benchmark for the academic year that includes the To Date.
** If the range spans multiple academic years:
*** Always use the academic year of the To Date.

----

h2. 5. School Performance Grid Updates

* Add a new column labeled Turnover.
* Location: Last column in the grid.
* Formatting:
** Decimal with two digits.
* Updates dynamically when time filter changes.
* Horizontal scrolling must be enabled due to column count.
* Clicking a Turnover value opens the Inventory Turnover Rate Details drawer.

----

h2. 6. Performance Trends Updates

* Add Inventory Turnover Rate to the KPI dropdown list.
* KPI dropdown ordering must match the benchmark configuration column order.
* Formatting: Decimal with two digits.
* Trend updates when:
** Time filter changes
** Location filter changes
* Popup/help text is supported (copy TBD).

----

h2. 7. Schoolie AI Updates

* Inventory Turnover Rate must be included in inputs for:
** Dashboard AI analysis
** Homepage daily recap AI analysis
* AI should receive both:
** Actual turnover value
** Benchmark goal context (if configured)

----

h2. 8. Configure KPIs Drawer Updates

* Inventory Turnover Rate must appear as an option in Configure KPIs.
* Users can:
** Toggle it on/off
** Sort/reorder it
* Preferences persist and apply immediately:
** Toggling off removes the card, grid column, and trend option.
** Sorting changes card order accordingly.

----

h2. 9. Validation and Edge Cases

* If denominator ((Beginning + Ending)/2) is zero:
** Display 0.00 or N/A based on existing KPI conventions.
** Prevent divide-by-zero errors.
* If benchmark is missing for one or more schools in multi-site selection:
** Treat missing schools as “no benchmark configured” and show Grey state unless defined otherwise.
* Values must never render as null/undefined.

----

h2. 10. Performance and UX

* Adding this KPI must not degrade dashboard load performance beyond existing thresholds.
* All KPI refreshes from filters must complete within normal UX expectations.
