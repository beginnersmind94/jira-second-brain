# NXT-67676 — Transaction Snapshot Logic

## Description
*As a* system
*I need to* snapshot account details when posting transactions
*So that* historical reports remain accurate even when accounts are edited

## Acceptance Criteria
* When posting a transaction, capture the current account code and description at that moment in time
* Save this snapshot with the transaction permanently
* When generating reports, display the snapshotted account code and description from when the transaction was posted
* If a user edits an account after posting, old transactions continue showing the old account details
* Future postings capture and use the updated account details
* Historical reports always show what was true at the time of posting
