# NXT-67534 — Story 3 — Apply State Template (District)

## Description
*As a* district business administrator
*I want to* apply my state's template to my Account Code Structure
*So that* I can start from a compliant baseline and customize it

h3. Background

Districts need a starting point for their account code structure. We provide drafts of state-specific templates (from Story 2) that they can apply as their foundation.

!image-20251029-150858.png|width=1082,alt="image-20251029-150858.png"!

## Acceptance Criteria
Loading a template:

* *"Load State-Defined Template" section includes:*
** -State dropdown (pre-populated with the district's state, shows all available states)-
*** Only show the state associated with that district and that cannot be edited. Hitting apply should still be an action the user takes. 
** "Apply Template" button
** The pre-selected state's template is not automatically loaded; the user must click "Apply Template" to load it.
** -Provide a banner saying “Quickly start with a draft based on publicly available state requirements. Please confirm with your finance department before saving.”-
*** Banner saying “This draft is based on publicly available state guidance. Please confirm with your finance department before saving.”
** -If the selected template is generic, display a warning: "We don't have a state-specific template on file. This is a commonly used structure, but please verify it matches your finance department’s requirements." (generic mapping done in separate story)-
*** No generic state in dropdown, no separate warning for generic template being applied 

Applying a template:

* If segments exist beyond the Site segment, clicking "Apply Template" opens a confirmation modal: "Applying this template will replace your current segments. Are you sure?"
* If only the Site segment exists, apply the template directly without confirmation.
* Upon confirmation, template segments are copied into the district's structure, replacing existing segments.
* After applying, all segments can be edited according to Story 1 rules.
* A new version is created only when the user clicks "Save Changes" (per Story 1 versioning).

Delete all segments:

* Clicking the "Delete All Segments" button opens a confirmation: "This will delete all segments except Site. Are you sure?"
* Upon confirmation, all segments except Site are deleted.
