# NXT-20510 — Income Survey - Edit Survey Details - Process/Make Pending/Delete/Restore Survey

## Description
As a district admin, I would like to be able to have a variety of ways to classify Income Surveys that are submitted, so that I can make sure that I have processed only those surveys that are truly complete.

*Summary*

Allow users to mark a Survey as Processed, Pending, and Deleted, as well as be able to restore a deleted survey.

*Requirements*

* In Eligibility → Forms → Surveys → Survey Details, allow a user to select one of 3 buttons in order to move the Survey forward or backward in status, plus a 4th button to restore a deleted survey.
* Button behavior:
** *Process Survey*
*** This button is always enabled unless the survey is in Deleted status.
*** When selected, prompt the user to enter a comment. Without a comment, they should not be able to mark the survey as processed. Example:
*** Process Button: 
**** Close the prompt 
**** Display a toast message (Survey has been processed successfully\!) 
**** Beneath the ‘Survey Number', display ‘Processed’
**** Insert this comment into the ‘Comments’ list of the survey
**** Put the Survey in Processed status
** *Mark as Pending Survey*
*** This button should only be enabled if the survey is in ‘Pending’, ‘Notified’, or ‘Processed’ status. 
*** When selected, prompt the user to enter a comment. Without a comment, they should not be able to mark the survey as pending. Example:
*** Mark Pending Button: 
**** Close the prompt 
**** Display a toast message (Survey has been marked as ‘Pending’) 
**** Beneath the ‘Survey Number', display ‘Pending’
**** Insert this comment into the ‘Comments’ list of the survey
**** Put the Survey in Pending status
** *Delete Survey*
*** This button should only be enabled if the survey is in ‘Processed’, ‘Notified’, or ‘Pending’ status.
*** When selected, prompt the user to enter a comment. Without a comment, they should not be able to mark the survey as deleted.
*** Also, warn the user that any changes they have made will be discarded. (May not be reflected in mock-up).
*** Delete Button: 
**** Close the prompt 
**** Display a toast message (Survey has been deleted successfully) 
**** Beneath the ‘Survey Number', display ‘Deleted’
**** Insert this comment into the ‘Comments’ list of the survey
**** Put the Survey in Deleted status
**** Disable the edit pencils on the Survey Details page
**** Enable the ‘Restore’ button
** *Restore Survey*
*** Only enable this button when a survey is in ‘Deleted’ status
*** When selected, prompt the user to leave a comment in order to restore the survey. Without a comment, they cannot restore the survey. We can use a similar prompt to those above.
*** When the user selects the Restore button:
**** Close the prompt 
**** Display a toast message (Survey has been restored and marked as ‘Pending’) 
**** Beneath the ‘Survey Number', display ‘Pending’
**** Insert this comment into the ‘Comments’ list of the survey
**** Put the Survey in Pending status
* This story is complete when all 4 of these buttons are functional.
