# NXT-49875 — LMA - License Keys - Generator

## Description
As a License Management (LMA) role member, I would like to create license keys for districts after they buy them so that I can control licenses across all our customers in one place.

*Notes:*

* Scope, like the License Key dashboard, it is for ALL districts at once, not a single district
* This is like the Add functionality we usually do, we are just calling by another name, so use that concept in creating the story

*Flow:*

* LMA role user browses to System > License Management
* User clicks Generate License Key
* User redirected to the Generate License Key drawer
* User enter in the needed information and hits Generate Key button
* System generates the key
* User goes back to the License Key dashboard and sees the new key without a district
* Later, when the district is created, the implementation team will apply the license key to the district, establishing the link

*Requirements:*

* On System > License Management do the following:
** When click on the + Generate button, open the Generate New License Key drawer

## Acceptance Criteria
h3. *1. Access and Navigation*

* From System > License Management, clicking the + Generate button opens the Generate New License Key drawer.
* Drawer opens with a sliding motion and is half-page width.

h3. *2. Basic Information Section*

* *District*
** Single-select dropdown listing all districts.
** Optional field.
** If selected, links license key to that district.
* *Purchase Date*
** Date picker.
** Must be today or in the past.
** Required field.
** Date only, no time.
* *Effective Date*
** Date picker.
** Can be past or future.
** Required field.
** Tooltip: “The day the license takes into effect.”
* *Expiration Date*
** Date picker.
** Must be after the Effective Date.
** Required field.
** Tooltip: “The day the license expires.”
* *Description*
** Single-line textbox.
** Required field.
** Tooltip: “Who and what is this for.”
* *Is Renewal?*
** Checkbox, default = No.
** If Yes, show *Original License Key* field.
** Activity value set: Yes = Renew, No = New.
* *Original License Key*
** Single-select dropdown listing existing license keys.
** Shown only if Renewal is Yes.
** Selection saved in DB for reporting.
* *Notes*
** Multi-line textbox.
** Required field.
* *Total Revenue*
** Read-only field.
** Auto-summed from license type revenue fields in real-time.

h3. *3. License Types Section*

* Displayed as a table of panels with inline editing.
* On page load:
** Workspace and System module rows are automatically added.
** Cannot be deleted by the user.
* *Panel Fields*
** License Type dropdown (filtered by Module).
*** Required if row is checked.
*** Disabled once selected; re-enabled if row is deleted.
** Deployment dropdown (filtered by license type, default applied if only one option available).
*** Required if row is checked.
*** Read-only if only one viable option exists.
** License Count
*** Required numeric field.
*** Read-only and auto-set to 1 if Deployment = District.
*** Editable if Deployment = Site or Device.
** Revenue
*** Optional numeric field, formatted as money.

h3. *4. License Types Table Actions*

* *Add License Button*
** Validates required fields.
** Adds new row to the grid.
** Disables selected License Type from dropdown list.
* *Add All Button*
** Adds all licenses with default values to the grid.
** Correctly applies deployment rules per license type.

h3. *5. Licenses Grid*

* Displays read-only values for License Type, Deployment, and Count.
* *Edit Button*
** Opens edit view with fields as per License Types section rules.
* *Cancel Button*
** Closes edit view without saving.
* *Save Button*
** Updates Total Revenue.
** Saves changes to DB.
** Returns card to read-only view.
* *Delete Button*
** Prompts for confirmation.
** Removes row if confirmed.
** Re-enables deleted License Type in the dropdown.

h3. *6. Generate License Button*

* Saves the following to DB:
** Generated Date = Now().
** Generated By = current user account name.
** Status = New.
** License # = 16-character alphanumeric string (mixed case, random, official-looking).
** Start and End dates inherited from license key.
** Save record to History table.

h3. *7. Security*

* Drawer and page only accessible by LMA role users.
