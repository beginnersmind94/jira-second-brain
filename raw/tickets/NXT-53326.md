# NXT-53326 — VND - SC - Transactions and Sessions

## Description
As Sally the Site Manager I want to see sessions and transactions from the vending machine in SchoolCafe.

*Requirements*

One of the Vendnovation API endpoints can result in sessions needing to be opened, and transactions recorded - /sale-completed

* Check to see if a session already exists for the terminal, at the site and current date.
** If a session already exists and it is Open, add sales / transactions to the session.
** If a session already exists and it is not Open, open a new session and add sales/transactions.
** If no session exists, open a new session and add sales/transactions.



[^VendNovation REST Authentication Service - Vendor (1).pdf]

*Discussion Refinement 10/10/24*

* Source column on Transaction can be used to note these are vending transactions
* Future items will consider revealing the Transaction.Source column and/or the TerminalType for Sessions and Transactions/History grids.

## Acceptance Criteria
* SchoolCafe handles the payloads from the /sale-completed endpoints appropriately and inserts into our databases.
** Sessions are opened when needed at the site the vending terminal is at, with the correct date.
** Sales/Transactions are recorded into Sessions as expected.
** All triggers work from there re: account balances.
* User can run reporting to see sales/transactions from the vending machines.
