# NXT-60315 — Digitize AI - Document codes within reports

## Description
As a user I want the ability to upload/capture an image/PDF to upload without having to specify which form I am using so that it saves me time and possible error.

* To properly detect form types, we should update supported form types (reports) to include a specific document code as part of the header or footer. This will allow Digitize AI to quickly identify the form type and reference its details from the database to be used in association with the summary display and when updating.
* Recommended to have a form type prefix + the instance (unique ID) of the form. Ex) PR-334567 : The PR = “Production Record” 
* It is probably best to have the form code somewhere in the head of the document as users will most likely capture at least the top of the page if using a mobile device. If we put in the footer, it’s possible they don’t capture it.

In this example, putting the new row for “Form: PR-99081245” would be what is scanned and identified.

## Acceptance Criteria
* Each form/report that is enabled for digitize AI is required to have a form reference field to be used for tracking. This reduces errors and complexity with trying to scan the header information and reduces issues with additional pages not containing that information.
* Each page of the form/report must contain the form reference field.
