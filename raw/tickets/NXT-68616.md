# NXT-68616 — Track Login & Logout - UX

## Description
As a SchoolCafé Admin, I would like to see when a user logged into or out of the app so that i can properly verify usage.

*Notes*

* this is the follow up to the backend story that did the tracking
* In this one, we want to be able to see the results

*Requirements*

* On the System > Security page, do the following:
* Add a new tab
** Authentication History
** Put it last
** On this tab, do the following:
* 
* Add a filters section
** Date Range
*** Use our usual smart date filter
*** Ensure it allows the user to put in a custom date range 
*** Default to Last 24 hours
** Apply button
** Clear button
** Ensure the filters section is expandable/collapsible
*** default collapsed
* Add a grid below the filters
** Use the usual kendo grid
** Has paging, sorting, filtering
** Do the show/hide filters icon like we have done elsewhere
** Data:
*** Default the initial load to just last 24 hours so that it loads quick
*** Data is filtered to the region logged into
** Columns:
*** User
*** Position
*** LogIn
*** LogOut
**** If null, show blank
*** IP Address
*** Authentication Type

## Acceptance Criteria
Able to view log the login & logout details for each user
