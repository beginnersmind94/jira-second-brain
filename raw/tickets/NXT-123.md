# NXT-123 — Accountability - Transactions - Adjustments page and Account Adjustments report

## Description
As a Central Office Admin I want to see what adjustments are being made to accounts and sales so I can see if anything is being done incorrectly.

As a Site Manager I want to see the same, but for the sites I have access to.

*requirements*

Add an option to the Transactions page that lets the user search for Adjustments. 

* If selected, the user has the following search criteria:
** The user can select the Area, Site, Device/Terminal, User, and Date.
*** User needs an ‘All’ option. Don’t search for instances ahead of time on this one like we did with Suspicious Transactions.
** User has the standard date search filters. 
** User can select the Adjustment Type they’re looking for:
*** Each one corresponds with an attached report. 
* Add the Account Adjustments report as part of this:
** Use the search criteria to pull back any ‘account adjustments’ transactions that have been performed which match.
** The report shows the following information:
*** *Enrollment Site*
**** Code + Name for the student’s enrollment site on that date.
*** *Account Name*
**** Firstname + Lastname
*** *Account ID*
**** Account ID #
*** *Amount*
**** $$ amount, go to 2 decimals always. $0.00, $50.00
*** *Adjustment Date*
**** Full datetime for when the transaction was performed. Time included.
*** *User*
**** Firstname + Lastname of the user who performed the adjustment
*** *Comments*
**** If a comment was entered when the adjustment was done, show that string.
** Show a *total* for the *Amount* column.
** Sort order:
*** Enrollment site, Adjustment date
** Report should show the typical headers and footers:
*** Generation datetime and user.
*** Report name.
*** Search criteria.
*** Footer about the report by SchoolCafe.
