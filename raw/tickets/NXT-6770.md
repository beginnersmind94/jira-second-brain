# NXT-6770 — Users Phase 2 - Add "Manage Users" page (View ALL users)

## Description
As a District Admin, I would like to view all of the users in my organization at a glance, in a list I can sort and filter, so that I can quickly find users who need their details updated.

+Summary+:

Utilize the existing breadcrumbs (System → Sites & Users → Users) and update the existing Manage Users page to use the new UI template.

+Details+:

* Replace the existing Users in System → Sites & Users → Users page with one that utilizes the new template, (mock-up below) with some exceptions:
** Username column is removed
** Page is now sorted by Last Name (desc) by default
** Email column should display first 
** Every row will be selectable/a hyperlink used to view the user details
** Authentication mode column is removed
** -Site field is added and will display any site(s) that the user can access. We can display list in whatever format works best from an architectural standpoint (mock-up shows comma delimiting)--no preference.-(due to KeyCloak constraints, not practical. Moved to additional Filter instead)
*** -Role can be displayed the same way.- (due to KeyCloak constraints, not practical. Moved to additional Filter instead)
** A filter should be added to the top of the page and contain the list of roles that belong to the tenant (both Standard and Custom) which, when selected, filter the list of users by that role. This drawer should also allow the user to filter by users' sites in the same way.
* One other difference is the addition of a new status, ‘Locked Out’ (for users who repeatedly fail to log in with a proper password), which will be handled when we implement this through KeyCloak. (separate story)
* User should still be able to search and sort within the screen.

## Acceptance Criteria
Verify that when a user navigates to System -> Configuration -> Users, they are presented with the new design template.

Verify existing behavior from 1.0 page applies with the following exceptions:
-Sorted by Last Name (desc) by default
-Username column is removed
-Email column can display first and will be the hyperlink used to view the user details
-Authentication mode column is removed
-Site field is added

Verify user is still able to search and sort.

Verify user can filter the list by Role or Site
