# NXT-52713 — PR- Plan/Record- Preserving Record Complete Status when 0 count item has changed on Plan 

## Description
As a production manager, there are times where last minute I realized that even though I planned not to produce an item, we changed our mind after the fact of my record being completed. Understanding that record complete statuses can not be changed, I would like the system to use the count that I enter on my planned count for an item previously 0 to also be used as my produced count.

## Acceptance Criteria
* When record is complete, if user changes a 0 count menu item from the plan, use the count they added to the planned counts as their produced counts 
** Preserve record complete status and should only use this logic if record is complete
* IF 0 count menu item is a prepared item, then user changes the count from 0 to a number, reclassify prepared item as non prepared because consolidated and prep site distribution will have been already completed
