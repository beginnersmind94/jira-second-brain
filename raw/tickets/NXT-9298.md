# NXT-9298 — Data Exchange - SIS Import - Activate (re-enroll) a student account

## Description
As a district I want to activate (re-enroll) a student account that is currently inactive if the student is identified on the student information file.

*“Handle Activation (Re-enroll) only as part of this story “*

More Detail:

* The most common approach for activating/deactivating student accounts is the presence of the student on the import file. This will be the default and initial approach to activation/deactivation. An alternate method is to provide an enrollment flag on the file (Active, Inactive). 
* Activation (re-enrollment) of a student means that the Student ID (account) already exists within the system. The account simply moves from the inactive (withdrawn) status to being made active.
* This scenario often occurs if a student moves physical addresses and begins attending a new school district (becoming withdrawn) and then returns to the school district at a later time (at least 1 day of separation from inactive to active status). This may occur within the same school year or a new school year.

Exploration and Ideas:

* To avoid the re-usage of an existing student ID on a new student, we should explore how to manage. An idea would be that if an activation is going to occur but the student’s name does not match, we should create a type of review process for the administrator of the system.
* We could also handle this challenge by moving students to a “graduated” status if they are in fact no longer to be active in the system under the current student ID number.



Dev note - there is no column or  staging table available for Active/Inactive

## Acceptance Criteria
1. If a student is currently inactive (withdrawn) and appears within the student import file, they should be activated (re-enrolled).
