# NXT-67060 — Ignore, old (SC - MT - Central Screen)

## Description
As a Customer Support role user, I want to be able to view data related to a Central realm so that I can manage org level and users.

*Notes*

* This story covers the creation of the page. Other stories will cover the content on the page.

*Requirements*

* Add a new resource to Resource Catalogue 
** Name: Central
** All scopes checked
** Ensure routes to the new page
* Create a new page:
** Name: Central
** Location: In System module, above Districts
** Ensure Customer Support and Technical Support roles have access and can do anything
** Ensure View permission for the resource is enforced
* Roles and Permissions
** Ensure the new resource shows up in the roles and permissions 
** Ensure it is checked by default for Director and SC Admin roles for new districts
** Ensure it is updated for existing districts to be checked for Director and SC Admin roles 
*

## Acceptance Criteria
h2. Resource Catalogue

* Add a new resource named Central.
* All scopes (view, create, edit, delete, etc.) are created and available for assignment.
* The Central resource appears in the Resource Catalogue list.
* Routing is configured so the Central resource routes to the new Central page.

h2. Page Creation & Navigation

* Create a new page named Central.
* Location: System module, positioned above Districts in the navigation hierarchy.
* Direct navigation to System > Central loads the Central page.
* If a user lacks View permission for Central, the page and its route are not accessible and do not appear in navigation.

h2. Access Control

* Customer Support and Technical Support roles can access the Central page and perform all actions available on the page.
* View permission for the Central resource is enforced at route and UI level:
** With View permission: page loads.
** Without View permission: page is hidden from menu and direct route access is blocked.

h2. Roles & Permissions Configuration

* The Central resource appears in the Roles & Permissions (R&P) configuration UI.
* For new districts, Director and SC Admin roles have Central resource View (and other default) permissions checked by default.
* For existing districts, Director and SC Admin roles are updated to include the default Central resource permissions.
* Changes to role permissions for Central persist and are reflected immediately in access checks.

h2. Non-Functional

* Navigation to the Central page renders within standard performance thresholds.
* Feature works under existing app theming and layout without visual regressions.
