# NXT-2703 — System - Management - Program Configuration - Add Sites tab

## Description
As a sponsor administrator I want the ability to configure each site and their participating programs.

*Summary*: Allow the user to configure the feeding site status, as well as the reimbursable status and serving times, of all meal types at one or all sites.

Requirements:

* Allow the user to select the ‘Sites’ tab of the Program Configuration page and be able to manage individual site configuration settings.
*# By default, only display the Site Type and Site Name fields (we can hide the “custom meal type” row by default until a Site Type is chosen)
*# Allow the user to select a Site Type. Based on the site type selected, allow the user to select a site also. 
*## When a Site Type is selected, make ‘All’ the default Site option selected. This should allow the user to mange all sites within the site type.
*### In this view, the ‘Feeding Site’ and ‘Custom Meal Timings’ toggles should display but should be turned off by default. (Every time the user loads the page, the default view for ‘All’ sites should be that both toggles display as off, as there could be a mixture of on/off settings for the individual sites in the site type).
*### allow the user to indicate whether all sites in the site type are feeding site(s) or not
*### with the “custom meal type” row, allow the user to toggle on/off whether custom settings will be in use for all sites in that site type
*## The Sites drop-down should then show only those sites within the selected site type.
*## When the site is selected: 
*### display default or the _current_ custom meal type timings. (display the default timing until the user edits the timing)
*### allow the user to toggle on/off whether it is a feeding site
*### with in the “custom meal type” row, allow the user to toggle on/off whether custom site settings will be in use for that site
*# For the Custom Meal Timings section:
*## display all meal types in the list along with their timings and settings, regardless of whether the site is set up to serve them (this will make more sense when we complete the Programs tab of the page). 
*### This section can be sorted in ascending order by start time
*## allow the user to toggle meal reimbursements on/off for individual meal types for one/all sites.
*## edit the timing of Breakfast/Lunch/etc. for a site. (Formerly this functionality existed in Site settings as well as the Meal Types page, but we are moving the functionality to one location.)
*### Timings between meal types should not overlap
*### Timings should be within a single 24-hour period, so that days do not overlap either
* All toggle updates should instantly save and display a confirmation, with one exception:
** When ‘All’ sites are selected and the user is changing the ‘Feeding Site’ toggle, the meal timings, or the Reimbursable toggles, display a warning message:
*** “You are modifying settings for all sites in this site type group. Proceed?” Y/N
*** If 'No', close the warning and do not commit the updates.
*** If ‘Yes’, close the warning and commit the updates successfully.

## Acceptance Criteria
Verify an individual site can be selected, with the Site Name field updating based on the Site Type field selection.

Verify that 'All' sites within a type can be selected.

Verify feeding site can be toggled on/off for one or all sites.

Verify custom meal settings can be set to show/hide meal types when toggled on/off

Verify default timings are restored when custom meal settings is toggled back off. (except reimbursable toggles)

Verify meal type can be toggled on/off as Reimbursable for one or all sites

Verify meal type timing can be changed for each meal type for one or all sites

Verify a warning message appears whenever doing a bulk edit for all sites

Verify Spanish language support for all text/fields
