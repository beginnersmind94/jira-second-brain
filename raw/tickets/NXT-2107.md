# NXT-2107 — Production - Production Plan - Strict Batching

## Description
As a site manager, who needs to prepare and serve food to students and/or staff at my location (and possibly other locations), I need to use recipes that are assigned to my location based on the menu item list that I’ve been given. Some of these recipes may have strict batching included.

!image-20210706-154908.png|width=100%!



When a recipe has Strict Batch marked as part of its criteria, I must adhere to planning quantities that coordinate with that quantity. 

If the strict batch quantity is 30, then I must plan in increments of 30, such as 30, 60, 90, 120, 150, etc…The menu item should display clarity in that regards (i.e. if the batch size is 30 portions, this should be identified easily on the production plan page, and one subsequent reports - technicians worksheet and production tickets, particularly.)

## Acceptance Criteria
1.) Page will display the batch size next to the menu item, if “strict batching” is part of the criteria of the recipe.

2.) Page will block user from being able to use increments that are outside of the range of the batch quantities (i.e. if batch size is 30, I should adhere to 30, 60, 90, 120, 150, etc counts… and should be prevented from planning a quantity of 45, or any other number not evenly divisible by the batch size.)

3.) A message should appear identifying why I am unable to enter numbers outside of the assigned increments (i.e. identifying the fact that it is a strict batch recipe, and needs to be in increments of…XX servings”

In this example - the recipe is “batched” for 60 portions… therefore, the page is preventing me from being able to generate 250 portions… I can do 240, or 300… as those are in increments of 60.
