# NXT-20042 — IM - Item Details - Hierarchy - Add ability to designate child items with 'AND' condition

## Description
As a FSD I want the ability to designate multiple child items as all being mandatory parts of a parent item, so I can handle variety pack items.

*Requirements*

* In the details card, add an additional column to the left of Status
** this column will only apply to *Child* items 
** this small column will contain a label indicating whether or not a child item is an “optional” child or part of a collective group of child items (variety pack)
*** This will be indicated with a label of ‘AND' or ‘OR’
*** If items are all part of the parent item, those items will have an ‘AND’ condition
*** If there are multiple child items but those items are different options for the parent item, those will have an ‘OR’ condition
**** this is the assumption unless the items are configured otherwise
** this column will also have an Edit mode which will change to contain checkboxes
*** Edit mode will be toggled by a button in the column header
*** items which are checked will be indicated as being part of an ‘AND’ condition hierarchy
**** every item which is checked is considered to be a mandatory part of the child items
**** this would be used for “variety pack” items; Ex: multiple flavors of a single item
**** Ex: Belvita Crackers (case of 16) has 4 child items: Orange, Blueberry, Vanilla, Cranberry
***** Each of these child items would be represented in the grid
***** Each of these child items would be ‘checked’ and therefore have an ‘AND’ condition
