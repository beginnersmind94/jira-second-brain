# NXT-52712 — Authentication - One-Time Passcode - Notify Process

## Description
As a SchoolCafe admin, I want to be able to tell a user to update their authentication so that they can login using it

*Notes*:

* This story is for how to handle the notification process
* Any One-Time Passcode type can use this process
* Will be used on the Edit Users screen and the District setup screens
* It covers the following use cases:
** District has setup a One-Time Passcode process previously but the user has not configured it
** District is setting up a new One-Time Passcode method and users need to be instructed on what to do
** District is removing a One-Time Passcode method that previously existed
*** This is covered in the delete story
** A new user is added in Add New Users
** A new user is added via import and a One-Time Passcode is the district’s preferred method
** An existing user is seen on Edit screen and told they need to configure it

*Flow (Happy Path):*

* Admin browses to System > Security page
* Admin sets up a New One-Time Passcode method
* System notifies users of change and provides instructions on how to configure
* User follows instructions, updating their One-Time Passcode method in their Profile page



*Requirements*:

*Security Screen*

* These requirements apply to the System > Security page > Authentication tab:
* 
* When adding a new user manually
* On save:
** Note: currently, there are 2 types of authentication emails that can go out to a user
*** One if they use SchoolCafe authentication which handles the password stuff
*** The other is for any other kind of method such as SSO
*** We are going to leverage the existing SSO email to communicate the One-Time Passcode changes
** If the district has enabled any One-Time Passcode method:
*** Update the authentication email sent to the user (not the password one) as described in the New User Authentication Email below



*User Imports*

* These requirements apply to when a new user account is created via imports
* Do exactly the same thing and communications as when creating a user manually
** This includes sending the New User Authentication Email below



*District Adds One-Time Passcode as a new method:*

* These requirements apply to the System > Security page > Authentication tab:
* 
* When the district enables a One-Time Passcode method by using the Enable button, do the following:
** After confirmation:
** Update the authentication email sent to the user when the district has added a new method as described in the New Authentication Added by District Email below





*User needs to Config a One-Time Passcode method:*

* These requirements apply to the System > Users > View User:
* When an admin views the user page and notices that a method is not configured:
* Admin clicks the Notify button
** After confirmation:
*** Update the email sent to the user when they have an unconfigured method as described in the User Unconfigured Authentication Email below





*User needs to Config a One-Time Passcode method:*

* These requirements apply to the System > Security page > Authentication tab:
* When an admin wants to warn all impacted users to configure their authentication if they haven’t yet:
* Admin clicks the Notify button
** After confirmation:
*** Update the email sent to the user when they have an unconfigured method as described in the User Unconfigured Authentication Email below
*** Use the same email as for an individual, just do it for all impacted individuals
