# NXT-48255 — IM - Menu Items - Preserve list page state when navigating back

## Description
As an Item Management user I want the list pages to remember my filter criteria and results paging/sorting when I navigate back to the page from a Details page, so I don’t have to reset my criteria each time over and over.

## Acceptance Criteria
* Header filter criteria should be preserved when navigating back to the Menu Items list page from a specific menu item
** extra logic is required for the Menu Items page because it can open any ‘Details’ screen, including the Item and Recipe screens which can also be opened from their respective list pages
*** all 4 details screens
* Retain user inputs and selections
** applied header filters
** applied column selections 
** applied column filters
** pagination page
** pagination size
* Accommodate both Basic and Advanced filter versions of list pages
** Perhaps a dependency on [https://primeroedgenext.atlassian.net/browse/NXT-48226|https://primeroedgenext.atlassian.net/browse/NXT-48226|smart-link] 
* All ‘Details’ pages will require a dedicated Back button in the UI
** cannot maintain state using browser back button/cache
