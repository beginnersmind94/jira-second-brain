# NXT-48075 — API – Validate Data (only UI framework changes)

## Description
As a SchoolCafé admin, I would like to validate the data pulled via the API call so that I can see what issues I will have before implementing the import.

*Note:*

* This story is leveraging a few concepts that have already been developed:
** The connection to pull API data
** The validators that produce the exceptions grid in History log
** The notifications to user’s inbox like on the Download data feature once complete
** The download for exceptions tab in History log
** Leverage this existing code
* The use case is that the user wants to validate the data that is pulled by calling the API and see what problems they will face before they run the import
* Since it could take a long time to validate, we will instead save the exception log to a file and notify the user when it is ready for their viewing
* There is a corresponding story for file uploads

*Flow:*

* Browse to API
* Click on Validate for one of the rows
* Shows popup confirmation
* User says yes and system shows message that it will be sent to their inbox and they will be notified once complete
* System runs the API, gets the data and then performs a validation of the data
* System generates an excel file of the exceptions data like that in the import history log
* System notifies the user when the file is ready by sending message to inbox and notification update
* User browses to inbox and reviews the file 
* 
* Browse to API
* Click on Validate for one of the rows that already have an import configuration completed
* User redirected to the Validate drawer
* User clicks Validate button
* System shows message that it will be sent to their inbox and they will be notified once complete
* System performs a validation of the data as if importing, but doesn’t import
* System saves the exception analysis to the database
* If the user is still on the page, the system updates the list of previously run analysis
* The user then clicks to view the analysis button and the system loads the results in a preview grid
* User optionally downloads the results
* System also notifies the user when the file is ready by sending message to inbox and notification update
* User optionally browses to inbox and reviews the file

## Acceptance Criteria
* User can click the button to start the validate process
* User is informed of the process and can confirm
* PII view restrictions are respected for import types that are marked with it
* Data is validated, but not imported
* User is notified when the file is ready to view
* User can open the file from their inbox
* Overlay shows the history of last 30 validations
* User is able to view the exception validations from previous analysis
* User can view results by clicking view
* User can view source data
