# NXT-37461 — Student Import – Code validations

## Description
As a SchoolCafé Admin I want to ensure that code fields are validated so that I can more gracefully handle import failures.

*Notes:*

* We have been working on updating the Student Import validations. These are additional ones that have come from onboarding new customers
* They relate to validating fields that have codes associated with them. The idea is to ensure that the codes are accurate when we import.

*Requirements*:

* When importing a Student Import, do the following:
* Validate that the codes / values for these columns are correct:
** GradeCode
*** If fails, do not import the row since this is a required field and we have no default to use, and log the exception
*** Currently, it will import unvalidated codes
*** Validate by region id
** Gender
*** -If fails, substitute “N” for the value as it is the default and import the row, and log the exception-
*** If fails, substitute “U” for the value as it is the default and import the row, and log the exception
*** JAS Note: Was going to be N, but we changed to U since N was changed to “Non-Binary”
** Ethnicity
*** If fails, substitute “7” for the value as it is the default and import the row, and log the exception
** Race
*** If fails, substitute “7” for the value as it is the default and import the row, and log the exception
** Language
*** If fails, substitute “ENG” for the value as it is the default and import the row, and log the exception
*** Note: I assume ENG is the default, if it isn’t, then use the actual default
** Site Code
*** If fails, do not import the row since this is a required field and we have no default to use, and log the exception
*** Validate by region id
* For each one, check that the value in the import file matches an acceptable value in our database and then do the action in the bullet point below it
* This validation should be for both manual and automatic imports

## Acceptance Criteria
* Codes that do not exist are not imported
* If the code has a default value, it is used in place of the bad code
* GradeCode rows that fail are not imported
* All other rows are fixed imported
* Works for both manual and automatic imported
