# NXT-25683 — INV - Orders - Production Ordering - Parameters + Production Validations (Step 1)

## Description
As an Ordering Manager I want a Production Orders page so I can select a date range for production records and begin ordering.

This is the first step in a multi-step process to place Production Orders. Production Orders are orders that are generated based on quantities of items used as part of serving. The quantities used on production records will drive the required order quantity.

*Requirements*

* Create a new Production Ordering page
** this should be in the navigation under the Orders category
* Provide a header for initial selections
** Site selection common control
*** *single selection*
** Date range
*** this will determine the set of production data we will retrieve
*** do not allow past dates 
*** default range
**** From Date: Today + 1
**** End Date: Today + 6
** Apply button 
*** this will search and retrieve production record item data according to the date range
* Alert user of any Incomplete Production Plans for the selected date range
** This should provide an option to direct the user to the plan to complete it (more details TBD)
** Incomplete plans prevent the user from production ordering entirely; do not load items or execute the remaining order item logic ([NXT-25684|https://primeroedgenext.atlassian.net/browse/NXT-25684])
* Alert for no data
** "No production data found for the selected date range"
