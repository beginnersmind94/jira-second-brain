# NXT-50417 — Roles & Permissions - SchoolCafe Roles

## Description
As a Customer Support role user, I would like to see what SchoolCafe only roles there are and who is in them so that I can know who has access to what page

*Notes:*

* The existing R&P is for districts and their roles
* We need a page that shows us the Admin roles like Customer Support role
* We want to know who is in the role and what rights it has
* The requirements are for information only; we are not editing permissions, cloning or modifying

*Requirements:*

* These requirements apply to the new version of Keycloak
* On the Roles & Permissions page:
* Add  -tab- button
** Name: Admin Roles
** Security: Visible to Customer Support role and Tech Support role users only
** Invisible to District users
** On click, open the Admin Roles page
* Add a button
** Name: History
** On click, show the History overlay
** On History Overlay, show all the additions, deletions related to these roles



Admin Roles -tab- page

* We will copy the existing UI for Districts to save time and effort
* Do not include the Add New button or its functionality (no create new role functionality)
* Do not include the Clone button or its functionality
* Do not include the Delete button or its functionality
* History
** If we are currently tracking history, then do it, but if we aren’t then don’t include it
** Show this button on each row
* Grid:
** Don’t need sorting, filtering or paging but can include it
** Data: List of Admin roles like Customer Support role, Tech Support role, License Management role, and any others
* Edit button:
** keep and open the Edit screen
* 
* Security
** Only users with Customer Support or Tech Support role can see or do anything on this tab



Edit Screen

** Show all the users that belong to the role
** Be able to add a user to the role

## Acceptance Criteria
* User can see what Admin roles there are
* User can see who is in the roles
* Only Customer Support role and Tech Support role can see this data
* User can add or remove users from the role
* User is able to view history of changes
