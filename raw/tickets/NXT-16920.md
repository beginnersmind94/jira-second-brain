# NXT-16920 — Notification Scheduler

## Description
As a system I need a scheduler that runs at a certain intervals and does the following: 

#  It should fetches a list of records from NT_Config of type 'Scheduled'  with its associated NT_ConfigSchedule with it from the System’s Db.  
# Within these records,  check if any config’s schedule has met the criteria and if so, we need to log the config to the db and publish it  as a message to rabbit mq  for the topic “NotificationRequest“ with the entire notification config record. _Once the message is published the modules would be listening to this topic and process it an publish the response as a message to “NotificationResponse“ topic(needs to be done by the module teams)_.
# Need a listener that will subscribe to the “NotificationResponse“ topic. Once a message is received the details should be saved to Db.
