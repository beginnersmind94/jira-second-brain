# NXT-24910 — IM - Items Details Import - Business Rules Only

## Description
IM - Onboarding Items - Basic Details

As  an Item Management Super User / Administrator, I need to be able to onboard new Items using an excel file in order to speed the process of bringing customers working with SchoolCafe’s Backoffice.  

*Entry Point:*

* System → Import Dashboard → Import Set Up
The user should able to onboard the data based on the system templates defined by the system Item template.

*The Items details data onboarding process shall follow the following procedure/steps*

*Step 1 - Pre-Upload (Platform Team will manage this)*

* Select Import Type
** Options
*** *Item Details*
** Select Data Source
*** By Default, set to Local
Out of Scope - other data sources other than local
The user shall be prompted (Would you like to download a template?) at this step either to download a template or not

 *Step 2 - File Upload (Platform Team will manage this)*

* If user selects Yes, the system will download a template (excel format) and the user will use the template to fill the data
If the No, then template then the user shall be prompted to upload a file
Once file is uploaded the system shall provide the user with visual confirmation of success or failure. Any empty file gets filtered at this stage
The user shall be prompted to perform Step 3 - Field Mapping

*Step 3 - Field Mapping (Platform Team will manage this)*

* Fields shall be mapped by selecting fields from source file (loaded template) and dragging and dropping to the mapped area
All mandatory or optional fields can be mapped at this stage
* Once fields are mapped the user shall be provided with:
** an option to preview the file data as mapped;
an option to return to the previous step
an option to continue to next step (Step 4 - Review)
* If user selects the previous option the system shall remove the uploaded file and move to Step 2
if the user selects continue then the system shall move to step 4

 

*Step 4 – Review (Platform Team will manage this)*

* The system shall display:
**  map fields between Source and Target (Source - SchoolCafé)
** Current File Name (Data File use)
*** User shall be able to rename the file
* The system shall provide the user with an option
**  to enable (Activate) the file mapping;
 an option to edit the mapping;
 and an option to save the mapping
* If user selects to save the mapping then the user shall receive visual confirmation of success or failure of the saving process and ask the user for acknowledgement
* If the mapping is successful then the user shall be directed to the Import dashboard where the user shall have the option:
** to disable the mapping
** to import the data according to mapping;
*** If user selects Import then the system shall use created mapping for import process
**** The user shall load data file (drag and drop area or selecting from specified file location)
Once file is loaded the system shall display data as mapped
**** Then user shall provide the user with option
***** Cancel the import process
Move the previous page
***** Complete the import file process
****** User shall be presented with information about file import and be presented with options
******* to go to import dashboard
To go to Monitoring page where the user can view the progress of file import and view log file

## Acceptance Criteria
*The following are the acceptance Criteria (please refer the attached excel sheet for all the log messages)*

*Business Rules:*

* *Base Rules*
** Scope of the Import – only INSERT OR Update only
*** A Record should be inserted only when the following qualifiers are passed
**** Item Code provided does not exist in local data source
All the Mandatory Fields are provided
Values provided against Configuration related fields are absolutely matched
No data type mismatch is found against one or more fields
Any conditional validation is fully met
*** A Record should be Updated only when the following qualifiers are passed
**** Explicit Item Code must be provided; Mandatory fields are not applicable In case of Update transaction
Item Code should already exist in the Local data source
Values provided against Configuration related fields are absolutely matched
No data type mismatch is found against one or more fields
Any conditional validation is fully met
** The default data source = Local, other data sources are out of scope (Ex: CNDB, Global Catalog), Creation of Menu items Or Single Item Recipe are out of scope
All fields mark as Mandatory = yes in the Sheet (Exception Message) are *MANDATORY*
*** Fields that are conditionally Mandatory should be applied as Mandatory based on the scenario listed in the sheet



* Default Status of the imported record = Active
The imported data for this is editable via UI once imported.
* *Field Level Rules*
** No duplication allowed on following Fields:
*** Item Code, Item Name with in local data source
** Item Category should be a valid item category (Validate against Item Category Configuration)
** Manufacture Name
*** Validate against Item Manufacture Configuration
** Brand Name
*** Validate against Item Brand Configuration
Brand Association with manufacturer need NOT be followed
**  Storage Category should be a valid storage category (Validate against Storage Category Configuration)
Valuation Group should be a valid Valuation Group (Validate against valuation group Configuration)
Unit Weight - Only Numeric values, no negative values allowed; Zero is allowed, Decimals are allowed
Unit weight measure should be a valid unit weight measure (Values Allowed: Grams, Ounce, Pound)
Fair Market Value / Standard Price – Only Numeric, No negative values allowed; Zero is allowed
** “Is Food” field is a flag indicating if the item is a food or non-food item (Flag) – Values Allowed Yes or No
*** Conditional Validations are applied on other fields if Is Food = yes (pls refer the attached excel sheet)
** Tags - If more than one Tag needed then they shall be comma separated
Edible Yield Factor set to 100 by default unless explicitly specified otherwise
** If Food Item = Yes and Nutrient fields are left blank then mark as (M) except for Mandatory Fields
*** Mandatory Nutrient Fields:
**** Calories, Total Fat, Saturated Fat, Carbohydrate, Protein
*** If Food Item = No, then whole Nutrient section is treated as optional for that record. Any value provided against the field should still be imported unless there a is data type mismatch
** Allergen Section is optional for all the records, the value should be imported as per the standard list
 

*Error Log Rules*

* One record in the excel template may encounter multiple logs, all the errors against a row should be captured
If multiple logs are captured for a record, then all the log messages should be reflected as separated by semicolon (Ex: \[Field Name] –Mandatory Field; \[Field Name] – Configuration Mismatch
Each Log should be shown a new line in UI, so that user can review fix them easily
* If field mark as mandatory is not provided then ignore record and continue with next
** Capture the Names of the Fields that are not provided
* If the mandatory or optional fields are found with data type mismatch, then ignore record and continue with next
** Capture the Names of the Fields that are not matched with Data Type
* If mandatory field or optional is based on the values of configurations and did not match, ignore record otherwise
** Capture the Names of the Fields that are not matched with configuration
* If Item ID is not defined, create new item code
** Capture the Item Code created for each row, this would be download by the user for verification (Excel Download option)
* Item ID provided and not conflicting with existing records then create new item code. If not then
** Capture the Codes as “\[Item ID] - Duplicate”
* Fields Name between brackets represents the field value used in the template. for example, for \[Item ID] represents the Field Value in Excel or Mapped Field in Staging Table
* *Mandatory Fields*
** If mandatory field is not provided log in transaction file with exception message "\[Field Name] – Mandatory” and proceed with next record
** Field Applicability
*** Non-Food Item (Is Food = No)
**** Item Name
**** At least one Item Hierarchy is defined (in Unit Info section)
**** Product Info (Item Category, Storage Category, Valuation Group)
**** At least one pack size configuration (Unit Hierarchy) is Mandatory
***** The user should be able to see the calculated price or weight of the other units once it is imported (Story Ref # NXT-30046)
**** Fair Market Value
**** Lowest Unit Weight & Weight Unit of Measure
*** Food Item (Is Food = Yes)
**** Item Name
**** At least one Item Hierarchy is defined (in Unit Info section)
**** Product Info (Item Category, Storage Category, Valuation Group)
**** Mandatory Nutrient Fields:
***** Calories, Total Fat, Saturated Fat, Carbohydrate, Protein
**** At least one pack size configuration (Unit Hierarchy) is Mandatory
***** The user should be able to see the calculated price or weight of the other units once it is imported (Story Ref # NXT-30046)
**** Fair Market Value (always assigned to ‘Highest’ unit)
**** Lowest Unit Weight & Weight Unit of Measure
**** Serving Measure Fields

                             §  Serving Size Quantity, Serving Size Unit, Serving Size Weight, Serving, Size Weight Unit



* *Value Already Exist / Duplicates*
** Log error on Import Transaction Log file with message "\[Field Name] - Duplicate"
*** Please refer the excel sheet for potential list of messages
* *If Item ID is not defined*
** Create new Item ID if no conflict with existing record

* *If Item ID is defined*
** Create new Item ID if no conflict with existing record

* *Configuration Mismatch (Mandatory or Optional Fields)*
** If value for a field is provided is not found in the configuration " \[Field Name] – Configuration Mismatch"
*** Please refer the excel sheet for potential list of messages

 *Data Type Mismatch (Mandatory or Optional Fields)*

* If value for a field is provided is invalid with data type then log the message as " \[Field Name] – Data Type Mismatch"
* Field Applicability
** Please refer the excel sheet for potential list of messages

See attached Item Import sheet for all possible validation messages that are required



*Data Length Specific Validations:* 

GTIN - should be 14 digits, any length which is less or more than 14 digits should be captured as exception as listed in the sheet



*General Rule:*

All the white spaces - Start and End of the fields should be trimmed

All the exception messages are available in the attached excel sheet, and it should be captured based on exception conditions as mentioned

*-Excel Download Option-* -- user should be able to download Excel with logs with all the original data + Import Status (Success/Fail) + exception messages associated with that record - Sample format attached-

Sorting on View Log Page
Page Filters
Pagination
Grid / Column Alignments should be aligned for optimum view
