# NXT-5171 — Device Management – Device Management Page

## Description
_Phase 2, System team Step 1_



As a SchoolCafé admin, I want to see my list of POS devices so that I can monitor and action them.

*Flow:*

* User logs in and browses to System > Management > Device Manager
* User views the list of devices registered
* User optionally adds a new POS device
* User optionally edits an existing POS device
* User optionally suspends or restarts a POS device
* User optionally deletes a POS device
* User optionally views the history of changes for a POS device
* User optionally views the details of a POS device

*Notes:*

* We created a generic device manager last year and started with the Face & Food Detection Edge terminals. That code was completed but is most likely obsolete now. In any event, that project is on hold and most likely is years away from being done
* What we will do in these POS device management stories is leverage the existing code that has been written and fit it to the POS terminals and ignore the FFD stuff
* Note: this feature requires the work of both System and FO teams.
** To achieve this, we will divide the work up into 3 phases
** Phase 1: FO team to update their data table and write APIs
** Phase 2: System team to write all front end code and leverage FO APIs
** Phase 3: Both teams do integration testing to ensure it is working
* The data table that will be used is in the front office database
** So all data will be pushed and pulled from that data table
** Screens will need to be reconfigured to use that data
** In order to save work, we will simply disable the existing Add New and Edit screens and start over with clean code for POS terminals
** The existing Add and Edit screens can be kept in case we come back to FFD in the future

*Requirements*:

* All of these requirements are for the Device Management page
* *Face & Food Detection Code:*
** Remove any code that is specific to the FFD project (There may not be on this page, it is mostly on the Add and Edit screens)



*Filters:*

* Move filters up to this section
* Have it collapsible per Figma
* FO team will provide the APIs to populate the fields lists
* 
* Fields:
* Device Name - Textbox, filters as start typing
* Site ID - Textbox, filters as start typing
** Note the column is not visible in the grid
* Site Name - Multi-select list
* Device Type – Multi-select list
* Status – multi-select list
* POS Version – would prefer a multi-select list, but might be hard; if difficult, then a textbox that filters as you type is acceptable
* Last Communication –
** Single select List
** Source value is a datetime datatype
** What we want to do is filter results by looking for devices that haven’t checked in for a long time
** We could do this with a date range, but what would work best is like the filter we have in import monitor where it has 24 hours, 7 days, etc., but do it in reverse
** The filter would work against the last communication date
** Values:
*** All - sets end time of the date range to now(); this is the default value
*** Yesterday – sets end time to midnight the day before
*** 7 days – sets end time to midnight 7 days before
*** 14 days – sets end time to midnight 14 days before
*** 30 days – sets end time to midnight 30 days before
** Don’t’ bother setting a start date time, just filter by end date time
* Description – Textbox, filters as start typing
