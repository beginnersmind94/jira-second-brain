# NXT-48860 — Users - Managed Views - Saving - Not Required

## Description
As a SchoolCafé users, I would like to be able to see more information about users than what is in the default list using Managed Views, so that I can see other information.

Notes:

* This is related to the Global Managed Views story
* We have broken out the saving a Global Managed View into this separate story to make it easier

*Requirements:*

* On System > Users, do the following:
* 
* Add the Manage Views icon above the grid if it is not already there
* Add the ellipsis icon next to the Manage Views icon if it is not already there
* Change the font and colors of button labels to be consistent with Figma
* 
* Add the Save As button to the ellipsis
** On click, open the Save As drawer
* 
* Add the Save button to the ellipsis
** Users can save only their own managed views, so disable Save for anything they can’t save, except …
** Customer Support role can save any global managed view change
** Save only applies to a Saved view, so disable if not viewing a saved view
** On click, do the same as you do now already for Save
** Ensure the filter criteria is included in the saved view



*Managed View Save As Drawer*

* Title: Save Managed View
* Fields:
* View Name
** Required
** Same max length as original field
** Currently doesn’t allow spaces, but we would like it to allow spaces
* Save as Global Managed View
** Checkbox
** Default unchecked
** Note: This is a new field added and is need in order to create Global Managed Views
** If this is checked, then the view will be saved as a Global Managed View
** If checked, disable or hide the sharing stuff fields below it since these don’t apply to Global Managed Views 
** This checkbox should only be visible to Customer Support role
* Save as Default View
** Checkbox
** Default unchecked
** Always visible
** Note: current name is Save as favorite
** Keep the functionality for this field as is, no updates
* Share with Others
** Checkbox
** Default unchecked
** Uncheck and disable if Save as Global View is checked
** If check, then uncheck Global Managed View checkbox since they can’t both be done together
** Keep the functionality for this field as is
* Users
** Note: current Save has this already, so keep as is, just change the control to match what we do now for sharing with users
** Hide or disable if Save as Global View is checked
** Hide or disable if Share with Others is unchecked
* Roles
** Note: this is new … sharing with Roles, but being added to Managed Views sharing, so add in the same way as do for other screens
** Hide or disable if Save as Global View is checked
** Hide or disable if Share with Others is unchecked
* Save button
** Ensure the filter criteria is included in the saved view
** Ensure save as Global Managed View if that checkbox is checked



*Backend*

* Create a default global view for the Users dashboard
** Name: Default View
** Copy over all the columns and settings from our default view
** Ensure this is in the list of Saved Views as a System View
** Make this the default view
** Ensure the user can always click on this view to go back to our default view
* 
* Create one global managed view
** Name: Users Audit Fields
** Columns to show:
*** Email
*** First Name
*** Last Name
*** Creation Date
*** LastLoginDate
*** LastFailedLoginDate
*** UpdateBy
*** UpdateDate
*** Actions
** Ensure the user can select the Users Audit Fields in the list in Saved Views and that updates the grid accordingly

## Acceptance Criteria
* Ellipsis icon added next to the Manage Views icon
* Save As button added to the ellipsis and opens Save As drawer
* User can see global views in the Manage Views drawer
* Customer Support role can save a view as a default
* Only Customer Support role can save a saved global managed view
* Only Customer Support role can edit a saved global managed view 
* Only Customer Support role can delete a saved global managed view
* Actions column always visible in grid
* Default View created as a global view with the specs mentioned and is visible in saved views
* Users Audit Fields View created as a global view with the specs mentioned and is visible in saved views
