# NXT-70966 — Insights - KPI - ENP - Drawer - PE

## Description
*I am a* district user *that I want to* open the Eligible Not Participating (ENP) details drawer from Insights
*so that I* can review the actual value, expected value, program-level breakdowns, and school-level detail behind the KPI.

----

h2. *Notes*

* This story applies to *Insights – PrimeroEdge data source*.
* ENP (Eligible Not Participating) is part of the *front office KPIs* being integrated into Insights.
* This story covers the *ENP Details Drawer only*.
* The user reaches this drawer by clicking:
** The *ENP KPI card* on the dashboard
** The *ENP value* in the School Performance grid
* The drawer UI should follow the same pattern used by existing Insights details drawers.

----

h3. *Data & Source of Truth*

* The source of truth for ENP data is available in:
** *POS → Daily Reports → Patron Participation*
* To generate the correct dataset:
** Filter *Eligibility* to *Free and Reduced only*
** Generate report
* This must be used by developers and QA to validate ENP calculations and displayed values.

----

h3. *Stored Procedure Integration*

* The front office team has already created the stored procedure:
** PrimeroEdge: [https://dev.azure.com/Cybersoft-Technologies-Inc/PrimeroEdge%20Classic/_workitems/edit/109336|https://dev.azure.com/Cybersoft-Technologies-Inc/PrimeroEdge%20Classic/_workitems/edit/109336|smart-link] 
* The details drawer must use this stored procedure as the data source for ENP values and supporting datasets.

----

h3. *KPI Definition & Formatting*

* ENP is a *percentage-based KPI*.
* Format must be:
** *0.00% (two decimal places)* on the drawer
* Values must be:
** Rounded to two decimal places
** Between *0 and 100*

----

h3. *KPI Direction*

* This KPI is *Lower is Better*:
** Values closer to *0% indicate better performance*
** Higher values indicate worse performance

h2. *Out of Scope*

* SchoolCafé implementation (separate story)

h2. *User Flow*

# A user logs into PrimeroEdge.
# The user navigates to *Insights*.
# The user lands on the Insights dashboard.
# The user filters the dashboard by timeframe and/or site.
# The user clicks either:
#* The *ENP KPI card*, or
#* The *ENP value* in the School Performance grid
# The *ENP Details Drawer* opens.
# The user views:
#* Drawer header with metric title and analysis date range
#* ENP Overview section
#* Program by Eligibility section
#* Details by School section
#* About this Metric section
# The user reviews program-level and school-level breakdowns.
# The user closes the drawer and returns to the dashboard.

----

h2. *Requirements*

h3. *1. Drawer Access & Header*

* Clicking the ENP KPI card must open the ENP Details Drawer.
* Clicking the ENP value in the School Performance grid must open the ENP Details Drawer.
* The drawer header must display:
** *Eligible Not Participating*
** *Analysis for [date range]*
* The displayed date range must reflect the currently selected dashboard timeframe.

----

h3. *2. ENP Overview Section*

* The drawer must include an *ENP Overview* section.
* The overview section must display:
** *Actual Value*
** *Expected Value*
** *Difference*
* Values must use percentage formatting of *0.00%*.
* Difference must display:
** Numeric difference
** Percentage difference (if supported by existing pattern)
* The expected value comes from the benchmark configuration.

----

h3. *3. Program by Eligibility Section*

* The drawer must include a *Program by Eligibility* section between Overview and Details by School.
* The section must display a matrix with:
** *Eligibility* as rows:
*** Free
*** Reduced
*** Paid
** *Programs* as columns:
*** SNP
*** SFSP
*** CACFP
* Each cell must display:
** *Count*
** *Percent*
* The matrix must include:
** Row totals
** Column totals
** Grand totals
* Columns must be *sortable*.

----

h3. *4. Details by School Section*

* The drawer must include a *Details by School* section.
* The section must display a matrix with:
** *School* as rows
** *Programs* as columns:
*** SNP
*** SFSP
*** CACFP
* Each cell must display:
** *Count*
** *Percent*
* Columns must be *sortable*.
* Pagination must be implemented:
** Default page size: *5 schools per page*
* Default sorting:
** *Highest percent Paid descending*
* Launch behavior:
** If opened from KPI card → show full filtered dataset
** If opened from grid row → show only selected school

----

h3. *5. About This Metric Section*

* The drawer must include an *About this Metric* section.
* This section must display:

*Eligible Not Participating = TBD (final calculation to be defined)*

*ENP is the percentage of students who are officially certified for Free or Reduced-price meals but choose not to eat the school meal on a given day. These are students whose paperwork is complete and approved, yet they are still bringing lunch from home or skipping the meal entirely.*

----

h3. *6. Data Integration*

* The details drawer must use the *PrimeroEdge ENP stored procedure*.
* Data returned must support:
** Actual value
** Expected value
** Difference calculation
** Program by Eligibility matrix
** Details by School matrix
** Sorting and pagination

----

h3. *7. Formatting & Consistency*

* The drawer must follow existing Insights drawer patterns.
* All percentages must be formatted to *0.00%*.
* Counts must use standard number formatting with commas.

----

h3. *8. Performance Requirement*

* The details drawer must load without performance degradation.
* Must support large datasets (many schools and programs).
