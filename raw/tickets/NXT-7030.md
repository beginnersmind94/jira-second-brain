# NXT-7030 — Deposits - Modify Deposit Slip

## Description
As the site manager or director I need to be able to modify deposit slips for the bank based on the days transactions. Sometimes I deposit multiple days at one time.

*Mockup:*

[https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=5834%3A730&scaling=min-zoom|https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=5834%3A730&scaling=min-zoom]

*Requirements:*

* Select a deposit slip from the Deposit Slip List
* Data is displayed based on the data saved
* Only the sessions in the original deposit slip are included
* The Date of Deposit remains the same
* Checks in for the deposit slip are auto selected from the sessions in the date range. [https://primeroedgenext.atlassian.net/browse/NXT-5373|https://primeroedgenext.atlassian.net/browse/NXT-5373|smart-link] 
* Checks can be added [https://primeroedgenext.atlassian.net/browse/NXT-5373|https://primeroedgenext.atlassian.net/browse/NXT-5373|smart-link] 
* Denominations are entered by the manager [https://primeroedgenext.atlassian.net/browse/NXT-5372|https://primeroedgenext.atlassian.net/browse/NXT-5372|smart-link] 
* The Summary displays the Cash Total, Check Total and calculates the Actual Deposit Total by adding them together. [https://primeroedgenext.atlassian.net/browse/NXT-85|https://primeroedgenext.atlassian.net/browse/NXT-85|smart-link] 
* The Summary also includes the Expected Deposit.
* The Expected Deposit is calculated by adding all the Cash and Checks from the sessions included in the deposit. Cash is calculated by subtracting the opening balance cash from the closing balance cash for each session included in the deposit. 
* IF there is a difference between the Expected Deposit and Actual Deposit Total a comment must be entered to save the deposit slip, A message is displayed saying a comment must be entered.
* The EDGE Reference number Version is incremented when saved (Its an increment of the previous reference number within the district) FORMAT SSSSMMDDYYNN-V
** SSSS = Site ID
** MMDDYY = Deposit Date
** NN = Deposit Number (one up number for by site for the deposit)
** V = Version of this deposit slip 0 for initial deposit slip. Plus one for each modification..

Flow:

# User Selects previously created deposit slip from Deposit Lists Page
# Deposit Detail page is displayed with Denominations, Checks, and comments populated from saved deposit slip data
# User changes cash to deposit, verifies checks, enters comment if necessary and clicks save, or makes no changes and clicks cancel.
# System returns to Deposit Lists page

## Acceptance Criteria
User can enter a Bank Deposit Slip number (optional)
Only Reconciled sessions are included in the calculation for Expected Deposit and Checks 
The status for all sessions included is changed to DEPOSITED when the deposit slip is saved.
A comment is required for all deposit slips where the Expected Deposit and Deposit Total are not the same.
The Deposit Total is Total Amount of All Checks in all Sessions PLUS the (Closing Balance - Opening Balance) of all sessions included in the deposit slip.
When saved the Session IDs are saved with the Deposit Slip
When an existing deposit slip is modified the Reference ID 'SSSSMMDDYYNN' is kept and the Version # is incremented by 1.

When a Deposit Slip is modified a Comment must be entered to save.
The SAVE button is not enabled unless a change has been made and a comment entered.
CANCEL can be selected at any time and all changes or modifications are ignored.
