# NXT-69127 — Change GL Account Mapping

## Description
Given a system account is mapped to a GL account When the user changes the mapping to a different GL account Then the change only affects future posting operations And existing ledger entries retain their original GL account assignments

## Acceptance Criteria
Make message global for story and combine [https://primeroedgenext.atlassian.net/browse/NXT-69084|https://primeroedgenext.atlassian.net/browse/NXT-69084|smart-link] 

*Scenario 1: Change mapping with no posted activity*

* User can select new GL account from dropdown
* User confirms change
* Mapping is updated immediately

*Scenario 2: Change mapping with posted activity in closed periods*

* User can select new GL account from dropdown
* System displays warning: "This account has posted activity in previous periods. Past ledger entries will remain on account [old account]. Future postings will use account [new account]. Reports spanning both periods will show both accounts."
* User confirms change with "I Understand, Change Mapping" button
* Mapping is updated for future postings only

*Scenario 3: Change mapping with unposted transactions in current open period*

* User can select new GL account from dropdown
* System displays message: "The current period [month/year] has not been posted yet. When you post [month], ALL transactions will use the new account: [new account]."
* User confirms change
* Mapping is updated immediately
