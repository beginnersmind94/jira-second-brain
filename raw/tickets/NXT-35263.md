# NXT-35263 — INV - Orders - Receipts - Create/Edit Receipt performance improvements (+ R&D)

## Description
As a user I want the Create/Edit Receipt screen to be as quick and snappy as possible so I can quickly complete one of the most laborious business processes. 

h4. Observations 

* Second+ delay on click of most buttons and some fields (Bin Line icon, Comments icon, Order Delivered By field, etc.)
* All actions in Bin Line slideout have a delay
* Scrolling delays
* Big delay on click of Confirm checkbox
* Save takes a few seconds depending on item quantity
* *Wrap* the Item Description

h4. Performance Notes (from discussion)

* Considering alternate Kendo grid
** flat table instead of K grid so we don’t have to rebind the grid 
** no filtering, sorting, nor Manage Views
* Future read-only (labels) version of Price and Charge/Discount fields
* Future option to show/hide Charge/Discount columns
* Alternate icon for when items have additional Bin Line records added (Icon decision with [~accountid:5f99ce5044658b0071265de9] )
* UX Design adjustment considerations
