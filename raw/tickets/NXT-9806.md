# NXT-9806 — Data Exchange - File Import Dashboard: File Import Setup DELETE

## Description
As a PrimeroEdge user, I would like to have an intuitive screen that allows me to view all file import setup configurations that are available for me with an option to DELETE FILE IMPORT SETUP configurations so that I can remove unnecessary or invalid configurations from the system.

*Mockup:*

[https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=388758%3A238848&scaling=scale-down|https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=388758%3A238848&scaling=scale-down|smart-link] 

!image-20210112-143456.png|width=734,height=234!

*Requirements:*

* When the user has clicks on “*File Import Dashboard*” under configuration, file import dashboard is loaded to view, create and import configurations.
* File Imports configurations table is displayed with all file configurations available for that user (districts)
* Each row has “DELETE” buttons available when user clicks on ellipsis icon
* when user clicks on DELETE:
** Confirmation prompt is displayed with message 
*** *Title*: File Import Setup Delete Confirmation
*** *Message*: “Are you sure you want to delete this file import setup?
*** Buttons: *CANCEL* on left, *CONFIRM DELETE* on right
** When user clicks on CONFIRM DELETE, that specific file import setup config row is deleted from database/system
** When user clicks on CANCEL, PROMPT disappears and returns to dashboard page
* User is redirected with refreshed file imports configuration table

## Acceptance Criteria
* Verify that when the user has clicks on “File Import Dashboard” under configuration, loads the dashboard page with DELETE action available in ellipsis icon for each row of the file import configuration table. 

* Verify when user clicks on DELETE action, confirmation prompt with 2 buttons are displayed as specified in requirements

* Verify that once File Import Setup row is deleted, user is presented with refreshed file import setup configurations table
