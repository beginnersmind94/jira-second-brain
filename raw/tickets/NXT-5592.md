# NXT-5592 — Applications - Process Applications Mode

## Description
*User Story*

As an applications processor I want to process applications that are in need of review.

*Mockup*

[https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=421304%3A268450&scaling=min-zoom|https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=421304%3A268450&scaling=min-zoom|smart-link] 

*Requirements*

* User has a ‘Process Applications’ button available to them which will take them into a continuous-flow version of the Process Application window.
* User is taken to the oldest application that needs to be processed. 
* User can skip an application and be taken to the next oldest application.
* User can process the application and be taken to the next application.
** This would be the ‘Process’ button in the mockup.
* Two users engaged in this flow should ideally never be taken to the same application for processing.
** We may need a ‘check out’ flag for each app.
** If a 2nd user has the same app open for processing and attempts to process it, we should give the user a warning prompt.
*** ‘The processing window for this application is currently opened by Firstname Lastname. Are you sure you want to process this application?’
*** User can confirm that they’ll process the app or cancel back.
* User can go back to the last application they processed and re-process / review what was done.
* User has a way to ‘Finish Processing’ and exit the continuous-flow processing mode.
** A ‘Process & Quit’ action, and the Cancel button.
** User should be taken to a summary page after this - handled in another story.
* System must track this processing session:
** User
** Start time
** Finish time
** Applications processed during this session



Discussion from 3/2/21

* Applications need to be checked out and locked (via SQL) to a single user before they are presented with the processing screen.
** Two users should not end up at the same application.
* This lock should also happen when the user:
** attempts to process via Details > Process.
** navigates back to a former app via Previous Application.
* No time-out for these locks right now - a new story will be added to allow users to manually check an application back in.

## Acceptance Criteria
User able to enter into a 'Process Applications' continual processing page.
User is taken to the oldest application for the current academic year that requires processing. Status <> (Processed, Notified)
User can skip to the next application.
User can process an application and be taken to the next.
User can go back to applications they've processed in this session.

Apps are tracked in a session log that can be reviewed later (in another story)

Two users in this mode should not be taken to the same application number. If 2 users have this application open for processing, the system should warn both users that another person (first name + last name of user) is also reviewing this application right now.
