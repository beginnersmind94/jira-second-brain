# NXT-54930 — MP- Menu/Menu Cycle- Persistence on List pages when using cancel/back (UI ONLY)

## Description
As a menu planner, I would like my search and or my filters to stick if I am going back my list pages if I am quickly making corrections to menus/menu cycles

## Acceptance Criteria
* Apply Persistence to Menu List and Menu Cycle List page
** Default to not show filters and add to the manage views (see Item List page)
* Preserve all filters and searched criteria applied by user and views if applied
** This includes the inactive filter
* This applies if they use the close, or cancel/back button on the menu, menu cycle
** This applies to the menu planner if the user the *save and close* or *cancel*
