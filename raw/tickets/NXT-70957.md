# NXT-70957 — Insights - KPI - PNA - Dashboard - PE

## Description
*I am a* district user *that I want to* view Paid Not Applied (PNA) performance in the Insights dashboard
*so that I* can monitor actual performance against expected benchmarks across sites and timeframes.

----

h2. *Notes*

* This story applies to *Insights – PrimeroEdge data source*.
* PNA (Paid Not Applied) is part of the *front office KPIs* being integrated into Insights.
* This story covers the *Dashboard screen only*.

h3. *Data & Source of Truth*

* The source of truth for PNA data is available in:
** *Student Eligibility → Eligibility Reports → Eligibility Roster*
* To generate the correct dataset:
** Eligibility Type = *Paid*
** Reason = *Default*
* This must be used by developers and QA to validate PNA values.

h3. *Stored Procedure Integration*

* The front office team has already created the stored procedure:
** PrimeroEdge: [https://dev.azure.com/Cybersoft-Technologies-Inc/PrimeroEdge%20Classic/_workitems/edit/109335|https://dev.azure.com/Cybersoft-Technologies-Inc/PrimeroEdge%20Classic/_workitems/edit/109335]
* The dashboard must use this stored procedure as the data source for PNA values.

h3. *KPI Definition & Formatting*

* PNA is a *percentage-based KPI*.
* Format must be: *0.00% (two decimal places)*.
* Values must be:
** Rounded to two decimal places
** Between *0 and 100*
* Calculation:
** *(Count of students where Eligibility = Paid AND Reason = Default) ÷ Total relevant student population × 100*
* The dashboard must display:
** Actual value
** Expected value (*PNA Goal from benchmark configuration*)
* PNA is a *lower-is-better KPI*:
** Values closer to *0% indicate better performance*
** Higher values indicate worse performance

h3. *Filtering Behavior*

* All PNA components must respond to *Timeframe and Site filters*:
** KPI card
** School Performance grid
** Performance Trends chart

h3. *Interaction Behavior*

* Clicking on PNA values (card or grid) must open a *details drawer* (covered in a separate story).

h2. *Requirements*

h3. *1. KPI Card – PNA*

* Add a new KPI card:
** Label: *PNA*
** Location: *added to end by default*
* Card must display:
** PNA percentage (actual value) for selected filters
** PNA Goal (from benchmark configuration)
** Trend indicator
* Values must be formatted as *0.00%*
* Card must update based on:
** Timeframe filter
** Site filter
* On click:
** Open *PNA Details Drawer*

----

h3. *2. KPI Card – Performance Indicators*

* Card must display color states:
** Green → Actual ≤ Goal (better performance)
** Red → Actual > Goal (worse performance)
** Grey → No benchmark configured
* This KPI follows inverse performance logic compared to other percentage-based KPIs:
** Lower values are better
** Higher values are worse
* If no benchmark exists:
** Card displays grey

----

h3. *3. Filters (Timeframe & Site)*

* Dashboard must include:
** Timeframe selector
** Site selector
* Changing filters must update:
** PNA KPI card
** School Performance grid
** Performance Trends chart

----

h3. *4. Screen Configurator*

* Ensure PNA is added to *Configure KPIs drawer*
* User must be able to:
** Toggle PNA card on/off
** Reorder KPI cards via drag-and-drop
* Changes must:
** Persist per user
