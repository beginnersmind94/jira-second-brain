## Description
Introduce a Clear Balance tool similar to the PrimeroEdge side.

Improvements:

* Allow user to run it multiple times.
* Set balance ranges.
* Select person types.

## Acceptance Criteria
* Page/Tool added to AcctMgt under the Tools/Settings Clear Account Balances
* Access to the page/tool requires a role permission:
** Clear Account Balances
** Give the permission to Director role by default along with any customer care roles
* Step 1 - User can set search parameters, multi-select where possible:
** Site - multi-select, all option
** Grade - multi-select
** Homeroom
** Roster
** Balance Range
*** Similar to the Reminders balance search criteria.
**** Above, Below, Between
**** Text boxes for $ amounts
** Account Status - Active, Inactive, All
** Account Type - Student, Adult, Special Accounts, All
** Pricing - Free, Reduced, Paid, Staff - multi-select
** Explanation text above filters: This tool will help you find and zero out account balances. It will create a session with balance adjustments. The history tab above keeps track of usage, and will keep a copy of all affected accounts.
** User should be informed if a job is already queued/running and that they need to wait for it to finish.
*** Notification text: Please wait for the current Clear Balances request in the queue to complete. Last adjustment on: {{DATETIME}} by {{USER}}.
**** Example: Please wait for the current Clear Balances request in the queue to complete. Last adjustment on: 08:40 AM by Matt Smithers.
* Step 2 - User sees search results.
** User can export the results to Excel.
** User can sort and filter the result columns.
* Step 3 - User can see a summary:
** Number of accounts being cleared
** Total $$ that is being cleared - sum of balances
* User has a confirmation prompt before anything executes.
** User can enter a comment to be used as part of the adjustment.
*** Balance Cleared. {{Custom Comment}}
* After confirming:
** Balances for accounts in the search results are updated to $0
** Balance Adjustments are added with the comment above.
*** Use the user’s ID for the adjustment.
** User is informed once the balances are removed.
* Only 1 adjustment per district should be possible at the time.
** Lock the tool until the last “job” is completed.
** Inform the user is handled in Step 1 above.
* Text on the page:
** Instruction text at the top - Matt to provide
* 2nd tab on the page - History.
** Track who ran the tool, full timestamp, copy of the excel download
* Remove the System Settings side-nav link in Account Management
