# NXT-31556 — Add 3 times tray detection failure notification in Training Mode

## Description
As a cashier, I want to see the success or failure of when the Edge device detects a tray properly in training mode so that I can know to make corrections.

*Application*: POS

*Summary*

This story works just like the other story for adding in notices for 3 failures, except it has an additional coding requirement.  Currently, the terminal receives no results file from Edge device when in training mode.  Now it will, so we need to write code to get that result and then take the same actions as in the other story.

*Flow:*

* While cashier is using POS in Training Mode:
* Detection cycle begins when student walks up to cashier
** Cashier selects student
** Edge detection triggered
** Edge does not detect food
** Edge notifies POS that food is not detected (new)
** POS “records” that food not detected (new)
** transaction is completed
* Detection cycle with no detection happens a 2^nd^ time in a row
* Detection cycle with no detection happens a 3^rd^ time in a row
* POS notifies the cashier in a message that food detection has failed 3 times in a row (new)

*Assumptions:*

* This was completed in classic and you can copy the code from it
* Get with Saketh on the classic code
* Do the other story “Add 3 times tray detection failure notification to cart screen” first or at least at the same time
* Once you have written the code to receive the results, leverage all the existing code from the other story to store results and notify the cashier
* You will need to wait on Amniscient to finish their new MQTT message for results when in training mode
* For the purposes of this story, a negative result that needs to be actioned will be called Food Not Detected results, but the actual name from Amniscient may be different

*Requirements:*

* Receive result message from Edge device when in training mode
** Write code to receive call and get data
** Once you have this message and result, the rest of the actions are exactly the same as the story “Add 3 times tray detection failure notification to cart screen”
* Take action on Food Not Detected results
** Do exactly the same thing as in the other story
* Update to existing Edge Results notification
** Do exactly the same thing as in the other story
* Show message
** Do exactly the same thing as in the other story
*  
* Security – no additional requirements necessary

*Mockup*:

* None

## Acceptance Criteria
* We can receive the results file from Edge device when in training mode
* If we receive the Food Not Detected results, we “store” it and count it
* If we receive the Food Not Detected results, we count it up to 3 and then displays the message
* If we receive the Food Not Detected results, we count it and if less than 3 do nothing
* If we display the message to the cashier, it resets the count to 0
* If we receive a normal detection results, it resets the count to 0
* The user can close out of the message return to work with the click on one button
* Existing code continues to work
