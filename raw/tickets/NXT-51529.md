# NXT-51529 — GDSN Connect - Item Upload - API Changes

## Description
Currently customers do not get an opportunity to review the translated data

Navigation: IM - items - Upload Icon

User should have two radio buttons  as shown in the wireframe

User can click only one radio button at a time

By selecting GDSN Template, the user should be able to upload the current base GDSN Template (sample attached)

The existing API shall be called to push the data into staging table

Once the data parsing is completed, the data shall be parsed into SC from the new staging table where the translated data is stored

The Stored Procedure will call the ADF from this new staging table

## Acceptance Criteria
Once the data parsing is completed, the data shall be parsed into SC from the new staging table where the translated data is stored

The Stored Procedure will call the ADF from this new staging table

The Status shall show as “Data Parsing Completed” and move on with existing ADF flow as-is

Regression:

File Type validations, Field Mismatch Validations etc..
