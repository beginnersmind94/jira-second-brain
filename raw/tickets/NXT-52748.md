# NXT-52748 — DC Import – CalPads (FD - 251638)

## Description
As a SchoolCafé Admin I want to be able to import the CalPads import file for DC Import so that I do our direct certification

*Notes:*

* This comes from a customer request: [https://primeroedge.freshdesk.com/a/tickets/251638|https://primeroedge.freshdesk.com/a/tickets/251638] 
* Customers use a predefined template from California to do imports
* We need to accommodate this template
* It uses Caret delimited files, with no headers and a specific date format 
* These requirements are designed to be able to handle these unchangeable format requirements

*Requirements*:

* For *Add New* and for *Edit* screens, when a user selects DC Import, do the following:
* 
* *Pre-Upload tab*
* Note: When the user selects Direct Certified, it show the File format field below it
* Add a value to the File Format field when the user selects Direct Certified
** Label: CalPads
* If the user selects CalPads:
** Do not show the template download section
* 
* *File Upload tab*
* If the user selected CalPads on the previous screen, do the following:
* Note: Make the assumption that they are using the correctly formatted file
* Delimiter is auto set to Caret and made read only
* Qualifier is auto set to none but user can change
* 
* If you can, validate the file is a caret delimited file
** If it is not, then show an error message like you do for other formats
* 
* When they upload the file, insert as the first row the predefined column headers we stored in the database (see below)
** Note: This will enable us to continue the rest of the import functionality without modification
* Note: no need to transform the date column until we import the data
* 
* File Preview button
** Note: the file has no headers, so need to handle
** If the user wants to preview the data, use our predefined column headers (see below) in the grid
* 
* *Field Mapping tab*
* Since you inserted the column names as the first row, you can use this tab normally without changes
* Automap the fields as follows, since they should always be the same
** Student ID = Student ID
** State ID = State ID
** First Name = First Name
** Middle Name = Middle Name
** Last Name = Last Name
** Certification Date = Effective Date
** Certification Status = DC Type
* 
* For DC Type field, if not already done, do auto code mapping as follows:
** M = DCMA
** R = DCMAR
** S = SNAP
** T = TANF
* 
* *Review tab*
* No changes



* For *Impot Monitor* screen, when a user selects to import CalPads DC Import, do the following:
* If needed, insert as the first row the column names the same way you did for the configuration
* When you do the import routine, transform the date fields to the required format our database needs



* Setup column names in the database:
* In the database, save the file configuration format settings by specifying the column names we want to use and their order 
** Remember, file has no headers, but it is always in same format, so we will specify the column names ourselves
* Name and order:
** Academic Year
** District
** School of Attendance
** Student ID
** State ID
** First Name
** Middle Name
** Last Name
** Certification Date
** Certification Status

## Acceptance Criteria
* User can select CalPads in the file format if doing Direct Certification
* If user selects CalPads, on next screen, file preview uses our predefined column header names
* On file mapping screen, the page shows the predefined names in the source column
* On file mapping screen, the page auto maps the source fields to the database fields
* On file mapping screen, the column xxx has auto code mapping
* When importing, the import routine is able to handle not having column headers
* When importing, the import routine transforms the date field as needed
