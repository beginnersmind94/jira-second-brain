# NXT-47961 — FO - Document Center - Documents Center page and search

## Description
As Dora the Director I want to have a single page to collect my bulk prints from.

*requirements*

* Add a Documents Center page under System > Management
* Basic page - date range search, apply button.
** On page load, load the last 7 days for the user. 
* User can see a grid showing the print jobs from the date range they asked for.
** User, Page, Letter Type, Templates, Status, Print Date, Number of Letters
*** User: Lastname, Firstname
*** -Page: The page/area the print job originated from. Reminders, Applications, etc.-
*** Letter Type: Reminders, Direct Certification, Applications, Notifications
*** -Templates: The template names used as part of the print job-
*** Status: Queued, In Progress, Completed, Deleted, Failed
*** Print Date: MM/DD/YYYY HH:MM:SS AM/PM
*** Letters Generated: 5, 10, 11,300
**** Also show how many emails and prints.
**** Example: (Print 99, Email 1) - 80/100
** Grid columns should be filterable and searchable.
** Default sort: Print Date descending
* User has a toggle to Show Deleted batches
** If enabled, Deleted (status = Deleted) show in the results
** Toggle is off by default

*Discussion 5/21 with Harsha and Hussain*

* Removing the Page column
* Moving the Templates into the View action on another story 
* Letter Type to be the tab that the print was done on
* Letters Generated to show how many were generated to email and print.
** Download button will only provide Prints.
** Sorting is not needed for the Letters Generated column

## Acceptance Criteria
* Page added. 
* Page has a date search.
* Page has a results grid that shows print jobs done in the date range specified.
* Grid shows:
** User, Letter Type, Status, Print Date, Letters Generated
* Toggle for showing deleted print jobs
