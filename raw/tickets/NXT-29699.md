# NXT-29699 — Research - POS - Ways to avoid directing user to clear cache and avoid losing transactions

## Description
It isn’t viable for us to have users clear their local browser cache - too advanced for most users, and it will lead to support calls.

We need to research how to handle these scenarios without a manual clear cache happening.

* Data in local cache is incorrect and needs to be refreshed.
* All data is compared and pulled off the device / sync’d up before cache is cleared to avoid lost transactions. 

Is there a way to force a sync when the user logs in, and avoid these scenarios?

## Acceptance Criteria
Research how we can achieve ‘perfect data’ for the local POS cache, without requiring the user to delete their local cache/storage.

Research how to make sure we don’t lose any transactions during this process.

Research how to improve data sync as part of this.
