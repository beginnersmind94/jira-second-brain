# NXT-27401 — Edit Permissions page to support Special Permissions Assignment - API

## Description
The “Roles” needs to be updated to support the implementation of “Special Permissions” (also known as Functions).

## Acceptance Criteria
* A new section shall be added to the “Roles” page that will allow an end-user to see which “Special Permissions” have been assigned  to the selected role for the given module.
* The end-user shall have the ability to remove a special permission assignment from a role
** When the user removes an assignment, he/she will be asked to confirm the action
* The end-user shall have the ability to add a special permission assignment to a role
** When adding an assignment, the user shall have the ability to easily choose from a list of predefined special permissions.
* The “Permissions” column header on the existing “CRUD” grid shall be renamed to “Permissions”
* The existing “Select All” functionality shall be a fixed header at the top of the “CRUD” grid so that it does not sort with the data.
* The existing “Select All” functionality shall be corrected so that it works.
* If a permission/page checkbox is disabled per the setting in Resource Catalog, the checkbox shall be invisible.
* Each page shall have tags to detect available “Special permissions” applicable for that page at API level
* The list of resources (pages) within the CRUD grid shall be sorted ascending by default on page load
* The list of available special permissions shall be populated at deployment time
* The list of available permissions shall be defined by the Product Owners for each module and can be found here:  [https://primeroedgenext.atlassian.net/wiki/spaces/NEXT/pages/2328887519/Module+Special+Permissions|https://primeroedgenext.atlassian.net/wiki/spaces/NEXT/pages/2328887519/Module+Special+Permissions|smart-link]
