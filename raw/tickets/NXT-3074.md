# NXT-3074 — System - Settings - Prompt user for comment when changing setting values

## Description
To provide better auditing, prompt a user who attempts to change a setting to provide a comment as to why they are changing the setting. This could be in the form of a pop-up when the user attempts to update an internal setting value:

* Display a prompt/pop-up with a heading (that includes the selected setting name) and a field, along with OK and Cancel buttons.
* If the user adds a comment and clicks OK, allow them to save the setting value. 
** Comment can be limited to a fixed number of chars and must have at least 1 char.
* Otherwise (or if user clicks Cancel), do not allow the setting value change.
* Comment should be added to setting history as well.
* +*Common*+ setting changes should not require comments. (they should be optional)
* +*Internal*+ setting changes are *required*.

## Acceptance Criteria
Verify that page prompts users to add a comment when changing settings 

Verify that *common* settings do not require comments

Verify that *internal* settings do require comments

Verify comments must be within a character limit.

Verify that if the user adds a comment for an internal setting, we allow them to save the setting value. Otherwise, we do not allow the setting value to go through.

Verify that common setting changes can be saved in any case

Verify that the user's comments are added to setting history as well.
