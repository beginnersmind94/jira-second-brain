# NXT-11583 — Accounts - Account Details - Payment

## Description
*User Story*

As an administrator, I need to be able to apply payments to a patron account so that their account balance is increased for the purpose of purchasing food in the cafeteria.

*Requirements*

* In the Account Info page of Accounts, where there is a ‘More Actions’ drop-down menu, when a user selects the menu, show a button for *Payment*.
* When that button is selected, open the action slide out panel where the user can make payments to the selected account.
* Details: 
** Display group buttons corresponding to the type of payment being made. These will be the ‘Transaction Type’ of the transaction:
*** Cash
*** Check
**** When this is selected, show a ‘Check Number’ field to receive the check number
**** This button should only be enabled if check payments are allowed for *all* of the patrons added to the table have check payments allowed (this is a setting that exists in every patron account). Otherwise, disable it.
*** Card
**** When this is selected, show a ‘Last 4 digits of Card’ field to receive the card number. Field should only accept 4 numerical values.
*** By default, no option should be selected, but we should require the user to make a selection
** Payment Amount
*** Allow a user to enter an amount that is between 0 and the maximum payment amount (setting code MAXPYMTAMT).
*** Tooltip - Info icon?
**** Get the maximum payments allowed setting value (setting code MAXPYMTAMT) for the district and display a message:  _“The payment amount must be between $0 and  $\{\{settingValue}}”_
** Check Number
*** Only displays if Check payment type is selected
** Card Number (Last 4 Digits)
*** Only displays if Card payment type is selected
*** Can we use input type as credit card?
** Previous Balance
*** The current account balance
** Payment Amount
*** Amount entered within the input box
** Bonus Amount
*** Amount applied based on the bonus setting
*** Need more information (may remove for initial payments)
** New Balance
*** When a number is entered into the ‘Payment Amount’ field, display what this patron’s balance will be after the user completes the transaction.
*** Previous Balance + Payment Amount
* Once the requirements above are met, the user should be allowed to select the *Save* button to complete the transaction.
* ‘Close’ and/or ‘Cancel’ 
** Returns them to the Account Info page without saving
** Prompt the user if they want to cancel without saving changes.
* Upon save, the action panel closes and the balance on the account details should be updated and visible to the user in a lazy loading fashion (should note require refresh of the page)
* Error handling:
** IF _Payment Amount_ <= 0 OR _Amount_ > _MAXPYMTAMT setting value_
*** Display “Please enter an amount between $0.00 and $\{\{settingValue}}.”
* When ‘Save’ is selected, a *transaction* should then be entered with a transaction type related to one of the 3 buttons selected above.
* A *session* is also created if an open session for that site and user is not open, and a single transaction is entered into that session.
* If an *OPEN session* exists for that site and user, the transaction should be added to the open session.
* This story is complete when a user can navigate to the Payments menu from the Student Info page and perform a payment for the account, with setting display, balance display changes, and error handling where expected, and a transaction and session being created.

## Acceptance Criteria
After selecting an account the user can apply a cash payment to increase to balance of the account.
After selecting an account the user can apply a check payment to increase to balance of the account.
After selecting an account the user can apply a card payment to increase to balance of the account.

Payment amount must be greater than $0.

Appropriate error handling as described in the story description should exist.

User can back out of a transaction by using Close/Cancel buttons.

A transaction and session are created with these details:
-The transaction date is the current date
-The transaction type is CASH, CHECK, or CARD
-The account has a transaction entered
-The transaction is added to a session
