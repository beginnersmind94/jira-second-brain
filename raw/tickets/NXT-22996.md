# NXT-22996 — Base URL refactoring for UI framework

## Description
This story is to cover the tasks related to Base URL refactoring for UI framework.

Format for all API end points is "<Base_Url>/<Service_Name>/<Version>/<Path>". Define the base url once in the environment files. Declare services names as constants and version as constants. 

When calling API in function, call single base_url variable and change the service name as required.DO NOT declare API base url multiple times in the .env files.Use constant file to declare service names, api version. In the transport layer you can declare end point path are string and use it. 

Also in transport file, make sure everyone checks for response status code. Include timeout if needed.

In Case If all Api calls have same function declaration, write one function that takes url, api method and body as parameter and returns response. Then call the function directly in saga file.  Finally don't write same function with different names.

## Acceptance Criteria
Base URL refactoring for UI framework.

# Verify and validate that there is use of constant file to declare service names, api version. 
# Verify and validate that @ transport layer there is provision to declare end point path are string and use it. 
# Verify and validate@ transport file, that everyone checks for response status code and Include timeout if needed.
# Verify and Validate that all Api calls have same function declaration.
# Verify and Validate that every one writes one function that takes url, api method and body as parameter and returns response and Then call the function directly in saga file. 
# Verify and Validate that there are NO same functions with different names. 




This will help in reducing files and file sizes.
