# NXT-5511 — System - Sites - Move Sites to new section within Site Configuration page

## Description
Discussed in [NXT-3516|https://primeroedgenext.atlassian.net/browse/NXT-3516], let’s move the existing Sites page so that it is no longer a page on its own, but is a new menu option within the Site Configuration page.

* Delete Sites operation from the navigation
* Move Sites page content to within Site Configuration page
* When the user lands on Site Configuration, the Sites page should be the default section displayed
* Sites functionality itself should be unaffected and unchanged
* The ‘Add New' button on the page should be modified to allow the user to perform all 3 ‘adds’: adding a site, site type, or grade, and displaying the appropriate drawer based on the user’s selection.
** Functionality of each ‘add’ should continue to work as expected.
* Add a search button to search through the content of each tab as well.
* Story is complete when old page is removed, new page is added with Sites content moved over, ‘Add New’ button is modified, and Search functionality added.

Mock-up: [https://nqyc9v.axshare.com/start.html#p=site_configuration&id=mv2s8z&g=1|https://nqyc9v.axshare.com/start.html#p=site_configuration&id=mv2s8z&g=1]

## Acceptance Criteria
