# NXT-26285 — INV/SYS - Manage Periods - Flag site period as 'Needs Regeneration'

## Description
As a Site Manager I want the system to prompt me to regenerate our valuation if a change occurred in a period which was already generated, so we can regenerate the valuation and ensure it is accurate.

**Note*: 

* check regarding the logic in PrimeroEdge
** Needs Regeneration status is being triggered on any transaction change
* Dependent on [https://primeroedgenext.atlassian.net/browse/NXT-26253|https://primeroedgenext.atlassian.net/browse/NXT-26253|smart-link] 
* Related to [https://primeroedgenext.atlassian.net/browse/NXT-26269|https://primeroedgenext.atlassian.net/browse/NXT-26269|smart-link] 

*Requirements*

* Flag a Site’s period valuation with ‘Needs Regeneration’ status if anything occurs in a period whose valuation has already been Generated
** this is considered a “dirty” period now, because we don’t know if the valuation is correct for the old period after it’s been changed retroactively
** Note: we thought this was a simple check for a reopened period, but it is actually detecting any change in transactions (check PE logic w/ Michael)
* Note: this will be reflected on Period Valuation page ([26269|https://primeroedgenext.atlassian.net/browse/NXT-26269]) and Valuation Report ([26248|https://primeroedgenext.atlassian.net/browse/NXT-26248])
