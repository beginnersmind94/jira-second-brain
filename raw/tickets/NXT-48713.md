# NXT-48713 — OpenID - District Authentication - OpenID Edit

## Description
As a Customer Support user, I want to be able to edit a district’s OpenID authentication provider so that they can use SSO

*Notes*:

* This is the screen that allows a user to set configuration for Open ID provider

*Flow:*

* User browses to System > Security page
* User clicks Edit button on an OpenID provider
* User redirected to this drawer and completes the fields
* User tests the configuration and then saves the changes

*Requirements*:

* These requirements apply to the System > Security > OpenID Config drawer
* When a user clicks the Edit button for a row with OpenID Config, open this drawer
** Title: Edit OpenID Configuration
** Width: half size
** Add the sliding animation to the drawer open
* Fields:
* Authentication Name
** Single line textbox
** Required
** Matches to the AuthenticationName in the grid on the previous page
** Max length: 100
* Description
** Single line textbox
** Required
** Matches to the Description column in the grid on the previous page
** Max length: 50?
* OpenId Configuration Endpoint
** Textbox
** Single line, but needs to allow for a long URL
** Required
* Client Id
** Textbox
** Single line
** Required
* Client Secret
** Textbox
** Single line
** Required
** Masked like a password field
** Do the show / hide icon like a password and default to masked
* 
* {color:#ff5630}*_The rest of the fields will be determined by Dev Manager_*{color}
* 
* Cancel
* Save
** On click:
** Verify all required fields are entered
** Updated the registration with Keycloak
** If there are errors, show them to the user
** If successful, show the user the Success confirmation message and close the drawer
** The grid on the parent page is updated to show this authentication type
** Don’t close the drawer after saving
* {color:#ff5630}-Test-{color}
** {color:#ff5630}-Always enabled in Edit since they successfully saved the configuration to Keycloak already and they might just want to verify it works-{color}
** {color:#ff5630}-Button-{color}
** {color:#ff5630}-On click, do the same authentication test as for the Add New drawer-{color}
* 
* Ensure changes are saved to History log
* 
* Security:
** Security screen Edit permission required
** Customer Support role can always use
** Grant Security screen Edit permission to SchoolCafe Admin role by default
** Deny permission to all other district roles by default

## Acceptance Criteria
* Users are able to update the registration with Keycloak
* User is informed of success or failure
