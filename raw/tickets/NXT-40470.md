# NXT-40470 — MP- Site Configuration- Site Config page should not allow prep site to also be receiving site

## Description
As a Prep site, If I am making and preparing food for other sites, it does not make sense that a different site would prepare food for me.

## Acceptance Criteria
* Do not allow a prep site to also be a receiving site 
* !image-20230608-211642.png|width=1605,height=427!
When site is a prep site, user can not edit their prep site configuration 
** For example above, Since Aly Highschool is the prep site for _SiteforDemo, edit button on Aly Highschool should be disabled until Aly Highschool is no longer a prep site for anybody except themselves. 
* Provide a tooltip on the disabled Edit “Schoolname is a prep site for other sites”
* IF user is able to edit the prep site, provide a section for Current Effective Date with the Current effective date based on the current prep site configuration that is disabled and only for informational purposes
* Display another row that provides the New Effective Date selection that the user can change. Only allow this to be enabled and use if they switch prep sites
** Once user clicks and selects a new prep site, remove the Current Effective Date and allow the New effective Date selection
** New effective date should only be current date or future
* !image-20230608-221651.png|width=1291,height=729!
Date selector is not allowing user to select days in the past (which is correct), however user can manually type in the date in the past and it will save (this is incorrect), make changes to not allow past dates to be selected for new effective date
* Move save button to allow user to not have to scroll down based on zoom levels, there is no need for the save button to be that low
