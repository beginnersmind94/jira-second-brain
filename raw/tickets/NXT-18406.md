# NXT-18406 — Eligibility - Reports - Income Surveys-Income Range Count

## Description
As a district admin, I need to be able to view the list of accounts at my school who have submitted income surveys, based on their household size and/or income, so that I can provide the necessary data required by my state.

*Summary*

Add the “Income Range Count” Income Survey report to the Eligibility - Reports - Income Survey section. The report should display differently based on the *Income Range Type* setting value.

We can display this an HTML report or grid, as long as the results can be cleanly exported so that it functions like a report.



*Requirements*

* In Eligibility - Reports, add a section for *Income Surveys*
* In Eligibility - Reports - Income Surveys, do the following:
** Enable the “Income Range Count” Report Type option.
*** Do not show a ‘Date as of’ option
*** Site(s) should be selectable. A survey is counted as relating to a Site as long as at least one non-Pending student on the survey belongs to that site.
* When the user selects ‘Generate Report’ within the Income Range Count context, display the Income Range Count report. 
** Column data should change based on the site selection, as well as on the income survey eligibility guidelines of the district:
*** District Income Ranges
**** In this case, display the Household Sizes and Income Ranges of each row in the Eligibility → Settings → Income Eligibility Guidelines, as long as there is at least one survey with Household Income data matching that size/income range combination. Example:
**** !image-20210907-211338 (fb76c361-ef91-4def-9a2e-207991ffb6d3).png|width=658,height=364!
*** USDA Income Ranges
**** In this case, display how many surveys were submitted with each Household Size corresponding to each row in the Eligibility → Settings → Survey Ranges, as long as there is at least one survey with Household Income data matching that row. Example: 
!image-20211025-150732.png|width=527,height=370!

If the district is using the Exact Amount setting, disable this report type option and display a message: “This report type is only valid for districts using the USDA or District Income Range settings.”



* At the footer, continue displaying this text: _“_*_Note:_* _When there are no counts for the household size or income range the row or column will not be displayed.”_
* Also, the report should display the report option selections in the header or footer of the report
* User should also be able to download/export the report as well

## Acceptance Criteria
* Verify that the new Eligibility - Reports - Income Surveys tab has the _Income Range Count_ report option enabled
* Verify that when the option is selected, the user can only select the Site(s). 'Date as of' should be disabled
** Verify the default selection for ‘Site(s)’ is ‘ALL’
* Verify that when the user generates the report, the 'Income Range Count' report is displaying accurately based on their selections
* Verify that the report is displaying the report option selections in the header and/or footer of the report
* Verify that the column data changes based on the income survey eligibility guidelines of the district
