# NXT-53218 — INV - Physical Inventory Details - Counts tab & Sheet - Load items with null instead of zero

## Description
As a Site Manager I want to be able to differentiate between items we intentionally zeroed out and items we skipped, so I don’t have items falsely reported as “not counted.”



{quote}I think we need to change Physical Inventory items to load with *null / no value* instead of loading with zero.

I believe the logic behind making items load with zero is because of the looming reconciliation. In PE, the logic is like "either you count the item or it gets zeroed out during reconciliation." But since SC doesn't work that way and items can be skipped over during reconciliation, we can actually make a distinction between "not counted" and zero. Currently we're treating all zeroes as "not counted" which is just not actually true, since some items may have been "counted" as zero. If items are left null, then they are genuinely "not counted" and that will be reflected on the new Review screen, rather than being submitted as a zero quantity item.

We would perhaps require one new validation on the Counts tab alerting users to the items missing counts before they complete the inventory.{quote}

## Acceptance Criteria
* Change the Counts tab to load items with no input value for quantity 
** Update 3 APIs
* Consider items with no input as “not counted”
** Ensure the Summary tab values are properly differentiating 
* Consider items with zero value as counted
* Update Count Sheet to properly handle null values for Physical Quantity
** leave blank when null
** put zero quantity when zero value exists
