# NXT-60354 — IM- Item Onboarding- Pack Size Info- Inventory Options- Serving-Pack Link (Sub- Quantities) (Step 1) 

## Description
As a district admin, I would like a way for me to bulk update serving/pack size relationships. These could be either deciding to use serving as a base unit, provide how many servings are in the base unit, or provide weight.

!image-20250429-070236.png|width=1390,height=747,alt="image-20250429-070236.png"!

## Acceptance Criteria
* This step will be accessed through the Inventory Options (Pack- Serving Link) on the Review Pack Size page or through the (Units & Sub-Qty) step
* Filter and only display items on this page that are eligible to either be used as a base unit or provide Servings per BU ( In Mock up shows all items to show the difference)
** Do not show any items that have not been configured to have pack sizes (In mock up only showing to show the full flow)
* Provide Size Overview (This will be displayed on every step to emphasize changes) and should always dynamically update based on changes on the page
* Provide a toggle to “Use Serving as Base Unit” 
** IF toggle is Yes, Use serving description as the new base unit description and inherit the Sub Qty
*** Display “Dual Unit” chip on the pack size overview for user to see what the selection is doing, the dual unit is not confirmed until user saves
*** Gray out the ability to add Servings Per BU
** IF Toggle is No, user must either enter “Servings per BU” or provide Weight (Next Step)
*** IF “Servings per BU” is entered, display a servings chip on the pack size overview along with the sub qty relationship
* On this step, only Serving as Base Unit and servings per BU are captured, weight (mandatory or optional) is on step 2
* User must Save Linked Items before they can move on to review weight items
** Provide a Unsaved changes prompt if user has made changes or entered new information on the sub-quantities step
* User can also skip this whole sub-quantities step and go straight to the weights option ([Step 2|https://primeroedgenext.atlassian.net/browse/NXT-60355])
