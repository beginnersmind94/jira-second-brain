# NXT-69972 — FIN - Posting Activity Log Notifications

## Description
As a finance user, I want to see notifications when I log in about new, failed, or review-needed posting jobs since my last login, so that I can quickly address issues and stay informed.

## Acceptance Criteria
h3. Notification Trigger

*Given* the user has access to Financial Oversight, *When* the user logs in (or navigates to Financial Oversight for the first time in a session), *Then:*

* System checks for posting jobs since the user's last login
* If new activity exists, a notification banner displays

h3. Notification Content

*Given* there are new completed jobs since last login, *Then:* notification displays: "X posting jobs completed since your last login. View Activity Log →"

*Given* there are failed jobs since last login, *Then:* notification displays with warning style: "X posting jobs failed. View Activity Log →"

*Given* there is a mix of completed, failed, and requires-review jobs, *Then:* notification prioritizes warnings: "X jobs need attention, Y completed. View Activity Log →"

h3. Notification Behavior

*Given* the user clicks "View Activity Log →", *Then:* user navigates to Posting Activity Log, filtered to jobs since last login.

*Given* the user dismisses the notification, *Then:* notification does not reappear until next login with new activity.

*Given* no new posting jobs since last login, *Then:* no notification displays.

h3. Activity Log Error Messages

*Given* a posting job fails due to missing GL account mapping, *Then:* expanded row shows: "Missing GL account mapping for [Account Name]"

*Given* a posting job fails due to closed financial period, *Then:* expanded row shows: "Financial period [Month Year] is closed. Reopen the period to post."

*Given* a posting job fails due to missing reimbursement rates, *Then:* expanded row shows: "Reimbursement rates not configured for [Program Name] - [Federal/State]"

*Given* a posting job has status "Requires Review" due to existing system-generated entry, *Then:* expanded row shows: "A system-generated posting already exists for [Period]. Review and manually reverse if needed before reposting."

*Given* a posting job has status "Requires Review" due to existing claim posting, *Then:* expanded row shows: "A posting already exists for this claim. Review and manually reverse if needed before reposting."

h3. Job Types Included

Notification includes all posting job types:

* POS Posting
* Inventory Posting
* Reimbursement Posting

h3. Permissions

Access as per roles and permissions.

h2. Success Criteria

* Finance users are immediately aware of posting activity without manually checking Activity Log
* Failed and review-needed jobs surface prominently
* Clear, actionable error messages guide resolution
* Notification is non-intrusive and dismissible
