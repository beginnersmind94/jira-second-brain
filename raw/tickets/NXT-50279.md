# NXT-50279 — Reports - License Management

## Description
As a License Management (LMA) role or Customer Support role member, I would like to view reports on licenses we have created for any district in one page so that I can manage licenses across all our customers in one place.

*Notes:*

* The scope of this page is for ALL districts at once, not a single district
** It is like a global dashboard, used for managing all licenses
** Do not restrict visibility to just data for a single district

*Flow:*

* LMA role user browses to Reports > Global Dashboard > License Management Reports Dashboard
* User sees a KPIs, graphs and data points for Districts

*Requirements:*

* Add a new page to the resource catalog
** Title: License Management Reports Dashboard
** Put it under Global Dashboard
** Disable it from being managed in the Role and Permissions front end
* 
* On Reports > Global Dashboard:
* Create a card
** Title: License Management
** Description: View key data points related to licensing for all districts
** On click: go to the License Management Reports Dashboard page
* 
* License Management Reports Dashboard page
* These requirements apply to Reports > Global Dashboard > License Management Reports Dashboard:
* Create the License Management Reports Dashboard page if it does not already exist:
** Title: License Management Reports Dashboard
* 
* Add a back button back to the Global Dashboard
* 
* Filters
* Add filters section on top of page with the usual collapse button
* Fields
** District
*** Multi-select list
*** Default: If need to default, then select all; if can, no selection = All
** Status
*** Multi-select list
*** Default: Active
** Module
*** Multi-select list
*** Values: The module the license is for
*** Default: If need to default, then select all; if can, no selection = All
** Licenses
*** Multi-select list
*** Values: The licenses we offer, same list as Add new
*** Default: If need to default, then select all; if can, no selection = All
** Apply button
** Clear button
* 
* -KPIs-
* -Create a collapsible section-
** -Title: Active Districts-
* -Default show this section-
* -Add KPI graphs and cards as described here-
* -The Filters section updates the KPIs in the same way it updates the rows in the grid-
** -Keep the data in sync for the KPIs as with the data-
* 
* Donut Chart
** Name: Licenses
** Data:
*** Outer Ring: Count of license keys by license name
*** Inner Ring: Count of license keys by Module
*** For both, ensure we can see the label, count (and % if you can)
** Drilldown:
*** If click on the inner ring of keys by module:
**** Show a pie chart of count of license keys by license type for just that module
**** Opens in drawer
*** If click on the outer ring of keys by license name
**** Show a grid with District name, license type, license count and license used as the columns for each license key in that pie section
**** Opens in drawer

* 
* Bar Chart
** Name: Expirations
** Data: Count of license keys by end date by rolling quarters
*** Excludes IsDemo districts
*** We use End date as expiration date
*** Use Start date as starting date
** Vertical axis: Count
** Horizontal Axis:
*** In Level 1, show rolling quarters like this:
*** First is whatever quarter we are in:
**** Ex: Q324

*** Keep going until all the quarters are accounted for all active license keys
** Bars in each time frame:
*** One bar = count of expiring license keys
*** One bar = count of starting license keys
** Include labels and counts for each bar
** Drilldown:
*** In level 2, show grid of License Keys in that time frame
**** Columns: District, License Key, License Name
*** Ensure user can go back up to previous level

* Security
** Global Reports role and License Management role can do anything
** Everyone else blocked
