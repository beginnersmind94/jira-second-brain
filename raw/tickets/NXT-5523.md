# NXT-5523 — Workspace - My Tasks (Landing Page  Summary View)

## Description
*User Story*
As a PrimeroEdge User, I can see user created tasks under the My Tasks heading of the Workspace landing page.

*Mockup*
[https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=5636%3A1161&scaling=scale-down-width|https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=5636%3A1161&scaling=scale-down-width]

*Requirements*

* Option to select time period to display tasks based on due date
** Today, Next 7 Days, Next 30 Days
* -Gauge to show total number of tasks in the selected time period and how many have been marked as completed.- {color:#ff5630}Deferred{color}
* Resize vertically to include all tasks in the selected time period.
** Maximum of 5 tasks are shown, prioritized by due soonest
** Completed tasks are only shown here if there are fewer than 10 uncompleted tasks
** -Pagination may be needed if more than 10 tasks exist for the period.- {color:#ff5630}Deferred{color}
* Display Task Name and whether a task has been completed via status chips.
** Clicking Task Name will take user to a popup with a detailed description of the task.
** If clicked, status can be changed and a task is no longer marked as complete
** -Clicking on the check mark icon of an uncompleted Task will change the status of the Task to completed and change the icon.- {color:#ff5630}Deferred{color}
** In Next 7 and Next 30 Days, a due date is shown for each task
* New Task button to create a user Task in a pop-up window
** This is covered by NXT-6347
* View All button to navigate to a separate Task page that shows all Tasks assigned to the logged in user.
** Covered by another story - NXT-6337

## Acceptance Criteria
* Tasks section present on workspace
* Tasks can be updated or marked complete
* Task status is properly updated and tracked
