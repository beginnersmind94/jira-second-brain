# NXT-52754 — POS - Pause the Queue, allow user to lookup/correct account

## Description
!image-20240717-201347.png|width=1281,height=272,alt="image-20240717-201347.png"!

*Scenario*

* Student has entered a wrong ID number.
* When they hit the top of the Queue, it can’t find them.
* Other students have entered their IDs in the meantime - queue is up to #5.
* Student is told to re-enter their ID number - they are now #6 in the queue. 
* Student has to wait around while 5 other accounts are served.

*Requirements*

* User needs an option to pause the queue if it is going.
** Add a Pause button near the Queue button.
** If paused, the next ID entered should be loaded immediately.
*** Either by keyboard/pad entry or lookup.
* User needs an option to start the queue if it is paused.
** This will resume the queue and pull the next ID off the list.
* User has an option at POS to automatically pause the queue when an account cannot be found.
** Setting Name: Auto-Pause Queue when Account Not Found
** If enabled, the queue should pause when an account cannot be found
* User is more clearly informed when an account cannot be found.
** User is shown the ID that was entered.
** User is shown the next ID in the queue (in case the student already corrected it)
** User can dismiss this pop-up.
** User can pause the queue from the pop-up.
*** If auto-pause is enabled, no need for this action.
** The pop-up disappears after 6 seconds.

## Acceptance Criteria
* User can pause the queue
** User can then bring up an account to serve it immediately
* User can un-pause the queue
* User has an option to auto-pause the queue if an account cannot be found
* User is better informed when an account cannot be found:
** User can see the ID number that was entered
** User can see the next ID in the queue if it exists
** User can pause the queue from this pop-up
** User can dismiss the pop-up
** Pop-up dismisses automatically after 6 seconds
