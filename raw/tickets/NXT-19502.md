# NXT-19502 — INV - Quick Receipts - Back button should return to Order step when opening an Incomplete quick receipt

## Description
Inventory > Quick Receipts > When save as Incomplete - Back button functionality is not working as expected.

1. Login to Inventory
2. Navigate to Orders > Manage Orders
3. In view Orders list click on the Action button  for Quick receipt in Incomplete status
4. Click on the back button from 'Edit Quick Receipt' screen
5. Verify the Order step

Issue 1: Header sections must be disabled.
Issue 2: Item quantities will be populated from previously entered quantities

Lines in story:

* If the Quick Receipt has been saved as Incomplete:
* Back button will navigate back to Order step
* Header selections will be disabled
** Site
** Order Date
** Vendor
* Item quantities will be populated from previously entered quantities
* Next button will proceed to the Receipt step once again
