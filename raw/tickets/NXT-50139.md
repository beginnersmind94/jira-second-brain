# NXT-50139 — LMA - License Keys - Renew

## Description
As a License Management (LMA) role member, I would like to renew a license key so that I can establish the link between license keys for districts.

*Notes:*

* The use case for the story is a district wants to continue to use our software with the same licenses and we need to grant it

*Flow:*

* District’s license key expires soon and they want to continue
* They call and we we sell them an extension for the licenses
* LMA role user browses to System > License Management
* User sees a list of licenses generated for the software in the grid
* User clicks Renew License Key button on a row and Renew License Key drawer opens
* User completes fields and saves
* System generates a new license key with settings on the page
* License Keys grid updates with the new information
* When district users login after original expiration, they still have access

*Requirements:*

* On License tab grid, do the following:
* Grid Actions column:
** Renew License
*** On click:
**** Open the Renew License drawer
**** -Enabled for New, Active and Expired status (not Cancelled)-
**** Updated[10/17/2025 -Ravi]: Enabled for Active and Expired status (not Cancelled, New)
*** Only License Management role can click this button



*Renew License drawer*

* If the drawer does not already exist, create it
** Title: Renew License
** Width 1/2
** Use sliding motion



*Header*

* Show these fields are read only:
** District
** License Key
** Status
** Licenses
*** Comma delimited list of licenses on the key
** Usage message (see bolt code)
* Show these fields as editable and required:
** Effective Date
*** Default to the End date of the existing license
** End Date
*** Default to one year from Effective Date
** Notes
*** Same as Generate New screen
* Renew License button
** Only LMA roles can click this button
** On click:
*** Validate required fields
*** Show this message: This will generate a new license key for the selected date and existing licenses. Are you sure?
*** If confirmed:
**** Start Date = start date field
**** End Date = end date field
**** Purchase Date = Original Purchase Date field
**** Generated Date = Now()
**** Generated By = The name of the user account that generated the license
**** Status = Active
**** License #
***** Generate a new one 
*** Show the success message, close the drawer and return the user to the License Keys page with the grid updated
*** Log in the History table as an Update
*  
* Security
** License Management role only has rights to take actions on this page



*_Note to Devs_*

* _When renewing, it is also required to update the links for Sites and Devices_
** _For example, all sites linked to the old license need to be delinked from it and linked to the new one_
** _Same for Devices_
* _Problem is that it is most likely that functionality coding comes after this functionality coding_
* _Therefore the requirements ignore it, but you should be aware it will have to be done at some point_
* _If you develop that functionality first, then including the linking of sites and devices when you renew_

## Acceptance Criteria
h3. 1. Access and Navigation

* From the License tab grid, clicking Renew License opens the Apply License drawer.
* Renew License button is enabled only for New, Active, and Expired status licenses.
* Disabled for Cancelled licenses.
* Only License Management (LMA) role can click the button.

h3. 2. Drawer Layout

* Title: Apply License
* Width: half page
* Opens with a sliding motion.

h3. 3. Header Section

* Read-only fields:
** District
** License Key
** Status
** Licenses (comma-delimited list of all licenses on the key)
** Usage message (from Bolt code)
* Editable and required fields:
** Effective Date
*** Default to the End Date of the existing license.
** End Date
*** Default to one year from Effective Date.
** Notes
*** Same behavior as Generate New License screen.

h3. 4. Renew License Button

* Only LMA role can click.
* On click:
** Validate all required fields are filled.
** Show confirmation message:
_"This will generate a new license key for the selected date and existing licenses. Are you sure?"_
** If confirmed:
*** Set Start Date = Effective Date field
*** Set End Date = End Date field
*** Set Purchase Date = Original Purchase Date
*** Set Generated Date = Now()
*** Set Generated By = current user account
*** Set Status = Active
*** Generate a new License #
*** Show success message, close the drawer, and return the user to the License Keys page with the updated grid.
*** Log the action in the History table as an Update.

h3. 5. Security

* Only License Management (LMA) role can take actions on this page.
