# NXT-5008 — System - Roles - Temporary Role Access

## Description
*User Story*
As a PrimeroEdge admin, I have a way to add a temporary role to a person with an expiration date and time.
As a PrimeroEdge user, I can access all features related to a role while it's temporarily added and no longer have access once that role expires.

*Requirements*

* When adding a role to a user, an admin can set the role to be temporary
** This is present when selecting the role, indicated by the text "Make Temporary?"
** If clicked, a popup appears
*** Header: Assign Role as Temporary
*** User selects a time and date for the role to expire
**** Time must be at least 1 hour after the current time
**** Date can be the same date or later
* When viewing a user, a temporary role is marked with a clock icon to differentiate it
** Clicking the icon brings up the popup that shows the expiration time
** User can adjust the time and date
** User can remove the temporary setting, making it permanent
* When the time and date are reached, the role is automatically removed from the user's profile
** An entry is made in the history indicating that the role was removed because it expired

*Notes*

* This is covered by USDA requirement NR67

## Acceptance Criteria
* Temporary roles can be added and edited
* User can access permissions associated with the role while it is active
* The role is appropriately removed when it expires
