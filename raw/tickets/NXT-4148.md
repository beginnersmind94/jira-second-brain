# NXT-4148 — Security - Multi-Factor Authentication Settings

## Description
*User Story*
As a PrimeroEdge admin, I can manage which multi-factor authentication options are available, as well as enabling or disabling it.

*Requirements*

* There is a section in the Security Policy that allows admins to manage their organization's use of multi-factor authentication
* Admins can enable or disable multi-factor authentication
** If enabled, it applies the next time the user attempts to log in
*** There is a message at the top of the screen for this - "Any changes made will be applied on next login."
* Each available multi-factor authentication method can be enabled or disabled
** Options appear as checkboxes
** Any checked option is available to users as a choice
** Validation is present when saving Security Policy to ensure at least one option is checked when multi-factor authentication is enabled
** If a method people are using is unchecked, they are required to set a new one the next time they log in
* If one of the top level toggles (Multi-factor authentication or One Auth) is toggled off, the child toggles are saved, but are grayed out
** All the child toggles are disabled from a functionality standpoint, regardless of how they appear while grayed out
* If a top level toggle is enabled, but no child toggles are enabled, none of the child toggles are applied
