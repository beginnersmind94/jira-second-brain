# NXT-42807 — API – Download API Data - POC + Development

## Description
As a SchoolCafé admin, I would like to preview / download the data pulled via the API call so that I can debug issues and use it to help in configuring manual updates.

*Note:*

* The output of this download shows the API source data, not our data
* The use case is that the user wants to see what data is pulled by calling the API
* Since it could take a long time to load a preview grid, we will instead download it and notify the user when it is ready for their viewing

*Flow:*

* Browse to API
* Click on Download for one of the rows
* Shows popup confirmation
* User says yes and shows Processing message
* System generates an excel file of the data that comes back from the API calls
* System notifies the user when the file is ready by sending message to inbox and notification update
* User browses to inbox and clicks on file 

*Requirements*:

* On the System > Imports & Exports > API tab > Rows, do the following:
* 
* Add a button to preview the data
** Add it to each row
** Title = Download
* On click:
** Note: This basically is a Run Now without doing the actual import
** First, validate PII
** Check if the API’s Has PII flag is true
*** If true, then check the user’s permissions to see if they have PII permission (from roles and permissions table)
*** If they do have PII permission, then proceed
*** If they do not have PII permission, then do not allow them to view the data, but rather show them a message: You do not have permission to view Personally Identifiable Information (PII). Ask your administrator to grant you permission before continuing.
** Show the user a confirmation screen if they have PII permission
*** Text: This will call the source APIs, fetch the data and save in a file. You will be notified in your Inbox once complete. Do you whish to continue?
*** Show them Yes / No buttons
*** If they say No, do nothing
*** If they say Yes, then proceed
* 
* If confirm they want to preview, then do the following:
** Close out the confirmation box
** Show a message: Process started. You will be notified when the preview file is ready
** allow them to close it out or auto close like we normally do
* Then kick off the process to generate the file
** do all that you do for imports to get the data, but don’t import it 
** Instead, generate an excel file (or delimited that opens in Excel)
** file name is import name (scrubbed so only valid name characters are in it) and date if you can
** Send that file to the user’s workspace inbox (see Inbox section below)
* 
* Sending file to user’s inbox
** Don’t rewrite code to send a file to the inbox
** We have this already from System team
** See their code on sending files to Inbox that they did as part of Custom Reports
** Inbox values you will need:
*** From: System account
*** Subject: Preview [import name]
*** -Attachment: the file-
*** Body: 

Hi [user first name], 

Find attached the import preview that you requested.

## Acceptance Criteria
* User can click the button to start the preview process
* Use is informed of the process and can confirm
* PII view restrictions are respected for APIs that are marked with it
* File is sent to the user’s inbox
* User is notified when the file is ready to view using Notification Animation
* User can open the file from their inbox
