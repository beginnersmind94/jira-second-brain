# NXT-22590 — INV - Standard Ordering - Update for Pack Size Info

## Description
As a Site Manager I want to see all the orderable units within an item’s pack size in a treelist type format so I can see all the available units when ordering but still order them as individual items.

**Notes:* 

* do not consider Commodity Contracts in this story
** [NXT-32246|https://primeroedgenext.atlassian.net/browse/NXT-32246]  is created for Commodity Contract flow
* we will not consider Par Level Quantity in this story either 
** need another story for applying updated Par Level logic to Standard Ordering

*Requirements*

* Update the Standard Ordering screen to reflect the new pack size configuration
* This page must still *consider Contract Items* and *Ordering Conditions*
** For Orders, other pack size units will not be shown unless those specific units are also on the contract and the site is configured to order them
* Show item units on individual rows but indicate items which are linked together
** It does not need to be an actual TreeList grid, but if we can simply indicate when items are child units this will be sufficient
* Update hierarchical representation as items are added to cart
** remove units which are already added to cart
* -Replace Unit Type with Unit Description-
** Unit Description is displayed after the order quantity field; we just need to make sure it’s completely visible and obvious to the user
*** Put Unit Description in its own column
* Display Item # only once for the highest orderable unit
* Show conversion (sub) quantity (wishlist enhancement)
* Re-add ‘Date’ to ‘Delivery Date’ field label
* Remove header column search for Par Calculated Quantity

h4. +UI/UX Ideas+

* Implement Vendor as a grouping instead of column
* Decrease font to accommodate lengthy data
** Reference Production for how smaller font is used to manage item data
** This could done in many item screens
* Implement sticky footer for Add to Cart button
