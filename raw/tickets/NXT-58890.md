# NXT-58890 — FOH Dash - Claim Readiness - Claim Summary

## Description
As Dana the Director I want to generate and see details from the reimbursement claim.

*mockup*

!image-20250119-201159.png|width=1100,height=737,alt="image-20250119-201159.png"!

*requirements*

* User has a Claim Summary area
* User can trigger Claim Creation / Recreation
** Disable the generation button if a claim is already being generated
** Tool-tip for disabled button: Claim is currently generating. Please wait.
* User is informed of Claim Progress
** User does not need to stay on the page
* User can see high-level totals for the claim broken out by:
** meal type
** eligibility
* User can see a total reimbursable meal count (no second meal)
* User can see number of operating days per meal type
* User can see Reimbursement total per meal type
* User can see Reimbursement grand total
* User can see when the last claim was generated (full timestamp)



*Text:*

* Claim Generation Date
** Placement: Next to the Create Claim button
** Text if a claim exists: Claim Generated On: DD/MM/YYYY HH:MM:SS AM/PM
** Text if no claim exists: Claim has not been generated.

* Create/Recreate Claim button has been clicked:
** Placement: Toast msg should be fine for this, top-center area like the other ones
** Initial message: Claim generation is underway, we’ll let you know once it’s completed.
** Claim generation finished if the user is still on the page: Claim generation complete!
* We wanted to add an info icon + text to the Summary section to explain that the counts the user is seeing are based on rollups, not the actual claim:
** Placement: Header if possible, we can put the first sentence in and then put the whole text into a tool-tip if it’s too long.
** Text: 
{noformat}Summary counts shown below are from reports and not the actual reimbursement claim. Verify with the Claim Reports on the Claims page. 

If there is a gap between the counts shown here vs the Claim try the following steps. 

- Run Report Rollups off of the Reports page. 
- Recreate the Claim.{noformat}

## Acceptance Criteria
* User can see operating days, total meal count, claim summary (meal type x eligibility) and totals, grand total
* User can trigger claim creation & recreation
* User can see last generation time
* User has a Details button to take them to the Claims page if the program is SNP and SFSP
