# NXT-3332 — Inspection Template - Validation

## Description
*User Story*
As an Inspections user, any questions I enter into a template I'm creating or editing are validated to ensure I'm not producing duplicate questions or categories.

*Requirements*
* Categories and Questions cannot have duplicate entries
** Customers are informed if a duplicate entry exists
* Validation is not case sensitive
** For example, "Is the kitchen clean?" is the same as "is the kitchen clean?"
* Category validation occurs across all categories
** Validation occurs as user attempts to enter a category name
*** If the name matches another Category, the field is highlighted and the user is informed that two Categories cannot have the same name
** User is unable to save the template if the error is present
** Validation does not occur for Category description
* Question validation occurs within Categories
** For example, a question can be the exact same if it appears in Category 1 and Category 2, but not if both appear in Category 1
** Validation occurs across all questions not assigned to a Category
** Validation occurs as user attempts to enter a question
*** If the question matches another question, the field is highlighted and the user is informed that two questions in the same Category cannot be the same
** Validation does not occur for any other part of the Question (Help, Question Type, etc.)

## Acceptance Criteria
* Validation occurs for Categories when creating or editing
* Validation occurs for Questions when creating or editing
