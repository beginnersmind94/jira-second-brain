# NXT-61042 — GC > GDSN Connect Updates(SC)- Review Changes - Single Item at a time (Contribution / Allergen Tabs)

## Description
_(none)_

## Acceptance Criteria
This Story covers the following:

* Add two new tabs for Allergens and meal credits. 
* Allergens tab will display available updates for standard allergens
* Meal credits tab display the available updates for meal components
* Grid shows only allergens with updates for standard allergens, same for meal components. only meal components with updates will be listed (it wont list the allergen/meal components if it doesn't have any update).
* Select all available updates by default. Checkbox is selected by default. and allows user to deselect the checkbox if they dont want to accept that change. 
* The serving measure and nutrition and contribution box should be coupled - if the user deselects serving size related fields, then other section should be deselected as well and vice versa
* Hide the UPDATE NOW button we have on Images tab.



Navigation: Item Management - Global Catalog - Pending Updates

Default Data Provider - 3rd Party Source(s)

The UI should automatically load the items which has updates compared to local items

Upon clicking on item name (hyper link), the UI should show the comparable page that shows current and available update

Item Slide out panel:

By default - all the attributes are selected

User should be able to deselect the updates by clicking on Overwrite check box (derived from settings)

The serving measure and nutrition and contribution box should be coupled - if the user deselects serving size related fields, then other section should be deselected as well and vice versa

+_The wireframes are available in attached word document, PE screens are provided for reference purposes only_+



Images and Doc Tab: Regression (Asis)

The UI should load the one or more images in the sequence in which it came as update

User should be able to select the images and chose the desired option and click save

Clicking Update button shall override both attributes and images into a local item

Pending Updates page - One the update is completed, the UI removes that item from the list until it finds an update against a local item (if attribute count is >=1)
