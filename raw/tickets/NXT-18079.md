# NXT-18079 — INV - Orders - Manage Orders - Receipts - Quick Receipt - Receive Step - Items

## Description
As a site manager I want the ability to create a receipt without placing an order in the system so I can receive items that I purchased outside the system.

This is the second overall step of the ‘Quick Receipt’ process. This screen will function very similarly to the existing Create Receipt screen, with some minor differences.

*Requirements*

* Header and Charges/Discounts section handled on [https://primeroedgenext.atlassian.net/browse/NXT-18046|https://primeroedgenext.atlassian.net/browse/NXT-18046|smart-link] 
* Items section
** Since the user specifies the quantity on the previous step, this section is primarily for adding Date on Product info
** Remove the Order Quantity column
*** For Quick Receipts, order quantity is the same as received quantity
** Auto-fill the Receive Quantity fields with the quantities provided on the previous step
*** user can still edit quantity on this step
** Remove the Confirm column
*** there is no need for this since the user has already received the product by the time they are creating this
** Maintain the same logic for the Unit Price and Charge/Discount columns
*** these should all be editable
*** reference the permission the same way it is referenced by the usual receipt screen
** Date on Product sub-grid should be available as usual
* Provide a Back button to return to the Order step ([NXT-7257|https://primeroedgenext.atlassian.net/browse/NXT-7257])
* Provide Save buttons ([NXT-18048|https://primeroedgenext.atlassian.net/browse/NXT-18048])
** Save as Incomplete button
** Confirm button
