# NXT-50258 — Resource Catalog & R&P Verification

## Description
Background:

* We have upgraded to a new version of Keycloak
* As part of that, we are reviewing permissions for existing accounts and reviewed our resource catalog
* Resource Catalog is the list of Keycloak resources (pages, tabs, etc.) that we can apply security settings to (ex: Imports & Exports page)
* It appears that there are many orphaned resources (pages) from old Data Exchange work
* We need to review these resources and also what permissions we are actually using so that we can update the catalog and update our new roles and permissions page properly

*Requirements*:

* Go through the attached list of resources
* On the ones I have guessed at, verify
* For the blank ones, go through the code and see if we have any resources (pages, tabs, etc.) using those resource names
** If you find it in use, record what page, tab, etc. is using it
** If you don’t find it, then it is probably orphaned, so record that it is not found
** See table below
* 
* As a second part of this, we need to know if we are using permissions for security trimming and if so, which ones
* Our goal is to use the permissions for Imports & Exports for all sub pages, drawers, tabs, functions etc. 
** Ex: We can say that if you have Add permissions to Imports & Exports, all tabs can just use the same Add permission and don’t need to manage separately
** But the code may have been done differently; it might be coded to the APIs tab’s Add permission
** So the goal is to identify exactly which if any permissions are used everywhere under Imports & Exports
* For each tab, document where security trimming is being used and document what it is doing
** Ex: Add New button on File Imports - Is there a security check for permissions? If yes, what resource? what permission (Add?)?
*** Note: There is a resource name like API Configuration and then there are 4 actions for each (Add, Edit, Delete, View)
** Check each page, tab and action button
*** Drawers open from the action button, check them too but should match the action button if there is any security check
