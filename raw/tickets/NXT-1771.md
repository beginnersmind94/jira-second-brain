# NXT-1771 — INV - Orders - Create Orders - Apply Par Levels

## Description
As an ordering manager I want the ability to populate order quantities during the Create Order process by referencing my par levels so I can let the system tell me how much to order.

Par Level Quantity - (On Hand + In Transit) + Committed = Suggested Order Quantity

When calculating On Hand quantity, ignore Broken Units
When calculating Committed quantity 

||*Par Level Quantity*||*On Hand*||*In Transit (On Order)*||*Committed*||*Suggested Order Qty*||
|300 cases|247 cases (minus)|50 cases (minus)|17 cases (plus)|*20 cases*|

*Requirements*

* +on the Enter Item Quantities screen of the Create Order process:+
* Add an additional column called ‘Par Calculated Quantity’ 
** This would show the suggested order quantity for an item calculated from Par Level
** Show this column to the left of the Order Quantity column
** If none of the items have Par Level Quantities configured, hide the column by default
* Provide an icon/column/option to allow the user to populate Order Quantity with the calculated par level quantity
** When clicked, this would insert the calculated quantity into the Order Quantity field (whole units only)
* Provide a popup link for each Par Level Quantity, for the user to see the details of the calculation
** On this popup, it would simply show the following quantities for the selected items
*** Par Level Quantity
*** On Hand
*** In Transit
*** Committed
*** Suggested Order Quantity

## Acceptance Criteria
* Calculate suggested order quantity from par level 
* Display additional column on Create Order screen
* Provide a popup for viewing each item’s par calculated quantity
* Provide an option to populate Order Qty with the calculated quantity
