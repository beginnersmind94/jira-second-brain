# NXT-70193 — FIN - Bulk Actions Mode

## Description
_(none)_

## Acceptance Criteria
h2. 1. Entering Bulk Actions Mode

A "Bulk Actions" button is added to the page action area, near the existing "Add Data Point" button.

Clicking "Bulk Actions" enters selection mode:

* The "Bulk Actions" button changes to a highlighted/active state.
* A context banner appears at the top of the table area: "Select groups to apply bulk actions. [Exit Bulk Actions ✕]"
* A checkbox appears on every category group header across all sections.
* A "select all" checkbox appears on each section divider row (Assets, Liabilities, Equity, Revenue, Expenses). Checking a section divider selects every group in that section. Unchecking deselects them all. If only some groups in a section are checked, the section checkbox shows an indeterminate (—) state.
* Edit buttons, inline mapping, and Rules buttons are visually dimmed and cannot be clicked while in bulk actions mode.
* *These are override operations — they apply uniformly to all data points in the group, overwriting any existing mappings or statuses.*

----

h2. 2. Group Selection

* Clicking a group's checkbox selects it. The group doesn't need to be expanded.
* Users can select groups from different sections at the same time.
* If filters are active, checkboxes only appear on groups that are currently visible. Bulk actions only affect data points the user can see on screen.
* At least one group must be selected before any bulk action button becomes clickable.

----

h2. 3. Floating Action Bar

When one or more groups are selected, a floating action bar appears pinned to the bottom of the screen:

*"{X} groups selected · {Y} data points — [Map All] [Activate All] [Deactivate All] | [Cancel]"*

----

h2. 4. Bulk Actions

h3. Map All

Map All only works when every selected group belongs to the same section (e.g., all Revenue, or all Assets). If the user selects groups from different sections, Map All is grayed out with a tooltip: "Select groups within the same section to map to a single GL account."

Opens a confirmation modal:

* Header: "Bulk Map {Y} Data Points"
* Shows a list of every selected group by name and how many data points are in each one.
* Message: "All data points in the selected groups will be mapped to the chosen GL account, replacing any existing mappings. Future transactions will route through this account."
* GL account typeahead selector, showing only accounts that match the section's type (same filtering as the edit drawer).
* Actions: *Apply* / *Cancel*

On Apply: Maps every data point in the selected groups — including inactive ones — to the selected GL account. Changes go into the existing pending changes queue and don't commit until the user clicks "Save Changes."

After Apply: Toast: "Mapped {Y} data points across {X} groups to {GL Account Name}." Checkboxes uncheck. Page stays in bulk actions mode.

h3. Activate All

* Works no matter which sections the selected groups come from.
* Confirmation dialog: "Activate all {Y} data points across {X} selected groups? Activated data points become eligible for posting." Shows the list of selected groups.
* Actions: *Activate* / *Cancel*
* Changes go into the pending queue. Toast: "Activated {Y} data points across {X} groups." Checkboxes uncheck. Mode persists.

h3. Deactivate All

* Works no matter which sections the selected groups come from.
* Confirmation dialog: "Deactivate all {Y} data points across {X} selected groups? Deactivated data points will not generate journal entries during posting." Shows the list of selected groups.
* Actions: *Deactivate* / *Cancel*
* Changes go into the pending queue. Toast: "Deactivated {Y} data points across {X} groups." Checkboxes uncheck. Mode persists.

----

h2. 5. Feedback & Change Tracking

* -All bulk changes go into the existing pending changes queue — nothing commits until the user clicks "Save Changes."-
* All changes show up in the configuration change history after saving.
* -Pending indicator dots appear on affected rows, same as they do for individual edits.-

----

h2. 6. Exiting Bulk Actions Mode

Two ways to exit: "Cancel" in the floating bar OR "Exit Bulk Actions ✕"

On exit:

* Checkboxes disappear. Floating bar and banner disappear. Edit buttons, inline mapping, and Rules buttons go back to normal.
* Modified on {{2026-02-17}} after discussion with developer.

----

h2. 7. Status Change Behavior

Changes only commit when the user clicks "Save Changes." 

*Note: Bulk action counts and operations are filter-aware, so when search or any table filter is active, they only reflect and apply to the currently visible matching data points rather than the entire underlying groups (updated on* {{2026-02-11}})
