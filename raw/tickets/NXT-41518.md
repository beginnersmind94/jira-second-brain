# NXT-41518 — Acc - Reconciliation - Bulk Adjustment Workflow

## Description
As Simon the Site Manager I want to adjust multiple transactions at one time to save a lot of time for myself, and for Support + Development.

*requirements*

* Reconciliation → Add a Bulk Adjustment option to the More Actions list.
* Bulk Adjustment option opens a drawer with a 3-part stepper.

Step 1 - Search

* User can search the Session with a range of dropdown filters.
* User can see the results of the search + information on the transactions.
* User can select the transactions they want to edit.
* User can select all transactions with 1 click.
* User can move further into the workflow.

## Acceptance Criteria
* Bulk Adjustment (BA) option added.
** Option is disabled if the POS Period is closed.
* User permission added for Session Bulk Adjustment.
* BA option opens a drawer with a 3-part stepper.

Step 1 - Search

* User has dropdowns that let them filter the Transactions list.
** Meal Type
** Patron Type
** Pricing
** Menu Item
** Grade
* Dropdown options are based off of what is available inside the Transactions list.
* User can leave the dropdowns blank (or ‘All’) and return all transactions inside the session.
* Dropdowns are multi-select.
* User can execute a search using their selections.
* User can see the results of the search:
** Columns: Transaction #, ID number, Meal Type, Patron Type, Pricing, Menu Item, Price, Grade
* User can select transactions in the results.
* User can select all transactions with a single click.



Step 2

* Edit Transactions option added to the Bulk Adjustments workflow.
* Delete Transactions option also added.
* For Editing:
** User has a dropdown that they can pick any meal type from.
** User has a dropdown that they can pick any menu item from.
** These are single selection dropdowns.



Step 3 added to the Bulk Adjustment workflow.

* User is given a summary of the impact:
** For deleting transactions, outline how many will be removed.
** For editing, outline how many will be edited.
*** Data broken down by Person Type (Students vs Adults) and then Price Type (F/R/P/SecondMeal, Staff/ProgAdult/Visitor)
*** Show the # of transactions being edited.
*** Show the current meal type + item + price, show the new meal type + item + price.
* User must add a comment.
** Comment will be used for the adjustment + session history.
* User can action the bulk adjustment.
** These must all be done as Transaction Adjustments.
** Use the comment as part of the adjustment.
* User can see the adjustments under Transactions > Adjustments.
* User needs a confirmation prompt on Save.
