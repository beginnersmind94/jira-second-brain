# NXT-40555 — Monitoring Scheduler – P8 View Job Details - QA

## Description
As a SchoolCafé admin, I want to be able to view the details of monitoring jobs so that I can proactively monitor the application and ensure higher customer satisfaction.

*Flow:*

* User logs in and browses to Support > Monitoring > Job Scheduler
* User clicks the History tab
* User views a list of monitoring jobs that has completed
* User clicks View Log button on a row
* User is redirected to the Job Details page and views the details there

*Notes:*

* This is a simplified version of what we will do in the future

*Requirements*:

* On the History tab of the Job Scheduler page, do the following using the View Log page as the example UI/UX:
* 
* Navigation: System > Support > Monitoring > History
* Page title: Job Details
* 
* Create an html table with key / value pairs like the Figma
* Fields to show:
** Module
** Monitor
** Start Time
** End Time
** Status
** Message
* If the data is missing any of these fields, then show the fields you have
* 
* Permissions:
* In General resources screen for Roles & Permissions
** Application: applies History Tab view permission
** Write the security code to apply that permission to this button
* 
* The actual data for these fields and tabs will be included in other stories

## Acceptance Criteria
* Page opens when click on View Details from History
* Page shows an html table of the key / value pairs of the available fields (listed in the story)
* User can close the drawer
