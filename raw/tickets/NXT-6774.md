# NXT-6774 — System - Users Phase 2 - Add User  - Organization & Permissions section

## Description
As a District Admin, I would like to be able to easily grant PrimeroEdge access to a new user in my organization, so that they can quickly begin working in the system.

+Summary+:

Utilize the existing breadcrumbs (System → Configuration → Users → Add User button) and update the existing Add User page to use the new UI template *for the Organization & Permissions section only*.

+Details+:

* In the Add User screen, add an *Organization & Permissions* section to the bottom of the screen.
* This section should display the following fields. All are required:
** Organization Level
*** Single-selection drop-down can include District, County, Area, Sub-Region and Site options.
**** County can display within Region(s) drop-down as a tree structure if needed
**** Sub-Region can display within Area drop-down as a tree structure if needed
** Authentication Mode (Read-Only)
*** Displays the Authentication Mode chosen by the Tenant/District. Can be PRIMERO by default if needed.
** Region(s) - At least 1 required. When a Region is selected, 2 other fields may display:
*** Area - 
**** This field is only required if the user's Organization Level is "Area"
**** Single-selection drop-down. 
**** Areas must belong to the user’s selected Region.
*** Site(s) - 
**** +Only display this field if user’s Organization Level is not “District”+. 
**** This field is only required if the user's Organization Level is "Site"
**** Multi-select drop-down. Smart search is preferable.
**** IF an Area is selected, sites in this list must belong to that Area. Otherwise, this drop-down should display all sites within the district. 
** Role(s)
*** Multi-select drop-down. Smart search is preferable.
*** At least 1 role required.
*** Displays all Roles (Standard & Custom) that have been created by/for the district.

## Acceptance Criteria
Verify that when a user navigates to System -> Configuration -> Users and selects Add User, they are presented with the new design template for the Add User page and that the page contains an Organization & Permissions  section that displays the following (required) fields:

Organization Level -Single-selection drop-down includes District, Area, and Site options.

Authentication Mode (Read-Only) - Displays the Authentication Mode chosen by the Tenant/District. Can be PRIMERO by default if needed.

Region(s) - At least 1 required. When a Region is selected, 2 other fields may display:

Area - Single-selection drop-down. Optional field unless the user's Organization Level is "Area". Areas must belong to the user’s selected Region.

Site(s) - Only display this field if user’s Organization Level is not “District”.  Multi-select drop-down.

Verify that IF an Area is selected, sites in the Site(s) list must belong to that Area. Otherwise, this drop-down should display all sites within the district. 

Role(s) - Multi-select drop-down. Displays all Roles (Standard & Custom) that have been created by/for the district. At least 1 must be selected.
