# NXT-27895 — INV - Orders - View Orders - Research Data Query Custom View - UI Only

## Description
Research the implementation of the Data Query custom view framework on the View Orders screen.

* Currently, views and columns are static view, user won't be able save them as custom view
* The change is required to build the custom view so that user can customize their views as needed
* Full implementation story for reference: [https://primeroedgenext.atlassian.net/browse/NXT-24861|https://primeroedgenext.atlassian.net/browse/NXT-24861|smart-link]

## Acceptance Criteria
Creation of Views:
User should be able to select all the field as available
User should be able to save the view as a new view with user defined name
User should be able to select a view and flag them as default view (Check Box)
If the user select a view as default view, then system should show the fields as per the saved view (not the original view)

Viewing Saved view:
User should be able to view the �Saved Views� through an additional tab in Slide out
For each view saved, the user should be able to flag one view as default view
User should have an option to edit or delete a view

Editing a View:
User should be able to select the saved view
User should be able to select or deselect one or more columns
Once the target fields are selected, the user able to click the �Save� button and save the view

Deleting a View:
User should be able to delete one view at a time by clicking the delete icon
Upon deleting all the view, application should reinstate the original view
