# NXT-25480 — Accountability - Transactions - Adjustments - Session Adjustments

## Description
As a Central Office Admin I want to see what adjustments are being made to accounts and sales so I can see if anything is being done incorrectly.

As a Site Manager I want to see the same, but for the sites I have access to.

*requirements*

* Add the Session Adjustments report as an option to the Adjustments page:
** Use the search criteria to pull back any ‘session adjustments’ that have been performed which match. 
*** This is when the user has to adjust a session by an amount in order to make it hit a $0.00 over/under so they can reconcile it.
** The report shows the following information:
*** *Site*
**** Code + Name of the site the session was recorded at.
*** *Terminal*
**** Name of the terminal for that session.
*** *Session Date*
**** Date the session was opened. Include time as well.
*** *Adjustment Date*
**** Date the session was adjusted.  include time.
*** *User*
**** Firstname + Lastname of the user who performed the adjustment
*** *Comments*
**** If a comment was entered when the adjustment was done, show that string.
** -Show a- *-total-* -for the- *-Amount-* -column per site and for all sites together.-
** Sort order:
*** Site, Session Date
** Report should show the typical headers and footers:
*** Generation datetime and user.
*** Report name.
*** Search criteria.
*** Footer about the report by SchoolCafe.
