# NXT-37568 — IM - Items - Units - Save & Continue - Costing

## Description
As a user configuring items I want a series of completion tasks in the new Units card so I can configure any serving or pack size data points that aren’t input on a specific row or add flow.

*Requirements*

* Provide questions/prompts to complete the overall required fields for saving the initial Units card and proceeding to item creation
** this will have a lot of overlap with the [Text Guidance story|https://primeroedgenext.atlassian.net/browse/NXT-37068]
* Execute required checks before enabling official Save
** 1. Confirm Inventory readiness / Base Unit - [https://primeroedgenext.atlassian.net/browse/NXT-37111|https://primeroedgenext.atlassian.net/browse/NXT-37111|smart-link] 
** 2. Prompt for Costing (*this story)*
** These checks will always apply when saving the card, but these are presented as “steps” when creating the first time
** Once saved, these 2 checks will be handled differently in edit/update scenarios.
*** The checks will still occur, but will be managed by icons/banners
* Provide a Costing prompt - “Please add costing for the most relevant unit”
** Enable the Cost/Value field to be edited for all Pack Size units or a linked Serving Size (linked via Sub-Qty)
*** *if only Serving Sizes exist,* allow user to enter a cost for any serving sizes with '1' as the Amount
** Provide the ability to Skip 
*** Flag the item as ‘Missing Costing’ if the user skips the costing step (this will be indicated on the card header in view mode)
* Consider Inventory Readiness for Costing validations
** -if the Valuation Group selected has a ‘Standard Price’ for Valuation Method, Costing is required for Inventory-
*** -this is because FMV is used for the base valuation within Inventory-
*** -other Valuation Methods do not reference FMV-
** Require FMV for Inventory readiness
*** we may “loosen” this in the future so items can be brought into Inventory without FMV

----

*General Rules:*

* Sub-Qty is for pack sizes, except the one case where serving size is linked to the base unit
* Global Info (Brand, Manufacturer, Product Code) is always optional, but ideally available
* Inventory flag is triggered if Pack Sizes are added
* Item Categorizations are only required if Inventory is enabled 
* at least one serving size should be configured before moving on
