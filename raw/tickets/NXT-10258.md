# NXT-10258 — Template Integration - Sending - Print Batching

## Description
*User Story*

As a district admin I just sent 5,000 letters to print. I would like them organized in neat batches that I can re-print if needed.

*mockup*

N/A

*requirements*

We have customers sending 1,000s of emails to print at one time. Rather than create a single print job, we need to intelligently batch the letters into smaller print jobs and queue them up.

* System setting for Print Batch Sizing must be used.
* Large print jobs must be broken into smaller print jobs, using the system setting.
* Print jobs must be sent to local printer sequentially.
* We must keep track of these batch prints -
** If the printer jams halfway through a batch, customer needs a way to get back to that particular batch and re-print it.
*** This leads to a very nice-to-have would be if customers could see recent print jobs out of 2.0 and reprint them from a particular batch onward.
