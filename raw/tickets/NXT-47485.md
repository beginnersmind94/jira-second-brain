# NXT-47485 — Roles & Permissions - New Screen - Users Tab

## Description
As a SchoolCafé admin, I want a more intuitive way to set permissions for given roles so that I can control who has access to what.

*Flow:*

* Open SchoolCafé and browse to Settings > Roles & Permissions (R&P)
* View list of roles
* Click on Edit on a role to edit permissions
* Edit drawer opens
* User clicks on Users tab
* User sees a list of users in the role
* User optionally adds a user
* User optionally removes a user

*Requirements*:

Updates to the Edit Permissions drawer

* For the System > Roles & Permissions > Edit button click > Edit drawer, do the following:
* Add a new tab
** Title: Users
* 
* Top of page:
* + New button
** On click, open the Add User to Role drawer (see below)
* 
* Grid:
** Data: All the users with that given role
** Grid has usual filtering, paging, sorting
** Grid has download button (can be excel, delimited, or whatever works) which downloads the whole list of users for that role
** Columns:
*** Name
*** Action(s)
* 
* Actions column buttons
** View
*** On click, show their user profile like on Edit User Drawer
** Delete
*** On click, do the usual confirmation box
*** If confirmed, remove the user from the role (Don’t delete the account 🙂 )
* 

* Add User to Role drawer
** Title: Add User to Role
** Width: the smaller one
* Fields:
* Users
** Multi-Select list
** Values: list of all the active/enabled user accounts for the district that are not already added 
** Be able to start typing and filter the results
* Save button
** On click: add users to the role you are editing
** Refresh the grid on the Users tab to show the new users in it
* Cancel button
* 
* Security for tab and drawer
** Use the Edit permission for the R&P page to control adding and deleting users
** Use the View permission for the R&P page to control who can see the page



* Display Validation message if user have only one role assigned

## Acceptance Criteria
* All the requirements are met from above
* On edit screen, users can add or remove a user to the role
* Only users with R&P page Edit permission can add or delete
* Only users with R&P page View permission can view the page in read-only mode
