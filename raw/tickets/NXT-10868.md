# NXT-10868 — Verification - Sampling - Schedule sampling

## Description
As a district admin I want the sample to be created automatically, based on the options I picked from the wizard.

*mockup*

n/a

*requirements*

* Use the scheduler for the sample(s) to be created automatically at an appointed time.
** The mechanism should automatically create claims on the selected sample date, at the time from the system setting.
** When looking at dates, we don’t want to generate any for the current date or a future date.
*** Example: 
**** Sample Date is set to 6/20/21. Current date is 6/14/21.
**** The sample should not be generated until 6/21/21 to allow any applications from 6/20 to be included.
** When gathering applications, we need to be mindful of oversampling a particular application.
*** This is for rolling verification.



Some examples from refinement to get an idea on generation timing:

* User performed Setup on 7/15 and set 2 rolling samples + final sample:
** 8/1 - generate at 8/2 12:00 AM
9/1 - generate at 9/2 12:00 AM
** Final 10/1 - generate at 10/2 12:00 AM
* Setup 9/30
** 7/1 - 8/1 - generate on 10/1 12:00 AM
8/2 - 9/1 - generate on 10/1 12:00 AM
** FINAL 10/1 - generate on 10/2 12:00 AM

## Acceptance Criteria
Background mechanism to automate sample creation created and working.
System setting for time added, and used for sample creation.
Future samples should be triggered at the appropriate date.
Past samples should also work properly.

If we're doing this with Rolling Verification in mind, then we need to be sure that we're not oversampling applications - the gathered app number would need to check for a date range starting at the previous sample date + 1, if one exists, up until the new sample date at the timestamp.
