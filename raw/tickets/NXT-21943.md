# NXT-21943 — Serving Measure Configuration

## Description
As a user I need to configure all possible or most use Measuring sizes

## Acceptance Criteria
This story is done when:



*Entry Point*

* A  Serving Measure configuration option is added  under IM configuration  area
* Option Item Configuration Configuration
* Data source as dropdown box default to local
* Configuration parameters should list the field name in which the configuration is been added. E.g Serving Measure

*Main Area*

* Button  (ADD) to add Serving Measure
* When ADD is clicked
** A Slide Out Card is displayed
** Option to select Data Source where the Serving Measure is to saved (Smart drop down box) 
** One data source is to be selected. If adding a new Serving Measure it is mandatory to be populated 
** Field to enter new Serving Measure (Free Text box) . If adding new Serving Measure it is mandatory to be populated
** New Serving Measure should be validated for duplicates at the time of entry.  (No duplicates allow, if duplicate if found give warning message “Serving Measure Already Exist, Please enter different  description)
** Duplicate is allow when across multiple data sources
* SAVE button 
** When clicked the New Serving Measure is stored 
** New Serving Measure is displayed in the result grid
** Closes Slide Out Card and returns control to the Serving Measures page
* X Icon
** When clicking the X Icon
*** Cancels additions and closes Slide Out Card

*Result Grid*

* Should the display fields as per mockup below
* User should be able to filter and sort fields on result grid 
* User should be able to navigate as per mockup below
* All existing Serving Measures are display with options to edit or delete
* Serving Measure can only be discontinued if it is not used by any item/recipe
* Only Serving Measure description is editable
* Data Source

Only user created items can be edited
