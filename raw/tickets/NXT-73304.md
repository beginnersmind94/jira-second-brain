# NXT-73304 — FIN — Reverse Import: Payroll Import

## Description
Adds Reverse import support for **Payroll Import** on the Import & Export Log page (NXT-73286).\
\
Payroll Import follows the ledger-posting import reverse pattern defined in NXT-73286 sec 5A. Batches for non-overlapping PostedDate ranges are independently reversible.\
\
**Gates:** Year-End Close (hard block), overwrite per NXT-72753 (hard block), closed period (prerequisite — reopen to unblock).\
\
All UI patterns (confirmation dialog, mandatory comment, Cancel / Reverse import buttons, Status = Reversed, audit trail) follow NXT-73286."

## Acceptance Criteria
**AC1 — Reverse available on Payroll Import.** Given the Import & Export Log page (NXT-73286) is built, Reverse import appears on Done Payroll Import rows.

**AC2 — Independent reversibility.** Payroll Import batches for non-overlapping PostedDate ranges are independently reversible. Batch A (Jan 1–15) and Batch B (Jan 16–31) can both be reversed without blocking each other.

**AC3 — Closed period prerequisite.** If any period affected by the Payroll Import is closed, Reverse is disabled with tooltip directing the user to reopen the period(s).

**AC4 — Year-End Close hard block.** If Year-End Close has been run for any FY the import's entries touch, Reverse is permanently disabled.

**AC5 — Overwrite hard block.** If the Payroll Import batch was overwritten by a later import (per NXT-72753), Reverse is blocked.

**AC6 — Confirmation dialog.** Dialog shows import name, entry count, line count, affected FYs, total debits/credits. Warning per NXT-73286 sec 4.4 ledger import pattern. Mandatory comment. Buttons: Cancel (default) / Reverse import (destructive).

**AC7 — Hard delete.** On confirm, BL permanently removes all Payroll journal entry lines, headers, and batch record. Atomic. DE status updated to Reversed.

**AC8 — Rollups recalculated.** After deletion, rollups and balances for affected fiscal periods are re-run.

**AC9 — Duplicate detection.** After reversal, subsequent Payroll Imports for the same PostedDate range do not flag the deleted batch as a duplicate.
