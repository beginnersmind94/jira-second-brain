# NXT-46268 — Sync Agent - Track Errors - API

## Description
As a SchoolCafé Customer Support user, I want to be able to view sync agent issues so that I can quickly diagnose errors.

*Notes:*

* This is a backend only story, front end covered elsewhere
* Dev manager will ultimately define the requirements for this story

*Requirements:*

* -create db tables to store sync agent errors / issues-
* create APIs to log (create) them
** should be usable by other modules as well
* create APIs to provide (view) show them
* try to do in a way that is extensible and flexible, framework style so that others can leverage
** try not to hardcode stuff 



*Dev:*

* This ticket consists of data retrevial from 2 sources (one from Deadletter and one from app insights)
* Currently all the exceptions are logged into appinsights and there is a controller that retrieves the data from appinsigts [https://dev.azure.com/Cybersoft-Technologies-Inc/Platform/_git/PrimeroEdge.Administration?path=/PrimeroEdge.Administration.WebAPI/Controllers/AppInsightsController.cs|https://dev.azure.com/Cybersoft-Technologies-Inc/Platform/_git/PrimeroEdge.Administration?path=/PrimeroEdge.Administration.WebAPI/Controllers/AppInsightsController.cs]
* We need to retrieve the data from deadletters from rabbitmq and append that dataset to data retreived from app insights
* Model Needs to be
** ModuleName
** Exchange
** RoutingKey
** Date/Time
** Actions
*** Details (Exchange, Routing Key, Exceptions, Payload, DateTime)

## Acceptance Criteria
* We are able to capture and store sync agent errors / issues
