# NXT-64749 — Insights - KPI - MPLH - Dashboard - PE

## Description
*I am a* district user
*that I want to* view Meals Per Labor Hour (MPLH) performance in the Insights dashboard
*so that I* can monitor actual performance against expected benchmarks across sites and timeframes.

----

h2. *Notes*

* This story applies to *Insights – PrimeroEdge data source*.
* MPLH (Meals Per Labor Hour) is part of the *front office KPIs* being integrated into Insights.
* This story covers the *Dashboard screen only*.

h3. *Data & Source of Truth*

* The source of truth for MPLH data is available in:
** *POS → Management → Meals Per Labor Hour*
* This must be used by developers and QA to validate MPLH calculations and displayed values.

h3. *Stored Procedure Integration*

* The front office team has already created the stored procedure:
** PrimeroEdge: [https://dev.azure.com/Cybersoft-Technologies-Inc/PrimeroEdge%20Classic/_workitems/edit/109334|https://dev.azure.com/Cybersoft-Technologies-Inc/PrimeroEdge%20Classic/_workitems/edit/109334|smart-link] 
* The dashboard must use this stored procedure as the data source for MPLH values.

h3. *KPI Definition & Formatting*

* MPLH is a *rate-based KPI*.
* Format must be: *0.00 (two decimal places)*.
* Values must be *rounded to two decimal places*.
* Calculation:
** *MPLH = Equivalent Meals ÷ Actual Labor Hours*
* The dashboard must display:
** Actual value
** Expected value (*MLPH Goal from benchmark configuration*)

h3. *Filtering Behavior*

* All MPLH components must respond to *Timeframe and Site filters*:
** KPI card
** School Performance grid
** Performance Trends chart

h3. *Interaction Behavior*

* Clicking on MPLH values (card or grid) must open a *details drawer* (covered in a separate story).

----

h2. *Out of Scope*

* SchoolCafé implementation (separate story)
* Benchmark configuration UI (covered in separate story)
* MPLH detail drawer implementation
* Schoolie Update
* Paid Not Applied (P&A) KPI
* Eligible Not Participating (EMP) KPI

----

h2. *User Flow*

# A user logs into PrimeroEdge.
# The user navigates to Insights.
# The user lands on the Insights dashboard.
# The user selects timeframe and site filters.
# The system displays KPI cards including MPLH.
# The user views MPLH:
#* Actual value
#* Expected value
#* Performance indicator
# The user may:
#* Adjust filters → dashboard updates
#* Open configurator → show/hide/reorder cards
#* Click MPLH card → opens detail drawer
# The user scrolls to:
#* School Performance grid → sees MPLH column
#* Performance Trends chart → selects MPLH
# The system updates all views based on filters and selections.

----

h2. *Requirements*

h3. *1. KPI Card – MPLH*

* Add a new KPI card:
** Label: *MPLH*
** Location: *added to end by default*
* Card must display:
** MPLH rate (actual value) for selected filters
** MPLH Goal (from benchmark configuration)
** Trend indicator
* Values must be formatted as *0.00 rate*
* Card must update based on:
** Timeframe filter
** Site filter
* On click:
** Open *MPLH Details Drawer*

----

h3. *2. KPI Card – Performance Indicators*

* Card must display color states:
** *Green* → Actual ≤ Goal
** *Red* → Actual > Goal
** *Grey* → No benchmark configured
* If no benchmark exists:
** Card displays grey

----

h3. *3. Filters (Timeframe & Site)*

* Dashboard must include:
** Timeframe selector
** Site selector
* Changing filters must update:
** MPLH KPI card
** School Performance grid
** Performance Trends chart

----

h3. *4. Screen Configurator*

* Ensure MPLH is added to *Configure KPIs drawer*
* User must be able to:
** Toggle MPLH card on/off
** Reorder KPI cards via drag-and-drop
* Changes must:
** Persist per user
** Apply to dashboard layout

----

h3. *5. School Performance Grid*

* Add *MPLH* *column*:
** Label: *MPLH*
** Location: *last column*
** Format: *0.00 rate*
* Column must display MPLH per site
* Grid must update based on:
** Timeframe filter
** Site filter
* Grid must support:
** *Horizontal scrolling when columns exceed screen width*
* On click:
** Open *MPLH* *Details Drawer*

----

h3. *6. Performance Trends Chart*

* Add  *MPLH* to KPI dropdown:
** Location: *bottom of dropdown list*
** Order must align with benchmark configuration screen
* Selecting MPLH must:
** Display MPLH trend line
** Update based on timeframe and site filters

----

h3. *7. Trend Hover Behavior*

* Hovering over MPLH trend line must display tooltip including:
** Date
** MEQs
** Hours
** MPLH (formatted as n.nn)

----

h3. *8. Data Integration*

* Ensure MPLH is added to:
** Backend API responses
** Dataset used by dashboard
* Data must support:
** Actual MPLH values
** MPLH Goal (benchmark)
** Trend calculations
** Chart rendering

----

h3. *9. Performance Requirement*

* Dashboard must load and update without performance degradation
* Must support large districts with many sites and long timeframes
