# NXT-28661 — IM - Permissions (ADD/EDIT)

## Description
As item Manager user I shall be able to see Item Management site pages according to user' s permissions (VIEW)

*Entry Points*

* Item Management → Items (List and Split views)
* Item Management → Recipes (List and Split views)
* Item Management → Menu Items
* Item Management → Item Configuration
* Item Management → Item Configuration → Menu Item
* Item Management → Item Configuration → Item Category
* Item Management → Item Configuration → Storage Category
* Item Management → Item Configuration → Serving Measures
* Item Management → Item Configuration → Custom Allergen Configuration
* Item Management → Item Configuration → Valuation Groups
* Item Management → Item Configuration → Recipe Categories

*This story applies to:*

* *Items*
** Items Landing Page (List and Split views) - Applicability: (View, Add,)
* *Recipes*
** Recipes Landing Page (List and Split views) - Applicability: (View, Add)
* *Menu Items*
** Menu Items Landing Page - Applicability: (View, Add)
* *Configuration*
* Item Configuration Landing Page (User should have View access)
** Menu Item Category - Applicability: (View, Add)
** Item Categories - Applicability: (View, Add)
** Storage Categories - Applicability: (View, Add)
** Serving Measures - Applicability: (View, Add)
** Custom Allergens - Applicability: (View, Add)
** Valuation Groups - Applicability: (View, Add)
** Recipe Categories - Applicability: (View, Add)

 

* *ADD* *Permission* -
** If enabled
*** If the he user has add permissions to the page the user shall be able to ADD  page content
** Else hide  \[+ NEW] add option  

*General:*

* For pages where user has add permissions the user should also have VIEW permissions
* For pages where user does not has view permissions then ADD option shall not be available to user
