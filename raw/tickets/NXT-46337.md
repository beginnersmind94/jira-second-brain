# NXT-46337 — MP- Find and Replace- Inactive and Pending Deactivation Menu Items

## Description
As a menu planner, I should never see inactive menu items on the find and replace feature and should only see pending deactivation menu items along with normal menu items. This feature should help me replace all the pending deactivation menu items to get them into inactive status if they are designated as such.

## Acceptance Criteria
* Do not show Inactive menu items in the drop downs for find or replace
* Do not allow inactive menu items to be searched in either drop down
* Pending deactivation menu items should be able to be searched for in both drop downs
** Provide a warning sign with tool tip in the search results “Pending deactivation”
* Allow pending deactivation menu items to be used for replace (This is up to the user but we will provide the warning)

!image-20240111-062336.png|width=1202,height=400!

* After Menu item selection, we should continue to display the warning sign and the tool tip for user
* Provide a filter to show Pending Deactivation Items only for find (This will allow users to quickly go through those items and remove them from menus)
** The drop down should reset with only pending deactivation items and should be searchable with only pending deactivation items

!image-20240111-062749.png|width=1197,height=436!

* Lock all filters (Find, Replace, Pending Deactivation Items only) once user triggers Replace button
* Do status check on final action trigger of replace against both “Find” Menu item and “Replace” Menu Item 
** IF “Replace” menu item has been made inactive, clear the Replace with Field
*** Display message “(*Menu Item*) has now been made inactive, Menu item can not be replaced” ( These are error messages)
** IF “Find” or “Find” and “Replace” have been made inactive, do a hard reset on the page but continue to display error message
*** Display message “(*Menu Item*) has now been made inactive, Menu item can not be found or replaced” (These are error messages)
