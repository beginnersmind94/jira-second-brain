# NXT-2631 — Access(Manage Operations)

## Description
Access consists of permission to view, add, edit, update, and/or delete pages and functional areas throughout the solution.  

Potential 2.0 Approach - Beyond permissions for Operations (pages), major functional areas (change Eligibility Status), etc. we should also include specific access settings for things such as “PII”. We should designate pages/features in the system to have a flag that indicates it contains PII. If the user does not have access to PII, these pages/features would be hidden.
