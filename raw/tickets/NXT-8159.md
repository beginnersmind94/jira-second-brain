# NXT-8159 — INV - Physical Inventory - Save as Complete (Back end)

## Description
As an inventory user I want a transaction that captures saving an inventory as complete so my perpetual is properly calculated and reflected on the inventory.

Calling a function that grabs the perpetual inventory, calculates and saves that to the physical inventory table.

Save as Complete button click
If all of the validation pass then we will save a snapshot of the perpetual inventory into IM_PhysicalInventoryDetails and update the status of the physical inventory to complete and pending reconciliation.
The snapshot is calculated using IM_SiteInventory minus all transactions after the counted on time.

Validations
Reconciliation date cannot be before another physical inventory for the site and must be at least an hour later than the previous physical inventory
Use the setting “Physical Inventory Close day” to determine if we are checking the current date(counted date time) or prior date
Check if any open, pending approval, or incomplete orders exist before the date. If so then throw error
Check if any incomplete receipts or transfers
Check if any incomplete production withdrawals
Check if any distributions or central productions open

Period option
Check setting “Physical inventory – Enable Period Option setting”

If yes
Can only create one physical inventory for a period
If they want more that one physical inventory during the same period it will need to be created as interim
When saving as complete if the period is completed (current date is greater than period end date) then the counted date and time will be set to end date 11:45:00:000 pm
If the current date is less than period end date then the date will be set to the current date and the time will be set to the current time rounded down to nearest 15 minutes
If using interim option they will have a date and time picker to set their counted time

If no
The user will have a date and time picker to selected the counted on date

## Acceptance Criteria
User can save the inventory as complete
User is provided a popup when saving
Popup displays the current period if the inventory is done by period
Popup provides a date and time entry field if the inventory is not done by period (interim)
