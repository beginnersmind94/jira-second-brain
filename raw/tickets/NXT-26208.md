# NXT-26208 — SIS Auto Sync - Give End Users Ability to See Errors

## Description
Original Request:

The SIS API shall allow an end-user the ability to see any errors it throws.  I recently ran a SIS API import in QA and it threw an error.  However, I was not able to find what the errors was.  In my scenario, I believe there were no valid API licenses.  However, the “Test Connection” button returned a success.



JAS Note: 

When I click on an SIS import that has a status of Error, I get a blank screen in the overlay.
