# NXT-54012 — Acc/AM - Edit Transactions - Allow for Partial Payments (FD: 255617)

## Description
[https://primeroedge.freshdesk.com/a/tickets/255617|https://primeroedge.freshdesk.com/a/tickets/255617] 



[https://www.loom.com/share/4fe004ca712d44898526e313cf87bf37?sid=0041552f-6022-49d0-a112-8a85b0af47ac|https://www.loom.com/share/4fe004ca712d44898526e313cf87bf37?sid=0041552f-6022-49d0-a112-8a85b0af47ac|smart-embed]

Need to allow Edit Transaction to save a partial payment.

*Requirements*

* User can Edit a transaction and leave a partial payment if Payment Method is Cash/Check
** Amount Tendered < New Sale Amount
* User cannot leave a partial payment for Payment Method = Card
** Amount Tendered must = New Sale Amount
* Checks and validations are required for this when the user tries to Save the transaction
** Check the Charge Limits for the district to see if this exceeds the allowed charge limit
*** Check the System Setting BABECL
**** Decision to allow sale with user → Confirmation prompt like POS
**** Sale only with some cash → partial payment, allowed
**** No sale with out sufficent funds → block, error message
** Check the account’s purchase restrictions for debit restrictions on ALC items, if an ALC item is in the list
*** Respect the restrictions - inform the user about the restriction preventing the edit
** If no checks/restrictions are triggered → allow the partial payment to go through and update transaction accordingly.
* Add a special role permission to override restrictions
** Accountability
*** Name: Override Restrictions/Charge Limits When Editing Transactions
*** Description: This will allow a special override feature on the website (not at POS) when adding/editing transactions. The override option is available when purchase restrictions or district charge limits may be triggered while editing transactions.
** If the user has this permission, they get an Override option for these scenarios:
*** BABECL triggered on ‘No Sale with out sufficent funds’
*** Purchase Restrictions

*Discussion Notes 8/21*

* This applies for Add Transaction as well.

## Acceptance Criteria
* Editing Transactions → Partial Payment is possible, if BABECL and Purchase Restrictions are not triggered
* User is given an error message if BABECL or Purchase Restrictions are blocking it when they try to Save
* Special Permission for Override is added
** If user has the permission, they get an Override/Cancel option on the error message
** If they Override, allow the edit to go through
