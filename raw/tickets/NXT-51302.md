# NXT-51302 — Imports - Queuing Imports

## Description
We need to put in a queuing system for the imports to limit how many run at one time

*Notes:*

* Currently, many sites are putting their imports at the exact same time of day and it is causing issues with the database
* We want to implement queuing so that we can control how many run at the same time

*Requirements*:

* Put a configuration setting that is global (applies to all districts collectively) on how many imports can run at once
* Start with 2 at once
* Update the imports:
** When it is time, check to see if the threshold has been met (already 2 running)
** If it hasn’t, then proceed as normal
** If it has, then postpone the import for one minute and try again
* Note: we need a way to prevent a runaway process that indefinitely runs
** My suggestion is to put in a control that will auto kill any more attempts after a certain time limit like an hour or a certain count of retries

## Acceptance Criteria
* System is able to limit how many imports run at the same time
* System is able to que them so that they run sequentially
* Queuing is for all districts combined, not just for one district
