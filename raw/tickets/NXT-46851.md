# NXT-46851 — GDSN Connect Pack Sizes (SC)-Non Food Items

## Description
_(none)_

## Acceptance Criteria
The system should render the Pack Size details for Non-Food Items based on the following:

Coverage: SQL Script, Writing the results in unit table

*Non-Food Item*

Condition: If _Purchase Description Equals to Inventory Unit Description_ and if _IUs_Per_PU=1_

Result - Consider it as 1 Tier; Insert only Highest Unit; Serving Measure – Not applicable

Highest Unit:

·         Unit Description = Purchase_Unit_Description

·         Weight = Purchase_Unit_Weight

·         Weight UOM = Purchase_Unit_Weight_UOM

*  Sub Quantity = NULL/Blank

·         -_Identifiers = P, denotes Pack Size_-

·         Base Unit Flag = 1

GTIN = GTIN_PU

 

Condition: _If Purchase Description_ +_Not Equals to_+ _Inventory Unit Description_ and if _IUs_Per_PU >=1 (i.e. 1 or above))_

Result – Consider it as 2 Tier; Insert Highest Unit & Lowest Unit; Serving Measure – Not applicable

Highest Unit:

·         Unit Description = Purchase_Unit_Description

·         Weight = Purchase_Unit_Weight

·         Weight UOM = Purchase_Unit_Weight_UOM

         Sub Quantity = IUs_Per_PU (only If value is >1)

              If IUs_Per_PU is 1, then Sub Quantity=Null / Blank

* GTIN = GTIN_PU

Lowest Unit

·         Unit Description = Inventory_Unit_Type

·         Sub Quantity =  Quantity=Null / Blank

·         Weight = Purchase Unit Weight / IUs_Per_PU

·         Weight UOM = Purchase_Unit_Weight_UOM

·         -_Identifiers = P, denotes Pack Size_-

·         Base Unit Flag = Send it as BU

* GTIN = GTIN_Base
