# NXT-179 — Misc - Edit Check List

## Description
*Story*

As a user I need to be able to see all edit check that need to be processed so a comment can be entered as to why it occurred. An edit check is created when the percentage of students for an eligibility type exceeds the attendance factor.

This list aims to show users the enrollment count, the enrollment count impacted by the attendance factor, and the meal sales for any particular date.

It is summed up when run for a date range / month.

It highlights any exceptions - where they have sold more meals than they have students.

*Requirements*

* A list of Edit Checks can be shown, default status is UNRESOLVED
* One or more comments can be entered for an Edit Check. [https://primeroedgenext.atlassian.net/browse/NXT-7467|https://primeroedgenext.atlassian.net/browse/NXT-7467|smart-link] 
* The user and timestamp are part of the comment [https://primeroedgenext.atlassian.net/browse/NXT-7467|https://primeroedgenext.atlassian.net/browse/NXT-7467|smart-link] 
* When a comment is saved the Edit Check is marked as RESOLVED
* When a resolved EC is opened the comments are visible [https://primeroedgenext.atlassian.net/browse/NXT-7467|https://primeroedgenext.atlassian.net/browse/NXT-7467|smart-link]

## Acceptance Criteria
The Selection Criteria includes Area, Site Code, Site Name, Date Start and Date End, Edit Check Status
Default selection criteria is All except for Date Start and Date End.
Default selection criteria for Edit Check Status is Unresolved 
Date Start and Date End are todays date.
Edit Check Status is 'Resolved/Unresolved'
When Edit Checks is selected a list showing the Unresolved Edit Checks is shown for the selection criteria
A row shows 'Site Code/Site Name, Date, Meal Type, Eligibility, Edit Check Status
Selecting the Edit Check opens a comment window allowing the user to enter a resolution comment.
