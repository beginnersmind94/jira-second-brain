# NXT-36008 — MP- Menu Planner- Dynamically Updating choose counts to reflect Menu items and analysis (UI ONLY)

## Description
As a Menu planner, I may decide to change my mind when I am making choices regarding my choose count when I am creating or editing my menu on the menu planner. Currently, when I make any changes, the changes do not reflect until I save the menu and come back. 

This should also reflect the counts that are calculated so that it reflects to 100% of the category. 

Ex: 

I add 2 fruits and the choose count is currently one. The system should divide the counts evenly between the meal count. For this example we will use 100. It will split the counts to 50 and 50. I then decide to change the choose count to 2. It should repopulate the counts to 100 and 100 to fulfill the 100% meal count that is needed.

## Acceptance Criteria
* Allow changes to the choose count on the menu planner dynamically update the analysis as the user is using the menu planner.
* This should apply on both the create and edit scenarios

Ex: 

I add 2 fruits and the choose count is currently one. The system should divide the counts evenly between the meal count. For this example we will use 100. It will split the counts to 50 and 50. I then decide to change the choose count to 2. It should repopulate the counts to 100 and 100 to fulfill the 100% meal count that is needed.
