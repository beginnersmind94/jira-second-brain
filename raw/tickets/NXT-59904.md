# NXT-59904 — TH - Sec - District Setup Screen

## Description
*As a*: TeamHub user *I want to*: configure my district settings *So that*: I can access the correct district data

*Comments*:

* Screen: District Setup screen ({{app/(auth)/district-setup.tsx}})
* Configures district and platform settings
* Users cannot log in without this information

*User Flow*:

* User opens the application and is directed to login screen
* App checks if they have configured the Platform and District in the District Setup screen before
* If no, app redirects them to the District Setup screen
* User selects platform (SchoolCafé or PrimeroEdge)
* App filters districts per platform selection
* User selects district from available options
* User saves configuration
* User is redirected to login screen

*Requirements*:

_Front End_:

* On District Configuration screen, do the following:
* Ensure the screen supports dark mode from the app settings
* On screen load,
** Go to asynchronous state management datastore and get the values from the last time this app on this device saved
** If never have saved, use defaults
* Fields:
* Platform
** Selection buttons
** Required
** Values: PrimeroEdge, SchoolCafé
** -On selection, filter the District field- 
** Default value: SchoolCafé
** If user has already selected a platform previously, select it
* -District-
** -Dropdown with text search-
** -Required-
** -Values: SC Districts API or PE Districts API-
** -Ensure the user has selected a Platform-
** -Allow user to start typing the name to filter the list-
** -If user has already selected a district previously, select it-
* Mobile App Code
** Display as a textbox
** Mark as a required field.
** Put a text message next to it:  Go to your use profile on the website and get the mobile app code and enter it here to link this device to your user account.
* Save button
** Enabled only when both Platform is selected and Mobile App Code has a value.
** On click:
*** Validate that both Platform is selected and Mobile App Code has a value.
*** Perform the mobile app code check described below
** If successful:
*** Call the APIs for SC or PE to validate the mobile app code
**** Note: APIs covered in separate stories
*** Store the platform, district, username values in the asynchronous datastore for future use in the login screen.
*** Redirect the user to the login screen.
** If failure:
*** Show the error message
*** Try to show a specific message such as “Incorrect code” or “Connection Error”
* Cancel button
** On Click:
*** Redirect user to login screen with no validations or saving anything

-_Back End_--:-

* -Fetch available districts by platform-
** -2 APIS:-
** -Get Districts from PE-
** -Get Districts from SC-
* -Be able to read and save the 2 values from/to the asynch datastore-

_Security_:

* All users can view this page, no security trimming

## Acceptance Criteria
h3. *Front-End Requirements*

* Ensure the screen supports dark mode from the app settings

h4. *On Screen Load*

* Retrieve the previously saved *Platform* and *District* values from the asynchronous state management datastore.
* If no values are found, use the default values.
* Pre-select the previously chosen values if they exist.

h4. *Fields & UI Components*

h5. *Platform Selection*

* Display as selection buttons.
* Mark as a required field.
* Available options:
** *PrimeroEdge*
** *SchoolCafé*
* Default selection: *SchoolCafé*.

h5. *Mobile App Code*

* Display as a textbox
* Mark as a required field.
* Put a text message next to it:  Go to your use profile on the website and get the mobile app code and enter it here to link this device to your user account.
