# NXT-5051 — Messages - Alerts

## Description
*User Story*

As a PrimeroEdge user, I have a way to receive alerts when specific events occur in the product.

*Requirements*

* There is a method for connecting events in PrimeroEdge and enabling an alert when the event occurs
* Alerts are accessed through the alert icon on the user’s navigation bar
** If an unread alert is present, there is an indicator present to show that new alerts are present
* Clicking on the Alert icon shows all recent alerts
** There is a visible difference between read and unread alerts
** User is able to mark all alerts as read using the button at the bottom
** Clicking on an alert takes the user to the page the alert is related to
** Once an alert is clicked, it is considered “read”
* When a new alert is received, a toast message appears
** Toast is up for 5 seconds, then automatically closes
* There is an API that developers can hook into to determine when an alert should be sent and where to direct the user to
* For the purposes of this story, only a dummy alert is required

## Acceptance Criteria
* Alerts dropdown is present
* Alerts can be triggered and viewed in the dropdown
* Alerts can be marked as read
* User can follow an alert to the related page
