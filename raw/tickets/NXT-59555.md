# NXT-59555 — SC - Settings to Enable API Integration

## Description
As a GDSN user, I want to be able to see/hide the export and check buttons based upon a system setting so that I don’t try to export if I can’t.

*Notes:*

* This comes from KP and is needed for the Global Catalogue roadmap item

*Requirements:*

* In System Settings, add a new property
** Setting Name: Enable GDSN Export Button
** Code: GDSNSSOE
** Value: Yes/No
*** Default is No
** Category: Configuration
** Module: Item Management
* 
* Make changes to the Global Catalogue page (see screenshot):
* Add in a check for the property you just created on the logon button
** If yes, show both the Export Items button and the Check for Update button
** If no, don’t show the buttons

## Acceptance Criteria
Currently there is no option to restrict the usage of GDSN API Integration

Changes Needed:

Introduce a new setting as “Enable GDSN API Integration”, Default Toggle Value: No, User can change to Yes if needed

If the toggle value is *Yes, then SHOW the Export Items Button and Check for Update Buttons* in the UI 

If the toggle value is *NO, then HIDE the Export Items Button and Check for Update Buttons* in the UI

The *GDSN Items Export*  Report drop down list under Items - the dropdown value should also be hidden

This option should be accessible only if primary GDSN permission is enabled

*QA Considerations:*

Turn on the setting and try clicking the corresponding buttons in the UI

Turn Of the setting and try clicking the corresponding buttons in the UI
