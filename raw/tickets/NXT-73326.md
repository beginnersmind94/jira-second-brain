# NXT-73326 — FIN — Reverse Import: GL Accounts Import

## Description
Adds **Reverse import** to **GL Accounts Import** rows on the [Import & Export Log](https://primeroedgenext.atlassian.net/browse/NXT-73286) page.

Reverse import deletes the GL accounts an import created that have **no posted activity**. Accounts that have posted transactions are kept. **Categories are never deleted** — reversal operates on GL accounts and their non-ledger dependencies only.

## When Reverse is available

Reverse import is shown in the row menu for a completed GL Accounts Import, gated by the Reverse Import permission (hidden if the user lacks it). It appears only when the Reversal Status is **empty** (no reversal yet) or **Reverse Failed** (to retry); it is **not shown** while a reversal is **In Progress** or once it has **completed** (Reversed / Partially Reversed).

When shown, it is **disabled** (greyed, with a tooltip) if this import no longer owns any accounts — every account it created was re-imported in a later run, so their import history now points to that later import: _"No accounts available to reverse — all accounts re-imported in a later run."_ It is never allowed to run and fail.

## Reverse drawer — account summary

Opening Reverse import shows a side drawer summarizing the accounts this import created:

* Total accounts created by this import: **N**
* No transactions — will be deleted: **X**
* Has postings — will be kept: **Y**

If at least one account has no transactions, **Proceed with Reversal** is enabled. If all accounts have postings, Proceed is disabled with: _"All accounts have existing postings and cannot be deleted."_

The drawer shows no category listings, blocker cards, or unassign-accounts flow — categories are not part of reversal.

## What gets deleted

On confirm, BL deletes only the **zero-transaction** accounts this import created, and their non-ledger dependencies, in order: data point mappings, GL entry template rules, sub-account mappings, [site-level sub-accounts](https://primeroedgenext.atlassian.net/browse/NXT-68312), then the accounts. Accounts with posted transactions and their dependencies are untouched. No journal entries are deleted. **Categories are not deleted** (a category created by the import remains even if all its accounts were removed).

If every account the import created had zero transactions, all are deleted → Status = **Reversed**. If some had postings and are kept, Status = **Partially Reversed**.

## Confirmation

Mandatory comment (min 10 chars). Warning: _"{DeleteCount} accounts with no posted activity will be permanently deleted."_ — plus, when any are kept, _"{KeepCount} accounts with posted transactions will be kept."_ — and _"This cannot be undone."_ Buttons: Cancel (default) / Reverse import (destructive).

On confirm, Status → **Reversal In Progress** → **Reversed** or **Partially Reversed**. On failure, Status → **Reverse Failed** with a Retry button (re-executes without re-showing the confirmation). Reversal is atomic — if any part fails, nothing is deleted.

## After reversal

The reversal is recorded with **Initiated by, Date/Time, Comment, and Status**, shown in the View details drawer's **Reversal section** on the Import & Export Log ([NXT-73286](https://primeroedgenext.atlassian.net/browse/NXT-73286)).

Re-importing the correct file recreates the deleted accounts via create/update on Account Code (per [NXT-69421](https://primeroedgenext.atlassian.net/browse/NXT-69421)) — no duplicate conflict; kept accounts are updated in place.

Post-reversal guidance (in the View details drawer): _"To fix account details, re-import the correct file, then check Configuration for any mappings or rules that need to be set up again. To correct posted entries, use a manual journal entry. For larger changes, contact Support."_

## Config page impact

After reversal, the [GL Configurations page](https://primeroedgenext.atlassian.net/browse/NXT-64975) reflects the deletions. **System Accounts** and **Mapped GL Accounts** tabs re-surface warnings for unmapped system accounts and data points. **System Rules** that referenced deleted accounts are flagged with posting blocked until remapped. **Imported GL Accounts** tab reflects current state — deleted accounts gone, kept accounts remain. The **Account Categories** tab is unchanged (categories are not deleted).

## Acceptance Criteria
**AC1 — Reverse availability & permission.** Reverse import appears in the row menu for a completed GL Accounts Import, gated by the Reverse Import permission (hidden if the user lacks it). It is shown only when the Reversal Status is empty (no reversal yet) or Reverse Failed (to retry); it is not shown while a reversal is In Progress or once it has completed (Reversed or Partially Reversed).

**AC2 — Disabled when nothing to reverse.** When Reverse import is shown but every account this import created was re-imported in a later run (their import history now points to the later import), it is disabled (greyed) with the tooltip "No accounts available to reverse — all accounts re-imported in a later run." It is not hidden and cannot be run.

**AC3 — Account summary drawer.** Opening Reverse import shows a drawer with: Total accounts created by this import (N), No transactions — will be deleted (X), Has postings — will be kept (Y). No category listings, blocker cards, or unassign-accounts flow appear.

**AC4 — Proceed enable/disable.** If at least one account has no transactions, Proceed with Reversal is enabled. If all accounts have postings, Proceed is disabled with "All accounts have existing postings and cannot be deleted."

**AC5 — Confirmation.** On Proceed, a confirmation requires a comment (min 10 chars) and warns "{DeleteCount} accounts with no posted activity will be permanently deleted." plus, when any are kept, "{KeepCount} accounts with posted transactions will be kept." and "This cannot be undone." Buttons: Cancel (default) / Reverse import (destructive).

**AC6 — What gets deleted.** On confirm, Status → Reversal In Progress, and BL deletes only the zero-transaction accounts and their dependencies in order: data point mappings, GL entry template rules, sub-account mappings, site-level sub-accounts, then the accounts. Accounts with posted activity are untouched. No journal entries are deleted.

**AC7 — Categories are not deleted.** Reversal never deletes categories. A category created by the import remains even if all of its accounts were deleted.

**AC8 — Status after reversal.** If all accounts the import created are deleted, Status = Reversed; if some are kept (they have postings), Status = Partially Reversed. The row stays visible with comment, user, and timestamp; View details still works.

**AC9 — Atomic; Reverse Failed retry.** If any part fails, nothing is deleted; Status → Reverse Failed; a Retry button appears and re-executes without re-showing the confirmation.

**AC10 — Reversal recorded for the log drawer.** On reversal, the system captures Initiated by, Date/Time, Comment, and Status; these display in the View details drawer's Reversal section on the Import & Export Log (NXT-73286).

**AC11 — Re-import after reversal.** Re-importing the correct file recreates the deleted accounts via create/update on Account Code (per NXT-69421); no duplicate conflict; kept accounts are updated in place.

**AC12 — Post-reversal guidance.** The View details drawer shows which accounts were deleted and which were kept, with: "To fix account details, re-import the correct file, then check Configuration for any mappings or rules that need to be set up again. To correct posted entries, use a manual journal entry. For larger changes, contact Support."

**AC13 — System Accounts tab.** After reversal, if any deleted accounts were system-required, those warnings re-appear on the System Accounts tab.

**AC14 — Mapped GL Accounts tab.** After reversal, data points that referenced deleted accounts show as unmapped; their posting rules will not fire until remapped.

**AC15 — System Rules tab.** After reversal, any System Rule referencing a deleted account is flagged; posting for that rule is blocked until a valid account is assigned.

**AC16 — Imported GL Accounts & Account Categories tabs.** After reversal, deleted accounts no longer appear on the Imported GL Accounts tab; kept accounts remain. The Account Categories tab is unchanged.
