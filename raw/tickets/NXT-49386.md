# NXT-49386 — OpenID - District Authentication - Delete

## Description
As a Customer Support user, I want to be able to delete a district’s OpenID authentication provider so that they can use SSO

*Notes*:

* Covers delete for SchoolCafe authentication and OpenID
* Other methods covered in other stories

*Flow:*

* User browses to System > Security page
* User chooses which auth types they want to delete
* System deletes the authentication method if it is valid

*Requirements*:

* These requirements apply to the System > Security page > Authentication tab
* On the grid:
** Action Column update:
** Add the Delete button if not already there
** 
** One row rule:
*** Disable button if there is only one row
*** There must always be at least one authentication type
** Preferred Method rule:
*** They cannot delete the preferred method
*** First, validate that they have more than one row
*** Next, check if the row they are deleting is the Preferred method
**** If it is, then make them select another preferred method and don’t allow them to proceed, by showing this message: You are trying to delete your preferred authentication method. Please select another preferred method before deleting. 
**** Give them an OK button to click on to close the message and do not delete.
** {color:#ff5630}-SMS 2FA rule:-{color}
*** {color:#ff5630}-SMS 2FA cannot be the only preferred method, so-{color}
*** {color:#ff5630}-User cannot delete an authentication method if it would leave the SMS 2FA as the only remaining method-{color}
*** {color:#ff5630}-First, validate that they have more than one row-{color}
*** {color:#ff5630}-2nd, validate that the SMS 2FA is not the only remaining row-{color}
**** {color:#ff5630}-If it is, then make them select another preferred method and don’t allow them to proceed, by showing this message: Deleting this method would leave SMS 2FA as the only remaining method which is not allowed. Please add another method before deleting.- {color}
**** {color:#ff5630}-Give them an OK button to click on to close the message and do not delete.-{color}

## Acceptance Criteria
* User is able to delete an auth type if there is more than one
* User cannot delete the Preferred auth type
* -If user deletes SchoolCafe Authentication, then it updates all users with that auth type to use the new preferred auth type-
* -If user deletes SchoolCafe Authentication, then system notifies impacted users of the change-
* If user deletes OpenID Authentication, then it updates all users with that auth type to use the new preferred auth type
* {color:#ff5630}-If user deletes OpenID Authentication, then system notifies impacted users of the change-{color}
* {color:#ff5630}-If user deletes OpenID Authentication and the preferred auth type is SchoolCafe Authentication, then it goes through the new password process for those who don’t have one and the change password process for those who have an expired one-{color}
* Deletion is logged in history log
