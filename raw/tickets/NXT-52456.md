# NXT-52456 — Roles & Permissions - Default Permissions

## Description
We need to be able to setup default permissions for each default role for each resource so that implementation team does not have to do it themselves each time.

*Notes:*

* Currently, only SchoolCafe Admin role is working when setting up a new district
* We need to review these settings and then setup for all the other roles

*Requirements:*

* Create a new demo district in prod
** Title: SchoolCafe Template District
** Is Demo = True
* Get the district to fully onboarded status
* Add Customer Support and Tech Support roles to the district
* Copy the permissions from a production district that has already been fully setup
* Once ready, notify product team to review / update the settings
* Once Product team says they are finished:
** Ensure any new district that is created uses these settings when setup
** Apply these settings to the recently created districts in prod
*** List of districts to be provided by PO
** Apply the same permissions to all districts in all lower environments

Ensure lower environment districts also apply the same settings when created

*Acceptance criteria:*

* New districts apply the permissions using the template district settings

* All lower environments updated with the same settings
* Settings applied to list of districts supplied by PO

## Acceptance Criteria
* prod demo site is created
* default permissions are created in the demo site
* KeyCloak updates new districts with the same permissions as the demo site when they are created
