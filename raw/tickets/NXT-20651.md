# NXT-20651 — INV - Quick Receipts - Receive step - Provide message for Physical Inventory check on pageload if Transaction Date is earlier than completed Physical Inventory date

## Description
As a Site Manager I want to see the Physical Inventory check message (regarding most recently completed Physical Inventory) when the Quick Receipt receive step is loaded, so I do not waste time filling in mandatory fields if the receipt cannot be saved.

This is an enhancement for [https://primeroedgenext.atlassian.net/browse/NXT-19431|https://primeroedgenext.atlassian.net/browse/NXT-19431|smart-link] 

*Requirements*

* Show the ‘Physical Inventory has been completed’ error message on pageload if the preloaded Transaction Date is earlier than most recent physical inventory completion date
** “This Quick Receipt contains items from a previously reconciled Physical Inventory. Please remove these items or change the transaction date.”
** This will save time for the user so they don’t have to click Confirm button to see the message

## Acceptance Criteria
Show the Physical Inventory message on pageload if it applies to the Quick Receipt
