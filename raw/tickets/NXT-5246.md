# NXT-5246 — Direct Certification - Matched - Actions - History & Unmatch

## Description
*User Story*

As a district admin I want to see more information on a particular direct cert match, and have an option to unmatch or notify this student.

*Mockup*

[https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=133668%3A0&scaling=scale-down-width|https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=133668%3A0&scaling=scale-down-width|smart-link] 

*requirements*

* Add a couple of in-line options to the matched grid.
* User needs a way to see more Information on a match.
** Some kind of magnifying / analysis icon button for the grid.
** Pop-up showing more details on the DC match.
*** Student Information
**** ID #, Full Name, DOB
*** Guardian Information
**** Name, Address, Contact Info (Phones, Emails)
*** DC Information
**** Approval Type, File Number, Approval Date
*** Matching Criteria
**** Manual Entry, Extension, Automatic
***** Extension - show the details on who it was extended from and their match criteria. Address, household, guardian.
***** (FUTURE ITEM, MOVE TO ANOTHER STORY) Automatic - 
****** Show the percentage match with an option to view the full match scorecard. 
****** Full scorecard shows the matching information as of the approval date - file data vs student data, and the points that went with it.
****** This must be recorded in stone - the ‘student data’ should not be from their current record, but what their information was at the time of matching.
*** History
**** Log of who processed this match, dismissed it, when, etc.
** User should have an option to Unmatch the student.
*** Prompted for confirmation and a comment.
*** They should also have the option there to add this student to a ‘no match’ list.
**** ‘Block this student from being directly certified for the rest of the academic year.'
***** Checkbox.
***** Should block future DC matching for this student unless done manually.
**** Students who are unmatched this way should not be matched again against the *same data from a DC File Import.*
*** If confirmed, student needs to have their eligibility rolled back appropriately.
*** History tracking needs to happen for this match.
**** Add an entry indicating that this was unmatched, who did it, the comment, etc.

## Acceptance Criteria
User can analyze a particular DC match to see more information.
User can unmatch any DC match but must enter a comment.
Student eligibility data is rolled back successfully.
User has the option to block the student from being DC matched again for the school year.
System notes the matching data that was used and will prevent it from being matched again. This is mostly for File Imports.
