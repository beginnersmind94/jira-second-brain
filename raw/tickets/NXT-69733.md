# NXT-69733 — Sites Column: Clickable Site Status Drilldown

## Description
*As a* district finance user reviewing financial periods, *I want to* click on the site counts in the Sites column to see each site's POS and Inventory status, *so that I can* identify exactly which upstream periods are holding up closure for each site.

## Acceptance Criteria
# *Clickable site counts*

Given the Sites column displays counts (e.g., "8 Open, 7 Closed"), When the user clicks on either count, Then a popover opens showing the site-level breakdown.

# *Popover displays site status table*

Given the popover is open, Then it displays a table with columns: Site | POS | Inventory, And each cell shows status with visual indicator (e.g., ✓ Closed, ○ Open).

# *Site closure logic*

Given a site appears in the popover, Then that site shows as "Closed" in the aggregate count only when BOTH its POS AND Inventory periods are closed, And if either POS or Inventory is Open, the site counts toward "Open."

# *Sorting/filtering (optional)*

Given the popover is open, Then sites are sorted alphabetically by default, Or optionally filterable/sortable by status to surface open items first.
