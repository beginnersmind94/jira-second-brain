*Requirements:*

* The user can select the Area, Site, Patron, and Date.
* The date can be a day/month/date range
* A list of all transactions by selection criteria are displayed
* The list can be grouped by columns
* The headers can be sorted/filtered

## Acceptance Criteria
The user can generate the list by Area/Site/Patron/ and Date
The date is a day/month/ or date range. 
The date range is upto one year
The list has the paging and scrolling
The list can be organized by grouping headers
The list can be sorted
The list can be filtered
Headers can be added or removed
The column headers are Transaction Date/Patron ID/Patron Name/Description/Check #/Card #/Debit/Credit/Balance/Terminal/User/Comments
Totals are provided for the Debit and Credit columns
There are two versions a summary and detail.
Default view is summary
Detail include the transaction details
Transaction Details shows Description/Debit/Credit

There is a Toggle to switch between a summary and detail view of transactions. The default for the Toggle is Summary.
