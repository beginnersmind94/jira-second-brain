# NXT-37051 — IM - Create Item - Update initial flow (General Info)

## Description
As a District Administrator I need a revised item creation flow to manage the revised item type structure and facilitate the combined Servings + Pack section.

*Requirements*

* Update the Item creation flow to capture data in the updated concept of the item master 
** For initial flow, this card can be wider than the mockup represents
* Change the way we’re capturing the initial data
** Product Info will be optional, but will determine whether or not an item is “Inventory ready”
*** Note: Inventory ready is a new concept
*** We should also build a manual control for Inventory availability
** Some flags (questions) will be determined _before_ accessing the Serving/Pack grid
*** The available flags could be altered by initial questions or by Serving Pack grid
** Remove the ability to “skip” pack/serving sizes for POS info
*** We will create a special “POS only” screen for these types of “items”
* Provide a section to capture all general info
** Re-combine the “Global Info” section with the categorizations from Product Info
** Item Description is the only immediately required field
*** Continue checking for duplicates
** Tags are available and optional
** Brand, Manufacturer, Product Code - all optional
** Item Category, Storage Category, Valuation Group - conditionally optional
*** These will be optional but no item can be Inventory ready without these selections
*** Display an alert icon in the header bar “Item categorizations required for Inventory readiness”
* Capture important interactions with the Units ([NXT-35898|https://primeroedgenext.atlassian.net/browse/NXT-35898])
** If Pack Size units are provided for the Units grid, we indicate that the item is intended for Inventory
** If items are identified as being intended for Inventory, we indicate the 3 categorization fields are now required
*** Again, not required for save, but required for Inventory availability
** Proceed to Units card with Save button (create flow)
