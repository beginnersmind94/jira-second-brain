# NXT-70111 — Auto-Reprocess Failed Posting Jobs After GL Mapping Is Completed

## Description
As a district finance user, when I complete GL mapping while POS/Inventory periods were closed, I want previously failed posting jobs caused by missing mapping to rerun automatically, so I do not need to manually retry each job.

## Acceptance Criteria
* When a user saves missing GL mappings in {{Financials > Account Mapping}}, the system automatically reruns all previously failed posting jobs whose failure reason was missing mapping.

* The rerun is system-triggered (no manual retry), and each job logs the new result (success or new failure reason) in Posting Activity Log.
