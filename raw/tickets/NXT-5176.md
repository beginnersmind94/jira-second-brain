# NXT-5176 — System - Reimbursement Rates - Add Performance-Based Incentives tab

## Description
As a district administrator, I would like to be able to manage my district’s Performance Based Incentive, so that I can quickly view and update the additional rates that can be received over a particular date range.

Summary: Allow a user to be able to view the ‘Performance Based Incentive’ tab within the ‘Reimbursement Rates' section of the Program Configuration page. Allow users to view, edit, add, and delete incentives for a specified date range.

Details:

* Within the ‘Reimbursement Rates' section of the Program Configuration page, display the details of the *Performance-Based Incentives* option (rename tab from Classic to add hyphen and 's’ at the end).
* In this page, the functionality for those tabs will remain the same as Classic.
* Test Cases:
** Allow user to view existing performance-based incentive records, including Starting/Ending Month and Additional Rates.
*** Additional Rate can only be between 0.0001 and 99.9999.
*** 4 decimal places should be supported.
** Allow user to add a new record by specifying the info above (all fields required except Ending Month--Ending Month can be optional).
** Allow user to edit existing records via an edit pencil.
*** For both above points, months _can_ overlap when inserting or editing a new record. But we should not allow duplicate records (where Starting and Ending Month are the same).
**** For month fields, let’s have a *1/1/1950* minimum and a *12/31/2099* maximum. The error message can display something like ‘Year must be between 1950 and 2099’.
*** Additional Rate field should only allow numerical values and decimals.
** Allow user to hide existing records via a trash bin. We may need historical values for old reports/claim generation, so we likely need to maintain the database values.

## Acceptance Criteria
Verify that a user is able to view the 'Performance-Based Incentives' tab within the ‘Reimbursement Rates' section of the Program Configuration page

Verify that the user can view existing incentives

Verify that the user can edit an existing incentives details

Verify that the user can add a new incentive

Verify that only Starting Month and Additional Rate are required fields. 

Verify Additional Rate can only be between 0.0001 and 99.9999.

Verify Starting Month can be any date, even past, as long as it is before End Date (if End Date has data)

Verify that Additional Rate value can only be a decimal (dollar) amount

Verify that a user cannot add records where Starting Month and Ending Month values are the same. We can display an error message like, 'Duplicate start and end dates are not allowed.'

Verify that the user can hide an existing incentive
