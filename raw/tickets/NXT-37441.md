# NXT-37441 — IM - Items - Units - Link Servings to Pack Size

## Description
As a user configuring items I want the requirement to link servings and packs when both exist so that my inventory item is always linked to the MP/PD version of the item.

*Requirements*

* Continuing from [https://primeroedgenext.atlassian.net/browse/NXT-37110|https://primeroedgenext.atlassian.net/browse/NXT-37110|smart-link] 
* Emphasize Sub-Qty or Weight as required to ensure Pack Sizes and Servings are linked
** *Sub-Qty* 
*** if the Serving Size is potentially inventory-able (Amount = 1), enable the Sub-Qty for the Serving Size row
*** “Please add another unit, enter *sub-quantity*, or enter pack size weight”
**** Sub-Qty: “How many [_description of serving_] are in [_description_ _of pack unit_]?”
*** *Weight* 
*** For Pack Size units, Weight is optional, but may be used to link to a weight-based serving instead of Sub-Qty
*** “Please add another unit, enter sub-quantity, or enter pack size *weight*”
**** Weight: “Enter weight for the  [_description_ _of pack unit_]
*** +If a serving size is already added+
*** if this is a second or third pack size being added, but it’s being added as a “lower” unit (again, assuming this may be the base unit, we’ll need weight) below the existing pack size unit(s)
*** +If it’s the lowest unit in a 3-tier pack size+
***# If they indicated the item as Food and should be adding servings later, we should emphasize obtaining weight on the lowest unit
* Require linking for Inventory readiness
** We will allow users to add pack units which are unlinked from servings, but these items will be Inventory Ineligible ([Inventory Readiness|https://primeroedgenext.atlassian.net/browse/NXT-37068])
*** If servings and pack units are not linked, this prompt will replace the Confirm Base Unit step

## Acceptance Criteria
* Provide Sub-Qty on the Serving Size row in valid scenarios 
* Emphasize Weight field for Pack Size base units when Sub-Qty is not used for linking
* Trigger Inventory readiness based on proper pack-serving links
