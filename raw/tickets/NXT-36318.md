# NXT-36318 — Family Hub Payments - Monitoring Page

## Description
As a SchoolCafé admin, I want to be able to view the output of the reconciliation of Family Hub Payments so that I can proactively monitor the application and ensure higher customer satisfaction.

*Flow:*

* User logs in and browses to Support > Monitoring Dashboard > Family Hub Payments
* User views a list of payments with their reconciliation status
* Optionally, the user can reconcile a payment

*Notes:*

* This story assumes the Monitoring Dashboard has been created
* This story’s UI/UX is based off the Exports page and is following that page’s flow, layout and functionality
* We are only changing the appropriate fields as necessary, so when in doubt, copy what that page does
* This page is the user front end way of reconciling the Family Hub Payments
* There is a schedule job that also reconciles on a nightly basis, so you can get a lot of the code and logic from that if needed



*Requirements*:

* *Monitoring Dashboard Page updates:*
* Add a card for the Monitoring Dashboard page
* Card should look exactly like the other cards on the page and like those on Item Management > Reports
* Card title: Family Hub Payments
* Description: View results of the Family Hub Payments reconciliation
* On click, open the Family Hub Payments page
* 
* *Family Hub Page Requirements*:
* Create the page and link to it from the Monitoring Dashboard
* Base the UI/UX off the Exports page
* 
* Sections:
* *Header Section*
* Title: Family Hub Payment Monitor
* Navigation: System > Support > Family Hub Payments
* 
* *Filters Section*
* Create a filters section at the top of the screen like System Configuration Notification page
* Filter fields:
* Start Date
** Start date and time to filter the results
** Default to today at 1 minute after midnight
* End Date
** End date and time of when to stop
** Default to Today at 1 minute before midnight
* Exceptions Only
** A yes/ no toggle or radio button
** Default selection is no
** If toggle yes, filter the grid to show only rows with Not Matched status
* o    Reset button
** Resets the filters same as for the System Configuration Notification page
* o    Apply button
** Applies the filters same as for the System Configuration Notification page
* o    Collapse / expand arrows
** Expands and collapses the filters section same as for the System Configuration Notification page
* 
* *KPI Section*
* This section is to the right of the filters section, at the same level, above the grid
* Show a simple card with a label / value format
** Title: Exceptions
** Value: count of exceptions
* The filters in the filter column update the count when the user hits Apply
* 
* *Grid Section*
* Show a grid of the family hub payments and their reconciliation status
** Dataset: Note that it is possible that a payment exists in the Family Hub side, but not on the SchoolCafé side. Ensure that the dataset includes this payment and leave the fields from the SchoolCafé null or empty if you have to.
* Columns:
* Student ID
* Name
** Name of the student
* Payment ID
** Comes from Family Hub payment
* FH Payment
** Family Hub Payment Amount
** Comes from Family Hub payment
* SC Payment Amount
** School Café Payment Amount
** Comes from SchoolCafé
** Can be null if payment is missing
* FH Date/Time
** Comes from Family Hub payment
** Date/time that the payment was made
* SC Date/Time
** Comes from SchoolCafé
** Date/time that the payment was logged in SchoolCafé
* Reconciliation
** Matched, Not Matched
** Highlight or draw attention to it if not matched
** Leave normal text if matched
* Action
** Button
** Label: Reconcile
** On click, try to push the transaction from FH to SC again
** There should already be a stored procedure or API from the nightly reconciliation that does the same thing. Use that same code
* 
* Permissions:
* In General resources screen for Roles & Permissions
** Visible = true
** Hierarchy: put under System
** Title: Family Hub Payment Monitor
** Default permissions: all four checkboxes checked for SchoolCafé Admin
