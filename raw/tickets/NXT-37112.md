# NXT-37112 — IM - Items - Reintroduce Split View

## Description
As an Item Management user I want a split view component for Items just like I have for Recipes so I can have a cohesive user experience.

This will work together with the new Item details screen, which has a very busy header. It’s important that we are able to resize or collapse the split view and other components to manage the width.

*Requirements*

* Add the new and improved split view component from the recent Recipe Rework and apply to the new Item flow
* Set expand/collapse triggers and parameters
** collapsed mode when doing any editing on the item details
** collapsed during item creation
** collapsed if side navigation is opened
** reduce size and/or collapsed based on page sizing
* Manage the crowded data in the general Item card by creating an option to switch between item categorizations and brand info
** *If needed for saving space (width)* 
** Since the Units card takes much width, we can adjust the general card by only showing one of these groups of data at one time
** Since Recipes and Menu Items will have similarly lightweight headers (4 or fewer main attributes), we should have a corresponding Item view
** This toggle could be defaulted based on the item info
*** Example: if the item is explicitly non-Inventory, we could hide Categories by default
* Add quick toggle chips 
** These should be *independent* quick filters. By default you always see 'All' so it's not needed as a chip. If you click SIR it will filter only those which are SIR. If you then click POS, it should further filter those SIR into only the ones which are also POS items.
*** Obviously some selections are mutually exclusive but that's okay; the worst that will happen is the user will see 0 results.
** Single Item Recipes
*** We may need to use ‘SIR’ acronym or some other shorter term
** POS Items
*** If we use ‘SIR’ then we should use ‘POS’ instead of POS Item
** Food/Non-Food
*** The Food/Non-Food is an enhancement in which the chip has multiple states: _Unselected_ (both), Food (only), and Non-Food (only). It should cycle through those 3 states when clicked.
* Maintain filters from current list view page
** in expand section just like Recipes
** exclude Quick Filters (Date Range related fields)
