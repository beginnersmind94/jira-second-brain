# NXT-2106 — API and QA Production - Production Record - Strict Batching

## Description
As a site manager, who has prepared and served food to students and/or staff at my location (and possibly other locations), I use recipes that are assigned to my location based on the menu item list that I’ve been given. Some of these recipes may have strict batching included.

When a recipe has Strict Batch marked as part of its criteria, I must adhere to serving quantities that coordinate with that quantity.

If the strict batch quantity is 30, then I must produce that recipe in increments of 30, such as 30, 60, 90, 120, 150, etc…The menu item should display clarity in that regards (i.e. if the batch size is 30 portions, this should be identified easily on the production record page, and on subsequent reports - technicians worksheet, production form, production record, and production tickets.)

## Acceptance Criteria
1.) Page will display the batch size next to the menu item, if “strict batching” is part of the criteria of the recipe.

2.) Page will block user from being able to use increments that are outside of the range of the batch quantities (i.e. if batch size is 30, I should adhere to 30, 60, 90, 120, 150, etc counts… and should be prevented from planning a quantity of 45, or any other number not evenly divisible by the batch size.)

3.) A message should appear identifying why I am unable to enter numbers outside of the assigned increments (i.e. identifying the fact that it is a strict batch recipe, and needs to be in increments of…XX servings”

You’ll see the notation for this recipe is in batches of 60 servings… I’ve entered 250, which is NOT a multiple of 60, and the page blocks me from being able to save. There’s a message in a red bar indicating that strict batching is being enforced.

!image-20210615-175020.png|width=1248,height=287!

Entering 240 servings, which is a multiple of 60, removes the red message, and allows me to save the page.
