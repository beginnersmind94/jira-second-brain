# NXT-2896 — **System - Login with PrimeroEdge authentication

## Description
As a user i want to login into PrimeroEdge with my PrimeroEdge username and password so that I can perform my assigned roles, activities, and responsibilities.

Usernames and passwords are stored within the PrimeroEdge database.

Password-related settings to keep in mind are attached. For this story, please keep the highlighted items especially in mind.

## Acceptance Criteria
1. Only active users should be able to sign in
2. Username and Password are correct and user is active – authenticate user and allow access
3. Username and Password are correct and user is inactive – identify user is in system, but do not authenticate and display a message that “Your account is inactive.”
4. Password is incorrect – do not authenticate user and display message that “The password you have entered is incorrect. Please try again.”
5. Username is incorrect – do not authenticate user and display message that “The Username you have entered is not identified.”
6. After "X" login attempts (see first setting below), the user should be locked for "Y" minutes (see second setting below) and a message that says "Z" should display (see 3rd setting below) until "Y" minutes have elapsed.
