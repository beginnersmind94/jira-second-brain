# NXT-5292 — [TaggedDashboard] - Dashboard (Insights) - Eligibility - Call to Actions

## Description
As a central office eligibility user I want to see important information regarding applications that have action items such as needing processing (10 day threshold), needing notification (been 24 hrs), and/or are stuck in pending so that I am able to get students onto the program as quickly and efficiently as possible without having to go to the application list.

We should have actionable launchers that direct the user to the page that is pre-populated with filtered information. If the pre-populated functionality is too complicated for pilot, let’s split this story and handle that within R3. Message(s) alone still have value.

## Acceptance Criteria
User has an area on the dashboard that provides a call-to-action with useful information. 

Areas for calls to action:

* New Applications that need to be processed 
* New Applications that are approaching the 10 day validation threshold (>7 days)
* Pending applications that need attention.
* Processed applications with no notification sent.
* Days since the last Direct Cert import
* Number of Potential Matches on the Unmatched page

These can be small panels that show a label and the number of items that require attention.

(Discuss/split) User is able to click onto these warnings and be taken to the respective page with filters applied.

* New apps > Applications List > Filtered for New applications, sorted by Received Date descending
* Pending apps > Applications List > Filtered by status = Pending
* Notifications > Notifications > Applications > Filtered by processed date descending
