# NXT-49108 — DMF - Backend Framework

## Description
Do the backend work in order to enable the front end for the Data Migration Framework



Original Story:

As a Customer Support role person, I would like to be able to migrate data from PrimeroEdge to SchoolCafe using an intuitive dashboard so that I can more quickly resolve the issues

*Note:*

* This story covers the Migration tab and another story covers the Monitor tab

*Requirements:*

* Add a menu item under Imports & Exports on the left side
** Name: Data Migration
* Add 2 tabs to the Data Migration page
** Migration
** Monitor
* 
* In the Migration tab, do the following:
* Add a grid
** No paging, sorting, filtering
** No managed views
** Columns:
*** Database
*** Source
*** Destination
** There are four rows in the grid and these are the values for Database column:
*** System
*** Item Recipe Management
*** Eligibility
*** SchoolCafe Family Hub
** Source and Destination values:
*** The default value = Not Started
*** When one starts, show the Status of the import (Ex: Running, Failed, etc.)
**** Use the same status values as File Imports
*** Eventually, all will succeed and the status for all will be Success
*** Or, one will fail and the status for the previous will Success and it will be Failed and the remainder will be Not Started
*** When a user kicks off an import:
**** The status for it is updated to Running
**** All before it, if any, the status is left alone and should be Success
**** All after it are reset to Not Started even if they imported successfully or failed
*** If a user has successfully imported all and they go back and start again on the first one, reset the status of all after it to not started
** 
** Source and Destination columns have an action button on each row
*** The action button kicks off the migration flow for that database
*** When new, the first one will be enabled and rest disabled
*** When one starts and is running, they are all disabled
*** When one completes successfully, it is disabled
*** When one fails, it’s start button becomes enabled so the user can start up again at the same spot
*** When one fails, all previous start buttons are also enabled in case then need to start in a previous one
*** There can never be more than one start button enabled at any time
*** When a user starts an import, no matter where it is in the flow, disable all start buttons 
* 
* Rules for migration flow
** On each row, the Source action button must successfully complete before the Destination migration can start
** On the 2nd, 3rd, 4th rows, the row before it must have completed successfully before it can start
** Only one import can run at a time and only when the one before it was successful
** If the one before it completed successfully, then the next one will kick off automatically
** If one fails, then the whole process terminates
** Users can start over where they failed manually
** Users can start over at a point in time before they failed manually
* 
* Security and resources:
** Ensure that it is in the resources 
** Ensure that it is included in Keycloak, both old and new
** Ensure that security is enabled for it
** Ensure it is not viewable in the Roles & Permissions views (both of them)
** Ensure that Customer Service role has full access to this page
** Ensure that end users do not have access to this page
