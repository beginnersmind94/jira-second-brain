# NXT-13312 — Accountability - Sessions - Reconcile single or multiple sessions

## Description
As a user I want the ability to reconcile a single or multiple sessions from the Sessions list if the session is closed and does not have an over/under variance so that I can save time.

Details:

* “Reconcile” button should perform reconciliation on sessions selected by the user IF the session is closed and has no over/under variance.
* Checkboxes should only be enabled to the user IF the session is in a CLOSED status with no over/under variance. Otherwise, the checkbox will be disabled. 
* Currently checkboxes would require a user to select each individually. Create a checkbox within the first column header that could be used for selecting all, it would give a better user experience.
* Checking the header checkbox will select all enabled checkboxes.
* Within Reconciliation, if the session is in a closed status with a over/under variance of $0.00, the user should have a “Reconcile” button to perform the action.

## Acceptance Criteria
* “Reconcile” button should perform reconciliation on sessions selected by the user IF the session is closed and has no over/under variance.
* Checkbox for selection is only available for sessions with CLOSED status and no over/under variance.
* Multiple checkbox is available for the user to select all or unselect all enabled checkboxes.
* Within Reconciliation, if the session is in a closed status with a over/under variance of $0.00, the user should have a “Reconcile” button to perform the action.
