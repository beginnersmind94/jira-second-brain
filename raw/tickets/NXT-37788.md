# NXT-37788 — Payment Import – Separate Adult and Student Payment Type

## Description
As a SchoolCafé admin, I want to be able to run separately the adult and student Payment Imports so that we can run them concurrently.

*Flow:*

* Open SchoolCafé and browse to Settings > Imports & Exports > Imports
* Click on + New on the right
* Select Payment Import - Adults in the Import Type or select Payment Import – Students
* Continue the process until complete

 

*Notes:*

* We created the story and wrote the code to add an import type field where the user selects whether adult or student, but afterwards we received word that we need to be able to run adults and students at the same time, so we were told to create 2 payment imports, one for students and one for adults. These requirements will accomplish that

 

*Requirements*:

* On the existing Imports selector screen, do the following:
** Change the title of the import to “Payment Import – Students”
** Remove the toggle for Student or Adult selection
** Remove the code written in the last story that stores whether it is adult or student
** In the code that was added to send to the API/SPROC if it is an adult or student, hard code it to student
* 
* Create a new Payment Import for Adults:
** Add a new import type in the import selector called “Payment Import – Adults”
** Copy all the code from Payment Import – Adults
** Update the code to send Adult as person type instead of student to the API/SPROC
** Everything is basically the same, just Adult instead of Student
* 
* Permissions: No changes necessary

## Acceptance Criteria
* Two Payment Import Types exist, one for Students and one for Adults
* The Students one sends student info to FO
* The Adults one sends adult info to FO
