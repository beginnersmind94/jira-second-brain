# NXT-7015 — Users Phase 2 - My Profile - Changes based on new UI mock-up

## Description
*User Story*
As a PrimeroEdge User, I would like to be able to view and edit my picture, name, and basic details as well as confirm advanced details, so that I can make sure various parties in the organization always have the most accurate information about me. 

*Summary*

Allow a user to access their User Profile from their main navigation, view their basic profile information and make certain changes. *This story is based on the new Figma mock-up, which has some changes from the original User Profile story of NXT-5354.*

*Requirements*

* Add a button in Workspace or the main navigation menu to allow the logged-in user to go to their unique *My Profile* page.
* Display a basic {color:#6554c0}*view* {color}of the users information (see mock-up for layout):
** Profile Picture + Status
** Name
*** First + Last
** Email address
** Cell Phone number
** Work Phone number
*** Phone numbers should be in at least 7 digits and then automatically formatted to (xxx) xxx-xxxx
** ({color:#00b8d9}*NEW*{color}) *Remove* ‘Organization Access’ row.
** -(-{color:#00b8d9}*-NEW-*{color}-) Information about their password-
*** -Display a warning: ‘User’s password will expire in \[x] days.'-  (to be covered as an enhancement)
** Role
** ({color:#00b8d9}*NEW*{color}) Organization Level
** Theme
** Preferred Language
* -Display a sidebar/vertical navigation that the user can click to immediately scroll up/down to a particular section-
* Display an {color:#36b37e}*Edit* {color}button allows the user to update specific profile items when selected. (Email Address will not be editable because it is a primary key)
** Allow user to upload/change their profile picture
** First Name
** Middle Name (OPTIONAL field)
** Last Name
** Cell Phone number (OPTIONAL field)
** Work Phone number
* -Password-
** Main effort is being handled in a separate story (NXT-3918)
** Role (*Disabled/read-only*)
** Organization Level (*Disabled/read-only*)
** Access + pop-up  (*Disabled/read-only*)
*** When user selects number of sites, display them in a pop-up. We can display the first 10, followed by a View More button. Sites can ascend in alphabetical order.
** Theme
*** Default (default selection)
*** Dark Mode 
** Preferred Language
*** English (default selection)
*** Spanish
*** More options will be added in the future.
* Display Save and Cancel buttons
** When the user selects Save, commit the changes
** When the user selects Cancel, do not commit the changes and return to View mode

## Acceptance Criteria
Verify that a button is added in the Workspace or Navigation menu that allows the logged-in user to go to their unique My Profile page.

Verify that upon landing, the page shows a basic view of the user's information (see mock-up for layout).

Verify the fields present match the description above, especially the (NEW) changes from previous

Verify that we show when this user’s password is expiring with a warning: ‘User’s password will expire in [x] days.'

Verify that the page displays an Edit button that allows the user to update specific profile items when selected (as specified in the description).

Verify that user is allowed to upload/change their profile picture according to our standard file type/size restrictions (same as User Details screen)

Verify that Theme options include Default and Dark Mode

Verify that Preferred Language options include English or Spanish

Verify that Save and Cancel buttons are present in Edit Mode

Verify that when the user selects Save, we commit the changes

Verify that when the user selects Cancel, we do not commit the changes and return the user to View mode
