# NXT-65475 — Income Statement Pt. 1 (POS Posting)

## Description
As a district administrator,
I need to view revenue and cost-per-meal metrics for the different types of meals served by the district,
So that I can compare current period performance against prior periods, monitor year-to-date trends, and evaluate the percentage contribution of each meal type to total revenue.

## Acceptance Criteria
*Data Collection & Metrics*

* For *every program and meal type defined in the district’s configuration* (e.g., NSLP Lunch, SBP Breakfast, CACFP Supper, SFSP Snack, Adult Meals, Second Meals, Milk, or any other configured type), display the following columns:
** Revenue for *Current Period*
** Revenue for *Prior Period*
** *Current Cost-per-Meal*
** *Prior Cost-per-Meal*
** *Percentage of Total Revenue* (current period)
** *Change from Prior Period* (percentage and/or dollar change)
** *Year-to-Date Total Revenue*
** *Year-to-Date Cost-per-Meal*
** *Percentage of Meal Sales Revenue* (YTD contribution)

*Period & Scope Controls*

* User selects *Fiscal Year* and *Period (month)*.
* Report includes only *closed POS periods* within the selected financial period.
* Results reflect scope selection: *District total (default)* with option to filter to *Area(s)* or *Site(s)*.

*Data Integrity Rules*

* Report reflects only postings generated with *required GL account mappings* (Cash in Bank, Meal Sales System Accounts, etc.) configured.
* Posting dates follow POS posting rules: last day of the POS period is used as the posting date.
* If parameters return no data (e.g., no closed periods for that range), display a *“No data available”* message.

*Output Requirements*

* Display metrics in a *tabular format*, grouped first by *Program*, then by *Meal Type*, with clear labels for Current Period, Prior Period, and YTD.
* Inline calculations must show:
** *Percentage of total revenue*
** *Percentage change from prior period*
* Allow *export* to CSV and PDF, preserving program + meal type groupings.

*Functional Notes*

* Report uses the same dataset that drives the *Accountability → Revenue Report* for consistency.
* Metrics must reconcile with POS postings in the General Ledger (ensuring alignment with financial statements).
* Drill-down supported from *District totals → Program totals → Meal type → Site level* when multiple sites are included.
