# NXT-3438 — System - Notification Templates - Add Notification Templates page

## Description
As a district administrator, I would like to be able to easily manage all of the letter templates (for letters that will be going out to my parents and students) in one place, so that I can easily make changes as needed.



*Summary:* This page is a repository of notification templates to be referenced when generating notifications of various types throughout the suite. It should be divided by module and have a list of notifications and versions for each letter type (including the active template and standard out-of-the-box templates). Language selection is the last level but will be handled in another user story.

* Add page
* Divide letter types by module (and language, but just English is fine for now)
* Default module selected: Point of Service
* First selected letter template should be first letter by alpha
* Letters should display as: Active letter first, then all other letters by alpha
* View letters created for each type, for each module
* View ‘Last updated by’ information
* Basically, everything except what is in the letter editor section of the mock-up

Mock-up: [https://nqyc9v.axshare.com/start.html#p=templates&id=x5t8yr&c=1|https://nqyc9v.axshare.com/start.html#p=templates&id=x5t8yr&c=1]



*Full Details:*

* Currently in PrimeroEdge Classic, the letter templates pages exist in both the Point of Service and Student Eligibility modules, but this page will combine them into one. 
* In the _Management_ category, add a *Notification Templates* page
* When the user lands on this page, allow them to view the combined list of POS and Stud. Elig. letter templates.
* When the user selects a letter type, allow them to see the list of letters (active and inactive) accordingly. 
* This page will eventually contain the Letter Template history page as a tab (see current page in System/EditLetterTemplateHtmlVersion.aspx). Please design with that in mind. *NXT-3451*.
* HTML letters may be standard moving forward. We will come back to the ‘Convert to HTML’ functionality at a later time. (Or is it unnecessary?)
