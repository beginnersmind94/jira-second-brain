# NXT-8911 — INV - Orders - Orders List - View Orders - Search Header

## Description
As an Inventory user I want an Orders list page so I can filter and refine my list of orders to manage or open the orders I want to see.

This story is the first part of two stories for the main Orders List. This story is for the search/filter header containing dropdown selections and filters. The grid results will be handled in story [https://primeroedgenext.atlassian.net/browse/NXT-15557|https://primeroedgenext.atlassian.net/browse/NXT-15557|smart-link].

*Requirements*

* Add new page in navigation called ‘*Manage Orders*’ under the Orders category
** remove ‘Create Orders’ page if it exists
*** Create Orders will be accessed via the Create button on this page

+Search/Filter Header:+

* Site selection
** Area, Site (use control)
** Default to ‘All’ option
* Date Range search 
** From (date picker)
** To (date picker)
** Radio button to toggle between: (example below)
*** Delivery Date, OR
*** Order Date 
** Default Date Range: past 7 days to 7 days in the future
** To Date always greater than or equal to From Date
* Status dropdown - multi-selection
** Default to ‘All’ option
** Open
** Closed
** Pending Review 
** Incomplete
** Cancelled
* Vendor dropdown - multi-selection
** Default to ‘All’ option
* Reset button
* Apply button, to load/update results in grid ([NXT-15557|https://primeroedgenext.atlassian.net/browse/NXT-15557])



* UX Notes: ‘Area’ before ‘Site’

## Acceptance Criteria
* Add new ‘View Orders’ page in navigation under Orders category 
* Default date range will be one week in past to one week in future
* Multi-select dropdowns should be used for all dropdowns
* ‘All’ is default value for dropdowns
* Toggle will change whether Date Range selection applies to Order Date or Delivery Date
* Apply button will load/update grid results
