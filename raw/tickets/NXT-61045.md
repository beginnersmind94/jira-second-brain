## Acceptance Criteria
Navigation: Backend / Export Settings Table (to be created for SC)

Changes Required:

A backend job that calls the sync process for each region (a configuration table to have region ID, Export Schedule (Day), Last Run Date, NExt Run Date

On the given day of the month or week as defined, the system should call the sync API for the corresponding region

After 4 Hours of that day, the full items API should be called to fetch the updates

The fetched updates should be inserted into GDSN Connect_SC_Updates table 

Upon login, the user should be able to see the updates in the UI
