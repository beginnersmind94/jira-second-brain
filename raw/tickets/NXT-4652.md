# NXT-4652 — AcctMgt - Patron Search - Short and Blank Search

## Description
*User Story*

As a district admin I want to search for patrons with less than 3 characters, and want to see the entire patron population.

*Mockup*

N/A

*Requirements*

* Update the search pages to allow for search with the following:
** 0 characters entered, user hits the Search button or enter.
*** Result: The entire patron population loads as results.
** Less than 3 characters entered, user hits Search or Enter.
*** Result: Search results should load appropriately, but there would be no predictive results right now.
*** Example: User could enter ‘br’ and hit enter, and see results for Brook, Broom, Brush, etc.

Discussion point:

* We may want Lazy Loading to be present before we action this to avoid a very large response from the DB.

## Acceptance Criteria
User can perform a blank search and see all patrons.
User can enter less than 3 characters and see appropriate search results.
