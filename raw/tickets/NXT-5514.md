# NXT-5514 — System - Periods - Allow user to Preview and Create Periods

## Description
As a a district financials administrator, I would like to be able to specify my district's fiscal year and view or manage periods within that year, so that I can view reports and claims for the right date ranges.

+Summary+:

* Allow the user to preview periods for a variety of options and then fully create those periods with those default options.

+Details+:

* On the Generate Periods page, when a user has selected a fiscal year with a saved Fiscal Configuration and chooses their Period Configuration details, allow them to select a Preview Periods button.
* When the button is selected, display a table with period details based on their previous selections. (*No functionality required in this story--will be covered in NXT-5515. Elements below can be in place but remain disabled to QA for now if needed)*
** Period Names must match the frequency selected and ‘Include Date Range in Period Name’ checkbox.
*** If Weekly is selected, Period Name should be ‘Week \[X]’, where X is a number between 1 and the max number of periods spanning the selected year.
*** If Biweekly is selected, Period Name should be ‘Period \[X]’
*** If Semimonthly is selected, Period Name should be ‘\[Month] A’ or ‘\[Month] B’, depending on if it is the first half of a month or the second half of that month.
**** In Classic, it was shown as ‘\[Month] MM' or '\[Month] EOM’ but we can show it differently in 2.0.
*** If Monthly is selected, Period Name should be ‘\[Month] \[Year]’
** Start and End Date must match frequency for each period, for the fiscal year selected. 
** There will be an option in the next story to delete an individual period or split one. If we can, we should implement the UI for these options in this story.
** Period Types:
*** POS Periods
*** Sales Invoice Periods
*** Inventory Periods
*** Financial Periods
* Add a ‘Create Periods’ button that, when selected, allows the user to save the period details from this table to the database. 
** Only ‘Create Periods’ commits the data. ‘Preview Periods’ is simply a display for the user.
* If a user changes frequency or changes a checkbox selection, hide the grid/table so that the user knows that they must click Preview again to see new results.

## Acceptance Criteria
Verify Create Periods section can be accessed on the Generate Periods page when: Fiscal Year dates are set, Period Configuration are set, and Preview Periods button is selected.

Verify that previewed period details display in a table that matches selections from Period Configuration section.

Verify (disabled) UI elements within the table are in place for the functionality to be implemented in the next story.

Verify that user can commit the default details (based on their selections in the Period Configuration section) by clicking a 'Create Periods' button.

Verify that the periods are then saved into the database.
