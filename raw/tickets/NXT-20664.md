# NXT-20664 —  MP- Menu cycles- Search criteria Display

## Description
As a user, I should be able to do a smart search for my menu cycles without having to wait for all the menu cycle results to populate.

Would like the same functionality that is occurring on the search page in Item Managment when using the filters.

## Acceptance Criteria
When searching Menu cycles, all menu cycles disappear and regenerate a new search.
No loader is displayed (must display loader while waiting for results)
Find a way to allow smart search (seems that API call is being made everytime a new character is being entered)
All menu cycle information should already be available when user starts trying to search.
If we are displaying total number of menu cycles, we know how many menu cycles so we should be able to have all menu cycle names available.
Information for menu can be loaded once user selects a specific menu cycle. 

When searching to add a menu to a menu cycle, there will be a smart search occurring once I arrive at a 3 letter minimum in my criteria, which will continue narrowing as I add quantities of letters. In addition, there will be a display of 30 results to select from.
