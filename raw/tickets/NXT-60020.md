# NXT-60020 — TH - WS - Team

## Description
*As a* TeamHub user *I want* a visually appealing screen that provides quick access to view and contact my team members *So that* I can efficiently communicate changes with them.

h2. Comments

* Team Members = District Users in SC, so any reference to team members means the list of users from the district 
* This story is for SC, will do PE in another story

h2. User Flow

* User opens the application and is directed to login screen
* On login success, user is redirected to home screen
* User clicks Workspace and is redirected there
* User clicks on Team and is redirected here
* User is able to view team members and optionally contact them via Inbox, SMS or Email

h2. Requirements

_Login screen:_

* Add to the Login process the following:
* Add API calls to download the team member list
** Team members should be the last API called since is the least likely to change
** If doing the first login of the day, call the full team member list API
** If logging in after the first time:
*** Check that 4 hours have past since the last one
**** If no, do nothing
**** If yes, then call the Users Delta API
*** The objective is to limit the number of API calls as this data doesn’t change much



_Workspace screen:_

* Ensure the Team quick action card links to the Team screen



_Team Screen:_

* Ensure the screen supports dark mode

h4. _Header Section_

* Screen name: Team
* Back button goes back to Workspace

*_Filters Section_*

* Search textbox
** Filters team members list
** Must filter against the name, but would be nice if it can filter against the site and position as well

*_Body Section_*

* List of team members
** Be able to view a list of district users
** Data: see below
** View: 
*** Avatar
*** Name
*** Online Status (another story)
*** Site
*** Phone
*** Email
** Be able to:
*** Call using mobile devices functionality
**** On click: open the device's call dialogue box
*** Message using mobile devices SMS functionality
**** On click: open the device's SMS screen with the user’s mobile number in the To box
*** Send an Inbox message to the user
**** On click: open the app’s Compose Inbox Message screen with the user in the To section
*** Assign a task to the user (missing from screen)
**** On click: open the app’s Compose task screen with the user in the To section



h4. *Data Requirements*

* Local Storage
** There should be local storage that has the list of users with the fields necessary for this screen
** The storage should be updated when the user logs in 
* APIs:
** Get Users All
*** Called when doing a full download to get all the users for the district
*** Called on Login the first time of the day
*** Send the user id and any other details needed
*** Should return a success/failure code and the list of all active users
** Get Users Delta
*** Called on Login after the first time of the day
*** Do just a delta to save bandwidth and time
*** Send and receive same as for all users, except just since the last time
** Online Status
*** Not covered in this story, but covered in another story and will be added to this screen when done
* There should be no API calls on this screen for team members except the Status API
* All data should be pulled from the asynchronous storage on the phone for team members

h4. *Performance & Usability*

* The screen must load within 2 seconds and the layout be responsive and render correctly on: Mobile, Tablet, Desktop, Compact and expanded views
* All tap interactions must be responsive, with clear visual feedback

h4. *Error Handling*

* Use try/catch in all asynchronous calls and use error boundaries for any unexpected render failures.
* Errors must be communicated using user-friendly messages and use toast notifications, modals, or inline messages without disrupting the user experience
* Log any API or rendering errors to the error log system/database for further analysis



_Security_

* Any authenticated user can view this screen and take the actions on it

## Acceptance Criteria
h4. *Login Screen: Team Member Sync*

* On the first login of the day, the app:
** Calls the Get Users All API to download the full list of team members for the district.
** Stores the team member list in local asynchronous storage on the device.
** Ensures the Team Members API is the last API called in the login sequence.
* On subsequent logins:
** The app checks the last time the team member list was retrieved.
** If it has been less than 4 hours, the app does not call any team sync API.
** If it has been 4 hours or more, the app calls the Get Users Delta API to fetch only changed users since the last update.
** After the API call, the app updates the local storage with any deltas.
* The team data retrieved must include all required fields for the Team screen:
** Name, Site, Position, Phone, Email, etc.

h4. *Workspace Screen: Navigation*

* The Team card in the Quick Actions section:
** Must be visible to authenticated users.
** On tap, it navigates the user to the Team screen.

----

h4. *Team Screen: UI and Functionality*

_Dark Mode Support_

* The screen fully supports dark mode, including headers, text, icons, and background elements.

_Header Section_

* The screen title is "Team".
* Includes a Back button that returns the user to the Workspace screen.

_Filters Section_

* Includes a search textbox.
* Filters the team member list based on:
** Name (required)
** Site (optional)
** Position (optional)

_Body Section_

* Displays a list of district team members.
* Each team member card shows:
** Avatar
** Full Name
** Site
** Phone number
** Email address
** Online Status (placeholder for now, implemented in a separate story)
* For each team member, the user can:
** *Call*:
*** Tapping the phone icon opens the mobile device’s call dialog with the number pre-filled.
** *Message via SMS*:
*** Tapping the SMS icon opens the device's native messaging app with the phone number in the "To" field.
** *Send an Inbox Message*:
*** Tapping the message icon opens the app's Compose Message screen with the selected user pre-filled in the "To" field.
** *Assign a Task*:
*** Tapping the task icon opens the Compose Task screen with the selected user in the "To" field.

----

h4. *Data Requirements*

_Local Storage_

* The team member list is stored in local async storage and is used to render the Team screen.
* The local store is updated at login as described in the login criteria.
* Team screen reads from local storage only, except for status API, which is handled in another story.

_APIs_

* {{Get Users All}}:
** Called on first login of the day.
** Returns full user list with required metadata.
* {{Get Users Delta}}:
** Called if more than 4 hours have passed since the last full or delta sync.
** Returns only updated users since the last successful fetch.

----

h4. *Performance & Usability*

* The screen must load in under 2 seconds under normal operating conditions.
* Must render properly across all device types and layout modes: Mobile, Tablet, Desktop, Compact and expanded views
* All interactions (tap, scroll, navigation) must be smooth and responsive with visual feedback.

h4. *Error Handling*

* All async logic must be wrapped in try/catch blocks.
* UI failures must be handled using error boundaries.
* Any error encountered must be:

* No additional permissions or roles are required to view or interact with this screen.
