# NXT-36168 — Student Purchase History - Create Export - Dev

## Description
As a SchoolCafé admin, I want to be able to export the student Purchase History so that I can upload it into another system like My School Buck.

*Flow:*

* Open SchoolCafé and browse to Settings > Imports & Exports > Exports
* Click on + New on the right
* Select student Purchase History in the Export Type
* Hit Next
* Verify Source and Target fields and hit Next
* Review the last tab and make any changes
* Hit Save

 

*Notes:*

* The way it works should be like Claims Export (with comma delimiter), same flow, UI, UX
* The customer wants the header removed from the file, but we don’t support that right now; it will have to be done in a future date if they really need it

 

*Requirements*:

* Here is what we received from the customer:
** Student Purchase History (SchoolCafé to My School Buck):
** Columns: “SID”, ”LAST NAME”, ”FIRST NAME”, ”ITEM NAME”, ”ITEM PRICE’, ”SCHOOL”, ”DATE/TIME”, ”GUID” 
** Example: "654321", "JONES", "MOLLY", "SSO Breakfast - B", "0.00", "RED HIGH SCHOOL", "2021-04-08 08:18:00", "92E5618E-BEC8-47A9-9E3A-572F97151234"
* We need to map our internal fields to theirs
** Use our names for the headers, just make sure they are in the same order
* Note that they want the header not included in the file
* 
* If user selects Student Purchase History Export on the first screen:
** Show the Field Mapping - Student Purchase History tab when click next
* 
* Field Mapping - Student Purchase History tab
** Section title: Field Mapping - Student Purchase History tab
** SOURCE - SchoolCafé Fields: Our equivalent names of these fields: SID, LAST NAME, FIRST NAME, ITEM NAME, ITEM PRICE, SCHOOL, DATE/TIME, GUID
** Target - SchoolCafé Fields: same as source
** All the same functionality as the Claims Export (with comma delimiter)
** This screen must have the Action column with the edit and trash can action buttons just like the Claims Export (with comma delimiter)
** The edit and trash can buttons have the exact same functionality as with the Claims Export (with comma delimiter)
* Review - Student Purchase History tab
** Section title: Review - Student Purchase History tab
** All the same functionality as the Claims Export (with comma delimiter)
** If possible, can we default name the file StudentPurchaseHistoryExport
* 
* Permissions:
** No changes necessary

## Acceptance Criteria
* We are able to export the file successfully with all the required columns and accurate data
* Export screens have consistent UI / UX as Students Export with correct text and action buttons
