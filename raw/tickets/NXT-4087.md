# NXT-4087 — OTP - Users - SMS

## Description
As a SchoolCafe admin, I would like to be able to apply the SMS One-Time Passcode for a new User so that they are consistent with the rest of the company

*Notes*:

* Authentication Type is chosen by a user in their profile page but can also be set here by an admin when creating a new user
* A user can only choose the type of authentication enabled by the district

*Requirements*:

*Add User Screen*

* On the System > Users > Add New User drawer, do the following:
* In the grid:
** If the district has enabled this authentication method, ensure it is listed in the grid
** Columns:
** Enabled
*** Checkbox
*** Default Yes for each new row unless:
*** If district has specified that one method is required:
**** Check that row
**** Uncheck and disable all others
** Preferred
*** Checkbox
*** Default to the district’s setting for this type
*** If district has specified that one method is required:
**** Check that row
**** Uncheck and disable all others
** Method
*** Text: SMS One-Time Passcode 
** Description
*** Show the description from the District’s authentication type grid
* Ensure changes saved to History log and visible on history tab



*Edit User Screen > View Mode*

* On the System > Users > Edit User drawer, do the following:
* When in View mode:
** Ensure that this authentication method shows in the list if it is enabled by the district and the user
** If this method is not configured by the user but is enabled by the district, show this info message:
*** SMS One-Time Passcode authentication not configured yet by user. Notify?
** Notify is a button
*** Tooltip: Send user message to configure SMS One-Time Passcode 
*** On click, it kicks off the Update SMS One-Time Passcode process
**** Covered in separate story





*Edit User Screen > Edit Mode*

* On the System > Users > Edit User drawer, do the following:
* When in Edit mode:
** Ensure that this authentication method shows in the grid if it is enabled by the district
*** Doesn’t matter if enabled or not by user
** Default all properties to how they are configured
** Ensure existing requirements for Preferred and Enabled columns apply to this method also
** Action column:
*** If Configuration column = No, show the Notify button:
**** Tooltip: Send user message to configure SMS One-Time Passcode 
**** On click, it kicks off the Update SMS One-Time Passcode  process
***** Covered in separate story
*** {color:#ff5630}Edit button:{color}
**** {color:#ff5630}On click, open the Edit SMS Drawer{color}
**** Edit rights to the page are required to be able to click this button, so disable if they don’t have it
** Save button
*** Ensure validations and save requirements work for this auth method also
*** Ensure changes saved to History log and visible on history tab

## Acceptance Criteria
* This method added to Add New User screen
* Existing rules about enabled and preferred columns respected
* On Edit Users View mode, this method is in the list if enabled by district and user
* On Edit Users View mode, admin can send the user a message asking them to configure this method
* On Edit Users Edit mode, this method is in the grid if enabled by district
* On Edit Users Edit mode, admin can send the user a message asking them to configure this method
* On Edit Users Edit mode, existing rules about enabled and preferred columns respected
* On Edit Users Edit mode, existing rules about validation respected
* On Edit Users Edit mode, admin can set their phone number for them
