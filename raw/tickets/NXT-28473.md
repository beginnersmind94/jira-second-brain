# NXT-28473 — INV - Receipts - Create Receipt - Reduce QOH for Commodity Contract Items - API Only

## Description
As a FSD I want the system to reduce *Available Quantity* when I receive an order for Commodity Contract items, so that the vendor’s commodity quantity on hand is up-to-date. 

*Requirements*

* Create trigger upon Site receipt for *Commodity Contract Items*
* Reduce Available Quantity at the Commodity Contract according to the quantity received
** hit the API to reduce quantity
* Do not limit the receipt to only the available quantity
** this could cause the warehouse to go negative quantity
** this may be a setting in the future, to allow or disallow receiving more than available quantity
