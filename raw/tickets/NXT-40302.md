# NXT-40302 — Email – Framework - Spike

## Description
Investigate how to do the Email Framework.

Here is the original Email Framework Requirements:



We need to create an Email Framework so that all modules have a consistent way of distributing documents via email.

*Notes:*

* We need this for Data Query, eOrder and several other places
* No front end for this story
* There are conceptually 2 parts to the framework:
** Configuration
** Execution
* Users will either send a message on the fly or use a configuration to execute sending an email

*Flow:*

* Source page needs to send something via email
* It inserts a record into the email framework
* Email framework gets the configuration needed to send the file
* Email framework sends the file and notifies the source that it is complete

*Configuration Requirements*:

* This needs to be framework style, meaning that it can be used by multiple modules and features
* There needs to be a way to save an Email Configuration
* Store in a database table the following configuration at a minimum:
** Config ID
** Source
** Source ID
** From
** SchoolCafé Users
*** List of user accounts in SchoolCafé
** Roles
*** List of roles from the Roles and Permissions feature
** External Users
*** List of email addresses
** Subject
** Body
**  Attachments
** Date / Time Created
** Status
*** Values: Sent, Pending
* There needs to be a history table for changes to the configuration

 

*Execution Requirements*:

* Flow:
** Source tells email framework it wants to send a file
** Source identifies if it wants to use a configuration or if it wants to provide the details
*** If a configuration, then the framework grabs those details
*** If not a configuration, then the framework uses the details provided
** Framework consolidates SchoolCafé users and users from roles ensuring no duplication and gets email addresses and combines to external email addresses list
** Framework then sends the file
** Framework records in a history table what it did
* 
* There needs to be a way to tell the framework it needs to execute an email by a scheduled job
* There needs to be a way to tell the framework it needs to execute an email at runtime (send now)
* 
* There needs to be a way to tell the framework to use a certain email configuration to send the file
* There needs to be a way to provide the framework the necessary values instead of using a configuration file
* 
* The framework needs to be able to take in a list of roles and pull out the user accounts that apply to them
* The framework needs to be able to take in a list of user accounts and get the emails from it
* The framework needs to be able to take in a list of email addresses (should be external accounts, but they might add email addresses of SchoolCafé users)
* The framework needs to be able to consolidate the users list, users from roles and external email addresses and ensure that it does not send out duplicate emails
* 
* All attempts to email files must be documented in a History table regardless of success
* The framework needs to save the values used in a history table so that it can be reviewed later
** Save the values passed in such as if they select Cashier role and also save the actual list of email accounts emailed to
** Be able to track which users were passed in as SchoolCafé users, roles or external
* 
* There needs to be a way to save files somewhere and link them to the execution
* 
* The framework needs to be able to notify the source of the completion status if done via Runtime
* 
* Modifications to the email configuration need to be saved into the History table
