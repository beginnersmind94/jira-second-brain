# NXT-64496 — SCTV+ - Implement Backend Heartbeat Monitoring with Reconnection Triggers for Stale Devices

## Description
*As a* backend development team
*I want* to implement heartbeat monitoring that responds to the existing frontend heartbeat emissions
*So that* I can proactively trigger reconnections for devices that may have stale connections

*Context*

The frontend React Native app is already emitting heartbeats every 20 minutes. This story implements the backend response system to maintain connection health and trigger reconnections when needed.

## Acceptance Criteria
h3. Backend Implementation Requirements

* *Listen for Existing Heartbeats*: Backend receives {{HEARTBEAT}} events already being emitted by frontend every 20 minutes with payload:
** {{deviceId}}
** {{timestamp}} (ISO string)
** Optional metadata ({{appVersion}}, {{platform}})
* *Track Device Status*: Update {{lastSeenAt}} timestamp for each device upon receiving heartbeat
* *Background Monitoring*: Run periodic checks (every few minutes) to identify devices with stale connections
* *Maintain Online Status*: Do NOT mark devices as offline when sending reconnection triggers

h3. Technical Specifications

* *Expected Heartbeat Frequency*: Every 20 minutes (already implemented in frontend)
* *Trigger Threshold*: 25 minutes without heartbeat
* *Response Event*: (clarified that this comes from the device)

Note: ‘*Send Reconnection Triggers*: When a device hasn't sent heartbeat in *25 minutes*, send {{RECONNECT}} event to that device’ moved to another story
