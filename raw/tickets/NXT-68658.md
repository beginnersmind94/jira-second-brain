# NXT-68658 — FIN - Implement Financial Periods Management (Generate + Manage Tabs)

## Description
Districts need Financial periods to configure the start and end dates for their fiscal year, track monthly financial data, control manual entry backdating, and identify when source data has changed. 

Mockup attached here: [^Financial_Periods_Mockup.html]

## Acceptance Criteria
*Location:* Financials > Configuration > Periods

h3. Fiscal Year

* User configures Fiscal Year with *Start Date* and *End Date*
* System auto-generates periods based on selected frequency:
** Weekly (ends on selected weekday)
** Biweekly (ends on selected weekday)
** Semi-monthly (ends on 14th/15th/16th and last day)
** Monthly (ends last day of month)
* Option to include date range in period name
* No gaps: new FY start = prior FY end + 1 day
* Lock FY dates as soon as user sets and saves them, same for frequency
* User can split, edit, add, or delete periods after generation (no overlaps allowed)

h3. Period States

* *Open* – manual entries allowed, data flowing in
* *Closed* – manual entries blocked
** POS/Inventory re-closed since; show warning badge with source + timestamp
** User can reopen that period, make adjustments and close it
* Reversals can happen at any time and will post on the date the action was performed

h3. Data Flow

* When POS/Inventory closes, entries post to Financial period containing the close date
* POS/Inventory data always flows in regardless of period state
* If period was Closed → mark Dirty, notify user
* Reversal + new entry both dated to last day of original period (entered date = today)

h3. Automatic Entry Adjustments (Setting)

* Default: *OFF*
* When ON: auto-reverse and repost when source data changes for closed periods
* User controls whether this happens automatically or manually
* History tracking of changes

h3. Closed Fiscal Year (moved to separate story)

* Incoming retroactive data creates reversal + new entry in current period
* Flagged as prior period adjustment

h3. Manual Entries

* Allowed in Open periods only and once financial periods are configured
* Blocked in Closed periods and Closed fiscal years

h3. Period Actions

* Close / Reopen
* Close All / Reopen All
* Split, Edit, Delete
* View History (drawer)

h3. Manage Periods Table Columns

* Period (name + date range)
* Sites (POS & Inventory) – e.g., "8 Open, 7 Closed" (both POS AND Inventory must be open/closed)
* Data Changed (warning if source changed since close)
* Status
* Actions

h3. History Drawer

* Status changes with timestamp + user
* Source data changes for Dirty periods (what changed, when, from which source)
