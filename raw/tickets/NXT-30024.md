# NXT-30024 — INV - Vendors - Items - Enable 'Current Price' field in Edit mode if item has no price - UI Only

## Description
As a District Admin I want the price fields to be editable in the side panel if the item does not have a price, so that I don’t need to use the Price Conditions slideout to enter the first price.

Since we now allow users to add items to a contract without specifying the mandatory attributes, users would fill in that info in the items list. However, based on the prior logic, we are requiring all price fields to be managed in the Price Conditions slideout. We should change this to allow users to use the side panel for the first time price fields are being entered.

*Requirements*

* Allow all price fields to be entered in the slideout in Edit mode if no values exist yet
** this scenario only occurs if the multi-add method is used
** this is the same logic applied when adding items the first time
** all price fields should be enabled/disabled as a group
*** Current Price
*** Delivery Fee
*** Processing Fee
*** Net Off Invoice
* Determine appropriate ‘Effective Date’ value for date range when price is added after being added to the contract
** Use the Contract Start Date, so there will always be a price for any date for which the item is orderable
* Require the Price Conditions slide-out to be used once a price exists
** disable the ability to edit in the side panel
