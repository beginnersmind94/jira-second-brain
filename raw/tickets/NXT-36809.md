# NXT-36809 — Payment Import - Add Person Type to Pre-Upload Screen

## Description
As a SchoolCafé admin, I want to be able define if a person is an adult or student in the Payment Import screen so that database can put it into the right bucket.

*Flow:*

* Open SchoolCafé and browse to Settings > Imports & Exports > Imports
* Click on + New on the right
* Select Payment Imports in the Import Type
* Select Student or Adult in the Person Type
* Continue the process until complete

 

*Notes:*

* This story comes from Front Office team. They need us to update the way we are sending them this data after they moved from Couch to SQL. See linked story.

 

*Requirements*:

* On the Pre-Upload screen of the Add New Import page, do the following:
** Add a radio button under the Import Type
* Values: Student, Adult
* Use the existing Balance Import UI and copy that exactly
* Save this value and use it later
* 
* Update your code that sends the import information to the FO database table to include this Person Type value
* 
* Permissions: No changes necessary

## Acceptance Criteria
* The radio button is added and looks the same as Balance Import
* The value is passed on to FO team as they need it
