# NXT-71514 — SC - PBIRD - Delete

## Description
As a Director, I want to delete a Power BI report configuration so that I can remove reports that are no longer needed.

----

h2. *Description*

Allow users to delete a Power BI report from the Power BI Reports dashboard.

The delete behavior must follow the same *business rules and safeguards* as Managed View Reports to prevent deletion of reports that are actively in use.

----

h2. *User Flow*

# User navigates to:
#* *Reports → Power BI Reports*
# User clicks the *ellipsis (⋮)* on a report row
# User selects *Delete*
# System displays confirmation dialog
# User selects:
#* *Yes* → System attempts to delete report
#* *No* → Dialog closes, no action taken
# If allowed:
#* Report is deleted
#* User remains on dashboard
# If not allowed:
#* System displays appropriate message

----

h2. *Out of Scope*

* Bulk delete
* Workspace Dashboard updates _(covered in separate story)_
* Mobile app behavior

----

h2. *Requirements*

h3. *Delete Access*

* *Delete* is available as an option in the row ellipsis menu _(covered in separate story)_

----

h3. *Permissions*

* Users must have *Delete permission* to perform this action
* If the user does NOT have Delete permission:
** Delete option is visible but *disabled (grayed out)*
* Exception:
** *Customer Support* and *Technical Support* roles can always perform delete

----

h3. *Confirmation Dialog*

* When user selects *Delete*, display confirmation dialog
* Dialog should clearly indicate that the report will be removed
* Must include:
** *Yes*
** *No*

----

h3. *Deletion Business Logic (Critical)*

* Deletion must follow the same business rules as *Managed View Reports*
* The system must *prevent deletion* of reports that are actively in use, including (but not limited to):
** Reports with active *distribution schedules*
** Reports with configured *recipients or distribution settings*
* Users must:
** Remove or disable these configurations before deletion is allowed

----

h3. *User Feedback*

* If deletion is blocked:
** Display message explaining why deletion is not allowed
** Guide user to remove dependencies before retrying

----

h3. *Successful Deletion*

* If deletion is allowed:
** Remove the report configuration from the system
** Remove it from the dashboard grid immediately

----

h3. *Business Logic Reuse*

* All deletion-related behavior must align with Managed View Reports, including:
** Validation rules
** Dependency checks
** Any notifications triggered as part of deletion

----

h3. *Audit Logging*

* Deletion must be recorded in the audit log:
** Include user and timestamp
** Available in History _(if applicable prior to deletion)_

## Acceptance Criteria
* Given a user selects Delete, when the confirmation dialog appears, then the user can confirm or cancel the action
* Given the user selects No, when the dialog closes, then no deletion occurs
* Given the user selects Yes, when the report is not in use, then the report is deleted successfully and removed from the dashboard
* Given the report has active distribution or recipients, when the user attempts to delete, then the system prevents deletion and displays an appropriate message
* Given the report is deleted, when the action completes, then the deletion is recorded in the audit log
