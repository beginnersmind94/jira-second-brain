# NXT-63852 — SCTV+ - Fix Serving Line Data Mismatch between TV and Family Hub

## Description
*As a* backend developer maintaining the SCTV+ API
*I need* to ensure serving line filtering endpoints return the same menu data as the [http://schoolcafe.com|http://schoolcafe.com|smart-link]  family hub
*So that* frontend applications can display consistent menu information across all platforms

*Root Cause*: API endpoint mismatch between SCTV+ (.tv) and [http://schoolcafe.com|http://schoolcafe.com|smart-link]  (.com)

For example, in the image below, on selecting the ‘Lunch’ serving line we see no options but [schoolcafe.com|http://schoolcafe.com] shows the full menu on selecting the serving line. 

!image-20250708-183445.png|width=1536,height=864,alt="image-20250708-183445.png"!

## Acceptance Criteria
* Serving line filtering returns actual menu data instead of "No Options"
* SCTV+ Designer shows the same menu items as [http://schoolcafe.com|http://schoolcafe.com|smart-link]  for the same serving line
* Both .tv and .com API endpoints pull from the same data source
