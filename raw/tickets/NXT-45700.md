## Acceptance Criteria
Navigation: Insights → Front Office → KPI Reports

The KPI report - FO should be loaded based on the API data (currently using other APIs from AI/ML)

Scope - Site Level and Site Type level only

Input Parameters: followed as-is , however the data should be extracted based on Command Centre API source when user clicks on Apply

!image-20231011-214824.png|width=1581,height=364!

The UI should have following parameters:

# District - Disabled view and shows the current region as logged on
# Site - Specific Site or Site Type (Include Regression for - All Site Type as well)
# KPI - All as default option ( +_include ADP Breakfast and ADP Lunch as additional parameters), user can select custom KPI as needed_+
# Report Type - Monthly and Weekly
# Date options (Monthly) - Academic Year to Date, Year to Date and Custom Range (Regression)
# Date Options (Weekly) - Current Month, Last Month, Custom Date Range (Change)
# Date Range - Disabled view should populate the date ranges as applicable

The system should return the matching data with command center on this KPI page 

UI Loader option should be available to user to know the page is still loading

Threshold settings - MEQ and Revenue should reflect the threshold value from Command center (additional component development) - applicable for district level, site type and site level

All the threshold color should be in alignment with the range (Ex: Green)

Threshold Colors:

* Benchmark column should reflect green as default pointing to the defined range (against corresponding district level, site level or site type level) 

The format of the report stay as-is
