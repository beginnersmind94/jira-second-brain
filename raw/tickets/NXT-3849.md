# NXT-3849 — AcctMgt - Homerooms - Details - Edit/Delete/Add/Validations

## Description
*User Story*

As a district admin I want to edit homerooms and their contents.

*Mockup*

[https://nqyc9v.axshare.com/start.html#p=homeroom&id=spdi2j&c=1|https://nqyc9v.axshare.com/start.html#p=homeroom&id=spdi2j&c=1]

*Requirements*

* On the Homeroom Information panel…
** Add an Edit button in the corner.
*** Clicking will load the side-drawer where some information can be edited and saved.
*** Editable:
**** Homeroom Name
**** Homeroom Description (optional)
***** Character count ‘X/250’ also available and counts as the user types.
**** Grade(s) (Added 8/21/20)
***** Multi-select possible. Must be selected from grades available at the site.
**** Site the homeroom is assigned to.
*** Save and Cancel buttons on the same line.
*** Add a validation check when saving - the Homeroom Name must be unique per the site that it’s at.
**** Example: 'Brown A' can only exist 1 time at Site A. 
**** It can also exist at Site B, and would be a separate homeroom. 
**** Check with Sravani Netturu (Tech Writer) for appropriate error messaging.
** Add a Delete action button next to each student in the table.
*** This will remove the student from the homeroom.
*** This will require a confirmation prompt.
**** 'Are you sure you want to remove FirstName LastName (ID #12345) from Homeroom Brown B? 
**** Cancel / Confirm buttons.
** Add a way to deactivate a homeroom.
*** Active/Inactive toggle.
*** Confirmation prompt for the user.
**** ‘Are you sure you want to inactivate this homeroom?’
*** If students are still present in the homeroom, the prompt should also ask the user if they want to move students to another homeroom.

## Acceptance Criteria
Edits available for name, description and grade(s) available.
Duplicate homeroom names at the same site are not possible.
Students can be added / removed.
Homeroom can be deactivated / activated.
  User is prompted to move students to another homeroom if they want to.
Multi-language support added.
