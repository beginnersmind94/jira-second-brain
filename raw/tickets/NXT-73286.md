# NXT-73286 — FIN — Import & Export Log

## Description
As a district finance user, I want to see — without leaving Financials — a log of every import, export, and report download I have run, with the same monitoring detail System's Import Monitor provides, so that I can review activity, open run logs, check exceptions, and download reports from one place.

---

## TL;DR

New page under **Financials > Configuration**: **Import & Export Log**. Three tabs — **Imports | Exports | Reports** — each with its own grid and filter bar; switching tabs swaps the content and resets filters to that tab's defaults. Import rows pull from Import Monitor. Report rows pull from Document Center and Power BI export events. The Exports tab is present from day one but shows an empty state until configured exports are built.

This story builds the **log page infrastructure only**. Reverse import is added by linked stories per import type:

* **NXT-73326** — GL Accounts Import
* **NXT-73303** — Opening Balance Import
* **NXT-73304** — Payroll Import

**Why this matters:** Today, GL Accounts Import redirects the user to System and leaves them there (NXT-69421). Report downloads land in Document Center or are exported from Power BI. Finance users have no centralized view within Financials. This page eliminates the module hop.

---

## Scope

**In scope:** Import & Export Log page under Financials > Configuration. Three tabs (Imports / Exports / Reports), each with its own grid and filters. Imports and Reports populated; Exports an empty-state placeholder. View details side drawer (Imports) with summary stats, a deep-link to System, and a Reversal section. Download on report rows. Page header with deep-links to System Import Monitor and Export Monitor. Filterable, searchable, sortable, paginated.

**Out of scope:** Reverse import behavior (see linked stories — this page only displays reversal status and the drawer's Reversal section). Creating new imports, exports, or reports from this page. Changes to System Import/Export Monitor or Document Center.

## Requirements

### 1) Page and Navigation

1.1 Add to Financials > Configuration: **Import & Export Log**.

1.2 **Page header copy:**

**Import & Export Log**

_All your Financials imports and exports in one place. View a run's summary, download a report, or reverse an import. For full run logs, open \[Import Monitor\]({deep-link, filtered to module = Financials, new tab}) or \[Export Monitor\]({deep-link, Financials report types pre-selected, new tab})._

1.3 Roles with access: Director, Central Office, SchoolCafe Admin. Customer/Technical Support always full access. Finance Coordinator: read-only.

### 2) Tabs

2.1 Three tabs — **Imports | Exports | Reports** — replace the prior single grid and Category filter. Each tab has its own grid and filter bar. Switching tabs swaps the visible content and resets that tab's filters to default. There is no combined "All" view and no Category column — the tab is the category.

2.2 **Data sources.**

* Imports tab: from Import Monitor, filtered to Financials import types. Currently only **GL Accounts Import** exists; additional Financials import types appear automatically as they are configured.
* Reports tab: report downloads. Document Center reports (P&L, Trial Balance, Balance Sheet) appear when generated. Power BI reports (GL Report, Journal Report) appear when the user clicks **Export** in Power BI (not Generate), in the format the user selected.
* Exports tab: configured export jobs. None exist yet — empty-state placeholder (see 2.10).

#### Imports tab

2.3 Columns: Name, Type (Import Type name, e.g. "GL Accounts Import"), Date, User, **Import Status**, **Reversal Status**, Actions. Default sort: Date descending.

2.4 **Two status columns.**

* **Import Status** — read-only badge, always prefixed "Import ": Import Queued, Import In Progress, Import Completed, Import Error.
* **Reversal Status** — blank when no reversal has been initiated (blank, not "N/A"). Once a reversal runs: Reversal In Progress, Reversed, Partially Reversed, Reverse Failed. Reverse Failed shows a Retry action. (Reversal Status values are produced by the reverse-import stories — NXT-73326 / NXT-73303 / NXT-73304.)

2.5 Filters (Imports tab): Type, Date Range, User, Import Status, Reversal Status — the two status filters each offer only their own values. Free-text search across Name, Type, User. Columns sortable; standard pagination.

#### Reports tab

2.6 Columns: **Type** (Report name, e.g. "P&L Report", "GL Report"), Started By, Date / Time, Status, Download. No Name column. View only — no reverse action. Same structure as the Document Center report list. Default sort: Date descending.

2.7 Status (report-specific): Queued, Generating, Ready, Failed. The Download control is active only when Status = Ready.

2.8 Report Types: GL Report, Journal Report, P&L Report, Trial Balance, Balance Sheet.

2.9 Filters (Reports tab): Type, Date Range, Started By, Status. Search across Type and Started By. Sortable; standard pagination.

#### Exports tab

2.10 Placeholder tab — no table, no filters. Empty state only: "No exports have been configured yet. Exports will appear here once they are set up." The tab scaffolding must exist for the upcoming configured-exports story.

### 3) Actions and Drawers

3.1 **Imports tab — View details** (kebab). Opens a side drawer:

* **Title:** Run details — {ImportName}
* **Summary stats** (from Import Monitor): Total Records, Added, Updated, Processed, Exceptions, Status, Start, End.
* **Deep-link:** "Open full log in System ↗" — opens Import Monitor detail view in a new tab. Tooltip: "Opens the full import log in System > Import Monitor." Hidden if the user lacks System access. Disabled while running or no log data.
* **Reversal section (shown only when a reversal has occurred):** Initiated by, Date/Time, Comment, Status. Lives only in this drawer — a reversal never appears as its own row in the grid. (Reversal data is captured by the reverse-import stories.)

3.2 **Reports tab — Download.** Re-downloads the file; active only when Status = Ready. Document Center reports from Document Center; Power BI reports in the format selected at export time. No View details, no Reverse.

3.3 **Exports tab —** no actions (placeholder).

### 4) Permissions

4.1 View page: Director, Central Office, SchoolCafe Admin, Finance Coordinator. Customer/Technical Support full access.

4.2 Finance Coordinator: read-only (can view and download; cannot reverse once reverse stories land).

**AC4 — Import Status column.** Import Status is a read-only badge, always prefixed "Import ": Import Queued, Import In Progress, Import Completed, Import Error.

These values are produced by the reverse-import stories (NXT-73326 / NXT-73303 / NXT-73304).

**AC6 — Imports tab filters.** Type, Date Range, User, Import Status, and Reversal Status (each status filter offers only its own values), plus free-text search across Name, Type, and User. Columns sortable; standard pagination.

**AC7 — Reports tab.** The Reports tab lists report downloads. Columns: Type, Started By, Date / Time, Status, Download. No Name column. View only — no reverse action. Same structure as the Document Center report list.

**AC10 — Exports tab empty state.** The Exports tab exists but shows no table or filters — only the empty state: "No exports have been configured yet. Exports will appear here once they are set up."

**AC11 — Page header deep-links.** The page header includes inline links to Import Monitor (pre-filtered to module = Financials, new tab) and Export Monitor (pre-filtered to Financials report types, new tab).

It appears only when a reversal exists, and only inside the drawer — never as a row in the grid. Reversal data is captured by the reverse-import stories.

**AC14 — Permissions.** View page: Director, Central Office, SchoolCafe Admin, Finance Coordinator. Customer/Technical Support full access. Finance Coordinator read-only.

**AC15 — Empty states.** Each tab shows an empty state when it has no rows; the Exports tab shows the configured-exports message in AC10.
