# NXT-8182 — INV - Physical Inventory - Perform transaction checks on Physical Inventory

## Description
As an operations manager I want the system to check for any open transactions when I am attempting to complete a Physical Inventory, so that I do not save an inventory which may be inaccurate.

*Requirements*

* Physical Inventory screen will check these transactions before allowing the user to save as complete
** open orders
*** this validation will depend on a setting that specifies whether the validation for open orders applies to the current date or the prior date (day before)
*** any order status other than Closed or Cancelled
** pending review orders
** incomplete transfers
** incomplete receipts
*** this likely does not apply since there is no such thing as an “incomplete” regular receipt
** incomplete quick receipts
** -incomplete production orders-
*** this will be done once Production Orders are implemented
* Logic for which date to use:
** For Period inventories for the former period, use the period end date
** For Period inventories in the current period, use the current date
** For Interim inventories, use the current date
* On both Summary and Items tab, move the buttons to be right-aligned
** also unbind the buttons from the “Action” box and get rid of the action box
* Display a banner on the Summary tab to the left of the buttons (similar to Order/Receipt screens)
** Display banner message: “Some transactions may be preventing this inventory from being completed. \[View icon]”
** Open a slide-out when the user clicks the view icon
* Display a slide-out drawer containing the list of all transactions which would block inventory completion
** At the top of the slide-out display a *blue* banner type message: “The following transactions may be preventing this inventory from being completed. Please close all incomplete transactions which are dated on or before 8/31/2021.”
** In a two column grid, show a list of all the blocking transactions that the user can view any time during the inventory
*** First column is the description. Ex: “Open Order #12345 for 8/11/2021”
**** Some descriptions may be longer, so ensure there is adequate width for this column
**** Another example: “Plan Incomplete, Meal Type: Breakfast, Menu Line: Harrison -Pizzabar for the Serving Date: 06/22/2015”
*** Second column is a simple column containing a View icon for each transaction
**** the view icon will open the specific transaction in a new tab
* Automatically open the slide-out drawer if a user clicks ‘Save as Complete’ button while blocking transactions still exist.

## Acceptance Criteria
* Check for all transactions for the site that are dated on or before the inventory close date
* Provide a message if transactions are blocking the inventory completion
* List the blocking transactions so user can find and close them
* Prevent the user from completing the Physical Inventory if transactions are blocking
