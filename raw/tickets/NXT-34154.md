# NXT-34154 — Team Hub - Update on Logout

## Description
As a school admin, I want to ensure pending messages are sent so that my coworkers will get my messages.

 

*Flow:*

·         After logged in, Admin clicks on Settings button

·         Click Logout

 

*Background:*

·         Logout has already been done

·         We are updating it to include sending pending outbound messages

 

*Requirements*:

·         On Settings screen:

·         Fields: No new fields

·         Update business logic on Logout button to include:

o   Execute an incremental pull / push of messages to force the app to clear out any pending outbound messages

o   Upon successful completion of push / pull, then go on and finish logout

o   If error, skip and continue with logout

## Acceptance Criteria
·         On logout, the app runs incremental push / pull of messages and then completes logout
