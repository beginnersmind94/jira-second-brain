# NXT-48078 — Imports – Subscriptions

## Description
As a SchoolCafé admin, I would like to subscribe to notifications of imports so that I can be informed if there is an issue with the import.

*Note:*

* This story is leveraging the notification concepts that have already been developed for Download and Validate, so leverage this existing code
* The use case is that the user wants to be able to optionally subscribe to events such as failed imports or imports with exceptions and be notified when they occur
* This story covers the notifications for all import types for both Files and API

*Flow:*

* Browse to API or Files
* Click on Notifications for one of the rows
* Notifications drawer opens
* Optionally select users or roles to be notified
* Optionally select under which scenario to be notified (status, exceptions, etc.)
* Optionally see which users & roles are notified
* Optionally edit the notification configuration
* Optionally delete the notification 
* Save
* Later, when the import is run, if one of the criterion are met, the System triggers the notification process
* System notifies by sending message to inbox and notification update
* User browses to inbox and reviews the notification

## Acceptance Criteria
* User can click the button to start the subscription process
* User is informed of the process and can confirm
* User is able to add, edit and delete subscriptions
* User is able to add and delete people from the subscription
* User can define subscription criteria for status and exceptions
* Subscriptions can run immediately or on schedule
* User is notified in their inbox when the subscription event has taken place
