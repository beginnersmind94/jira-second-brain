# NXT-67290 — Publish React Vendor Files to Azure Cloud and Serve via FrontDoor

## Description
h3. *Description / Objective:*

Currently, during the React app build, all vendor JS files are copied to the {{dist}} folder and stored within the pod. When the page loads, these files are served from the pod, causing latency and slower page load times.

The goal of this ticket is to *publish all vendor files to Azure Cloud Storage* and serve them via *Azure FrontDoor*, removing dependency on pod storage and improving performance.

Additionally, the DevOps team will handle portions of the publishing process in the CI/CD pipeline.

----

h3.

## Acceptance Criteria
h3. *Acceptance Criteria:*

# *Metrics Collection – Before Implementation:*
#* Measure and document the current download time and latency of vendor files served from pods.
# *Implementation:*
#* Modify the React build process to publish all vendor files to *Azure Cloud Storage*.
#* Update the application logic to load vendor files via *Azure FrontDoor* instead of from the pod.
#* Ensure integration with the CI/CD pipeline for automated publishing.
# *Metrics Collection – After Implementation:*
#* Measure and document the download time and latency of vendor files served via Azure FrontDoor.
#* Compare metrics with pre-implementation values to confirm performance improvement.
# *Functional Verification:*
#* Verify that all pages of the application load correctly without errors.
#* Ensure no functionality or assets are broken due to the change in vendor file location.

----

h3. *Additional Notes:*

* Coordinate with the DevOps team for the pipeline changes related to vendor file publishing.
* Ensure cache settings on Azure FrontDoor are configured to maximize performance.
