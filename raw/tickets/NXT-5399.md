*Requirements:*

Add an option to the Transactions page that lets the user search for Suspicious Activity. This could be a Tab, with the existing page the default and labelled Patron History.

* If selected, the user has the following search criteria:
** The user can select the Area, Site, User, and Date.
** The date can be a day/month/date range
* A list of all suspicious transactions by selection criteria are displayed
** Transactions that were changed in POS displayed Canceled/Void/Cleared Item are listed.
* The headers can be sorted/filtered
* The results grid can be Printed or exported to Excel.

## Acceptance Criteria
The user can generate the list by Area/Site/User/ and Date
The date is a day/month/ or date range. 
The date range is upto one year
The above report is generated as a display.
List Displayed can be Printed or Exported (When Print/Export functionality is available)
Voided transactions are shown
Canceled payments are shown
Removed items are shown
Items are grouped by Area/Site/User/Date

The results grid can be Printed or exported to Excel.
