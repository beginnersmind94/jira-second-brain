# NXT-41877 — Workspace - Tasks - UI/UX improvements

## Description
*Summary*

Perform useful UI/UX enhancements to Workspace Tasks in order to support Onboarding wizard enhancements.

*Mockup*

N/A

*Details*

* In Workspace → Tasks, refactor the page and do the following: 
** Allow user to expand/reduce column widths
** Reduce the default width of the Status, Due Date, and Action(s) columns by 50% and distribute the remaining width between the Task Name and Description columns.
** _Edit Task_ drawer: 
*** Update the description field to accept up to *500* characters (a character counter is always helpful, but not required).
*** Make this drawer a *3/4-drawer* so that the user can view the full title and description more easily.
*** When a user selects the Save button, change the success toast message from “Task has been created successfully!” to “Task has been updated successfully!”
*** When a user updates the Status of a Workspace task to _Done_, make the task become read-only. 
**** If we need to disable the edit pencil while still allowing row click and having that open a ‘Task Details’ drawer instead, that would be fine.

## Acceptance Criteria
Verify that Workspace page has been redesigned.

* Verify default column widths have been adjusted
** Verify columns widths can be customized by the user as well
* Verify that the Edit Task drawer is now a 3/4-width drawer
* Verify that the Edit Task drawer’s description field now accepts up to 500 characters.
* Verify that the Edit Task drawer’s Save toast message has been updated
* Verify that when users update a task to ‘Done’ status, it becomes read-only
