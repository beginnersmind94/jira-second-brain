# NXT-57292 — UI Changes: SC - GDSN Connect - Image Updates (API Based)

## Description
_(none)_

## Acceptance Criteria
Navigation: Item Upload - Pending Updates Tab

Acceptance Criteria:

Include a new button as “Export Items”

Upon clicking the Button, The system should perform the following:

# *Sync all the unique vendor names into GDSN Connect using Vendors/Sync API:* 

Upon clicking the button, The API should be able to send all the *Vendor* data in SC for a specific region and return the results

Vendors should be parsed as UNSPECIFIED for all the items

Once the data is stored, the user should be able to render the the Vendor list in GDSN Connect UI

# *Sync all the local Items into GDSN Connect using Items/Sync API:* 

Export all the existing items in selected data source (Local) and translate them into message structure as defined in Swagger - [https://api.gdsnconnect.com/docs/cybersoft/#/Items/startItemsSync|https://api.gdsnconnect.com/docs/cybersoft/#/Items/startItemsSync|smart-link] 

Upon successful sync (Return of Sync ID as response), show an UI message says, “Items Export process is successful”, if failed - show as “Items: Sync Process Failed”

Enable Check for Updates button in this step

# *Get Item Updates from GDSN Connect using Items/getItemsFull API:* 

Insert all the updates in to SC - Item updates staging table by calling the end point: [https://api.gdsnconnect.com/docs/cybersoft/#/Items/getItemsFull|https://api.gdsnconnect.com/docs/cybersoft/#/Items/getItemsFull|smart-link] 

Upon successful insertion of data, show an UI message says, “Item update(s) are successfully completed. Please click refresh to view updates”, if failed - show as “Items Update Process Failed”

Once the step is completed, the user should be able to click “Refresh”, the user should be able view the updates populating in the grid with new item images as counts - if applicable
