# NXT-7258 — INV - Physical Inventory - Start new Physical Inventory - Selections

## Description
As an inventory user I want the ability to make the selections for starting a new Physical Inventory so I can create a new inventory.

[Figma|https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=80104%3A148411&viewport=325%2C-4288%2C0.08251962065696716&scaling=scale-down]

To start a new Physical Inventory, can select by:

* Site Selection 
** Area
** Site Code
** Site Name
** Start button

Once Start is selected, additional selection criteria is provided

* Item Rules
** All Items
*** until perpetual inventory exists, this option will be used
*** by ‘All Items’ it means all items for which this site has some ordering condition or has had on hand at one point in time.
**** for initial implementation, the site-item record will need to be created in the back end
** Only Items with Positive/Negative QOH
*** only the items for which the system has some positive (or negative) quantity on hand
** Blank - add items on the fly
* Storage Category (area)
** start by ‘Cooler’ for example
** add areas/categories on the fly
** Item Categories
*** filtered based on the Storage Category selection
* OR Valuation Groups
** if ‘Valuation Groups is selected’ load the Valuation Groups instead of Storage and Item Categories
* Start button
** Physical Inventory is created once this is clicked

Once the new Physical Inventory selections are made (as part of NXT-7258) and the (second) Start button is clicked, this will create the new Physical Inventory and validate the selections.

*History* - save a history of the selections that were made when creating the inventory (split into separate story)

* save a snapshot of every option that was selected
