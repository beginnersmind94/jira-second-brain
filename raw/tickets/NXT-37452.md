# NXT-37452 — IM - Items - Units - Edit/Update

## Description
As an Item Management user I want the ability to edit/update the Units card so I can expand the pack/serving details or manage card-level flags or options.

*Unit card edit scenarios*

* *Fix Weight/Volume* for Pack Sizes (spillover from [NXT-37110|https://primeroedgenext.atlassian.net/browse/NXT-37110])
** Optional pack size field is always in Weight
** Only pack size that will ever use Volume is the Pack Size being linked via Weight/Volume (not sub-qty) when the item is fluid
* Add/update costing
** This should be a clickable action which triggers the costing step
** the costing step enables the cost field for all applicable units
** if a cost exists, load the cost in the unit for which it was originally applied
*** if user wants to assign the cost to a different unit, they should clear this cost first
** Connect to the ‘Missing Costing’ alert if the item is flagged as such
*** some clickable action on the Missing Costing alert could also trigger the edit
* Inventory Readiness (in edit mode)
** Provide an bannered icon in the Units header that presents live information about current Inventory readiness status
*** By default this will communicate the current status (probably icon only)
**** provide a tooltip to show which status the icon represents
** make the icon expandable into a banner which provides the corrective action for the status
*** [https://primeroedgenext.atlassian.net/browse/NXT-37982|https://primeroedgenext.atlassian.net/browse/NXT-37982|smart-link] 
** Manual Control
*** Some card/header level flag that allows the user to manually flag the item as being withheld from Inventory
*** Since the manual option would only be available once an item is inventory eligible, this control can simply take the place of the banner which would be indicating readiness
*** Example: the ‘Inventory readiness’ banner icon could turn green with the Inventory Ready status (assuming categories are also selected) and clicking the banner could open the publish toggle (publish by default)
**** The user could uncheck/toggle and we could change the label to ‘Do not publish to Inventory’
