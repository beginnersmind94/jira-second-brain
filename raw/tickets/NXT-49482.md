Write a new stored procedure that filters the record and move the records to GDSN_Connect table post following validations (Fire & Forget Method):

# GDSNImportKey - Create it as new field, capture this as batch ID for each import
# Duplicate Ingredient code - Insert only one entry into target table, mark the duplicate record as “Duplicate” at staging table
# Duplicate stock item ID - Insert only one record, mark the duplicate record as “Duplicate” at staging table
# Data Type Validations - if any of the field is not meeting the data type , pls mark it as “Invalid Data” Type
# +Field Level Validations:+
## DEX_Local_Menu_ID - validate for any duplicates with in region and mark the duplicate entries as “Duplicate”
## Item_Name_Menu - validate for any duplicates with in region and mark the duplicate entries as “Duplicate”
## District_Menu_ID is NOT NULL and if Item_Type=Food Item and if Serving_Size_Description is blank - mark them as “Missing Serving Size”
## District_Menu_ID is NOT NULL and if Item_Type=Food Item and if Serving_Size_Weight is blank - mark them as “Missing Serving Size Unit Weight”
# Multi Region - Rule 1, 2, 5.a should be exempted if the same data is uploaded in different region  

 All the passed records should be moved to GDSN_Connect table, the existing API should render the details as-is

+Data Corrections:+

# Nutrition Fields - write zero as zero, blank as blank at target and actual values as-is (USDA Compliance)
# Allergens, Images, Meal Credits - if two consecutive Pipe symbols (||||) found then consider as one (||)

# District_Menu_ID is NULL & District_Item_ID is NOT NULL - remark them as “Stock Item only”
