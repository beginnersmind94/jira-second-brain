# NXT-19087 — Income Surveys - Add New Survey - Details Step

## Description
As a district admin, I would like the ability to quickly and easily add a manual income survey on behalf of a household, so that their income data is entered into our system for reference at any time.

*Summary*

Within the ‘Add New Survey’ function, add a step to help add a new survey record for a set of students by allowing them to indicate whether they want to participate in optional programs or receive other benefits, as well as provide other optional information about themselves.

*Requirements*

* *IF* the district’s _Show Income Surveys Benefit_ setting value is set to ‘Yes’, display an *Details* step that displays the following:
** Ethnicity 
*** Radio buttons
**** Hispanic or Latino
**** Not Hispanic or Latino
**** Not Provided (default selection)
** Racial Identity 
*** Radio buttons:
**** Asian
**** American Indian or Alaskan Native
**** Black or African American
**** Native Hawaiian or Other Pacific Islander
**** White
** Other Benefits 
*** Checkboxes
*** If the district has any other benefits set up, allow the user to select one or more
**** This list should *only* display benefits for the current academic year
**** If none exist, we can hide the section or show '_None provided_'
** Consent to Release Meal Eligibility/Other Benefits 
*** Checkbox
****  Default: unchecked
*** Label: _By checking this box, you voluntarily consent to let your child(ren)’s school share your child(ren)’s meal eligibility with any program(s) indicated above. You acknowledge that you have read and understood all information listed._
* *IF* the district’s _Show Income Surveys Benefit_ setting value is set to ‘No’, +*hide*+ this step.
** The step should be hidden from the stepper, and the user should be taken to the next step (Contact Info) instead. 
** The same goes for when trying to go back to the previous step from Contact Info. If this setting is set to ‘No’, the user should be taken back to Household.
* IF the district’s OTHRBENFT setting value is disabled/No, +*hide*+ just the Race, Ethnicity, and Other Benefits sections (only showing the Info Sharing question).
* This story is complete when a user can view any optional benefits offered by the district for the current academic year as well indicate their ethnicity, racial identity, and information sharing preferences. This step should only be visible when the _Show Income Surveys Benefit_ setting value is ‘Yes’. They can also show/hide the list of other benefits based on the other setting value.

## Acceptance Criteria
Verify that in the Income Survey page, when a user selects ‘Add New Survey’, the wizard page displays

Verify that the user can proceed through the survey and access this step, *IF* the district’s _Show Income Surveys Benefit_ setting value is set to ‘Yes’

Verify that the step functions as outlined in the requirements above

Verify that ethnicity can be indicated, and ‘Not Provided’ is the default selection

Verify that racial identity can be indicated

Verify that the list of optional benefits for the current academic year displays, as long as the other setting value is set to indicate showing of benefits

Verify that this list is hidden or ‘Not Provided’ is shown, if there are no benefits for the current academic year

Verify that information sharing preference can be indicated, and the box is unchecked by default

Verify that if OTHRBENFT is set to ‘No’, we hide just the Race, Ethnicity, and Other Benefits sections (only showing the Info Sharing question)

Verify that *IF* the district’s _Show Income Surveys Benefit_ setting value is set to ‘No’, we +*hide*+ this whole step, even from the stepper. 

Verify that this step is skipped (forwards and backwards) in that case also.

Verify that the selections that the user makes on the step are retained if the user proceeds to the next step and then goes back to this previous step, or vice-versa
