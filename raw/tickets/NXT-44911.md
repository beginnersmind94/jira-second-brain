# NXT-44911 — Ignore - SIS Auto Sync – Custom – Add New - API Tab - Joins

## Description
As a SchoolCafé admin, I would like to treat the SIS Auto Sync the same way as manual imports by adding the field mapping tab along with all its functionality so that I can have more control over the import.

*Notes:*

* On this screen, you will:
** Optionally allow the use to “Join APIs” so that they can combine them to customize their import
* This story is written as an update to the existing screen

*Requirements*:

* On the System > Imports & Exports > SIS Auto Sync tab > Add New and Edit, do the following:
* 
*  APIs tab updates:
* Fields:
* Radio Button List
** No changes
* Pre-Configured SIS Data
** No Changes
** We will not join on our pre-configured data sets
* SIS Provider APIs
** Change it to a multi-select list
** Keep same values
** If the select more than one value, then show the Join section
* 
* View Column List
** If have more than one API selected, show the columns for all the APIs selected
* 
* Joins Section
* Add a Grid
** Add New button
*** on click, add a row to the grid
** Columns:
*** First Source API
*** First Source Column
*** Second Source API
*** Second Source Column
**  Rules: 
*** they can’t join a API Column pair to itself
*** they can’t join the same pair twice
*** they can join the same APIs together on different columns
*** Every row must be complete (value selected in all four fields) or else it is invalid
** If they select more than one API, they must “JOIN” them like SQL on a field
*** Example Students.StudentID to Contacts.StudentID to match a student to his guardians
** Delete button on each row
** no paging, sorting, filtering for the grid
** -Try to be smart about it:-
*** -If both datasets have StudentID (or equivalent) auto select it for them as this is the most probable-
* 
* Preview Data button
** Name = Preview
** On click:
*** Keep doing what doing already, but does this also:
*** Validate the Join
*** If they have more than one table selected, verify they have done the JOIN properly
**** selected at least one field from each table
**** rows have values in all 4 columns
*** If they have, allow them to proceed
*** If they haven’t, show them an error message that they must join the tables and don’t let them proceed
*** If more than one select, call each API and then join the data together in code per the join values they selected
***  Show a preview of rows (top 10?) from the API call in the grid below
* Preview Grid
** Source = top x (10?) rows from API call
** Grid does not need paging, sorting or filtering since so few rows
** Columns are variable, depending on columns from API call
** Use API’s field names for column titles and column count
** Rows are the payload from the API call
* 
* Previous
* Next
** Put validators on the Next button
** Validate that at least one API is selected from either list
** If more than one, add the JOIN validator described in Preview button
* 
* On the Field Mapping tab
** the Source Table shows all the fields from the joined tables
** the user can see which API which column comes from

## Acceptance Criteria
* All existing code still works
* existing validators still in place
* User is able to join multiple SIS APIs
* Once joined, user is able to see a data structure of the joined APIs in the preview columns and preview data
* Field Mapping screen Source table shows all the joined data
