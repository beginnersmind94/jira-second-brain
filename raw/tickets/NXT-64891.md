# NXT-64891 — GC Portal - API / SQL Job for Importable Item (Indicator/Tool Tip)

## Description
_(none)_

## Acceptance Criteria
Navigation: DB Enabler

Changes Required:

Additional 2 fields should be inserted in USDA_Consolidated and Data Mart DB as “Importable Item” and Importable_item_Remarks”

This should be a sql job that identifies each record and check all the basic elements that requires for an ingredient:

Item Name + Serving Size (Quantity, Measure, Unit Weight and Unit Weight Measure) + Mandatory Nutrient Elements (Calorie, Total Fat, Saturated Fat, Protein and Carb) + Meal Credits + Stock Item Information

If all the elements are found, then the importable status should be 1, otherwise it should be zero, along with remarks captured


If the mandatory is missing, then update the Importable_item_Remarks as Comma Separated (Ex: Missing Serving Size or Nutrient), update flag as 0, update the Importable_item_Remarks as “Not an Importable Item” 

If serving size + Nutrient are available AND if some of the Meal Credit Elements are missing - then update flag as 1 and update the Importable_item_Remarks as as “Missing Meal Credits”

If serving size + Nutrient + Meal credits are available, and If Stock Item Details are missing, then update flag as 1, and update the Importable_item_Remarks as “Missing Stock Item Details” 


If all are missing, then update flag as 0, and update the Importable_item_Remarks as “Not an Importable Item” 

*View Local Item Scenario in SC*: The optional missing Nutrient elements other than mandatory should be flagged as missing (M) in the UI (Regression)

*Changes in Search API:*
Show an additional Columns in the API result “Importable”=1 or 0 along with importable item remarks
