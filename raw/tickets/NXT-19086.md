# NXT-19086 — Income Surveys - Add New Survey - Household Step

## Description
As a district admin, I would like the ability to quickly and easily add a manual income survey on behalf of a household, so that their income data is entered into our system for reference at any time.

*Summary*

Within the ‘Add New Survey’ function, add a step to help add a new survey record for a set of students by allowing them to indicate Household income or participation in special assistance programs.

*Requirements*

* Add two survey Response Types (*Unselected* by default. One option must be selected to proceed):
** Special Assistance
*** Display the Eligibility Description Assistance Programs for the district. We should display the Local Description for this field.
*** Require the user to enter in their Case #. We should validate it according to the district’s ‘case number format’ settings (System > Configuration > Settings > Internal-Eligibility-Applications). 
** Income
*** Display ‘Household’ Size as an input field:
**** No need to validate this number, unless it is less than the number of students they’ve displayed. In that case we can display an error: “Size exceeded by students”
*** Display ‘Income Frequency’ as a drop-down menu:
**** Get the income frequencies that correspond to the specific Eligibility Guidelines set up in Survey Ranges.
*** The third field changes based on the _Income Range Type_ setting value:
**** If set to ‘Exact Amount’, Display ‘Total Income Amount’ as a plain input field with a $ sign:
***** This menu can cascade from ‘Income Frequency’.
***** User can enter an exact $ amount.
**** If set to either of the other two setting values, display this as a drop-down menu:
***** Based on the selected Frequency, get the income ranges that correspond to the specific Eligibility Guidelines set up in Survey Ranges (USDA or District, based on the setting).
** This story is complete when a user can indicate their response as special assistance (where they select a program and enter a case number in accordance with their district’s descriptions and setting) or income, where they enter a valid household size and an income frequency and range in accordance with their district’s eligibility guidelines.

## Acceptance Criteria
Verify that in the Income Survey page, when a user selects ‘Add New Survey’, the wizard page displays

Verify that the user can proceed through the survey and access this step

Verify that the step functions as outlined in the requirements above

Verify that special assistance programs and case number formats are valid according to the district configuration

Verify that household size is validated to be > # of students added to the survey

Verify that income frequency list is based on the district’s eligibility guidelines

Verify that total income amount ranges are based on the income frequency and then based on the district’s eligibility guidelines 

Verify that the selections that the user makes on the step are retained if the user proceeds to the next step and then goes back to this previous step, or vice-versa
