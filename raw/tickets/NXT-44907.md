# NXT-44907 — API – Custom – Add New - API Tab

## Description
As a SchoolCafé admin, I would like to treat the SIS Auto Sync the same way as manual imports by adding the field mapping tab along with all its functionality so that I can have more control over the import.

*Notes:*

* On this screen, you will:
** Select which API you want to call
** Call the API to get the headers
** Optionally allow the user to preview the data by calling the API and getting the data

*Requirements*:

* On the System > Imports & Exports > SIS Auto Sync tab > Add New and Edit, do the following:
* 
*  APIs tab:
** Doing APIs instead of  File Upload
** Don’t copy the File Upload tab from Manual import, since don’t need it
* Fields:
* Radio Button List
** Pre-Configured
*** If selected, enable the Pre-Configured SIS Data list
*** Default selected
** SIS Provider APIs
*** If selected, enable the SIS Provider list
* Pre-Configured SIS Data
** Single select list
** Source = list of datasets we have already (and will more in the future) configured such as Student Import, Adult Import, Sites Import, Photos Import, etc.)
** List is restricted to the provider selected for the district (Aeries for Aeries, etc.)
** Based upon the value selected in Import Type on the Pre-Upload screen, auto select the API that we normally use for it
*** ex: Student Import for Student Import
** List is hidden if SIS Provider list is shown
* SIS Provider APIs
** Single select list
*** will make multi-select when we do the joins
** Source = list of APIs we will allow the user to consume from the provider that are stored in our DB
*** list is covered in another story
** List is restricted to the provider selected for the district
** List label values are the name of API
** list is hidden if Pre-Configured is shown
* View Column List
** Button
** On click, show the user the list of columns in the selected Pre-Config data or SIS API
*** The list of columns should be loaded in the DB for all of our SIS calls (See other story)
*** Can show in whatever way works best
* o   Preview Data button
** Name = Preview
** On click:
*** -First, validate PII-
**** -Check if the API’s Has PII flag is true-
**** -If true, then check the user’s permissions to see if they have PII permission (from roles and permissions table)-
**** -If they do have PII permission, then allow the user to continue-
**** -If they do not have PII permission, then do not allow them to view the data, but rather show them a message: You do not have permission to view Personally Identifiable Information (PII). Ask your administrator to grant you permission before continuing.-
*** If the user is allowed to preview, show the data in the preview grid
*** If the user is not allowed, then show a message: This data potentially contains Personally Identifiable Information (PII) and your account is not enabled to view PII. Please contact your administrator to grant you PII access if you believe you should have access.
*** Use the Loading circle until you get the data back
*** 
* Grid
** Source = top x (10?) rows from API call
** Grid does not need paging, sorting or filtering since so few rows
** Columns are variable, depending on columns from API call
** Use API’s field names for column titles and column count
** Rows are the payload from the API call
* 
* Previous
* Next
** Put validators on the Next button
** Validate that at least one API is selected from either list

## Acceptance Criteria
* Has radio button for two options
* list of option selected is shown and the other one is hidden
* each list shows the correct data
* user must select at least one value from either list in order to proceed
* User can preview the columns 
* User can preview sample rows
