# NXT-41875 — Onboarding - Save completed Onboarding tours as Workspace Tasks

## Description
As a district administrator, I would like to be able to reference Onboarding tours that I have already finished, so that I have a place to keep track of what I learned while onboarding.

*Summary*

For every tour in the Onboarding wizard that a user has *_finished_*, save that tour as a completed task which can then be referenced in their Workspace.

*Mockup*

N/A

*Details*

* In the Onboarding wizard, when a user goes to any tour where they can click the green {color:#36b37e}*Finish Tour* {color}button (in the Feature selection screen, System and Front Office task screens, all individual task screens), do the following: 
** Reference the data of the first step of the tour from the database
** {color:#97a0af}*_Save the content of the tour as a Workspace task for the individual user_*{color}
*** _Task Title:_ concatenate “*Onboarding” + {{the location of the tour}} + {{the title of the first step of the tour}}*
**** i.e. “Onboarding - Feature Selection - The Recipe for Success”
*** _Due Date:_ The district’s current Go-Live Date.
*** _Status:_ Done
*** _Description:_ For each step in the tour, where *X* = the step number, concatenate “*Step X: ” + “step X title - ” + “step X description”*
**** i.e. “Step 3: Tasks to Complete - This is a list of tasks that need to be complete for this particular feature before you can move to the next phase. Each one has a short description and time estimate to complete.” 
** {color:#97a0af}*_User can then view the task in their_* {color}[{color:#97a0af}*_Workspace tasks page_*{color}|https://qa.perseusedge.com/workspace/tasks]{color:#97a0af}*_._* {color}
*** Task details and status should be disabled.
** *_Note:_* _if a user skips or otherwise does not complete a tour, a task does not need to be created for them._

## Acceptance Criteria
Verify that finished tours can be viewed in Workspace Tasks.

* Verify that when a user selects the ‘Finish Tour’ button anywhere, that tour is saved as a completed task for the user
* Verify that the task shows the title, due date, status, and description as referenced above
* Verify that the user can view the task details but cannot modify them (read-only)
* Verify that if a user skips or otherwise does not complete a tour, a task is not created for them
