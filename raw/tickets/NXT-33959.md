# NXT-33959 — Site Dropdown - SPIKE - Configurable Sorting and Grouping

## Description
Enable the Site dropdown to allow the users to determine how the dropdown should be sorted.



!image-20220823-132439.png|width=409,height=130!

!image-20220823-132534.png|width=1043,height=866!

## Acceptance Criteria
* As a director, I would like the ability to specify via a “Common” setting how the “Site” dropdown UI control is sorted for my district’s users.
* As a Director, I would like to have a setting to control how the “Site” dropdown UI control is sorted for my district’s users:
** Setting Name: *Site UI Dropdown Style*
** Dropdown options:
*** “Sort By Site Code”
*** “Sort By Site Name”
*** -“Sort By Site Type”-
** Default value: “Sort By Site Code”
** Code: SITEDROPSTYLE
** Category: Configuration
** Module: System
** Type: Common
* The Site UI control shall be sorted by default depending upon the setting value specified
** -If “Sort By Site Type” is specified, the Site dropdown shall have the sites grouped by Site Type and sorted alphanumerically within each group-
*** -Sites Types shall displayed in this order (Can this be defined in a configuration somewhere versus hardcoded):-
**** -Elementary School-
**** -Middle School-
**** -High School-
**** -Central Office-
**** -Any other site types shall be displayed in alphanumeric order ascending after the above 4 mentioned types-
**** JAS 11/22/22 change - In order to facilitate this story, we will change the Site Type to a filterable column in the grid. This can achieve the ultimate goal and minimize the significant recoding necessary for the original request.
* -If a Site Type does not contain any Sites, it shall not display in the dropdown.-
