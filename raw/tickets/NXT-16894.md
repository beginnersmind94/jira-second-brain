# NXT-16894 — INV - Manage Orders - Order Detail - Approve or Reject Order

## Description
As an order reviewer I want Approve and Reject buttons on the Order Detail page of any Pending Review orders so I can approve orders or send them back for adjustments.

*Requirements*

* On the Order Detail page, provide an action bar at the top of the page that will contain any buttons relevant to the order
* For Orders that are in Pending Review status, display Approve and Reject buttons
** this is only for users with the ‘Review Orders’ permission (TBA)
** the user who is reviewing the order would not see the Cancel Order button
* Upon click of Approve button
** a comments popup will be displayed, in case the reviewer wants to add notes
*** this will be optional
** the order will be saved in Open status
** the Approve/Reject buttons will be removed
* Upon click of Reject button
** a comments popup will be displayed, so the reviewer can explain why the order is rejected
*** this will be mandatory
** the order will be saved in ‘Resubmit’ status 
*** this new status is covered in [https://primeroedgenext.atlassian.net/browse/NXT-17504|https://primeroedgenext.atlassian.net/browse/NXT-17504|smart-link] 
** the Approve/Reject buttons will be removed
* Regarding Comments upon Approval/Rejection
** this will use the Order Comments field (currently limited to 200 characters)
** if comments already exist, first try to add the review comments to the existing comments
** if the combined comments would exceed 200 characters, overwrite the previous comments, giving priority to the review comments

## Acceptance Criteria
* Approve and Reject buttons will be available on the Order Detail
* On click of either button, comments popup will be provided
* On click of Reject, comments are required
* Rejecting an order will change the status to ‘Resubmit’
* Approving an order will change the status to ‘Open'
