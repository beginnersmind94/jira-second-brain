# NXT-5705 — INV - Add New Ordering Conditions - Select Items (Step 2)

## Description
As a District Supervisor I want an Add Ordering Conditions page so I can select items and begin to assign Site Ordering Conditions.

This page will allow users to select new items and assign them to sites in order to create Ordering Conditions. This story will cover the first half of this process, which is to search and select items.

*Requirements*

Once user makes selections via [NXT-9016|https://primeroedgenext.atlassian.net/browse/NXT-9016], they will click Search and load the items.

+Items+

* No items are loaded when page is first opened (too many items)
* Load the items that are returned by the selections
* Display only the items which exist on at least one active vendor contract
* Do *not* display items which already have some ordering conditions (i.e. exist on the Summary page)

+Item List:+

* Provide ‘Select All’ checkbox so user can select all items in the selected Item Category
* +Columns:+
* Item Number
** searchable column header
* Item Name
** searchable column header
* Vendor attribute dropdown or label (combination of Vendor-Contract and Vendor Item Number)
** column label: *Vendor (Contract # ) - VI#*
** if multiple active vendors provide item, display as a dropdown:
*** display all currently *active* vendor contracts containing that item
**** vendor contract end date should not be passed
*** ‘Select’ option is default (no default vendor selected)
*** Vendor (Contract Number)
**** Example: Oak Farms (C12345)
*** Vendor Item #
*** Example display: “Oak Farms (C12345) - OF-Milk1”
** if the item is only provided by one active vendor contract, display as a label:
*** Vendor (Contract Number)
**** Example: Oak Farms (C12345) - OF-Milk1
*** Vendor Item #
* checkbox for items
** items can be multi-selected
* Next button
* display total number of items selected in the header
* do *not* use pagination

+Validations on Next button+:

* user must select at least one item
* when user clicks Next button, every selected (checked) Item should have a Vendor selection
** provide error message at the top of the page
*** “Please select a vendor for each selected item.”
** highlight items which do not have a vendor selected

Items without current ordering conditions are displayed
Only items on an active vendor contract are displayed
User can select (checkbox) items within the list
User must select a single vendor-contract for each selected (checked) item
