# NXT-16326 — Verification - Tracking - Response workflow - Updating Application Info

## Description
*User Story*

As a district admin I need to complete the response / information I have on a verification application.

*mockup*

[https://www.figma.com/proto/Y3tlwN2yvRc67304adzyyB/Eligibility-3.0?node-id=1129%3A433&scaling=scale-down-width&page-id=1110%3A0|https://www.figma.com/proto/Y3tlwN2yvRc67304adzyyB/Eligibility-3.0?node-id=1129%3A433&scaling=scale-down-width&page-id=1110%3A0|smart-link] 

*requirements*

This story covers the add/edit/delete functions for the Application Info section.

* Update Application Information
** This gives the user the option to adjust the following - *this does not update the application itself\! It must be tracked separately\!*
*** Students
**** Add additional students.
**** Edit existing students, including a change for their Change categorical status on a student.
**** If they’re using the Child Income setting to track income on each student, it can be adjusted.
**** We can show grade / school editing to save space.
*** Categorical Benefits
**** Case Number, Case Type, Recipient Name (optional)
***** This scenario could come up if a household did NOT have categorical benefits originally, but does as of the time they respond to a verification request. It would be more common for this section to have no details available originally, and then a user would be adding them in.
*** Household Members
**** Add additional members.
**** Change reported income amounts.
**** PFD for Alaska.
*** Child Income (if setting is on)
*** Display the Household Size and Total Income + Frequency.
** User can then select Verification Complete.
*** Application is updated, marked as verification complete.
*** The outcomes here can be all over the place - 
**** they’re essentially re-submitting the application information, so the app could go from free to reduced, reduced to free, r to p, f to p, no change, etc.
**** change reasons need to be tracked appropriately here. strongly recommend getting the list from PrimeroEdge for guidance here.
*** Eligibility for students on the application must be updated appropriately.
