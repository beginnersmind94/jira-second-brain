# NXT-28651 — INV - Transfers - Create Transfer - Check items for Physical Inventory

## Description
As a Site Manager I want the Create Transfer screen to warn me when an item is part of a recent completed Physical Inventory so I can adjust transfers accordingly.

*Requirements*

* Warn the user from transferring items which are part of a completed physical inventory
** this is the current behavior for other functions such as Quick Receipts
** check at the item level so that we don’t block the entire transaction for the user
** Implement action on click of *Confirm* (on next screen)
*** unless we want to display in-line after user enters quantity, or indicate those items in the UI on pageload
* Display appropriate messages 
** Message should be consistent with same checks on other transaction types (consider message story [NXT-28652|https://primeroedgenext.atlassian.net/browse/NXT-28652])
* Update UI
** move buttons to save vertical space
** consider implementing Comments differently
** update button names (Reset Fields)
** consolidate Comments and Delete columns into single Action column
** size columns better
*** Unit Type is too wide (Case, Each, or Package 99% of the time; no need for 25+ characters)
*** Item Description needs greater percentage of the width
*** Storage Category and Item Category could be smaller if needed
**** these could be used for custom (or static) grouping instead
** move up Next button so user doesn’t have to scroll forever
*** sticky footer would work well when issues are cleared
