# NXT-71221 — IM - Expand Data Sources for Multi Tenancy support

## Description
As a multi-tenant (statewide) user I want the ability to manage data sources and how the multiple districts I manage will interact with the data in those data sources.

## Acceptance Criteria
* Reintroduce Data Sources as a user-managed field for Multi-Tenancy (statewide/central management)
* Consider User access to higher “realm” districts (State, County, Management, etc.)
* Build support for higher-level data sources: Shared, Global, etc.
* Use data sources throughout Item Management to determine user access for view, edit, etc.
** e.g. users with view-only access to State level data source cannot edit data housed in the State data source
** Users with shared data source access cannot create data in the Local data source
