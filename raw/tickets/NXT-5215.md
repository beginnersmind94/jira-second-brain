# NXT-5215 — Application Details - Actions - Add to Verification

## Description
*User Story* 

As a district admin I suspect a household may be providing the wrong information on an application and want to that application to the verification sample for cause. 

*Mock Ups* 

n/a → add to this dropdown:

!image-20210629-140654.png|width=1832,height=452!

*Requirements* 

* Add the ‘Add to Verification’ button back in under Application Details → More Actions dropdown.
** The button is only available for the following:
*** A processed or notified application.
*** Eligibility of free or reduced.
*** An application that has students still receiving benefits.
**** It’s an active application.
** The button is deactivated otherwise, and if the application has already been added to verification.
** A confirmation box will be needed for this action.
* Program the Add to Verification button to add the app to the sample as a ‘Cause’ type. 
** {color:#ff5630}This doesn’t necessarily need to create the sample itself - but the application will need to be available for viewing under Tracking.{color}
** This should be added to the Final Sample.



Messages:

* Confirmation box: Are you sure you want to add this application to verification for cause?
* Success message: Application added to verification for cause.

## Acceptance Criteria
'Add to Verification' button programmed to add an application to the final verification sample.
Application is added for Cause, not Sampled.

Button is available:

* If the application is not already part of a verification sample.
* If the application is processed / notified.
* If the eligibility is free / reduced.
* If the application is active - students are still actively receiving benefits from it.

A confirmation box is required for the action.
