## Acceptance Criteria
* Upon selection of New E-Order, display a modified version of the Orders list on the Orders tab
* Provide basic input required for showing eligible orders
** Site - default to All; allow single selection 
** Vendor - require single selection
** Date Range - 
*** default to +7 days for From Date
*** add 7 days for To Date
** ‘Include Already Sent Orders’ checkbox option
*** default to off
* Display a ‘Back to View Orders’ button in the sticky footer
* Provide a Preview Orders button to show orders from header selections
* Load the list of eligible orders, if any, in the Orders grid
** Only load orders in *Open* status for the selected vendor and selected date range
** *Dev decision:* use pagination or scroll; as orders can be in the hundreds 
* +Provide basic columns+ (no Manage Views)
* Order #
* Site
* Delivery Date
* Order Amount
* Last E-Order Sent On
** this only needs to be displayed if ‘Include Already Sent Orders’ is applied
** this would show the date and E-Order # of the last time that order was part of a successfully submitted E-Order
* Selection & Exception column
** provide a Select All checkbox in the header row
** provide a checkbox for each eligible order
** display the exception message for any ineligible orders
*** Site not configured for E-Ordering
*** Customer Number not configured for site
* Display a sticky footer Cancel button to reset the page
* Upon selection of orders, enable the ‘Send Electronic Order(s)’ button 
** also display the help message next to the button, dynamically showing the count of selected orders
* Once the E-Order is sent, navigate the user to the E-Orders tab with a highlight over the E-Order which they just submitted (most recent E-Orders at the top of the grid)
