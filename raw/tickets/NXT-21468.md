# NXT-21468 — Skyward SIS Import - Front Office Implementation

## Description
This story is to cover the triggering mechanism of the API Calls.

As a end user, I would like to have a way to trigger (IMPORT DATA) the API calls. 

Once the API's are set up (in the current cases, inbound), there must be a triggering mechanism (button) to trigger the Inbound API and this trigger must be controlled. 

There is a pretty good chance that the user might click multiple times and so there must be warning messages tool tips.

Also, the user must be prevented for clicking more than once (deprecation of the click)until the initial trigger is completed.

## Acceptance Criteria
Verify and validate that the new control “Import Data” is within the configuration module.

Verify and validate that the new control is only available to trigger only if the API is enabled.

Verify and Validate that the new control is deprecated as soon as it is triggered and while it is processing a status reflector like an hour glass must be displayed.

Verify and validate that appropriate messages are displayed once the API is triggered (Status that API is successfully triggered or failed etc based on the feed back from the vendors)

Verify and validate that the new control is available ofter the get data is has fetched some data.

Verify and validate that all the warning messages, error messages and tool tips are in place.

Verify and validate that this triggering mechanism is set up for all the three vendors (power school, aries and skyward)
