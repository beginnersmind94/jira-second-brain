# NXT-70053 — POS - Credit Cards > Square Integration > Square Transaction Feature > Additional Messaging and Alerts

## Description
[https://developer.squareup.com/docs/oauth-api/overview|https://developer.squareup.com/docs/oauth-api/overview|smart-link] 
When A terminal is in use that is configured for square, we will need to let the user know all of the important information that will assist them using the feature.

This includes all additional helper texts and alerts for login, authorization status, and an offline scenario.

If a user tries to initiate a square transaction when the authorization token is expired, we will display a bright message area with helper text to let the user know that Card transactions are disabled and that they need to reauthorize.

Text:

Credit Card Transactions Disabled

Square Authorization has Expired. 

Please contact system Administrator to Reauthorize.

## Acceptance Criteria
A banner on the Login screen is shown if the Token Authorization has expired.
A banner on open session screen is presented in red if Token Authorization has expired.
If Authorization is expired, the credit card tab on the “PAY” pop up displays a disabled message.
If a terminal is running in offline mode, show a message on the Pay tab and disable credit cards.
Show Message on login for expiration warning.
Add Credit Card Icon to Welcome screen and Open Session screen with tooltip

When we detect new terminal name is in use and has a Square Configuration use this warning pop up:

”{color:#36b37e}That device name was recently used. Continuing will deactivate the original terminal and remove its 3rd party Credit Card Processing configuration. Credit card processing will need to be set up again if required.{color}

{color:#36b37e}Are you sure you want to deactivate the existing terminal and remove 3rd party credit card configuration?{color} “
