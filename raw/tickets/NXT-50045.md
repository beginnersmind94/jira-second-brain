# NXT-50045 — PR- Consolidated Tab- Final Check before save

## Description
As a production manager, sometimes as I am reviewing my consolidated plan, other sites are continuing to make changes to the counts or adding new menu items. I would like for the system to do one final check before I click save to ensure that I get the latest data available.

## Acceptance Criteria
* When user clicks save, run a final check to make sure that the data available on the current consolidated plan is the latest data available
** This includes if prep site opens a new tab and changes their own plan while the other tab still has the consolidated tab open, apply final as check as well
* If there are changes available, refresh the page and display a banner “New changes have been made to this plan” 
* Display the newest data available whether it is counts or new menu items or if counts were removed or menu items were removed
* Also consider any new/deleted plans from “receiving sites” that prep site is responsible for
* This only happens before they click the first save to lock in the consolidated plan
