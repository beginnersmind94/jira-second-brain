# NXT-15263 — [POS] - Offline Support/Operation

## Description
Regression and development for offline support/operation of POS. Any functionality within connected mode should be supported within offline usage. Below is the list of items that should be reviewed. Any identified features that are not working should be created as associate bugs to the parent epic of this user story.

Also note that there is a limit to how long a POS can be operated offline before data needs to be refreshed. (check with DEV to confirm)

Required:

* Sign In - This will be handled in cloned story 17933
* Open service/session
* Perform menu service
* Open accounts based on key entry or manual lookup
* Display account details, image, restrictions, allergies, and notes
* Display menus and items
* Make sales
* Make payments
* Identify second meal for on the same terminal
* Close service/session
* Perform bulk sales
* Perform summary sales
* Synchronize information to/from server when connection is re-established

Not required:

* Second meal from other terminals
* Identification of students from other sites

*Note: This functionality will be required for pilot customers as well, therefore, will need to be a hotfix as part of Release 2.0.*
