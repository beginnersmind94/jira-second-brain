# NXT-3197 — Sessions - List

## Description
*User Story*

As a site manager I want to view a list of sessions tied to the sites I have access to so that I can ensure sessions are balanced.

As a district admin I want to view all sessions across the district to ensure that sessions are balanced and accounted for.

*Mock-up*

[+https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=1219%3A373&scaling=min-zoom+|https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=1219%3A373&scaling=min-zoom]

*Requirements*

POS Module > Session takes us to the Sessions list page.

The list of sessions is determined by the Search Criteria entered. 

* Create a page where a list of sessions can be seen along with basic session information.
** Terminal Name
** Site Name / Site Code
** Opening Balance
** Opening Date
** Closing Balance
** Closing Date
** Status
** Over/Under
*** Opening Balance - Closing Balance
** View Session button*
** Reconcile Session button*
*** These 2 buttons are handled in separate cases.
* Add filters to narrow results:
** Area
** Site Code
** Site Name
** Date Fields (To - From)
** Status
*** Open, Closed, Reconciled, Deposited

The ability to select and open multiple sessions at one time

The ability to select and Auto Reconcile multiple sessions

Not implemented in this story

* Export, 
* Print, 
* By Status, 
* More Filters, 
* Reconcile Selected.
* Session Cards for Open, Closed, Reconciled, Deposited and the Pie Chart are in and can be added later [https://primeroedgenext.atlassian.net/browse/NXT-6696|https://primeroedgenext.atlassian.net/browse/NXT-6696|smart-link]

## Acceptance Criteria
User has access to a page that shows sessions.
User can select a list of sessions based on search criteria
Default Date range is for the previous 3 days, Including today. Example 12/26, 12/27. 12/28 if today is the Dec 28th.
Any past date can be selected.
User can filter sessions in the list based on filter selections
User can only see sessions for sites they have access to.
User can select and open sessions for processing or viewing.
As one session is closed manually or by changing status to Reconciled it is closed and the next session is opened for processing or viewing.
When Over/Under balance is within MAXAMTRC the session can be selected for Auto Reconcile.
All columns are sortable
Only Dates are required for search.
Auto Select Yesterday and Today as the From and To dates.
Sort order Site Code, Terminal, Opening Date, Closing Date, Status
Multiple Sessions can be selected from the list and reconciled.
