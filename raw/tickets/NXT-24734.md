# NXT-24734 — POS - Dynamic Menus - Dynamic menu grid based on Production data

## Description
As a cashier I want to display a menu grid based on what was entered in Menu Planning and Production rather than relying on a previously built static menu. 

* The Menu grid is determined by the Site, Date, Meal Type and Serving Line
* There are two options to display the dynamic menu Vertical or Horizontal
** clicking or touching the menu grid for longer than 3 seconds when a patron is not selected changes the orientation.
** The default is vertical
* There are five default categories, Entree, Fruit, Vegetable, Grain, and Milk
** Depending on the orientation of the grid the categories are displayed at the top or left side of the grid
* The items that are listed in order based on the amount produced from production
** The Items are listed in order by production count first then by alphabetical order within the appropriate category.
** The background color of the item is determined by the POS category color of the item
* The grid is expanded as required to display all menu items 
* If required scrolling is enabled.
* The same rules for allergens, pricing and other restrictions are applied to being able to sell a menu item to a patron as in static menu sales.
* When a visitor is added to the Student Transaction the allergen restrictions are removed.
* If an item is restricted the item button is disabled and a slash icon is overlayed.
