# NXT-23388 — INV - Additions/Withdrawals - Edit mode - Unsaved Changes + Fixes

## Description
As a Site Manager I want improvements to the process of managing Additions or Withdrawals.

*Note:* may require coordination with team 2 regarding Additions. The goal is for both to function very similar to each other.

*Requirements*

* Add unsaved changes prompts (standard wording) to both Additions and Withdrawals when editing (*BOTH)*
** if any value has changed
** if user clicks on any other nav link and exists the page
* Always display most recent (previous) comments (*BOTH*)
* Show Comments as mandatory if an edit has been made
** User is still required to enter or modify Comments if edited
*** for Edit, Comments simply need to be updated
*** (compared to Cancel comments will overwrite)
*** indicate required Comments by highlighting Comments box
* Hide the ‘Back to Addition/Withdrawal List’ button when in Edit mode (*WITHDRAWALS ONLY*)
** this is to avoid confusion as a Save button
** -perhaps change styling- 
** *Note:* Additions is already doing that
* Do not disable the Save button in Edit mode (*BOTH*)
** allow the user to click to understand why they can’t Save
** show relevant messages:
*** ‘Please enter adjustment comments’
*** ‘Please correct Date on Product information for Item #1234'
