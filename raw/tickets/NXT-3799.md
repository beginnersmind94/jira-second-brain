# NXT-3799 — AcctMgt - Patron Split View - Enrollment

## Description
*User Story*

As a district administrator I want to see the enrollment history of a patron.

*Mockup*

[https://nqyc9v.axshare.com/start.html#p=category_based_on_patron_view&id=jytwv5&g=1|https://nqyc9v.axshare.com/start.html#p=category_based_on_patron_view&id=jytwv5&g=1]

*Requirements*

* Add an Enrollment section to the Patron Details panel.
* Section will show information on historical enrollment in chronological order, latest first.
** Grid showing:
*** School (Site Code + Site Name)
*** Grade
*** Start Date
*** End Date
*** Processor
*** Process Date
** Show the top 3 entries only on the panel. The rest are buried under a ‘View more’ button that should load the full history in a drawer.
* Edit functionality has been adjusted as of 7/17/20:
** While viewing the full history in the side-drawer, the top entry should have an edit option available.
*** Only the top entry can be edited.
*** Only the Start Date can be adjusted.
**** Start Date cannot go past the start of the Academic Year (7/1/XX)
** When saving, the 2nd from top record will update to have an End Date of ‘Start Date - 1’ if there is a need to adjust the dates.

* -Add an Edit button to the Enrollment section.-
** -This activates an edit mode where all existing lines can be edited/removed.-
** -Removing should be a soft-delete. We want to keep the data in the DB but inactivate the record.-
** -User should receive a confirmation dialog when deleting a record.-
** -Editing entries should be done appropriately - available grades depends on the site selected.-
** -Following fields can be edited:-
*** -School (Site Code + Site Name)-
*** -Grade-
*** -Start Date-
*** -End Date-

* -Dates should not be allowed to overlap for start and end dates. Students should not be in 2 places at once.-
** -User needs appropriate messaging / feedback if the dates are going to overlap.-
*** -The enrollment dates cannot overlap. Edit the existing records so that there is an open date range available.-
** -We may want to have the grid update dynamically when an entry is edited so everything is in chronological order.-
** -Standard save messaging if there are no conflicts.-

## Acceptance Criteria
Enrollment section added.
History of enrollment shown as a grid. 
Top entry can have its start date edited - 2nd from top entry shifts its end date to accommodate.
