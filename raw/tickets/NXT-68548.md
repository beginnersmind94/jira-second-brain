# NXT-68548 — Fin - General Ledger Excel Export (Per Account)

## Description
When the user runs the General Ledger report and clicks *Export to Excel*, the system must generate an {{.xlsx}} workbook that contains *one worksheet per GL account* included in the report, using the *Ledger Template* layout we defined:

* Account-level summary at the top
* Detailed transaction lines in a table below
* Balances consistent with the GL calculation rules (assets/liabilities/equity permanent; revenue/expense fiscal-year based).

Parameter selection (which accounts, date range, etc.) is handled by another story; this story starts from the moment the backend has already computed the GL dataset.

## Acceptance Criteria
* *Workbook and sheets*
1.1. System generates a single {{.xlsx}} file per export action.
1.2. The workbook must contain *one worksheet per account* returned in the GL report.
1.3. Sheet name pattern: {{<AccountNumber> - <ShortAccountName>}} (truncated if needed to fit Excel’s 31-character limit).
* *Header section (per account sheet)*
Each sheet must follow the ledger template layout:
** Row 1 – merged title {{General Ledger}} --- [Account Number] [Account Name].-
** {{Account info}} block (cells A3:B6):
*** A3: “Account Name” | B3: account name
*** A4: “Account Number” | B4: full account code
*** A5: “Account Type” | B5: Asset / Liability / Equity / Revenue / Expense
*** A6: “Account Category”
*** -A6: “Fiscal Year” | B6: fiscal year label (e.g., FY 2024–2025 or 2024-07-01 – 2025-06-30).-
** {{Report range & balances}} block (cells D3:E8):
*** D3: “Posted Date (From)” | E3: report {{start_date}}
*** D4: “Posted Date (To)” | E4: report {{end_date}}
*** D5: “Beginning Balance” | E5: balance from GL engine = _ending balance as of the day before_ {{start_date}} (i.e., after applying all transactions with {{posted_date ≤ start_date − 1}}, per account-type rules)
*** D6: “Period Debits” | E6: total debits in [start_date, end_date] for this account.
*** D7: “Period Credits” | E7: total credits in [start_date, end_date] for this account.
*** D8: “Ending Balance” | E8: ending balance from GL engine (as of {{end_date}}).
2.1. Values must match exactly the on-screen GL report for that account.
*** D9: “Current Balance” separate field – ending balance as of date the report is run
* *Transactions table (per account sheet)*
3.1. Row 9: label {{Journal Lines}} (or {{Transactions}}) in A9.
3.2. Row 10 must contain the column headers (frozen for scrolling):
** A10: Date (posted date used for GL calculations)
** -B10: Fiscal Year (e.g., FY 2024–2025)-
** C10: Transaction ID (unique line or document identifier)
** D10: Description (line-level description / memo)
** E10: Debit/Credit (single character or word: {{D}} or {{C}})
** F10: Amount (absolute value of the entry)
** G10: Running Balance
* 3.3. All GL lines for this account within the selected date range must be written from row 11 downward, *sorted by*:
** Posted Date ascending
* *Running Balance calculation*
1. Which accounts use which rule
** *Group A (Debit-normal accounts)*
*** Asset
*** Expense
*** *For Assets/Expenses, debits add, credits subtract from the balance.* 
** *Group B (Credit-normal accounts)*
*** Liability
*** Equity / Fund Balance
*** Revenue
*** *For Liabilities/Equity/Revenue, credits add, debits subtract from the balance.*
* The backend already knows the account type for each sheet.
** Starting point for the first line
*** Let:
**** *BeginningBalance* = value in header cell *E5* for that account.
**** For the *first transaction line* (row 13):
***** We will calculate *G13 (Running Balance)* using the rules below (Group A or Group B).
***** Group A – Assets & Expenses (debit-normal)
****** Use this logic *only if account type is Asset or Expense*:
****** *First line (row 13)*
****** If *E13 = "D"* → {{G13 = BeginningBalance + F13}}
****** f *E13 = "C"* → {{G13 = BeginningBalance - F13}}
****** *Each next line (row n > 13)*
****** If *En = "D"* → {{Gn = G(n-1) + Fn}}
****** If *En = "C"* → {{Gn = G(n-1) - Fn}}
***** For Assets/Expenses, *debits add*, *credits subtract* from the balance.
***** Group B – Liabilities, Equity, Revenue (credit-normal)
****** Use this logic *only if account type is Liability, Equity/Fund Balance, or Revenue*:
****** *First line (row 13)*
****** If *E13 = "D"* → {{G13 = BeginningBalance - F13}}
****** If *E13 = "C"* → {{G13 = BeginningBalance + F13}}
****** *Each next line (row n > 13)*
****** If *En = "D"* → {{Gn = G(n-1) - Fn}}
****** If *En = "C"* → {{Gn = G(n-1) + Fn}}
***** For Liabilities/Equity/Revenue, *credits add*, *debits subtract* from the balance.
***** What must always be true
****** If there is at least one transaction line: The *last Running Balance* value (bottom of column G) *must equal* the *Ending Balance* in header cell *E8* for that sheet.
****** If there are *no transaction lines* for that account in the period:
****** Leave column G empty.
****** *BeginningBalance = EndingBalance* in the header (E5 = E8).
* *Fiscal-year consistency*
5.1. This export assumes the date range has already passed the “single fiscal year” validation (that logic is handled by the GL UI story).
5.2. Export must *not* re-implement date validation, but it must include the correct Fiscal Year label based on the backend’s fiscal-year configuration (assume a default of 7/1 to 6/30 if nothing is set
* *Numeric formatting*
6.1. Amount and balance columns (F, G) must use an accounting/number format with 2 decimal places and right alignment.
6.2. Negative balances should appear with a leading minus sign (or parentheses if consistent with rest of product).
* *File naming*
7.1. Default filename pattern: {{GL_Export_AccountCode.xlsx}}
