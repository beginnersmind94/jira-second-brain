# NXT-10916 — [TaggedNotification] - Notifications - Framework - Common Drawer Component

## Description
*User Story*

As a central office user I want to send multiple notifications at once (bulk) with options to choose between email or print, sorting, etc.



*Summary*

* Create a common component to give user options:
** Format (Individual letter for each student vs grouping students by household)
** Email/Print (choose between sending emails, printing, the diff. of both)
** Sorting (based on student information)
* Show the user which active letter templates will be used based on  (this is a nice-to-have)
* Call the Notification APIs for sending letters out via print/email.

*Details*

* Create a common drawer component that can be used across multiple modules and calls the Notification APIs at the end.
* The following options should be available for each team to pick and choose from:
** List of templates:
*** Show the list of Reminder templates that will be used to notify the current selection of students. 
**** To do this, get the primary language for each student. If there is a corresponding template in that language for the student, display the name of it in this list. Otherwise, use the active English template. 
**** i.e., if 2 students selected in the grid have a default language of Spanish, 2 have Chinese as their language, and 5 are English, then this list should show 3 different templates (as long as Chinese and Spanish are active). If an active template does not exist for Chinese, it should show 2 different templates--English and Spanish.
** Letter Format
*** Individual
**** 1 notification is printed for every individual student in the grid selection
*** Household
**** If any of the students in the grid selection belong to the same household, combine them on the letter.
** Email/Print Options - (this should display the total number of ‘sends’ that would happen)
*** Email/Print
**** This option gathers all the emails, and then prints for the remainder. Number should highlight how many emails will be sent, and how many prints will happen, along with a total.
*** Email Only
**** This option collects all grid entries that have an available email address. Number should be based on the selected Letter Format.
*** Print All
**** This option sends a print for each grid result, based on the selected Letter Format.
*** Print Only
**** This option collects all grid entries that have no email address. Number should be based on the selected Letter Format.
** Sorting Options
*** Site Code
**** Sort letters by the Site Code first, then Grade, Last Name
*** Last Name
**** Sort letters by Last Name first
*** Grade
**** Sort letters by Grade first
*** Zip
**** Sort letters by Zip/PostalCode first

## Acceptance Criteria
Drawer component is available for any team to bolt onto a page.

Options available:

* Template List (nice-to-have, lowest priority item)
* Format (individual vs household)
* Email/Print options
* Sorting options

Options are optional when being used elsewhere.
