# NXT-11217 — Acct Mgmt - Account Details - Add History

## Description
As a district admin, I would like to be able to view a log of any changes made to an account, so that I can easily find where potential mistakes might have been made (and by whom\!).

*Summary*

Add a History button to each patron’s details page, which when selected opens a page that allows the user to see a timestamped log of any field changes made to that account.

*Mockup*

[View History|https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=422500%3A277157&scaling=scale-down-width]

*Details*

* In the Account Management - Accounts page, when a user goes to view the details of an individual account, add a History icon to the view mode.
* When selected, open the History of this account.
* This page should display a table that should follow the look and feel of the Role History page, displaying data in these columns: 
** _Updated On_: the date & timestamp of when any user last hit a ‘Save’ button after modifying this patron’s account.
*** By default, sort the grid by this column, in descending order
** _Updated By:_ the user (first name and last name) of whoever selected a ‘Save’ button when modifying the patron’s account at that time.
** _Previous Value:_ the old value. Can also be blank for optional fields.
*** This should follow the convention of ‘\{field}-\{oldVal}’. Ex. Eligibility-Free
** _New Value:_ the new value that was added or changed by the user.
*** This should follow the same convention as Previous Value.
*** This only captures field-level changes, not enrollment history.
** Values should display the first 7 rows (or whatever the standard is), and then expansion should be possible.
** This table will also need pagination.
* Add a Close button that takes the user back to that patron’s details when selected.
* This should apply to both Students and Adults.

## Acceptance Criteria
This story is complete when a user can find a History icon on every account detail page which, when selected, shows a log of all old and new changes made to that account by a user. It should capture every potential field that can be changed for that account, and be sorted on Updated On in descending order by default, with pagination.
