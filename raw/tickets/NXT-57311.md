# NXT-57311 — MP- Menu Planner- Considering Counts from Standard based on percentages for analysis calculations (UI ONLY)

## Description
As a menu planner, sometimes I use the percentages as a way to calculate my counts when I am on the standard menu. However, when I provide those numbers, it uses those numbers as counts instead of a percentage.

## Acceptance Criteria
* Update menu planner to consider percentage flag set on the standard 
* Counts set on the standard if the percentage flag should reflect on the analysis set on the menu planner
** EX if on standard meal count is set to 200 and user uses percentage flag, and adds 50 and 50 for two menu items, this should calculate on menu planner as 100 and 100 for the counts
* If user adds and removes menu items, the counts again should switch to a percentage instead of a count and continue following the percentage flag
** EX if on standard meal count is set to 200 and user uses percentage flag, and adds 50 and 50 for two menu items, this should calculate on menu planner as 100 and 100 for the counts, however if 1 more menu item is added on the menu planner, then percentages would be split 34,33,33.
