# NXT-66392 — Login page remember me feature implementation

## Description
h2. Overview

The "Remember Me" functionality enhances the user experience by allowing the system to store the user's email address after a successful login, provided the "Remember Me" checkbox is selected. This feature ensures that the email field is auto-filled on subsequent visits to the login page, even after the user logs out. The functionality also supports rendering additional authentication modes based on the email and allows users to edit the stored email if needed. When the "Remember Me" checkbox is unchecked, the functionality is disabled, and no email is stored or auto-filled.

h2. Functional Requirements

# *"Remember Me" Checkbox*:
#* A "Remember Me" checkbox is presented on the login page, allowing the user to opt-in to the functionality.
#* By default, the checkbox may be unchecked to avoid unintended storage of user information.
# *Storing Email on Successful Login*:
#* If the "Remember Me" checkbox is checked and the user successfully logs in (i.e., the provided email and credentials are validated), the entered email address is stored in the browser's localStorage under the key RememberedUserEmail.
#* The email is only stored after a successful login to ensure that only valid emails are saved.
# *Auto-Filling Email*:
#* When the login page loads, the system checks localStorage for a stored RememberedUserEmail.
#* If a stored email exists and the "Remember Me" functionality was previously enabled, the email field is automatically populated with the stored email.
#* This behavior persists even after the user logs out, as long as the email was stored during a previous successful login with "Remember Me" checked.
# *Rendering Authentication Modes*:
#* Upon auto-filling the email field or when the user enters an email, the system triggers the fetchAuthTypes function to retrieve and display available authentication modes (e.g., password, SSO) associated with the email.
#* This ensures that the login page dynamically updates to show relevant authentication options based on the email, whether auto-filled or manually entered.
#* Remove the "Login using Single Sign-On" button if OpenID authentication is not enabled.
# *Editing the Email*:
#* Users can edit the auto-filled email in the email field at any time.
#* If the email is edited, the system validates the new email and updates the authentication modes accordingly by calling fetchAuthTypes with the new email value.
#* If the user logs in successfully with a new email and "Remember Me" is checked, the new email overwrites the previously stored RememberedUserEmail in localStorage.
# *Disabling "Remember Me"*:
#* If the "Remember Me" checkbox is unchecked during login, the system does not store the entered email in localStorage.
#* If an email was previously stored, unchecking "Remember Me" and logging in does not clear the stored email unless explicitly implemented (e.g., clearing localStorage on logout when "Remember Me" is unchecked).
#* When "Remember Me" is unchecked, the email field does not auto-fill on subsequent visits, and the system behaves as if no email was stored.
# *Logout Behavior*:
#* After logout, if an email was previously stored with "Remember Me" checked, the email field on the login page is auto-filled with the stored RememberedUserEmail.
#* The system continues to retrieve and display authentication modes for the auto-filled email.
# *Technical Considerations*:
#* The email is stored in localStorage to persist across browser sessions and after logout.
#* The system uses a controlled TextField component (e.g., Material-UI TextField) to manage the email input, ensuring the field reflects the stored email or user edits.
#* Email validation ensures that only valid email formats are processed, with error messages displayed for blank or incorrectly formatted emails.
#* The useEffect hook is used to check for a stored email when the login component mounts, populating the email field if applicable.
#* The fetchAuthTypes function is called on email blur or change to retrieve authentication modes, ensuring dynamic updates based on the email.

h2. User Flow

# *User Visits Login Page*:
#* If a RememberedUserEmail exists in localStorage, the email field is auto-filled, and authentication modes are fetched.
#* If no stored email exists, the email field is empty.
# *User Enters Email and Credentials*:
#* The user enters an email and checks/unchecks the "Remember Me" checkbox.
#* If the email is edited, validation occurs, and authentication modes are updated.
# *Successful Login*:
#* If "Remember Me" is checked, the email is stored in localStorage under RememberedUserEmail.
#* If "Remember Me" is unchecked, no email is stored.
# *Logout*:
#* On logout, the login page reloads, and the email field is auto-filled with the stored RememberedUserEmail (if it exists).
#* Authentication modes are fetched for the auto-filled email.
# *Subsequent Visits*:
#* The email field auto-fills with the stored email if "Remember Me" was previously enabled.
#* The user can edit the email, and the system updates authentication modes accordingly.

## Acceptance Criteria
1. Auto-fill Email on Login Page Load (with Remembered Email)

Given a value exists in localStorage under the key RememberedUserEmail
When the user visits the login page
Then the email input field should be auto-filled with the stored email
And the system should fetch authentication modes for the auto-filled email

2. Empty Email Field on Login Page Load (no Remembered Email)

Given no value exists in localStorage under the key RememberedUserEmail
When the user visits the login page
Then the email input field should be empty
And no authentication modes are fetched initially

3. Email Entry and Remember Me Behavior

Given the user enters an email
And toggles the "Remember Me" checkbox
When the user logs in successfully
Then if "Remember Me" is checked, the email is saved in localStorage under RememberedUserEmail
And if "Remember Me" is unchecked, no email is saved in localStorage

4. Email Validation and Authentication Modes Fetch on Edit

Given the user has entered an email
When the email field is modified
Then the system validates the email format
And if the format is valid, authentication modes are re-fetched for the new email

5. Auto-fill and Auth Modes Fetch on Logout

Given the user has logged out
When the login page reloads
Then the email field should be auto-filled with the value from localStorage.RememberedUserEmail (if it exists)
And authentication modes should be fetched for that email

6. Subsequent Visits Remember Email if Stored

Given a user had previously logged in with "Remember Me" checked
When they return to the login page
Then the email field should be auto-filled with the stored email
And authentication modes should be fetched for that email

7. User Can Edit Auto-filled Email and Trigger Auth Modes Update

Given the email field is auto-filled from localStorage
When the user edits the email
Then the system should validate the new email format
And if valid, authentication modes should be re-fetched based on the new email
