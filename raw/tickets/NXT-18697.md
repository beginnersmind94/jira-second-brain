# NXT-18697 — IM- API - Remove Parenthetical serving measure information from CNDB Items

## Description
USDA has requested that we remove Parenthetical serving measure information when information becomes inaccurate based on the change in unit of measure or quantity of measure. 

Example: Item #1053 "1 cup (yields 2 cups whipped) when added to a recipe in a different volume than 1 cup, say 2 cups, it will still display (yields 2 cups whipped). This information is inaccurate and must be removed. 

Side note:
Researching on Classic, determined that this removal is already happening on some items but not all. There is no way to pinpoint how this removal is being determined but could possibly be helpful if analyzed. 



More notes and examples.  

The requirement is for any CN item if it’s measure was given with some information in parenthesis , then clean it by removing it and import the measure.

Problem is for this item #1053 : one item measure is given as 1 cup, fluid (yields 2 cups whipped) = 238 grams. so, per USDA “yields 2 cups of Whipped” is valid only for 1 cup, fluid.

lets say, you used 10 cups , fluid in a recipe then showing that as 10 cup, fluid (yields 2 cups whipped) is incorrect. like its shown in the below screenshot. thats the problem we are trying to avoid by cleaning the measure description and importing it as 1 cup, fluid = 238 grams.

Example 2: for this item #10130 item measure says 2 slices (6 per 6-oz pkg.) , but when clicked on possible measures it is showing 1 slice also 6 per 6oz pkg. which is wrong.

## Acceptance Criteria
Remove Parenthetical information from 1 cup (yields 2 cups whipped).

Classic import of this information displays correctly but no determination on how certain items are keeping parenthetical information and some are not.
