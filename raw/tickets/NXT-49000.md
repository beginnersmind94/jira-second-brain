# NXT-49000 — AM - Reminders - Scheduler - View Details / Send History

## Description
As Dana the Director I want to see the sending history of the recurring event so I have confidence and proof that the system is sending the automatic reminders.

*requirements*

* User can click into a recurring event and see when the system has sent the emails out.
* User can download the letters associated with the event.
* User can click into the event and see a list of accounts who were sent an email.
* User can download the individual letter associated with a single account.

## Acceptance Criteria
* User can click into a recurring event and see when the system has sent the emails out.
** Drawer containing a grid with columns:
*** Start Date
*** End Date
*** Status - In Progress, Complete, Failed, Queued
*** Letters Generated - number of letters that were created and sent as part of the recurring event.
**** Failed - Show how many were sent before failure: ex: 500/1000, 0/1000
**** In Progress - Show how many have sent so far ex: 0/1000, 400/1000
**** Completed - Show how many were sent: 1000, 500, 450
**** Queued - No need to show any numbers
* -User can download the letters associated with the event.-
** -Download action - provide 1 file to the user with all letters inside of it. Either a merged pdf or a zip file.-
* User can click into the event and see a list of accounts who were sent an email.
** View action, another drawer with a grid and columns:
*** Account ID #
*** Name - Format: Lastname, Firstname
** Columns have filters.
** User can download the individual letter associated with a single account.
*** Download action inside the 2nd drawer. Serve up the 1 letter that was sent to that account.
