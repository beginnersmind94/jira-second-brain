*Requirements:*

* User has a page where they can launch the Edit Check report off of.
* User can select whether they have a Detailed or a Summary version of the report.
** The Detailed version would provide a day-by-day breakdown, while the Summary would not.
* User has options for site selection and dates.
** Site Selection
*** Areas
**** Groups of sites that are configured elsewhere
*** Site Type
**** A characteristic on sites that is configured elsewhere.
*** Sites
**** Needs an ‘All’ option or a ‘Select All’ button if we can’t do the all option.
**** Dropdown, multi-select possible.
** Dates
*** User can run the report for a single day, a date range, or a month.
*** 1 year max date range selection to prevent people pulling too much data.
* User can run the report, which contains the following:
** It shows each day of the selected month.
** Breaks down by meal type.
** Breaks down further by eligibility (free/reduced/paid)
*** Columns:
**** Approved (how many students are approved for that eligibility for that date)
**** Eligible (V/A) - this takes into account the approved number + visiting students - away students.
***** This relates to the enrollment site of the student at the time of sale. Visiting student is enrolled at another site, an away student had a meal at another site that day.
**** Eligible X AF (eligible number above multiplied by the attendance factor for that month)
**** Count (meal count for that meal for that eligibility for that date)
**** % (what percentage of the ‘Eligibility X AF’ number had a meal? (eligibleXAF / count) * 100) 
**** It also shows totals.
***** Served total
***** Site total
***** District total
**** Attendance Factor 
** Report also shows the attendance factor which is used for a calculation above.
** It also shows the total enrollment count - the highest number of students active for that time period.

* User can see problems that need to be resolved. 
** Two scenarios here:
*** Meal Count > Eligible (V/A) count.
*** Meal Count > Attendance Factor x AF count
** User needs a way to discern when this happens.
* User can export the report out to print or PDF.

## Acceptance Criteria
Edit check data is selected by Area/Site Type/SIte Code/Site/Programs/Meal Type
The list can be shown in Summary view or Detailed view
Counts that exceed the Eligibility Count or Attendance factor are highlighted
The date is a day/month/ or date range. 
The date range is upto one year
The list has the paging and scrolling
The list can be organized by grouping headers
The list can be sorted
The list can be filtered
Headers can be added or removed
There are sections for each Eligibility Type Free/Reduced/Paid with a Total and Attendance Factor percentage.
Each Section Free/Paid/Reduced/Total has sub headers for Approved/Eligible (V/A)/AA Eligible/Count/Percent
Apprv = Approved students enrolled at site. 
AF = Attendance Factor
Elig (V/A) = Students Eligible to be served. V = Students served visiting from other sites. A = Students served away at other sites. Calculation = (Apprv + V - A)
AA Elig = Attendance Adjusted Eligible. The number of meals allowed before the Attendance Factor is exceeded. Calculation = (Elig (V/A) X AF)
