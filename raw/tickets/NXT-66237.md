# NXT-66237 — ACC - Labor Hours - Delete actions, History tracking

## Description
As Dana the Director I want the ability to clean up my configuration values so that inactives are not cluttering my screen. I need an audit log of these actions later so I do not blame Cybersoft for my data disappearing.

*requirements*

* For Settings > Roles AND Categories
** {color:#ff991f}Add a Delete Action within the action column next to the edit icon{color}
*** Delete action is only available if the role has *NEVER* had any data recorded against it (if any value is greater than 0)
*** Inactive/disabled button tooltip:
**** {noformat}This role cannot be deleted as it has been used to record hours.{noformat}
*** Confirmation prompt
**** Text:
***** {noformat}This will delete the role and it cannot be recovered. Are you sure you want to delete this role? 

CANCEL / DELETE{noformat}
* Add a Show Inactive toggle for Setting > Roles 
** Default: Off
** If On, show the roles where Status = Inactive
* History Tracking
** Add History / Audit Trail tracking for
*** Labor Hour Entry
*** Setting Changes
** User can view history via a History action
*** Add the button to the main Labor Hours page to show Labor Hour Entry history
*** Add new tab next to entry type “History” to the Settings drawer to show Settings history
*** Tracking history will be by academic year
** Standard history tracking should be fine for this

{color:#ff991f}10/22 Mini Demo discussion.{color}
{color:#ff991f}Remove “Jobs” from inactive toggle text{color}
{color:#ff991f}Edge case for adding an already deleted Role from Precise prevents deleting.{color}
{color:#ff991f}Add History Entry for history drop down on the settings page{color}
{color:#ff991f}- One for Precise History tracking{color}
{color:#ff991f}- One for Standard History tracking{color}

{color:#ff991f}Only track cell changes after initial Null > Value.{color}
{color:#ff991f}First entry can simply say user name entered values.{color}
{color:#ff991f}After first value track data changes.{color}

{color:#ff991f}Settings page > When any change is detected, display small warning banner with message “Heads up! Changes will not apply until saved.” {color}

## Acceptance Criteria
* Delete action added to Settings
** Confirmation prompt
** Button is only active if the role/category has never had data recorded against it
* Show Inactive toggle added to Settings > Roles
** Inactives are not shown by default, only when the toggle is on
* History Tracking
** Available for Labor Hours, shows changes in labor hour entry
** Available for Settings, shows changes in Method, Roles, Meal Equivalents
