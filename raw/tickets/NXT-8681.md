# NXT-8681 — ExpressPoint Website Close Session Verification

## Description
When the user selects the Close Session button the user is prompted to enter closing balance information and the data is saved. The session status is updated to Closed. 

*Mockup:*

N/A

*Requirements:*

* The use can enter closing denomination counts.
* The user can verify checks and credit card payment transactions
* The session status is marked as closed.

## Acceptance Criteria
A session is Closed for the session date and site selected when the Close Service button is selected.
The session close date is the current date.
The close session data is saved and the status is set to closed

Perseus Verification
Verify Couch DB is updated correctly
After a session is closed the user will go to the Perseus website and open the Reconciliation page.
Enter the search criteria for the site and date it was opened. There should be a session in the open state with the Opening balance and denominations entered from ExpressPoint.
The Terminal Name will be the ExpressPoint Device name, the Site and Site name will be for the site the session was opened, Opening balance will be Opening balance from ExpressPoint the Opening Date will be the date and time the session was created. 
Closing balance will be the Closing Balance from ExpressPoint, Closing date will be todays date(date the session was closed), the status will be Closed, Over/Under will be Closing Balance - Opening Balance.

When the session is Closed the denominations in the opening balance are the same as what was entered in ExpressPoint, the denominations in the closing balance are the same as what was entered in ExpressPoint.
