# NXT-67005 — Page 4: Smart Mapping

## Description
*As a* nutrition director setting up Financial Oversight
*I want to* review and confirm system-suggested mappings between my district's accounts and the system's required accounts
*So that* I can quickly complete the mapping process while maintaining control over which accounts are used for each financial category

## Acceptance Criteria
* Display header: "Map Required System Accounts"
* Include explanatory text:
** *What is this?* "Mapping connects your district's specific account codes to our system's standard categories. It's like creating a translation guide between your filing system and ours."
** *Why needed?* "This ensures your financial data flows to the right places in reports and helps track federal program requirements accurately."
* Display progress indicator: "Progress: 0 of X required accounts mapped" (updates in real-time as user confirms mappings)

*System Account Cards:*

* Display each of the X required system accounts as individual cards containing:
** System account name (e.g., "Cash in Bank", "Food Purchases")
** Description of the account's purpose (e.g., "Primary operating cash account for food service")
** Confidence badge when suggestion available (e.g., "High Confidence", "Medium Confidence", "Low Confidence")
** Suggested mapping or dropdown selector

*For accounts with confident suggestions:*

* Display "Suggested: [Account Code] - [Account Description]" (e.g., "Suggested: 11000 - Operating Cash Account")
* Show two action buttons:
** "Confirm" button (primary/purple styling)
** "Choose Different" button (secondary/gray styling)

*For accounts without confident suggestions:*

* Display dropdown selector with placeholder text "Select an account"
* Dropdown options filtered by matching base type (Asset, Liability, Revenue, Expense, Equity) of the system account
* Dropdown shows: [Account Code] - [Account Description] for each option
* Dropdown includes search/filter capability

*When user clicks "Confirm":*

* Mapping is saved
* Card updates to show confirmed mapping with green checkmark icon
* Progress counter increments (e.g., "Progress: 1 of 28 required accounts mapped")
* "Confirm" and "Choose Different" buttons are replaced with "Change Mapping" button

*When user clicks "Choose Different":*

* Dropdown expands showing all imported accounts filtered by matching base type
* User selects an account from the dropdown
* Once selected, mapping is saved automatically
* Card updates to show confirmed mapping with green checkmark icon
* Progress counter increments
* Display "Change Mapping" button

*When user clicks "Change Mapping":*

* Card returns to selection state
* If there was an original suggestion, show "Suggested:" line again with "Confirm" and "Choose Different" buttons
* If there was no original suggestion, show dropdown selector
* Progress counter decrements

*Skipping accounts:*

* User can scroll past unmapped accounts without selecting anything
* "Next" button remains enabled even if not all X accounts are mapped
* When user clicks "Next" with unmapped accounts remaining, display warning modal:
** Title: "Incomplete Mapping Detected"
** Message: "You have [Y] required accounts that are not mapped. This may affect functionality and report accuracy. Reports relying on these accounts will show incomplete or incorrect data."
** List unmapped account names in the modal
** Two action buttons:
*** "Go Back and Complete" (secondary button)
*** "Continue Anyway" (primary button with warning styling)

*If user chooses "Continue Anyway":*

* Store list of unmapped accounts
* Proceed to Step 4
* Display persistent warning banner on Step 4: "[Y] required accounts remain unmapped. You can complete mapping later in Settings > GL Account Mapping"

*Navigation:*

* Include "Exit Wizard", "Back", and "Next" buttons
* "Back" button returns to Step 2 (Import Accounts) without losing confirmed mappings
* Store timestamp when user completes this step (when they click "Next")

*Technical notes:*

* System uses semantic matching to suggest mappings based on:
** Similar keywords in account names/descriptions
** Account type matching (Asset to Asset, Expense to Expense, etc.)
* Confidence level displayed based on match strength (specific algorithm TBD with dev team)
* All confirmed mappings persist if user exits wizard and returns later
