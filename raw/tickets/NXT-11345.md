Technical Requirements:

We must remove all of the pre-determined, hardcoded column widths (px or %) for data grids. Kendo grids will automatically size the column widths. This prevents horizontal scrolling that we are seeing all throughout the system. Because we have the ability to dynamically add/remove columns, it has a poor results to the actual grid result.

We should note down each column’s representation of data. If we have many columns within the grid, the information may overflow. In this case, we should determine if the data should wrap or if we can use an overflow visual that displays first characters with a “…”

We will face the challenge of large amounts of columns. At some point, we may have horizontal scroll. However, we do have the ability to “lock” certain columns and make them always visible. We should always “lock” the action column as this is the user’s way to interact with the item. Other information can expand and scroll as needed. 



+Miscellaneous:+

Fix existing issues, such as paging & sorting not working on the grid, and the grid’s Site column not showing the full list of sites configured for that roster.
