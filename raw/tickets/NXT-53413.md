# NXT-53413 — SYS - Program Config - Overlapping dates for SSO, Configuration Options

## Description
Issue raised by Healdsberg and a ticket support - let’s link it here:

*requirements*

* Prevent overlapping dates for SSO and NSLP/SBP
** POS Meal Types should not be impacted by this - we may need to check SSO dates to allow for the SNP programs (NSLP/SBP/ASP) to be served if the pos setting for meal types via program config is on.
* Add a Configured toggle for sites in SSO
** Only Configured sites need to have dates set
** Only Configured sites will have the date boxes
* Add a text string to the SSO pop-up:
** Text: Only sites participating in SSO need to be configured. Dates in this window should not overlap with dates under NSLP/SBP/ASP. You will need to change the serving dates for those programs before setting the dates for SSO service. 
* Double check claims afterwards - we should only end up making SSO adjustments for sites that are configured + have dates

## Acceptance Criteria
* Program Config → Dates cannot overlap between SSO and NSLP/SBP
* Configured toggle added for sites in SSO
* Claims still working properly for SSO sites afterwards re: adjusting to Free
