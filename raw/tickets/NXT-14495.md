# NXT-14495 — Import-Payments 3rd Party

## Description
This Story is to cover the 3rd Party Payments @ Imports.

As an end user, the main goal is to import the 3rd party payment file as frequently as possible (currently every 10 min 24x7) with the last 4 days of data. At given point, the data in the file must of n-4 when n is the current date.

This is a simple import with 5 mandatory fields. 

File Specifications are mentioned in [https://primeroedgenext.atlassian.net/browse/NXT-19748|https://primeroedgenext.atlassian.net/browse/NXT-19748|smart-link] 

Currently the file is a text file, comma delimited, with double quotes for every field. The file is named *DistrictName_Payment.txt* (where the district name is an actual district name, NOT DistrictName: i.e. *OwenJRoberts_Payment.txt*).  

The file can be comma, tab, or Pipe (|) delimited and will have an end-of-line character (\[CR]\[LF]) at the end of every row. However if needed, we must be able to support other formats as needed.

*_Note :_* 

# The balance field is [DD.cc|http://DD.cc] and negatives are preceded by dash.  i.e. -5.25 and 105.97
# Please do *not* add headers to the file.
# Payment/transaction Date (MM/DD/YYYY) *and* (separated by a space) Payment/transaction time: Military format (HH:MM:SS) where HH is 23 hours (i.e. "02/13/2016 16:24:09")
#  Current file has end-of-line (\[CR]\[LF]) at the end of every record
