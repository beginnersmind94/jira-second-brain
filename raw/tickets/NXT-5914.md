# NXT-5914 — Messages - Broadcast Message Control Panel

## Description
*User Story*

As a PrimeroEdge admin, I would like to have a control panel that allows me to view and send Broadcast Messages to any persons or roles within my organization, so that we can all communicate more effectively.

*Summary*

Add a Broadcast Message page that allows a user to post new broadcasts (Banner messages) and view ones they have already posted.

*Requirements*

* Within the System → Administration category, add a *Broadcast Message* operation
** This is the screen for viewing and sending Broadcast Messages. 
* On the page, display a table with previously posted broadcasts. Columns:
** Audience (“Display To”)
*** Recipients/audience of the message. 
*** This can include both individual users as well as Roles.
*** Display the first 25 characters of recipients before ellipses.
**** Also display how many additional recipients are  listed but not shown. (i.e. “+5 others”)
** Message
*** Description of the sent message.
*** Display first 75 characters by default before ellipses.
** Posted On
*** *Sort the page by this column in descending order by default*
*** Date that the broadcast was posted.
** Special options:
*** -Select All-
**** -Allow user to select one, more, or all messages to perform an action on them.- {color:#ff5630}Deferred{color}
*** Action buttons
**** Allow user to Delete or -Copy- ({color:#ff5630}Deferred{color}) a broadcast when one is selected.
**** When multiple messages are selected, the only option available to the user should be   ‘Delete’.
* Also on the page, display a Broadcast Message button.
** This button will open a ‘New Broadcast’ pop-up for posting a new broadcast message ({color:#ff5630}will be handled in separate story{color})
* When clicking to view a message, a similar popup should appear with the message’s content.

## Acceptance Criteria
Verify that within the System → Administration category, we display a Broadcast Message operation

Verify that on the page, we display a table with previously posted broadcasts. 

Verify that table contains the following columns displaying the content explained in the description:
-Audience (“Shown To”)
-Message
-Posted On

Verify that we sort the page by the Posted On column in descending order by default

Verify Action (Copy and/or Delete) options are available

Verify that Action button has the options to Copy and/or Delete when only a single message is selected (functionality can be covered in a separate story if needed)

Verify that when multiple messages are selected, the only option available to the user should be ‘Delete’.

***Actions themselves are covered in separate stories***

Verify that on the page we display a Broadcast Message button.

***New Broadcast window itself is covered in separate story***

Verify that this button will open a ‘New Broadcast’ pop-up for posting a new broadcast message (UI only)

Verify that when clicking to edit a message, a similar popup should appear with the message’s content (will be handled in separate stories).
