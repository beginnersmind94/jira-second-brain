# NXT-60453 — TH - WS - Tasks Add & Edit Screen

## Description
*As a* TeamHub user *I want* a screen to assign new tasks *So that* I can stay on track of my duties

h2. Comments

* This story is for SchoolCafé (SC) districts
** PrimeroEdge districts screen stories will come in another story
* Basically, we want to mimic the tasks add new screen from SC
* This story covers 3 scenarios:
** User wants to add a new task
** User wants to update a task
** User wants to assign a task to a team member
* The app uses the same screen for both Add New and Edit and just passes a parameter to it for which mode it is in

h2. User Flows

* #1, Compose button from Tasks Screen:
** User opens the application and is directed to login screen
** On login success, user is redirected to home screen
** User clicks Workspace and is redirected there
** User clicks on Tasks and is redirected there
** User clicks the Add New button and is redirected to this screen
** System loads a blank screen (no presets) except for Assigned To
* #2, Card click from the Tasks screen:
** User opens the application and is directed to login screen
** On login success, user is redirected to home screen
** User clicks Workspace and is redirected there
** User clicks on Tasks and is redirected there
** User sees a list of Tasks 
** User clicks on a task and is redirected to the Edit Task screen
** System loads all the fields from the task
** System also shows the status
** System also shows the Assigned By field
* #3, Assign Tasks button on the Team screen:
** User opens the application and is directed to login screen
** On login success, user is redirected to home screen
** User clicks Workspace and is redirected there
** User clicks on Team and is redirected there
** User sees a list of Team members
** User clicks the Assign Task button and is redirected to this screen
** System loads all blank fields except the To field and prepopulates the To with the name of the team member

h2. Requirements

* Ensure the screen supports dark mode

_Header Section_

* Have a back button to Tasks
* Page Title: New Task

_Body Section_

* Usual Task fields:
* Title
** Textbox
** Required
** Max Length same as Title field in SC Compose screen
** Sanitize data entry
** Default blank if coming from Add New on Task screen or Assign Task button from Team screen
** Default with current value if in edit screen
* Description
** Multi-line Textbox
** Required
** Max Length same as description field in SC Compose screen
** Sanitize data entry
** Default blank if coming from Add New on Task screen or Assign Task button from Team screen
** Default with current value if in edit screen
* Due Date
** Default 7 days if coming from Add New on Task screen or Assign Task button from Team screen
** Default with current value if in edit screen
** Don’t allow past dates
** Required
* Assigned To
** List from Team Members screen
** Required
** Multi-select
** Do not allow external email addresses or ad hoc email addresses
** Ensure can’t select same team member twice
** Default current user if coming from Add New on Task screen 
** Default user from card if coming from Assign Task button from Team screen
** Default with current value if in edit screen
* Save button
** On click:
** Validate the required fields
*** If no, indicate to the user what is missing
*** If yes, continue
** Store the message in the asynchronous data store
** Trigger in code for the Task async job to run immediately
*** Note: rather than sending the task directly to the server, it saves it to the local storage and then tells the sync agent to run
** Show the user a success message
** Redirect the user to the Tasks screen

h4. *Data Requirements*

* There should be no API calls on this screen 
* All data should be pulled from the asynchronous storage on the phone
* The Save button should write to the asynchronous storage on the phone, not connect to the external server
* There should be separate code (another story) that runs independently from this screen that syncs the tasks to the server and this screen should be agnostic to that.

h4. *Performance & Usability*

* The screen must load within 2 seconds and the layout be responsive and render correctly on: Mobile, Tablet, Desktop, Compact and expanded views
* All tap interactions must be responsive, with clear visual feedback

h4. *Error Handling*

* Use try/catch in all asynchronous calls and use error boundaries for any unexpected render failures.
* Errors must be communicated using user-friendly messages:
** Use toast notifications, modals, or inline messages without disrupting the user experience
* Log any API or rendering errors to the error log system/database for further analysis



_Security_

* Any authenticated user can view this screen and take the actions on it

## Acceptance Criteria
h4. *Theme Support*

* The screen fully supports dark mode, including all headers, fields, text, and interactive elements.
* All visual elements must remain accessible and readable in both light and dark modes.

----

h4. *Header Section*

* Displays a Back button that navigates to the Tasks screen.
* Displays the page title: "New Task".

----

h4. *Body Section: Task Fields*

*Title Field*

* Rendered as a single-line textbox.
* Required field.
* Maximum length must match the Title field limit in the SC Compose screen.
* Input must be sanitized to prevent formatting issues or injections.
* Defaults:
** Blank if launched from Add New or Assign Task button.
** Current task value if opened in edit mode.

*Description Field*

* Rendered as a multi-line textbox.
* Required field.
* Maximum length must match the Description field limit in the SC Compose screen.
* Input must be sanitized.
* Defaults:
** Blank if launched from Add New or Assign Task button.
** Current task value if opened in edit mode.

*Due Date Field*

* Rendered using a date picker.
* Required field.
* Defaults:
** 7 days from today if coming from Add New or Assign Task button.
** Current due date if opened in edit mode.
* Past dates are not allowed for selection.

*Assigned To Field*

* Displays a multi-select list of team members retrieved from local storage.
* Required field.
* Users cannot:
** Add external email addresses.
** Select the same team member more than once.
* Defaults:
** Current user if launched from Add New button.
** Targeted user if launched from Assign Task button.
** Current assigned values if opened in edit mode.

----

h4. *Save Button Behavior*

* When the Save button is tapped:
** Validates all required fields (Title, Description, Due Date, Assigned To).
** If any required field is missing:
*** Displays a user-friendly error message indicating what is missing.
** If all validations pass:
*** Saves the task data to asynchronous local storage.
*** Triggers the Task sync agent to run immediately (sync logic handled separately).
*** Displays a success toast message.
*** Redirects the user to the Tasks screen.

----

h4. *Data Requirements*

* No direct API calls are made on this screen.
* All team member data and defaults are pulled from local async storage.
* Task data is saved to local async storage and queued for sync.
* Syncing to the server is handled by a separate background job, not part of this screen.

----

h4. *Performance & Usability*

* The screen must load within 2 seconds under normal conditions.
* The screen must display and function correctly across:
** Mobile
** Tablet
** Desktop
** Compact and expanded views
* All tap and form interactions must be:
** Smooth
** Responsive
** Provide clear visual feedback

----

h4. *Error Handling*

* All asynchronous calls must use try/catch for error handling.
* UI rendering issues must be caught using error boundaries.
* Any errors should be:
** Displayed to the user via toast, modal, or inline messages
** Logged to the central error log system for monitoring and analysis

----

h4. *Security*

* This screen is available to any authenticated user.
* No additional role-based permissions are required to access or use this screen
