# NXT-31558 — Add Detection Button to Camera Alignment Screen

## Description
As a cashier, I want the system to be able to be able to test food detection using a simple button rather than having to simulate a new student so that I can see if food detection is working.

*Application*: POS

*Background*

We created a similar button on the camera alignment tool page that triggers a training scan of the image.  It has been a very successful feature.  We want to do a similar thing and be able to trigger a detection and show the results.  Currently, we have to trigger a detection by going through the whole process of selecting a student, etc. and it is annoying and time consuming.

*Requirements:*

* Location: POS > Admin Screens > Camera Alignment Tool screen
*  
* We already have the upload functionality on the cart screen, so you can copy from that
* See the code in classic and copy it
* Get with Saketh and he can show you how it was done
*  
* Just like we did for the upload button, add a Detection button to the camera alignment screen
* When it detects, show the results on the screen close to the button
*  
* Security – no additional requirements necessary

*Mockup*:

* None

## Acceptance Criteria
* Detects just like on cart screen and shows the results accurately
* No new bugs
