# NXT-48692 — OpenID - District Authentication - OpenID Add New

## Description
As a Customer Support user, I want to be able to setup a district’s OpenID authentication provider so that they can use SSO

*Notes*:

* This is the screen that allows a user to set configuration for Open ID provider
* Note that they can have more than one OpenID provider

*Flow:*

* User browses to System > Security page
* User clicks Edit button on an OpenID provider
* User redirected to this drawer and completes the fields
* User tests the configuration and then saves the changes

*Requirements*:

* These requirements apply to the System > Security > OpenID Config drawer
* When a user clicks the Add New OpenID button, open this drawer
** Title: Add New OpenID Configuration
** Width: half size
** Add the sliding animation to the drawer open
* 
* Check if they have already setup this configuration in the past
** If they did, then copy that configuration and set the values for it accordingly on page load to save them time
** Maybe you do soft delete and can just re-enable the configuration type rather then doing a new one?
* 
* Fields:
* Authentication Name
** Single line textbox
** Required
** Matches to the AuthenticationName in the grid on the previous page
** Max length: 100
* Description
** Single line textbox
** Required
** Matches to the Description column in the grid on the previous page
** Max length: 50?
* OpenId Configuration Endpoint
** Textbox
** Single line, but needs to allow for a long URL
** Required
* Client Id
** Textbox
** Single line
** Required
* Client Secret
** Textbox
** Single line
** Required
** Masked like a password field
** Do the show / hide icon like a password and default to masked
* 
* {color:#ff5630}*_The rest of the fields will be determined by Dev Manager_*{color}
* 
* Cancel
* Save
** On click:
** Verify all required fields are entered
** Register with Keycloak the OpenID Connect client
** If there are errors, show them to the user
** If successful, show the user the Success confirmation message
** The grid on the parent page is updated to show this authentication type
** Don’t close the drawer after saving
** -Enable the test button-
* Test (*Hold on the testing, as it needs be configured for Keycloak*.) 
** Disabled until they successfully save the configuration to Keycloak because Keycloak must be configured first before it will work
** Button
** On click, start the login process the same way you would on the Login page whereby they have to enter their username / password for the OpenID provider and authenticate
*** But just get a successful result, do not do an actual Login
** If they are successful in logging into the provider, then everything is good
** If they aren’t able to, then they need to update the settings

* 
* Ensure changes are saved to History log
* 
* Security:
** Security screen Add permission required
** Customer Support role can always use
** Grant Security screen Add permission to SchoolCafe Admin role by default
** Deny permission to all other district roles by default



****Points to be Noted*****

* Test Authentication can be implemented afterwards as it have some dependencies on other stories. So please ignore Test Authentication points in the story.

## Acceptance Criteria
* Users are able to register an OpenID client with Keycloak
* The authentication configuration shows in the authentication grids on the parent page
* User is informed of success or failure
