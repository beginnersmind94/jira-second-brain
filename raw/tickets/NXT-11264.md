# NXT-11264 — Dashboard (Insights) - Accountability - Revenue Widget (Simple)

## Description
As a central office user or cafeteria manager I want the ability to see the revenue dollar amount generated by my site or district for the day, week, month, or year based on the time range selected for the dashboard.

The simple revenue widget will just pull in the overall dollar value of the calculated revenue. If the dashboard has a time range selection, we should pull that total value to reflect. See linked issues for the more advanced revenue widget that we will take in with release 3+.

Dashboard date range and district/site selectors may need to be created before we can update this component dynamically. If that is agreed upon, we can simply display the daily revenue amount for the district for this user story.

If a user has access to a single site only (for example, a site kitchen manager) then we can display this calculation based on the site.

If date range selector is not available, utilize current day revenue calculation.

Input value for calculating should take a date range - From/To and Site List array.

Calculation is the combined payment amount or meal pay amount (non charge) when POS transaction is performed for the sites array based on the date range. Revenue should pull in all payment types of cash, check, card, and online payment.
