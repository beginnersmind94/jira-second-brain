# NXT-2968 — System - Roles - View & Save Page Permissions for System module

## Description
As a central office administrator I want to view the details of a role to see its permissions, so that I can make changes to it that reflect the needs of my organization.

+Summary+:

When a user selects a role, allow them to view the ‘Page Permissions’ for every page within the System module and make changes to the access level of each page for that role.

+Details+:

* When a user selects a role from the Roles grid, open the drawer for that role
* The System module should be selected by default
* With the System module chosen, display all of the page permissions, which includes every page that currently exists.
* For each page permission, display checkboxes indicating the levels of control: Full Control, View, Add, Edit, and Delete. 
* Every page will have Full Control, but other permissions will vary for certain pages (which can be displayed in alphabetical order):
*# Languages - View, Add, Edit, Delete
*# Program Configuration - View, Edit
*# Reimbursement Rates - View, Add, Edit, Delete
*# Regions - View, Add, Edit
*# Security - View, Add, Edit
*# Settings - View, Edit
*# Site Configuration - View, Add, Edit, Delete 
*# Charge Limits - View, Add, Edit
*# Licensing Summary - View, Edit
*# Generate Periods - View, Add, Edit
*# Special Assistance Programs - View, Add, Edit, Delete
*# Templates - View, Edit, Delete*
*# Users - View, Add, Edit
*# Roles - View, Add, Edit, Delete
*# Data Exchange - Add, Edit
** *Some functionality and pages have not yet been added to the suite will be in upcoming Phase 1 stories.
* For the following pages, display a ‘More' button along with the following checkboxes:
** Program Configuration - Manage Sites, Manage Programs, Manage Rates
** Site Configuration - Manage Sites, Manage Grade Groups, Manage Site Types
** Templates - Manage Officials
** Users - Manage All Users
** Security - Authentication Types, Passwords
** Data Exchange - Overwrite existing configuration
* When a user checks the box of the page permissions or associated more permissions and selects 'Save’, update the permissions for the role in the database.
* Behavior/application of each permission for related pages will be in a separate story.
