## Acceptance Criteria
* Allow the placeholder units to be edited, indicating which required fields and optional fields are available for each unit being edited
* Display a banner message with instructions that update as the user enters different fields with their cursor
** e.g. in Highest Unit field “Enter the description of the highest or biggest unit of the pack sizes.”
** Base Unit: “Enter the description of the smallest pack size unit being counted in Inventory.”
** Serving: “Enter the primary serving size quantity and measure.”
** Sub-Qty: “Enter the quantity of [base units] in each [higher unit]”
** Weight/Volume: “Enter the [weight/volume] of the [pack/serving]” 
* In the editable mode, display ghost text in the Description input fields
** Highest Unit: “e.g. Case, Box, Pallet”
** Middle Unit: “e.g. Package, Bag”
** Base Unit: “e.g. Each, Carton, Bun” 
* Display the +Pack and +Serving buttons as appropriate, but hide/disable other buttons until Units are populated (or deleted) as required
** disable +Serving button until placeholder serving is populated
