# NXT-27832 — INV - Production Ordering - Display Orders outside of grid (Redesign)

## Description
As an Ordering Manager I want the placed production orders to appear on the Production Ordering screen even when I still have orderable items so I can add those items to existing orders.

Continuation from [https://primeroedgenext.atlassian.net/browse/NXT-25240|https://primeroedgenext.atlassian.net/browse/NXT-25240|smart-link] 

It is possible that orders may be placed for vendors while some items are still not yet ordered (this scenario does not occur in PE). 

*Requirements*

* On the Production Ordering screen, if a vendor is selected which has both production items still available to be ordered, but also has some orders already placed, show both sections on the screen
* Provide an easy way for users to add items to the existing orders if the items can still be added
** Ideally the items should remain the primary point of the page, but orders should be visible so the user can know whether they can add the remaining items to the existing orders
*** Note: if we do the optional banner, this would be a great place to put this info (e.g. ‘These items can be added to Order #12345’)
* Only high-level order info is necessary in this scenario, at least for immediately available info
** Important attributes:
*** *Order #* 
*** *Delivery Date*
*** Vendor (this is obviously not necessary if we continue working one vendor at a time)
** Semi-important attributes:
*** Order Total $
*** Production Start and End Date
*** Ordered By (user)
*** Order Date



Both sections (rough mockup - JM); we don’t need that much order info

## Acceptance Criteria
* Find an optimal way to display both *Production Items* and *Placed Orders* on the Production Ordering screen
* Allow users to easily add remaining items to existing orders
