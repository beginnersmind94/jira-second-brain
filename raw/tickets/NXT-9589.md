# NXT-9589 — Top Navigation - Create Search function to allow user to go directly to any page

## Description
As any SchoolCafé user, I would like to be able to quickly redirect to any page that I want to go to via a universal search option, so that I can save time by not having to get to that page via navigation menus.

*Summary*

In order to speed up site navigation, the Search menu along the top navigation can be greatly utilized. Ideally, it should search any page or person through the website.

In the first phase though, let’s at least make it so that any page, in any module, can be searched and selected from this search menu. Phase 2 can expand on this by allowing users to search for people, users, items, etc.

*Details*

* In the top-navigation, add a Search field next to the help and user icons.
** Placeholder text: Search for any page
* When the user clicks into this page and begins typing, display the top-10 results that match their text. Rules:
** Search should query the sitemap of pages within any module, as long as there is no dependency (i.e. a user cannot go directly to the Account Info page because they must look up an account first in order to get there)
** List of pages should be a ‘contains’ search. So any page where the string of text matches the page title/description should populate the list.
** For page names that are common across modules (Dashboard, Reports, Configuration, etc.), concatenate the module + page name. i.e. ‘Accountabilty Reports’, ‘Inventory Reports’, ‘System Dashboard’, ‘Workspace Dashboard’, etc.
* When a user clicks on a result in that list, redirect them to that page in the same browser tab.
** User should be able to click on a page as long as the user has permission to access that page. If they do not, we should not display that page (preferred), or make the result disabled or throw an error when selected.

## Acceptance Criteria
When a user begins typing in the Search field of the top navigation, allow them to find any page in the system **as long as they have access to it via their Role permissions**

Search should be speedy enough to be advantageous over navigating to the page through the left navigation

Verify that when a page is selected in the list, the user is redirected to that page.

Verify that if a user does not have permission to access that page, it is either hidden (preferred), disabled, or throws an error when selected
