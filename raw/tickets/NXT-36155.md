# NXT-36155 — System - Terminals - API to provide terminal data

## Description
As Matt the PM I want to help build a Terminal Management page so that they can be monitored and centrally managed to save time.

The Platform team is going to be building a Terminals page to help with monitoring all POS machines by district. We need to build the API to provide the information on what terminals are being used at the district. Future work will include transforming this data.

*requirements*

* Build an API that provides the following information for the selected district:
** Unique Terminal ID
** Terminal Name
** Device Name
** Site Name
** Site Code
** App Version
** Last Check In



*Notes*

* Last Check In
** Timestamp for the last time we ‘saw’ the POS.
* App Version
** The POS version the device was last seen using.

## Acceptance Criteria
API available for Platform team to use.

Following data is available to pull by district:

* Unique Terminal ID
* Terminal Name
* Device Name
* Site Name
* Site Code
* App Version
* Last Check In
