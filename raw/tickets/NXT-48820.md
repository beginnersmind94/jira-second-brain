# NXT-48820 — Temp User - Add New User Settings

## Description
As a SchoolCafe admin, I would like to be able to add a new temporary User so that we can have auditors in the site

*Notes*:

* These requirements assume you are doing at the same time as the OpenID story as have completed that story

*Requirements*:

* On the System > Users > Add New User drawer, do the following:
** Right now you are getting a list of all enabled authentication methods for the district
** You need to add to that the account type
** Then you need to look and see if there are any account type authentication methods for temporary and for permanent and use this as a filter for the Account Type field and the grid
* Add a Field:
** Account Type
*** Radio Button list
*** Values: Permanent, Temporary
*** Default selected: Permanent
*** Filter the values:
**** If there are no auth methods that allow Temporary:
***** Disable the temporary button
***** Put tooltip: No methods allow Temporary users. First enable temp users in a Security > Authentication > Method
**** If there are no auth methods that allow Permanent:
***** Disable the Permanent button
***** Put tooltip: No methods allow Permanent users. First enable perm users in a Security > Authentication > Method
*** Use the values selected here to filter the grid of authentication methods
**** If Permanent selected, filter grid to only rows with permanent rows account type
**** If Temporary selected, filter grid to only rows with temporary account type
* 
* Make these changes to the Authentication Type field:
** If they have selected Temporary, do the following:
** Show the following temporary fields:
*** Start Date
**** Date picker
**** Default to Today()
**** no time, just date
**** Required if visible
**** Cannot be in the past
**** Must be before End Date
*** End Date
**** Date picker
**** No default date
**** No time, just date
**** Required if visible
**** Cannot be in the past, but can be Today() or Now()
** If they have selected Permanent, do the following:
*** Hide the temporary date fields
* 
* On save:
** Ensure temporary accounts can override the District’s SSO requirements by enabling SchoolCafe simple authentication for them
* 
* Ensure changes to Account Type are added to History log

## Acceptance Criteria
* Permanent and Temporary account type is added to user stuff in the DB and Keycloak
* User can select Start and End dates and they need to be applied to Temporary accounts and used in authorization
* Existing users are considered Permanent 
* Cannot select Temp account if there is no district authentication method that uses it
* Cannot select Perm account if there is no district authentication method that uses it
* Temp account cannot select an authentication method in the grid that does not allow temp users
* Perm account cannot select an authentication method in the grid that does not allow perm users
