# NXT-30893 — Grid Component - Consistent Items Per Page Behavior

## Description
Currently, whenever a user changes the “Items Per Page” dropdown selection in grids throughout the application, changes focus to another page, then returns, the value gets reset to a default value. This results in the user having to change the “Items Per Page” value too often.

## Acceptance Criteria
* As an end-user, when I change the “Items Per Page” dropdown selection in grids throughout the application, I would like the selection for that grid to persist so that I don’t have to change it every time I return to the page.
* As a module product team member, I would like to have a grid control that has consistent and common behavior across all grids that I employ such that I don’t need to write custom code for every grid to achieve this common behavior.
* The selected value may be persisted in the browser (Meaning if the browser's data/cache is cleared, it is acceptable to return to the default value).
