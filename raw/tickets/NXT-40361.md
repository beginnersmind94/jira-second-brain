# NXT-40361 — CR & MVR - PDF - Custom Header & Footer

## Description
As a SchoolCafé user, I would like to be able to add custom headers and footers so that I can use them in our custom reports.

*Notes:*

* In this design, we have decided to create one default header and one default footer to be used by all HTML and PDF reports rather than let each report have its own formatting
* This creates some limitations but facilitates dev work and enforces UI conformity on reports
* Note that the header and footer will not be used by Excel and CSV docs

*Flow:*

* Browse to System > Settings Configuration
* Open the properties for header and footer custom configuration and edit them
* Save the configuration and test it on a custom report
* System uses this text for the header and foot when generating pdf and html reports

*Requirements*:

* On System > Configuration > Settings page, do the following:
* Add a setting:
** Setting Name: Custom Report Header
** Code: whatever you want, maybe CUSREPHEAD?
** Setting Value: the text created when edit that field
** Category: -Administration- Custom Reports (This is technically custom reports only not the administration) 
** Module: System
** Configuration Type: Common
** Default Value: to be provided in screenshot
** Actions: only history
* Add a setting:
** Setting Name: Custom Report Footer
** Code: whatever you want, maybe CUSREPFOOT?
** Setting Value: the text created when edit that field
** Category: Administration
** Module: System
** Configuration Type: Common
** Default Value: to be created when we can edit the property
** Actions: only history
* For both settings, do the following when the user clicks edit icon:
** UI/UX should be consistent with other settings
** There needs to be a description text on the popup that tells the user what system fields they can use in their custom header.
*** Example text: Available system fields: [ReportName], [Owner], [Date/Time], [DistrictName], [Module], [DataSet]
** Popup should have a WYSIWYG Editor that allows the user to format the text and insert a logo
** A comments textbox
** Cancel
** Save
* 
* When generating a pdf and html document, do the following:
** Insert the custom header
** Insert the custom footer
** Substitute system fields like [ReportName] with whatever they name the report

h3. {color:#ff5630}*Changes to the story*{color}

Initially thought was to have headers and footers in the settings page. But due the max chat limit constraints it was not recommended to have this in settings. The better approach is to move this our Templates page and combine both Header & Footer in a single template. This will allow the user to view everything in one page and also easier to make the changes. Anything above *{{body}}* tag will be considered as Header and any content below *{{body}}* tag will be considered as footer.  

Need to add a warning message saying don’t modify/remove {{body}} tag.
