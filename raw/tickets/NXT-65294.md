# NXT-65294 — ACC - Labor Hours Feature - New Page

## Description
A key metric for districts like any business operation is the ability to track labor costs and efficiency.  In order for SchoolCafe to offer this feature, we will need to implement a way to store, update, and manage labor hour entry.

Ideally there are 2 way we can support entry: One for the standard(quick) method and another for precision.

This boils down to 2 entirely different storage structures which user need to be able to seamlessly choose between based on whether they can find the time/effort worth revealing this key metric through later support for data visualizations.

This story contains the need to create the 2 storage structures(determined via in-page side bar configuration) and how they are represented on the page.

SIDE BAR options:

To handle the need for configuring the labor hours feature to the 2 main ways of use, we will be implementing an on page side bar settings.
We will need to include a new permission to be able to access this.

Add new user permission: Labor Hours - Settings. (View, Edit, Add, Delete)
When view - user can view side bar

When edit - user can select the entry method and Meal Equivalent configurations.
When add - user can add new roles/categories
When delete - user can “deactivate” roles/categories

Add drop down for Academic year in the settings bar to view each year’s configuration (default to current Academic year) at the top of the slide out.

Splash note:
Use these settings to control how Labor Hour Entry and Meal Equivalents Ratios are utilized.
Standard Entry - Simple table layout to record number of hours worked by defining and then entering number of shifts worked.
Precise Entry - In depth entry for configurable categories recorded by hours worked.
Meal Equivalents - Configure by modifying how many units required (U) to make how many equivalents of a meal (M). Used in meal per labor hour reporting.
