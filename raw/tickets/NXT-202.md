# NXT-202 — Configuration - Mark for Review Reasons

## Description
*Story*

As a FSD I need to set the POS review reasons that can be used by Cashiers to aid in reconciliation and to communicate to the manager.

*Requirements*

* There are 12 default Review Reasons
* The user can edit the text displayed for the review reason
* The user can restore the default reasons

## Acceptance Criteria
Review Reasons is launched from the Configuration from the left panel
Review Reasons is reached through the tabs of the Configuration page.
The default Review Reasons are
1 Should be Cash
2 Should be Check
3 Should be Charge
4 Wrong Payment Amount
5 Return Change
6 Add Item
7 Remove Item
8 2nd Meal Question
9 Patron Detail Error
10 Note Question
11 Restriction Question
12 Other

The text for the reason can be edited by the user.
Duplicate reasons shall not be allowed.
The reasons can be reset to the defaults if changed
Min 4 Characters, Max Text Length is 25 Characters

Message if new reason text is < 4 or > 25 "The reason text must be at least 4 characters and no more than 25 characters."

Message confirmation when resetting to defaults. "You are changing all review reasons to the default text. Are you sure?"   yes or no option
