# NXT-7124 — Menu Grid Builder

## Description
*Story*

As a user I need to be able to create menu grids for use on ExpressPoint. There are two types of menu grids, static which have menu items that don’t change and dynamic which has items that change based on menu planning. This story handles Static menu grids. 

*Mockup*

[https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=29545%3A1331&scaling=scale-down-width|https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=29545%3A1331&scaling=scale-down-width]

*Requirements*

* Grids are named, have a meal type, has at least one site type, has at least one serving day,
* Grids are enabled or disabled (disabled is default)
* Grid can be copied (Requires a New Menu Name)
* Ver 2 can be orderable.
* Ver 2 can be Meal Combo

## Acceptance Criteria
When launched by Add New all data fields are empty/blank
When launched by Edit button the data for the menu grid is loaded from the DB
Menus have a unique name Within the district.
Menu can be copied to create a new menu grid. 
A unique Menu Name is required (within the district) Message "Please enter a Name for the Menu before saving"
Meal type is required
One site type is required
One day allowed is required
Clicking cancel returns the user to the menu grid list and makes no changes
Toggle for Enabled/Disabled Default = Disabled
Toggle for Is Template Yes/No default No (Version 2)

Message for disabled menu ‘You have disabled this menu. It will not be available for serving on ExpressPoint’

Message for enabling a menu ‘You have enabled this menu. It will be available for serving on ExpressPoint 

If they try to create a menu with an existing name ‘The menu name is already in use. Please enter a new menu name’

When a menu grid is saved. 'You have saved the menu grid, please make sure to enable it when ready to use.
