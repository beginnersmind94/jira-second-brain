# NXT-56793 — Acc - Edit Check Report - Reverse Edit Check

## Description
As Dana the Director I want a way to focus on sites that are well under the expected meal count based on their monthly average, and a list of sites that fall into this category would speed up my workflow.

*requirements*

*Page Updates*

* Update the date picker to the new one
* Change the site dropdown out for a multi-select
* User can see/configure the rolling average number of meals per site created as part of [https://primeroedgenext.atlassian.net/browse/NXT-58225|https://primeroedgenext.atlassian.net/browse/NXT-58225|smart-link] 
* Add a new Additional Option to the report page:
** Highlight Reverse Edit Checks
*** User can set the % Under Average they want to highlight like they can in the other story.

*Report Updates*

* Change how we’re highlight days with meal counts over AA and over total:
** Meal count is over AA: 
*** Bold and color the following cells: Date, AA Elig, Count
*** Color: {{#FFAA84}}
** Meal count is over Eligible count:
*** Bold and color the following cells: Date, Elig (V/A), Count
*** {{Color: #FF6666}}
* When the user runs the report with the new option enabled:
** Colorize days with meal counts that fall into the % Under Average bucket, same as if they had generated an item for the Results Grid in the other story.
*** Bold and color the following cells:
*** Color: Date, Count
*** Color: {{#E3E0F8}}
* Add this information to the report footer:
** Cell colorizations: [xxx.xx] Exceeds Attendance Factor [xxx.xx] Exceeds Eligible Count [xxx.xx] Low Meal Count / Reverse Edit Check
* Show the user the selected sites when they multi-select
** Can leave it at ‘All’ if they picked all sites to save on some space

## Acceptance Criteria
* Date picker changed to the same one as the new Dashboard
* Site selector changed to multi-select 
* Reverse Edit Check option added
** User can set the % Under Average amount to be used for highlighting Reverse Edit Checks
* User can view/set the Average Meal Counts from related story
* Cells are colorized when they fall into any of the 3 exceptions - Over AA, Over Elig, Reverse Edit Check
* Review why 0 meal counts with 0 eligible students are getting flagged.
** These should only be flagged if the new Reverse Edit Check feature is enabled by the user.
* User can see the selected sites on the report, as part of report criteria
