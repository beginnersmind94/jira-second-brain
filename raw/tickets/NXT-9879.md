# NXT-9879 — Eligibility - Notifications - Applications

## Description
As a central office user I want a central location to send approval/denial notices for free and reduced applications so that families can receive the results of their application.

Utilize the Eligibility > Notifications page (see related link for page/routing of the Notifications page)

 *mockup*

!image-20210305-222242.png|width=1501,height=852!



*Requirements*

* Displayed within Eligibility (module) > Notifications (page) > Applications (tab)
* Filtering:
** -User can select an Academic Year (if needed). By default, load the current academic year.-
*** *Note 3/10/21: Academic Year selector not required, only use current academic year.*
** User can  see applications that are in status in Processed and/or Notified. Only these application statuses are available for notification.
* User has the following options:
** Exclude previously notified applications
*** This would omit any application that has previously been sent a notification after the last processed date.
** Processed Date Range
*** User can specify an optional date range that will filter results based on the dates selected vs the Processed Date for applications.
* User sees search results in a grid with the following columns:
** Application #
** Eligibility
** Basis
** Status
** Processed Date
** Processed By (First Name and Last Name of the Processor)
** Notified On (if available)
** Language
** View
*** Should display a letter icon to launch the notification viewer in order to preview.
** Notification Options
*** Print
*** Email
** Hidden columns under Filter. Guardian information should be per the Applicant.
*** Student Names
*** Applicant Name
*** Address
*** Email
*** Phone #

*Applications - Notify (Current screens for reference)*

Applications will contain the list of applications available for notification (meaning, they have been validated/processed and have not yet been notified).

!image-20210215-222727.png|width=1034,height=825!

Notification Options - We need to review and add a user story for giving notification options (as shown below). Heavily used by customers when large quantities of notifications are done (ex - during Back to School season and 1,000 applications per day are being processed). This assists the user in grouping those applications that need to be printed.

!image-20210215-222848.png|width=598,height=356!

*Future Functionality (NOT NEEDED NOW)* - Options for exporting the lists of both application and DC notifications to excel/csv. Some very large districts do not want to use the food service solution for printing, but want to send to a print shop within the school district for mass printing. This could be because they choose to print all notifications, instead of taking advantage of the email option. They will do a large mail merge with the exported file and use their own “templates”. This functionality is up for discussion and is not needed for the pilot (R2).

!image-20210215-223815.png|width=363,height=327!

## Acceptance Criteria
User has a Applications area under Notifications.

User can select an approval date range and see applications that were approved / processed in that time.
