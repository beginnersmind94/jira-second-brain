# NXT-16915 — INV - Orders - Create Orders - Apply Order Review rule checks to order creation

## Description
As an operations manager I want the Order Review rules to be applied to orders when they’re being created, so that all applicable orders will go into Pending Review status.

This story is heavy on the back end and QA. The only UI work is displaying the ‘Pending Review’ status label.

*Requirements*

* When user creates an order or multiple orders, via the Create Orders process, apply the Order Review rules to the orders.
** on Create button on Confirm Orders screen
* For each rule, check each component of the rule (And condition)
* The date range of the Order Review rules will apply to the order’s *_Delivery Date_* (not Order Date)
* If an order review rule applies to an order, put that order into Pending Review status
** these orders will be Pending Review instead of Open

!image-20210809-161134.png|width=91.66666666666666%!

## Acceptance Criteria
* Review Order rules will be checked as orders are created
* Order statuses will change from Incomplete to Pending Review if an order review rule applies
* Pending Review orders will be treated the same as Open orders (no editing from this screen, possible Merge icon, etc.)
