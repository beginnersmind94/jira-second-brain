# NXT-39435 — Student & Adult Import – Validate Email Addresses

## Description
As a SchoolCafé admin, I want to ensure we import valid email addresses so that we can have cleaner data in the system.

*Flow:*

* Open SchoolCafé and browse to Settings > Imports & Exports > Imports
* Click Add New or click Edit for a Student or Adult Import
* Complete the process of adding or editing a student or an adult

*Notes:*

* This story comes from Dallas and Customer Care team during implementation of Salem
* The goal is to try and ensure that we have valid email addresses

*Requirements*:

* The following changes are for both Student and Adult Imports, related to email addresses
*   
* Student Import
* Make the changes for the following fields:
** Email
** GuardianEmail
* 
* Put validators on the import routine that do the following:
** Require a @ character
** Require a . character
*  If either the @ or . are missing:
** Make the field empty since they do not have a valid email address format
** Log as an Optional exception so the user can know we modified their data
** Treat it the way we do exceptions for optional fields by marking as Optional exception and still importing the row
** This exception will show up in the exceptions table
* 
* Adult Import
** Do the thing for the field Email on the Adult Import that you did for the Student Import
* 
* Permissions: No changes necessary

## Acceptance Criteria
* The fields above will make the field empty if they are missing the @ or . and mark as an optional exception
* Optional exceptions still allow the row to import
* Exceptions show in the exceptions table
