# NXT-51052 — DMF - Dashboard - Migration Tab

## Description
As a Customer Support role person, I would like to be able to migrate data from PrimeroEdge to SchoolCafe using an intuitive dashboard so that I can more quickly resolve the issues

*Note:*

* This story covers the Migration tab and another story covers the Monitor tab

*Requirements:*

* Add a menu item under Imports & Exports on the left side
** Name: Data Migration
* Add 2 tabs to the Data Migration page
** Migration
** Monitor

* In the Migration tab, do the following:
* Follow the Files tab from Imports Monitor as your guide
* Include:
** Add New button
** Managed Views
** Grid
** Row Edit Button
** Row Delete button
* Exclude:
** View History (can do later)
** View Config (can do later)
** Clone (don’t need)
** Row Import button (functionality covered in Edit screen)
** Row Status button (Don’t need)
* Change:
** Column: Import Type
*** Rename to District
* Edit button:
** On click, open the Run Data Migration drawer (separate story)
* Delete button:
** Do the usual confirmation
** If yes, delete all the migration data from all the migration tables
* Add New button
** Open the Add New Data Migration drawer
* 
* Add New Data Migration drawer
* If it doesn’t exist, create it
* Fields:
* District
** single select drop down list
** Required
* Save button
** On click:
*** Verify that the district is not already in the grid
**** Don’t let them insert the same district twice
*** Save to the database and ensure that the district is inserted into the row on the grid
* 
* Security and resources:
** Ensure that Data Migration is in the resource catalogue
** Ensure that it is included in Keycloak
** Ensure it is not viewable in the Roles & Permissions
** Ensure that Customer Service and Technical Support roles have full access to this page and its drawers
** Ensure that end users do not have access to this page

## Acceptance Criteria
* Data Migration created in resource catalogue and added to the navigation
* Customer Support role and Technical Support roles only can access this page
* User can create a new data migration by clicking new
* There can be only one migration per district
* User can click on Edit and go to the Run Data Migration screen
* User can delete the data migration
