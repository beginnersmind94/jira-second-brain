## Acceptance Criteria
Navigation: GC - Item Catalog - Child Nutrition Database & Cybersoft Catalog

*Changes Required:*

Add Data Source Column in the GRID (USDA Vs CNDB Vs Cybersoft)

User should be able to search based on Search Key That has Item Name, Short Name, Product Code, GTIN, Brand, Manufacturer separated by Pipe Symbol

Use the API to render the values from the corresponding DB (CNDB_Consolidated and USDA_Consolidated)



*Usage of Serving Size:*

Serving Size API:

[https://devinsightsservice.perseusedge.com/serving_size_multiple_measures|https://devinsightsservice.perseusedge.com/serving_size_multiple_measures]

[https://qainsightsservice.perseusedge.com/serving_size_multiple_measures|https://qainsightsservice.perseusedge.com/serving_size_multiple_measures]

Integrate Serving Size API and Parse the element

The UI should reflect the Quantity 1 , Measure 1 elements, along with Unit weight

Example: GTIN # 10038034570203

 Quantity 1: 1

Measure 1: Piece

Unit Weight: 96

Weight Unit of Measure: Grams

*Single Item Import*
The user should be able to view the item at a time and click on import, the item should be created as local item
In case of CNDB or USDA Source - show ingredient only (Stock Item Tab should be hidden)

*Bulk Item Import*: User should be able to select one or more records and import 

In case of CNDB or USDA Source - create ingredient only, as stock item details are not available at the moment

*The download Exceptions* should capture all the records that are imported irrespective of any data source
