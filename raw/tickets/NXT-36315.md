# NXT-36315 — Monitoring Dashboard

## Description
As a SchoolCafé admin, I want to be able to browse to a monitoring section so that I can more easily see errors and other monitoring and health related items.

*Flow:*

* Open SchoolCafé and browse to Settings > Support > Monitoring
* Optionally click on Web Errors tab and browse to that screen
* Optionally click on other tabs and view other support information

 

*Notes:*

* This is a new page that does not exist
* We need to ensure that permissions are accounted for and that it is added to the General Permissions once done
* Make sure the new Support section is created in the left navigation

 

*Requirements*:

* Create a new page under System > Support called Monitoring Dashboard
* Add Monitoring to the left side navigation menu under Support
* Page navigation at the top is System > Support
* Page title is Monitoring Dashboard
* 
* UI / UX will eventually have cards for each sub page, but not for this sprint
** Cards will show page title and description
** Cards will link to the sub page
** Cards will show a relevant metric or data point for each sub page
* But for the purpose of this story, leave the body blank
** All the sub-page stories will address the body content on this page
* 
* Permissions:
** Only SchoolCafé admins can view this page
** In General resources screen for Roles & Permissions
*** Visible = true
*** Hierarchy: put under System
*** Title: Support
*** Default permissions: all four checkboxes checked for SchoolCafé Admin
