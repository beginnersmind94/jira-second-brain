# NXT-66796 — GL Account Import Processing and Error Handling

## Description
As a district finance administrator, I want all my Chart of Accounts to be imported even if some have validation errors, so that I don't lose my work and can fix problematic accounts later while still being able to use the valid ones.

## Acceptance Criteria
* *All accounts are imported regardless of validation results* (valid and invalid accounts alike)
* *If all accounts pass validation*: User receives success confirmation and all imported accounts are ready to use
* *If any accounts fail validation*:
** User sees specific error messages like "Expected XXX-XXX-XXX format, got XXX-XXXXX format for 77 accounts" (customized to their actual account code structure)
** User receives detailed error report showing exactly which accounts failed and why
** *User is presented with two options:*
*** *Fix and re-upload now*: User can correct the file and upload again immediately
*** *Continue with current import*: User accepts the current import with flagged accounts
* *If user chooses "Continue with current import"*:
** Display prominent warning: "X accounts were imported with validation errors and are flagged for correction. This may affect report accuracy until fixed."
** User can download their original file with an additional column indicating validation status for each account
** User proceeds to use the system with flagged accounts (can fix individual accounts later in the UI)
* *System stores audit trail data* for compliance (UI in separate story)
