## Description
On the Regions page:

# Update the page/drawer layout according to the new mock-up below
# In the ‘Parent Region’ field, display all values with their associated organization level in parentheses. i.e. ‘Texas (State)’, ‘Harris County (County)’, ‘Houston ISD (District)’, etc. We can carry forward the tree design from classic if necessary.
# In the ‘Organization Level’ field, when the user chooses a value, display the following options in the ‘Parent’ field accordingly: 

* Hierarchy:
** -If user selects ‘State’, display all Regions, Counties, Districts, Areas, and Sub-Regions related to that state.-
**(Daimien’s Note (6/3/2020): Ignore this item and button. No need to develop it since we will not handle State additions at the moment) (Daimien’s Note (8/6/2020): The 5 buttons at the top should be Sub-Region, Area, District, County, and Region.*
** If user selects ‘Region’, display a ‘State’ drop-down menu with all states and territories carried over from Classic.
** If user selects ‘County’, display ‘State’, plus ‘Region’ drop-downs
** If user selects ‘District’, display ‘State’, ‘Region’ and ‘County’ drop-downs.
** If user selects ‘Area’, display ‘State’, ‘Region’, ‘County’, and ‘District’ drop-downs.
** If user selects ‘Sub-Region’, display ‘State’, ‘Region’, ‘County’, ‘District’, and ‘Area’ drop-downs.
** The default option selected in this pop-up can be ‘Region’ (if a default is required)

4. If the user selects, ‘District’, in the ‘Organization Level’ field, display fields to input address. The required fields and validations should be the same as in Classic.

5. ‘County’ will not always have data. The drop-down menu can be optional.

6. When editing regions, only Region Name and Description can be edited. No Parent Region fields can be updated.
