# NXT-18046 — INV - Orders - Manage Orders - Receipts - Quick Receipt - Receive Step - Header

## Description
As a site manager I want the ability to create a receipt without placing an order in the system so I can receive items that I purchased outside the system.

This is the second overall step of the ‘Quick Receipt’ process. This screen will function very similarly to the existing Create Receipt screen, with some minor differences.

*Requirements*

* After the user has proceeded from the Order step of Quick Receipt ([NXT-7257|https://primeroedgenext.atlassian.net/browse/NXT-7257]), navigate to the Create Receipt screen
** this will contain all the usual section of a receipt (header, charges/discounts, items)
* Do not display ‘Add Items’ button
* Header section - order-related fields
** If the site has Delivery Locations configured, the label in the header should instead be a dropdown, since at this point there has not been a place for the user to select it.
** Order # field should display ‘TBD’ 
* Header section - same receipt-related input fields
** Transaction Date/Time (required)
*** this will function as both the Delivery Date/Time as well as the Receipt Date/Time
*** Default to current date/time
** Order Delivered By (optional)
** Invoice # (required)
** Invoice Amount (required)
*** user-input order/receipt dollar amount
** Comments
* Charges/Discounts section
** This entire section should function as it usually does
* Items section continued on [https://primeroedgenext.atlassian.net/browse/NXT-18079|https://primeroedgenext.atlassian.net/browse/NXT-18079|smart-link]
