# NXT-4052 — Security - IP Whitelist Framework

## Description
*User Story*
As a PrimeroEdge admin, I have a way to list individual IPs or a range of IPs that can specifically access the product.

*Requirements*

* System has the ability to whitelist IPs
** This is a feature that can be toggled on or off
** If enabled, only whitelisted IPs can access the site
* IPs can be added either individually or in a range
** If added in a range, all IPs included in the range have access
* Individual IPs and ranges and both be removed
* If user attempts to access from a restricted IP, they are taken to a screen saying they cannot access the site and to contact an administrator
* PrimeroEdge company IPs can always access sites
* Note: UI for this will be covered separately

!image-20201120-205830.png|width=648,height=573!

## Acceptance Criteria
* IP whitelisting can be toggled (backend only)
* IPs can be added either individually or in a range (backend only)
* IPs in whitelist can properly access
* IPs outside of whitelist cannot access
* Universal access if whitelisting is toggled off
