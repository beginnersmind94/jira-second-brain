# NXT-46264 — Imports - GDSN Connect - API Connection Config

## Description
this is unfinished, still in progress





As a SchoolCafé admin, I would like to be able to import the GDSN Connect Items Export via API from InTeam so that I can include them in our Global Catalogue.

*Notes:*

* This story is for the connection configuration part of the GDSN Connect Items feature
* The back end, import configuration and import execution parts are in other stories
* This story is for the InTeams Items export only and not the Recipes export part
* This story covers API import only, not manual or sFTP
* A lot of these requirements reference the InTeam specifications which are attached to this story

*Requirements:*

* These requirements are for the Add New SIS Configuration when click + New Config button on API (AKA Sys Auto Sync) tab:
* In the drawer, add a new Import Type
** Label: GDSN Connect
** On click: show the GDSN Connect fields and hide the SIS Provider fields
** If click other provider, then hide GDSN Connect fields and show the SIS provider fields
* 
* GDSN Connect Fields:
* GDSN Connect District ID
** Textbox
** Max length = 100
** Required
** Note: Is unique to each district we support
** Note: End user will get this from the provider website and paste it in
* User ID
** Textbox
** Max length = 100
** Required
** Note: Is unique in provider database
** Note: End user will get this from the provider website and paste it in
* First Name
** Textbox
** Max length = 100
** Required
** Note: End user will get this from the provider website and paste it in
* Last Name
** Textbox
** Max length = 100
** Required
** Note: End user will get this from the provider website and paste it in
* Test button
** On button click, use the settings and connect and verify receive authentication
** Use the same button UI/UX as for SIS Provider config
** Do the same things as for SIS Provider config
** Show the same messages
* Save
** Do same as for SIS Providers, commit settings to the database
** Don’t save w/o successfully testing the authentication
* Cancel
** Discard the configuration



Hidden Fields needed for configuration:

* End Point
** Get from provider and save it in DB or Config file somewhere
** User does not need to see this
* Client ID
** This is SchoolCafé’s ID in GDSN Connect
** Get from provider and save it in DB or Config file somewhere
** Use same for all districts
* API Key
** Needed for token, get from provider and save with Client ID
** Use same for all districts
* Grant Type
** Get from provider and save it in DB or Config file somewhere
** Use same for all districts
