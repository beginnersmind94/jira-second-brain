# NXT-54851 — Student Import - API - Preferred Name Mapping

## Description
As Dana the Director I want to update preferred name fields via the automatic import process so that students are addressed by name.

*Notes:*

* Once we are complete, FO team needs to update their code to handle the new column, so notify them once it is available
* This is basically the same thing as the File Import changes, except for the API

*Requirements*

* These requirements apply to API - Student Import
* Back End:
* Add Preferred Name to the list of fields for Student Import
* Add Preferred Name to APIs
* Add Preferred Name to all the preview and import tables
* For Validations, there are no additional validations for this field
* It does not throw any exceptions, do like Middle Name
* 
* Front End:
* On the Field Mapping tab:
** Add a Field/Row to the Mapped table and to the SchoolCafé Fields table
*** Label: Preferred Name
*** Optional (no asterisk and does not affect Save button)
*** It is not a code mapping field
*** Treat it the same way you do Middle Name on validations
*** Locate it between First Name and Middle Name
** 
** Check each provider and see if there is a preferred name field in the APIs and be sure to include it
*** If there is a preferred name, match it to the Preferred Name in Mapped table when auto mapping
* 
* View Configuration:
** Preferred Name is included in the View Configuration screen
* View History:
** Preferred Name is included in any additions and updates recorded in the history log
* Download:
** Preferred Name is in the columns when downloading if applicable

## Acceptance Criteria
* User can see and map the Preferred Name field via Student Imports
* Existing fields and functionality are unaffected
* Preferred Name is included in the View Configuration screen
* Preferred Name is included in any additions and updates recorded in the history log
* Preferred Name is in the list of columns, if applicable, when downloading
* Preferred Name is in the list of fields in the downloadable template
* Provider APIs are reviewed and if field exists, it is included in the list of fields in the first table
* If Preferred Name field exists, it is auto-mapped
