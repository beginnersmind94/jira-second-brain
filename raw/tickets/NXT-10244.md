# NXT-10244 — Accounts - Student Info - Enable Transfer Funds option

## Description
*User Story*

As an administrator, I need to be able to transfer funds between patron accounts upon request, so that parents can be able to have their children share money when one of them runs out.

*Mock-up*

[https://www.figma.com/proto/bG3v4WZvmPnKENe3Yewe7j/Eligibility-5.0?node-id=176%3A10999&scaling=scale-down-width&page-id=139%3A62442&starting-point-node-id=176%3A10999|https://www.figma.com/proto/bG3v4WZvmPnKENe3Yewe7j/Eligibility-5.0?node-id=176%3A10999&scaling=scale-down-width&page-id=139%3A62442&starting-point-node-id=176%3A10999|smart-link]

(Need to update both the Account Info → Actions and Quick Preview)

*Requirements*

* In the Account Info page of Accounts (for Students only), where there is a ‘More Actions’ drop-down menu, when a user selects the menu, show a button for *Transfer Funds*. 
** Add a Transfer Funds button to the Quick Preview as well, with the same functionality. This button should only show if the account is a Student.
* When that button is selected, open a new page where the user can transfer funds from the account selected to another account.
* Details:
** Balance
*** Display the TransferFrom patron’s current balance
*** When a number is entered into the amount field, display what this patron’s balance will be after the user completes the transaction.
*** Display the potential balance in green if it will make the patron’s balance stay above 0.00. Display it in red if it will make them go negative (<0.00). Display it in black if it will make their balance = 0.00.
** Transfer Amount
*** Allow a user to enter an amount that is less than or equal to the balance in the TransferFrom/source account. 
*** The amount should be between 0.01 and the current balance of that account.
** Transfer To
*** Allow the user to then select an account to transfer money to. (It can be any _student_ account, though Adults/Special Account search will be deferred to a future story if needed).
*** This field should allow a user to type in an ID or name of any patron in the current school district and instantly populate the field with their name and ID.
** Comments
*** A comment is required for any transaction.
* Once the requirements above are met, the user should be allow to hit ‘Apply' ('Save') button to complete the transfer.
** Or, there should be ‘Close’ and/or ‘Cancel’ buttons that return them to the Account Info page without saving anything.
* Error handling:
** IF Comments IS NULL
*** Display “A comment is required in order to transfer funds.”
** IF _Amount_ <= 0 OR _Amount_ > _Balance_ (of the TransferFrom account)
*** Display “Insufficient funds for transfer. Please enter an amount between $0.01 and the current balance of this patron.”
** IF Balance <= 0
*** Immediately hide the amount & comment fields.
*** Display “This patron cannot transfer funds because they currently have a negative or zero balance”
* When ‘Apply’ is selected, an un-editable *transaction* should then be entered into both the TransferFrom and TransferTo accounts with a transaction type of 'Transfer'.
* A *session* is also created if an open session for that site and user is not open, and a single transaction is entered into that session.
* If an *OPEN session* exists for that site and user, the transaction should be added to the open session.
* This story is complete when a user can navigate to the Transfer Funds menu from the Student Info page and perform a transfer to another account, with balance display changes and error handling where expected, and a transaction and session being created.

## Acceptance Criteria
After selecting a student account the user can select an account to transfer money to.

That field (TransferTo) displays the user's name and ID.

The user can enter an amount to transfer that is less than or equal to the source account balance, greater than 0.01.

Appropriate error handling as described in the story description should exist.

User can back out of a transaction by using Close/Cancel buttons.

Potential Balance should display and change color based on value entered into amount field

A transaction and session are created with these details:
-The transaction date is the current date
-The transaction type is Transfer Out for the source account
-The transaction type is Transfer In for the receiving account
-The comment is saved
-Both accounts have a transaction entered
-The transaction is added to the session

The Transfer Funds button is added to the Quick Preview as well. This button should only show if the account is a Student.
