# NXT-25163 — Accountability -  Default Restrictions - Allow user to set as 'Items per day' as blank (no restriction) or '0' (none allowed )

## Description
*Summary:*

The ‘Items(s) per day' field in Default Restriction is causing confusion, as well as issues in POS.

*Mockup:*
TBA

*Details:*
In Accountability → Settings → Configuration → Default Restrictions, users are unable to clear the ‘Maximum number of items per day' field, which is showing as a ‘0' 
Two options:
Simple - we should allow the field to be blank, treating a '0' entered into this field as 0 items allowed, and blank as no restriction. OR 
The note says *‘If blank, there will be no restrictions on Ala-carte menu item sales’*

The Heading is changed to '*Maximum number of Ala-carte item sales allowed per day*
Restriction takes the ‘Items(s) per day' field value if it is not blank.
If ‘Items(s) per day' field is blank, indicates that the user can purchase an infinite/unrestricted number of a la carte items per day. This would be the default value for all new students as well. 

Change the header from ‘Maximum number of restricted items allowed per day’ to ‘Maximum number of A-la-carte item sales allowed per day’

Concept UI Based on current screen notice banner/note is removed

## Acceptance Criteria
Verify that a user can indicate that there are no a la carte restrictions for a site type (infinite)

Verify that a user can indicate that a site type can have a restriction of '0' a la carte purchases allowed per day (none allowed)

Verify any value can be entered in between

Verify that the POS then takes the correct value when determining whether the site type limits are applied for alacarte purchases

Verify the header is changed.

Verify the says ‘If blank, there will be no restrictions on Ala-carte menu item sales’

Verify the Heading is changed to '*Maximum number of Ala-carte items allowed per day*
