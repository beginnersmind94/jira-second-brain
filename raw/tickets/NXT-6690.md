# NXT-6690 — Application Entry - 7. Submit: Conditionals

## Description
*Story* 

As an applications processor, i can see the eligibility outcome based on settings and configurations specific to this region.

*Requirements*

{color:#6554c0}*_INTEGRATION WITH SYSTEM MODULE FOR SETTINGS_*{color}

* ‘Last 4 Digits of SSN’ field > must enter 4 digits or check box
** Can enter SSN digits or check no SSN box
** For +income based+ apps with blank or less than 4 digits > show warning “invalid SSN”
*** SSN only required for +income based+ applications (p.81 and 42) 
** If all students are categorical or benefits using assistance program, no SSN required.
*** Show info message in the section - _This section is not required due to the basis of the application._

* Application status determination: Processed v. Pending 
** All information complete > Submit is enabled> Application goes into *Processed* status 6950
* Eligibility Panel for complete applications (without missing required fields or pending reasons) - 5228

*Settings*

* Application - Effective Date (setting) 
** Received Date > Date selector > can go as far back as the start of the program year 
*** (can back date an application)
** Processed Date > Date picker is hidden “start date” is Populated with Today’s date 
* Look at current AY price types for student 
* Start/End Dates for eligibility window should update accordingly

## Acceptance Criteria
- admin can submit a complete application
- SSN required for income based apps 
- admin has option to go to previous step 
- submit info includes sections listed above
- Multi-language, ADA compliant.
