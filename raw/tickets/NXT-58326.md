# NXT-58326 — PR - Plan/Record - History Capture of Menu changes on Plan/Record (API ONLY)

## Description
As a production manager, If changes happen on the menu, I would like to be able to see what changes were done on the menu through my plan page history so I can account for any changes and make adjustments as needed

## Acceptance Criteria
* Capture any information relevant to the Plan/Record that could be changed from the menu
* This information as follows in this order
** Menu Name Change
** Serving Groups change NOT count change
** Added Menu Item along with Serving Sizes and groups
*** Whether it is added from Menu, Menu Item Quicklist or Find and Replace, capturing this information is most important so the differentiation does not need to happen on the Plan/Record History
** Removed Menu Item
* Follow order previously implemented on the UI capture of history
