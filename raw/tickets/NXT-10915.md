# NXT-10915 — Notifications - Framework - Viewer (Preview & Send)

## Description
*User Story*

As a central office user I want the ability to view an individual notice to preview it and then have the option to trigger an official notice via email and/or print.

*Mockup*

Current proposed design from Jaiganesh.

[https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=152560%3A181926&scaling=min-zoom|https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=152560%3A181926&scaling=min-zoom|smart-link] 

(Select the Notify button)

*Requirements*

* Launch:
** Display a letter icon (visual representation) that can be selected to launch the notification viewer within grid/list views
** Notify button within the Application Details page
** Other button actions within the product
* View:
** Notification viewer should contain the letter template that is filled with the dynamic data (as a recipient would see it) based on the chosen template. By default, the active template is selected and populated.
* Template: (Can defer to a later release - R3)
** Allow for the selection of a new template that is associated with the notification type
** Selecting a new template will load the template and dynamic information to populate the notification
* Actions:
** Notification view should contain Email, Print, and Download buttons. 
*** Email is available if the application or the contained students have an email. 
*** Print option is always available
*** Download option is always available (can defer to a later release)

*Sample Viewer from PrimeroEdge Classic:*

Icon is displayed for the user to select for preview/notification. We should create a separate column for “View” (or something similar). Preview icon should not just be on the Print column.

View displays the letter as a recipient would see. Contains the notification methods of Email and Print. If email is not available, it should be disabled.

## Acceptance Criteria
User has an action on the list of notifications that lets them preview the approval letter.
Active template is shown and fields are populated with that student / application's data.
User can send the letter to print (print to PDF can be done locally).
User can send the letter out via email.

Status of the DC Match / Application should update to Notified.

Copies of the notifications are stored as-is for future reference.
These notifications should be tied to the application and student(s) involved.
