# NXT-36134 — POS - Account Sync Changes

## Description
The Accounts on the POS Terminal are currently updated using sync between Couch to Pouch at regular intervals. 

This is done using pouchdb sync function which is slower since it also syncs all document revisions and a reason of causing delay on terminals. 

We are changing this by removing the sync and updating the pouchdb for POS_Accounts using an API endpoint. 

The api would send all accounts with pagination (for removing the overload of large response data in single call) at initial load of POS Accounts. Henceforth there would be a way to send the last sync time to the server and the API would check a difference between current and the time stamp sent on payload and send only those accounts that are updated in between the difference time. 

This changes would also help reduce down the sync of revision files and only the latest/winning revision would be send on the response and would be more faster and efficient. This needs to be changed across all places on POS whenever accounts are synced including login, manual sync using device admin screen and auto sync happening at regular intervals.

Add 2 district wide settings to make the sync interval configurable for accounts and previous sales. Settings for these are POSACCSYTM & POSPSSYNTM respectively.

The changes for the current story should be enabled based on site setting ‘POSACCSYNC’. If this is not enabled, the sync for POS_Accounts would work by syncing docs with Couchbase.

## Acceptance Criteria
POS_Accounts uses an API endpoint instead of sync.

Account information updates quickly at POS at regular intervals.

Account information updates automatically while the user is still logged in.

Account information can be manually updated with the sync features under Device Admin.

Make sure the sync is happening for accounts and previous meals based on the new settings.

Make sure sync works with Couchbase when POSACCSYNC is not enabled

Make sure sync works with SQL through API when POSACCSYNC is enabled.
