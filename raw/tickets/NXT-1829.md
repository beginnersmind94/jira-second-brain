# NXT-1829 — INV - Withdrawals - Create Withdrawal - POS A La Carte items

## Description
As a Site Manager I want to be able to designate a Withdrawal as a ‘POS’ withdrawal so it can grab the list of items from the POS module for the selected date along with their quantities.

*Tentative Plan*

* POS A La Carte Withdrawals will be eliminated and those items will be routed through Production Withdrawals.
* Changes need to occur in Item Management to facilitate this functionality. These changes relate to designation of Menu Item or POS Item.

*Cross-module:* Check with POS team

Needs to be integration with the Point of Service team. This will be a direct connection with Item Management as well. The goal is to withdraw counts based on Point of Service sales done for non-reimbursable (A La Carte).

*Requirements*

* for POS withdrawals, grab all items from the selected date along with the pre-populated quantities

We need to differentiate between A La Carte items which are added to production plans vs. A La Carte items which are only through POS. 

* Speak with Matt, Muhammad, etc. about our intended way to handle these. Should all go through Production? Is there a need for other A La Carte items which are not a menu item?
