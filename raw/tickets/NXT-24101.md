# NXT-24101 — Imports - Allergen Import

## Description
This story is to cover the tasks related to Allergens-Patron Import. There are two imports related to Allergens. They Patrons and Items. This import is related to Patron.

The file is in CSV or excel formal as decided by the school districts with only two fields. Student ID and Allergy or Allergens or Allergies. Student ID is the mandatory field while the later is not. 

This is a simple and straight import. Below are the main types of allergies. Provision must be given for code mapping where they can assign the values (force mapping in case of typos like eggs to egg, fishes to fish etc) in code mapping.

# Egg
# Fish
# Gluten
# Milk
# Peanuts
# Soy
# Tree Nuts
# Wheat
# Cautesian Fish

{color:#ff5630}Note: Incase there are more Allergens than the above list, Warning label (given below) must be present at the bottom of code mapping page with steps of how to add missing allergens.{color}

{color:#00b8d9}_“Missing Allergens must be added via Item Management-->Item Configuration-->Custom Allergen Configuration (Configuration Items tab)”_{color}

{color:#ff5630}Student ID is Tab separated, Carat or Pipe delimited and allergens are always separated by semicolon.{color}

{color:#ff5630}Current email list to send the exceptions is not finalized so the only user who will get email is the user who uploads the file.{color}

## Acceptance Criteria
# Verify and Validate that Allergen Import is in the right module.
# Verify and Validate that the the fields (Student ID Mandatory and Allergens-Optional) to be mapped are per requirement for Allergen-Patron Import.
# Verify and Validate that mandatory fields are marked.
# Verify and Validate that the download template option is given after selecting Allergen-Patron Import.
# Verify and Validate that Allergen-Patron Import file should be of the format (student ID  is tab separated, and allergens as semicolon separated. )
# Verify and Validate that in Allergen-Patron Import, code look can be provided so that alternate values can be assigned to the Allergen categories.
# Verify and validate that the the configuration can be edited from Landing page or Import Dashboard
# Verify and Validate that the transactions logs can be monitored at every step using Import Monitor.
# Verify and Validate that the new Allergens are not filtered and are updated and logged accordingly.
# Verify and Validate that the presence of new allergen is reflected in the status email.
# Verify and Validate that the delimiter for Allergen Import is only Pipe, Caret, Tab delimited and Excel.
# Verify and Validate that the adding of missing allergens for the first time, option to add missing allergens is successful and the import is successful.
# Verify and Validate that during the second import, if any new allergens other than the above, the new allergen must be logged in the history - transaction log and exception must be sent in email to the user who updated the file.
