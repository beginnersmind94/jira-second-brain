# NXT-58846 — PEI - Filters and Settings - Date and Date Range controls

## Description
!image-20250131-235442.png|width=1589,height=149,alt="image-20250131-235442.png"!



Create a new API to populate the Date dropdown with these values:

!image-20250131-190604.png|width=425,height=626,alt="image-20250131-190604.png"!

Following the way we have implemented the Date dropdown in Import Monitor UI, we need to do same here.
[https://perseusapimuat.azure-api.net/dataexchange/v1.0/api/Monitor/DateRange|https://perseusapimuat.azure-api.net/dataexchange/v1.0/api/Monitor/DateRange]
Assuming that the week starts on Monday and ends on Sunday:
Following is the payload the API should return.
The startDate and endDate values from the dropdown will be used by the API to get the respective data.
For the Current Month and Academic Year To Date, the end date should be the current date.
Unfortunately, today is 01/31/2025 so the end date is the last day of the month.
When Custom Range is selected:
           Show the Start Date and End Date selector (as done in Import Monitor UI). Hide the Start Date and End Date selector for other selections.
           Make use of startDateValidation and endDateValidation values in the payload to add validation to the Start Date and End Date selector.               
            Start Date selected should be greater or equal to startDateValidation
            End Date selected should be less or equal to endDateValidation and should be greater than Start Date selected.
On page load, select Current Month as the default date.

Use this query to get the Academic Year:

{code:sql}select 
	ay.Description as AcademicYear
	,cast(ay.StartDate as Date) as StartDate
	,cast(ay.EndDate as Date) as EndDate
from 
	dbo.PR_AcademicYear ay order by ay.StartDate desc
