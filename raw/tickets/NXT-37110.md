# NXT-37110 — IM - Items - Units - Pack Sizes & Sub-Qty

## Description
As a user configuring items I want the ability to add pack size units in the Units section so I can begin building the pack size for my Inventory item.

*Requirements*

* Provide the ability to add a pack size unit in the new Units grid
* Display an ‘+ Pack’ button which would insert a new row in the Units grid
* Capture only the initial mandatory fields:

# Description
## this should be a hybrid smart text field ([NXT-37052|https://primeroedgenext.atlassian.net/browse/NXT-37052])
## we should match pack size units in the same way we will match serving description
## recommend common measures configured (Case, Bag, etc.)
### consider adding a setting to restrict users to only the configured options
### *Require at least one character before showing suggestions*
# Sub-Qty *-* only in some scenarios
## +*Sub-Qty on the same row*+
### if the current pack size being added is a second pack size, being added as a child item to (below) the existing pack size
## +*Sub-Qty on the child item row*+
### if the current pack size being added is a second pack size, being added as a parent to (above) the existing pack size
## Do not enable Sub-Qty for the first pack size being added

*General Rules:*

* Sub-Qty is for pack sizes, except the one case where serving size is linked to the base unit
* Inventory flag is triggered if Pack Sizes are added
* at least one serving size or pack size unit should be configured before moving on
