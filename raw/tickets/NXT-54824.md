# NXT-54824 — POS - Keylogging

## Description
As Matt the Product Manager I want a way to reconcile the number of IDs that have been input vs the number of completed sales/transactions. The first step here is logging all input at the POS and making that data available for review.

*requirements*

* Add a POS Setting to enable keylogging
** Setting: Enable Input Logging
** Default: Off
** Setting Code: POSKEYLOG
* Add an option to turn this on via the Device Management page for a terminal
* If enabled:
** Log all key entry at the POS
*** All ‘text’-entry needs to be logged, ideally with carriage returns for easier parsing
*** Log all input strings from devices
** Transmit keylogging up and stored with the session it was part of
** Logs available for review as part of Session Details
*** Button available to view data
**** Overlay with grid
**** Excel download
*** Tie button visibility + access to the SchoolCafe Administrator role
*** If no log is available, button status = Disabled
**** Tooltip: No device input log is available for this session.
** Data to track:
*** SessionID
*** Timestamp
*** Text Input
*** Input Device
