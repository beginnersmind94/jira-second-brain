# NXT-61057 — AI Onboarding - Digital Recipe - AI Analysis (TITAN)

## Description
As a customer support person, I would like to use AI to digitize a recipe document so that I can speed up onboarding

*Notes:*

* This story covers the backend process of analyzing a recipe using AI
* It is triggered when the user clicks the Analyze button on the Recipe AI screen

*User Flow:*

* User browses to Item Management and then Recipes
* System checks if the district has enabled AI and shows the AI button if they have
* User clicks on the AI button and it opens the AI Recipe drawer
* User ensures the New chip is selected
* User selects what type of file they are uploading
* User selects a file to upload and clicks upload
* System saves the file (in blob storage?) so that it can be used in the AI analysis and Previous files process
* System looks at the format selected and chooses which prepared prompt to pass to the AI engine
* System passes the file and selected prompt to AI engine for analysis
* System gets back the analysis from the AI engine
* System saves the file upload info
** Records the file upload and meta data in the database generating a unique ID
** Saves all the data needed to support the Previous files process 
** Saves the results of the analysis so that it is usable for the edit recipe process
* Note: this story goes only this far. What happens for previous and editing are covered in other stories

*Requirements:*

* *Documents*
** Get copies of real recipes from competitor systems
*** Most important is Titan an Nutrikids
** Note: Documents can have N number of recipes
** Note: document formats are usually PDF, but not always, could be word or even images
** Excel:
*** We will create and provide a sample Excel document with headers that users can manually enter data  that you can use to train for Excel
** Custom:
*** When user selects custom, it is a format we don’t support, but the user wants to try it anyways and the results are whatever they are, so we will upload it without format specific training model
* 
* *Training*
** For each format type: 
*** Train the AI engine on how to read them
**** Note document formats don’t change per vendor, just content
*** Save the training to make it reusable 
** Ensure the AI engine returns the results in a predefined format (JSON?) that is repeatable and usable in our code so that the results can be saved to the database and used for the editing screen
* 
* *UX update*
* On the Report AI screen, do the following:
* Update the Analyze button:
** On click: 
** Get the right prompt based upon the format type
** Send the file and correct prompt to the AI engine
** Show a Analyzing loader message on the right side
** Get the results back
* When you get the results back
** Commit the file upload and analysis to the database
** Send the user a toast message that the analysis is complete
** Note: Don’t commit before getting the results back, or it will create additional headaches about what to do if it didn’t work, etc.
** Note: Do not try to save any data into existing Recipe database tables
*** You need your own Recipe AI tables for all of this
*** Do not attempt to save to real recipe tables yet
*** You will save to the real recipe tables using the submit button (another story)
*** These tables are working tables for the Recipe AI feature
** Save the results to the database
*** Save the file upload information
**** File to blob with link to it
**** File name and info and link
**** User and date/time stamp
**** Format selected
**** Default a Status to New
**** How many recipes there were found in the file
*** Save the recipe information
**** There is a many to one relationship of recipe to file
**** Each recipe should have a status (default to New)
**** Save the basic Information and Settings values per recipe
**** Saves the steps per recipe
**** Save the ingredients per recipe and per step
* 
* Note: this story stops here. What to do with the saved information picks up in the Previous, File Analysis stories
