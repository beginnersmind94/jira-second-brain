# NXT-23323 — IM - Storage  Category Maintenance

## Description
As a user I need to maintain the Storage Categories that are going to be use in order to keep consistency thru processes assignment of storage categories as well as to keep up-to-information

## Acceptance Criteria
This story is done when:



*Entry Point*

* A  Storage Categories option is added  under IM configuration  area
* Option Storage Categories

*Result Grid*

* Should display fields as per mockup in Figma link provided  on comments
* -User should be able to filter and sort fields on result grid- 
* User should be able to edit Storage Category  \[Description Only]
** Only for user created storage categories (e.g  CN cannot be edited or deleted)
* User should be able to delete the Storage Category only if there are no items/recipes using it
** User should be presented with a confirmation message “Are you sure you want to delete this Storage Category” is Yes proceed with deletion if no cancel process

When Slide Out Card is displayed

* Fields on slide out card:
** Storage Category Description \[Free text box] 
*** Is a Mandatory field
*** Only can be added to user/district created data source (e.g. Local)
** Data Source \[Dropdown box  that list available data sources to the user]  
*** Is a Mandatory field
*** Only one Data Source can be selected
* When pressing the *SAVE* button 
** Validation occurs
*** Storage Category is Mandatory and no duplicates are allowed
*** If the Storage Category already exist warn the user with a message “Storage Category already in use, please use a different Category name and allow the user to enter a different name”
*** One Data Source is Selected (Mandatory)
*** The newly added Storage Category is listed on the result grid
***  system returns to Storage Category screen
* If User clicks on the *X icon* then the Slide Out Card is close nd transactions are canceled
* Pagination should be present
* *Export* \[PDF Format]
** Title \[Storage Category Configuration]
** Should cover same fields as the result grid \[Storage Category Description, Data Source Name]
** Header
*** Updated by \[User Name]
*** Generated on \{System date/time]
** Footer
*** Power by SchoolCafe for \{School District Name]
*** Pagination should be present

*General:*
Once user selects configuration a po-up window is presented to the user. This pop-up windows asks for data source and  configuration option desired. When user click/press apply button then system re-directs the user to appropriate configuration area
