# NXT-4977 — AcctMgt - Patron - Adult Details - Documents/Notes

## Description
*User Story*

As a district admin I want to see the information on an adult’s account.

*Mock-up*

[https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=418732%3A10&scaling=min-zoom|https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=418732%3A10&scaling=min-zoom|smart-link] 

*Requirements*

* Add a Documents & Notes panel for adult accounts.
** *UI look and feel can essentially be the same ‘Add Adult’ document upload:* [https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=418711%3A2046&scaling=min-zoom|https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=418711%3A2046&scaling=min-zoom|smart-link]
* Section will show any documents / comments attached to the account. 
** Show information in a grid with the following columns: 
*** Documents (show size)
**** In View mode, allow user to click the icon to download the file without having to go to Edit mode.
*** -Type-
**** {color:#ff5630}Can be ignored{color}
*** Comments / Notes
**** Can also be shown on separate rows, since they do not necessarily need to be associated with a document.
*** Added
**** User FirstName + LastName + Timestamp
* User needs a way to edit the available documents. 
** Each item has a delete option
*** Deleted items should remain in the database and be flagged as deleted.
* Attachments
** Show the file name, file size.
** Provide a thumbnail preview of the document.
** Clicking the thumbnail should download the document.
* User can add new comments / attachments.
** User can enter just a note / comment.
** User can select if this is a general (default) or a medical note / attachment.
** User can attach a single document.
** User has the option to post or discard.
** New items are added to the grid.
* Actions here need to be logged in history. If files are removed, what was the file name, file size, who, when, etc.

## Acceptance Criteria
Documents/Notes panel added for adult accounts.
Comments/docs can be viewed appropriately and interacted with per requirements.
Multi-language support enabled (LANGUAGENAME_ENGTEXT ie: FR_Students, FR_ADULTS)
Accessibility / ADA Compliance programmed appropriately (full KB nav possible, etc.)
