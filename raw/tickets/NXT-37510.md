# NXT-37510 — Serving Size  - API Integration (DB - Enabler)

## Description
_(none)_

## Acceptance Criteria
Navigation: Menu Planning - Item Catalog

The serving size API should be integrated in the backend

The serving size value and measure should be populated based on the API (attached below) outcome

The mapped values in UI should be used when user click on Import (Regression)

The created serving measure should reflect in Ingredients screen (Regression) 

Approach:

Integrate the API at 1WS interface level and update the serving measure for all the new entries

Add a new DB which has revised serving measure created against each item ID

The item shall have multiple entries against each item ID. 

The API may return up to 5 measures
