# NXT-14492 — Eligibility Import

## Description
As Vera the Director I want to import a snapshot of eligibility for all my students so that their benefits can continue with the new system and there are no empty stomachs at the cafeteria.

*requirements*

* User can select an eligibility start date for the file.
** This would allow the user to backdate information.
** User cannot select dates in the future.
** User cannot select dates outside of the current academic year.
** When inserting new eligibility records, this will be the start date.
* User can choose to retain better benefits or overwrite everything.
** If selected, any better benefits would be retained:
*** A student who is already Free would not be updated to Free, Reduced ,or Paid, or have their Reason updated.
*** A student who is already Reduced would not be updated to Reduced or Paid, or have their reason updated.
** If the user selects to overwrite, then all students are updated.
* User can upload a file (file example attached) that contains:
** Student ID numbers
** Student names
** Eligibility
*** This could be either a code or text.
**** 1 / F / Free
**** 2 / R / Reduced
**** 3 / P / Paid
** *It would be nice if this was configurable instead of being a set format.*
* *_User can see the impact of the import and a summary of information before approving it._*
** *_Current figures:_*
*** *_# of students, how many are free, reduced, paid_*
** *_File analysis:_*
*** *_# of students in the file, how many are free / reduced / paid._*
*** *_How many actually exist in the DB (identified) vs not found (unidentified)_*
** *_Changes if the import is approved:_*
*** *_How many students receive a better benefit?_*
**** *_How many changes - how many move Paid → Free, Paid → Reduced, Reduced → Free_*
**** *_How many reductions? Free → Reduced, Free → Paid, Reduced → Paid_*
*** *_How many updates will be performed?_*
* *_User can approve or reject the import after reviewing the statistics / impact report._*
