# NXT-66238 — ACC - Labor Hours - Copy Settings

## Description
As Dana the Director I do not want to re-write my roles and settings each year. It would save time and increase my product satisfaction if I could copy my setup year over year.

*requirements*

* User can copy configuration from one academic year to another.
** Standard copy format used for pages like Program Configuration & Other Benefits is fine for this. (follow from and to limits)

!image-20251007-135635.png|width=637,alt="image-20251007-135635.png"!
** Copy over the Settings
*** Entry Method 
*** Roles
*** Meal Equivalents 
** User can only copy TO current or future academic years.
** History Tracking should have an entry for this action taking place.

What to do with existing data?
Roles - Exception if same name offer choice to keep or rename by appending (Old)
Meal Equivalent - just overwrite

Common text for copy pop up:
“This will let you copy configured Category/Roles and Meal Equivalents from one academic year to another. Please note that this cannot be undone.”

Give confirmation message “are you sure” when copying.

{color:#ff991f}You are about to copy the settings from [DATE “2025”] to [DATE “2026”]. All existing configuration in [DATE “2026”] will be lost. Are you sure you wish to proceed in copying? “Yes Copy and Save” “No/Cancel”{color}

## Acceptance Criteria
* User can copy configuration configuration year over year
** Can only copy TO current or future academic years.
* Copies:
** Entry Method, Roles, Meal Equivalents
* History entries added if the user performs the copy action
