# NXT-5249 — Other Benefits - Add/Edit/Delete

## Description
*User Story*

As a district admin I want to be edit or configure a new benefit for applicants.

*Mockup*

[https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=5343%3A3&scaling=min-zoom|https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=5343%3A3&scaling=min-zoom]

*Requirements*

* Add an Edit option to all entries in the Departments and Benefits grids.
** This should load the existing data into an editable side-drawer (or the new template equivalent) for editing.
** Editable details:
*** Departments
**** Name
**** Code
**** Active status
*** Benefits
**** Name
**** Description
**** Department
***** Likely want a drop-down here with all active departments for that academic year shown.
**** Active status
** Cancel/Save buttons.
* Add an Add option to the Departments grid.
** User needs to fill out the following:
*** Name
**** No duplicates for the same year.
*** Code
**** No duplicates for the same year.
** Cancel/Save buttons.
* Add an Add option to the Benefits grid.
** User needs to fill out the following:
*** Name
**** No duplicates for the same year should be possible.
*** Department
**** A selection must be made.
*** Description
**** English is the default here, but they’ll need a field for every active language configured for that district.
**** If they have 5 languages setup, they’ll need a field for each language.
***** English Description, Spanish Description, French, etc.
** Cancel/Save buttons.
* Add a Delete option to all entries in both grids.
** This needs to check for dependencies.
*** Departments - is there a Benefit that is tied to this? If so, it can’t be deleted.
**** Delete button should be disabled with hover text:
***** Please remove this department from all benefits before deleting.
*** Benefits - is there a student who has this benefit on their account? If so, it can’t be deleted.
** This will also need a confirmation box.
*** Are you sure you want to delete this benefit/department?
**** Options: Cancel, Delete.



Discussion 11/12/20:

* Only active departments are available while adding a Benefit.
* Inactive departments may show while editing a Benefit, IF the currently selected department is inactive.
** Otherwise, only active departments should be available.
** We need a visual indicator for the selected department being Inactive.
*** Example: Libraries (Inactive)
* If no text exists for a particular language description, show the following:
** No description is available for this language.
* The delete icon for a department needs to be disabled (this might change) if the action cannot be taken.

## Acceptance Criteria
Departments/Benefits can be added.
Departments/Benefits can be edited.
Departments/Benefits can be deleted.
Dependencies for blocking deletion are in place.
Benefit descriptions pull based on active languages.
