# NXT-68633 — SC - MT - Resource Catalogue Update

## Description
As a Customer Support role user, I want to be able to control visibility of pages related to a Central realm so that users don’t see pages they don’t need.

*Notes*

* This story covers which resources (pages) Central Users can see in their own domain
* It does not cover what they can see in children domains
* This value will be used by pages in order to show/hide pages to Central users in their own domain

*Requirements*

* On the System > Management > Resource Catalogue page, do the following:
* On the + New and Edit screens, do the following:
* 
* Add a new field
** Type: Multi-Select
** Label: Realm Types
** Values: District, Central
** Default select District on + New
* On all existing resources, select the District value
* 
* Update the menu navigation to look for this field
* If the user is logged into a district, ensure the resources that aren’t district are hidden from navigation
* Do the same for Central - only show resources that are selected for Central

## Acceptance Criteria
h1. 

h2. *1. Resource Catalogue Enhancements*

h3. + New / Edit Screens

* Add a new field to the *Resource Catalogue* form:
** *Type:* Multi-select dropdown
** *Label:* {{Realm Types}}
** *Values:*
*** {{District}}
*** {{Central}}
** *Default Selection (on + New):* {{District}}
** *Behavior for Existing Records:*
*** Automatically assign the value *District* to all existing resources.

h3. Field Behavior

* Multiple realm types can be selected to indicate which types of users (by realm) the resource should be visible to.
* The field should be editable by users with permission to manage resources (e.g., Customer Support role).

----

h2. *2. Menu Navigation Updates*

h3. Navigation Logic

* Update the application’s *menu-building logic* to filter visible resources based on the {{Realm Types}} field for the currently logged-in user.

h4. *When User is in a District Realm:*

* Show only resources (pages) that include {{District}} in their Realm Types selection.
* Hide all resources that do not include {{District}}.

h4. *When User is in a Central Realm:*

* Show only resources (pages) that include {{Central}} in their Realm Types selection.
* Hide all resources that do not include {{Central}}.

h3. Mixed Realm Access (if applicable)

* If a resource has both {{District}} and {{Central}} selected in its Realm Types, it should be visible to users in either type of realm.

----

h2. *3. Data Management*

* Ensure {{Realm Types}} data is stored in the Resource Catalogue metadata.
* The value should be retrievable via the existing resource API for use in navigation logic.
* Ensure all related systems that load or cache navigation respect this field.

----

h2. *4. Permissions & Security*

* Only *Customer Support* and *Technical Support* roles can edit the {{Realm Types}} field.
* Other users (e.g., District or Central Admins) can view the field but cannot edit it.
* Changes to {{Realm Types}} must be logged in the *History Log*, capturing:
** Resource name
** Old value and new value
** User who made the change
** Timestamp

----

h2. *5. Validation & UI*

* If no realm type is selected during resource creation, show a validation message:
** _“Please select at least one realm type.”_
* Ensure the new field appears directly under existing fields like _Category_ or _Module_ for visibility.
* When editing, the previously selected realm types should prepopulate correctly.

----

h2. *6. Performance*

* Menu rendering with realm-based filtering must complete in under *2 seconds*.
* Navigation updates dynamically when switching between realms (e.g., from Central to District context).
