# NXT-67532 — Story 1 — Account Code Structure (UI Build)

## Description
*As* a district business administrator
*I want* to define and manage the account code structure
*So that* my district stays compliant and can report at site level

## Acceptance Criteria
----

h3. *A. Header Panel (Informational Banner)*

* Title: *Account Code Structure*
* Collapsible info box at top with this text (copy/paste-ready):

{quote}*Important Setup:* Think of your account code structure like creating a filing system for all your school nutrition transactions. This determines how every dollar coming in and going out gets categorized and tracked.

*What this affects:*

* How your financial data appears in state reports and board presentations
* The way meal program transactions are recorded and tracked
* Your ability to see spending and revenue by school, program, etc.

*State-Based Setup:* We’ll suggest a structure based on your state’s requirements, but you can modify it to match your district’s specific needs.

*Required Site Tracking:* Every account code includes a Site segment so you can track transactions by individual school location. This cannot be removed but can be repositioned in your code structure.

*Example:* {{199-3500-6100-001}} might represent General Fund – Food Services – Food Purchases – Lincoln Elementary

*Why this matters:* This structure helps you answer questions like “How much did Lincoln Elementary spend on food last month?” or “What’s our district’s total breakfast revenue?” The better organized this is up front, the easier your reporting will be all year.

*Before you start:* Check that this matches your district’s chart of accounts and state reporting requirements.{quote}

----

h3. *B. Add New Segment Form*

||Field||Details||
|*Description*|Text, required, ≤ 50 characters, unique. Character counter shows remaining chars.|
|*Type*|Dropdown: Numeric / Alphanumeric.|
|*Segment Length*|Positive integer.|
|*Separator*|Single character or “none”.|
|*Add Segment* button|Appends row to Segments table (bottom of current list).|
|*Load State-Defined Template* section|Dropdown “Select State” + disabled “Apply” button until state is chosen (handled in later stories).|
|*Character counter for segment*|Displays “X of 50 characters used”.|

----

h3. *C. Segments Table*

||Column||Behavior||
|Position|Drag handle (::) to reorder segments; order immediately reflected in preview.|
|Segment Description|Text (label). Site row is read-only for name.|
|Type|Badge (Numeric/Alphanumeric).|
|Segment Length|Displays configured integer.|
|Separator|Shows character or “(none)”. (special character only)|
|Segment Characters|Auto-calculated (length)|
|Edit|Pencil icon → inline edit modal.|
|Delete|Trash icon → confirmation modal; disabled for Site.|

Additional Rules:

* *Site segment* always present on first load as the first and only segment; cannot be deleted or renamed but may be reordered.
* Adding/editing segments re-computes total characters per row and preview in real time.
* Inline validation appears on blur or Save.
* No character limit on the total structure.

----

h3. *D. Live Account Code Preview*

* Shown below segments table.
* Updates immediately when segments change (order, separator, type, length, or Site settings).
* Displays two examples:
** *-Account Code Example:-* -structure without Site.-
** *Site Account Code Example:* appears only when “Include Site segment” is checked and shows Site placement + separator toggle effect.
* Help text below:
Site segment uses numeric site code (e.g., 345 for NHS). Site sub-codes are appended to create sub-accounts (e.g., 1000-2000-3000-345-50000.NHS).

----

h3. *-E. Site Segment Options-*

-(two checkboxes immediately below preview)-

# *-Include the Separator that comes before the Site segment in the Account Code-*
- – Visible only when “Include Site” is checked.-
# *-Include the Site segment in the Account Code-*
- – Checked by default.-
- – Unchecking hides the Site example and disables #1.-

----

h3. *F. Validation & Warnings*

* Description ≤ 50 chars and unique.
* Structure check before Save
** If the *total number of segments (including Site)* is *less than or equal to 3*, display this *gentle warning* (non-blocking):
**“Please make sure this structure matches what your finance team expects.”*
** This message appears just above the Save button in a yellow banner (not a popup) and does *not* block saving.
* Inline errors for invalid length or separator (> 1 char).
* Unsaved-changes guard (on navigate away).

----

h3. *G. History Panel (separate story but FYI):*

Located below the Add New Segment form or as a side drawer.

* A new version is created *only when the user clicks “Save Changes.”*
* Display at bottom of page: “"Last Updated on September 24, 2025 at 4:22 PM. View changes in the [History] tab."
(hyperlink on "History" opens the History tab in General Ledger Accounts page)
