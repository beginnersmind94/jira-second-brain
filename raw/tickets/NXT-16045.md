# NXT-16045 — INV - Create Orders - Order Review - Allow Delivery Date to be overridden by permission

## Description
As a district admin I want the ability to override the Delivery Date so I can place orders to vendors outside the calculated delivery schedule.

*Note:* check with Platform team about creating new permission, or other page in System module containing non-page access permissions

*Requirements*

* Add new permission titled ‘Override Delivery Schedule’ 
* If the user has the Override Delivery Schedule permission…
** on the Order Review screen:
*** Provide an Edit icon next to the Delivery Date field
**** *Only for orders in Incomplete status
** on the Order Details screen: 
*** Provide an Edit icon next to the Delivery Date field
**** Do not display this if the order is Closed or Cancelled
**** Do not display this if any receipts exist for the order
* If user clicks the Edit icon:
** convert the label into a date picker
** replace Edit icon with green checkmark (save) and red X (cancel)
* Allow the user to pick any date from the date picker
** Show a warning icon (or highlight in red) if the newly selected date does not correspond to a valid delivery date
*** (!) “The selected Delivery Date is not valid according to the vendor delivery schedule.”
*** User can still save the new date
** Show another warning if any of the product lead times are being overridden by the change in delivery date
*** (!) “The selected Delivery Date is not valid according to some product lead times.”
*** User can still save the new date
** Do not allow user to save delivery dates outside of the contract date range ([NXT-16396|https://primeroedgenext.atlassian.net/browse/NXT-16396])
*** (!) “The selected Delivery Date is outside of the contract date range.”
*** User *cannot* save in this case
