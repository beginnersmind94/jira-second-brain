# NXT-58289 — LMA - District - Add Devices

## Description
As a SchoolCafé Admin role member, I would like to view a list of device linked to a license so that I can know what we have access to and when they will expire.

*Notes:*

* This is a continuation of the Dashboard story and covers everything that happens when a user clicks on the Show Devices button
** It works like the Show Sites
* Users are District users and Customer Support role
* In this area, we can see what devices a license is linked to
* Requirements are written assuming the drawer will be only usable for Device licenses (Not Sites, covered in another story)
* There are currently only 2 licenses with a device type: Vending and SCTV+
** Vending licenses are managed with the device manager screen under System
** SCTV+ are not managed in SchoolCafé currently at the time of writing this story, so may not be available when coding, so TBD if it will be included in the code

*Flow:*

* LMA role user browses to System > District Licenses page
* User sees a list of licenses generated for the district
* User clicks the Show Devices button which opens the View Devices panel
* User sees which devices are linked to the license
* User optionally adds a device
* User optionally removes a device

*Requirements:*

* These requirements apply to System > District Licenses page:
* Add a button to the License Types card if not already there
** Label: Show Devices
** On click:
*** Expand the Devices panel
*** Toggle the label to Hide Devices
*** If click Hide Devices, hide the panel and toggle the label back to Show Devices
** Only applies to licenses with a device license type (Vending, SCTV+)

*Show Devices panel*

* If there are no Devices assigned, show a message explaining to click the Add Devices button
* Add Devices button
** Permission:
*** Customer Support and LMA role users can click it
*** District users with Edit permission to the page can click it
*** Else, disable the button
** On click:
*** Show the Add Devices popup (see below)
** Business logic:
*** If the district has filled the license count (used all the available licenses), we cannot allow them to add more devices to the license. They can’t add more devices than they have purchased. So, they must remove a devices to add a devices.
** If there are no available licenses, hide or disable the Show Devices button and instead show a message that the max has been achieved
* Assigned Devices list
** Data: list of sites that have been linked to the license
** Fields:
*** Name of the device
*** Site
*** Delete button
**** On Click:
***** Show confirmation button
***** If confirmed:
****** Unlink the device from the license
****** If they had reached the max licenses, show the Show Devices button again

*Add Devices popup*

* If the popup does not already exist, create it
** Title: Add Devices 
** Use animation
* 
* Fields:
* Available Devices availability
** Indicate in text how many devices they can add
* Add All
** Business Logic: User cannot add more devices than available licenses
** On click:
*** If there are enough available licenses:
**** Add all the devices to the license
**** Close the popup
**** Update the linked devices list on the main page with the new devices 
*** If not, then disable the Add All button
* List of Available Devices 
** Data: list of devices that have not been linked to the license
** Shows:
*** Name
*** Site
*** Add Button
** If a device is already added, do not show it
** On Click:
*** Link the device to the license
*** Hide it from the available devices list
*** Show it on the list of linked devices in the main page
*** If you reach the max count of available, close the popup

*Security*

* Customer Support role and License Management role can do anything
* District Licenses page permissions from R&P, so ensure user is a role with that permissions
** Edit permission required for Add Devices

## Acceptance Criteria
h3. *1. Show Devices Button*

* Add a *button* to the *License Types card* if it doesn’t exist.
* *Label*: {{Show Devices}}
* *On click*:
** Expand the *Devices panel*
** Toggle label to {{Hide Devices}}
* *If clicked again* (Hide Devices):
** Collapse the panel
** Toggle label back to {{Show Devices}}
* *Applies only* to licenses with a *device license type* (e.g., Vending, SCTV+)

----

h3. *2. Devices Panel*

* *If no devices are assigned*, show a *message*: “No devices assigned. Click Add Devices to link devices to this license.”
* *Assigned Devices list*:
** *Fields*:
*** Device Name
*** Site
*** Delete Button
** *Delete Button behavior*:
*** Show confirmation popup
*** If confirmed:
**** Unlink device from license
**** If user had reached max licenses, re-enable {{Show Devices}} button

----

h3. *3. Add Devices Button*

* *Permissions*:
** Clickable by *Customer Support* and *License Management* roles
** District users with *Edit* permission for this page
** Disabled for others
* *On click*:
** Open *Add Devices popup*

----

h3. *4. Add Devices Popup*

* If it doesn’t exist, *create it*
* *Title*: {{Add Devices}}
* *Animation / sliding motion*

h4. *Fields / Logic*

* *Available Devices*:
** Indicate how many devices can be added (available licenses left)
* *Add All button*:
** *Business Logic*: Cannot add more devices than available licenses
** On click:
*** If enough licenses → add all devices, close popup, update main page assigned devices list
*** If not enough → disable the button
* *List of Available Devices*:
** Show *Name*, *Site*, and *Add Button*
** If a device is already added → do not show it
** On *Add click*:
*** Link device to license
*** Hide from available list
*** Show in main page assigned devices list
*** If max license count reached → close the popup

----

h3. *5. Business Logic*

* *Cannot assign more devices than purchased licenses*
* *If max reached*:
** Hide or disable {{Show Devices}} button
** Show message that maximum has been achieved

----

h3. *6. Security / Permissions*

* Roles allowed: *Customer Support* and *License Management*
* District Licenses page permissions apply (from R&P)
* *Edit permission required* for *Add Devices* and device assignment
* Users without required role/permission → buttons disabled
