# NXT-38997 — GC Database - Items Publishing Qualifiers (Filter usable items based on exceptions)

## Description
_(none)_

## Acceptance Criteria
Introduce a new table as “I*tem Publishing Qualifiers*” and capture the flag against each item ID

The following flags should be introduced against each item irrespective of any data source:

Item ID / Code

*Is missing Description:* Flag as 1 if either one of the field is missing at item level:

·        Ingredient Name is not provided



*Is missing Serving Size Info:* Flag as 1 if an 

 item is missing with serving size information (Description)



I*s Missing nutrition Info:* Flag as 1 if an item does have entry on Nutrition Basis Table but missing with following fields:

 ·       Food Energy (KCal) is not provided

·        Total Fat is not provided

·        Saturated Fat is not provided

·        Carbohydrate is not provided

·        Protein is not provided

*Is missing Manufacturer* info: Flag as 1 if an item is missing the manufacturer 



*Is missing Brand Info:* Flag as 1 if an item is missing the Brand



*Is invalid product type:* flag as 1 for product type: DS, PL and MX (applicable for 1WS data source only) 

*is discontinued:* Flag as 1 if either of the following is met

·       for CNDB -  if item Status = d

·       for USDA Branded Food = discontinued_date !=NULL (based on max (available date))

·       for 1WS = gdsnregistorystatus = Discontinued or cancelled

*Is exception:* Flag as 1 if either of the above qualifier is 1

Data Source value should be captured against each record (Ex: 1,2,3)
