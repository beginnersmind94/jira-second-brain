# NXT-5715 — INV - Create Orders - Enter Item Quantities (Step 3)

## Description
*User Story*
As an Inventory user I want a list of the items returned by my delivery and item selections so I can create an order.

Third step of the Create Order process, after user has provided Delivery information and Item Population.

+*Smart Search/Add Item*+

User should have access to a smart search & add feature that could be used for the Quick Entry method, so user knows which item they’re intending to order and types it and enters order quantities at the same time.

* Item #
* Vendor Item #

+*Item List:*+
Use the grid which allows users to select or deselect columns, and allows users to custom group/sort items by dragging columns to the top. 

Use pagination for grid.

*Bold* columns are default. Others can be selected using the ‘Columns’ button.

* *Item Number*
** searchable
* *Item Description*
** searchable
* *Vendor*
** should be able to sort/group items by Vendor
** should be able to search by Vendor
* *Vendor Item Number*
** searchable
* Delivery Date
** shows first available delivery date for each item
* Storage Category
** should be able to sort/group items by Storage Category
* Item Category 
** should be able to sort/group items by Item Category
* *Whole Unit Quantity* - user input
** *Validation*: alert the user if the quantity entered does not meet the Minimum Order Quantity or exceeds the Maximum Order Quantity, according to the vendor contract
*** Note: the validation should consider the total order quantity (whole unit + broken unit)
*** Show the warning symbol (!) if quantity does not meet minimum or exceeds max
*** Provide hover tooltip to explain error:
**** if doesn’t meet minimum quantity: “Quantity entered does not meet minimum order quantity.”
**** if exceeds maximum quantity: “Quantity entered exceeds maximum order quantity.”
* *Broken Unit Quantity* - user input
* *Price*
** per unit
*** from vendor contract
** Total (unit price * quantity)

## Acceptance Criteria
* User should be able to search item list with column headers
* User should be able to enter quantity in the WU field if item is configured to allow ordering whole units
* User should be able to enter quantity in the BU field if item is configured to allow ordering broken units
* System will display alert/warning message if an item’s order quantity is outside the Min/Max Order Quantity parameters of the contract.
* The selection checkbox for an item should be checked if any quantity is entered
* User should be able to add items to cart if they are selected (checked)
* If an item is selected (checked), it should have some quantity entered before it can be added to the cart
* Price should be calculated dynamically based on the order quantity entered
