# NXT-6835 — Recents – Track Pages Visited

## Description
As a SchoolCafé user, I would like the pages I visit to be tracked so that I can quickly browse the pages I use most.

*Notes:*

* -Use the Recents Framework to complete this story-
* -Recent page views is tracked by Google Analytics (GA) on our pages, so will use that rather than track manually to our DB-
* Track the recent entries and store it in Azure Table Storage

*Flow:*

* Browse to any page in SchoolCafé
* Page visit it recorded in database
* Page visit is recorded by GA
* Optionally, user goes to workspace and see the list of 10 most recently visited

*Requirements*:

* -Add in the GA code for all pages the ability to track the user visiting that page if not already done-
* Allow the page developers to “opt out” in code to not track the visit
** Example: we do not want Workspace Dashboard to record visits b/c it is where the user sees the most recently viewed list anyways
* We do not want drawers and overlays that are for add/edit/delete functions to record their page views
* 
* -To track, save to the Recents database the user id, page name, url, datetime stamp and whatever else needed when the user visits a page-
* Do this silently and in the background so as to not affect page load time
* 
* The last 10 pages viewed should be available to the Workspace dashboard for that section

## Acceptance Criteria
* User visits are tracked
* Front End developers can “opt-out” of logging a visit to a particular page so that we don’t show drawers and overlays out of context
