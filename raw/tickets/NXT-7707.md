# NXT-7707 — INV - Physical Inventory - Reconciliation (back end) - Part 1

## Description
As an inventory user I need a reconciliation transaction so I can update my perpetual inventory when I save and reconcile a physical inventory.

Per Michael/Kapil, this is a big back end change and there are several dependencies with Physical Inventory stories.

Reconciliation
IM_SiteInventoryCaptureHeader_UpdateStatus--1.0 procedure
This transaction will usually be created from a physical inventory. Its also used whenever we need to make any adjustments to the inventory (ex: remove negative batches).

First step is to check what the setting for FIFO is. If its Yes then the system tries to prevent negatives so we need to check if the physical inventory adjustment would cause any future transactions to go negative.

If Perpetual – physical > qoh then show error saying item has been withdrawn

If no errors or no check required because FIFO is no then
Sync bin inventory and site inventory –this should already be in sync but in case it isn’t we should sync
Remove negative batches

Create reconciliation transaction for all items based on Physical – Perpetual.
Use the FIFO procedure described above to find the batches to fill TransactionBinLines of the transaction
After all the bin lines are populated take the weighted average price and use that to calculate the line amount on transaction lines.

Update the discrepancy on the physical inventory using the weighted average price of the batches selected

Update the reconciliation transaction id in physical inventory header with the newly recreated transaction and change the status to reconciled.

## Acceptance Criteria
User can click Reconcile button
User will see Reconciliation confirmation prompt
Reconciliation will update Perpetual Inventory
Reconciliation will change PI status to Reconciled
