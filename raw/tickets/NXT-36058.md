# NXT-36058 — Device Management – Suspend / Activate POS Device

## Description
_Phase 2, System Step 3_



As a SchoolCafé admin, I want to be able to suspend or reactive an existing POS device so that I can disable usage of devices with issues.

*Flow:*

* User logs in and browses to System > Management > Device Manager
* User clicks the suspend button on a record
* Suspend confirmation box opens
* User clicks yes
* System calls Suspend POS Device API from FO
* Later, user clicks Activate button on the same record
* Activate confirmation box opens
* User clicks yes
* System calls Activate POS Device API from FO

*Notes:*

* This is the continuation of the Device Manager stories
** 1^st^, we created the APIs in Front Office
** 2^nd^, we updated the device manager screen
** 3^rd^, we created the Add New POS drawer and are able to add new devices
** Now, we will do the suspend and activate functionality for a POS device
*** Note: this feature requires the work of both System and FO teams
** By the time we start this story, FO team must have written the relative APIs for button
** And the Add New functionality must have been completed

*Requirements*:

* *Suspend POS Device Button*
* All of these requirements are for the Suspend POS Device button on the Device Manager page
* Suspend button:
** This button shows when the device is in Active status
** If this button is visible, Activate button is not
** On click:
** Show a confirmation popup
** If they confirm, then do the following:
** Call the Suspend POS Device API from FO
** When call API:
*** add User to it for logging purposes
** 
** Show a message to the user that the POS Device is being suspended
*** Wait a few seconds for the success response and
*** then show them a message for a second that it suspended successfully
*** then close the notification and update the device status
* 
* *Activate POS Device Button*
* All of these requirements are for the Activate POS Device button on the Device Manager page
* Activate button:
** This button shows when the device is not in Active status
** If this button is visible, Suspend button is not
** On click:
** Show a confirmation popup
** If they confirm, then do the following:
** Call the Suspend POS Device API from FO
** When call API:
*** add User to it for logging purposes
** 
** Show a message to the user that the POS Device is being activated
*** Wait a few seconds for the success response and
*** then show them a message for a second that it activated successfully
*** then close the notification and update the device status
* 
* The following fields/values are not seen on the screen, but needed for the POS object
** User = account logged in for logging purposes
* 
* Security:
* Roles & Permissions actions:
* Only those with Edit rights to Device Manager & Cybersoft admins can use these buttons
