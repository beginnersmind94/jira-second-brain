# NXT-58225 — Acc - Edit Checks Page - Reverse Edit Checks - Page Updates + Results

## Description
As Dana the Director I want a way to focus on sites that are well under the expected meal count based on their monthly average, and a list of sites that fall into this category would speed up my workflow.

*requirements*

* Update the date picker to the new one from the FOH Dashboard.
* Change the site dropdown out for a multi-select

* *Edit Checks*
** Existing page and existing workflow
** Add a link to Sessions for the site + date in the results grid.
* *Reverse Edit Checks*
** New workflow, aimed at allowing the user to:
*** Configure the average number of meals served by Meal Type and by site
*** Search for sites that are below that average by a certain % to make corrections.
** User can set and save the average number of meals by site, by meal type.
*** User can also ask the system to calculate this number over the last 30 days.
*** User can see the meal types the district has served over the last 30 days - these will be the only meal types shown to the user.
*** User can leave an average meal count blank for a site type + meal type combo.
**** Example: Texas Elementary, Breakfast, can be left blank. This will then be ignored for search results.
** User can set the % under average meals they are searching for.
** User can perform the search and see results.
** In the results grid, user can see sites that fall within the % Under Average - some examples are below.
** User can go to the Sessions for the Site + Date in the grid.
** Explanation text for the top of the page: 
*** {noformat}Reverse Edit Checks are a way to find service sites that have low meal counts for any given day. Configure an average meal count for each site, per meal type. Then search for serving days that fall under that average count by a certain percentage.{noformat}
* *Results Grid*

||*Site Code*||*Date*||*Meal Type*||*Meal Count*||*Site Avg Count*||*% Under Avg*||*Options*||
|163 - Texas Elm|01/17/2025|Breakfast|0|89|100%|Sessions|
|163 - Texas Elm|01/17/2025|Lunch|150|300|50%|Sessions|

So for 163 - Texas Elementary:

* User has configured the Site Average Count for:
** Breakfast → 89
** Lunch → 300
* They could’ve had the % Under Average Meals set to 50% and hit Apply.
* The results grid would’ve shown them that Breakfast was 50% under, and Breakfast is 100% under average.
* If they set the % Under Average Meals to 51% they would have only seen Breakfast.



*Calculations*

* Calculating Average Meals for the user:
** Check the last 30 days
** Sum how many days they’ve served each meal type
** Sum the meal count
** Average = (Total Meals by Meal Type) / Number of Service Days
** Save this number.
** User can override and manually set a different number.
* % Under Avg
** (Avg Meal Count - Meal Count) / Avg Meal Count * 100
*** (89 - 0) / 89 * 100
**** 89 / 89 * 100
**** 1 * 100
**** 100%
*** (300 - 150) / 300 * 100
**** 150 / 300 * 100
**** 0.5 * 100
**** 50%
** User is setting their search filter - if they set it to 51%, they want results that are between/including 51% - 100%. We’d only show the 89 meal count for this.

Discussion at Refinement (1/21/25)

* Rounding → go up to the nearest whole number
* User can leave a site average blank - we will ignore this meal type when finding results.
* For meal counts - we are only counting student meals
** Free, Reduced, Paid
** We will not include Second Meals
** New Student account would be included in this - no other special accounts
** No adults
