# NXT-72232 — Enhance GL Balance Fields to Display Debit/Credit Direction

## Description
Update the existing GL account detail screen so balance fields show whether the calculated balance is a debit balance or a credit balance.

This story is limited to display behavior for the existing calculated balances.

Apply Dr/Cr display to these three balance fields:

* Current Balance
* Beginning Balance
* Ending Balance

Example:

text

```
Current Balance: $5,164.52 Cr
```

Do not change journal posting, stored debit/credit values, transaction balancing, account categorization, or the existing balance formulas as part of this story.

---

## Requirements

### 1. Preserve Existing Balance Calculations

The system must continue using the existing balance calculation rules by account type.

No changes are required to:

* Journal entry posting
* Debit and credit storage
* Transaction-level balancing
* Chart of accounts category assignment
* Existing account type derivation from account category
* Total Debits calculation
* Total Credits calculation
* Current Balance calculation
* Beginning Balance calculation
* Ending Balance calculation

---

### 2. Display Dr/Cr Direction for the Three Balance Fields

The GL account detail screen must display debit/credit direction for:

* Current Balance
* Beginning Balance
* Ending Balance

Display format:

| Balance Direction | Display |
| --- | --- |
| Debit balance | `$X.XX Dr` |
| Credit balance | `$X.XX Cr` |
| Zero balance | `$0.00` |

The displayed amount must use the absolute value of the calculated balance.

---

### 3. Dr/Cr Replaces Negative Parentheses/Red Formatting for Balance Fields

For Current Balance, Beginning Balance, and Ending Balance, the Dr/Cr suffix is the indicator of balance direction.

For these three balance fields:

* Do not display negative balances using parentheses.
* Do not display negative balances in red only because the calculated value is negative.
* Do not combine parentheses with Dr/Cr.

Example:

text

```
$1,250.00 Dr
```

Do not display:

text

```
($1,250.00)
```

or:

text

```
($1,250.00) Dr
```

This requirement intentionally overrides the negative-balance display rule from NXT-65124 for these three balance fields only.

---

### 4. Do Not Add Dr/Cr to Total Debits or Total Credits

The following fields must remain currency-only:

* Total Debits
* Total Credits

These fields must not display `Dr` or `Cr` because the field labels already identify the accounting side.

---

### 5. Map Calculated Balance to Display Direction

Account type is derived from the account's chart of accounts category. Apply this display mapping to the existing calculated balance:

| Account Type | Calculated Balance > 0 | Calculated Balance < 0 | Calculated Balance = 0 |
| --- | --- | --- | --- |
| Asset | `$X.XX Dr` | `$X.XX Cr` | `$0.00` |
| Expense | `$X.XX Dr` | `$X.XX Cr` | `$0.00` |
| Liability | `$X.XX Cr` | `$X.XX Dr` | `$0.00` |
| Equity | `$X.XX Cr` | `$X.XX Dr` | `$0.00` |
| Revenue | `$X.XX Cr` | `$X.XX Dr` | `$0.00` |

The displayed amount uses the absolute value of the calculated balance. The underlying calculated balance value must not be changed, reversed, or forced into a positive display direction.

Reference — existing formulas not modified by this story:

| Account Type | Ending Balance Formula |
| --- | --- |
| Asset, Expense | Beginning Balance + Total Debits − Total Credits |
| Liability, Equity, Revenue | Beginning Balance + Total Credits − Total Debits |

---

## Defaults

* If a calculated balance is exactly zero, display `$0.00` without `Dr` or `Cr`.
* If Current Balance, Beginning Balance, or Ending Balance is null, treat the value as `0.00` for display.
* If Total Debits or Total Credits is null, display `$0.00`.
* If account type cannot be determined, display the balance as signed currency and log the missing account type/category mapping.

---

## Edge / Empty / Error States

### Empty Transaction Set

If no transactions exist in the selected date range:

* Total Debits displays `$0.00`
* Total Credits displays `$0.00`
* Ending Balance equals Beginning Balance
* Current Balance, Beginning Balance, and Ending Balance still show the correct Dr/Cr direction when non-zero

### Opposite-Side Balance

If the calculated balance is opposite the usual Dr/Cr direction for that account type, display the actual Dr/Cr direction.

Examples:

| Scenario | Display |
| --- | --- |
| Liability account with debit balance | `$X.XX Dr` |
| Asset account with credit balance | `$X.XX Cr` |

Do not hide, reverse, or convert opposite-side balances into the usual direction for that account type.

### Missing Account Type Mapping

If the account's chart of accounts category does not map to Asset, Liability, Equity, Revenue, or Expense:

* Do not infer Dr/Cr direction.
* Display the balance as signed currency.
* Log the missing account type/category mapping for investigation.

## Acceptance Criteria
### AC1: Liability Account Shows Credit Balance

Given a liability account with:

- Beginning Balance: `$0.00`
- Total Debits: `$62.50`
- Total Credits: `$500.00`
When the GL account detail screen is displayed,

Then Ending Balance displays:

text

```
$437.50 Cr
```

And Total Debits displays:

text

```
$62.50
```

And Total Credits displays:

text

```
$500.00
```

---

### AC2: Liability Account Shows Debit Balance When Calculation Is Negative

Given a liability account where the existing balance calculation returns:

text

```
-500.00
```

When the GL account detail screen is displayed,

Then the balance displays:

text

```
$500.00 Dr
```

And the balance does not display as:

text

```
($500.00)
```

### AC7: Zero Balance Has No Direction Suffix

Given any account type where Current Balance, Beginning Balance, or Ending Balance is exactly zero,

When the GL account detail screen is displayed,

text

```

```

And it does not display `Dr` or `Cr`.

---

### AC8: Total Debits and Credits Remain Currency-Only

Given any account type,

When Total Debits and Total Credits are displayed,

Then they display as currency amounts only,

And they do not include `Dr` or `Cr` suffixes.

---

### AC9: Dr/Cr Applies to Current, Beginning, and Ending Balance

Given the GL account detail screen displays Current Balance, Beginning Balance, and Ending Balance,

When each balance field has a non-zero calculated value,

Then each field displays the absolute amount plus the correct `Dr` or `Cr` suffix.

---

### AC10: Missing Account Type Mapping Falls Back to Signed Currency

Given an account whose chart of accounts category does not map to Asset, Liability, Equity, Revenue, or Expense,

When the GL account detail screen is displayed,

Then the balance displays as signed currency,

And no `Dr` or `Cr` suffix is inferred,

And the missing mapping is logged.


## Out of Scope

- Posting logic changes
- Database schema changes
- Changes to stored debit/credit values
- Changes to transaction balancing rules
- Changes to account category mapping
- Changes to Current Balance, Beginning Balance, or Ending Balance formulas
- Revenue/expense fiscal-year beginning balance changes
- Year-End Close behavior changes
- Reconciliation workflows
- Opposite-side balance alerts
- Export format changes unless separately requested
- Negative-parentheses/red formatting outside the three balance fields covered by this story

Open question for that work:

When a user selects a report From date inside a closed prior fiscal year, should revenue/expense Beginning Balance show activity from the fiscal year start through the day before the From date, or should it reflect the closed-year result after year-end closing entries?

This story does not answer or change that behavior.
