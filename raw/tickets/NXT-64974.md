# NXT-64974 — SCTV - Auto-update menus in response to auto-publishing of menus 

## Description
As a user, I want to make sure my device displays the latest menus

## Acceptance Criteria
Since the auto-publish feature will be pushing out new menus every 5 minutes, we need to perform a periodic check to ensure that we are working with the latest data

* Perform a fresh pull of data every 15 minutes
* Check if there are any changes in the menu items compared to before and only make updates accordingly, items with no change in availability should not be cached again
* On transition to a new meal type (based on schedule timings), trigger an immediate fresh pull equivalent to the "Reload Menu" command from the admin app
* When "Reload Menu" is manually triggered from the admin app, bypass the 15-minute cycle and perform an immediate fresh pull with full cache refresh

Technical Notes on the New Points

Meal type transition trigger:

* Monitor current time against breakfast/lunch schedule boundaries
* When crossing from one meal period to another, force immediate data refresh
* Ensures menus are current when meal service periods change
* Critical for accuracy during transition periods (e.g., 10:30 AM breakfast→lunch switchover)
