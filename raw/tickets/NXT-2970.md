# NXT-2970 — System - Roles - View History drawer

## Description
As a sponsor central office administrator I want the ability to view the history of changes for a role so that I can determine who and when changes were made regarding access to various parts of the system. 

+Summary+:

Allow a user to select a button that shows them all changes made to a role over time.

+Details+:

* In the Roles grid, in the menu containing the ‘Copy’ button, add a ‘History’ button as well.
* When the button is selected, open a History drawer that contains the following information:
** Role Selected (drawer header)
** -Comments- (Deferred to future story)
** Columns:
*** Updated On
*** Updated By
*** Module Name
*** Permission
**** For pages: display the page + permission + change. For example: *Site Types - View - Enabled* or *Users - Full Control - Disabled*
**** For ‘More’ page permissions: display the permission + change. For example: *Manage* *Programs - Enabled*
**** For miscellaneous permissions, display the permission + change. For example: *Support Tickets - Enabled*
** All changes made at once can go in the same record, being separated by the details above on different lines. One row should look something like this:
*** 8/24/2020 : John Smith : System : Sites- Edit - Enabled
* Like other History drawers, items should be sorted in descending order from the most recent update.

PrimeroEdge Classic Reference: System > Sites and Users > Roles > Select a Role > History (tab)

## Acceptance Criteria
1. History button is added alongside each role, which displays a History drawer when selected
2. User can see new values of the permission modified in the format specified above
3. User can see who made the change and when
4. Changes are sorted in descending order from most to least recent
