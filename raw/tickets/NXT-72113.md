# NXT-72113 — SC - MT/Central - State Student File upload

## Description
Story for the feature for the ability to upload and process State Student file.

Introduction of new upload modes which our competitors and standalone SIS systems already have: Additive and Upsert logic.

Job
Processing
Results
Verifying the student data - Testing show a prompt that this is a state level import.

## Acceptance Criteria
SIS Student Upload (Central, populate districts)
new page available in Central/State Realm to upload a student file.
When a file is uploaded student data is propagated down to each respective SFA/District
User can choose upload behavior
-{color:#0747a6}Standard{color}
-- Updates, Inserts, and withdraws
-{color:#4c9aff}Additive Only{color}
--If Id exists: ignore/skip the record
--If Id does not exist: insert
--{color:#ff991f}No Withdrawals{color}
-{color:#4c9aff}Upsert{color}
--If Id exists: update
--If Id does not exist: insert
--{color:#ff991f}No withdrawals{color}

Standard follows withdraw threshold setting.
Upload can be canceled

Upload can be undone within 48 hours
-- Track all the changes that were done, what do we do with DC matching that may have happened since then, move them to a “deleted/DE” status

Upload status can be tracked in ~real-time (whatever is feasible).

Show history in the Data Exchange page.
