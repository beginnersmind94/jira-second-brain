# NXT-4382 — System - Roles - Allow user to Delete a Role

## Description
As a district or central office administrator, I would like to be able to delete a role that is no longer necessary, so that I can avoid confusion when trying to assign relevant roles to users.

+Summary+:

Allow a user to select a button that lets them delete any role that is not already assigned to a user.

+Details+:

* In the Roles grid, in the menu containing the ‘Copy’ button, add a ‘Delete’ button as well.
* This button should *only exist for Custom Roles.* Standard roles cannot be deleted.
* When this button is selected, perform a validation: 
** If there exists >=1 users (active OR inactive) that are assigned the role, display an error message: “This role is currently assigned to one or more user(s). Please reassign these users in order to delete this role.”  
** If there are no users that are assigned the role, display a Y/N prompt allowing the user to confirm deletion.
*** If the user says No/Cancel, return them to the roles grid.
*** If the user says Yes, hide the role from the roles grid.
* Role is preserved in the backend (soft delete)

## Acceptance Criteria
Verify user can access a 'Delete' button for any *Custom* role on the Roles grid

Verify user cannot delete Standard roles

Verify validation is performed when 'Delete' button is checked for a role: 
-If one or more users are assigned the role, we display the error message
-If no users are assigned the role, we allow the user to delete the role as needed or back out

Verify that roles can then be deleted from the list but maintained in KeyCloak
