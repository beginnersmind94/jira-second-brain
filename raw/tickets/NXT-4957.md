# NXT-4957 — AcctMgt - Split/Table View - Search Filters

## Description
*User Story*

As a district admin I want a way to filter my searches to save time.

*Requirements*

* Add the filter icon with dropdown area next to the Search By bar.
** This is both Split and Table view.
** For Table view, match the positioning that Item Management has. 
** We may need to move our Search field also to match.
* Filters available in the area:
** Site Code / Site Name
** Grade
** Patron Type
*** Options: All (default), Students, Adults
**** Multi-select available.
**** Filters the search results to just the selected account/patron/person type.
*** This filter is only available on the Split/Table pages, not on the landing page.
* Defaults for all are blank
** If the user is brought here after searching off the landing page, then any search filters will be passed through however.
* User can type into each filter to filter the available results.
** -Example: User can type ‘Elementary' into the Site Code / Name field and the results will dynamically shift to only include sites with a name that includes ‘elementary’, or ‘501’ to pull up any site code (or name) with 501 in it.-
* Multi-select is possible on these filters. 
* Selected values should appear as chips that can be removed.
* Grade is populated based off site(s) selected. 
** If all sites are selected, all grades are available for selection.
** If Site A has grades PK-5 and is selected, then the Grade dropdown has PK, KG, 1, 2, 3, 4, 5 available.
* Add the filters to the Table view also with similar functionality.
* Selected filters impact the search results.
* Predictive search does not need to be impacted by the filters right now.

## Acceptance Criteria
Filters added to both split and table views.
Multi-select available on the two dropdowns.
Grade behaves appropriately based on selected sites.
Search results are appropriately filtered based on selections.
Multi-language support enabled (LANGUAGENAME_ENGTEXT ie: FR_Students, FR_ADULTS)
Accessibility / ADA Compliance programmed appropriately (full KB nav possible, etc.)
Predictive search does not need to be impacted by the filters right now.
