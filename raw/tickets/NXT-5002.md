# NXT-5002 — Data Exchange - Smart Mapping (framework)

## Description
*User Story*

As a PrimeroEdge user importing a flat file, I have a system that will make an attempt at automatically mapping the columns in my file to the available fields in PrimeroEdge.

*Requirements*

* The system attempts to automatically match the column header with the appropriate field in PrimeroEdge based on the Import being done
* A message is displayed next to the column preview saying the field that has been matched
** The selection is automatically made for the field in the column section
* User has the option to confirm the mapping or select their own
** User can edit the selection
** User can also choose to ignore the column
* System checks for keywords in header names to match with fields
* System checks for patterns in column content to match with fields
** E.g. Email addresses, phone numbers, addresses
* If system cannot make a match, no field is selected
** User can still manually select a field or choose to ignore the column
** Message: A matching field could not be determined

## Acceptance Criteria
* System checks for field matches based on criteria listed in the requirements
* Fields are automatically mapped when the user enters the Field Mapping screen
* User can confirm matching in every individual column or in bulk
* Manual matches can still be made
