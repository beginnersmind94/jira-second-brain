# NXT-32138 — INV - Config - Ordering Conditions - Add Condition - Update for Pack Size Config

## Description
As a District Admin I want the Add Ordering Condition function to consider pack size units so I can configure orderability for the right units.

*Note:* Page is not currently working correctly anyway; need to fix several issues observed in UAT

* Items appearing without Vendor/Contract
* Items appearing multiple times

*Requirements*

* Update to resolve issues with logic
** Items should only appear on this screen if they:
*** on an active contract
*** are *not* currently assigned to an an existing ordering condition
*** have a Vendor Item #
*** have an active (current) Purchase Price
* Update the Add Ordering Condition screen to consider pack size configuration updates
** Provide the hierarchical representation for related items
** Implement business rules as they exist on the main Ordering Conditions page
*** units cannot be orderable from multiple vendors by the same site 
*** use same validations from main Ordering Conditions slideout
** **New* Show the below message for a child unit which cannot be assigned (checked) because its parent unit is assigned to a different vendor contract
*** All item units must be assigned to the same vendor contract to be enabled. Please first change the higher unit.
** If one unit of an item is assigned at least one ordering condition, move the entire item (including all units) to the main Ordering Conditions list page
*** this will prevent showing units on the Add screen which are separated from their related unit
** Checkboxes will be per unit, not per item
*** but Item # only shown once for hierarchy
* UI Changes
** Move header filters into Item card
*** Remove Data Source dropdown (always Local) 
** Increase the height of both Items and Sites card to accommodate lots of data
*** Implement a sticky footer to take full advantage of page height
