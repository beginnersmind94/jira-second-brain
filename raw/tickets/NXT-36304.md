# NXT-36304 — Acc - Reports - Rollups Page

## Description
As David the Support Manager I want to let the user refresh reports by themselves so we do not have to escalate tickets through our Support team. This will save time for customers and for us internally.

*mockup*

[https://www.figma.com/proto/QRowEx99f8nKGMmpZPeLRq/Front-Office-6.3?node-id=151%3A4040&scaling=scale-down-width&page-id=150%3A3556&starting-point-node-id=169%3A4251|https://www.figma.com/proto/QRowEx99f8nKGMmpZPeLRq/Front-Office-6.3?node-id=151%3A4040&scaling=scale-down-width&page-id=150%3A3556&starting-point-node-id=169%3A4251|smart-link] 

*Requirements*

* Add a page off the Reports page
** Page name: Report Rollups
* Page has a search component:
** Site
*** Can include all sites or select multiple sites
** Date Range
*** Can pick standard date ranges
** Apply, Reset buttons
* Results grid
** Columns
*** Site Name + Site Code
*** Date
**** Format: MM/DD/YYYY
*** Last Modified
**** Full timestamp: MM/DD/YYYY HH:MM:SS
**** Tool-tip for this: This timestamp shows the last time anything transactional (Sessions, Transactions, etc.) was updated for this site.
**** This should represent the last time data was touched for that date + site.
***** Sessions updated, transactions updated. What’s the last time stamp available?
*** Reports Updated On
**** Full timestamp: MM/DD/YYYY HH:MM:SS
**** Tool-tip for this: This timestamp shows the last time report data was updated for this site and date.
**** This is the last time rollups were run for this site.
***** Do we still have multiple? let’s show this based off the standard rollup and not person rollup if so.
*** Actions
**** Refresh
** Data ordered by site, date.
** Highlight any rows where the Last Modified date is greater than the Reports Updated On date 
*** This is checking to see if the Reports data is stall and needs to be refreshed.
*** For current date, only highlight if the difference is more than 15 minutes.
*** For prior dates, always highlight if the difference exists.
* Actions
** Multiple actions on the page:
*** Each site should have a ‘Refresh Site’ button
*** Each site + date should have a Refresh button.
*** The grid should have a ‘Refresh All’ button
** These will add the site + date for rollups. *We should do the Person Rollups as well as part of this.*
*** Success message for Refresh button: Report data will update shortly. Please check back in 15 minutes.
*** Success message for Refresh Site button: Report data for the site will update shortly. Please check back in 15 minutes.
*** Success message for Refresh All button: Report data for all sites will update shortly. Please check back in 15 minutes.
* Instructional text to put near the top of the page:
** This page shows when report data was last updated. If any report data is stale or out of sync use the *Refresh* buttons to update the reports. This process can take 15 minutes or longer depending on the number of dates added to the queue.
* Role Permission
** Need to add a permission tied to page access + button availability.

## Acceptance Criteria
* Page added off the Reports page
* User can search by site and date.
** Multiple sites, multiple dates.
* User can see data on when rollups were last run for each site + date.
* User can see when transaction/session data was last updated for each site + date.
* User can request a Refresh on a site + date.
* User can request a Refresh for all sites + dates in the search results.
* User can request a refresh for all dates for a particular site.
* Role permission added to access this site.
