# NXT-5275 — Verification - Tracking - Response workflow

## Description
*User Story*

As a district admin I need to complete the response / information I have on a verification application.

*mockup*

[https://www.figma.com/proto/Y3tlwN2yvRc67304adzyyB/Eligibility-3.0?node-id=1129%3A433&scaling=scale-down-width&page-id=1110%3A0|https://www.figma.com/proto/Y3tlwN2yvRc67304adzyyB/Eligibility-3.0?node-id=1129%3A433&scaling=scale-down-width&page-id=1110%3A0|smart-link] 

*requirements*

Off the App. Info page, the user needs a way to complete verification for a single application. This should be a series of questions and actions that the user can answer as part of the Response pop-up. User can change their selections as they go. 

The below is a vague attempt at the work flow for this. Screenshots above may help.

* Were all students directly verified? Yes/No
** YES > What benefit level were they verified at? Free/Reduced
*** EITHER ONE> Application can be marked as ‘Verification Complete’, new action becomes available.
**** If user selects the new action, then the application must be updated as being complete.
!image-20201225-000023.png|width=342,height=256!
!image-20201225-000130.png|width=345,height=266!

**** Update the Result, Reason for Change, Change Date if applicable (keep adverse action days in mind), Completion Date. History tracking for this action being taken.
**** Eligibility for students on the application must be updated appropriately.
** NO > Next question…
* Was notification received of refusal to respond? Yes/No
** YES > User needs to select a start date for the new benefits (adverse action days taken into account), can select a reduction in benefits (free to reduced OR paid, reduced to paid), and enter comments.
** User can then select Verification Complete.
*** Application can be updated similar to question 1, and marked as verification complete.
*** Eligibility for students on the application must be updated appropriately.
*** !image-20201225-000552.png|width=851,height=485!

** NO > Next question…
* Is the Case Number confirmed? YES/NO
** This question should only be available if the application was categorical with a case number / case type to begin with.
** This needs to use the RegEx settings.
** YES > User should be shown the existing case number, and be asked to enter the confirmed case number and select the case type. 
** User can then select Verification Complete.
*** Application is updated, marked as verification complete.
*** !image-20201225-001009.png|width=931,height=494!

** NO > Next question…
* Categorical status confirmed through other means? YES/NO
** This question should only be available if the application was categorical with a case number / case type to begin with.
** YES > User can enter a case number, select the type, and has a comment box (150 characters).
** User can then select Verification Complete.
*** Application is updated, marked as verification complete.
*** !image-20201225-001318.png|width=1005,height=555!

** NO > Next section…
* Update Application Information
** *Display data from the application only - editing is handled in the 2nd story.*
*** Students
**** Add additional students.
**** Edit existing students, including a change for their Change categorical status on a student.
**** If they’re using the Child Income setting to track income on each student, it can be adjusted.
**** We can show grade / school editing to save space.
*** Household Members
**** Add additional members.
**** Change reported income amounts.
**** PFD for Alaska.
*** Child Income (if setting is on)
*** Display the Household Size and Total Income + Frequency.
* User can add comments at any point in this response flow. Comments are required. 500 character length.
* User should be able to cancel out of this flow with a confirmation prompt if they’ve begun answering.
* User should be able to change their action mid-stream - if they clicked ‘No’ to an earlier question, they need a way to go back and change that.
