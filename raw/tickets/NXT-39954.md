# NXT-39954 — SIS Auto Sync – Configuration Drawer -Dev

## Description
As a SchoolCafé admin, I would like to configure the SIS Auto Sync connection fields on a separate drawer so that we don’t have to reconfigure them every time we do an SIS import configuration.

*Notes:*

* These fields currently exist in the Add New and Edit SIS Import configuration screen
* We need to move them to their own drawer so that it can be reused for multiple imports and not have to be re-setup every time

*Requirements*:

* On the System > Imports & Exports > SIS Auto Sync tab, do the following:
* 
* Add a button:
** Title: Configuration
** On click, open the Configuration drawer
** Only shows on the SIS auto sync tab, not the others
* 
* Create the SIS Configuration drawer
** Opens from the Configuration button click
** Title: SIS Configuration
* 
* Add 4 new settings:
*  Provider
** Drop down list
** Values
*** Aeries
*** Skyward
*** PowerSchool
** Default Value = empty
** Required
* 
* URL
** Textbox
** Required
* 
* Key
** Textbox
** Required
** Mask text and have eyeball button to show like would do with a password just like it currently is done
* 
* Secret
** Textbox
** Required
** Mask text and have eyeball button to show like would do with a password just like it currently is done
* 
* Test Connection
** Copy over the code from the existing screen and do all the same things that you do there including the success and failure messages
* 
* Save
** On click, validate required fields and test the connection
** If successful, save the values to the database
* 
* Close
** On click, close the drawer
* 
* Allow the user to only configure one provider at a time by:
** If there are no configurations, show all three providers and allow them to configure one
** If there is a configuration, then they can only edit or delete that configuration in edit screen so don’t allow them to select a second provider in the provider drop down

## Acceptance Criteria
* Configuration button added to the SIS page and doesn’t show up for other tabs
* New configuration drawer has the 4 config fields and test connection button and they work the same as is currently in production
* Test connection tests the connection button
* Config fields saved to the database when click save
* User can have only one provider at a time
