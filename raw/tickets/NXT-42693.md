# NXT-42693 — IM - All Types - Replace all instances of Cancel and Save buttons on autosaving cards

## Description
As a user I don’t want misleading buttons like Cancel and (often disabled) Save when editing my recipe steps, my item pack sizes, or my menu serving exceptions, because I want confidence that my data will save when performing important actions.

h2. Recipe Steps

!Recipe Steps Save and Close-20230824-203255.png|width=91.66666666666666%!

## Acceptance Criteria
* Replace all instances where autosave is requiring users to use the ‘Cancel’ button like a ‘Close’ button
** Recipe Steps, SIR, CMI Ingredients
** Units card
*** all tabs
** VMI Menu Items
* If the autosave causes the Save to be disabled, both Cancel and Save should be replaced with a simple ‘Save & Close’
** Do *not* show ‘Save & Close’ until some change is made; first time edit should display Cancel and a disabled Save (current behavior)
* If some additional edit is made requiring the main Cancel/Save buttons, show those buttons instead of Save & Close
