# NXT-45496 — IM - All screens - Update styling for disabled/incomplete states

## Description
As an Item Management user I want the sub-type tabs to behave the same across all IM screens so I can have a consistent user experience when attempting to enable new sub-types.

## Acceptance Criteria
* Find a way to show the tabs and tab cards for items/recipes/VMIs which are not yet “complete”
* Allow the tabs to be visible but unclickable while in initial create/edit modes (typical unfocused behavior)
* Allow the tabs to be clicked in view mode but show uneditable versions of the cards until mandatory info is provided (usually a serving size)
