# NXT-49592 — Data - Dashboard

## Description
As a SchoolCafé admin, I would like to be able to execute API calls from SIS providers w/o having to import the data so that I can gather information and debug issues.

*Notes:*

* We are leveraging the existing import features to be able to pull and view data w/o having to import
** Therefore on config screens, we do not need Field Mapping, Schedule or review screens
** We can consolidate the rest of the screens onto one
* On this dashboard, user can:
** Add / edit / delete a power query
** Add / edit / delete a data only API call
** View list of configs for both
** Run Now / Download both

*Requirements*:

* On the System > Imports & Exports, do the following:
* Add a new tab
** Title: Data
** Make it the last one





* On the System > Imports & Exports > Data tab, do the following:
* Add a + New button on the top right like the other tabs
** Make it a select + New button like on custom reports
** Values:
*** PowerQuery
*** SIS Source Data
** If user clicks PowerQuery, open the Add New PowerQuery drawer
** If user clicks SIS Source Data, open the Add New SIS Source Data drawer
* 
* Add a grid to the page
** Data:
*** List of PowerQueries and Data Only queries that are Active
*** Note: 
**** Show only active / enabled querries (this is different from imports)
**** There is no “make active” or “make inactive” for these; they can add new ones or delete old ones, but will not toggle active / Inactive
** Columns:
** Provider
** Name
** Source
*** If PowerQuery, then source is PowerQuery
*** Else, source is the Import Type (Student Import, Adult Import, etc.)
** Actions
*** Download
**** Do PII Validation (see below)
**** Note: this is the equivalent of Download on Imports
**** On click:
***** Show the Download dialogue and process just like you do for API Imports
**** Note: Make Download the button outside the ellipsis if you show any; it can be a text button instead of icon
*** Edit
**** Validate has Edit rights to Imports & Exports
**** On click:
***** Show the Edit drawer (separate story)
*** Delete
**** Validate has Delete rights to Imports & Exports
**** Covered in a separate story
*** View History
**** On click:
***** Show the View History drawer (separate story)
** 
** Note: Do not show Import, IsScheduled columns
*** There is no import or scheduling
*** Note: Scheduling could be useful but we would then have to setup Distribution like custom reports and this would take time and resources, so we will skip it for now
* Manage Views
** Show the Manage Views for the grid
** In addition to the default columns, add these as possible columns:
*** Created On
*** Created By
*** Updated On
*** Updated By
*** Status
*** Description
** Like on Imports, we will not do the the Save views functionality, just the columns part
* 
* Security
** Add tab to Resources
*** Ensure added when promoting to higher environments
*** Set settings so that it is not visible in roles and permissions
*** Use the parent (Imports & Exports page) permissions instead of having separate permissions for this tab
** Parent’s view permission required to view tab
** Parent’s add permission required to click on Add New
** Parent’s edit permission required to click on Edit button
** Parent’s delete permission required to click on delete permission
* 
* PII
** IsPII is required to click the Download button for Source Data (Import Types) that have personally identifiable information
** Do it like you are doing for the Download button on API rows
*** Pull from the table
** Note:
*** For PowerQuery, we have no idea if data has PII or not, so we will ask the user to designate it and you can use that field

## Acceptance Criteria
* Data tab is created
* User can click to create a new PowerQuery
* User can click to create a new Data Only 
* User can view the list of enabled Data configs
* User can delete one if they have rights
* User can edit one if they have rights
* User click to view history
* User can click Download and it starts the process
* Only IsPII users can view Data files with PII data
* Tab uses parent’s security permissions
