# NXT-11651 — Applications - Details - Forced Check-in

## Description
As a district admin I want a way to remove the lock on an application so that it can be processed by another member of staff.

*requirements*

* User needs to know if an application is currently checked out to someone, and who that someone is.
** In Application Details, add information to the side panel to indicator if the app is currently checked out.
** Show which user (first name + last name) who has the app checked out.
*** If possible, show the time it was checked out. This should use local user time.
* Add an action to check the application back in.
** This should remove the lock from the application and give the user a success message.
** Prompt the user with a confirmation message for whether they really want to check this application back in.
*** Text: This will check the application back in and it will be available for anyone to process. Are you sure you want to check this in?



Placement: As this will prevent the user from processing, it makes sense to show this information and the check-in option as part of the side panel.

## Acceptance Criteria
User can see if an app is checked out.
User can see who has the app checked out (And when it was checked out, if possible)
User can check the application back in so someone else can process it.
