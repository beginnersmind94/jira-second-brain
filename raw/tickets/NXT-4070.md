# NXT-4070 — Security - Password Policy UI

## Description
*User Story*
As a PrimeroEdge admin, I have a screen for setting the password policy as part of the security policy to ensure passwords meet minimum standards.

*Requirements*

* Within the Configuration category, add a page navigation called *Security* (tentative name and location), which allows the user to manage their settings for multi-factor authentication and passwords/activity.
* The header of the page itself should be titled *Login and Security*
* There is a section in security policy for creating a password policy
* The following fields are available in the following sections:
** Descriptions available on [https://primeroedgenext.atlassian.net/wiki/spaces/PS/pages/584384522/Password+Policy|https://primeroedgenext.atlassian.net/wiki/spaces/PS/pages/584384522/Password+Policy|smart-link] 
** Password Complexity
*** Minimum Password Length (Dropdown)
*** Require Upper and Lower Case (Toggle)
*** Minimum Special Characters (Dropdown)
*** Minimum Numeric Characters (Dropdown)
** Password Settings
*** Maximum Password Age (Dropdown)
*** Refusal of Previously Used Passwords (Dropdown)
*** Max Failed Attempts (Dropdown)
*** Failed Attempt Lockout Time (Dropdown)
*** Inactivity Timeout (Dropdown)
*** Inactivity Warning (Dropdown)
* There are presets that can be selected and apply settings automatically
** Fair, Good, Strong - All are non-editable for non-PrimeroEdge users (Good should be the default toggle selected)
** Settings to be determined
** If a Cybersoft user manually changes any settings for Fair, Good, or Strong, the preset will change also.
** For other users, the only values that can be changed is Custom (though for the purposes of this story we can ignore this for now)
* Any settings are applied to users on next password change
** Maximum password age is retroactively applied
*** If users have a password that is greater than the maximum age, they are prompted to change it on next login
*** If users have a password that is less than the maximum age, the age is applied based on when they created their password
