# NXT-39059 — PR- Record- Variety Menu item on serve with Functionality

## Description
As a production manager, I do not like the fact that when I am trying to use the serve with functionality, it is giving me the option for the variety menu item parent and not the actual menu items that I have toggled on.

## Acceptance Criteria
* When user tries to use the serve with function, update the list of the pop up with the menu items that are available to be used including the toggled on Child menu items of the VMI
** Do not show the parent menu item as it is not truly a menu item that can be served
* IF user toggles off the child variety menu item, remove the serve with link with the primary item 
* Same logic applies if the child menu item is already the primary menu item with another menu item, do not show the child menu item on the serve with list
* All of the serve with logic should flow through to the plan and vice versa
* FOR UI- there needs to be a serve with option for the child menu items and remove the serve with option from the parent ( Same logic applies for primary/complimentary items)
