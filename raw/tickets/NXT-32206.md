# NXT-32206 — SPIKE ONLY- MP/PR- Deleting/ Overriding Menus from MP affect on PR

## Description
As a Menu Planner, There are scenarios that happen where I may delete an assigned menu or override a current menu assigned. Even though I want to, the system should not always allow me to do those actions.

Currently: 

When deleting a menu from an assigned menu, we are keeping the line on the production side. If a user were to add another menu on the same line, it now shows in the production calendar as two records to check.

## Acceptance Criteria
* Allow menu to be deleted from assign Menu page if production status is in Plan complete or less. Do not allow deletion if it is in RECORD COMPLETE
** Remove the X from being available on days that a Record has already been complete for that Menu Line
** In the case of a *select all* scenario, if any site has their production record complete, remove the X from being available. User must go to individual sites and remove if they are not in record complete
* Allow menu to be overridden from assign Menu page if production status is in Plan complete or less. Do not allow override if it is in RECORD COMPLETE
** The pop up should display “*ONE OR MORE PRODUCTION RECORDS HAVE BEEN COMPLETED, THIS MENU CAN NOT BE OVERRIDDEN.*” in white font and red border
!image-20220621-195509.png|width=874,height=394!

** Disable the Assign Button as the user will not have the ability to assign or override a menu. 
* The warning that currently appears needs to be changed to Yellow border and black font to differentiate between a warning vs a blocker message. “*WARNING: ANY MENUS THAT ARE CURRENTLY ASSIGNED WILL BE OVERRIDDEN.”*
* Allow menus with menu cycles to be overridden from assign Menu page if production status is in Plan complete or less. Do not allow ANY override if it is in RECORD COMPLETE
** Remove the override existing menus toggle
** Remove Use all Menus toggle
** Display Warning on Menu Cycle Pop up in yellow border and black font to differentiate between a warning vs a blocker message “*WARNING: ANY MENUS THAT ARE CURRENTLY ASSIGNED WILL BE OVERRIDDEN.”*
!image-20220621-223905.png|width=813,height=742!
** If any day that will land in the menu cycle has a menu that is in record complete status, do not allow the assignment of the menu cycle or overriding 
** The pop up should display “*ONE OR MORE PRODUCTION RECORDS HAVE BEEN COMPLETED WITHIN YOUR NEW MENU CYCLE ASSIGNMENT DAYS, THIS MENU CYCLE CAN NOT BE ASSIGNED.*” in white font and red border
* Any menu deleted, should be a hard delete from both the MP and PR side
** Do not keep the menu line assignment in production and add another line like we currently do
* Any menu override should remove the old menu from MP and PR and replace with new menu in MP and PR
** Do not keep the menu line assignment in production and add another line like we currently do
