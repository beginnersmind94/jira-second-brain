# NXT-44448 — Multilevel Logging

## Description
We need to find a way to do multi-level history logging.

*Notes:*

* Dev manager will ultimately define the requirements for this story
* Front end for the story will be done in another story
** For frontend - needs to expose a new component flavor with the capability of using existing one and switch to newer one

*BackEnd*

* Currently all the Audit is a flat structure, we need to enhance to make it tree structure to support parent-child relation
* Create Models/Controller/Repos with V1 version and new below new properties
** ParentAuditId
** LogType (Possibile Values: INSERT, UPDATE, DELETE)



*FrontEnd*

Below image is a sample screenshot how the multilevel audit control looks like.

Basically the new control will allow to group the audit based on the parent and all related children needs to be displayed under each parent. The right side details view will be same as how the current control is.

* If the audit record doesn’t have any children then don’t expand (it will be flat record)
* If the audit record has children then it should expand and show all the children below the parent record
* if the child audit record still have children, it should further drilldown to expand to this children

## Acceptance Criteria
* Able to log parent-child audit data
* new solution should not break existing logging
* new solution can be used by all modules
