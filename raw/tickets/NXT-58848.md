# NXT-58848 — MVR - Optionally Save Column Order

## Description
As a MVR user, I would like to optionally set the column order for my managed view (MV) and managed view report (MVR) so that I can use set the column order I want

*Notes:*

* We want to give the user the choice to set the column order of the fields in the grid
* And then for the MVR to respect that when generating the report
* This request comes from FO team

*Requirements:*

* Use the Users grid to work on this and make sure it works
* On a managed view (MV) grid:
** Allow users to put the grid columns in the order that they want
** Use drag and drop if you can
* On a MV, when click the save or save as buttons, it opens the save screen
* On that Save screen, put a new field:
** Save Column Order
*** Toggle
*** Default Yes
*** If user selects yes:
**** include the column order in the MV configuration saved to the DB
*** If no:
**** Do not include the column order in the MV config in the DB
* When opening that managed view in the future, it sets the columns in the order of the managed view
* 
* When running an MVR, make sure that it puts the data in the order of the columns

## Acceptance Criteria
* User is able to decide if they want to save column order on any managed view
* Default is Enabled for the toggle
* When viewing that MV, the grid places the columns in the correct order
* When running an MVR, the data is in the order specified
* All existing managed views continue to work as is (don’t break)
* All existing MVRs continue to work and don’t break
