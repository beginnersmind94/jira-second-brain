# NXT-33674 — Import - Improved Import Exception Management - Item Details - API

## Description
When imports are performed, many exceptions can occur preventing numerous records of data from being imported. The objective of this story is to help narrow the importing user’s focus onto the records that could not be imported so that she/he can easily make corrections to the data and complete the import as fast as possible.

## Acceptance Criteria
* The Improved Import Exception Management UI shall be applied to the “Item Details Import” import.
* A UI shall be provided as a tab on the “Imports Monitor” page.
** The tab name shall be “Exceptions”
** The tab shall contain a table with the same columns as the import file
** 2 additional columns shall be pre-pended to the table
*** Row number - This shall correspond to the row in the import file
*** Message - This shall be the validation message returned by the module performing the business rules/validation
* If any row in the import file generates an error, the “Import File Data” tab shall be enabled and the table shall be populated with the data that is in the import file.
** Column-fields that are invalidated shall have their background-color set to light red.
** Import exceptions shall be made available for download in a csv file that matches the import template.
** The filename shall be the name of the import file with a “-exceptions” suffix
* The user shall be able to export the data in the table
