## Acceptance Criteria
* On Bulk Update, provide an edit for Item Description on Summary Info section
** Check item description uniqueness against items in the session and local items
** Provide alert that highlights items that have a duplicate description
* Update Categories or Branding remain the same for Bulk update
* Introduce column level edit which will only enable that column to be edited (this needs to be updated on Onboarding flow as well)
** Consume inventory status information to ensure proper preservation of selected categories
*** Synced and locked status - DO not allow to remove categories, they can only change categories
**** Do not give the clear X option on the drop down if in either synced or locked status
*** Published, Unpublished & incomplete - Allow removal or change of categories but display red alert that categories are missing (this needs to be updated on Onboarding flow as well)
* Provide a ‘Bulk Apply’ action to all columns to allow a single value to be applied for all selected items (this needs to be updated on Onboarding flow as well)
** All 3 categories
** Brand, Manufacturer, Tags
* “Bulk Apply” is only available in edit mode and is a single column edit and can not be used on the update categories or branding edit functions (this needs to be updated on Onboarding flow as well)
** All other columns in view/label mode
** Once user select or types information to bulk apply, allow user to select items and enable the “Apply to # items” button
* Once completed, user must save all information with the page level save to ensure all changes are saved
