# NXT-54083 — MP- Assign Menus- Assigning Menu Cycles to site group & delete scenarios

## Description
As a menu planner, I would like for the pop up to be more clear when assigning menus as to what menus have been previously assigned to particular sites so I know whether I am overriding menus or not

## Acceptance Criteria
* When assigning menu to all within a site group with a particular menu line, affect all sites and display that information on the pop up
* Display all sites affected and provide an accordion for all sites 
** Default accordion to closed with the accordion title the name of the site
** Each site should display menu being assigned, the day of the week, serving date and if there is any existing menu belonging to that site on that particular day and menu line



!image-20240821-062431.png|width=1013,height=632,alt="image-20240821-062431.png"!

* Start Date should be a label as there is no ability to change date on the pop up
* If a particular site has a menu already assigned and it is record complete or consolidated plan complete, do not allow menu to be assigned to particular site ALL OR INDIVIDUAL cases
** Display a strikethrough for that day and provide (RC square icon) next to existing menu name on the left
*** (RC square icon) with tooltip (Record Complete) then menu name
** Display a strikethrough for that day and provide (PC Square Icon) next to existing menu name
*** (PC square Icon) with tool tip (Consolidated Plan Complete) then menu name
* IF all sites (rare case) have different menus and are all trying to be overridden and are all record or consolidated plan complete, provide strikethrough for all menus on all sites and also disable the assign button as there is no action to be performed
* A menu can not replaced with the same menu, only a different menu can replace a menu
** Display a strikethrough on any date that happens to have a menu replacing an existing menu that is the same 
*** Provide (Same Menu) along with menu name and strikethrough
* IF a particular site has a holiday and other sites do not (rare case), check site that has holiday and do not allow assignment to be completed for that site
** Display a strikethrough for that day and provide (Holiday) like it is done when assigning for menu cycles
** User will have to go individually to that site if they would like to assign this menu 
* IF all sites have holiday then the holiday should display on the calendar and the user knows there is a holiday on that day and is still choosing to assign, allow all assignments to be completed
* After assignment, if any site deletes from individual site deletion, ONLY affect that site and not all sites in the site group and menu line, however if the deletion happens on the all selection, affect all sites that are within that site group and menu line
** EX for rare case: 
*** Menu was assigned for all selection, then individual site replaces menu, IF All selection is now deleted, DO NOT effect the replaced menu on individual menu on individual site
* IF user untoggles the skip holidays, currently we are treating the weekends as holidays and this is not correct, holidays are completely separate and should not affect anything related to weekends
* IN ANY of these cases, if the user uses “use all menus” toggle, system should only apply it to add more days for holidays only  
* WORST CASE SCENARIO: IF there happens to be a time where multiple menus are assigned on the same day/menu line, disable the delete option to preserve the data and we will have to handle manually
* Only consider assignment from effective date and moving forward
** Continue assignment for “new” site with all other sites 
*** “Use all menus” should not be considered in effective date case
** If there happens to also be an end date, hault assignment on end date
