# NXT-61850 — IM - Bulk Item - Serving Info - Header filters and UI enhancements

## Description
As an onboarding district admin I want filters within the Serving Info section so I can select a subset of items for a specific update, or find items with specific data conditions such as fluid/non-fluid.

!image-20250926-202132.png|width=366,alt="image-20250926-202132.png"!

## Acceptance Criteria
* Add filters to the Review Serving Info section (_More filters will be added with later, enhanced versions of the Serving Info section)_
** Provide dropdown for Non-Fluid/Fluid
*** All (_default_)
*** Non-Fluid (G)
*** Fluid (ML)
** Provide toggle to filter for Descriptions that are weight/volume “Show Items with Weight / Volume Description”
*** This will filter all items that have used a weight or volume, this is to encourage users to use proper descriptions instead of weight/volume for their serving descriptions
* Provide Instructions in a instruction help button and fly out (Mockup has it to the side just to show it), and keep instruction on update page as well
** *Select appropriate designation for an item as a Non- Fluid or Fluid Item along with providing Serving details such as serving amount and description and weight/volume for entered serving size.*
* Provide an toggle for missing data “Show Items with Incomplete/Missing Serving Info”
** This should work along with the main filter and the selections should be held for update
* On updating serving info, error highlights on the text boxes should not appear as soon as user starts typing information for a particular row and if the highlight is a must it should only highlight the text box in the same section, the two serving info sections can be saved independently
* Prefilled weight/volume should be left aligned (Currently center aligned)
