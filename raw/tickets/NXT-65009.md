# NXT-65009 — Account Categories Management (#2)

## Description
*As a* district business administrator configuring Financial Oversight 

*I want* to view and manage account categories imported from our Chart of Accounts 

*So that* I can organize GL accounts under the correct categories (Assets, Liabilities, Revenue, Expenses) and ensure proper financial reporting structure

Note: This is Parent Accounts in the PrimeroEdge version but renamed

## Acceptance Criteria
* Users are able to see the account categories that they imported into the system and their base type (sorted by alphabetical order based on category description)
* Users are able to edit the list of account categories
* When adding new account categories, a base type must be assigned (Revenue, Expenditures, Assets, Liabilities, Fund Balance) along with a unique description with a 50-character limit
* Add collapsible accordion at the top that reads "📁 Account Categories: Required custom groupings for detailed financial reporting.
** Every GL account must be assigned to a category within its base type (Assets, Liabilities, Revenue, Expenditures, Fund Balance)
** Create sub-groups where needed (e.g., split Revenue into 'Food Sales', 'Catering', 'Vending')
** Account codes for categories are optional - we recommend matching your defined account code structure for better reporting
** Transactions post to individual accounts only, not categories
* Users can search account categories by code or description using the search box
* Users can filter account categories by base type using the dropdown filter
* Display shows total count of account categories and child accounts for each category
* Account categories are displayed in a paginated table format with columns:
** Category Description (name of the account category, required, must be unique)
** Base Type (Revenue, Expenditures, Assets, Liabilities, Fund Balance in dropdown, required)
** Category Code (optional, may be blank)
** Linked Accounts (count of accounts under this category)
** Actions (Edit icon, Delete icon)

Empty State (No Account Categories):

* When no account categories exist (imported or created), display:
** The three help text accordions/textboxes as described above
** Empty table with column headers visible
** "Add Account Category" button
** No special empty state message or banner

Add Account Category Functionality:

* "Add Account Category" button positioned top-right of table near search/filter controls
* Clicking opens side drawer with fields:
** Category Description* (required, max 50 chars, must be unique)
** Base Type* (required dropdown: Revenue, Expenditures, Assets, Liabilities, Fund Balance)
** Category Code (optional)
*** If provided and doesn't match district's configured format: shows warning
*** Warning message: "⚠️ This code doesn't match your configured format. You can still save, but we recommend using the same structure for better reporting"
*** If provided, must be unique

Edit Account Category Functionality:

* Click edit icon in Actions column to open edit sidedrawer
* Edit side drawer pre-populates with existing values
* Same fields and validation as Add side drawer
* Success notification after edit: "Account category updated successfully"

What Blocks Saving:

* Category Description already exists
* Category Code already exists (only if code is provided)
* Required fields are empty

What Shows Warning But Allows Saving:

* Category Code doesn't match the configured account code structure (if provided)

Table Display:

* Successfully created/edited categories appear in table
* Categories without codes show blank in Category Code column
* Any category with non-matching format code shows ⚠️ icon next to Category Code
* Tooltip on warning icon: "This category code doesn't match your configured account structure format"

Success Messages:

* Add: "Account category created successfully"
* Edit: "Account category updated successfully"

Error Messages:

* Duplicate description: "This category description already exists. Please use a unique description."
* Duplicate code: "This category code already exists. Please use a unique code."
* Save failure: "Unable to save account category. Please try again."
