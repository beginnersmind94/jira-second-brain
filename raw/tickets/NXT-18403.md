# NXT-18403 — Income Surveys - Add New Survey - Students Step

## Description
As a district admin, I would like the ability to quickly and easily add Students to a manual income survey on behalf of a household, so that their income data is entered into our system for reference at any time.

*Summary*

Develop ‘Add New Survey’ wizard, with stepper as detailed below. Display the initial Students step as the landing step.

*Requirements*

* When the user selects ‘Add Survey’, redirect them to the survey wizard.
** Display the horizontal stepper with all steps: Students, Household, Details, Contact Info, Comments, Review
** Initial landing step: *Students*
* On the page, display a tooltip: _Search to add a student using the field in the top-left, or select 'Add New' to manually enter the details of a new student._
* Give the user the same 2 tools they have in the add application Students step:
** Find a Student
*** Unlike the Application wizard, we will handle this through a search field within the page rather than a modal (as the UX team is trying to reduce modal usage throughout the application).
*** {color:#6554c0}*This field does not need to be functional in this first story, just visible.*{color}
** Add a Student
*** This button should open a drawer for ‘Add Student’ with a form
*** We should display the following fields in the form:
**** Student ID
**** First Name _(required)_
**** Middle Name
**** Last Name _(required)_
**** Suffix
**** Date of Birth
**** Site
**** Grade
*** Display an ‘Add’ and ‘X’ button (which can close the drawer)
*** {color:#6554c0}*This drawer does not need to be functional in this story.*{color}
* These 2 functions need not be functional in this story, as adding a student to the survey will be handled in a separate story.
