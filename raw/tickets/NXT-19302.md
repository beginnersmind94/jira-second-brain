# NXT-19302 — Eligibility - Applications - Add support for Student Benefits search

## Description
As a district admin, I would like to be able to view, group, and export lists of students whose applications indicated that their households wish to receive Optional Benefits for a given academic year or have unique Info Sharing preferences, so that I can choose which data to provide to any appropriate third-parties.

*Summary*

# Add a multi-select ‘Benefits’ filter to the Applications page
# Add a ‘Benefits’ column to the Applications page grid.
# Add an ‘Info Sharing’ column to the Applications page grid.

*Requirements*

* Same objective as [https://primeroedgenext.atlassian.net/browse/NXT-18534|https://primeroedgenext.atlassian.net/browse/NXT-18534|smart-link], but on the Applications page instead. 
* In Eligibility → Forms → Applications, do the following:
** {color:#6554c0}*(May not yet be shown in mock-up)*{color} In the list of filters, add a new drop-down filter: _Benefits_
*** Menu should be *multi*-select
*** Menu should show all of the benefits associated +only with the selected Academic Year+, even inactive ones
** In the list of columns, add a new column: _Benefits_
*** This column should be hidden by default
*** When the user chooses to show this column, the column should show any benefits that were selected in association with each application record
**** If none exist for the application, this column can be blank
** In the list of columns, add a new column: _Info Sharing_
*** This column should be hidden by default
*** When the user chooses to show this column, the column should show Yes/No in association with each survey record
**** If no data exists for the survey, this column can be blank
** When a user applies the changes to either filter, only show those applications where those particular benefits were accepted or info sharing preference was indicated.

## Acceptance Criteria
Verify that both a ‘Benefits’ filter has been added to the Applications page

Verify that both a ‘Benefits’ column and ‘Info Sharing’ column have been added to the applications column search

Verify that these columns display any benefits/info sharing preference tied to the application record, and are hidden by default

Verify that when a user applies these options, the grid only shows those applications where those benefits were accepted or info sharing filter is set to the selected value (even if the columns are hidden).
