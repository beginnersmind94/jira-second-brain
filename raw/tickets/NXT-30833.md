# NXT-30833 — IM - Items - Create New Item - Inventory changes for Unit Configuration

## Description
As a District Admin I need the Inventory Items to be saved according to the Item Management Unit Info configuration so that I am working with the item units on Inventory transactions instead of the main item.

*Note:* this story has large impacts on the way items and units are saved to the DB. This also has impact on the syncing between IM and Inventory.

*Requirements*

* Move the hierarchy table (now “Unit Info”) to Item Management from the Inventory DB
* Change the item references to consider the item’s unit (sub-item) configuration instead of referencing the Item Management “main item”
* Modify the schema
* Translate the existing inventory data (in both Dev & QA)
** to ItemUnit (Technical in DB Schema InventoryItemId)
* QA team will go through and identify the broken pages (since we are depending on the item hierarchy on some - like perpetual inventory and production orders) 
** Based on the input we can create future stories and work on them.
