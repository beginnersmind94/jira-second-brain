# NXT-14853 — Platform - PII - Implement PII component at Section level

## Description
*Persona*: Perseus User

*Access level:* School District/Site

*Description*:

As a Perseus user, I should be able to view the sensitive data at field/section/page level only when I have access to a role that has view permissions. 

*PII Fields:*

* First Name
* Last Name 
* Parents Name
* Address
* Student Number
* Birth Date (DOB)
* Gender
* Ethnicity
* Race
* Eligibility

*Sensitive Sections: (HIPAA, FERPA compliance)*

* Allergens
* Eligibility
* Household Information

*Requirements:*

* All the fields identified in the above sections needs to be masked/hide for user who do not have PII access roles/permission
* Sections will be marked as PII under resource catalog
* PII sections are enabled only for roles with PII attribute enabled.

!SectionLevelPermision-20220426-135329 (b5fe5f34-7347-431c-8454-fd7a5e5785f9).png|width=1536,height=1075!

## Acceptance Criteria
* User shall be able to see sensitive sections only when permissions are enabled for the roles specific to PII
* User shall not see the sensitive data even at page source or downloadable files
