# NXT-47959 — POS - Transactions - Report option to show menu items sold, added to main menu

## Description
As Sonya the Site Manager I want to see the meal count report off the main POS page for ease of access.

*mockup*

!87b71d61-1bf9-499d-9590-713bd3db8e45#media-blob-url=true&id=a2e1d2cb-bd05-461d-bf29-aebede68aeb8&contextId=71075&collection=|width=2365,height=651!

This already exists in the POS - we’re goin to add a copy of it to the main menu.

*requirements*

* Add a new main POS menu option:
** Item Sales Counts
* Copy the Meal Counts report option off POS > Transactions, but we’ll do some enhancements:
** Rename it to Item Sales Counts
** User can see a summary of meal counts (existing feature) for the selected date (default)
*** Currently this is all tied to current session - we enhance this to look at all sessions for the date.
** User can see what menu items were sold, and how many were sold.
*** Replace the ‘Meal Type’ with the Menu Item
*** Show how many were sold to each price type
*** Show totals for Reimbursable and Total
** User can see a list of sessions that were sold for the day.
*** User can click into each to see the Item Sales Counts for that specific session.

## Acceptance Criteria
* Item Sales Counts option added to POS main menu.
* User can see summary of meal counts for all sessions on the selected date 
* User can see menu items sold + sale counts
* User can see individual sessions and click into them
** User sees meal counts + menu item sales for that session in particular
