# NXT-25268 — Custom User view enhancement

## Description
During the integration on Custom Views on the accounts screen the following requirements were discovered:

# Custom Views API needed to ensure that for a given data model or screen and a user there should only be one default view. 
# The view name of a view should be unique in a list of views for a user and shouldn’t be reused for the same screen by the user. 

We need to enhance the Custom View’s api and enhance this functionality.

## Acceptance Criteria
I know this is done when:

# Users is allowed to set only one view as their default on a screen.
# Users should not be allowed to repeat the the same view name on a screen.
