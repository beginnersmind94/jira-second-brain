# NXT-54110 — POS - Offline Mode - Login

## Description
As Cena the Cafeteria Manager I have no internet connection but need to log into the POS.

*requirements*

Support offline login and other findings from research task → [https://primeroedgenext.atlassian.net/browse/NXT-27570|https://primeroedgenext.atlassian.net/browse/NXT-27570|smart-link] 

* User can login to the POS while offline.
* User has access to ionic icons.
* We can ignore the POS Period check while they’re offline.
* User sees an offline banner at the bottom of POS.
** Text: You are currently working Offline.
* User sees a ‘back online’ banner at the bottom of POS when they’re back online.
** Text: You’re back online. Session data will begin streaming to the server shortly.
** This banner can be dismissed. It can also automatically dismiss after 15 secs.

----

*Research Notes*

*Authentication, Terminal and User Profile Data:*
To make it work offline from login to opening session flow, we need to save the recent success responses from APIs in the browser (PouchDb/LocalStorage) and use them in offline mode.
Currently, we have the offline API data bucket in PouchDB, which is used when APIs are down. We can use the same at the development stage and store other required data offline.

*CheckPOSPeriodStatus API data:*
CheckPOSPeriodStatus API data can also be saved with the date selected. but changing the session date may trigger a new API, and you won't be able to get details.
It's good to have at least a week of checkPOSPeriodStatus data in the PouchDB to continue offline for more than a day.

*Icons:*
All Ionic icons can be downloaded prior to the login.

*Other limitations we still have are:*
We cannot get visiting student's data.
Switching sites may not be possible offline.
Sessions, transactions, and payments will be in the PouchDB. Once the website connected to the server, those will be synced.
