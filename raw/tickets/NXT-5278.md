# NXT-5278 — Verification - Collection Report (FNS-742) - Add report details (Part 1)

## Description
As a district admin, since I worry a lot about my numbers being accurate during verification, I would like to have reporting that backs up the numbers (it would be a welcome relief).

*Mockup*

[https://www.figma.com/proto/Yo99YhQm5Xk91Ie7b0PRkD/Front-Office-6.0?node-id=250%3A10219&scaling=scale-down-width&page-id=250%3A10218&starting-point-node-id=250%3A10219&hide-ui=1|https://www.figma.com/proto/Yo99YhQm5Xk91Ie7b0PRkD/Front-Office-6.0?node-id=250%3A10219&scaling=scale-down-width&page-id=250%3A10218&starting-point-node-id=250%3A10219&hide-ui=1|smart-link] 

*Requirements*

* In Eligibility → Verification → Reporting → Collection Report, add a detailed view that displays a breakdown of the data shown in the Collection Worksheet.
** This view will involve using multiple grids which breakdown the data in a more understandable format for the user, to allow them to cross-check the data from the worksheet. 
** This view (‘report’) will only be accessible when clicked through the Collection Worksheet. It can be given a separate URL but does not need to have a dedicated page/nav-item.
* Within this view, user should also be able to export the grid data to PDF or Excel (whichever is more feasible).



For each tab:

* Display app/student/site totals in the tab header (note that this number *must* absolutely +match+ the cell the user clicked from on the form). May need to check this as part of [https://primeroedgenext.atlassian.net/browse/NXT-30235|https://primeroedgenext.atlassian.net/browse/NXT-30235|smart-link] .


Information to show for sites:

* Site Code
* Site Name



Information to show for students:

* ID #
* First Name
* Last Name
* Grade
* DOB
* Eligibility Code
* Reason
* App #
* Case #
* Eligibility Start Date
* Eligibility End Date
* Action icon that should redirect user to student’s Account Info page.


Information to show for applications:

* App #
* Student ID #s
* Student Names
* Action icon that should redirect user to app’s Application Details page.


Information to show for results (section 5):

* App #
* ID #
* Completion Date
* Results / Changes
* Reason for Changes
* Action icon should redirect user to Verification Results page.

## Acceptance Criteria
* Every number on the Collection Report can be clicked on to show detailed information of what made up that number.
** Including new hover+clickable icons in the 4 sections above.
* Numbers shown detailed information views should match the Collection Report.
* User can export any of these reports out to PDF or Excel.
* 'Back' button added to return.
