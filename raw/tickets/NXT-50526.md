# NXT-50526 — INV/PR- Production Orders- Consuming Prepared Items on Prep Site 

## Description
As a Prep Site Manager, when I am pulling a production order for a particular data range, I would like to be able to pull all prepared items from the consolidated plan on the production side to ensure that I am ordering enough items for all the plans that I need to prepare for.

## Acceptance Criteria
* Pull Prepared Menu Items based on Prep Site to receiving site relationship based on prep site configuration or single menu line prep site config
** IF site is its own prep site and does not have any other receiving sites associated with it, treat as normal menu item and pull from plan (Current Functionality)
** IF site is its own prep site and HAS other receiving sites associated with it, pull information from the consolidated plan and combine all total based on saved consolidated plan data and disregard counts from the plan page for all prepared items for both prep site and receiving sites
* In the case that a prep site were to go ahead and finish there distribution and all items are marked as shipped before a production order is placed, do not show items on the production orders to be ordered
* For receiving sites, *only* pull non prepared menu items and display items needed to order without including prepared items
* For prep sites, pull all consolidated plan prepared item information as well as non prepared menu items from the plans and display all items needed to order
* Only pull allow production orders to be created for a particular data range if all plans and consolidated plans are completed (Currently only looking at plans)
** Display Message “*Site Name has following incomplete production plans for the period selected”* and display a (Consolidated Plan) for consolidated plans on the individual lines
*** Provide separate lines if both the plan for the prep site and the consolidated plan for the same serving date, meal type, and menu line
