# NXT-71354 — Insights - KPI - PNA - Drawer - SC

## Description
*I am a* district user *that I want to* open the Paid Not Applied (PNA) details drawer from Insights using SchoolCafé data
*so that I* can review the actual value, expected value, difference, and school-level detail behind the KPI.

----

h2. *Notes*

* This story applies to *Insights – SchoolCafé data source*.
* PNA (Paid Not Applied) is part of the *front office KPIs* being integrated into Insights.
* This story covers the *PNA Details Drawer only*.
* This story is a *companion to the PrimeroEdge PNA Details Drawer story*.
* The *frontend behavior, layout, interactions, formatting, sections, and drawer pattern must remain the same as the PrimeroEdge implementation*.
* Frontend developers should *not need to build new UI logic* for SchoolCafé.
* This implementation must reuse the same:
** drawer layout
** header behavior
** PNA Overview section
** Details by School section
** About this Metric section
** formatting behavior
** open/close behavior

h3. *Data & Source of Truth*

* The source of truth for PNA data is available in:
** *SchoolCafé → Eligibility Module → Reports → Roster Report*
* To generate the correct dataset:
** Eligibility = *Paid*
** Basis = *Default*
** Click *Apply*
* This must be used by developers and QA to validate PNA calculations and displayed values.

h3. *Stored Procedure Integration*

* The front office team has already created the stored procedure:
** SchoolCafé: [https://primeroedgenext.atlassian.net/browse/NXT-67455|https://primeroedgenext.atlassian.net/browse/NXT-67455|smart-link] 
* The details drawer must use the *SchoolCafé stored procedure* as the data source instead of the PrimeroEdge stored procedure.

h3. *Launch Behavior*

* If launched from the *PNA KPI card*, the drawer must show data for the full filtered set.
* If launched from a *School Performance grid row*, the drawer must show only the data for that selected school row.

----

h2. *Out of Scope*

* PrimeroEdge implementation (covered in separate story)

----

h2. *User Flow*

* Same as *PrimeroEdge PNA Details Drawer story*, except data is sourced from SchoolCafé.

----

h2. *Requirements*

h3. *1. Shared Frontend Implementation*

* The SchoolCafé implementation must reuse the *same frontend behavior and UI* as the PrimeroEdge details drawer implementation.
* No new or separate frontend workflow should be introduced for:
** drawer layout
** overview section
** school detail grid
** about-this-metric section
** open/close interactions
** formatting behavior

----

h3. *2. Backend Data Source Change*

* The backend must use the *SchoolCafé PNA stored procedure* instead of the PrimeroEdge stored procedure.
* This data source change applies to:
** header date context
** actual value
** expected value
** difference calculation
** school-level detail rows

----

h3. *3. Functional Consistency*

* All SchoolCafé PNA details drawer behavior must remain functionally identical to the PrimeroEdge implementation

----

h3. *4. Launch Context Behavior*

* If the drawer is opened from the *PNA KPI card*, it must show the full filtered dataset.
* If the drawer is opened from a *School Performance grid row*, it must show only the selected school row.

----

h3. *5. Data Validation*

* SchoolCafé PNA values must match:
** SchoolCafé stored procedure output
** SchoolCafé Eligibility Roster Report

----

h3. *6. Performance Requirement*

* The SchoolCafé-backed implementation must perform at the same level as the PrimeroEdge implementation and must not introduce performance degradation.

## Acceptance Criteria
h3. *1. Shared Frontend Implementation*

* SchoolCafé PNA details drawer uses the same UI and behavior as PrimeroEdge.
* No separate frontend implementation is introduced for SchoolCafé.

----

h3. *2. Backend Data Source Change*

* Drawer uses the SchoolCafé PNA stored procedure.
* Overview values come from SchoolCafé data.
* School detail grid values come from SchoolCafé data.

----

h3. *3. Functional Consistency*

* Header behavior matches PrimeroEdge.
* Overview section matches PrimeroEdge.
* Details by School section matches PrimeroEdge.
* About this Metric section matches PrimeroEdge.
* Formatting matches PrimeroEdge.

----

h3. *4. Launch Context Behavior*

* Opening from the KPI card shows the full filtered dataset.
* Opening from a school row shows only that selected school.

----

h3. *5. Data Validation*

* PNA values match SchoolCafé stored procedure output.
* PNA values match the SchoolCafé source of truth report.

----

h3. *6. Performance Requirement*

* Drawer performance is consistent with PrimeroEdge implementation.
* No performance degradation is introduced.
