# NXT-2985 — Employee Mobility 

## Description
Admin has the ability change the primary site of an employee (add/remove secondary sites) to associate the employee with a new district as needed.

* Ability to recognize this is a current employee and not create duplicate employee records in different districts within same state
** Need to research state transfers (if state is using primeroedge, theoretically we should be able to transfer them to a different ‘district’ within another state. 
** If the state has their own server, then need the ability to EXPORT a record and IMPORT it into the state’s system/server  
* Central Realm v. District Realm 
** At central realm, admin (with permission and state ID setting enabled) can move an employee to a site within another district. 
** at district realm, admin can move an employee to another site within the same district. (State ID setting does NOT have to be enabled for in-district mobility)
* State ID identifies the employee across state 
** State ID setting must be enabled to transfer?
* Scenario: Employee gets a promotion and moves to another district -
** need to transfer that employees records to the corresponding district 
** history tracks the move 
** reporting for employee data previous to move will show the prior district 
** reporting after the transfer, will display current data 
* Ability to EXPORT an employee record for tracking (new PBI?)
** Record includes data from all sites/districts tied to that ID number

## Acceptance Criteria
Employees have the ability to move sites and districts with their entire employee record
Ability to export employee data (professional, education, training record, history, etc.)
