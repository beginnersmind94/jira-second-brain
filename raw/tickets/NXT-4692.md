# NXT-4692 — Item Management - Allergen Information - Validation and Checks

## Description
*User Story*

As a user, I need to be able to not fill in any allergen information, specify there are no allergens indicated, or select standard / custom allergens.

*Mockup*

!image-20200804-125506.png|width=1108,height=748!



!image-20200804-124819.png|width=1047,height=622!



 

*Requirements*

* When navigating to the Allergen section on 'Add new food item' the default is as the picture in the mockup
** Default value is 'null' or nothing selected
* If the user selects 'No Allergens Indicated' then the grid below disappears
* The user can only select one value per allergen (standard or custom)
** Multiple values are not allowed
* Order of the columns to be as follows: This should appear on the view and the add/edit screen [~accountid:5dc4cd94c424110de525c2b3] [~accountid:5dd323b81cf3c20ef5ff2ce0] 
** Contains
** Processed in a facility that also processes
** May Contain
** Free From
* When adding an item sort by allergen name
** example: Contains are sorted, then a-z
* When editing an item, sort by indicator first in the order below
** Contains, Processed in a facility that also processes, May Contain, Free From
** After this, then sort by name a-z
* Show the same disclaimer on the add allergen screen [~accountid:5dc4cd94c424110de525c2b3] Please update the mockups
** Always visible and not hidden (not the same as view page)
** Below Custom Allergens
* When adding a custom allergen, but no indicator is selected when trying to click ‘ADD'
** Inline error stating:
*** Please select an Allergen Indicator
** Allowing the user to delete / remove data is coming in story [https://primeroedgenext.atlassian.net/browse/NXT-4690|https://primeroedgenext.atlassian.net/browse/NXT-4690|smart-link]

## Acceptance Criteria
* The user can create or edit an item to not have any allergen information filled out, indicate there are no allergens, or that there are standard or custom allergens
