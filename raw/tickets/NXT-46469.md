# NXT-46469 — Data - Source Data - Add New

## Description
As a SchoolCafé admin, I would like to be able to execute API calls from SIS providers w/o having to import the data so that I can gather information and debug issues.

*Notes:*

* We are leveraging the existing import features to be able to pull and view data w/o having to import
** Therefore on this screen, we do not need Field Mapping, Schedule or review screens
** We can consolidate the rest of the screens onto one

*Requirements*:

* On the System > Imports & Exports > Data tab, do the following:
** If user clicks SIS Source Data in the + New button, open the Add New SIS Source Data drawer



*Add New SIS Source Data drawer*

* If the drawer does not already exist, create it
** Title: Add New SIS Source Data
** Width: full page because of the grid
** Use sliding motion to open
* 
* Fields:
** Name
*** single line textbox
*** Use same requirements as Name field on Review tab for + New Imports
** Description
*** Multi-line textbox
*** Optional
** Source Data
*** Single Select drop down list
*** Required
*** Values:
**** Student Import
**** Adult Import
**** Sites
**** Photos (if provider is Aeries and if done for PowerSchool)
** Source Data Has Personally Identifiable Information (PII)
*** Checkbox
*** Set value based upon DB IsPII for the import type
*** Make read only
** Grid or multi-select list
*** This is the list of fields for the source data
*** List of fields depends on the Source Data selected
**** Update the list when the user changes the source data
*** Show all the fields available
**** You can use the same we use for Show all fields for the API Imports
*** The selected fields are the ones that will be shown in the excel file when generate the download
*** If grid:
**** Checkbox column
**** Field Name
*** If list:
**** Show name and be able to select many
** Preview
*** Button
*** Do just like the Preview button in the API Add New
*** Show first 10 rows in a grid below like API Add New
*** Ensure IsPII if Source Data has IsPII setting in DB
**** User cannot preview data if data IsPII and user is not (but they can still create the config)
**** If they try to preview and can’t, show them the IsPII violation message like we do for API imports
** Save
*** On click:
**** Validate fields
**** Save to DB
**** Show green success message
**** Refresh grid in Data tab to show now row
* 
* When saving, save to the history audit log as well so that it will show up in the View History drawer
* 
* Security
** Need Add rights to Imports & Exports page

## Acceptance Criteria
* User is able to select a data source
* User is able to see the fields from that data source and select which ones they want
* User is able to view a preview of the data
* IsPII is respected for previewing data
* User must have Add rights to parent page
