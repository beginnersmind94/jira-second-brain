# NXT-46468 — PR- Production Adjustment- Inactive and Pending Deactivation Menu items

## Description
As a production manager or admin, when I am using the production adjustment feature, I would like to make sure that the system lets me know which menu items are marked for pending deactivation so I can decide if I would like to still use that item or not when making adjustments.

## Acceptance Criteria
* Do not show inactive menu items in the drop down
** Do not let user search for inactive menu items
* Pending Deactivation Menu items should show in drop down and be able to be searched
** Provide warning icon and tool tip “Pending Deactivation”
* Allow pending deactivation menu items to be used for adjustment (This is up to the user but we will provide the warning)

!image-20240116-193737.png|width=1014,height=412!

* Once user selects a menu item, continue to provide the warning icon and tool tip
* Do a status check once user clicks on “Select Plans” to make sure the menu item is available and not inactive 
** IF menu item is not available and is inactive, Provide an error message “ (*Menu Item*) has now been made inactive, Menu item can not be added.
* Unlock the menu item so that user can select a different menu item and remove the menu item from the list
* Once user selects plans with a available menu item and selects all the plans they would like to adjust, when user clicks submit adjustments, again provide a status check
**  IF menu item is not available and is inactive, Provide an error message “ (*Menu Item*) has now been made inactive, Menu item can not be added.
