# NXT-9378 — INV - Orders - Orders List - Merge Orders

## Description
As an Inventory manager I want a Merge Orders tab on the Orders List so I can select orders with common criteria to potentially be merged.

+*Search/selection criteria:*+

* Site
** only allow single selection
** only orders for the same site
* Vendor
** allow All selection
** only orders for the same vendor
* Delivery Date Range
** Default to a single day 
*** default to one week ahead
** Allow user to expand date range
** only orders with the same delivery date (current functionality)
** could possibly allow orders with different delivery dates to be merged, as long as users change the delivery dates to match
* Status
** allow All selection
** Statuses:
*** Open (only if no receipts are created)
*** Incomplete
*** Pending Review
** only orders with the same status can be merged

+*Order list:*+
(bolded attributes must be the same for orders to be mergeable) 

* Order #
* *Site*
* *Vendor*
* *Status*
* *Delivery Date*
* *Delivery Location*
* Order Date
* Order Amount
* User  

+Feature+

* reorganize/group the Orders list by similar orders
** Same Site 
** Same Vendor
** Same Delivery Location
*** we could display orders with different Delivery Locations, but the user can only merge orders that have the same Delivery Location
** Same Status
*** We will display all orders regardless of status (Open, Incomplete) but user can only merge orders that have the same status
*** in Merge Mode, orders with the following statuses would *not* be included:
**** Closed
**** Cancelled
**** Open but w/ Receipt (at least one Receipt)
** Same Delivery Date
*** Orders must have the same Delivery Date to be merged
* Provide a Merge button for each mergeable grouping
** Provide a checkbox for each Order within the grouping
* *Second Step* - when user merges orders, they will remain on the Merge Orders tab and see a success message indicating the updated Order #
** Validations:
*** The Delivery Dates for merged orders should all be the same
*** The Delivery Location for merged orders should all be the same
** Pick one of the Order Numbers to be the new Order # for the merged orders
*** Use the Order # of the most recent order as the new “Primary Order Number”
** Show a success message
*** “The selected orders have been merged into Order #5678.”
