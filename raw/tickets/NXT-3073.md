# NXT-3073 — *** System - Sites - Site Types - Add New Site Type

## Description
As an administrator, I would like to be able to add a new site type for my district, so that I can manage non-traditional site types within PrimeroEdge.



* In the Site Types page, add an *Add Site Type* button
* When the button is selected, open an interface that allows the user (having appropriate permission) to add a new Site Type using the following 3 fields:

# ({color:#ff5630}required{color}) Site Type Description - Form field
# ({color:#ff5630}required{color}) Site Type Type - Drop-down. Default value should be ‘School Site Types’
# ({color:#00b8d9}optional{color}) Grades - Multi-select field that should allow the user to add/remove grades from the site type. List of grades should come from the database.  

* Existing field validation rules for description, as well as existing site type types and default grades can be carried over from PrimeroEdge Classic
* Within the ‘Add Site Types’ form, add an *Add New Site Type* button and *Cancel* button. 
** When the user selects the 'Add New Site Type' button and field validations are successful, add the new site type to the district and return the user to the main Site Types list/view.
** When the user selects the ‘Cancel’ button, return the user to the Site Types main page.



We can use the PrimeroEdge Classic layout for the most part, with the additional multi-select for adding grades to the site type. Previously, this was done in the main site types list/grid.

## Acceptance Criteria
1. Within the Site Types page, an Add New Site Type button has been added and is accessible to users with the proper role permission
2. When that button is selected, the user is presented with three fields: Site Type Description (req.), Site Type Type (req.), and Grades (optional). 
3. The default option in the Site Type Type drop-down menu is 'School Site Types' (Database code SCHL)
4. When the user selects the 'Add New Site Type' button and field validations are successful, the new site type is added to the district and the logged-in user is returned to the main Site Types list/view.
5. If the validation does not pass or the description field is blank, the 'Add New Site Type' button is disabled.
6. When the user selects the ‘Cancel’ button, they are returned to the main Site Types list/view.
