# NXT-18602 — IM - Items - Display Commodity Code per USDA Requirement

## Description
Per USDA approval requirements, the Commodity Code field, which is provided by the CN Database, must be displayed as a field on items as well as a search option (column) in the Item list page.

USDA Wording: *_Show the commodity code from the Commodity or Recipe Code field of the FDES table, for source code 4 food items on the item screen._*  *From nutrient analysis specifications document:The software must show the commodity code from the Commodity or Recipe Code field of the FDES table. The user must be able to search by the commodity code and the commodity code must also be displayed on search results.*

+*Requirements*+

* Search needs to be a smart search 
* Commodity code filed should allow special characters to be used
* Commodity Code should be displayed on the Item details when it exists for an item
* Commodity Code should be obtained from the CN Database
* Commodity Code should be a column option in the Items list
** Field Search should be enabled for this column, to meet search requirement
**  Commodity code should also be displayed on the ingredient report
* This field should be optional
* This field should be unique
* This field should take the location of the ‘Broken Unit GTIN' field, which is being removed by the INV1 team in Sprint 51

To meet these USDA requirements I need  to able to:

* When adding a new Item and the item is a food item then ask is this a Commodity Item 
** If the Item is a commodity Item then ask to enter the commodity code 
** Commodity Code is a free text box and is unique to the item
