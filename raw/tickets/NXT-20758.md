# NXT-20758 — Update FFD Website to Use 2.0 Data

## Description
There needs to be a way to determine if a site is from classic or 2.0 and then call the appropriate APIs. 

*Application*: FFD Website

*Flow:*

* User browses to page from source system
* System determines if from classic or 2.0
* System makes calls to APIs based upon if from classic or 2.0

 

*Assumptions:*

* This story assumes the user is logged in from the source site and browsing here by clicking on the FFD link and not logging in on the Login page of the FFD website
** Login from FFD website is covered in a separate story
* It is assumed the APIs from classic have been created in 2.0 already

 

*Requirements:*

* However you want to do it, determine if the user is coming from classic or 2.0 and store that
* Call the APIs from whichever one you need to
*

* N/A

## Acceptance Criteria
* If user is on a 2.0 site, it calls the APIs from the 2.0 site
* If user is on a classic site, it calls the APIs from the classic site
