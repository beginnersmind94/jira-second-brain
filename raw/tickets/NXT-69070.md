# NXT-69070 — Account Mapping Wizard

## Description
*As a* district financial administrator,

*I want* a guided wizard to map system accounts to GL accounts,

*So that* SchoolCafe can generate accurate ledger entries for revenue, reimbursements, and expenses.

## Acceptance Criteria
_Page Header & Navigation_

* Page accessible from sidebar under Financials > Configuration > General Ledger Configuration

_Status Banner_

* When unmapped accounts exist, show warning banner with text: "[X] mapping(s) incomplete — Ledger entries cannot post until all accounts are mapped."
* Warning banner includes secondary button "Complete Mapping" that opens wizard
* When all accounts mapped, show success banner with text: "All accounts mapped! — Ledger entries are ready to post."

_Summary Cards (4 cards in horizontal row)_

* Card 1: "Total Accounts" with dynamic count and subtext "System accounts configured"
* Card 2: "Mapped" with dynamic count (styled green) and subtext "Ready for posting"
* Card 3: "Unmapped" with dynamic count (styled amber) and subtext "Action required"
* Card 4: "GL Accounts Used" with count of unique GL codes and subtext "Unique GL accounts"
* Regular accounts count as 1 toward total
* Valuation categories count as 2 toward total (Asset mapping + Expense mapping)
* Valuation category only counts as "mapped" when BOTH Asset and Expense accounts are selected

_Filters_

* Section dropdown: All Sections, Revenue & Reimbursements, Expenses
* Base Type dropdown: All Types, Asset, Revenue, Liability, Expense
* Status dropdown: All, Mapped, Unmapped
* Search text input with placeholder "Search system accounts..."
* Reset button to clear all filters
* Filters apply immediately on change

_Account Table Structure_

* Table columns: System Account, GL Account, Status, Actions
* Accounts grouped into collapsible sections
* All groups collapsed by default on page load

_Section Dividers_

* Display section divider row when transitioning between Revenue & Reimbursements and Expenses
* Section divider shows badge with section name, description text, and mapping progress (e.g., "13/16 mapped")
* Revenue & Reimbursements description: "POS transactions, program meals, and federal reimbursements"
* Expenses description: "Inventory valuation and cost of goods"

_Collapsible Group Headers_

* Each group has a header row showing: expand/collapse chevron, group title, account count, mapping status summary
* Account count displays "[X] accounts" for regular groups or "[X] categories" for valuation groups
* Status summary shows green dot with "[X] mapped" and amber dot with "[X] unmapped" (if any unmapped)
* Clicking group header toggles visibility of accounts within that group
* Chevron rotates to indicate collapsed/expanded state

_System Accounts (from backend)_

* Each system account has a unique appAccountCode identifier
* Each system account has a baseType: Asset, Revenue, Liability, or Expense
* GL Account dropdowns filtered to only show GL accounts matching the system account's baseType

_System Account List by Group_

Bank & Receivables (Revenue & Reimbursements section):

* Cash in Bank (appAccountCode: CASHBNK, baseType: Asset)
* Due from Federal Funds (appAccountCode: DUEFEDF, baseType: Asset)

Meal Sales (Revenue & Reimbursements section):

* Meal Sales Accounts (appAccountCode: SALES, baseType: Revenue)

Reimbursements & Commodities (Revenue & Reimbursements section):

* Reimbursement Program Accounts (appAccountCode: RMBRSMNT, baseType: Revenue)
* Commodity Revenue (appAccountCode: COMMREV, baseType: Revenue)
* Planned Assistance Level (PAL) (appAccountCode: COMMAST, baseType: Revenue)
* PAL Estimated Revenue (appAccountCode: ESTREV, baseType: Liability)

POS Revenue Adjustments (Revenue & Reimbursements section):

* Additional Income (appAccountCode: ADDINCM, baseType: Revenue)
* Bank Fees (appAccountCode: BNKFEE, baseType: Revenue)
* Deposit Variances (appAccountCode: DPSTVAR, baseType: Revenue)

POS Liabilities (Revenue & Reimbursements section):

* Prepaid Account (appAccountCode: PPACC, baseType: Liability)
* Refunds (appAccountCode: REFUND, baseType: Liability)
* Returned Checks (appAccountCode: RTNCHCK, baseType: Liability)
* Sales Tax Owed (appAccountCode: STTAXOW, baseType: Liability)

POS Expenses (Revenue & Reimbursements section):

* Account Adjustments (appAccountCode: ACCTADJ, baseType: Expense)
* Bonus (appAccountCode: BONUS, baseType: Expense)

Payables (Expenses section):

* Accounts Payable (appAccountCode: AP, baseType: Liability)

Inventory Valuation (Expenses section):

* Valuation Group Inventory Asset (appAccountCode: INVENTORY, baseType: Asset)
* Valuation Group Inventory Expense (appAccountCode: INVEXP, baseType: Expense)
* Note: Each valuation category from Item Management maps to both an Asset GL account and an Expense GL account

_Validation Rules_

* Due from Federal Funds (DUEFEDF) and Cash in Bank (CASHBNK) must be mapped to different GL accounts
* Prepaid Account (PPACC) and Refunds (REFUND) must be mapped to different GL accounts
* Prepaid Account (PPACC) and Returned Checks (RTNCHCK) must be mapped to different GL accounts
* Prepaid Account (PPACC) and Sales Tax Owed (STTAXOW) must be mapped to different GL accounts
* Display inline error message when validation rule is violated
* Prevent saving when validation rules are violated

_Inline Editing with Typeahead_

* Each account row has a searchable text input for GL Account selection
* Clicking or focusing the input reveals a dropdown of matching GL accounts
* Typing in the input filters the dropdown to show only GL accounts whose name contains the typed text
* Dropdown only shows GL accounts that match the system account's baseType
* Selecting an option from the dropdown populates the input and saves the mapping
* Clearing the input and clicking away removes the mapping
* Unmapped inputs display with dashed border styling and placeholder text "Type to search GL accounts..."

_Valuation Category Rows_

* Display category name
* Two inline typeahead inputs side by side: Asset Account and Expense Account
* Asset input filters to show only Asset-type GL accounts with placeholder "Type to search Asset accounts..."
* Expense input filters to show only Expense-type GL accounts with placeholder "Type to search Expense accounts..."
* Status shows "Complete" when both mapped, "Incomplete" when one or both missing

_Auto-Save Behavior_

* Selecting a GL account from the dropdown immediately saves the mapping
* Table and all summary statistics refresh after each change
* Status banner updates based on new mapping totals

_Actions Column_

* Each row includes a history button to view mapping change history
