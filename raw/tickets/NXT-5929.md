# NXT-5929 — Eligibility - DC Precedence

## Description
*User Story*

As a district admin I want to control the order of direct certifications & approvals for approval purposes.

*Mockup*

[https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=6544%3A100491&scaling=scale-down-width|https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=6544%3A100491&scaling=scale-down-width]

*Business Information*

* This page will contain the available Direct Certification programs along with their precedence order.
** This is saved by each district but we’ll have a master order.
** This data is used to determine if a student’s existing eligibility needs to be overwritten.
*** Example: Student is currently Free, reason of Migrant.
*** Any approvals higher than that in the order would impact their price type / eligibility.
**** DC SNAP, TANF, etc.
*** Anything lower would result in no change to the student.
**** Homeless, Runaway, etc.

*Requirements*

* Add a page called ‘DC Precedence’
* Show a grid on the table outlining the programs, and their precedence order.
** Columns:
*** Reason
**** Level 1 (Assistance Programs)
***** DC SNAP
***** DC TANF
***** DC FDPIR
***** DC Medicaid (Free)
***** DC Foster
**** Level 2 (Other Source Categorical Programs)
***** Migrant
***** Homeless
***** Runaway
***** Head Start
***** Even Start
***** RCCI
***** Principal Approved
***** Pre-K
*** Precedence Order
*** Status
**** Active/Inactive
**** Show as a chip/tag, will be a switch while editing.
*** Last Updated (Datetime + User)
* Add an Edit option that will allow the user to change the order.
** Each row can be moved up or down in the Precedence ordering.
** Entries can’t jump between levels. You can’t elevate a Level 2 approval up to a Level 1.
** Save / Cancel options available.
** Any saved changes will need to update the moved entries.



Notes based on discussion 11/18 with Nandish

* Rearranged the default ordering for Level 2 above. Homeless and Runaway moved up 2 places.

Notes based on daily discussion 11/20

* The updated timestamp does not need to be shown on the Level itself, only the programs inside each level.

## Acceptance Criteria
DC/DA Precedence page added.
Users can adjust the ordering, make active/inactive.
Programs cannot escape their Level while re-ordering (example: Migrant can't move from Level 2 to Level 1).
