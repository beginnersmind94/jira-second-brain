# NXT-66898 — Page 3: Import accounts

## Description
*As a* nutrition director setting up Financial Oversight *I want to* import my district's Chart of Accounts and see validation results within the guided setup *So that* I can ensure my account codes match the expected format and have all my accounts available for mapping without leaving the setup wizard

## Acceptance Criteria
* Display header: "Import Accounts"
* Include explanatory text:
** *What is this?* "Your Chart of Accounts is a complete list of all the 'buckets' your district uses to categorize money. It's like a filing system for every dollar that comes in or goes out."
** *Why import?* "Instead of typing hundreds of accounts manually, you can upload your district's existing list to save hours of setup time."
* Display the System import workflow with these steps:
** *Pre-Upload step:* Import Type dropdown (defaulted to "GL Accounts")
** *File Upload step:* Drag and drop file upload interface with "Drag and drop your file here or click to browse" text
** *Field Mapping step:* Interface for mapping uploaded file columns to system fields
** *Review step:* Final review and validation of the import

*File Upload:*

* When user clicks upload area or "click to browse", file explorer opens
* Progress bar displays while file uploads
* System automatically validates file against configured account code structure

*Validation Results:*

* Display validation results summary showing:
** Total accounts processed
** Number passed validation
** Number failed validation with specific error details
* Show detailed error messages like "Expected XXX-XXX-XXX format, got XXX-XXXXX format for 77 accounts" (customized to their account structure)

*If all accounts pass validation:*

* Accounts automatically imported
* Success confirmation displayed
* "Next" button enabled to proceed to Step 3

*If any accounts fail validation:*

* Display two options:
** "Fix and re-upload now" (allows immediate file correction and re-upload)
** "Continue with current import" (proceeds with flagged accounts)

*If user chooses "Continue with current import":*

* Display prominent warning: "X accounts were imported with validation errors and are flagged for correction. This may affect report accuracy until fixed."
* User can download original file with additional validation status column
* "Next" button enabled to proceed to Step 3

*Additional features:*

* User can download detailed error report
* All accounts imported regardless of validation status (valid and invalid)
* System stores audit trail (upload timestamp, username, validation results)
* If user re-uploads, new file replaces previous active Chart of Accounts

*Navigation:*

* Include "Exit Wizard", "Back", and "Next" buttons
* Store timestamp when user completes this step
