## Acceptance Criteria
h3. 1. Page Availability

* A new License Management page must be created in parallel to the existing License Management page.
* Page title: License Management.

----

h3. 2. Header Buttons

h4. Bulk Renew Button

* Label: Bulk Renew
* On click:
** Opens the Bulk Renew drawer (covered in another story).
* Security:
** Enabled only for License Management role.
** Disabled for all other roles, including Customer Support role.

h4. + Generate Button

* Label: + Generate
* Functionality:
** Mimic the existing Add New button.
** Opens the Generate License Key drawer (covered in another story).
* Security:
** Enabled only for License Management role.
** Disabled for all other roles, including Customer Support role.

h4. Subscriptions Button

* Label: Subscriptions
* On click:
** Opens the Subscribe to License Key Events drawer (covered in another story).
* Security:
** Enabled for License Management (LMA) and Customer Support (CS) roles.
** Disabled for all other roles.

----

h3. 3. Filters Section

* Collapsible section at the top of the page.
* Search Box
** No name field.
** Real-time filtering of all rows after 3 characters typed.
* Status Filter
** Single-select dropdown.
** Values: New, Active, Cancelled, Expired.
** Default: Select all or no selection = All.
* Licenses Filter
** Single-select dropdown.
** Values: All licenses offered (from Bolt mockup).
** Default: Select all or no selection = All.
* Deployments Filter
** Single-select dropdown.
** Values: District, Site, Device.
** Default: Select all or no selection = All.

----

h3. 4. Grid Section

* Displays all generated license keys with paging and filtering enabled.
* Default Sort:
** Expired descending.
** District ascending.

h4. Columns

* District
** Empty for unassigned license keys.
** Optionally display as “Unassigned” if empty.
* License Key: Display actual key value.
* Status
** Values: New, Active, Cancelled, Expired.
** Status should use colors from Bolt code.
* Licenses: Numeric count of licenses attached to the key.
* Start: License effective date.
* Expires: License end date.
* Revenue: Total revenue for the key (may be blank).
* Actions: Buttons as detailed below.

h4. Action Buttons

* Edit
** Opens Edit License Key drawer.
** Enabled only for New or Active status.
** Accessible by Customer Support, License Management, and Tech Support roles.
* Add Licenses
** Opens Add License Key drawer.
** Enabled only for New or Active status.
** Accessible by Customer Support, License Management, and Tech Support roles.
* Remove Licenses
** Opens Remove License Key drawer.
** Enabled only for Active status.
** Accessible only by License Management role.
* Cancel License Key
** Opens Cancel License Key drawer.
** Enabled only for New or Active status.
** Accessible only by License Management role.
* Renew License Key
** Opens Renew License drawer.
** Enabled only for Expired or Active status.
** Accessible only by License Management role.
* History
** Opens History drawer.
** Always enabled.
** Accessible by Customer Support, License Management, and Tech Support roles.
* No Delete actions are available on this page.

----

h3. 5. Security

* Page visible to Customer Support, License Management, and Tech Support roles.
* Each action button has role-specific access as detailed above.

----

h3. 6. Functional Requirements

* Filters must work in real-time for all grid rows.
* Paging and sorting must be functional and default to the Expired descending / District ascending order.
* All actions must open their respective drawers as defined in separate stories.
* The grid should display all license keys ever generated, including multiple rows for a district if it has multiple licenses.
