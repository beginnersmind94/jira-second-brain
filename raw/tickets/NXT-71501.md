# NXT-71501 — SC - PBIRD - Dashboard

## Description
As a Director, I want to view and manage all Power BI distributed reports in a centralized dashboard so that I can monitor, access, and manage report configurations in one place.

----

h2. *Description*

Create a new *Power BI Reports dashboard* within the Reports module that allows users to view and manage Power BI distributed reports.

This dashboard will follow the same *look, feel, fields, functionality, and general business rules* as the existing *Managed View Reports dashboard*, while operating as a *separate page dedicated to Power BI reports*.

----

h2. *User Flow*

# User logs into SchoolCafé
# Navigates to:
#* *Reports → Power BI Reports*
# User views list of Power BI reports
# User applies filters as needed
# User reviews report details in the grid
# User may download grid data

----

h2. *Out of Scope*

* Favorites functionality (star column) _(separate story)_
* Edit functionality _(separate story)_
* Row-level actions _(separate stories)_
* All ellipsis menu actions _(separate stories)_

----

h2. *Requirements*

h3. *Navigation & Placement*

* Add new submenu item under Reports:
** *Custom Reports*
** *Managed View Reports*
** *Power BI Reports* _(new)_
* Page title:
** *Power BI Reports*

----

h3. *Dashboard Structure*

* The Power BI Reports dashboard must:
** Match the *layout, styling, and behavior* of the Managed View Reports dashboard
** Be implemented as a *separate page specific to Power BI reports*
* The implementation should *follow general business rules* established in Managed View Reports where applicable

----

h3. *Filters Section*

* Filters must match Managed View Reports exactly:
** Name
** Module
** Shared
** Scheduled
** Distributed
** Owner
* Include:
** *Apply* button
** *Reset* button
* Behavior must match existing implementation

----

h3. *Grid*

* Grid must match Managed View Reports structure and behavior
* Columns:
** Favorite (star)
** Name
** Module
** Shared
** Scheduled
** Distributed
** Owner
** Actions
* Data displayed must reflect *Power BI report configurations only*

----

h3. *Shared Column (Important Note)*

* If the *Sharing step is removed* from the Power BI Add New flow:
** The *Shared column should also be removed from this grid*
* Grid should stay consistent with the configuration experience

----

h3. *Download Functionality (In Scope)*

* Include *download/export button* for the grid
* Behavior should match existing download functionality from Managed View Reports
* Export should reflect current filtered dataset

----

h3. *Data Scope*

* Only display:
** Power BI report configurations
** Reports the user has permission to view

----

h3. *Consistency & Reuse*

* Reuse existing components and patterns from Managed View Reports where applicable:
** Filters
** Grid
** Download behavior
* Ensure consistency in:
** Styling
** Interaction patterns
** Labeling

----

h3. *Security & Resource Catalog*

* The Power BI Reports dashboard must be registered in the *Resource Catalog* as a new page resource
* Standard screen-level permissions must be applied using the Resource Catalog configuration

----

h3. *Screen Permissions*

* Users with *View permission*:
** Can access and view the Power BI Reports dashboard
** Can see grid data and filters
* Users with *Edit permission*:
** Can interact with editable actions on the page (e.g., Edit button)
* Users with *Delete permission*:
** Can interact with delete actions (when implemented in future stories)

----

h3. *UI Behavior Based on Permissions*

* If a user does NOT have *View permission*:
** The user cannot access the Power BI Reports dashboard
* If a user does NOT have *Edit permission*:
** Edit buttons are visible but *disabled (grayed out)*
* If a user does NOT have *Delete permission*:
** Delete actions are visible but *disabled (grayed out)* _(when implemented)_

----

h3. *Consistency*

* Security behavior must align with existing patterns used in:
** Managed View Reports
** Custom Reports

----

h3. *Future Actions (Reference Only)*

* The Actions (ellipsis) menu will include options such as:
** View Configuration
** View History _(single history option only)_
** Recently Run Reports
** Email Now
** Download
** Delete
* Each action will be implemented in *separate stories*
* Note: Existing duplicate history options should be consolidated into a *single history experience*

## Acceptance Criteria
* Given a user navigates to Reports, when the submenu is displayed, then Power BI Reports is available as an option
* Given the user selects Power BI Reports, when the page loads, then the dashboard displays with filters and grid matching Managed View Reports
* Given the dashboard is loaded, when data is displayed, then only Power BI reports are shown
* Given the user applies filters, when they click Apply, then the grid updates accordingly
* Given the user clicks Reset, when filters are cleared, then the grid returns to default state
* Given the user clicks the download button, when the export is generated, then the file reflects the current filtered dataset
