# NXT-59641 — AcctMgt - PINs - Split Max Length setting, update to make PIN settings a ranges (FD 269114)

## Description
[https://primeroedge.freshdesk.com/a/tickets/269114|https://primeroedge.freshdesk.com/a/tickets/269114]

As Dana the Director I use different PIN lengths for students vs staff and the system is not allowing for this to happen without flipping settings back and forth. An update here will save me time and frustration.

*requirements*

*Settings*

!image-20250303-200442.png|width=603,height=64,alt="image-20250303-200442.png"!

* Add a second setting for Staff
** Name: PIN - Maximum Length for Staff/Adults (up to 12 characters)
** Description: This setting controls the maximum length a staff/adult PIN can be set to. Ranges can be controlled on the PIN Number Settings page in Account Management.
* Rename the existing:
** Name: PIN - Maximum Length for Students (up to 12 characters)
** Description: This setting controls the maximum length a student PIN can be set to. Ranges can be controlled on the PIN Number Settings page in Account Management.

*PIN Number Settings*

* Treat the MAXLENGTH settings as the longest string length possible.
* Update to allow the MIN to be treated as a range.
* Example:
** MAXLENGTH for Students can be set to 5.
** Student Range set from 100 to 50000
** Valid PIN range would start at 100, end at 50000
** No padding required - lengths from 3 to 5, within 100 and 50000, are all acceptable
* Update the text on the page:
** This page controls the PIN Number ranges used if Random PINs need to be generated. Random PINs can be enabled for Students and/or Adults and the maximum PIN length can also be configured. The PIN Number ranges for Students and Adults cannot overlap.

*Account Management*

* Ignore the ranges when the PIN # is being manually edited.
** Warn the user, but do not block them, if the PIN # does not comply with range settings.
** Text: PIN # does not match Pin Range Settings.
*** Make Pin Range Settings a link to the PIN Number Settings page, open in new tab.
** Ultimately - user can set the PIN # to whatever they want, but it must be unique.
* Manual Account Creation/Entry
** If the user does not specify a PIN number during create/update let’s give them a small msg to let them know we’re going to make one for them based on setting.
* PIN field should be required.

*Imports*

* Update the Import workflow for Random PINs to check for the respective settings and ranges.
