## Acceptance Criteria
Changes:

New - Maintain a new table as “Export Settings” that will have 2 Fields: 1. Region ID 2. ExportGDSNItemsOnly (Values: 0 or 1)



Changes in Export GDSN Script:

 The system to check the setting for the corresponding region if the a value set to 1

     If the value  = 1 - export only items that are created via GDSN Item IMport (Join based on Item Code from GDSNItems_SC and Local Items table)

    If  the value = 0 or no entry found in the table for the corresponding region, then export all the items as-is (current scenario)

Changes Applicable in Manual Export and End point as well.
