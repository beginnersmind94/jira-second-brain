# NXT-5239 — Eligibility - Direct Certification - File Upload

## Description
*User Story*

As a district admin I want to upload my direct cert file via the Eligibility module and have it process matches.

*mockup*

!image-20210226-195516.png|width=1227,height=915!



*Requirements*

The main aim here is to connect the DataExchange process with the Eligibility module, and allow users to start the DC process from there.

* -User is able to select any configured file formats for the selected DC type.-
** -Example: They select Other Source Categorical → Homeless.-
** -User is shown all Homeless configurations and can select one to use.-
* User is shown the name of the saved configuration for the selected DC type.
** Example: User selects Other Source Categorical → Homeless.
** User is shown the name of the current file configuration.
** If no prior job/configuration has been created for this DC type, show the user that information:
*** tbd
* User is able to upload a file from their local system.
* After user submits the file:
** Standard procedure for Eligiblity - a file number/reference is generated with selected options recorded.
*** Approval date, type, etc. are stored.
*** A file/reference number is allocated.
*** Status for the file in Eligibility is marked as ‘Processing' until completed.
** System processes the file per the DC Matching process.
*** Automatic matching, potential matches, all generated.
*** Batch is updated to Processed once complete.

## Acceptance Criteria
User is able to upload a file from their local machine.
- file is taken in via DataExchange
- DC Matching happens as expected - students receive eligibility, potential matches are generated.
- batch is created when the user uploads a file
- batch status updates as the DC matching process fires and finishes
