# NXT-52454 — Notifications - Profile Preferences - Security Trimming

## Description
We need to enhance the original story to capture security trimming so that users do not see subscriptions that they do not have rights to

*Notes*

* This story assumes the notifications section has already been created and works
* It is about security trimming the grid

*Requirements*

* On Workspace > Profile do the following for add and edit
* For the Subscriptions table, do the following:
* 
* Do not show subscriptions to the user for modules not purchased by the district
** Ex: Do not show any Inventory subscriptions if they do not have the inventory module license
* Do not show subscriptions to resources they do not have rights to via roles and permissions
** Ex: If the user does not have rights to view Users Dashboard, then do not show it to them
* 
* To do this, when you register the subscription event in the DB, you will also need to register what resource / module it is for
* 
* Security: no updates

## Acceptance Criteria
* Users cannot see subscription events for modules their district has not purchased
* User s cannot see subscription events for resources they don’t have permissions to
