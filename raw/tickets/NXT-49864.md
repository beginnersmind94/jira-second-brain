# NXT-49864 — Roles & Permissions - Hierarchy & Actions & Resources Page Modifications

## Description
As a SchoolCafé Dev Team leader, I want to ensure multi-level nesting and functions are added to my resource catalogue so that I can control who has access to what.

*Requirements*:

*Hierarchy changes:*

On System > Resource Catalogue, do the following:

* Ensure that we can nest resources up to 5 levels
* Use Case: 
** System > Inventory module
** Click on Orders
** View orders and click on view Order Detail
** From there click on Order Receipts
** Do hierarchy is Orders > Order Detail > Order Receipts



Updates to the Edit Permissions drawer

* For the System > Roles & Permissions > Edit button click > Edit drawer, do the following:
* Ensure that the nested resources display properly in a hierarchical format



*Function Name change:*

On System > Resource Catalogue, do the following:

* Right now, the Function sub component seems to be tied to Special Permissions somehow
* It should be independent of special permissions
* It should work like a tab or sub-page
* So, de-couple it from the special permissions
* If needed, call it Action
* Ensure it can be used like a sub-page or tab, meaning it can be nested under another resource



Updates to the Edit Permissions drawer

* Ensure the Action labeled items show up in the list and in proper nesting like the other resources

## Acceptance Criteria
* Roles and Permissions show multi-level hierarchy for resources
* Resource Catalogue functions are treated independently of special permissions
