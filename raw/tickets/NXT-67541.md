# NXT-67541 — Story 6: Save Account Code Structure with Effective Dates (UI)

## Description
*As a* district business administrator
*I want to* save my account code structure changes with effective dates
*So that* the correct structure applies to transactions from specific time periods

h3. Background

When users save changes to their account code structure (from Story 1), they need to specify when this version becomes active. This allows for retroactive application and mid-year changes.

## Acceptance Criteria
*Save Changes:*

* Clicking "Save Changes" button opens confirmation modal
* Modal includes:
** Header: "Save Account Code Structure"
** Warning: "This will create a new version and apply this structure to all past and future transactions. This cannot be undone."
** Version number that will be created (e.g., "This will be saved as Version 4")

*Save Behavior:*

* When user confirms:
** Create new version (Story 5 handles backend)
** Recode all past transactions and apply to all future transactions
** Close modal and return to Account Code Structure tab
** Show success message: "Account Code Structure saved successfully as Version [X]"
* "Cancel" button closes modal without saving
