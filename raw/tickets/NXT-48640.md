# NXT-48640 — MVR - Users Add New

## Description
As a SchoolCafé user, I would like to be able to setup my managed view for recurring distribution to a set of users so that I can get periodic notifications like on custom reports.

*Notes:*

* We are doing this for Users page, but code as if it were being done by any page in front office or back office
* Do not hard code assumptions that it is only for Users page or only for System module
* This is part of the Custom Reports Framework and will work like a Custom Report



*Requirements:*

* When a user clicks the Distribute button on System > Users and is redirected to the drawer, do the following:
* -Note that there are 2 possible scenarios:-
** -The user clicked Distribute on a saved managed view-
** -The user clicked Distribute when the view is not saved-
** -We will handle both in the same flow-
*** -Basically, if not saved, we will create a saved manage view for them in the process-
* 
* Add a new drawer
** Title: Managed View Distribution
** Use the sliding open action
** Width is full page
** Note: Use the Custom Reports (CR) + New interface for this drawer
*** We will have the same Stepper tabs except Config tab
* Stepper Tabs:
* Selection tab
** Name 
*** Same requirements as CR
*** If saved MV, default text to name of the Managed View
*** If not saved MV, default text to name of the page + Managed View Report [ex: Users Managed View Report]
** Description
*** Same requirements as CR
*** For both saved and unsaved MV, default text = Managed View Report for [page name]
** Owner
*** Same requirements as CR
*** For both saved and unsaved MV, default = user account logged in
*** Allow them to select another user
** Module
*** Same requirements as CR
*** Make read-only
*** Default = the module the page is in, which is System in this case
** Data Source
*** Same requirements as CR
*** Make read-only
*** Default = Name of the data source feeding the grid
** Contains PII
*** Same requirements as CR
*** Default selection is set by dev based upon data source, which is No in this case
** Dynamic Query
*** Don’t show, doesn’t apply to MVR
*** Default = No if need to account for it in the DB
** Global Templated fields
*** Don’t show, doesn’t apply to MVR
** 
* Config tab
** The Config tab functionality is completely different or MVR than a CR
** So don’t show the graphical query builder from CR
** Instead, show the managed view criteria some way
** It doesn’t need to be fancy, maybe just show the JSON file or something
** 
* Sharing
** Use the same Sharing tab as CR
*** If saved MV, copy users and Roles selected from the Managed View and populate
**** If they select sharing to people or roles here, then be sure to save it back to the managed view when Save
** 
* Distribution
** Use the same Distribution tab as CR
** 
* Scheduling
** Use the same Scheduling tab as CR
** 
* Review
** Use the same Review tab as CR
* 
* When you save it
** -First check if the user is saving a managed view or not-
*** -If not, then first save as a managed view-
**** -Use the Name field from the first tab to name the MV-
**** -Create an MV configuration the same way you would if you had clicked the Save button on Managed Views-
**** -Ensure this new MV shows up in the list of MVs for the grid-
**** -If any person or role added on Sharing tab, add to MV-
**** -MV type is Mine for the owner and shared for everyone else-
**** -Then go on to save as MVR-
*** -If yes, then proceed to save as MVR-
** Save it as a Managed View Report
*** It will have a different data structure than the CR on the configuration tab, but everything else the same
*** Ensure it is linked to the Managed View
**** All MVRs need to be linked to MVs
**** They are 2 separate objects
*** Ensure it shows up in the Managed Views Dashboard after you save it
* 
* Ensure you save it to the History table
* 
* Security:
** For distribute button: Use Edit rights on R&P for the page it is on, which is Users in this case
*** So, user must have Edit rights to Users page

## Acceptance Criteria
* Drawer exists with correct name and tabs
* User is able to set the properties on each tab
* MVR is able to get the MV configuration from the source page and save it
* MVR is linked to MV
* MVR shows in the Managed Views list of managed views
* MVR shows in the MVR dashboard
