# NXT-49349 — OTP - District - Test Authentication

## Description
As a Customer Support user, I want to be able to test a district’s One-Time Passcode configuration to make sure it works well

*Notes*:

* This story is for testing the One-Time Passcode Keycloak configuration while adding or editing any One-Time Passcode method
* Each type of One-Time Passcode will be discussed in separate stories
* To get to this screen, you have to be in the district’s add new or edit One-Time Passcode screen

*Flow:*

* User browses to System > Security page
* User clicks the Add New One-Time Passcode button
* User is redirected to One-Time Passcode Add New drawer and completes it
* User saves the config to Keycloak
* User tests the config on this screen:
** Selects test method
** Select user
** Ensures the necessary values are there
** Hits submit button
** System sends the code via the method selected
** User gets and enters the code
** User hits confirmation button
** System “logs in” with Keycloak to get an authentication response
** System shows the user a success or failure message
** System does not update the user authentication token or modify their login status
* User closes the screen

*Requirements*:

*Add New and Edit Drawer*

* These requirements apply to the System > Security > Authentication tab > Add New or Edit drawers > Test One-Time Passcode Authentication:
* On all Add New and Edit One-Time Passcode drawers in district setup, do the following:
* Add a test button if not already there (should be on each one already)
** If not on page, go to that story for requirements



*Test Screen*

* Create a new screen to test One-Time Passcode authentication
** Can be drawer, popup as needed
** Title: Test One-Time Passcode Authentication

** Single select drop down list
** Values: Users for the district
** On click, load the user fields in the section below

** Fields to show to verify the one-time passcode:
** Textbox
*** Label: One-Time Code
*** Use same as on login page
*** Numeric only
*** Required length = number of digits of the code sent
*** User must enter the code they received
** Button
*** Label: Verify
*** On click:
**** We need to simulate the login with One-Time Passcode process
**** Get the code and then try to “authenticate” with Keycloak
**** Get the response and see if it was successful
**** If yes, then everything is setup right and working

** Deny permission to all other district roles by default

## Acceptance Criteria
* User is able to select which method to test
* User is able to enter the required fields to test
* User is able to scan the QRC code for the apps if testing those
* User is able to send the test to themselves or to some other user
* The test simulates the authentication of a One-Time Passcode but does not save or authenticate the user (doesn’t change their login)
