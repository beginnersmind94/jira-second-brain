# NXT-40345 — Custom Reports – Add New Report – Distribution Tab

## Description
As a SchoolCafé user, I would like to be able to distribute my custom reports so that others can receive them without having to run reports.

*Notes:*

* We are updating the existing Add New Custom Reports and adding in the distribution functionality to it
* This functionality will be used by action buttons on the dashboard and grid pages
* It would be best to develop the sharing functionality before this

*Flow:*

* Open SchoolCafé and browse to Settings > Dashboard > Custom Reports
* View list of reports
* Click + New
* Create the configuration and set distribution and then save
* Return to the updated list of custom reports

*Requirements*:

* These requirements are for the Add New Custom Report overlay
* 
* Add a tab to the Stepper for the Distribution tab
** Title: Distribution
** Location: 4^th^ step, behind Sharing
* 
* Tab requirements:
* Flow:
** Ask the user if they want to distribute this report?
** If yes
*** Ask how to distribute
*** Ask in what format if email, ftp or inbox
*** If they select email, do the following:
*** Ask if same list as sharing?
*** If yes, hide or make read only the role and user selector, but let them continue to add external if they want
*** If no, then proceed below
*** Give them a user selector
*** Give them a people selector
*** Give them the external email textbox
*** If they select Inbox, do the same for email except external addresses
*** If they select FTP, do the following, ask them for the FTP information we need in order to send the report via FTP to a location of their choosing
*** If they select API, do the following, provide them with what they need to connect into our External facing APIs
** If no, hide or make read only this section
* 
* Fields:
* Distribute Report
** Radio Buttons
** Values – Yes / No
** Required
** Default = No
** If yes, show or enable the people and role selectors
** If No, hide or disable the people and role selectors
* 
* Fields to show if they say yes to distribute:
* Distribution Method
** Multi select list
** Values: Email, FTP, API, SchoolCafé Inbox
** Required if distribute = Yes
** No default value
* 
* Fields to show if they select Email or Inbox or FTP:
* Format
** Single select list
** Values: CSV, Excel, HTML????, PDF??? , JSON
** Need dev team to tell us our options
** Required if distribute = Yes
** No default value
** If they select API, don’t show / enable this field
** Note: if they select HTML and Email, the html should be in the body of the email
* 
* Fields to show if they select Email -or Inbox-:
* Use Sharing List
** Radio Buttons
** Values – Yes / No
** Required if Email = Yes
** Default = No
** If yes, show or enable the users and role selectors
** If No, hide or disable the users and role selectors
** Note: this field has no impact on the External Emails field, they can complete that fields regardless of this answer
* SchoolCafé Users
** Same as on Share screen
** Multi-select list
** Values – active user accounts for the district
** Required if Sharing List = No
** No default values
* SchoolCafé Roles
** Same as on Share screen
** Multi-select list
** Values – Roles for the district
** Required if Sharing List = No
** No default values
* 
* Fields to show if they select Email:
* Other Non-SchoolCafé User Email Addresses
** Textbox
** Always optional
** No default value
** Tooltip: Optionally add comma delimited list of email addresses for users who are not registered in SchoolCafé
** The user can type in a list of email addresses separated by commas
** Note: make this a larger textbox and ability to scroll since they may have a lot of email addresses they want to use
** Note: only shown for Emails, not other types
** 
** If the Selection screen field PII Data = Yes:
*** Show a message near this field
*** Message: Caution! This report contains PII data. Do not add recipients that are not allowed to view PII data.
* 
* Fields to show if they select FTP:
* FTP Host
** Textbox
** Required if FTP = Yes
* FTP Port
** Textbox
** Required if FTP = Yes
** Default = 21
** Validate: Must be a number
* FTP Folder ???
** Textbox
** Required if FTP = Yes
** Note sure if we need this
* Username
** Textbox
** Required if FTP = Yes
* Password
** Textbox
** Required if FTP = Yes
** Use usual hide / show text eye button
* Note: Developers: add the fields you need, It is my understanding that these are the ones needed, but up to dev team
* 
* Fields to show if they select API:
* URL
** Textbox
** Read only, not editable
** Value: API end point to server for these reports given by developers
* Key
** Textbox
** Required if API = Yes
** Use usual hide / show text eye button
* Secret
** Textbox
** Required if API = Yes
** Use usual hide / show text eye button
* Notes:
** I grabbed these fields off the SIS Auto Sync Add New screen
** These may not be the fields we need, developers will need to tell us
** Not sure if need other fields like a report id parameter, etc.
** End user will have to configure on their end how to pull this
* 
* If the Selection screen field PII Data = Yes:
** Show a message below the fields
** Message: This report contains PII data. Users that are not in the PII Role will not be able to view or receive the report even if it is shared with them.
* If the PII Data = No, then do not show a message
* Note: allow them to add any user or role. We will control access on the View screen and when distributing
* 
* Previous
* Next
** Put a validator on the Type:
** If they select HTML, it can only be used as the body of an email, so validate they have only selected email as a distribution type. If they did anything other than select email (ex: they select FTP or select Email and FTP), then show them an error message “HTML format is valid for Email distribution only” and do not let them click next into the next screen
* 
* Other Notes:
** If they say that they want to use the same list as sharing, you will use those users and roles for distribution too
** Somehow, you will have to track and manage this
** In the worst case scenario, copy the sharing users and roles to the distribution users and roles and maintain 2 separate lists after they save to the database

** Note: Don’t send more than once to a given user (remember they might be in a role)

* 
* Update to the *Edit button on Dashboard*
** Ensure they are there or add all these requirements to the Edit screen

* Update to the *View Configuration Details*
