# NXT-23317 — AcctMgt - Groups - Homerooms - Allow user to view and assign/remove tags

## Description
As a district admin, I would like to be able to easily view and categorize my Homerooms, so that I can serve them accordingly.

*Summary*

* In Homerooms, add a Tags filter and column which the user can use to search for specific Homerooms. Also, allow them to edit a homeroom’s tags.

*Details*

* In Groups → Homerooms, add a Tags drop-down menu to the list of filters.
** Tags menu should be *multi-select*
** In this list, pull up all active Tags that have been created for this particular district (+*_if_*+ we have implemented active/inactive status for Tags. Otherwise just show all Tags for the district)
* When a user selects one or more Tags and then clicks ‘Apply’, only display those Homerooms in the grid which have been assigned those Tags.
* In the Grid, add a Tags column which shows those assigned tags for each Homeroom.
** If the number of Tags is too wide for the standard column width, display the overflow as ‘+1','+2','+3', etc. 
*** When hovering over that content, display all Tags for the homeroom in a floating tooltip.
* When a user clicks the edit button in the grid, open the Edit Homeroom control as a drawer instead of as a pop-up.
** Within the drawer, add the Tags field. (other fields remain unchanged)
** The user should be able to use this field to assign/remove Tags and then save the details for this individual homeroom.
* Other UI/UX changes to note:
** Filter design should be changed to an expansion panel with the title: ‘Filters’ (expanded by default).
** Add a Reset button, which when selected, returns all of the filters to their default selections (if we have to also clear the grid results, that’s fine).
** Change the ‘Find Homeroom(s)’ button label to ‘Apply’. Behavior remains unchanged.
** In the grid, change the edit icon from ‘Edit’ to ‘Edit2’
** Replace the Edit Homeroom buttons ‘Update’ and ‘Cancel' with a X and 'Save’ buttons.

## Acceptance Criteria
* Verify that a multi-select Tags field is added to the list of Homerooms filters, containing the active Tags for this specific district
* Verify that a Tags column is added to the list of Homerooms filters 
* Verify that this column shows a condensed view of the tags relating to each homeroom, with overflow showing as +X,X+1,X+2, etc. and the full list showing in a tooltip while hovering over it
* Verify that the Edit Homeroom functionality opens as a drawer instead of a pop-up, and that it now contains a Tags field that the user can use to assign/remove tags from the homeroom
* Verify that the other general UI/UX improvements have been made to the page:
** Expansion panel
** Reset/Apply
** Grid edit icon ‘Edit’ → ‘Edit2’
** Edit Homeroom buttons ‘Update’ and ‘Cancel' → X and 'Save’
