# NXT-5915 — Messages - Compose New Message

## Description
*User Story*

As a PrimeroEdge user, I have the ability to compose a message and send it to another user within my organization.

*Requirements*

* There is a compose function available that brings up a dialogue box for a user to enter a message and send it to anyone in the organization
** Any messages sent will appear in the respective user’s inbox
** Compose is only accessible where the Inbox is present (Inbox screen or Workspace)
** Messages cannot be sent outside of the organization or product
* There is an autofill mechanism for the *TO* section
** Typing letters narrows it down
** All users in the organization are available
** Multiple users can be entered into this section
** User names are shown based only on users available in the system for the organization
*** User name must be selected by user
*** Users are displayed as First Name Last Name in the To section
** -Allow users to search for “-*-Groups-*-” (distribution list) with hierarchical structure to be included-  {color:#ff5630}Deferred{color}
* User can enter a subject
* -There’s an option to mark a message as “High Importance”- {color:#ff5630} Deferred{color}
** -Messages are either Regular or High Importance-
* User can enter a message
** Formatting options as displayed in the mockup are available and can be used to format the message text
*** Heading Settings
*** Bold
*** Italics
*** Underline
*** Embedded Link
*** Numbered List
*** Bulleted List
*** Clear Formatting
* There is a confirmation message if user tries to close without sending
** Messages are not saved if user continues
* Only one window can be open at once
* Once Send is hit, the message is delivered to the user or users in the To section
** Message cannot be sent without a valid recipient in the To section
** Message cannot be sent if the Subject or Message sections are empty
* Unicode/UTF-16 is supported for language support
* Message is displayed in Sent folder of Inbox once sent
* Out of Scope:
** Reply
** Read Receipt
** Any external messaging
** Attachments

## Acceptance Criteria
* A message can be composed using the template's compose function
* All fields can be filled out while composing a message
* Message content can be properly formatted as described in the story
* Messages are saved and can be sent to users
* Disable SEND button if user does not add VALID email address under TO section.
