# NXT-53072 — OpenID - Notify - New User

## Description
As a SchoolCafe admin, I want to be able to tell a user to update their authentication so that they can login using it

*Notes*:

* This story is for how to handle the notification process for when adding a new user manually or via imports

*Flow (Happy Path):*

* Admin browses to System > Security page
* Admin sets up OpenID method
* System notifies users of change and provides instructions on how to configure
* User follows instructions, updating their OpenID method in their Profile page



*Requirements*:

*Add User Screen*

* These requirements apply to the System > Users > Add New
* On save:
** Note: currently, there are 2 types of authentication emails that can go out to a user
*** One if they use SchoolCafe authentication which handles the password stuff
*** The other is for any other kind of method such as SSO
** If the district has enabled OpenID method:
*** Update the authentication email sent to the user (not the password one) as described in the New User Authentication Email below



*User Imports*

* These requirements apply to when a new user account is created via imports
* Do exactly the same thing and communications as when creating a user manually
** This includes sending the New User Authentication Email below





*New User Authentication Email:*

[first name],

A SchoolCafe user account has been setup for you for [district name]. 

The login page is [hyperlink to the login page].

Your SchoolCafe username is this email address.

The district has setup the following authentication methods.

[if SchoolCafe authentication, show this:]

SchoolCafe Authentication:

Website simple authentication using email and password. You will receive another email with instructions on how to set up your password. [if it is the preferred method, insert this text] This is the district’s preferred method.

[if OpenID SSO, show this:]

Single Sign On:

Login using your district username and password with your district’s authentication provider. No additional setup by you is required for this method. [if it is the preferred method, insert this text] This is the district’s preferred method.

## Acceptance Criteria
* Users are notified of the authentication method the district is using
* Notifications are sent for both imports and manually adding new
