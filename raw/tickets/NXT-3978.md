# NXT-3978 — System - Notification Templates - Allow user to save current template or save as a new template

## Description
As a district administrator, I would like to be able to easily manage all of the letter templates (for letters that will be going out to my parents and students) in one place, so that I can easily make changes as needed.

*Summary:* Allow the user to save a letter template in its current save or save it under a new filename. ‘Save As’ is the only way for a user to add a custom template. Letter editor does not have to be done by this point.

* Make the ‘Save’ button functional for each letter
* Save letter appropriately
* Make the ‘Save As’ button functional for each letter, requiring unique filename. (A prompt that provides the user with the option make the template active will be in a separate story)
* Create new letter that can be seen in the list
* Update ‘Last updated by’ info as it relates to the letter

Mock-up: [https://nqyc9v.axshare.com/start.html#p=templates&id=x5t8yr&c=1|https://nqyc9v.axshare.com/start.html#p=templates&id=x5t8yr&c=1]



*Full Details:*

* In the *Notification Templates* page when the user selects a letter, allow them to save the details of the letters (active and inactive) accordingly. 
** Selecting the *Save* button should save the details and update the ‘Last updated by’ info for that letter.
** Standard templates cannot be overwritten. The *Save* button should be disabled for those, forcing the user to select *Save As* in order to create a new template.
* Other notes:
** Letter types are standard, but users can add languages in and set up their letters in that particular language as well. (again, separate story but please design with this in mind)
** Letter templates cannot be added from scratch.
