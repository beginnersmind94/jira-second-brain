# NXT-20749 — Food Detection Trigger

## Description
As a Cashier, while doing a transaction, I want to trigger the Edge device to scan the tray so that it can perform the food detection.

*Application*: POS

*Flow:*

* While using the POS transaction screen with Food Detection enabled:
* Food Detection event is triggered by selecting a student
* Edge scans food and returns results
* System takes relevant actions per other user stories

*Assumptions:*

* This page exists in the classic ExpressPoint
* You can copy all the code and try to copy as-is
* Talk to Saketh if you have any questions

 *Requirements*:

* Copy everything as-is from classic
* While using the POS transaction screen with Food Detection enabled, Food Detection event is triggered by selecting a student
* When food Detection event is triggered:
* System notifies Edge to scan tray
* System establishes IDs, Tokens or whatever is necessary so that the food scan can be linked to the transaction and user
* If System can’t establish connection to the Edge
** Shows message: Edge Not Found
** Logs in history log
* Security
** Leverage existing security for this screen

## Acceptance Criteria
* Food Detection event is triggered by selecting a student when Food Detection is enabled in admin screen
* Food Detection event is NOT triggered by selecting a student when Food Detection is disabled in admin screen
* When food Detection event is triggered:
** System notifies Edge to scan food
** System establishes IDs, Tokens or whatever is necessary so that the food scan can be linked to the transaction and user
* When system can’t establish connection to the Edge and Food Detection is enabled in admin screen
** shows message: Edge Not Found
** logs in history log
