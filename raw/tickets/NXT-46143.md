# NXT-46143 — Elig - Applications - SIGs - Verification updates, actions

## Description
As Anna the Application Processor I want to perform Verification actions on State Eligibility applications and I need the system to behave appropriately.

*requirements*

This story is to handle workflow updates for State Income Guidelines (SIGs) and Verification actions. These scenarios only apply if the setting is turned on.

* Show the State Eligibility on the following pages:
** Tracking
*** Eligibility column should show as USDA Eligibility
*** Add the State Eligibility column
**** Use the local descriptions
**** Filterable column
*** State Result column for the Complete tracking grid (1/26/24 addition after discussion with Harsha)
**** Use the local descriptions
**** Show changes in this for State Eligibility
***** Examples: N/A to State-Free, State-Free to N/A, State-Reduced to State-Free, etc.
** Verification Details
*** Reminder: Use the local descriptions whenever we are referencing them
*** Outcome 
**** Show USDA/State on separate lines
**** Example: 
***** USDA: Reduced (Income) to Paid (Income)
***** State: N/A to Free (Income)
***** If State is N/A to N/A, just show
****** State: N/A
*** Pre-Verification Application Information + Post-Verification Application Information
**** Show USDA Eligibility + State Eligibility in the section titles
***** Example: Pre-Verification Application Information (USDA: Denied - Income, State: Free - Income)
***** Post-Verification Application Information (Denied - Income, State: Free - Income)
*** Verification Results
**** Add a State Result section to the grid, show the change
***** Examples: N/A to Free, Reduced to Free, Free to N/A



* Process State Eligibility as part of verification workflow actions
