# NXT-59535 — SchoolCafé - Menu Planning - Publish Menus - Add "Auto-Publish" Feature

## Description
{panel:bgColor=#deebff}
{color:#6554c0}*NOTE*{color}: Mockup is for Classic, but can be adopted for 2.0 as well since the pages are nearly identical.
{panel}



It is of popular opinion that the publish menus task should be able to be automated as the people working with menus are not always the same as who manages SchoolCafé content. Even if they are, users find it tedious that is required to have to remember to go to a page every time their menus are ready for publishing to SchoolCafé.



# Add new tab called *Auto Publish*
# Add a background job for every district which can be turned on if the district wants auto-publishing
# Follow similar design as Eligibility Notifications (below). Required fields:
## Publish ID
## Last Publish
## Next Publish
## Status
## Action(s)
### Enable or Disable. May add more options over time. 
### History (of users enabling/disabling)
# See AC for other details. No frequency field is required since publishing will happen daily based on the job timing (whatever we determine for performance-sake).

## Acceptance Criteria
Verify new tab is added to Publish Menus page

Verify background job is added

Verify district can enable auto publishing using a variety of settings

Verify menus get published to Family Hub

Verify that this function allows menu publishing to still work the same way as it does manually, in that the auto-publish would take whatever is in the queue for the day: menus assigned to the calendar, or production planned items having counts > 1.
