# NXT-3798 — AcctMgt - Patron Split View - Household

## Description
*User Story*

As a district administrator I want to see other accounts that are in the same household.

*Mockup*

[https://c7dihe.axshare.com/#id=2jywmg&p=patron_detailed|https://c7dihe.axshare.com/#id=2jywmg&p=patron_detailed] - example of multiple students showing

[https://nqyc9v.axshare.com/start.html#p=category_based_on_patron_view&id=jytwv5&g=1|https://nqyc9v.axshare.com/start.html#p=category_based_on_patron_view&id=jytwv5&g=1] - actual wireframes

*Requirements*

* Add a Household section to the Patron Details panel.
* Section will show information on who is in the household together, including a Household Pin #.
** Household Pin #
*** Unique number across the district.
** A grid showing:
*** Name of the other patrons in the household
*** ID #
*** School (Site Code + Site Name)
*** Grade
*** -Homeroom-
** User can navigate to the other patron account by clicking on the ID #.
* Add an Edit button to the Household panel.
** This enables an Edit mode, side-anchored drawer.
** Household PIN # *(Not editable)*
*** This should be randomly generated for students when they are first added.
*** The number should be unique across the installation per PrimeroEdge standards.
** Household Members can be removed. 
*** Removed entries should be tracked somewhere- consider it a soft-delete.
** Add a way to Add another patron. 
*** This will require some form of lookup feature - search by ID #, first name, last name. Maybe use the ‘search by name / ID' bar that’s just been developed.
**** Predictive text - show results in the dropdown, select an option there.
**** Example: Searching by name comes back with 3 other students. A + button on each that performs the add without closing the results.
**** *Only students can be added to households.*
**** *Students can only be in 1 household at a time.* 
***** We need a dialog box if a user selects a student who is currently in another household.
****** This student is already assigned to another household. Saving will remove them from that household.
**** Add the student to the grid in the drawer with a save / discard option.
**** The added student’s pin # should update to be the same as the loaded student. IE: We’re looking at Student A, and add student B to the household. Student B’s pin # updates to be the same as Student A’s.

* -We need to track all changes that are made.-
** -Timestamp of the change, who was added or removed, who did it.-
** -For the pin # - old value, new value.-

## Acceptance Criteria
Household section added.
User can add/delete household members.
Pin # is unique across the installation.

REMOVED: History of changes tracked.
REMOVED: User can navigate to the other patrons. (per Karthik?)
