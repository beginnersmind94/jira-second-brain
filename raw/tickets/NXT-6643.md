# NXT-6643 — Application Entry - Guided Process (1. Students) - Student Popup Edit View

## Description
*Story* 

As an admin, i have the option to edit students and add linked household students to the application from the same screen. 

*Mockup* 

[https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=3869%3A67555&scaling=min-zoom|https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=3869%3A67555&scaling=min-zoom]

*Requirements* 

* _Select 'Find a Student' to search for an existing student in the system. Can't find the student? Select 'Add a Student' to manually enter the pending student details. Then, answer the categorical and income questions to save the information._
* *Student Grid* *> Active/Inactive (matched student using lookup) > Edit Button*
** Add Student popup shows without the options to Add/Find a Student since they are editing the selected student only
** ‘Find a Student’ option is preselected or hidden at the top 
*** Grid is collapsed (since the student has already been selected) 
**** -option to open lookup and change student-
** Entry fields are populated with the matched student info available: ID, First Name, Middle Name, Last Name, Suffix, DOB, Site, Grade
*** Entry fields are read-only
** Ability to edit the last 2 questions 
*** Leave answers from matched student and admin can update if needed
** Save > student is added/updated in the grid 
** Cancel > changes are discarded 

* *Student Grid* *> Pending Student > Edit Button*
** Add Student popup shows without the options to Add/Find a Student since they are editing the selected student only
** Add a Student’ option is preselected or hidden at the top
*** Allows editing on the entry fields: ID, First Name, Middle Name, Last Name, Suffix, DOB, Site, Grade
** -Ability to switch to ‘Find a Student’ using the lookup > switches to add student using lookup view (6022)-
*** -replaces manually entered data with selected student information-
*** -Leave answers from pending student and admin can update if needed- _(11/11 discussion)_
** Ability to edit the last 2 questions 
** Save > student is added/updated in the grid 
** Cancel > changes are discarded
* *Household Student Suggestion > Add Button* 
** Add a student popup shows
** Entry fields are populated with the matched student info available: ID, First Name, Middle Name, Last Name, Suffix, DOB, Site, Grade
*** Entry fields are read-only 
** Must answer the last 2 questions (Categorical and Income - default no)
*** Defaults options if yes 
**** Categorical - blank until selected (selection is required if answering yes)
**** Income - Frequency is blank until/unless a value greater than 0 is entered in the corresponding amount box, Frequency for an amount greater than 0 is required.
** Save > Suggested student is added in the top grid and removed from the suggestions 
** Cancel > changes are discarded and suggestion remains in grid

## Acceptance Criteria
- Multi-language, ADA compliant.
- on edit and add linked hh member, student popup shows collapsed grid with fields populated with matched student info 
- adding suggested household students must answer the last 2 questions to save
- edit button for pending/manual entry students shows editable entry fields 
- admin had the ability to switch back and forth via Find a Student (lookup) or Add a student (manual entry) if they so desire
- saving adds the linked hh student to the application and removes suggestion from box
- saving updates the existing student with the edits made in the popup
- cancel discards changes and leaves suggestion in box
