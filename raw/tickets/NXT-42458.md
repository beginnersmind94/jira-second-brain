# NXT-42458 — Custom Reports – Add New Report – Configuration Tab - Source Table

## Description
As a SchoolCafé user, I would like to be able to use a graphical interface to build my custom reports so that others can save time from creating their own.

*Notes:*

* We are updating the existing Add New Custom Reports and adding in the configuration functionality to it

*Flow:*

* On the configuration tab:
* Select which source fields to use and optionally specify a different name and any aggregates for them
* Optionally set the order via drag and drop
* Optionally set the filter values to use
* Optionally set the Arguments, Sort by, Group by, Having sections
* Optionally view the auto generated SQL statement
* Optionally preview the output of the SQL statement

*Requirements*:

* These requirements are for the Configuration Tab
* 
* Add a tab to the Stepper for the Configuration tab if it is not already there
** Title: Configuration
** Location: 2^nd^ step, behind Selection
* 
* Sub tabs in the screen:
** Source Table
** Default Sort
** Arguments
** Group By
** Having
** Parameters
** SQL

 

h4. Source Table Section

* The Source Table will be used to build the SELECT clause and the FROM clause
* Data = Select * from the selected data source field from the Selection screen
* The table shows all the fields from the source database table
* 
* It is a drag and drop table just like the Student Export table (copy that one to save time)
* User can drag and drop the rows to set the order of the columns
** When building the query, use the order of the rows when building your select statement to get them in the right order
* 
* Table column Names:
** Source Field
** Alt Name
** Aggregate
** Operator
** Value
** Enabled
* 
* In each row, the user can create an Alias by typing in a new name for the column
** To do this, add a column in the Source table
** Column Title: Alt Name
** Each row has a textbox
** Textbox has no name
** Required = False
** Max Length: 50
** If they leave blank, ignore
** If they enter in a value, create an Alias clause in the SQL statement (ex: Select DistrictId AS [District] FROM Students)
* 
* In each row, the user can create an Aggregate function on the field by selecting from a list
** To do this, add a column in the Source table
** Column Title: Aggregates
** Each row has a single select drop down list
** List box has no name
** Required = False
** Values:
*** Empty text (Default Value)
*** APPROX_COUNT_DISTINCT
*** AVG
*** CHECKSUM_AGG
*** COUNT
*** COUNT_BIG
*** GROUPING
*** GROUPING_ID
*** MAX
*** MIN
*** STDEV
*** STDEVP
*** STRING_AGG
*** SUM
*** VAR
*** VARP
** If they leave blank, ignore
** If they select a value, apply the function to the field in the SQL statement (ex: SELECT AVG(UnitPrice) FROM Sales)
*    
* The next 2 fields are for the WHERE clause
* In each row, the user can create an Operator on the field by selecting from a list
** To do this, add a column
** Column Title: Operators
** Each row has a single select drop down list
** Required = True
** Values:
*** Empty text (Default Value)
*** =
*** <> 
*** > 
*** >=
*** < 
*** <=
** This field is used in the Where clause along with the field name and the Value
* 
* In each row, the user can set a value for the Operator on the field
** To do this, add a column
** Column Title: Value
** Each row’s field type is determined by the source field’s type
** If it is list
*** show as a multi-select list
*** values are the list
** If it is a text
*** Show as a textbox
*** Max length is per database field
** If it is a Boolean
*** Show as a toggle
** Required = True
** This field is used in the Where clause along with the field name and the Operator
* 
* In each row, the user can decide to use the column or not by toggling an Enabled / Disabled field
** To do this, add a column in the Source table
** Column Title: Enabled
** Each row has a toggle with no label
** Required = True
** Default Value = True
** Values:
*** Enabled
*** Disabled
** If they select Enabled, the field is used by the export
** If they don’t, then the field is ignored
* 
* We need the ability to join tables
** Add a button above or on the Source Table
** Title: Join
** On click, show a drawer
** 
** Drawer title: Join Tables
** Fields:
** Current Data Source
*** Show a single select list of data sources currently in the configuration screen
*** Auto select the first one
*** They must select one
*** Then they must select the column to use in the join
** Current Data Source Column
*** Show only after they select a current data source
*** Shows the columns for the data source selected from the Selection screen
*** They must select one
** 
** Join Type
*** Values:
*** Inner join
*** Left outer join
*** Right outer join
*** Full outer join
*** Cross join
** New Data Source
*** Show a list of data sources
*** They must select one
*** Then they must select the column to use in the join
**  New Data Source Column
*** Show only after they select a data source
*** Shows the columns for the data source
** 
** These values are then used to form a JOIN clause
** 
** They can join more than one table, ie they can do multiple joins
** Show a table underneath that lists the joins that have already been created
*** Text value: description of join, ex: current data source name current data source column join type new data source name new data source column
*** Delete button
**** On click, ask to confirm

**** -They cannot choose the last 2 options w/o selecting TOP-
** -Example:-
*** -SELECT TOP(5) PERCENT JobTitle FROM Employees-
