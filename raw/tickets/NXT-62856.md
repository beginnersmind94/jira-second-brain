# NXT-62856 — SchoolCafé - Menu Publishing/Settings - Integrate setting to publish via Production Counts

## Description
As a production planner, I would like to be able to publish the details of my current plan to the Family Portal, so that I don’t have to navigate to the menu calendar and update it each time there is a last-minute change whereby the menu calendar does not reflect what is actually being produced.

*Summary*

The menu publishing method setting carried over from Classic (either publish via Menu Calendar or Planned Counts) has not been properly integrated, since only the Menu Calendar setting value is being respected. Fix this so that, when the user chooses to publish via Planned Counts and then publishes menus/auto-publishes menus, the items sent to the Family Portal are based on Production, instead of Menu Planning.

* Consult the same procedure/setting in Classic for more information. 
* Setting Name: *MNUPUBMTHD*
