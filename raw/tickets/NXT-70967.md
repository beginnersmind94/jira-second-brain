# NXT-70967 — Insights - KPI - ENP - Schoolie

## Description
*I am a* district user *that I want to* have Eligible Not Participating (ENP) included in Schoolie AI analysis
*so that I* can receive insights that reflect student participation gaps across sites and timeframes.

----

h2. *Notes*

* This story applies to *Schoolie AI integration within the Insights dashboard*.
* Schoolie is triggered from the dashboard and performs analysis on the *visible KPI cards* for the selected:
** Timeframe
** Site filters
* ENP (Eligible Not Participating) is a *front office KPI* that must be included in AI analysis when applicable.
* ENP is a *Lower is Better KPI*, and AI analysis should interpret it accordingly:
** Higher ENP → negative performance (more missed participation)
** Lower ENP → positive performance

----

h2. *Out of Scope*

* Any KPI other than ENP

----

h2. *User Flow*

# A user logs into the system and navigates to the *Insights dashboard*.
# The user configures:
#* Timeframe
#* Site filters
#* Visible KPI cards
# The user clicks the *Schoolie AI button*.
# The system sends a prompt to the AI engine including the visible KPI data.
# The AI returns an analysis.
# The user views insights, including ENP if it is part of the visible KPIs.

----

h2. *Requirements*

h3. *1. Include ENP in AI Prompt*

* The backend must include *ENP data* in the prompt sent to the AI engine.
* ENP data must reflect:
** Current timeframe
** Selected site(s)
** Current KPI values
** KPI direction (lower is better)

----

h3. *2. Conditional Inclusion Based on Visibility*

* ENP must only be included in the AI prompt if:
** The ENP KPI card is *visible on the dashboard*
* If ENP is not visible:
** It must *not be included* in the AI prompt

----

h3. *3. AI Response Inclusion*

* When ENP is included in the prompt:
** The AI response must include analysis related to ENP
** The analysis should:
*** Highlight high ENP values as risk areas
*** Highlight low ENP values as positive performance
*** Identify trends across sites and timeframes
* When ENP is not included:
** The AI response must not reference ENP

----

h3. *4. Consistency with Existing KPI Handling*

* ENP must follow the same prompt structure and handling as existing KPIs (MPLH, PNA).
* No custom or separate logic should be introduced beyond inclusion and KPI direction handling.

## Acceptance Criteria
h3. *1. ENP Included When Visible*

* When ENP card is visible:
** ENP data is included in the AI prompt
** AI response includes ENP analysis

----

h3. *2. ENP Excluded When Not Visible*

* When ENP card is hidden:
** ENP is not included in the AI prompt
** AI response does not include ENP

----

h3. *3. Data Accuracy*

* ENP values included in the prompt match dashboard values for:
** Timeframe
** Site filters

----

h3. *4. KPI Direction Awareness*

* AI analysis correctly interprets ENP:
** Higher values = worse performance
** Lower values = better performance

----

h3. *5. No Regression*

* Existing KPI analysis behavior remains unchanged
* Schoolie continues to function for all other KPI
