# NXT-71764 — SchoolCafé - Forms - (Part 4) Family Hub-related changes for Automatically Processed Forms, Other Settings

## Description
As a Family Hub parent/applicant, I would like to be able to apply for Summer EBT (as well as any other non-traditional form) applications with minimal steps, so that I do not have to use an external system just for this purpose.

*Summary*

* Integrate the SchoolCafé setting for automatically processing a Form
** This might be based on the Form type, depending on how we implemented auto-processing (by Form or using the existing setting)
* Integrate the SchoolCafé ADDINCSUR setting value for Manual Entry Only
* Add a new section within the ‘Eligibility Info’ page to allow applicants to review their forms the same way they can review their applications, including the application’s result

*Details*

# Integrate the setting/configuration for automatically processing forms, so that whenever a complete Form is submitted in the ‘Fill Out a Form’ page (i.e. all required fields are inputted, no pending students), the form gets automatically processed.
## We can let the parent know as soon as the application is submitted, that the result will be ‘Qualified or Unqualified’. 
## These form applications should immediately show as Processed in SchoolCafé. 

# Integrate the ADDINCSUR setting value for Manual Entry Only [(this has already been completed in Classic)|https://dev.azure.com/Cybersoft-Technologies-Inc/PrimeroEdge%20Classic/_workitems/edit/107862]. When this setting value is selected, do the following:
#* In SchoolCafé family portal Apply page, do not ever show Income Survey-related steps
#** Essentially, MANUAL ENTRY ONLY will behave like NO.
#** Parents/applicants can still see manually-entered Surveys in their Eligibility Info page in the portal.

# In Eligibility Info > My Applications, do the following to add support for Forms:
## Change ‘My Applications’ tab label to ‘My Meal Applications’
## Add a new tab: ‘My Forms’
### Mimic the current My Applications tab, except add a ‘Form Name’ column which corresponds to the submitted application. Also, result will be ‘Qualified’ (Free or Reduced) or ‘Unqualified’ (Paid).
## Allow user to view the submitted form and verify that it shows the specific Form Image Template linked to the application’s form
