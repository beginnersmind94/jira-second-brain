# NXT-57920 — MP - Schema Changes for Menu Planning and Production Planning with Data Migration

## Description
As a menu planner, allow me to alter menus any time I choose and store the information properly to my records that have already been completed so that my auditor can see the menu that was assigned associated to my record

## Acceptance Criteria
* Allow user to always make edits to menus regardless if they are production record complete
** Menus will be displayed as is with a Chip to indicate version if applicable
** Users will not have the ability to alternate the version assigned by the system 
* Assign version to menu associated with historical production records that have already been completed 
* Scenarios for versioning:
** Record Complete = will always require a new version
** Plan Complete/Incomplete = Will be user decided, If user decides to affect current plans, no need for new version, if they decide not to, create new version
** If no assignment associated with menu, there is never a need to create a version
