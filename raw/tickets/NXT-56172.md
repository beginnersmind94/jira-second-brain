# NXT-56172 — SC - Bulk Image Updates - API Changes

## Description
_(none)_

## Acceptance Criteria
Changes needed in API (Backend)

API should have a record status as New, Queued, Completed



*Single Image Update:*

The image shall be updated to Queued and Processed Immediately, the records is marked as completed 



*Bulk Image Update:*

All the selected records (lets say 50), they shall be updated with status as Queued

All the records marked for bulk updates shall be processed in the backend (a auto refresh shall be used in UI to clear them) . Once all the images are processed, the record will be marked as completed

The API shall return all the records that are New + Queued for filtering purpose in UI

When a new file is updated if the new images are included - the record for that status will be reset to new (it will reflect in API as New)

Download Exceptions - will reflect the records as-is (regression)

When a new file is updated and there are no images or same image that is already updated to local item - the record status will remain as completed, no new records will reflect in the UI
