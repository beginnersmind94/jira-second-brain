# NXT-50041 — MP- Site Groups- Ability to make Site groups inactive vs delete

## Description
As a district manager, when I am making my site groups or no longer using a site group, I need the ability to make a site group inactive vs deleting it. Deleting it should not happen if there is historical records available.

## Acceptance Criteria
* Site Group should have the ability to be deactivated
** Deactivate icon should only show up if no sites are associated with site group
** Display tool tip for Deactivate 
* Do not show delete icon if there is historical records associated with site group (Menus, Menu Cycles, Menu Templates)
** Only show delete icon if no records are associated with site group
*** No need to check name uniqueness if user re creates a new site group with same name
*** IF user creates a new site group and has no sites associated and decides at any point to come back and make changes to name, system should always to save
*** Show save button as close if no changes are made (Similar to standard menu)
* Add a new column for status to display Active and Inactive 
** Allow Inactive status to be clickable to be able to make it active again
** Display a confirm box “Do you want to make *Site Group Name* Active?” Cancel/ Yes
