# NXT-71772 — SchoolCafé - Forms - New Settings for Form Configuration and Display (SC Only)

## Description
*Summary*

* Integrate a SchoolCafé setting for automatically processing a Form: 
** Either using the existing ONLAPPROCE setting, or by adding a question to the Form Configuration process.
* Add a new setting value to the ADDINCSUR setting that allows districts to hide Income Surveys from the Family Portal (basically, allows the manual entry method only)
** Update the ADDINCSUR setting to add a new setting option, ‘Manual Entry Only’ 
** When this option is selected, hide Income Survey functions and behavior from within the family portal
** Continue to allow users to view the Income Survey category and operations within SchoolCafé 

*Details*

# Choose +one+ of the following two options for allowing forms to be auto-processed in the Family Hub or SchoolCafé:

## Integrate the setting, *Automatically Validate Online Applications if All Required Fields are Completed* (ONLAPPROCE), so that whenever a complete Form is submitted in the ‘Fill Out a Form’ page (i.e. all required fields are inputted, no pending students), the form gets automatically processed just like applications do under this setting. 
### Rename this setting to *Automatically Validate Online Applications or Forms if All Required Fields are Completed*

{color:#0747a6}*OR:*{color}

## Add a new question to the questionnaire part of Form Configuration, inserted as question #6:
### Question text: {{Form applications should be auto-processed if all required fields are completed}}
### Default value: Yes
### Behavior: The same as the ONLAPPROCE setting, but +*only*+ applies to applications belonging to this specific form type. 
!image-20260407-193612.png|width=1011,alt="image-20260407-193612.png"!
### If this option is selected, update the Form Configuration grid:
#### Remove the ‘End Date’ column and add the end date to the ‘Start Date’ column as a date range instead.
#### Change the ‘Start Date’ label to ‘Availability’
#### Add a new column for ‘Auto-Processing’ which corresponds to the answer to the question above for the specified form.

!image-20260407-194106.png|width=987,alt="image-20260407-194106.png"!

# Either way, these form applications should immediately show as ‘Processed’ in all relevant locations, as soon as they are submitted with all required info. 
## We can let the parent know as soon as the application is submitted, what the result will be ‘Qualified or Unqualified’ (there is a separate story for this: [https://primeroedgenext.atlassian.net/browse/NXT-71764|https://primeroedgenext.atlassian.net/browse/NXT-71764|smart-link]).

# In System Settings, in the Eligibility section, for the setting "Enables Income Surveys with License", add a new (optional) setting value: *Manual Entry Only*
## Additionally, update the tooltip/description: "When set to YES parents will enter Income Surveys only, When MIXED parents will enter a Meal Application or Income survey based on the enrollment sites of students. When NO parents will enter Meal Applications only. MANUAL ENTRY ONLY will make it so that the feature can only be utilized by district staff entering surveys manually.”
## In _Eligibility_, continue showing the Income Surveys category as usual, allowing districts to still manually add Income Surveys as needed. 
!image-20260407-195903.png|width=234,alt="image-20260407-195903.png"!
## We will handle Family Portal implementation of this setting (where Income Survey submission will be blocked in this scenario) in a separate story.
