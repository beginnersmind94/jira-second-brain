# NXT-58550 — Elig - Reviews - Search + Results Grid

## Description
As Dana the Director I want to generate a student roster for the auditor, and then be able to flag students for review on that list based on the configuration numbers.

*mockup*

!image-20250128-210158.png|width=1081,height=739,alt="image-20250128-210158.png"!

*requirements*

* User can select an Academic Year and an As Of Date and hit Apply.
** If the Statistic Sample stats haven’t been put in for that year, give the user a warning message:
*** Please configure the Statistical Sample first.
** Either way, load a results grid that is almost a copy of the Eligibility Roster
*** Find all Free and Reduced students as of the date
*** Sort by Last Name
*** User can export the grid out to Excel
* If the Statistical Sample data has been entered, we can fill in the Selected column based on the following:
** Use the Random Starting point and select that student.
** Add the Interval, select that student.
** Repeat until the number of selected = the Number of Eligible Students that MUST be reviewed.
* User can filter the grid to just the Selected students.
* User can click on the Application # or DC File # to be taken to the Application or DC File Details. 
** Open in New Tab

## Acceptance Criteria
* User can perform a search
** Select an Academic Year
** Select an As Of Date
* User is warned if they haven’t filled out the Statistical Sample Generator for that academic year.
* Results grid is populated either way with Free and Reduced students as of the selected date.
** We can only populate the Selected column if they have filled out the Sample Generator data entry.
** Default sort: Last Name
** Cannot change the sorting of the grid.
** Filters available on columns with an (F) in the example.
* User can filter the grid to Selected entries only.
* User can access the Application or DC File Details for students.
* User can export the grid.
