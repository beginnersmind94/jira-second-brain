# NXT-43488 — MP- Menu Planner- Allow Create menu/menu cycle to have multi select for Serving Groups

## Description
As a menu planner, I would like to be able to create menu or menu cycles with multiple serving groups from the menu planner.

## Acceptance Criteria
* Allow user when they are creating menu/menu cycle to multi select serving groups
* !image-20230928-114427.png|width=1478,height=508!

* Display serving groups available based on meal services provided 

|USDA CACFP Meal Pattern| |
|Ages 1-2|Breakfast, Lunch, Snack, Supper|
|Ages 3-5|Breakfast, Lunch, Snack, Supper|
|Ages 6-18|Breakfast, Lunch, Snack, Supper|
|Adults|Breakfast, Lunch, Snack, Supper|

|USDA Meal Pattern SY 2014+| |
|PK|Breakfast, Lunch, Snack, Supper|
|K-5|Breakfast, Lunch, Supper|
|K-8|Breakfast, Lunch, Supper|
|K-12|Breakfast, Snack|
|6-8|Breakfast, Lunch, Supper|
|9-12|Breakfast, Lunch, Supper|
|Adult|Breakfast, Lunch, Supper|
|Ala Carte|Breakfast, Lunch, Supper|

|USDA SFSP Meal Pattern| |
|All Ages|Breakfast, Lunch, Snack, Supper|

* Once selected and user clicks apply, only display the serving groups that were selected 
** Default to whatever serving group shows in order (Similar to edit mode when user adds serving groups from standard menu)
*** EX: User selects K-5 and 6-8, default to select k-5 once user clicks apply
* While in the create mode, any menu item added will apply to all serving groups
** Apply default serving size for all serving groups if applicable 
*** User can still switch serving groups and change serving size from the menu item serving size drop down if needed
** Users if they do not need to serve that item for a particular serving group will need to change the serving size to NOT SERVED
* Serving Group exceptions must be handled all on the fly if there happens to be menus created with multiple serving groups. This is a continuation from story ([https://primeroedgenext.atlassian.net/browse/NXT-43544|https://primeroedgenext.atlassian.net/browse/NXT-43544|smart-link] )
