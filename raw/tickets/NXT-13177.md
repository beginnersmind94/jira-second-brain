# NXT-13177 — [Cancelled] Platform - AI/ML - Authenticate Device

## Description
*Persona*: Student/Parent

*Access level:* School District/Site

*Description*:

As a Operational site admin, I should be able to register and configure a new EdgeBox with UUID identifier and assign to a school district. 

*Mockup:*

[https://www.figma.com/proto/tpwyiu5Ynk6Bh0cX0tsG7Q/Systems-3.0?node-id=7%3A3203&scaling=scale-down-width&page-id=0%3A1|https://www.figma.com/proto/tpwyiu5Ynk6Bh0cX0tsG7Q/Systems-3.0?node-id=7%3A3203&scaling=scale-down-width&page-id=0%3A1|smart-link] 

*Requirements:*

* Edge Device with specific serial number triggers SaaS API to authenticate the device with a specific token
* Valid token must be provided for activation/registration
* When EdgeBox sends a request to authenticate, system should validate the authentication token assigned to that EdgeBox belonging to an ISD
* Same Authentication token shall be used for camera devices along attached to the EdgeBox. 
* EdgeBox handshake with POS will be bi-directional based on device UUID and POS ID
** SSE - Service & Events (LAN only - offline mode)
* When authentication fails, System shall generate an email notification for the site admins
* System admin shall be able to RESET the authentication token and update the EdgeBox with new token
