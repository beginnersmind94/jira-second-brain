# NXT-6467 — Eligibility/POS - Patron Lookup Feature

## Description
*User Story*

As an admin I want a way to search for a particular patron and be able to select them for interactions on other pages.

*Mockup*

[https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=3869%3A67555&scaling=min-zoom|https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=3869%3A67555&scaling=min-zoom]

*Requirements*

* Add “Find a Student” button that opens the lookup feature
* Add a lookup feature that can be used for multiple modules.
** Primarily used as a pop-up to perform a filtered search for a patron account.
** A single account can be selected from the results grid and passed back as the selected account.
* Options for the lookup should be somewhat customizable when being used.
** When being implemented elsewhere, allow for search fields and result columns to be toggled on/off or set values forced through.
** Example - there’s a Patron Type field, but we may have some pages where (in Applications especially) where we only want to show Students in the results. There’s no need for the Patron Type field to be available for selection in that case.
** Option for Advanced Search > DOB, Area, Site, Site Code, Grade, Homeroom, Status
** 5 max results in lookup and pagination in the lookup
* Available search fields/filters:
** Name  
*** user can search first, middle or last name 
** ID #
*** Search against ExternalPersonID, State ID and PIN #.
*** For SE, we are looking at Student ID 
** Area
*** Drop down - dependent on district
*** dynamic - hide if no areas exist
*** Relationship: site, site code and grade
** Birth Date
*** User can enter a date in DD/MM/YYYY format without having to enter the slashes.
** Site
*** Dropdown selection, but user can type in a site name
*** Relationship: area, site code and grade
** Site Code
*** enter or select a site code to narrow the dropdown list
*** Relationship: area, site, and grade
** Grade
*** If a site is selected, this should filter to the grades available for that site. Otherwise, all grades are available. Like the Roster page.
*** Relationship: area, site, site code
** Status (default: active)
*** Dropdown - Active / Inactive / All
** Roster (not needed for SE)
*** If a site is selected, this should filter appropriately.
** Patron Type (not needed for SE popup)
*** Dropdown - Student / Adult / All
* Results grid 
** All sortable column headers, all results
** Show ‘No results found.' in the grid if the search doesnt yield any results
** Shows the following data in each line item, if needed we can do expand/collapse 
*** Name (FN LN Suffix)
*** ID #
*** Site Name
*** Grade
*** Homeroom 
*** Birth Date
* Actions
** Search - searches based on the criteria entered. Must enter info in one of the fields… Name, ID (partial is okay)
*** Please enter the student Name or ID
** Advanced Search icon - DOB, Area, Site, Site Code, Grade, Homeroom, Status
** “Clear All” option - resets/clears the filters to default values
** Selection - user can select the student from the grid and the UI indicates that selection with ADA compliant colors and populates the selected student info in the fields 
* Selection Expandable Fields 
** Address, Guardian Name, Eligibility (most recent), Status

PrimeroEdge current lookup fields example

## Acceptance Criteria
Lookup feature added that can be integrated onto other pages.
Search and filters available as detailed in the requirements.
Users can select a single account and send it to another page.
