# NXT-64747 — WS - Insights - KPI - Inventory Value - Benchmark Config Screen

## Description
As a district nutrition manager, I want to see an *Inventory Value card* on the Insights dashboard, benchmarked against the *Inventory Total Goal* for each site, so that, I can quickly evaluate whether schools are staying within acceptable inventory levels.

*Notes:*

* This story is for the benchmark screen. The Dashboard screen will be done separately.

*Requirements:*

*Grid Updates:*

* Add Inventory Total as a column to the grid
** Column name: Inventory Total ($)
** Location: After Waste (keep grouped with other dollar columns)
** Type and formatting: money, same as Revenue
* Ensure the column has the Actuals value field next to it like all the other columns
** Show the actual value for Inventory Total using the same rules and formatting as for Revenue and Waste
* Ensure the Total Inventory value is added to the API and dataset

*Years List Updates:*

* Ensure the Total Inventory textbox value is updated when changing years
* Ensure the Total Inventory actual value is updated when changing years

*AutoFill List Updates:*

* Ensure the Total Inventory textbox value is updated when updating using the autofill button

*Bulk Update Updates:*

* Add the Total Inventory textbox to the bulk update popup
** Label: Inventory Total ($)
** Location: Put it after Waste and bump the next one down a row
** Type and formatting: money, same as Revenue and Waste
** Ensure users can update the field the same as Revenue and Waste
** Update the Apply Actions button to ensure the Inventory Total button is updated on the benchmark screen the same way Revenue and waste are
* Ensure the Total Inventory textbox value is updated when the user has changed it in the benchmark popup

*View History Updates:*

* Ensure Inventory Total is accounted for in the backend API and database for all Add, Edit and Delete actions, same as Revenue and Waste
* Changes to the Inventory Value appear in the History Log when viewing it

*Save Changes Updates:*

* Ensure Inventory Total is saved or updated in the database when the user clicks save
* Ensure this new value is reflected on the dashboard when closing the configuration

## Acceptance Criteria
h2. 1. Grid Updates

* The benchmark grid includes a new column named Inventory Total ($).
* The Inventory Total ($) column is positioned after Waste, grouped with other dollar-based columns.
* The column uses currency formatting, matching the formatting rules used for Revenue and Waste.
* The column displays:
** The Actuals value field next to the column, consistent with other benchmark columns.
** The actual Inventory Total value using the same calculation and display rules as Revenue and Waste.
* The Total Inventory value is included in the backend API response and dataset used by the benchmark screen.

----

h2. 2. Year Selection Updates

* When the user changes the selected year:
** The Inventory Total textbox value updates to reflect the selected year.
** The Inventory Total actual value in the grid updates accordingly.
* Year changes trigger the same refresh behavior for Inventory Total as they do for Revenue and Waste.

----

h2. 3. Autofill Updates

* When the user uses the Autofill feature:
** The Inventory Total textbox value is updated based on autofill rules.
** The value is recalculated and displayed consistently with Revenue and Waste.
* Autofill updates do not require a page refresh to reflect Inventory Total changes.

----

h2. 4. Bulk Update Popup

* The Bulk Update popup includes a new field:
** Label: Inventory Total ($)
** Type: Currency (money)
** Formatting: Same as Revenue and Waste
* The Inventory Total field is positioned:
** After Waste, with subsequent fields shifted down one row.
* Users can:
** Enter, edit, and clear the Inventory Total value using the same interaction rules as Revenue and Waste.
* Clicking Apply Actions:
** Updates the Inventory Total value on the benchmark screen.
** Applies changes using the same logic as Revenue and Waste.
* After applying:
** The Inventory Total textbox reflects the updated value immediately.

----

h2. 5. View History Updates

* Inventory Total is included in:
** Backend APIs
** Database schema
** History tracking
* Inventory Total changes are logged for:
** Add actions
** Edit actions
** Delete actions
* History entries for Inventory Total follow the same structure and visibility rules as Revenue and Waste.

----

h2. 6. Save Changes Behavior

* When the user clicks Save:
** Inventory Total values are persisted to the database.
** Updates apply at the correct site and year level.
* After saving and closing the configuration:
** The updated Inventory Total value is reflected on the Insights dashboard.
** No manual refresh is required to see the updated value.

----

h2. 7. Data Consistency & Validation

* Inventory Total values:
** Must be non-negative currency values.
** Follow the same validation rules as Revenue and Waste.
* If a save or update fails:
** Inventory Total values are not partially saved.
** The user is shown the standard error message used for benchmark save failures.

----

h2. 8. Performance & UX

* Adding Inventory Total does not degrade benchmark grid performance.
* Grid loads, year switching, autofill, and bulk updates complete within existing performance thresholds.
* Inventory Total visually aligns with existing dollar columns and follows current UI standards.
