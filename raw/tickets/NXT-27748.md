# NXT-27748 — POS - Print receipts function changes

## Description
POS when the Print All Transaction Toggle is ON the POS should print the receipt without the user having to confirm if the receipt should be printed. When the toggle is OFF the user should not be prompted if the receipt should be printed. 

Current function is that from Meal Service, Pay, Add Funds, Payments the user is prompted to answer Yes or No when the transaction is complete. This significantly slows the line.

!Screen Shot 2022-03-03 at 1.03.23 PM.png|width=1332,height=600!

*Acceptance Criteria*

When the Toggle is ON the receipt automatically prints to the default printer

When the Toggle is OFF there is no receipt printed AND there is no prompt to print a receipt.

## Acceptance Criteria
When the Toggle is ON the receipt automatically prints to the default printer

When the Toggle is OFF there is no receipt printed AND there is no prompt to print a receipt.
