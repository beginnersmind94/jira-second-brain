# NXT-26977 — PR- Carryover Calendar- Meal Type Order/ Comment Box 

## Description
As a user, I should not be able to carryover the same meal type to the same day regardless of menu line. We can also not carryover backwards.

## Acceptance Criteria
Carryover should not happen from same meal type to same meal type on the same day

This should effect *ALL Menu Items.*

Display error message: “You are not allowed to carryover to the same meal type or menu line on the same day”

Default the meal type and menu line to the same line and meal type you are on if there are production plans planned within the carryover day window but do not allow same day selection. However, if there is not that menu line available in the future, do not have a default selection for both and only the meal type.

Order of Meal types

Breakfast

Lunch

Snack

Supper

WE can not carryover backwards meaning Lunch can not carryover to Breakfast on the same day.

Supper can not carryover to any meal type on the same day and so forth.

Disable meal types and menu line that are in the past on the same day, but reenable if user selects a different day. 

*Where to find meal type timings : select * from MealType - this has the start and end times for a meal type*

ALLOW user to enter a comment when entering carryover information on the pop up display. Make this field required to enter and display that it is required.
