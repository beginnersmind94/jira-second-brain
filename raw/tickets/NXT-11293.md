# NXT-11293 — Notification - Sending through email

## Description
As a platform admin, I would like to provide a means to send the notification through email. 

We will use the existing email component to communicate.

Integrate with the letter generation to get content if the notification content type is letter. 

This is purely back end work.

*Inputs* - Module, Business Process, Channel, Notification Type, Content Type, List of business objects (like application or student data), List of email(s)

Steps for sending notification through email:

* Based on the module, business process, channel, notification type and content type identify the notification in the list of notifications (it should be unique with this combination). Get only active notification. (Status field)
*  If the content type is letter, based on the module, business process (letter type), status (active only), call the generate letter routine. Based on the module pass the business data to the letter generator. For example, For student eligibility approval send application / student data to generate an approval letter. In the backend API make the list of business objects as a parameter that will be used for generating the letter. Like list of applications, students etc. It can be generic objects that this framework just takes and passes the information to letter generation library.
* Pick the content that is returned from the previous step (letter generation library) and call the Twilio library with list of email(s) got from the input.

Need to support multiple notifications based on the size of the lists that come in the input. Please store each record of the notification instance in a separate table for history purposes.

None required as its a service in the back end.
