# NXT-40331 — eOrders Configuration – Add New Part 4 – Sites Tab

## Description
This is a continuation of the eOrders Configuration – Add New Part 1 and covers the Sites Tab

*Notes:*

* The data for this tab resides in Back Office (BO)
* BO team must provide APIs for the following:
** Getting dataset to load grid
** Post the result of the grid to save to the database
** Post the result of the grid to update the database
* When we save or update, call their API and have them save it using the Vendor ID

*Requirements:*

* If the Sites tab is not already created, then create it and do the following on it:
* Show this tab when the user selects which sites can use the configuration and assigns the unique ID to the site
* Review the Site EOrder Settings grid in the ManageVendorEDISettings page for reference
* 
* Add a Grid to the tab
** Unlike most grids, this one will need to be able to edit values for some of the columns
** The grid shows the list of sites for the district
* Columns:
** Site
*** Name of the site
** Enabled
*** Yes / No
*** Toggle like on Status column on Imports Grid
*** Default to No
** Customer Number
*** Textbox
*** No default value
*** Required if Enabled = True
** Effective Date
*** Date picker
*** Date only, no time
*** Can pick backwards in time
*** Required if Enabled = True
*** No default value
* Grid has paging, but no sorting or filtering
* 
* Previous
** Go back to the previous tab
* Next
** Validate required fields and show the user validation error messages until they are all resolved
** Cannot advance unless they have selected all the required fields
** Go to next tab
