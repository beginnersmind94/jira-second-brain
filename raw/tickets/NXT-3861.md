# NXT-3861 — Add Item - Ingredient - Add Item Info

## Description
*User Story*

As a Item Management Super User/Menu Planner/Food Service Director/Nutritionist, I can use the ‘Add New’ button to manually add an item and input the general fields

*Mockup*

[https://nqyc9v.axshare.com/start.html#p=itemmanagement_-_ingredient&id=00ehm1&g=1|https://nqyc9v.axshare.com/start.html#p=itemmanagement_-_ingredient&id=00ehm1&g=1]

*Requirements*

* Ingredient Name
** Required 
** User defined
** Free form field that describes the name of the item entered.
* Do you need a Short Name?
** Checkbox
** If checked, add field ‘Short Name’
*** Free form text field to input Short Name
*** Only appears if checkbox is selected
**** _Note_ - The mockup is hiding behind Ingredient Category
* Item Code (Not visible)
** Autogenerated field on save based off the datasource
** Read-Only
** Required
* Category
** Dropdown list
** Recommended
** User defined - Default Category list:
*** Meat & Meat Alternatives
*** Vegetables
*** Fruits
*** Grains
*** Dairy
*** Uncategorized (Default selection)
* Ingredient Form
** As Purchased
** As Served

* Item Form
** Optional
** User defined
** Options: “As served” or “As purchased”
* Sub Ingredient
** Free form text field
** Optional
** User defined
* Standard Recipe Direction
** Free form text field
** Optional
** User defined
* Locally Grown
** Checkbox
** Optional 
** User defined
** Describes where the food is sourced from. Could describe “Sourced Locally” which is configurable stating if it was sourced within a certain range of the district.

## Acceptance Criteria
* Above Requirement Fields are visible and modifiable by user
* All fields accessible by Super User
* When Short Name is checked, the field 'Short Name' should show up
* Item Code - Item code generation needs to follow the data source code prefix. Takes the prefix from the datasource and add a number 1 to it. This ensures that it will always be unique Product Prefix - DST Ingredients Prefix - LI
* Validation on Item name (current validation in classic is it should be unique per data source)
