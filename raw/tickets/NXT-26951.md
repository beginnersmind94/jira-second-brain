# NXT-26951 — INV - Pack Sizes - Develop a common Quantity On Hand view

## Description
As an Inventory user I need to be able to quickly see and understand my QOH so I can know how much I have and how much is needed.

*Requirements*

* Develop a common API to reference QOH, calculate according to item hierarchy, and display in a clear way
* True Quantity = exactly whichever quantities are represented from the total batches
** This will not round up to higher units
** Ex: 4 Cases, 12 Packages, 16 Bottles
* Provide alternate representations of Quantity On Hand by providing an overlay which displays the QOH expressed in other pack sizes
** See image below as example
*** True: 3 Case, 4 Bag
*** Alternate: 34 Bags
** Round up to the nearest whole unit quantity for each unit
*** i.e. do not display 30.45 Cases
** [Storybook: Popover Component|https://devuicomponents.perseusedge.com/?path=/docs/components-popover--default]
* Popover labels
** Actual QOH
** Highest Unit QOH
*** we don’t need to show this if Actual is the same as Highest
*** this is used when the user may have many lowest units and want to see them in the equivalence of higher units
** Middle Unit QOH
*** this line will only show for 3-tier hierarchies
** Lowest Unit QOH
* Add the control to the Perpetual Inventory page
