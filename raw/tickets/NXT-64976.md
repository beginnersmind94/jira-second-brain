# NXT-64976 — Define Account Code Structure (Proposed #1)

## Description
*As a* district business administrator configuring the Financial Oversight module
*I want* to configure the state-required accounting code structure
*So that* I can stay in compliance with audit requirements

h3. Context: 

This page is similar to the GL Code Segments page in PrimeroEdge’s Financials module. There are a few key differences, however:

* We will refer to the page as ‘Account Code Structure’ instead of ‘GL Code Segments’
* We will provide helpful information in a text box so the user understands what is expected of them here
* Let users pick from state-mandated templates to speed up onboarding

Note that there will always be a Site segment that the user cannot delete and will help them see transactions at the site level

## Acceptance Criteria
h3. *Acceptance Criteria*

* *State Template Selection*: I can select my state from a dropdown to auto-populate segments with state-compliant structure
* *Template Modification*: After selecting a state template, I can edit, add, or remove segments as needed
* *Reset to Template*: I can reset back to the original state template if I made changes
* I can add any number of segments with custom descriptions (Fund, Location, Department, etc.)
* For each segment, I can set: description, type (numeric/alphanumeric), length, and separator character
* Real-time preview shows how my segments build the complete account code format
* I can reorder segments by changing position numbers
* Site segment configuration includes options for separators and inclusion in account codes
* "Preview Account Codes" generates examples using my actual segment structure
* Can edit or delete segments at any time
* Real-time character count updates as segments are added/modified
* Account code examples update immediately when segments change
* Drag and drop reordering works for position changes
* Includes a mandatory 'Site' segment that can be reordered but not deleted or renamed

Note: Use branding elements (e.g. fonts, colors) as per design guidelines for SchoolCafe 2.0, consistent with the rest of the product.
