# NXT-9778 — Data Exchange - File Import Setup: Field Mapping Screen Source & Target

## Description
As a PrimeroEdge user, I would like to have an intuitive screen that allows me to easily associate fields in PrimeroEdge with the fields in my uploaded file, so that I am not overwhelmed by the amount of information on the screen.

*Mockup*:

[https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=392877%3A594&scaling=scale-down-width|https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=392877%3A594&scaling=scale-down-width|smart-link] 

*Requirements:*

* In File Import Setup page, add a *Field Mapping* screen which user can perform in stepper format after “File Upload” step
** This screen shows the details of a successfully uploaded file and allows the user to both map the fields
* Display *SOURCE* table- which displays all *PrimeroEdge* fields in the first column
** Displays all mandatory fields *required* by *Import Type*
** Required fields are displayed first
** These fields has draggable icon to their left which allows to map to target
** Source column should be displayed in GREEN

* Display *TARGET* section with Mapped and File fields columns:
** Mapped fields should be empty with “Not Mapped” status which allows user to drop the primeroedge fields from SOURCE.
** File Fields includes the fields (file columns) from the file uploaded by the user
**  order of the fields should start from 1-N columns in the csv/xls file
** Each row should align with PrimeroEdge rows accordingly
* *Mapped Fields:*
** This column should allow user to drag and drop fields from PrimeroEdge fields column
* DRAG & DROP with arrows should be displayed between SOURCE and TARGET sections
* *File Data Preview* table should be displayed below the SOURCE and TARGET sections
** Display File Fields, File Data Preview columns
** Render all fields from the file under File Fields column
** Render all values from the first row of the file under File Data Preview
** Display “NO DATA” if file does not consists any data
** If file does not contain atleast 3 columns, DO NOT display this File Preview section
* Display Legends in user friendly format to help user to navigate around
* Next button is greyed out without any validations completed in “Mapped Fields” section
* User should still be able to go back to the previous screen via a Previous button.

## Acceptance Criteria
* Verify user can proceed to this step from File Upload

* Verify SOURCE and TARGET sections are displayed properly with alignment

* Verify TARGET section has 2 columns with Mapped fields and File Fields displayed from uploaded file

* Verify PrimeroEdge fields are displayed based on the Import Type including the mandatory fields 

* Verify File Data Preview table is displayed based on the columns and data from the uploaded file

* Verify user can go back to File Upload step if needed and retain config changes

* Verify NEXT button is disabled when mapped fields has not be validated

* Verify screen Legend text and DRAG & DROP labels is displayed

* Mapped fields should be empty with “Not Mapped” status which allows user to drop the primeroedge fields from SOURCE.
* The “PrimeroEdge” field name shall be populated in the “Mapped Fields” column, by default, with orange font color to indicate that the field has not been mapped.  When the source field is dragged/dropped to the “Mapped Fields” cell, the font color shall become green (indicating it is mapped). The legend shall contain an entry to communicate this to the user.
