# NXT-71830 — GC - PE - My items page - Mapping district items to Cybersoft catalog items. 

## Description
GC - PE - My items page - Allow mapping of district items to Cybersoft catalog items. 

Scope of this story

# Add Data provider Dropdown with Cybersoft catalog option - only one option, defaults to cybersoft catalog.
# Add Items dropdown with ALL, Mapped and Unmapped options, 
## default the items dropdown to ALL option, 
# when clicked on apply button for a selected data source ,data provider and items dropdown option: 
## Display the grid with Columns  
##* Item# - Stock item number 
##* PrimeroEdge Item Description - Stock item description
##* Ingredient Code - Ingredient code  
##* PrimeroEdge Ingredient Name --Ingredient name
##* Manufacturer - if ingredient manufacturer and stock item manufacturer names are available display stock item manufacturer. other wise display the name available from ingredient or stock item. 
##* WU GTIN, – display Whole unit GTIN from stock item 
##* BU GTIN - Display Broken unit GTIN from stock item 
##** if ingredient GTIN is available and matching with one of WU GTIN or BU GTIN then ignore it , if its not matching with any , then display it in WU GTIN column next to existing WU GTIN code separated by a comma. – ideally it should be matching with one of WU GTIN or BU GTIN
##* Catalog Item Description  – display the currently linked ingredient name from Cybersoft catalog. otherwise it will be blank.
##* Mapped - Yes when there is a link exists for that item in Cybersoft catalog. No when it no link exists. 
## List All Items from selected Data source with or without GTINs 
### for linked items and ingredients all columns for ingredient and stock item sections will display data 
###* mp_ingredientitem is the table where you can see the link information - use the same datasourceid from this table same as of ingredient and item
### if there are stock items without links to ingredients only stock item columns will show data and other ingredient code and ingredient name will be blank. 
### if there are ingredients without stock items then only column data related to ingredient information will show data
## Item Counts on the results grid header  : Display the summary of item counts for Total items , mapped items, and unmapped items count as shown in the below mockup. 
# Enable/disable Map items button
## When ALL option is selected in Items dropdown: Disable the map items button and add a tooltip on the button “ Please select mapped or unmapped item status in Items dropdown to enable this button” .
## Enable the button if mapped or unmapped item status is selected. 
# When clicked on Map items button for Mapped  or Unmapped items selection:
## Save button is available next to map items button. 
## select column with check boxes in the grid. 
## Hide the “Mapped” column.
## when mapped selected user can remove the link and edit the link by selecting a different item from catalog
## when unmapped is selected user can create a link between district item and Cybersoft catalog item. 
## Catalog item description column: Item will be auto selected in the dropdown based on District GTIN(s) if we found a match in Cybersoft catalog , user can search and change the item if needed. 
## user can search by name and select one item from dropdown.
### scan thru the subset of catalog item results based on  the *keywords* we have in district ingredient name or district stock item name. 
###* Example chicken fajita, strips - all search results we get when searched with keyword chicken  or fajita or strips etc..
###* search support only by name from this page as its going to be a performance hit if we allow multiple columns. 
### display 50 items at a time and load more with lazy loading technique?? – need dev input on this. 
# Selecting the checkboxes for items and clicking on Save button will map the selected data source items with Cybersoft catalog items. 
## it should work if user selected items from multiple pages  when pagination is used on the page. 
# After saving is completed, grid loads in read only mode showing the catalog item description as a label. and checkboxes column disappear from the grid,
## save the GTINs from District and GTINs from Catalog along with ItemIDs on both sides of district and catalog.
### i am thinking we would save CatalogItemID, RegionID, DatasourceID, PrimeroIngredientID, PrimeroStockitemID, CatalogGTIN, CatalogGTINCode, CatalogBrokenUnitCode, CatalogVendorUPC, CatalogVendorUPCBroken, District GTIN, DistrictWUGTIN, DistrictBUGTIN, Product.
###* we discussed the above data to be saved in both primero and GC databases? 
### if district item has no GTIN information, and user manually selects an item from Cybersoft catalog search dropdown then assign the GTIN from Cybersoft catalog for ingredient GTIN  
### We can see the {color:#0747a6}+Linked to Global Catalog+ {color}on ingredient details page as shown in the 3rd screenshot. – currently the link is not working, please restore the old functionality, Michael from BO team told me its  calling api but not returning the data. please let me know what is its current behavior. 
# track the history for mapping items for new links and for editing links, 
## district data source item A is mapped with Cybersoft catalog item A for the first time– ingredient detail history tab will display the history record for it 
## if district data source item A is changed from Cybersoft catalog item A to AA then history will show the this change.
