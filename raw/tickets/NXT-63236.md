# NXT-63236 — API - GTIN Matching on Multi Data Source

## Description
_(none)_

## Acceptance Criteria
Navigation: GC Portal - Search

Changes Required:

*Catalog – Matching*
User Selects Provider Dropdown – Cybersoft

User paste the GTIN in comma separate format

The matching should go based on the following sequence: 

*Search for records in USDA Branded Foods*

Add a search Key as : USDA BF: Name || Shor Name || Category || GTIN || Brand || Manufacturer || Product Code || FDC ID

Use the branded foods table and match the GTINs based on last 10 digit of each GTIN pasted (like condition)


Fetch info from DB for such GTINs and insert the response in Staging DB (call it as – GTIN Match Logs)
Consume all the available fields such as Item Basic Details and Serving Size and Nutrient Information

 *Search for records in CNDB*

Add a search Key as : Name || Shor Name || Category || GTIN || Brand || Manufacturer || Product Code || CN Code

Use the consolidated CNDB foods table and match the GTINs based on last 10 digit of each GTIN pasted (like condition)

Consume all the available fields such as Item Basic Details and Serving Size and Nutrient Information

*Search for records in DataMart*

For the unmatched records:

Use Data mart for matching the last 10 digits of GTIN (add an additional field (as search key) that’s based on Product Code||GTIN||Purchase GTIN||Broken Unit GTIN||Vendor Product Code)

The system should match the records in unique data set – if not found, then it should looks at rest of the records and fetch the data. In such cases the GTIN should return the top 1 match from the DB irrespective of the where conditions used (basically where conditions apply for multiple GTIN)

for records matched in Data Mart, the allergen, meal credits can be included as-is



Upon Exporting the data:

The user should be able to choose PE Format or SC Format. The system should export the data based on selected format and download the data in XLSX or XLS (no CSV as it will lose GTIN formatting)



Fetching of Additional Data: Such as Images, Allergen and Pack Size will derived from other external data source tables (Future Use cases)
