# NXT-69386 — Update Profit & Loss Revenue Section

## Description
As a district user, I want the revenue section of the Profit & Loss report to include meal sales revenue, Additional Income under Revenue and calculate totals so revenue excludes sales tax while total collected includes it.

## Acceptance Criteria
h2. Meal Sales Revenue

* Create a row corresponding to every single meal sales revenue data point with the status “Active” in the mapping page.
** If a program is active but has no revenue, it should still display with $0.00 with the income in the current period section. 
* *Meal Sales Revenue (Student) value* = (total credits - total debits) for all ledger entry lines attributed to meal sales revenue data points with the status “Active” created for student programs, included in the report when the posting date (last day of the Period) falls within the report date range (Monthly / Quarterly / YTD), regardless of how many Financial periods are included in that report date range.
* *Meal Sales Revenue (Adults) value* = (total credits - total debits) for all ledger entry lines attributed to meal sales revenue data points with the status “Active” created for adult sale types, included in the report when the posting date (last day of the Period) falls within the report date range (Monthly / Quarterly / YTD), regardless of how many Financial periods are included in that report date range.

h2. Reimbursements 

* Create a row for Reimbursement Claim - Federal Sources.
** This row is the aggregation of all reimbursement data points with the status “Active” created for Federal Sources.
** *Reimbursement Claim - Federal Sources value* = (total credits - total debits) for all ledger entry lines attributed to reimbursement data points with the status “Active” created for Federal Sources, included in the report when the posting date (last day of the Claim Period) falls within the report date range (Monthly / Quarterly / YTD), regardless of how many Financial periods are included in that report date range.
* Create a row for Reimbursement Claim - State Sources.
** This row is the aggregation of all reimbursement data points with the status “Active” created for State Sources.
** *Reimbursement Claim - State Sources value* = (total credits - total debits) for all ledger entry lines attributed to reimbursement data points with the status “Active” created for State Sources, included in the report when the posting date (last day of the Claim Period) falls within the report date range (Monthly / Quarterly / YTD), regardless of how many Financial periods are included in that report date range.

h2. Other Revenue and Totals 

* Add *Additional Income* under the *Revenue* section.
* *Net Revenue (excludes sales tax)* = Cash Sales + Debit Sales + Additional Income
* *Sales Tax Collected* = Cash Tax + Debit Tax
* *Total Collected (includes sales tax)* = Net Revenue + Sales Tax Collected
* Update the Revenue section totals to use these definitions.
