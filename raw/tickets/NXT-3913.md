# NXT-3913 — System - Users - Edit User - Set user as active/inactive

## Description
As an administrator, I would like to be able to activate or deactivate a PrimeroEdge user, so that only authorized users are accessing PrimeroEdge as needed.

* In the Users page, when selecting an individual user, add the functionality to make users active or inactive.
** Add a toggle for individual users, through their User Details drawer, that performs this function.
* When the admin makes the user active, update their value in the database appropriately and display a success message.
* The same goes for when the admin makes a user inactive.
* The success message should vary appropriately. i.e. if a user is active, message should reference ‘User deactivated’ and if user is inactive, button should say ‘User activated’. 

Mock-up: [https://nqyc9v.axshare.com/start.html#p=users&id=yfr8gz|https://nqyc9v.axshare.com/start.html#p=users&id=yfr8gz]

*Mockup Scenario*
For an individual user (User Details page) the Active Toggle Button can be toggled on or off

## Acceptance Criteria
Update UI/visual indicator in User grid and card views for the district

Allow an admin to activate an inactive user with user details toggle
Allow an admin to deactivate an active user with user details toggle

Save database values appropriately

Display success messages appropriately
