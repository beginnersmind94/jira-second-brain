# NXT-17685 — INV - Review Order Settings - Cleanup

## Description
This story is for a collection of enhancement or smaller changes to make throughout the Add Rule process of the Review Order Settings.

*Requirements*

Make the following changes:

* All steps 
** remove Confirm button (it can too easily be confused as a Save button)
*** we won’t need this button anyway once the steps are clickable
** Call the delete API when a user exits the Add Rule process by clicking ‘Back to Rules List’ button (8/16/2021)
* User(s)
** default to the User Search tab if a brand new rule is being created
** default to the Selected Users if an existing rule is being opened
** make tabs appear consistent with Items step
***  they should not be connected to the grid
** Provide grid heading on the row that contains the buttons
*** “User List”
* Items
** default to the Add Items to Rule tab if a brand new rule is being created
** default to the Selected Items if an existing rule is being opened
** Fix the tab appearance when on ‘Selected Items’ tab
*** there is not appropriate spacing between the tab selection and the grid. it should look like the Add Items to Rule tab
*** app a little spacing to the tabs (check with Gary for any questions)
** Fix issue with Edit state. Save and Cancel buttons are shown when not really in Edit state
** Remove the label under Min/Max Order Qty fields (“Should be <= Max Purchase Quantity”)
*** this only needs to be displayed if user tries to save an incorrect value; does not need to be a permanent label
*** If user tries to save a Min Order Qty value that is not *less than* the Max Order Qty, display “Should be less than Maximum Order Quantity.”
*** if user tries to save a Max Order Qty value that is not *more than* the Min Qty, display “Should be greater than Minimum Order Quantity.”
** Fix Misspelled ‘Quantity’ in columns

* Vendor(s)
** -Remove vendors from the dropdown when they are added to the rule-
** -Vendor dropdown functions only as “add” action. Not as an “update” to the list-
