# NXT-4908 — SSO - Google (Social)

## Description
As a Customer Support user, I want to be able to setup Google as a district’s authentication provider so that they can use SSO

*Notes*:

* This story is for setting up Google Social in Keycloak, not for ADFS
* This feature will be done mostly the same way as OpenID and the requirements will reference this

*Flow:*

* User browses to System > Security page
* User clicks the Google auth type Add new button
* The system adds Google to the grid like it does for SchoolCafe simple authentication
* System saves the config to Keycloak
* User tests the config
* User optionally sets Google as the default SSO provider
* User optionally views History
* User optionally sets Google as their own SSO provider
* User optionally sets Google as someone else’s SSO provider if they have rights
* User optionally makes Google active or inactive
* User optionally deletes Google

## Acceptance Criteria
* Admin is able to add Google as an authentication method for their district
* Admin is able to assign Google as the preferred method
* Admin is able to enable / disable Google method
* Admin is able to delete Google method if there are other rows in the grid
* Users are able to set Google in their profile if it is a valid option
* Admins are able to set Google for a user if it a valid option
* Changes to Google auth show up in History page
* Passwords are handled the same was as for Google 
* If user is Google method, they redirect to their provider when click next on login screen
* User is redirected back to our website with valid auth token after logging in
