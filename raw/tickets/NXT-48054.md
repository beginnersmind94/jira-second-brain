# NXT-48054 — Users - Managed Views

## Description
As a SchoolCafé users, I would like to be able to see more information about users than what is in the default list using Managed Views, so that I can see other information.

*Requirements:*

* On System > Users, do the following
* Add the Managed Views functionality to the grid on the page
** Users should be able to do all the managed view stuff such as is on the Advanced tab of Item Management > Items
** This includes:
** Show Saved Views tab in the Manage Views drawer
** Users can see shared views someone has shared with them
** Users can see their own saved views
** Users can save a view as their favorite
** Users can share with others
** Users can edit their saved view (but not the saved view of others)
** Users can delete their saved view
** Note: Copy over the existing code as much as possible



*Manage Views drawer*

* Add the Saved Views tab with the relevant functionality mentioned above that we already have developed elsewhere
* 
* Additionally:
* Add a letter on each row in the Saved Views tab
* Personal, Shared or System view
** If they saved it, it is Mine
** Letter = M
** Tooltip = Mine
* Shared as User
** If shared with them as a user
** Letter = U
** Tooltip = User
* Shared as Role
** If shared with them as a role
** Letter = R
** Tooltip = Role
* Shared as System view
** If it is a global view, it is System
** Letter = G
** Tooltip = Global
* Put a tooltip over the letters with these labels
* Ensure the Saved Views tab shows the global views
* 
* New Requirement:
* Allow the user to remove themselves from a shared manage view
** You can use the delete button for this
** When they click on delete button for a view that is shared with them as an individual, then just remove their name from the view’s shared list
* If the user deletes a shared view for roles, show this confirmation message: Deleting a shared view removes it for everyone. Are you sure?
* 
* On the columns tab, add in the extra columns available for the grid besides the ones already in the grid:
** Cell Phone
** Creation Date
** LastLoginDate
** LastFailedLoginDate
** UpdateBy
** UpdateDate
** UserName
** WorkPhone



*Create 2 global views*

* Create a default global view for the Users dashboard
** Name: Default View
** Copy over all the columns and settings from our default view
** Ensure this is in the list of Saved Views as a System View
** Make this the default view
** Ensure the user can always click on this view to go back to our default view
* 
* Create one global managed view
** Name: Users Audit Fields
** Columns to show:
*** Email
*** First Name
*** Last Name
*** Creation Date
*** LastLoginDate
*** LastFailedLoginDate
*** UpdateBy
*** UpdateDate
*** Actions
** Ensure the user can select the Users Audit Fields in the list in Saved Views and that updates the grid accordingly



*Users Page*

* Add the Manage Views icon above the grid if not already there
** On click, open the Manage Views drawer
** Developer is able to decided if want to show the full text version of the button or show just the icon
* Add the ellipsis next to the Manage Views icon if not already there
* Add the Save As button to the ellipsis if not already there
** Functionality covered in the Save View story
* Add the Save button to the ellipsis if not already there
** Functionality covered in the Save View story
* -Add the Hide / Show filters button to the ellipsis-
** -Show or hide the filters row based upon the button click-
* 
* Grid:
** Actions column must always be visible, do not allow them to hide it

*Technical Details*

* FrontEnd
** Enable ManagedView for Users Grid
** Add the additional columns as mentioned above
* Backend
** Not Required

## Acceptance Criteria
* User can see Saved Views tab in the Manage Views drawer
* Users can see shared views someone has shared with them
* Users can see their own saved views
* User can see global views
* Users can save a view as their default
* Users can share with others
* Users can edit their saved view (but not the saved view of others)
* Users can delete their saved view
* Users can view an icon on each row in the Saved Views tab to indicate if it a Personal, Shared or System view
* On the columns tab, the extra columns are added besides the ones already in the grid
* Manage Views icon added above the grid
* On click, it opens the Manage Views drawer
* Ellipsis icon added next to the Manage Views icon
* Save As button added to the ellipsis and opens Save As drawer
* User able to save a view
* -Hide / Show filters button added to the ellipsis and shows or hides the filters row based upon the button click-
* Actions column always visible in grid
