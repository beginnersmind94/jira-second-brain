# NXT-30284 — Import - Item Details - Platform File Upload to Staging Only (1/3)

## Description
{color:#ff5630}This story is intended to cover the Platform team’s portion of the import.  BackOffice will have to complete their piece for the import to work end-to-end.{color}

_“As an Item Management Super User / Administrator, I need to be able to onboard new Items using an excel file in order to speed the process of bringing customers working with SchoolCafe’s Backoffice. “_

There are 3 steps involved in an import:

# Platform uploads the file to a staging repository
# The module team exposes an API to be notified that step 1 is complete
# The Platform notifies the module mechanism via the API identified in step 2.

This story is for step 1.

## Acceptance Criteria
*Step: Pre-Upload*

* A new “Import Type” shall be used for this import named “Item Details Import”.
* A dropdown parameter named “Data Source” shall be present containing one entry named “Local”.
** The “Local” Data Source entry shall be selected by default
* The user shall be prompted to download a template using the standard verbiage
** The “No” radio button shall be set by default
** The file template to be used shall be attached to and provided by the Back Office team in story [https://primeroedgenext.atlassian.net/browse/NXT-24910|https://primeroedgenext.atlassian.net/browse/NXT-24910|smart-link].  The fields marked in red font in the attachment are mandatory.

*Step: File Upload*

* No changes from existing behavior

*Step: Field Mapping*

* No changes from existing behavior
* No code mapping required

*Step: Review*

* No changes from existing behavior

*File Types allowed*

* File types accepted shall be CSV, XLS, and XSLX by default.

*Business Rules and Error Log Rules*

* The “Business Rules” and “Error Log Rules” as stated in [https://primeroedgenext.atlassian.net/browse/NXT-24910|https://primeroedgenext.atlassian.net/browse/NXT-24910|smart-link] shall be the responsibility of the Back Office team.
