# NXT-3209 — Reconciliation - Reconcile Info - Transactions - Add Transaction/Payment

## Description
*User Story*

As a user I need to be able to add a transaction to session if it was not able to be entered before the session was closed.

*Requirements*

* Launch Transaction Page with active session information. (New functionality) Click Add Transaction button in Transaction List.
* The user can add a transaction
* The user can search and select a student or adult.
* The user can select the meal type
* The user can add or remove menu items served
* The user can add or remove a payment

Notes from discussion,

* Transaction Type is not needed
* Original Sale Amount is not needed
* Original Amount for Payments is not needed
* If there is a payment ONLY Meal Type is not required. 
* Review Reason is not needed
* Any remaining amount from a check payment is automatically deposited.
** Check Payment - Sale Amount = Deposit Amount

## Acceptance Criteria
When Add is selected from the Transaction View the New Transaction page is launched with the active session information, Date, SessionId, User, etc
The user can add a transaction
The user can search for and select a student or adult.
The user can select the meal type
The user can add or remove menu items served during the transaction
The user can add or remove a payment added during the transaction
When the patron is an adult tax is calculated
Second meals are flagged for students
The user can enter quantity counts for menu items.
When the transaction is added it is included in the session totals.
Credit and Food restrictions are indicated with a warning popup and verification to continue.
User can override credit and food restrictions
