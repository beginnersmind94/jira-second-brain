# NXT-15142 — Import API End Point Configuration

## Description
As a school district administrator, I would like to have an API configuration page where I can plug in the API End point URL’s / End point alias, update user ID’s, Passwords, Pass-phrases, Certificates.

# This API Configuration page must be under System Configuration API Configuration.
# Once in API Configuration, User should be able to add new vendors, edit existing vendors.
# From the list of Vendors (grid page) user must have option to enable any API connection (only one per sub account type) 
# While creating the connection or API calls, the type of API (Import/export) must be tied up with a vendor (Power school, aries etc)appropriate template (Student import, Items import).
# Provision to save the Vendor Name (Nick name), save user ID, URL, Hide/show password must be provided.
# Export API configuration will be not activated.
# Provision for Certificates and other technical details will be updated once full testing with vendor starts.

Details:
Key and Secret textboxes should have a toggle to show and hide the values just like we have it on the Login page-Password textbox.
Url and Secret textboxes should be mandatory, while the Key textbox is optional.
Also special characters for Key and Secrets.
Set max length for Key textbox=100 and that of Secret to 300.


!image-20210712-213815.png|width=532,height=88!

The action button on the grid will have an Edit option enabled and Delete as disabled in the context menu for active provider configuration and will have Edit and Delete options enabled for inactive provider configuration. Just like we have on Import Dashboard.

When a new configuration is created, it needs to be saved with Status=1, so far we are not providing any toggle switch to enable/disable the configuration on the Save/Edit page. Enabling/disabling of the configuration will be performed from the toggle switch on the grid.
If a district has added API Provider, then at least one needs to be enabled at a given point in time.

## Acceptance Criteria
# Verify and validate that the new page “API Configuration” is within the configuration module.
# Verify and validate that the user can save the URL and also save with an alias (save as).
# Verify and validate that the user can save the user ID.
# Verify and validate that the user can save the password / Pass phrase.
# Verify and validate that the user can see the password  / Pass phrase before saving.
# Verify and validate that the user CANNOT see the saved user ID and password  / Pass phrase.
# Export API Configuration will not be activated.
