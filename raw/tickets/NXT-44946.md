# NXT-44946 — Notifications - Profile Preferences 

## Description
As a SchoolCafé user, I would like to be able to manage how I am notified by the system of things like banner notifications so that I can personalize the user experience more to my taste.

*Notes*

* This works like the Settings page, but is for users instead of the system

*Flow*

* User logs in and clicks on Profile in the Workspace sub menu: Workspace > Profile
* User clicks on the Preferences tab
* User makes any modifications they want
* System uses these settings when sending notifications

*Requirements*

* On Workspace > Profile do the following for add and edit
* Add a tab to the page if not already there
** Title: Preferences
* Add a group
** Ensure has expand / collapse arrow
** Title: Subscriptions and Notification Settings
* In this group:
** Put an info box above the list:
*** Text: Select which events you would like to subscribe to and how you would like to be notified
** Grid:
*** Inline editing
*** No paging, sorting, filtering
** Data
*** List of all the registered event type from the Subscription framework event list
*** Do it dynamically, don’t hard code the list
** Columns;
*** No name
**** Checkbox
**** If yes:
***** Want to receive notifications for this event type
***** Must complete all the other fields for the row to be able to save
**** If no, don’t send notifications for this event type
**** Default = No
*** Component
**** Read-Only
**** Values: Sites & Users, Management, Configuration, Integration
**** In first example, use Sites & Users
*** Event
**** Read-Only
**** Name of the Event from the Events table
**** In first example, use User Accounts
*** Criteria
**** Single select dropdown list
**** Values: comes from the criteria list created when creating the event id
***** Do not hard code
**** In first example, use Additions, Changes, Deletions 
***** Note: these are the criteria for User Accounts and almost all event types
**** No default selected
**** Required if row is checked
**** Do not do the +1 thing when doing the chips, show all the values in View mode
*** Distribution
**** Single select dropdown list
**** Values:
***** Email
***** Inbox
***** Toast
***** You can hard code these, they will likely never change except maybe when we do mobile messaging
**** No default selected
**** Required if row is checked
**** Do not do the +1 thing when doing the chips, show all the values in View mode
*** Timing
**** Single select dropdown list
**** Values:
***** Immediately
***** End of Day
***** End of Week
***** You can hard code these
**** No default selected
**** Required if row is checked
**** If do End of Day, schedule to distribute at midnight every day
**** If do End of Week, schedule to distribute at midnight every Friday
*** Save / Edit buttons
**** usual edit and save buttons on an inline grid
**** I am designing this way instead of one save button for whole grid b/c dev team will have to do all the registrations or de-registrations for each one, so just easier to do one at a time for dev team
**** When saving:
**** If they uncheck a row:
***** Remove the user from the subscription (this is opt-out)
***** If they are the only user, remove the subscription completely
**** If they check a row:
***** All editable fields are required to Save
***** Do the user Subscription registration for that row the same way you would do if they did it from that corresponding screen
** Note: No delete or add for this grid
* 
* Security: 
** Customer Support role can edit
** Any user can edit their own tab
