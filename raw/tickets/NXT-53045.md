# NXT-53045 — OpenID - District Authentication - Notify Button

## Description
As a Customer Support user, I want to be able to setup a district’s OpenID authentication provider so that they can use SSO

*Notes*:

* This story covers the missing Notify button functionality from the parent story



*Requirements*:

* These requirements apply to the System > Security page
* 
* Add a grid to the tab
** Title: Authentication methods
** No paging, filtering or sorting
** 
** Columns:
*** Actions
**** Notify
***** Note: this button is so that an admin can send an email to all users who have not configured this method to do so
***** Note: for now, this applies only to SchoolCafe Authentication sense OpenID has no user configuration anymore
***** Tooltip: Send notification to users who haven’t configured yet
***** On click:
****** Get the list of users who have not set a password for this method yet or have an expired password
****** For each one, send them the email for users with unconfigured authentication methods below
****** Show the user a message: Message sent to all users that have not configured this authentication message
**** Disabled for OpenID
**** Enabled for SchoolCafe Auth
