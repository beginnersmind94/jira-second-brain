# NXT-66135 — LMA - License Keys - Bulk Renew

## Description
As a License Management (LMA) role member, I would like to renew in bulk multiple license keys so that I can save a lot of time doing them one by one.

*Notes:*

* The use case for the story we want to bulk renew our software for multiple districts with their same licenses

*Flow:*

* LMA role user browses to System > License Management
* User clicks Bulk Renew button and drawer opens
* User completes fields and saves
* System generates a new license key with settings on the page for each selected district
* License Keys grid updates with the new information
* When district users login after original expiration, they still have access

*Requirements:*

* On License management screen, do the following:
* Add a button
** Bulk Renew
*** On click:
**** Open the Bulk Renew drawer
*** Only License Management role can click this button



*Bulk Renew drawer*

* If the drawer does not already exist, create it
** Title: Bulk Renew
** Width 1/2
** Use sliding motion



*Header*

* Show these fields are read only:
** Usage message (see bolt code)
* Show these fields as editable and required:
** Effective Date
*** Default to the End date of the existing license
** End Date
*** Default to one year from Effective Date
** Notes
*** Same as Generate New screen
** Select All
*** On click, select the checkbox on all the cards in the table



*Table*

* No paging, sorting or filtering
* List of keys with Active Status
** Only Active status, not others
** Columns:
*** Checkbox
*** District
*** Key
*** Start and End Dates
*** Deployment
*** Count
* 
* Bulk Renew button
** Only LMA roles can click this button
** Note:
*** There is a start date problem
*** It is possible a current license has an end date before the user’s selected Start Date
*** This would cause  gap in usage
*** To mitigate it, we will add this logic:
**** If current End Date < Selected Start Date, set new Start Date = current End Date to prevent the gap
**** Else, use new Start Date
** On click:
*** Validate required fields
*** Show this message: This will generate a new license key with existing licenses for each selected license key for chosen dates. If the Start Date is after the current end date, the existing End Date will be used for the new Start Date . Are you sure?
*** If confirmed:
**** Start Date = new start date field or existing end date, depending on business logic
**** End Date = end date field
**** Purchase Date = Original Purchase Date field for each
**** Generated Date = Now()
**** Generated By = The name of the user account that generated the license
**** Status = Active
**** License #
***** Generate a new one 
**** Update Notes field with that message
*** Show the success message, close the drawer and return the user to the License Keys page with the grid updated
*** Log in the History table as an Update
*  
* Security
** License Management role only has rights to take actions on this page



*_Note to Devs_*

* _When bulk renewing, it is also required to update the links for Sites and Devices the same as when doing just one_
* _See the Renew story for a discussion on that issue_

## Acceptance Criteria
h3. *1. Access and Navigation*

* On the *License Management screen*, add a *Bulk Renew* button.
* Only *License Management (LMA)* role can click the button.
* On click, open the *Bulk Renew* drawer.

h3. *2. Drawer Layout*

* Title: *Bulk Renew*
* Width: half page
* Opens with a *sliding motion*.

h3. *3. Header Section*

* Read-only fields:
** Usage message (from Bolt code)
* Editable and required fields:
** Effective Date
*** Default to the *End Date of the existing license*.
** End Date
*** Default to *one year from Effective Date*.
** Notes
*** Same behavior as *Generate New License* screen.

h3. *4. Select All*

* Checkbox that, when clicked, selects the checkbox on all the cards in the table.

h3. *5. Table of Licenses*

* Only show licenses with *Active Status*.
* No paging, sorting, or filtering.
* Columns:
** Checkbox
** District
** License Key
** Start Date
** End Date
** Deployment
** Count

h3. *6. Bulk Renew Button*

* Only *LMA role* can click.
* Business logic for Start Date:
** If *current End Date < selected Start Date*, use *current End Date* as the new Start Date to prevent gaps.
** Else, use the *selected Start Date*.
* On click:
** Validate required fields.
** Show confirmation message:
_"This will generate a new license key with existing licenses for each selected license key for chosen dates. If the Start Date is after the current End Date, the existing End Date will be used for the new Start Date. Are you sure?"_
** If confirmed:
*** Start Date = *new start date field* or *existing end date*, depending on the business logic.
*** End Date = *end date field*.
*** Purchase Date = *Original Purchase Date* for each license key.
*** Generated Date = *Now()*
*** Generated By = *current user account*
*** Status = *Active*
*** Generate a new *License #* for each renewed license.
*** Update Notes field with any relevant message.
*** Show success message, close the drawer, and return the user to the License Keys page with the *updated grid*.
*** Log the action in the *History table* as an *Update*.

h3. *7. Security*

* Only *License Management (LMA)* role can take actions on this page.
