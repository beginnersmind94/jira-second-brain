# NXT-62703 — POS - Served Sites - POS Updates

## Description
As Cassandra the Cashier I want to see account info for Served Sites without having to check the server for photos/data so the line keeps moving fast.

*requirements*

* Update POS to load Served Sites as part of initial load.
** If any Served Sites exist, download account data + photos for the accounts at the other sites.
** Update the ‘Students’ text on the loader to ‘Students Including Served Sites’
* Update POS to pull local account info for Served Sites.
** Suppress the Visiting Student alert if the student is part of a Served Site.
** The alert should still pop-up for visiting students who are not at a Served Site.
* Update dataflow APIs to account for Served Sites.

!image-20250527-170505.png|width=1660,height=993!

*Discussion 7/2/25* 

* Need to handle local data updates if a Served Site is removed.
** Example: Elem 2 is removed as a Served Site for Elem 1.
** Local account data for Elem 2 should be removed from the terminals at Elem 1 on login.
** Need to be mindful of login processing times.

## Acceptance Criteria
* POS dataflow updated to account for Served Site updates.
* Served Sites are part of initial data preload.
* POS pulls Served Site account info from local if available.
* Suppress the Visiting Student pop-up only if the student is at a Served Site.
* Handle local data clean-up after a Served Site is removed.
* No changes for transactions re: visiting students.
