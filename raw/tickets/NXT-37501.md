# NXT-37501 — IM - Items/Recipes - Move Tags to the header level

## Description
As an Item Management user I want Tags to be in a consistent UI location and mostly “out of the way” so I can use the tags in the context of the overall product and they don’t interfere with the IM specific attributes.

*Requirements*

* For all IM entities (Item, Recipe, -Menu Item, POS, etc.-) move the Tags to a common location, ideally in the header, that will never interfere with the entity-specific attributes
** Tags are a product-wide feature, used to tag anything and everything. They should be available on the Item screens, but they are not part of the “General Info” of anything in IM
* Change the UI element used to manage tags
** Rather than a lengthy text field in the main card, which is often unused or under-used, we could manage in a popup, slideout, flyout,
