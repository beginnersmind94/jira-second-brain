# NXT-22705 — INV - Withdrawal Details - History tab - Cancelled transactions should show as Cancelled

## Description
As a Site Manager I want the transaction details page to accurately reflect the most recent iteration of the transaction so I don’t need to view the history to understand.

*Question:* we are doing this for Additions. Is this also needed for Withdrawals? Should the same team do both? *Yes*, the WD history has the same need.

*Requirements*

* In the History grid on the tab
** Cancelled transactions should show as cancelled (instead of “reversal”): 
*** Transaction Details: “Withdrawal transaction #12345 cancelled.”
*** Combine the cancellation and reversal
**** The cancellation is the reversal
**** When looking at a cancelled transaction, the details should reflect the cancellation and display as negative values
** Original Comments do not need to be overwritten when cancelled
*** just display the user-entered comments at the time of that transaction being saved
*** comments can be blank for a particular history entry
* The negative or highlighted/red quantities on each details should show correctly.
