# NXT-35088 — Device Management – View POS Details

## Description
_Phase 2, System Step 4_



As a SchoolCafé admin, I want to see the details of a POS device so that I can use it in debugging any outstanding issues.

*Flow:*

* User logs in and browses to System > Management > Device Manager
* User clicks on the View Details button for a record
* The View Details drawer opens up and user sees the information

*Notes:*

* This is the continuation of the Device Manager stories.
** 1^st^, we created the APIs in Front Office
** 2^nd^, we updated the device manager screen
** 3^rd^, we created the Add New, Edit, Delete, Suspend, Activate functionality
** Now, we will do the view details for a POS device
* Note: this feature requires the work of both System and FO teams.
** By the time we start this story, FO team must have written the relative APIs for button
** And the Add New functionality must have been completed

*Requirements*:

* All of these requirements are for the POS View Details drawer
* Create the drawer per the Figma
** Title = POS Details
** All the page is read only, so don’t need input controls like textbox or drop down list
** Page can be simply html
** Fields (in no particular order)
** Device Type
*** Not in source db
*** Source has TerminalTypeId, so link to get the name
** Device Name
*** Use TerminalName from data table
** Site ID
*** Use SchoolId from data table
** Site
*** Not in source db
*** Source has SiteId, so link to get the name
** MAC Address = MACAddress
** Machine Name = MachineName              
** Touch Screen
*** Use TouchScreen
*** Display values are Yes or No
** Setup Complete
*** Use IsSetupComplete
*** Display values are Yes or No
** Last Communication
*** Use LastCommunication Date
*** Is date time field
*** Show the date and time to the second (if they have it)
** App Version = App Version
** Status = Status
** Description = Description
** Self Serve
*** Use IsSelfServe Terminal
*** Display values are Yes or No
** Satellite
*** Use isSatellite
*** Display values are Yes or No
** {color:#ff5630}o   Use POS Beta Site{color}
*** {color:#ff5630}Display values are Yes or No{color}
** -TerminalId = don’t show this, they don’t need it-
** -DistrictRegionId DatabaseName = don’t show this, they don’t need it-
* 
* Security:
* Roles & Permissions actions:
* Only those with View rights to Device Manager & Cybersoft admins can use

## Acceptance Criteria
* When click View details, opens this drawer
* Has the fields listed above with the correct data
* Only users with View rights can see the page
