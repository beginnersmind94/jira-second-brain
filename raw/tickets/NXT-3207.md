# NXT-3207 — Reconciliation - Reconcile Info - Adjust Session

## Description
*User Story*

As a manager I need to be able to enter an adjustment amount for the session to make it balance to zero if after entering corrections it still doesn’t balance to zero.

*Mock-up*

[https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=1137%3A22760&scaling=scale-down-width|https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=1137%3A22760&scaling=scale-down-width]

!Screen Shot 2020-10-16 at 2.47.40 PM.png|width=1388,height=635!

*Requirements*

* An adjustment amount is calculated to correct the total balance to zero
* A note MUST be entered to explain why the session was adjusted.
* The date, time, and user entering the adjustment amount is recorded
** Comment Structure “User Name, Date Time, User Entered Comment'
* Instructions mention ‘Comments must be entered.'
* If save is clicked without a comment a message is displayed requesting a comment be entered.
** Message displayed 'Please enter comment before saving'
* Cashier name and amount are not editable.
* Add record to session history.

## Acceptance Criteria
Select 'Adjust Session' when Total Over/Under value is not $0.00
An adjustment amount to zero over/under for the session is automatically calculated.
After a comment is entered and saved the adjustment amount is entered for the session
When the Over/Under Total is zero the session status will be changed to RECONCILED
Session history is updated
