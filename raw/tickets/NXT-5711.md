# NXT-5711 — INV - Create Orders - Edit Order - Add Additional Items

## Description
*User Story*
As an inventory user creating an order I need an Add Additional Items function when editing an order so I can add items to my order.

* if the user needs to add additional items to a specific order, they will be taken to Step 3
* on the Step 3 screen, they will see the item list but there will be specific selections for all of the filter fields (Delivery Date, Vendor, etc.) and they cannot change them. This would mean that the user is not able to accidentally add items that would generate a new/different order.

*Context*

The Edit Order ([NXT-9678|https://primeroedgenext.atlassian.net/browse/NXT-9678]) functionality occurs on the Confirm Orders screen and allows the user to modify quantities or remove items without leaving the screen. However, if the user wants to add items, they will need to be navigated to the Enter Item Quantities (step 3) screen. This story is for the logic required in navigating the user back to step 3 with the updated order criteria.

Normally on the Enter Item Quantities screen, the user does not have to specify a Delivery Date or Vendor (or Order Template), which allows the user to potentially create multiple orders at a time. However, when the user would be navigated to the screen for editing, they are editing only a single order. Therefore, when the user is navigated back to the screen for editing, we will need to enforce the order selections.

For example, if the user is editing an order for ABC Vendor with a Delivery Date of 8/1/2021, when they click Add Items they will be navigated back to the Enter Item Quantities screen. On this screen where the dropdowns are usually optional, they will instead be pre-populated and not editable. In this case, the Vendor dropdown would have ‘ABC Vendor' as the selection and the user could not change it. The Delivery Date would be ‘8/1/2021’ and they could not change it. However, Order Templates could be selected and applied.

*Requirements*

* When a user is editing an order and clicks Add Items, navigate the user to the Enter Item Quantities screen.
** save any order item changes that have been made by the user before navigating to this screen
* On the Enter Item Quantities screen, the Site, Order Date, Delivery Date and Vendor fields would all be selected according to the order being edited.
* Site, Order Date, Delivery Date, and Vendor should be disabled so the user cannot change them.
* Order Templates field would be editable and user could apply templates if desired.
** when initially loading the screen, no Order Template will be selected by default
* Provide a *Save* button so the user can return to the Confirm Orders screen after adding any items.
** do not need the Shopping Cart for this. The Save button will return the user directly to the Confirm screen.

## Acceptance Criteria
* Provide Add Items button in the Edit state of an order
* Add Items button will navigate user to the Enter Item Quantities screen
* User cannot change Site, Order Date, Delivery Date and Vendor selections when on this edit screen
* User can add items or change quantities from this screen
* User can save changes and will be navigated back to the Confirm Orders screen
