# NXT-5233 — Applications - Sessions - Session Summary

## Description
*User Story* 

As an applications processor I want to see what I just worked on so I can print a relevant report and sign off on my work.

*Mock Ups* 

[https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=421061%3A0&scaling=scale-down-width|https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=421061%3A0&scaling=scale-down-width|smart-link] 

*Requirements* 

When a user finishes processing and leaves the continuous processing flow, they should be taken to a Session Summary showing them what they just did.

* Session info to show:
** Determining Official (fname + lname of processor)
** Start Date
** End Date
** Applications Processed
* Grid showing the applications that were processed:
** App #
** Students
** Eligibility
** Reason
** Entry Method
** Processed Date
* User should be able to click the application # and be taken to the application details of that app.



*Note*

* Session data is per day per user
** George processed 90 applications across 2/22/21. Just 1 session will show on the list for George for 2/22/21.







*Discussion on 2/25/21*

* Harsha, Ragav and Matt discussed grid and date ranges.

## Acceptance Criteria
User is taken to a summary screen when they finish processing applications.

User is shown session info as outlined in story.

User has a grid showing the apps processed as part of that session.

User can click on an application and be taken to application details for that application.
