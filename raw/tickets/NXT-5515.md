# NXT-5515 — System - Periods - Allow user to Update Periods, including Fiscal Years

## Description
As a a district financials administrator, I would like to be able to specify my district's fiscal year and view or manage periods within that year, so that I can view reports and claims for the right date ranges.

+Summary+:

* Allow a user to update period details when previewing periods, before AND after creating periods.

+Details+:

* In the Create Periods section of the Generate Periods page, allow the user to perform several actions to update previewed or previously saved periods:
** Rename ‘Period Name’:
*** New period name must be unique and less than 50 chars.
** Change Period dates:
*** Dates should not overlap between periods, and remaining future period dates within the fiscal year should adjust accordingly so that the whole year is covered. (see Classic for example)
*** *Note:* Start and End Dates can only be modified before creating the period. Once periods have been created, these can no longer be changed and the user has to split them instead.
*** (see Classic for examples)
* When the user selects the ‘Create Periods’ button, allow them to commit the changes and update the period data for the selected fiscal year.
* Additionally, this story should accomplish the dynamic Fiscal Year dates, so that the user can modify the Start and End Year dates before creating a period:
** Default Start Date should be 7/1/XXXX and default End Date should be 6/30/YYYY, where XXXX is the current year and YYYY is next year.
** Start Date should also be greater than the End Date of the previous Fiscal Year. No other restrictions should apply to dates.
** Dates must be future (starting from Today) and Start Date must be before End Date.

## Acceptance Criteria
Verify Create Periods section can be accessed on the Generate Periods page when: Fiscal Year dates are set, Period Configuration are set, and Preview Periods button is selected.

Verify that previewed period details display in a table that matches selections from Period Configuration section.

Verify user can modify any Period Name and that it must be unique

Verify user can modify an End Date for a period and update it, and that updates all future date ranges so that the year is covered.

Verify user can commit the updated details by clicking the 'Create Periods' button.

Verify that the periods are then saved into the database.

Verify that Fiscal Year dates can be modified when dates have not already been set for a year. 

Verify that dates must be future (starting from Today) and Start Date must be before End Date.

Verify that we allow the user to save the Fiscal Year and it is saved to the database.
