h3. Settings

* Modify the setting for displaying available quantity to sites [https://primeroedgenext.atlassian.net/browse/NXT-66208|https://primeroedgenext.atlassian.net/browse/NXT-66208|smart-link] 
** Change toggle to a dropdown
*** label: ‘Display Available Quantity to Sites’
*** 3 options: Projected QOH (default), Actual QOH, Do Not Display QOH
** Provide an info icon with a clickable (not mouseover) fly-out explainer: “This setting determines which value for the distribution site’s Available Quantity is displayed to sites when ordering. Projected QOH will consider future transactions, while Actual QOH will strictly reference the perpetual value at the time. You may also choose to hide the Quantity Available value from ordering sites.”
* Provide a new toggle setting for whether or not sites can order more than the distribution site QOH
** label: ‘Allow Orders to Exceed Available Quantity'
** default to ‘Allow’ (enabled)
** if ‘Do not display QOH’ is selected for the ‘Display Available Quantity to Sites’ setting in the row above, forcibly set the toggle to ‘Allow’ and disable the toggle switch
** Provide an info icon with a clickable explainer: “This setting determines whether or not sites can order quantities greater than the distribution site’s Available Quantity. The value of the ‘Display Available Quantity to Sites’ setting will be considered for determining which QOH value is used to limit orders. Please note, this setting cannot be disabled if ‘Do Not Display QOH’ is selected for the prior setting.”
* Introduce a new setting for ‘Source of Substitutions’ [https://primeroedgenext.atlassian.net/browse/NXT-66209|https://primeroedgenext.atlassian.net/browse/NXT-66209|smart-link] 
** Dropdown w/ 2 options: Site Perpetual, Contract Items
** Default: Site Perpetual
** currently this is hardcoded to consider only contract items
** Provide an info icon with a clickable explainer: “When this option is set to Site Perpetual, any item in the distribution site’s inventory may be added to distributions for item substitutions or additions. When this option is set to Contract Items, only items on the vendor contract may be added to distributions.”
* Provide a toggle setting for whether or not items can be added without being a substitution
** label: 'Allow Non-Substitution Added Items'
** if this is set to ‘Do not allow' then all added items must be substituted
** Provide an info icon with a clickable explainer: “This setting determines whether items can be manually added to distributions. When this option is set to Allow, the distribution site can freely add items to a distribution without substituting for another item. When this option is set to Do Not Allow, the distribution site can only add items as part of a substitution.”
* Provide a setting for how orders will be received when the distribution for a day is already Assigned [https://primeroedgenext.atlassian.net/browse/NXT-66210|https://primeroedgenext.atlassian.net/browse/NXT-66210|smart-link] 
** this would override the existing Approve/Reject framework
** label: Allow Orders After Assignment
** 3 options:
*** Prompt Each Time (default) (current behavior)
*** Auto-Create New Distribution
*** Disallow orders after Assignment
** Provide an info icon with a clickable explainer: 
*** “This setting determines what should happen when orders are placed on a date for which a Distribution is already in Assignment status. 
Prompt Each Time will tentatively add orders to the existing distribution and allow the distribution site to decide whether to include them or move them to a new distribution.
Auto-Create New Distribution will accept orders after assignment, but they will automatically go to a new distribution transaction.
Disallow Orders After Assignment will completely prevent sites from ordering once that date is in Assignment status.”
