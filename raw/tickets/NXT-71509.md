# NXT-71509 — SC - PBIRD - Dashboard – View Configuration

## Description
As a Director, I want to view the full configuration of a Power BI report in one place so that I can quickly understand its setup without entering edit mode.

----

h2. *Description*

Allow users to view the full configuration of a Power BI report from the dashboard using a read-only viewer.

This view will display all configuration fields and values in a structured format and will reuse the same component used in the *Review tab* of the report configuration flow and the *Managed View Reports View Configuration* experience.

----

h2. *User Flow*

# User navigates to:
#* *Reports → Power BI Reports*
# User clicks the *ellipsis (⋮)* on a report row
# User selects *View Configuration*
# System opens a *View Configuration drawer/modal*
# User reviews configuration details
# User may download the configuration
# User closes the drawer

----

h2. *Out of Scope*

* Editing configuration
* Configuration stepper navigation
* History or audit details
* Other ellipsis actions

----

h2. *Requirements*

h3. *View Configuration Access*

* “View Configuration” is available as an option in the row ellipsis menu _(covered in separate story)_
* Selecting this option opens a *read-only configuration viewer*

----

h3. *Viewer Component (Reuse)*

* Use the same *configuration viewer component* as:
** Managed View Reports → View Configuration
** Review tab in report configuration flow
* Do not create a new custom viewer

----

h3. *Display*

* The viewer must display:
** All configuration sections and fields
** Corresponding values in a structured, readable format
* Sections should follow the same structure as:
** Selection
** Configuration
** (Sharing if applicable)
** Distribution
** Scheduling

----

h3. *Download Functionality*

* Include *Download option* within the viewer
* Behavior must match Managed View Reports:
** Downloads the configuration details shown in the viewer

----

h3. *Read-Only Behavior*

* No fields are editable in this view
* No navigation into edit flow from this screen

----

h3. *Consistency & Reuse*

* Styling, layout, and behavior must match Managed View Reports implementation
* Ensure consistency with Review tab display

## Acceptance Criteria
* Given a user selects View Configuration, when the viewer opens, then all configuration fields and values are displayed in a structured format
* Given the viewer is open, when the user reviews the data, then no fields are editable
* Given the viewer is open, when the user clicks Download, then the configuration is exported successfully
* Given the user closes the viewer, when the action completes, then the user returns to the Power BI Reports dashboard
