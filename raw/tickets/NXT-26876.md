# NXT-26876 — Platform - Permissions - Implement Page Level View Permissions

## Description
As a Primeroedge admin, I should be able to manage permissions for VIEW of the pages under any specific module

*Requirements:*

* When user updates the permissions of a page from roles & permissions, system shall reflect scope of the page when user reloads the screen.
* When VIEW permission is removed for any role, that specific resource/page must be removed/hidden from navigation screen effective immediately.
** All tabs should be hidden as well within that page
** Page/link should be removed from side-nav also
** Page should be removed/hidden for users for that specific users only
* When VIEW permission is enabled for any role, that specific resource/page must be visible from navigation screen effective immediately.

## Acceptance Criteria
* District/Cybersoft admin shall be able to update VIEW permissions of any page type resource from roles & permissions 
* User shall not be able to see the page if the role he has been assigned does not have “VIEW” permissions enabled
* User shall not be able to see the page from side-nav if the role he has been assigned does not have “VIEW” permissions enabled
* User shall not be able to see the tabs/sub-pages under the page that has been disabled access.
