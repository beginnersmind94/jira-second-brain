# NXT-51116 — POS - Dynamic Entrees - Data for Dynamic Entrees category + Usage

## Description
As Connie the Cashier I want to see the entrees from production plans after selecting the Dynamic Entrees (DE) category so that I can sell meals to students.

*mockup*

Not needed - this is to make sure we’re sending data down to populate the DE category and DE button(s).

*requirements*

* Populate the DE Category and DE menu buttons with items from the production plan:
** From the date selected for opening service.
*** If the user picks 5/27/24, check the Production Plans for 5/27/24 for the site they’re serving at.
*** Pool all items on the Production Plan for that site and date that: 
**** IsMeal / IsEntree flag set to 1
**** Production Plan count >0
**** POS pricing
*** Grab the pricing + pos button description
** DE Buttons on the main grid need to be ‘updated’ based on the sorting method ([https://primeroedgenext.atlassian.net/browse/NXT-51119|https://primeroedgenext.atlassian.net/browse/NXT-51119|smart-link] ) for the terminal.
*** Replace the name of the button with the name from item info, populate pricing, service restrictions, etc.
* User can see the items in the category grid
** Default sorting: Alphabetical.
*** User can change the sorting method later with another story → [https://primeroedgenext.atlassian.net/browse/NXT-51119|https://primeroedgenext.atlassian.net/browse/NXT-51119|smart-link] 
* User can see the Dynamic Entree buttons on the main grid have been updated with data from the Production Plan → Item Info.
** Name updated to use the POS Button name.
* User can select an item.
** Pricing is shown appropriately for the item.
* User can complete the transaction and sell the item, it flows up like normal.
