# NXT-56235 — PE - GDSN Connect - API Based Updates

## Description
_(none)_

## Acceptance Criteria
Navigation: MP - GC-Pending Updates 

Acceptance Criteria:

Pre-Requisite: User should login in GDSN Connect and process the items that are matched

User should have a new button in the Pending Updates screen as “*Check for Updates*”

Upon clicking the button, The API should be able to send all the data in PE for a specific region and return the results from GDSN Connect API - [https://api.gdsnconnect.com/docs/cybersoft/#/Items/getItemsFull|https://api.gdsnconnect.com/docs/cybersoft/#/Items/getItemsFull|smart-link] 

All the extracted API information should be stored in staging table for parsing of data

The exception engine should be run on a stored data in staging table (As-Is)

The refined data should be moved to GDSN Connect table (As-Is)

Once the data is stored, the user should be able to render the the updates in UI as-is

The user can download the exceptions for the latest entries updates in DB
