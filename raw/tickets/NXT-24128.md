# NXT-24128 — IM - Custom Allergen Configuration

## Description
As an authorized Nutrition service user I need a way to create custom allergen  in order to help identify items with allergens

## Acceptance Criteria
h2. Acceptance Criteria

This story is done when:

*Entry Point*

* A Custom allergens option is added under IM configuration area
* Option Custom Allergen

*Result Grid*

* Should display fields as per mockup 

User should be able to search and sort fields on result grid

* User should be able to edit Custom Allergen Description \[Description Only]
** Only for user created Custom Allergen (e.g CN cannot be edited or Deactivated)
* User should be able to deactivate Custom Allergen only if there are no items/recipes/menus using it
** User should be presented with a confirmation message “Are you sure you want to deactivate this Custom Allergen” is Yes proceed with deletion if no cancel process

When Slide Out Card is displayed

* Fields on slide out card:
** Custom Allergen Description \[Free text box]
*** Is a Mandatory field
*** Only can be added to user/district created data source (e.g. Local)
** Data Source \[Dropdown box that list available data sources to the user] \[Local]
*** Is a Mandatory field
*** Only one Data Source can be selected \[Local]
* When pressing the *SAVE* button
** Validation occurs
*** Custom Allergen is Mandatory and no duplicates are allowed
*** If Custom Allergen already exist warn the user with a message “Custom Allergen already in use, please use a different Custom Allergen name and allow the user to enter a different name”
*** One Data Source is Selected (Mandatory)
*** The newly added Custom Allergen is listed on the result grid
*** System returns to Custom Allergen screen
* If User clicks on the *X icon* then the Slide Out Card is close and transactions are canceled
* Pagination should be present
* *Export* \[PDF Format]
** Title \[Custom Allergen Configuration]
** Should cover same fields as the result grid \[Custom Allergen Description, Data Source Name]
** Header
*** Updated by \[User Name]
*** Generated on \{System date/time UTC]
** Footer
*** Power by SchoolCafe for \{School District Name]
*** Pagination should be present

*General:*
Once user selects configuration a pop-up window is presented to the user. This pop-up windows asks for data source and configuration option desired. When user click/press apply button then system re-directs the user to appropriate configuration area

Only use Local data source
