# NXT-30235 — Verification - Collection Report (FNS-742) - Add report details (Part 2)

## Description
As a district admin, since I worry a lot about my numbers being accurate during verification, I would like to have reporting that backs up the numbers (it would be a welcome relief).

*Mockup*

[https://www.figma.com/proto/Yo99YhQm5Xk91Ie7b0PRkD/Front-Office-6.0?node-id=250%3A10219&scaling=scale-down-width&page-id=250%3A10218&starting-point-node-id=250%3A10219&hide-ui=1|https://www.figma.com/proto/Yo99YhQm5Xk91Ie7b0PRkD/Front-Office-6.0?node-id=250%3A10219&scaling=scale-down-width&page-id=250%3A10218&starting-point-node-id=250%3A10219&hide-ui=1|smart-link] 

*Requirements*

* In Eligibility → Verification → Reporting → Collection Report, add a way for users to verify the information in every section by making the links clickable.
* To do this, add a document icon which is a hybrid hover+clickable icon:
** This icon should only display when a count is > 0.
** When a user hovers over it, displays a tooltip related to the section itself. Tooltip texts (*if different than mockup, use these*):
*** Section 1 A: The number of sites (not including RCCIs) established when creating the verification sample. Select Sites make up this count. Click on this icon to see a list of sites.
*** Section 1 B: The number of students (not including those from RCCIs) established when creating the verification sample. Select students make up this count. Click on this icon to see a list of applications.
*** Section 5-7 A: The number of Applications established when creating the verification sample. Select Applications make up this count. Click on this icon to see a list of applications.
*** Section 5-7 B: The number of students established when creating the verification sample. Select students make up this count. Click on this icon to see a list of students.
*** *Note that everywhere we show a value, there needs to be a corresponding document icon link. Sections other than the 4 above do not need to have a tooltip, but should still link the user to a page with either the related student, site, or application data.*
** When clicked, redirects the user to other sections/tabs within the Collection Report page
*** Update the title of the grid to reflect the section where the user clicked the details button
*** Disable tab navigation, because otherwise the numbers could be confusing
*** Add a ‘Back’ button to return to the Collection Worksheet.
** Examples:
*** User should be able to click on the 82 for 1-1-A and see a list of the 82 sites.
*** User should be able to click on the 63,502 for 1-1-B, and be shown the students that made up that number.
* *Additionally*, (not pictured in mock-up) If we cannot embed the report within the tab and automatically open it there (instead of as an overlay), add a ‘Generate Report’ button within the tab so that the user can click it to generate the report instead of having to click the ‘Apply’ button.

## Acceptance Criteria
Verify that:

* Every number on the Collection Report can be clicked on to show detailed information of what made up that number.
** Including new hover+clickable icons in the 4 sections above.
** All other sections just need to be clickable.
* Numbers shown in the worksheet columns should match the data in the Collection Report detail grids.
* 'Back' button added to return to the worksheet from the details.
* Worksheet is now either embedded within the tab or is accessed via a new ‘Generate Report’ button (instead of using the existing ‘Apply’ button above the tabs)
