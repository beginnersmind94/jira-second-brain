# NXT-3700 — Add Item - Ingredient - Add Nutrient Information - Serving Measures

## Description
*User Story*

As a Item Management User, when creating an Ingredient, I want to add the Nutrient Information

*Mockup*

[https://nqyc9v.axshare.com/start.html#p=itemmanagement_-_ingredient&id=00ehm1|https://nqyc9v.axshare.com/start.html#p=itemmanagement_-_ingredient&id=00ehm1]

*Requirements*

* Header called ‘Serving Measures'
** A Table that can list the different serving sizes
** User will click the Add or Minus for each serving size added
*** Serving
**** Serving Size
***** A fraction enabled numerical field allowing the user to select the service size. Ex:
****** 0.5 translates to 1/2
****** 1/2 can be entered
***** Minimum Fractional denominators allowed: 2, 4, 8 (others may be allowed or disallowed in future PBI)
***** Allow the user to select different measurement types - dropdown
*** Weight in Grams
**** A Decimal / Numerical field allowing the user to select the weight
**** Allow the user to select different weights - dropdown
*** Preferred Measurement Checkbox
*** Selection to add or remove the measurement on the right
**** After hitting the ‘Add’ button, the record is displayed in the grid
*** Converted Measurements
**** Calculate a standard list of measurements based off the input provided
***** if the user provided half a cup, what are all the convertible measurements related to that measurement
**** User can click on ‘Converted Measurements’ and select the ‘Preferred Measurement' for that measurement to show up in the grid
*** User can ‘Add’ a new measurement, which allows the user to input the information they need

## Acceptance Criteria
+ Serving Measures can be entered: Unit (Volume only, no weights), Quantity (decimal, whole or fraction), and Weight (decimal, usually in grams but user may be more comfortable in ounces)
+ Serving Measures can be edited (see also future INFO requirements, below)
+ Serving Measures can be removed (see also future INFO requirements, below)
+ Weight units cannot be selected for Serving Measures (we already know the weight for these!)
+ if Quantity <> 1, also display the weight for 1 of the Unit
+ if the Unit is convertible to other Units via standard conversion, also display the weights of all convertible Units on add (EX: 1 pint = 2 cups)
+ display convertible Units as such. EX: assuming 1 cup = 100 grams; display 1 pint (conv. from 1 cup) = 200 grams. Repeat for each convertible Unit
+ convertible Units are NOT allowed to be entered as well (once 1 cup is entered we already know the weight of 1 pint)
+ system is tolerant of Serving Measures that are not allowed to be entered from the UI (rule above), and displays them as they are. (some data from CNDB comes this way and we have to display it, without modification even if it doesn't add up)
+ Units can be marked as Preferred Measures - not all convertible Units are practical in the kitchen for preparation purposes. EX: They need to be able to indicate Cup and no. 10 Can are preferred measures. Here it is just an indication, but this will be used in more detail in Recipe Scaling (FUTURE).
* INFO: Serving Measures that are "in use" are not removable. (Note: This will be defined in Recipe Steps.)
* INFO: Modifying a Serving Measure, where Ingredient and Unit are "in use" on Recipe Steps will automatically trigger 
nutrient recalculation of Recipe Nutrients. (Note: This will be defined in Recipe Steps.)
* When a serving measure is added or removed from the grid, then add or delete from the database
