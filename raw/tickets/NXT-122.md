# NXT-122 — Accountability - Reports - Account Balance

## Description
*User Story*

As a district admin I want a way to bring up account balances for everyone on a particular date.

*Mockup*

[https://www.figma.com/proto/Yo99YhQm5Xk91Ie7b0PRkD/Front-Office-6.0?node-id=1974%3A21431&scaling=scale-down-width&page-id=1974%3A21430&starting-point-node-id=1974%3A21431|https://www.figma.com/proto/Yo99YhQm5Xk91Ie7b0PRkD/Front-Office-6.0?node-id=1974%3A21431&scaling=scale-down-width&page-id=1974%3A21430&starting-point-node-id=1974%3A21431|smart-link] 

*Requirements*

* In the Accountability → Reports landing page (not Account Management--ignore that detail in the mockup for now), add a new report with the following details:
** Report Name: *Account Balance*
** Description: _Lists account details and balances as of a selected date._
** Verify listings are still in alphabetical order
* On the report page, user can select:
** Site(s) (use the Site Code+Site Name dropdown)
*** Default option: All
** Grade(s)
*** Default option: All
** Date
*** Default option: Today’s date
** Account (any combo of the below)
*** Type (student/adult/special accounts)
**** Default option: All
*** Eligibility/Pricing (free/reduced/paid, staff/visitor/employee)
**** Default option: All
*** Status (active/inactive)
**** Default option: All
** Balance Criteria
*** Amount <, >, < >, > <, All
*** Option to exclude $0.00 balances
*** Default option: All
** Options: 
*** Include Primary Contact Details
**** Default option: Deselected
* User is shown accounts that meet the selected criteria.
** Data can be the same as the Classic canned report, including contact information.
*** Primary Contact Info = Primary Guardian Name, Address (Street, City, State, ZIP Code), Email Address, Phone Number
** Negative balances will be parentheses, i.e. Negative $5 would show as *($5.00)* 
** User can sort by site, name - or just name.
** Report can be exported to Excel or PDF

## Acceptance Criteria
Account Balance report added.

Report page has options:

* Site(s) - dropdown should include sitecode + site name
* Grade(s)
* Date - default, current date
* Account
** Student, Adult, Special Account
*** Can choose to include/exclude any, included by default.
** Eligibility/Pricing
*** Free/Reduced/Paid, Staff/Employee/Visitor
** Account Status
*** Active, Inactive
* Balance Criteria
** Can select if they want to search for balances above, below, or between a certain number(s). Or all balances.
*** Can enter an amount, or 2 amounts if they use between. 
** Can choose to exclude $0.00 balances.
* Options:
** Can choose to include Primary Contact Details.
*** This should pull the info for the Primary contact if they exist. If not, leave blank.
** Can choose to sort by Site, or Last Name.
*** Site will go by Site Code, and then Last Name.
*** Last Name will just go last name.

Report added that shows the above info.

Report can be exported out to Excel or PDF.
