# NXT-11022 — Direct Certification - Automation - Household Extensions

## Description
*user story*

As a district admin I want the system to automatically perform DC Household Extensions if enough matching criteria are met.

*mockup*

n/a

*requirements*

This is tied to the following system settings:

!image-20210212-215615.png|width=1196,height=232!

* If the setting DCSBAUTO is set to Yes, then we should check for potential household extensions -after every DC match happens- once a day.
* The second setting is the number of matching criteria. 
** Criteria:
*** Same Household
*** Same Guardian
*** Same Address
** Available options in the setting are 2, 3. At least 2 of the criteria must match.
*** The system setting values will need to be updated as part of this - right now it contains 2, 3, 4.
* The system should perform these checks between the freshly DC matched student and every other student in the database.
* If enough criteria are met, the DC Extension should happen automatically.

*req added 2/25*

* History tracking needs to be added for these automated extensions, like we do already for extensions.
** The processor would be the System in this scenario.
** Matching criteria should be tracked and visible.

## Acceptance Criteria
System uses the settings appropriately to perform DC extensions after every DC match.
System setting values updated from 2,3,4 to 2,3.
Eligibility for the extended student is updated appropriately.
The search for and execution of extensions happens once a day, automatically, if the setting is turned on.
