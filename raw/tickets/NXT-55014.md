# NXT-55014 — RevTrak - Student Balance Export

## Description
As a SchoolCafé Admin I want to be able to export student balances to RevTrak in the format it needs so that I can get our payments back in their system

*Notes:*

* This comes from a customer request to be able to export student balances back to RevTrak the way PrimeroEdge does
* We have to match a predefined template from RevTrak
* It uses pipped delimited files with no header row
* We currently only support comma delimited with header rows
* These requirements are designed to be able to handle these new format requirements
* Note: the immediate need is for Student Balance Export, but these changes will been done generically and so will apply to all export types except Claim export

*Requirements*:

*Add New Export drawer*

* For each export type except Claim Export, do the follwoing:
* For *Add New* screen, do the following:
* *Select tab:*
* When a user selects Student Balance Export:
** Make visible 2 new fields:
*** File Format
**** Single Select drop down list
**** Values:
***** Comma (,)
***** Pipped (|)
***** Tab 
***** Caret (^)
***** Semi-Colon (;)
**** Required if user selects Student Balance Export
**** Default value is Comma
*** Include Headers
**** Toggle
**** Default value is Yes
** -If user selects any thing else besides Student Balance Export, these fields are hidden-
* 
* *Field Mapping tab*
** No changes
* 
* *Review tab*
** On Save:
*** Ensure the 2 new value selections are saved to the configuration
*** -Ensure the 2 new fields are recorded in the History log-



*Edit Drawer*

* On Save:
** Include the 2 new fields in the Edit drawer
** Ensure that save button updates the values
** -Ensure the 2 new fields are recorded in the History log-



*Backend Work*

* Use the Has Headers and File Format field values and dynamically update the ADF to run the export accordingly

## Acceptance Criteria
* User can select select if they want to have headers when doing this export type
* User can select if they want to do a pipped delimited export when doing this export type
* System will generate a pipped delimited txt file with no headers if that is what the user selected
* Changes do no break existing pre-configured exports
* User can still do comma delimited exports with headers
* Edit screen allows user to change these values and they are saved
* Changes applied for all export types except Claims Export
