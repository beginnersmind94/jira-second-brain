# NXT-26113 — Platform - Permissions - Function to Module Implementation

## Description
As a Primeroedge admin, I should be able to add a “Function” resource type to gain access at a module scope

*Mockups:*
[https://www.figma.com/proto/vEncfcQPT3DYGVKOVAagis/System-%26-Platforms-5.0?node-id=169%3A8794&scaling=scale-down-width&page-id=169%3A8142&starting-point-node-id=169%3A8794|https://www.figma.com/proto/vEncfcQPT3DYGVKOVAagis/System-%26-Platforms-5.0?node-id=169%3A8794&scaling=scale-down-width&page-id=169%3A8142&starting-point-node-id=169%3A8794|smart-link] 

*Requirements:*

* New resource catalog to be added with 
** Resource Type*
** Module Name*
** Resource Name*
** Description
* Remove Categories mapping (out of scope for R5)
* Every resource must contain a resource Id in backend with display name separate
* Each resource added must be available under that module in “Roles & Permissions” page

## Acceptance Criteria
* User shall be able to add a resource specific to a module
* User shall be able to select resource type as “function” mapped to module based on the given resource 
* User shall not be able to select category for any given resource
