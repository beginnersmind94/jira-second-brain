# NXT-16396 — INV - Orders - Receipt Detail - Add columns for item level Charges / Discounts

## Description
As an ordering manager I want columns for the contract-driven item-specific charges/discounts, so I can see the math in how each item price is calculated.

**Note:* it is assumed in the effort of this story that the Confirm checkbox logic will be delivered before this story. ([NXT-17417|https://primeroedgenext.atlassian.net/browse/NXT-17417])

*Requirements*

* On the Receipt Detail screen in the item grid, add 3 additional columns for the contract charges/discounts
** Processing Fee (charge)
** Delivery Fee (charge)
** Net Off Invoice (discount)
* Dynamically load the columns
** Only display a column for a charge or discount if at least one item on the receipt has a charge/discount configured
** Example: only one item has a Delivery Fee, so show the Delivery Fee column. No items have a Processing Fee or NOI, so do not display those columns
* Show the charge/discount value per item
** In the top of the cell, show the value of the charge/discount for that item, as it is configured on the contract
** editable based on permission (future, separate story)
*** for this story the field will always be editable
*** this will only change the value on the receipt. it will not update the configured value on the contract
* Show the charge/discount value for the item row (quantity * charge/discount)
** in the bottom of the cell, show the value of the charge/discount multiplied by the quantity of items
* Display negative numbers with a minus sign 
* Display dollar sign ($) for every value

## Acceptance Criteria
* Add columns for item-level contract charges and discount
* Dynamically load the column(s) based on whether or not items have configured values for the charge/discount
* Show both the per-item value and the total value for all quantity
* Allow per-item value to be editable if user has appropriate permission
