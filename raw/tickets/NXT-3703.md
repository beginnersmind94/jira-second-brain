# NXT-3703 — Add Item - Ingredient - Add Nutrient Information - Allergens

## Description
*User Story*

As a Item Management User, when creating an Ingredient, I want to add the Nutrient Information for Allergens

*Mockup*

[https://nqyc9v.axshare.com/start.html#p=itemmanagement_-_ingredient&id=00ehm1|https://nqyc9v.axshare.com/start.html#p=itemmanagement_-_ingredient&id=00ehm1]

*Requirements*

* Header
** Allergens
* Default of None showing up
* Allergens - Checkbox ‘None’
** When unchecked, display standard allergens configured by the district
* Standard Allergen list
** Crustacean Shellfish
** Egg
** Fish
** Gluten
** Milk
** Peanuts
** Soy
** Wheat
* All Allergens have *one* selection (and only one) available and must be as follows:
** Processed in a Facility that also processes
** Contains
** Does Not Contain
* Allow the user to 'Add' an allergen
** List is pulled from the datasource as the item is being created
** If an allergy exists in the datasource, then it would show up in this list. Each datasource will have it’s list of allergens added to it
* Allow the user to create a Custom Allergen
** Have a button at the bottom labelled ‘Create Custom Allergen’
** When clicked, allow the user to enter in the allergen
*** User would then only be able to select from one of the available options and only one
** Click ‘Add’ under custom allergen to add the allergen to the corresponding datasource, and to this item
* If at any time the user selects ‘None’ it clears out the allergen information and it must be selected

## Acceptance Criteria
* Default Allergens as 'None'
* When you uncheck 'None' then it will display the standard Allergen list
* When clicking 'Add' at the bottom, user will be able to type in an allergen. This allergen would then store at the datasource level if they have access to it. For example, they are creating an item on the local datasource, the allergen would be created at the local datasource so it would show up for other users that use this datasource. Example. Strawberry allergy would be added, once added at the local datasource, it shows up for other items 
* When clicking 'Add Custom Allergen' once the user clicks 'Add', then it stores the value in the corresponding datasource. 
** If this is a shared / global item and the user doesn't have access, then put the allergen in the local datasource
* If user selects 'None' then it would not use any allergen information and display as no allergens.
** Store previous results so if the user accidentally clicks ‘None’ and then unchecks it, it can keep the previous results
