# NXT-49535 — MVR - Dashboard - Download

## Description
As a SchoolCafé user, I would like to be able to download a managed view report so that I can see the current report.

*Notes:*

* In CR, we had a preview button that the user would go to the screen and look at it and then be able to download, but this is not working well b/c of data latency
* Since it could take a long time to load a preview grid, we will instead download it and notify the user when it is ready for their viewing
* So we will change the approach to do the download option like we do in Imports
* The output of this download shows the data for Run Now of the MVR

*Flow:*

* Browse to Reports > Managed View Reports
* Click on Download for one of the rows
* Shows popup confirmation
* User says yes and shows Processing message
* System generates an excel file of the MVR
* System notifies the user when the file is ready by sending message to inbox and notification update
* User browses to inbox and clicks on file 

*Requirements:*

* On the Edit button of the grid in Reports > Managed View Reports, do the following:
* Action column updates:
** Label = Download
** If button not there yet, add it
** On click:
** Note: This basically is a Run Now
** First, validate PII
** Check if the MVR Has PII flag is true
*** If true, then check the user’s permissions to see if they have PII permission (from roles and permissions table)
*** If they do have PII permission, then proceed
*** If they do not have PII permission, then do not allow them to view the data, but rather show them a message: You do not have permission to view Personally Identifiable Information (PII). Ask your administrator to grant you permission before continuing.
** Show the user a confirmation screen if they have PII permission
*** Text: This will run the report now and save the data in a file. You will be notified in your Inbox once complete. Do you whish to continue?
*** Show them Yes / No buttons
*** If they say No, do nothing
*** If they say Yes, then proceed
*  
* If confirm, then do the following:
** Close out the confirmation box
** Show a message: Process started. You will be notified when the preview file is ready
** Allow them to close it out or auto close like we normally do
* Then kick off the process to generate the file
** Do what you do to create the MVR for distribution but:
** Instead, generate an excel file (or delimited that opens in Excel)
** Create meaningful file name like you for CR
** Send that file to the user’s workspace inbox (see Inbox section below)
*  
* Sending file to user’s inbox
** Don’t rewrite code to send a file to the inbox, use the way we already do
** Inbox values you will need:
*** From: System account
*** Subject: [MVR name] Download
*** Link to file
*** Body: [note: do the body like for CR emails]

Hi [user first name],

Clink link to view the download file for the [MVR name] managed view report that you requested.

[link]

## Acceptance Criteria
* User can click the button to start the download process
* Use is informed of the process and can confirm
* PII view restrictions are respected for APIs that are marked with it
* File is sent to the user’s inbox
* User is notified when the file is ready to view using Notification Animation
* User can open the file from their inbox

*Technical Details*

This will behave the same way as Distribute Now. The only difference is we need to send the downloaded file directly the user Inbox (we don’t want to consider the Distribution channel for the MVR)

* Add a new bool Property to the CustomReportMessage model (DownloadAndNotify(
* Modify the logic in Notification Scheduler when this property exists and only send the download file to Inbox (for the UserId sent in the above model)
* Remaining everything will work same
