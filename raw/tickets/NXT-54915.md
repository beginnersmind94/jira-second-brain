# NXT-54915 — PR- Carryover- Consider carryover days if produced count is 0

## Description
As a production manager, If carryover an item from one day to another and I do not produce any extra and only use what I have and want to carryover those items again, consider the original carryover days so that I am not constantly restarting the number of days I can carryover.

## Acceptance Criteria
* Consider Carryover days if produced count is 0 
** Currently, if carryover moves from 1 day to another and then is being carried over again, we are starting the max carryover days all over again
* Carryover days need to continue using the original day if no extras are produced
** EX: Carryover was added from Monday ---> Tuesday with a max carryover for 3 days, it should allow the carryover to be carried over to max Thursday. IF on Tuesday, no new servings were produced, when carrying over again still consider the max 3 days from Monday since it is the same items and the system can confirm that information since no new servings were added to the carryovers 
*** Display: Carryover From: 08-01-2024 (Originally from: 7-31-2024)
* If there are multiple carryovers from different days to the same day, use the max carryover day of the *shortest* remaining
** EX: Max days 6
*** Carryover 1 from Previous Friday to Tuesday (4 days) (50 count)
*** Carryover 2 From Monday to Tuesday (1 day) (30 count) now we have total 80 count
*** Now, user wants to carryover remaining (60 count) on Tuesday to any day moving forward, only Wednesday or Thursday should be available since the system will not know which of the carryovers remaining need to be used
