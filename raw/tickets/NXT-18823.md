# NXT-18823 — Production- Reports- Post Production Analysis Criteria Page

## Description
This Analysis will mimic the Weekly Menu Analysis with all the same characteristics but instead of using menu counts, it will use Produced Production counts.

## Acceptance Criteria
All the same characteristics of the Weekly Menu Analysis but this function will use the produced Production counts. 

How to calculate counts since there are no serving groups:

1.) Get the "Produced" count from each menu item on each day
2.) Multiply that number by the nutrient information 
3.) Add each days results together into one number
4.) Divide that result by the total meal count on that day to get the single day average 
 a.) If there are multiple serving groups, add the TOTAL counts of the serving groups together, then determine the percentage of each serving group on each day
 b.) Multiply the results from step 4 by those percentages to get the daily average for each serving group
5.) Take all of the single days averages, add them together, and divide by the total number of days
 a.) Do this for each serving group, if there are multiple

This will only affect Meal Pattern SY 2014+ 

It will have all 3 reports same as the weekly analysis. Nutrition report, Nutrition Summary Report, and Food Component Report. 

Criteria for Page:
Meal Type
Site Group
Site
Menu Line
Start Date- End Date ( Must select between 3-7 serving days for the analysis to work)

When You select Start Date, the end date should automatically populate a date one week ahead 
EX: if user selects 09/01/2021 the end date should display 9/7/2021.

When making selections for Meal type, Site Group and Site, this information should be saved for future searches if user leaves the page or if they go to the pre-production analysis page as well. 

Analysis must be printable same as weekly analysis layout.
