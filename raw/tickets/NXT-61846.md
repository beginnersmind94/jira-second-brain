# NXT-61846 — IM - Item Onboarding - Column headers and data fields for Excel Import/Export

## Description
As a district manager trying to use the onboarding tool, sometimes it would be better for me to work in the excel version as I am more comfortable working through an excel format workflow. When looking at the columns I want to make sure that the information is clear and is translated correctly when imported back into the onboarding tool.

## Acceptance Criteria
* On Initial Info, Item Type (Food/Non Food) should have selections of (Food or Non Food) and is mandatory
** Format the cells to be greyed out that are not needed for Non Food Items (Nutrients, Components, Ingredient Info, Allergens)
* “# of Inventory Units” (Currently, % of Inventory Units) should only appear on the excel if user has enabled the “Pack Size Info” section and must be entered
** Default to None/Decide Later if section is not enabled
** Mandatory to answer if enabled
* “Is this a Direct Menu Item” should only appear if the user has enabled the “Menu Item Info” section and must be entered
** Default to None/Decide later if section is not enabled
** Mandatory to answer if enabled
* Keep Non-Fluid/ Fluid Item column and provide (Non-Fluid/Fluid) drop down and is a mandatory field to select 
** IF *Fluid* is selected, provide Fluid Measure in Measure drop down (*Milliliter, Fluid Ounce, Gallon*)
** IF *Non-Fluid* is selected, provide Weight Measure (*Gram, Ounce, Pound*)
** Remove The conditional grey out formatting for fluid and non fluid
* Format Serving Size Quantity and Serving Size Weight to only be integer and decimal values
* Format Edible Yield Factor to be an integer and decimal values and should never exceed 100
* Format Nutrient values to only be integer and decimal values
* Food Components need to ONLY include 1 field for each component and or variety and just call it the name of the component or variety 
** Columns should just be the name of the component or sub component
** Include Variety field as the last field for that component section with Yes/ No selections (Fruit Juice and Vegetable Juice
*** EX: Fruit, Fruit Juice OR Whole Grain, Non Whole Grain, Grain Based Dessert
*** For Milk, Include a “Milk Variety” column and the drop down values are the variety selection
*** For Grains, No need to have a grain component, provide a column for each sub grain 
*** For Veg sub groups, provide column for each sub group and no need to have a “Vegetable” component column
* Remove Custom Allergens Column (User must use the UI to add custom allergens)
* Add Conditional formatting, if user marks None Indicated as Yes, grey out the allergen columns 
* Remove all Asterisks as we are allowing partial data and verifying on the UI tool
* For Downloading Data from existing session,
** For Non Food Items, none of the “food” Fields should be available for the user, grey out the columns associated
*** Do not prepopulate information that was never entered by the user as the UI does not allow it
*** EX: Non food item being exported with Edible Yield Factor, Locally Grown & Buy American and Components marked as no
* IF a value has not been entered, DO not export with 0 value and leave as blank
** EX: Serving Size Weight not entered on the UI but downloaded as 0
* IF None Indicated for Allergens is selected as Yes, Grey out the selections for allergens entry
** Currently we are displaying No for every item regardless of food/non food
