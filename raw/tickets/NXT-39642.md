# NXT-39642 — IM - MI Consolidation - POS Only Item - Summary

## Description
As a menu planner I want the ability to create a POS only item so I can simply configure buttons to appear on the POS terminal without creating an entire food item or menu item.

*Create POS Item - Summary* [https://www.figma.com/proto/eDsxhWPpQikc74wptrgntb/Item-Management-6.4?page-id=769%3A44444&type=design&node-id=913-53110&viewport=1619%2C460%2C0.24&scaling=min-zoom&starting-point-node-id=913%3A53110|https://www.figma.com/proto/eDsxhWPpQikc74wptrgntb/Item-Management-6.4?page-id=769%3A44444&type=design&node-id=913-53110&viewport=1619%2C460%2C0.24&scaling=min-zoom&starting-point-node-id=913%3A53110|smart-link]

## Acceptance Criteria
* Add an option to the New menu item button titled “POS Only”
** these are POS items which will never be connected to any other item type 
* Create a special details page for these POS Only items
* Display a Summary card similar to other IM details pages
** Capture existing fields
*** POS Button Name (English & Spanish)
**** in view mode, display together
**** in edit mode, display separately
*** POS Item Category
**** this is just Menu Item Category with an alias
**** this will filter according to the POS categories, per the [merged configuration story|https://primeroedgenext.atlassian.net/browse/NXT-30763]
*** Meal Type
*** 'Is a Meal' flag (this field only exists on POS only items)
