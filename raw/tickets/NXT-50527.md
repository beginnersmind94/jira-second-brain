# NXT-50527 — PR- Update Flags for Consolidated and Distribution complete

## Description
As a production manager, when doing production orders or withdrawals as a prep site, I need the information to pull over to the respective functionality based on whether I have completed my consolidated plan or prep site distribution.

## Acceptance Criteria
* Capture a flag that tells the system when a consolidated plan is completed
** This happens once consolidated plan is saved and consolidated plan flag as true
** If deleted, mark consolidated plan flag false 
* Capture a flag that tells the system when a prep site distribution is completed
** This happens once all menu items have the *status: Shipped* and counts can no longer be changed and mark flag as true
** Any status before Shipped for all menu items the flag will remain false
* Update current job to include checking if all status is shipped to on the fly change the distribution flag as true
* Shipped planned counts toggle should not disabled unless all items are marked as shipped
** IF 1 item or more is marked as shipped and others are not, allow user to use ship planned counts but do not disturb the counts that are already shipped
* Disable save button if all items are shipped
* Validate strict batch rules when shipping, shipping total counts should fall within the strict batch validation
