# NXT-70508 — Add endpoint to fetch Custom Reports by Region ID

## Description
*As a* consumer of the Custom Reports API, *I want* an endpoint that returns all custom reports filtered by a given region ID, *So that* I can retrieve region-specific reports without fetching the entire dataset.

h3. Background

The existing {{CustomReportController}} already provides CRUD operations for custom reports using the established manager and repository layers. We need to extend it with a new GET endpoint that accepts a {{regionId}} path parameter and returns all matching custom reports.



h3. Acceptance Criteria

* A new {{GET /api/custom-reports/region/{regionId}}} endpoint is added to {{CustomReportController}}
* The endpoint delegates to the manager/service layer, which queries the repository
* A new repository method {{findByRegionId(Long regionId)}} is implemented
* Returns {{200 OK}} with a list of custom reports (empty list if none found)
* Returns {{400 Bad Request}} if {{regionId}} is invalid
* Unit tests cover controller, service, and repository layers

h3. Technical Notes

* Follow existing patterns in {{CustomReportController}}
* Add {{findByRegionId}} to the repository interface
* Add corresponding method in the manager/service layer
