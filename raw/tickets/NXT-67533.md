# NXT-67533 — Story 2 — Store & Maintain State Templates (Admin)

## Description
*As a* system administrator
*I want to* store state-defined account code templates
*So that* districts can copy them as a starting point for their account code structure

h3. Background

Each state has its own required account code structure for school nutrition reporting. We need to store these templates as reference data so districts can select and copy them into their own configuration (Story 3).

## Acceptance Criteria
h3. Template Storage:

* System stores one account code template per US state (identified by 2-letter state code)
* Each template contains three separate account structures:
** Revenue account structure
** Expense account structure
** Fund balance account structure
* For initial launch, all three structures will be identical within each state template
* The three-structure design supports future state requirements where revenue/expense/fund balance may differ

h3. Account Structure Definition:

Each account structure (revenue, expense, or fund balance) is defined by a list of segments. Each segment has:

* *Description* - Label for the segment (e.g., "Fund", "Function", "Object")
** Must be unique within that account structure
** Can repeat across the three structures (revenue can have "Fund" and expenses can also have "Fund")
* *Type* - Either "Numeric" or "Alphanumeric"
* *Segment Length* - How many characters this segment contains (must be at least 1)
* *Separator* - Character that appears after this segment (e.g., "-" or "."), or none if it's the last segment
* *Position* - Order of the segment (1st, 2nd, 3rd, etc.)
** Must start at 1 and increase sequentially with no gaps

h3. Business Rules:

* Each account structure must have at least one segment
* No limit on maximum number of segments
* Position numbers must be sequential (1, 2, 3...) with no skipped numbers
* Each state can only have one template

h3. Generic Templates:

* Some templates can be marked as "generic" for states where we don't have state-specific requirements
* When a district selects a generic template, they see this warning: _"We don't have a state-specific template on file. This is a commonly-used structure, but please verify it matches your finance department's requirements."_

h3. Template Usage:

* When districts set up their accounts (Story 3), they copy one of these state templates as their starting point
* Once copied, districts own their configuration—future updates to state templates don't affect them
* Districts always get the current version of the template when they copy it
