# NXT-41919 — DataExchange Import – Fixed Width – Field Mapping Update

## Description
As a SchoolCafé Admin I want to import a fixed width file so that I can use my state’s existing format.

*Flow:*

* User logs in and browses to System > Imports & Exports > Imports
* User clicks + New Import button
* User selects DC Import on Pre-Upload screen
* User selects the Configuration Type they want to use
* User clicks the Next button
* User uploads the fixed width file template in the File Uploads screen and clicks Next
* Field Mapping tab loads the columns from the fixed width format template and hides the + New and Delete buttons
* User completes the import process as normal

*Notes:*

* -Note that since it is fixed width, we do not want to allow the user to add or delete or modify the File Fields table-

*Requirements*:

* For the Field Mapping tab when click + New or click the edit icon, do the following:
* If the user selected Fixed Width as file format from the Pre-Upload screen:
** Make all the changes below
** Else, leave everything alone
* 
* On page load, load the fields from the format file template as the field names in the File Fields
** Note that this is different as you are not reading the column headers from the file but reading the values from a list in the file
* 
* Do not show the + New or delete for fixed width imports
* 
* The rest of the page usage will remain the same

## Acceptance Criteria
* Page loads the name of the fix width fields from the fixed width file format selected
* + New and delete buttons are hidden if it is a fixed width file
