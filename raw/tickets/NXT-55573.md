# NXT-55573 — FH - Accounts - Delete Account option

## Description
As Andy from Apple App Reviews I’m not going to approve the Family Hub app until a delete account option is added, so you better add one.

*mockup*

n/a

*requirements*

* Update the text on the ‘Deactivate My Account’ button
** Text change: ‘Delete Account’
* User needs to confirm they want to delete the account
* If user confirms:
** We can treat this as a soft-delete, but we need to free up the email address for new account usage.
*** Right now the ‘deactivate’ option lets the user restore their account. 
*** We will block the restore functionality, but keep the remaining functionality for deactivating an account.
*** This will likely mean we have to change the email address so that the user can create a new account.
* Language translations need to be added for the confirmation text and the button if possible.

## Acceptance Criteria
* Deactivate updated to Delete, so user can delete their account.
* User is given a confirmation prompt and a soft test (type DELETE in this box) to avoid misclick scenarios.
* Language translation should be used for the prompt and associated text.
* Update the deactivation workflow so this acts as a soft-delete:
** CC details removed.
** No app notifications, no emails.
** Unlink Google/Apple accounts.
** Email address is renamed and freed up for new account creation.
** Restore option does not work (user can’t just log back in with the same credentials)
** Confirmation email of account deletion should be sent to the user before the email address is removed/scrambled.
