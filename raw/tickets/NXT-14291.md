# NXT-14291 — Import and Export Dashboard-Ability to make a configuration active and inactive

## Description
Currently, we don't have a way to make a configuration active or inactive.
Need a way to provide this functionality. In order to achieve this functionality will add a label by the name IsActive and a toggle below the Configuration Name textbox on the Review step. This value will be persisted in the DB table. On  Import and Export dashboard, will show the Import or Export button for only active configuration. For all the inactive configurations the button will be invisible.
Only one configuration should be active for a given import. e.g. If there is only one configuration that is active and then this configuration should not be allowed to be made inactive.
For DC Import each Configuration Type should be considered as an individual import.
So for example, DC Import-DC SNAP there should be only one configuration active and likewise the same all other Configuration Type.

## Acceptance Criteria
* All the active configurations take priority and must be displayed first, all inactive configurations must be at the bottom. (Further refinement on this is needed to include filter condition if possible).
* In Import file set up, 
** For Import Type  as Student Import,
*** Only One configuration must be active at any given time.
*** Incase if there is already a configuration which is active already, the new configuration when set up as active will turn the other existing active configurations into inactive mode.
* In Import file set up, 
** For Import Type  as DC Import,
*** There can be several configurations in active under DC import, however each sub category is limited to one.
**** Only One sub-category configuration cab be active at any given time.
***** Incase if there is already a sub-category configuration which is active already, the new configuration when set up as active will turn the other existing active configurations into inactive mode.
* In Export file set up, 
** For Export Type  as Student Eligibility,
*** Only One configuration must be active at any given time.
*** Incase if there is already a configuration which is active already, the new configuration when set up as active will turn the other existing active configurations into inactive mode.
* In case of of Import File set up, Import functionality must be allowed only for Active configurations. All the inactive configurations will have download functionality and delete option disabled (This needs to be refined further to deprecate the download functionality incase of inactive).
* In case of of Export File set up, Export and Download functionalities must be allowed only for Active configurations. All the inactive configurations will have download functionality and delete option disabled (This needs to be refined further to deprecate the download functionality incase of inactive).
* Tool Tips Messages must be activated.
* Data posted (Direct upload) using The HTTP(S) URL must be validated.
