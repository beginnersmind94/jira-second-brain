# NXT-53433 — DMF - Trigger Framework SQL Scripts and individual DBs migration

## Description
!image-20240731-160822.png|width=1226,height=518,alt="image-20240731-160822.png"!

!image-20240731-161423.png|width=1453,height=275,alt="image-20240731-161423.png"!

In order, to re-run the Dat Migration Framework SQL Scripts Source and Destination, call the below API with the required MigrationId and RegionName

!image-20240723-224209.png|width=513,height=360,alt="image-20240723-224209.png"!

In order, to start the Source migration, call the below API and with the required MigrationId and RegionName
For System DB use migrationRunOrderId=5
ItemRecipe Management DB use migrationRunOrderId=7
Eligibility DB use migrationRunOrderId=9

!image-20240723-224414.png|width=524,height=367,alt="image-20240723-224414.png"!

In order, to start the Destination migration, call the below API and with the required MigrationId and RegionName
For System DB use migrationRunOrderId=6
ItemRecipe Management DB use migrationRunOrderId=8
Eligibility DB use migrationRunOrderId=10

!image-20240723-225354.png|width=539,height=378,alt="image-20240723-225354.png"!

Data Migration Framework SQL Scripts for Source and Destination will start when the new migration is started.
System DB Source and others should not start until the Data Migration Framework SQL Scripts for Source and Destination is completed successfully.
Once the Data Migration Framework SQL Scripts for Source and Destination is completed successfully, then only the ‘Not Started’ button on System DB Source should be enabled and the user can start it.
The ‘Not Started’ button on the System DB Destination should be disabled when the System DB Source migration is running and enabled only when the System DB Source is completed successfully.
The same steps need to be followed for all ItemRecipeManagement and Eligibility DB.
