# NXT-34146 — Team Hub - Add Authentication Type Prompt to Login Screen

## Description
As a school admin, I want to set the platform setting so that the system can correctly authenticate.

 

*Flow:*

·         When logging in, if user hasn’t already set product type, prompt them to select what platform they are on

·         Result is saved to Settings > Product value and used for authentication type

 

*Assumptions:*

·         Login screen already exists per previous development and is for Classic only

 

*Background:*

·         Login screen already exists

·         In this story, we are adding new functionality that will allow the user to select which product they have, PrimeroEdge or SchoolCafé and based upon their choice, we will determine the type of authentication to use

 

*Requirements*:

·         When logging in, App checks if they have previously selected what platform they are using

·         If it has, then skip and do nothing further

·         If it hasn’t, do the following:

o   Show a popup screen with the following:

o   Message: Please select what platform you are on:

o   List box

§  Name: Platform

§  Values: PrimeroEdge, SchoolCafé

o   Submit Button

§  Upon click, save the value to Settings > Product

§  Close the popup screen and allow the user to continue

·         Use the platform value behind the scenes to determine what type of authentication to use when logging in as Classic and 2.0 have different types

·          

·         Security – already covered in previous stories

## Acceptance Criteria
·         If user hasn’t selected their platform, the screen forces them to select it before proceeding to login

·         2.0 district users can properly authenticate
