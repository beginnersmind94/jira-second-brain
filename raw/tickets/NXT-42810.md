# NXT-42810 — SIS Auto Sync – Guardians – Execution & Exceptions

## Description
As a SchoolCafé admin, I would like to import multiple guardians so that I can have that information on the student record when I need it

*Requirements*:

* When running the Guardians SIS Import, do the following:
* Skip rows that have exceptions for mandatory fields that can’t be repaired
* Skip/blank fields on rows if the field can’t be repaired
* 
* Mandatory fields:
** First Name*
** Last Name*
** Street Address*
** City*
** Zip*
** These fields are text fields
** Validate their length, if too long, trim it to the max length, and allow to import
** If missing, then skip the row
** 
** State*
** Relationship*
** These fields have list values
** Validate that the value is acceptable to our database, if not, skip the row
* 
* Optional fields:
** Middle Name
** Apt./Suite
** Validate their length, if too long, trim it to the max length, and allow to import
** 
** Email Address
** Alternate Email Address
** Validate their length, if too long, trim it to the max length, and allow to import
** Validate they are well formed email addresses like Student Import, if not, blank it out and allow the row to import
** 
** Cell Phone
** Home Phone
** Work Phone
** Strip out non numbers like on Student Import, then
** Validate their length, strip out anything more than 10, and allow to import
** 
** Prefix
** Suffix
** Preferred Communication Mode
** These are list values
** Validate against our database to ensure the values are acceptable; if not, blank them out and allow the row to import
*      
* Log exceptions like you do for the other imports
* Log the history like you do for the other imports
* Log the source data like you do for the other imports
* Execution steps are logged

## Acceptance Criteria
* Skips rows that have exceptions for mandatory fields that can’t be repaired
* Skips/blanks fields on rows if the field can’t be repaired
* Exceptions are logged
* History is logged
* Source Data is logged
* Execution steps are logged
