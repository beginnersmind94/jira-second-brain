# NXT-33416 — Acct Mgmt - Notifications - Add filters to view latest PNA students only, consolidate checkboxes

## Description
As a central office admin, I would like to be able to easily notify a list of students who are ‘Paid’ status but have ‘Not Applied’ for free or reduced benefits, so that I can contact them directly encouraging them to apply.

*Summary*

Add a new drop-down to the ‘Notifications’ page which allows users to immediately view only those accounts which are ‘Paid Not Applied’ (These are Paid students who have not submitted an application for the current academic year), based on the Insights data. Additionally, move the existing 2 checkboxes into a separate drop-down menu.

*Details*

* In Account Management → Notifications, hide or remove the existing checkboxes for '{{Show only eligible "Carryover" students}}' and '{{Show only eligible "Default" students}}'
* Replace them by adding a new drop-down to the Filters: *Basis* 
** Type: Multi-selection
** Options:
*** *All* 
**** Default value
*** *Carryover*
**** This is the same as the existing checkbox filter on the page
*** *Default*
**** This is the same as the existing checkbox filter on the page
* When ‘Default’ and/or ‘Carryover’ are selected and the user clicks Apply, continue to only show those students who are Default or Carryover in the grid according to the site(s) selected
* Add another new drop-down menu to the Filters: *Insights*
** Type: Single-selection
** Options:
*** *All* 
**** Default value
*** *Paid Not Applied*
**** We can pull this list directly from the Insights module
* When ‘All’ is selected, continue to pull up all students in the site(s) selected when the user clicks Apply
* If ‘Paid Not Applied’ is selected, do the following:
** Hide and ignore the Basis field
** When the user clicks Apply, get the list of PNA students from Insights for the site(s) selected and populate the grid

* Verify that the existing 2 checkboxes for showing Default and Carryover students have been consolidated into a new ‘Basis’ drop-down menu, with their behavior still mimicking the current page
