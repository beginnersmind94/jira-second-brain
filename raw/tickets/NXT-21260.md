# NXT-21260 — Decouple Publishing from Model Building

## Description
We need to update the build model code to stop publishing the model immediately and create a Model Publisher that is disconnected from building models

*Application*: Face and Food Detection Website

*Flow:*

* No user flow for this one

 

*Background:*

* Originally, when a model is built, it is then immediately sent out (published) to the sites
* This creates an unwanted dependency to send code out to a site, in that it forces you to build a model first
* We want to decouple building models and publishing models
** One set of code for each, independent of each other, with each doing its own set of functions
* Therefore, we want to create a Model Publisher object that will manage sending models to sites and removing models from sites and other publishing related things

 

*Assumptions:*

* All the Model Publishing code already exists
* In this story, we are not asking you to create new functionality, but rather take the existing functionality and encapsulate it in its own Model Publisher object
* To do this, it is assumed that you will need some kind of asynchronous code that either runs in the background or can be triggered / called by code to publish
** You are going to need both in the future

 

*Requirements:*

* Find the code that selects which sites to send a model to
* You will find it from the Create button on the Build Model page
* Update that code to look at the Model Sites table where sites are linked to a Model
* Get the devices for those sites and send the model to those sites
* Update the Start Date of the Model Sites table with the date/time stamp so that we know when it was sent to the site
* Remove the code to blast out to all registered sites
*  
* Security – already covered

* Model sent to selected sites from Build Model page
* Model is not sent to any other sites other than those selected from Build Model Page
