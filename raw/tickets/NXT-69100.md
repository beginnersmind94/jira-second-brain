# NXT-69100 — Fin - User-Created Data Points

## Description
Users can manually create additional GL accounts for tracking activity not covered by the auto-generated accounts.

## Acceptance Criteria
* Opens when clicking "Add Data Point" button
* Drawer slides in from right side
* Header displays: "Add Data Point" with X close button
* Description text: "Create a custom data point for manual journal entries. This won't affect automatic postings from POS or inventory."

Form Fields:

Data Point Name:

* Label: "Data Point Name" with red asterisk (required)
* Input placeholder: "Enter a unique name..."
* Validates that name is unique across all existing data points
* Error message for empty: "Please enter a name"
* Error message for duplicate: "A data point with this name already exists"

Base Type:

* Label: "Base Type" with red asterisk (required)
* Dropdown with options: Select a type..., Asset, Liability, Equity, Revenue, Expense

GL Account dropdown for that base type and required field before saving

Buttons:

* "Cancel" button (ghost style) - closes drawer without creating
* "Create Data Point" button (primary style) - validates and creates

On Creation:

* New data point added to "Other [Type]s" group (e.g., "Other Expenses", "Other Assets")
* "Other" group only appears if user-created accounts exist for that type (except for “Other Revenue” which always exists due to data points from the system)
* Created By value set to "User"
* Drawer closes
* "Other" group auto-expands
* Page scrolls to new data point
* New row briefly highlighted with purple background
* GL Account shows as unmapped (needs mapping)

Edit/Delete button:

* Provide an edit and delete button
