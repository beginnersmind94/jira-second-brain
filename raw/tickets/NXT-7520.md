# NXT-7520 — Applications - Processing Mode - Statistics

## Description
*User Story* 

As an application processor I want to see how many applications are left to process and how many I’ve done so far.

*Mock Ups* 

*Requirements* 

* At the top of the continuous processing mode, show the user some high level statistics so they get a sense of progress.
** How many unprocessed applications remain
** How many they’ve processed in this session
** Who else is processing applications right now (firstname + lastname + current application # )
* These statistics should update in real time if possible.



Discussion 3/2/21

* The Processed Applications number should be a count of how many apps that particular user has processed for that particular day.
** This includes Manual applications - any ‘processed’ status update for an application.

## Acceptance Criteria
User can see how many applications are still pending processing.
User can see how many they've processed this session.

User can see who else is processing apps and what app # they are on.

Stats update in real time (nice to have)
