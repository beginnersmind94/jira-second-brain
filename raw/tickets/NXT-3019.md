# NXT-3019 — System - Configuration - Periods - Add Periods page and Fiscal Configuration section

## Description
As a a district financials administrator, I would like to be able to specify my district's fiscal year and view or manage periods within that year, so that I can view reports and claims for the right date ranges.

Additional Information:

* Periods are needed to determine financial and operational periods of times to be used in the system. It helps to prevent certain usability and “lock” data/information (ex - POS Claims, Postings, etc).
* This page has integration with POS, Inventory, Financials, and Sales Invoice.

Summary:

* Add the Generate Periods page and basic layout for setting and saving a Fiscal Year in a ‘Fiscal Year Configuration’ section.

Details:

* In the Management category of System, add a *Generate Periods* page
** Columns/behavior for this page will mimic Classic
* For districts where Fiscal Year dates have not yet been set:
** Upon landing on the page, allow the user to choose a Fiscal Year. 
*** Current academic year (current year + next year) should be selected by default. 
*** Drop-down should include last 10 academic years.
** -Display the Start and End Dates and allow the user to change them.- {color:#ff5630}This is being handled in NXT-5515{color}
*** -Default Start Date should be 7/1/XXXX and default End Date should be 6/30/YYYY, where XXXX is the current year and YYYY is next year.-
*** -Start Date should also be greater than the End Date of the previous Fiscal Year. No other restrictions should apply to dates.-
*** -Dates must be future (starting from Today) and Start Date must be before End Date.-
** Allow the user to save the Fiscal Year.
* When Fiscal Year dates have already been set: When the Fiscal Year is changed (or the default current year is chosen), display the Start and End Dates that were already set.
** Start Date and End Date fields should be disabled until the user chooses a different year.
** *Fiscal Year cannot be modified once set*.

## Acceptance Criteria
Verify that Generate Periods page is added to the Management Category of System

Verify that upon landing on the page, the Fiscal Year Configuration section is displaying

Verify that current academic year is displaying by default (current year + next year)

Verify that for districts where Fiscal Year dates have not yet been set, upon landing on the page, we allow the user to choose a Fiscal Year. 

Verify that fiscal year drop-down includes at least last 10 academic years.

Verify that default Start Date should be 7/1/XXXX and default End Date should be 6/30/YYYY, where XXXX is the current year and YYYY is next year.

Verify that when the Fiscal Year is changed (or the default current year is chosen and dates are already set), we display the Start and End Dates that were already set.

Verify that when dates for the selected fiscal year have been saved, the related Start Date and End Date fields should be disabled until the user chooses a different year.

Verify that Fiscal Year cannot be modified once set.
