# NXT-25413 — Tag component Enhancements

## Description
As a system when I create a tag that already exist, I should get an validation message from api informing me that "Tag already exist". Also, regionId should be extracted from the user's  claims rather than being expected in the route.

## Acceptance Criteria
This will be done when:
# When I am not required to send the region id in the request but rather than that its extracted from the token.
# When I get the validation message from the backend if I try to create a tag that already exists.
