# NXT-69488 — Fin - Create Custom Data Points with Business Rules

## Description
*As a* financial administrator using the Account Mapping interface,
*I want to* create custom data points with associated business rules through an "Add Data Point" action,
*So that* I can define district-specific financial entries that aren't covered by standard system data points and ensure they map correctly to our GL accounts.

## Acceptance Criteria
h3. 1. Post-Creation Success Message

* Upon successful data point creation, a toast notification appears at the bottom of the screen
* *Exact message*: "[Data Point Name] created. Use *Edit* for GL mapping, then *Rules* for business rules."
* Toast auto-dismisses after 5 seconds
* -The newly created row is highlighted briefly (green tint) to help user locate it-

h3. 2. Accessing Business Rules (Rules Button)

* User-created data points display a *Rules* button in the row actions (document icon)
* Clicking the Rules button opens the Rules side drawer
* The drawer title shows "Data Point Rules" with the data point name as subtitle

h3. 3. Rules Drawer for User-Created Data Points

When opening the Rules drawer for a user-created data point:

* Shows "User-defined journal entry" as the description
* Displays the current rule configuration (if any):
* Rule Name
* Base Type
* Entry Type (Manual)
* Journal entry template showing Debit and Credit accounts
* If no rules are configured yet, displays placeholder text indicating rules need to be set up
* The data point must be involved in the rule 

h3. 4. Editing Business Rules (via Edit Drawer)

For user-created data points, the Edit drawer includes a "Journal Entry Template" section allowing:

* *Rule Name* (required) - Name for the journal entry rule
* *Debit Account* selection (required) - Dropdown of all data points
* *Credit Account* selection (required) - Dropdown of all data points

Users can modify these fields at any time (unlike system-created data points which have fixed rules).

h3. 5. Delete Capability

* -User-created data points display a- *-Delete-* -button (trash icon) in the row actions-
* -Delete is also available in the Edit drawer footer-
* -Clicking Delete stages the data point for deletion (row shows strikethrough with "Undo" button)-
* -Deletion is finalized when user clicks "Save Changes" in the floating save bar-
* *-Note-*-: Deletion removes the routing configuration only; historical GL entries remain unaffected-
* *-Check if that data point is involved in any rule. If so, block it and inform the user and add a warning icon next to those rules.-* 

h3. 6. Validation

* Rule Name is required when configuring journal entry templates
* Rule Name must be unique across all data points
* Debit and Credit accounts are required when saving rule configuration
* The data point in which the rule is created is involved at least once on either debit and credit

The system must allow multiple journal lines with the same GL account regardless of side; no auto-merge, no blocking; validation at journal level (debits = credits) and account level (active + mapped) only; when 2+ lines resolve to the same account, display once per session: "X lines reference the same account [*account  name.*]." This is:

* Informational only — does not prevent save or post.
* Persistent: stays visible for the duration of the editing session,cannot be dismissed by the user. 
* Auto-removes only if the user edits the lines such that the duplicate condition no longer exists.
* Re-appears if the user later re-creates the condition.
