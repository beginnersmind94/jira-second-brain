# NXT-11086 — Eligibility - Applications - Pending Students - Actions

## Description
*User Story*

As an applications processor I want the system to help me find good matches for pending students on applications, matching them against students in the database.

*Mockup*

[https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=424768%3A861&scaling=min-zoom|https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=424768%3A861&scaling=min-zoom|smart-link] 

*Requirements*

* *Actions*
** Find Matches
*** For all the pending students in the list - attempt to find a student in the student database to match this against.
*** Use the same matching criteria as we do for identifying a student.
*** If a match is found: 
**** Show the information on the potential match.
***** ID number, name, etc. Same information that we’d be showing for the Potential student so the user can do a visual confirmation.
**** Show a visible indicator if a single match has been found or selected. (A green check mark?)
**** Show a caution indicator if multiple matches have been found.
** Process Matches
*** User has a checkbox on any row with a potential student match.
**** User also has a checkbox for the column header that will serve to select every potential student match regardless of pagination.
*** The Process Match button will execute any potential matches that are selected.
**** Link the student to the application, process eligibility appropriately.
**** Add a comment into the application’s Comments for the successful identification and the benefits being extended.
***** Pending student successfully linked to Student #ID (FirstName LastName).
***** Comment should be appropriately timestamped and have the user who actioned this.
*** User is given a success message and told how many students were processed.
**** Text: Processing complete for selected pending students. (5 students)
*** Confirmation Prompt
**** User should be given a confirmation prompt when they try to initiate this action.
**** Text: Process the selected pending students?

* Actions for individual pending students - 
** Search
*** This is the look-up feature in Classic.
*** User can search the student database by site, grade, last name, first name, ID #, date of birth, active/inactive status.
*** User can select a student from the results.
*** Options: Cancel, Match Student
**** Cancel
**** Match Student
***** This should identify the pending student as the selected student, and process the match / extend benefits appropriately.
** Edit
*** This should allow the user to edit the information about the pending student via a side drawer.
*** Editable fields:
**** Student ID
**** Last Name
**** First Name
**** Grade
**** Site
**** Date of Birth
*** Update / Cancel options
!image-20210216-180147.png|width=1036,height=777!

* Compare
** If a match is found (or selected by the user) then show a Compare icon that lets the user review the information from the application vs student details.
*** Student ID
*** First Name
*** Last Name
*** Site Code
*** Grade
*** Date of Birth

## Acceptance Criteria
User can see a list of unidentified pending students
User can see a list of potential match students
User can see a list of identified students
User can search for enrolled student record for association and link/match
User can view/edit the details of pending student information to assist with better potential matching opportunities
Multi-language, ADA compliant.
