# NXT-9781 — Data Exchange - File Import Setup: Review Step

## Description
As a PrimeroEdge user, I would like to have an intuitive screen that allows me to verify all of the fields in PrimeroEdge are rightly associated with the fields in my first uploaded file, so that I can confidently save the import configurations for an data import later.

*Mockup:*

[https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=5269%3A82698&scaling=scale-down-width|https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=5269%3A82698&scaling=scale-down-width|smart-link] 

*Requirements:*

* When the user has clicked ‘Next’ from the Field Mapping step, redirect them to a *Review* step.
* On this step, display a preview that shows a table that contains:
** File Fields (fields from uploaded file)
*** Ideally, we should display the header values as the fields that they will mapped to in PrimeroEdge. i.e. if there is a column on the config file called ‘DOB’ and the user has mapped it to ‘Date of Birth’ in PrimeroEdge, display the header for that column as ‘Date of Birth’ in the preview.
** Mapped Fields
*** PrimeroEdge fields displayed in GREEN inline with file fields
* Display Review table with 10 rows with vertical scroll bar for user to navigate and review
* Display *Current File Name* labelled field with default name with a *EDIT* & *TOOLTIP* icon
** File Name format should be <Districts selected>_<Import Type>_Config_1
** This file name should be editable by user (Inline edit - no pop-up)
** System should *AUTO-SAVE* the file name edits by the user
** "Default Name - Edit to update the File Import Setup Name" is the tooltip text
* Display “*EDIT MAPPING*” button for user to navigate back to mapping screen
** This action should retain file name edits by user
** 
** 
** +Save Configuration+ 
**# Save the configuration to the database
**# Display a pop-up with a message explaining that the configuration has been saved.
**## *Title*: Success\!
**## *Message*: The file configuration has been saved successfully.
**# Display an ‘OK’ button that, when selected, redirects the user back to the Pre-Upload step of the page.
* *SAVE* option is displayed for user to save the File Import Setup configuration based on the file name.

## Acceptance Criteria
* Verify that when the user has clicked ‘Next’ on the Field Mapping step, we redirect them to a Review step.

* Verify that on this step, we display a REVIEW that shows a table that contains:
1. The header columns of the file are displayed under "File Fields" column
2. PrimeroEdge fields that are mapped by the user are displayed under "Mapped Fields" column

* Verify that 10 rows are displayed with file fields and mapped fields aligns on each row with a scroll bar enabled for user to nagivate & review

* Verify that beneath the table, we display 2 buttons:
1. EDIT MAPPING - Allow the user to go back to the Field Mapping step.
2. SAVE - Save the configuration to the database with the user defined or default file name

* Verify Current File Name field is pre-populated with a default name (specified format requirements above)

*Verify Current File Name field is editable and AUO-SAVE upon user edits

* Verify TOOLTIP is displayed for current file name labelled field

* Verify EDIT MAPPING button is enabled for user to navigate back which retains changes on this screen


* Verify that SAVE button is enabled
