# NXT-63473 — API Change - Fetch Missing Information (View API / Import API)

## Description
_(none)_

## Acceptance Criteria
Navigation: GC Portal - Search

Changes Required:

  

*Catalog – Matching*

The matching should go based on the following sequence:

User Selects Provider Dropdown – Cybersoft

User paste the GTIN in comma separate format

 

*Search for records in USDA Branded Foods*

Use the branded foods table and match the GTINs based on last 10 digit of each GTIN pasted

Call the USDA API or Fetch info from DB for such GTINs and insert the response in Staging DB (call it as – USDA Matched)

Consume all the available fields such as Item Basic Details and Serving Size and Nutrient Information

 *Search for records in CNDB*

Use the branded foods table and match the GTINs based on last 10 digit of each GTIN pasted

Call the USDA API or Fetch info from DB for such GTINs and insert the response in Staging DB (call it as – USDA Matched)

Consume all the available fields such as Item Basic Details and Serving Size and Nutrient Information

*Search for records in DataMart*

For the unmatched records:

Use Data mart for matching the last 10 digits of GTIN (add an additional field (as search key) that’s based on Product Code||GTIN||Purchase GTIN||Broken Unit GTIN||Vendor Product Code)

The system should match the records in unique data set – if not found, then it should looks at rest of the records and fetch the data

 

Item Import Batch Creation:

The system should create import batch with additional data identified based on GDSN Format

Fetching of Additional Data: Such as Images, Allergen and Pack Size will derived from other external data source tables
