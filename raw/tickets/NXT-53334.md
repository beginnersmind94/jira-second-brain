# NXT-53334 — VND - Settings - Sell meals as ala carte

## Description
As Dana the Director I want to allow students to authenticate themselves at the vending machine with different methods to fit our district’s needs.

*Requirements*

* Add a new System Setting:
** Setting Name: Sell Meals as A La Carte on Non-Operating Days
** Setting Code: VSELLALANO
** Module: Accountability
** Category: Vending
** Setting Description: Controls whether Reimbursable Meals are sold at A La Carte pricing on weekends and other non-operating days.
** Values: Yes, No
* Setting VSELLALANO needs to be used as part of our response to the /selection endpoint.
** If No, there’s no change.
** If Yes → 
*** Check to see if it’s an operating day for the district.
**** Weekends are not operating days.
**** May need to check the Holiday calendar as well.
*** If it’s not an operating day, send the “NewPrice” back with ala carte pricing (2nd meal for students)
* We will need to handle this for transactions as well
** It will not be a reimbursable meal
* 
* ---------------------If operating days False------------------------------------------------------------------
and Use operating days system setting is off
Record Reimbursable meals as A la Carte on weekends
--------------------If operating days True------------------------------------------------------------------
and if Use operating days system setting is on
Record Reimbursable meals as A la Carte on configured Non-Operating Days
** If it is > MAXVEND value - “Approved” sent back is false.

## Acceptance Criteria
* System Setting AUTHMETH checked as part of the /customer endpoint.

* Student ID/PIN → “UserCredentials” checks against the Student ID #. “PINNumber” sent back is the PIN.
* Student ID/Birthday → “UserCredentials” checks against the Student ID #. “PINNumber” sent back is the DOB.
* PIN/Birthday → “UserCredentials” checks against the account PIN #. “PINNumber” sent back is the DOB.
