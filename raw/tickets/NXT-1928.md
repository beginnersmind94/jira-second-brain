# NXT-1928 — Menu Planning - Publish Menus

## Description
This is for the process of publishing to external applications, like: SchoolCafé Menus and SchoolCafé TV. ExpressPoint pulls from real-time connection to PRs. 

NOTE: Could be a Production function, as well - many districts choose to publish their menus based on production plan items that have a count other than zero, thus allowing the exclusion of menu items that the site is not inteding to serve. Particularly in instances of “Managers Choice” items, or when the nutritionist has assigned multiple fruits or entrees with an instruction of “select one to serve” to the site. 

Explore opportunities to have this as a specific action AND/OR automated (scheduled, etc).

If a PR has a substitution, ask the user if they want to “re-publish” the changes/substitutions that have been made.

Has a dependency on the assignment of the menu to sites.

Note: Need to have ability to set order for menu items that get published to SchoolCafe - on a site by site basis…

## Acceptance Criteria
Publish all items to from Assign Menu data per site

Give history for users to see a history of Publishing

Who published, what days did they publish, how many menus, how many menu items and how many details did they publish

This should send all information of a menu item to Family Hub

ALL menu information should be sent, date, site, meal type, menu line.

Allow auto publish for any new assigned menus and display any new publishing in the history.

Display that auto publishing is on on the publish page

Turn Auto Publishing On (Yes or No) (System setting in platform)

Max days is 99.

No restriction on the date to publish from.
