# NXT-66897 — Page 2: Account Code Structure

## Description
As a user setting up configuration,

I want to understand what this step does

So that I know what information to provide

## Acceptance Criteria
* The UI includes a banner at the top with the text at the bottom of the acceptance criteria
* The account code structure is set to the default structure for the customer's state based on mapping in the backend, or a general default template if no state-specific default exists
* The user can reorder, rename, edit, delete existing segments, add new segments
** The site segment cannot be deleted
* Display an example account number that updates as user makes changes to the structure
* Add a save, reset, and "skip for now" button:
** Save: saves changes and proceeds
** Reset: resets to the default account code structure
** Skip for now: displays a warning modal with the following text, then proceeds without saving: *Warning Modal Text:* "Warning: Proceeding without proper account structure setup may result in audit compliance issues and manual work for your finance team. You can complete this setup anytime under Configuration > Account Structure." *Modal should have two buttons:*
*** "Go Back and Set Up Now" (closes modal, returns to setup)
*** "Skip Anyway" (closes modal, proceeds without saving)
* The timestamp for when the user saves their account code structure and finishes the configuration wizard should be stored in the backend (will be used in another story)
* Include the following text in a banner at the top:

*"Set Up Your Account Code Structure"*
