# NXT-1651 — INV - Orders - Edit Orders - Apply QOH check on Commodity items

## Description
As a Site Manager I need the system to check for available QOH for commodity contracts, to ensure I don’t order more than what is available.

*Requirements*

* Implement the check for available quantity for the items on Commodity Contracts
** on Edit Orders (Open, Pending Review)
** adding items to an order
*** this should already be handled in Create Order story
* Show alert (!) icon next to Order Quantity
** tooltip: “Item order quantity cannot exceed 36 cases available.”
** *Note: it is _possible_ to have multiple contract-related messages in the tooltip, but unlikely (e.g. Available Quantity *and* Contract Min/Max)
* Show message if needed:
** “Item order quantity cannot exceed available quantity.”
** Alternate: “Item order quantity cannot exceed available quantity for Items #LI-2933, #LI-90212”
* Fix tooltip messages:
** Total Order Amount message should reference the Contract Min/Max *Purchase Amount* (dollar value)
*** e.g. “Increase order amount by $140.00 to meet minimum purchase amount.”
