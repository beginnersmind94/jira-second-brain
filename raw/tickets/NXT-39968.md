# NXT-39968 — Elig - Verification - Online Apps after Verification

## Description
We need to handle a couple of scenarios where a household reapplies after previously having been selected for verification.

Screenshots from the PrimeroEdge side:

!image-20230523-210729.png|width=1135,height=848!

!image-20230523-210755.png|width=852,height=782!

!image-20230523-211332.png|width=506,height=613!

!image-20230523-211349.png|width=544,height=115!

*requirements*

Scenario: Household is reapplying after being chosen for verification and ending up with a verification result that was the termination of their benefits. (They went from F/R → Paid with Reason: Verification)

* Track whether students who are part of the academic year’s verification pool have had proof of income provided.
* If a new application comes in, we need to check if the students were part of verification.
** If they were, let’s alert the user. We have a warning already - let’s add to it. Keep the existing notes, append:
*** Text: This student was selected for Verification on Application #12345.

!image-20230523-213135.png|width=1965,height=522!

* If proof of income has not been provided for the household then on the Household Member screens we want to prompt the user that they need to do this.
** Add a ‘Proof’ checkbox down by the Total Income box.
*** Add a warning icon next to it with the following text: Income documentation or proof of program participation required.
** If the user checks the box, show them the following text with an OK action:
*** Please scan or attach income documentation / proof of program participation under the Documents section after processing.
* If proof of income has been provided for the household, then automatically fill the ‘Proof’ checkbox.

!image-20230523-213748.png|width=1975,height=888!

* If the Proof box is not checked, then the application cannot be processed / completed. It is Incomplete.
** Add / use pending reason: Proof required after Verification
** Cannot Submit the app, can only mark as Pending.

!image-20230523-214537.png|width=1565,height=538!

* If an online application comes in for a student who was involved in verification, and proof has not been established, then the application must go to Pending with the pending reason attached.
* Application Processing / Details
** We need to make the same changes and show the same information here as in Manual Entry.
*** Show pending reason as per normal.
*** Students → Show the warning for the student having been part of Verification.
*** Household Members → Show Proof checkbox, allow Edit to check or uncheck it.
**** Unchecking it should lead to the app being un-processable, can only go to Pending.
**** Same as manual entry - show tooltip, show confirmation prompt on checking the checkbox.

## Acceptance Criteria
* Online Apps
** Apps for students with no provided proof go to Pending, are not automatically processed.
** Make use of the proof Pending Reason.
