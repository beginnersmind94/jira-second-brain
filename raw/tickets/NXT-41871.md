# NXT-41871 — System - Content Management (Part 2: Add Content Management page UI)

## Description
As a Customer Success administrator, I would like to be able to quickly update the content that users are seeing in the Onboarding wizard, with individual text strings and for all districts at once, so that I do not need to depend on a production release in order to push out updated information to users.

*Summary*

Add a ‘Content Management’ page which can be accessed only by users with the Customer Support role. This page will contain all of the text related to Onboarding and will allow the user to make updates which then cascade to every district on the server. The page will be expanded upon later to manage other text updates throughout the application.

*Details*

* In System → Support, add a new page: *Content Management*
* In this new Content Management page, include the following: 
** {color:#97a0af}*_a drop-down menu similar to Item Management’s_* {color}[{color:#97a0af}*_Item Configuration page_*{color}|https://qa.perseusedge.com/item-management/catalog/item-configuration]{color:#97a0af}*_._* {color}
*** An option for “Onboarding wizard” can be selected by default, but in the future this will allow us to manage texts in other pages using one central screen.
** {color:#97a0af}*_A table showing each of the texts that exist (by page and by area) in the Onboarding wizard_*{color}
*** It would be great to have search fields to search on the page, but if this is too much effort it is not required right now. 
*** Pagination required.
*** Current field headers for Location, Area, and Text should correspond to the data records, but these headers and their descriptions can be changed if needed.
** {color:#97a0af}*_An edit pencil next to each text, which_* {color}*_either_*{color:#97a0af} *_opens an “Edit Text” drawer_* {color}*_OR_*{color:#97a0af} *_allows for inline editing of the text._* {color}
*** When a successful change is made, then we should display a success message: “Text updated successfully!” 
*** Fields cannot be blank.
*** All changes should be reflected immediately when navigating to the Onboarding wizard for *any* district on the server. We do not want custom instructions for each district at this time.
** Not pictured: we need to capture audit history. It would be great to show the history icon as well, but we need to at least capture the user’s timestamp and name when making a change.
* Additionally, move the existing Customer Dashboard into the Support category and update the breadcrumbs for that page. This will better consolidate the Support pages in the navigation.

## Acceptance Criteria
Verify that a new Content Management page has been added to the System module

Verify that this page has a drop-down menu which has “Onboarding wizard” as the default and only option

Verify that the page contains a table displaying all text from the Onboarding wizard categorized by page, area, etc.

Verify that each text field can be edited successfully and cannot be blank

Verify that audit history is captured even if icon is not displayed

Verify that changes made are immediately reflected when navigating to the Onboarding wizard for any district

Verify that Customer Dashboard has been nested under the Support category in the System module and had its breadcrumbs updated as well
