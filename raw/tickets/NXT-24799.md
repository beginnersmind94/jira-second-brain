# NXT-24799 — INV - Create Receipt - Update Prices on Transaction Date Change  

## Description
As a district admin I want the receipt price to consider the transaction date so that my items populate with the correct price if I am using Received Date for prices.

Setting is added as part of [https://primeroedgenext.atlassian.net/browse/NXT-16356|https://primeroedgenext.atlassian.net/browse/NXT-16356|smart-link] 

*Requirements*

* Receipt item prices should be updated if Transaction Date is changed to a date with a different price condition
** this only applies if the ‘Receipt Item Price Condition’ is set to use Received Date per the vendor contract
** this can only be done the first time the receipt is being created, before it has been saved
* Display an alert if the changed date resulted in different pricing
** “The selected Transaction Date has resulted in some item prices being updated.”

## Acceptance Criteria
Automatically update item prices on the receipt if the transaction date is changed (dependent on setting value)
