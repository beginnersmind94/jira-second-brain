# NXT-71432 — SC - MT - District - Central Boundaries History

## Description
As a *Customer Support or Technical Support user*, I want to view a history of all changes made on the *Central Boundaries tab*, so that I can audit parent-level boundary propagation and role assignment changes.

----

h1. Background / Notes

This follows the same audit pattern used across the system.

For Central Boundaries:

* Audit history is *global to the entire screen*
* Focused on *parent-to-child configuration changes*
* Tracks both:
** boundary propagation
** role/permission propagation

We will:

* Use the *existing View History drawer*
* Use the *Tailwind version*
* Maintain consistency with District Boundaries and Realm screens

----

h1. Current Navigation

*System > Districts > Central Boundaries tab*

----

h1. Technical / Implementation Notes

* Use existing audit logging framework
* Use Tailwind View History drawer
* Trigger on *Save Changes*
* Track all parent-level configuration changes
* Scope = entire screen (not per boundary row)

Audit events should include:

* enabling/disabling boundaries
* setting boundary requirement (Optional/Mandatory)
* setting role & permission requirement
* selecting/changing linked roles
* selecting/changing target districts
* applying changes to all vs subset of children

----

h1. Out of Scope

* Row-level history
* Advanced filtering beyond existing drawer
* Export functionality

----

h1. Functional Requirements

h2. 1. Add View History button

* Display on Central Boundaries screen
* Top-right placement

----

h2. 2. Open history drawer

* On click, open drawer
* Use Tailwind template

----

h2. 3. Global audit scope

* Display all audit entries for:
** central boundary configuration
** role propagation
** district targeting

----

h2. 4. Track Save-driven changes

Audit must capture:

* boundary enable/disable
* boundary requirement changes
* role & permission requirement changes
* linked role changes
* target district changes

----

h2. 5. Audit entry details

Each entry includes:

* User
* Timestamp
* Action type (Add / Edit / Delete)
* Entity affected:
** boundary
** role mapping
** target districts
* Before values
* After values

----

h2. 6. Consistent audit format

* Must match system standard

----

h2. 7. No additional security

* No additional permissions required

## Acceptance Criteria
* View History button is visible
* Drawer opens on click
* Drawer uses Tailwind component
* Displays all central boundary changes
* Add/Edit/Delete actions are logged
* Includes user, timestamp, before/after values
* Matches existing audit UI
