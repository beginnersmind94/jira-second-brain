# NXT-19507 — MP - Menu Planner - Build a Tray Tool (UI Only)

## Description
When the Menu Planner, Nutritionist, Food Service Director, or other appropriate members of the district are dragging/dropping menu items in the Menu Planner tool, there needs to be a tool that fills in components based on the menu items selected to add to the menu. 

Component information is assigned to each menu item, which will allow a small "tray" to fill in according to those components. This functionality is in the current SchoolCafe Parent Portal, so a visual correlation with that will be a strong help for the person building the menu to know that they've included all of the components they intend to onto each day of the menu cycle.

Story [https://primeroedgenext.atlassian.net/browse/NXT-17959|https://primeroedgenext.atlassian.net/browse/NXT-17959] has these default categories listed, with the appropriate colors... so that the menu items themselves are colored to match the categories when placed on a menu. As each menu item/component is added and the component requirements are met, the small tray will fill in with the matching color.

There will be a tray for each day of the cycle that is being built... it can remain blank until menu items are dragged to the day it belongs to.

## Acceptance Criteria
The small tray above each day of the menu cycle will fill in appropriately with the colors of the components as I add menu items to each day. The tray will correlate with USDA Nutrient Requirements for the Meal Pattern selected for the menu cycle... so that the minimum requirement must be met before the coordinating spot on the tray will be filled. It will also take into consideration any "Choose 1" or "Choose 2" defaults that are created for this particular page. (Those will be in a different story).
