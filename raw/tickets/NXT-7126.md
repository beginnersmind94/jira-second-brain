# NXT-7126 — Data Exchange - Field Mapping Screen - Allow user to link file fields with PrimeroEdge mapped fields

## Description
As a PrimeroEdge user, I would like to have an intuitive screen that allows me to associate fields in PrimeroEdge with fields in my uploaded file, so that I do not get overwhelmed by all the data on the screen.

*Summary*

# Allow a user to link fields in the PrimeroEdge mapped field section with fields in file details table (using drag-and-drop action) and update the respective Confirm Mapping column.
# In the PrimeroEdge mapped field section, mark a 'used' column in green.

*Details*

* In Data Exchange → Field Mapping, allow a user to drag and drop fields from the PrimeroEdge mapped field section onto a cell in the Confirm Mapping column of the file details table.
** When the field is dropped onto the field’s Confirm Mapping cell, replace ‘Not Validated’ with the \{\{PrimeroEdgeMappedField}} value as a green chip.
* In the PrimeroEdge mapped field section, change the font color of dropped fields to green so that the user knows that they have been used.
** Exception: If the dropped field is ‘Not In Use’ no color change is necessary in the section. In the table, display the chip in gray.
* Allow the user to drag and drop fields that have already been linked, to another field. In this case, just remove the link from the previous column so that it goes back to ‘Not Validated’.

## Acceptance Criteria
* Verify that in Data Exchange → Field Mapping, we allow a user to drag and drop fields from the PrimeroEdge mapped field section onto a cell in the Confirm Mapping column of the file details table.

* Verify that when the field is dropped onto the field’s Confirm Mapping cell, we replace ‘Not Validated’ with the {{PrimeroEdgeMappedField}} value as a green checkmark.

* Verify that in the PrimeroEdge mapped field section, we change the font color of dropped fields to green so that the user knows that they have been used.

* Verify that exception: If the dropped field is ‘Not In Use’ then no color change is necessary in the section. 

* Verify that in the table, we display 'Not in Use' chips in gray.

* Verify that we allow the user to move fields that have already been mapped, to other fields by removing the mapping from the previous field so that it goes back to ‘Un Mapped’ state
