# NXT-16425 — Platform-UX- Mock up for Reimbursement Claims @ Export

## Description
This story is to cover the tasks related to the mock up screens @  Reimbursement Claims.

The user tires to access the Claims Reimbursement-Export by navigating System--> Configuration-->Export Dashboard.

Once on the landing page,  (similar to that of Import dash board which is currently being redesigned to incorporate file/api based inputs), the user based on the access level (will see either the school ISD assigned or list of schools under their umbrella) will choose either File or API (check for naming consistency). Once selected, the user can select the template either student or DC or Claims (Claims is the new one).

Once claims as option is selected, they should be able to select the claim type. Each state has about 4-5 types if claims as standard so let’s name them as Claim 1 to claim 5 (can be renamed later).



Once the claim sub type is selected, the user lands into a page similar to that of the classic type shown here (attached and named as classic mapping screen shot).



The landing page, lets call this as the details page, must have the below options. Check for consistency across the platform

Export name

Export Data Category 

Enabled (Check box or radio) this is to keep the template enabled.

State Default (Check box or radio) this is to keep the all the settings to a default as that of the state. 

IF ANY DEFAULT setting is modified or edited, the state default must be indicated to reflect that is not in default anymore.

Save, Save as, Print, cancel --these must be vetted and applied only if needed.

Also provision must be given to include the below

delimiter, qualifier, option to include header (Check box or radio), Mappings

filters (more details needed)

History (more details needed)

File location / Transfer  Must include transfer Method, File Name, Format, Name Match Criteria (More details needed)
