# NXT-16885 — INV - Orders - Review Order Settings - Add New Rule - User (Step 2)

## Description
As an inventory operations manager I want a User option on the Add New Order Review Rule process so I can potentially configure order reviews for orders from a specific user or users.

*Requirements*

* This is the second step of the Add New Rule process of the Review Order settings
** this step is optional
* Provide a control to select one or more users
** Provide a search field to search for users
*** the label will be ‘Search by Name’
*** this will find all users that match the search criteria
*** Search button
** Provide a small grid below that lists all users returned by the search
*** only return Active users
*** columns:
**** First Name
**** Last Name
**** Email
**** Roles
***** some dependency on System module
***** show a popup if the number of roles exceeds the width limit of the cell
**** Site(s)
***** some dependency on System module
***** show a popup if the number of roles exceeds the width limit of the cell
**** Select
***** this will officially select the user and add them to the Review Order Rule
***** this will add the user to a second grid below (or a tab) which will list only the users added to the rule
*** Once a user has been selected, reload the grid to show only the selected user
**** disable the Select column
** Display another grid listing only the users which have been added to the rule
*** *TBD / Mocked up*
* Display a Back to Rules List button in the bottom left corner, to cancel and return to the list page
** Prompt user: “Discard rule configuration and exit setup?” Yes/Cancel
* Display a Previous button in the bottom left corner, to return to the previous step
* Display a Next button in the bottom right corner to advance to the next step
* Display a Complete button in the bottom right corner, to skip the remaining steps and jump ahead to the confirmation step

## Acceptance Criteria
* Can search for users by name (single field for first or last name)
* Can view user results returned by search
* User attributes include First Name, Last Name, Email (Roles + Sites TBA later)
* Can add users to the rule
* Can view all added users in separate grid/view
* This step is optional/skippable
