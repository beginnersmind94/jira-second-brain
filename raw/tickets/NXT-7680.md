# NXT-7680 — Direct Certification - Extensions - Manual

## Description
*User Story*

As a district admin I want a way to manually extend benefits from one student to another-  the search just isn’t finding the student.

*Mockup*

[https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=172630%3A889&scaling=scale-down-width|https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=172630%3A889&scaling=scale-down-width|smart-link] 

*Requirements*

* Add a Manual option to the Extensions page.
* User can select 2 students with the Student Lookup option.
** 1st selection is the Eligible Student.
*** This student must be Directly Certified. If they are not, give the user an appropriate error message.
**** The selected student is not directly certified.
** 2nd selection is the Candidate.
* Show the user current information on both students so they can review this.
** Names, ID #s, Eligibility
** Addresses
** Guardians 
* User has an option to Match or Dismiss this.
** History tracking should happen if they go ahead and Match this. the Matching Criteria would be ‘Manual Entry’ for this.
** If they dismiss this without ever acting on it, we don’t need to worry about tracking that.
** If they match, the students need to be placed in the same household.

## Acceptance Criteria
User can manually select a DC student and extend benefits to another student.
Eligibility extends appropriately with history tracking.
Households are updated appropriately - students are placed together.
