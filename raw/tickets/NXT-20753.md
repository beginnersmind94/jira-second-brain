# NXT-20753 — Logging

## Description
As an administrator, I want to be able to view reports on system usage, counts, timing and accuracy so that I can find areas of improvement.

*Application*: POS

*Flow:*

* While using the POS transaction screen with Food Detection enabled:
* At key points data is logged into a transaction table in the background

*Assumptions:*

* This page exists in the classic ExpressPoint
* You can copy all the code and try to copy as-is
* Talk to Saketh if you have any questions

 *Requirements*:

* Copy everything as-is from classic
* While using the POS transaction screen with Food Detection enabled, do the following:
** Ideally, there would be one row in database table per transaction with these as the columns that correspond to the values above:
*** SiteID, TerminalID, ScanID, TransactionID, StartTime, ReturnTime, CountItemsFound, FoodItems
*** If do this, when register start, leave other columns null/empty and then do an update on those columns when get results back
** Start of process when a Food Detection Event is triggered
*** Key Values:
**** SiteID – id of school so that can join to site table in later reporting
**** TerminalID – id of terminal so that can join to terminal table in later reporting
**** TransactionID – transaction id of the onscreen transaction, need it to join to in reporting so can get food items and count
**** StartTime – date/time stamp
** When it receives the results back from Edge, update the row from the trigger with these values
*** Key Values:
**** Scan ID – the id from the Edge device for the scan
**** ReturnTime – date/time stamp
**** CountItemsFound – small integer, count of food items found
**** FoodItems -
***** Note: FoodItems will come back as a list array in text form, probably
***** Need to store in a way that can be transformed into a many to one relationship in reporting app later on.
***** It is very important that we be able to easily pull out the list values in reporting to compare against the foods sold
* Need a 2^nd^ logging table for Food Detection Exceptions
** This gets updated when there is a problem
** Needed Columns:
*** TransactionID – same as above
*** ExceptionID – small integer, tells what part of part generated exception
*** DateTime – date time stamp to capture when, call it whatever you want
** Note, this will keep the table as small as possible
** Can Have another lookup table that can be joined to with these key / value pairs:
** ExceptionID and Text Values:
*** 1 – Edge Not Found
*** 2 – Food Detection Failed
*** 3 – Food Detection Cancelled - this is not required in this story, ignore
*** 4 – Food Detection Killed - this is not required in this story, ignore
*** 5 – Edge Results Not Returned - this is not required in this story, ignore
*** 6 – Food Detection Enabled
**** Created when user toggles on the Food Detection in admin screen
**** Leave TransactionID null/empty
*** 7 – Food Detection Disabled
**** Created when user toggles off the Food Detection in admin screen
**** Leave TransactionID null/empty
*** 8 – Food Detection Session Enabled
**** Ignore in this story, no currently in use
**** Created when user clicks Food Detection On in transaction screen
**** Leave TransactionID null/empty
*** 9 – Food Detection Session Disabled
**** Ignore in this story, no currently in use
**** Created when user clicks Food Detection Off in transaction screen
**** Leaves Results Screen there, doesn’t hide it
**** Leave TransactionID null/empty
*** 10 – Food Detection Transaction Enabled
**** See Story 20747 for when used
**** Created when user toggles on the Add Detected Items in Cart field in admin screen
**** Leave TransactionID null/empty
*** 11 – Food Detection Transaction Disabled
**** See Story 20747 for when used
**** Created when user toggles off the Add Detected Items in Cart field in admin screen
**** Leave TransactionID null/empty
*** 12 – Food Detection Configuration Failure
**** Created when the app starts and Food Detection is On and the app cannot match the Model Configuration in the edge device and the POS
**** Leave TransactionID null/empty
* If Food Detection is disabled, do not log anything above
* Security
** Not required
