# NXT-50537 — PR- Plan/Record- Save button change to close when no changes are available (UI ONLY)

## Description
As a production manager, I would love for a way to quickly know if there something that needs to be saved on the production plan or record page based on changes that have been made on either page.

## Acceptance Criteria
* Save button should only be available if changes have been made and need to be saved
** Changes include, change in counts, Offered with addition or removal or menu item deletion
* Provide a close button if no changes are needed to be saved and or if changes have been reverted to the original state of when the plan or record was opened (similar to standard menu)
* All unsaved changes prompts should remain the same if user leaves the page or tries to switch tabs or click back button
* If user clicks on close, redirect back to the calendar page on the month that the user was on with the site that they were on
* If user refreshes plan or record page, always still hold the information to redirect through close or back button to the appropriate calendar month and site
