# NXT-39052 — MP- Menu- Standard- Variety menu item sub item check for add menu items

## Description
As a Menu Planner, When I have added a variety menu item, the system should not allow me to add any of the menu items within the variety menu item separately. Also, if the user happens to add the menu item first, then the variety menu item which includes that menu item, only include it on the variety menu item and remove from the menu.

## Acceptance Criteria
* When user adds variety menu item onto menu, system should validate if there are any of the child menu items already on the menu and remove them since they will now be added on the menu from the variety menu item. 
** Variety menu item (child Items) should take priority in terms of which one stays on the menu 
** Info message “ Menu item(s): '*Menu Item Name”*, “*Menu Item Name*” has now been added as part of the Variety Menu item” (green Font)
*** Display at the top of the page once page reloads with newly added menu items, if there happens to be multiple menu items that were already on the menu then display the multiple menu item names within the message 
*** Only display on current refresh, if user leaves page and comes back do not display again, allow user to manually remove the message
!image-20230427-143120.png|width=284,height=52!
* When user adds variety menu item onto the menu first, then wants to add more menu items that are part of the variety separately, system should not allow those menu items to be added and gray out the selection of being able to add that menu item (Add menu item screen)
** Display a tool tip on the checkbox that explains that this menu item is already part of an added variety menu item 
*** “Menu item added on Variety Menu Item”
