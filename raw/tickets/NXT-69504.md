# NXT-69504 — Fin - Inactive Data Points

## Description
_(none)_

## Acceptance Criteria
h2. Toggle Data Point to Inactive — Core Behavior

_When user attempts to toggle a data point to Inactive:_

* Provide a toggle button inside the rules side drawer to set the status of the data point (active/inactive)
* The status in the main grid updates accordingly and that data point is greyed out (inactive appears with an orange circle)
* Check dependencies - System queries GL Entry Types using this data point

h3. If dependencies exist:

* Show modal: "{{This data point is used in {X} entry type(s). If you make it inactive, posting will be blocked anywhere it’s referenced until you reactivate it.}}"
* -List the affected entry types (clickable links)-
* Two actions:
* {{Simplified modal with count only and 2 buttons (Cancel, Make Inactive).}}

*-"View Entry Types" → navigates to Entry Types page filtered to affected items-*

*"*{{Make Inactive Anyway}}*" → proceeds with warning logged*

h3. Downstream behavior (already implemented in separate story):

* -Entry types containing inactive data points get a warning badge-
* -Automatic posting attempts for those entry types fail gracefully with clear error message-
* -For user-created data points, the user can reassign position of that data point used within its associated rules to other data points-
* -Any reassignment behavior is out of scope for this story and happens in the rules side drawer only for user created rules.-

h2. Rules Drawer — Status Toggle

_Given_ a user opens the rules drawer for any data point

_Then_ display a status toggle at the top of the drawer:

* Active state: Green background, "● Active" + "{{Journal entries using this data point can post.}}"
* Inactive state: Red background, "○ Inactive" + "{{Posting is blocked for journal entries using this data point. Activate it to allow posting.}}"
* Toggle switch allows immediate status change

h3. Rules Drawer — Inactive Warning Banner

_Given_ a user views the rules drawer for an entry type that references an inactive data point

_Then_ display a warning banner at the top of the journal entries section (inside the scrollable area):

* Format: Warning icon + "{{Inactive Data Point Warning"}} title + "{{{X} journal entry/entries reference inactive data points. Activate them to allow posting.}}"
* Styling: Amber gradient background, rounded border, shadow for visibility
* Position: Sticky at top of scroll area (stays visible while scrolling through rules)
* Condition: Only displayed when inactive references exist — hidden otherwise

h3. Rules Drawer — Inactive Data Point Display in Rules

_Given_ a journal entry references an inactive data point in debit or credit

_Then_ the inactive data point displays in the debit/credit table:

* Warning icon (⚠️) inline before the data point name
** This warning should appear in *any* rule drawer where inactive data points are referenced—not only in the specific drawer where we intentionally made a data point inactive during testing.
* Strikethrough styling with muted opacity (0.6)

_Note:_ This applies to ALL rules where the inactive data point appears

h3. Rules Drawer — Legend (Bottom)

A legend is displayed at the bottom of the drawer showing:

* Warning icon + strikethrough "Name" example
* Text: "{{Inactive data point — entry won't post until active."}}

h3. Main Table — Inactive Row Display

_Given_ a data point is marked inactive

_Then_ the table row displays:

* Status badge shows "{{Inactive}}" with red styling
* Row has muted appearance

h3. -Posting Behavior-

* -No automatic posting until all referenced data points are active.-
** Removed as this is current behavior

h3. System Rules — Read-Only

_Given_ a user views the rules drawer for a system-created data point

_Then:_

* No edit button is displayed on journal entry cards
* Rules are read-only and cannot be modified
* Only user-created data points have editable rules
