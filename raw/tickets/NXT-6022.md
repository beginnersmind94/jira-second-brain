# NXT-6022 — Application Entry - Guided Process (1. Students) - Add Student by Lookup

## Description
*Story* 

In step one of the guided application process, the admin can easily add/find students for the application using the lookup feature.

*Mockups* 

[https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=1296%3A25854&scaling=min-zoom|https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=1296%3A25854&scaling=min-zoom]

*Requirements* 

* Instructions: _Select 'Find a Student' to search for an existing student in the system. Can't find the student? Select 'Add a Student' to manually enter the pending student details. Then, answer the categorical and income questions to save the information._
* Add a student generates popup 
** Add a ‘Find a Student’ button to the Add a student pop-up.
*** 'Find a student’ button leads to the Patron Lookup that was made in another story. 
*** [https://primeroedgenext.atlassian.net/browse/NXT-6467|https://primeroedgenext.atlassian.net/browse/NXT-6467|smart-link] 
* If a student is selected in the Lookup - 
** Show the selected student’s picture, full name, ID number, Status (Active/Inactive) 
** Entry fields are populated with the matched student info available: ID, First Name, Middle Name, Last Name, Suffix, DOB, Site, Grade
** Entry fields are read-only - cannot be edited.
* User can deselect the student in the lookup box
** If this happens, entry fields are cleared out 
* Option for Advanced Search 
** Shows the additional lookup fields other than name and ID > DOB, Area, Site, Site Code, Grade, Homeroom, Status
* 5 max results in lookup and pagination in the lookup
* at least 1 student on app to go Next
* Answer 2 questions - Categorical and Income 
* Adds student to grid

## Acceptance Criteria
- admin can add students using the lookup feature 
- lookup feature searches db based on information entered by the admin (similar to classic functionality)
- when the student selected from the lookup is collapsed to show only the selected student
- user can find a student or manually enter data in the fields (pending student story 6445 takes care of manual entry)
- Multi-language, ADA compliant.
