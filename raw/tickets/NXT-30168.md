# NXT-30168 — IM - Items - Allow Pack Size changes

## Description
As a District Admin I want to be able to modify my Pack Size configuration so I can configure items as single units and expand upon their configuration later.

*Note:* this story is related to [https://primeroedgenext.atlassian.net/browse/NXT-31281|https://primeroedgenext.atlassian.net/browse/NXT-31281|smart-link] 

*Requirements*

* Allow units to be added or removed for items unused in Inventory
** Allow item Pack Size configuration to be expanded as long as the item has not been used in transactions in Inventory
** Allow units to be removed from a Pack Size configuration if the units are not consumed in Inventory
* Provide ‘Add Higher Unit’ button upon Edit if item is not yet used in Inventory batches 
** Follow similar flow for ‘Lowest’ unit creation flow
*** We are always saving single unit as ‘Lowest' so any expanded Pack Size will be built upward
** Hide the button if item’s pack size cannot be changed
*** Only allow Description, Weight, FMV changes
* Weight should be editable on the lowest unit
* Allow Sub-Quantity to adjusted if item is not yet used in Inventory batches
** The sub-quantity could potentially be changed as long as the item is not yet consumed within Inventory

## Acceptance Criteria
* Allow units to be added/removed to Pack Size configuration for items not yet consumed by Inventory
** Provide a unique flow similar to ‘Lowest’ item creation
* Allow sub-quantity to be changed for items which are not yet part of Inventory transactions
