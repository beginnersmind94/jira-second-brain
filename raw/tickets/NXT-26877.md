# NXT-26877 — Platform - Permissions - Implement Page Level ADD Permissions

## Description
As a Primeroedge admin, I should be able to manage permissions for ADD of the pages under any specific module

*Requirements:*

* When user updates the permissions of a page from roles & permissions, system shall reflect scope of the page when user reloads the screen.
* When ADD permission is removed for any role, that specific resource/page must be disabled(but visible) with any add functionality effective immediately.
** All tabs should be derive the same permissions as well within that page
** -disabled ADD button shall include “Info” icon that displays below text when user hover over-
*** -“You do not have permissions to perform this action”-
* When ADD permission is enabled for any role, that specific resource/page ADD buttons must be visible from navigation screen effective immediately.

## Acceptance Criteria
* District/Cybersoft admin shall be able to update ADD permissions of any page type resource from roles & permissions 
* User shall able to see the disabled ADD button if the role he has been assigned does NOT have “ADD” permissions enabled
* User shall not be able to see the tabs/sub-pages with disabled ADD functions under the page that has been disabled ADD permissions.
