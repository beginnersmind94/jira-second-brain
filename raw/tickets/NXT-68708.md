# NXT-68708 — Add Required “Entry Type” Field to Manual Journal Entries

## Description
To correctly categorize manual journal entries for the P&L and Inventory reports, we need to add a required *“Entry Type”* field to the Manual Entry screen.
This is an *enhancement* to the existing workflow and users must now select an Entry Type whenever creating a manual entry.

## Acceptance Criteria
h3. *1. New Required Field*

* Add a new field labeled *“Entry Type”* to the Manual Entry form.
* Field is *required* to save or submit an entry.
* If left blank, show:
*“Please select an Entry Type.”*

----

h3. *2. Dropdown Values (Final List)*

h4. *INVENTORY*

* Inventory – Purchase
* Inventory – Usage (COGS)
* Inventory – Transfer In (from Warehouse)
* Inventory – Transfer Out (to Warehouse)

h4. *EXPENSE*

* Expense – SG&A
* Expense – Payroll
* Expense – Other

h4. *REVENUE*

* Revenue – Catering
* Revenue – Breakfast A La Carte
* Revenue – Breakfast Adult/Visitor
* Revenue – Breakfast Paid
* Revenue – Breakfast Reduced
* Revenue – Lunch A La Carte
* Revenue – Lunch Adult/Visitor
* Revenue – Lunch Paid
* Revenue – Lunch Reduced

h4. *REIMBURSEMENT*

* Revenue – Reimbursement (Breakfast, District)
* Revenue – Reimbursement (Lunch, District)

----

h3. *3. Editing Existing Entries*

* Existing manual entries without an Entry Type must prompt the user to select one before saving.

----

h3. *4. Persistence*

* Store the selected Entry Type with the manual entry record.
* Display it in the Manual Entry detail view.
