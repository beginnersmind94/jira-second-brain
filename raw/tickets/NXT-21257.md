# NXT-21257 — Add Tabs to Build Model

## Description
We need to separate Build Model into a two-step process, one for configuration and one for publishing

*Application*: Face and Food Detection Website

*Flow:*

* Browse to the Food Detection Settings page and then the Model Management Page
* Click on the Create Model button and come to this page
* On the first tab, do the configuration and click next
* On the second tab, select the publishing information
* Select Create and it does everything it normally does

 

*Background:*

* This page already exists, we adding in publishing functionality to it
* Leave all the existing code in place and alone, only add in the code I specify here
* Everything that is on the existing page except the Create button is a part of configuration and will go on the configuration page

 

*Assumptions:*

* Here you will need to know all the sites for a district
* To do this, it is assumed you can do a DB lookup for the district of the user logged in
* It is assumed that where I refer to tabs, I am talking about a 2-step workflow

 

*Requirements:*

* Add a workflow or tabs at the top of page content under the header
* There are two tabs, with names as follows
** 1 Configuration
** 2 Publish
* Tabs need to be consistent with 2.0 (see attachment)
*  
* Configuration Tab
* At the bottom of the Configuration have a next button that takes you to the Publish tab, UI consistent with 2.0
* All the content on the existing page will go on this tab except the Create button
* Data maintains state values on this tab when goes to the next tab
*  
* Publish Tab
* At the bottom of the Configuration have a Back button that takes you to the Configuration tab
* Also at the bottom, where the “Next” button would go, put the Create button from the original page
* The rest of the content in this page will be covered in the next story
*  
* Security – already done

## Acceptance Criteria
* All the original code still works and does what it already did
* There are two tabs with names and UI as per requirements
* Can navigate back and forth between tabs and retain state / data
