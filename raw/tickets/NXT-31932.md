# NXT-31932 — Roles and Permissions - Add User Confirmation on Changes

## Description
Currently, a user is able to make permissions changes, but then navigate away from the page without being reminded that they have not saved their changes.  This story is to add functionality to confirm with the user whether they want to save or abandon their changes.

## Acceptance Criteria
* After a user modifies permissions, if they have pending changes that have not been saved and they attempt to navigate away from the page (or change roles, or change the module), they will be prompted to save their changes.
** The user shall be presented with a popup window asking “You have unsaved changes.  Would you like to continue without saving?” and containing “Continue” and “Cancel” buttons.  If they press “Continue”, their changes will not be saved and their navigation shall continue to where they originally intended.  If they press “Cancel”, the user shall be returned to the permissions page (where they have the option of saving their changes).  
* A “Cancel” button shall be added to the UI to allow the user to explicitly abandon any changes that they had made.
