# NXT-36059 — Device Management – View Log

## Description
_Phase 2, System Step 4_



As a SchoolCafé admin, I want to see the history of the updates to the devices so that I can use it in debugging any outstanding issues.

*Flow:*

* User logs in and browses to System > Management > Device Manager
* User clicks on the History button for a record
* The History drawer opens up and user sees the information

*Notes:*

* This is the continuation of the Device Manager stories.
** 1^st^, we created the APIs in Front Office
** 2^nd^, we updated the device manager screen
** 3^rd^, we created the Add New, Edit, Delete, Suspend, Activate functionality
** Now, we will do the view history functionality for a POS device
* Note: this feature requires the work of both System and FO teams.
** By the time we start this story, FO team must have written the relative APIs for button
** And the Add New functionality must have been completed

*Requirements*:

* All of these requirements are for the POS Device History drawer
* Update the drawer per the Figma
* Currently it is a page, but needs to be a drawer
* 
* Grid has the following columns:
* Device
** The name of the device
* Event
** Values:
*** Add
*** Edit
*** Delete
*** Suspend
*** Activate
* Description
** Value:
** The text of the values from the add and edit drawers
** Format: Device=xxx; Site=xxx; Description=xxx
** Example text:
** Device=NC34-01; Site=Tavola Elementary; description=Left lane in main serving lines
* User
** The user account logged in that made the change
* Date/Time
** The date / time stamp of the change from the database
* 
* Grid has paging
* Add sorting and filtering if you can

## Acceptance Criteria
* History is updated to be a drawer
* Grid has columns and values as per the requirements above
* Grid has paging
