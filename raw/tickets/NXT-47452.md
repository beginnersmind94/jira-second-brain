# NXT-47452 — PR- Autofill- Confirm Autofill and Plan Page Load

## Description
As a production manager, when I use the autofill feature, I would like for the numbers to be inserted into the production plan but I want an opportunity to review the counts before I save.

## Acceptance Criteria
* This applies for all versions of autofill 
** Allow user to click the autofill 
* Close the slideout once the action is clicked and reload the the plan page with the counts but not saved
* Provide a warning banner on the top of the plan page with a save button
** “All Counts have been autofilled but not saved, please review counts and save”
** DO not let message disappear until the user resaves the plan page with the new autofilled counts
* IF there are any VMIs that have been autofilled provide message “Menu Items that belong to VMI: *Menu Item Name* have been autofilled”
** This will be a separate banner
* Allow banners to float as user slides down the plan page
