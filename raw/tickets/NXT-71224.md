# NXT-71224 — IM - Items - Data Sources

## Description
As a multi-tenant user I want Data Source as a filter on the Items page so I can separately view and manage items in higher data sources and items in the local data source.

## Acceptance Criteria
* Add Data Source (back) to the Items page as a filter
** Make the Data Source and Status dropdowns share a single quadrant in the header, so that the layout is not disrupted
** Display Data Source first
* The Data Source dropdown should consider user level access to multiple data sources
** if the user does only has access to the Local data source, do not show the dropdown
* Add Data Source as a column
** hidden by default unless there are multiple data sources in the item list results
* Show Data Source on Item Details for items not in the local DS
** display a chip on the header of the Summary Info card
** use the abbreviation
** on mouseover tooltip show the full data source name
* For multi-tenant users, include Data Source as a mandatory attribute during item creation
** Add in the header area of the Summary info card to not disturb existing design
** Remove tags label “*( Max character length are allowed to type - 30 )*”
