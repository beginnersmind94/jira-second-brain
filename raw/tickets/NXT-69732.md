# NXT-69732 — Data Changed Column: Clickable Site Drilldown

## Description
*As a* district finance user reviewing financial periods, *I want to* click on the site count in the Data Changed column to see which specific sites are affected, *so that I can* quickly identify where adjustments may be needed and navigate directly to relevant journal entries.

## Acceptance Criteria
# *Reopened source (adjustments may be in progress)*

Given a financial period where a POS or Inventory period has reopened but not yet re-closed, When the user clicks the site count link (e.g., "7 sites"), Then a popover opens listing the affected site names.

# *Re-closed source (review journal entries)*

Given a financial period where a POS or Inventory period has re-closed, When the user clicks the site count link, Then the system navigates to the Journal view, pre-filtered to:

* This financial period
* This source (POS or Inventory)
* Only entries from the affected sites

# *Multi-source scenario - both reopened*

Given both POS and Inventory periods have reopened (displayed on separate lines), When the user clicks either site count link, Then a popover opens listing only the affected sites for that specific source.

# *Multi-source scenario - both re-closed*

Given both POS and Inventory periods have re-closed, When the user clicks either site count link, Then the system navigates to the Journal view pre-filtered for that source's affected sites.

# *Multi-source scenario - mixed states*

Given one source is reopened and the other is re-closed, When the user clicks the reopened source's site count, Then a popover opens listing affected sites.

When the user clicks the re-closed source's site count, Then the system navigates to the pre-filtered Journal view.
