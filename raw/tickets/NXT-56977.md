# NXT-56977 — System - Management - Year Begin - Add new page to control Year Begin process

## Description
As a Cybersoft or district admin, I would like to be able to have a central location that explains every task which is necessary to have a complete year begin process, so that users will have smooth transitions between academic years.

*Summary*

* Add a basic Year Begin page, with grid and filters, to the System module

[*Mockup link*|https://www.figma.com/proto/TxIF5PvP9D0IBNiTPkA2m5/UX-2024-25?page-id=105%3A1277&node-id=105-1362&node-type=canvas&viewport=226%2C415%2C0.13&t=uolVIp2PntEbLBpx-1&scaling=scale-down-width&content-scaling=fixed&starting-point-node-id=105%3A1362]

*Details*

* In _System → Management_, add a new page at the bottom of the nav: *Year Begin*
* Within the page, add the following elements:
** _+NEW_ button *(separate story)*
** Filters
*** Academic Year
**** Options: List of current and previous academic year
**** Default value: Current academic year
*** Module
**** Options: List of all individual modules which are licensed by the district (plus ‘All’)
**** Default value: All
*** Active Flag
**** Options: Yes, No
**** Default value: Yes
** Status bar (in terms of task completion)
*** Display number of tasks completed based on current grid results
*** Display legend for ‘Standard Tasks’ (Cybersoft-created) or ‘Custom Tasks’ (District-created)
**** *Ask Pavan G for HTML color codes*
** Grid
*** Sortable (default sort is ascending by Task Order, Task Name), filterable
** Grid Columns
*** Task color (for Standard or Custom tasks)
*** Task Name
*** Module
*** Description
*** Priority
**** Low, Medium, High
*** Status
**** New, Complete, In Progress
*** Assigned To
**** First Name + Last Name of any active user in the district
*** Due Date
**** Can be current or future date
** Actions
*** Open Task  (hover text)
**** Open relevant page in a new tab
*** Edit (hover text)
**** Open Edit Task drawer *(separate story)*
*** Delete (hover text)
**** User can delete locally-created/Custom tasks regardless of status
**** Cybersoft users can delete *any* task
**** Deleted tasks will be removed from the grid entirely (hard-delete).
* Additionally, add some generic Standard and Custom tasks to the grid to make QA testing easier.
* Default page permissions for all users who are:
** Customer Support
** License Management
** Tech Support
** SchoolCafé Administrator
** Central Office Administrator
** Director
* Future features
** History 
** Custom Views
** Ties to workspace tasks
** Notifications

## Acceptance Criteria
Verify that new page has been added to the System module

Verify that there is a grid which shows internal and custom tasks (some generic ones by default)

Verify that page filters update the grid

Verify that links can be opened in a new tab
