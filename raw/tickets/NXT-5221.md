# NXT-5221 — Applications - Pending - Pending Students Workflow Update

## Description
*User Story*

As an applications processor I want to review the student I just selected against the application information so that I am sure that I’m making an informed decision.

*Mockup*

[https://www.figma.com/proto/bG3v4WZvmPnKENe3Yewe7j/Eligibility-5.0?page-id=104%3A2&node-id=104%3A2&viewport=275%2C48%2C1.42&scaling=scale-down-width&starting-point-node-id=104%3A666|https://www.figma.com/proto/bG3v4WZvmPnKENe3Yewe7j/Eligibility-5.0?page-id=104%3A2&node-id=104%3A2&viewport=275%2C48%2C1.42&scaling=scale-down-width&starting-point-node-id=104%3A666|smart-link] 

*Requirements*

Tab

* Add the count of pending students to the tab in parentheses, just like we show the count of pending applications in that tab’s title.

Grid

* When a user manually identifies a student for matching, they should be shown the comparison window as part of the process.
* Workflow:
** User navigates to Pending Students.
** User selects the magnifying glass icon next to a student
** User performs a Student Lookup/search.
** User selects a student.
** *(NEW)* Instead of immediately matching, the user is then shown another drawer page first: the Comparison. This shows the student information vs the application information (and also shows the Match/Cancel buttons).
** User _then_ selects Match or Cancel.
** If the user selects Match, then the Application Information can be updated to match the student’s info (current behavior).
* For auto matches, we can still have the ‘View’ icon (Compare drawer) but rename the drawer title to “Match Details”

## Acceptance Criteria
Tab title is updated to show the count.

User is shown the compare window as part of manually selecting a Pending student.

Application Information is only updated if the user matches off the comparison window.

View drawer is renamed from ‘Compare’ to ‘Match Details’
