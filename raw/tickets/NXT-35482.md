# NXT-35482 — Command Centre - Threshold API (UI Changes)

## Description
Currently, the threshold is calculated based on district data (scripts) and stored as CSV file.

As we decided to use the API to extract the data from source as FO Dashboard, this may not get refreshed.

Data Requirement: Site and District Details

Change: The system should  refresh the threshold values with the new API approach and save the values in CSV/DB so that there is no impact the end user experience in UI when they apply threshold settings in UI

## Acceptance Criteria
Scope:

Consumption of new API created for Threshold calculations and implement them in UI (3 end points + UI Changes)



Common Changes on all the pages:

Site Group  - to be renamed as Site Type

     Upon Selecting a School Type - All Values (All Elementary) should be selected by default

Sites Code / Site Name Dropdown- to show the “Code” - “Site Name “
