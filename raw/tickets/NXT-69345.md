# NXT-69345 — For a central user there should be a screen to view all the districts and countys under that state.

## Description
As a *central user* or *county user*,
I want to view and navigate the hierarchy of *states, counties, and districts*,
so that I can manage and access the correct administrative level without modifying data.

----

h2. *Central User Requirements*

h3. *State Screen*

* A screen should be available to view *all counties and districts under the selected state*.
* The State screen should contain *two tabs*:
** *General*
** *Realm*

----

h3. *General Tab (Central User)*

* Editing is *not allowed*.
* Deleting is *not allowed*.
* Only *View* and *History* actions are available.
* A *Change District* icon should be displayed.
** On clicking the icon:
*** The user can select a *county or district* to navigate to.
*** The dropdown should populate *only the counties and districts under the selected state*.
* The selected county or district should open its respective screen.
* *Note:* The same behavior applies to *county-level login* where applicable.

----

h3. *Realm Tab (Central User)*

* Two dropdowns should be available:
** *Realm Type* (State / County)
** *State*
* When *State* is selected as the realm type:
** All states are populated in the State dropdown.
** After selecting a state:
*** All districts under that state are displayed in the *Unassigned* box.
* When *County* is selected as the realm type:
** All districts directly under the selected state are displayed in the *Unassigned* box.
** All districts under the selected county are displayed in the *Assigned* box.

----

h2. *County User Requirements*

h3. *State Screen*

* A screen should be available to view *all counties and districts under the state*.
* Only the *General* tab should be visible.

----

h3. *General Tab (County User)*

* Editing is *not allowed*.
* Deleting is *not allowed*.
* Only *View* and *History* actions are available.
* A *Change District* icon should be displayed.
** On clicking the icon:
*** The dropdown should populate *only the districts under the selected state*.
*** The user can navigate to the selected district.

## Acceptance Criteria
* Users can only *view and navigate*, not modify data.
* Dropdowns are populated *only with valid entities under the selected state*.
* Navigation correctly opens the selected county or district screen.
* Tabs and functionality differ correctly based on *user role (Central vs County)*.
