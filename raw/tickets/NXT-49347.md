# NXT-49347 — OTP - District - Add & Edit - Google Authenticator

## Description
As a Customer Support user, I want to be able to setup a district’s Google Authenticator One-Time Passcode authentication so that they can use a code sent to them to complete login

*Notes*:

* This story is for adding and editing One-Time Passcode for Google Authenticator 
* Each type of One-Time Passcode will be discussed in separate stories
* Note: all these requirements are the same for FreeOTP, just the name is different, so I recommend doing both at the same time

*Flow:*

* User browses to System > Security page
* User clicks the Add New Google Authenticator One-Time Passcode button
* User is redirected to Google Authenticator One-Time Passcode Add New drawer and completes it
* User saves the config to Keycloak
* User tests the config
* User returns and optionally sets One-Time Passcode as the default authentication type

*Requirements*:

*Security Screen*

* These requirements apply to the System > Security page > Authentication tab:
* 
* Add Google Authenticator One-Time Passcode to the + New drop down button if it is not already there
** On click for Google Authenticator One-Time Passcode :
*** Open the Add New Google Authenticator One-Time Passcode Authentication drawer
* 
* On the grid, 
** Authentication Type column 
*** Ensure it includes Google Authenticator One-Time Passcode 
** Description
*** Default the text with: Login using Google Authenticator One-Time Passcode authentication
** Can mark preferred and Active/Inactive the same as any other method
** Actions
** Edit
*** On click for Google Authenticator One-Time Passcode row, open the Edit Google Authenticator One-Time Passcode drawer
** Delete
*** Covered in delete One-Time Passcode story



*Add New Drawer*

* Create an Add New Google Authenticator One-Time Passcode drawer
** Title: Add New Google Authenticator One-Time Passcode 
* Open this drawer when the user clicks Add New Google Authenticator One-Time Passcode from the dashboard
* 
* Fields
** Behind the scene, set Authentication Method to Google Authenticator One-Time Passcode 
** Note: these are exactly the same fields as FreeOTP, so copy that if you have done it already
** Description
*** Login using Google Authenticator One-Time Passcode 
*** Use same field and requirements as OpenID screen
** Note: the remainder of the fields come from Keycloak, so copy as needed from there
** Have an info button with a tooltip next to each fields
** OTP Type
*** Dropdown list
*** Single select
*** values: Time Based, Counter Based
*** Default: Time Based
*** Required
*** Info button text: Time based is a little more secure. Counter based is a little more user friendly.
** OTP Hash Algorithm
*** Dropdown list
*** Single select
*** values: SHA1, SHA256, SHA512
*** Default: SHA1
*** Required
*** Info button text: SHA1 is least secure.
** Number of Digits
*** Single select list
*** Values:
**** Numbers 4 - 6
*** Default selection is 4
*** Required
*** Info button text: The amount of characters the user must type.
** Look Ahead Window
*** Show only if OTP type = Time Based
*** Dropdown list
*** Single select
*** values: 1-5
*** Default: 1
*** Required
*** Info button text: Extends the OTP Token period as a multiple. Ex: if Period is 30 seconds and you select 2, user will have 60 seconds to complete the task instead of 30.
** Token Period
*** Show only if OTP type = Time Based
*** Dropdown list
*** Single select
*** Values: Numbers 1 - 120
*** Default: 30
*** Required
*** Info button text: Time interval in seconds before a new token will be generated and the user has to enter the code
** Initial Counter
*** Show only if OTP type = Counter Based
*** Dropdown list
*** Single select
*** values: 1-5
*** Default: 2
*** Required
*** Info button text: First counter value
** Dev manager to provide any additional fields needed
** Note: we are managing Enabled and Preferred from the grid on the Authentication dashboard, so not showing them here. If you do show them, then show them read-only
** Set Enabled = False
*** Want to force them to intentionally enable on the grid because there is a lot of business logic to this event and they might want time to test and configure
** Set Preferred = False
*** Same as enabled, want to let them create and test and plan before triggering this logic
* 
* Save button
** On click:
** Verify all required fields are entered
** Register with Keycloak the Google Authenticator One-Time Passcode for the client’s realm
** If there are errors, show them to the user
** If successful, show the user the Success confirmation message
** The grid on the parent page is updated to show this authentication type
** Don’t close the drawer after saving
** Enable the test button
* 
* Test
** Disabled until they successfully save the configuration to Keycloak because Keycloak must be configured first before it will work
** Button
** On click:
*** Open the Test One-Time Passcode screen
*** Follow the process on that screen
*** If they are successful in testing the One-Time Passcode , then everything is good
* 
* Ensure changes are saved to History log
* 
* Security:
** Security screen Add permission required
** Customer Support role can always use
** Grant Security screen Add permission to SchoolCafe Admin role by default
** Deny permission to all other district roles by default



*Edit Drawer*

* Do the same things, but update instead of save new
* 
* Ensure changes are saved to History log
* 
* Security:
** Security screen Edit permission required
** Customer Support role can always use
** Grant Security screen Edit permission to SchoolCafe Admin role by default
** Deny permission to all other district roles by default



*User Imports*

* Ensure that this One-Time Passcode does not impact User import routines



*History tab*

* Ensure that this One-Time Passcode add/edit/delete activities appear in History page

## Acceptance Criteria
* User is able to open the add new drawer for this type
* User is able to open the edit drawer for this type
* User is able to add a new method for this type and have it appear in the grid
* Authentication method is set behind the scenes
* Preferred and Enabled are set according to the requirements
* User is able to test the authentication
* Security page Add permission is required to use the Add New drawer
* Security page Edit permission is required to use the Edit drawer
* Add and Edit changes are logged in History table
