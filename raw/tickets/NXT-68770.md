# NXT-68770 — Backend Calculations for Profit & Loss Report

## Description
This story deals with the profit and loss report that will be generated based on dev work done in Phase 1. Since we are only doing manual entries at this stage, the final report may look different. 

Sample report attached[^sample_profit_loss.pdf]

## Acceptance Criteria
* user selects a reporting period (example: january 1 to january 31)
* create a revenue table
* revenue table must include one row for each gl account that meets both of these conditions:
** account type = revenue
** account appears in at least one manual entry line (debit or credit) where the posting date is inside the selected reporting period
* row label must use the gl account name exactly as stored in the gl accounts table
* for each revenue gl account row, system must calculate:
** description = gl account name
** current period = (sum of credits in period) – (sum of debits in period)
** prior period = (sum of credits in prior period) – (sum of debits in prior period)
** year-to-date = cumulative (credits – debits) from fiscal year start up to the end date of the selected reporting period
** % of revenue = current period ÷ total revenue
** % change from prior period =
*** if prior period ≠ 0 → (current – prior) ÷ prior
*** if prior period = 0 → -
* total revenue = sum of all current period revenue values across all revenue gl accounts
* these exact revenue values (each row + totals) must be used by the dashboard
* create an expenses table
* expenses table must include one row for each gl account that meets both of these conditions:
** account type = expense
** account appears in at least one manual entry line (debit or credit) where the posting date is inside the selected reporting period
* row label must use the gl account name exactly as stored in the gl accounts table
* for each expense gl account row, system must calculate:
** description = gl account name
** current period = (sum of debits in period) – (sum of credits in period)
** prior period = (sum of debits in prior period) – (sum of credits in prior period)
** year-to-date = cumulative (debits – credits) from fiscal year start up to the end date of the selected reporting period
** % of expenses = current period ÷ total expenses
** % change from prior period =
*** if prior period ≠ 0 → (current – prior) ÷ prior
*** if prior period = 0 → -
* total expenses = sum of all current period expense values across all expense gl accounts
* these exact expense values (each row + totals) must be used by the dashboard
* display net income = total revenue – total expenses and subtotals
