# NXT-69493 — Fin - Posting Activity Log

## Description
As a Child Nutrition Director,
I want to view a log of all background posting jobs,
so that I can confirm my financial data is being posted correctly.

h2. Objectives

Provide visibility into background posting jobs so users can quickly identify failures and trust their ledger is current.

## Acceptance Criteria
h1. 

# *Navigation*: Financials > Activity > Posting Activity Log. Page subtitle: "Monitor POS and Inventory posting jobs and their ledger entries"
# *Summary cards*: Four cards at top — Total Jobs (30 Days), Completed (green), Failed (red), Running (orange). Values calculated from last 30 days. If zero jobs, all cards show 0.
# *Status filter*: Chips above table for All / Completed / Failed / Running. Default: All.
# *Job table columns*: Job Name, Type (pill: "POS Posting" or "Inventory Posting"), Status (Completed green, Running orange with spinner, Failed red), Last Run (format: "Jan 2, 2026 at 2:30 PM"), Triggered By ("System" or username).
# *Table behavior*: Default 10 rows, paginated. Shows "Showing 1-10 of X jobs". Refresh button reloads data. Sorted reverse-chronological (newest first), not user-sortable.
# *Expanded row (success)*: Click row to expand. Shows: Ledger Entries Created (count), Total Amount (green, sum of entry amounts), Duration. Below that, "Entries by GL Entry Type" table with columns: Entry Type, Rule (format: "Debit: [Account] → Credit: [Account]" with Debit purple, Credit green), Count, Amount. Below table: "View in Journal →" link navigates to Journal Entries filtered by job date range and entry types.
# *Expanded row (failed)*: Shows red warning container with error icon, "Posting Failed" heading, and specific failure reason (e.g., "Missing GL account mapping for Site: Washington Elementary"). No entries table shown.
## If the user runs the job again and its in progress or it is successful, make a separate entry and disable the action buttons in the failed job. 
# *Empty state*: If no jobs in last 30 days, show centered: "No posting jobs in the last 30 days" with muted icon. Summary cards all show 0.
# *Partial failure*: Jobs that partially succeed then fail display as "Failed" with error message. Partial entries are not shown.
# *Manual trigger*: Out of scope — jobs triggered from Fiscal Periods page, not here.Tech Notes

*Amount calculation:*
Each ledger entry is a balanced double-entry transaction (debit = credit). The "Amount" displayed is the value of one side, not the sum of both. For example, if a Meal Sales Revenue entry debits Cash in Bank $45,000 and credits Meal Sales $45,000, the Amount shown is $45,000.

*Total Amount:*
Sum of all individual entry amounts for the posting job.

*Entry Type Amount:*
Sum of entry amounts grouped by GL Entry Type (e.g., all Meal Sales Revenue entries for this job).
