# NXT-41629 — Student Import - Auto Code Mapping – Use Name from File

## Description
As a SchoolCafé Admin I want to be able to load with button click the codes from the file in the table of mapped codes in the same way as the auto code mapping so that I don’t have to map them manually.

*Requirements*:

* If we can do this for all the imports, would be great, but at a minimum do for Student Import
* 
* On the File Mapping tab, when you click edit on a mapping row, do the following in the drawer that opens up:
* 
* Add a button
* Title = Add from File
* Visibility
** Only show the button if the code mapping was not able to load the values from the file
** If the code mapping did the auto mapping, then hide the button
* 
* On click:
** Load the rows in the mapped values table on the screen with the values from the file the same way you do for auto mapping
** In auto mapping, you are using the column name to match to, but here you are using the link / association created from the drag and drop
** If a row already exists with the value, don’t add it
** If the value you find is a “correct” value (ex: F for Gender), don’t add it

## Acceptance Criteria
* Button exists and it loads the table with unique values from the import file when clicked
* Existing codes are not added as rows to the code mapping drawer
* Correct codes are not added as rows to the code mapping drawer
