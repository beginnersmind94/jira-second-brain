# NXT-33303 — IM - Item Details - Food tab - Allow nutrient entry by percentage

## Description
As a Nutrition Manager I want the ability to enter nutrient values by percentage, at least for nutrients like vitamins, so I can enter nutritionals exactly as they are on their label. 

Users cannot be expected to figure out what the milligram equivalent of 2% of iron is.

+All percentages are based on the+ [+recently updated Daily Values+|https://www.fda.gov/food/new-nutrition-facts-label/daily-value-new-nutrition-and-supplement-facts-labels] +established by the FDA+

* Iron - *18mg*
* Calcium - *1300mg*
* Vitamin A - *900mcg RAE*
** we’ll need to change this from IU to mcg
** I think we can ignore the ‘RAE’
* Vitamin C - *90mg*
* Vitamin D - *20mcg*
** we’ll need to change this from IU to mcg
* Potassium - *4700mg* 
* Moisture - no daily value ?
* Ash - no daily value ?

*UX* [~accountid:5f99ce5044658b0071265de9] *:*

* Reset the grid headers after Protein
** put a bolder line or something to distinguish macronutrients from the vitamins
** Change the ‘% of Calories’ column to ‘Daily Value %’ column for Iron and everything below
* Provide a numerical percentage input field in the Daily Value column
* Keep the existing mg/mcg fields in the Per Serving column
** Implement logic which allows users to use either field
** one field will automatically update based on user input in the other field
* Make sure View and Edit are similar enough
** main point is showing some separation between macronutrients and vitamin nutrients
* In View mode, Cholesterol should not be indented under Total Fat
** it is its own separate nutrient
* +Other Improvements+
** *Disable* the ability to increase numbers by scrolling within a text field. This is a very dangerous area to allow this functionality
** Ensure it continues to be keyboard friendly
