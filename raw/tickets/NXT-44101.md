# NXT-44101 — Custom Reports – Add New Report - Sharing Tab Updates

## Description
We need to make the following enhancements to the Sharing tab

*Notes:*

* What is in the sharing tab is correct per requirements
* However, upon testing, we realized we have a logic error in the original requirements
* Currently, the user is forced to select to share the report with both a user and a role if sharing is enabled
* We shouldn’t do this, they may not want to share with a role, only a user
* This story is to correct that issue.

*Requirements*:

* Please change the required field validation as follows
* Instead of making each field required, can we make it so it requires at least one field have at least one value before they click Next
* Possible outcomes:
** Select at least one User
** Select at least one Role
** Select at least one User and Role
* Invalid Outcomes
** Do not select a User or a Role
* 
* If it doesn’t validate, show an error message:
** Validation text: You must select at least one User or Role, or unselect the Share checkbox.

 

h3.

## Acceptance Criteria
* User can select Users and Roles independently of each other and is not required to select both
* The user is shown an error message if select Shared but neither a User or Role
