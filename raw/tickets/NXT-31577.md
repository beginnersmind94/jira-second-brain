# NXT-31577 — Create Model Management Page

## Description
As a site administrator, I want to view information about my generated models and optionally create or delete one so that I can manage and deploy my models.

*Application*: SchoolCafé

*Flow:*

* Browse to SchoolCafé > Accountability > Food Detection > Model Management
* View builds
* Optionally click Create Model, View Model or Delete buttons

*Background:*

* This was completed in FFD website and matches to the [https://foodtrainingprod.azurewebsites.net/build-model|https://foodtrainingprod.azurewebsites.net/build-model|smart-link]  page
* Copy the page as it is and don’t make any changes
* Get with Saketh if you need help with the code or understanding any technical challenges
* *CRITICAL:* You must copy all of the model building code AS-IS
** It comes from Amniscient team and is very sensitive to changes
** Even slight changes in the past have broken it
** We work with them to maintain this page’s code
** You can update UI stuff, but leave all the model stuff alone

*Requirements:*

* Location: SchoolCafé > Accountability > Food Detection > Model Management
*  
* Page is visible only if the main page is visible
*  
* Copy all the code from the FFD website page called Model Management
* Make your page look the same or as UX team says if they want changes
*  
* Ensure that the report grid has sorting, paging, filtering
* Ensure it exports the grid to excel (currently missing)
* Default sort by Date, most recent first
* Default paging 10 records at a time
*  
* Ensure when click on Create Model, it goes to the Create Model page
* Ensure when click on View Model, it goes to the View Model page
* Ensure when click on Delete, it Deletes the model
*  
* Changes to existing:
** Remove the Menu Items column (just takes up space and adds no value)
** Nice to have: if you could make the columns where we can pull sideways to see more of the text, it would be nice, but can live without
** Progress and Action columns have too much width, would be nice if you could contract them a little bit
*  
* Security Roles:
