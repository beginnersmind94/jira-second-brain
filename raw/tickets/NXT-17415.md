# NXT-17415 — UI-Hide Scheduler controls and IsScheduled column in API Configuration UI

## Description
This story is to cover the tasks needed to build scheduler mechanism for API Calls.

This screen should reflect the settings needed for setting up scheduler on the page where API configuration can be edited.

Provision to enable and disable the scheduler must be included.

Provision to set the scheduler as given below must be made.
1. All 7 days of the week.
2. Start Time and End Time.
3. Repeat to Schedule @ 1,2,3 and 4 hours respectively.

Provision must be made on the API configuration landing page to reflect the scheduler status (enabled or disabled).

## Acceptance Criteria
1. Verify and Validate that the Scheduler set up is in the right page (system-->configuration-->API Configuration-->Actions-->Edit).

2. Verify and validate that the status of the scheduler is reflected on the landing page (system-->configuration-->API Configuration)

3. Verify and validate that the scheduler can be scheduler for all the options available
a. All days of the week, alternate day, one day a week, random days of the week.
b. Verify and validate that the Start Time , End Time and Repeat are mandatory to enable a scheduled trigger.
c. Verify and validate that the Scheduler can be selected to repeat every 1,2,3 and 4 hours respectively.

d. Verify and Validate that the start time and end time is between the school working hours (like 6 am to 5pm etc or 7:00 am to 5:00 PM etc.).

e. Verify and validate that the scheduler runs ONLY between the start and end time.

f. Verify and validate that at any give point, only one vendor is to be enabled. Two vendors cannot be enabled at same time.

g. Verify and validate that scheduler can be enabled only for enabled Vendors.

h. Verify and validate that scheduler configuration will be deprecated if the scheduler is disabled.

4. Verify and Validate that the scheduler can be enabled and disabled from the configuration Page

5. Verify and Validate that multiple configurations can be scheduled at the same time and they are triggered appropriately.

6. Verify and Validate that the scheduler can be enabled and disabled from the landing Page

7. Verify and Validate that during the setting up of scheduler, API configurations are not disturbed and are intact.
