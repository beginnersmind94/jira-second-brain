# NXT-60449 — K12 - WS - Team Online Status Functionality

## Description
*As a* TeamHub user *I want* a way to determine online status of my team members *So that* I can efficiently communicate with them.

h2. Summary Description

TeamHub is a mobile app that displays a list of team members on a dedicated "Team" screen. Each team member is represented by a card showing their name, position, site, mobile, email, and online status. The online status is indicated by:

* *Green:* The user is actively using the app (online).
* *-Red:-* -The user is offline.-
* *Yellow:* The user’s status is unknown (for example, when a status update fails).

This feature is implemented using a real-time WebSocket connection (via [http://socket.io|http://socket.io] ) between the mobile app (built with React Native, Expo, and TypeScript) and the SchoolCafé SaaS server (using Node/Express and MS SQL). A heartbeat mechanism pings the server regularly to ensure that the online presence is up-to-date. The unique identity of a user is defined by a composite key of UserID, PlatformID, and DistrictID (all integers), accommodating multiple platforms and multi-tenant environments, and a responsive UI on the Team screen that handles errors gracefully.  This comprehensive user story, along with the provided code and explanations, offers a complete blueprint for implementing the online presence feature in the TeamHub app.

h2. Requirements:

* *Real-time Detection:*
The mobile app must detect when it moves to the foreground or background and emit appropriate {{userOnline}} and {{userOffline}} events along with the composite key (UserID, PlatformID, DistrictID).
* *Heartbeat Mechanism:*
While the app is active, it should emit a heartbeat event every 30 seconds to update the server with the latest activity timestamp.
* *Data Persistence:*
The server must persist the online status in an MS SQL database using a table and stored procedures that leverage the composite key (UserID, PlatformID, DistrictID).
* *Team Screen Updates:*
The Team screen must fetch team member statuses from the server API every 60 seconds and update the UI.
** In case of an API error, all team members’ online status should be set to “unknown” (displayed as yellow).
* *Error Handling:*
If the status API call fails, the error message should be displayed on the Team screen, and the online status for all members is set to “unknown.”

h2. Flows

h3. Flow 1: Mobile App Online Detection

# *Initialization:*
The mobile app establishes a [http://socket.io|http://socket.io]  connection when launched.
# *AppState Changes:*
#* When the app comes to the foreground ({{active}}), it emits a {{userOnline}} event with the composite key.
#* When the app moves to the background ({{inactive}} or {{background}}), it emits a {{userOffline}} event.

h3. Flow 2: Heartbeat

# *Interval Check:*
Every 30 seconds, while the app is active, it sends a {{heartbeat}} event with the composite key and current timestamp.
# *Server Processing:*
The server logs the heartbeat and can optionally update the database with the latest activity timestamp.

h3. Flow 3: Server Side Processing

# *Socket Events:*
The server receives {{userOnline}}, {{userOffline}}, and {{heartbeat}} events, updates an in-memory store, and broadcasts status updates if needed.
# *API Endpoints:*
#* A REST API endpoint ({{/api/updateStatus}}) uses a stored procedure to update the database.
#* Another endpoint ({{/api/teamStatuses}}) fetches the current statuses from the database.

h3. Flow 4: Team Screen Refresh

# *API Call:*
The Team screen calls the {{/api/teamStatuses}} endpoint every 60 seconds.
# *UI Update:*
#* On success: The app matches each dummy team member’s composite key to update the online status.
#* On failure: The app displays an error message and sets all statuses to “unknown” (yellow).

h3. 1. MS SQL Scripts

h4. Create the Table

Defines the {{UserStatus}} table with a composite key of *UserId*, *PlatformID*, and *DistrictID* (all integers):

{noformat}CREATE TABLE UserStatus (
    UserId INT NOT NULL,
    PlatformID INT NOT NULL,
    DistrictID INT NOT NULL,
    IsOnline BIT NOT NULL,
    LastActive DATETIME NOT NULL DEFAULT GETDATE(),
    PRIMARY KEY (UserId, PlatformID, DistrictID)
);
{noformat}

h4. Stored Procedures

*UpdateUserStatus:*
This procedure updates or inserts a record in {{UserStatus}} based on the composite key.

{noformat}CREATE PROCEDURE UpdateUserStatus
    @UserId INT,
    @PlatformID INT,
    @DistrictID INT,
    @IsOnline BIT
AS
BEGIN
    SET NOCOUNT ON;

    IF EXISTS (
        SELECT 1 
        FROM UserStatus 
        WHERE UserId = @UserId 
          AND PlatformID = @PlatformID 
          AND DistrictID = @DistrictID
    )
    BEGIN
        UPDATE UserStatus
        SET IsOnline = @IsOnline,
            LastActive = GETDATE()
        WHERE UserId = @UserId
          AND PlatformID = @PlatformID
          AND DistrictID = @DistrictID;
    END
    ELSE
    BEGIN
        INSERT INTO UserStatus (UserId, PlatformID, DistrictID, IsOnline, LastActive)
        VALUES (@UserId, @PlatformID, @DistrictID, @IsOnline, GETDATE());
    END
END
{noformat}

*GetTeamStatuses:*
Returns the online status for all team members.

{noformat}CREATE PROCEDURE GetTeamStatuses
AS
BEGIN
    SET NOCOUNT ON;

    SELECT UserId, PlatformID, DistrictID, IsOnline, LastActive 
    FROM UserStatus;
END
{noformat}

_Optional: GetUserStatus procedure to retrieve a single user’s status._

{noformat}CREATE PROCEDURE GetUserStatus
    @UserId NVARCHAR(50),
    @PlatformID NVARCHAR(50),
    @DistrictID NVARCHAR(50)
AS
BEGIN
    SET NOCOUNT ON;
SELECT UserId, PlatformID, DistrictID, IsOnline, LastActive 
FROM UserStatus
WHERE UserId = @UserId 
  AND PlatformID = @PlatformID 
  AND DistrictID = @DistrictID;
END{noformat}

h3. 2. Server-Side Code (Node/Express with [http://Socket.io|http://Socket.io] )

This server code listens for socket events and provides REST API endpoints to update and retrieve statuses from MS SQL.

{noformat}// server.js
const express = require('express');
const sql = require('mssql');
const socketIo = require('socket.io');
const http = require('http');

const app = express();
app.use(express.json());

// MS SQL DB configuration
const dbConfig = {
  user: 'dbUser',
  password: 'dbPassword',
  server: 'localhost', // adjust as necessary
  database: 'TeamHubDB',
  options: {
    encrypt: true, // for Azure
    trustServerCertificate: true // for local development
  }
};

// Create HTTP server and setup Socket.io
const server = http.createServer(app);
const io = socketIo(server, {
  cors: { origin: '*' } // Adjust CORS as needed
});

// In-memory store (for demonstration) keyed by composite key "UserId:PlatformID:DistrictID"
const onlineUsers = {};

io.on('connection', (socket) => {
  console.log('New client connected');

  // User comes online
  socket.on('userOnline', (data) => {
    const { userId, platformId, districtId } = data;
    const key = `${userId}:${platformId}:${districtId}`;
    onlineUsers[key] = true;
    console.log(`User ${key} is online.`);
    io.emit('updateUserStatus', { userId, platformId, districtId, online: true });
  });

  // User goes offline
  socket.on('userOffline', (data) => {
    const { userId, platformId, districtId } = data;
    const key = `${userId}:${platformId}:${districtId}`;
    onlineUsers[key] = false;
    console.log(`User ${key} is offline.`);
    io.emit('updateUserStatus', { userId, platformId, districtId, online: false });
  });

  // Heartbeat event to update last active time
  socket.on('heartbeat', (data) => {
    const { userId, platformId, districtId, timestamp } = data;
    const key = `${userId}:${platformId}:${districtId}`;
    console.log(`Heartbeat received from ${key} at ${timestamp}`);
    // Optionally, update an in-memory timestamp or trigger a DB update here.
  });

  socket.on('disconnect', () => {
    console.log('Client disconnected');
    // Optionally, determine which user disconnected and update their status.
  });
});

// API endpoint to update user status via stored procedure
app.post('/api/updateStatus', async (req, res) => {
  const { userId, platformId, districtId, isOnline } = req.body;
  try {
    const pool = await sql.connect(dbConfig);
    await pool.request()
      .input('UserId', sql.Int, userId)
      .input('PlatformID', sql.Int, platformId)
      .input('DistrictID', sql.Int, districtId)
      .input('IsOnline', sql.Bit, isOnline)
      .execute('UpdateUserStatus');
    res.json({ message: 'Status updated in DB' });
  } catch (err) {
    console.error(err);
    res.status(500).json({ error: 'Database error' });
  }
});

// API endpoint to retrieve team statuses
app.get('/api/teamStatuses', async (req, res) => {
  try {
    const pool = await sql.connect(dbConfig);
    const result = await pool.request().execute('GetTeamStatuses');
    res.json(result.recordset);
  } catch (err) {
    console.error(err);
    res.status(500).json({ error: 'Database error' });
  }
});

server.listen(3000, () => {
  console.log('Server listening on port 3000');
});
{noformat}

*Explanation:*

* The server sets up a [http://socket.io|http://socket.io|smart-link]  connection that listens for {{userOnline}}, {{userOffline}}, and {{heartbeat}} events.
* Each event includes the composite key (*UserId*, *PlatformID*, *DistrictID*).
* REST API endpoints ({{/api/updateStatus}} and {{/api/teamStatuses}}) use stored procedures to update and retrieve user statuses from MS SQL.

h3. 3. Mobile App Code Sample with Heartbeat (React Native with Expo & TypeScript)

This mobile app sample sends online status events and heartbeats to the server using the composite key.

{noformat}// App.tsx
import React, { useEffect, useState } from 'react';
import { AppState, AppStateStatus, Text, View } from 'react-native';
import io from 'socket.io-client';

const SOCKET_SERVER_URL = 'http://your-server-url.com'; // Replace with your server URL

// Replace these with the actual logged-in user's identifiers (as integers)
const USER_ID = 1;
const PLATFORM_ID = 1;
const DISTRICT_ID = 100;

export default function App() {
  const [appState, setAppState] = useState<AppStateStatus>(AppState.currentState);
  const [socket, setSocket] = useState<any>(null);

  // Establish socket connection on mount
  useEffect(() => {
    const newSocket = io(SOCKET_SERVER_URL, {
      transports: ['websocket'],
      reconnection: true,
    });
    setSocket(newSocket);

    // Clean up on unmount
    return () => {
      newSocket.disconnect();
    };
  }, []);

  // Listen for AppState changes to update online presence
  useEffect(() => {
    const subscription = AppState.addEventListener('change', (nextAppState: AppStateStatus) => {
      setAppState(nextAppState);
      if (nextAppState === 'active') {
        // Notify server the user is online
        socket?.emit('userOnline', {
          userId: USER_ID,
          platformId: PLATFORM_ID,
          districtId: DISTRICT_ID,
        });
      } else {
        // Notify server the user is offline
        socket?.emit('userOffline', {
          userId: USER_ID,
          platformId: PLATFORM_ID,
          districtId: DISTRICT_ID,
        });
      }
    });

    return () => {
      subscription.remove();
    };
  }, [socket]);

  // Heartbeat: ping the server every 30 seconds while app is active
  useEffect(() => {
    const heartbeatInterval = setInterval(() => {
      if (appState === 'active' && socket) {
        socket.emit('heartbeat', {
          userId: USER_ID,
          platformId: PLATFORM_ID,
          districtId: DISTRICT_ID,
          timestamp: new Date().toISOString(),
        });
        // Optionally, also update the DB via the REST API.
      }
    }, 30000); // every 30 seconds

    return () => clearInterval(heartbeatInterval);
  }, [socket, appState]);

  return (
    <View style={{ padding: 20 }}>
      <Text style={{ fontSize: 18 }}>Current App State: {appState}</Text>
      {/* Other UI components can be rendered here */}
    </View>
  );
}
{noformat}

*Explanation:*

* The app initializes a socket connection on mount and listens to app state changes.
* It emits {{userOnline}} and {{userOffline}} events with the composite key.
* A heartbeat event is sent every 30 seconds when the app is active.

h3. 4. Team Member Screen Example

This screen fetches team member statuses from the server API every 60 seconds and displays each member’s details. In case of an error, it shows an error message and sets the online status to "unknown" (yellow).

{noformat}// TeamScreen.tsx
import React, { useEffect, useState } from 'react';
import { View, Text, FlatList, StyleSheet, ActivityIndicator } from 'react-native';

interface TeamMember {
  userId: number;
  platformId: number;
  districtId: number;
  name: string;
  position: string;
  site: string;
  mobile: string;
  email: string;
  // online can be true (online), false (offline) or 'unknown'
  online: boolean | 'unknown';
}

// Dummy team member details (could come from another API or local data)
const dummyTeamMembers: TeamMember[] = [
  {
    userId: 1,
    platformId: 1,
    districtId: 100,
    name: 'Alice',
    position: 'Manager',
    site: 'Headquarters',
    mobile: '123-456-7890',
    email: 'alice@example.com',
    online: false,
  },
  {
    userId: 2,
    platformId: 1,
    districtId: 100,
    name: 'Bob',
    position: 'Developer',
    site: 'Branch Office',
    mobile: '234-567-8901',
    email: 'bob@example.com',
    online: false,
  },
  {
    userId: 3,
    platformId: 2,
    districtId: 200,
    name: 'Charlie',
    position: 'Designer',
    site: 'Remote',
    mobile: '345-678-9012',
    email: 'charlie@example.com',
    online: false,
  },
];

export default function TeamScreen() {
  const [teamMembers, setTeamMembers] = useState<TeamMember[]>(dummyTeamMembers);
  const [loading, setLoading] = useState<boolean>(true);
  const [errorMessage, setErrorMessage] = useState<string>('');

  // Fetch team statuses from the server and update the dummy team members
  const fetchTeamStatuses = async () => {
    try {
      const response = await fetch('http://your-server-url.com/api/teamStatuses');
      const statuses = await response.json();
      // Update dummy team members' online status by matching composite keys
      const updatedMembers = dummyTeamMembers.map((member) => {
        const status = statuses.find(
          (s: any) =>
            s.UserId === member.userId &&
            s.PlatformID === member.platformId &&
            s.DistrictID === member.districtId
        );
        return {
          ...member,
          online: status ? Boolean(status.IsOnline) : false,
        };
      });
      setTeamMembers(updatedMembers);
      setErrorMessage('');
    } catch (error) {
      console.error('Error fetching team statuses:', error);
      setErrorMessage('Failed to update team statuses.');
      // On error, update each member to "unknown" online status
      const updatedMembers = dummyTeamMembers.map((member) => ({
        ...member,
        online: 'unknown',
      }));
      setTeamMembers(updatedMembers);
    } finally {
      setLoading(false);
    }
  };

  useEffect(() => {
    fetchTeamStatuses();
  }, []);

  // Set an interval to refresh statuses every 60 seconds
  useEffect(() => {
    const intervalId = setInterval(fetchTeamStatuses, 60000);
    return () => clearInterval(intervalId);
  }, []);

  if (loading) {
    return (
      <View style={styles.centered}>
        <ActivityIndicator size="large" />
      </View>
    );
  }

  const renderItem = ({ item }: { item: TeamMember }) => {
    // Determine the color based on online status
    const statusColor =
      item.online === true ? 'green' : item.online === false ? 'red' : 'yellow';

    return (
      <View style={styles.card}>
        <View style={styles.header}>
          <Text style={styles.name}>{item.name}</Text>
          <View style={[styles.statusDot, { backgroundColor: statusColor }]} />
        </View>
        <Text>Position: {item.position}</Text>
        <Text>Site: {item.site}</Text>
        <Text>Mobile: {item.mobile}</Text>
        <Text>Email: {item.email}</Text>
      </View>
    );
  };

  return (
    <View style={styles.container}>
      <Text style={styles.title}>Team Members</Text>
      {errorMessage ? (
        <Text style={styles.error}>{errorMessage}</Text>
      ) : null}
      <FlatList
        data={teamMembers}
        keyExtractor={(item) =>
          `${item.userId}-${item.platformId}-${item.districtId}`
        }
        renderItem={renderItem}
      />
    </View>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1, padding: 16, backgroundColor: '#fff' },
  title: { fontSize: 24, marginBottom: 16 },
  card: {
    padding: 16,
    marginBottom: 12,
    borderRadius: 8,
    backgroundColor: '#f2f2f2',
  },
  header: {
    flexDirection: 'row',
    justifyContent: 'space-between',
    alignItems: 'center',
    marginBottom: 8,
  },
  name: { fontSize: 18, fontWeight: 'bold' },
  statusDot: { width: 12, height: 12, borderRadius: 6 },
  centered: { flex: 1, justifyContent: 'center', alignItems: 'center' },
  error: { color: 'red', marginBottom: 12 },
});
{noformat}
