# NXT-68295 — Track transaction reversals in data model (Backend)

## Description
Need to link reversed transactions to their originals so we can show reversal status and audit trail in the UI.

* {{transaction_date: 2025-11-01}}

* {{reversed: 0}}

## Acceptance Criteria
* System knows which transaction reversed another transaction (queryable from either direction)
* System knows if a transaction has been reversed
* Reversal metadata is stored: reason, user, timestamp
