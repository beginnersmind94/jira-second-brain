# NXT-6347 — Workspace - New Task Popup

## Description
*User Story*
As a PrimeroEdge User, I have a pop up available for creating, editing, and scheduling new tasks

*Mockup*
[https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=724%3A21152&scaling=scale-down-width&hide-ui=1|https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=724%3A21152&scaling=scale-down-width&hide-ui=1]

*Requirements*

* New Task button appears in the Workspace and the Task screens
* New Task button brings up a pop-up window. (All fields required)
** Task Name - Max 25 chars
** Description - Max 75 chars
** -Assignee- ({color:#ff5630}UI deferred to a future story but back-end {color}{color:#ff5630}*_can_*{color}{color:#ff5630} be covered here if possible. If not, no problem{color})
*** Only visible in Task Hub screen for users with the assign task permission
*** By default, set to user creating the task
** Tasks can have their status updated:
*** New
*** In Progress
*** Done (Marked as complete)
** Due Date
*** Can be current or future date only
** -Recurrence- ({color:#ff5630}Deferred to a future story{color})
* If a task is created in the Workspace, it is assigned to the user by default (we will implement assignees in the future)

## Acceptance Criteria
* Create task pop up present in Tasks screen (if created. If not, item can be deferred)
* Create task pop up present in Workspace
* Tasks can be created and edited using the pop up
* Task details apply as shown above (max char limit, current/future dates only, all fields required)
* Tasks display in the user's My Tasks section of their Workspace, with details according to mock-ups
