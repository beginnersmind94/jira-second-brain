# NXT-6424 — INV - Vendor - Add Vendor Contract - 1st step

## Description
As a Procurement Specialist, I want to be able to create and add a contract to a vendor so that I can define the terms and conditions agreed upon between the vendor and District as it relates to the items to be supplied by the vendor. 

 criteria of the vendor item contract.

*Requirements*:

List contracts on the Vendors page, with an edit icon for each contract

!image-20201027-123407.png|width=91.66666666666666%!

Upon clicking Add Contract, view the first step of the add contract stepper:

!image-20201102-140703.png|width=91.66666666666666%!



* Contract ID
** This a Mandatory entry field
** Alphanumeric data type
** Unique within vendor (regardless of contract status)
** Maximum length - 16 characters
** Special characters allowed - Yes (hyphen, slash, underscore, etc.)
*** “Contract ID already exists. Please enter new Contract ID.”
* Date Range
** These are both Mandatory fields. They define the effective start and end date of the contract. (No purchase order against this contract should be created or received if the order date is outside the defined date range).
*** Start Date
**** Can be a future date but cannot be a past date
**** End Date
**** if user does not enter an End Date, default to 12/31/9999
*** Dates cannot overlap with another contract for that vendor
**** One active contract at a time for a vendor
**** If user is adding the Start Date for a new contract that conflicts with the End Date for an existing contract, require the user to modify the End Date of the existing contract.
**** “Date range conflicts with an existing Active contract. Please adjust dates.”
*** Both dates displayed in MM/DD/YYYY format
*** Use inline calendar picker
**** see comment below (or storybook) for reference of calendar picker
* Status
**  This field is Mandatory
*** The user has the option of selecting one of the below options. And authorized user should have the ability to hide and existing option or add a new one
**** Active
**** Inactive
**** Suspended
**** Marked for Deletion
*** Default to Active for the first new contract of a vendor
* Contract Category
** This field is a Mandatory field. The user has the option of selecting one of the below options. Any authorized user should have the ability to hide and existing option or add a new one.
** Require selection - no default value
*** Dry Goods
*** Baked Goods
*** Dairy 
*** Meats
*** Kitchen Supplies
*** Office Supplies
*** Services
*** Multiple
* Comments
** section for adding any comments related to the contract
** Character limit: 250
** Special characters allowed
*** Example: “Please copy all order emails to mary@vendor.com”
** Allow multi-line entries
* Cancel button
** will exit the Add Contract screen/process
* Next button
** will validate the fields of this screen and check for mandatory attributes
** will proceed to the next step/screen of Add Contract
