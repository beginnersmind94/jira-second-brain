# NXT-36444 — Timeouts - Display Error Message Part 1

## Description
As a SchoolCafé user, I want to be quickly notified if the page has timed out so that I don’t have to sit there waiting on the timeout spinning wheel that never stops.

*Flow:*

* A timeout error occurs
* Code recognizes it and stops the spinning wheel
* System shows the user a message the timeout has occurred with an OK button
* They click OK and the screen stops trying to execute the code that caused a timeout
* An email is sent with the error details to our error email address
* The error is logged and available for error monitoring screen

 

*Notes:*

* This story is part of the stories to expose errors to support staff and follows the story to create an error handling email account
* This story is requested by Bhanu and comes out of the Columbia implementation
* This story is for System only, other modules will do their own after this one is done

*Assumptions:*

* It is assumed that the story to create the error reporting email has been completed
* It is assumed that we have a common way of handling error reporting

 

*Requirements*:

* On all our system pages where a timeout can occur, we need to add error handling for timeouts (not user inactivity, but code) in order to prevent the user to keep waiting on the turning circle
* 
* Do the following:
* Trap the error or otherwise detect that it has or is going to timeout
* 
* *Log Error:*
* Log the error like we normally do, ensuring that it will show up in the Web Error Monitor
* 
* *Show Error Message:*
* Show the user a message:
** Title: Timeout Issue
** Body Short Description: A timeout has occurred. Refresh the screen to continue.
** Under the short description: Error: [insert error code] – [insert error code description]
** From Bhanu: We should have error codes so if user sends a screen shot we may be available to tell what's happening looking at the code. Code can be a mix of module+page+functionality (Ex: ModifyItem recipe: ITMRECMOD). Developers can come up with codes.
* 
* *Send Error Email:*
* We already are sending error emails, so just make sure this email is sent like the others with the same formatting, etc.
* 
* Permissions: no additional requirements necessary
