# NXT-54832 — Family Hub - Super Admin - Payment Guidelines - New Edit Payment Guidelines layout

## Description
As a Cybersoft financial administrator, I would like to make it easier to identify as well as customize a district's convenience fee configuration, so that fewer mistakes are made and custom configurations can be added for certain customers.

*Summary*

When the user clicks the Edit button on this page, open a drawer that combines payment guidelines, convenience/district fee, and bonuses into a single form.

*Details*

* In Family Hub as a Super Admin, in _Configuration -> Payment Guidelines_, do 3 things:

** Update the page layout: 
*** Reorder the page sections in the following way:
***# Convenience Fee Details
***# Payment Limits
***# Bonus Details
***# Payment Gateway Details 
*** Remove the edit pencil icons from Payment Limits and Bonus Details
** When the user clicks the + icon next to Convenience Fee Details, open a new drawer with a new form (see mockups and mind-map below).
*** Changes are mostly UI/default value related.
*** When user saves a new configuration, update the same forms on the page as usual (Payment Limits, Convenience Fee Details, Bonuses should all continue to be updated separately).
** In the new drawer, display a dynamic Sample Receipt that updates based on user selections (more below).

* Requirements for new Convenience Fee Details UI:
** Texts:
*** *Which payment category?*
**** Options:
***** General (Default)
***** Food Service
***** Non Food Service
*** *Who pays the convenience fee on each transaction?*
**** Users
***** Sub-options:
****** Percentage
******* Automatically set to 5%
****** Flat
******* 2 sub-option radio buttons: $2.25 *OR* $2.95
******** Default = $2.25
******** NOTE: When selected category is NFS payments, only $2.95 is available
**** District
***** Sub-options:
****** Percentage
******* Automatically set to 5%
****** Flat
******* 2 sub-option radio buttons: $2.25 *OR* $2.95
****** Default = $2.25
**** Mixed
***** User Percentage
****** Field: set to 0.00% by default, can be modified, must have a value
***** User Flat
****** Field: set to $0.00 by default, can be modified, must have a value
***** District Percentage
****** Field set to 0.00% by default, can be modified, must have a value
***** District Flat
****** Field set to $0.00 by default, can be modified, must have a value
***** Calculations: 
****** Total percentages *must* >=5% if +only+ User Percentage & District Percentage have values. 
****** Total flat fees *must <=$2.95 if* +*only*+ *User Flat & District Flat have values.* 
****** *No restrictions for the user if they select a combination of Flat and Percentage fields have values, as we cannot calculate that accurately and will leave it up to the Super Admin user/4th Floor to determine if the combination is viable.*
*** *The convenience fee will be paid via _____ ,* 
**** Read-only text
***** Minimum
****** If User or District Flat selected, set to $0.01
****** If Percentage set in any (User, District, Mixed) scenario, set to $20.00
***** Maximum
****** If User or District Flat selected and set to $2.25, set to $100
****** If User or District Flat selected and set to $2.95, set to $500
****** If Percentage set in any (User, District, Mixed) scenario, set to $0.00
******* User can/must update this particular value in this scenario
******* Maximum value entered must <= $1000
*** *effective ________*

**** Date must start from tomorrow

*** *Can a bonus amount be applied to payments above a certain dollar amount?*
**** Options: 
***** No (Default)
***** Yes
*** *What type of bonus amount will be added?*
**** Options:
***** Flat
***** Percentage
*** *Apply a bonus amount of $ ______ to payments over $_______* OR 
***Apply a bonus amount of ______ % to payments over $_______*
**** Amount must be <= Threshold
*** *Apply the bonus from ______ to _______*
**** Dates must start from tomorrow
** Other details:
*** Add a Sample Receipt to the drawer so that users can quickly see what a theoretical payment would look like within the current configuration
**** Receipt does NOT need to display when the selected payment category is Non Food Service.
**** Receipt can be copied from existing receipts, with fixed data everywhere except for 4 locations that will change based on user selections throughout the form:
****# Convenience Fee
****## Only displays if convenience fee payer is 'Users' or some User value is entered when selection is 'Mixed'
****## Updates based on fee type and amount
****# Bonus
****## Only displays if bonus selection is 'Yes'
****## Updates based on fee type and amount
****# Total (2 locations on receipt)
****## Only displays if fee amounts are selected
****## Updates based on fee type and amount
****### Is not impacted by District fee or Bonus fee
****## Is always $100.00 + whatever fee type and amount is selected
****### Be careful to properly calculate this when 'Mixed' is selected especially
* Future additions to the page: History. Tooltip next to Bonus (Explain what a bonus is and where funds come from)
* Next steps: add to Agreement wizard
