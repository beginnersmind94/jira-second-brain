# NXT-58930 — SC - Inclusion of Vendor Name and Vendor ID /DEX IDs from GDSN Source 

## Description
_(none)_

## Acceptance Criteria
Navigation: SC - Items - Reports - GDSN Items Export and API

Currently the vendor names are parsed as Unspecified and vendor IDs are parsed as NULL 

For the corresponding Region, when the user clicks Export button, the system should look for Item Codes or Item IDs from GDSN Connect SC table and capture the following:

# Name of the vendor and the vendor Product ID in the corresponding columns
# Dex ID and Dex Local Menu ID

If the data is not available at source (GDSNConnect_SC) we shall parse it as Blank

The change is applicable for both Manual Expor and API Calls 

*GDSN Connect - To check if the items are matched*

Upon login to GDSN Connect - Items, the created items should reflect the corresponding Dex ID, Dex Local Menu ID, Vendor name and vendor Item ID. this will increase the better matching percentage
