# NXT-21248 — Trigger Edge Device to Start Learning Scan

## Description
As an enhancement, we need to add in functionality that will trigger the Edge device to start the scan and upload the image into the cloud when in Learning mode.

*Application*: ExpressPoint

*Background:*

* The Edge device can be in detection mode or learning mode
* When in learning mode, we need to send the Edge device a different message than when in detection mode
* The Edge device needs to be told when to start a scan and then told to upload into the cloud
* This story covers the code update in Edge device to start and stop the scan and trigger the upload

 

*Flow:*

* While using the POS transaction screen with Food Detection enabled and in Learning mode:
* Food Detection event is triggered by selecting a student
* POS send a message to Edge to scan food
* Edge device uploads the scan to the cloud
* POS continues operations as normal

*Assumptions:*

* Edge device has already the code to perform the scan and upload into the cloud

 

*Requirements*:

* Must be using the POS transaction screen with Food Detection enabled and Learning Mode enabled
* Food Detection event is triggered by selecting a student, then:
*  
* POS sends a message to Edge device to start a scan
* Edge device uploads the scan to the cloud
*  
* If System can’t establish connection to the Edge
** Shows message: Edge Not Found
** Logs in history log
*  
* Security
** N/A, leverage existing security for this screen

 

*Mockup*:

* None

## Acceptance Criteria
* With the POS is in Food Detection enabled and Learning Mode enabled
* Food Detection event is triggered by selecting a student
* POS sends a message to Edge device to start a scan
* Edge device uploads the scan to the cloud and is visible in New Images
* The Transaction screen in POS is not affected by any of this and there is no delay noticed
*  
* If System can’t establish connection to the Edge
** Shows message: Edge Not Found
** Logs in history log
