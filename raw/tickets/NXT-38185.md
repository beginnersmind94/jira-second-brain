# NXT-38185 — Device Management – FO – Terminal History Table, APIs ( API ONLY)

## Description
_This is Device Management Phase 1 for Front Office team_



As a SchoolCafé admin, I want to see the history of the updates to the devices so that I can use it in debugging any outstanding issues.

*Flow:*

* FO API is called to Add, edit, delete, suspend or activate a POS device from Device Manager
* API does the relevant action in the API
* API then inserts a record in the History table with the relevant data

*Notes:*

* Device Management has a history screen that enables users to see a history of what has happened for a given POS device
* We need to create a history table to store this data
* This table will be updated when a user does one of the following: Adds, edits, deletes, suspends, activates a POS device from the Device Manager pages
* This table will be written to by FO team when one of these actions occur
* This table will be read from on the POS View History button on the Device Manager page
* System team has multiple dependencies on this table and its APIs being created in order to finish the Device Manager feature

*Requirements*:

* Create a History table for POS Device Management
* At a minimum, the table needs the following columns:
** TerminalID
*** linked to Terminal table’s TerminalID
** Event Type
*** This is which action caused the record to be inserted
*** Values:
**** Add
**** Edit
**** Delete
**** Suspend
**** Activate
** Description
*** Value:
**** The text of the values from the Add and Edit APIs
*** Format: DeviceType=xxx; Site=xxx; Description=xxx
**** Example text:
**** DeviceType=Whatever; Site=Tavola Elementary; description=Left lane in main serving lines
**** For the others, you can leave Description blank
** User
*** The user account logged in that made the change
** Date/Time
*** The date / time stamp of the change

*Stored Procedures*

* Create stored procedure that takes the columns as input values and inserts them into the data table
* Create a stored procedure that takes the device ID and returns all the rows with all the columns for that given device ID
* No update stored procedure needed

*APIs*

* *POS Device History API*
** Used to show the POS Device History Log
*** Parameters:
**** District ID
*** Fields provide:
**** All the columns in the table for that district id
** Get POS Device History Stored Procedure
*** Create or update existing stored procedure to fetch all records for that District ID

## Acceptance Criteria
* History table exists and has the columns mentioned above
* APIs available to surface all history for a given terminal and any edits
