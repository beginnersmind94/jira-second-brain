# NXT-7255 — INV - Create Orders - Apply the Contract Date Range to Delivery Date instead of Order Date

## Description
As an inventory order manager, I want the Delivery Date to be checked against the Contract Date Range instead of the Order Date, so I can ensure that all my orders will be delivered within the contract’s active date range.

This story consists of only back end changes. No UI work is needed.

*Requirements*

* Validate Delivery Date against the contract’s active date range
* Do not validate Order Date against the contract’s active date range
* Update the existing logic to consider Delivery Date instead of Order Date

## Acceptance Criteria
* Delivery Date should be within Contract date range
* Order Date does not have to be within contract date range
