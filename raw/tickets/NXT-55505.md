# NXT-55505 — DMF - Dashboard - License Check

## Description
As a Customer Support role user, I would like to be informed if the district does not have a valid license for the modules I am importing so that I can skip them

*Notes:*

* After our first prototype review, Dallas asked that we be able to avoid importing modules the user is not licenses for
* In practical terms, this really means to decide if we want to import front office and back office sinch the framework and system must be imported no matter what

*Requirements:*

* On the main grid, make these changes:
* When click Add New button, do the following:
* Query the source data and get their list of valid licenses
** Exclude old or expired licenses
** Check if they have a license for Front Office
*** If no
**** Unselect Front Office by default
**** Add an Info note next to that check box: No valid source license
**** Allow the user to reselect it, so don’t make it read only
*** If yes, do nothing
** Check if they have a license for Back Office
*** If no
**** Unselect Back Office by default
**** Add an Info note next to that check box: No valid source license
**** Allow the user to reselect it, so don’t make it read only
*** If yes, do nothing
* The rest of the requirements to skip migrating unlicensed modules are covered in the story for optional migrations, so do those before or at the same time as this one

## Acceptance Criteria
* System auto checks if the district has a valid license for the front office and back office
* User is able to optionally select if they want to migrate front office or back office data
