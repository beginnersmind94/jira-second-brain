# NXT-27927 — INV - Physical Inventory - Valuation Report (Reconciled)

## Description
As a Food Service Director I want a Valuation Report on the Physical Inventory review screen so I can view the financial details of a particular physical inventory.

*Note:* this story should share logic with the main Inventory Valuation Report story 

* [https://primeroedgenext.atlassian.net/browse/NXT-1761|https://primeroedgenext.atlassian.net/browse/NXT-1761|smart-link] 

*Requirements*

* Add a new report to the Physical Inventory screen for inventories which have been counted/completed (Pending Reconciliation, Reconciled)
** Pending Reconciliation: *Projected* Inventory Valuation
** Reconciled: Inventory Valuation
* Move the Discrepancy Report button to a proper location
** (UX) we may not be able to do a single ‘Reports’ button because we require a second selection for the report type for the Valuation report

+Generate the Projected Inventory Valuation Report+ (*Pending Reconciliation* status):

* Done in [https://primeroedgenext.atlassian.net/browse/NXT-7570|https://primeroedgenext.atlassian.net/browse/NXT-7570|smart-link] 

+Generate the Inventory Valuation Report+ (*Reconciled* status):

* This report is provided when the inventory *is reconciled*
* Provide a way to let the user pick which report type (Item Category, Storage Category, Valuation Group, Valuation Group w/ Market Value)
** these were introduced in the original story [https://primeroedgenext.atlassian.net/browse/NXT-1761|https://primeroedgenext.atlassian.net/browse/NXT-1761|smart-link] 
** unfortunately this requires a second selection after selecting the report, because we decided to change this from a system setting (in PrimeroEdge) to a report setting
* This will just be the existing Valuation Report, from the dedicated Reports page
