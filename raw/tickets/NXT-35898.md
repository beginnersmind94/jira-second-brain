# NXT-35898 — IM - Items - Units - Card & Add Serving

## Description
Combine Pack Size and Serving Measures section into a single cohesive section so that the pack-to-serving conversion is not entirely dependent on weight.

This is the main story for building the new combined section to manage all pack size units and serving sizes in one place, while still managing the unique contexts of these separate units.

**Note:* this card will have many view-mode features and will also manage all the business rules for editing, but these initial stories are in the context of the new *create flow*. We will revise as needed the further we get into the item flow.

*Requirements*

* Display a ‘Units’ card in the header of Items next to the general info card
** This card should be editable once the general card is saved
* Provide initial questions in the header bar of the card before enabling the grid
*# Food Item? Y/N
*## if Yes, ask Fluid Item
*## if No, disable Serving Sizes
*# Fluid Item? Y/N
*## if Yes, switch to ‘Volume’ for Serving Sizes
* After questions, load the grid with Add Serving Size and Add Pack Size actions (based on question input)
* Display a header bar area for various prompts or questions provided to the user [https://primeroedgenext.atlassian.net/browse/NXT-37068|https://primeroedgenext.atlassian.net/browse/NXT-37068|smart-link] 
* Provide a button for adding a Serving Size unit - [https://primeroedgenext.atlassian.net/browse/NXT-37109|https://primeroedgenext.atlassian.net/browse/NXT-37109|smart-link] 
** Allow the user to add independent serving sizes
** Amount field
**# if = 1, eligible for having sub-quantity *and* for possibly becoming a pack size (base unit)
**## “Potentially inventory-able serving”
**# if anything other than 1, it’s a serving size only (current behavior)
** Description
**# this should be a hybrid smart text field instead of a dropdown ([NXT-37052|https://primeroedgenext.atlassian.net/browse/NXT-37052])
**# we should still have the standard measures and try to match them in the text field (hence the ‘smart’ text field)
** Weight / Volume
**# Column header label rules
**## Use ‘Weight’ if the item is not fluid
**## Use ‘Weight / Volume’ if the item is fluid ([NXT-37054|https://primeroedgenext.atlassian.net/browse/NXT-37054])
**### this is because we will use fluid measures for the servings but weight for (most of the) pack sizes
**# Provide the separate number and measure dropdown control
* Provide a button for adding Pack Size units - [https://primeroedgenext.atlassian.net/browse/NXT-37110|https://primeroedgenext.atlassian.net/browse/NXT-37110|smart-link] 
* Display the following columns for the grid
** Amount - a field only applicable to Serving Sizes
** Description - a smart text field used for both serving and pack size units 
*** this will reference the new Common Measures page
** Sub-Qty - a field used to link pack size units, and in some rare cases, used to link a serving size to a pack size base unit
** Weight / Volume - field used to define the measurable unit of the item 
*** Use ‘Weight’ header label if the item is not fluid
**** this is the current ‘Weight in Grams’ column
*** Use ‘Weight / Volume’ header label if the item is fluid, because servings will be volume but packs will be weight
**** this Volume column essentially replaces ‘Weight in Grams’
**** this can be thought of like ‘Volume in Milliliter' 
***** and just like the Grams column has Ounce and Pound, the Volume column will also have all the scalable volume units (Quart, Tbsp, etc.)
** Cost / Value - the replacement field for Fair Market Value
*** this will now be editable on all eligible units (pack sizes and servings linked with sub-qty)
*** *Note: may need to allow on serving sizes for ingredient-only items
** Identifiers - a field to show unit level indicators or attributes important for understanding the overall units card
*** this may not be necessary to show in edit/create mode; TBD
** Action - typical column for various edit/delete/save icons
* Require at least one serving size (or pack size if the item is inventory only) before proceeding to next step (*future)
