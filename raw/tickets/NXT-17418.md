# NXT-17418 — INV - Orders - Order Detail - Receipts tab - Edit Receipt Details - Save Receipt

## Description
As an operations manager I want the ability to edit items on an existing receipt so I can update item received quantities or lot info.

This is for saving an updated receipt. There are separate stories for the details of editing a specific part of the receipt: the header ([NXT-15379|https://primeroedgenext.atlassian.net/browse/NXT-15379]), Charges & Discounts section ([NXT-16998|https://primeroedgenext.atlassian.net/browse/NXT-16998]), and Items ([NXT-16997|https://primeroedgenext.atlassian.net/browse/NXT-16997]).

Note: this functionality is dependent on the user having the ‘Edit Receipt’ permission (separate story)

*Requirements*

* While in Edit mode, the Edit button will be replaced with a Save button and a Cancel button
* Cancel button will discard changes and revert to previously saved state.
* Save (Confirm) button will perform validations and save updated receipt
** Confirm button will not be available until all mandatory fields are populated
* Validations:
** Invoice # should be unique across district/region
*** validate uniqueness on Save
*** Show message if not unique: “Invoice # has already been used. Please enter a unique Invoice #.”
** Alert user via prompt if Receipt Amount and Invoice Amount do not match
*** Alert user on click of Save
*** Prompt: If the Receipt Amount ($) does not match the Invoice Amount ($) that the user added, provide a prompt alert: “Are you sure you want to continue? The receipt amount and invoice amount do not match.”
** Prevent negative batches
*** if user has reduced the received quantity for an item, we need to check the batch in case some of that product has been used
*** if product has been used from this batch, prevent the user from saving the updated quantity
**** “Sorry, this receipt cannot be adjusted because some of the received product has already been used.”
*** Example: User previously received 10 cases. Of those 10 cases, 7 have already been used. If user attempts to edit the Received Quantity, it cannot be reduced any lower than 7 cases. 
** Check against closed period (TBD)
