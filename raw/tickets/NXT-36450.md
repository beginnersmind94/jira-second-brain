# NXT-36450 — Monitoring Scheduler – P2 Add New Monitor Page

## Description
As a SchoolCafé admin, I want to be able to add a new monitoring job so that I can proactively monitor the application and ensure higher customer satisfaction.

*Flow:*

* User logs in and browses to Support > Monitoring > Job Scheduler
* User clicks on + New button and is redirected to the Add New Scheduled Job page
* User completes the form and saves the new job

*Notes:*

* This is a continuation of the story to create the Job Scheduler page
* This story assumes that story has been created and that the Monitor tab was created
* This story is based off the SIS Auto Sync Add New page and is following that page’s flow, layout and functionality
* We are only changing the appropriate fields as necessary, so when in doubt, copy what that page does
* This page is the user front end to adding scheduled jobs
** These jobs are created in other stories and not covered here
** Those other stories will include requirements to show on this page

*Requirements*:

* On the Job Scheduler page, do the following using the SIS Auto Sync + New page as the example UI/UX when the user clicks the + New page:
* 
* Sections:
* *Header Section*
** Navigation: System > Support > Monitoring
** Title: Job Scheduler
* 
* *Fields Section*
* Title: Monitors
** This is the section tile in place of API Settings
* Fields:
* Module
** List of modules
** Values will be added by other stories, so leave blank for now
** Instead of Import Type from Export Monitor, have Module
** Its value will then filter the list of values for Monitor list
** Ex: Select Front Office and then only Front Office Monitors are shown
* Monitor
** List of Monitors
** Values will be added by other stories, so leave blank for now
** Instead of Provider from Export Monitor, have Monitor
** Its value is filtered by the list of values from Module list
** All fields required
* 
* *Schedule Section*
* Title: Scheduler
* Show the schedule section like on Syst Auto-Sync page just like it is and with all the existing functionality for that page
* All fields required
* Cancel button
** Just like on Syst Auto-Sync page
** On click, forget what you are doing and go back to the Job Scheduler page
* Save button
** Just like on Syst Auto-Sync page
** The requirements for this button are covered in other stories, for now just have the button and do nothing when clicked
** Be sure to set the Last Updated and Last Updated by values so that the report has the value it needs 
* 
* Permissions:
* In General resources screen for Roles & Permissions
** Visible = false
** Inheritance: Inherits Job Scheduler Add permission
