# NXT-69084 — Fin - Account Mapping Management View

## Description
*As a* district financial administrator, 

*I want* a centralized view to see all system account mappings and edit them inline, 

*so that* I can quickly review mapping status, identify gaps, and make changes without navigating through the wizard flow.

Mockup attached here: [^Account_Mapping_View.html]

## Acceptance Criteria
h2. PAGE HEADER & NAVIGATION

• Page accessible from sidebar under Financials > Configuration > General Ledger Configuration
• Breadcrumb displays: Financials › Configuration › General Ledger Configuration
• Page title: "Account Mapping"

h2. IMPORTANT SETUP ACCORDION

• Collapsible accordion appears at top of content area, collapsed by default
• Header displays:

* Chevron icon (>) on left that rotates 90° when expanded
* Warning triangle icon
* Bold text: "Important Setup:"
* Subtitle text: "This is where you connect SchoolCafe to your district's accounting system. Each data point represents a type of transaction—just tell us which GL account it should post to."
• Expanded content contains the following sections:

What this affects:
• How transactions flow from SchoolCafe to your general ledger
• The accuracy of your financial reports and state submissions
• Period-end closing and ledger entry generation

Automatic Data Points:
• "Meal sales and reimbursement accounts are created from System → Management → Program Configuration. Inventory accounts come from Item Management → Valuation Categories. These post automatically when you close POS or inventory periods."

Manual Data Points:
• "Accounts like salaries, wages, and benefits are not generated by SchoolCafe but are needed for complete financial reporting. Add these using manual journal entries."

Posting dates:
• "The posted date will be the last day of the closed POS or Inventory period. The entered date will be the day you performed the close."

h2. MAPPING PROGRESS BANNER

• Displays below the accordion
• Shows warning state (yellow background, amber border) when mappings incomplete
• Shows success state (green background, green border) when all mappings complete

Warning State:
• Warning triangle icon
• Headline: "[X] system data points unmapped"
• Subtext: "Ledger entries cannot post until all system-generated data points are mapped to a GL account"
• "Complete Mapping" button on right side
• Progress bar fills proportionally with amber/orange color
• Right side displays: "[X] of [Y] system data points mapped"

Success State:
• Checkmark icon
• Headline: "All system data points mapped!"
• Subtext: "Automatic ledger entries are ready to post"
• Progress bar shows 100% filled with green color
• "Complete Mapping" button hidden

Counting Logic:
• Only system-created data points count toward mapping progress
• User-created data points do NOT count toward mapping progress
• Each data point counts as 1 toward total

h2. ACCOUNT ROW STRUCTURE

Each row displays:

* Data Point name (read-only, shows pending indicator dot if changes staged)
* GL Account name (read-only, displays "Not mapped" in gray italic if unmapped)
* Entry Type badge (Automatic/Manual)
* Status badge (read-only) - displays "Active" with green dot or "Inactive" with amber dot
* Created By value ("System" for system data points, “User” for user-created)
* "Edit" button - opens Edit Data Point side drawer

h2. STATUS COLUMN - READ ONLY DISPLAY

* Each account row displays a read-only status badge
* Badge displays:
** Green dot + "Active" text when active
** Amber dot + "Inactive" text when inactive
* Status can only be changed via the Edit Data Point side drawer
* By default, all data points are Active

h2. CREATED BY COLUMN

* Displays who created the data point
• Value: "System" for all pre-installed data points
• Value: "User" for user-created data points

h2. ACTION COLUMN

* ACTION COLUMN
** Each row displays an "Edit" button that opens the Edit Data Point side drawer
* Edit Data Point Drawer:
** Rename: Users can rename any data point (user-created only) via the Name field
** GL Mapping: Users can change GL account mapping via typeahead dropdown
** Delete: "Delete" button appears only for user-created data points; system data points cannot be deleted
* Batch Save:
** All changes (renames, GL mapping updates, deletions) are staged as pending
** Pending changes are applied together when user clicks "Save Changes" in the save bar
** -Status toggle changes apply immediately (not batched)-

h2. INLINE EDITING WITH TYPEAHEAD

• Each account row has a searchable text input for GL Account selection

* EDIT DATA POINT SIDE DRAWER:
**  Opens when user clicks "Edit" button on any account row
** Drawer header: "Edit Data Point"
** Drawer content displays:
** Entry Type badge and Base Type badge (read-only info)
** Data Point Name input (editable, required)
** GL Account typeahead input (required, filtered by base type)
** Created By info (read-only)
** Drawer footer contains:
** "Delete" button (left side, only visible for user-created data points)
** "Cancel" button (closes drawer without saving)
** "Apply Changes" button (stages changes to pending queue)
** GL Account typeahead only shows accounts matching the data point's base type
** Validation: Name must be unique, GL Account is required
** Close drawer with Escape key or clicking overlay
