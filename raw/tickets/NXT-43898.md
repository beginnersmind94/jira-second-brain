# NXT-43898 — MP- Menu Template/ Menus -Check when adding menu item for VMI and Child at the same time (UI ONLY)

## Description
As a menu planner, there needs to be a check against me being able to add a VMI and a child of that VMI at the same time when I am adding menu items to a menu or menu template.

## Acceptance Criteria
* Check when menu items are being added from the add menu item screen that a VMI and a child of that VMI can not be added separately
* If menu items are added together at the same time, only insert the VMI and display a message like the one we display for when a VMI is added after a Child menu item has been added in the past (Reference story [https://primeroedgenext.atlassian.net/browse/NXT-39052|https://primeroedgenext.atlassian.net/browse/NXT-39052|smart-link] 
** Variety menu item (child Items) should take priority in terms of which one stays on the menu 
** Info message “ Menu item(s): '*Menu Item Name”*, “*Menu Item Name*” has now been added as part of the Variety Menu item” (green Font)
*** Display at the top of the page once page reloads with newly added menu items, if there happens to be multiple menu items that were already on the variety menu item show the multiple menu item names within the message 
*** Only display on current refresh, if user leaves page and comes back do not display again, allow user to manually remove the message
