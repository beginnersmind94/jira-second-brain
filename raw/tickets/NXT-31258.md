# NXT-31258 — Platform - HIPAA and FERPA Information Protection

## Description
*Persona*: Perseus User

*Access level:* School District/Site

*Description*:

As a Perseus user, I should be able to enable/disable PII access to any role under role management

* *Sensitive Sections: (HIPAA, FERPA compliance)*
** Allergens
** Eligibility
** Household Information

*Requirements:*

* Each role under role management must have a PII access column so that at each page level, it can tagged whether a specific role has access to PII
* Scope hierarchy is maintained as Page > Section > Fields with implicit permissions
* PII enabled roles are identified in the table with new “PII Access” column

!image-20220426-123645.png|width=1252,height=402!

## Acceptance Criteria
* User shall see the and have ability to enable/disable PII access for any role under role management in system
* Resources with no PII data should be greyed out for “PII Access” flag
