# NXT-60040 — TH - Sec - SC Auth - Verify App Code

## Description
*As a* TeamHub user *I want* to securely link my mobile app to my SchoolCafé (SC) account using a mobile app code *So that* I can enable login on the app.

h3. Comments:

* We need a way to figure out what platform and district to use to login the user on TeamHub
* We have decided that the best way to do that is create a “handshake” using a mobile app code in SC and TeamHub
* There are several parts to this:
** TeamHub District Config screen where the user enters the code (another story)
** SC System Settings where the district decides if it will use TeamHub or not (another story)
** User Profile where the user gets the mobile app code to enter on TeamHub District Config screen (another story) 
** API for TeamHub District Config screen to call that matches the code from the mobile to the user profile (this story) 
* Key security and best practice measures include:
** *Input validation and sanitization* to ensure data integrity and prevent injection attacks.
** *Consistent error handling* with standardized error responses.
** *API versioning* (e.g., using {{/api/v1/}}) for future-proofing.
** *Audit logging* of all requests that captures the inputs and links them to the user ID.
** *Rate limiting* to mitigate potential abuse.

h3. Flow:

# *TeamHub Flow:*
#* User downloads and launches TeamHub.
#* On first launch, the login screen redirects to the District Configuration screen.
#* The screen instructs the user to retrieve their mobile app code from their SC User Profile.
#* User enters the mobile app code into TeamHub.
#* TeamHub sends a POST request to the SC API endpoint, providing:
#** {{code}}: The mobile app code.
#** {{appVersion}}: The current version of the TeamHub app.
#** {{deviceInfo}}: An object containing device details (e.g., device ID, OS type, OS version).
#** {{authToken}}: A token to authenticate the request’s origin.
# *SchoolCafé Flow:*
#* The API endpoint (e.g., {{/api/v1/mobile-handshake}}) verifies the authentication token to ensure the request is coming from an authorized client.
#* It performs robust input validation and sanitization on all incoming data.
#* The endpoint checks if the provided mobile app code exists in the SC mapping table and is associated with an active user account.
#* If valid, the API retrieves and returns the following user details:
#** *platform*: Hardcoded as "SchoolCafé".
#** *district*: The district identifier linked to the user.
#** *userId*: The unique identifier of the user.
#** *email*: The user’s email address.
#* If the code is invalid or the account is inactive, the API returns a standardized error response.
# *TeamHub Flow:*
#* User is redirected to the login screen

h3. Requirements:

* *API Endpoint Setup:*
** The endpoint is available via a secure HTTPS POST request at {{/api/v1/mobile-handshake}}.
** API versioning is implemented to allow future modifications without breaking current integrations.
* *Input Requirements:*
** Request payload must include at a minimum:
*** {{code}} (string, required): The mobile app code.
*** {{appVersion}} (string, required): The version of the TeamHub app.
*** {{deviceInfo}} (object, required): Must include {{deviceId}}, {{osType}}, and {{osVersion}}.
*** {{authToken}} (string, required): Used to authenticate the request.
*** Other fields as necessary
** The API must return an HTTP 400 error if any required field is missing or invalid.
* *Validation & Processing:*
** All incoming data must be validated and sanitized to prevent injection attacks.
** The mobile app code is checked for existence in the mobile app code mapping table and verified to be associated with an active user.
** If valid, the API returns an HTTP 200 response with a JSON payload that includes platform, distict, userID, email and other fields as necessary 
** If invalid, return a consistent error response with an appropriate HTTP status (e.g., 401 Unauthorized or 404 Not Found) and a generic error message.
* *Audit Logging:*
** Every API request must be logged into an audit history that records:
*** All input parameters (excluding sensitive data like raw passwords).
*** The timestamp of the request.
*** The associated user ID.
*** The source of the request (e.g., TeamHub app details).
** Audit logs must be secured and accessible only to authorized personnel.
** Note: later, we will create a monitor screen to review this data
* *Security & Rate Limiting:*
** All communications must occur over HTTPS.
** The API must enforce token-based authorization via the {{authToken}}.
** Implement rate limiting (e.g., maximum requests per IP per minute) to mitigate brute-force and DoS attacks.
* *Performance & Documentation:*
** The API should respond within 2 seconds under normal operating conditions.
** The endpoint must be scalable to handle multiple simultaneous requests.
