# NXT-36766 — IM - Recipe Rework - Split View - Collapsed Mode

## Description
As an Item Management user I want the split view search to be collapsible so I can have a thumbnail only view when wanting to see more of the item/recipe details.

Figma: [https://www.figma.com/proto/OAZ7qTRgdG5bFFGuhDmkoZ/Item-Management-6.0?page-id=3190%3A40604&node-id=3599%3A61887&viewport=2213%2C371%2C0.21&scaling=min-zoom&starting-point-node-id=3190%3A40605|https://www.figma.com/proto/OAZ7qTRgdG5bFFGuhDmkoZ/Item-Management-6.0?page-id=3190%3A40604&node-id=3599%3A61887&viewport=2213%2C371%2C0.21&scaling=min-zoom&starting-point-node-id=3190%3A40605|smart-link] 

*Requirements*

* Provide a collapsible version of the split view which is smaller in width
** Split view should dynamically resize to an extent (percentage)
** Ensure the side panel is still vertically fixed and does not scroll with the item/recipe
* Use image thumbnails only in the collapsed view
** these will still be the selection area, in collapsed mode
** if a new item is selected, navigate to that item, and expand the side panel search
* Provide collapsed icons
** Expand/collapse icon - just expands/collapses the side panel
** Search icon
*** Search icon will open the slideout with focus on search field
** Filter icon
*** Filter icon will open the slideout with the filter section visible
* Leverage the collapsed view state to accommodate full screen modes of Item/Recipe details 
** [https://primeroedgenext.atlassian.net/browse/NXT-36799|https://primeroedgenext.atlassian.net/browse/NXT-36799|smart-link] 
* Define possible thresholds on page size that could automatically trigger collapse/expand states of the side panel search
** e.g. automatically collapse the side panel when page width is too small
* Maintain different interactive states (disabled, etc.)
** [https://primeroedgenext.atlassian.net/browse/NXT-36800|https://primeroedgenext.atlassian.net/browse/NXT-36800|smart-link]
