# NXT-41231 — IM - Item Onboarding - Stock Item Import (Pack Size, Serving linkage)

## Description
As an onboarding district I want a dedicated stock item/pack size import so I can add pack size units to my items in a separate import that allows me to link packs to servings and also allows me to create non-food items.

## Acceptance Criteria
* Provide a separate “stock item” import that handles the non-ingredient related item info
* Categories are provided and mandatory (also provided in Ingredient Import but not mandatory)
* For existing/exported items, populate the primary/default Serving Size in a read-only 
* Provide new ‘# of Servings per next higher unit’ column
** just like the UI, this field is only available if the serving is an ‘inventory-able unit’ (Amount=1, Description not standard measure)
* Provide new ‘Is Serving the Base Unit?' (Yes/No)
** just like UI, this is only available if Sub-Qty (# of servings per next higher unit) is provided
* Provide the remaining Pack Size columns (Base Unit, Sub-Qty, GTIN, FMV)
* Populate the Base Unit columns with the values of new columns if the serving was indicated as the Base Unit
** e.g. ‘1 Sandwich’ is a dual unit, therefore Base Unit fields should be auto-populated with serving info
