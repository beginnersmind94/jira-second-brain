# NXT-47958 — POS - Void Transaction

## Description
As Carmen the Cashier I want to void a transaction I just made to clean up an error quickly and ensure funds are back on the account. This will save time at the end of the day and lower daily staffing costs.

*mockup*

!image-20240312-222502.png|width=1098,height=550!

*requirements*

* Add a special role permission tied to this new option:
** Title: POS - Void Transactions from Transactions List 
* If the user has the permission:
** User can void a transaction via the transactions list.
** User receives a confirmation prompt.
*** Text: Are you sure you want to remove this transaction?
** If user confirms, roll back the transaction.
*** Remove sale, add funds back, etc.
*** Add a suspicious transaction for the void.

## Acceptance Criteria
* Special role permission added
* User can void a transaction after performing the sale if they have the permission.
* User gets a confirmation prompt.
* Transaction is rolled back immediately.
** User should be able to sell to the same account right after and see funds and not get a second meal.
* Voids show up under Suspicious Transactions with transaction details as per usual.
