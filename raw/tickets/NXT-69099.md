# NXT-69099 — Fin - Dynamic Inventory Data Points (Config-Based)

## Description
These data points are created based on valuation categories configured in Item Management.

*Source:* Item Management > Item Configuration > Valuation Categories

## Acceptance Criteria
*For each valuation category, create two system accounts:*

* [Valuation Category Name] — Asset (inventory value)
* [Valuation Category Name] — Expense (cost of goods/usage)

*Business rules:*

* Both asset and expense accounts must be mapped before the category can be used in postings
* Each asset GL account can only link to one valuation category
* Each expense GL account can only link to one valuation category

*Note: This list is independent of active status within Item Management.*
