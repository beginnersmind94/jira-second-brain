# NXT-36955 — Web Error - Create API to Pull Errors from Application Insights

## Description
As a Cybersoft employee supporting customers, I want to view the error log related to my customer so that I can quickly see if I need to address a problem.

*Flow:*

* Browse to the System > Support page
* Click on link to view the error log
* View errors

 

*Notes:*

* This story is part of the stories to expose errors to support staff
* In this story, it is strictly back end development, no UI
* UI will be coved in another story

*Assumptions:*

* It is assumed that we have a common way of handling errors and saving to the couch db

 

*Requirements*:

* Create an API call that provides the error information out of the couch database
* API should include the relevant details at a minimum:
** What environment
** What District
** What module (if you can)
** What page
** Error Type (Web, API, etc.)
** Error Code
** Long description (the specific error)
** Date / Time
** User account logged in (if exists)
** Anything else that is available

## Acceptance Criteria
* API exists and is consumable and has at least these fields:
** What environment
** What District
** What module (if you can)
** What page
** Error Type (Web, API, etc.)
** Error Code
** Long description (the specific error)
** Date / Time
** User account logged in (if exists)
