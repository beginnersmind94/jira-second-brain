# NXT-20422 — UI - Platform - Security - Field Level Resource permissions mapping updates - System

## Description
As a primero edge admin, I should be able to view list of resources and easily filter based on the resource type. Based on the resource type, appropriate ACLs are enabled

+*Requirements:*+

* Module/categories are mapped with specific resources for user to update ACLs/permissions
* Default permissions based on resource type should be VIEW only
* Disable default Full control permissions
* Every resource should be clearly testing and ACLs are applied immediately so that based on ACL, user access is controlled.



Mockup - [Systems 4.0 – Figma|https://www.figma.com/file/PLmXuylGMphY0Bz7WBDCOp/Systems-4.0?node-id=1041%3A14610]

## Acceptance Criteria
# User shall be able to see list of resources for specific modules/categories
# Each resource permissions are loaded as per resource type
# User who does not have appropriate permissions should not be able to access the resource
# User shall be able to see only VIEW permissions
