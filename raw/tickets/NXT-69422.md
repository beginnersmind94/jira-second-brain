# NXT-69422 — Fin - Editing GL accounts

## Description
As the administrator for this application, I want to prevent users from deleting GL accounts without reassigning the associated mapped data point if any to a new GL account.

## Acceptance Criteria
* define “transactions” as: any posted ledger entry line that references the GL account
* if the user attempts to edit a GL account that has *no transactions* but is *actively mapped* to one or more data points, ask the user to reassign those data points in the Account Mapping page
** *if the user tries to change the category it is grouped under, restrict this to account categories with the same base type as the data point*
** Inform user “This account is mapped to a data point(s) so you can only reassign it to a category with the same base type as that data point(s).”
*** Same as applicable when deleting and editing the account category. 
* display an explicit confirmation message: reassignment routes *future postings* for those data points into the newly selected GL account(s)
* after completion, show a summary of changes (data point → old GL account → new GL account)
