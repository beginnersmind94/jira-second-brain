# NXT-70866 —  Blocked Posting Notification

## Description
*As a* district finance manager, *I want to* be notified when automated postings cannot run due to incomplete GL mappings, *so that* I can resolve the gaps immediately and keep my financial data complete.

*Bell Notification*

* Fires once when a posting job is blocked by the readiness gate
* Appears in POS, Inventory, and Financial Oversight — wherever the user is
* Message: _"Financial postings could not run. 2 required GL mappings are incomplete."_
* Count is dynamic — always reflects current number of gaps
* Tapping navigates to Account Mapping page, pre-filtered to show only incomplete mappings
* Does not re-fire for the same gap state — one notification per gate activation
* New notification fires only if the gap count changes

*On-Screen Banner (Posting Activity Log)*

* Yellow informational banner pinned to top of Posting Activity Log while gate is active
* Same message, same count, same one-click link to filtered Account Mapping
* Disappears automatically when all required mappings are complete — no dismiss button while gaps remain

*What we are NOT building*

* No email alerts or escalation chains
* No toast that auto-dismisses
* No notification per individual missing mapping
* No notification when the gate clears — postings just resume silently
* No inline warnings on other pages beyond the banner on Posting Activity Log

## Acceptance Criteria
*Bell Notification*

* When the readiness gate blocks a posting job, a bell notification is created
* Notification appears in POS, Inventory, and Financial Oversight modules
* Notification text: _"Financial postings could not run. [X] required GL mappings are incomplete."_ where [X] = current count of incomplete mappings
* Tapping the notification navigates to Account Mapping page filtered to show only incomplete mappings
* Only one notification per gate activation — do not create a new notification for each blocked job
* If the gap count changes (e.g., user fixes one mapping but a new gap appears), fire a new notification
* If the user resolves all gaps, do not send a "resolved" notification — postings resume silently on the next cycle

*On-Screen Banner*

* When the readiness gate is active, display a yellow informational banner pinned to the top of the Posting Activity Log page
* Banner text: same message and dynamic count as the bell notification
* Banner includes a link to Account Mapping page, pre-filtered to incomplete mappings only
* Banner has no dismiss button — it remains visible as long as gaps exist
* Banner disappears automatically when all required mappings are complete

*Edge Cases*

* If the user has no access to the Account Mapping page, the notification and banner still appear but the link is not shown
* If multiple posting jobs are blocked in the same cycle, only one notification fires — not one per job
* If the system is in Not Ready state at the time of the very first posting job after module setup, the notification fires immediately — do not wait for a second cycle
