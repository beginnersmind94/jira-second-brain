*reqs*

* User needs a way to select applications from any of the grids.
** multi-select checkboxes for each line
** 'select all' option at the top of the grid.
*** this should select all items in the grid, including pagination
* If an app is selected, a Notify button becomes available.
** We’ll need to make an update to the notification drawer to allow the user to select a letter to send first.
** The available options is available depending on which tab they’re on.
** Incomplete Options:
*** Selection Notice
*** Follow-up Notice
**** This is only available for applications that have had the Selection Notice sent first.
** Show the user how many of each letter are available to send.
*** For example - user selects 10 apps, 6 have the Selection Notice sent, 1 does not have a Selection Notice sent. If the user selects all 10 apps and clicks Notify, they’d see:
**** Selection Notice (10)
**** Follow-up Notice (9)
** Depending on the selection, a different letter template needs to be used:
*** Selection Notice → Verification Notice
*** Follow-up Notice → Verification Subsequent Notice (can we rename this to Follow-up Notice?)
** After making a selection, the typical Notify drawer continues.
** User can change their letter selection and counts should update appropriately.
* As part of this, we should perform a failover check for contact information:
** If the applicant has no address/email, start checking the students on the app.
** Start with the youngest student by DOB, check their primary guardian.
** Cursor through each student until contact address or email can be found.
* The letter needs to be added to the notification history for the application.
** Who, when, letter, method (email/print)

## Acceptance Criteria
User can select applications from the various lists, select all applications or individual.

If a selection is made, the Notify button is available for selection.

* User can select which letter they are sending:
** Selection Notice
** Follow-up Notice
* User can send notifications per the usual Notify drawer.
* User can change the letter they’re sending and counts should refresh appropriately.

The selection options for Letter (that aren’t the Selection Notice, which is always available) are available if an application meets requirements:

* Follow-up Notice → a Selection Notice has been sent already.

As part of this, we should perform a failover check for contact information:

* If the applicant has no address/email, start checking the students on the app.
* Start with the youngest student, check their primary guardian.
* Cursor through each student until contact address or email can be found.

Letters send appropriately via email or print.

Copy of the letter is saved to the application and to the student(s) involved.
