# NXT-26386 — Acct Mgmt - Accounts - Add filters and column to support Patron Balance view

## Description
As a central office admin, I would like to be able to generate a list of accounts whose balances were within a certain range, so that I can have a report of those accounts for my staff to reach out to them as needed.

*Summary*

In order to allow users to pull up this list (formerly called the ‘Patron Balance’ report in Classic), we will need to integrate a new filter and column in the Accounts page.

*Details*

In Account Management → Accounts, add the following:

# In the list of filters: 
## Add a drop-down filter called *Account Balance* which contains 4 options:
###  All (default option selected. No change in behavior from currently)
### Above
#### When this option is selected, display a Minimum Balance field as well. In that field, display a placeholder amount of $0.00 which can be edited by the user also.
#### When the user clicks Apply, display only those accounts whose Balance is > the Minimum Balance value.
### Below
#### When this option is selected, display a Maximum Balance field as well. In that field, display a placeholder amount of $5.00 which can be edited by the user also.
#### When the user clicks Apply, display only those accounts whose Balance is < the Maximum Balance value.
### Between
#### When this option is selected, display a Minimum Balance and a Maximum Balance field as well. In the Minimum Balance field, display a placeholder amount of $0.00 which can be edited by the user also. In the Maximum Balance field, display a placeholder amount of $5.00 which can be edited by the user also.
#### When the user clicks Apply, display only those accounts whose Balance is between the Minimum Balance and Maximum Balance values.
## Add 2 checkboxes (both *unchecked* by default):
### Exclude $0.00
#### When this box is checked and the user clicks Apply, display only those accounts whose Balance does not equal $0.00.
### Exclude No Charge
#### When this box is checked and the user clicks Apply, display only those accounts who do not have the ‘No Charge’ restriction.
# In the grid, add a new column to the ‘Columns & Views’ button list: *Can Charge*.
## In this column, display No Charge restriction status for any applicable has account: Yes (no restriction) or No (account cannot charge)
## The user should be able to use field search to filter this column
## This column should be hidden by default
# The Export button should now include the Can Charge column if it is added.

## Acceptance Criteria
Verify that the user can still search for any account regardless of balance 

Verify that the user can search for any account whose balance is above a given amount

Verify that the user can search for any account whose balance is below a given amount

Verify that the user can search for any account whose balance is between two given amounts

Verify that the user can exclude any account whose balance is $0.00

Verify that the user can exclude any account with a ‘No Charge’ restriction

Verify that the user can show or hide the ‘Can Charge’ column within the grid

Verify that this column can be filtered through Field Search
