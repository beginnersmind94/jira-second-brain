# NXT-4381 — System - Roles - Edit Role Screen - Layout

## Description
*User Story*

As a PrimeroEdge admin, I have a screen where I can either view or edit permissions assigned to a role.

*Requirements*

* Summary: Create edit role view for all modules.
* The Edit Role Screen can be accessed by clicking on any role on the Role Screen
** If the user only has view access, they can still reach this screen, but cannot edit anything
*** By default, all Standard Roles are view-only for everyone
** If user has edit access, they can make changes and save them
* The Edit Role Screen has an “All” view that displays permissions for all available modules in collapsable sections
** By default, the sections are collapsed
** This is the view the user is presented with when clicking on the role to edit
** User can select individual modules or global settings from a dropdown to only view specific module permissions
*** These can be reached directly from the role screen by clicking on the individual module beneath the role
* Each module has different sections of permissions
** Page Permissions - Determines level of access on a per page basis - Full Access, View, Edit, Add, and Delete
*** Some pages will not have all options depending on need
*** Each module has checkboxes that allow for selecting all of any one of these permissions at once.
** General Permissions - Permissions that are related to the module but don’t fit within page access permissions
*** Note: The API should be built in a way that allows developers working on other modules to add additional sections as needed here
* Changes are retained when moving between modules, but only saved when the user clicks the “Save Changes” button
** If the user attempts to navigate away, a message appears warning of unsaved changes if they exist
* This is a general story for creating the layout of the page, with specific permissions/saving for the System module being handled in [https://primeroedgenext.atlassian.net/browse/NXT-2968|https://primeroedgenext.atlassian.net/browse/NXT-2968|smart-link] & [https://primeroedgenext.atlassian.net/browse/NXT-2969|https://primeroedgenext.atlassian.net/browse/NXT-2969|smart-link]

## Acceptance Criteria
* Edit Role Screen can be accessed and get the list of related permissions for a module
* Permissions can be selected/removed and saved
* User can switch between the all, global, and module pages to view/update their various permissions
