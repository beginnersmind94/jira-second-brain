# NXT-43897 — MP- Find and Replace- Replacing menu item with a child menu item already on Menu as part of VMI

## Description
As a menu planner, I should not be able to replace a menu item with a child menu item that already belongs on a Menu as part of a VMI.

## Acceptance Criteria
* Do not allow menu item to be replaced with a child menu item of VMI already on a particular menu 
* Gray out the check box and provide tool tip “Menu Item already on Menu as part of VMI: Name”
* Allow users to still select all and select all the menus to be replaced but skip the ones that can not be allowed
* All menus should be able to be find and replaced at the same time
* These changes should sync to production and need to be confirmed for any plan incomplete or plan complete records
** We should not effect any plans that are already part of a consolidated plan *complete*
