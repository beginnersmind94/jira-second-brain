# NXT-9303 — Data Exchange - SIS Import - Trigger Direct Certification check against new and updated students

## Description
(This may already be handled by the work the SIS dev is doing)

As a district admin I want the system to automatically check freshly enrolled and freshly reactivated students to see if there’s a DC match waiting for them.

*mockup*

n/a

*requirements*

* System makes a call to DC matching for a student after the student import completes and when they are:
** created
** made active again in the system
** updated information is applied to the student (dirty record flag) - such as a name change that may trigger a match

## Acceptance Criteria
System attempts to DC match students who are newly created or reactivated.

If enough points are given, student is automatically DC matched.
If not enough points, student is available as a potential DC match - if enough points are available.
Student's eligibility is not updated unless an automatic match happens.
