# NXT-42749 — Custom Reports – Add New Report – Configuration Tab - Remainder Tabs

## Description
*Notes:*

* This is a continuation of the Custom Reports Configuration tab
* The story has been broken down into smaller parts
* The full story can be seen under the Source Table story
* This part covers the Default Sort, Arguments, Group By and Having tabs



*Requirements*:

 *Default Sort Section*

* Add a small table to the page for the ORDER BY clause
** Table Name = Default Sort
** Table column Names:
*  Source Field
** Rows have:
** Single select list
** No default value selected
** Values = list of field names of the source table
** First value of the list is empty field
** Required if shown, not required if not shown
* Direction
** Rows have:
** Single select list
** Values: ASC, DSC
** Default Value is ASC
** Required if shown, not required if not shown
* Actions
** Rows have:
** Delete button
** Don’t have to show the Actions column label
** On click, ask the user to confirm they want to delete and then delete that row from the table
* User can drag and drop the rows to set the order of the sorting
* Table has an add button to add a new row
** Default has no rows
** Don’t allow more than 3 rows in the table so that they don’t get carried away with it
* If table is blank, ignore the ORDER BY clause
* If has values, validate the required fields

·          

 

h4. Arguments Section

* Add the ability to add Arguments to the Select clause
* Add a field:
** Check Box list
** Title: Arguments
** Required = True
** Values:
*** None (Default Value)
**** If None selected, can’t select others
**** If unselected, then show others
*** ALL
*** DISTINCT
**** Put these on the same row
**** They are mutually exclusive, can only select one of them or none of them
**** So if select one, unselect the other
**** They can also unselect the selected one and leave both blank
*** TOP
*** Count
**** Textbox that shows how many to use for Top
*** PERCENTAGE
*** WITH TIES
**** These should be on one row
**** If they select TOP, then they can optionally select none, both or one of the other 2 choices
**** They cannot choose the last 2 options w/o selecting TOP
** Example:
*** SELECT TOP(5) PERCENT JobTitle FROM Employees

 

h4. Group By section

* Add the ability to add a GROUP BY clause
* Add a field:
** Multi-select list
** Title: GROUP BY
** Required = False
** Values: list of source fields
** Add as chips in the box
** The order they add them is the order of the fields for the GROUP BY clause

 

h4. Having Section

* Add a small table
** Title: Having
* Data = Select field name from the selected data source field from the Selection screen
* The table default to no rows
* User will have to add rows by clicking the add button
* Table column Names:
** Source Field
** Aggregate
** Argument
** Operator
** Value
** Delete button
* Table has an add button
** On click, add a row
* In each row, the user can select the source field by selecting from a list
** Column Title: Source Field
** Each row has:
** Single select list
** Values: list of source field names
** Required = true if shown
** First value is empty string
* In each row, the user can create an Aggregate function on the field by selecting from a list
** To do this, add a column in the Source table
** Column Title: Aggregates
** Each row has a single select drop down list
** Required = False
** Values:
*** Empty text (Default Value)
*** APPROX_COUNT_DISTINCT
*** AVG
*** CHECKSUM_AGG
*** COUNT
*** COUNT_BIG
*** GROUPING
*** GROUPING_ID
*** MAX
*** MIN
*** STDEV
*** STDEVP
*** STRING_AGG
*** SUM
*** VAR
*** VARP
** If they leave blank, ignore
** If they select a value, apply the function to the field in the SQL statement (ex: SELECT AVG(UnitPrice) FROM Sales)
* In each row, the user can create an Argument on the field by selecting from a list
** To do this, add a column in the Source table
** Column Title: Arguments
** Each row has a single select drop down list
** Required = False
** Values:
*** Empty text (Default Value)
*** ALL
*** DISTINCT
** If they leave blank, ignore
** If they select a value, apply the argument to the field in the SQL statement (ex: SELECT DISTINCT ProductName  FROM Products)
** To use the Aggregate and Argument together, the syntax is like this: (SELECT AVG(DISTINCT ListPrice)  FROM Products)
* In each row, the user can create an Operator on the field by selecting from a list
** To do this, add a column in the Source table
** Column Title: Operators
** Each row has a single select drop down list
** Required = True
** Values:
*** Empty text (Default Value)
*** =
*** <> 
*** > 
*** >=
*** < 
*** <=
* In each row, the user can set a value for the Operator on the field
** To do this, add a column in the Source table
** Column Title: Value
** Each row has a textbox
** Required = True
** Allow the user to type in something
* Example: SELECT Department, sum(Salary) FROM employee GROUP BY department HAVING SUM(Salary) >= 50; 
** Note: Sum is the Aggregate, Salary is the source field, >= is the Operator and 50 is the Value (there is no Argument in this example)
* Delete button
** Do the usual confirmation and remove from the row if they confirm
* Put a limit of 5 rows on this table so that they don’t go crazy
* If they add more than one row, use the AND statement for the Having Clause
