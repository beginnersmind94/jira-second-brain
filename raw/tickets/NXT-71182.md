# NXT-71182 — FIN — Reports: Balance Sheet

## Description
As a district finance user, I want to run a balance sheet report that shows the district's assets, liabilities, and equity/fund balance for a selected period, with beginning balances, period activity, and ending balances clearly presented, so that I can confirm the financial position of the nutrition fund and reconcile with my bank statement.

{quote}*A sample report is attached with this story for visual reference.*{quote}

----

h2. Context

The balance sheet is a formal financial statement that presents permanent accounts (Assets, Liabilities, Equity) alongside the net effect of temporary accounts (Revenue and Expenditures) in a structured, stakeholder-readable format.

The report is governed by the *Expanded Accounting Equation*:

{noformat}Assets = Liabilities + Equity + Revenue − Expenditures{noformat}

Which can also be expressed as:

{noformat}Assets = Liabilities + Equity + Net Income{noformat}

Where:

||Term||Definition||
|*Assets*|What the district owns (Cash, Receivables, Inventory, Fixed Assets, etc.)|
|*Liabilities*|What the district owes (Accounts Payable, Accrued Liabilities, Unearned Income, etc.)|
|*Equity*|Fund Balance accounts — the district's accumulated net position from prior periods|
|*Revenue*|Income earned during the selected period (meal sales, reimbursements, etc.)|
|*Expenditures*|Costs incurred during the selected period (food, labor, supplies, etc.)|
|*Net Income*|Revenue − Expenditures from the fiscal year start date through the report end date|

Revenue and Expenditure accounts do not appear as individual line items on the balance sheet. Their net effect flows into the Equity section as a single calculated line: *Net Income*.

The report can be built and run regardless of whether an opening balance import has been performed. All values are derived from ledger entries for the selected date range.

----

h2. Requirements

h3. 1) Outcome

1.1 A district finance user can run a balance sheet report showing Assets, Liabilities, and Equity/Fund Balance for a selected period.

1.2 The report shows beginning balance, period activity, and ending balance for each account so the user can trace movement during the period.

1.3 The report satisfies the expanded accounting equation at the ending balance level:

{noformat}Total Assets = Total Liabilities + Total Equity + Net Income{noformat}

If this equation does not balance, the report must display a prominent warning.

h3. 2) Report Parameters Page

h4. 2.1 — Page Title and Guidance

* *Page title:* "Balance Sheet Report"
* *Helper text* (directly under title): "Choose a fiscal year, reporting period, and sites. When multiple sites are selected, each site gets its own Balance Sheet worksheet in the Excel file."
* *Tooltip* (on the export/generate button): "You'll be notified once your file is ready and can download it from the Document Center."

h4. 2.2 — Parameters

||Parameter||Required||Default||Selection||Notes||
|Fiscal Year|Yes|—|Single select|Dropdown of available fiscal years|
|Reporting Period|Yes|—|Single select|Determines the date range for the report; see 2.3|
|Site Type|No|All|Multiselect|Filters which sites are included; see 2.4|
|Site|No|All|Multiselect|Filters to specific sites within the selected Site Type(s); see 2.4|

h4. 2.3 — Reporting Period Options

The Reporting Period dropdown offers four options. Each option resolves to a specific start date and end date that defines the report's date range.

||Option||Additional Input Required||Start Date||End Date||
|*Single month*|Month selector|First day of the selected month|Last day of the selected month|
|*Quarter*|Quarter selector (Q1, Q2, Q3, Q4)|First day of the quarter's first month|Last day of the quarter's last month|
|*Fiscal YTD*|None|First day of the selected fiscal year|Last day of the most recent closed period, or today if no periods are closed|
|*Custom date range*|Start date and end date pickers|User-selected start date|User-selected end date|

All calculations throughout this report (Beginning Balance, Period Activity, Ending Balance, Net Income) use the resolved *start date* and *end date* from the selected Reporting Period option.

* *Beginning Balance* = cumulative ledger position as of the day before the start date
* *Period Activity* = net change from start date through end date (inclusive)
* *Ending Balance* = Beginning Balance + Period Activity
* *Net Income* = Revenue − Expenditures from start date through end date (inclusive)

h4. 2.4 — Site Type and Site (Cascading Multiselect Filters)

Site Type and Site are both *multiselect* dropdowns that operate as a *cascading filter pair*. Site Type narrows the pool of available sites; Site optionally selects specific sites within that pool.

*Filter behavior:*

||Site Type||Site||Report Scope||
|All (default)|All (default)|All sites in the district — full district rollup|
|All|One or more specific sites selected|Only the selected sites|
|One or more types selected (e.g., Elementary + Middle School)|All|All sites belonging to any of the selected types|
|One or more types selected|One or more specific sites selected|Only the selected sites (which must belong to the selected types)|

*Cascading logic:*

* When the user changes the Site Type selection, the Site dropdown refreshes to show only sites that belong to the selected type(s), plus the "All" option.
* When Site Type is "All", the Site dropdown shows all sites in the district.
* If specific sites were previously selected and the user changes Site Type such that one or more of those sites no longer belong to the selected type(s), those sites are deselected. If no sites remain selected, Site resets to "All".
* The user can select multiple Site Types (e.g., Elementary + Middle School) and then further narrow to specific sites within those types.

*"All" behavior:*

* "All" is the default for both dropdowns.
* Selecting "All" in Site Type clears any previously selected types and shows all sites in the Site dropdown.
* Selecting "All" in Site clears any previously selected individual sites.
* Selecting a specific value when "All" is currently selected automatically deselects "All".

h3. 3) Report Structure

3.1 The report lists accounts grouped into three top-level sections in order:

*Assets*

* Account category groups (expandable), each with a subtotal row
* *Total Assets*

*Liabilities*

* Account category groups (expandable), each with a subtotal row
* *Total Liabilities*

*Equity / Fund Balance*

* Account category groups (expandable), each with a subtotal row
* *Net Income* — a calculated line (not a GL account); see formula in Section 4
* *Total Equity* — sum of all Equity account ending balances + Net Income

*Total Liabilities and Equity* = Total Liabilities + Total Equity

h4. 3.2 — Account Category Grouping

Within each top-level section (Assets, Liabilities, Equity), accounts are organized into *account category groups* based on the category assigned to each account in the chart of accounts (e.g., Current Assets, Fixed Assets, Current Liabilities, Long-Term Liabilities, etc.).

Each category group:

* Is *expandable/collapsible* — collapsed by default, showing only the category name and its subtotal row
* When expanded, reveals the individual account rows belonging to that category
* Displays a *subtotal row* showing the sum of Beginning Balance, Period Activity, and Ending Balance for all accounts in the category

Example when expanded:

{noformat}Assets
  ▼ Current Assets
      1010  Cash                          10,000.00    2,000.00    12,000.00
      1020  Accounts Receivable            5,000.00    1,000.00     6,000.00
      Total Current Assets               15,000.00    3,000.00    18,000.00
  ▼ Fixed Assets
      1510  Equipment                     25,000.00        0.00    25,000.00
      Total Fixed Assets                  25,000.00        0.00    25,000.00
  Total Assets                            40,000.00    3,000.00    43,000.00{noformat}

h4. 3.3 — Accounts Without a Category

If an account does not have a category assigned in the chart of accounts, it appears under an *"Other"* group at the bottom of its section (e.g., "Other Assets", "Other Liabilities", "Other Equity"). This group behaves identically to any other category group — expandable with a subtotal row.

h4. 3.4 — Balance Check

The report must satisfy this check at the ending balance level:

{noformat}Total Assets = Total Liabilities and Equity{noformat}

If they are not equal, a prominent warning is displayed showing the difference.

h4. 3.5 — Net Income Placement

Revenue and Expenditure accounts do not appear as individual line items. Their net effect, cumulative from the fiscal year start date through the report end date, is:

{noformat}Net Income = (Revenue Credits − Revenue Debits) − (Expenditure Debits − Expenditure Credits){noformat}

This value appears as a single line within the Equity section, below all Equity category groups and above Total Equity. Net Income is not part of any category group.

h3. 4) Column Structure

||Column||Description||Formula||
|Account Code|GL account code|—|
|Account Description|Account name|—|
|Beginning Balance|Cumulative ledger position as of the day before the report start date|See 4.2|
|Period Activity|Net change from report start date through report end date|See 4.3|
|Ending Balance|Position as of the report end date|Beginning Balance + Period Activity|

h4. 4.1 — Balance Presentation

The balance sheet expresses values as net amounts per account — not split into separate debit and credit columns.

h4. 4.2 — Beginning Balance

Beginning Balance is the cumulative net position of the account from all ledger entries *prior to* the report start date.

||Account Type||Beginning Balance Formula||
|Assets|Sum of all Debits − Sum of all Credits for all entries before the report start date|
|Liabilities|Sum of all Credits − Sum of all Debits for all entries before the report start date|
|Equity / Fund Balance|Sum of all Credits − Sum of all Debits for all entries before the report start date|

When Site Type(s) or specific Site(s) are selected, Beginning Balance is scoped to ledger entries for the matching sites only.

A positive Beginning Balance means the account carries its expected (normal) balance. A negative Beginning Balance means the account carries a balance opposite to its normal balance.

h4. 4.3 — Period Activity

Period Activity is the net change in the account from the report start date through the report end date (inclusive).

||Account Type||Normal Balance||Period Activity Formula||Positive Value Means||
|Assets|Debit|Period Debits − Period Credits|Asset increased|
|Liabilities|Credit|Period Credits − Period Debits|Liability increased|
|Equity / Fund Balance|Credit|Period Credits − Period Debits|Equity increased|

When Site Type(s) or specific Site(s) are selected, Period Activity is scoped to ledger entries for the matching sites only.

h4. 4.4 — Ending Balance

{noformat}Ending Balance = Beginning Balance + Period Activity{noformat}

This applies to every account row individually.

h4. 4.5 — Net Income (calculated line)

Net Income is not a GL account. It is a calculated value that appears in the Equity section.

{noformat}Net Income = (Revenue Credits − Revenue Debits) − (Expenditure Debits − Expenditure Credits){noformat}

Scoped from the fiscal year start date (of the selected Fiscal Year) through the report end date, inclusive. This captures the unclosed result of Revenue and Expenditure activity — the portion that has accumulated inside Asset and Liability account balances but has not yet been swept into Fund Balance by Year End Close.

Where:

* *Revenue Credits* / *Revenue Debits* = Sum of all credit / debit entries posted to Revenue accounts from the fiscal year start date through the report end date
* *Expenditure Debits* / *Expenditure Credits* = Sum of all debit / credit entries posted to Expenditure accounts from the fiscal year start date through the report end date

When Site Type(s) or specific Site(s) are selected, Net Income is scoped to ledger entries for the matching sites only.

A positive Net Income means revenues exceeded expenditures. A negative Net Income (displayed in parentheses) means expenditures exceeded revenues.

h4. 4.6 — Category Subtotals

Each account category group displays a subtotal row:

{noformat}Category Subtotal (Beginning Balance) = Sum of Beginning Balances for all accounts in the category
Category Subtotal (Period Activity)   = Sum of Period Activity for all accounts in the category
Category Subtotal (Ending Balance)    = Sum of Ending Balances for all accounts in the category{noformat}

h4. 4.7 — Section Totals

{noformat}Total Assets         = Sum of Ending Balances for all Asset accounts
Total Liabilities    = Sum of Ending Balances for all Liability accounts
Total Equity         = Sum of Ending Balances for all Equity accounts + Net Income{noformat}

Note: Section totals must equal the sum of their category subtotals. Total Assets = Sum of all Asset category subtotal ending balances.

h4. 4.8 — Grand Total and Balance Check

{noformat}Total Liabilities and Equity = Total Liabilities + Total Equity{noformat}

Balance check:

{noformat}Total Assets = Total Liabilities and Equity{noformat}

If this check fails, display a warning with the difference:

{noformat}Out of Balance Amount = Total Assets − Total Liabilities and Equity{noformat}

h4. 4.9 — Negative Value Display

All negative values must be displayed in parentheses. Example: $(1,234.56)

h3. 5) Display and Formatting

5.1 All currency values must be formatted with two decimal places and thousands separators. Example: $12,345.67

5.2 The report header (on the rendered report and on each export page/worksheet) must display: District Name, Report Title, Fiscal Year, Reporting Period (with resolved date range), Site Type(s) (if not "All"), Site(s) (if not "All"), and the date and time the report was generated.

5.3 If the balance sheet does not balance, a prominent warning must appear at the top of the report showing the out-of-balance amount.

5.4 Accounts with zero beginning balance, zero period activity, and zero ending balance may be hidden by default with an option to show them.

5.5 Account category groups are collapsed by default. The user can expand any group to see its individual accounts.

h3. 6) Export

h4. 6.1 — General

Report generation is *asynchronous*. When the user clicks the generate/export button, the report is queued for processing. The user is notified once the file is ready and can download it from the *Document Center*.

h4. 6.2 — Excel (XLSX) Export

6.2.1 When *"All" sites or a single site* is selected, the Excel file contains one worksheet with the full balance sheet for that scope.

6.2.2 When *multiple specific sites* are selected, each site gets its own worksheet within the same Excel file. Each worksheet:

* Is named with the site name
* Contains a complete balance sheet for that site (same section structure, category groups, subtotals, and formulas — all scoped to that site's ledger entries)
* Includes the report header with the site name

6.2.3 When multiple sites are selected via Site Type (e.g., "All Elementary sites"), each individual site within that type gets its own worksheet — the same as selecting those sites individually.

6.2.4 All worksheets must preserve the section groupings, category subtotals, and all column values with correct number formatting.

h4. 6.3 — PDF Export

6.3.1 When *"All" sites or a single site* is selected, the PDF contains one balance sheet for that scope.

6.3.2 When *multiple specific sites* are selected, each site gets its own page (or page group) within the same PDF, with a page break between sites.

6.3.3 The PDF must render in standard balance sheet format suitable for sharing with administrators or auditors.

## Acceptance Criteria
h3. Parameters Page

*AC1 — Report Accessible* Given an authorized Finance user opens the Reports section, When they navigate to Balance Sheet, Then the report parameters page is displayed with the title "Balance Sheet Report" and helper text "Choose a fiscal year, reporting period, and sites. When multiple sites are selected, each site gets its own Balance Sheet worksheet in the Excel file."

*AC2 — Export Tooltip* Given the user hovers over the generate/export button, When the tooltip appears, Then it reads "You'll be notified once your file is ready and can download it from the Document Center."

*AC3 — Reporting Period Options* Given the user clicks the Reporting Period dropdown, When the dropdown opens, Then it displays four options: Single month, Quarter, Fiscal YTD, and Custom date range.

*AC4 — Single Month Resolves Correctly* Given the user selects Single month and picks a month, When the report generates, Then the date range spans the first day through the last day of that month.

*AC5 — Quarter Resolves Correctly* Given the user selects Quarter and picks a quarter, When the report generates, Then the date range spans the first day of the quarter's first month through the last day of the quarter's last month.

*AC6 — Fiscal YTD Resolves Correctly* Given the user selects Fiscal YTD, When the report generates, Then the date range spans the first day of the fiscal year through the last day of the most recent closed period (or today if no periods are closed).

*AC7 — Custom Date Range* Given the user selects Custom date range and enters a start and end date, When the report generates, Then the date range matches exactly what the user entered.

*AC8 — Site Type Defaults to All and Supports Multiselect* Given the user opens the report parameters, When the Site Type dropdown loads, Then "All" is selected by default and the user can select multiple types.

*AC9 — Site Defaults to All and Supports Multiselect* Given the user opens the report parameters, When the Site dropdown loads, Then "All" is selected by default and the user can select multiple sites.

*AC10 — Site Dropdown Cascades from Site Type* Given the user selects one or more Site Types (e.g., Elementary + Middle School), When the Site dropdown refreshes, Then it shows only sites belonging to the selected types plus the "All" option.

*AC11 — Site Deselects on Site Type Change* Given the user has selected specific sites, When they change Site Type such that one or more selected sites no longer belong to the selected type(s), Then those sites are deselected; if no sites remain selected, Site resets to "All".

*AC12 — Selecting All Clears Individual Selections* Given the user has selected specific Site Types or Sites, When they select "All", Then all individual selections are cleared and "All" becomes the only selection.

*AC13 — Selecting a Specific Value Clears All* Given "All" is currently selected in Site Type or Site, When the user selects a specific value, Then "All" is automatically deselected.

h3. Report Content

*AC14 — Site Type and Site Filter Scopes Data* Given the user selects one or more Site Types and/or specific Sites, When the report generates, Then all values (Beginning Balance, Period Activity, Ending Balance, Net Income) reflect only ledger entries from the matching sites.

*AC15 — Correct Section Structure* Given the report generates, When the user views the report, Then it displays Assets, Liabilities, and Equity sections in order with category groups, category subtotals, section totals, and a Net Income line within Equity.

*AC16 — Category Groups Expandable* Given the report renders with category groups collapsed, When the user expands a category group, Then individual account rows within that category are revealed with the category subtotal row visible.

*AC17 — Category Subtotals Correct* Given a category group contains multiple accounts, When the user views the category subtotal row, Then each column (Beginning Balance, Period Activity, Ending Balance) equals the sum of the individual account values in that category.

*AC18 — Uncategorized Accounts in Other Group* Given an account does not have a category assigned, When the report generates, Then the account appears under an "Other" group at the bottom of its section with the same expandable and subtotal behavior.

*AC19 — Expanded Equation Satisfied* Given the report generates for a period with complete and correct data, When the user views the grand totals, Then Total Assets = Total Liabilities + Total Equity + Net Income.

*AC20 — Out of Balance Warning* Given Total Assets does not equal Total Liabilities and Equity, When the report generates, Then a prominent warning is displayed showing the out-of-balance amount.

*AC21 — Revenue and Expenditure Not Shown as Line Items* Given the report generates, When the user views the report, Then no individual Revenue or Expenditure accounts appear as rows; their net effect appears only as the Net Income line within the Equity section.

*AC22 — Net Income Correct* Given the report generates for any reporting period within a fiscal year, When the user views the Equity section, Then Net Income = (Revenue Credits − Revenue Debits) − (Expenditure Debits − Expenditure Credits), scoped from the fiscal year start date (of the selected Fiscal Year) through the report end date inclusive.

*AC23 — Period Activity Sign Convention* Given the report generates, When the user views Period Activity for any account, Then Asset accounts show Debits − Credits, and Liability and Equity accounts show Credits − Debits.

*AC24 — Beginning Balance Period-Relative* Given the user selects any reporting period, When the report generates, Then Beginning Balance reflects the cumulative ledger position of each account as of the day before the report start date.

*AC25 — Ending Balance Correct* Given the report generates, When the user views any account row, Then Ending Balance = Beginning Balance + Period Activity.

*AC26 — Total Equity Includes Net Income* Given the report generates, When the user views Total Equity, Then Total Equity = Sum of all Equity account ending balances + Net Income.

*AC27 — Negative Values in Parentheses* Given an account has a balance opposite to its normal balance, When the report generates, Then the value is displayed in parentheses.

*AC28 — Report Header Shows Filters* Given the user has selected specific Site Type(s) and/or Site(s), When the report generates, Then the report header displays the selected filter values alongside the other header fields.

h3. Export and Document Center

*AC29 — Async Generation* Given the user clicks the generate/export button, When the report is submitted, Then the report is queued for processing and the user is notified when it is ready in the Document Center.

*AC30 — Excel Single Site or All* Given the user selects "All" sites or a single site and exports to Excel, When the file is generated, Then the Excel file contains one worksheet with the full balance sheet.

*AC35 — PDF Multiple Sites* Given the user selects multiple specific sites and exports to PDF, When the file is generated, Then the PDF contains one balance sheet per site with page breaks between sites.

h2. Edge / Empty / Error States

||Scenario||Behavior||
|No transactions for selected date range|Report generates with zero Period Activity; beginning and ending balances shown based on prior cumulative ledger activity|
|No ledger entries exist at all|Report generates with all zeros; no error|
|Balance sheet does not balance|Prominent out-of-balance warning displayed with the difference amount|
|Negative Net Income|Displayed in parentheses within the Equity section; Total Equity is reduced accordingly|
|Single site district|Site Type and Site filters available but effectively show one site; Excel export has one worksheet|
|Period not yet closed|Report generates with available data; indicator shown that the period is still open|
|Custom date range crosses fiscal years|Report runs using the entered dates; Beginning Balance reflects all activity prior to the start date regardless of fiscal year|

|Account category has no accounts with activity|Category group is hidden if all accounts have zero balances (per 5.4); visible if "show zero balances" is enabled|
|All accounts in a section are uncategorized|The "Other" group is the only group in that section; section total equals the "Other" subtotal|
|Selected Site Type(s) have no sites with ledger entries|Report generates with all zeros for the selected scope; no error|
|Site Type changed after specific Sites were selected|Sites not belonging to the new type(s) are deselected; if none remain, Site resets to "All"|
|Multiple Site Types selected|Site dropdown shows the union of all sites belonging to any of the selected types|
|50+ sites selected|Excel file generates with 50+ worksheets; Document Center handles async delivery|

h2. Notes

* This report covers Assets, Liabilities, and Equity only. For Revenue and Expenditure detail, see the P&L report.
* Accounts are grouped first by section (Asset, Liability, Equity), then by their account category as defined in the chart of accounts. The system does not require the user to manually configure which accounts belong to which category group on this report — it reads from the chart of accounts.
* The balance sheet is a net-amount report — not split into debit/credit columns — consistent with standard financial statement presentation.
* The report is built using Power BI Report Builder (paginated reports). Category grouping uses row groups with toggle visibility for expand/collapse behavior.
* Site-level breakdown with drilldown is handled in the Trial Balance report, not here.
* The multi-site worksheet pattern (one worksheet per site) is consistent with the P&L report export behavior.
