# NXT-119 — [TaggedReport] - Accountability - Reports - Patron Participation

## Description
*User Story*

As a district admin I want to see who is participating in meal service along with information about them, their meal, and be able to manipulate this information for different outcomes. I also want to do the opposite - see who isn’t participating.

*Requirements*

* User can make selections for the following:
** Report type: Summary, Detailed
*** Summary shows the summary chart from the 2nd image.
*** Detailed shows the full details per the 1st image.
** Site: single, multiple, entire district ('All')
** Site type: single, multiple. pull from district configuration.
** Grade: all, select any, but this is constrained by any selected Site (ie: select an Elementary school, and the Grade dropdown only pulls up the grades for elementary)
** Dates: use the dropdown
*** !image-20210810-151208.png|width=541,height=334!
** Meal: any configured meal for the district, multiple, ‘All’
** Eligibility: free, reduced, paid, employee, visitor, staff
** Reason: any price type reason in the DB for that district.
* User is shown different information depending on the selected report type. The examples are above - we’d want to show the same information in the reports.

## Acceptance Criteria
User can run a report with date range options that shows who did participate in a particular meal program.
