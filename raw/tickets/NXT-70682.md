# NXT-70682 — School Nutrition Profit & Loss Promotional Report

## Description
As a district finance stakeholder, I want a printable Profit & Loss report with executive summary, detailed revenue and expenditure statements, and supporting analysis so I can review performance and share a clear example with internal teams and external stakeholders.

## Acceptance Criteria
# The report shows header information: district name, report title, reporting period, fiscal year, and “generated by” name/date-time.
# The report includes an *Executive Summary* table with columns {{Line Item}} and {{Amount}}, containing:
#* Net Gain/(Loss)
#* Total Revenue
#* Total Expenditures
#* Beginning Fund Balance
#* Ending Fund Balance
# The report includes a *Revenue* table with columns:
#* Description
#* Current Period
#* YTD
#* Budget
#* % of Revenue
#* % Change
#* Variance
# The Revenue table includes grouped line items and a {{Total Revenue}} row.
# The report includes an *Expenditures* table with columns:
#* Description
#* Current Period
#* YTD
#* Budget
#* % of Expenses
#* % Change
#* Variance
# The Expenditures table includes grouped line items and a {{Total Expenditures}} row.
# The report includes a *Summary* table with columns:
#* Line Item
#* Current Period
#* YTD
#* Budget
#* Variance
#* Prior Year
# The Summary table contains:
#* Total Revenue
#* Total Expenditures
#* Net Gain/(Loss)
#* Beginning Fund Balance
#* Ending Fund Balance
# The report includes a *Compliance Checks* table with columns:
#* Metric
#* Value
#* Status
#* Benchmark

# The report includes a *KPI Analysis* table with columns:
#* Metric
#* Current
#* vs Prior Year
#* Signal
# The report includes a legend defining data source markers: {{System-generated}} and {{Manually entered}}.
# The report includes a disclosure statement that this is not a substitute for a fully audited financial statement.

*Requirements*

# Support report metadata inputs: district, period, fiscal year, generated-by name, generated date-time.
# Support line-item source classification (system-generated vs manually entered).
# Support grouped financial categories within Revenue and Expenditures.
# Support subtotal/total rows in financial statements.
# Support benchmark/status reporting for compliance metrics.
# Support explanatory “Signal” text for KPI rows.
# Support inclusion of formula notes on applicable line items (for example, indirect costs formula).
# Generate output suitable for stakeholder distribution as a printable report/PDF.
