# NXT-28717 — INV - Transfers - Transfer Detail - Enable Edit/Cancel (Require Approval: YES)

## Description
*Update 12/2022 -* At this point this story should not be done until the relevant permission functionality is available

As a Site Manager, or District admin, I want to be able to Edit or Cancel a transfer even in Approved status, so I can make adjustments until an inventory is completed or the period is closed.

Currently, only Cancel is provided and only when the transfer is pending approval. Once the transfer is ‘Approved’ (final status) it is locked for changes. However, this behavior is inconsistent with Additions, Withdrawals, etc. which can be edited more easily.

*Requirements*

* Add the Edit button on Transfers in Pending status
** only if user has access to the sites and has Approve Transfers permission (and of course Edit Transfer permission as well)
* Provide the Edit and Cancel buttons in Approved status
** only if user has access to the sites and has Approve Transfers permission
* Do not allow site level users to Edit or Cancel if they do not have Approve Transfers permission
** only in the context of the system setting being set to YES
* Restrict the user from editing or cancelling according to the typical transaction rules 
** Physical Inventory completed since the transaction
** Period is closed
* Display appropriate messages 
** For preventing Edit/Cancel if Period is closed
** For preventing Edit/Cancel if items are part of a postdated completed physical inventory

## Acceptance Criteria
* Users should be edit or cancel a Transfer if it did not require Approval (permission conflict)
* Users cannot edit or cancel a Transfer if a Physical Inventory has been completed for that site since
* Users cannot edit or cancel a Transfer if the period for the transaction has been closed
* Users will see messages explaining the above blockers
