# NXT-58552 — AI Summary on Insight screen

## Description
As an Insights role user, I would like to see a summary of my performance generated by AI in natural language so that I can get more understandable feedback.

*Notes:*

* This story is for adding the natural language summary to the top of the Insights screen
** There is also a story for a drawer and another for a summary on another page, but they are covered elsewhere
* Both backend and frontend are covered in this story

*Flow:*

* User browses to the page
* System preps the data using the site and time filters
* System sends the data and a prepared AI prompt to the AI engine
* AI engine analyzes data using prompt and returns a paragraph style text summary
* System shows the summary to the user

*Requirements:*

* On the Insights screen in Workspace, do the following:
* 
* Summary Insights section
* Add an icon to the page next to the other icons
** Icon can be the schoolie or queriosity icon
** On click:
*** Show the AI summary
*** Use a sliding motion
*** If user clicks it when expanded, use the sliding motion to close it
*** Do not show the icon if the district has AI turned off
**** There is a system setting for districts to enable\disable AI
**** If disabled, don’t show the AI section
* Add the AI summary section to the page per Dallas’s Bolt mockup
** Section title: AI Recap
** It is a summary (paragraph) text section
** Should have show/hide ability based upon the icon click
** Default to hidden
** Data Set
*** The results are dependent on the user’s site access
**** If the user only has access to one site, results show only for that one site, etc.
*** The results are dependent on the time frame selected
*** Pass in a dataset with these site and time filters
* On page load:
** System gets the dataset using the site and time filters
** System sends the dataset and a previously prepared AI prompt to the AI engine to perform the analysis 
** AI engine does its analysis and returns the result
** System loads the result in the AI section
* Show a loader icon while waiting on the AI engine. If you can, include text to let them know we are waiting on the the AI engine.
* 
* Prompt Text:
** Ask AI to summarize in paragraph format what went well and what didn’t go well and any trends if the data is for multiple days
* 
* Add security to the button:
** This section inherits the page’s role based security, no additional security needed beyond the AI district check
* Error Handling:
** If there is an error in retrieving or processing the AI summary (e.g., failure to contact the AI engine), an appropriate error message should be displayed to the user.
* System Performance:
** The system must load the AI-generated summary within a reasonable amount of time, aiming for a smooth user experience without significant delays.

## Acceptance Criteria
# *AI Summary Section Icon*:
#* A new icon must be added next to the other icons on the Insights screen.
#* The icon can be either the schoolie or queriosity icon, as per design guidelines.
#* When the user clicks the icon, the AI summary section should appear with a sliding motion.
#* If the AI summary is already expanded, clicking the icon should close it with a sliding motion.
#* The icon must not be visible if AI is disabled for the district (controlled by a system setting for districts to enable/disable AI).
#* Section title: AI Recap
# *AI Summary Visibility*:
#* The AI summary section should be added to the page, as per Dallas’s Bolt mockup.
#* The AI summary must be a paragraph text section.
#* By default, the AI summary section should be hidden and will only appear when the icon is clicked.
#* The section must have the ability to show/hide based on the icon click (expand/collapse functionality).
# *Data Set Handling*:
#* The AI summary content must depend on the user’s site access:
#** If the user has access to multiple sites, the summary should include aggregated results from all sites they have access to.
#** If the user has access to only one site, the summary should display only results for that specific site.
#* The summary data must depend on the selected time frame:
#** The system should pass in the dataset using the selected site and time filters (as per the user’s settings).
#* The system must send the dataset and the prepared AI prompt to the AI engine for analysis.
# *AI Engine Integration*:
#* On page load, the system must:
#*# Retrieve the dataset using the site and time filters.
#*# Send the dataset along with a pre-defined AI prompt to the AI engine.
#*# The AI engine should analyze the data and return the result.
#*# The system should load the returned result into the AI summary section.
#* A loader icon must be displayed while waiting for the AI engine’s response.
#* Optional: The system may show text next to the loader indicating that it is waiting for the AI engine to process the data.
# *Role-Based Security*:
#* The AI summary section should inherit the page’s role-based security, with no additional security required beyond the district’s AI setting check.
#* Only users with the appropriate permissions, based on the district’s AI setting, should be able to see the AI summary section.
# *Error Handling*:
#* If an error occurs while retrieving or processing the AI summary (e.g., failure to contact the AI engine), the system must display an appropriate error message to the user.
# *System Performance*:
#* The system must load the AI-generated summary within a reasonable amount of time to ensure a smooth user experience.
#* The AI summary section must not cause significant delays or performance issues in the overall page load process.
# *Prompt Text*:
#* The prompt text that will be sent to the AI engine needs to be defined and should be included when sending the dataset to the AI engine for analysis. 
# *Behavior on Icon Click*:
#* Clicking the icon should toggle the AI summary's visibility using a smooth sliding motion.
#* If the AI summary is expanded, clicking the icon should collapse it, and vice versa.
