# NXT-34145 — Team Hub - Add Authentication Type to Settings Screen

## Description
As a school admin, I want to set the platform setting so that the system can correctly authenticate.

 

*Flow:*

·         After logged in, Admin clicks on Settings button

·         Change authentication type per product

 

*Assumptions:*

·         Settings screen already exists per previous development

 

*Background:*

·         Settings screen already exists

·         In this story, we are adding new functionality that will allow the user to select which product they have, PrimeroEdge or SchoolCafé and based upon their choice, we will determine the type of authentication to use

·         In the previous version, it assumed Classic only

·         Code will use this value at login to determine what type of authentication to use

 

*Requirements*:

·         From any screen, Admin click on Settings button at top of screen

·         Opens flyout for settings typical of smart devices

·          

·         Add this Field:

o   Platform – list

§  Values: PrimeroEdge, SchoolCafé

§  Use the value behind the scenes to story what type of authentication to use when logging in

·          

·         Security – already covered in previous stories

## Acceptance Criteria
·         Field added to screen as per above

·         Login screen uses the value to determine authentication type
