# NXT-42298 — Homepage - Log user in to their 'Primary' district

## Description
Currently, when users log in, we direct them to the last district they were in before logging out. Instead, let’s direct them to the district where their account was first created. 

This will prevent unnecessary issues such as changing the wrong district’s data and wasting multi-district users' time redirecting back to their home district, as well as make it easier for a user to change their own password.

If there is a network issue or some other reason we cannot track the user’s logout, we can just go with their last logged-in district.

## Acceptance Criteria
Verify that when a user logs in to SchoolCafé, they land in the district for which they were first created.
