# NXT-39056 — PR- Plan- Add Menu item /Variety Menu item- Validation against Toggled or Untoggled Child Items and added menu items

## Description
As a production manager, When there is already a variety menu item with menu items, toggled or untoggled, do not allow me to be able to add any of those items onto my plan as they are already available to be added or are already added on the plan through the variety menu item. IF there is already a menu item, and then a variety menu item gets added, toggle that menu item on and add the counts to that menu item and remove the original menu item from being there,

## Acceptance Criteria
* When user adds a variety menu item on the plan, toggled or untoggled child menu items should also not be able to be added separately from the add menu item function
* Do not allow individual menu items to be added to the plan if these menu items are part of VMI (Already on plan)
** Grey out menu items and display tool tip “Menu Item is part of a added Variety Menu Item” 
* IF a menu item is added previously, and a user decides to add a variety menu item that happens to include that menu item or menu items (if there are more than one already on the plan), add the variety menu item and automatically toggle onto those menu items and remove the previously added individual menu items
** If there happens to be already counts for those menu items previously, populate the counts with the toggled child menu item
** Use the category of the VMI (or whatever was selected when adding the VMI) and not what was previously added for the individual menu item 
** User can not toggle these child menu items off as they were already previously added on the menu, all they can do at that point is 0 out the menu item if they happen to not want to use the menu item ( Same logic as they would have to previously do had they not added the VMI)
*** Provide tool tip “ Menu Item originally on Menu”
* IF Menu Item happens to be added via carryover and that was the reason the menu item was on the plan, then the user removes the carryover from the “source” plan, the system should toggle off the child menu item 
* No consideration for different serving sizes (For Now)
