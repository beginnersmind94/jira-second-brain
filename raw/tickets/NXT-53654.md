# NXT-53654 — MVR - Users Grid - Distribute Now

## Description
As a SchoolCafé user, I would like to be able to setup my managed view for recurring distribution to a set of users so that I can get periodic notifications like on custom reports.

*Notes:*

* This is a continuation of the users grid MVR story
* We split the distribute now part into a separate story that can be done after the Add New is complete
* 

*Requirements:*

* On System > Users, do the following:
* If not already there, add Distribute button to the Managed View ellipsis
** On click:
** First, check if an MVR has been setup for it already
*** If no, all requirements covered in original story
*** If yes, then we want to distribute it, so follow distribute section below



*Distribute now*

* If the user click Distribute button on a linked MVR, then do the following:
* Immediately, or schedule for a few seconds from now, to run the report and distribute it per its Distribution settings
** Basically it is the same thing as Run Now from CR, so do it like you did that
** Ensure it shows up in the recently distributed page
** Ensure it shows up in the appropriate distribution monitor pages [ex: email monitor if it was emailed]
** Ensure it shows up in the Inbox if it was distributed that way
** Ensure it shows up in the email if it was distributed that way
* If successfully started:
** Show message: Distribution process started. It may take a few minutes
** When the process has completed:
*** Show the user a notification framework toast message: [MVR name] distributed successfully
* If failed for some reason:
** Show message: Distribution process failed. Try again.
** Ensure failure is captured in error log and viewable in Monitors

## Acceptance Criteria
* If the user click Distribute button on a linked MVR, it executes the query and distributes the MVR immediately to the recipients in the config
* Success or failure messages are displayed
