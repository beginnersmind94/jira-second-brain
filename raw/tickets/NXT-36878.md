# NXT-36878 — Student Import – Don’t Abort Import If Row Fails

## Description
As a SchoolCafé admin, I want to be able to import all the successful student rows even if some fail so that I can get started with using the system.

*Flow:*

* Open SchoolCafé and browse to Settings > Imports & Exports > Imports
* Find a Student Import you want to use
* Click on Import for that row
* Follow the import process
* When the import routine comes across a row that fails because of a required field, skip it, log it as an exception and keep on importing the rest of the records
* Notify the user when the import finishes

 

*Notes:*

* This story comes from Dallas and Customer Care team during implementation of River Road. They have concluded that it is better to have a successful import with exceptions rather than completely abort the import because of a few bad records.
* Therefore, the point of this story is that we can’t let a bad import record cause the whole import to fail.
* Note this story applies only to failures caused by required fields; optional field issues are covered in another story

 

*Requirements*:

* When click on Import button for a Student Import:
* When running, if there is a record that fails because of a required field, do the following:
** Currently it causes the import to fail – don’t do this, keep going
** Skip that record in the import file
** Add that failure to import that record as an exception
** Continue processing the remaining records
* 
* When creating the import file and testing if the import file is accurate
** Continue to test the same way you do currently, no changes
* 
* When viewing the History log from Import Monitor:
** Ensure the exception is logged and visible in the import log
** If there is not already an exception text and code for this, use these:
** Exception text: Record failed to import
** Assign it a code
* 
* Ensure that the number of exceptions are countable
** Future stories will need to display the amount of exceptions
*   
* Permissions: No changes necessary

## Acceptance Criteria
* When importing, if there is a record that fails, the import routine skips it and continues
* Failed record imports are marked as exceptions and visible in the history log
