# NXT-36652 — Monitoring Scheduler – P4 Delete Monitor

## Description
As a SchoolCafé admin, I want to be able to delete existing monitoring jobs so that I can proactively monitor the application and ensure higher customer satisfaction.

*Flow:*

* User logs in and browses to Support > Monitoring > Job Scheduler
* User views a list of scheduled monitoring jobs
* User clicks delete icon on an existing scheduled job

*Notes:*

* This is a continuation of the story to create the Job Scheduler page
* This story assumes that story has been created and that the Monitor tab was created and the grid on that page has the edit button
* This story assumes that the Add New Monitor story has been completed and that users are able to add new monitoring jobs already
* Note that each row in the grid already has an delete button that does not do anything yet

*Requirements*:

* Requirements for the Job Scheduler > Monitors tab
* Requirements are for what happens when a user clicks the delete button on a row in the grid of scheduled jobs
* For the delete button, on click do the following:
* Open the Edit Monitor Page
* Like what is done on the System Configuration Notification page, show a confirmation popup with the same UI, fields and wording (see screenshot)
* If they select yes, delete the scheduled job
** Soft delete, not hard delete
** Who canceled and when is logged so that later can debug if there are issues
* If they say no, then cancel the popup and do nothing
* When deleted, the grid no longer shows the job
* 
*  Permissions:
* In General resources screen for Roles & Permissions
** Application: applies Job Scheduler delete permission
** Write the security code to apply that permission to this button
