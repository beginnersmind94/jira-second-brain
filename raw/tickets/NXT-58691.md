## Acceptance Criteria
Average Daily Participation - Lunch card should be loaded in a progressive way.
We need to show the progress indicator (loader) on the card as shown in the Figma.
When the page loads show the the progress indicator (loader) and hide the the progress indicator (loader) when the API returns a successful response.
User should not be able to click and view the drawer while the progress indicator (loader) is visible.
No need to show the progress indicator (loader) for the values in the drawer because by that time API will have the data in the response.
When the API returns  Http Status Code other than 200 (failure), then show all the decimal values as 0 and percentage values as 0.00%.
All numeric and percentage values need to be right aligned in the UI.
Example of numeric values that are right aligned in the UI.
