# NXT-68096 — SC - MT - Settings Update

## Description
As a Customer Support role user, I want to be able to control settings related to a Central realm so that I can manage children districts.

*Notes*

* This story covers a central realm managing system settings for children districts

*User Flow*

* Central realm user browses to system settings
* User updates a setting that has a type of Children Required and hits save
* System applies the new setting value to all children realms

## Acceptance Criteria
h1. 

h2. *1. Filters and Layout Updates (System > Settings Page)*

h3. Filters Section

* *Move Reset Filters* next to the existing filter controls.
* *Move Refresh View* down to the same row as the Filter icon.
* Update *Refresh View* to display only a *refresh icon* (no text label).

h3. Column Order and Formatting

* *Reorder Columns:*
** Move {{Module}} before {{Category}}.
** Move {{Configuration Type}} before {{Module}}.
* *Rename Columns:*
** {{Configuration Type}} → {{Config}}
** {{Default Value}} → {{Default}}
* *Resize Columns:*
** Narrow the following columns: {{Setting Value}}, {{Config}}, {{Default}}, {{Actions}}.

h3. Add Configuration Type Value

* Add *Central* as a valid value in {{Configuration Type}}.
* Treat *Central* exactly the same as *Common* in terms of security trimming and visibility.

h3. Add New Column

* *Label:* {{Scope}}
* *Location:* Before {{Default}} column.
* *Values:* Derived from the {{Scope}} field in the Scope drawer.

----

h2. *2. Manage Scope Button*

* *Label:* {{Manage Scope}}
* *Location:* Where the {{Refresh View}} button was previously located.
* *Visibility:*
** Display only if the current realm is *not a District*.
** Hidden for District-level realms.
* *On Click:* Opens the *Scope Popup*.

----

h2. *3. Scope Popup – Manage Settings Scope*

* *Title:* {{Manage Settings Scope}}
* *Info Button:* Displays message: _“On this page set the scope of settings for child realms.”_

h3. Filters Section

* Replicate the same filter controls from the main Settings page.

h3. Grid Configuration

* *Columns:*
** Checkbox
** Setting Name (without info icon)
** Scope
** Config
** Module
** Category
* *Behavior:*
** Sorting enabled on all columns.
** Filtering controlled by the Filters section.
** No paging — show all matching results.

----

h3. Grid-Level Buttons

* *Select All:* Selects all visible (filtered) rows.
* *Unselect All:* Deselects all visible (filtered) rows.
* *Mark Selected as Self Only:* Updates all selected rows’ scope to *Self*.
* *Mark Selected as Self + Children – Required:* Updates all selected rows’ scope to *Child Req*.
* *Mark Selected as Self + Children – Optional:* Updates all selected rows’ scope to *Child Opt*.
* *Cancel:* Closes the popup without saving.

----

h3. -Save Button-

* *-Label:-* {{Save}}
* *-Behavior:-*
** -On click, show confirmation message:-
-_“This saves setting, but does not apply it to children. Are you sure?”_-
** -If confirmed:-
*** -Save the updated scope values.-
*** -Close the popup.-
*** -Refresh the main grid to show changes.-

----

h3. Apply Scope Button

* *Label:* {{Apply Scope}}
* *Tooltip:* “Apply required settings to children realms.”
* *Behavior:*
** On click, show confirmation dialog:
_“This will apply required setting values to all children realms. Are you sure?”_
** If confirmed:
*** Apply all *Child Required* setting values to all child realms.
*** Save the updates.
*** Display a *green success popup* confirming completion.

----

h2. *4. History Logging*

* Log all configuration changes made in the Settings page, including:
** Manual edits (Save action).
** Automated propagation (Apply Scope action).
* Each history log entry must include:
** Setting name and ID.
** Old value and new value.
** User who made the change.
** Timestamp.
** Whether it was a manual or automated update.

----

h2. *5. Permissions*

* Only *Customer Support role users* can access and manage these settings.
* Users without the necessary permissions cannot view or interact with the *Manage Scope* button.

----

h2. *6. Validation & Error Handling*

* If a save or apply operation fails:
** Display an inline message: _“Unable to save changes. Please try again.”_
** Log the error for support diagnostics.
* Prevent concurrent edits on the same record to avoid data conflicts.

----

h2. *7. Performance*

* Scope popup loads within *2 seconds* of button click.
* Save and Apply operations complete within *5 seconds* for average datasets.
* All updates are reflected immediately in the Settings grid after completion.
