# NXT-70075 — FIN - Period Context in Action Modals

## Description
*Description:* Display specific period information in all action modals so users understand exactly which Financial period they're modifying.

*Given* I am performing an action on a Financial period
*When* a confirmation modal appears
*Then* the modal includes the period name, date range, and period type

## Acceptance Criteria
* All period action modals (Close, Reopen, Lock, etc.) must include period context
* Format: "Are you sure you would like to [action] the [Period Name] ([Start Date] - [End Date]) Financials period?"
* Example: "Are you sure you would like to re-open the September 2025 (Sep 1 - Sep 30, 2025) Financials period?"
* Date format: MMM D - MMM D, YYYY (or MMM D, YYYY - MMM D, YYYY if spanning months)
* "Financials period" appears in all modals to distinguish from POS/Inventory periods
* Bulk actions: If user selects multiple periods, use generic text without listing specific periods: "Are you sure you would like to [action] the selected Financials periods?"
* Apply to all existing action modals: Close, Reopen, Lock, Unlock, Delete
