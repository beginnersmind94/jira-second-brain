# NXT-14876 — Platform - PII - User Experience(UX) for PII data

## Description
*Persona**: Perseus User*

*Access level:* *School District/Site*

*Description**:*

*As a Perseus user, I should be able to view the sensitive data at field/section/page level only when I have access to a role that has view permissions.* 

*PII Fields:*

* *First Name*
* *Last Name* 
* *Parents Name*
* *Address*
* *Student Number*
* *Birth Date (DOB)*
* *Gender*
* *Ethnicity*
* *Race*
* *Eligibility*

*Sensitive Sections: (HIPAA, FERPA compliance)*

* *Allergens*
* *Eligibility*
* *Household Information*

*Requirements:*

* *All the fields identified in the above data fields needs to be masked/hide for user who do not have PII access roles/permission*
* *Fields must be masked with <Sensitive> field tag with blurred preview of the data field*
* *Fields will be marked as PII under resource catalog*



*References:*

* [https://studentprivacy.ed.gov/node/548/|https://studentprivacy.ed.gov/node/548/|smart-link]

## Acceptance Criteria
* User shall be able to see sensitive fields only when permissions are enabled for the roles specific to PII
