# NXT-12620 — Dashboard - Common Component - Context Panel

## Description
As a user I want the ability to set a timeframe and site context for my dashboard visualization so that I can see my data represented over time and scope.

Add a new shared component that contains a Title, Context (Sites), and Timeframe (Date Range).

*Proposed Design:*

!image-20210323-024038.png|width=900,height=505!

!image-20210323-024133.png|width=900,height=505!

!image-20210323-024205.png|width=494,height=505!

In the examples above, we see that the context panel should allow options for the context (site) and timeframe (dates) to be shown/hidden based on the need. Chart controls are able to be included inside the control which will pass the filtered parameters to the content. We will need the ability to set an initial filter context so that information loads by default. Users can then modify information.

*Context:*

* Displays option for “All Sites” or a multi-site selector of sites that the user has access to.
* If all sites are selected, show “All Sites”
* If one site is select, show \[site code - site name]
* If more than one site is selected, show \[X sites]

*Timeframe:*

* Preset date values for simplicity and consistency
* Preset options: Today, Current Week, Last 7 Days, Current Month, Current Program Year
** Today - Current Day
** Current Week - Current range of weekdays in the so far. Ex - If it is a Wednesday, current week will display M, T, W data
** Last 7 Days - Last 7 days starting with current day. Ex - If it is Wednesday, current day - 7 (Wed - Wed)
** Current Month - Current range of weekdays within the month so far.
** Current Program Year - Current range of week days within the current program year (academic year which starts 7/1 and ends 6/30).
* Future options: We can also add (in the future) options for “Custom Range”. This option can allow a user to set the from/to as the same for a single date in the past or set a range for longer periods calculation.

Context and Timeframe may actually be custom components in and of themselves. 

Note: Panel will allow us to expand as we go to eventually include Programs (NSLP, SFSP, etc).
