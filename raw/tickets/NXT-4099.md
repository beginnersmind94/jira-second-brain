# NXT-4099 — IM - Copy Item

## Description
*User Story*

As a Item Management User, I copy an item from an existing item

*Mockup*

[https://nqyc9v.axshare.com/start.html#p=itemmanagement_-_super_user&id=n0hwa7&g=1|https://nqyc9v.axshare.com/start.html#p=itemmanagement_-_super_user&id=n0hwa7&g=1]

!image-20200521-133427.png|width=858,height=426!

*Requirements*

* Header of the item record
** Have the option for ‘Copy Item’ *(Mockup mistakenly says ‘Ingredient’)*
** When clicking on the Copy Item Information, bring up a popup for copying the information
* Header - 'Copy Item'
** Item Name - Default ‘Copy of 'Item Name’ *(Mockup just shows name, use Item Name instead)*
*** Allow the user to put in a unique Item Name here
**** Follows the same rules for duplicate check, does NOT allow duplicates
*** If user is typing in an item name that already exists, display ‘(item name that exists) already exists. Please type in a unique name’
** Text - “Please check or uncheck the sections to be copied”
* Checkboxes for the information they want to copy, text to use below 
** Default them all to unchecked
*** If no information is available for the sections, then they would not be clickable to copy (ie, if there are no allergens, then it would be greyed out)
**** General Information
**** Images and attachments
**** Tags
**** Nutrition and Contribution Information
**** Allergens
**** Product Info
* Cancel - Return to the previous screen
* Copy & Save - Create the new item record and bring the user to the add new screen (past duplicate check, since that happened already)
** Datasource defaults to the datasource from the original item, unless the user does not have access to that datasource, which then would be defaulted to ‘*Local*’
*** Most common scenario is the datasource is going to be local

## Acceptance Criteria
* User must be able to click ‘Copy Ingredient’ button on an item that is already created
* Not allowed to make a duplicate item
