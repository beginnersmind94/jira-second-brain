# NXT-34248 — DataQuery Framework - Reports Execution Page - Architecture Refactor for Data Retrieval

## Description
On the Reports execution page, the data retrieval mechanism is not performant with large amounts of data.  This story will be used to drive the refactoring of the architecture to perform paging, filtering, sorting, and other methods so that data retrieval is performant and scalable.

## Acceptance Criteria
* As an end-user, I would like the ability to turn multi-column sorting on and off during runtime of a report
* Filtering shall work for all data types (including dates)
* Sorting shall function properly
* Single and multi-column sorting shall be available where the end-user shall be able to turn multi-column sorting on or off
* Page loads shall be performant and shall load within 1.8 seconds
