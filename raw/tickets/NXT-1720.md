# NXT-1720 — INV - E-Orders - Generate E-Order - Populate File Schema

## Description
As an Inventory manager I want the E-Ordering tool to generate a file containing all my selected order data so I can send the orders electronically.

{noformat}public class EOrder
{
    public int EOrderId { get; set; }
    public List<Order> Orders { get; set; }
    public string VendorId { get; set; }
}

public class Order
{
    public int OrderId { get; set; }
    public string OrderNumber { get; set; }
    public DateTime OrderDate { get; set; }
    public string ShipToDescription { get; set; }
    public string BillToDescription { get; set; }
    public DateTime DeliveryDate { get; set; }
    public string TotalAmount { get; set; }
    public string Comments { get; set; }
    public DateTime LastEOrderSentOn { get; set; }
    public string LastEOrderNumber { get; set; }
    public List<OrderLineItem> LineItems { get; set; }
}

public class OrderLineItem
{
    public string DistrictItemNumber { get; set; } // ItemCode
    public string VendorItemNumber { get; set; }
    public string ItemDescription { get; set; }
    public string PurchaseUnit { get; set; }
    public string UnitPrice { get; set; }
    public string Quantity { get; set; }
    public string TotalAmount { get; set; }
    public string Comments { get; set; }
    public bool IsCommodity { get; set; }
}{noformat}

## Acceptance Criteria
* Populate the E-Order schema with the selected orders for the E-Order being generated
* Pass the populated order information to the System module to generate the file
** System processes details once E-Order information is sent by Inventory
* (For now) always send ‘CA’ as the Purchase Unit Type
** Not supporting “broken units” in the file; each unit is a separate vendor item
* Send IsCommodity as false for now
