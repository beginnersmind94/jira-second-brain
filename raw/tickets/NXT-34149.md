# NXT-34149 — Team Hub - View a Message

## Description
As a school admin, I want to view the specific message details and optionally action it so that I can stay up to date with the app and coworkers.

*Flow:*

·         After logging in, user redirected to Home Screen

·         On Home Screen, click the View All Messages Icon and redirected to Messages Screen and on that screen click on a message, or

·         On Home Screen, click on a message in the messages control

·         This screen opens up and user views automatically loaded message details of selected message

·         Optionally Admin can click on delete

·         Optionally Admin can click on forward, reply or reply all

 

*Assumptions:*

·         User has logged in

·         To code this, you will have to have completed the push / pull messages stories and one of the source triggers (Home Screen or Messages Screen)

 

*Requirements*:

·         Fields:

·         Screen Name: Message

·         Back – Image button

o   Use same back button as rest of site

o   On click, redirect user to Messages Screen

·         Forward – Image button

o   Make look like traditional email Forward icon

o   All requirements in the Reply or Forward a Message story

o   Just put the icon in its place in this story

·         Reply – Image button

o   Make look like traditional email Reply icon

o   All requirements in the Reply or Forward a Message story

o   Just put the icon in its place in this story

·         Reply All – Image button

o   Make look like traditional email Reply All icon

o   All requirements in the Reply or Forward a Message story

o   Just put the icon in its place in this story

·         Compose – Image button

o   Use a traditional writing, composing icon

o   All requirements in the Reply or Forward a Message story

o   Just put the icon in its place in this story

·         Delete – Image button  

o   Use a traditional delete or trash icon

o   All requirements in the Delete a Message story

o   Just put the icon in its place in this story

·         From – Label

o   Make look like traditional email From field

o   Read-only

o   Shows who the message is from

§  Show Name field from Profile

·         To – Label

o   Make look like traditional email To field

o   Read-only

o   Shows who the message is to

§  Show Name field from Profile

·         Date / Time – Label

o   Make look like traditional email date / time stamp field

o   Read-only

o   Shows the date / time stamp of the message with no label

·         Subject – Label

o   Make look like traditional email subject field

o   Read-only

o   Shows the subject of the message

·         Body – Label

o   Make look like traditional email body field

o   Read-only

o   Shows the body of the message

·          

·         Biz Logic:

o   Upon page load, update the message State field to “Read” if it is “unread”

§  State field is not visible to user on this screen, but used behind the scenes

·          

·         Security

o   Copy all the security on the Home Screen and do the same for this screen

o   If user not logged in, redirects the user to the login screen

o   User can’t see messages for another user
