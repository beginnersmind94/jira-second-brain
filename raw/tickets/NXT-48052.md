# NXT-48052 — Users - Managed Views - Hide or Show Filters Row

## Description
As a SchoolCafé user I want to be able to hide or show the filters row of my grids, so that I can save the space and clutter if I don’t need to use the filter.

h3. *Notes:*

* The purpose of this story is to manage the hide / show filters row that is in the grid
* Sometimes a user wants to use the filters and should be able to see that row
* At other times they don’t, so they should be able to hide the row

*Requirements:*

* In the Grid components, add the ability to easily hide or show the filters row
* Should be like a property for the developer to show or hide the button
* If they want it, then a button shows to hide / show the rows
* If they don’t want it, then the button should not show and the developer can hide or show the row in code by default
* The default should be to hide the button
* The default row visibility should be to not show
* If the button is shown, then toggle the row visibility accordingly
* Button text is View Filters
* Enable the user to show the full button with the text or to show the “Short version” with just the icon
* If show just the icon, show a tooltip with text: View Filters
* 
* Put this button and functionality on the Users grid: System > Sites & Users > Users
** Add the ellipsis next to the Manage Views icon if not already there
** Add the Hide / Show filters button to the ellipsis
*** Show or hide the filters row based upon the button click
*** Default row visibility = false



*Technical Details*

FrontEnd

* Expose a function out of Managed Views to respect show/hide grid filters. It should control the grid filters to hide or show

## Acceptance Criteria
* Hide / Show filters button added to the ellipsis and shows or hides the filters row based upon the button click
* Developers can easily set to show or hide the View Filters button
* If user clicks on View Filters button, it toggles the visibility of the edit row
* Default button visibility is No
* Default row visibility is No
