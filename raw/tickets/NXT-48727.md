# NXT-48727 — PR- Record- Not allowing Record complete status to be reverted back to lesser status

## Description
As a production manager, once I have made a record into record complete, the system should not allow me to go back down to a lesser status as everything that I would need to enter should already be there and only need to be altered.

## Acceptance Criteria
* Do not allow Record complete status to be changed back to plan complete
* Only allow actual counts, menu item counts to be changed but not removed
* When adding menu item, user should be *required* to be asked for the PRODUCED count on the add menu page and must enter counts or 0 to preserve status
** Display Serving groups and only display serving size and produced count
*** Add a total row to sum all the counts for all serving groups
** Consider strict batching on TOTAL
* Introduce Strict batch rules on record (Same as plan)
** This should apply to all serving sizes produced for 1 menu item
** Message “Strict Batching is required for this recipe. Please correct the counts to add up to full batches.”
* IF user tries to add same menu item from the plan page, gray out the menu item on the add menu item page 
** Display tool tip “Menu Item added from Record Page”
* On plan page, never display added from record menu item, in the background do not consider as part of status
* Do not allow Add menu item from Plan page if Record is already complete, this will ensure that items added have to be added from the record as the record is now complete
