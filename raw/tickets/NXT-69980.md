## Acceptance Criteria
When a Square terminal is configured the credit card tab on the “PAY” pop up is replaced with Square functionality.
POS terminals that are not configured for Square will see the existing credit card tab + workflow.
When clicked >ExpressPoint will initialize a Square transaction that can then be completed on the Square Device.
When a credit card transaction is completed a user is presented with a brief green check on the pop up which will then auto close the pop up and display success message toast.
Users are presented with Helper text on each step of the Square Transaction.
Users can cancel at any time even if it means we handle delays on the backend.
A timer is presented which will automatically timeout/cancel the transaction if nothing has been returned by the square terminal. (Default 3 Minutes)
Users can initialize another Square transaction after completion or even after one has been canceled.
Users are alerted with a toast message of the status of a Square transaction after completion, canceled, or timeout.
Complete Transaction Button is disabled during a credit card transaction for Square(auto closes with success message when finished)
UI Dollar and Number buttons are disabled during a credit card transaction for Square (reenables if user cancels)
If a user clicks Cash or Check tab, Cancel the Credit Card Transactions.
Log all Cancel interactions for future reporting.
