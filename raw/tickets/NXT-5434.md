# NXT-5434 — System - Licensing Summary - View Device-Level License Details

## Description
As a Cybersoft or district administrator, I would like the ability to manage our currently-assigned devices licenses, so that I can make sure that every device subscription within the district can be accounted for.

+Summary+

* Within the Licensing Summary page, for multiple licensed modules, allow the user to view the Terminal *and* Device-level module details, as well as add or remove licenses as needed.

+Details+:

* When the user selects a module in the Device-Level Licensing details section, display: 
*# Information on number of licenses in use vs. number of licenses available, for the selected module
*# The devices that are currently licensed to use that module
*# The device that are *not* currently licensed to use that module (but have been enrolled and were possibly licensed previously)
*# The ability to jump to other modules
* Page content/columns can be carried over from existing solution, with new UI/UX controls such as toggles.
** Devices should also display in alphabetical order by Device Name.
* An unlicensed device can have a module license added as long as at least 1 license remains available, otherwise we should prevent the user from adding that license and/or display an error.
* Removing a license can be done for as many devices as needed.
* Overall license count for the module should adjust as the user adds and removes individual licenses for each device. Values should update in the database.
* The user should be able to search for licenses on the page, and easily filter out licensed vs. unlicensed devices
* When a user jumps to another module, we should display the device licensing details for that module and allow the user to perform the operations above

## Acceptance Criteria
Verify that device-level licensing details can be viewed from within the Licensing Summary page

Verify that when a module is selected, the user is immediately presented with a detailed view of their licenses for each device for that module, including:
-Overall license count
-Licensed devices
-Unlicensed devices

Verify that devices are displayed in alphabetical order by Device Name

Verify that an unlicensed device can have a module license added as long as at least 1 license remains available

Verify that licenses can be removed from any or all devices as needed

Verify that the overall license count for the module adjusts as the user adds and removes individual licenses for each device. Values should update in the database.

Verify that the user can jump to another module to view device license details and perform the functions above
