# NXT-52710 — PR- Plan/Record- Prep site drop down validations for prep site and receiving sites

## Description
As a production manager, If I am a prep site, I should not be able to also become a receiving site even on an indiviidual day. Also when I consolidate a plan, the receiving sites should not be able to change their prep site either.

## Acceptance Criteria
* If site is a prep site for other sites, never allow for them to change their prep site on the plan/record page and they should always be prepping for themselves
** Grey out the selections so they can never be a receiving site IF any plans are associated with that prep site for that particular Serving date, meal type & Menu Line combination 
*** Tool tip: “Site is currently preparing for other sites”
** EX: bellview- config harper
**forest- Config Harper*
**longview- Config Harper*
**morrison- Config Harper*
**Harper- Prep Site*
7/18/2024
Individual plan - Lunch Main Line
Bellview- king high
Forest- King high
Longview- Aly High
Morrison- Aly High
Prep Site Harper should be allowed to become receiving site ONLY for this serving date, meal type and menu line
* Once a plan has been consolidated by the prep site, receiving sites should also no longer be able to change their prep site from the plan/record page
** Provide Tooltip when hovering over greyed out selection on plan “Plan has already been consolidated by prep site”
** The current banner message for the receiving sites that says Plan has already been consolidated should always appear and not disappear like it currently does, user should have the ability to click X to close it however it should appear the next time they try to come to the plan page again
*** Banner Message: *The selected Plan was previously consolidated by Prep site. Please notify the Prep site of the changes made to this production Plan. Status changed to (Current Status of receiving site).*
** IF user adds new menu items to receiving site plan that is a prepared item, treat as a normal menu item if consolidated plan has already been completed
*** This should effect whether added from plan page or menu
*** QA Point: Test to make sure Prepared Item flag does not show up on any reports
* If a user still decides to go and change the prep site through the prep site config or the site config page, only effect plans that have not be consolidated 
** New effective date should be priority when changing plans 
** “Plans that have already been *consolidated* will not be effected and previous prep site will remain the same”
** If config already completed for future date and consolidated was already completed on or after the future date, then the consolidated plan was deleted, we are will keep the same selections (aka old prep site) but drop downs will be available to select if needed to change prep site 
*** _Manual Config change and nothing will be done automatically based on new config from site config page_
