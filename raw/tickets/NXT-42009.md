# NXT-42009 — Recents – Create Framework

## Description
We need to create a framework to save pages as a Recent so that end users can quickly browse to the pages they just previously viewed.

*Notes:*

* This is a backend only story
* Need to do first so front-end stories can be completed

 

*Requirements*:

* Create a backend structure in the database and supporting APIs that will allow they system to save pages as a recently viewed in SchoolCafé
* 
* At a minimum, the database needs to be able to save:
** User ID
** Page Name
** URL
** Something to order them (datetime stamp?)
* 
* At a minimum, the following needs to be exposed to the Front End
* FE needs to be a way to get a list of the last 10 pages viewed (URLs and names) by user id and sorted by most recently viewed first
* System needs a way to save a single viewed page (User ID, name and URL) to the list
* System needs a way to purge (delete) any viewed pages older than the 10^th^ from the list
* 
* Consider:
** It might be best to have the system just log during the day and then at night run a job to purge older pages rather than transactionally maintaining just 10 as you go

## Acceptance Criteria
* Database exists
* Front End team can use the framework to log and view last 10 pages visited
