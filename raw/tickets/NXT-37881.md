# NXT-37881 — IM - Items - Single Item Recipe - Select Serving Size

## Description
As a Nutrionist I want Single Item Recipe tab to interact with the serving sizes in the header Units card so I can select the appropriate serving size for my single item recipe.

*Goal:* Provide the ability to select a specific serving size from the Units card, if there are multiple servings, before configuring the SIR Steps and other info.

*Requirements*

* Require a selection of serving size from the Units card for the Single Item Recipe before Steps & Ingredients can be configured
** this is because we need to know which Serving Size is for the SIR
** if there is only one Serving Size, this step may be irrelevant and that single serving size can be auto-selected
* Provide some interactions with the header card for Serving Size selection
** Enable a semi-editable state of the Units card where only the serving sizes are selectable
*** Require a selection of one specific serving size
** Enable a ‘Single Serve’ flag once a serving size is selected
*** Single Serve means the ‘# of Servings’ is equal to *1 or greater*, which will often be the case
*** This flag will be enabled by default
** Allow the user to disable the Single Serve option
*** If the user does this, add a child row below the selected serving size
*** Hide ‘Amount’ column (it is fixed to '1')
*** Provide the Description field
*** Provide an editable ‘*# of Servings*’ field on the added row (empty by default)
**** Require the user to input a number greater than 1 to proceed (otherwise the Single Serve flag must be re-enabled)
**** this maps with the *Sub-Qty* column
**** Example: the existing selected serving size is ‘1 Frozen Pizza’, but the SIR serving size is 1/8 Pizza. The user must disable the ‘Single Serve’ flag, enter 1/8 Pizza as the serving size on the below row and 8 for *# of Servings* 
*** Once the user has modified the SIR serving size, allow the row to be saved
*** This new serving size will be added to the grid, but it will be inherited from the main serving
** Require confirmation of the selected/configured serving size on the Units card level
*** use the Units header area
* Move focus to the Single Item Recipe Summary card under the tab once the SIR serving size is configured

+*PMUX Notes*+

3 scenarios for the value of ‘# of Servings’

* 1 (default) 
** By default, Single Serve enabled. # of Servings is enabled but is set to 1, therefore Strict Batching is inapplicable
* Less than 1 = child row (1/6 Pizza → Slice)
** If user disables Single Serve toggle, that row becomes locked/not editable and a child row is added which has the editable fields
*** on this row the Strict Batching is also provided, but default to on
* Greater than 1 = scaled (20 serving of 5 nuggets)
** If user changes to # of Servings to 2 or more, Strict Batching option becomes available and is default to of

## Acceptance Criteria
* Modify the Units card to serve as a Serving Size header selection for configuring Single Item Recipes
* Provide appropriate interactions between Units and the SIR configuration to manage and view Serving Sizes associated with the SIR
* Accommodate the somewhat rare changing of ‘# of Servings’ for Single Item Recipes
* Introduce a ‘Single Serve’ flag, which means the item is served directly and # of Servings will be 1 or greater
* Provide the ability to disable the Single Serve flag and load a child row where a “sub serving” can be input
