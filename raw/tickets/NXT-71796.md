*Description:*                                                                                                                                                                                      
  Display a Menu Analysis bar on the Menu Detail page (View Mode) between the Menu Header and Menu Items sections, as shown in the attached mockup image.                                                                                                       
 

*Requirements* (refer to attached image):

# Analysis bar shows two rows — a header row with column labels and a data row with values
# Columns: Menu Analysis label, Calories, Sat. Fat, Sodium, Carbs, Fruit, M/MA, Grain, Milk, Veg, and Analysis result
# Data row values are displayed as solid colored chips — green for Pass, orange for High/Fail, darker orange for Exceeded — with white text (not outlined boxes)
# Legend chips (Pass, High/Fail, Exceeded) appear in the first column of the data row
# Food component columns (Fruit, M/MA, Grain, Milk, Veg) show colored circle icons next to the label, with values in brackets like [10.00]
# Last column shows overall PASS or FAIL with an eye icon and the serving group name, e.g., "Analysis (K-5)"
# Serving group selection is synced — when a user clicks a serving group column header (e.g., PK(25), K-5(200)) in the Menu Items grid below, the Analysis bar updates to show that serving
group's data. The first serving group should be pre-selected on page load so both the grid column and the analysis bar are active by default.

  API: GET /api/Menus/{menuId}/Analysis — returns analysis data per serving group. 



**Use Mock Data for now **

## Acceptance Criteria
* Add analysis bar to the standard menu page
* Calculate all analysis related calculations in the API, no more UI calculations
* Allow user to click between  serving groups and refresh the analysis based on the selected serving groups
* Provide a clickable icon on the fail or pass to open the nutrition report in a new tab
