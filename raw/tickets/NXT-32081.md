# NXT-32081 — INV - Create Addition - Items - Pack Size Updates

## Description
As a Site Manager I want the Additions functionality to allow me to submit an addition for any unit in an item’s pack size configuration so I can enter the exact quantities I am adding to my perpetual inventory.

*Requirements*

* Update the Additions screens to reflect the updated item hierarchy configuration
** Items will be searched and managed at the Item # level throughout the Inventory category
*** Ex: I need to add to inventory 2 cases and 4 boxes of Item 100 - Orange Juice
* Display Quantity on Hand
** Present the full string
** Provide Alternate QOH popover ([NXT-26951|https://primeroedgenext.atlassian.net/browse/NXT-26951])
* Other Improvements
** Step 1 selections should be retained when returning from Step 2
* +Update the UI to accommodate multiple unit types+
* Replace Unit Type with Unit Description
** remove the dedicated column, as Unit will be selected after Quantity
* Provide a Unit dropdown after the Addition Quantity
** Display a label instead of the dropdown if there is only one unit in the pack size configuration
** By default, pick the highest unit 
* Provide an icon to expand the pack size hierarchy
** Only if the item has multiple possible units
*** this adds rows below for each sub-unit in the pack size hierarchy
** Load a default price for sub-units based on the Sub Quantity
*** i.e. if a Case is $10 and there are 10 Bottles per case, load $1 as default for the Bottle
** Remove the dropdowns and lock the row to the specific unit label
** Provide an icon to collapse the pack size hierarchy if the sub-units aren’t being used
* Fix styling to be consistent with Withdrawals and Transfers
**  Selection labels are not bolded like on Withdrawals
** Retain left alignment
