# NXT-49560 — Cancelled API – Show All Fields – PowerSchool – Student - Front End

## Description
As a SchoolCafé admin, I would like to see all available source fields of the custom API call so that I can map whatever I want.

*Notes:*

* This is the story for the front end part of the Student Show All Fields Feature
* This feature is for the API Student Import only
* Backend is covered in a previous story

*Requirements*:

* For PowerSchool Student API Import Add New and Edit, do the following:
* On the Field Mapping screen, do the following:
* Add a button to the API Fields table (first one) next to Map / Unmap button
** Label = More Fields
** On click, open the More API Fields drawer



More API Fields drawer

* Add a drawer:
** Title = More API Fields
** Width = 1/4
** Do the sliding open motion
* Fields:
** Information text: This drawer contains less commonly used fields. You can add additional fields by clicking the checkbox and they will show up at the bottom of the API Fields table. You can drag and drop the field on the Mapped table to include in your import configuration. To remove the field, uncheck it.
** Grid
*** Note: This can be a multi-select list with checkboxes as well, whichever works best
**** If doing the multi-select list, don’t do the +1 thing, show all the fields by expanding the list
*** Data:
**** List of fields from the source API
**** Note: you have to make a choice:
***** Show all fields including the original ones
***** Show only the additional fields
***** Both have their plus and minus:
***** The underlying problem is how to remove a field from the list
***** If you show all fields, we would be allowing the user to remove fields that are needed like StudentID
***** If you show only the new ones, you would have to put logic to distinguish between the original list and the extra fields
***** I am ok with either way, as long as the requirements are met but these requirements assume you will only show the additional columns in the list
*** Columns:
**** No column name
***** Checkbox
****** If checked, show in the API Field table
****** If unchecked, remove from the API Field table
**** Name
***** the name of the field that shows in the table
*** Validators:
**** If a field is mapped (dragged to a field in the Mapped table):
***** Either we don’t let them uncheck by making it read only, or:
***** we show a confirmation box and then automatically do the unmapping for them
***** I am in favor of making a mapped field read-only with a tooltip: Unmap field first
**** A field can’t be added twice to the table
*** Save
**** On click, update the API Field table by adding or removing the fields 
** 
** Note: there is no “Delete” button. The unchecking of a field equals deleting

## Acceptance Criteria
* User can add extra fields to the API Fields table
* The extra fields can be used like any other field such as drag and drop, used in imports, etc.
* User can remove the extra field
* If user removes a field from API Fields table, it cannot continue to be mapped to a field in the mapped table
