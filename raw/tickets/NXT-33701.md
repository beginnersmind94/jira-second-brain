# NXT-33701 — INV - Reports - Usage Report - Update to include Pack Size info

## Description
As a District Admin I want an updated Usage Report which includes appropriate labelling details about pack size configuration so I can accurately read the quantity values and understand according to the pack size units.

*Note:* this will be dependent on some changes to the Period Valuation page’s Generation function, which is currently captured as part of the [Valuation Report Changes Story|https://primeroedgenext.atlassian.net/browse/NXT-33501]

*Requirements*

* Update the Usage Report to accommodate and communicate Pack Size configuration
** This requirement is different for Usage, which shows the Quantity fields for every transaction type (7+), therefore requiring that we do *not* show the full string each time for each Usage t
* Add Pack Size info somewhere
** Could add to Item Description cell (like PE) or somehow add as dedicated column (probably overkill)
* Add split Quantity sub-columns for different unit types (like PE)
** To be named: Higher Unit, *Whole Unit (WU)*, Purchase Unit?
*** this will always represent a higher unit, assuming it is a 2 or 3 tier hierarchy item
** To be named: *BU (Base Unit)*
*** Base Unit column will display the decimal values
*** this will be the only column populated for items with only *one* pack size unit (1-tier)
* Display 3-tier hierarchy items in modified 2-tier format
** For 3-tier items, convert the lowest unit quantity into middle unit and display as decimals
** Display an asterisk or something to denote as a 3-tier item, so user knows it’s being converted for display purposes
*** Display a footnote for this asterisk: “Items with 3 pack size units are being converted to be displayed as 2 units.”
* Update report values
** Update Quantity and Value calculations to consider all item units 
** Update for all Report Type variations
*** Storage Category
*** Valuation Group
