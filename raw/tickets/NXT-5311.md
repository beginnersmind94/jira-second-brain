# NXT-5311 — Onboarding - Feature Selection - Add Tour

## Description
As a new SchoolCafé user, I would like to have the option to view a tutorial that explains in detail how my district can go live with a particular feature suite, so that we can eventually complete our onboarding of SchoolCafé.

*Summary*

Implement Shepherd.js tour steps on the Feature Selection page. This will explain what the user needs to be aware of and what they need to do on this initial screen.

*Requirements*

* *UI:* When the user first navigates to the Feature Selection/Set Up page, automatically call a Tour with the following steps, anchoring them in the location and direction shown in the image above. 

# +*Initial Step*+ (centered on the page):
## Title - *The Recipe for Success*
## Description - Check out our short, step-by-step process to learn how you can quickly get started with SchoolCafé by selecting *Start Tour* below! Otherwise, select *Skip*.
## Footer - 
### Button: *Start Tour*: Proceed to step 1. Color: {color:#6554c0}*Purple* {color}(*#665AD8*).
### Button: *Skip*: Close the tour and allow the user to continue using SchoolCafé as normal. Color: {color:#97a0af}Gray {color}(*#737373*)
# +*Step 1*+ (left of Go-Live field): 
## Title - *Step 1 of 3: Choose a Go-Live Date*
## Description - Select a target date to go-live with SchoolCafé (don’t worry, this can be modified at any time). It is recommended that you set this to at least 1 week from today.
## Footer - 
### Button: *Next*: Proceed to step 2. Color: {color:#6554c0}*Purple* {color}(*#665AD8*).
### Button: *Exit*: Close the tour and allow the user to continue using SchoolCafé as normal. Color: {color:#97a0af}Gray {color}(*#737373*)
# +*Step 2*+ (above System card): 
## Title - *Step 2 of 3: Choose a Feature*
## Description - Click or tap on the *System* feature. This will allow you to configure things like sites, users, and role permissions (not required, but optional. More on that later!). After you have completed System set up, you can return to this screen to begin onboarding other features.
## Footer - 
### Button: *Previous*: Return to step 1. Color: {color:#97a0af}Gray {color}(*#737373*)
### Button: *Next*: Proceed to step 3. Color: {color:#6554c0}*Purple* {color}(*#665AD8*).
### Button: *Exit*: Close the tour and allow the user to continue using SchoolCafé as normal. Color: {color:#97a0af}Gray {color}(*#737373*)
# +*Step 3*+ (left of 'Let’s Get Started' button): 
## Title - *Step 3 of 3: Get Started!*
## Description - Once you have selected the *System* feature, the *Let’s Get Started!* button will become available. Selecting this button will take you to the list of tasks to complete for the feature you have selected.
## Footer - 
### Button: *Previous*: Return to step 2. Color: {color:#97a0af}Gray {color}(*#737373*)
### Button: *Finish Tour*: Close the tour and allow the user to continue using SchoolCafé as normal. Color: {color:#36b37e}*Green*{color} (*#5DE6B2*)

* *API (not required?):* We will save the user’s tour progress/button selections in the next story. For now, display the tour every time the user returns to this page or refreshes it.

## Acceptance Criteria
Verify that when a user comes to the Feature Selection screen, they immediately see the initial step allowing them to start the Tour

Verify that the user can immediately Skip the tour

Verify that the user can click Start Tour and be taken through the 3 additional steps

Verify that the UI/UX of the text and button colors for the 3 steps is according to the description shown above

Verify that the user can go backwards or forwards in the tour or end it at any time

Verify that the user can see it again if they navigate back to this page or refresh it (not through finishing)
