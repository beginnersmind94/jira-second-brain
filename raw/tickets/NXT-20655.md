# NXT-20655 — IM- Variety Menu Item

## Description
As a Menu Planner, I would like to have a way to "group" menu items together under a single "variety Menu Item. 
EX: Kitchens sometimes have 10 different types of cereal and want to have them all available on the menu, however they do not want to have to add them one by one and want them on the menu under "Cereal Variety".

## Acceptance Criteria
*Entry Point*

* Item Management → Menu Items → Add → Variety Menu Item



* When user selects "Variety Menu Item" check, disable Leftover Category, Maximum Carryover Days,  Preparation Site Item, Show in Summary, Favorite.
* Only fields available to configure are:
** Menu Item Name (required)
** Marketing Name
** Marketing Description
* *Once saved, this item will not be able to be used to add recipes to it*, but the purpose is to add menu items to it. 
* This menu Item will act primarily as a "placeholder" on menus for the menu items added to it. 
* Display all menu items in a list view that are added to this "variety menu Item" 
* Variety Menu Item Contribution that is set will only allow users to search and select menu items with matching component data in any serving size.
* Display all items in search but gray out items that do not match Variety Menu Item Contributions at any serving size level.
* Allow user to remove items if added incorrectly or user comes back later to make changes.
* Menu Item Category (required) 
* -Menu Item Link box: (display menu items that  Match serving size from contribution)-
* -Serving Size (Use Menu Item serving size from contribution)-
* -Units (Use standard from contribution)-

General:

* Use Local Data Source
* Flow: 
** SAVE / EDIT / CANCEL  buttons needs to be added at sections: Main Card, 
** Selected menu items of Menu Items
** If there is at least a menu item added contribution accordion should be disable
* Nutrition information and Allergen information are only auto calculated
