# NXT-34143 — Team Hub - Perform Incremental Push & Pull of Messages

## Description
At various points in the mobile application, it will need to do an incremental push / pull and reconciliation of messages with the server so that the app stays in sync with the server.

 

*Flow:*

·         Upon timer trigger, or

·         Upon Sending a message

·         Incremental Push / Pull is called, and app messages are reconciled to server’s messages

 

*Background:*

·         This code is called from the message timer and the send message screen

·         Put here as a separate story so can be reused for both

·         This does the same thing as the full push / pull, therefore consider just updating that code and adding a property to do full or incremental and updating accordingly

 

*Requirements*:

·         Fields: None, backend code only

·          

·         Source triggers this code to run:

·         The app does exactly all the same things as the full push / pull, except with the following restrictions:

o   Use the timer to restrict which messages to pull

o   Only pull messages since the last time you pulled

o   This effectively creates an incremental download and saves on system resources

·         Like the full push / pull, even in incremental mode you will still:

o   Send all un-pushed messages

o   Save messages locally

o   Reset the timer once done

o   Etc.

·          

·         Security – already covered in previous stories
