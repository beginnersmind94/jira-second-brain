# NXT-67540 — User Story 5: Store and Manage Account Code Structure Versions (Backend)

## Description
*As a* system
*I want to* track all changes to account code structure, categories, accounts, mappings, and system accounts
*So that* users can view a complete audit trail of modifications

*Background:* Users need visibility into what changed, when, who made the change, and what the before/after values were across all general ledger configuration areas.

## Acceptance Criteria
*What Gets Tracked:*

* Account Code Structure changes (segment additions, modifications, deletions)
* Category changes (creates, updates, deletes, imports)
* Account changes (creates, updates, deletes, imports)
* Mapping changes (creates, updates, deletes)
* System Account changes (creates, updates, deletes)

*History Record Storage:*

* Each change creates a history record with:
** Change Type (Account Code Structure, Category, Account, Mapping, System Account)
** Action (Created, Updated, Deleted, Imported)
** Changed By (user ID)
** Change Date (auto-set timestamp)
** Record values based on action type:
*** For Creates: Store the new values (after state)
*** For Updates: Store both before and after values of all changed fields
*** For Deletes: Store the values that existed before deletion (before state)
*** For Imports: Store the new values (after state) and batch ID to group related imports
** Batch ID (for imports only, to group related changes)

*History Retrieval:*

* System can retrieve all history records for a district
* Records can be filtered by Change Type, Action, and Date Range
* Records are sorted by Change Date descending (newest first) by default
* System supports pagination for large history sets

*Detail Display:*

* For Account Code Structure: Store segment configuration (before/after based on action)
* For Categories: Store category code, description, base type (before/after based on action)
* For Accounts: Store account code, description, category (before/after based on action)
* For Mappings: Store system account code, GL account ID/code (before/after based on action)
* For System Accounts: Store system account name (before/after based on action)
* For Imports: Store batch ID to retrieve all items imported together, along with their values
