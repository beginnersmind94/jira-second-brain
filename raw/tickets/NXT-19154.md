# NXT-19154 — Family Hub SC - Apply/Application Template - Add full support for Income Surveys

## Description
As a household, I would like to be able to respond to income surveys online, so that I can easily provide my school with the income information they need to be in compliance.

*Summary*

Use this story to take care of any needs related to the Income Survey step of the Family Hub, from SchoolCafé setting integration to additional UI needs.

*Requirements*

* +Application Template side:+
** In the Family Hub → Benefits → Application Template, add the infrastructure for Income Surveys.
*** This is already complete in Classic. We can copy the same details to the 2.0 Family Hub.
**** ‘Income Survey' should be added as a conditional step (see behavior in ‘Apply’ section below). It is labelled as 'Household’ on the Apply page, but contains unique fields for the user:
!Capture 2021-09-28 at 8.18.34 PM.png|width=819,height=436!

!Capture 2021-09-28 at 8.20.04 PM.png|width=1427,height=571!

**** The Review step should hide the traditional Household step and instead show Household Income data when an Income Survey was filled out (also see behavior below). Link should take the user back to the Income Survey step. (again, it is labelled as ‘Household’ on the Apply page):
!Capture 2021-09-28 at 8.45.57 PM.png|width=1167,height=252!

**** Submit step shows new message confirming that the user submitted an Income Survey and not an application. 
**** !Capture 2021-09-28 at 8.47.44 PM.png|width=538,height=242!
All of these should still be editable to some degree by the user, with the changes being reflected on the ‘Apply’ page for a given language.



* +Apply side:+
** This should already be complete in Classic as well. We can copy the same details to the 2.0 Family Hub.
** In Benefits → Apply, change the application experience for the user +based on the 3 possible+ +setting values+ for the *_Enable Income Surveys (with License)_* setting: 
*** No
**** IF this setting value is set to ‘No’, the current Apply page is unchanged and the user should only be able to see and submit a Meal Application as they already can. Do not display the ‘Income Survey’-related steps or messages.
*** Yes
**** IF this setting value is set to ‘Yes’, always display the ‘Income Survey’ version of the Household step when the user selects ‘No’ on the ‘Assistance’ step and then clicks the Next button. Other logic should follow from there.
*** Mixed
**** IF this setting value is set to ‘Mixed’, *only* go on to display the ‘Income Survey’ version of the Household step as above +*IF*+ all students added on the Students step are enrolled in CEP, Provision I, or Provision II sites. If any of the students added there are enrolled in non-CEP/Provision sites, direct the user through the Meal Application as normal, and not the Income Survey verison. Other logic should follow from there.
*** Look, feel, and behavior can match Classic SchoolCafé. i.e. get the exact household amount from the parent rather than income range, along with household size and income frequency:
** !Capture 2021-09-28 at 8.58.14 PM.png|width=1170,height=545!
!Capture 2021-09-28 at 8.21.29 PM.png|width=85.65737051792829%!
** Upon submission, the survey image does not have to show, if we have not yet developed one.
** !Capture 2021-09-28 at 8.21.53 PM.png|width=1200,height=403!
** When the survey is submitted successfully, it should show up in the Surveys grid just like any manual survey.
*** Income Surveys should be processed automatically that are submitted through online (FamilyHub)
*** Status should be 'Processed'

*No UX Required*

## Acceptance Criteria
Verify that the Application Template page experience matches Classic SchoolCafé in how it treats districts that can accept Income Surveys

* User can view Income Survey step
* User can view Income Surveys in Review step
* Messages can be edited on both of the above in any language

Verify that the Apply page experience matches Classic SchoolCafé in how it treats districts that can accept Income Surveys

* User can view Income Survey (‘Household’) step
* User can view Income Surveys (‘Household’) in Review step
* All of the above differs based on the setting values and languages selected
