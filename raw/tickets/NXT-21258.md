# NXT-21258 — Add Publishing Models to Sites

## Description
As a site administrator, I want to specify which sites to add to the model I build so that I can have different models for different schools

*Application*: Face and Food Detection Website

*Flow:*

* Browse to the Food Detection Settings page and then the Model Management Page
* Click on the Create Model button and come to this page
* On the first tab, do the configuration and click next
* On the second tab, select the publishing information
* Select Create and it does everything it normally does

 

*Background:*

* Originally, the code blasts out the model to all registered sites
* We want to be more specific about which sites we want to send a model to
* To do that, we are creating the concept of a Publisher
* This publisher will handle sending models to sites and removing models from sites

 

*Assumptions:*

* You must have done the Add Tabs to Build Model before doing this story, although you can do it as the same time if you want
* Here you will need to know all the sites for a district
* To do this, it is assumed you can do a DB lookup for the district of the user logged in
* At this time, we have decided to not filter the sites to only those with Edge devices

 

*Requirements:*

* All of these requirements are for the Publish tab
*  
* Clear – button, unchecks all the sits
* Select All – button
** When clicked, select all the sites
*  
* Sites – grid
** Dataset = all the sites that have a registered Edge device for the district
*** Not all the sites in a district, but all the sites with an Edge device
** Be able to sort, paginate, filter upon columns, export to excel
**  
** Columns:
** Include – checkbox, clickable
*** If checked, add to the list of sites
** Site – text name of the site, RO
*  
* Create button Changes:
* Remember, this button already exists, so copy over all its functionality
* Keep original validators but add this one:
** Sites – There must be at least one site selected
** If not, show error message “Select at least one site”
* Additional to the existing Create Model code functionality, also need to now save the sites to the Model Site database table (see Update requirements below)
*  
* Security – already covered

 

*Update to Model Configuration Data Structures*:

* There needs to be changes to the Model Configuration as follows
** See Add Model Configuration to Database story for further info if needed
* Add a new data structure and link it to the Models Data Structure:
** Model Sites table linked to Sites table:
** Model ID – Foreign Key to Models table Model ID
** Site ID – unique ID of the site
** Site Name – text, may not need if prefer to join to that table when doing select statements
** Start Date – date/time stamp, will be blank in beginning
** End Date – date/time stamp, will be blank in beginning
