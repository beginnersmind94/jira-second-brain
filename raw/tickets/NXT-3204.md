# NXT-3204 — Reconciliation - Reconcile Info - Opening Balance

## Description
*User Story*

As a manager I want to see the Cash entered as the opening balance for the session.

*Mock-up*

[https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=1137%3A22760&scaling=scale-down-width|https://www.figma.com/proto/yrkJeGfd3f9hWjqm0qvfir/Edge2.0_V1?node-id=1137%3A22760&scaling=scale-down-width]

*Requirements*

* The user can view the opening cash entered for a session
* The user can modify and save the opening cash for a session
* The History of the session is saved [https://primeroedgenext.atlassian.net/browse/NXT-6745|https://primeroedgenext.atlassian.net/browse/NXT-6745|smart-link] 
* Running total of Denominations shows in total under Total Opening Balance
* User can use UP/DOWN to change denomination counts.
* Reusable Component from UI side.

## Acceptance Criteria
The denominations are listed
The user can enter denominations OR a total opening balance
The user can enter denomination counts OR denomination Amounts
If the user enters a count the amount is calculated. and displayed
If the user enters an amount the count is calculated and displayed
When amounts are entered that result in a partial coin amount the user will be displayed a message "The entered amount results in a incorrect coin count. Please recount and enter the correct amount" The amount is not accepted until corrected. Example Entering $5.20 for Quarters results in 20.8 as the count for quarters.
Running total of denominations entered.
User can increment and decrement denomination counts with a click.
All amounts and counts can be cleared with a single click
Changes can be saved
Changes can be canceled (prior to saving)
Session history is saved when opening balance is changed and saved.
