# NXT-49316 — OpenID - Login - More Options Screen

## Description
As a SchoolCafe user, I want to be able to login to the site when my SSO authentication fails so that I can still use the app

*Notes*:

* This story is for the following use case:
** User has SSO as their default auth method, but the district also allows other methods like SchoolCafe and 2FA
** User is unable to authenticate for whatever reason with the SSO provider
** We need to enable them to login by another way
* We will be adjusting the flow of the login page to account for this use case

*Flow:*

* User browses to Login page
* User asked for username and hits Continue
* System checks the authentication type for the username and it is SSO 
* System redirects the user to their SSO provider, they fail to authenticate there
* SSO provider sends a failure message
* System redirects the user to this page so that they can choose another method to login
* User clicks on another method and goes to that screen
* Optionally, the user can click back and try again with the SSO provider
