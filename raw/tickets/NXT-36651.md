# NXT-36651 — Monitoring Scheduler – P3 Edit Monitor Page

## Description
As a SchoolCafé admin, I want to be able to edit existing monitoring jobs so that I can proactively monitor the application and ensure higher customer satisfaction.

*Flow:*

* User logs in and browses to Support > Monitoring > Job Scheduler
* User views a list of scheduled monitoring jobs
* User clicks edit on an existing scheduled job

*Notes:*

* This is a continuation of the story to create the Job Scheduler page
* This story assumes that story has been created and that the Monitor tab was created and the grid on that page has the edit button
* This story assumes that the Add New Monitor story has been completed and that users are able to add new monitoring jobs already
* Note that each row in the grid already has an edit button that does not do anything yet

*Requirements*:

* Requirements for the Job Scheduler > Monitors tab
* Requirements are for what happens when a user clicks the edit button on a row in the grid of scheduled jobs
* For the edit button, on click do the following:
** Open the Edit Monitor Page
* 
* Requirements for the Job Scheduler > Edit Monitor Page:
** Try to leverage the existing add new page and just using logic to determine if adding a new one or editing an existing one
** Page title: Edit Monitor instead of Add New Monitor
** After a user clicks the edit button on the Monitor Tab, this page opens up
** This is the exact same page as the Add New Monitor Page, exactly the same in every way
** The only difference is that on clicking the save button, you update the existing job instead of creating a new one and record the changes in the history log
* 
* History Log Requirements:
** Any time we edit, we need to save the changes to the history log
** If a history log doesn’t exist, create one like Front Office uses for the System Notifications edit notification page
** Get the logic from that page and do it the way they do it
** Save the following to the log when updating:
*** Field Name – which field was changed
*** New Value – the new value in the changed field
*** Old Value – the old value in the changed field
*** Updated By – who is doing it, used for the last updated by value in the Monitoring grid
*** Updated On – date / time stamp of when make the update, used for last updated in the Monitoring grid
* 
* Permissions:
* If you are doing a real new page, then does these permissions, else leave them alone as it will inherit the existing pages’ permissions:
* In General resources screen for Roles & Permissions
** Application: applies Job Scheduler edit permission
** Write the security code to apply that permission to this button
