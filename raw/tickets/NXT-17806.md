# NXT-17806 — Verification - Reports - Inactive Applications 

## Description
As a district admin I want a way to see collated information on what happened during verification.

*mockup*

[https://www.figma.com/proto/Y3tlwN2yvRc67304adzyyB/Eligibility-3.0?node-id=5133%3A21410&scaling=min-zoom&page-id=1%3A254608&starting-point-node-id=1%3A258308|https://www.figma.com/proto/Y3tlwN2yvRc67304adzyyB/Eligibility-3.0?node-id=5133%3A21410&scaling=min-zoom&page-id=1%3A254608&starting-point-node-id=1%3A258308|smart-link] 

*reqs*

* Add an Inactive Applications tab to the Verification → Reports page, using the academic year and apply to search for results.
* This should just show the inactive apps that are counted as part of the 10/1 count that have gone inactive at any time after 10/1.
** This 10/1 date would be the Final Verification Sample date.
* Results in a grid, columns:
** Application #
** Student ID #s
** Application Process Date
** Eligibility Start Date
** Inactive Date
*** This would need to be the date the last student on the app moved to another source of eligibility.
* Add an explanation to the page: The following applications have gone inactive (all students have moved to another source of eligibility) after the Application Count Date (10/1/XX).
** Pull the date from system settings for this text, or if the user set alternate dates, use those instead.
** It’s the final sampling date.
