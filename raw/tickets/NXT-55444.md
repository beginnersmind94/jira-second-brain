# NXT-55444 — MP- Menu Planner- Final check on save for Record Complete Menus

## Description
As a Menu Planner, I would like for the system to check when I am saving a menu and not allow changes to that I am making to a menu if the menu is now associated with a record complete production record.

## Acceptance Criteria
* Do Final Check on *Save or Save and Close* to check if menu has a record complete record associated 
* Provide a banner error message “Menu: Menu Name could not be saved! One or more Production Records have been completed associated with this menu.” on *save* change
* If more than one menu has been affected at the same time when on the menu planner, provide error message “Menus: Menu Name, Menu Name, Menu Name could not be saved! One or more Production Records have been completed associated with these menus”
* ON save and Close scenario, do not lead user back to the list page if there are the errors above and provide the errors
* ON both scenarios, clear all changes that were made for the menus affected and leave and *SAVE* changes that were made on other menus
* Menus after this should be greyed out and not allowed for changes to be made
