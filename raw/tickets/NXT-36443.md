# NXT-36443 — Meal Pricing – Scheduled Job

## Description
As a SchoolCafé admin, I want to ensure that Meal Pricing is reconciled on a nightly basis so that I can proactively monitor the application and ensure higher customer satisfaction.

*Flow:*

* User browses to Support > Monitoring Dashboard > Job Scheduler
* User clicks the + New button
* User Selects Accountability Module and selects Meal Prices as a Monitor
* User sets the schedule and saves
* The system sets up a recurring job at the time and days selected by the user
* The system runs the jobs per schedule and output is viewable in the Monitoring pages

*Notes:*

* This story assumes that the screens for Job Scheduler and Add New Job have been created
* We have been told that the script already exists that does the checks, we just need to update it
* The script compares the price charged for a purchase versus what should have been charged

*Requirements*:

* *Job Scheduler Page updates:*
* Ensure that Meal Prices is in the Monitor drop down list
** Via API, not hard coded
* Ensure that after a Monitor job is created, that it shows up in the grid with the correct values
* 
* *Add New Monitor Page updates:*
* Add or ensure that Accountability is a value in the Modules list
* Add Meal Prices to the list of values for the Monitor drop down list
* When the user selects Accountability, the list of values for Monitor are restricted to that Module so ensure that Meal Prices is one of the values visible for Accountability
* 
* Schedule section: no additional requirements
* Save button:
** On click, create the scheduled job per the dates and time selection in the schedule section (what to do in detail follows below) and then redirect to Job Scheduler screen
* 
* *Job Requirements*:
* Take the existing script and make the following updates:
* Change the email it sends errors to the new support email we created in the previous story
** remember that there are 2 emails, one for prod, one for non-prod
* 
* For every charge that does not reconcile:
* Automatically call a script to force the system to update to the correct charges
** This script already exists, just use it
* If successful in pushing the update, then reconcile again
* If successful in reconciling that charge, then keep on going for the remaining charges
** If reconciles, se the status in the email record to the email that shows it failed but was fixed using the status column, Status = Repaired
** If fail to push or reconcile the student, then add him to the email exception body (described below)
*** Status = Failed
* 
* If there is any record not reconciled or if it was repaired, send an email
* Email:
** Subject: Account Balances Exceptions – District Name
** Body: List all the exceptions
** Table Columns / Fields:
** Student ID
** Student Name
** Student School
** Transaction Date
** Transaction Price Before
** This is what was actually paid
** Transaction Price Required
** This is what should have been paid
** Menu Item(s)
** Status
*** Values: Failed, Repaired
* 
* Permissions: no extra requirements
