# Accountability guide edits — review (strict citation standard)

**14 proposed edits across 8 guides.** One earlier edit (NXT-65306, GUIDE-021) was **withdrawn** under the strict standard because its AC field is `-` (empty) and Release Notes are blank — no verbatim source to quote.

**Standard applied to every edit in this document:**

- Every proposed claim traces to a **verbatim quote** from Acceptance Criteria or Release Notes.
- Verbatim quotes appear in blockquotes (`>`).
- Anything not directly quotable is either labeled as **Interpretation** or flagged as an **Open question**.
- Where Description supplies a specific detail (UI label, menu path, value) not echoed in AC or RN, the detail is **not** silently included — it is surfaced as an open question.
- **Internal jargon is translated, not quoted.** AC sometimes uses dev-team shorthand (system-setting codes like `BABECL`, abbreviations like `CRUD`). These are flagged in an Interpretation note and rendered into customer-facing language in the proposed text. The original code is preserved in the Interpretation note and Open questions so the SME knows what to look up.

## Edit 3 — GUIDE-013 (Reconciliation)

**Section:** New subsection appended after *Reconcile a Closed Session*.
**Citations:** `NXT-36439` (resolved 2023-04-18) and `NXT-53996` (resolved 2024-08-23).

**Source — NXT-36439 AC (verbatim):**

> * Delete Session option added.
> ** Requires a confirmation from the user.
> ** Requires a comment.
> * Undo button available for 30 seconds before deletion begins.
> * Deletion to be done via adjustments.
> * Adjustments of all types must be available under the Transactions > Adjustments page.
> * Adjustments should use the comment from the user in this format: Session Deleted: comment
> * Rollups run afterwards.
> * Role Permission added.

**Source — NXT-36439 Release Notes (verbatim):**

> An option to delete a session has been added. Role permissions for this have also been added.
> A 30 second 'Undo' option is available after confirming the delete, but after that all transactions in the session will be adjusted off and the session will be updated to a new status of Deleted.
> Support will not be able to reverse the deletion; we advise being very certain before starting the deletion process.

**Source — NXT-53996 AC (verbatim):**

> * Online Payment sessions cannot be deleted.
> ** User is informed why.
> * Delete Session Confirmation prompt updated to tell the user how many transactions and $$ payments are in the session.

**Source — NXT-53996 Release Notes (verbatim):**

> We have temporarily removed permissions to delete sessions. If you need a session removed, please contact our Support team for assistance. The permission will be returned in the future along with an option to restore a deleted session.
> However, we have added in a permanent blocker to prevent sessions at Central Office from being deleted. These sessions typically contain online payments from Family Hub, along with refunds and other account-related actions taken by staff at the Nutrition Office. You will need to contact Support if one of these sessions does need to be removed.

**Current text in the guide:** *(no equivalent — this is a new subsection)*

**Proposed new subsection:**

> ### Delete a Session
>
> An option to delete a session has been added, gated by a role permission. Online Payment sessions cannot be deleted, and a permanent blocker prevents sessions at Central Office (which typically contain Family Hub online payments, refunds, and other account-related actions) from being deleted; contact Support if one of these needs to be removed.
>
> 1. Select the Delete Session option
>    A confirmation prompt requires a comment. The prompt tells the user how many transactions and dollar payments are in the session.
> 2. Confirm the delete
>    A 30-second Undo is available after confirming. After the 30 seconds, all transactions in the session are adjusted off, the session is updated to a new status of Deleted, and rollups run afterwards. Support will not be able to reverse the deletion — be very certain before starting the deletion process.
> 3. After deletion, the transactions appear as adjustments on the Transactions / Adjustments page
>    The adjustment uses the user's comment in this format: "Session Deleted: <comment>".

**Source contradiction surfaced:** NXT-53996's RN says delete permissions were **"temporarily removed"** pending restore-session capability; NXT-53996's AC speaks only of the **permanent** Central Office block. NXT-36439's RN treats delete as available. The proposed text reflects what AC and the permanent-block portion of NXT-53996 RN both agree on, and omits the temporary-removal sentence because NXT-54040 (Restore Session, 2024-10-25) is the path forward referenced by NXT-53996's RN but is not part of this pass.

**Open questions:**

- Where exactly is the Delete Session control in the UI? *(Description: "More Actions dropdown on the Reconciliation page" — not in AC or RN of either ticket.)*
- Should Restore Session (NXT-54040) be documented alongside Delete? NXT-53996's RN explicitly anticipates it; without it, this subsection may overstate finality.

**Proposed replacement:**

> The End of Day function is a self-management tool for site managers, allowing them to review and complete Daily Tasks. These Daily Tasks include Sessions, Marked for Review, Suspicious Transactions, Edit Checks, Serving Exceptions, and Deposit Slip. An additional End of Day task is available when the Attendance entry method is set to Daily and at least one site is configured for SNP; this task is toggled off by default. Once enabled, its Summary shows "Count: <n>" or "Not Recorded"; Status is Not Started if no count has been entered, and Complete once any count (including 0) has been entered. The task's action is a Record button that lets the user record or edit the attendance count for the site + date. The task count on EOD and the FOH Dashboard increases when the task is turned on. Also, the history for the individual task and the site and date can be viewed as needed. Reports can also be generated.
