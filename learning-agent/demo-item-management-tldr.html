Item Management — TLDR

Module: Item Management  |  Template: TLDR One-Pager  |  Source: REC-2026-0515-IM-001 (May 15, 2026)

What this module does

Item Management is the central catalog where district staff create and configure every product — food and non-food — that feeds Inventory, Menu Planning, Point of Sale, and Production Records.

Key features

FeatureWhat it does
Non-Food Item creationCreates supply items (cups, trays, gloves) with pack sizes and costing.
Food Item creationCreates served-food items with full nutrient, allergen, and meal-pattern configuration.
Multi-tier Pack SizesSupports up to three pack tiers (e.g., Box → Sleeve → Cup) linked by Sub-Qty.
Serving SizesDefines the serving weight/volume used in recipes and menu planning (food items only).
Serving-to-Pack LinkingConnects serving sizes to pack tiers via Sub-Qty to unlock Inventory readiness.
CostingEnters cost for one pack tier; system auto-calculates all other tiers from Sub-Qty.
Nutrient InfoRecords label nutrients per serving; drives nutrient analysis in Menu Planning.
Allergen Info + Custom AllergensAssigns standard and custom allergens with Contains / May Contain / Free From status.
Contribution InfoSets meal-pattern credit (Meat/Meat Alt, Grains, Vegetables, Fruits, Fluid Milk).
Direct Menu Item (DMI)Makes an item placeable on a menu without a recipe; enables POS Pricing tab.
POS PricingConfigures per-eligibility-type prices (Free, Reduced, Paid, Second Meal) by site type.
Steps Card (HACCP)Assigns HACCP process (No Cook, Same Day, Complex) and critical control points.

Who uses it

District Admins and Implementation Specialists create and configure items; Menu Planners and Nutritionists manage nutrient, allergen, and contribution data.

Common workflows

  • Create a non-food supply item: Item Card → Units Card (select Non-Food) → add Pack Sizes → enter Costing
  • Create a food item: Item Card → Units Card (Food / Fluid?) → add Serving Sizes → add Pack Sizes → link servings to packs → Costing
  • Configure nutrients, allergens, and meal-pattern contributions on the food item's detail tabs
  • Create a custom allergen in Item Management > Configuration > Item Configuration > Custom Allergens before applying it to any item
  • Enable Show on POS on the Menu Item card, then configure POS Pricing by eligibility type and site

Important gotchas

  • Base Unit locks after first Inventory transaction. Once an item appears in any count, receipt, or adjustment, Base Unit and Whole Numbers/Decimals are permanently locked — contact Cybersoft support to correct.
  • Save the Units Card before detail tabs activate. Nutrient Info, Menu Info, and Inventory Info tabs stay grayed out until the Units Card is saved.
  • Custom allergens must exist before you can apply them. An allergen not created in Item Configuration first will produce an empty dropdown on the item's allergen card.
  • Item Category, Storage Category, and Valuation Group are required when Inventory is enabled. Configure these in district settings before creating items to avoid retroactive data entry.
