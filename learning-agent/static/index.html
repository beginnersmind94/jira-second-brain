    How Learning Studio works

    Every guide begins with a real training transcript. Learning Studio researches the matching Jira tickets, runs a four-stage pipeline to write the guide, and then refuses to ship anything until an automated grounding check confirms each claim traces back to a genuine source.

    Every guide flows from a training transcript through grounded research and a four-stage pipeline, then must pass automated grounding checks before reaching the Library.

    1 Map

    Parses the uploaded transcript and matches each feature to the right Jira tickets for the selected module.

    2 Plan

    Chooses which workflows to cover and how deep — mining stories, bugs, tasks and epics for the right material.

    3 Verify

    Pulls Acceptance Criteria and Release Notes to confirm every claim against its true source and trust tier.

    4 Draft

    Writes the guide and cites each claim by source ID — exact quotes are inserted from the registry, never typed by the model.

    Grounding

    Jira search → load epics → read tickets supplies the verbatim, correctly-tiered evidence the pipeline cites.

    Eval gate

    Automated grounding checks score the draft; only provably-grounded guides reach the Library, marked Pending SME Review.
