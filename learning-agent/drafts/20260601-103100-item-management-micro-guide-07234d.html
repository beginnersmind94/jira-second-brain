Item Management: Creating Food and Non-Food Items — Micro Guide

Purpose

After reading this, you can create a non-food supply item, create a food item with serving sizes and pack size links, configure standard and custom allergens, and set up a Direct Menu Item (DMI) with POS pricing — giving each item the configuration it needs to flow correctly into Inventory, Menu Planning, and Point of Sale.

Who This Is For

District implementation specialists and district admins onboarding to SchoolCafé 2.0. Menu planners and nutritionists also use this guide when configuring Nutrient Info and Allergen Info on food items.

Before You Start

  • Item Category, Storage Category, and Valuation Group must be configured in district settings before creating items. These fields are required when Inventory is enabled.
  • Valuation Group method matters: if the selected Valuation Group uses the Standard Price valuation method, costing is required for Inventory readiness.
  • Custom Allergens (any allergen not in the standard list) must be created first at Item Management > Configuration > Item Configuration > Custom Allergens before they can be applied to item records.
  • Login credentials must be provisioned by the Cybersoft implementation team before staff can access the module.

Top Workflows

1. Create a Non-Food Item

Build a supply item (cups, trays, napkins, gloves) with pack sizes and costing so it is ready for Inventory.

  1. Navigate to Item Management > Items and click + NEW.
  2. On the Item Card, enter Item Description (required), Item Category, Storage Category, and Valuation Group. Optionally add Brand Name, Manufacturer, and Product Code for searchability. Click SAVE.
  3. On the Units Card, select NON-FOOD. This disables the Fluid/Non-Fluid and Direct Menu Item questions, which do not apply to supply items.
  4. Click + PACK, enter a Description (e.g., Box), and click the checkmark to save the row. A Missing Costing alert will appear — this is expected; do not address it until all pack sizes are added.
  5. To add sub-tiers, click + PACK again, select LOWER, and fill in Description and Sub-Qty (units of this tier contained in the one above). Repeat up to three tiers maximum.
  6. Click the Missing Costing alert, then the pencil icon. Enter a cost for one pack tier — the system calculates the remaining tiers using Sub-Qty. Click SAVE. A green Inventory Ready flag confirms completion.
⚠ Watch out: Sub-Qty must reflect the standard pack specification, not an occasional short-ship quantity from a vendor. Receiving discrepancies are handled during Inventory receiving — never by changing Sub-Qty.

2. Create a Food Item

Build a food item with serving sizes, pack sizes, and the serving-to-pack link required for Inventory readiness.

  1. Complete the Item Card (same fields as non-food) and click SAVE.
  2. On the Units Card, select FOOD, then NON-FLUID if the nutrition label weight is in grams (e.g., yogurt), or FLUID if measured in milliliters (e.g., milk, juice).
  3. Click + SERVING. Enter Amount (typically 1), Description (e.g., Each), and Weight in grams. Click the checkmark. Add all serving sizes before adding pack sizes.
  4. Click + PACK and add pack tiers (same process as Workflow 1). Maximum three tiers.
  5. Click the Inventory Ready alert ("Servings and Pack Units must be linked"), then the pencil icon. On the serving size row, enter the Sub-Qty — the number of servings contained in the lowest pack tier (e.g., 16 yogurt cups per Carton). Click SAVE.
  6. Enter costing (same as Workflow 1). Confirm the Base Unit, select Whole Numbers or Decimals for inventory counting, then click CONTINUE > SAVE & CLOSE.
⚠ Watch out: The Nutrient Info, Menu Info, and Inventory Info tabs remain grayed out until the Units Card is saved. Save the Units Card before attempting to open any of those tabs.

3. Configure Allergens (Standard and Custom)

Record allergen statuses on a food item so that data flows to Family Hub parent menus and production records.

  1. On the item's Allergen Info card, click the three-dot (⋮) menu and select Edit.
  2. Review the Allergen Feature Disclaimer at the top of the panel: SchoolCafé is a data entry tool and the district retains responsibility for verifying allergen accuracy.
  3. For each standard allergen (Crustacean Shellfish, Egg, Fish, Gluten, Milk, Peanuts, Sesame, Soy, Tree Nuts, Wheat), select one status: Contains, Processed in a Facility That Also Processes, May Contain, or Free From.
  4. To add a custom allergen, first create it: navigate to Item Management > Configuration > Item Configuration, select Custom Allergens, click + NEW, enter the allergen name, optionally toggle Show on Family Hub Menus, and click SAVE.
  5. Back on the item's allergen card, click + ADD CUSTOM ALLERGEN, select the allergen from the dropdown, choose a status (e.g., Contains), click ADD, then click SAVE.
⚠ Watch out: If a custom allergen has not been created in Item Configuration first, the dropdown will be empty when you try to apply it to an item. Always create the allergen at the district level before opening an item record.

4. Create a Direct Menu Item (DMI) and Set POS Pricing

Configure an item so it can be placed directly on a menu and displayed at POS without being part of a recipe — for example, a yogurt cup served as-is.

  1. From the Units Card banner, click CREATE MENU ITEM.
  2. Enter Menu Item Name (required; shown on menus and reports), Button Name (English) (required; displayed on the POS touchscreen), and Menu Item Category (required).
  3. If applicable, set Leftover Category. If anything other than Unspecified is selected, Max Days becomes required.
  4. Check POS Entree if the item counts toward a reimbursable meal; leave unchecked for à la carte extras.
  5. Toggle Show on POS on to reveal the POS Pricing tab. Click SAVE. The tab label updates to Menu & POS Info.
  6. On the POS Pricing tab, click the three-dot menu and select Edit. Enter prices per eligibility type (Free, Reduced, Paid, Second Meal) and site type. Click SAVE.
⚠ Watch out: Verify the POS Entree checkbox before saving. An unchecked item is treated as à la carte at POS and will not count toward a reimbursable meal — this affects meal reimbursement eligibility for students.

Common Mistakes

  • Skipping categorization fields at item creation. Item Category, Storage Category, and Valuation Group feel optional early on but are required once Inventory is enabled. Setting them on every item from day one avoids painful retroactive data entry across a large catalog.
  • Adding pack sizes before serving sizes on food items. The workflow is smoothest when serving sizes are added first, pack sizes second. Reversing the order creates confusion in the serving-to-pack linking step.
  • Entering zero instead of checking Missing Value on the Nutrient Info tab. Zero means the item contains none of that nutrient. If the data is simply unavailable, check the Missing Value checkbox — it displays as (M) in nutrient analysis and correctly distinguishes unknown from zero.
  • Setting the wrong Base Unit or Whole/Decimal setting before Inventory use. Once an item appears in any Inventory transaction (count, receipt, or adjustment), the Base Unit and Whole Numbers/Decimals settings are permanently locked. Correction requires Cybersoft support intervention.
