Item Management: Learning Guide

Overview

Item Management is the foundation of item data in SchoolCafé. Every item you create here carries classification, unit, and nutrient information that downstream processes depend on — so getting setup right matters from the first save.

Food vs. Non-Food: The Defining Question

Every item is classified as either a food or non-food item at creation. That single answer gates a cascade of conditional field requirements throughout the rest of item setup. Specifically:

  • If an item is not a food item, serving sizes are disabled entirely.
  • If an item is a food item, the system also asks whether it is a fluid item, which then switches serving size measurement to volume.

These questions appear in the Units card header before the configuration grid is enabled, so users cannot accidentally skip them.

The Units Card

Pack size and serving size configuration are managed together in a single Units card — a deliberate redesign that keeps the pack-to-serving relationship visible in one place. Both food and non-food items require at least one pack size configuration.

Bulk Onboarding via Import

For large initial loads, items can be onboarded through an Excel-based import workflow rather than one-by-one UI entry. Key import behaviors to know:

Roles & Permissions

Access to Item Management pages is governed by per-page permission levels. The two primary levels that control day-to-day use are View and Add.

How permissions affect the UI

  • When a user holds Add permission for a page, they can create new records on that page.
  • When Add permission is absent, the + NEW button is hidden entirely — it is not disabled or greyed out.
  • Add permission implicitly requires View permission: any page granting Add access also grants View.
  • Conversely, a user without View access will not see the Add option for that page.

In practice, staff who only need to review items should be granted View only. Staff who create or maintain items, recipes, or configuration records require Add in addition to View.

Prerequisites

Before you can create or import items, five configuration records must already exist in the system. The import engine validates each item row against these records at runtime — a mismatch causes the row to be skipped and logged as an error, not silently accepted.

Required Configuration Records

Configuration Where it is set up Validation rule
Item Category Item Management → Item Configuration → Item Category Value must match an existing Item Category record
Storage Category Item Management → Item Configuration → Storage Category Value must match an existing Storage Category record
Valuation Group Item Management → Item Configuration → Valuation Groups Value must match an existing Valuation Group record
Manufacturer Item Management → Item Configuration Value must match an existing Manufacturer record
Brand Item Management → Item Configuration Value must match an existing Brand record

Item Category, Storage Category, and Valuation Group are mandatory for every item — both food and non-food. If any of these three values are missing or do not match configuration, the record fails.

Manufacturer and Brand are optional fields, but if supplied, the values must exactly match their respective configuration tables. Note that Brand association with a Manufacturer is not enforced — a Brand can be assigned independently.

Configuration mismatch error: If a value provided for any configuration-backed field is not found in the corresponding configuration table, the system logs the row with a configuration mismatch message and skips it. Resolve missing configuration records before attempting a bulk import.

Workflow: Creating a Non-Food Item

Non-food (supplies) items follow a distinct path through the Units card because serving sizes do not apply. Understanding where and why the workflow branches keeps setup fast and error-free.

Step 1 — Save the General Info Card First

The Units card sits in the item creation header alongside the general info card. It becomes editable only after the general card is saved.

Step 2 — Answer the Header Questions

Before the Units grid activates, the card presents initial questions in its header bar. For a non-food item, answer No to the Food Item question. This disables serving sizes entirely. After the questions are answered, the grid loads with the appropriate Add Pack Size action.

Step 3 — Build the Pack Size Hierarchy

Use the + Pack button to insert each pack size row into the Units grid. The Description field is a smart text field that matches against configured common measures such as Case, Bag, and similar units.

Pack Level Sub-Qty Required? Notes
First (base) pack No Sub-Qty is not enabled for the first pack size added.
Second pack (child or parent) Yes Sub-Qty links the new level to the existing pack size.
Additional levels Yes Each added level uses Sub-Qty to link pack size units in sequence.

The inventory flag is triggered automatically once pack sizes are added.

Step 4 — Enter Costing

Cost / Value entry is enabled during the finish tasks at the end of the create flow. Select the specific pack size unit on which to enter pricing.

Completion Gate

At least one pack size unit must be configured before the item can proceed to the next step.

Workflow: Creating a Food Item – Units & Costing

The Units card sits in the item creation header and combines pack size and serving size management in one place. It becomes editable only after the General Info card is saved.

Step 1 — Answer the header questions

Two questions gate the grid.

  • Food Item? — Answering No disables Serving Sizes entirely.
  • Fluid Item? — Answering Yes switches the column header to Weight / Volume and enables volume measures for servings.

Once both questions are answered, the grid loads with Add Serving Size and Add Pack Size actions.

Step 2 — Add a serving size

Click + Serving. Fill in the required fields. The Amount field controls eligibility for inventory linking:

Amount valueResult
1Potentially inventory-able — Sub-Qty and base-unit promotion become available
Any other valueServing-only; no pack link possible

Step 3 — Add a pack size

Click + Pack. The first pack size row does not require Sub-Qty. For a second pack added below the first, Sub-Qty is required on the child row; if added above, Sub-Qty appears on the existing (lower) row.

Step 4 — Link servings to packs (Inventory Readiness)

When a serving with Amount = 1 exists, enable Sub-Qty on that serving row and answer: "How many [serving] are in [pack unit]?" Alternatively, enter the pack's Weight to create a weight-based link instead. Items where packs and servings remain unlinked are marked Inventory Ineligible.

Step 5 — Resolve the Missing Costing alert

Cost / Value is editable on pack sizes and any serving linked via Sub-Qty. Select the target unit and enter pricing during the finish-tasks step.

Optional — Promote a serving to Base Unit

If the serving size is potentially inventory-able and should act as the base unit, entering weight on the lowest pack row signals this intent. This step replaces the Confirm Base Unit prompt when servings and packs are not yet linked.

Workflow: Configuring Nutrient & Ingredient Information

Nutrient data requirements depend on whether an item is flagged as a food item. The Is Food flag controls which nutrient fields become mandatory.

Required Nutrient Fields (Food Items only)

When Is Food = Yes, the following five fields must be populated :

  • Calories
  • Total Fat
  • Saturated Fat
  • Carbohydrate
  • Protein

Serving Size Fields

For food items, the following serving measure fields are conditionally required : Serving Size Quantity, Serving Size Unit, Serving Size Weight, and Serving Size Weight Unit .

Edible Yield Factor

The Edible Yield Factor defaults to 100 unless a different value is explicitly provided.

Allergen Section

The allergen section is optional for all records regardless of the Is Food setting; values are imported against the standard list.

Workflow: Configuring Allergens

Allergen configuration in Item Management operates on two levels: standard allergen statuses applied per item on the Allergen Info card, and custom allergens defined centrally and reused across items. The allergen section is optional for all item records.

Standard Allergen Statuses

On any item's Allergen Info card, you can set a status for each standard allergen. The available status values are: [TO VERIFY — presenter named these as Contains, May Contain, Free From, and Processed In Facility; confirm against current UI labels before publishing]. Apply the appropriate status for each allergen relevant to the item.

Creating a Custom Allergen

Custom allergens are managed centrally before they can be applied to individual items. Users with the appropriate permission can view and add custom allergens.

  1. Navigate to Item Management → Configuration → Custom Allergens.
  2. Select the option to add a new custom allergen.
  3. Enter the allergen name and save.

Adding a Custom Allergen to an Item

  1. Open the item record and locate the Allergen Info card.
  2. Select the custom allergen from the available list.
  3. Set the appropriate status and save the item.

Step 2 — Enable POS and Set the POS Entree Flag

  1. Toggle Show on POS to make the item available at terminals.
  2. Check POS Entree to designate the item as a reimbursable entree within POS sales tracking. This checkbox is available on any POS-enabled menu item.

  • Use Set All Prices to apply one value across every cell in the grid simultaneously, then adjust individual cells as needed.

Workflow: Configuring HACCP & Recipe Steps

Before You Begin

The Steps & Ingredients tab becomes active only after a Description, Serving Size, and Number of Servings have been saved on the summary card. The Steps card displays on the left side of that tab.

Step-by-Step: Configure HACCP and Add Steps

  1. Enter edit mode. Select the edit action on the Steps card to enable Edit Recipe Steps mode. The HACCP Process selection becomes available in this mode.
  2. Select a HACCP Process. Choose from the processes configured in your system — in practice this is almost always No Cook, Same Day Service, or Complex Food. This selection is the only required field in the HACCP section.
  3. Add recipe direction steps. Each step's direction text is the primary visual row in the Steps card. Step ingredients are available in an expandable sub-section beneath each step.
  4. Add Critical Control Points. Use the Add CCP action — located at the same level as the Add Step action — to attach CCPs from the configured library. The system validates saves against the added control points.
  5. Save or cancel independently. The HACCP section supports its own Save/Cancel, separate from the steps.

Changing the HACCP Process After CCPs Are Added

SituationSystem Behavior
Existing CCPs are not associated with the newly selected process A warning is shown before clearing those control points.
Existing CCPs are compatible with the newly selected process Those CCPs are retained automatically.

The HACCP section is designed to expand over time and may later include additional pre-preparation fields.

Key Fields & Statuses

Units Card

The Units card appears in the item creation header and consolidates pack size and serving size configuration into a single section. It becomes editable only after the general info card has been saved.

Before the grid loads, the card presents two sequential setup questions:

  • Food Item? — If No, serving sizes are disabled.
  • Fluid Item? — If Yes, serving size measurement switches from weight to volume.

Grid Field Reference

Field Applies To Key Behavior
Amount Serving sizes only Not applicable to pack size units.
Description Servings and pack sizes Smart text field matched against common measures for both unit types.
Sub-Qty Pack sizes; inventory-able servings Links pack size units together; also links a serving to its base unit in specific scenarios.
Weight / Volume Both Column header reads "Weight" for non-fluid items and "Weight / Volume" for fluid items.
Cost / Value Pack sizes and eligible servings Replaces the legacy Fair Market Value field; editable on all eligible units.
Identifiers All units Displays unit-level indicators and attributes relevant to understanding the full Units card.

Inventory Readiness

Inventory Readiness is a status concept built into the item creation flow to reflect whether an item is fully configured for inventory use. The inventory flag is triggered when pack sizes are added. Pack units that remain unlinked from serving sizes place the item in an inventory-ineligible state. At least one serving size or pack size unit must be configured before the user can proceed to the next step.

Reports & Outputs

An item record drives several downstream views and reports. The two primary outputs available in the Item Management module are the Steps & Ingredients summary and the POS Entrée flag for sales tracking.

Steps & Ingredients Summary

When viewing a recipe in view mode, a right-side Ingredients card aggregates every ingredient added to the recipe across all steps. The quantity shown for each ingredient is the sum of that ingredient across the entire recipe, not just a single step. A label or icon on the card makes clear that these figures represent recipe-level totals. Quantities are also scaled up to higher units to keep totals readable at recipe scale.

Key point: The per-step expandable view shows ingredients for one step at a time; the Ingredients card is the correct place to see the aggregated whole-recipe picture.

POS Entrée Flag

The POS Entrée checkbox enables POS sales tracking for designated menu items. The flag was previously called "Is a Meal" and was limited to POS-Only items; it is now available on any POS-enabled menu item under its new label. The consistent label applies across all item types.

Behavior Detail
Availability Any POS-enabled menu item
POS transmission Flag is sent to POS when set
Audit trail Changes are captured in item history

Exceptions & Troubleshooting

Inventory Readiness Blocked — Unlinked Servings and Packs

When pack size units and serving sizes exist but are not linked, the item becomes Inventory Ineligible. Inventory Readiness requires that a proper pack-serving link is established before the item can proceed.

How to resolve: The system prompts you to either enter a sub-quantity or provide a pack size weight. Sub-Qty answers how many of the serving unit fit inside the pack unit. Weight on the pack row is the alternative path when linking to a weight-based serving.

Pack Size Hierarchy Limits

ScenarioWhat to do
Adding the first pack sizeSub-Qty is not enabled — description and weight only.
Adding a second or lower pack sizeSub-Qty becomes available on the appropriate row to establish the parent-child link.
Lowest unit in a three-tier structurePrioritize entering weight to ensure linking is possible.

Nutrient Save Blocked — Missing Required Fields

For food items, five nutrient fields are mandatory and will cause the record to be skipped if absent. The required fields are Calories, Total Fat, Saturated Fat, Carbohydrate, and Protein. For non-food items, the entire nutrient section is optional; values provided are still imported unless a data type mismatch is detected.

Missing Value vs. Zero-Value Nutrients

These are not equivalent. On a food item, blank nutrient fields are flagged as missing. A zero is a valid numeric entry and imports as-is. If a nutrient is genuinely zero, enter 0 explicitly — do not leave the cell blank.
