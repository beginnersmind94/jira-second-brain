Purpose

After reading this guide you can: create a non-food supply item; create a food item with a linked serving and pack; configure Menu Info and POS Pricing so the item appears on the register; and set up allergens including custom allergens.

Who this is for

District Administrators and Nutrition Managers setting up items in SchoolCafé 2.0 for the first time.

Before you start

  • You need login credentials from the implementation team before you can access Item Management.
  • Gather item details: description, brand, manufacturer, product code, and pack-size structure.
  • Know whether each item is food or non-food. Food items require serving sizes, nutrients, and allergen data.
  • For food items, have the nutrition label available.
  • For POS items, confirm your district's student and adult pricing before the POS Pricing step.

Top workflows

Workflow 1 — Create a non-food item

For supply items (cups, trays, napkins) tracked in Inventory but never served as food.

  1. Go to Item Management > Items and click +NEW.
  2. Enter Item Description (required); optionally add Tags, Brand Name [TO VERIFY], Manufacturer [TO VERIFY], Product Code [TO VERIFY].
  3. Select all three product dropdowns — Item Category [TO VERIFY], Storage Category [TO VERIFY], Valuation Group [TO VERIFY] — or skip (required later for Inventory).
  4. Click SAVE. Item is created and assigned an Item number.
  5. On the Units card, select NON-FOOD.
  6. Click +PACK; enter Description; for lower tiers add Sub-Qty. Weight optional for non-food [TO VERIFY].
  7. Optional: enter cost for one unit; system scales all tiers.
  8. Review Inventory Readiness banner; confirm Base Unit; set Whole Numbers vs. Decimals.

[!warning] Watch out: According to the training session, Sub-Qty must reflect the standard spec-sheet quantity, not a short-ship quantity. Handle short-ships at receiving in Inventory, not here.

Workflow 2 — Create a food item with serving and pack linked for Inventory

For items you will serve and track in Inventory. Linking serving to pack unlocks Inventory eligibility.

  1. Click +NEW, complete the Item card (same as Workflow 1, Steps 1–4).
  2. On the Units card, select FOOD; answer Fluid Item question (Fluid = milliliters; Non-Fluid = grams).
  3. Click +SERVING; enter Amount (1 = inventory-eligible), Description, Weight.
  4. Click +PACK; add Description and Sub-Qty for lower tiers.
  5. On the serving row enter Sub-Qty — "How many [serving] are in [pack]?" This triggers Inventory eligibility.
  6. If pack Description or weight matches the serving (dual-unit scenario), click the Merge button to confirm; serving Sub-Qty promotes to the new base unit. [TO VERIFY: exact dialog text not in NXT-40778 AC — use "Merge" per AC, not presenter's "BASE UNIT chip"/"promote"]
  7. Review Inventory Readiness banner, confirm Base Unit, set Whole Numbers vs. Decimals.

[!warning] Watch out: Both rows plus a Sub-Qty on the serving row are required for Inventory eligibility. Rows without the Sub-Qty link leave the item Inventory Ineligible.

Workflow 3 — Set up Menu Info and POS Pricing

Make a food item available on the POS and in menu planning.

  1. On the item card, navigate to the Menu Info tab.
  2. Click CREATE MENU ITEM [TO VERIFY: exact label is Description-only].
  3. Enter Menu Item Name [TO VERIFY], Button Name English [TO VERIFY], Menu Item Category [TO VERIFY]. All field names are Description-only.
  4. Toggle Show on POS. Enables the POS Info tab and loads POS attributes and Pricing.
  5. On Students tab: enter pricing by student type and site level.
  6. On Adults tab: enter adult pricing; check Taxable if applicable (Adults tab only).
  7. Click SAVE.

[!warning] Watch out: The Taxable checkbox appears only on the Adults pricing tab — it is not shown on the Students tab. POS does not apply tax to student meals.

Workflow 4 — Configure allergens (standard and custom)

Record allergens for a food item, including allergens not on the standard list.

Standard allergens:

  1. Navigate to Allergen Info. Allergens can be entered without Nutrient or Contribution data.
  2. Click Edit; select applicable allergen forms. Priority order: Contains, Processed (Facility), May Contain, Free From.
  3. Click SAVE. Only applicable allergens appear.

Custom allergens (when the allergen is not on the standard list):

  1. Go to Item Management > Configuration > Item Configuration. Select Custom Allergen, choose Local data source, click Apply.
  2. Click +NEW. Enter Custom Allergen Description (mandatory) and Data Source: Local (mandatory).
  3. Click SAVE. Duplicate names are rejected with: "Custom Allergen already in use, please use a different Custom Allergen name."

[!warning] Watch out: The allergen card only shows allergens applicable to the item — it no longer lists all allergens with empty states. An absent allergen means not applicable, not missing or unknown.

Common mistakes

  • Skipping the serving-to-pack Sub-Qty link. Both rows must exist AND the serving row must have a Sub-Qty. Without it the item is Inventory Ineligible.
  • Entering a short-ship quantity as Sub-Qty. According to the training session, Sub-Qty must match the standard spec-sheet quantity, not an exception delivery.
  • Assuming a three-tier pack cap is a hard rule [TO VERIFY — stated by presenter at transcript [07:05]–[07:30]; not documented in NXT-37110 AC].
  • Setting Yield Factor on the Ingredient Info card. Per NXT-47249 (Done Done), Yield Factor has moved to the Units card. Do not follow the presenter's Ingredient Info workflow — confirm with the product team.
  • Looking for Taxable on the Students pricing tab. It appears only on the Adults tab — not the Students tab.
