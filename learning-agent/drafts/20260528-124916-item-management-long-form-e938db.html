Item Management: Creating Food and Non-Food Items

Module: Item Management  |  Template: Long-form guide  |  Platform: SchoolCafé 2.0  |  Draft generated: 2026-05-28

Overview

Item Management is the central catalog for everything your district purchases, serves, or tracks — from individual food items like yogurt cups to non-food supplies like disposable gloves. Every item you create here flows downstream into Inventory (for ordering and receiving), Menu Planning (for nutrient analysis and menu building), and Point of Sale (for student and adult transactions). Getting item data right at the start prevents cascading problems across all connected modules.

The module organizes items into two top-level types: food and non-food. Non-food items — supplies such as cups, trays, napkins, and gloves — are simpler: they require an item description, category fields, and pack size units with costing. Food items carry all of that plus nutritional values, allergen declarations, meal-pattern contribution data, and the option to become a Direct Menu Item — a food item served as-is on a menu, without being wrapped in a recipe.

The Item card at the top of every item record is identical for both types. Differences begin on the Units card, which configures pack sizes and (for food items) serving sizes, and continue through the food-only tabs: Nutrient Info, Menu & POS Info, and the Steps card.

A fully configured food item will have: a costed inventory unit, verified nutrient data, declared allergens, a meal-pattern credit, and — if it is served directly — a Direct Menu Item with cashier-screen button names and POS pricing. When all sections are complete, the item is ready to be published to Inventory and placed on menus and production records.

  2. Storage Categories — Required for the Storage Category field. Path: same Configuration page, select Storage Categories. Examples: Dry, Cooler, Freezer.

  4.   5. HACCP Processes and Control Points (optional — food items only) — The system ships with three default HACCP processes (No Cook, Same Day Service, Complex Food). 4.1 Creating a Non-Food Item

Non-food items are supplies: cups, trays, napkins, gloves, and similar goods. They do not carry nutrient information or menu data. The creation process involves the Item card and the Units card only.

Step 1 — Open a new item

  1. Navigate to Item Management → Items. The list shows every item in your district's catalog.
  2. Click + NEW. The Item Details page opens with the Item card displayed.

Step 2 — Complete the Item card

The Item card is identical for food and non-food items.

  1. Enter an Item Description. This field is required. Include the item name and size for searchability — for example, Dixie Cup, 12 oz. [TO VERIFY: exact character limit — presenter stated "I think it's a 250 character limit" at [02:01]; this limit is not confirmed in available Jira AC]
  2. Select an Item Category. Required if using Inventory.
  3. Select a Storage Category. Required if using Inventory.
  4. Select a Valuation Group. Required if using Inventory.
  5. Optionally enter Tags, Brand Name, Manufacturer, and Product Code. Brand Name and Manufacturer are especially useful as search filters.
  6. Click SAVE. The page refreshes and the Units card appears.
Tip — fill the category fields even if Inventory is not yet active. If you enable Inventory later, you will need to return to every item and fill in the category fields manually. Filling them now prevents that rework.

Step 3 — Select Non-Food on the Units card

  1. On the Units card, click NON-FOOD. Non-food items skip the fluid/non-fluid question; that question appears only for food items.
  2. Proceed directly to adding pack sizes.

Step 4 — Add pack sizes (up to three tiers)

Pack sizes define the units used for inventory counts, ordering, and receiving. The system supports a maximum of three levels (for example, Box → Sleeve → Cup). The lowest level is automatically assigned as the Base Unit, which is the unit used when counting inventory.

  1. Click + PACK. A new editable row appears in the units grid.
  2. Enter a Description for the first (largest) pack size — for example, Box. Description is required.
  3. For non-food items, Weight is optional. Enter it if you need it for shipping or receiving reference; it does not affect inventory quantity counts.
  4. Click the checkmark to save the row. A yellow Missing Costing banner appears at the top. Do not act on it yet — add all pack sizes first.
  5. To add a second (lower) tier: click + PACK again. The system asks Higher or Lower?
    • Select HIGHER if the new unit is larger than the existing one (it will contain the existing unit).
    • Select LOWER if the new unit is smaller (the existing unit contains it).
  6. Enter the Description and Sub-Qty for the second tier. Sub-Qty is the number of the lower units of this tier that fit in the tier above it — for example, 20 sleeves per box.
  7. Click the checkmark. Repeat for a third tier if needed. After three tiers are added, the + PACK button becomes inactive — the maximum is enforced by the system.
What to do when you have four supply-chain levels: Configure the three tiers that matter most for day-to-day operations — typically the purchase unit, an intermediate unit, and the individual piece. Handle pallet-level receiving by counting the number of purchase units on the pallet during the Inventory receiving workflow.

Step 5 — Add GTIN/UPC barcodes (optional)

  1. Click the GTIN/UPC button on the Units card.
  2. A barcode field appears for each pack size row. Enter the GTIN or UPC code for each unit that has one.
  3. Click SAVE. A barcode icon appears in the Identifiers column for each row that now has a code assigned.

Step 6 — Enter costing

  1. Click the Missing Costing banner to expand it, then click the Edit (pencil) icon.
  2. The cost fields for all tiers are shown, but only one can be entered at a time. Enter the cost for the unit whose price you know — typically the unit you purchase. While you type in that field, all other tier fields are disabled.
  3. Click SAVE. The system divides the entered cost by each tier's Sub-Qty to calculate the cost for every other tier automatically.
  4. The Missing Costing banner disappears. The item gains an Inventory Ready flag and the Publish toggle is automatically enabled.
Updating cost later: After costing is saved, [TO VERIFY: button label — presenter called it "Manage Costing" at [08:52]] replaces the Missing Costing banner. Click it at any time to update the price; the system recalculates all tiers.

Step 7 — Set Whole Numbers or Decimals (optional)

  1. Click the Inventory Ready flag to expand it. The banner shows which unit is the Base Unit and its current counting mode (default: Whole Numbers).
  2. To allow partial-unit counts, click the WHOLE NUMBERS chip and switch to DECIMALS. For most non-food items, whole numbers is appropriate.

4.2 Creating a Food Item

Food items carry all the fields of a non-food item plus serving sizes, nutritional values, allergen data, meal-pattern contributions, and the ability to become a Direct Menu Item with POS pricing. The Item card is identical; differences begin at the Units card.

Steps 1–2 — Item card (same as non-food)

Follow Steps 1–2 from Section 4.1. Example values for a food item: Item Description Yogurt, Strawberry, 4 oz · Item Category Entrees · Storage Category Cooler · Valuation Group Non-Commodity · Brand Yoplait · Manufacturer GENERAL MILLS SALES INC.

Step 3 — Select Food and the fluid type on the Units card

  1. On the Units card, click FOOD.
  2. The system immediately asks: Non-Fluid Item (Grams) or Fluid Item (Milliliters)?
    • Select NON-FLUID when the product's nutrition label expresses weight in grams — this covers most solid foods and semi-solids such as yogurt.
    • Select FLUID when the label uses milliliters — juice boxes, milk cartons, liquid condiments.

Step 4 — Add a serving size

Food items require at least one serving size. Serving sizes are the basis for recipe ingredient usage and menu planning. Pack sizes are the basis for inventory ordering and receiving. Both are needed and must ultimately be linked.

  1. Click + SERVING.
  2. Enter:
    • Amount — the numeric amount of one serving (for example, 1)
    • Description — a text label (for example, Each)
    • Weight and its unit of measure (for example, 4 and Ounce)
  3. Click the checkmark. The Nutrient Info, Menu Info, and Inventory Info tabs appear at the bottom of the page but remain grayed out. They activate only after the Units card is saved.
Serving-first is the recommended order. The system accepts either serving sizes first or pack sizes first. Adding the serving size first makes the linking step (Step 6) more straightforward.

Step 5 — Add pack sizes

Follow the same + PACK process described in Section 4.1, Steps 4–5. For food items, the system may require a weight value on the lowest pack size if Sub-Qty is not used to link it to the serving size.

Step 6 — Link serving sizes to pack sizes

For a food item to be inventory-eligible, the system must know how many servings are in the lowest pack. If packs and servings are not linked, a banner reads: "Servings and pack units must be linked for Inventory eligibility."

  1. Click the Inventory Ready banner, then click the Edit icon.
  2. In the serving size row, enter the Sub-Qty: how many individual servings fit in the lowest pack tier. For example, if there are 16 yogurts in a carton, enter 16.
  3. Click SAVE. The linking alert clears. Proceed to costing.

Step 7 — Enter costing (same process as non-food)

Click the Missing Costing banner → Edit → enter the cost for the one unit you know → Save. The system calculates all other tiers. Rounding is applied to the calculated values.

Step 8 — Confirm or promote the Base Unit (food items only)

After costing, the system defaults to the lowest pack size as the Base Unit. For food items only, you can promote the serving size to act as the Base Unit instead — useful when your physical inventory count tracks individual portions rather than packs.

  1. Click the BASE UNIT chip in the Inventory Ready banner.
  2. The system shows the current Base Unit and the option to promote the serving size.
  3. Click the serving size row to select it as the candidate Base Unit.
  4. Click CONFIRM. The serving size row now carries both the Serving Size (S) identifier and the Base Unit (BU) identifier — this combination is called a dual unit.

Step 9 — Set Whole Numbers or Decimals, then save

Set the Base Unit counting mode (see Section 4.1, Step 7 — same process). Then click CONTINUE → SAVE & CLOSE to finalize the Units card. The Nutrient Info, Menu Info, and Inventory Info tabs are now active.

4.3 Completing Nutrient Info (Food Items Only)

The Nutrient Info tab contains four data areas: Nutrient Details, Ingredient Info, Contribution Info, and Allergen Info. Completing this tab is required for items used in menu nutrient analysis. Items can be used in Inventory without completing nutrients; only menu-planning analysis is affected.

Nutrient Details card

  1. Click the three-dot menu on the Nutrient Details card → Edit.
  2. The serving size is pre-populated at the top — for example, Serving Size: 1 Each (113.400 g). The gram value is auto-converted from the ounce weight entered on the Units card.
  3. Enter values for all required nutrient fields (marked with an asterisk). Standard required nutrients include Calories, Total Fat, Saturated Fat, Trans Fat, Cholesterol, Sodium, Total Carbohydrate, Dietary Fiber, Total Sugars, Added Sugars, Protein, Vitamin D, Calcium, Iron, and Potassium.
  4. For any nutrient that does not appear on the product label, check the Missing Value checkbox rather than leaving the field blank.
  5. Click SAVE. The system auto-calculates the Per 100 Gram column and daily-value percentages. Nutrients flagged Missing Value display as (M) in the card.
Missing Value vs. zero: Entering zero tells the system the item contains none of that nutrient. Checking Missing Value marks it (M) — flagging the data as unknown rather than reporting a false zero in nutrient analysis.

Ingredient Info card

  1. Click the three-dot menu on the Ingredient Info card → Edit.
  2. Edible Yield Factor — the percentage of the item that is edible. For fully packaged foods it is 100%. For produce with inedible parts (for example, a banana), enter the actual edible percentage.
  3. Sub Ingredients — the ingredient list text. Optional.
  4. Standard Recipe Directions — instructions that will appear on any multi-item recipe that uses this item as an ingredient (for example, Thaw for 30 minutes before serving). Optional.
  5. Locally Grown checkbox — check if applicable for compliance tracking.
  6. Buy American checkbox — check if the item meets the Buy American requirement. Important: checking this disables the Buy American Exemption Letter upload. An exemption letter can only be uploaded when this box is unchecked.
  7. Click SAVE.

Contribution Info

  1. Click the three-dot menu on the Contribution Info card → Edit. A slide-out panel opens.
  2. Set the meal-pattern credits for this item — for example, 1 oz equivalent of Meat/Meat Alternates. Fill in any applicable food components (Grains, Vegetables, Fruit, Milk, Meat/Meat Alternates).
  3. Click SAVE.

Allergen Info card

  1. Click the three-dot menu on the Allergen Info card → Edit. A slide-out panel opens.
  2. Read the Allergen Feature Disclaimer at the top. SchoolCafé is a tool; the district remains legally responsible for verifying allergen accuracy.
  3. For each standard allergen — Crustacean Shellfish, Egg, Fish, Gluten, Milk, Peanuts, Sesame, Soy, Tree Nuts, Wheat — select the applicable status:
    • Contains
    • Processed in a Facility That Also Processes
    • May Contain
    • Free From
  4. To add a custom allergen (one already created in Configuration): click + ADD CUSTOM ALLERGEN, select it from the dropdown, set its status, then click ADD.
  5. Click SAVE.

4.6 Configuring HACCP and Recipe Steps

For complex items, you can build out full step-by-step preparation directions.

  1. On the Item Details page, click ENABLE RECIPE on the Steps card. (The label reads "Recipe" even on Direct Menu Items — this is expected.)

  3. Select a HACCP Process:

  4. Optionally enter Pre-Preparation Instructions (for example, Remove from cooler 15 minutes before service), Prep Time, and Cook Time.

  7. To add a Critical Control Point: click the dropdown arrow on + ADD → select Control Point → choose the CCP type (for example, Cold Hold for No Cook items). The system auto-fills the Critical Limit and Corrective Action; edit them if your district's standards differ. 4.8 Viewing the Inventory Info Tab

The Inventory Info tab is a read-only view within Item Management that shows vendor contract details for the item once it has been published to Inventory and added to a contract in the Inventory module. Item Management staff do not edit contracts here — that work is done in Inventory.

Reports and Outputs

Item Management does not produce standalone reports. The data entered here feeds reports in connected modules:

  • Nutrient analysis reports (Menu Planning) — Consume nutrient values and meal-pattern contributions from each food item. Items with Missing Value nutrients display data-gap flags rather than zero values.
  • Menu Calendar Report (Menu Planning) — The Show in Summary field on the Menu Item card controls whether the item appears in the report's summary view.
  • Inventory Ordering cart (Inventory) — Displays the Preferred Inventory Image for each item.
  • Production records (Production) — Use the Menu Item Name, Leftover Category, Max Days, and assigned HACCP process from Item Management.
  • POS meal-count and sales reports (Point of Sale) — Items with the POS Entrée checkbox enabled are tracked as reimbursable meals.

Exceptions

Three-tier pack size maximum enforced by the system

The system enforces a hard limit of three pack size tiers per item. After three tiers are added, the + PACK button becomes inactive. If your supply chain has four levels (for example, Pallet → Case → Carton → Each), configure the three levels most useful for daily operations — typically Case, Carton, Each. Record pallet-level receiving in Inventory by counting the number of cases on each pallet.

Base Unit and counting mode lock after first Inventory transaction

The Base Unit assignment and Whole Numbers / Decimals setting for an item become locked once the item has been used in an Inventory transaction (count, receipt, or adjustment). Configure these correctly before the item goes live in Inventory. If these settings are wrong after the item has been transacted, contact your Inventory module administrator.

Serving-pack link required for food item Inventory eligibility

A food item that has both serving sizes and pack sizes configured but not linked will remain Inventory Ineligible. The item can still be saved and costing can still be entered in this state, but the item cannot be published to Inventory until the link is established.

Buy American checkbox disables exemption letter upload

Checking Buy American on the Ingredient Info card disables the document upload button for Buy American Exemption Letters. If you need to attach an exemption letter, the item must not have Buy American checked.

Nutrient tabs inactive until Units card is saved

The Nutrient Info, Menu Info, and Inventory Info tabs appear after a serving size is added but remain grayed out until the Units card has been saved and closed. If these tabs appear inaccessible, complete and save the Units card first.

Required nutrient fields block Nutrient Details save

The Nutrient Details card will not save if any required (asterisk) nutrient field is blank and the Missing Value checkbox is not checked. Either enter a value or check Missing Value. The item can still be used in Inventory without completing nutrient data; only menu nutrient analysis is blocked.

Sub-quantity represents the standard spec, not variable shipments

The Sub-Qty value on a pack size should reflect what is on the product specification sheet — the standard count. If a vendor occasionally ships short (for example, 18 sleeves instead of the specified 20), that is a receiving discrepancy to handle in the Inventory receiving workflow. Do not adjust the item's Sub-Qty to match a non-standard shipment.

Troubleshooting

SymptomLikely CauseFix
Missing Costing banner appears after adding all pack sizes Expected behavior — costing is deliberately held until all pack sizes are configured. Click Missing Costing banner → Edit → enter cost for the one unit you know → Save. Do not attempt to fix this banner before adding all pack sizes.
Inventory Info tab shows Incomplete One or more Inventory Readiness requirements are unresolved. Address each banner alert in turn: (1) item categories, (2) pack-serving link for food items, (3) costing. The Incomplete status resolves when all three are satisfied.
Nutrient Info / Menu Info / Inventory Info tabs are grayed out and cannot be clicked Units card has not been saved. Click CONTINUE → SAVE & CLOSE on the Units card.
+ PACK button is inactive after adding two pack sizes Three-tier maximum has been reached. This is by design. Use the three tiers added. If a fourth level is operationally needed, manage it at the Inventory receiving level.
Base Unit or Whole Numbers/Decimals is locked and cannot be changed Item has been used in an Inventory transaction. Contact your Inventory module administrator. Prevent recurrence by confirming these settings before the item is activated in Inventory.
Nutrient Details card will not save One or more required (asterisk) nutrient fields are blank without a Missing Value checkbox checked. Enter a value for each required nutrient, or check the Missing Value checkbox if the value is not on the product label.
Food item shows "Servings and pack units must be linked for Inventory eligibility" Serving sizes and pack sizes have been added but not linked via Sub-Qty or weight. Click the Inventory Ready banner → Edit → enter Sub-Qty on the serving size row → Save.
Custom allergen does not appear in the allergen panel dropdown The custom allergen has not yet been created in Configuration. Go to Item Management → Configuration → Item Configuration → Custom Allergens → + NEW, create the allergen, then return to the item's allergen panel.
POS Pricing section does not appear on the Menu & POS Info tab The Show on POS toggle is disabled. On the Menu Item card, enable the Show on POS toggle and save. The POS Pricing section will appear.
HACCP process change clears existing Control Points The new HACCP process does not include some CCPs that were added under the previous process. This is by design. The system warns before clearing. If the CCPs are still needed, re-add them under the new process.
