Item Management: Creating Food and Non-Food Items — Long-Form Learning Guide

Module: Item Management · Platform: SchoolCafé 2.0 · Audience: Implementation Specialists, District Nutrition Directors, Menu Planners · Draft generated: 2026-06-01

1. Overview

The Item Management module is the foundational catalog for SchoolCafé 2.0. Every food product, supply, and ingredient your district uses must be configured here before it can appear in Inventory, on a menu, in a recipe, or at the point-of-sale register. Getting items right at setup time prevents problems across every downstream module — a mis-configured item will surface as bad inventory counts, incorrect nutrient analysis, or a wrong POS price.

The module supports two item types: food items and non-food items. Non-food items (supplies such as cups, trays, napkins, and gloves) follow a shorter setup path — Item card, pack sizes, and costing. Food items follow the full path and add serving sizes, nutrient information, allergen data, meal pattern contribution, menu item configuration, HACCP and recipe steps, and POS pricing.

The Item card — which captures the item's name, category, and identification fields — is identical for both item types and must be saved first before any other configuration becomes available. In scope: creating and fully configuring food and non-food items from scratch. Out of scope: recipe creation, menu planning, and Inventory order management, which are separate modules that consume item records created here.

2. Roles and Permissions

The following roles interact with Item Management during item creation and configuration:

Role Actions
Implementation Specialist / Cybersoft Staff Full item creation and configuration during district onboarding. Trains district staff on the workflow.
Nutrition Director / Item Management Super User Full item creation, editing, and publishing. Manages Item Configuration settings including Custom Allergen setup.
Nutritionist / Menu Planner Enters nutrient details, contribution info, allergen info, and configures Direct Menu Items and recipe steps.
Menu Planning Manager Manages Menu Item configuration including POS pricing and menu category assignments.

Item Category, Storage Category, and Valuation Group are required fields only when the district has the Inventory module enabled. If Inventory is not enabled, these fields are optional — but completing them upfront is strongly recommended because retrofitting hundreds of items later is labor-intensive.

3. Prerequisites

  • Login credentials and module access: Staff must have active SchoolCafé 2.0 credentials with Item Management module permissions before beginning any item creation.
  • Item Configuration settings: Before assigning custom allergens to items, the custom allergens must first be created in Item Management → Configuration → Item Configuration. See the Custom Allergen Configuration workflow below.
  • HACCP process configuration: HACCP Processes and Control Points are configured system-wide in Item Configuration before they appear as options on the Steps card.
  • Inventory module enablement: The Inventory module must be enabled at the district level for item categorizations and the Publish to Inventory toggle to have effect.
  • Common Measures configuration: Pack size descriptions (Case, Box, Sleeve, etc.) are drawn from a Common Measures list. Ensure this list is configured before item creation.

4. Full Workflows

4.1 Creating a Non-Food Item

Purpose: Configure a supply item (cups, trays, napkins, gloves, etc.) for use in Inventory. Non-food items do not have nutrition, allergen, or menu data.

Step A — Item Card

  1. Navigate to Item Management → Items.
  2. Click + NEW to open the Item Details page.
  3. On the Item Card, complete the following fields:
    • Item Description — Required. Enter the item name and size (e.g., Dixie Cup, 12 oz). Keep it descriptive but concise. [TO VERIFY: 250-character limit — presenter stated this at [02:01] but was uncertain; not confirmed in Jira AC]
    • Item Category — Required when Inventory is enabled.
    • Storage Category — Required when Inventory is enabled (e.g., Dry).
    • Valuation Group — Required when Inventory is enabled (e.g., Non-Food).
    • Tags, Brand Name, Manufacturer, Product Code — Optional. Completing Brand Name and Manufacturer makes items easier to filter when searching.
  4. Click SAVE. The page refreshes and the Units Card appears.

Step B — Units Card: Item Type and Pack Sizes

  1. On the Units Card header, click NON-FOOD.
  2. For non-food items, the Fluid/Non-Fluid question does not appear. Proceed directly to adding pack sizes.
  3. Click + PACK to add your first (highest) pack size:
    • Description — Required. Enter the unit name (e.g., Box).
    • Weight — Optional for non-food items. Enter if needed for shipping or receiving reference.
  4. Click the checkmark icon to save the row. A Missing Costing alert appears — do not address it yet.
  5. To add a second (smaller) level, click + PACK again. The system asks Higher or Lower:
    • Select LOWER to add a unit that is contained within the existing pack (e.g., a Sleeve inside a Box).
    • Select HIGHER to add a unit that contains the existing pack (e.g., a Pallet above a Box).
    Fill in Description and Sub-Qty (e.g., Sleeve, Sub-Qty: 20 means 20 sleeves per box). Click the checkmark.
  6. Repeat for a third level if needed (e.g., Cup, Sub-Qty: 25). After three pack sizes, the + PACK button becomes unavailable — three tiers is the system maximum.
  7. Review the Identifiers column:
    • P badge — marks each pack size level.
    • BU badge — marks the Base Unit (automatically assigned to the lowest level). This is the unit used for inventory counting.

Step C — Costing

  1. Click the Missing Costing alert to expand it, then click the Edit (pencil) icon.
  2. Enter the cost for the one pack size unit you know the price for (typically the purchase unit, e.g., Box = $80.00). The other unit fields are disabled while you type.
  3. Click SAVE. The system calculates the remaining unit costs by dividing proportionally through the Sub-Qty chain (e.g., Sleeve = $80 ÷ 20 = $4.00; Cup = $4 ÷ 25 = $0.16).
  4. The Missing Costing alert is replaced by a Manage Costing button. Use this button at any time to update pricing — all dependent unit costs recalculate automatically.
  5. The Inventory Ready flag turns green and the Publish toggle is automatically enabled. The item is ready to be published to Inventory.

Step D — GTIN/UPC Barcodes (Optional)

  1. Click the GTIN/UPC button on the Units Card.
  2. Each pack size row now displays a GTIN code field. Enter the barcode for each applicable unit.
  3. Click SAVE. Barcode icons appear in the Identifiers column for all units with a code entered.

GTIN/UPC codes are optional but strongly recommended if the district uses barcode scanners for receiving.

Step E — Inventory Count Method

  1. Expand the Inventory Ready banner. The system displays the current Base Unit and count method.
  2. By default, the system counts in Whole Numbers. If partial units are needed, click the WHOLE NUMBERS chip to switch to DECIMALS.
⚠ Important — Base Unit and Decimals lock after first Inventory use. Once an item has been used in an inventory count, receipt, or adjustment, the Base Unit assignment and the Whole Numbers/Decimals setting become locked. Configure these correctly before activating the item in Inventory.

4.2 Creating a Food Item — Item Card and Units Card

Purpose: Configure an item that will be served to students or used as a recipe ingredient. Food items require a full configuration path: Units Card, Nutrient Info, Menu Info, Steps, and POS Pricing.

Step A — Item Card (Food)

  1. Navigate to Item Management → Items and click + NEW.
  2. Complete the Item Card with the same fields as non-food:
    • Item Description — e.g., Yogurt, Strawberry, 4 oz
    • Item Category — e.g., Entrees
    • Storage Category — e.g., Cooler for refrigerated items
    • Valuation Group — e.g., Non-Commodity
    • Brand Name, Manufacturer, Product Code — optional but recommended
  3. Click SAVE.

Step B — Units Card: Food Type and Fluid/Non-Fluid

  1. On the Units Card header, click FOOD.
  2. A second question appears — Non-Fluid (Grams) or Fluid (Milliliters)?
    • Select NON-FLUID for items weighed in grams (most solid and semi-solid foods — yogurt, bread, fruit).
    • Select FLUID for items measured by volume (milk, juice, soups).
    The choice controls the Weight/Volume column label and the unit of measure used for serving sizes throughout the item.

Step C — Adding Serving Sizes

  1. Click + SERVING to add the first serving size row:
    • Amount — The number of units in the serving (e.g., 1).
    • Description — e.g., Each.
    • Weight — Weight in grams (or volume in mL for fluid items). e.g., 4 oz — the system converts oz to grams automatically.
  2. Click the checkmark. The NUTRIENT INFO, MENU INFO, and INVENTORY INFO tabs appear at the bottom of the page but remain grayed out until the Units Card is saved.

Step D — Adding Pack Sizes (Food)

  1. Click + PACK. For food items, the pack sizes operate the same as for non-food:
    • First pack size: enter Description (e.g., Case). Weight is optional.
    • Subsequent pack sizes: select Higher or Lower, enter Description and Sub-Qty (e.g., Carton, Sub-Qty: 3 = 3 cartons per case).
  2. Maximum of three pack size tiers applies to food items as well. The + PACK button becomes unavailable after three levels are added.

Step E — Linking Servings to Pack Sizes (Food Only)

Food items display a second alert alongside Missing Costing: Inventory Ready — "Servings and Pack Units must be linked for Inventory Eligibility." This alert does not appear for non-food items.

  1. Click the Inventory Ready alert, then click the Edit icon.
  2. Enter the Sub-Qty on the serving size row: how many of that serving size are in the lowest pack size (e.g., 16 yogurts per Carton → Sub-Qty: 16).
  3. Click SAVE. The Inventory Ready alert clears, confirming the serving and pack sizes are now linked.

Step F — Costing (Food)

Identical to the non-food costing process. Enter cost on the one unit whose price you know; the system calculates all others by dividing through the Sub-Qty chain. A Manage Costing button replaces the alert after the initial save.

Step G — Base Unit Selection (Food)

  1. After costing is complete, expand the Inventory Ready banner. The system displays the current Base Unit — by default, the lowest pack size (e.g., Carton).
  2. Click the BASE UNIT chip to see the promotion option. For food items, if a serving size with Amount = 1 exists and is linked via Sub-Qty, the system offers to promote that serving size to Base Unit.
  3. Select the preferred Base Unit row (e.g., Each) and click CONFIRM. The promoted serving size now displays both Serving and BU (Base Unit) identifiers — it is a dual unit.
  4. Select Whole Numbers or Decimals for inventory counting (same as non-food).
  5. Click CONTINUE, then SAVE & CLOSE.

The Base Unit promotion option is only available for food items (because non-food items have no serving sizes). This step is also not available once the item has been used in Inventory tables.

4.3 Nutrient Info Tab

Purpose: Record all nutritional data for a food item. This data drives nutrient analysis in menu planning and is required for any menu item nutrient report. This tab is only available on food items.

The Nutrient Info tab becomes active after the Units Card is saved. The serving size selected on the Units Card is displayed at the top of the tab (e.g., Serving Size: 1 Each (113.400 g) — the system converts oz to grams automatically).

Nutrient Details Card

  1. Click the View More Options (three-dot) icon on the Nutrient Details card, then select Edit.
  2. Complete all required fields (marked with an asterisk): Calories, Total Fat, Saturated Fat, Trans Fat, Cholesterol, Sodium, Carbohydrate, Dietary Fiber, Total Sugars, Added Sugars, Protein, and the required vitamins/minerals.
  3. For any nutrient that does not appear on your product label, check the Missing Value checkbox rather than leaving the field blank. This marks the nutrient as (M) — Missing — in all reports. A blank field would imply zero; a Missing Value flag accurately communicates that the data is unknown.
  4. Click SAVE. The system automatically calculates the Per 100 Gram column and daily-value percentages. You do not need to enter these manually.

Saving the Nutrient Details card is enforced — you cannot save if required (asterisk) fields are missing. However, an item with an incomplete Nutrient Info tab can still be used in Inventory. Nutrient completeness only matters when the item is used in menu planning, where nutrient analysis is performed.

Ingredient Info Card

  1. Click the three-dot icon on the Ingredient Info card, then Edit.
  2. Set the Edible Yield Factor — the percentage of the item that is actually consumed. For a yogurt: 100%. For a banana: approximately 85% (peel discarded). This field drives recipe yield calculations.
  3. Optionally enter Sub Ingredients — the ingredient list for this item when it is used as an ingredient in a recipe.
  4. Optionally enter Standard Recipe Directions — instructions that print on any recipe that uses this item as an ingredient (e.g., Thaw for 30 minutes before serving).
  5. Set compliance checkboxes:
    • Locally Grown — check if the item qualifies for locally-grown reporting.
    • Buy American — check if the item is domestically sourced. Checking this field disables the exemption letter upload icon, because marking an item as Buy American means no exemption is needed. If the item is not Buy American, leave this unchecked — the upload icon remains active for attaching an exemption letter.

Contribution Info Card

  1. Click the three-dot icon on the Contribution Info card, then Edit. A slide-out panel opens.
  2. Enter the meal pattern credit for this item. For example, a 4 oz yogurt credits as 1 oz equivalent of Meat/Meat Alternates.
  3. If the item counts toward multiple component categories (e.g., also Grains), complete those fields as well.
  4. Click SAVE.

Allergen Info Card

  1. Click the three-dot icon on the Allergen Info card, then Edit. A slide-out panel opens.
  2. Read the Allergen Feature Disclaimer at the top of the panel. The disclaimer states that SchoolCafé is a tool for recording allergen data and is not a guarantee — the district remains responsible for verifying all allergen information.
  3. For each of the standard allergens listed, mark the appropriate status:
    • Contains
    • Processed in a Facility That Also Processes
    • May Contain
    • Free From
    Standard allergens include: Crustacean Shellfish, Egg, Fish, Gluten, Milk, Peanuts, Sesame, Soy, Tree Nuts, Wheat.
  4. For allergens not in the standard list, click + ADD CUSTOM ALLERGEN. The custom allergen must be pre-configured in Item Configuration before it appears in this dropdown. See section 4.8 for setup steps.
  5. Select the custom allergen from the dropdown, check the appropriate status, click ADD, then SAVE.

4.4 Menu Info Tab — Creating a Direct Menu Item

Purpose: Convert a food item into a Direct Menu Item (DMI) — a single item that can be placed directly on a menu or production record without being wrapped in a recipe. Typical DMIs include individually served items like yogurt cups, pre-packaged snacks, or bottled beverages.

  1. Click the MENU INFO tab (active only after the Units Card is saved). It displays a Disabled status until configured.
  2. If only one serving size exists on the Units Card, the system auto-selects it as the Menu Item Serving Size. If multiple serving sizes exist, you will be prompted to choose one.
  3. Click CREATE MENU ITEM on the Units Card banner. The Menu Item card opens.
  4. Complete the required fields:
    • Menu Item Name — Required. Appears on menus, production records, and reports (e.g., Strawberry Yogurt).
    • Button Name (English) — Required. What the cashier sees on the POS touchscreen.
    • Menu Item Category — Required. (e.g., Entree).
    • Max Days — How many days this item can be carried over on a production record (applies only when Unspecified or Carryover leftover categories are used).
  5. Complete the optional fields as applicable:
    • Marketing Name & Description — Appears in Family Hub (the parent-facing portal). Use descriptive, appealing language.
    • Button Name (Spanish) — For bilingual POS screens.
    • Meal Type — Lunch, Breakfast, etc. Used for menu grid filtering.
    • Leftover Category — What happens to unsold inventory: Return to Stock or Discard.
    • Tags — Custom classification tags for filtering.
    • Preparation Site Item — Check if the item is prepared at a central kitchen and shipped to serving sites.
    • Show in Summary — Controls visibility in the Menu Calendar Report.
    • POS Entree — Check this if the item counts as part of a reimbursable meal. Leave unchecked for à la carte items.
  6. Toggle Show on POS ON if students can purchase this item at the register. Additional POS configuration fields appear and the POS PRICING tab activates.
  7. Click SAVE. The tab is renamed MENU & POS INFO. An MI badge appears on the serving row in the Units Card indicating this serving size is the Menu Item Serving.

Adding Menu Serving Sizes and Exceptions

  1. From the Units Card in Edit mode, click + MENU SERVING to add additional serving sizes for the same menu item (e.g., a larger or smaller portion).
  2. Click EXCEPTIONS to configure serving exceptions — for example, restricting a serving size to specific meal pattern groups or grade levels. Assign a Meal Pattern & Serving Group, then click NOT SERVED and select which group cannot receive that serving size.

4.5 Steps Card — HACCP and Recipe Directions

Purpose: Record HACCP food safety information and preparation directions for a Direct Menu Item. For simple, uncooked items, this means selecting the HACCP process and optionally adding a control point. For complex prepared items, this includes full step-by-step recipe directions.

  1. On the Item Details page, locate the Steps card. Click ENABLE RECIPE.
  2. Click the three-dot icon, then Edit. Step 0 is the HACCP configuration step.
  3. Select the HACCP Process from the dropdown:
    • No Cook — for items that are never heated (e.g., fresh fruit, pre-packaged yogurt).
    • Same Day Service — for items cooked and served the same day.
    • Complex Food — for items that go through multiple temperature stages, cooling and reheating cycles.
  4. Optionally enter Pre-Preparation Instructions, Prep Time, and Cook Time.
  5. Click SAVE.

Adding Recipe Steps and Control Points

  1. Click + ADD to insert a recipe direction step. Type the preparation instruction text.
  2. To add a Control Point (CCP), click the dropdown arrow on the + ADD button and select Control Point. Pick the control point type (e.g., Cold Hold) from the dropdown. The system auto-fills the Critical Limit and Corrective Action based on the configured control point definition. Edit these values if the district's procedures differ.
  3. To reorder steps, drag the row handle (the dots on the left side of each row) to the desired position. Save after any reordering.
ℹ Note: If you change the HACCP Process after Control Points have already been added, the system warns you that CCPs not associated with the new process will be cleared. Confirm the change only if you intend to remove those control points.

4.6 POS Pricing Tab

Purpose: Define what students and adults pay for this menu item at the point of sale. This tab is only available after Show on POS is enabled on the Menu Info tab.

  1. Click the three-dot icon on the POS Pricing card, then Edit.
  2. The Students grid displays eligibility types (Free, Reduced, Paid, Second Meal) on the left and site types (Elementary, Middle, High School, Alternative) across the top. Enter the price for each cell relevant to your district.
  3. If prices are uniform across all site types, use the SET ALL PRICES shortcut: enter the price in the field at the top and click the button — all cells populate at once.
  4. The Allow Sale toggle auto-enables for each eligibility type when a price is entered. Toggle it off to prevent sales to a specific group even when a price is set.
  5. Click the ADULTS tab to set pricing for staff, visitor, and program adult categories. The same grid and SET ALL PRICES shortcut apply.
  6. If the item is taxable (e.g., certain à la carte items), check the Taxable checkbox at the top of the card. Most school meals are not taxable. Consult your district's business office for tax rate configuration details.
  7. Click SAVE.

4.7 Images and Documents

Purpose: Attach product images and supporting documentation to an item record. This section applies to both food and non-food items.

  1. On the Item Details page, click the expand icon on the Images & Docs card. The Manage Images & Documents panel opens.
  2. Drag and drop files onto the panel, or click SELECT FILES to browse. Supported formats include JPEG, PNG, PDF, Word documents, and spreadsheets. [TO VERIFY: 5 MB per file limit — stated by presenter at [11:02] but not confirmed in Jira AC]
  3. Once an image uploads, a preview appears. Select a value from the Use as dropdown (e.g., Preferred Inventory Image to display the image when browsing items in Inventory).
  4. Add optional comments per file if needed.
  5. Multiple files can be uploaded and assigned different Use as values.
  6. Click SAVE. The image appears on the Item Details card.

4.8 Custom Allergen Configuration

Purpose: Create district-specific allergens that do not appear in the standard allergen list. Custom allergens must exist in Item Configuration before they can be assigned to individual items.

  1. Navigate to Item Management → Configuration → Item Configuration.
  2. Select Custom Allergens from the dropdown, then click APPLY.
  3. Click + NEW. Enter the allergen name (e.g., Strawberry).
  4. Toggle Show on Family Hub Menus on or off. When enabled, this custom allergen appears in the allergen legend visible to parents viewing menus in Family Hub.
  5. Click SAVE. The custom allergen is now available in the allergen slide-out panel on any food item's Allergen Info card.

4.9 Inventory Info Tab

Purpose: Review the active vendor contracts associated with this item, once the item has been published to Inventory and added to a contract in the Inventory module. This tab is read-only for contract data; it also provides shortcuts for editing Units Card settings.

When contracts exist, this tab displays: Vendor Name, Vendor Item Number, Unit, Lead Time, Min/Max Order Quantities, and Current Price. A shortcut icon navigates directly to the Contract Items page in the Inventory module.

Pack sizes and GTIN codes can also be managed from this tab — clicking the edit controls here is equivalent to editing the Units Card directly.

6. Reports and Outputs

Item Management does not directly produce standalone reports, but item data drives outputs in connected modules:

  • Nutrient Analysis Reports — Generated in the Menu Planning module. Uses nutrient data entered on the Nutrient Info tab. Nutrients flagged as Missing Value (M) appear as incomplete in these reports.
  • Production Records — Generated in Menu Planning. Use the Menu Item Name, Max Days, and Leftover Category fields configured on the Menu Info tab.
  • POS Transaction Records — Generated by the POS module. Use the Button Name, POS pricing grid, and Allow Sale toggles configured on the POS Pricing tab.
  • Inventory Counts and Receiving Reports — Generated in the Inventory module. Use the Base Unit, Sub-Qty chain, GTIN/UPC codes, and Whole Numbers/Decimals setting from the Units Card.
  • Family Hub Menu Display — The parent-facing portal displays the Marketing Name & Description and Custom Allergen flags (if the "Show on Family Hub" toggle is enabled for the allergen) configured on the item.
  • Recipe Reports — Generated in Item Management (Recipes section). Use ingredient data, HACCP process, and recipe steps from the Steps card.

7. Exceptions

Maximum Pack Size Tiers (Three-tier limit)

The system supports a maximum of three pack size tiers per item (e.g., Pallet → Case → Each). The + PACK button is unavailable once three tiers are added. If your district's supply chain has a fourth level (e.g., Pallet), configure the three levels most relevant to daily operations and handle the higher level at receiving time in the Inventory module — entering received quantities in terms of the highest configured unit.

Sub-Quantity Must Reflect Standard Pack Size

The Sub-Qty field should record the standard pack size as stated on the product specification sheet — not an occasional short-ship quantity. If a vendor ships a non-standard quantity (e.g., 18 sleeves instead of 20), that discrepancy is handled during the receiving process in the Inventory module, not by changing the item record.

Base Unit and Count Method Lock After Inventory Activity

Once an item participates in any Inventory transaction (count, receipt, adjustment), the Base Unit assignment and Whole Numbers/Decimals setting are locked. If a wrong setting is discovered after Inventory use, correcting it requires intervention beyond routine editing — coordinate with your Cybersoft Implementation Specialist.

HACCP Process Change Clears Incompatible Control Points

If you change the HACCP Process on the Steps card after Control Points have already been added, the system warns that CCPs not associated with the new process will be removed. Confirm the change only if you intend to clear those control points. Associated CCPs (compatible with both processes) are retained.

Allergen Disclaimer — District Responsibility

Allergen data in SchoolCafé is a recording tool, not a certification. The system displays an Allergen Feature Disclaimer on the allergen panel to this effect. The district remains legally responsible for verifying allergen accuracy with product manufacturers.

Buy American and Exemption Letters

Checking the Buy American checkbox on the Ingredient Info card disables the exemption letter upload icon. These two fields are mutually exclusive by design: an item marked as Buy American is domestic and requires no exemption. To upload an exemption letter, the Buy American checkbox must be unchecked.
