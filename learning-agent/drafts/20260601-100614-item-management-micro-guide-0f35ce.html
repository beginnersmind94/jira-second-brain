Item Management: Creating Food and Non-Food Items — Micro Guide

Module: Item Management  |  Template: Micro guide  |  Source recording: REC-2026-0515-IM-001 (May 15, 2026)  |  Presenter: Sarah Chen, Implementation Specialist

Purpose

After reading this, you can create and configure food and non-food items in SchoolCafé 2.0 so they are ready for Inventory, Menu Planning, and POS. You will complete the Item card, set up pack sizes and costing, enter nutrient and allergen data for food items, and create a Direct Menu Item with POS pricing.

Who This Is For

District administrators and menu planners creating or maintaining items in SchoolCafé 2.0. Nutritionists responsible for nutrient, allergen, and meal pattern contribution data will also use Workflow 3.

Before You Start

  • Your account has Item Management module permissions and the district environment is provisioned in SchoolCafé 2.0.
  • Item categories, storage categories, and valuation groups already exist in the system — required for Inventory-enabled districts.
  • Any non-standard allergen created first at Item Management → Configuration → Item Configuration → Custom Allergens.
  • HACCP processes configured in Item Management Configuration if you will use the Steps card.

Top Workflows

1. Create a Non-Food Item

Configure a supply item (cups, trays, napkins) for Inventory tracking.

  1. Go to Item Management → Items → click + NEW.
  2. On the Item card, enter Item Description (required), Item Category, Storage Category, and Valuation Group. Optionally add Brand Name, Manufacturer, and Product Code to improve search filtering. Click SAVE.
  3. On the Units card, click NON-FOOD. This disables serving size entry.
  4. Click + PACK to add the highest-level unit (e.g., Box). Click the checkmark. For sub-levels, click + PACK → LOWER and enter a description and Sub-Qty (units per parent). Maximum three tiers.
  5. Click the Missing Costing alert → pencil icon. Enter the price for one purchased unit only; the system calculates all other tiers. Click SAVE.
  6. Click the Inventory Ready flag to confirm the Base Unit and set counting mode (Whole Numbers or Decimals).

⚠ Watch out: Add all pack sizes before entering costing. Adding a pack size afterward requires re-entering all prices.

2. Set Up a Food Item's Units Card

Select food type, add serving sizes, link them to pack sizes, and confirm costing and Base Unit.

  1. Complete the Item card with food-appropriate categories (e.g., Item Category: Entrees, Storage Category: Cooler). Click SAVE.
  2. On the Units card, click FOOD. Select NON-FLUID (grams) or FLUID (milliliters).
  3. Click + SERVING. Enter amount, description, and weight (e.g., 1 Each, 4 oz). Click the checkmark.
  4. Add pack sizes using + PACK (same process as non-food).
  5. Click the Inventory Ready alert → Edit. On the serving size row, enter Sub-Qty — the number of servings per lowest pack unit. Click SAVE. This linking step is required only for food items.
  6. Resolve Missing Costing: enter price for one purchased unit; system divides down through all tiers.
  7. Click the BASE UNIT chip on the Inventory Ready flag. Optionally promote the serving size to Base Unit to count individual servings in Inventory. Choose Whole Numbers or Decimals → CONTINUE → SAVE & CLOSE.

⚠ Watch out: The Nutrient Info and Menu Info tabs remain grayed out until the Units card is saved. Save the card before navigating to either tab.

3. Enter Nutrition and Allergen Data

Complete required nutrient fields and allergen statuses so the item contributes correctly to menu analysis.

  1. Click the NUTRIENT INFO tab → three-dot menu on the Nutrient Details card → Edit.
  2. Complete all asterisk-marked fields. For nutrients absent from the product label, check Missing Value — do not leave the field blank and do not enter zero. Click SAVE. The system populates the Per 100g column and Daily Value percentages automatically.
  3. On the Contribution Info card → three-dot menu → Edit. Enter meal component credits (e.g., 1 oz eq Meat/Meat Alternates). Click SAVE.
  4. On the Allergen Info card → three-dot menu → Edit. Read the disclaimer — the district remains responsible for verifying allergen information.
  5. For each allergen, select Contains, May Contain, Processed in a Facility That Also Processes It, or Free From. To add a custom allergen, click + ADD CUSTOM ALLERGEN, select from the dropdown (must exist in Configuration first — see Prerequisites), choose a status, click ADD → SAVE. Use Skip for now if data is not yet available.

⚠ Watch out: A blank nutrient field is not the same as checking Missing Value. Blanks block the save; Missing Value marks the nutrient as (M) in analysis outputs so auditors know data is incomplete rather than zero.

4. Create a Direct Menu Item and Set POS Pricing

Configure an item served as-is (not in a recipe) for menus, the POS terminal, and Family Hub.

  1. Click the MENU INFO tab. Status shows Disabled — this is normal before configuration.
  2. Click CREATE MENU ITEM on the Units card banner.
  3. Enter required fields: Menu Item Name, Button Name (English), and Menu Item Category.
  4. Enter optional fields as applicable: Marketing Name & Description (displayed in Family Hub), Button Name (Spanish), Meal Type, Leftover Category, and POS Entree (check if item counts toward a reimbursable meal; leave unchecked for à la carte).
  5. Toggle Show on POS on — the POS PRICING tab appears. Click SAVE. The tab label changes to MENU & POS INFO — this is expected.
  6. Click the POS PRICING tab → three-dot menu → Edit. Enter prices by eligibility row (Free, Reduced, Paid, Second Meal) and site type column, or use SET ALL PRICES to apply one price across an entire row. Click SAVE.

⚠ Watch out: The POS PRICING tab does not appear until Show on POS is toggled on and the Menu Item card is saved. Enable the toggle first, then navigate to that tab.

Not covered here — HACCP configuration (Steps card): Click ENABLE RECIPE on the Steps card and complete Step 0 to assign a HACCP process (No Cook, Same Day Service, or Complex Food). See the long-form guide for full step-by-step instructions.

Common Mistakes

  • Adding a pack size after costing is entered — always complete all pack sizes first, then add costing. A pack size added afterward will carry no cost and requires re-entering all prices via Manage Costing.
  • Skipping the serving-to-pack link for food items — food items require a Sub-Qty on the serving size row; the Inventory Ready linking alert will not clear until this is done.
  • Leaving a nutrient field blank instead of checking Missing Value — blanks block the save and misrepresent data as zero in analysis outputs; Missing Value correctly flags the nutrient as (M).
  • Not confirming Base Unit before first Inventory use — Base Unit and counting mode lock permanently after the first Inventory transaction (count, receipt, or adjustment). Set them correctly before beginning any inventory operations. If a correction is needed after that point, contact Cybersoft support — it is not a self-service fix.
  • Custom allergen not appearing in the dropdown — a non-standard allergen must be created in Item Management Configuration before it is selectable on an item's Allergen card.
