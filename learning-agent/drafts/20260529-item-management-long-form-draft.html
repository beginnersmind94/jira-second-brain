Overview

Item Management is where food and non-food items are created, configured, and maintained in SchoolCafé 2.0. Every downstream operation — inventory counts and ordering, menu planning, POS transactions, and production records — depends on the accuracy of the item record here. Two item types exist: non-food items (supplies such as cups, trays, and gloves) and food items (everything served to students). Non-food items require pack sizes, costing, and identifiers. Food items require all of that plus serving sizes, nutrient details, allergen and contribution information, menu configuration, HACCP steps, and POS pricing. The outcome a user is trying to reach is a fully configured item that is Inventory Ready — at least one pack size, a confirmed base unit, and an entered cost — and, for food items served directly, optionally a Direct Menu Item (DMI) with menu and POS configuration complete. In scope: creating and configuring food and non-food items including pack/serving sizes, costing, identifiers, images, nutrient details, allergen and contribution info, menu item setup, HACCP steps, POS pricing, and inventory info. Out of scope: vendor contracts (Inventory), running menus (Menu Planning), POS transactions (POS), multi-ingredient recipes (Recipes), and production records (Production Records).

Full Workflows

4.1 Creating a Non-Food Item (end-to-end)

Purpose: Create a supply item — cups, trays, napkins, gloves — ready for Inventory. Example: Dixie Cup, 12 oz.

  1. Navigate to Item Management > Items, click +NEW, and the Item card opens.
  2. Fill in Item Description (required) [TO VERIFY: 250-character limit — NXT-30036 Description only]; optionally add Tags, Brand Name, Manufacturer, and Product Code [TO VERIFY: field labels — NXT-30036 Description only].
  3. Fill in Item Category, Storage Category, and Valuation Group (required for Inventory use), or use Skip to bypass them.
  4. Click SAVE; the Units card appears.
  5. On the Units card select NON-FOOD; this disables the serving size section.
  6. Click +PACK, fill in Description (e.g., Box), and click the checkmark; a yellow Missing Costing alert appears [TO VERIFY: alert label — NXT-37568 Description only].
  7. Add lower-level pack sizes by clicking +PACK → LOWER, entering Description and Sub-Qty for each tier [TO VERIFY: three-tier maximum — stated by presenter at [07:05]; NXT-37110 AC text does not state a cap].
  8. Review the Identifiers column; the lowest pack size is automatically designated the Base Unit (BU).
  9. Click the Missing Costing alert, click the Edit icon, enter cost for the one unit you know (e.g., $80 for the Box), and click SAVE; the system scales costs to other units automatically. [TO VERIFY: scaling formula described by presenter at [08:32] appears in NXT-37568 Description only, not AC]
  10. The Inventory Ready flag appears and the Publish toggle auto-enables.
  11. Optionally click GTIN/UPC, enter barcode numbers per pack size row (8–14 digits), and click SAVE; a barcode icon appears in the Identifiers column.
  12. Optionally configure Whole Numbers vs. Decimals for the Base Unit [TO VERIFY: UI control label — presenter described "click the WHOLE NUMBERS chip" at [09:48]; not in NXT-33518 AC]; this setting locks after the first Inventory transaction [TO VERIFY: locking trigger — NXT-54229 Description only].
  13. Optionally add Images and Documents via the expand icon on the Images & Documents card: drag-and-drop files, select a Use As option for each file, and click SAVE; up to 10 files can be uploaded [TO VERIFY: 5 MB per-file limit and specific file types stated by presenter at [11:02] — not in NXT-41273 AC or RN].

The non-food item is now Inventory Ready and can be published to Inventory via the Publish toggle.

4.2 Creating a Food Item — Item Card and Units Card

Purpose: Set up the item record and unit structure for a consumable food item. Example: Yoplait Strawberry Yogurt, 4 oz. Complete the Item card (steps 1–4 of Section 4.1) with the food item details, then click SAVE.

  1. On the Units card select FOOD, then answer the Fluid / Non-Fluid question (Non-Fluid = grams; Fluid = milliliters for volume-measured items such as juice or milk).
  2. Click +SERVING first and fill in Amount, Description, and Weight; click the checkmark — serving sizes are recommended before pack sizes for a smoother workflow (not system-enforced).
  3. Add pack sizes using +PACK (same process as Section 4.1, steps 6–7); note that Nutrient Info, Menu Info, and Inventory Info tabs appear but are grayed out until the Units card is saved.
  4. Click the Inventory Ready alert, then Edit, and enter the Sub-Qty on the serving size row (e.g., 16 yogurts per Carton) to link the serving to the pack; click SAVE when the alert clears.
  5. Enter costing (same process as Section 4.1, step 9).
  6. Optionally promote the serving size to Base Unit (dual unit workflow) via the Merge mechanism (AC term): when a new pack size description or weight matches an existing serving, the system prompts a Merge button; clicking it confirms the serving as the base inventory unit.

Troubleshooting

Symptom: Nutrient Info, Menu Info, and Inventory Info tabs are grayed out.
Cause/Fix: The Units card has not been saved yet — complete all pack and serving sizes, then click SAVE on the Units card to activate the tabs.
Symptom: Inventory Ready alert says servings and packs must be linked.
Cause/Fix: No Sub-Qty on the serving size row — click the Inventory Ready alert, click Edit, enter how many individual servings are in the lowest pack unit, then click SAVE.
Symptom: Nutrient Details card will not save — required field error.
Cause/Fix: A required nutrient field is blank without a Missing Value checkbox — enter a value or check Missing Value; for Vitamin A and Vitamin C specifically, leaving them blank auto-enables Is Missing with no manual action needed.
Symptom: Need to change sub-quantity after item has been used in Inventory.
Cause/Fix: Changing sub-quantities is a restricted long-term editing operation per NXT-54229 AC — contact the product team for the correct recovery procedure. [TO VERIFY: exact system message or restriction behavior]
