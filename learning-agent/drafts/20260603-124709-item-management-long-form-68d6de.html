Excel Bulk Import — Entry Point

For large catalog onboarding, use the Excel import path rather than manual item creation. The process follows four steps managed through the Import Dashboard:

  1. Pre-Upload — Select import type and data source. The data source defaults to Local; other sources are out of scope. Download the system template at this step if needed.

  4. Review & Activate — Confirm mapped fields, save the mapping, then trigger the import from the dashboard.

Once submitted, track progress and review any row-level errors on the Monitoring page.

Workflow: Create a Non-Food Item

Step 1 — Complete the Item Card

Fill in all required fields on the Item card before moving to Units. Optional fields can remain non-editable until needed.

Step 2 — Build Pack Sizes on the Units Card

The Units card is where you define the pack size structure for the item. Use the add button to insert a new row in the Units grid for each pack size you need.

Sub-Qty rules determine how pack sizes are linked:

  • The first pack size you add does not have Sub-Qty enabled.
  • When adding a second pack size below an existing one (as a child), Sub-Qty appears on that same row.
  • When adding a pack size above an existing one (as a parent), Sub-Qty appears on the child item row.

The system highlights the correct Sub-Qty fields so you always know where to enter the linking quantity.

Weight is optional in most cases, but becomes required in specific scenarios (such as linking to a serving base unit).

Step 3 — Resolve the Inventory Readiness Banner

An Inventory Readiness banner is always displayed and operates as a live status indicator — not a one-time gate. Each banner version carries clear, specific messaging so you know exactly what is still required. The banner covers all scenarios, including a Non-Inventory state when the item is ineligible or disabled.

Step 4 — Enter Cost

Cost and value fields are completed during the final finish tasks, not during initial pack size setup. You select the specific pack size unit on which to record the price.

Workflow: Create a Food Item

The Units card is the single place to manage both serving sizes and pack sizes for a food item. It becomes editable once the general info card is saved.

Step 1 — Answer the header questions

Before the grid activates, two questions appear in the header bar.

  1. Food Item? — Answering No disables Serving Sizes entirely.
  2. Fluid Item? — Answering Yes switches serving size measurement to Volume; the column header changes to Weight / Volume because servings use fluid measures while pack sizes still use weight.

After both questions are answered, the grid loads with Add Serving Size and Add Pack Size actions.

Step 2 — Add serving sizes

Each serving size row captures three fields.

FieldKey rule
AmountAmount = 1 makes the serving "potentially inventory-able" and eligible for Sub-Qty linking. Any other value is a serving-only unit.
DescriptionSmart text field matched against standard measures.
Weight / VolumeDefines the measurable unit; label switches based on Fluid answer.

Step 3 — Add pack sizes and set the hierarchy

Pack sizes are added via the + Pack button. When a second pack size exists, Sub-Qty is required on the child row (lower unit) or the parent row (upper unit) to establish the Higher/Lower hierarchy. Sub-Qty is not enabled for the very first pack size.

Step 4 — Link servings to packs (Inventory Readiness)

To achieve Inventory Readiness, a serving size with Amount = 1 must be linked to a pack size via Sub-Qty. The system prompts: how many serving units fit inside the pack unit. Unlinked items remain inventory-ineligible. An Inventory Readiness banner is always displayed so the status is visible at a glance.

Step 5 — Resolve Missing Costing

Cost / Value becomes editable during finish tasks on pack sizes and on inventory-eligible serving sizes linked via Sub-Qty.

Serving vs. pack — purpose summary

  • Serving sizes — drive Menu Planning and production quantities.
  • Pack sizes — drive Inventory tracking; linking to a serving ensures the inventory unit maps to the correct menu-planning version.

Workflow: Configure Allergen Information

Authorized nutrition service users can define custom allergens to supplement the standard allergen set, ensuring items are accurately flagged for site-specific dietary requirements.

Add a Custom Allergen

Custom allergens are managed through a dedicated configuration area inside Item Management. After selecting the configuration option, a pop-up window prompts for the data source and the configuration option desired; pressing Apply loads the relevant screen.

  1. Navigate to Item Management > Configuration > Item Configuration > Custom Allergens.
  2. On the result grid, open the slide-out card to create a new record.
  3. Enter a Custom Allergen Description (free-text, mandatory, no duplicates permitted).
  4. Select a Data Source from the dropdown — only the Local data source is available; exactly one must be chosen.
  5. Click Save. Validation runs; if the name already exists the system prompts for a different name. On success the new allergen appears immediately in the result grid.

Tip: Closing the slide-out card with the X icon cancels all unsaved changes.

Step-by-Step

  1. Save the Units card to unlock the Menu Info tab.
  2. Enter all three required fields: Menu Item Name, Button Name, and Menu Item Category.
  3. Fill in any applicable optional fields for your site's needs.
  4. Under Serving Exceptions, configure overrides by Meal Pattern and Serving Group as needed.
  5. Open POS Pricing and set prices for each eligibility group (student and adult). Use the Set All Prices shortcut to populate multiple groups at once.
  6. Toggle Allow Sale on or off to control whether the item is available for point-of-sale transactions.
  7. Check Taxable if the item is subject to tax at the point of sale.
  8. Save the record.

Common mistake: Attempting to access Menu Info before saving the Units card — the tab will not appear.

Workflow: Configure HACCP and Recipe Steps

Prerequisites

The Steps & Ingredients tab activates only after a description, serving size, and number of servings have been saved on the summary card. The Steps card is displayed on the left side of this tab.

Step 1 — Select the HACCP Process

A HACCP Process section sits above or alongside the Steps card. The three standard process options are No Cook, Same Day Service, and Complex Food. This selection is the only required field in the section. Use the independent Save and Cancel controls to commit the choice before continuing.

Step 2 — Add Steps and Directions

In Edit Steps mode, directions serve as the primary visual element of each step row. Step ingredients are available in an expandable sub-section, but only one step's ingredients may be expanded at a time.

Step 3 — Insert Critical Control Points (CCPs)

An Add CCP action appears at the same level as the Add Step action inside Edit Steps mode. The system validates the save against the control points associated with the selected process.

Changing the Process After CCPs Exist

SituationSystem behavior
Switched process conflicts with existing CCPs Warning prompt shown before any CCPs are cleared
Switched process is still compatible with existing CCPs Those CCPs are retained automatically

Ingredients Summary (View Mode)

A right-side Ingredients card aggregates every ingredient added across all steps, displaying the recipe-level total quantity. The quantity is scaled to higher units for the recipe total.

Key Fields & Statuses

Units Card

The Units card sits in the item creation header alongside the general info card, consolidating pack size and serving size configuration in one place. It becomes editable only after the general card is saved. Before the grid loads, a header bar presents two setup questions that control which fields and actions become available.

  • Food Item? — if No, serving sizes are disabled entirely.
  • Fluid Item? (appears only when Food = Yes) — if Yes, the measurement column switches to volume-based units for servings.

Inventory Readiness Banners

Inventory Readiness operates as a persistent status rather than a sequential gate. A banner is always visible, with distinct messaging for every scenario.

  • Inventory Ready — triggered when pack sizes are added and properly linked.
  • Non-Inventory — shown when the item is ineligible or inventory is disabled.
  • Inventory Ineligible — results when pack units and servings exist but are not linked to each other.

To resolve an Ineligible status, enter Sub-Qty on a serving row with Amount = 1, or enter Weight on the pack size base unit.

Nutrient Data → Menu Planning Reports

Nutrient values entered on the item flow into menu planning and are aggregated in nutrient analysis reports. This means incomplete nutrient setup on even one item can create gaps in a menu's nutritional totals.

Menu Item & POS Pricing → Production and Display

Menu Item configuration and POS Pricing data flow to three downstream outputs:

  • Production records — item details are pulled into production documentation.
  • Menu Calendar Report — scheduled menu items appear with their configured attributes.
  • POS touchscreen — pricing and item display information render at the point of sale.

Because these outputs share the same source record, a correction made on the item propagates to all three without separate updates.

Exceptions & Troubleshooting

Sub-Quantity Is Disabled on the First Pack Size

Sub-Qty is intentionally unavailable when adding the first pack size to an item. It activates only when a second pack size is added — either as a child unit below or a parent unit above the existing pack size. Where Sub-Qty cannot be used, Weight serves as the alternative linking method.

Pack–Serving Linking and Inventory Readiness

Items whose pack sizes are not linked to serving sizes are flagged as Inventory Ineligible. Linking requires either a Sub-Qty or Weight entry; both options satisfy the requirement before Inventory Readiness can be achieved. The Inventory Readiness banner remains visible at all times and provides clear status messaging for every scenario, including a Non-Inventory state for items that are ineligible or disabled.

Food Items: Configure Serving Sizes Before Pack Sizes

For food items, add serving sizes before building pack sizes. If a pack is added before servings exist, the system emphasizes capturing Weight on the lowest pack unit so that linking can complete once the serving is added later.
