Item Management: Creating Food and Non-Food Items — Micro Guide

Purpose

After reading this guide, you can create and fully configure food and non-food items in SchoolCafé 2.0's Item Management module — from the initial Item Card through pack sizes, costing, Nutrient Info, Direct Menu Item creation, and POS Pricing. Correctly configured items prevent errors across Inventory, Menu Planning, and POS.

Who This Is For

Implementation Specialists and Cybersoft staff configuring a new district, and Nutrition Directors or Item Management Super Users who create and maintain items. Nutritionists/Menu Planners handle Nutrient Info and Steps; Menu Planning Managers configure POS pricing.

Before You Start

  • Active credentials with Item Management module permissions are required before creating any item.
  • Item Category, Storage Category, and Valuation Group are required only when the Inventory module is enabled — but fill them in during setup regardless; retrofitting hundreds of items later is labor-intensive.
  • Common Measures must be configured (Case, Box, Sleeve, etc.) before pack size descriptions can be named.
  • Custom allergens must be created in Item Configuration before they can be assigned to individual items. Navigate to Item Management → Configuration → Item Configuration → Custom Allergens.
  • HACCP Processes and Control Points must be configured system-wide in Item Configuration before they appear as options on the Steps card.

Top Workflows

1. Create a Non-Food Item

Configure a supply item (cups, trays, napkins) for use in Inventory. Non-food items have no nutrition, allergen, or menu data.

  1. Go to Item Management → Items and click + NEW.
  2. On the Item Card, enter Item Description (required; [TO VERIFY: 250-character limit — stated at [02:01] with uncertainty, not confirmed in Jira AC]), Item Category, Storage Category, Valuation Group (required if Inventory enabled), and optional Brand Name/Manufacturer. Click SAVE.
  3. On the Units Card, click NON-FOOD. Click + PACK to add your first pack size. For each additional tier, click + PACK, choose HIGHER or LOWER, and enter Description and Sub-Qty. Maximum three tiers.
  4. After all pack sizes are added, click the Missing Costing alert → pencil icon. Enter cost on one unit only; the system calculates the rest through the Sub-Qty chain. Click SAVE.
  5. Verify the Inventory Ready green flag appears and the Publish toggle auto-enables.
⚠ Watch out: Do not click the Missing Costing alert until all pack sizes are added. Acting early leaves costs missing for any tiers added afterward.

2. Create a Food Item (Units Card)

Configure a food item's unit structure — item type, serving sizes, pack sizes, serving-pack link, costing, and base unit assignment.

  1. Go to Item Management → Items, click + NEW, complete the Item Card (same fields as non-food), and click SAVE.
  2. On the Units Card, click FOOD, then choose NON-FLUID (grams) or FLUID (milliliters) based on how the nutrition label measures weight/volume.
  3. Click + SERVING. Enter Amount, Description, and Weight (oz is converted to grams automatically). Click the checkmark.
  4. Click + PACK and add pack size tiers using the same Higher/Lower flow as non-food (max three tiers).
  5. Click the Inventory Ready alert → Edit icon. Enter Sub-Qty on the serving row (how many servings are in the lowest pack size). Click SAVE.
  6. Complete costing on one unit (same as non-food). Expand the Inventory Ready banner, confirm or promote the Base Unit (if a serving with Amount = 1 is linked, the system offers to promote it), select Whole Numbers or Decimals, click CONTINUE → SAVE & CLOSE.

3. Configure Nutrient Info (Food Items Only)

Record nutrition data, ingredient details, meal pattern contribution, and allergen status. This tab is active only after the Units Card is saved.

  1. Click the NUTRIENT INFO tab. On the Nutrient Details card, click three-dot → Edit. Enter all asterisk-required fields (Calories, Total Fat, Saturated Fat, Trans Fat, Cholesterol, Sodium, Carbohydrate, Dietary Fiber, Total Sugars, Added Sugars, Protein, and required vitamins/minerals). For any nutrient absent from the product label, check Missing Value — do not leave the field blank. Click SAVE. The system calculates Per 100 Gram values and percentages automatically.
  2. On the Ingredient Info card: click three-dot → Edit. Set Edible Yield Factor (e.g., 100% for yogurt, ~85% for banana). Check Locally Grown and/or Buy American as applicable. Click SAVE.
  3. On the Contribution Info card: click three-dot → Edit. Enter meal pattern credit (e.g., 1 oz equivalent Meat/Meat Alternate). Click SAVE.
  4. On the Allergen Info card: click three-dot → Edit. Read the disclaimer (district remains responsible for verifying allergen accuracy). Mark status for each standard allergen (Contains / May Contain / Free From / Processed In a Facility). To add a non-standard allergen, click + ADD CUSTOM ALLERGEN, select from the dropdown, click ADD, then SAVE. 4. Create a Direct Menu Item (DMI)

Convert a food item into a single item that can be placed directly on a menu or production record without being wrapped in a recipe.

  1. Click the MENU INFO tab. A Disabled status is normal before configuration.
  2. Click CREATE MENU ITEM from the Units Card banner.
  3. Enter required fields: Menu Item Name (appears on menus and production records), Button Name (English) (appears on the cashier's POS screen), and Menu Item Category. Check POS Entree if the item counts toward a reimbursable meal.
  4. Toggle Show on POS ON to enable POS sales; this activates the POS Pricing tab.
  5. Click SAVE. The tab renames to MENU & POS INFO and an MI badge appears on the serving row in the Units Card.
⚠ Watch out: If Show on POS is not toggled ON before saving, the POS Pricing tab does not activate and the tab name remains MENU INFO.

5. Set POS Pricing

Define what students and adults pay for this item at the register. Only available after Show on POS is enabled.

  1. Click the POS PRICING tab, then click three-dot → Edit on the POS Pricing card.
  2. In the Students grid, enter prices by eligibility type (Free, Reduced, Paid, Second Meal) and site type (Elementary, Middle, High School, Alternative). Use SET ALL PRICES to fill uniform prices across all site types at once.
  3. Click the ADULTS tab and repeat for staff, visitor, and program adult categories.
  4. Check Taxable if applicable (most school meals are not). Click SAVE.
⚠ Watch out: The Allow Sale toggle auto-enables when a price is entered. Manually toggle it off to block sales to a specific eligibility group even when a price is set.

Additional configuration (not covered at full depth): The Steps card supports HACCP process assignment (No Cook / Same Day Service / Complex Food) and step-by-step recipe directions for Direct Menu Items. Common Mistakes

  • Skipping Item Category, Storage Category, and Valuation Group because Inventory isn't enabled yet — if your district turns on Inventory later, you'll have to retrofit every item, which is labor-intensive.
  • Clicking Missing Costing before all pack sizes are added — costs for tiers added afterward will not be calculated. Complete all pack size tiers first.
  • Setting the wrong Base Unit or Whole Numbers/Decimals before the item goes live in Inventory — these settings lock permanently once any Inventory transaction (count, receipt, or adjustment) occurs.
  • Leaving a required nutrient field blank instead of checking Missing Value — a blank field implies zero; Missing Value correctly marks the nutrient as unknown, displayed as (M) in nutrient analysis reports.
  • Trying to assign a custom allergen to an item before creating it in Item Configuration — the allergen must first exist at Item Management → Configuration → Item Configuration → Custom Allergens before it appears in the item's allergen dropdown.
