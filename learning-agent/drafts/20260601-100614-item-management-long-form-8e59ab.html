Item Management: Creating Food and Non-Food Items

Module: Item Management  |  Template: Long-form guide  |  Source recording: REC-2026-0515-IM-001 (May 15, 2026)  |  Presenter: Sarah Chen, Implementation Specialist  |  Generated: 2026-06-01

1. Overview

Item Management is the foundational module in SchoolCafé 2.0 where all items — food and non-food alike — are created and maintained. Every downstream module draws its data from here: Inventory uses pack sizes and costing, Menu Planning uses serving sizes and nutrient data, POS uses pricing and button names, and Family Hub uses marketing descriptions. Errors introduced at the item level propagate throughout all of those modules — incorrect pack sizes create receiving problems, incomplete nutrient data breaks menu analysis, and missing POS configuration prevents cashiers from completing transactions.

There are two item types: food items (anything served as a meal component or used as a recipe ingredient) and non-food items (consumable supplies such as cups, trays, napkins, and gloves). Non-food items require a smaller set of configuration steps. Food items require everything non-food items have, plus nutrients, allergens, meal pattern contributions, Direct Menu Item configuration, POS pricing, and HACCP steps. The Item card is identical for both types; configuration diverges beginning at the Units card.

The expected outcome for a new district is a complete, published item catalog — every item correctly categorized, costed, and linked — before any Inventory counts or menu planning begins.

2. Roles and Permissions

Role Typical actions in Item Management
Item Management Super User / Administrator Creates and edits all item types; manages module configuration (custom allergens, HACCP processes); publishes items to Inventory.
District Admin Creates new items; enters product information; configures pack sizes and costing.
Menu Planner Configures the Menu Info card to create Direct Menu Items; sets POS pricing.
Nutritionist Enters nutrient data, allergen information, and meal pattern contributions; assigns the HACCP process on the Steps card.
Cybersoft Implementation Specialist Guides district staff through initial item setup during onboarding; does not own ongoing item maintenance.

Inventory-related fields note: Item Category, Storage Category, and Valuation Group are required for districts that have Inventory enabled. For districts not yet using Inventory, these fields are technically optional — but filling them in at creation avoids retroactive data entry if Inventory is turned on later.

3. Prerequisites

  • District environment provisioned. The district must be fully set up in SchoolCafé 2.0 before items can be created. Consult your implementation team if the district is not yet onboarded.
  • Item categories, storage categories, and valuation groups configured. These values must exist in the system before they can be selected on an item. They are typically pre-loaded during onboarding.
  • Custom allergens created (if applicable). Any allergen not on the standard list must be created in Item Management → Configuration → Item Configuration → Custom Allergens before it can be attached to an item.
  • HACCP processes configured (if using the Steps card). HACCP process options and control point templates are defined in Item Management Configuration before they appear on the Steps card.
  • User login credentials. Access to Item Management requires an account with appropriate module permissions. Contact your district administrator if the module is unavailable after login.

4. Full Workflows

4.1 Creating a Non-Food Item

Non-food items are supplies tracked in Inventory but never placed on menus or included in nutrient analysis. This workflow is the simpler of the two.

Step 1 — Navigate to item creation

  1. Go to Item Management → Items.
  2. Click + NEW. The Item Details page opens, displaying the Item card.

Step 2 — Complete the Item card

  1. Item Description (required) — Enter a descriptive name that includes the item name and size. Example: Dixie Cup, 12 oz. [TO VERIFY: presenter estimated a 250-character limit at transcript [02:01] — "I think it's a 250 character limit" — but this was not confirmed in Jira AC. Verify before advising staff on maximum length.]
  2. Item Category (required for Inventory) — Select the appropriate category. Example: Non-Food Supplies.
  3. Storage Category (required for Inventory) — Select where the item is stored. Example: Dry.
  4. Valuation Group (required for Inventory) — Select the appropriate group. Example: Non-Food.
  5. Brand Name, Manufacturer, Product Code, Tags — All optional. Recommended because Brand Name and Manufacturer are available as search filters, making it easier to locate items later (e.g., find all Dixie products).
  6. Click SAVE. The page refreshes and the Units card becomes active.

Step 3 — Set item type on the Units card

  1. On the Units card, click NON-FOOD. Selecting NON-FOOD disables serving size entry and bypasses the fluid/non-fluid question — both of which apply only to food items.

Step 4 — Add pack sizes

Pack sizes define the purchasing and storage hierarchy used for inventory counts, ordering, and receiving. The system supports a maximum of three tiers.

  1. Click + PACK to add the first (highest) level. Enter a Description (required) — e.g., Box. Weight is optional for non-food items. Click the checkmark to save the row.
  2. To add a sub-level, click + PACK again. The system prompts: Higher or Lower? — select LOWER to add a child unit inside the existing one. Enter Description (e.g., Sleeve) and Sub-Qty — the number of this unit per parent unit (e.g., 20 sleeves per box). Click the checkmark.
  3. Repeat for a third tier if needed. Example: Cup, Sub-Qty 25 (cups per sleeve).
  4. After the third tier is added, the + PACK button becomes grayed out. No further tiers can be added.

Identifier badges: After saving each row, a badge appears in the Identifiers column. P = Pack Size. BU = Base Unit. The system automatically assigns BU to the lowest-level pack size. The Base Unit is the unit used when conducting inventory counts.

Step 5 — Resolve the Missing Costing alert

After saving pack sizes, a yellow Missing Costing alert appears. Add all pack sizes before entering costing — adding a new pack size after costing requires re-entry.

  1. Click the Missing Costing alert to expand it, then click the Edit (pencil) icon.
  2. Enter the price for one unit only — typically the unit you purchase (e.g., Box = $80.00). The fields for all other units are grayed out while you type.
  3. Click SAVE. The system calculates costs for all other tiers by dividing down through the Sub-Qty hierarchy. Example: Box $80.00 ÷ 20 sleeves = $4.00 per Sleeve; $4.00 ÷ 25 cups = $0.16 per Cup.
  4. The Missing Costing alert is replaced by a Manage Costing button. Click it at any time to update pricing; the system recalculates all tiers.

Step 6 — Add GTIN/UPC barcodes (optional)

  1. Click GTIN/UPC on the Units card. Each pack size row gains a GTIN entry field.
  2. Enter the barcode number for any pack size used with a scanner during receiving.
  3. Click SAVE. Barcode icons appear in the Identifiers column for any rows with codes.

Step 7 — Confirm Inventory Options

  1. Click the Inventory Ready flag (green) to expand it. The banner confirms the Base Unit and the counting mode.
  2. The default counting mode is WHOLE NUMBERS. If partial units are needed, click the WHOLE NUMBERS chip to switch to DECIMALS. For most non-food items, whole numbers is correct.
  3. Important: Base Unit and counting mode can be changed freely until the item is first used in an Inventory transaction (count, receipt, or adjustment). After first Inventory use, both settings are locked. Set these correctly before beginning inventory operations.

Step 8 — Upload images and documents (optional)

  1. Click the expand icon on the Images & Docs card. The Manage Images & Documents panel opens.
  2. Drag and drop files or click SELECT FILES. Supported formats include JPEG, PNG, PDF, Word documents, and spreadsheets. Maximum file size: 5 MB per file. [TO VERIFY: stated by presenter at transcript [11:02]; not confirmed in Jira AC.]
  3. After upload, select a use from the "Use as" dropdown — e.g., Preferred Inventory Image. Add an optional comment. Multiple images with different assigned uses are supported.
  4. Click SAVE. The image appears on the Item Details page.

The non-food item is now fully configured and ready for Inventory.

4.2 Creating a Food Item

Food items follow the same initial steps as non-food — Item card, then Units card — but require substantially more configuration: serving sizes, serving-to-pack linking, nutrients, allergens, meal pattern contributions, ingredient info, Direct Menu Item setup, POS pricing, and HACCP steps. The example used throughout this section is Yoplait Strawberry Yogurt, 4 oz.

Step 1 — Item card

Complete the Item card following the same process as Section 4.1, Steps 1–2. Use food-appropriate values:

  • Item Category: Entrees
  • Storage Category: Cooler (for refrigerated items)
  • Valuation Group: Non-Commodity

Step 2 — Food and Fluid/Non-Fluid selection on Units card

  1. Click FOOD on the Units card.
  2. A second question appears: Non-Fluid Item (Grams) or Fluid Item (Milliliters)? This question appears only for food items — non-food items skip it entirely.
  3. Select NON-FLUID if the nutrition label measures weight in grams (most solid and semi-solid foods, including yogurt). Select FLUID if the label uses milliliters (beverages such as juice, milk, or other drinks).

Step 3 — Add serving sizes (food items only)

Serving sizes are the unit of measure used for recipe ingredients and menu planning. Pack sizes are for Inventory. The two serve different purposes. Add serving sizes before pack sizes for the smoothest workflow.

  1. Click + SERVING. Fill in:
    • Amount: 1
    • Description: Each
    • Weight: 4 Ounce
  2. Click the checkmark. The Nutrient Info, Menu Info, and Inventory Info tabs appear at the bottom of the page but remain grayed out until the Units card is saved.

Step 4 — Add pack sizes (food items)

  1. Click + PACK. Enter Description: Case. Weight is optional. Click the checkmark.
  2. Click + PACK again → select LOWER. Enter Description: Carton, Sub-Qty: 3 (3 cartons per case). Click the checkmark.

Step 5 — Link serving sizes to pack sizes (food items only)

Food items display an Inventory Ready alert: "Servings and Pack Units must be linked for Inventory Eligibility." This alert is unique to food items; non-food items do not require linking.

  1. Click the Inventory Ready alert → click the Edit icon.
  2. In the Sub-Qty field on the serving size row, enter how many servings are in the lowest pack unit. Example: 16 yogurts per carton.
  3. Click SAVE. The Inventory Ready linking alert disappears.

Step 6 — Resolve Missing Costing (same process as non-food)

Enter the price for the unit you purchase. Example: Case = $45.02. The system divides down to calculate Carton ($15.01) and Each ($0.94) automatically. Note that the system rounds calculated values.

Step 7 — Confirm or promote the Base Unit (food items)

  1. Click the BASE UNIT chip on the Inventory Ready flag.
  2. The system shows the current default Base Unit (the lowest pack size — e.g., Carton) and, because the serving size (Each) is linked via Sub-Qty, offers the option to promote it.
  3. To count individual servings during inventory (e.g., count individual yogurt containers rather than whole cartons), click the serving size row and then CONFIRM. The serving size becomes a dual unit, carrying both the Serving and Base Unit (BU) identifiers.
  4. Select Whole Numbers or Decimals, then click CONTINUE → SAVE & CLOSE.

Lock-out reminder: The Base Unit choice and Whole Numbers/Decimals setting lock permanently after the item's first Inventory transaction. Confirm these settings are correct before starting any inventory operations.

Step 8 — Nutrient Info tab

  1. Click the NUTRIENT INFO tab. The system shows the serving size at the top and converts the entered weight to grams automatically — e.g., 4 oz becomes 113.4 g.
  2. Click the three-dot (⋯) menu on the Nutrient Details card → Edit.
  3. Complete all fields marked with an asterisk — these are required. For nutrients not present on the item's label, check the Missing Value checkbox. Do not leave the field blank, and do not enter zero — blank implies unknown, zero implies the item literally contains none.
  4. Click SAVE. The system automatically populates the Per 100 Gram column and Daily Value percentages. Nutrients marked missing display (M).

What Missing Value means: Marking a nutrient Missing tells the system "data is not available" — distinct from zero. In nutrient analysis reports, missing values are flagged so staff and auditors know the data is incomplete, rather than misreading a blank as zero.

Step 9 — Ingredient Info card

  1. Click the three-dot (⋯) menu on the Ingredient Info card (within the Nutrient Info tab) → Edit.
  2. Edible Yield Factor — percentage of the item that is edible after preparation. Enter 100 for items with no waste. Enter a lower value for items with inedible portions (e.g., a banana's peel).
  3. Sub Ingredients — ingredient sub-list for when this item is used as a component in a recipe.
  4. Standard Recipe Directions — preparation notes that auto-populate on any recipe using this item as an ingredient (e.g., "Thaw for 30 minutes before serving").
  5. Locally Grown — check if applicable for compliance tracking.
  6. Buy American — check if the item is domestically sourced. [TO VERIFY: Presenter stated at transcript [21:22] that checking Buy American disables the exemption letter upload because checking asserts domestic compliance, while leaving it unchecked enables the upload icon. This behavior was not confirmed in Jira AC — verify before using in compliance training.]
  7. Click SAVE.

Step 10 — Contribution Info card (Meal Pattern)

  1. Click the three-dot (⋯) menu on the Contribution Info card → Edit. A slide-out panel opens.
  2. Enter the meal component credits this item provides — e.g., 1 oz equivalent Meat/Meat Alternates for yogurt. Enter additional credits (Grains, Vegetables, Fruit, Milk) if the item contributes to multiple components.
  3. Click SAVE.

Step 11 — Allergen Info card

  1. Click the three-dot (⋯) menu on the Allergen Info card → Edit. A slide-out panel opens.
  2. Read the Allergen Feature Disclaimer at the top. It states that SchoolCafé is a tool — the district remains responsible for verifying allergen information.
  3. For each standard allergen, select the appropriate status from: Contains, Processed in a Facility That Also Processes It, May Contain, or Free From.
  4. To add a non-standard allergen (e.g., Strawberry):
    • Click + ADD CUSTOM ALLERGEN at the bottom of the panel.
    • The allergen must already exist in Item Configuration (see Prerequisites). Select it from the dropdown, choose the status, click ADD.
    • Click SAVE on the panel.
  5. Allergen data can be entered independently — nutrient data and contributions do not need to be complete first. Use Skip for now if data is not yet available.

Step 12 — Menu Info tab: creating a Direct Menu Item

A Direct Menu Item is a single item served as-is — not cooked or combined into a recipe — such as individual yogurt containers or packaged fruit.

  1. Click the MENU INFO tab. Status shows Disabled — this is normal before the menu item is configured.
  2. Click CREATE MENU ITEM on the Units card banner. The Menu Item card opens.
  3. Complete required fields:
    • Menu Item Name (required) — appears on menus, production records, and reports. Example: Strawberry Yogurt.
    • Button Name (English) (required) — the label the cashier sees on the POS touchscreen.
    • Menu Item Category (required) — e.g., Entree.
    • Max Days — maximum days this item can carry over on a production record. Applies only to Unspecified or Carryover leftover categories.
  4. Complete optional fields as appropriate:
    • Marketing Name & Description — consumer-facing name and text displayed in Family Hub (the parent portal). Use language that is appealing to families.
    • Button Name (Spanish) — for bilingual POS displays.
    • Meal Type — Lunch, Breakfast, etc. Used for Menu Grid filtering.
    • Leftover Category — how unserved units are handled: Return to Stock (returned to Inventory) or Discard.
    • Tags — custom labels for filtering.
    • Preparation Site Item — check if the item is prepared at a central kitchen and shipped to serving sites.
    • Show in Summary — controls visibility in the Menu Calendar Report.
    • POS Entree — check if this item counts toward a reimbursable meal. Leave unchecked for à la carte items.
  5. Toggle Show on POS on to display the item on POS terminals. The POS PRICING tab appears once this toggle is enabled.
  6. Click SAVE. The tab name changes from MENU INFO to MENU & POS INFO. This is expected behavior.

Step 12a — Configure serving exceptions (optional)

  1. On the Units card, click Edit. Two new actions appear: + MENU SERVING (to add additional serving sizes for the menu item) and EXCEPTIONS.
  2. Click EXCEPTIONS to restrict which student groups receive this item. Assign a Meal Pattern & Serving Group, then click NOT SERVED to block the selected group from receiving it. Example use case: a 4 oz yogurt approved for elementary and middle school but not high school.

Step 13 — Steps card (HACCP)

  1. Click ENABLE RECIPE on the Steps card. Note: the label says "recipe" even for Direct Menu Items — this is expected.
  2. Click the three-dot (⋯) menu → Edit. This opens Step 0, the HACCP configuration step.
  3. Select the HACCP Process from the available options:
    • No Cook — items served without any cooking, such as yogurt or pre-packaged fruit.
    • Same Day Service — items cooked and served the same day.
    • Complex Food — items requiring multi-step temperature cycling or multi-day preparation.
    [TO VERIFY: The process names No Cook, Same Day Service, and Complex Food were stated clearly by the presenter at transcript [28:35] and are consistent with NXT-36583 description — but the exact names in the UI were not confirmed in Jira AC.]
  4. Optionally enter Pre-Preparation Instructions (e.g., "Remove from cooler 15 minutes before service"), Prep Time, and Cook Time.
  5. Click SAVE for Step 0.
  6. To add a recipe step, click + ADD and type the direction.
  7. To add a Critical Control Point (CCP), click the dropdown arrow on + ADD → select Control Point → choose the type (e.g., Cold Hold). The system pre-fills the Critical Limit and a Corrective Action; edit these values if your district's requirements differ. [TO VERIFY: Presenter stated at transcript [29:42] that the Cold Hold critical limit auto-fills as "≤42°F" — this specific value was not confirmed in Jira AC. Verify against your HACCP configuration.]
  8. Reorder steps by dragging the handle on the left side of each row. Save after any reordering.

Step 14 — POS Pricing tab

  1. Click the POS PRICING tab (visible only after Show on POS is enabled).
  2. Click the three-dot (⋯) menu → Edit. A pricing grid appears:
    • Rows: eligibility types — Free, Reduced, Paid, Second Meal.
    • Columns: site types — Elementary, Middle, High School, Alternative.
  3. Enter prices for each cell individually, or use the SET ALL PRICES field at the top to apply a single price across all site types for a given eligibility row.
  4. The Allow Sale toggle enables automatically when a price is entered. Manually disable it to block sales to a specific eligibility group.
  5. Click the ADULTS tab to enter pricing for staff, visitors, and program adults.
  6. Check Taxable if sales tax applies to this item. [TO VERIFY: The presenter was uncertain about how sales tax rates are configured in SchoolCafé 2.0 at transcript [31:20] — consult your business office and Cybersoft support before enabling this option.]
  7. Click SAVE.

When Nutrient Info, Menu & POS Info, POS Pricing, and Inventory Info all display green check indicators, the food item is fully configured.

Inventory Info tab (informational)

The Inventory Info tab is read-only within Item Management. Once the item is published to Inventory and added to a vendor contract in the Inventory module, this tab displays the active contract details: vendor name, vendor item number, unit, lead time, minimum and maximum order quantities, and current price. A shortcut icon on this tab links directly to the Contract Items page in Inventory. You can also access the Units card edit functions (pack size updates, GTIN management) from this tab.

6. Reports and Outputs

Item Management does not generate standalone printed reports. Items feed data into reports in downstream modules:

  • Nutrient analysis reports — generated in Menu Planning. Require all asterisk-marked nutrient fields to be completed. Nutrients marked Missing are flagged in analysis output and do not contribute as zero.
  • Production records — generated in Menu Planning. The Menu Item Name, Max Days, and Leftover Category entered on the Menu Item card drive how production records behave for this item.
  • Menu Calendar Report — the Show in Summary checkbox on the Menu Item card controls whether an item appears in this report.
  • Inventory cost and count reports — pack sizes, Sub-Qty, costing, and the Base Unit setting all determine how items appear in Inventory count sheets and cost reports.
  • HACCP and recipe reports — the Steps card data (HACCP process, control points, recipe directions) feeds into recipe and food-safety reports. These are run from the Recipes area within Item Management.

7. Exceptions

Maximum three pack size tiers

The system supports a maximum of three pack size tiers per item. After the third tier is saved, the + PACK button is grayed out. If your purchasing hierarchy has four levels (e.g., Pallet → Case → Carton → Each), configure the three tiers most relevant to day-to-day operations. Handle the highest-level unit (e.g., pallet) at the receiving entry step in Inventory — enter the quantity as the number of cases received.

Base Unit and counting mode lock after Inventory use

The Base Unit selection and the Whole Numbers/Decimals counting mode can be freely changed until the item is used in its first Inventory transaction (a count, receipt, or adjustment). After that first use, both settings are permanently locked. If an incorrect Base Unit is discovered after Inventory use has begun, contact Cybersoft support — it is not a self-service correction.

Custom allergens must be created in Configuration first

A non-standard allergen (one not in the default list) cannot be created directly from an item's Allergen card. It must first be created in Item Management → Configuration → Item Configuration → Custom Allergens, where you also set whether it appears on Family Hub menus. Only after that step does the allergen appear in the custom allergen dropdown on the item. Attempting to add a custom allergen that has not been configured results in it not appearing in the selection list.

Buy American checkbox and exemption letters

[TO VERIFY: Presenter stated at transcript [21:22] that checking the Buy American box asserts domestic compliance and disables the exemption letter upload, while leaving it unchecked enables the upload icon for attaching an exemption letter. This behavior was not confirmed in Jira AC and should be verified against current system behavior before including in compliance workflows.]

Sub-Qty reflects the standard specification, not actual shipments

The Sub-Qty field should reflect the standard pack specification — what is printed on the vendor's spec sheet. If a vendor occasionally ships a short count (e.g., 18 sleeves instead of 20), that discrepancy is addressed during receiving in Inventory, not by changing the item record. The item record should always reflect the standard.

8. Troubleshooting

Missing Costing alert persists after entering a price

Symptom: A price was entered but the Missing Costing alert does not clear.
Likely cause: A pack size row was added after costing was entered, leaving it without a cost.
Fix: Always add all pack sizes before entering costing. Then click Manage Costing to re-enter pricing for all tiers.

Nutrient Details card will not save

Symptom: Save is blocked on the Nutrient Details card.
Likely cause: One or more asterisk-marked (required) nutrient fields are empty.
Fix: Review all required fields. If a nutrient is absent from the item label, check the Missing Value checkbox — do not leave it blank. The Missing Value checkbox marks the field as (M) in analysis outputs; a blank field or a zero implies different things.

Nutrient Info and Menu Info tabs remain grayed out after adding serving sizes

Symptom: Tabs appear but cannot be clicked after a serving size is added.
Likely cause: The Units card has not yet been saved.
Fix: Save the Units card. The tabs become active only after the initial Units card save.

MENU INFO tab stays Disabled

Symptom: Tab status shows Disabled even after completing nutrients and other tabs.
Likely cause: CREATE MENU ITEM has not been clicked on the Units card banner.
Fix: Click CREATE MENU ITEM from the Units card banner. The tab activates once the Menu Item card is opened.

Inventory Ready alert persists for food items even after costing is done

Symptom: Food item shows "Servings and Pack Units must be linked" alert after costing is complete.
Likely cause: Serving size has not been linked to pack sizes via Sub-Qty.
Fix: Click the Inventory Ready alert → Edit, then enter Sub-Qty on the serving size row.

Steps card says "recipe" but the item is a Direct Menu Item

Symptom: The button label reads ENABLE RECIPE on the Steps card of a Direct Menu Item.
Likely cause: This is expected UI behavior — the Steps card uses the same label for all item types.
Fix: Click ENABLE RECIPE and proceed normally to configure HACCP process and control points.
