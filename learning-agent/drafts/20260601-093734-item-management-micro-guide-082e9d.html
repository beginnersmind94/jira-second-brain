Item Management: Creating Food and Non-Food Items — Micro Guide

Purpose

After reading this, you can create non-food supply items and food items in SchoolCafé 2.0, configure pack sizes and costing, enter nutrient and allergen data, and publish a Direct Menu Item with POS pricing — ensuring that Inventory, Menu Planning, and POS modules have accurate item records from day one.

Who This Is For

District Administrators who build and maintain the item catalog. Menu Planners (Nutritionists) who configure nutrient, allergen, and contribution data and create Direct Menu Items.

Before You Start

  • Valid SchoolCafé 2.0 login credentials required for each staff member.
  • Item Category, Storage Category, and Valuation Group must exist in the system before item creation — required only when the Inventory module is enabled.
  • Menu Item Category (e.g., "Entree") must be configured before a Direct Menu Item can be classified.
  • Any custom allergens (e.g., Strawberry) must be pre-built at Item Management > Configuration > Item Configuration > Custom Allergens before they can be assigned to items.

Top Workflows

1. Create a Non-Food Item

Build a supply item (cups, trays, gloves) with pack-size hierarchy and costing so it can be tracked in Inventory.

  1. Navigate to Item Management > Items, click + NEW.
  2. Fill in Item Description (required); add Item Category, Storage Category, and Valuation Group if Inventory is enabled. Click SAVE.
  3. On the Units card, click NON-FOOD.
  4. Click + PACK to add the highest pack tier (e.g., Box). For a lower tier, click + PACK again, select LOWER, and enter Sub-Qty (e.g., 20 sleeves per box). Maximum three tiers.
  5. After all pack sizes are saved, click the Missing Costing alert → Edit (pencil) → enter cost for one unit → SAVE. The system calculates the remaining tiers automatically.
  6. Confirm the green Inventory Ready flag appears; the Publish toggle is now enabled.
⚠ Watch out: Enter costing only after all pack-size tiers are finalized. Adding costing before all tiers exist causes the Missing Costing alert to persist after saving.

2. Create a Food Item (Units Card)

Set up a food item with serving sizes, pack sizes, and the sub-quantity link that makes it eligible for Inventory.

  1. Complete the Item card (same as non-food), click SAVE, then click FOOD on the Units card. Choose NON-FLUID (grams) for solid foods or FLUID (milliliters) for volume-measured items.
  2. Click + SERVING. Enter Amount (typically 1), Description (e.g., Each), and Weight from the nutrition label. Click the checkmark.
  3. Click + PACK and add pack tiers using the Higher/Lower hierarchy (same process as non-food).
  4. Click the yellow Inventory Ready alert → Edit → enter Sub-Qty on the serving size row (e.g., 16 yogurts per carton) → SAVE.
  5. Address Missing Costing using the same one-unit entry process as non-food. Click SAVE & CLOSE. The Nutrient Info, Menu Info, and Inventory Info tabs activate.
⚠ Watch out: Skipping the Sub-Qty link leaves the item marked Inventory Ineligible. It can still be used in menu planning, but it cannot participate in Inventory workflows until the serving-to-pack link is established.

3. Configure Nutrient Info and Allergens

Enter nutrition facts and allergen declarations so the item supports menu analysis, production records, and Family Hub display.

  1. Click the NUTRIENT INFO tab → three-dot menu on Nutrient Details → Edit. Fill all required (asterisk) fields. For any nutrient absent from the product label, check Missing Value to mark it (M) instead of zero — analysis reports flag (M) values as data-incomplete. Click SAVE.
  2. Click the three-dot menu on Allergen Info → Edit. Read the Allergen Feature Disclaimer at the top of the panel before proceeding.
  3. For each standard allergen (Crustacean Shellfish, Egg, Fish, Gluten, Milk, Peanuts, Sesame, Soy, Tree Nuts, Wheat) select Contains, May Contain, Processed in a Facility That Also Processes, or Free From.
  4. To add a custom allergen, click + ADD CUSTOM ALLERGEN, select from the dropdown (must be pre-configured — see Before You Start), and click ADD.
  5. Click SAVE.
⚠ Watch out: Unfilled required nutrient fields block saving the Nutrient Details card, but do not block saving the item or using it in Inventory. Nutrient completeness is enforced only at menu planning and analysis time.

4. Create a Direct Menu Item and Set POS Pricing

Convert a food item into a standalone menu item that appears on POS touchscreens with per-eligibility-group pricing.

  1. Click the MENU INFO tab → CREATE MENU ITEM from the banner.
  2. Fill required fields: Menu Item Name (appears on menus and production records), Button Name (English) (cashier POS touchscreen label), Menu Item Category, and Max Days (carryover limit on production records).
  3. Toggle on Show on POS; the POS Pricing tab becomes active. [TO VERIFY: tab label reported to change from "MENU INFO" to "MENU & POS INFO" after save — stated at [27:22]; not confirmed in Jira AC]
  4. Click SAVE on the Menu Item card.
  5. Click the three-dot menu on POS Pricing → Edit. Enter prices per eligibility type (Free, Reduced, Paid, Second Meal) across site types, or use SET ALL PRICES to apply one price across all site types for an eligibility group.
  6. Click the ADULTS tab to set staff and visitor pricing. Check Taxable if applicable. [TO VERIFY: sales tax rate configuration mechanism — presenter was uncertain at [31:20]; no Jira source found]
  7. Click SAVE.
⚠ Watch out: The POS Entree checkbox must be checked for items that count toward a reimbursable meal; leave it unchecked for à la carte extras. Misconfiguring this field affects meal reimbursability.

Common Mistakes

  • Costing entered before all pack sizes are finalized — always add every pack tier first; entering cost mid-setup causes the Missing Costing alert to persist after saving.
  • Serving sizes not linked to pack sizes on food items — a missing Sub-Qty leaves the item Inventory Ineligible; fix it via the yellow Inventory Ready alert.
  • Base Unit and count mode locked after first Inventory transaction — configure both the Base Unit designation and the Whole Numbers/Decimals setting before the item enters active Inventory use; changes afterward require administrator intervention.
  • Custom allergen missing from the allergen dropdown — the allergen must first be created at Item Management > Configuration > Item Configuration > Custom Allergens before it appears on the item allergen panel.
  • Nutrient Info and Menu Info tabs remain grayed out — these tabs unlock only after the Units card is fully completed and closed via SAVE & CLOSE.
