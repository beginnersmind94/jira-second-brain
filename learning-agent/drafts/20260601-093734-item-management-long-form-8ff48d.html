Item Management: Creating Food and Non-Food Items

Training source: REC-2026-0515-IM-001 · Presenter: Sarah Chen (Implementation Specialist) · Session date: 2026-05-15 · Platform: SchoolCafé 2.0 · File: raw/transcripts/20260529-070600-item-management-2026-05-28-item-management-create-items-training.md

1. Overview

Item Management is the master catalog for every product your district buys, serves, or tracks — from individual food items to non-food supplies like trays and gloves. All downstream modules — Inventory, Menu Planning, and POS — draw their item data from records created here, which means errors made at item setup propagate everywhere: incorrect pack sizes cause Inventory discrepancies, missing nutrients block menu analysis, and misconfigured POS fields affect cashier screens. Getting items right at the start is the single highest-leverage action in your onboarding.

The module recognizes two item types. Non-food items (supplies: cups, trays, gloves, etc.) follow a shorter path — Item card, pack sizes, and costing. Food items extend that foundation with nutritional data, allergen declarations, meal-pattern contribution settings, Direct Menu Item configuration, and POS pricing. A fully configured food item can be placed directly on a menu as a standalone item and appear on the POS touchscreen with per-eligibility-group pricing.

This guide covers the complete item-creation workflow as taught in the May 2026 Brunswick County Schools onboarding session. Downstream modules — Inventory, Menu Planning, POS, Family Hub — are referenced where item fields connect to them but are not covered in depth here.

2. Roles and Permissions

Item creation and editing require a District Administrator role or equivalent. Menu Planners (Nutritionists) can access and edit the Nutrient Info, Allergen, and Contribution sections and can configure Direct Menu Items.

The precise permission matrix — which roles can create vs. edit vs. publish items — was not covered in the training session. [TO VERIFY: full role-permission table for Item Management create/edit/publish actions]

Custom Allergen configuration (adding non-standard allergens to the district list) requires access to Item Management > Configuration > Item Configuration and is typically a District Administrator action.

3. Prerequisites

  • Active credentials: Each staff member needs valid SchoolCafé 2.0 login credentials before accessing Item Management.
  • Item categories configured: Item Category, Storage Category, and Valuation Group must exist in the system before they can be selected on an item card. These fields are required only when the Inventory module is enabled; if your district is not using Inventory they are technically optional, though filling them in from the start avoids retroactive data entry if Inventory is activated later.
  • Menu Item Categories configured: The Menu Item Category field (e.g., "Entree") must exist in configuration before a Direct Menu Item can be classified.
  • Custom allergens pre-built (if needed): Any non-standard allergen (e.g., Strawberry) must be added to the district's Custom Allergen list in Item Configuration before it can be assigned to an item. See workflow 4.9 below.

4. Full Workflows

4.1 Creating a Non-Food Item — End to End

Non-food items are any supplies your district purchases but does not serve as a meal component: cups, trays, napkins, gloves, utensils.

  1. Navigate to Item Management > Items.
  2. Click + NEW. The Item Details page opens with the Item card.
  3. Complete the Item card:
    • Item Description — required. Use the item name plus the size or distinguishing detail (e.g., Dixie Cup, 12 oz). [TO VERIFY: 250-character limit stated by presenter at [02:01]; not confirmed in Jira AC]
    • Item Category, Storage Category, Valuation Group — required if Inventory is enabled; optional otherwise.
    • Brand Name, Manufacturer, Product Code, Tags — optional, but Brand Name and Manufacturer are useful search filters when the catalog grows large.
  4. Click SAVE. The page refreshes and the Units card appears.
  5. On the Units card, click NON-FOOD.
  6. Click + PACK to add the first (highest) pack size:
    • Description — required (e.g., Box).
    • Weight — optional for non-food items; enter only if needed for shipping or receiving documentation.
    Click the checkmark icon to save the row.
  7. To add a lower-tier pack size (e.g., Sleeve inside Box), click + PACK again. When prompted Higher or Lower?, select LOWER. Fill in Description (e.g., Sleeve) and Sub-Qty (e.g., 20 = 20 sleeves per box). Click the checkmark.
  8. Repeat for a third tier if needed (e.g., Cup, Sub-Qty 25). After the third tier is added the + PACK button becomes unavailable — three tiers is the system maximum.
  9. The system automatically assigns the Base Unit (BU badge) to the lowest-tier pack size. Higher tiers display P (Pack) badges. The Base Unit is what staff count during inventory.
  10. Address Missing Costing only after all pack sizes are added. Click the Missing Costing alert to expand it, then click the Edit (pencil) icon. Enter the cost for the one unit whose price you know (e.g., $80 for Box). The fields for the other tiers are disabled while you type; after saving, the system calculates them from the sub-quantities automatically.
  11. Click SAVE. The Missing Costing alert disappears; the green Inventory Ready flag appears and the Publish toggle is enabled.
  12. To update costs later, click the Manage Costing button (which replaces the Missing Costing alert) at any time. The system recalculates all tiers.
Critical: The Base Unit designation and the Whole Numbers/Decimals count setting (see 4.3) become locked once the item is used in any Inventory transaction — count, receipt, or adjustment. Configure both correctly before the item enters active Inventory use.

4.2 Adding GTIN/UPC Barcodes

Barcodes are optional but strongly recommended if your district uses barcode scanners for receiving.

  1. On the Units card, click GTIN/UPC. Each pack-size row now shows a GTIN field.
  2. Enter the barcode number for each applicable tier.
  3. Click SAVE. Barcode icons appear in the Identifiers column confirming entry.

4.3 Setting Whole Numbers vs. Decimals for Inventory Counts

Click the green Inventory Ready flag to expand it. The banner shows whether counts use Whole Numbers or Decimals. Click the WHOLE NUMBERS chip to toggle between modes. For most non-food items whole numbers is appropriate.

4.4 Uploading Images and Documents

The Images & Docs card is identical for both food and non-food items.

  1. Click the expand icon (arrow) on the Images & Docs card to open the Manage Images & Documents panel.
  2. Drag and drop files into the panel, or click SELECT FILES. Supported formats include JPEG, PNG, PDF, Word documents, and spreadsheets. [TO VERIFY: 5 MB per-file limit stated by presenter at [11:02]; not confirmed in Jira AC]
  3. For each uploaded image, select a value from the Use as dropdown. The Preferred Inventory Image setting causes that image to display when browsing items in Inventory. Multiple images can be uploaded with different Use as designations.
  4. Add an optional comment, then click SAVE. The image or document appears on the Item Details page.

4.5 Creating a Food Item — Units Card

Food items begin identically to non-food (Item card → SAVE), but the Units card requires both serving sizes and pack sizes. Serving sizes are the basis for ingredient usage in recipes and menu planning. Pack sizes are the basis for Inventory ordering and receiving. These are two separate purposes within one combined card.

  1. After saving the Item card, click FOOD on the Units card.
  2. Choose fluid type:
    • NON-FLUID (Grams) — for items whose nutrition label weight is in grams (most solid foods, including yogurt).
    • FLUID (Milliliters) — for items measured by volume (juice, milk cartons). The system uses volume units for serving sizes when Fluid is selected.
  3. Click + SERVING to add the serving size. Fill in Amount (typically 1), Description (e.g., Each), and Weight with the unit from the nutrition label (e.g., 4 Ounce). The system converts to grams automatically. Click the checkmark.
  4. Click + PACK to add pack sizes following the same Higher/Lower hierarchy as non-food.
  5. Food items display an Inventory Ready alert stating "Servings and Pack Units must be linked for Inventory Eligibility." This is distinct from the Missing Costing alert.
  6. To link servings and packs: click the Inventory Ready alert → Edit icon. Enter the Sub-Qty on the serving size row — this answers "how many servings are in one [lowest pack]?" (e.g., 16 yogurts per carton). Click SAVE. The alert clears.
  7. Address Missing Costing using the same one-unit entry process as non-food.
  8. The system defaults the Base Unit to the lowest pack size. For food items, you have the option to promote the serving size to Base Unit instead (so staff count individual servings during inventory counts rather than cases or cartons):
    1. Click the BASE UNIT chip. The system prompts: "Confirm [lowest pack] as Base Unit, or promote [serving] to lowest Inventory Unit?"
    2. Click the serving size row, then click CONFIRM. The serving size now displays both the Serving identifier and the BU badge — this is a dual unit.
  9. Set Whole Numbers vs. Decimals as needed, then click CONTINUE → SAVE & CLOSE. The Nutrient Info, Menu Info, and Inventory Info tabs become active.
Order of entry: The system allows adding pack sizes before or after serving sizes. The presenter recommends adding the serving size first for a smoother workflow.

4.6 Configuring Nutrient Info

  1. Click the NUTRIENT INFO tab (active only after the Units card is saved).
  2. Click the three-dot menu on the Nutrient Details card → Edit.
  3. Fill in all nutrient fields. Fields marked with an asterisk are required and the card cannot be saved without them.
  4. For any nutrient not present on the product label, check the Missing Value checkbox. This marks the nutrient as (M) in the system, signaling "data not available" rather than "zero." Nutrient analysis reports flag (M) values to alert staff that data is incomplete.
  5. Click SAVE. The system auto-calculates the Per 100 Gram column and Daily Value percentages.
Note: Incomplete nutrient data does not block Inventory use. The Nutrient Info tab stays at Incomplete status until all required fields are filled. Nutrient completeness is required only for menu planning and nutrient analysis.

4.7 Configuring Ingredient Info

  1. On the Nutrient Info tab, click the three-dot menu on the Ingredient Info card → Edit.
  2. Edible Yield Factor — percentage of the item that is edible after preparation (e.g., 100% for pre-packaged yogurt; lower for items with inedible portions like banana peels).
  3. Sub Ingredients — component ingredients when this item is used inside a recipe.
  4. Standard Recipe Directions — instructions that auto-populate on any multi-item recipe that uses this item as an ingredient (e.g., "Thaw for 30 minutes before serving").
  5. Locally Grown checkbox — for compliance tracking of locally sourced items.
  6. Buy American checkbox — check when the item is domestically produced. When checked, no exemption letter upload is available (checking the box affirms domestic origin). When unchecked, an exemption letter can be uploaded via the upload icon. [TO VERIFY: exact UI behavior of Buy American toggle and exemption letter upload control not confirmed in Jira AC]

4.8 Configuring Contribution Info

  1. Click the three-dot menu on the Contribution Info card → Edit. A full-panel slide opens with meal-pattern contribution fields.
  2. Select which meal-pattern component(s) the item credits toward (e.g., Meat/Meat Alternates, Grains, Vegetables, Fluid Milk). Enter the applicable credit amounts.
  3. Click SAVE.

4.9 Configuring Allergen Info

  1. Click the three-dot menu on the Allergen Info card → Edit. An allergen panel opens.
  2. Read the Allergen Feature Disclaimer at the top of the panel. It states that SchoolCafé is a tool and not a guarantee; the district remains responsible for verifying allergen accuracy.
  3. Standard allergens available: Crustacean Shellfish, Egg, Fish, Gluten, Milk, Peanuts, Sesame, Soy, Tree Nuts, Wheat. For each allergen, mark one status: Contains · Processed in a Facility That Also Processes · May Contain · Free From.
  4. To assign a custom allergen (one not in the standard list): click + ADD CUSTOM ALLERGEN, select the allergen from the dropdown, check the appropriate status, click ADD. Custom allergens must be pre-configured (see 4.10).
  5. Click SAVE.
Bulk allergen import: A staff member asked whether allergens could be imported in bulk for large catalogs. The presenter was unable to confirm this feature exists. [TO VERIFY: bulk allergen import capability; not confirmed in Jira AC or KB]

4.10 Configuring Custom Allergens (Administrative Setup)

This step is completed once, before assigning the custom allergen to any item.

  1. Navigate to Item Management > Configuration > Item Configuration.
  2. From the dropdown, select Custom Allergens. Click APPLY.
  3. Click + NEW. Enter the allergen name (e.g., Strawberry).
  4. Toggle on Show on Family Hub menus if parents should see this allergen in the parent portal.
  5. Click SAVE. The allergen is now selectable in the item allergen panel.

4.11 Creating a Direct Menu Item

A Direct Menu Item (DMI) is a single food item placed on a menu without a recipe — it is served as-is. Yogurt, milk cartons, and pre-packaged items are typical examples.

  1. Click the MENU INFO tab. Initial status is Disabled.
  2. Click CREATE MENU ITEM from the banner on the Units card. The Menu Item card opens.
  3. Required fields:
    • Menu Item Name — appears on menus, production records, and reports (e.g., Strawberry Yogurt).
    • Button Name (English) — text the cashier sees on the POS touchscreen.
    • Menu Item Category — classifies the item (e.g., Entree).
    • Max Days — how many days this item can be carried over on a production record (applies when the Leftover Category is Unspecified or Carryover).
  4. Optional fields:
    • Marketing Name & Description — consumer-facing text displayed in Family Hub (the parent portal).
    • Button Name (Spanish) — for bilingual POS screens.
    • Meal Type — Lunch, Breakfast, etc.; used for Menu Grid filtering.
    • Leftover Category — what happens to unsold items: Return to Stock (back to Inventory) or Discard.
    • Preparation Site Item checkbox — check when the item is prepared at a central kitchen and shipped to serving sites.
    • Show in Summary — filter controlling whether this item appears in the Menu Calendar Report summary view.
    • POS Entree checkbox — check if this item counts toward a reimbursable meal; leave unchecked for à la carte extras.
  5. Toggle on Show on POS to have the item appear on the POS terminal. Additional POS fields appear and the POS Pricing tab becomes active.
  6. Click SAVE. [TO VERIFY: tab label reported to change from "MENU INFO" to "MENU & POS INFO" after save — stated at [27:22]; not confirmed in Jira AC]

Serving Exceptions (optional)

From the Units card in Edit mode, click EXCEPTIONS to restrict which student groups can receive the item at specific meal sites — for example, a 4 oz yogurt approved for elementary and middle but not high school. Assign the Meal Pattern & Serving Group, then configure Not Served for the relevant group.

Adding Menu Serving Sizes (optional)

From the Units card in Edit mode, click + MENU SERVING to add additional serving sizes for the menu item (e.g., both a 4 oz and an 8 oz option). The MI badge in the Identifiers column marks which row is the active menu serving.

4.12 Setting POS Pricing

  1. Click the three-dot menu on the POS Pricing card → Edit.
  2. A grid shows student eligibility types (Free, Reduced, Paid, Second Meal) across site types (Elementary, Middle, High School, Alternative). Enter a price in each applicable cell.
  3. To apply a single price across all site types for one eligibility type, enter the amount in the top field and click SET ALL PRICES.
  4. The Allow Sale toggle enables automatically when a price is entered. Manually disable it to prevent sales to a specific group at a specific site.
  5. Click the ADULTS tab to set pricing for staff, visitors, and program adults using the same grid approach.
  6. Check Taxable if sales tax applies (most reimbursable school meals are tax-exempt; certain à la carte items may not be). [TO VERIFY: sales tax rate configuration mechanism in SchoolCafé 2.0 — presenter was uncertain at [31:20]]
  7. Click SAVE.

4.13 Configuring HACCP Steps

  1. On the item's Steps card, click ENABLE RECIPE.
  2. Click the three-dot menu → Edit. You are in Step 0 (configuration).
  3. Select the HACCP Process:
    • No Cook — items served without cooking (pre-packaged, ready-to-eat).
    • Same Day Service — cooked and served the same day.
    • Complex Food — multi-day preparation or complex cooking processes.
  4. Enter Pre-Preparation Instructions, Prep Time, and Cook Time as needed.
  5. Click SAVE.
  6. To add recipe steps, click + ADD and type the direction text. To add a Control Point (e.g., Cold Hold), click the dropdown arrow on + ADD → Control Point → select the type. The system auto-fills the critical limit and corrective action (both editable). [TO VERIFY: Cold Hold critical limit auto-fill value ≤42°F stated at [29:29]; not confirmed in Jira AC]
  7. Reorder steps by dragging the handle on the left side of each row. Reordering requires a save.

6. Reports and Outputs

  • Nutrient Analysis Reports: Driven by the Nutrient Info tab. Nutrients marked (M) (Missing Value) are flagged as data-incomplete in analysis output, distinguishing genuine zeroes from unknown values.
  • Menu Calendar Report: The Show in Summary field on the Menu Item card controls whether an item appears in this report's summary view.
  • Production Records: Display the Menu Item Name. The Max Days field governs how many days an item can be carried over on a production record.
  • Family Hub (parent portal): Displays the Marketing Name & Description configured on the Menu Item card. Custom allergens display in Family Hub when the Show on Family Hub menus toggle is enabled for that allergen.
  • POS Touchscreen: Displays Button Name (English) and Button Name (Spanish) to cashiers.
  • Inventory Info Tab (on item record): Once an item is published to Inventory and linked to a vendor contract, this tab displays the active contract details: vendor name, vendor item number, unit, lead time, min/max order quantities, and current contract price. A shortcut icon opens the Contract Items page in the Inventory module directly.

7. Exceptions

  • Three-tier pack size maximum: The system supports a maximum of three pack-size tiers (e.g., Case → Carton → Each). The + PACK button becomes unavailable after the third tier is added. If your district receives in a fourth tier (e.g., Pallet), enter quantities in terms of the highest configured tier at receiving time in Inventory.
  • Base Unit and count mode lock after first Inventory use: The Base Unit designation and the Whole Numbers/Decimals setting are locked once the item appears in any Inventory table (count, receipt, or adjustment). Correct these before the item enters active use.
  • Unlinked servings and packs → Inventory Ineligible: A food item with serving sizes and pack sizes that are not linked via sub-quantity is flagged as Inventory Ineligible. It can still be used in menu planning but cannot participate in full Inventory workflows until the link is established.
  • Required nutrient fields block nutrient save only: Unfilled asterisk-marked nutrient fields prevent saving the Nutrient Details card, but they do not block saving the item itself or using it in Inventory. Nutrient completeness is enforced only at menu planning and analysis time.
  • Non-food items cannot have serving sizes: The Food/Non-Food selection on the Units card controls whether serving size entry is available. Selecting Non-Food disables serving size functionality entirely.
  • Sub-quantity reflects the standard specification, not actual received quantities: Enter the manufacturer's spec (what appears on the spec sheet) as sub-quantity. If a vendor occasionally ships short, handle the discrepancy during receiving in Inventory rather than changing the item record.
  • Custom allergen must be pre-configured: The custom allergen dropdown on the item's allergen card only shows allergens already added in Item Configuration. Attempting to add an unlisted custom allergen directly from the item is not supported.

8. Troubleshooting

SymptomLikely CauseFix
Missing Costing alert persists after entering a price Cost was entered before all pack sizes were added, or the save was not completed. Ensure all pack-size tiers are finalized first. Click the Missing Costing alert → Edit → enter cost → SAVE.
Nutrient Info, Menu Info, and Inventory Info tabs are grayed out The Units card has not been fully saved. Tabs are locked until the Units card is complete and closed. Finish all Units card steps (costing, linking, base unit confirmation) and click SAVE & CLOSE.
Food item shows Inventory Ready alert (yellow) after adding packs Serving sizes and pack sizes are both present but no sub-quantity links them. Click the Inventory Ready alert → Edit icon → enter Sub-Qty on the serving size row → SAVE.
Custom allergen does not appear in the item allergen dropdown The allergen has not been created in Item Configuration. Go to Item Management → Configuration → Item Configuration → Custom Allergens → + NEW. Create the allergen, then return to the item.
+ PACK button is grayed out Three pack-size tiers already exist — the system maximum. Use the three tiers that best match day-to-day operations. Handle higher-level units (e.g., pallets) at receiving in Inventory.
Base Unit or Whole Numbers/Decimals setting cannot be changed The item has been used in an Inventory transaction, locking those settings. Contact your Inventory administrator. Changing a locked base unit typically requires a data correction process.
