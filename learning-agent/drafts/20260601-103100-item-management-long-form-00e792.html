Item Management: Creating Food and Non-Food Items

Module: Item Management  |  Template: Long-form Guide  |  Source recording: REC-2026-0515-IM-001 (May 15, 2026)  |  Presenter: Sarah Chen, Implementation Specialist

1. Overview

Item Management is the central catalog for every product a district uses in SchoolCafé — from yogurt cups served at lunch to the paper napkins that line trays. Every other module (Inventory, Menu Planning, Point of Sale, Production Records) reads from the item records built here. Configuring items correctly at the start eliminates cascading problems in ordering, nutritional analysis, and POS operations. Two item types exist: non-food items (supplies — cups, trays, gloves, napkins) and food items (anything served to students). Food items carry substantially more configuration — nutrients, allergens, meal-pattern contributions, menu placement, and POS pricing — while non-food items are simpler and make the best starting point for new staff. This guide covers creating both item types from scratch, including every tab and card on the Item Details page.

3. Prerequisites

  • Item Category, Storage Category, and Valuation Group must be configured in district settings before creating items. These three fields are required when Inventory is enabled; without them, the item cannot achieve Inventory readiness.
  • Custom Allergens (any allergens not in the standard list) must be created in Item Management > Configuration > Item Configuration > Custom Allergens before they can be applied to item records. See Section 4.7.
  • Login credentials must be provisioned by the Cybersoft implementation team before staff can access the module.
  • Related modules: Inventory module setup affects costing requirements (Valuation Method on the Valuation Group determines whether Fair Market Value / costing is required for Inventory eligibility).

4. Full Workflows

4.1 Navigation to the Item List

From the SchoolCafé 2.0 main menu, navigate to Item Management > Items. The list page shows all items in the district's catalog. For a new district, this list will initially be empty. Use the + NEW button at the top of the page to begin creating an item.

4.2 Creating a Non-Food Item

4.2.1 Step 1 — Item Card

Clicking + NEW opens the Item Details page at the Item Card. This card is identical for food and non-food items. Fill in the following fields:

  • Item Description — Required. The item name displayed throughout the system (e.g., Dixie Cup, 12 oz). Include item name and size for clarity. [TO VERIFY: Presenter stated a 250-character limit on this field; no AC found to confirm this limit.]
  • Item Category, Storage Category, Valuation Group — Required when Inventory is enabled; strongly recommended even when Inventory is not yet active, to avoid retroactive data entry.
  • Tags, Brand Name, Manufacturer, Product Code — Optional. Brand Name and Manufacturer are useful for filtering items later (e.g., find all Dixie products).

Click SAVE. The page refreshes and the Units Card becomes active.

4.2.2 Step 2 — Units Card (Non-Food)

The Units Card is where you configure pack sizes. For non-food items, the card presents the Food / Non-Food attribute question. Select NON-FOOD. Selecting Non-Food disables the Fluid/Non-Fluid question and the Direct Menu Item question, which do not apply to supplies.

Adding pack sizes: Click + PACK to insert the first pack size row. Fill in:

  • Description — Required. The name of this pack level (e.g., Box).
  • Weight — Optional for non-food items. Enter if needed for shipping or receiving reference; it does not affect inventory counts.

Click the checkmark to save the row. A Missing Costing alert and an Inventory Info tab with Incomplete status will appear — both are expected at this point. Do not address costing until all pack sizes are added.

Adding lower-tier pack sizes (multi-tier structure): The system supports up to three pack size tiers. Click + PACK again. The system asks whether the new unit is Higher (a larger containing unit) or Lower (a smaller sub-unit). Select LOWER to add a sub-unit beneath the existing pack. Fill in Description and Sub-Qty (the number of this unit contained in the one above it). Example three-tier structure for Dixie cups:

TierDescriptionSub-QtyMeaning
1 (highest)Box—Top-level purchasing unit
2Sleeve2020 sleeves per box
3 (lowest / Base Unit)Cup2525 cups per sleeve

Once three pack sizes are added, the + PACK button becomes inactive — three tiers is the system maximum.

Unit Identifiers: Each row displays colored badge identifiers in the Identifiers column. P badges denote Pack Size units. The BU badge marks the Base Unit — the system automatically assigns the lowest-tier pack size as the Base Unit. The Base Unit is the unit used for inventory counts.

GTIN / UPC barcodes: Click the GTIN/UPC button to expand barcode fields on each pack size row. Enter the barcode number for each tier used with a scanner (optional). Barcode icons appear in the Identifiers column after saving.

Whole Numbers vs. Decimals: Expand the Inventory Ready flag banner. By default, the item is counted in whole numbers. To allow fractional inventory counts, click the WHOLE NUMBERS chip and switch to DECIMALS. For most non-food items, whole numbers is appropriate.

4.2.3 Step 3 — Costing

After adding all pack sizes, address the Missing Costing alert. Click the alert to expand it, then click the Edit (pencil) icon. Enter a cost for one pack size — typically the unit you actually purchase. The fields for the other pack sizes are disabled while you type; the system calculates their costs automatically using the Sub-Qty conversions. Example: entering $80 for the Box automatically calculates Sleeve = $4.00 (80 ÷ 20) and Cup = $0.16 (4.00 ÷ 25). Click SAVE. The Missing Costing alert is replaced by a green Inventory Ready flag, and the system enables the Publish toggle.

Costing can be updated at any time using the Manage Costing button that replaces the Missing Costing alert after the first save. All tier costs recalculate automatically.

4.2.4 Step 4 — Images and Documents

Click the expand icon on the Images & Docs card to open the Manage Images & Documents panel. Drag and drop files or click SELECT FILES to upload. Supported types: JPEG, PNG, PDF, Word documents, spreadsheets. [TO VERIFY: Presenter stated a 5 MB maximum file size per file; no AC found to confirm this limit.] After uploading, select an assignment from the Use as dropdown (e.g., Preferred Inventory Image). Multiple images can be uploaded and assigned different uses. Add comments as needed. Click SAVE.

4.3 Creating a Food Item — Item Card and Units Card

4.3.1 Item Card

Same process as non-food (Section 4.2.1). For a food item, choose appropriate values: Item Category (e.g., Entrees), Storage Category (e.g., Cooler for refrigerated items), Valuation Group (e.g., Non-Commodity). Include Brand, Manufacturer, and Product Code for searchability. Click SAVE.

4.3.2 Units Card Initial Questions

On the Units Card, answer the initial attribute questions:

  1. Food or Non-Food? — Select FOOD.
  2. Non-Fluid or Fluid? — Select NON-FLUID if the nutrition label weight is in grams (e.g., yogurt, a grain item). Select FLUID if the item is measured in milliliters (e.g., juice, milk).
  3. Number of countable inventory units (1, 2, 3, or None/Decide Later) and Direct Menu Item (Yes/No) — Additional configuration questions introduced in the Units Card Overhaul. Default is None/Decide Later and No respectively. [TO VERIFY: Sarah did not demonstrate these two additional questions in the training session; they may reflect a later UI update under NXT-45562.]

4.3.3 Adding Serving Sizes

Food items require at least one serving size. Serving sizes are the basis for ingredient usage in recipes and menu planning. Click + SERVING. Fill in:

  • Amount — Typically 1 (e.g., 1 Each). When Amount = 1, the serving size is eligible to become a Pack Size Base Unit (see Section 4.3.6).
  • Description — Free text (e.g., Each).
  • Weight — The weight of one serving in grams (or volume in mL for fluid items). Example: 4 ounces for a 4 oz yogurt cup.

Click the checkmark. The Nutrient Info, Menu Info, and Inventory Info tabs appear at the bottom but remain inactive until the Units Card is saved.

Add serving sizes before pack sizes — the system workflow is smoothest in this order.

4.3.4 Adding Pack Sizes

Pack sizes are the basis for Inventory ordering and receiving. Click + PACK and fill in Description and optionally Weight. Add subsequent tiers using + PACK > LOWER, filling in Sub-Qty for each child tier. The same three-tier maximum applies as for non-food items.

4.3.5 Linking Servings to Packs (Food Items Only)

After adding both serving sizes and pack sizes, the system displays an Inventory Ready alert stating that servings and pack units must be linked for Inventory eligibility. This link tells the system how many serving-size units are contained in the lowest pack tier.

  1. Click the Inventory Ready alert, then the Edit icon.
  2. Enter the Sub-Qty on the serving size row: the number of servings in the lowest pack size tier (e.g., 16 yogurts per Carton).
  3. Click SAVE. The Inventory Ready alert resolves.

4.3.6 Costing (Food Items)

Same process as non-food (Section 4.2.3). Enter cost for the most relevant pack size; the system propagates values through Sub-Qty conversions. Costing is optional but required for Inventory readiness when the Valuation Group uses the Standard Price valuation method.

4.3.7 Base Unit and Inventory Options

After costing, the system confirms the Base Unit — the unit used for inventory counting. By default this is the lowest pack tier. For food items only, the system may offer to promote the serving size to the Base Unit if the serving size is linked via Sub-Qty to the lowest pack tier.

To promote the serving size to Base Unit: click the BASE UNIT chip, select the serving size row, then click CONFIRM. The serving size row then carries both the BU (Base Unit) and MS/serving identifiers — a "dual unit."

Select Whole Numbers or Decimals for inventory counting, then click CONTINUE > SAVE & CLOSE to finalize the Units Card.

⚠ Important: Once the item is used in any Inventory transaction (count, receipt, or adjustment), the Base Unit and Whole Numbers/Decimals settings are locked and cannot be changed. Verify these settings before the item enters active Inventory use.

4.4 Nutrient Info Tab

The Nutrient Info tab is active only for food items after the Units Card is saved. Click the tab, then click the three-dot (⋮) menu on the Nutrient Details card and select Edit. The system pre-populates the serving size at the top, converted to grams (e.g., 4 oz displays as 113.4 g).

Enter values from the product's nutrition label. Fields marked with an asterisk (*) are required before the card can be saved. If a nutrient does not appear on the label, check the Missing Value checkbox rather than leaving the field blank or entering zero. A zero implies the item contains none of that nutrient; Missing Value indicates the data is genuinely unknown. [TO VERIFY: The functional behavior of the Missing Value checkbox (how it is flagged in nutrient analysis reports) is described by the presenter but not found in a TIER 1 AC source.]

After saving, the system auto-calculates and displays the Per 100 Gram column and Daily Value percentages. Missing Value nutrients display an (M) indicator.

The Nutrient Info tab remains at Incomplete status if required fields are not filled in. This does not block other module uses (e.g., Inventory) but is required for nutrient analysis in Menu Planning.

4.5 Ingredient Info Card

Within the Nutrient Info tab, the Ingredient Info card captures additional food-handling details. Click the three-dot menu and select Edit:

  • Edible Yield Factor — Percentage of the item that is edible after prep (e.g., 100% for yogurt, ~85% for a banana once the peel is removed).
  • Sub Ingredients — Listed when this item is used as an ingredient in a recipe.
  • Standard Recipe Directions — Instructions that auto-populate on any multi-item recipe using this item (e.g., "Thaw for 30 minutes before serving").
  • Locally Grown — Checkbox for compliance tracking.
  • Buy American — Checkbox indicating the item is domestically sourced. Checking this box removes the option to upload an exemption letter. Leave unchecked for non-domestic items; the exemption letter upload icon then becomes available.

4.6 Contribution Info Card

The Contribution Info card sets the meal pattern credit the item earns when served. Click the three-dot menu and select Edit to open the Contribution slide-out panel. Enter the appropriate credit amounts by meal component (Meat/Meat Alternates, Grains, Vegetables, Fruits, Fluid Milk, etc.). An item may contribute to multiple components. Example: a 4 oz strawberry yogurt contributes 1 oz equivalent of Meat/Meat Alternates. Click SAVE.

4.7 Allergen Info Card

Click the three-dot menu on the Allergen Info card and select Edit to open the Allergen slide-out panel.

Note: The Allergen Feature Disclaimer at the top of the panel states that SchoolCafé is a data entry tool and that the district retains responsibility for verifying allergen accuracy. Review this disclaimer before using allergen data for parent communication.

The standard allergen list includes: Crustacean Shellfish, Egg, Fish, Gluten, Milk, Peanuts, Sesame, Soy, Tree Nuts, Wheat. For each allergen, select one status:

  • Contains
  • Processed in a Facility That Also Processes
  • May Contain
  • Free From

Custom Allergens

To declare an allergen not in the standard list (e.g., Strawberry), the custom allergen must be created first in Item Configuration, then applied to the item.

Creating a custom allergen:

  1. Navigate to Item Management > Configuration > Item Configuration.
  2. Select Custom Allergens from the dropdown and click APPLY.
  3. Click + NEW. Enter the allergen name (e.g., Strawberry).
  4. Toggle on Show on Family Hub Menus if the allergen should appear in parent-facing menus.
  5. Click SAVE. The allergen is now available in item allergen cards across the district.

Applying a custom allergen to an item: In the item's Allergen slide-out, click + ADD CUSTOM ALLERGEN, select the allergen from the dropdown, check the appropriate status (e.g., Contains), click ADD, then click SAVE.

4.9 Steps Card — HACCP and Recipe Directions

The Steps Card holds HACCP process information and recipe directions. For Direct Menu Items like a yogurt cup, recipe directions may be minimal, but HACCP process assignment is still recommended.

  1. Click ENABLE RECIPE on the Steps Card.
  2. Click the three-dot menu and select Edit. This opens Step 0 — the HACCP configuration step.
  3. Select the HACCP Process:
    • No Cook — Item is served without cooking (e.g., yogurt, fresh fruit)
    • Same Day Service — Cooked and served same day
    • Complex Food — Multi-day or multi-step cooking process
  4. Enter optional Pre-Preparation Instructions, Prep Time, and Cook Time.
  5. Click SAVE.

Adding Control Points: Click the dropdown on + ADD and select Control Point. For No Cook items, select Cold Hold. The system auto-fills the Critical Limit and a default Corrective Action. [TO VERIFY: Presenter stated Cold Hold critical limit is "less than or equal to 42°F"; no AC found to confirm this specific temperature threshold.] Edit as needed, then click SAVE.

Steps and control points support drag-and-drop reordering using the handle on the left of each row. Any reordering requires a save.

4.10 POS Pricing Tab

The POS Pricing tab appears after enabling Show on POS in the Menu Info card. Click the three-dot menu and select Edit.

A pricing grid displays student eligibility types (Free, Reduced, Paid, Second Meal) across site types (Elementary, Middle, High School, Alternative). Enter prices for each applicable cell. The Allow Sale toggle auto-enables when a price is entered but can be manually turned off to block sales to a specific group.

Set All Prices shortcut: [TO VERIFY: Presenter described entering an amount in a top field and clicking "SET ALL PRICES" to populate all cells at once. This shortcut was not found in any reviewed AC — verify with Cybersoft.]

Click the ADULTS tab to configure pricing for staff, visitors, and program adults.

A Taxable checkbox at the top of the tab marks the item for sales tax. Most school meal items are not taxable; à la carte items that are subject to tax require this checkbox. [TO VERIFY: The mechanism for configuring the tax rate in SchoolCafé 2.0 was not covered in the training and no AC was found; verify with the business office and Cybersoft.]

Click SAVE when pricing is complete. A fully configured food item shows green completion indicators on all tabs: Nutrient Info, Menu & POS Info, and Inventory Info.

4.11 Inventory Info Tab

The Inventory Info tab displays the item's active vendor contracts. For a newly created item, this tab is empty. Contract details — vendor name, vendor item number, unit, lead time, min/max order quantities, and current price — populate here once the item is published to Inventory and added to a vendor contract within the Inventory module. A shortcut icon on this tab links directly to the Contract Items page in Inventory.

Pack sizes and GTIN codes can also be managed from this tab without navigating back to the Units Card.

6. Reports and Outputs

Item Management produces the following outputs used by downstream modules:

  • Item List export — A filterable export of all items in the district catalog. Accessible from the Items list page.
  • Food Item Report — Filtered report of food items; includes nutrient and allergen data. Supports grid column filters.
  • Nutrient analysis — Downstream output in Menu Planning, driven by the Nutrient Info tab data configured here. Incomplete nutrient tabs produce partial analysis.
  • Allergen declarations — Consumed by Family Hub (parent-facing menus) and production records. Custom allergens can be toggled to display or hide on Family Hub.
  • POS display — Menu Item Name and Button Name fields appear on the cashier POS touchscreen for items with Show on POS enabled.

7. Exceptions

  • Maximum three pack size tiers. The system allows 1, 2, or 3 countable inventory units — no more. When a fourth level exists (e.g., Pallet), configure only the three tiers most relevant to day-to-day operations and handle the omitted tier during receiving in Inventory.
  • Base Unit lock after Inventory use. Once an item has been used in any Inventory transaction (count, receipt, or adjustment), the Base Unit and Whole Numbers/Decimals settings are permanently locked. If the wrong settings were saved, contact Cybersoft support — correction requires manual intervention.
  • Serving size and pack size are the same unit (food items). When one physical unit functions as both the serving size and the inventory count unit (e.g., individual milk cartons), add the serving size first, then complete the linking step. The system provides the option to promote the serving size to the Base Unit, making it a dual unit.
  • Custom allergen must exist before it can be applied. Custom allergens must be created in Item Configuration before they appear in the item's allergen card. Attempting to add a custom allergen that has not been configured in Item Management > Configuration > Item Configuration will result in an empty dropdown.
  • Sub-Qty must reflect the standard pack quantity. If a vendor occasionally ships fewer units than the standard pack (short-shipping), do not adjust Sub-Qty. Sub-Qty represents the standard specification. Discrepancies are handled during Inventory receiving.
