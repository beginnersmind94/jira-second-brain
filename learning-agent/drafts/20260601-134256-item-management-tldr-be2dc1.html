Item Management: Learning Guide

What This Module Does

Item Management is the central configuration hub where all food and non-food items are created and fully built out — covering basic descriptions and pack sizes through nutrients, allergens, meal-pattern contributions, POS pricing, and HACCP steps — before those items become available in Inventory, Menu Planning, or POS.

Key Features

Feature What it does
Units Card A single card combining pack size and serving size management. Food/Non-Food and Fluid/Non-Fluid flags in the header control which unit types and measure labels are enabled.
Pack Sizes Users can add pack size units to the Units grid. Sub-Qty fields link pack sizes to each other; the system highlights required Sub-Qty fields automatically.
Serving Sizes Independent serving sizes can be added for food items; serving sizes are disabled when the item is marked Non-Food. A serving with an amount of 1 becomes eligible to link to a pack size as a base unit.
Inventory Readiness & Base Unit The Inventory flag is triggered when pack sizes are added. A completion step confirms the Base Unit before the Units card can be saved.
Global Item Info Brand, Manufacturer, and Product Code are optional fields available on all items.
Nutrient Info Nutrition information fields mirror those available on Menu Items.
Allergen Info Allergen information fields mirror those available on Menu Items.
Contribution Info Contribution information fields mirror those available on Menu Items.
Menu Info & Marketing Fields Direct Menu Items support a Marketing Name and Marketing Description. Menu Item status controls eligibility for Menu Planning.
Images & Docs One image can be uploaded per item; location/file selection is provided and the field is not mandatory.
Item & Sub-Type Deactivation A confirmation prompt is shown before deactivating; the system checks all sub-types and assigns Pending Deactivation status to any that cannot be made inactive immediately.

Common Workflows

  • Create a non-food item: Fill the Item card, then select Non-Food in the Units card to disable serving sizes. Add up to three pack tiers using Higher/Lower positioning and Sub-Qty to link them. Enter cost on one unit and let the system calculate the rest; optionally attach GTIN codes and images.
  • Create a food item: Fill the Item card, then select Food and Non-Fluid or Fluid in the Units card. Add serving size(s) first, then add pack tiers and link servings to packs via Sub-Qty — or promote a serving to the Base Unit. Complete costing, then fill Nutrient Info, Allergen Info, and Contribution Info, and enable Menu Item and POS Pricing.
  • Add a custom allergen: Navigate to Item Management → Configuration → Item Configuration, select Custom Allergens, create the allergen, then return to the item's Allergen card to apply it.

Gotchas

  • Item categorizations are only required if Inventory is enabled — configure them at item creation so you don't have to revisit the record once Inventory is activated.
  • The Base Unit can only be converted from a serving size while pack sizes remain unlocked; once the item enters Inventory tables, pack size and Base Unit are locked .
  • For food items, pack sizes must be linked to serving sizes for Inventory eligibility ; when that link is missing, expand the Inventory Ready banner for the specific corrective action .
  • Cost and value fields are only enabled during finish tasks — after all pack sizes are added ; you then select the specific pack size or inventory-able serving on which to enter pricing .
  • Deactivating an item places it in Pending Deactivation if any sub-type cannot be made inactive immediately ; the item becomes fully inactive only once every sub-type is resolved — each must be removed from active recipes, menus, or open orders first .

Where To Go Next

  • Inventory — manage ordering, receiving, and vendor contracts for items built here.
  • Menu Planning — place items and recipes on menus and swap items across menu cycles.
  • POS — configure menu grids using items marked for POS.
  • Recipes — use food items as ingredients and build multi-step recipes with HACCP tracking.
