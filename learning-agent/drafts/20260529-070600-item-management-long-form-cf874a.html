Item Management: Creating Food and Non-Food Items — Complete Reference Guide

Module: Item Management · Platform: SchoolCafé 2.0 · Audience: District Administrators, Nutritionists, Menu Planners · Source: Training recording REC-2026-0515-IM-001, presented by Sarah Chen (Implementation Specialist), May 15, 2026 · Brunswick County Schools onboarding cohort

1. Overview

Item Management is the central item catalog for SchoolCafé 2.0. Every food and non-food item your district uses — from entrées and side dishes to paper trays and gloves — must be created and configured here before it can appear in Inventory, Menu Planning, Production, or Point of Sale. Getting items right at initial setup is critical: errors in an item record propagate downstream into menus, inventory counts, purchasing, and nutrient analysis.

There are two top-level item types: food items (anything served to students) and non-food items (supplies such as cups, trays, napkins, and gloves). Food items are significantly more complex because they carry nutritional data, allergen information, meal-pattern contribution credits, menu configuration, and POS pricing in addition to the pack-size and costing information that both types share.

The workflow for both item types begins with the same Item card, proceeds through the Units card (pack sizes, serving sizes, costing, inventory readiness), and — for food items — continues through the Nutrient Info, Menu Info, and POS Pricing tabs. Non-food items are simpler: they have only pack sizes, costing, optional GTIN barcodes, and optional images or documents.

The outcome of item creation is a fully configured item record that the Inventory module can publish, that Menu Planning can include in menus and recipes, and that the Point of Sale can price and sell. Incomplete items can exist in the system with tab statuses of Incomplete, but will not be eligible for all downstream uses until all required sections are complete.

  • Login credentials: All staff must have active SchoolCafé 2.0 accounts with Item Management access before starting item creation.
  • Item Categories: Your district's item categories (e.g., Entrees, Non-Food Supplies) must be configured before items can be categorized. [TO VERIFY: where Item Categories are configured — the transcript does not specify the setup path]
  • Storage Categories: Dry, Cooler, Freezer, and other storage options must be available in the system. [TO VERIFY: whether these are system defaults or district-configurable]
  • Valuation Groups: Required fields for Inventory. Must be set up before items are created if your district uses the Inventory module.
  • Custom Allergens: Any allergen not in the standard list (e.g., Strawberry) must be created in Item Configuration before it can be assigned to an item. See Section 4.7.

4. Full Workflows

4.1 Creating a Non-Food Item (Step-by-Step)

Non-food items cover any supply the district tracks in Inventory but does not serve as a food item: cups, trays, napkins, gloves, utensils, and similar supplies.

Example used in training: Dixie Cup, 12 oz.

Step 1: Open the new item form

  1. Navigate to Item Management → Items. The list of all items in your system is displayed.
  2. Click the + NEW button.
  3. The Item card opens. This card is identical for food and non-food items.

Step 2: Complete the Item card

  1. Enter the Item Description (required). Use a descriptive format that includes both item name and size, e.g., Dixie Cup, 12 oz. [TO VERIFY: 250-character limit on Item Description — stated by presenter at [02:01] but not confirmed in any ticket AC]
  2. Select Item Category. Required if using Inventory. Example: Non-Food Supplies.
  3. Select Storage Category. Required if using Inventory. Example: Dry.
  4. Select Valuation Group. Required if using Inventory. Example: Non-Food.
  5. Optionally enter Tags, Brand Name, Manufacturer, and Product Code. These optional fields are useful for filtering the item list later. Example: Brand = Dixie, Manufacturer = Georgia-Pacific, Product Code = 7342DX.
  6. Click SAVE. The page refreshes and the Units card appears.

Step 3: Select Non-Food on the Units card

  1. On the Units card, two buttons appear: FOOD and NON-FOOD. Click NON-FOOD.
  2. Non-food items skip the Fluid/Non-Fluid question entirely — that question only applies to food items.

Step 4: Add pack size tiers

Pack sizes represent the purchasing and receiving hierarchy (e.g., Box → Sleeve → Cup). Add all tiers before addressing the costing alert.

  1. Click + PACK to add the first (highest) pack size. Enter a Description (required), e.g., Box. Weight is optional for non-food items. Click the checkmark to save the row.
  2. To add a lower level: click + PACK again. The system asks: Higher or Lower? Select LOWER (this new pack size is smaller than — and contained by — the existing one). Enter Description (Sleeve) and Sub-Qty (how many of this unit are inside the unit above it — e.g., 20 sleeves per box). Click the checkmark.
  3. Repeat for a third tier if needed: click + PACK, select LOWER, enter Description (Cup) and Sub-Qty (25 cups per sleeve). Click the checkmark.
Maximum three pack-size tiers: Once three tiers are configured, the + PACK button is grayed out. [TO VERIFY: three-tier maximum — stated by presenter at [07:05]; NXT-37110 description references second and third pack sizes, implying the constraint, but does not state it in AC verbatim.]

If your operation has four natural levels (e.g., Pallet, Case, Carton, Each), configure the three most relevant for day-to-day operations and handle the highest level during Inventory receiving — entering inbound pallets as however many cases they contain.

Step 5: Review unit identifiers

After saving pack sizes, the Identifiers column shows colored badge icons:

  • P — Pack Size identifier, shown on every pack-size row.
  • BU — Base Unit, shown on the lowest-level pack size. The system automatically assigns the lowest tier as the Base Unit. Inventory counts are performed in Base Unit quantities.

Step 6: Enter costing

  1. A Missing Costing alert appears after pack sizes are saved. Click the alert, then click the Edit (pencil) icon.
  2. Enter the cost for exactly one pack size — typically the unit you purchase (e.g., the Box). While you type in the chosen field, all other tier fields are disabled (grayed out).
  3. Click SAVE. The system automatically calculates the cost for all other tiers by dividing through the Sub-Qty values. Example: Box = $80.00 → Sleeve = $4.00 (80 ÷ 20) → Cup = $0.16 (4.00 ÷ 25).
  4. The Missing Costing alert is replaced by a Manage Costing button. Click this button at any time to update the price; all tiers recalculate on save.

Step 7: Inventory Ready flag

After costing is complete, a green Inventory Ready flag appears and the Publish toggle is automatically enabled. The item is now eligible to be published to the Inventory module.

Step 8: GTIN/UPC barcodes (optional)

  1. Click the GTIN/UPC button on the Units card. Each pack-size row gains a barcode entry field.
  2. Enter the GTIN code for each applicable pack size. Click SAVE.
  3. Saved codes display as barcode icons in the Identifiers column.

Step 9: Whole Numbers vs. Decimals

The Inventory Ready banner indicates whether the item will be counted in Whole Numbers (default) or Decimals. For most non-food items, whole numbers is correct. Click the WHOLE NUMBERS chip to toggle to Decimals when partial-unit counting is needed.

Lock point — configure before first Inventory transaction: The Base Unit and Whole Numbers / Decimals setting can be changed until the item has been used in an Inventory transaction (count, receipt, or adjustment). After first use, these settings lock. [TO VERIFY: NXT-33518 AC notes "TBD when this option is locked" — exact lock trigger not confirmed in AC.]

Step 10: Add images or documents (optional)

See Section 4.11 — Images & Documents. This card is identical for food and non-food items.

The non-food item is now complete.

4.2 Creating a Food Item (Step-by-Step)

Food items require significantly more configuration than non-food items. In addition to pack-size and costing information, food items carry nutritional data, allergen information, meal-pattern contribution credits, and optional menu and POS setup.

Example used in training: Yoplait Strawberry Yogurt, 4 oz.

Step 1: Complete the Item card

Follow the same process as for non-food items (Section 4.1, Steps 1–2). Example values: Description = Yogurt, Strawberry, 4 oz, Category = Entrees, Storage = Cooler, Valuation Group = Non-Commodity, Brand = Yoplait, Manufacturer = GENERAL MILLS SALES INC, Product Code = 208016. Click SAVE.

Step 2: Select Food and choose Fluid type

  1. On the Units card, click FOOD.
  2. An additional question appears — not present for non-food items: "Is this a Non-Fluid Item (Grams) or a Fluid Item (Milliliters)?"
  3. Select NON-FLUID for items whose nutrition label reports weight in grams (most solid and semi-solid foods, including yogurt). Select FLUID for items reported in milliliters (juice, milk, other liquids).

Step 3: Add serving size(s)

Serving sizes are the basis for all ingredient usage in recipes and Menu Planning nutrient analysis. Pack sizes are the basis for all Inventory usage such as ordering and receiving. These two concepts are separate and must be linked.

Best practice — serving before pack: Add serving sizes before pack sizes. The pack-to-serving linking workflow is smoother when serving sizes already exist.
  1. Click + SERVING.
  2. Enter Amount (e.g., 1), Description (e.g., Each), and Weight with unit (e.g., 4 Ounce). Click the checkmark to save the row.
  3. The system automatically converts the weight to grams (e.g., 4 oz displays as 113.400 g on the Nutrient Info tab).
  4. After saving, the Nutrient Info, Menu Info, and Inventory Info tabs appear at the bottom of the page but remain grayed out. They become editable only after the Units card is saved.

Step 4: Add pack sizes

Follow the same process as for non-food items (Section 4.1, Step 4). Food items can also have up to three pack-size tiers. Example: Case → Carton (Sub-Qty: 3).

Step 5: Link serving sizes to pack sizes

For food items, the system requires a link between serving sizes and pack sizes for Inventory eligibility. Before linking, an Inventory Ready alert displays: "Servings and Pack Units must be linked for Inventory Eligibility."

  1. Click the Inventory Ready alert, then the Edit icon.
  2. Enter the Sub-Qty for the serving size row — how many individual servings are in the lowest pack-size unit. Example: 16 yogurts per Carton → Sub-Qty = 16.
  3. Click SAVE. The Inventory Ready linking alert disappears. Servings and packs are now linked.

Step 6: Complete costing

Same process as non-food (Section 4.1, Step 6). Enter cost for one unit; the system calculates the rest. Example: Case = $45.02 → Carton = $15.01 → Each = $0.94.

Step 7: Confirm or promote the Base Unit

By default, the lowest pack-size tier is the Base Unit. For food items, the system offers an additional option: promote the serving size to become the Base Unit. This is useful when you count individual servings during Inventory (e.g., individual yogurt containers, not cartons).

  1. Click the BASE UNIT chip in the Inventory Ready banner. The system shows the current Base Unit (lowest pack tier) and an option to promote the serving size.
  2. Click the serving size row (e.g., Each) to select it as the new Base Unit.
  3. Click CONFIRM. The serving size row now shows both identifiers: the serving identifier and the BU (Base Unit) badge — this is called a dual unit.
Food-item exclusive: The Base Unit promotion option is only available for food items, because non-food items do not have serving sizes.
  1. Choose Whole Numbers or Decimals for Inventory counting (same as non-food, Step 9).
  2. Click SAVE & CLOSE. The Units card is complete. The Nutrient Info, Menu Info, and Inventory Info tabs become active.

4.3 Nutrient Info Tab

The Nutrient Info tab stores the item's nutritional data. This data drives nutrient analysis in Menu Planning. The tab can remain at Incomplete status — the item can still be used for Inventory — but it must be complete for accurate menu-planning nutrient analysis.

Entering Nutrient Details

  1. Click the Nutrient Info tab.
  2. The serving size from the Units card is displayed at the top (e.g., "Serving Size: 1 Each (113.400 g)"). The gram value is the system's automatic conversion of the ounce weight entered during serving-size setup.
  3. Click the three-dot menu (View More Options) on the Nutrient Details card, then Edit.
  4. Enter nutrient values. Fields marked with an asterisk (*) are required to save the card.
  5. If a nutrient is not listed on the product label, check the Missing Value checkbox next to that field — do not leave the field blank or enter zero.
  6. Click SAVE.

After saving, the system automatically calculates the Per 100 Gram column and the percentage Daily Values. You do not need to compute these manually. Nutrients marked as Missing Value display as (M) in the saved view.

Why Missing Value matters — do not use zero as a substitute: Checking Missing Value marks the nutrient as intentionally unknown, which displays as "(M)" in nutrient analysis reports and signals that data is incomplete. Entering zero (or leaving a field blank) implies the item contains none of that nutrient, which will silently misrepresent the item's nutritional profile in menu analysis.

4.5 Contribution Info Card

Contribution Info sets what meal-pattern credit the item contributes when served — the basis for USDA meal-pattern compliance tracking.

  1. Click the three-dot menu on the Contribution Info card, then Edit. A configuration panel opens.
  2. Set the meal-pattern contribution for the item. Example: Yoplait Strawberry Yogurt = 1 oz equivalent Meat/Meat Alternates.
  3. If the item credits multiple meal-pattern components (e.g., both Grain and Meat/Meat Alternates), fill in all applicable fields.
  4. Click SAVE.

4.10 HACCP / Steps Card

The Steps card records HACCP process information, pre-preparation instructions, recipe directions, and critical control points for a Direct Menu Item. It is an optional feature that must be explicitly enabled — it is not present by default when a Menu Item is first created.

Step 1: Enable the Steps card

  1. On the Menu & POS Info tab, locate the Steps card.
  2. Click ENABLE RECIPE.
  3. Click the three-dot menu on the Steps card, then Edit. The first step that appears is Step 0 — the HACCP configuration step.

Step 2: Configure Step 0 — HACCP Process

  1. Select the HACCP Process that applies:
ProcessWhen to use
No CookItem is served cold or at ambient temperature with no cooking step (e.g., yogurt, pre-packaged items).
Same Day ServiceItem is cooked and served the same day.
Complex FoodItem requires extended preparation — cooled, reheated, or held over multiple days.
  1. Enter Pre-Preparation Instructions if needed (e.g., "Remove from cooler 15 minutes before service.").
  2. Enter Prep Time and Cook Time. Both can be zero for No Cook items.
  3. Click SAVE.

Step 3: Add recipe directions and control points

  1. Click + ADD to insert a recipe direction step. Type the instruction text.
  2. To add a HACCP Critical Control Point (CCP): click the dropdown arrow on + ADD and select Control Point.
  3. Select a CCP from the list. Only CCPs configured for the selected HACCP Process are shown. The system pre-fills the Critical Limit and Corrective Action fields with your district's configured defaults; both are editable.   4. Click SAVE.
  5. To reorder steps or control points, grab the drag handle (dot icon) on the left of any row and drag it to the desired position. Save after reordering.

4.12 Inventory Info Tab

The Inventory Info tab is a read-only view of active vendor contracts associated with this item. For a newly created item with no contract history, the tab is empty.

Once the item is published to Inventory and added to a vendor contract (managed inside the Inventory module, not here), the following fields populate:

  • Vendor name
  • Vendor item number
  • Unit
  • Lead time
  • Minimum and maximum order quantities
  • Current price

A shortcut icon on this tab navigates directly to the Contract Items page in the Inventory module.

You can also edit the Units card and manage GTIN codes from this tab using the same process described in Sections 4.1 and 4.2.

6. Reports and Outputs

Item Management is primarily a configuration module; dedicated reporting tools live in the downstream modules that consume item data. Key outputs flowing from item records include:

  • Menu Calendar Report: Only items configured with Show in Summary checked on the Menu Info tab appear in this report.
  • Nutrient Analysis (Menu Planning module): Items with complete Nutrient Info tabs contribute to nutrient analysis generated in Menu Planning. Items with Missing Value nutrients display (M) flags in those reports — indicating intentionally missing data, not a zero value.
  • Family Hub allergen display: Custom allergens configured with the "show on Family Hub menus" toggle appear on the parent-facing menu view.
  • Inventory module outputs: Published items flow into inventory counts, purchase orders, and receiving records in the Inventory module. The Inventory Info tab on the item record provides a read-only view of current contract pricing for reference.
  • POS pricing: Prices configured on the POS Pricing tab are consumed directly by POS terminals for transaction processing.

7. Exceptions

7.1 Maximum three pack-size tiers

The + PACK button is grayed out once three pack-size tiers are configured. [TO VERIFY: three-tier maximum — stated by presenter at [07:05]; not stated verbatim in NXT-37110 or NXT-37441 AC, though the description references second and third pack sizes implying the limit.]

Workaround: Use the three tiers most relevant to day-to-day operations (typically Case, Carton, Each). Handle any higher level (e.g., Pallet) at the time of Inventory receiving by entering the pallet as a multiple of cases.

7.2 Food items — servings and packs must be linked for Inventory eligibility

A food item with serving sizes and pack sizes that are not linked via Sub-Qty is Inventory Ineligible. The system allows the item to exist in this state but it cannot be published for Inventory use until linking is complete.

7.3 Base Unit and Decimals settings lock after first Inventory transaction

Once an item has been used in any Inventory transaction (count, receipt, or adjustment), the Base Unit designation and the Whole Numbers / Decimals setting are locked and cannot be changed through the normal UI. Configure these correctly before publishing the item. [TO VERIFY: NXT-33518 AC notes "TBD when this option is locked" — exact trigger not confirmed in AC.]

7.4 Buy American checkbox and exemption letter are mutually exclusive

Checking the Buy American checkbox removes the exemption letter upload option. To attach an exemption letter, the Buy American checkbox must be unchecked.

7.5 Required nutrient fields block save on the Nutrient Details card — but not the item overall

The Nutrient Details card will not save if any asterisk-marked required fields are empty. However, this only blocks saving of that card — the item itself can still be used for Inventory purposes. The Nutrient Info tab will remain at Incomplete status. The item must have complete Nutrient Info before it can meaningfully contribute to Menu Planning nutrient analysis.

7.6 Sub-Qty reflects the standard pack specification, not actual shipment quantities

The Sub-Qty field must match the standard pack size as specified on the product spec sheet. Short-shipments or overages from vendors are handled as receiving discrepancies in the Inventory module — they do not require changing the item record.

7.7 Enable Recipe button only appears after a Direct Menu Item is created

The Steps card and the ENABLE RECIPE button are not available until a Menu Item has been created (i.e., CREATE MENU ITEM has been clicked and saved).

8. Troubleshooting

Symptom Likely Cause Fix
Nutrient Info and Menu Info tabs are grayed out after adding a serving size. The Units card has not been saved yet. Tabs become active only after a Units card save. Save the Units card first, then navigate to those tabs.
Missing Costing alert persists after entering a price. SAVE was not clicked after entering the cost, or pack sizes were added after the costing step. Ensure all pack sizes are added first, then open Missing Costing → Edit → enter cost for one tier → SAVE.
Inventory Ready alert persists on a food item even after entering a serving size. Serving sizes and pack sizes are not yet linked. The system requires an explicit Sub-Qty link. Open the Inventory Ready alert → Edit → enter Sub-Qty on the serving size row (how many servings fit in the lowest pack unit) → SAVE.
Custom allergen does not appear in the Allergen Info dropdown. The custom allergen was not created in Configuration before attempting to assign it to the item. Navigate to Item Management → Configuration → Item Configuration → Custom Allergens → create the allergen → return to the item.
The Enable Recipe button and Steps card are not visible on the Menu Info tab. The Direct Menu Item has not been created yet; the Steps card is only available after CREATE MENU ITEM is saved. Click CREATE MENU ITEM, fill in all required fields, click SAVE — then the Enable Recipe button becomes available.
Base Unit or Whole / Decimal setting cannot be changed. The item has already been used in an Inventory transaction and the settings are locked. Contact your system administrator. Changing locked settings after Inventory use requires data remediation steps.
Nutrient Details card will not save. One or more required (asterisk-marked) nutrient fields are blank. A blank field is not valid — entering zero implies zero content, and leaving it empty blocks save. Fill in all required fields. If a value is genuinely unknown or not on the label, check the Missing Value checkbox rather than leaving the field empty.
Cost amounts look wrong on lower pack tiers after entering the top-tier price. The system divides the entered cost by each tier's Sub-Qty. If Sub-Qty values are wrong, the calculated costs will be wrong. Verify Sub-Qty values for each pack tier, then re-enter the cost via Manage Costing.
