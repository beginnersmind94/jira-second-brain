Prerequisites

Before you begin creating an item, confirm the following conditions are in place. Skipping any step will block progress in the creation flow.

In-flow prerequisites (enforced by the system)

  1. General info card saved first. The Units card is only editable once the general card has been saved.
  2. Food Item / Fluid Item questions answered. The serving size and pack size grid does not activate until the header-bar questions are completed. Answering No to the Food Item question disables serving sizes entirely.
  3. At least one unit entry added. At minimum, one serving size — or one pack size for inventory-only items — must exist before you can advance to the next step.

Quick reference: question logic

Answer Effect on Units card
Food Item = No Serving sizes disabled
Food Item = Yes → Fluid Item = Yes Serving sizes switch to Volume measurement
Food Item = Yes → Fluid Item = No Weight column used for serving and pack size units

Note: The Units card combines pack size and serving size into a single section. Confirm your answers to the header-bar questions before adding any rows — changing them later may alter available columns.

Workflow: Creating a Non-Food Item

Non-food items use a streamlined Units card that disables serving sizes and focuses on pack size configuration to drive inventory readiness.

Step-by-Step

  1. Save the general info card first. The Units card becomes editable only after the general card is saved.
  2. Open the Units card. This combined card in the item header manages all pack size and serving information in one place.
  3. Answer the Food Item prompt — select No. Serving Sizes are disabled automatically for non-food items.
  4. Add pack sizes. Use the Add Pack Size button to build your pack structure. Link each pack size tier to the next using the Sub-Qty field.
  5. Enter costing. The Cost/Value field is editable on all eligible pack size units.
  6. Confirm the Base Unit. The Decimals preference for the item is set as part of this step.
  7. Publish to Inventory. The publish flag defaults to on when the item reaches inventory readiness.

Inventory Readiness Statuses

Status What it means Corrective action
Inventory Ineligible Pack sizes are missing or not linked to one another. Add and link pack sizes.
Inventory Incomplete Pack sizes are configured but costing or item categorization is missing. Add costing and item categories.
Do Not Publish Item is manually held from Inventory. Enable the publish flag when ready.
Inventory Ready Pack size tiers are complete; the header shows the tier count instead of a named status. No action required.

Important: Once an item is added to Inventory, pack sizes can no longer be added or changed and Sub-Qty fields become read-only.

Workflow: Creating a Food Item

The Units card consolidates pack size and serving size configuration into one place. It becomes editable only after the general information card has been saved.

Step 1 — Answer the header questions

Before the grid is enabled, the card presents two sequential questions. Your answers control which actions and column labels appear next.

QuestionAnswerResult
Food Item?NoServing sizes are disabled
Food Item?YesFluid Item question appears
Fluid Item?NoWeight column label used
Fluid Item?YesWeight/Volume column label used

Step 2 — Add a serving size

Use the Add Serving Size action to open a new row. Each serving size row captures three fields:

  • Amount — when set to 1, the serving becomes potentially inventory-able and may link to a pack size. Any other value treats it as a serving size only.
  • Description — a smart text field that matches against standard measures.
  • Weight / Volume — defines the measurable unit; column header reflects fluid status.

Multiple independent serving sizes can be saved on a single item.

Step 3 — Add pack sizes

Use the Add Pack Size action to define purchasing and inventory units. Serving sizes and pack sizes serve distinct downstream purposes — serving sizes support ingredient and menu planning, while pack sizes drive inventory and ordering.

If pack sizes and serving sizes are not linked via sub-quantity or weight, the item will be marked inventory ineligible. Proper linking triggers inventory readiness.

Workflow: Linking Servings to Pack Sizes and Publishing to Inventory

Before a food item can reach the Inventory module, it must pass through an Inventory Readiness check — a status displayed on the Units card that signals what, if anything, still blocks the item. The system uses pack size configuration as the primary driver of that status.

Inventory Readiness Statuses

Status What it means
Inventory Ineligible Pack sizes are missing or are not properly linked to servings.
Inventory Incomplete Pack sizes are configured but item categorization is missing.
Inventory Ready (tier count shown) All requirements met; the tier count replaces an explicit "Complete" label.
Do Not Publish User has manually held the item from transferring to Inventory.

Step 1 — Link Servings to Pack Sizes

When a food item has both serving sizes and pack sizes, they must be linked or the item remains Inventory Ineligible. There are two ways to create the link:

  • Sub-Qty — enter how many servings fit inside the pack unit. This field is enabled on the Serving Size row when the serving amount qualifies.
  • Weight — enter the weight for the pack unit to link via a weight-based serving instead. Weight is especially important on the lowest pack tier when a food item may be added later.

If neither field is populated, the banner prompts the user to resolve the gap before proceeding.

Step 2 — Enter Costing

Once pack-serving links are in place, the banner shifts to a missing costing alert if costs have not been entered. The corrective message directs staff to add costing for the most relevant unit.

Step 3 — Confirm the Base Unit and Decimal Setting

When all data requirements are met, the item enters an "Inventory Pre-Ready" state that calls the Base Unit confirmation step. This confirmation is treated as part of the publish flow — it does not fire on a routine card save. The Decimals (Whole Numbers vs. Decimals) setting is included in this same confirmation screen.

⚠ Lock warning: Once an item is added to Inventory, sub-quantity becomes non-editable and pack sizes cannot be added or changed. Set the Base Unit and decimal preference carefully before publishing.

Step 4 — Enable the Publish Toggle

After Base Unit confirmation, the Publish flag becomes available. It defaults to on for items that reach this point. Turning it off places the item in Do Not Publish status, holding it from the Inventory module until manually released.

Workflow: Completing Nutrient, Ingredient & Allergen Information

Three tabs collect the core nutritional, contribution, and allergen details a menu planner needs for each item. The system auto-populates data wherever possible; staff review and supplement manually.

Nutrient Information

Nutrition data is automatically populated for the item. Menu planners can view the populated values directly within the item record.

Contribution Information

Contribution data supports manual entry, following the same process as editing contribution information on an existing item. The same validation rules apply in both contexts. When the system has auto-populated this section, the header carries a system-generated label so planners know the values were not entered manually.

Allergen Information

Allergen data is automatically populated in a manner consistent with how allergens are handled during item creation. Menu planners can review allergen information for any item from this tab.

Tab Auto-Populated? Manual Entry? Key Note
Nutrient Info Yes View / review —
Contribution Info Yes Yes — same rules as item edit Header shows system-generated label when auto-filled
Allergen Info Yes View / review —

Common mistake: Assuming auto-populated contribution data is always correct. Check for the system-generated label and verify the values before saving.

Workflow: Adding HACCP Steps and Recipe Directions

The Steps card appears on both Direct Menu Items and more complex food items once the recipe is enabled. Use it to record preparation instructions, HACCP process classification, and individual recipe directions with food-safety control points.

Before You Start

  • Open the item record and navigate to the Steps card.
  • Toggle Enable Recipe to activate the fields below it.

Step-by-Step

  1. Select a HACCP Process. Choose one of the three classifications that matches how the item is prepared.
  2. Enter Pre-Preparation Instructions and fill in Prep Time and Cook Time fields.
  3. Add recipe direction steps. Each step gets its own row; add as many as the recipe requires.
  4. Add a Control Point to any step where a critical food-safety check occurs. When you select a Control Point type, the system auto-fills the Critical Limit and Corrective Action fields.
  5. Reorder steps by dragging and dropping rows into the correct sequence.

HACCP Process Quick Reference

Process Typical Use
No Cook Items served without a cooking step
Same Day Service Items cooked and served the same day
Complex Food Items requiring multi-step temperature control across days

Note: Auto-filled Critical Limit and Corrective Action values (for example, on a Cold Hold control point) are driven by the selected Control Point type — review them before saving to confirm they match your operation's standards.

Reports & Outputs

Item Management controls two key downstream outputs: the nutrition information visible to menu planners, and the Inventory Readiness status that gates whether an item flows into the Inventory module.

Nutrition Information

Nutrition data is automatically populated for items and is visible to menu planners directly within Item Management. Staff with the appropriate role can view nutrition information in the same format available during item creation.

Inventory Readiness Flag

Inventory Readiness is an explicit control over which items become available within the Inventory module. The status is displayed as a header-level indicator on the Units card. Tooltips guide users toward the corrective action required for each status.

Four statuses are maintained:

Status Condition Tooltip Guidance
Do Not Publish Manual hold by user
Inventory Ineligible Pack sizes missing or not linked
Inventory Incomplete Pack sizes configured but categorizations missing
Inventory Complete All data valid; displays pack size tier count instead of a label

Key point: Once an item reaches Inventory Complete and its pack sizes are locked, the tooltip confirms the item is published to Inventory. Items that do not reach this state are not transferable to the Inventory module.

Exceptions & Troubleshooting

Settings That Lock After Inventory Use

Two settings become permanent once an item is added to Inventory — treat them as irreversible decisions, not edits you can revisit later.

  • Sub-quantity and pack sizes: Once an item enters Inventory, sub-quantity is no longer editable and pack sizes cannot be added or changed.
  • Decimals setting: The Decimals setting is confirmed during Base Unit confirmation. Make this selection deliberately — it becomes part of the publish action.

Sub-Quantity: What It Measures

Sub-Qty answers the question: How many [serving] are in [pack unit]? It is the primary link between pack sizes and servings. Items with pack units not linked to servings are flagged Inventory Ineligible — they remain usable in Item Management but cannot flow to the Inventory module.

Inventory Status Quick Reference

StatusMeaningBlocks Inventory?
Inventory Ineligible Pack sizes missing or not linked to servings Yes — "Pack size units must be linked to servings for Inventory eligibility"
Inventory Incomplete Pack sizes configured but item categorizations missing Partial — item categorizations must be provided for Inventory readiness
Do Not Publish User is manually holding the item from Inventory Yes — manual hold

Food Item Workflow Order

For food items, answer the Food Item? and Fluid Item? questions first — the grid does not enable Serving Size actions until these are set. Configuring pack sizes before serving sizes can create linking gaps that require rework.
