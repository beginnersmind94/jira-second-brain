Item Management: Learning Guide

Purpose

Item Management is the foundational module of SchoolCafé 2.0. Every other operational area — Inventory, menus, recipes, and POS — pulls from the item records configured here. Getting items right at the source is not optional; it is a prerequisite for accurate downstream behavior.

The consequence of skipping or rushing this setup is cascading errors: a misconfigured item does not fail quietly in one place — it surfaces as incorrect counts in Inventory, wrong prices at POS, and broken recipe costs simultaneously.

In short: fix it here once, or troubleshoot it everywhere.

Who This Is For

This guide is written for three groups:

  • District nutritionists — staff responsible for maintaining nutritional accuracy and compliance across the item catalog
  • Food-service managers — site-level staff who work with items day-to-day and need to locate, review, or update catalog entries
  • Implementation staff — personnel onboarding a new district or performing initial catalog setup

If you manage, audit, or configure items in the catalog, this guide is for you. Staff in adjacent roles who only view items as part of a larger workflow should refer to their module-specific guide instead.

Before You Start

Confirm the following before working with items in this module:

  • Login access — you must have an active account with Item Management permissions assigned by your administrator.

Three fields must be configured on every item before it can be published to Inventory:

FieldWhy it matters
Item CategoryControls classification and downstream routing
Storage CategoryDetermines storage rules applied at publish
Valuation GroupDrives costing and financial reporting

Note: Set these fields even if Inventory is not yet active in your environment — doing so avoids rework when the module goes live.

Workflow: Create a Non-Food Item

For supplies and disposables, answer Food Item? No in the Units card header — this disables Serving Sizes and keeps the workflow focused on pack sizes only.

  1. Save the general info card first. The Units card becomes editable only after the general card is saved.
  2. Build the pack hierarchy. Use the Units grid to add pack size units. Sub-Qty is not enabled on the first pack size added. For each additional tier, Sub-Qty links the new pack size to the existing one.
  3. Enter cost. Cost/Value becomes editable during finish tasks. Select the specific pack size unit on which to enter pricing; linked units are calculated from that entry.

Adding at least one pack size triggers the Inventory Readiness flag for the item.

Workflow: Create a Food Item

Food items extend the standard item workflow by adding classification questions and serving sizes inside a combined Units card.

Steps

  1. Answer Food Item? — if Yes, answer Fluid Item? before the grid enables.
  2. The grid loads with both Add Serving Size and Add Pack Size actions.
  3. Add a serving size — provide Amount, Description, and Weight (or Weight/Volume for fluid items).
  4. If Amount equals 1, the serving becomes eligible to link to a pack via Sub-Qty.
  5. Add pack size(s); use Sub-Qty to link pack sizes together and to the serving.
  6. The Inventory Ready banner surfaces exactly what is missing and guides you to the corrective field.
  7. Enter costing on eligible units to finish.
Key serving-size fields
FieldPurpose
AmountValue of 1 enables Sub-Qty linking and pack-size promotion.
DescriptionSmart text field matched against common measures.
Weight / VolumeColumn label switches to Weight/Volume for fluid items.
Sub-QtyLinks a serving to a pack base unit to satisfy Inventory Readiness.

Workflow: Complete Nutrient, Allergen, and Contribution Info

Applies to food items only. This workflow spans tabs for nutrient data, allergen assignments, and meal-pattern contribution credits.

Allergens

Menu planners require allergen information visible on each item. Allergen data is automatically populated during item creation and editing. The full allergen panel is accessible directly from the item record.

Workflow: Set Up a Direct Menu Item and POS Pricing

1 — Create the item

  1. Go to the Menu Items landing page and choose Add → Menu Item.
  2. Enter the required fields: Menu Item Name and Category.
  3. Optionally add a Marketing Name for downstream display use.
  4. Save — the system creates the item and assigns its code.

2 — Configure HACCP on the Steps card

A HACCP Process selector appears above the Steps card. Choose a process — in practice almost always No Cook, Same Day Service, or Complex Food. This is the only required selection in this section.

  • Use Add CCP to attach control points to individual steps.
  • Switching to a different process when incompatible control points exist triggers a confirmation warning before any points are removed.

Common Mistakes

MistakeWhat happensHow to avoid it
Entering costing before all pack sizes are saved Cost fields are only enabled during finish tasks, so premature entry is blocked or lost. Complete the full pack size grid first, then move to pricing.
Skipping the Sub-Qty link between pack and serving The Inventory Ready banner flags the item as ineligible and directs you to the linking fields. After adding a second pack size, fill Sub-Qty to establish the pack-to-serving connection.
Attempting Sub-Qty on the very first pack size row Sub-Qty is intentionally disabled for the initial pack size. Add a second pack size; Sub-Qty only becomes available then.

The Inventory Readiness banner surfaces specific corrective actions for each unresolved gap — use it as your checklist before saving.
