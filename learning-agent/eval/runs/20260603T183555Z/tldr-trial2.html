Item Management: Learning Guide

What This Module Does

Item Management is the system of record for all food and non-food items. Data entered here flows directly into Inventory, Menu Planning, and POS — errors at this layer propagate downstream, so accuracy in this module is foundational to every dependent workflow.

Key Features

Feature What it does
Food / Non-food classification Item header prompts whether the item is a food item and, if so, whether it is a fluid item, controlling which unit fields are enabled.
Units card (pack size & serving sizes) A single combined card manages all pack size tiers and serving sizes in one place, supporting up to three unit tiers.
Automatic cost & weight scaling Fair Market Value is entered only on the highest unit and calculated for lower tiers; weight is entered only on the lowest unit and calculated upward via sub-quantity.
GTIN per unit Each unit row accepts a GTIN, which must be unique across all items.
Nutrient Details Redesigned card with condensed layout, nested-nutrient auto-zero rules, and updated optional/required field logic.
Allergen Info — standard View mode displays only applicable allergens grouped by form (Contains, Processed, May Contain, Free From) with color-coded icons.
Allergen Info — custom District admins can create and manage custom allergens via the IM configuration area; custom allergens appear after standard allergens.
Contribution Info Contribution information is captured on items and carries through to menu items.
Menu Info / Direct Menu Item Items can be flagged and configured directly as menu items, including serving size, category, recipe links, and marketing fields.
Images & Documents Replaces the previous images-only card; supports typed image roles (Ingredient, Inventory, Menu) and typed documents such as CN Labels and Buy American Exemption Letters.
Inventory Ready / Base Unit flag A guided confirmation step establishes which unit is the smallest inventory-able unit and sets the Base Unit before the Units card is saved.

Common Workflows

  • Non-food item creation: Create the Item card → configure Units/pack sizes (up to three tiers, entering weight on the lowest unit and cost on the highest) → confirm costing → add GTIN if applicable → publish.
  • Food item creation: Create the Item card → open the Units card and answer the Food Item / Fluid Item questions → add serving sizes and pack sizes → link servings to packs via sub-quantity → confirm inventory readiness and base unit → complete costing → enter Nutrient Info → enter Allergen Info → complete Menu Info and POS Pricing.
  • Create a custom allergen: Navigate to the Custom Allergens option in IM Configuration → open the slide-out card, enter a unique description, select the Local data source, and save ; the new allergen appears in the result grid immediately .
  • Add a Direct Menu Item with HACCP steps: Add a new Menu Item from the Menu Items landing page → complete the required fields (name, category, serving size and measure) → attach recipe links if needed → add HACCP steps and control points before saving.

Gotchas

  • Item categorizations are only required if Inventory is enabled — but populate them before first use to avoid blocking an inventory workflow later.
  • Pack size hierarchy is capped at 3 tiers. Enter Fair Market Value on the highest unit only and Weight on the lowest unit only — the system calculates the rest.
  • Pack size configuration locks once the item enters Inventory tables — set the Base Unit correctly the first time.
  • Custom allergens must be created in IM Configuration before they can be assigned to any item.
