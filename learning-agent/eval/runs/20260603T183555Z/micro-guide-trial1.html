Who This Is For

This guide is written for two primary audiences:

  • Implementation specialists setting up Item Management for a district for the first time
  • District nutritionists and administrators who will own ongoing item creation and maintenance after go-live

If you are maintaining an already-configured item catalog or exploring advanced reporting, the long-form guide is a better fit. This micro guide focuses on getting new users to task-completion confidence as quickly as possible.

Before You Start

The Units card only becomes editable after the general info card is saved. Have the following decisions ready before you open the card:

  • Food or Non-Food? The system asks this before enabling the units grid. Answering No disables Serving Sizes entirely.
  • Fluid or Non-Fluid? Only presented if the item is Food. Your answer controls whether the weight column switches to Weight / Volume.
  • Inventory intent? Adding at least one pack size triggers the Inventory flag. Confirm whether this item needs to be inventory-tracked before you begin.

You must configure at least one serving size or pack size unit before proceeding to the next step.

Workflow: Create a Non-Food Item

The Units card appears in the Item creation header alongside the general info card. It becomes editable once the general card is saved.

  1. Answer the Food Item question → No. This disables Serving Sizes and loads the grid with the Add Pack Size action.
  2. Add pack size rows. Click the + Pack button to insert a new row. Enter a Description using the smart text field.
  3. Link tiers with Sub-Qty. Sub-Qty connects pack size units to each other. The first pack size added does not require Sub-Qty.
  4. Enter Cost/Value at finish tasks. Cost/Value is enabled at the end of the create flow. Select the specific pack size unit on which to enter pricing.

Workflow: Build the Units Card for a Food Item

The Units card consolidates pack sizes and serving sizes into one combined grid. It becomes editable only after the general info card is saved.

Header FlagEffect on Grid
Food Item = NoServing sizes disabled
Fluid Item = YesWeight column switches to Weight / Volume
  1. Answer Food and Fluid flags in the card header bar — these must be answered before the grid enables.
  2. Add serving size first. Set Amount = 1 to qualify the serving for Sub-Qty linking and possible Base Unit promotion.
  3. Add pack size and enter Sub-Qty to link the pack to the serving.
  4. Enter Cost / Value on eligible units during the finish-tasks stage.

Key pitfall: Unlinked pack and serving units leave the item without Inventory Readiness — the item will be inventory ineligible until the link is resolved.

Workflow: Complete Nutrient, Allergen, and Contribution Info

After entering core menu item details, complete the three informational tabs before saving. Each mirrors its counterpart on the base Item record.

Tab What to know
Nutrition Information Fields match Item-Nutrition information.
Contribution Information Fields match Item-Contribution information.
Allergen Information Fields match Item-Allergen information. Allergen data populates automatically, mirroring the process used during item creation.

Menu planners can review allergen information directly on the menu item record.

Workflow: Configure Menu Info and POS Pricing

Use the Menu Info tab to promote a food item to a Direct Menu Item. The item type reflects the source of the menu item, not a category.

Common Mistakes

Mistake Why it matters
Entering cost before finishing pack setup Costing fields are enabled only during finish tasks, after pack configuration is complete.
Skipping the serving-to-pack link Unlinked pack units leave the item Inventory Ineligible.
Omitting HACCP Process on food items The HACCP Process selection on the Steps card is the only required field in that section and must not be left blank.
Adding Sub-Qty on the first pack size Sub-Qty is only applicable once a second pack size exists.
