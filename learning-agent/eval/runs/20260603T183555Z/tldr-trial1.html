Item Management: Learning Guide

What This Module Does

Item Management is where staff configure food and non-food items, classifying each as a food item and, if applicable, a fluid item — answers that control which unit types and measurement columns are available on the record. Pack sizes and serving sizes are managed together in a single combined Units section, so that pack-to-serving conversion is handled in one place rather than across separate screens. Adding pack sizes to an item triggers its inventory readiness flag, making correct item setup a prerequisite for downstream inventory use.

Key Features

Feature What it does
Item Card Captures core item identity: description, category, storage category, valuation group, and brand/manufacturer/product code fields.
Units Card — Food/Fluid classification A combined card for pack sizes and serving sizes. Initial prompts ask whether the item is a food item and, if so, whether it is a fluid item, which controls whether serving sizes use weight or volume.
Units Card — Pack sizes (up to 3 tiers) Users can add multiple pack size rows. Sub-Qty links pack size tiers to each other and, in select cases, links a serving size to the base unit.
Units Card — Serving sizes (food items only) Independent serving sizes can be added for food items. Serving sizes are disabled when the item is marked non-food.
Base Unit selection & Inventory Readiness A guided finish step confirms which unit is the smallest inventory-able unit (Base Unit) and validates pack-to-serving links before saving. An Inventory Ready banner in edit mode surfaces corrective actions when pack sizes or links are incomplete.
Whole Numbers vs. Decimals / Amount field A serving size row with an Amount of 1 is eligible to become a base unit or carry a sub-quantity link; any other amount behaves as a standard serving size only.
Nutrient Info tab Nutrition fields mirror the item creation nutrition section; information is automatically populated where available.
Allergen Info Allergen information (standard and custom, Contains/May Contain/Free From) is automatically populated and displayed.
Contribution Info Meal pattern contribution fields are available on both items and direct menu items, matching item creation contribution information.
Menu Info / Direct Menu Item A menu item can be created directly, with fields for name, marketing name/description, category, serving size/measure, and optional recipe links.
POS Pricing grid Pricing is configured per eligible pack or serving unit, including student-by-site-type and adult pricing, with a taxable toggle.
Images & Docs An image can be selected or deleted on a menu item; location/file selection is provided and only one image can be loaded.
Inventory Info tab Once an item is added to Inventory, its pack size configuration is locked; the Base Unit confirmed during Units Card setup cannot be changed from this tab.

Common Workflows

  • Create a non-food item: Fill out the Item Card, then in the Units card select the non-food path — serving sizes are disabled. Add up to three pack tiers, linking them via Sub-Qty. Add all packs before entering cost; enter cost on one unit and the system calculates the rest.
  • Create a food item: Fill out the Item Card, then answer the Food Item and Fluid Item prompts to configure the Units grid. Add serving size first, then pack sizes; link servings to packs using Sub-Qty. Complete costing, then optionally confirm an inventory-able serving as the Base Unit.
  • Complete nutrient and allergen info: Enter all required (asterisk) nutrient fields; use the Missing Value checkbox for any unknown values. Set standard allergens and any custom allergens on the item's Allergen card.
  • Configure a Direct Menu Item: Open the Menu Info tab and create the menu item with name, category, and required fields. Then set student and adult pricing in the POS Pricing tab.
  • Add custom allergens: Navigate to Item Management > Configuration > Item Configuration > Custom Allergens, create the allergen, then assign it on the item's Allergen card.

Gotchas

  • Enter costs only after all pack sizes are configured — cost and value fields are enabled during finish tasks, not mid-setup.
  • Pack size configuration is locked once the item appears in Inventory tables — confirm the Base Unit before first use or the option becomes unavailable.
  • Item categorizations are only required when Inventory is enabled, but skipping them now means back-filling every item later.
  • Sub-Qty is not enabled on the first pack size added; it only applies when a second pack size is introduced to establish the link.
