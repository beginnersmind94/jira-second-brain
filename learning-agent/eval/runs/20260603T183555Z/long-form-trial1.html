Item Management: Learning Guide

Overview

Item Management is the foundation of SchoolCafé 2.0's operations. A consolidated list page surfaces all variations of menu items — including POS items — so staff can see everything as it will be served and sold. The list page has been updated to improve the user experience and to provide additional clarity on the different menu item types available.

Menu Item Types

Every item in the module has a type that identifies its source within the system. Four types are supported:

TypeWhen to use
ItemA standalone purchasable unit
RecipeA prepared combination of ingredients
Variety Menu ItemA grouped offering presented as one choice
POS OnlySold at point of sale but not planned through menus

Food vs. Non-Food Items

During item creation, the system asks whether the item is a food item. This answer drives downstream behavior: if the item is not a food item, serving sizes are disabled. For food items, a further question determines whether the item is a fluid item, which switches serving size measurement to volume.

Units: Pack Size and Serving Size

Pack size and serving size configuration is combined into a single Units card on the item record, so the pack-to-serving conversion is managed in one place. This card becomes editable once the general item information is saved.

Correctly configured items — with accurate type, food classification, and units — are a prerequisite for reliable behavior in menus, inventory, POS, and reporting.

Prerequisites

Before working through item creation, confirm the following are in place.

System dependency

The Units card appears in the item creation header alongside the general info card. It becomes editable only after the general info card has been saved. Attempting to complete unit configuration before saving the general card will not be possible.

Card-level gates

Even once the Units card is editable, the grid does not load immediately. The card first presents item-classification questions in its header bar. The grid — along with its Add Serving Size and Add Pack Size actions — loads only after those questions have been answered.

Quick reference

Prerequisite Why it matters
General info card saved Units card remains read-only until this step is complete
Item-type questions answered Grid actions (serving size, pack size) are gated behind the header-bar prompts

Workflow: Create a Non-Food Item

Use this workflow for supplies such as cups, trays, and napkins — items that have pack sizes but no serving sizes.

Steps

  1. Complete and save the Item card (general info). The Units card becomes editable only after the general card is saved.
  2. Open the Units card. This card combines pack size and serving size management in one place. A header bar displays initial questions before the grid is enabled.
  3. Answer Food Item? → No. Selecting No disables serving sizes for this item. The grid then loads with the Add Pack Size action.
  4. Build the pack structure (up to three tiers). Use the Sub-Qty field to link pack size units to one another. When adding a second or third (lower) pack tier, the system emphasizes entering the weight for the lowest unit.
  5. Enter cost on one unit. The Cost / Value field is editable on eligible pack size units.
  6. Monitor the Inventory Readiness banner. The banner is always visible and updates as you complete required fields. Each banner state provides clear, user-friendly messaging to indicate what remains before publishing.

Key fields at a glance

FieldPurpose
Sub-QtyLinks pack size tiers to each other
Cost / ValueReplaces Fair Market Value; editable on eligible units
Inventory Readiness bannerTracks publishing eligibility in real time

Common mistake: Starting the Units card before saving general info — the card will not be editable until the save is complete.

Workflow: Create a Food Item

Step 1 — Complete the Item Card

Save the general information card first. The Units card sits alongside it in the item header and only becomes editable after the general card is saved.

Step 2 — Answer the Units Card Questions

Before the grid activates, the Units card presents classification questions. Answer in order:

  1. Food Item? — selecting No disables serving sizes entirely.
  2. Fluid Item? (shown only if Food = Yes) — selecting Yes switches serving size measurement to volume.

Step 3 — Build the Units Grid

After the questions, the grid loads with actions to add serving sizes and pack sizes. Key column behavior:

ColumnNotes
AmountAmount = 1 makes the row eligible for Sub-Qty and potential base-unit promotion. Any other value is a serving-only row.
DescriptionSmart text field — matches standard measures.
Weight / VolumeHeader reads Weight for non-fluid items and Weight / Volume for fluid items.
Sub-QtyLinks a serving size row to a pack size unit. For pack sizes, Weight can substitute for Sub-Qty to link via a weight-based serving instead.
Cost / ValueEditable on all eligible units (pack sizes and servings linked with Sub-Qty).

Step 4 — Link Servings to Packs (Inventory Readiness)

Unlinked pack units are permitted but result in the item being Inventory Ineligible. The Inventory Readiness banner is always visible and provides specific messaging for every scenario, including non-inventory states.

Step 5 — Promote to Base Unit (Optional)

A serving size row with Amount = 1 and a valid Sub-Qty or Weight link is eligible to become the pack-size base unit, enabling a dual-unit configuration.

Workflow: Enter Nutrient, Contribution & Allergen Information

The Nutrient Info tab organises nutritional data entry into three areas: Nutrient Details, Ingredient Info, and Allergens. Each area interacts with the serving sizes defined in the Units header card.

Nutrient Details Card

The Nutrient Details card appears at the top of the tab. Before entering values, select the target serving size from the dropdown at the top of the card. If only one serving size exists on the item, no dropdown is shown.

In view mode, the identifier on each serving size reflects the serving size to which nutrients were assigned.

Contribution Info Card

Contribution data follows the same serving-size selection pattern as Nutrient Details — a dropdown appears at the top of the card before data input. Nutrients and Contributions must each be assigned to a specific serving size, but they are not required to use the same one.

Ingredient Info Card

The Ingredient Info card has been reorganised to present its attributes more effectively. For items where ingredient information is not applicable, the card can be hidden. The show/hide toggle is found in the Item Type slideout and is only visible when the slideout is opened while on the Nutrient Info tab.

Allergens

Allergen assignment is managed within the same tab. The allergens section does not interact with serving sizes and requires no serving-size selection step.

Workflow: Configure Menu Info and POS Pricing

Direct Menu Items are one of the recognized menu item types in the module. To create one, use the New+ button on the Menu Items list page, which provides options corresponding directly to each menu item type.

Required Fields

Field Input Type
Menu Item Name Free text — mandatory
Menu Item Category Dropdown — mandatory

Optional Fields

  • Marketing Name — free text, not required.
  • Leftover Category — defaults to Unspecified. If you change it to any other value, Maximum Carryover Days becomes mandatory.
  • Preparation Site Item — checkbox.

Leftover Category dependency: leaving Leftover Category at its default avoids the Maximum Carryover Days requirement. Change it only if this item is intentionally tracked as a leftover.

Saving the Record

Clicking the save control commits the new menu item; the system then displays the assigned menu item code on the detail page. Canceling discards all entries and returns you to the Menu Items landing page.

Workflow: Add HACCP Steps and Control Points

Prerequisites

The Steps & Ingredients tab becomes active only after a Description, Serving Size, and Number of Servings have been saved on the summary card. Recipe steps are housed in a left-aligned Steps card on that tab.

Step 1 — Open Edit Mode and Select a HACCP Process

Trigger the Edit Recipe Steps action on the Steps card. In edit mode, a HACCP Process selection appears within the Steps section.

Choose one of the three standard processes:

  • No Cook
  • Same Day Service
  • Complex Food

Selecting a HACCP Process is the only required action in this section before saving.

Step 2 — Add Control Points (CCPs)

While still in edit mode, an Add CCP action is available at the same level as the Add Step button. Add as many control points as required for the selected process.

Step 3 — Save

The HACCP section supports independent Save and Cancel without affecting other parts of the recipe. Saves are validated against the control points currently added to the steps. Independent editing of the section is also permitted after initial save.

Changing the HACCP Process

The process can be changed after the initial selection. If any existing control points are not associated with the newly selected process, the system warns the user before proceeding. Control points that remain compatible with the new process are retained automatically.

Key Fields & Statuses

Units Card

The Units card appears in the item creation header, consolidating pack size and serving size configuration into one place. It becomes editable only after the general info card is saved.

Setup Questions (Header Bar)

Two questions must be answered before the units grid loads.

  • Food Item? — answering No disables serving sizes entirely.
  • Fluid Item? (appears only when Food = Yes) — answering Yes switches serving sizes to volume-based measurement.

Units Grid Columns

ColumnApplies ToKey Behavior
Amount Serving sizes only A value of 1 makes the serving potentially inventory-able and eligible for pack-size linking.
Description Both Smart text field referencing Common Measures.
Sub-Qty Both (link field) Links pack size units together and to serving sizes; enabled on a serving row only when Amount = 1.
Weight / Volume Both Header reads "Weight" for non-fluid items and "Weight / Volume" for fluid items, because servings use volume while pack sizes use weight.
Cost / Value Pack sizes + linked servings Replaces the former Fair Market Value field; editable on all eligible units.
Identifiers Both Displays unit-level indicators; Nutrients and Contributions badges map to the specific serving size they were assigned to.

Inventory Readiness

Inventory Readiness works as a persistent status banner, not a sequential checklist. A banner is always displayed, including a Non-Inventory state for ineligible or disabled configurations. When pack sizes and serving sizes are not linked via Sub-Qty or Weight, the item is flagged as inventory ineligible. Correct linking triggers inventory readiness.

Exceptions & Troubleshooting

Linking Is Required for Inventory Readiness

Pack sizes and serving sizes must be properly linked before an item can reach Inventory Readiness. Items with pack units that remain unlinked from servings are treated as Inventory Ineligible.

Two fields establish the link — use whichever applies:

  • Sub-Qty — enter on the Serving Size row in valid scenarios. It captures how many serving units are contained in the pack unit.
  • Weight — enter on the Pack Size base unit when Sub-Qty is not used. Weight is optional but can link the pack to a weight-based serving instead.

If neither Sub-Qty nor Weight is provided, the system will prompt the user to resolve the gap before proceeding. Inventory Readiness triggers only when pack-serving links are correctly in place.

Common Linking Mistakes

SituationResolution
Serving added but no Sub-Qty or Weight entered Enter Sub-Qty on the Serving Size row or Weight on the lowest Pack Size unit
Adding a new pack unit below existing pack sizes Enter Weight on the new lowest unit to maintain the link
Item marked as Food but servings not yet added Enter Weight on the lowest unit now so linking is possible when servings are added later

Reading the Inventory Readiness Banner

The banner is always visible — it no longer requires steps to be completed sequentially before appearing. It functions as a live status indicator with a distinct state for every scenario, including a Non-Inventory state for ineligible or disabled items. Each state carries specific, user-friendly messaging to guide next steps.
