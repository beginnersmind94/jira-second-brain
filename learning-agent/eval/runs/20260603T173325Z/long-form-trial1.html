Prerequisites

Before creating or configuring items, confirm the following are in place.

Access

  • Active login credentials with permission to create items in the system.

Sequencing rules you must understand

Item creation follows a strict card order. The Units card — which combines pack size and serving size configuration — only becomes editable after the general information card has been saved. Plan to complete the general card first; you cannot configure units in the same step.

Concepts to know before you start

Concept Why it matters
Food Item / Fluid Item classification Two questions in the Units card header gate which fields and actions load in the grid. Answering incorrectly affects serving size availability and column labels for the entire item.
Inventory Readiness A built-in status concept that tracks whether an item meets the conditions required to be used in Inventory. Adding pack sizes is the primary trigger.
Minimum unit requirement At least one serving size or pack size unit must be configured before you can advance past the Units step.

If you are unsure whether an item will be used in Inventory, clarify this before starting — it determines whether pack sizes are required and affects the entire Units setup path.

Workflow: Creating a Non-Food Item

A supply item is created through the Units card, which sits in the item creation header alongside the general information card. The Units card becomes editable only after the general card has been saved.

Step-by-step

  1. Answer the header questions. The Units card opens with a prompt bar before the grid is enabled. You will be asked whether the item is a food item. Select No — this disables the Serving Sizes section.
  2. Load the grid. After answering, the grid loads and the Add Pack Size action becomes available.
  3. Add your first pack size. Use the + Pack button to insert a new row. Enter a Description using the smart text field (e.g., Case, Bag). Sub-Qty is not enabled for the first pack size added.
  4. Build the tier structure (up to 3 tiers). When adding a second pack size, Sub-Qty is required to link the tiers. Sub-Qty appears on the same row if the new unit is being added below the existing one, or on the child row if being added above it.
  5. Enter Weight where needed. Weight is optional unless this unit will serve as the base unit for linking.
  6. Resolve costing. The Cost/Value field is enabled at the end during finish tasks; select the specific pack size unit on which to enter pricing.
  7. Reach Inventory Ready status. The inventory flag is triggered once pack sizes are added. At least one pack size unit must be configured before moving forward.
Sub-Qty quick reference
ScenarioSub-Qty required?
First pack size onlyNo
Second (or higher) pack size added below existingYes — on the same row
Second pack size added above existingYes — on the child row

Workflow: Creating a Food Item

The Units card sits in the item creation header and consolidates pack size and serving size configuration into one place. It becomes editable after the general info card is saved.

Step 1 — Answer the header questions

Before the grid activates, two questions appear in the card's header bar. Your answers control which rows and columns are available.

AnswerEffect
Food Item = NoServing Sizes are disabled.
Food Item = Yes → Fluid Item = YesServing size measurements switch to Volume; the column header becomes Weight/Volume.
Food Item = Yes → Fluid Item = NoColumn header remains Weight.

Step 2 — Add serving sizes

Use the Add Serving Size action to insert a serving row. Set the Amount field carefully: an Amount of 1 makes the serving potentially inventory-able and eligible for Sub-Qty linking. Any other Amount keeps it a serving-only unit.

Step 3 — Add pack sizes

Use the Add Pack Size action to insert a pack row. Sub-Qty fields are highlighted where required to link pack sizes to each other or to a serving. The first pack size added does not require Sub-Qty; subsequent packs do.

Step 4 — Link servings to packs (Inventory Readiness)

When a serving has Amount = 1, its Sub-Qty field becomes active to accept a link to the base pack unit. Unlinked pack units result in Inventory Ineligible status — the item cannot proceed to inventory workflows until the link is resolved. Inventory Readiness is triggered once pack sizes are added and properly linked.

Step 5 — Complete costing

Cost/Value entry is unlocked during the finish tasks step. You select the specific pack size or inventory-able serving unit on which to record the cost.

Workflow: Configuring Nutrient & Food Information

The Food Info tab is active only for items defined as food items; for non-food items the tab is disabled. On page load all cards display in view-only mode — values are readable but not editable until you explicitly open edit.

Cards on the Food Info Tab

Five cards make up the tab: Serving Measures, Ingredient Info, Nutrient Details, Contribution Info, and Allergen Information. The Ingredient Info card also loads in view mode by default.

Editing Nutrient Details

  • Select the edit action to unlock the card; changes are made in-line.
  • Nutrient values update automatically when a Serving Measure is selected.

Adding Custom Allergens

Custom allergens are managed from the Item Management configuration area, not within the item record itself. A slide-out card collects two mandatory fields:

  • Custom Allergen Description — free text; duplicates are not permitted.
  • Data Source — dropdown; only the Local data source is available for selection.

After saving, the allergen appears in the result grid. Only user-created allergens can have their description edited or be deactivated. Deactivation is blocked while the allergen is in use by any item, recipe, or menu. A confirmation prompt is displayed before the deactivation proceeds.

Workflow: Adding HACCP & Recipe Steps

Prerequisites

The Steps & Ingredients tab only becomes active after a Description, Serving Size, and Number of Servings have been saved on the summary card. Once enabled, a left-aligned card houses all recipe steps.

Step-by-Step: Building Steps with HACCP

  1. Open Edit Steps mode. Use the edit action on the Steps card to enter editing.
  2. Select a HACCP Process. A dedicated selection field appears above the Steps card. The available processes reflect system configuration but are almost always No Cook, Same Day Service, or Complex Food. This selection is required before saving.
  3. Add recipe direction steps. Directions are the primary visible element of each step. Step ingredients appear in a collapsible sub-section beneath each step.
  4. Add Critical Control Points (CCPs). An Add CCP action sits at the same level as the Add Step button. Saves are validated against the CCPs already added.

Changing the HACCP Process After CCPs Are Added

SituationBehaviour
CCP is not compatible with the new process System warns the user that switching will clear incompatible control points.
CCP is still associated with the new process Existing CCP is retained without interruption.

Save Behaviour

The HACCP section supports independent Save and Cancel actions, keeping changes isolated from other parts of the recipe. The process selection can be changed at any time.

Status Indicators

The Food Info tab tracks three completion states:

  • Incomplete — item flagged as food but no serving measures saved.
  • Partial — serving measures saved but no nutrient data entered.
  • Complete — nutrient details saved.

Inventory Readiness is a separate status tracked at the unit level. Adding pack sizes triggers the inventory flag. Items where servings and pack sizes remain unlinked are marked Inventory Ineligible.

Reports & Outputs

Item Management surfaces data in two primary output areas: the Food Info tab for nutrient reporting and the Steps & Ingredients view for recipe reporting.

Nutrient Output (Food Info Tab)

The Food Info tab is enabled only for items defined as food items; the tab is disabled and grayed out otherwise. On page load, all cards — including Serving Measures, Nutrient Details, Ingredient Info, Contribution Info, and Allergen Information — render in view mode. Nutrient values update dynamically whenever a different Serving Measure is selected.

Each food item carries one of three completeness states:

StateCondition
Incomplete Flagged as a food item but no Serving Measures saved
Partial Serving Measures saved but no Nutrients entered
Complete Nutrient Details saved

Recipe Report

For items with Steps configured, a dedicated Steps & Ingredients tab consolidates all recipe preparation data. In view mode, step ingredients can be expanded for only one step at a time; a separate report covers all steps together.

The right-side Ingredients card aggregates every ingredient added to the recipe, regardless of which step it belongs to. The quantity shown is the sum of each ingredient across the entire recipe, scaled to higher units where appropriate. This total view also drives the Scale Recipe report.

Exceptions & Troubleshooting

The scenarios below cover the edge cases most likely to cause confusion during item setup. Each maps a symptom to its cause and the correct response.

Symptom Cause Resolution
Sub-Qty field is greyed out on the first pack size row Expected behavior — Sub-Qty is intentionally disabled for the first pack size added. Add a second pack size; Sub-Qty will become available to link the two tiers.
Item is marked Inventory Ineligible after pack sizes are saved Pack size units and serving sizes exist but are not linked. Link via Sub-Qty or, for weight-based servings, enter weight on the pack unit instead.
Confirm Base Unit step is replaced by a different prompt Servings and pack units are unlinked, triggering a substituted prompt. Return to the Units grid, establish the Sub-Qty or Weight link, then re-attempt the finish flow.
Pack size hierarchy exceeds three tiers The system models pack size chains up to a lowest unit in a 3-tier structure. Flatten the real-world structure to fit within three tiers; consult your configuration team for edge cases that cannot be reduced.

Key reminders

  • Inventory Readiness is triggered once pack sizes are present. Items with unlinked pack-serving relationships will not reach Inventory Eligible status.
  • Weight on the lowest pack unit is the fallback linking mechanism when Sub-Qty cannot be used — for example, when linking to a weight-based serving. The field is optional in all other scenarios.
  • The system will prompt for Sub-Qty or Weight when a potential link is detected but not yet established.
