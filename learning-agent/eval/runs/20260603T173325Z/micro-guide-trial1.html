Who This Is For

This section targets two primary audiences:

  • Nutrition staff — responsible for building and maintaining the item catalog day-to-day, including adding new items and keeping existing records current.
  • District administrators — overseeing item setup during initial district onboarding and coordinating catalog standards across sites.

If you are not involved in creating or managing items in the catalog, this guide is background reading at most. If you are new to the district or onboarding a new site, start here before any other Item Management content.

Before You Start

Three configuration tables must be populated before you create or import items. The system validates each value at import time — a mismatch causes the record to be skipped.

Required ConfigurationWhy it matters
Item CategoryValidated against Item Category Configuration
Storage CategoryValidated against Storage Category Configuration
Valuation GroupValidated against Valuation Group Configuration

All three are mandatory fields for both food and non-food items.

For large onboarding efforts, use the Import Dashboard instead of manual entry. The import scope covers insert and update operations only.

Workflow: Create a Non-Food Item

  1. Item card — complete required fields, including the item Description.
  2. Units card — use the combined Units grid to add pack sizes. Each additional pack is inserted via the + Pack button.
  3. Sub-quantities — Sub-Qty fields are surfaced as required to link each pack tier to its parent or child. The first pack added does not enable Sub-Qty.
  4. Costing — cost entry is enabled after packs are configured; select the specific pack size on which to enter pricing and cost calculates across tiers.
  5. Inventory Readiness — a persistent banner displays throughout, providing clear status messaging at every stage until the item is ready.

Workflow: Create a Food Item — Units Card

The Units card combines pack size and serving size configuration into a single grid. It becomes editable after the general info card is saved.

Step 1 — Answer header questions

Two questions unlock the grid.

QuestionYesNo
Food Item? Proceed to Fluid Item question Serving sizes disabled
Fluid Item? Weight column switches to Volume for servings Weight column remains

Step 2 — Add a serving size

Click Add Serving Size and fill Amount, Description, and Weight. Setting Amount to 1 makes the serving inventory-eligible and enables Sub-Qty on that row.

Step 3 — Add pack size tiers

Use Add Pack to insert tiers (up to three). Sub-Qty is not enabled on the first pack added; it activates when a second tier is positioned above or below the existing one.

Step 4 — Link serving to pack (Inventory Readiness)

Enter Sub-Qty on the serving row to link it to a pack unit. Correct linking triggers Inventory Readiness; unlinked units remain inventory ineligible.

Step 5 — Resolve Missing Costing

Cost/Value becomes editable on eligible units at the finish tasks stage.

Step 6 — Promote Base Unit (optional)

A serving with Amount = 1 is eligible to be promoted as the base unit.

Workflow: Complete Food Item Details

After saving the Units card, work through each remaining detail tab in order.

Tab What to complete
Nutrition Information Enter the required nutrient fields.
Contribution Information Assign meal pattern credits.
Allergen Information Allergen data populates automatically. Review and confirm each allergen's status.

Save after completing each tab to preserve entries before navigating away.

Common Mistakes

  1. Attempting costing before all pack tiers are built. Cost and Value fields are only enabled during finish tasks — entering pricing earlier is not possible. Complete every pack level first.
  2. Skipping the Sub-Qty link between servings and packs. Unlinked pack units leave the item Inventory Ineligible. The interface highlights the Sub-Qty fields required to establish the link.
  3. Entering Sub-Qty on the first pack size. Sub-Qty is disabled on the first pack row by design. It activates only when a second pack size is introduced.
  4. Ignoring the Inventory Readiness banner. The banner is always visible and covers every state, including Non-Inventory for ineligible items. Confirm status here before publishing.
