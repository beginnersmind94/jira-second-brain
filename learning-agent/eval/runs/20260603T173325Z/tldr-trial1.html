Item Management: Learning Guide

What This Module Does

Item Management is the system of record for every food and non-food item in SchoolCafé. Configurations set here flow directly into Inventory (ordering and receiving), Menu Planning (recipes and nutrient analysis), and POS (pricing) — meaning errors at this stage cascade downstream across all three modules.

Key Features

Feature What it does
Item Card Captures core item details; brand, manufacturer, and product code are optional but recommended.
Units Card – Pack Sizes A combined Units grid replaces separate pack and serving sections, supporting multiple pack size units linked via Sub-Qty to establish a Base Unit.
Units Card – Serving Sizes Food and Fluid item flags gate serving size entry; answering No to "Food Item?" disables serving sizes entirely, while answering Yes to "Fluid Item?" switches measures to volume.
Serving-to-Pack Linking When a serving size Amount equals 1, Sub-Qty can link it to the pack size base unit, making it inventory-eligible (food items only).
Cost / Value Replaces Fair Market Value; editable on all eligible pack and linked serving units.
Inventory Readiness A persistent status banner guides users through pack size configuration requirements; every scenario — including non-inventory — displays a dedicated banner.
Nutrient / Contribution / Allergen Info The item record includes nutrition, contribution, and allergen information tabs.
Custom Allergen Configuration Authorized users can create district-level custom allergens from the Item Management configuration area; duplicates are blocked and deactivation is restricted while the allergen is in use.
Direct Menu Item A menu item can be created directly with a mandatory name and category, and linked to one or more recipes.
Item Images A single image can be uploaded per menu item via a file selection prompt.

Common Workflows

  • Create a non-food item: Save the Item Card, then open the Units Card and answer the Food Item prompt with No (which disables serving sizes). Add all required pack tiers using the Add Pack action , link tiers via Sub-Qty , then resolve any Missing Costing prompt and verify the Inventory Ready flag.
  • Create a food item: Save the Item Card, then on the Units Card answer Food Item → Yes and Fluid Item → Yes/No (Fluid switches serving size measure to Volume). Add a serving size first , then add pack tiers; link serving to a pack via Sub-Qty or pack weight. Resolve Missing Costing and confirm Inventory Readiness ; optionally promote the serving to Base Unit , then complete any remaining nutrition, allergen, contribution, and pricing tabs.
  • Add a custom allergen: Navigate to Item Management › Configuration › Item Configuration › Custom Allergens and select NEW. Enter a unique description and select the Local data source (both mandatory, no duplicates allowed) , save to return to the result grid , then assign the allergen on the item's Allergen Info card.

Gotchas

  • Complete all pack size tiers before addressing the costing step — the system executes required checks before enabling the official save, and the costing prompt is a deliberate completion check, not an error.
  • Pack size configuration supports up to three countable inventory units; the system's inventory readiness logic is built around a maximum of three tiers.
  • Once an item enters Inventory tables, the Base Unit choice is locked and cannot be changed — configure it correctly before first use.
  • For food items, servings and pack sizes must be linked (via Sub-Qty or weight) to achieve Inventory readiness; unlinked items will be marked Inventory Ineligible.
  • Item categorizations are only required when Inventory is enabled, but fill them in at item creation anyway to avoid rework later.
  • Custom allergens must be pre-created in the Item Management configuration area before they appear as selectable options on an item — they cannot be added inline from the item card.

Where To Go Next

  • Inventory module — vendor contracts, receiving, and order management; the Inventory Info tab links directly to Contract Items configured here.
  • Menu Planning module — recipes, menu grids, and nutrient analysis draw on serving sizes and nutrient data set up in Item Management.
  • POS module — student and adult pricing entered here flows to cashier touchscreens.

Quick-reference guides for creating non-food items and food items are available on the SchoolCafé support site; links will be provided by your implementation team.
