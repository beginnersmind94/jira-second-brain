Hello Roberta,     There is a bug identified with the Red items containing discontinued ingredients, which will be rectified in version 16.4.     The issue we found is that the code was not looking for ONLY the "current" ingredients in the menu item... it was looking at any ingredient that had EVER been associated with it. So if you (appropriately) previously replaced an "old" milk with a "new" milk on the menu item milk, it's flagging that milk. The update coming in 16.4 will look at only CURRENT ingredients in the menu items, so those instances will no longer be flagged, and only "actual" menu items that need to be looked into will be red.              Check out PrimeroEdge Training WEBINARS Available: Click Here  SchoolCafe Training WEBINARS Available: Click Here     Let us know if you have any questions regarding this ticket by

Thank you for the follow-up question. You are correct that, at first glance, both the Recipe tab and the Menu Item tab appeared to be using the same serving size of “4 fluid ounces,” which is why the behavior was confusing.     What we found is that the issue was not with the displayed serving size itself, but rather with how the serving measurements were configured internally behind the scenes for nutrient analysis processing.     In the Recipe tab, although the serving size displayed as “4 fluid ounces,” the recipe was still configured with a Quantity of “1 Each.” Because of this, the system was internally treating the recipe serving as an “Each”-based serving rather than a fluid ounce–based serving during nutrient analysis calculations.     At the same time, the Menu Item tab was using a serving size configuration based on “4 fluid ounces.” During nutrient analysis, the application attempts to convert the recipe serving measurement to the menu serving measurement. Since the recipe was internally tied to “Each” while the menu item was tied to a fluid ounce serving measure, the system could not properly convert the values, which caused the nutrient analysis report to return “(M)” for missing nutrient information.     Once we updated both areas so they were aligned consistently as:   Recipe tab = 1 Each  Menu Item default serving size = 1 Each  the nutrient analysis processed successfully because the serving measurements matched internally and no conversion was required.     So in simpler terms, the issue was not the visible “4 fluid ounces” value itself, but rather that the recipe and menu item were using different underlying serving measurement types internally, even though the displayed serving sizes looked similar.     For reference, I have attached a screenshot highlighting the Recipe tab configuration that was causing the issue.   Please let me know if you have any additional questions.
